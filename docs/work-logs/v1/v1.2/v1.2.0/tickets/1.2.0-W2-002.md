## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

popup 模組 17 個 JS 檔案 + popup.html 中，中文字串和配置常數散布各處，未集中管理。

### 影響範圍

| 檔案 | 中文行數 | 常數類型 |
|------|---------|---------|
| popup.js | 504 | MessageDictionary + MESSAGES + CONFIG + HANDSHAKE_CONFIG + STATUS_TYPES |
| popup-error-handler.js | 245 | 錯誤訊息 |
| popup-ui-manager.js | 240 | UI 文字 |
| popup-event-controller.js | 197 | 事件訊息 |
| popup-diagnostic-enhancer.js | 163 | 診斷訊息 |
| popup-controller.js | 154 | 控制邏輯訊息 |
| popup-ui-components.js | 151 | UI 狀態文字 |
| diagnostic-module.js | 151 | 診斷文字 |
| popup.html | 58 | HTML 文字 + inline style |
| 其餘 8 個 JS | 各 17-135 | 混合 |

### 既有基礎設施

- `src/core/design-system/` 已有 COLORS, SPACING, FONT_SIZES, SHADOWS 等 token
- `src/core/messages/MessageDictionary.js` 已有統一訊息管理機制
- popup.js 已有局部 `popupMessages`（MessageDictionary）和 `MESSAGES` 常數區塊

### Context Bundle

**目標架構**：
```
src/popup/constants/
  ui-text.js        -- 所有 popup UI 文字常數（按功能分類，非按元件）
  layout.js         -- popup-specific 佈局設定（寬高、間距、超時等）
  index.js          -- 統一匯出
```

**設計決策**：
1. UI 文字按功能分類（status / extraction / import / diagnostic / navigation），不按元件分類（AC3）
2. 佈局設定僅放 popup-specific 值；通用 spacing/typography 引用 `src/core/design-system/`
3. popup.js 中既有 `popupMessages`（MessageDictionary）保留為 Logger 專用；UI 顯示文字移入 `ui-text.js`
4. popup.html 中的中文文字移入 JS 常數，HTML 保留 data-attribute 或 JS 注入
5. 各檔案改為 `require('./constants')` 引用

**AC 對照**：
- AC1：HTML/JS 中文文字 → ui-text.js（popup.html 中 6 個按鈕 + 標題 + 狀態文字）
- AC2：尺寸/超時常數 → layout.js（CONFIG, HANDSHAKE_CONFIG, popup.html inline styles）
- AC3：按功能分類，非元件前綴
- AC4：CLAUDE.md 記錄路徑
- AC5：npm test 全通過

**注意**：AC5 有一個既有失敗（popup-interface ARIA test），已建 W2-005 追蹤，不在本 ticket 範圍。

**commit policy**：功能分支 feat/1.2.0-W2-002-popup-constants，完成後 PM 合併到 main

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 建立常數檔（src/popup/constants/）

| 檔案 | 內容 |
|------|------|
| `ui-text.js` | UI 文字按功能分類：HEADER_TEXT / STATUS_TEXT / ACTION_TEXT / EXTRACTION_TEXT / RESULTS_TEXT / ERROR_TEXT / PAGE_INFO_TEXT / VERSION_TEXT / DIALOG_TEXT；語意枚舉 STATUS_TYPES（補齊既有缺漏的 WARNING）/ MESSAGE_TYPES |
| `layout.js` | popup-specific：POPUP_DIMENSIONS（350x400）/ STATUS_CONFIG（輪詢間隔 + domain）/ HANDSHAKE_CONFIG。通用 spacing/typography 仍引用 design-system |
| `index.js` | 統一匯出（`require('./constants')`） |

### popup.js 改造

- 常數區改 `require('./constants')`，映射回既有變數名（STATUS_TYPES/MESSAGE_TYPES/MESSAGES/CONFIG/HANDSHAKE_CONFIG），下游使用點與 window re-export 完全不變（保留測試契約）。
- require 降級鏈：`require('./constants')` 失敗 → `window.PopupConstants` fallback（對應 esbuild bundle / eval 測試環境）。
- 新增 `applyStaticUIText()`，於 `initialize()` 步驟 1 後以常數注入 HTML 文字（含 aria-label），使常數檔成為運行時 SSOT。

### popup.html 改造

- 中文文字降為無 JS fallback，加註 SSOT 指向 constants/ui-text.js；尺寸註解指向 constants/layout.js。

### 測試環境配套

- `popup-interface.test.js`、`version-display.test.js`（皆 eval popup.js）補注入 `window.PopupConstants = require('src/popup/constants')`，對應 popup.js require 降級路徑。

### 設計取捨

- HTML 文字未採「清空 + JS 注入」：popup-interface.test.js 讀真實 popup.html 並斷言 extractBtn 初始 textContent 含「提取」，清空會造成時序回歸。改採「常數為運行時 SSOT + HTML 同值 fallback」，零回歸且符合 AC「集中管理（單一事實來源）」。
- Logger 專用 `popupMessages`（MessageDictionary）保留原位，不納入 ui-text.js。
