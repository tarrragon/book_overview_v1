## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

popup 模組 17 個 JS 檔案 + popup.html 中，中文字串和配置常數散布各處，未集中管理。

### 影響範圍

| 檔案 | 中文行數 | 常數類型 |
|------|---------|---------|
| popup.js | 504 | MessageDictionary + MESSAGES + CONFIG + HANDSHAKE_CONFIG + STATUS_TYPES |
| popup-error-handler.js | 245 | 錯誤訊息 |
| popup-ui-manager.js | 240 | UI 文字 |
| popup-event-controller.js | 197 | 事件訊息 |
| popup-diagnostic-enhancer.js | 163 | 診斷訊息 |
| popup-controller.js | 154 | 控制邏輯訊息 |
| popup-ui-components.js | 151 | UI 狀態文字 |
| diagnostic-module.js | 151 | 診斷文字 |
| popup.html | 58 | HTML 文字 + inline style |
| 其餘 8 個 JS | 各 17-135 | 混合 |

### 既有基礎設施

- `src/core/design-system/` 已有 COLORS, SPACING, FONT_SIZES, SHADOWS 等 token
- `src/core/messages/MessageDictionary.js` 已有統一訊息管理機制
- popup.js 已有局部 `popupMessages`（MessageDictionary）和 `MESSAGES` 常數區塊

### Context Bundle

**目標架構**：
```
src/popup/constants/
  ui-text.js        -- 所有 popup UI 文字常數（按功能分類，非按元件）
  layout.js         -- popup-specific 佈局設定（寬高、間距、超時等）
  index.js          -- 統一匯出
```

**設計決策**：
1. UI 文字按功能分類（status / extraction / import / diagnostic / navigation），不按元件分類（AC3）
2. 佈局設定僅放 popup-specific 值；通用 spacing/typography 引用 `src/core/design-system/`
3. popup.js 中既有 `popupMessages`（MessageDictionary）保留為 Logger 專用；UI 顯示文字移入 `ui-text.js`
4. popup.html 中的中文文字移入 JS 常數，HTML 保留 data-attribute 或 JS 注入
5. 各檔案改為 `require('./constants')` 引用

**AC 對照**：
- AC1：HTML/JS 中文文字 → ui-text.js（popup.html 中 6 個按鈕 + 標題 + 狀態文字）
- AC2：尺寸/超時常數 → layout.js（CONFIG, HANDSHAKE_CONFIG, popup.html inline styles）
- AC3：按功能分類，非元件前綴
- AC4：CLAUDE.md 記錄路徑
- AC5：npm test 全通過

**注意**：AC5 有一個既有失敗（popup-interface ARIA test），已建 W2-005 追蹤，不在本 ticket 範圍。

**commit policy**：功能分支 feat/1.2.0-W2-002-popup-constants，完成後 PM 合併到 main
