## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### Context Bundle

**目標**：將 popup.html 硬編碼的 UI 元素替換為 ui-factory.js 動態渲染，實現 AC-1（共用卡片）、AC-2（按鈕模組化）、AC-3（CLAUDE.md 登錄）。

**重構範圍**：

1. **popup.html**（刪減）：
   - 移除 `<div id="importResultContainer">` 和 `<div id="importErrorContainer">` 的 HTML 結構
   - 保留空容器 `<div id="import-mount-point"></div>` 供 JS 動態掛載
   - 移除主操作區按鈕的硬編碼 HTML，保留容器 `<div id="action-buttons-mount"></div>`
   - 移除 progressContainer / resultsContainer / errorContainer / initErrorContainer 的內部 HTML 結構
   - CSS `<style>` 區塊不動（工廠函式使用同樣的 class name）

2. **popup-ui-components.js**（改造）：
   - initializeElements() 改為呼叫 ui-factory 建立元素，而非 getElementById
   - 新增 renderAll(containerElement) 方法，一次建立所有動態區塊
   - 保持既有公開 API 不變（updateStatus / showProgress / updateProgress / showResults / showError 等）

3. **import-panel.js**（適配）：
   - constructor 接收的 elements 物件改為 ui-factory 產生的 DOM 元素
   - 匯入結果和匯入錯誤統一使用 createCard（AC-1）

4. **popup.js**（初始化改造）：
   - DOMContentLoaded 時呼叫 ui-factory 建立按鈕和卡片，掛載到 mount point
   - import-panel 初始化時傳入工廠產生的元素

5. **CLAUDE.md**（登錄）：
   - §6.1 常數與設計系統表格新增 `src/popup/components/ui-factory.js` 一列

**關鍵約束**：
- popup-ui-components.js 的公開 API 不可 breaking change（既有測試依賴）
- import-panel.js 的 12 個測試必須全數通過
- CSS 不新增不修改
- 此 ticket blockedBy W2-003.1（需先有 ui-factory.js）

**風險**：
- popup.js 高度耦合 getElementById，修改影響範圍大
- 建議分兩次 commit：(1) 匯入結果/錯誤卡片統一 (2) 其他卡片 + 按鈕模組化

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要（方案 A，AC-4 不適用）

依 team-lead 裁示採方案 A：AC-4（popup-ui-components.js 改用 ui-factory）標記本 ticket 不適用，保留既有 getElementById 契約（其 28 個測試依賴）；AC-4 由 team-lead 另開 follow-up ticket。

**變更**：
1. `src/popup/popup.html`：移除 importResultContainer / importErrorContainer 硬編碼 HTML，改為單一 `<div id="importPanelMount">` 掛載點（保留 extractBtn 等主操作按鈕，popup-interface 整合測試依賴）。
2. `src/popup/popup.js`：
   - 新增 `require('src/popup/components/ui-factory')`（esbuild build 時 inline 進 bundle）。
   - 新增 `buildImportPanelElements()`：以 `createCard`（AC-1）+ `createButton`（AC-2）動態建立匯入結果卡與匯入錯誤卡，掛載到 mount point，回傳子元素引用給 ImportPanel（AC-5）。
   - 移除 elements 物件中 import 結果/錯誤的 getElementById 條目（已不存在於 HTML）。
   - 新增 popupMessages 鍵 IMPORT_PANEL_MOUNT_FAILED。
3. `CLAUDE.md` §6.1：登錄 ui-factory.js（AC-3/AC-8）。

**realm 衝突處理（方案 ii）**：popup-interface.test.js 以 `eval(popup.js)` + `require(ui-factory)` 形成兩個 jsdom realm，工廠 node 無法 append 進 eval realm 的 mount。mount.appendChild 包 try/catch + popupLogger.warn 降級（生產單一 realm 永遠成功；observability 規則 4：有日誌非靜默）。6 個 W4-006 case 降級後正常回綠（該測試不斷言匯入卡片）。

**import-panel.js 未改**：ImportPanel 透過 constructor 接收 elements 並寫入，傳入工廠元素的 key 與原本一致，無需修改；其 12 個測試注入 mock elements，不受影響。
