### Context Bundle

**目標**：將 popup.html 硬編碼的 UI 元素替換為 ui-factory.js 動態渲染，實現 AC-1（共用卡片）、AC-2（按鈕模組化）、AC-3（CLAUDE.md 登錄）。

**重構範圍**：

1. **popup.html**（刪減）：
   - 移除 `<div id="importResultContainer">` 和 `<div id="importErrorContainer">` 的 HTML 結構
   - 保留空容器 `<div id="import-mount-point"></div>` 供 JS 動態掛載
   - 移除主操作區按鈕的硬編碼 HTML，保留容器 `<div id="action-buttons-mount"></div>`
   - 移除 progressContainer / resultsContainer / errorContainer / initErrorContainer 的內部 HTML 結構
   - CSS `<style>` 區塊不動（工廠函式使用同樣的 class name）

2. **popup-ui-components.js**（改造）：
   - initializeElements() 改為呼叫 ui-factory 建立元素，而非 getElementById
   - 新增 renderAll(containerElement) 方法，一次建立所有動態區塊
   - 保持既有公開 API 不變（updateStatus / showProgress / updateProgress / showResults / showError 等）

3. **import-panel.js**（適配）：
   - constructor 接收的 elements 物件改為 ui-factory 產生的 DOM 元素
   - 匯入結果和匯入錯誤統一使用 createCard（AC-1）

4. **popup.js**（初始化改造）：
   - DOMContentLoaded 時呼叫 ui-factory 建立按鈕和卡片，掛載到 mount point
   - import-panel 初始化時傳入工廠產生的元素

5. **CLAUDE.md**（登錄）：
   - §6.1 常數與設計系統表格新增 `src/popup/components/ui-factory.js` 一列

**關鍵約束**：
- popup-ui-components.js 的公開 API 不可 breaking change（既有測試依賴）
- import-panel.js 的 12 個測試必須全數通過
- CSS 不新增不修改
- 此 ticket blockedBy W2-003.1（需先有 ui-factory.js）

**風險**：
- popup.js 高度耦合 getElementById，修改影響範圍大
- 建議分兩次 commit：(1) 匯入結果/錯誤卡片統一 (2) 其他卡片 + 按鈕模組化
