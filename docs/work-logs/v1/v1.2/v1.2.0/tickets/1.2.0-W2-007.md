### 根因分析

**事件**：本 session W2-003.1 派發時，PM 在寫 Context Bundle 前先 claim ticket，導致代理人派發後無法認領。

**根因**：PM 使用 `ticket track claim 1.2.0-W2-003.1 --as rosemary-project-manager` 將 ticket 狀態改為 in_progress。後續派發代理人時，代理人嘗試 `claim --as javascript-developer` 被拒——ticket 已被 PM 持有。

**觸發條件**：PM 需要在派發前寫入 Context Bundle（PC-040 要求），而 `append-log` 對 pending 狀態的 Problem Analysis / Context Bundle 章節已豁免 status precondition（W1-058）。PM 不需要 claim 就能寫 Context Bundle，但習慣性地先 claim 了。

**影響**：代理人浪費 1 個回合嘗試 claim 並回報失敗，PM 需手動 release 再通知代理人重試，整體浪費 2-3 個 tool call 回合。
