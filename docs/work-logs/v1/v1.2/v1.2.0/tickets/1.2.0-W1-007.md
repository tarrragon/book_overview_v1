## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-002 完成了 JSON 匯入 domain 層（`src/import/json-importer.js`），但 Popup UI 端尚無匯入入口。需要新增 import-panel 元件串接 domain API。

### 影響範圍

- `src/popup/popup.html` — 新增匯入按鈕和結果顯示區塊
- `src/popup/components/import-panel.js` — 新建 import panel 元件
- `src/popup/popup.js` — 初始化和事件綁定整合

### 架構分析

1. **Domain API 已就緒**：`json-importer.js` 提供 `parseAndValidate`、`checkStaleness`、`executeImport` 三個公開函式
2. **元件模式參照**：`sync-panel.js` 是同類 panel 元件（`constructor(elements)` → `initialize()` → 業務方法），import-panel 應遵循此模式
3. **防舊蓋新流程**：`executeImport` 回傳 `IMPORT_STALE_DATA` 時 UI 需 `window.confirm` 確認，確認後以 `skipStalenessCheck: true` 重新匯入
4. **錯誤顯示**：複用既有 `errorContainer`（popup.html L315-336）
5. **Build**：popup.js 使用 CJS `require`，import-panel 若用 ESM import 需注意 build 配置

### 相關 Error Pattern

無直接相關 error pattern。注意 Chrome Extension 環境限制（quickref.md）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### Phase 1 功能規格

### 1. HTML 結構新增

**popup.html 主操作卡片**（primary-surface 區塊）：
- `viewLibraryBtn` 之後新增 `importBtn`（secondary 按鈕，文字「匯入書庫」）
- 隱藏 `<input type="file" id="importFileInput" accept=".json">`（不可見觸發元素）

**新增 importResultContainer**（status-card，預設隱藏）：
- 標題：「匯入結果」
- 摘要區：新增 `importedCount` / 更新 `updatedCount` / 未變更 `unchangedCount`
- 關閉按鈕

### 2. ImportPanel class（`src/popup/components/import-panel.js`）

遵循 sync-panel.js ESM class 模式：

```
constructor(elements: { importBtn, fileInput, resultContainer, importedCount, updatedCount, unchangedCount })
initialize()          — 綁定 importBtn click → triggerFileSelect; fileInput change → handleFileSelected
triggerFileSelect()   — fileInput.click()
handleFileSelected(e) — FileReader.readAsText → executeImport → 分流
handleImportResult(result)  — success → 更新摘要 / STALE_DATA → confirm → retry / 其他 error → showError
showResult(summary)   — 顯示 resultContainer + 填入 added/updated/unchanged
showError(error)      — 複用 popup.html 既有 errorContainer（由 popup.js 傳入 callback）
reset()               — 隱藏 resultContainer + 清空 fileInput.value
```

### 3. popup.js 整合

- elements 物件新增 importBtn/importFileInput/importResultContainer 等 DOM refs
- setupEventListeners 或 initialize 中建立 ImportPanel 實例並呼叫 initialize()
- 傳入 showError callback 供 import-panel 在匯入失敗時顯示錯誤

### 4. 防舊蓋新流程

executeImport 回傳 `{ success: false, error: { code: 'IMPORT_STALE_DATA' }, staleness }` 時：
1. 呼叫 `window.confirm` 顯示「匯入檔案（{exportedAt}）較本機上次匯入（{lastImportedAt}）舊，確定要覆蓋？」
2. 確認 → `executeImport(content, { skipStalenessCheck: true })`
3. 取消 → reset，不做任何事

### 5. 錯誤處理

| 錯誤碼 | UI 行為 |
|-------|---------|
| IMPORT_PARSE_ERROR | errorContainer 顯示「檔案格式錯誤，無法解析 JSON」 |
| IMPORT_UNKNOWN_FORMAT | errorContainer 顯示「無法識別的檔案格式」 |
| IMPORT_EMPTY_BOOKS | errorContainer 顯示「匯入檔案中沒有書籍資料」 |
| IMPORT_STORAGE_ERROR | errorContainer 顯示「儲存失敗，請重試」 |
| IMPORT_STALE_DATA | window.confirm 確認流程（非 error 顯示） |
| FILE_READ_ERROR | errorContainer 顯示「檔案讀取失敗」 |
