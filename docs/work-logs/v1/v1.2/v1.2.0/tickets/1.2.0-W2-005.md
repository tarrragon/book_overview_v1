## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

popup.html 第 314 行 closeImportResultBtn 和第 325 行 closeImportErrorBtn 是空 button 元素（W1-007 引入），無文字內容也無 JS 動態注入。測試 JSDOM 環境不執行 popup.js，遍歷所有 button 檢查 textContent.length > 0 時命中這兩個空按鈕。

### Context Bundle

修復：兩個空 button 加上 fallback 文字 "關閉"，同時新增 ui-text.js IMPORT_TEXT.CLOSE_RESULT / CLOSE_ERROR 常數。
檔案：src/popup/popup.html（2 行）、src/popup/constants/ui-text.js（2 行）
分支：feat/1.2.0-W2-005-aria-fix

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 根因
closeImportResultBtn / closeImportErrorBtn 的 textContent 由 ImportPanel.initialize() 在 runtime 動態注入（src/popup/components/import-panel.js:52-58，值為 IMPORT_MESSAGES.CLOSE = '關閉'）。測試 JSDOM 環境不執行 popup.js，故兩個 button 維持 HTML 靜態空字串，popup-interface.test.js:517 遍歷 button 斷言 textContent.length > 0 命中失敗。

### 修復
src/popup/popup.html:314 / :325 兩個 button 加上靜態 fallback 文字「關閉」，與 runtime 動態注入值一致（runtime 仍會覆寫成同值，無視覺/行為副作用）。

### 與 Context Bundle 的偏差說明
Context Bundle 建議新增 ui-text.js IMPORT_TEXT.CLOSE_RESULT / CLOSE_ERROR 常數，未採用。理由：(1) ui-text.js 無 IMPORT_TEXT 物件，分類軸為功能領域 *_TEXT；(2) close button 文字 SSOT 已是 import-panel.js 的 IMPORT_MESSAGES.CLOSE，新增重複常數違反 DRY；(3) HTML 靜態屬性無法引用 JS 常數。僅補 HTML 靜態 fallback 即可滿足 AC 且不引入重複。
