### Context Bundle

**目標**：建立 `src/popup/components/ui-factory.js` — Popup UI 元件工廠模組，提供 6 個工廠函式動態建立 DOM 元素。

**設計規格（Phase 1）**：

1. **createButton({ variant, text, id, ariaLabel, onClick, disabled, size })**
   - variant: 'primary' | 'secondary' | 'danger'（必填）
   - size: 'normal' | 'small'（預設 'normal'）
   - 回傳 HTMLButtonElement，class 為 `button ${variant}` + size 為 small 時加 `small`
   - 樣式使用 popup.html 既有的 `.button.primary` / `.button.secondary` / `.button.danger` CSS class

2. **createCard({ variant, title, id, content, actions })**
   - variant: 'default' | 'error'（預設 'default'）
   - title: 字串（選填，有則建立 header 含 `<strong>`）
   - content: HTMLElement | 字串（選填，字串建立 `<div class="info-text">`）
   - actions: HTMLElement[]（選填，放入 `<div class="action-buttons">`）
   - 回傳 HTMLDivElement，class 為 `status-card`；variant='error' 額外加 `error-card`

3. **createStatusIndicator({ type, text, info })**
   - type: 'loading' | 'ready' | 'error'
   - 回傳含 status-dot + statusText span + statusInfo div 的容器

4. **createProgressSection({ percentage, text })**
   - 回傳含 progress-header / progress-bar-container / progress-fill / progressText 的容器
   - progress-bar 需設 ARIA 屬性（role=progressbar, valuemin=0, valuemax=100）

5. **createResultsSection({ bookCount, time, successRate, onExport, onView })**
   - 回傳含 results-header + info-text + action-buttons 的結果卡片
   - onExport / onView 為按鈕 click callback

6. **createErrorSection({ message, onRetry, onReload, suggestions })**
   - 回傳含 error-header + error-message + action-buttons + suggestions ol 的錯誤卡片
   - suggestions 為字串陣列，有值時顯示

**技術約束**：
- 使用 CSS class 名稱與 popup.html 既有 `<style>` 完全一致，不新增 CSS
- CommonJS 匯出（`module.exports`），與既有 popup-ui-components.js 一致
- 不依賴任何外部 library，純 DOM API
- 所有 id 屬性透過參數傳入（不硬編碼 DOM id）

**現有 CSS class 參考**（popup.html 內 `<style>` 已定義）：
- `.button` / `.button.primary` / `.button.secondary` / `.button.danger` / `.button.small` / `.button:disabled`
- `.status-card` / `.status-card.primary-surface` / `.error-card`
- `.status-indicator` / `.status-dot` / `.status-dot.loading` / `.status-dot.error`
- `.progress-header` / `.progress-bar-container` / `.progress-bar` / `.progress-fill`
- `.results-header` / `.error-header` / `.error-message` / `.error-suggestions`
- `.action-buttons` / `.info-text`

**測試檔案**：`tests/unit/popup/ui-factory.test.js`（新建）
- 使用 jsdom 環境（Jest 預設）
- 每個工廠函式至少測試：正常呼叫、必填參數缺失、各 variant、callback 綁定

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
建立 src/popup/components/ui-factory.js，提供 6 個無狀態純函式工廠：

- createButton: button.{variant}（primary/secondary/danger）+ small；支援 id/ariaLabel/onClick/disabled，缺 variant 拋錯
- createCard: status-card（error 加 error-card）；title→strong header、content（字串→info-text／HTMLElement→直放）、actions→action-buttons
- createStatusIndicator: status-card > status-indicator(status-dot[.loading/.error] + span) + info-text
- createProgressSection: progress-header + progress-bar-container > progress-bar[role=progressbar,aria-*] > progress-fill(width=%) + info-text
- createResultsSection: 複用 createCard，info-text 列書籍數/時間/成功率 + 匯出/查看 按鈕
- createErrorSection: 複用 createCard(error)，error-message + 重試/重新載入 按鈕 + 選填 suggestions ol

設計要點：
- class 名稱與 popup.html 既有 style 完全一致，不新增 CSS
- CommonJS 匯出（與 popup-ui-components.js 一致），供 jsdom 測試與 esbuild bundle
- 純 DOM API，無外部依賴；所有 id 由參數傳入不硬編碼
- 安全性：createResultsSection 改用 appendLabeledValue（textContent）建構，移除 innerHTML 拼接避免 XSS（PostToolUse security hook 觸發後修正）

WRAP 三問：
- W：候選 (1) CommonJS 純函式工廠（採用，對齊 popup-ui-components.js）(2) ES Module。依 Context Bundle 指定選 (1)
- A：P2 模組化重構不阻塞核心功能，產出為可替換元件庫
- P：最可能失敗為工廠 DOM 結構與 popup.html 不一致致樣式錯位；防護＝測試斷言 class 與巢狀結構對齊既有 HTML
