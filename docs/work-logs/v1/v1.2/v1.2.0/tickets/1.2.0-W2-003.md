## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 現況分析

**問題根因**：popup.html 中 UI 元素（按鈕、結果卡片、匯入結果/錯誤卡片）直接以 HTML 定義，缺乏模組化。匯入結果卡片和匯入錯誤卡片結構相似但各自獨立定義，違反 DRY。

**現有元件架構**：
- `popup-ui-components.js`：元件「管理器」，查找既有 DOM 元素並操作狀態（show/hide/updateStatus），不建立 DOM
- `components/import-panel.js`：匯入面板控制器，接收外部傳入的 DOM 元素引用
- `popup.html`：所有 UI 結構硬編碼（6 個按鈕、5 個卡片區塊）

**影響範圍**：
- `src/popup/popup.html` — 按鈕和卡片 HTML 結構
- `src/popup/popup-ui-components.js` — DOM 元素引用初始化
- `src/popup/components/import-panel.js` — 匯入面板使用的 DOM 元素
- `src/popup/popup.js` — 初始化和事件綁定
- `tests/unit/popup/import-panel.test.js` — 12 個匯入面板測試
- `tests/unit/popup/popup-ui-components.test.js` — UI 元件測試

### WRAP 三問

**W（Widen）— 候選方案**：
1. **元件工廠函式**（選擇）：建立 `createButton()` / `createCard()` 工廠函式，popup.html 保留骨架，動態渲染內容區域
2. **Web Components**：使用 Custom Elements API。過重，Chrome Extension popup 不需要 Shadow DOM 隔離

**A（Attain distance）— 機會成本**：佔用 W2 容量，延後 W2 其他工作。但本 ticket 是 P2 且無下游 blocker，影響可控。

**P（Prepare to be wrong）— 最可能失敗原因**：重構後既有 12 個 import-panel 測試或 popup-ui-components 測試因 DOM 結構變更而 break。防護：漸進式重構，先建元件再逐步替換 HTML，每步確認測試全綠。

## Context Bundle

### Phase 1 功能規格（擴展範圍）

用戶選擇擴展範圍：除 AC 原文（共用卡片 + 按鈕 + CLAUDE.md）外，涵蓋 status-card / progress-bar / results-card / error-card 全面模組化。

#### 元件工廠設計

**新檔案**：`src/popup/components/ui-factory.js`

**工廠函式清單**：

| 函式 | 回傳 | 用途 |
|------|------|------|
| `createButton({ variant, text, id, ariaLabel, onClick, disabled, size })` | HTMLButtonElement | primary/secondary/danger 三種樣式 |
| `createCard({ variant, title, id, content, actions })` | HTMLDivElement | 通用卡片容器（default/error 兩種 variant） |
| `createStatusIndicator({ type, text, info })` | HTMLDivElement | 狀態指示器（loading/ready/error） |
| `createProgressSection({ percentage, text })` | HTMLDivElement | 進度條區塊 |
| `createResultsSection({ bookCount, time, successRate, onExport, onView })` | HTMLDivElement | 提取結果區塊 |
| `createErrorSection({ message, onRetry, onReload, suggestions })` | HTMLDivElement | 錯誤訊息區塊 |

**共用卡片元件（AC-1）**：
匯入結果和匯入錯誤統一使用 `createCard({ variant, title, content, actions })`：
- variant='default' → 匯入結果
- variant='error' → 匯入錯誤（套用 error-card 樣式）

**按鈕元件（AC-2）**：
`createButton` 支援 `variant: 'primary' | 'secondary' | 'danger'`，`size: 'normal' | 'small'`。

#### 重構策略（漸進式）

popup.html 保留骨架（`<div class="content">` 容器），卡片內容改由 JS 動態渲染。
popup-ui-components.js 改為使用 ui-factory 建立元素，不再 getElementById 查找 HTML 硬編碼元素。

#### 認知負擔評估

- 新檔案：1（ui-factory.js）+ 1（ui-factory.test.js）
- 修改檔案：popup.html, popup-ui-components.js, popup.js, import-panel.js, CLAUDE.md = 5
- 功能職責：3（工廠建立 / HTML 重構 / 測試適配）

建議拆為 2 個子 ticket：
- W2-003.1：建立元件工廠 ui-factory.js + 測試（純新增，不改既有）
- W2-003.2：重構 popup.html + 既有 JS 使用元件工廠（修改既有，依賴 .1）
