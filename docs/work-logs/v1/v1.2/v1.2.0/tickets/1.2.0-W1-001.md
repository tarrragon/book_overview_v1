### Phase 1 功能規格設計

**設計者**：lavender-interface-designer | **日期**：2026-06-20

### 1. Purpose（目的）

本模組解決「Web 端書庫資料單向傳輸至 Flutter App」的核心問題。使用者在 Extension Popup 內點擊「同步到 App」，系統自動將書庫資料壓縮、切塊、編碼為 QR 動畫，在 Canvas 上輪播顯示，供 App 端相機掃描接收。

**目標使用者**：已在 Readmoo 提取書庫的使用者，希望將書庫資料同步到手機 App。

**核心價值**：零網路依賴、零伺服器成本、零帳號配置的視覺直傳同步。

### 2. API Signatures（介面定義）

#### 2.1 src/sync/qr-encoder.js（純邏輯模組，無 DOM）

- `compressData(jsonString: string): Promise<Uint8Array>` — gzip 壓縮。CompressionStream 不可用時拋出 Error。
- `calculateCRC32(data: Uint8Array): number` — CRC32 校驗值（32-bit unsigned）。
- `createFrames(compressedData: Uint8Array, chunkSize = 800): Uint8Array[]` — 切塊 + 15 bytes frame header。
  - Frame header: [0-1] magic 0x51 0x52, [2] version 0x01, [3-4] total_frames BE uint16, [5-6] frame_index BE uint16 (0-based), [7-10] total_size BE uint32, [11-14] crc32 BE uint32, [15+] payload。
- `encodeFrameToQR(frameData: Uint8Array, options?): QRCode` — frame 二進位 → QR 物件。options: { errorCorrectionLevel: 'M', version: auto 20-25 }。
- `encodeBookDataToQRFrames(jsonString: string, options?): Promise<{frames: QRCode[], totalSize: number, isStatic: boolean}>` — 完整管線。isStatic: 壓縮後 <= chunkSize。

#### 2.2 src/popup/services/sync-qr-renderer.js（Canvas 輪播渲染器）

- `class SyncQRRenderer` — constructor(canvas, options?)。options: { fps: 8, canvasSize: 280, maxCycles: 3, onProgress: fn, onComplete: fn }。
- `start(frames: QRCode[], isStatic: boolean)` — 載入 frames 並開始播放/靜態顯示。
- `pause()` / `resume()` / `stop()` — 動畫控制。
- `getStatus(): {playing, currentFrame, totalFrames, currentCycle}` — 播放狀態查詢。

#### 2.3 src/popup/components/sync-panel.js（UI 元件）

- `class SyncPanel` — constructor(elements)。elements: { syncContainer, qrCanvas, syncStatus, syncButton, stopButton }。
- `initialize(): Promise<void>` — 綁定事件、載入上次同步資訊。
- `startSync(): Promise<void>` — 觸發完整同步流程（讀取書庫 → 壓縮 → 切塊 → QR → 動畫）。
- `stopSync()` — 停止動畫回到初始狀態。

#### 2.4 同步用 JSON 資料結構

```json
{
  "format_version": "2.0",
  "books": [ /* BookSchemaV2 陣列 */ ],
  "sync_meta": {
    "exported_at": "ISO-8601",
    "source_app": "chrome-extension",
    "source_device": "ext_{random}",
    "book_count": Number,
    "last_sync_summary": { "added": 0, "updated": 0, "deleted": 0 }
  }
}
```

### 3. GWT Scenarios（行為場景）

場景 1: 多幀動畫同步
- Given: 書庫 200 本書，壓縮後 > 800 bytes
- When: 點擊「同步到 App」
- Then: 壓縮切塊產生多幀 QR，Canvas 8fps 輪播，顯示「共 N 幀」

場景 2: 單張靜態 QR
- Given: 書庫 5 本書，壓縮後 <= 800 bytes
- When: 點擊「同步到 App」
- Then: 產生單張靜態 QR，不播放動畫

場景 3: 空書庫
- Given: books 陣列為空
- When: 點擊「同步到 App」
- Then: 顯示「書庫中沒有書籍，請先提取書庫資料」，不產生 QR

場景 4: 壓縮失敗
- Given: CompressionStream 不可用
- When: 嘗試壓縮
- Then: 顯示「壓縮功能不可用，請更新瀏覽器」，記錄 error 日誌

場景 5: QR 編碼失敗
- Given: 單幀資料超過 QR 容量上限
- When: 嘗試編碼為 QR
- Then: 顯示「資料編碼失敗」，記錄 error 日誌含 frame 大小

場景 6: 使用者中途停止
- Given: QR 動畫正在輪播
- When: 點擊「停止」
- Then: 動畫停止，Canvas 清空，UI 回初始狀態

場景 7: 循環播放完成
- Given: 動畫輪播中，maxCycles=3
- When: 完成 3 圈
- Then: 停止播放，顯示「播放完成，若 App 未收齊可重新播放」

### 4. Error Handling（錯誤處理）

| 錯誤情境 | 偵測方式 | 處理策略 | 使用者回饋 |
|---------|---------|---------|-----------|
| 書庫為空 | books.length === 0 | 阻止流程 | 「書庫中沒有書籍」 |
| CompressionStream 不可用 | typeof === 'undefined' | 拋出 Error | 「壓縮功能不可用」 |
| gzip 壓縮失敗 | catch | 拋出 Error | 「資料壓縮失敗」 |
| QR 編碼失敗 | catch | 拋出 Error | 「QR 編碼失敗」 |
| qrcode-generator 載入失敗 | import 失敗 | 拋出 Error | 「QR 元件載入失敗」 |
| Canvas 不可用 | getContext('2d') null | 拋出 Error | 「繪圖功能不可用」 |
| 書庫讀取失敗 | storage.get 失敗 | catch | 「讀取書庫失敗」 |

日誌策略：所有 catch 記錄 Logger.error（含錯誤訊息 + 元件名稱）。

### 5. Dependencies（依賴）

外部依賴：
- qrcode-generator (npm) — QR 產生，純 JS 無 eval()，需 spike 驗證 esbuild + CSP
- CompressionStream API — Chrome 80+ 原生

內部依賴：
- chrome.storage.local — 讀取 readmoo_books
- Logger / ErrorCodes — 日誌和錯誤碼

環境假設：Chrome 100+, Extension Popup context (有 DOM + Canvas), esbuild IIFE bundle。

### 6. Acceptance（驗收條件）

- [ ] compressData 正確壓縮 JSON 為 gzip Uint8Array
- [ ] calculateCRC32 產出正確 CRC32（已知測試向量驗證）
- [ ] createFrames 每幀含 15 bytes header，magic 0x51 0x52，version 0x01
- [ ] total_frames/frame_index/total_size/crc32 欄位正確（big-endian）
- [ ] 壓縮後 <= 800 bytes 時 isStatic: true 且 frames 長度 1
- [ ] 壓縮後 > 800 bytes 時正確切塊，最後一塊允許 < chunkSize
- [ ] SyncQRRenderer Canvas 上 ~8fps 輪播
- [ ] 靜態模式顯示單張 QR 不啟動動畫
- [ ] maxCycles 到達後自動停止觸發 onComplete
- [ ] SyncPanel 完整流程（讀取 → 壓縮 → 切塊 → QR → 動畫）
- [ ] 空書庫顯示提示不產生 QR
- [ ] CompressionStream 不可用時顯示明確錯誤
- [ ] 所有 catch 有 error 日誌

### 7. 行為單元清單

| 行為單元 | GWT | 邊界條件 | 獨立可測 |
|---------|-----|---------|---------|
| gzip 壓縮 | Given JSON → When compressData → Then Uint8Array | 空字串、超大字串、API 不可用 | 是 |
| CRC32 計算 | Given Uint8Array → When calculateCRC32 → Then 正確值 | 空陣列、單 byte、已知向量 | 是 |
| Frame 切塊+Header | Given 壓縮資料 → When createFrames → Then 正確 header+payload | 整除/不整除 chunkSize、單塊 | 是 |
| QR 編碼 | Given frame → When encodeFrameToQR → Then QR 物件 | 超過容量上限 | 是 |
| 完整管線 | Given JSON → When encodeBookDataToQRFrames → Then frames+isStatic | 空書庫、小書庫(靜態)、大書庫(多幀) | 是 |
| Canvas 靜態繪製 | Given 單張 QR → When start(f, true) → Then 繪製一次 | context 不可用 | 是 |
| Canvas 動畫輪播 | Given 多幀 → When start(f, false) → Then 8fps 循環 | 1幀、maxCycles、pause/resume | 是 |
| 同步面板互動 | Given popup → When 點擊同步 → Then 完整管線 | 空書庫、壓縮失敗、使用者停止 | 是 |

### 8. Decision Questions

- Q1（跨模組共用）：不適用。三模組職責完全不同，無相似邏輯需共用。
- Q3-Q4（ID 設計）：不適用。不涉及持久化 ID 生成。
- Q5（出錯通知）：所有 catch 記錄 Logger.error + UI 錯誤訊息。
- Q6a（最慢操作）：gzip 壓縮 < 50ms + QR 編碼 < 200ms，點擊到動畫開始 < 500ms。
- Q6b（寫入失敗）：不適用。純讀取+即時運算，無多步驟寫入。

### 11. 資料流

chrome.storage.local.readmoo_books → JSON.stringify（含 sync_meta）→ compressData（gzip）→ calculateCRC32 → createFrames（切塊+header）→ encodeFrameToQR（每幀→QR）→ SyncQRRenderer.start（Canvas 輪播）

### 12. src/sync/ 與 data-management/sync-* 職責邊界

| 模組 | 職責 | 層級 |
|------|------|------|
| src/sync/qr-encoder.js | QR 傳輸編碼（壓縮+切塊+CRC32+QR） | Domain（純邏輯） |
| src/background/.../sync-* | Storage 元資料管理 | Infrastructure |

兩者不互相依賴。qr-encoder 接收 JSON 字串，不直接存取 chrome.storage。
