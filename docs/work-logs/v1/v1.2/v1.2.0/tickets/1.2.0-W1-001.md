#### 1. 系統一致性審查

**1.1 src/sync/ 新目錄** — 有疑慮需調整

src/sync/ 作為頂層目錄與既有 src/export/ 平行，放置 presentation-agnostic 同步邏輯（壓縮+切塊+frame header+CRC32）可接受。但須注意：qr-encoder.js 應為純邏輯（無 DOM/canvas），Canvas 輪播應放 src/popup/ 下。與 src/background/domains/data-management/services/sync-*（Storage 層級元資料管理）邊界需明確。

**1.2 src/popup/ 結構** — 通過

已有 components/services/utils 子目錄。sync-panel UI 可放 components/，渲染服務放 services/。Ticket where.files 需修正：實際路徑為 src/popup/ 非 popup/。建議不另建 sync-panel.html，在 popup.html 中新增區段或 JS 動態渲染。

**1.3 CompressionStream** — 通過

Chrome 80+ 支援，build target chrome100 遠超最低要求。popup 為標準 web page context，完整支援。

**1.4 重複造輪子** — 無風險

既有 export/ 為格式轉換+檔案下載，與 QR 二進位編碼完全不同。data-management sync-* 為元資料管理非傳輸編碼。CRC32 自行實作合理。

**1.5 qrcode-generator MV3 相容性** — 需確認

純 JS 函式庫理論上相容，但：(1) package.json 無 dependencies，需新增；(2) esbuild IIFE bundle 需驗證；(3) 無 eval() 應符合 CSP。建議 Phase 1 初期先做 spike 驗證。
