### Phase 1 功能規格設計

**設計者**：lavender-interface-designer | **日期**：2026-06-20

### 1. Purpose（目的）

本模組解決「Web 端書庫資料單向傳輸至 Flutter App」的核心問題。使用者在 Extension Popup 內點擊「同步到 App」，系統自動將書庫資料壓縮、切塊、編碼為 QR 動畫，在 Canvas 上輪播顯示，供 App 端相機掃描接收。

**目標使用者**：已在 Readmoo 提取書庫的使用者，希望將書庫資料同步到手機 App。

**核心價值**：零網路依賴、零伺服器成本、零帳號配置的視覺直傳同步。

### 2. API Signatures（介面定義）

#### 2.1 src/sync/qr-encoder.js（純邏輯模組，無 DOM）

- `compressData(jsonString: string): Promise<Uint8Array>` — gzip 壓縮。CompressionStream 不可用時拋出 Error。
- `calculateCRC32(data: Uint8Array): number` — CRC32 校驗值（32-bit unsigned）。
- `createFrames(compressedData: Uint8Array, chunkSize = 800): Uint8Array[]` — 切塊 + 15 bytes frame header。
  - Frame header: [0-1] magic 0x51 0x52, [2] version 0x01, [3-4] total_frames BE uint16, [5-6] frame_index BE uint16 (0-based), [7-10] total_size BE uint32, [11-14] crc32 BE uint32, [15+] payload。
- `encodeFrameToQR(frameData: Uint8Array, options?): QRCode` — frame 二進位 → QR 物件。options: { errorCorrectionLevel: 'M', version: auto 20-25 }。
- `encodeBookDataToQRFrames(jsonString: string, options?): Promise<{frames: QRCode[], totalSize: number, isStatic: boolean}>` — 完整管線。isStatic: 壓縮後 <= chunkSize。

#### 2.2 src/popup/services/sync-qr-renderer.js（Canvas 輪播渲染器）

- `class SyncQRRenderer` — constructor(canvas, options?)。options: { fps: 8, canvasSize: 280, maxCycles: 3, onProgress: fn, onComplete: fn }。
- `start(frames: QRCode[], isStatic: boolean)` — 載入 frames 並開始播放/靜態顯示。
- `pause()` / `resume()` / `stop()` — 動畫控制。
- `getStatus(): {playing, currentFrame, totalFrames, currentCycle}` — 播放狀態查詢。

#### 2.3 src/popup/components/sync-panel.js（UI 元件）

- `class SyncPanel` — constructor(elements)。elements: { syncContainer, qrCanvas, syncStatus, syncButton, stopButton }。
- `initialize(): Promise<void>` — 綁定事件、載入上次同步資訊。
- `startSync(): Promise<void>` — 觸發完整同步流程（讀取書庫 → 壓縮 → 切塊 → QR → 動畫）。
- `stopSync()` — 停止動畫回到初始狀態。

#### 2.4 同步用 JSON 資料結構

```json
{
  "format_version": "2.0",
  "books": [ /* BookSchemaV2 陣列 */ ],
  "sync_meta": {
    "exported_at": "ISO-8601",
    "source_app": "chrome-extension",
    "source_device": "ext_{random}",
    "book_count": Number,
    "last_sync_summary": { "added": 0, "updated": 0, "deleted": 0 }
  }
}
```

### 3. GWT Scenarios（行為場景）

場景 1: 多幀動畫同步
- Given: 書庫 200 本書，壓縮後 > 800 bytes
- When: 點擊「同步到 App」
- Then: 壓縮切塊產生多幀 QR，Canvas 8fps 輪播，顯示「共 N 幀」

場景 2: 單張靜態 QR
- Given: 書庫 5 本書，壓縮後 <= 800 bytes
- When: 點擊「同步到 App」
- Then: 產生單張靜態 QR，不播放動畫

場景 3: 空書庫
- Given: books 陣列為空
- When: 點擊「同步到 App」
- Then: 顯示「書庫中沒有書籍，請先提取書庫資料」，不產生 QR

場景 4: 壓縮失敗
- Given: CompressionStream 不可用
- When: 嘗試壓縮
- Then: 顯示「壓縮功能不可用，請更新瀏覽器」，記錄 error 日誌

場景 5: QR 編碼失敗
- Given: 單幀資料超過 QR 容量上限
- When: 嘗試編碼為 QR
- Then: 顯示「資料編碼失敗」，記錄 error 日誌含 frame 大小

場景 6: 使用者中途停止
- Given: QR 動畫正在輪播
- When: 點擊「停止」
- Then: 動畫停止，Canvas 清空，UI 回初始狀態

場景 7: 循環播放完成
- Given: 動畫輪播中，maxCycles=3
- When: 完成 3 圈
- Then: 停止播放，顯示「播放完成，若 App 未收齊可重新播放」

### 4. Error Handling（錯誤處理）

| 錯誤情境 | 偵測方式 | 處理策略 | 使用者回饋 |
|---------|---------|---------|-----------|
| 書庫為空 | books.length === 0 | 阻止流程 | 「書庫中沒有書籍」 |
| CompressionStream 不可用 | typeof === 'undefined' | 拋出 Error | 「壓縮功能不可用」 |
| gzip 壓縮失敗 | catch | 拋出 Error | 「資料壓縮失敗」 |
| QR 編碼失敗 | catch | 拋出 Error | 「QR 編碼失敗」 |
| qrcode-generator 載入失敗 | import 失敗 | 拋出 Error | 「QR 元件載入失敗」 |
| Canvas 不可用 | getContext('2d') null | 拋出 Error | 「繪圖功能不可用」 |
| 書庫讀取失敗 | storage.get 失敗 | catch | 「讀取書庫失敗」 |

日誌策略：所有 catch 記錄 Logger.error（含錯誤訊息 + 元件名稱）。

### 6. Acceptance（驗收條件）

- [ ] compressData 正確壓縮 JSON 為 gzip Uint8Array
- [ ] calculateCRC32 產出正確 CRC32（已知測試向量驗證）
- [ ] createFrames 每幀含 15 bytes header，magic 0x51 0x52，version 0x01
- [ ] total_frames/frame_index/total_size/crc32 欄位正確（big-endian）
- [ ] 壓縮後 <= 800 bytes 時 isStatic: true 且 frames 長度 1
- [ ] 壓縮後 > 800 bytes 時正確切塊，最後一塊允許 < chunkSize
- [ ] SyncQRRenderer Canvas 上 ~8fps 輪播
- [ ] 靜態模式顯示單張 QR 不啟動動畫
- [ ] maxCycles 到達後自動停止觸發 onComplete
- [ ] SyncPanel 完整流程（讀取 → 壓縮 → 切塊 → QR → 動畫）
- [ ] 空書庫顯示提示不產生 QR
- [ ] CompressionStream 不可用時顯示明確錯誤
- [ ] 所有 catch 有 error 日誌

### 7. 行為單元清單

| 行為單元 | GWT | 邊界條件 | 獨立可測 |
|---------|-----|---------|---------|
| gzip 壓縮 | Given JSON → When compressData → Then Uint8Array | 空字串、超大字串、API 不可用 | 是 |
| CRC32 計算 | Given Uint8Array → When calculateCRC32 → Then 正確值 | 空陣列、單 byte、已知向量 | 是 |
| Frame 切塊+Header | Given 壓縮資料 → When createFrames → Then 正確 header+payload | 整除/不整除 chunkSize、單塊 | 是 |
| QR 編碼 | Given frame → When encodeFrameToQR → Then QR 物件 | 超過容量上限 | 是 |
| 完整管線 | Given JSON → When encodeBookDataToQRFrames → Then frames+isStatic | 空書庫、小書庫(靜態)、大書庫(多幀) | 是 |
| Canvas 靜態繪製 | Given 單張 QR → When start(f, true) → Then 繪製一次 | context 不可用 | 是 |
| Canvas 動畫輪播 | Given 多幀 → When start(f, false) → Then 8fps 循環 | 1幀、maxCycles、pause/resume | 是 |
| 同步面板互動 | Given popup → When 點擊同步 → Then 完整管線 | 空書庫、壓縮失敗、使用者停止 | 是 |

### 8. Decision Questions

- Q1（跨模組共用）：不適用。三模組職責完全不同，無相似邏輯需共用。
- Q3-Q4（ID 設計）：不適用。不涉及持久化 ID 生成。
- Q5（出錯通知）：所有 catch 記錄 Logger.error + UI 錯誤訊息。
- Q6a（最慢操作）：gzip 壓縮 < 50ms + QR 編碼 < 200ms，點擊到動畫開始 < 500ms。
- Q6b（寫入失敗）：不適用。純讀取+即時運算，無多步驟寫入。

### 4. GWT 測試案例（按行為單元分群）

#### 群組 A：gzip 壓縮（qr-encoder-compress.test.js）

**場景 N1：正常 JSON 壓縮**
Given: 有效的 JSON 字串（含 format_version + books 陣列 + sync_meta）
When: 呼叫 compressData(jsonString)
Then: 回傳 Uint8Array，長度 > 0 且小於原始 UTF-8 byte 長度
AND: 前兩 bytes 為 gzip magic number 0x1f 0x8b

**場景 E2：CompressionStream 不可用**
Given: globalThis.CompressionStream 為 undefined
When: 呼叫 compressData(jsonString)
Then: 拋出 Error，訊息含壓縮不可用描述
AND: Logger.error 被呼叫（spy 驗證 call count >= 1 且參數含元件名稱）

**場景 E3：壓縮過程失敗**
Given: CompressionStream 存在但 writable.write 拋出 TypeError
When: 呼叫 compressData(jsonString)

### 1. 模組架構與資料結構選擇

**三模組職責**：

| 模組 | 層級 | 輸入 | 輸出 | 狀態性 |
|------|------|------|------|--------|
| qr-encoder | Domain（純邏輯） | JSON 字串 | QR 物件陣列 | 無狀態（純函式） |
| sync-qr-renderer | Presentation | QR 物件陣列 | Canvas 繪製 | 有狀態（播放狀態機） |
| sync-panel | Presentation | 使用者事件 | UI 更新 | 有狀態（UI 控制器） |

**核心資料結構**：

- **Frame 二進位格式**（固定 15 bytes header + 可變 payload）：
  ```
  Offset  Size    Type        Field
  0-1     2B      uint8[2]    magic: [0x51, 0x52]
  2       1B      uint8       version: 0x01
  3-4     2B      BE uint16   total_frames
  5-6     2B      BE uint16   frame_index (0-based)
  7-10    4B      BE uint32   total_size (壓縮後原始長度)
  11-14   4B      BE uint32   crc32 (全資料 CRC32)
  15+     var     uint8[]     payload (chunk data)
  ```

- **CRC32 查找表**：256 entries，uint32 陣列，啟動時一次計算

- **渲染器狀態機**：
  ```
  IDLE → PLAYING → PAUSED → PLAYING（resume）
       → IDLE（stop）
  PLAYING → COMPLETED（maxCycles 到達）→ IDLE（reset）
  ```

### 2. 虛擬碼（按群組）

#### 群組 A：gzip 壓縮

```
FUNCTION compressData(jsonString):
    IF CompressionStream 不存在:
        Logger.error("壓縮不可用", 元件名稱)
        THROW Error("壓縮功能不可用")

    encoder = TextEncoder()
    inputBytes = encoder.encode(jsonString)

    stream = new CompressionStream("gzip")
    writer = stream.writable.getWriter()
    reader = stream.readable.getReader()

    chunks = []
    // 非同步讀寫管線
    writer.write(inputBytes)
    writer.close()

    LOOP:
        result = reader.read()
        IF result.done: BREAK
        chunks.append(result.value)

    RETURN concatenate(chunks) → Uint8Array
```

#### 群組 B：CRC32 計算

```
// 啟動時建構查找表（模組層級常數）
CRC32_TABLE = buildTable():
    table = Array(256)
    FOR i = 0 to 255:
        c = i
        FOR j = 0 to 7:
            IF c AND 1:
                c = 0xEDB88320 XOR (c >>> 1)
            ELSE:
                c = c >>> 1
        table[i] = c
    RETURN table

FUNCTION calculateCRC32(data: Uint8Array):
    crc = 0xFFFFFFFF
    FOR EACH byte IN data:
        crc = CRC32_TABLE[(crc XOR byte) AND 0xFF] XOR (crc >>> 8)
    RETURN (crc XOR 0xFFFFFFFF) >>> 0   // unsigned 32-bit
```

#### 群組 C：Frame 切塊 + Header

```
FUNCTION createFrames(compressedData, chunkSize = 800):
    totalSize = compressedData.length
    crc32 = calculateCRC32(compressedData)
    totalFrames = CEIL(totalSize / chunkSize)
    frames = []

    FOR i = 0 to totalFrames - 1:
        offset = i * chunkSize
        chunk = compressedData.slice(offset, offset + chunkSize)

        header = new Uint8Array(15)
        header[0] = 0x51  // magic
        header[1] = 0x52  // magic
        header[2] = 0x01  // version

        // Big-endian writes
        writeUint16BE(header, 3, totalFrames)
        writeUint16BE(header, 5, i)           // frame_index
        writeUint32BE(header, 7, totalSize)
        writeUint32BE(header, 11, crc32)

        frame = concatenate(header, chunk)
        frames.append(frame)

    RETURN frames

// 輔助：Big-endian 寫入
FUNCTION writeUint16BE(buf, offset, value):
    buf[offset]     = (value >>> 8) AND 0xFF
    buf[offset + 1] = value AND 0xFF

FUNCTION writeUint32BE(buf, offset, value):
    buf[offset]     = (value >>> 24) AND 0xFF
    buf[offset + 1] = (value >>> 16) AND 0xFF
    buf[offset + 2] = (value >>> 8) AND 0xFF
    buf[offset + 3] = value AND 0xFF
```

FUNCTION encodeFrameToQR(frameData, options = {}):

    TRY:
        qr = qrcodeGenerator.create(0, ecLevel)
        qr.addData(frameData, 'Byte')  // byte mode

```
FUNCTION encodeBookDataToQRFrames(jsonString, options = {}):
    parsed = JSON.parse(jsonString)
    IF parsed.books.length === 0:
        THROW Error("書庫為空")

    chunkSize = options.chunkSize OR 800
    compressed = AWAIT compressData(jsonString)

    isStatic = compressed.length <= chunkSize

    frames_raw = createFrames(compressed, chunkSize)
    qrFrames = []
    FOR EACH frame IN frames_raw:
        qr = encodeFrameToQR(frame, options)
        qrFrames.append(qr)

    RETURN {
        frames: qrFrames,
        totalSize: compressed.length,
        isStatic: isStatic
    }
```

```
CLASS SyncQRRenderer:
    CONSTRUCTOR(canvas, options = {}):
        context = canvas.getContext('2d')
        IF context IS null:
            Logger.error("Canvas context 不可用", 元件名稱)
            THROW Error("繪圖功能不可用")

        this.canvas = canvas
        this.ctx = context
        this.fps = options.fps OR 8
        this.canvasSize = options.canvasSize OR 280
        this.maxCycles = options.maxCycles OR 3
        this.onProgress = options.onProgress OR noop
        this.onComplete = options.onComplete OR noop
        this.state = IDLE
        this.currentFrame = 0
        this.currentCycle = 0
        this.frames = []
        this.animationId = null
        this.lastFrameTime = 0

    METHOD start(frames, isStatic):
        this.frames = frames
        this.currentFrame = 0
        this.currentCycle = 0

        IF isStatic:
            drawQRToCanvas(this.ctx, frames[0], this.canvasSize)
            this.state = IDLE
            RETURN

        this.state = PLAYING
        this.lastFrameTime = 0
        requestAnimationFrame(this.animate)

    METHOD animate(timestamp):
        IF this.state != PLAYING: RETURN

        elapsed = timestamp - this.lastFrameTime
        interval = 1000 / this.fps

        IF elapsed >= interval:
            drawQRToCanvas(this.ctx, this.frames[this.currentFrame], this.canvasSize)
            this.onProgress(this.currentFrame, this.frames.length)
            this.lastFrameTime = timestamp

            this.currentFrame++
            IF this.currentFrame >= this.frames.length:
                this.currentFrame = 0
                this.currentCycle++
                IF this.currentCycle >= this.maxCycles:
                    this.state = COMPLETED
                    this.onComplete()
                    RETURN

        this.animationId = requestAnimationFrame(this.animate)

    METHOD pause():
        IF this.state == PLAYING:
            this.state = PAUSED
            cancelAnimationFrame(this.animationId)

    METHOD resume():
        IF this.state == PAUSED:
            this.state = PLAYING
            this.lastFrameTime = 0
            requestAnimationFrame(this.animate)

    METHOD stop():
        this.state = IDLE
        cancelAnimationFrame(this.animationId)
        this.ctx.clearRect(0, 0, this.canvasSize, this.canvasSize)
        this.currentFrame = 0
        this.currentCycle = 0
        this.frames = []

    METHOD getStatus():
        RETURN {
            playing: this.state == PLAYING,
            currentFrame: this.currentFrame,
            totalFrames: this.frames.length,
            currentCycle: this.currentCycle
        }

// 輔助：QR 繪製到 Canvas
FUNCTION drawQRToCanvas(ctx, qrCode, canvasSize):
    moduleCount = qrCode.getModuleCount()
    cellSize = canvasSize / moduleCount
    ctx.clearRect(0, 0, canvasSize, canvasSize)
    FOR row = 0 to moduleCount - 1:
        FOR col = 0 to moduleCount - 1:
            ctx.fillStyle = qrCode.isDark(row, col) ? '#000' : '#FFF'
            ctx.fillRect(col * cellSize, row * cellSize, cellSize, cellSize)
```

**技術債務標記**：
- TD-001：QR 繪製效能（fillRect vs ImageData），影響低，Phase 4 評估
- TD-002：chunkSize 動態計算（依 QR version 最大容量），影響低，Phase 4 評估
- TD-003：Canvas 尺寸自適應，影響低，Phase 4 評估

### 7. JavaScript 特定實作注意事項

**CompressionStream 非同步管線**：
- WritableStream + ReadableStream 的 reader/writer 皆為 Promise API
- 必須在 write+close 後才 read，否則可能讀到不完整資料
- 測試中 Mock CompressionStream 需模擬完整 stream 行為

**Big-endian 寫入**：
- JS 原生為 little-endian，手動位移寫入或用 DataView
- 建議用 DataView.setUint16/setUint32 搭配 big-endian flag

**qrcode-generator 函式庫**：
- byte mode 需傳入 Uint8Array 或等效 byte 陣列
- QR version auto-selection 需注意：version 20-25 的 byte mode 容量為 816-1182 bytes (ECL M)
- esbuild bundle 時需確認 CJS/ESM 相容性

**requestAnimationFrame 測試**：
- Jest 需 jest.useFakeTimers + 手動推進
- rAF callback 收到 DOMHighResTimeStamp，Mock 時需提供遞增值
- 注意 test-assertion-design-rules：禁止計時門檻斷言，用 call count 驗證
