### 問題根因

Phase 4 多視角評估發現 QR 模組三個認知負擔問題：

1. **compressData（qr-encoder.js:39-74）認知負擔指數 23**：stream 讀取 + chunk 收集 + 位元組合併三個職責混在一個函式。變數數 8 + 分支數 3 + 巢狀深度 3 + 依賴數 2 = 16-23。
2. **animate（sync-qr-renderer.js:86-109）巢狀 4 層**：state guard > elapsed check > frame advance > cycle check，四層巢狀 if。
3. **_buildSyncJSON（sync-panel.js:114-124）Domain 知識外洩**：同步 JSON 格式（format_version 2.0 + sync_meta）是 Domain 知識，不應在 Presentation 層 SyncPanel 中定義。
4. **STORAGE_KEY 硬編碼**：sync-panel.js:19 本地定義 `STORAGE_KEY = 'readmoo_books'`，但 module-constants.js:521 已有 `STORAGE_KEYS.READMOO_BOOKS`。
