### 張力項 #1 的初步分析方向

- **既有假設**：pm-role.md 行為循環、agent-dispatch-template、PC-042（subagent ~20 tool call 限制 → PM 前台分析 + 代理人實作分工）皆以單層派發為前提
- **潛在收益**：大型任務可由協調型 agent 自行拆分子 agent，減少 PM 中轉回合
- **潛在風險**：(a) 嵌套層的 commit / ticket 歸屬鏈不可見（PC-166 ghost branch 風險放大）；(b) dispatch hook（prompt 行數上限、路徑分類）對嵌套層無法攔截；(c) token 預算失控（5 層 × 並行）
- **敏感性**：影響整個專案互動模式，採用方向屬用戶決策範圍（skill 反模式表：敏感規則不擅自落地）
