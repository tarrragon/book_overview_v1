### 跨 consumer 實證分析（已貼入 tarrragon/claude#3 comment）

掃描 15 個框架 consumer，識別五類專案類型：

| 類型 | 版本源 | worklog 樣式 | 現行 skill 支援 |
|------|--------|-------------|----------------|
| Chrome Extension | package.json + manifest.json | flat | 有（唯一） |
| Flutter | pubspec.yaml (X.Y.Z+build) | nested 3-level | 無 → 假陽性 |
| Go | git tag 或 VERSION 文字檔 | flat | 無 |
| PHP+npm monorepo | 子專案各自 package.json/composer.json | flat | 無 |
| Flutter+Go monorepo | ui/pubspec.yaml + server/go.mod | flat | 無 |

三設計維度：(1) 版本源多樣性 (2) worklog 路徑樣式 (3) monorepo 多版本源。
.version-release.yaml schema 方向與自動偵測 fallback 設計見 issue #3 comment。

關鍵事實：15 個 consumer 無一有 .version-release.yaml，全靠硬編碼 package.json。
