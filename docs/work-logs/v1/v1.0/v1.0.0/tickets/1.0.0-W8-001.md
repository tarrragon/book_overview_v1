## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

auto-commit graceful degrade 後 ticket body 退回「僅存於 working tree」的脆弱狀態（W1-017 原始風險）。degrade 觸發時唯一通知管道是 stderr，但自動序列（sync-push / worktree merge / 後續 append-log）不讀 stderr —— 等於無人接住。

### 影響範圍

- `.claude/skills/ticket/ticket_system/lib/git_utils.py` — auto-commit 實作
- `.claude/skills/ticket/ticket_system/commands/track_acceptance.py:757-772` — 呼叫端 graceful degrade 處理

### 相關 Error Pattern

- PC-148（hook 雙重註冊 shebang bypass）：類似「靜默失敗」風險但不同層級
- W7-001 否決方案 C（warning hook）的論證同樣適用本問題

### Degrade 觸發條件盤點

| 觸發條件 | return/exception | 發生情境 | 頻率估計 |
|---------|-----------------|---------|---------|
| index.lock 並發 | `git_failed`（git add/commit returncode≠0）或 `TimeoutExpired` exception | 並行 agent commit / PM commit / pre-commit hook 長時間鎖定 | 中（並行派發常見） |
| 非 git repo | `not_git_repo` | ticket CLI 於非 repo 環境執行（極罕見） | 極低 |
| git 未安裝 | OSError exception | 環境異常 | 極低 |
| commit timeout（30s） | `TimeoutExpired` exception | pre-commit hook 卡死 / 網路 credential helper hang | 低 |
| git add 失敗 | `git_failed` | 權限問題 / 磁碟滿 / 檔案被鎖 | 極低 |

**核心洞察**：唯一中頻觸發條件是 index.lock 並發（並行 agent 場景），其他均為極端邊緣。

### 實際後果鏈

1. append-log 寫入 → save_ticket 成功（body 在 working tree）
2. auto-commit degrade → stderr 警告
3. 後續若發生 `git checkout -- <file>` / `git reset --hard` / `git stash` → body 遺失
4. 但步驟 3 需要**人或 agent 主動執行破壞性 git 操作**才觸發

**關鍵事實**：degrade 後 body 不會**自動**遺失；需要額外破壞性操作才遺失。下一次 append-log 呼叫會再嘗試 auto-commit（含累積的所有 body 變更），形成自然重試。
