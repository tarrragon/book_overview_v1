## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

acceptance-gate-hook 步驟 6 呼叫 `check_execution_log_filled` 只查 Solution / Test Results 兩章節。ANA type ticket 的核心產出「重現實驗結果」章節未納入檢查；且 `_is_section_empty` 的剝除邏輯未涵蓋 H3 子標題骨架（`### 實驗方法`）與「（必填：...）」placeholder，使 ANA 重現實驗結果只剩骨架的空殼章節被誤判為非空，complete 放行（W7-003 reproduction placeholder 案例）。

### 影響範圍

- `.claude/hooks/acceptance_checkers/execution_log_checker.py`：`check_execution_log_filled` + `_is_section_empty`
- caller wiring：`.claude/skills/ticket/hooks/acceptance-gate-hook.py` 步驟 6（傳入 ticket_type 才能啟用 ANA 檢查，否則為 dead code）

### 相關 Error Pattern

PC-110 空殼 complete 同模式（schema section placeholder false negative）；W17-071 為前序修復（HTML 註解 + 分隔符 + 自定義 H2 邊界），本 ticket 延伸涵蓋 H3 骨架 + 必填 placeholder + ANA reproduction 檢查。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 三項修復

1. **ANA reproduction 檢查**：`check_execution_log_filled` 新增 `ticket_type` 參數（預設 None 維持向後相容）。當 `ticket_type == "ANA"` 時額外查「重現實驗結果」章節，空殼即視為未填寫（OR 邏輯：即使 Solution/Test Results 有內容，ANA reproduction 空殼仍 flag）。
2. **H3 骨架剝除**：`_is_section_empty` 新增 `^#{3,}\s.*$` 行剝除，使「### 實驗方法」等骨架標題不被當實質內容。
3. **必填 placeholder 剝除**：新增 `（必填[^）]*）` 剝除，與既有 `（待填寫...）` 同等處理。

### caller wiring

`acceptance-gate-hook.py` 步驟 6 改傳 `ticket_type=frontmatter.get("type", "")`，使 ANA 檢查生效（不接線則為 dead code）。此為單行向後相容 kwarg 傳遞，與並行 agent 的 git_utils.py 修改互不重疊。

### 設計決策

- ANA 採 OR 邏輯（reproduction 空殼即 flag），因重現實驗是 ANA 核心產出，不應被 Solution 有內容掩蓋。
- H3 剝除用 `#{3,}` 涵蓋 H3 及更深，避免遺漏。
- 不過度剝除：H3 標題下有實質內容時保留（test_h3_with_real_content_not_empty 驗證）。
