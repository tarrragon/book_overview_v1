### 文件變更摘要

**寫入位置決策**：補入 `docs/environment-recovery-guide.md`（既有環境恢復 SOP），不另建新檔。理由：(1) 該檔已是「環境異常排查」的權威入口（四層環境問題對照表、E2E 環境恢復皆在此）；(2) hook 系統異常屬同一「session 跑不動 / 行為異常」診斷情境；(3) dispatch 約束限 docs/ 範圍，`.claude/references/` 不在本票可改範圍。

**新增章節**：「Claude Code Hook 系統隔離除錯（--safe-mode）」，插於「恢復後驗證清單」與「相關 Ticket 與文件」之間，含四個子節：

| 子節 | 內容 |
|------|------|
| 使用時機 | 適用/不適用場景表 + 二分定位邏輯（safe mode 下問題消失 → 客製化層；仍在 → CC 本體或環境） |
| 兩種啟用方式 | `claude --safe-mode` flag 與 `CLAUDE_CODE_SAFE_MODE=1` 環境變數，兩者等效 |
| 與 hook-health-monitor 互補關係 | 對照表：health-monitor 平時偵測靜默失敗（SessionStart hooks 日誌目錄新鮮度）、safe mode 出事時全量隔離；建議先看零成本的啟動報告再動用 safe mode |
| 注意事項 | safe mode 下 90 個 hook + CLAUDE.md/rules 注入 + 自訂 settings 全失效；僅供診斷，不在 safe mode session 做 commit / complete 等持久副作用操作 |

**同步更新**：「相關 Ticket 與文件」表新增本票與 `hook-health-monitor.py` 兩列；footer bump 至 1.2.0。

**Acceptance 對應**：
- 指引含使用時機、兩種啟用方式、與 hook 健康檢查機制的關係 → 四子節齊備
- 寫入位置與既有文件結構一致（不重複建檔）→ 補入既有 guide，章節層級（## / ###）與全檔一致
