# Execution Log

## Task Summary

評估 disableBundledSkills 對 context 成本的實際收益量測

---

## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

CC 2.1.169 引入 `disableBundledSkills` 設定，可停用 CC 內建 bundled skills。本專案擁有 46 個自有 skill，bundled skills 是否佔用不必要 context 待量測。

### 影響範圍

- `.claude/settings.json`（設定檔）
- 每 session system-reminder 的 skills 清單注入量

### 相關 Error Pattern

無直接相關 error pattern。與 W16-003（skills 注入成本分析）為延續關係。

### 前置發現

W16-003（2026-04-19）已完成全量分析，W16-003.1 已卸載 document-skills + example-skills。但 `claude plugin list` 顯示兩者**目前已重新安裝**（可能用戶手動或 CC 更新自動恢復）。`disableBundledSkills` 是 CC 2.1.169 新增的 settings.json 層級控制，比 plugin uninstall 更持久且不影響其他專案。

## 重現實驗結果
### 實驗方法

1. **Token 成本估算**：以 W16-003 已量測的 skill description 平均字元數與 token 換算率（~2.8 char/token 中英混雜），計算 bundled skills 在 skill 清單的注入成本
2. **使用頻率盤點**：grep work-logs 與 transcripts 中 bundled skill 類型關鍵字（pdf/docx/pptx/xlsx/canvas-design/frontend-design/mcp-builder）的提及次數
3. **現況盤點**：`claude plugin list` 確認當前已安裝 plugin 與 bundled skills 狀態

### 實驗執行

**1. 當前 plugin 狀態（2026-06-11）**

`claude plugin list` 結果：16 個 plugin，其中 14 個啟用。與 bundled skills 相關者：

| Plugin | 狀態 | Skills 數（W16-003 估） | 本專案使用？ |
|--------|------|----------------------|------------|
| `document-skills@anthropic-agent-skills` | 啟用 | ~18 | 否 |
| `example-skills@anthropic-agent-skills` | 啟用 | ~14（與 document-skills 重複） | 否 |
| `frontend-design@claude-plugins-official` | 啟用 | 待確認 | 否（Chrome Extension 非 frontend-design 範疇） |

注意：W16-003.1（2026-04-19）已卸載 document-skills + example-skills，但目前已重新出現。

**2. Token 成本量測**

基於 W16-003 §3 已量測數據（估算方法：W16-003 抽樣 system-reminder skill description 字元數 → 除以 2.8 char/token 換算率）：

| 項目 | Description 平均字元 | 預估 tokens | 數量 | 小計 tokens |
|------|--------------------|-----------|----|-----------|
| document-skills skills | ~600 | ~160 | ~18 | ~2,880 |
| example-skills skills（重複） | ~600 | ~160 | ~14 | ~2,240 |
| CC 內建 skills（init/review/security-review） | ~300 | ~80 | ~3 | ~240 |
| **bundled 小計** | — | — | ~35 | **~5,360** |
| 本專案自有 46 skills | ~400 | ~110 | 46 | ~5,060 |
| **全量 skills 清單** | — | — | ~81+ | **~10,420+** |

**bundled skills 佔全量 skills 清單約 51%（5,360 / 10,420）**。

修正注意：W16-003 §3.2 修正估算全量 skills 為 ~7-10K tokens/session。以 8K 中值計，bundled 佔約 5K，自有佔約 3K（自有 skill descriptions 普遍較短）。

**3. 使用頻率盤點**

grep `docs/work-logs/` 中 bundled skill 相關關鍵字：

- `pdf`：6 筆命中，均為 W16-003 分析報告本身或 ticket 描述，**零實際使用**
- `docx`：散見於匯出功能討論，**零透過 bundled skill 處理**
- `pptx`/`xlsx`：零命中或僅 W16-003 分析引用
- `canvas-design`/`frontend-design`/`mcp-builder`：零命中
- `init`/`review`/`security-review`（CC 內建）：`review` 有命中但為本專案自有 code-review plugin，非 bundled；`security-review` 本專案自有同名 skill 覆蓋

**結論：本專案對 bundled skills 的實際使用頻率為零。**

### 實驗發現

**已驗證事實**：
1. document-skills + example-skills 已重新出現（W16-003.1 卸載後復裝）
2. bundled skills 佔 skill 清單 token 成本約 ~5K tokens（全量 ~8-10K 的 50-60%）
3. 本專案 46 個自有 skill 完全覆蓋工作流程需求，bundled skills 使用頻率為零
4. CC 內建 3 個 skills（init/review/security-review）中 security-review 已被自有同名 skill 覆蓋

**仍未驗證假設**：
1. `disableBundledSkills` 的確切作用範圍——是否僅停用 CC 內建 3 個，還是同時影響 plugin-provided skills？（需 CC 官方文件確認或實測）
2. skills 清單是否每 turn 重送或有 CC 側快取（W16-003 假設每 turn 重送但未實證）
3. plugin skills（document-skills 等）是否歸類為「bundled」——若不歸類，`disableBundledSkills` 僅影響 CC 內建 ~3 個 skills（~240 tokens），收益極低

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 採用建議

**建議：分兩層處理，合併收益最大化**

#### 層 1：再次卸載 document-skills + example-skills plugin（立即可做，確定收益）

| 指標 | 值 |
|------|-----|
| 可省 tokens | ~5K/session（skills 清單的 50-60%） |
| 使用頻率 | 零（7 個月跨 18+ Wave 無任何實際使用） |
| 風險 | 極低（一行指令可重裝） |
| 操作 | `claude plugin remove document-skills@anthropic-agent-skills` + `claude plugin remove example-skills@anthropic-agent-skills` |

W16-003.1 已執行過此操作但 plugin 復裝。建議同時在 `.claude/references/plugin-management.md` 記錄「已評估不需要」清單，防止未來重裝。

#### 層 2：啟用 `disableBundledSkills`（待確認作用範圍）

| 情境 | 收益 | 建議 |
|------|------|------|
| 若 `disableBundledSkills` 涵蓋 plugin-provided skills | ~5K tokens（與層 1 重疊，擇一即可） | 優先使用此設定（比 plugin uninstall 更持久、專案級隔離） |
| 若 `disableBundledSkills` 僅涵蓋 CC 內建 3 個 skills | ~240 tokens（微量） | 收益不值得啟用的複雜性；僅做層 1 |

**建議操作**：先做層 1（確定收益 ~5K tokens），再小規模實測 `disableBundledSkills: true` 確認作用範圍。若涵蓋 plugin skills，可改用層 2 取代層 1（更優雅）。

#### 量化摘要

| 方案 | 預估節省 | 確定性 | 建議 |
|------|---------|--------|------|
| 卸載 document-skills + example-skills | ~5K tokens/session | 高（W16-003 已量測） | 立即執行 |
| 額外卸載 frontend-design | ~0.5-1K tokens | 中 | 併入評估 |
| disableBundledSkills（若涵蓋 plugin） | ~5K（與卸載重疊） | 待驗證 | 先實測再決定 |
| disableBundledSkills（若僅 CC 內建） | ~240 tokens | 待驗證 | 收益過低不建議 |
