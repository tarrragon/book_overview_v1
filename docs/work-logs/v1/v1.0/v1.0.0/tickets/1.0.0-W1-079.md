分析 agent 定義 model 欄位行內註解的間歇性解析失敗（inherit + # 註解被當 model 值）

## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

直接原因：basil-writing-critic 派發時 runtime 拒絕其 model 值（錯誤回顯完整污染字串 `inherit  # 註解...`，含行內註解）。深層機制經受控實驗修正初始假設：(1) 失敗對 basil 是**確定性**而非間歇（6/6）；(2) 同字面的 thyme/linux 7/7 成功，故「行內註解被當 model 值」不是充分條件；(3) agent 定義在 session 啟動時快取，session 內檔案修改對 resolver 不可見——真正 discriminator 位於 runtime 快取/解析層，本 session 不可觀測（詳見重現實驗結果）。

### 影響範圍

- 直接：basil-writing-critic 本 session 無 model 覆寫即無法派發（Layer 2 審查流程受阻，已用覆寫參數 workaround）
- 潛在：16 個 agent 檔同 commit（b7f67fb5）引入同字面 model 行；現有證據顯示僅 basil 受影響，但 discriminator 未定位前無法排除其他 agent 在不同快取條件下踩中
- 框架知識：「agent 定義 session 快取、in-session 編輯不可見」影響所有 agent 定義修改的驗證流程（必須跨 session 驗證）

### 相關 Error Pattern

無既有 PC/IMP 直接對應（agent 定義解析層首例）。鄰近模式：PC-115（subagent .claude/ Edit transient deny，同屬 runtime 行為不可預測類）；快取知識已另寫入 memory 供跨 session 召回。

### 已驗證事實

1. **非間歇**：basil 同 session 內 6/6 確定性失敗；thyme/linux 同字面 7/7 成功
2. **agent 定義 session-start 快取**：session 內編輯 agent .md（tools / model 欄位）對 dispatch resolver 完全不可見，錯誤訊息持續回顯快取的舊字串——本 session 無法以檔案修改驗證任何修復
3. **model 覆寫參數為有效 workaround**（繞過快取的壞值路徑）
4. 三 agent 的 model 行同字面、同 commit 引入（b7f67fb5，16 agent opus→inherit 統一），排除字面差異與引入時間差
5. 失敗發生在 agent 啟動前（usage 0 tokens / 0 tool uses），錯誤為模型驗證拒絕

### 仍未驗證假設

- basil 與 thyme/linux 同字面不同命運的 discriminator（位於 runtime 快取/解析層，session 內不可觀測；候選：tools 含未連線 MCP server 工具、permissionMode 有無、欄位組合——H-tools 實驗因快取失效未獲證據）
- 乾淨 `model: inherit`（已寫入檔案）能否在新 session 修復——需新 session 驗證（spawn W1-079.1）

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 裁決

| 項目 | 決定 |
|------|------|
| basil 檔案處置 | model 行清理為 `model: inherit`（移除行內註解，本 commit）；tools 行還原含 mcp__zhtw-mcp__zhtw（H-tools 未獲證據，維持最小 diff） |
| 其餘 15 個同寫法 agent | **不批量修改**——thyme/linux 實證污染字面可正常運作，批量改動承擔回歸風險而無已證收益；待 W1-079.1 驗證結果再決定是否擴大清理 |
| 過渡 workaround | 修復驗證前，派發 basil 必帶 `model` 覆寫參數（sonnet/opus 依任務），已實證可完整工作 |
| 驗證 | spawn 1.0.0-W1-079.1（新 session 啟動後 probe basil；仍失敗則依序試移除 model 行 / 明確 model 值 / 移除 MCP 工具引用，逐步隔離 discriminator） |
| 上游回報 | 暫不報 CC issue——discriminator 未定位，報告無最小重現；W1-079.1 隔離出單一變因後再評估 |
