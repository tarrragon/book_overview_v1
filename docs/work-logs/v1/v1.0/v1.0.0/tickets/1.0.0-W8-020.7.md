## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->
### 決策：方案 A（合併）

以 parallel-evaluation-methodology 為存活核心，折入 multi-perspective-analysis 獨特核心，刪除 multi-perspective-analysis 並重導引用。

### 決策理由

兩方法論皆為「並行多視角派發 → 彙整 → 決策」，差異僅在報告格式（multi-perspective 偏理解型分析報告，parallel-evaluation 偏品質掃描清單），**派發機制完全相同**——這是典型應合併的重疊。選 A 而非 B 的三個依據：

1. **既有交叉依賴指向合併**：parallel-evaluation SKILL.md L19 原本就「詳見 multi-perspective「衝突處理策略」」，兩檔互相依賴同一概念，雙核心會持續維護兩份。
2. **執行層只有一套**：parallel-evaluation 有完整 skill（SKILL + 4 references）+ README Phase 4a/4c 引用；multi-perspective 無 skill backing，是純原則層。存活核心應選有執行層的一方。
3. **campaign 目標一致**：W8-020 campaign 針對「方法論重疊」收斂，兩檔覆蓋同一派發模式正是目標。
