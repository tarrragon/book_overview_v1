### WRAP（認領三問）
- W（Widen）：候選 (a) SKILL + shell/py 腳本包裝 gh【採用，對齊 /ticket /error-pattern 框架既有模式】；(b) 每次臨時拼 gh 指令【否決：無標準化、重複思考】；(c) GitHub MCP 工具直呼【否決：非可重用 skill 工作流】。
- A（Attain distance）：機會成本低，純 .claude/ 基礎建設，不擠壓 src 開發。
- P（Prepare to be wrong）：最可能失敗 = gh 降級路徑不夠健壯而 crash。防護：preflight 三態檢查（未裝/未登入/Issues 停用）+ subprocess 例外捕捉 + 各路徑測試覆蓋。

### 交付內容（首建範圍：scaffold + create/list）
- `.claude/skills/framework-issue/SKILL.md`：frontmatter（name: framework-issue，description 含觸發詞 framework issue / canonical / 跨 consumer 修復追蹤）+ commands/usage/降級/測試文件。
- `scripts/gh_common.py`：共用 preflight + run_gh，將 gh 三種不可用狀態轉為清楚 stderr 提示 + exit 3（EXIT_DEGRADED），不拋 traceback（可觀測性規則 4）。
- `scripts/create_issue.py`：包裝 `gh issue create --repo tarrragon/claude.git`（--title/--body/--label）。
- `scripts/list_issues.py`：包裝 `gh issue list --repo tarrragon/claude.git`（--state/--label/--limit/--search）。
- `tests/test_framework_issue.py`：6 測試（正常 create/list + 降級：未裝/未登入/Issues 停用/subprocess 例外），全程 mock gh，不真打 GitHub API。
