### 技術細節（PM 已獨立 grep 驗證）

檔案：.claude/skills/ticket/ticket_system/lib/file_lock.py（386 行）

三函式各定義兩次，Python 取最後 def，故後套為活碼、前套為死碼被 shadow：
| 函式 | 死碼(前/FileLock 版) | 活碼(後/fcntl 版) |
|------|------|------|
| reap_stale_locks | L98 | L241 |
| _try_acquire_create_lock | L171 (回傳 Optional[FileLock]) | L316 (回傳 Optional[IO[str]]) |
| create_id_allocation_lock | L199 | L344 |

根因：W9-001 把 file_lock() 主路徑遷移到 filelock 後端（FileLock 版），但未刪舊 fcntl 版；overlay sync 文字層合併把兩套都保留（無文字衝突）。後續補 fcntl gate 讓舊版能跑，但新版仍死碼並存。屬 ARCH-V1-002 模式。

### 後端決策（須先定）
- 選項 A（對齊 W9-001 遷移意圖）：保留 FileLock 版、刪 fcntl 版。需驗證 FileLock 版功能正確（它一直是死碼從未實跑）+ 跨平台（Windows graceful degrade）。
- 選項 B（保守，保留現行行為）：保留 fcntl 版、刪 FileLock 版。簡單但回退 W9-001 跨平台意圖。
- 決策依據：以「哪套通過完整測試 + 符合 filelock 跨平台目標」為準，於 Solution 記錄選擇理由。

### 要求
1. 收斂三函式各為單一定義（grep '^def <name>' 計數=1）。
2. 補回歸測試鎖定「同名 top-level 定義唯一」（防 ARCH-V1-002 復發）。
3. file_lock 模組 import + ticket create/claim/complete 全綠。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 後端選擇：選項 A（保留 FileLock 版，刪 fcntl 版）

**決策依據（evidence-driven）**：

1. **活碼 `file_lock()` 本身已是 FileLock 後端**（L55，單一定義無重複）。即「FileLock 後端從未實跑」只適用於 4 個重複的 helper，主鎖路徑 `file_lock()` 在生產環境一直跑 filelock，已驗證可用。保留 FileLock 版 helper 可與既有主路徑後端一致。
2. **FileLock 版 helper 不依賴 `_HAS_FCNTL` / `fcntl`**（grep 確認 L98-234 僅 docstring 提及 fcntl，code 無引用）。故選 A 後可一併移除整個 `try: import fcntl` gate（原 L42-51），徹底消除 POSIX-only 依賴，符合 W9-001 Windows graceful-degrade 跨平台意圖。
3. **選項 B（保留 fcntl）會回退 W9-001**：重新引入「無 fcntl 平台降級」分支與 POSIX-only 依賴，與模組 docstring（L17-32 跨平台設計宣告）矛盾。
4. **完整測試通過**：選 A 後 1870 passed（baseline 1868 + 2 新測試），ticket create/claim/complete/query 正常。

### 收斂內容

實際重複的 top-level 定義為 **4 函式 + 1 常數**（ticket 描述為 3，實際多 `_warn_create_lock_degraded` 與 `CREATE_LOCK_FILENAME`）：

| 定義 | 動作 |
|------|------|
| reap_stale_locks | 刪 fcntl 版（L241），保留 FileLock 版（L98） |
| _try_acquire_create_lock | 刪 fcntl 版（L316），保留 FileLock 版（L171） |
| create_id_allocation_lock | 刪 fcntl 版（L344），保留 FileLock 版（L199） |
| _warn_create_lock_degraded | 刪重複（L307），保留 L162 |
| CREATE_LOCK_FILENAME | 刪重複（L304），保留 L159 |
| `try: import fcntl` gate | 刪除（不再被任何 code 引用） |
| `from typing import IO` | 刪除 `IO`（去重後無用） |
