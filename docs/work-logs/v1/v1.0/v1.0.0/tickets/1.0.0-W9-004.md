## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復實作（framework issue #2）

### 變更檔案
- .claude/hooks/hook-completeness-check.py：新增反向檢查（原僅正向：檔存在未註冊）。
  - _resolve_command_path(command, project_root)：解析 $CLAUDE_PROJECT_DIR/${...} + interpreter 前綴（取首個 .py token），非 .py 回 None。
  - extract_registered_commands：回 (event_type, matcher, command) 三元組（保留 matcher）。
  - find_phantom_registrations：掃 settings.json + settings.local.json，已註冊但檔不存在者列出（runtime 崩潰類型）。
  - find_duplicate_registrations：偵測同 hook 同 (event, matcher) 跨檔/多次註冊。
  - main() 整合：幽靈註冊 + 重複註冊報 WARNING（幽靈額外寫 stderr，quality-baseline 規則 4）。
- .claude/hooks/tests/test_hook_completeness_reverse_check.py（新）：12 測試。

### 關鍵設計：matcher 防誤判
首版以 (event, path) 為 key，誤把同 hook 在 PreToolUse 不同 matcher（Edit/Write 多工具覆蓋）報為重複（runtime 實測 12+ 誤報）。修正為 key 含 matcher：不同 matcher → 不同 key → 不誤報；Stop（matcher=""）跨檔仍正確偵測（issue #2 真正關切）。

### issue #2 建議覆蓋
- 建議1 反向檢查：已實作（幽靈註冊偵測）。
- 建議2 搬遷遷移指引：由 WARNING 訊息提供（「移除殘留註冊...hook relocate 後常見於 settings.local.json」），SessionStart 主動暴露。
- 建議3 跨檔重複偵測：已實作（matcher-aware）。

### Runtime 驗證
本 repo 跑 hook：無幽靈/重複（settings.local.json 乾淨，不同於 wotel Windows 案例）；matcher 修正後多-matcher 誤報全消。

- [x] 測試通過率：15/15（12 反向檢查 + 3 既有 autocommit）
- [x] false-positive 回歸防護：test_different_matchers_not_flagged
- [x] 可觀測性：幽靈註冊 print + logger.warning + stderr 三通道（崩潰類型高可見）
