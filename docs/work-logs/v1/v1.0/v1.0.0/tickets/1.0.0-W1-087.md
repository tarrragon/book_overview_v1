**整合點（how.strategy）**：在 sync-claude-push.py 的 stage_tracked_tree() 後、git push 前新增 run_framework_smoke_test(staging_dir)：於 staging 樹 import ticket_system 全模組 + 跑關鍵測試子集，紅燈 sys.exit(1)，與既有 validate_version_bump 並列。

**重要——與近期變更整合**：sync-claude-push.py 剛新增 create_and_push_version_tag()（push main 成功後打 tag）。smoke test 閘門應在「push 之前」攔截，即位於 push（含 tag）動作之前；確認新函式的呼叫順序，smoke test 紅燈時不得進入 push/tag 任何階段。

**為何不用 CC hook 或人工 PR**：CC event-model 不支援此攔截點；人工 PR 對單人 repo 是空殼且把機械缺陷誤當流程問題（PC-V1-009）。故採 push 前自檢函式。

**框架約束**：改 .claude/scripts/ 框架碼會 sync 全 consumer——禁嵌 project-local ticket ID 於 code/註解（跨專案斷源，reference-stability-rules 規則 8）；註解用通用描述。
