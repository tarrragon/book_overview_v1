## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
新增 run_framework_smoke_test(staging_dir) 至 .claude/scripts/sync-claude-push.py，作為 C1 生產閘。

整合點：main() 中 stage_tracked_tree() 取得 git tracked 樹後、git push（含 create_and_push_version_tag）任何階段「之前」呼叫（行序確認：staging try 區塊內，push 在 1391、tag 在 1407，smoke test 在 staging 取得後立即執行）。紅燈 sys.exit(1)，與 validate_version_bump 並列為 push 前 fail-fast 防護，紅燈時不進入 push / tag。

機制：以 pkgutil.walk_packages 走訪 staging 樹核心套件 ticket_system 全生產模組逐一 import。
- 框架內部模組缺失（ModuleNotFoundError name 屬 ticket_system，即 import 殘留缺陷）/ 語法錯誤 / 其他 import 期例外 → 視為缺陷 → exit 1。
- 已知第三方依賴缺失（filelock / yaml / pyyaml / pytest，push 零依賴環境本就無）→ 容忍，零誤報。
- 測試模組（.tests）跳過（import pytest 等非下游執行依賴）。
- import 隔離：獨立 sys.path 前綴 + 結束後清理已 import 的 ticket_system 模組，不污染 push 腳本自身 import 狀態。

框架約束遵循：code/註解全程用通用描述（「機械缺陷」「import 殘留」「C1 生產閘」），未嵌任何 project-local ticket ID。
