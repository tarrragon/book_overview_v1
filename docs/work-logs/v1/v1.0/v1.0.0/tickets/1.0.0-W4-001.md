### 關鍵考量驗證結果（必先驗證項，已實證）

v3 canonical 為 everything-as-tags，`tagTree` 由 book 物件自身欄位衍生，**不需** storage tags/tagCategories（與 v2 不同），故 handleExportJSONv3 無 `_loadTagData` 步驟、無需 chrome.storage。

實證（node spike + Group H 測試）：
- `mapV1BookToCanonical(book)`：`book.tagIds` → `tags.custom = [{id,name}]`；`book.authors` → `tags.author`；`book.source` → `tags.platform`。
- `buildTagTree(canonicalBooks)`：聚合 `tags.custom` → `tagTree.custom` 扁平節點（locked:false）。
- `_getBooksForExport()` 回傳 `filteredBooks`，book 已攜帶 tagIds（filter 管線 line 597 已用 book.tagIds 為證），故實際匯出 `tagTree.custom` 非空（測試斷言 contains 科幻/長篇）。

結論：無需比照 v2 補載 storage tag。v3 路徑比 v2 更簡單（無 async storage 載入）。
