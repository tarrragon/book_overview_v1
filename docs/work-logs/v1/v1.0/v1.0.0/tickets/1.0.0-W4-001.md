## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因（PM 前台確認，2026-06-07）

單一根因：Overview 匯出 UI 從不傳 `options.formatVersion=3.0.0`，故 exporter 已實作的 v3 canonical 路徑永遠走不到。

- exporter v3 路徑**已存在**：`src/export/book-data-exporter.js`
  - `exportToJSON` line 413：`if (options.formatVersion === '3.0.0')` → `_exportToJSONCanonical`（line 573）
  - `_exportToJSONCanonical`：逐書 `mapV1BookToCanonical(book)`（everything-as-tags）→ `buildTagTree` → 組 root `{ format:'book-interchange-v1', formatVersion:'3.0.0', metadata, books, tagTree }`
- controller UI **硬編碼 v2**：`src/overview/overview-page-controller.js`
  - line 84：`EXPORT_V2.FORMAT_VERSION: '2.0.0'`
  - `handleExportJSONv2()`（line ~920）：傳 `formatVersion: CONSTANTS.EXPORT_V2.FORMAT_VERSION`（永遠 2.0.0）
  - 按鈕綁定 line 286-290：`exportJSONBtn` → `handleExportJSONv2()`

### 影響範圍

- `src/overview/overview-page-controller.js`（按鈕綁定 + 新增 handleExportJSONv3 + CONSTANTS.EXPORT_V3）
- `src/overview/overview.html`（新增 v2 相容按鈕元素）
- `tests/unit/overview/overview-page-controller.test.js`（RED：v3 wiring + v2 仍可達）
- 既有 exporter v3 canonical 已有測試 `tests/unit/export/book-data-exporter-canonical.test.js`（本票不改 exporter）

### 相關 Error Pattern

- 無直接 error-pattern。注意 PC-165（false-positive fix）：本票改 UI 行為，acceptance 已含 `npm test 全綠`，且 RED 須斷言匯出檔實際 `format`/`formatVersion` 字面（非僅檢查方法被呼叫）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 設計決策（用戶拍板：方案 B）

主「匯出 JSON」改 v3 canonical；新增「匯出 JSON (v2 相容)」次選鈕走既有 v2；v2 仍可達（acceptance 3 不破壞）。

UI 佈局（overview.html `.export-buttons`）：
```
[匯出 CSV] [匯出 JSON] [匯出 JSON (v2 相容)] [匯入 JSON] ...
                ↑ v3 canonical    ↑ 新增，走既有 handleExportJSONv2
```

### 實作步驟（語言無關策略，thyme 轉 JS）

1. `overview.html` line 34 後新增按鈕：`<button class="export-btn" id="exportJSONv2Btn">匯出 JSON (v2 相容)</button>`。
2. `overview-page-controller.js`：
   - CONSTANTS 新增 `EXPORT_V3 = { FORMAT_VERSION: '3.0.0', JSON_MIME: 'application/json' }`（或復用既有 MIME）。
   - 新增 `async handleExportJSONv3()`：`new BookDataExporter(books).exportToJSON({ formatVersion: CONSTANTS.EXPORT_V3.FORMAT_VERSION })` → `_triggerExportDownload(json, 'json', ...)`。
   - 按鈕綁定改：`exportJSONBtn` → `handleExportJSONv3()`；新增 `exportJSONv2Btn` → `handleExportJSONv2()`。
   - line 205 buttons 陣列加 `'exportJSONv2Btn'`（含 elements 快取）。
3. **關鍵考量（必先驗證）**：`_exportToJSONCanonical` 從 `this.books` 經 `mapV1BookToCanonical` 取 tag（everything-as-tags），不另傳 tags/tagCategories。須確認 `_getBooksForExport()` 回傳的 book 物件**已攜帶 tag 資料**；若 v3 canonical 需要 storage tag 補強（如 v2 的 `_loadTagData()`），handleExportJSONv3 須比照載入並確認 mapV1BookToCanonical 能正確產出 tagTree。以實際匯出 JSON 的 tagTree 非空（有 tag 的書）驗證。

### TDD 指引

  - `handleExportJSONv3()` 觸發後，下載內容 JSON.parse 後 `format === 'book-interchange-v1'` 且 `formatVersion === '3.0.0'` 且具 `tagTree`（斷言字面，非僅 spy 呼叫）。
  - `exportJSONBtn` click → 走 v3；`exportJSONv2Btn` click → 走 v2（`formatVersion === '2.0.0'`，acceptance 3）。
- Phase 3b GREEN：依上步驟實作。
