## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

`_build_decision_tree_path`（create.py:165）在非豁免且 decision-tree 全缺（`provided_count == 0`）時 `raise ValueError`，由 `_parse_cli_args_to_config` 的 `except ValueError: return None`（785-786）接住，使 `execute` 在 `if config is None: return 1`（1258）**提前退出**——早於 checklist 階段（1116 `_validate_create_checklist`）。

這與 WHY_REQUIRED 修復前同構（雙軌冗餘）：checklist 本身已含 `decision_tree_path` 完整性檢查（現於 `validate_create_checklist`），但 decision-tree 在 config 建構階段就先 fail，根本到不了 checklist。

後果：全欄位缺漏時用戶需多輪試錯（version → wave → decision-tree → checklist 各自獨立退出）。W1-024.1 A2 已對 why 做相同修復（796-799 註解），本 ticket 對 decision-tree 套用同手法。

### 影響範圍

- `.claude/skills/ticket/ticket_system/commands/create.py`（`_build_decision_tree_path` 的 `provided_count==0` 分支）
- `.claude/skills/ticket/ticket_system/tests/test_create_error_envelope.py`（missing_all 測試改寫 + 新增 only_missing case）

### 相關 Error Pattern

- source W1-024 對抗性複審挑戰 1：checklist 階段內已列全但跨階段分批仍在。
- 與 WHY_REQUIRED（W1-024.1 A2）、空字串漏判（W1-043）同家族：建票驗證的「提前退出 vs 匯總列全」一致性收斂。

### Wave 階段裁決（how.strategy 待評估項）

裁決：**wave 不併入 checklist**。Why：wave/version 是 CLI 結構性前置參數（決定 ticket ID 與目錄路徑），缺失時無法建構 ticket ID，本質與「內容欄位」（who/why/decision_tree_path）不同，必須前置攔截。runtime 實證 version 缺失報 `VERSION_NOT_REGISTERED` 早於 config 建構，印證此分層合理，無需併入。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復：全缺不提前退出，併入 checklist；PARTIAL/EMPTY 保留

`_build_decision_tree_path` 的 `provided_count == 0` 分支，移除非豁免的 print + raise，統一 `return None`：

```python
if provided_count == 0:
    # 豁免或非豁免皆 return None，不在此提前退出。非豁免全缺交給
    # validate_create_checklist 的 decision_tree_path 完整性檢查，與
    # when/who/how_strategy 等一次列全（W1-029，A2 同手法）。
    # PARTIAL（1-2 個）與 EMPTY_VALUE 仍即時精確報錯，不退化。
    return None
```

`config["decision_tree_path"] = None` 後，checklist 的完整性檢查（`dt = None or {}` → `has_complete_path` False）會將 `decision_tree_path` 列入缺失，與其他必填合併單一 `CHECKLIST_VALIDATION_FAILED`。

### 設計取捨

| 維度 | 決定 | 理由 |
|------|------|------|
| 全缺：提前退出 vs 併入 | 併入 checklist | 與 why（A2）一致，避免跨階段分批試錯（acceptance 1） |
| PARTIAL（1-2 參數） | 保留提前退出 + 參數級 hint | 「缺 final_decision, rationale」比「缺 decision_tree_path」精確，不退化（acceptance 2） |
| EMPTY_VALUE（空字串值） | 保留即時報錯 | 明確輸入錯誤，即時提示成本最低 |
| wave 階段 | 不併入（裁決見 Problem Analysis） | 結構性前置參數 vs 內容欄位的本質差異 |
