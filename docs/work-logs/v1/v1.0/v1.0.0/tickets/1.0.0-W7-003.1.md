## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

承接 W7-003 ANA 結論（TD-1 統一封裝否決；僅落地 TD-2 補 timeout + TD-3 修註解）。

| 檔案 | 變更 |
|------|------|
| `lib/git_utils.py` | `_run_git` 加 `timeout` 參數（預設 `_FAST_GIT_TIMEOUT=5s`）；新增 `_COMMIT_GIT_TIMEOUT=30s` 供 commit 步驟（含 pre-commit husky）；`_auto_commit_ticket_md` commit 呼叫改用 30s |
| `commands/lifecycle.py` | `_auto_stage_git_add` 加 `timeout=5`；逾時拋 TimeoutExpired，由呼叫端 `_auto_stage_completion_files` 既有 except 涵蓋（graceful degrade） |
| `commands/track_acceptance.py` | lazy import `from ticket_system.lib import git_utils` 補精確註解：非循環依賴規避（git_utils 僅 import subprocess+pathlib），真實理由為測試 patch 點 + graceful degrade 邊界 |

### 設計決策

- 快命令（rev-parse/add/diff）5s 對齊 track_snapshot._get_git_branch；commit 30s 容納 husky（P 防護：避免太短誤殺）。
- 維持四處薄封裝獨立（TD-1 否決），未做統一 runner，保留各模組 patch 隔離。
- 逾時走 TimeoutExpired → 呼叫端既有 try/except 捕獲，符合 graceful degrade（不中斷主流程）。

### 不在本 ticket 範圍

統一封裝（TD-1，W7-003 WRAP 否決）；track_snapshot / checkpoint_state 兩處已有 timeout，不動。
