## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因（PM 第一手觀察，2026-06-07 sync-pull session；根因同源性待 claim 後 WRAP 驗證 AC-4）

本 ANA 由實際 sync-pull 事件觸發，PM 親歷處理三議題（commit 47e54652），以下為第一手觀察素材：

**初步根因假設（待 WRAP 驗證）**：三議題可能同源於 `sync-claude-pull.py` 的「單向全量覆蓋」設計——將獨立 repo 內容完整覆蓋本地 `.claude/`，缺乏 (a) 衝突偵測；(b) 本專案演化方向感知（外移決策、結構重組、特有防護的識別）。

- **議題一（同步衝突解決）**：sync-pull 無 3-way merge。獨立 repo 與本專案分叉時（base SHA `e4d8d2ceb188`），衝突以「獨立 repo 全勝」靜默解決，本地演化被覆蓋而無提示。
- **議題二（多餘 hook）**：獨立 repo 採扁平 `.claude/hooks/` 結構，本專案已重組為 skill 自包含 `skills/{ticket,worktree,wrap-decision}/hooks/`。pull 將 27 個 hook 灌入 `.claude/hooks/` 頂層，與 skills/ 版本同名但內容不同，且 settings.json 未註冊 → 孤兒污染。根因：兩 repo hook 組織結構分叉，sync 無結構對齊機制。
- **議題三（pull 覆蓋進度）**：本專案演化內容被獨立 repo 舊版回灌/刪除——(a) 反方向回流：`project-specific-conventions.md`（已外移 docs/）、`project-integration/*.md`（W1-002 計畫外移）被拉回；(b) 特有防護被刪：PC-177、malformed-tool-call-detector-hook、context-depth-warning-hook（含 settings 取消註冊）。根因：sync 無法識別「本專案比獨立 repo 新」，盲目以舊覆新。

### 影響範圍

| 議題 | 確切影響（commit 47e54652 已處理） |
|------|------|
| 多餘 hook | 27 個孤兒 hook 灌入 `.claude/hooks/`（已移除） |
| 反方向回流 | `project-specific-conventions.md`（1）+ `project-integration/*.md`（8）（已移除） |
| 特有防護刪除 | PC-177 + 2 hook + 4 test + settings.json（已還原） |
| 衝突靜默 | 52 個 M 修改（已採納）、1 個 D（接受重組）需逐一人工判斷 |

範圍邊界：本次處理僅限 book_overview_v1；但 `sync-claude-pull.py` 為 `.claude/` 跨專案共用腳本，改善方案影響所有使用此腳本的專案。

## 重現實驗結果

### 實驗方法

執行 `sync-claude-pull.py` 從 tarrragon/claude.git 拉取，以 `git diff` 對照 HEAD `1223c40a` 分析變更（本專案 `.claude/` 受 git 追蹤，1825 檔，故 git 即為比對與還原基準）。

### 實驗執行

- 拉取：`python3 .claude/scripts/sync-claude-pull.py`（base SHA `e4d8d2ceb188`）
- 變更總量：104 檔（53 M + 45 untracked + 6 D）
- 分類查驗：`git status --porcelain` + 備份目錄 `/tmp/claude-backup-_1ib1ep4` 對照
- 重複偵測：`find .claude/skills -path "*/hooks/<name>"`
- 內容比對：`diff .claude/hooks/<name>` vs `skills/*/hooks/<name>`（抽樣 3 個全部內容不同）

### 實驗發現

**已驗證事實**：
1. 27 個 `.claude/hooks/` 頂層 hook 與 `skills/*/hooks/` 同名但內容不同，settings.json 未註冊（孤兒）
2. `project-specific-conventions.md` 備份中不存在（本專案已外移 docs/），含 9 個專案識別符
3. `project-integration/` pull 前僅 1 檔（yaml），pull 後 9 檔（8 個 .md 灌入）
4. PC-177 + 2 hook（malformed-tool-call-detector、context-depth-warning）被刪，settings.json 同步取消其註冊
5. 7+ 檔既有 broken link 指向 `project-integration/*.md`（pull 前即存在，非本次造成）

**待驗證假設（claim 後 WRAP）**：
- 三議題是否同源於「sync 單向全量覆蓋無方向感知」（AC-4）
- 改善方案的具體形態（候選：sync 前 git diff 預覽 + 確認 / 衝突偵測 hook / 結構對齊規則 / 方向感知排除白名單 / 改 pull 為雙向 reconcile）——僅列候選，定案需 WRAP 評估
