## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->

**變更摘要**：在 `malformed-tool-call-detector-hook.py` module docstring 現有 YAGNI 段後，新增「架構邊界」散文段（24 行）。內容三要素：

1. **孤兒結尾漏網路徑屬 Stop-hook 架構邊界**：harness 成功抽離 `<invoke>` 為 tool_use 後，該回合以 tool_use-only 訊息持久化，孤兒前綴併入或丟棄，`last_assistant_text()` 取到的是更早的乾淨 text 訊息——漏網路徑在持久化 transcript 層無 hook-visible 可掃文字，非 signature 覆蓋缺口。
2. **引用 W8-029 census 證據**：text+tooluse=0（text/tool_use block 不並存）、ends_short=0（無孤兒短 token 結尾的持久化 text）、invoke_literal_in_text=16（double-failure 命中型正常攔截）。
3. **重啟 trigger 記錄**：跨 session 累積 ≥2 反例（不同 session 實際取樣到「孤兒結尾 text + 後續 tool_use-only」兩段式持久化 entry）為唯一重啟 signature 設計的 trigger，在此之前新增孤兒 signature 屬攔不到目標且高誤報的負價值改動。

**未改動**：SIGNATURE_PATTERNS / detect() / strip_code_regions() / EXEMPT_MARKER 全數零改（git diff 確認 24 insertions / 0 deletions，僅 docstring）。
