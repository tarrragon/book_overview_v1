### 交付（實際檔案在 .claude/skills/framework-issue/，非原 where.files 的 sync-pull）
1. /framework-issue 新增 fix-status 命令：
   - `fix-status <issue-ref>`（view）：讀 issue body 的修復矩陣顯示哪些 consumer 修了。
   - `fix-status <issue-ref> --mark-fixed`（update）：標記「本 consumer」為已修，回寫 issue body。
2. 矩陣落點：issue body 內以標記區段管理（如 <!-- fix-matrix -->...<!-- /fix-matrix -->），內含 markdown 表格 `| consumer | status |`。read=gh issue view --json body 解析；write=更新該區段後 gh issue edit --body-file 回寫。
3. consumer 自我識別：沿用 .claude/error-patterns/_project-registry.yaml + git toplevel basename（同 error-pattern allocator 的 identify_project_code），勿要求手動傳 consumer 名。
4. 沿用 scripts/gh_common.py preflight 降級；gh 不可用清楚提示不 crash。
5. 測試：view 解析矩陣 / mark-fixed 更新本 consumer / 矩陣不存在時初始化 / 降級，至少 4 個（mock gh，勿真打 API）。
