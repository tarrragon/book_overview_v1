## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

在 `DEFAULT_VERSION_RELEASE_CONFIG` dict（L61-134）新增三個可選欄位：

1. `project_type`: None — 專案類型（chrome-ext/flutter/go/php/python/monorepo/npm），None 時由自動偵測判定
2. `version_source`: None — 版本源配置 dict（primary/parser/key/sync_targets），None 時依 `VERSION_FILE_CANDIDATES` 自動偵測
3. `subprojects`: None — monorepo 子專案配置，僅 `project_type: monorepo` 時使用

在 `load_version_release_config()` 的補值迴圈（L795-805）將三個新 key 加入缺漏欄位清單，確保 yaml 缺少這些欄位時由 DEFAULT 補值。

現有三欄位（release_workflow/tag_format/worklog_path_pattern）未修改，行為不變。
