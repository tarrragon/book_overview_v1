### 可行性評估結論：可行（< 30 行重用既有邏輯）→ 實作

評估 toolification-bias R3：--audit 唯讀稽核可重用既有 `collect_remote_files`（對任意目錄做同一 SKIP_DURING_SYNC/symlink 過濾）+ `clone_repo` + `should_exclude` + `load_preserve_list`，核心邏輯為全量差集，遠低於 30 行門檻。

### 實作（不動既有同步主流程）

1. `compute_orphan_candidates(claude_dir, upstream_dir, preserve)`（~10 行純函式）：
   - `local = collect_remote_files(claude_dir)`；`upstream = collect_remote_files(upstream_dir)`
   - 差集 `local - upstream`，再排除 preserve 與 should_exclude（LOCAL_ONLY/憑證）
   - 涵蓋早於 base 不在 delta 窗的孤兒（全量差集 vs delta 路徑只看 base..HEAD 窗）
2. `run_audit()`（~20 行）：clone 上游 → 計算孤兒候選 → stdout 列出（非阻擋措辭，鏡像 warn_upstream_deleted_residue）。純唯讀，不寫本地檔、不更新 base SHA。
3. `main()` 加 `--audit` argv 分支：見旗標即 run_audit() 後 return，不進同步主流程。

### 非阻擋措辭（W8-037 Premortem R1）

列出之檔可能是應清理孤兒，也可能刻意本地特化，腳本無法自動判別，僅提醒人工判斷，不自動刪、不阻擋、不視為失敗。

### 與缺口 1 區別

缺口 1（W8-037.1）：被動通報「上游 delta 刪除 + 本地分歧」（限 base..HEAD 窗）。
缺口 2（本 ticket）：主動全量稽核「本地有上游 HEAD 無」（不限窗），一鍵 `--audit` 觸發。
