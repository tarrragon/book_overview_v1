### 問題根因

W8-002 ANA 結論：auto-commit（W7-001）僅覆蓋 `track_acceptance.py:execute_append_log` 路徑。PM/agent 手動 Edit ticket md（body 補寫、AC 勾選）後 body 停留 working tree，最長暴露窗口至下次 commit；若中途觸發 git 還原（checkout/stash/sync-push）即遺失（PC-178）。auto-commit 製造「body 已自動保護」虛假安全感，降低手動 Edit 路徑的 commit 警覺（二階風險）。

### 影響範圍

- 新增 `.claude/hooks/uncommitted-ticket-md-reminder-hook.py`（PreToolUse Bash）
- 新增 `.claude/hooks/tests/test_uncommitted_ticket_md_reminder_hook.py`
- `.claude/settings.json` PreToolUse Bash matcher 註冊

### 相關 Error Pattern

PC-178（ticket body 未 commit 被 git 還原覆蓋）、IMP-048（logger.error 寫 stderr 觸發 UI hook error）、PC-086（subagent Write hook 缺 exec bit）

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要（採 W8-002 方案 4：狀態偵測，警告不阻擋）

新增 PreToolUse Bash hook，在 `git commit` 前偵測 working tree 內「已修改/暫存/新增但未 commit 的 ticket md」（`docs/work-logs/**/tickets/*.md`），輸出 stderr 警告並 exit 0（不阻擋）。

### 設計決策

| 決策 | 理由 |
|------|------|
| 觸發點 PreToolUse Bash + git commit 偵測 | 依 ticket strategy；commit 是提交前的最後可見決策點，提示「ticket md 是否一併納入」 |
| 涵蓋 modified + staged + untracked 三類 | 任一未進入 commit 的 ticket md 都屬遺失風險範圍 |
| 警告走 `print(..., file=sys.stderr)` 而非 logger.warning | IMP-048：logger.error/warning 寫 stderr 會被 CC runtime 當 hook 失敗觸發 UI hook error |
| exit 0 不阻擋 | W8-002 結論：殘留風險低（歷史 1 例 + 主路徑已防護），警告級防護即適當 |
| ImportError fail-open（no-op） | 不因 lib import 失敗阻斷流程，參考 branch-status-reminder 慣例 |
| 排除 git log/show/diff/status | 僅讀取命令不需提醒 |
| 路徑用 regex `docs/work-logs/.*/tickets/[^/]+\.md$` | 同時支援階層（v1/v1.0/v1.0.0/tickets/）與扁平（v0.29.0/tickets/）結構 |

### 防護落地

- chmod +x（避 PC-086 exec bit 缺失）
- settings.json PreToolUse Bash matcher 末位註冊
- 共用 `git_utils.get_uncommitted_files()` 高階 API（FileStatus），不自刻 porcelain 解析
