### 問題根因

兩個 SubagentStop hook（subagent-stop-dispatch-cleanup / agent-commit-verification）存在三層缺陷共振（parent 1.0.0-W1-055 ANA 已查明，confidence 0.95）：

1. **通道語意錯置（主因）**：0.19.1-W1-046 將輸出自 top-level `systemMessage`（純顯示）改為 `hookSpecificOutput.additionalContext`。W1-055 活體確證該欄位投遞對象是「停止中的 subagent」——注入其對話並令其繼續 → 再停止 → 事件再觸發（自激迴圈，H1）。
2. **缺斷路器**：CC hook 規格為 Stop/SubagentStop 提供 `stop_hook_active` 防迴圈欄位，兩 hook 均未檢查。
3. **缺去重**：每次事件全量重掃並無條件重播相同訊息——[WAIT] 廣播（remaining>0 即輸出）與 hook error 5 分鐘視窗摘要（W1-074/075/076 重播症狀）。

### 影響範圍

- `.claude/hooks/subagent-stop-dispatch-cleanup-hook.py`（[WAIT]/[OK] 廣播）
- `.claude/hooks/agent-commit-verification-hook.py`（commit 警告 + hook error 摘要）
- 症狀：同一 agent_id 事件 5-20 次重複、PM 主線程 11+ 次重複喚醒、每次喚醒 context 重載成本

### 相關 Error Pattern

- PC-148（雙重註冊）已被 parent ANA 排除（對 SubagentStop）；W1-040.1 孤兒 waiter 為同症狀不同觸發源
