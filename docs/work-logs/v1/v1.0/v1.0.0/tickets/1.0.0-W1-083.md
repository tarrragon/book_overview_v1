### 修改內容

三個寫入命令的 `check_identity` 呼叫端各補傳 `command` 具名參數，取代預設 `(unknown)`：

| 呼叫端 | 檔案 | 傳入值 |
|--------|------|--------|
| `_execute_complete` | `ticket_system/commands/track.py` | `complete` |
| `execute_check_acceptance` | `ticket_system/commands/track_acceptance.py` | `check-acceptance` |
| `execute_set_acceptance` | `ticket_system/commands/track_set_acceptance.py` | `set-acceptance` |

### 測試設計決策

呼叫端層級測試新增於 `tests/test_identity_guard.py`（parametrized 三命令）：以「身份不符 deny 早退」路徑觸發各呼叫端——deny 在 `check_identity` 內直接 return，不執行命令本體，無需鋪設完整 ticket 環境且零狀態副作用；斷言 usage.log 記錄的 command 欄位為真實命令名且非 `(unknown)`。

### 範圍確認

盤點全 repo `check_identity` 呼叫點僅上述三處（rg 驗證），與 how.strategy 一致；`identity_guard.py` 本體與 `(unknown)` 預設值不變（向後相容，未升級呼叫端仍可運作）。
