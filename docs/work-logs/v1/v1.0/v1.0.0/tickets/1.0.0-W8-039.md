## Solution

### 新定義邊界（核心結論）

舊定義（framework-meta / methodology-writing）把方法論定為「30 秒複習清單 / 電梯測試 / 不超過 1 頁 / 不超過 5 條要點」，把兩件本質不同的事混為一談：

- (a) 不把操作流程放方法論 —— 正確，**保留**（流程外移 skill，避免多檔同步）
- (b) 把判斷標準本身壓縮到 5 條要點 —— 對 AI 開發判斷有害，**移除**（壓縮會遮蔽判準細節導致誤判）

新定義：**方法論是 AI 開發時的核心判斷標準 / 框架運作核心規則，判準須明確完整可套用，篇幅以講清判準為度；可重複操作流程仍外移 skill。** 30 秒壓縮的 token 動機對方法論幾乎不成立（按需載入、不佔 45k 自動載入預算），故壓縮本屬過度套用；但「避免與 skill 重複」動機仍守住。
