### 實作摘要

新增 `--as <agent-name>` 選用旗標至三個寫入命令（complete / check-acceptance / set-acceptance），與 ticket frontmatter `who.current` 精確比對。

**新增模組**：`.claude/skills/ticket/ticket_system/lib/identity_guard.py`
- `check_identity(version, ticket_id, as_value)`：依規格表四情境回傳 deny exit code 或 None（放行）。
- `_resolve_who_current()`：讀取 frontmatter who.current（空值 / 缺欄位 / ticket 不存在皆視為 None）。
- 常數 `PM_AGENT_NAME = "rosemary-project-manager"`、`IDENTITY_DENY_EXIT = 1`。
- 所有提示走 stderr，不污染 stdout 消費者。

**判定邏輯**（依序，對應 Problem Analysis 規格表）：
1. 未提供 --as → stderr 警告一行，放行（向後相容）。
2. --as = rosemary-project-manager → 一律放行（PM bookkeeping 豁免，先於 who.current 解析）。
3. --as = who.current → 放行。
4. --as != who.current（含 who.current 空值 / 缺欄位 / ticket 不存在）→ deny（exit 1）+ stderr 引導回報 PM。

**接線位置**：
- `commands/track.py`：三 subparser 各加 `--as`（dest=as_agent）；`_execute_complete` 開頭呼叫 check_identity。
- `commands/track_acceptance.py::execute_check_acceptance` 開頭。
- `commands/track_set_acceptance.py::execute_set_acceptance` 開頭。

**設計約束遵循**：
- deny 為純前置檢查，short-circuit return，不寫入任何 ticket 狀態（CLI 實測 deny 後 acceptance 未變動）。
- frontmatter who.current 字串精確比對，無模糊匹配。
- 警告與 deny 訊息均走 stderr。
