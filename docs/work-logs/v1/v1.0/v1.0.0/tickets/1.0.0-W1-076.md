## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要（PC-V1-004 防護 C 方案 A：規則層補 Stop event 盲區）

**1. 前綴常數單一來源（hook_utils/hook_io.py）**

- 新增模組常數 `PM_ONLY_PREFIX = "[PM-ONLY] "`（含三明示 docstring），經 `hook_utils/__init__.py` re-export
- `emit_hook_output(audience="pm_only")` 在統一出口實作前綴：主線程觸發且訊息非空時自動前置（接上 W1-075 預留掛點，移除「本票不實作」過時註記）；subagent 觸發仍整則丟棄（既有過濾語意不變）；audience="all" 與預設呼叫不加前綴（向後相容）

**2. Stop 類 hook 前綴落地（settings.json Stop event 註冊項逐一盤點）**

| Stop hook | 注入通道 | 處置 |
|-----------|---------|------|
| stop-worklog-handoff-sync-check | top-level systemMessage | import 常數，warning 前置前綴 |
| handoff-auto-resume | decision:block 的 reason（單任務 + 多任務兩路徑） | import 常數，reason 前置前綴 |
| session-experience-persistence-reminder | stderr + exit 0 | 不注入模型 context（exit 0 stderr 僅 debug 可見），無需前綴 |
| evaluate-session | stderr + exit 0 | 同上，無需前綴 |
| context-depth-warning | stderr + exit 0（hook 明文「永遠 exit 0」） | 同上，無需前綴 |
| malformed-tool-call-detector | stderr + exit 2 | 注入受眾是「寫壞工具呼叫的當事 agent」（subagent 也適用修正指示），屬雙方可見訊息，不加前綴 |
| worktree-auto-commit | 無 stdout/stderr 注入 | 無需前綴 |

兩個加前綴的 hook 輸出 shape（systemMessage / decision-reason）非 emit_hook_output 可覆蓋（Stop event schema 不允許 hookSpecificOutput.additionalContext），故 import 常數自行前置；未複製字串字面（單一來源由回歸測試強制）。

**3. AGENT_PRELOAD.md 規則 10（v1.12.0）**

- 新增「忽略含 [PM-ONLY] 前綴的 hook 注入訊息」規則：三明示結構（Why：Stop event 無 agent_id 程式層不可偵測 / Consequence：指令性注入誘導越界，PC-V1-004 實證 / Action：四列處置表——不執行、不轉述、不中斷任務、唯一合法反應是無動作）
- 附判別準則（只比對訊息開頭字面；無前綴注入照常處理）；執行檢查清單同步補項
- 框架資產約束遵守：未引用專案 ticket ID，以 PC-V1-004 與功能性描述錨定

### 設計決策

- 與 W1-044 既有 subagent context 偵測 + suppressOutput 互補（雙層防護），未移除既有防護
- handoff-auto-resume 的 block reason 雖不在 ticket 列舉的 systemMessage / additionalContext 字面內，但屬 Stop event 對模型的指令性注入（恢復指令為 PM 專屬動作，正是 hook 內 W1-024 / W1-030 記載的劫持向量），納入前綴範圍符合本票意圖
- 遵循率實測不在本票範圍（觀測屬後續）
