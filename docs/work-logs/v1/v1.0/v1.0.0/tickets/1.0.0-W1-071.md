### 問題根因

4 個 PostToolUse:Bash hook（commit-msg-layer2-marker-check / post-test / needs-context-listener / ticket-skill-sync-check）缺 `is_subagent_environment()` 偵測，其 additionalContext 含 PM-only 動作性提示（開 Ticket、派發代理人、git merge、掃描 pm-rules 等），在 subagent 觸發時注入 subagent context，誘導唯讀委員越界（W1-060 實證 Explore 執行 git merge）。

### 影響範圍

- `.claude/hooks/commit-msg-layer2-marker-check-hook.py`（additionalContext 原 L232）
- `.claude/hooks/post-test-hook.py`（additionalContext 原 L477）
- `.claude/skills/ticket/hooks/needs-context-listener-hook.py`（additionalContext 原 L97）
- `.claude/skills/ticket/hooks/ticket-skill-sync-check-hook.py`（additionalContext 原 L281）

**路徑勘誤（PC-162 接手者驗證）**：ticket `where.files` 記載後兩個 hook 位於 `.claude/hooks/`，實際位於 `.claude/skills/ticket/hooks/`（W10-092 已遷移）。W1-060 重現實驗的行號盤點正確，僅路徑前綴 stale；已依實際路徑實作。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復內容

各 hook 在輸入解析後、業務邏輯前加入 `is_subagent_environment(input_data)`（`hook_utils/hook_io.py`，以 `agent_id` 欄位判定）早期跳過，並保持各 hook 既有的跳過輸出慣例：

| Hook | 插入位置 | 跳過輸出慣例 |
|------|---------|------------|
| commit-msg-layer2-marker-check | `tool_name != Bash` 檢查後 | 無輸出（return EXIT_SUCCESS） |
| post-test | `read_json_from_stdin` 後、`_is_test_command` 前 | 無輸出（return EXIT_SUCCESS） |
| needs-context-listener | `read_json_from_stdin` 後 | 無輸出（return EXIT_SUCCESS） |
| ticket-skill-sync-check | `tool_name != Bash` 檢查後 | 輸出 DEFAULT_OUTPUT JSON（沿用既有慣例） |

實作參照已修復 hook（commit-handoff-hook）的既有寫法：偵測 → log 記錄 agent_id → 早期 return。

### 回歸測試

新增 `.claude/hooks/tests/test_subagent_early_skip_post_tool_use_hooks.py`（8 tests）：每 hook 兩組情境——(a) subagent 環境（payload 含 agent_id）零注入且業務子邏輯不被呼叫（MagicMock sentinel 驗證早期跳過）；(b) PM 環境（無 agent_id）觸發條件成立時照常輸出 additionalContext。

### 自檢結果

- [x] 僅修改 4 個目標 hook + 新增 1 個測試檔（無越界）
- [x] 各 hook 跳過輸出慣例與既有 skip 分支一致（layer2/post-test/needs-context 無輸出；sync-check 印 DEFAULT_OUTPUT）
- [x] `is_subagent_environment` 來自 hook_utils 統一 helper，無平行實作（ARCH-020）
- [x] 跳過時 log 記錄 agent_id（quality-baseline 規則 4 可觀測性）
- [x] 註解標明業務情境（W1-071 / PC-V1-004 入口污染防護）而非語法描述
