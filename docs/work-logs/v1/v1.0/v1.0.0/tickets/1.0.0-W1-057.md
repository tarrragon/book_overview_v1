## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要

在 `identity_guard.py` 新增 telemetry 落盤管線，warn 與 deny 兩路徑各 append 一行結構化 JSON 記錄。

### 設計決策

- **落盤位置**：`.claude/hook-logs/identity-guard/usage.log`。基底目錄沿用既有 `HOOK_LOGS_DIR` env 慣例（與 force-usage log 相同，便於測試隔離），子目錄 `identity-guard` 區隔此管線。
- **記錄欄位**：`timestamp` / `command` / `ticket_id` / `has_as`（--as 有無）/ `result`（warn|deny）。僅記 `--as` 有無不記原文，避免將 agent 名稱寫入長期觀測檔。
- **command 參數**：`check_identity` 新增選填參數 `command`（預設 `(unknown)`），向後相容既有三個呼叫端（track.py / track_acceptance.py / track_set_acceptance.py 未傳則記 `(unknown)`）。範圍限制使本 ticket 不改呼叫端，後續可漸進 opt-in。
- **失敗降級**：`_write_telemetry` 以 try/except OSError 包裹寫入；落盤失敗時不阻斷 CLI 主流程（return 值不變），但寫 stderr 雙通道可見（observability 規則 4 擴充）。
- **放行路徑不落盤**：身份相符（情境 2/3）不記錄，telemetry 僅觀測 warn/deny 兩過渡期裁決點，符合 W1-049 統計目的。

### 變更檔案

- `.claude/skills/ticket/ticket_system/lib/identity_guard.py`：新增 `_resolve_identity_log_path` / `_write_telemetry` / `RESULT_WARN` / `RESULT_DENY` 常數；`check_identity` 加 command 參數並於 warn/deny 路徑呼叫 telemetry。
- `.claude/skills/ticket/tests/test_identity_guard.py`：新增 module-level autouse `_isolate_identity_log` fixture（隔離 HOOK_LOGS_DIR，因 tests/ 樹無 hook-logs autouse）+ 6 個 telemetry 測試。

### 不在本 ticket 範圍

- 呼叫端 opt-in 傳遞具體 command 名稱（後續可漸進補強）。
- warn-only 轉強制裁決（W1-049 本體，需本管線累積資料後評估）。
