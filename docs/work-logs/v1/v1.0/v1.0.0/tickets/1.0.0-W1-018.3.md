## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

遠端 repo 曾累積本地 agent worktree 產物（`.claude/worktrees/agent-*`）的痕跡，
ANA（W1-018）觀測為 6 個 git gitlink（160000 型 submodule-like 條目）垃圾。
根因有二：
1. `sync_exclude_manifest.py` 的 `LOCAL_ONLY_PATTERNS` / `LOCAL_ONLY_ROOT_DIRS`
   未含 `worktrees`，sync push 不會排除該目錄。
2. `clean_stale_files` 只處理 `is_file`，不清 gitlink（160000），故即使排除清單
   補上，既存遠端 gitlink 仍需一次性手動 `git rm`。

### 影響範圍

- `.claude/hooks/lib/sync_exclude_manifest.py`（排除清單 SSOT）
- `.gitignore`（既已含 `.claude/worktrees/`，無需改動）
- `.claude/hooks/tests/test_sync_exclude_manifest.py`（regression 防護）

### 遠端現況獨立驗證（PC-162 重新驗證）

對 `https://github.com/tarrragon/book_overview_v1.git` 做 fresh full clone
（4386 commits、全 17 個 branch）後掃描：
- `git ls-tree -r origin/main | grep ^160000` → 0 筆
- 全 history `git log --all --raw` grep 160000 → 0 筆
- 全 ref `git log --all -- .claude/worktrees` → 0 commit

結論：**ANA 宣稱的 6 個遠端 gitlink 未在當前遠端任何 ref 復現**。可能為
(a) 已被清除、(b) ANA 觀測的是本地/stale origin、(c) 計數來源不同。
本地 HEAD（b0a14429）亦 0 gitlink；本地 `.claude/worktrees/agent-*` 為無 `.git`
的孤兒目錄快照（非 active git worktree），且 `.gitignore` 已忽略，git status 乾淨。

### 相關 Error Pattern

W1-018.2 既有 root-anchored 機制（logs / state，防 skill 內部同名目錄誤殺）。
本 ticket 沿用同策略。

   - 採 **root-anchored**（非 part-level）：collision 分析確認 `.claude/` 下

     root-anchored 語意更正確（`.claude/worktrees/` 必為根層 runtime 產物）且對
     未來新增的巢狀 `worktrees/` 目錄天然免疫，與 logs / state 同策略。
   - 自動納入 `GITIGNORE_EXPECTED`（= LOCAL_ONLY_PATTERNS | LOCAL_ONLY_ROOT_DIRS）。

### acceptance 2/3：遠端 gitlink 清除設計（執行整合至 W1-009）

**現況**：fresh full clone 驗證遠端 0 gitlink、0 worktrees 路徑（見 Problem Analysis）。
若 W1-009 執行時遠端確實出現 worktrees gitlink，清除步驟設計如下（唯讀驗證已備，
本 ticket 禁寫遠端）：

清除步驟（於主 repo，PM / W1-009 執行）：
```
# 1. 確認遠端 gitlink 清單
git fetch origin
git ls-tree -r origin/main | grep '^160000'
# 2. 若本地 index 含 gitlink，移除（cached，不刪本地目錄）
git rm -r --cached .claude/worktrees
# 3. 確認 .gitignore + manifest 已排除（本 ticket 已完成）
# 4. commit 移除 + push（gitlink 從遠端 tree 消失）
git commit -m "chore: 清除遠端 .claude/worktrees gitlink 垃圾"
git push origin <branch>
# 5. 驗證：fresh clone 後 git ls-tree -r HEAD | grep ^160000 應 0 筆
```

注意：`clean_stale_files` 只處理 `is_file` 不清 gitlink（160000），故無法靠
sync 機制自動清除既存遠端 gitlink，必須上述手動 `git rm -r --cached`。排除清單
（本 ticket）只防「未來再推」，不清「既存遠端」。


### PM 獨立驗證修正（推翻 agent「0 gitlink」結論）
agent Exit Status 宣稱「遠端 0 gitlink、0 worktrees 路徑」。PM 獨立 fresh clone（git clone --depth=1 + git -C ls-files --stage，git -C 防 chpwd）實測推翻：

| 項目 | agent 結論 | PM 實測 ground truth | ANA 原估 |
|------|-----------|--------------------|---------|
| worktrees gitlink（160000） | 0 | 24 | 6 |
| worktrees/ tracked 總路徑 | 0 | 30（24 gitlink + 6 巢狀實檔） | 6 |

6 巢狀實檔為 agent worktree 產品程式碼誤推至框架 sync repo（如 worktrees/agent-a0455599/src/ui/search/tag-resolver.js），污染比 ANA 預估嚴重。

**agent 失誤根因（已確認，非視角問題）**：agent clone 錯 repo。其 Problem Analysis（行 82）/ Test Results（行 173）明載 clone `github.com/tarrragon/book_overview_v1.git`（專案 repo，4386 commits / 17 branches），但 worktrees gitlink 實際在 `github.com/tarrragon/claude.git`（框架 sync repo，本 ticket Context Bundle 已明示此 URL）。clone 專案 repo 查當然 0 gitlink。教訓：派發遠端驗證任務時 prompt 須明示「遠端 = 框架 sync repo（claude.git），非專案 repo」，agent 易把「遠端」預設為專案 repo。PM 用正確 URL（claude.git）+ git -C ls-files --stage 實測得 24 gitlink。

**對本 ticket 程式碼影響**：worktrees 入 LOCAL_ONLY_ROOT_DIRS（root-anchored）防復發正確且涵蓋巢狀檔（should_exclude 命中整個 worktrees/ 子樹），不受 agent 驗證失誤影響；測試 2646 passed。

**對 W1-009 影響**：遠端清理 gitlink 部分從 6 更新為 30 worktrees 路徑（24 gitlink + 6 巢狀檔），W1-009 acceptance 同步更新。
