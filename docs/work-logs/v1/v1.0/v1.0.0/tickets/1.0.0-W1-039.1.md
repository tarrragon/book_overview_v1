### 問題根因

PC-166 現行內容預設「confabulation = 執行從未發生」（純捏造工具結果），漏掉 W1-039 鑑識證實的正交維度——執行**真實發生過**但本執行體無記錄（ghost branch）。兩者在「我記得做過但 git status 找不到」當下症狀相同，處置卻相反（純幻覺停手、ghost 找回），現行 PC-166 無此鑑識判據。

### 變更摘要（PC-166 三處新增，整合既有架構不另立平行標籤）

| # | 位置 | 變更 |
|---|------|------|
| 1 | 症狀「幻覺分兩類」表後 | 新增「第二維度：執行是否真實發生」——純 confabulation（無副作用）vs ghost branch（有真實副作用、本執行體無記錄）對照表 + 處置相反說明 |
| 2 | 防護 D 表後 | 新增「防護 D 延伸：同 prompt 異執行體（ghost branch）」——雙向鑑識判據表（ghost agent 痕跡 / 無主 commit / birth time）+ 收斂表（有痕跡→真實副作用對帳；無痕跡→不可判回防護 A-E）+ 宿主機制未定誠實邊界 |
| 3 | 相關 | 新增 W1-039 source case 行，註明四視角審查修正（M1/M3 二分降為有痕跡 vs 不可判、宿主機制降為多候選未排除）|
