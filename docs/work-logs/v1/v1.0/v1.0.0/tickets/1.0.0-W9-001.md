## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復實作（framework issue #1 問題0，P0）

### 變更檔案
- ticket_system/lib/file_lock.py：移除 import fcntl 與 _HAS_FCNTL 平台分支，改 from filelock import FileLock, Timeout。
  - file_lock：FileLock(lock_path) blocking acquire/release（語義等同原 fcntl.LOCK_EX）。
  - reap_stale_locks：FileLock(timeout=0) non-blocking 偵測 active；搶到=stale→計數（綁搶鎖成功事件）+best-effort unlink；移除 _HAS_FCNTL gate。
  - create_id_allocation_lock：改 FileLock blocking，移除「無 fcntl 平台降級」分支，保留 OSError graceful degrade（不阻斷單 process create）。
- pyproject.toml：dependencies 加入 filelock>=3.12。
- uv.lock：uv lock 同步，Added filelock v3.19.1/v3.29.4。
- ticket_system/tests/test_create_id_allocation_race.py：移除 test_no_fcntl_platform_degrades_with_warning（前提失效）+ unused file_lock_mod import。OSError 降級仍由 test_lock_open_failure_degrades_with_warning 覆蓋。

### 語義保全關鍵
filelock POSIX 後端用 fcntl.flock（per-open-file-description），與原實作一致：兩 FileLock 實例=兩 fd，第二個 LOCK_NB 仍被同 process 第一鎖拒絕，故 reap 同 process active-lock 偵測（test #3/#4）行為不變。

- 鎖測試 13 passed（reap 7 + precondition 2 + create-race 4）。
- 完整 ticket 套件 2470 passed, 1 deselected（perf）。

### 公開 API（不變）
