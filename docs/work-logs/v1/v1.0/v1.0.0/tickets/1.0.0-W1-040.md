## 重現實驗結果

### 實驗方法

1. **現況盤點**（acceptance 1）：rg 掃描 `ticket_system/commands/create.py`、`bulk_create.py`、`constants.py` 的重複偵測實作與常數。
2. **量化 reality test**（acceptance 2 誤報率依據）：sys.path 載入 `_calculate_jaccard_similarity`，以五組真實場景標題（title+what 合併文字）量測相似度，對照現行閾值 0.3：TP 兩組（ghost 同語意重複：逐字相同 / 近似改寫）、FP 三組（dashboard 真實兄弟票 W1-034 vs W1-035、W1-036 vs W1-037、batch-create 同質模板任務）。

### 實驗執行

**現況盤點結果**：

| 元件 | 現況 |
|------|------|
| `create.py` `_detect_duplicate_tickets()`（:371） | 已有語意偵測：Jaccard（中文逐字 + 英文單詞 tokenize）比對 title+what，掃同版本 pending/in_progress/completed（7 天內），排除自身與 parent，閾值 `DUPLICATE_DETECTION_THRESHOLD = 0.3`（constants.py:356），達標輸出 `[WARNING] 發現 N 個可能重複的 Ticket`——**僅警告不阻擋**（容錯式設計，內部異常靜默不影響建立） |
| `bulk_create.py` | **零偵測**（rg `duplicate` 無命中） |
| ID collision | 獨立分層：max+1 序號配置 + 檔案系統掃描 + git main ref 掃描（ticket_builder.py，PC-152 鏈） |

**量化 reality test**（閾值 0.3 對照）：

| 場景 | 相似度 | 0.3 警告層 | 性質 |
|------|--------|-----------|------|
| TP 逐字相同（ghost 重複 spawn） | 1.000 | 觸發 | 正確 |
| TP 近似改寫（同語意異字面） | 0.688 | 觸發 | 正確 |
| FP 兄弟票 1（W1-034 vs W1-035 真實標題） | 0.031 | 不觸發 | 正確 |
| FP 兄弟票 2（W1-036 vs W1-037 真實標題） | 0.069 | 不觸發 | 正確 |
| FP batch 同質模板（實作 X Widget vs 實作 Y Page） | 0.400 | 觸發 | **誤報**（但僅警告，成本低） |

### 實驗發現

**已驗證的事實**：
1. 語意偵測已存在且對 ghost 威脅模型的 TP 全數命中（0.688-1.0 >> 0.3）；真實兄弟票零誤報（0.03-0.07）。
2. 缺口不在「偵測」在「強制力」：警告寫 stdout 依賴執行體自律消費——而 ghost 雙執行流場景正是自律不可依賴的場景（兩執行體各自autonomous，W1-024 事件中 ghost 的寫入全數成功落地）。
3. bulk_create 完全無偵測，為旁路缺口。
4. 閾值間隔可支撐兩層設計：TP 下界 0.688 與最高 FP 0.400 之間存在 0.28 的分隔帶，阻擋閾值取 0.6 可分離（TP 阻擋、batch FP 不阻擋）。

**仍未驗證的假設**：0.688 為合成樣本下界，真實 ghost 重複 spawn 的字面差異分佈未知（事發未留下本線程的would-be 標題）；阻擋閾值定 0.6 屬工程判斷，須在 IMP Phase 2 測試以更多樣本驗證並允許 constants 調整。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 評估結論：兩層防護設計（警告層保留 + 阻擋層新增）

| 層 | 條件 | 行為 | 設計依據 |
|----|------|------|---------|
| Tier 1 警告層（既有，保留） | 同版本 pending/in_progress/completed(7d)，相似度 >= 0.3 | stdout WARNING，不阻擋 | 現行實作已正確，TP 全中、兄弟票零誤報 |
| Tier 2 阻擋層（新增，W1-040.1） | 同版本 pending/in_progress + 相似度 >= 0.6 + **候選建立時間在短窗口內**（建議 60 分鐘，constants 可調） | exit 1 阻擋，`--allow-duplicate` 旁路 | 雙條件交集鎖定 ghost 簽名（雙執行流在同 turn 數分鐘內重複 spawn）；時間窗條件排除「長存 pending 與新票相似」的合法情境 |

**誤報率考量**（acceptance 2）：阻擋層誤報需同時滿足「高相似 + 短窗口」——實測兄弟票相似度 0.03-0.07（同分鐘連續建立也不會擋）、batch 同質 0.400 < 0.6 不擋；剩餘誤報情境為「短時間內刻意重建近似票」（如 create 失誤後重建），此時阻擋語意上正確（應 migrate 或沿用），且有 `--allow-duplicate` 明示旁路。bulk_create 補警告層即可（批次內部同質性高，阻擋層誤報風險大，不適用 Tier 2）。
