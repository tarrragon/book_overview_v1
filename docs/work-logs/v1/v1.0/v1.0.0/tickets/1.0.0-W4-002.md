### 問題根因（PM 前台確認，2026-06-07）

### 影響範圍（純測試新增，無 src 變更）

- 新增/擴充 `tests/integration/cross-device-round-trip.test.js`（v3 round-trip group）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### v3 round-trip building blocks（PM 已確認全部存在）

| 環節 | 元件 | 位置 |
|------|------|------|
| export v3 | `BookDataExporter.exportToJSON({formatVersion:'3.0.0'})` | src/export/book-data-exporter.js（W4-001 已驗證 CE-runtime 產出 book-interchange-v1/3.0.0/tagTree） |
| 偵測 canonical | `detectInterchangeSource(data)` → `'canonical'`（data.format==='book-interchange-v1'） | src/export/format-version-detector.js:48 |
| import 路由 | `ContentParser.parse(json,'json')` 內部 detectInterchangeSource='canonical' → mapCanonicalToV1Book 逐本 + dedup | src/overview/import/content-parser.js:415 |
| canonical→內部 v2 | `mapCanonicalToV1Book(canonical)` + `rebuildFromPassthrough` | src/export/book-interchange-v1-adapter.js:234 / :184 |

### 實作策略（語言無關，thyme 轉 JS）

於既有 cross-device-round-trip.test.js 新增 v3 group（鏡像既有 Group A/B/C 結構與 helper 風格）：

1. 新增 helper `exportToV3Json(books)`：`new BookDataExporter(books).exportToJSON({ formatVersion: '3.0.0', pretty: false })`（v3 不需 tags/tagCategories，everything-as-tags）。
2. round-trip 鏈：`exportToV3Json` → 既有 `simulateCloudTransport`（純字串 parse/stringify）→ 既有 `parseImportContent`（ContentParser，已自動路由 canonical）。
3. 斷言無損（acceptance 2）：
   - 還原後 books 數量 = 原始；逐本 id/title/progress/readingStatus/tagIds 對齊。
   - everything-as-tags：原 book 的 authors/source/tagIds 經 canonical 往返後正確還原（用 mapCanonicalToV1Book 的 converge 邏輯）。
   - `_passthrough` 欄位保留（rebuildFromPassthrough）：放入含非標準欄位的書，往返後該欄位不丟失。
4. fixture：可復用既有 device fixture 或自建含 tag/_passthrough 的小樣本。
