## Context Bundle

### 實作錨點（PM 已 grounding，避免重新探索）

**檔案**：`.claude/scripts/sync-claude-pull.py`（1903 行）

**缺口位置**：`three_way_merge_file`（L534-600）的 L573-577：
```python
# upstream 刪除此檔
if upstream_content is None:
    if local_content is None or local_content == base_content:
        return None, False        # 自動跟刪（OK）
    return local_content, False    # 上游刪+本地改 → 保留本地，conflict=False（靜默殘留！缺口）
```
此 `return local_content, False` 路徑保留本地但 conflict=False → 不進 .sync-conflicts、pull 不通報。

**呼叫端**：`apply_upstream_delta`（L1324）逐檔呼叫 three_way_merge_file（L1413）。需在此收集「upstream 刪除（status=='D' 或 upstream_path is None）且 merged 非 None（保留本地）」的案例。判別：status 為 D（delta 中刪除）+ three_way_merge_file 回 (非None, False) + 非 local_deleted。

**鏡像模式（直接參考）**：W1-084 既有殘留通報——`detect_conflict_residue`（L1049）+ `warn_conflict_residue`（L1084），main 在 L1891 呼叫。仿此在 pull 結尾印出「上游已刪但本地分歧而保留」清單。

**措辭約束（R1，非阻擋）**：「以下檔案上游已刪除，但本地版本已修改而保留——若為應清理的孤兒請手動移除，若為刻意客製可忽略」。不自動刪、不阻擋、不進 .sync-conflicts。

**reproduction test**：可直接呼叫 three_way_merge_file（importlib 載入 module，見 W8-037 重現實驗結果的 airtight 重現法）：
- 上游刪+本地改（local != base）→ 應被收集進通報清單
- 上游刪+本地未改（local == base）→ 自動跟刪，不在清單
測試位置依既有 sync 測試慣例（先 grep 確認 .claude/scripts/ 或對應 tests/ 是否已有 sync-claude-pull 測試檔）。

**驗證**：`npm run test:hooks` 或對應 pytest 全綠；不破壞既有 sync 測試。
