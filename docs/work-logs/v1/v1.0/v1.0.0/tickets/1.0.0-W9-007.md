## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W9-007 原描述將 5 個失敗統一歸因「gitignore 規則 7→8」，實際為兩組不同根因 + 一個 source bug：

**A 組 — gitignore 必要規則清單同步（4 個）**
GITIGNORE_REQUIRED_RULES 已增至 8 條（新增 .claude/dispatch-active 與 __pycache__），但測試斷言/fixture 落後：
- test_gitignore_empty_file: 動態計算空 gitignore 為 8 條（coverage/htmlcov 合計 2 條 + 其餘 6 條），斷言仍寫 == 7
- test_gitignore_all_rules_present / test_onboard_flutter_project_complete / test_onboard_with_all_files_present: fixture 缺 .claude/dispatch-active，致 all_required_complete=False

**B 組 — check 命令項目數同步（1 個，與 gitignore 無關）**
check 的 _collect_all_sections 已從 6 項增至 8 項（新增 _check_codebase_memory_mcp、_check_codegraph），但 test_check_with_all_ok 僅 mock 6 個偵測函式、斷言 "6/6"，實際輸出 summary='7/8 項目正常'（未 mock 的 mcp 偵測在測試環境 1 項 fail）。

**額外發現 — source 路徑不一致 bug（本票一併修，用戶確認含 source 修正）**
_create_missing_gitignore_result（not_exists / encoding_error 路徑）的 missing_rules 硬編碼 7 條、漏 .claude/dispatch-active，且 dataclass 漏列 has_dispatch_active_rule；與 check_gitignore_completeness 動態計算（8 條）對「全部規則缺失」回傳不一致。此不一致剛好讓 test_gitignore_not_exists（斷言 ==7）通過、test_gitignore_empty_file（動態 8）失敗，固化了矛盾斷言。

### 影響範圍
- source: .claude/skills/project-init/project_init/lib/onboard_checker.py（_create_missing_gitignore_result）
- test: test_onboard_checker.py / test_onboard.py / test_integration.py

### 相關 Error Pattern
ticket handoff 描述與環境漂移：W9-007（W9-002 跑套件時順手建）把 3 組獨立根因誤統一為單一 gitignore 根因。接手者依描述會漏掉 B 組（check 項目數）真因。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修法

**source（onboard_checker.py）**
_create_missing_gitignore_result 補 has_dispatch_active_rule=False 欄位 + missing_rules 加 .claude/dispatch-active.json，與 check_gitignore_completeness 動態計算路徑統一為 8 條，消除 not_exists/empty 兩路徑對「全缺」的不一致。

**A 組測試（gitignore 同步）**
- test_onboard_checker.test_gitignore_empty_file: missing 斷言 7→8
- test_onboard_checker.test_gitignore_not_exists: 斷言 7→8（source 修好後 not_exists 路徑也回 8）
- test_onboard_checker.test_gitignore_all_rules_present: fixture 補 .claude/dispatch-active
- test_onboard.test_onboard_flutter_project_complete / test_onboard_with_all_files_present: fixture 補 .claude/dispatch-active
- fixture 字面用 .claude/dispatch-active（無 .json）對齊 _has_gitignore_rule 精確比對 pattern；precise_matching 測試（不斷言 all_ok）刻意未動

**B 組測試（check 項目數）**
- test_integration.test_check_with_all_ok: 補 detect_codebase_memory_mcp / detect_codegraph 兩 mock（is_available=True 即判定 OK），斷言 6/6→8/8；8 個 mock 改 parenthesized context manager 避免巢狀過深

### 驗證
project-init 完整套件：5 failed → 149 passed（commit 14e159f3）

### Spawn 落地確認
修測試時發現 dispatch-active 三方字面不一致（pattern 無後綴 vs 實際檔名 .json vs missing 建議 .json，_has_gitignore_rule 精確比對致照 missing 建議加 .json 永遠判缺失）。此為獨立 source bug，非本票測試修復範圍：
