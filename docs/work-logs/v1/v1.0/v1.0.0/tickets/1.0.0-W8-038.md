## Context Bundle

### 目標檔與插入點

- 檔案：.claude/commands/sync-pull.md（177 行）
- 插入點：在「## pull 後檢查清單（強制）」節（L90 起，子節 1-5 至 ~L157）之後、「## 新專案初始化」（L158）之前，新增 `## 孤兒稽核（--audit）` 節
- 既有腳本呼叫慣例（對齊用）：`python3 ./.claude/scripts/sync-claude-pull.py`，故 --audit 為 `python3 ./.claude/scripts/sync-claude-pull.py --audit`

### --audit 功能事實（來源 W8-037.3 實作 + sync-claude-pull.py run_audit）

- 唯讀全量稽核：clone 上游 → 計算「本地 .claude/ 有、上游 HEAD 無」之全量差集（排除 preserve / LOCAL_ONLY / 憑證）→ stdout 列孤兒候選
- 純唯讀：不寫本地檔、不更新 base SHA、不進同步主流程（見旗標即 run_audit() 後 return）
- 非阻擋措辭：列出檔可能是應清理孤兒，也可能刻意本地特化，腳本不自動刪、不阻擋、不視為失敗
- 無孤兒時印「無孤兒候選（本地 .claude/ 皆存在於上游 HEAD）」
- 與缺口 1（W8-037.1 被動殘留通報）區別：缺口 1 限 base..HEAD delta 窗；--audit 主動全量稽核（不限窗），補早於 base 不在 delta 窗的孤兒盲區

### 內容要求（三明示，document-writing-style）

- Why：久未 pull 或長期本地演化會累積「上游已無、本地殘留」的孤兒；delta 窗通報只看 base..HEAD，不限窗的舊孤兒會漏掉
- Consequence：孤兒累積膨脹 .claude/ + 狀態模糊（刻意特化 vs 該清殘留難辨），且可能經 sync-push 傳播至其他專案
- Action：跑 `python3 ./.claude/scripts/sync-claude-pull.py --audit`，純唯讀列「本地有上游無」候選；逐項人工判別——應清理孤兒手動移除，刻意特化則忽略
