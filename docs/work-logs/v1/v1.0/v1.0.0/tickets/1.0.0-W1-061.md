## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 分析對象

`.claude/skills/ticket/tests/test_checkpoint_state/test_performance.py` 中的 `test_E1_checkpoint_state_performance_median_under_100ms`。

### 斷言設計

測試在真實 tmp git repo + mock ticket CLI 環境下執行 `checkpoint_state()` 30 次，計算 median 和 p95，以硬門檻斷言：

- `assert median < 100.0`（AC6 硬 AC）
- `assert p95 < 150.0`（sage E1 目標）

兩者皆為**絕對計時門檻**，使用 `time.perf_counter()` 差值作為斷言對象。

### 問題識別

1. **負載敏感性**：計時值受 CPU 負載、GC、IO 排隊影響。ticket why 記載「5 背景 agent 並行」高負載下全套件 1 RED / 1 GREEN，standalone 5/5 GREEN——典型的環境依賴 flaky 模式。
2. **規則覆蓋缺口**：`test-assertion-design-rules.md` 規則 1-4 以 Jest/npm test 為標的（`tests/unit/`、`tests/integration/`、`tests/perf/`），未涵蓋 `.claude/skills/ticket/tests/` 下的 Python pytest 計時斷言。原則相同（計時硬門檻不入主套件）但無具體條款。

### 影響範圍

- 受影響測試：僅此 1 個 test case（grep 確認 `test_performance.py` 唯一計時斷言檔）
- 受影響指令：`npm run test:hooks`（透過 `uv run pytest tests/`）
- 不影響：`npm test`（Jest 主套件）
