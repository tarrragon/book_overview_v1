## Problem Analysis

### 問題根因

W1-006 ticket 原始 why 將本專案混合 hook 結構描述為「兩結構分叉使每次 full overlay sync 都產生孤兒重複」，暗示需「統一方向」。**此前提經 Reality Test 推翻**：

1. **「分叉」是誤命名**——本專案的混合結構（89 扁平 + 30 skill 自包含）是 W10-016 ANA 有意確立的**雙層架構決策**，非意外分叉。依據「框架資產 vs 專案產物分離」原則（framework-asset-separation.md §3）：skill 是可獨立發佈單位，其專用 hook 應隨 skill 移動。
2. **孤兒重複真根因**：非「結構分叉」，而是**遷移刪除未透過 sync-push 傳播至獨立 repo**。W10-092~094 遷移時本專案 `git mv`（本地舊扁平位置刪除），sync-push 推送了新自包含位置（結構感知 ✓），但 push 機制只 add/overwrite 不傳播 deletion → 獨立 repo 殘留 27 個遷移前的扁平孤兒。
3. **觸發機制**：下次 pull 時，獨立 repo 扁平 `hooks/<x>.py` 相對 base 屬「remote 保留」，被 merge/overlay 帶回本專案 → 與本專案 `skills/<s>/hooks/<x>.py` 並存 → 孤兒重複重演。W1-003 議題二定位的 base SHA 缺失（full overlay）只是「放大器」，真源頭是獨立 repo 殘留孤兒。

### 影響範圍

- `.claude/scripts/sync-claude-push.py`：缺「遷移刪除傳播」邏輯（真正待修點）
- 獨立 repo（github.com/tarrragon/claude.git）`hooks/`：殘留 27 個扁平孤兒（待清理）
- 不影響：本專案當前工作區（檔名層級零重複，W1-003~005 止血已生效）
- 不影響：歸屬規則（framework-asset-separation.md §3 已完整文件化，無需補）

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 決策結論

**方向 = 維持雙層架構（不做統一遷移）；真正待修為 sync-push 遷移刪除傳播缺口。**

ticket 原命題「skill 自包含 vs 扁平二選一統一」是偽選擇——reality test 證明雙層是 W10-016 有意決策且已穩定，孤兒重複源於 sync bug 而非結構選擇。

### WRAP 四階段

**W（Widen）候選**：

結構方向：
- 方向 1：維持雙層（W10-016）+ 修 sync push 傳播刪除 ← **採用**
- 方向 2：全扁平統一（30 自包含移回扁平）← 否決（推翻 W10-016、破壞 skill portability）
- 方向 3：全自包含統一 ← 不可行（73 通用 hook 無 skill 可歸屬，強行歸入破壞 SRP）

sync 修復子選項：
- B1：sync-push 傳播遷移刪除（偵測「本地自包含有 + 本地扁平無 + 遠端扁平有」→ dry-run + 確認後 git rm 遠端孤兒）← **採用**（修源頭，一次 push 即清 27 孤兒）
- B2：sync-pull 偵測扁平孤兒跳過 ← 不採用（B1 修源頭後無孤兒對象，冗餘）
- B3：sync-preserve 反向排除清單 ← 不採用（手動清單違反「不依賴記憶」）

**R（Reality Test）**：已執行（見重現實驗結果）。關鍵推翻：「結構分叉需統一」前提 → 實為「雙層正確 + push 未傳播刪除」。clone 獨立 repo 實證 27 孤兒源。

**A（Attain distance）機會成本**：維持雙層成本=0（已穩定態）；修 sync-push 為跨專案共用腳本，惠及所有專案（高槓桿，與 W1-003 同判斷）。不修則每次遷移都新增孤兒，債務複利累積。不擠壓 v1.0.0 主線（sync 屬框架基建）。

**P（Prepare to be wrong）**：
- (a) B1「刪除遠端檔案」風險——誤判孤兒可能誤刪有效 hook。防護：嚴格三條件 AND（本地自包含有 AND 本地扁平無 AND 遠端扁平有）+ 強制 dry-run 預覽 + 互動確認。
- (b) full overlay 路徑下 B1 是否仍正確識別孤兒——下放 IMP TDD 驗證兩條 sync 路徑。
- (c) 其他 skill 未來新增自包含 hook 時同模式復發——B1 機制化後自動覆蓋，非一次性清理。
