## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 兩層防護設計（保留 Tier 1，新增 Tier 2）

| 層 | 條件（交集） | 行為 | 範圍 |
|----|-------------|------|------|
| Tier 1 警告層（既有保留） | 同版本 pending/in_progress/completed(7d) + 相似度 >= 0.3 | stdout WARNING，不阻擋 | create + bulk_create（本票補齊） |
| Tier 2 阻擋層（新增） | 同版本 pending/in_progress + 相似度 >= 0.6 + 候選建立時間 < 60min | exit 1 阻擋，`--allow-duplicate` 旁路 | 僅 create（bulk_create 不套用，批次同質誤報風險） |

### 實作要點

- `_find_blocking_duplicate`：三條件交集偵測，命中回傳 `(id, title, status, similarity)`，無命中或異常回傳 None（容錯，不阻斷建立）
- `_enforce_blocking_duplicate`：命中且未旁路 → 印 `[ERROR]` + 命中 ID/相似度 + `--allow-duplicate` 提示，回傳 False；旁路 → 印 `[INFO]`，回傳 True
- `_validate_before_persist` 加 `allow_duplicate` 參數，Tier 2 排在 Tier 1 之前；命中阻擋時 `execute` 回傳 1
- bulk_create 迴圈內每筆 target 呼叫 `_detect_duplicate_tickets`（複用 create，無複製貼上）
- 常數 `DUPLICATE_BLOCK_THRESHOLD = 0.6` / `DUPLICATE_BLOCK_WINDOW_MINUTES = 60` 集中於 constants.py 並附 W1-040 量測依據註解

### 不在本 ticket 範圍

- W1-039.3 方案 B（git pre-commit 訊息指紋去重）、方案 C（PostToolUse 審計 lease）：parent W1-040 Solution 已標記無需建 ticket
