## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 火星收斂驗證（任務 1）

火星收斂成立。主因 = 觸發鏈（sync → 裸 cd → chpwd → confabulation）的「啟動頻率近期上升」，**非三個並列獨立因子疊加**。

**修正用戶「三因疊加」框架**：實為「一觸發鏈 + 兩放大器」：
- 真火星（觸發鏈，母 ANA line 79-85 六環）：裸 cd → chpwd 淹沒 → 不信任 → 同訊息續寫 → 滾雪球
- 放大器 A（sync 機制）：W1-003~W1-018 sync 密集 → 裸 cd 行為頻率↑ → 第 1 環啟動機率↑
- 放大器 B（MCP / 高 tool-call 密度）：tool-call 密度↑ → result 邊界模糊↑ → 第 3 環（不信任）易發

| 候選因子 | 是否獨立火星 | 真實角色 |
|---------|------------|---------|
| CC 版本更新 | 否（同版本一發生一不發生自證，母 ANA line 113-115） | 環境噪音 |
| sync 機制 | 部分（非內容變更觸發，是裸 cd 行為頻率放大） | 觸發鏈第 1 環頻率放大器 |
| MCP 工具 | 否（無證據為獨立環） | 情境因子（PC-166 觸發條件 3 已涵蓋高 tool-call 密度） |

解釋「為何兩天才出現」（放大器 A 近期密集）+「為何同版本一發生一不發生」（觸發鏈是否啟動取決於是否裸 cd）。無第四隱藏因子。

### 防護覆蓋確認（任務 2）— 6 環逐環對照

| 環 | 動作 | 覆蓋防護（grep 驗證存在） | 缺口 |
|----|------|------------------------|------|
| 1 | 裸 cd | bash-rules 規則一升級（git -C 首選 line 22）+ tool-output-trust 規則 4 | 無 hook 強制（已由 W1-023.4 ANA 評估承接） |
| 2 | chpwd 淹沒 | IMP-056 + 受眾段（PM 自身適用 line 51-55）+ bash-rules 即時協議步驟 1-2 | 本質無法 head/tail 預防，掐源頭+承接=設計正確 |
| 3 | 開始不信任 | bash-rules 即時協議步驟 1/3 + tool-output-trust 規則 2 | 不信任是事後歸因非機制因果，防護正確地掐環 4 |
| 4 | 同訊息續寫（confabulation 核心） | tool-output-trust 規則 1 + 規則 1 邊界（保並行）+ 即時協議步驟 1 | 所有 CC hook 偵測不到（hookdesign F1）= 覆蓋天花板非缺口，規則層自律承接 |
| 5 | 當真實 result 推論 | tool-output-trust 規則 3 + PC-166 防護 E + 即時協議步驟 4 | 復原點（高熵 grounding），覆蓋完整 |
