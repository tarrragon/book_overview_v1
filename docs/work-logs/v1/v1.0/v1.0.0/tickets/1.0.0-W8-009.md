## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

代理人定義在長期演進中累積了「人格／授權」以外的內容。依 `knowledge-carrier-allocation-methodology` 的「一句話判定」——代理人定義回答「你是誰、你能做什麼、你偏好怎麼做」，skill 回答「這件事怎麼做」——但多數 agent 檔混入了「怎麼做」的可重複流程、品質清單全文、錯誤案例全文、工具教學長文。這些內容每次派發時隨 agent 定義注入，重複付費（attention 稀釋 + token），且與單一來源（quality-common / error-pattern / search-tools-guide / skills）漂移。

### 影響範圍

`.claude/agents/` 共 33 個 `.md`。扣除元文件（AGENT_PRELOAD、README）與範本（language-agent-template）3 檔，扣除已 DEPRECATED 2 檔（john-carmack、memory-network-builder）與第三方 vendored 1 檔（impeccable-manual-edit-applier），實質代理人定義 27 檔需盤點。總計 12280 行，最長 parsley 1227 行、cinnamon 865 行、oregano 698 行，明顯超出「人格與授權」應有體量。

### 相關 Error Pattern

- PC-V1-006（執法對齊）：方法論首次權威化「代理人定義內容規範」，存量錯置即此規範的病史。
- 本盤點為 W8-016（執法 hook）的前置——掃描器須先於存量清理上線才不會產生告警噪音（W8-016 `blockedBy` 本票）。
