### 變更摘要（DOC）

依規則 8 引用性質判準（`.claude/references/reference-stability-rules.md` L178-187「移除測試」）盤點全框架 footer。

**盤點規模**：grep 全框架 footer（`**Version**` / `**Last Updated**` 行）含 ticket ID 共 392 處。逐一套用移除測試後，絕大多數含變更摘要屬歷史錨點型（保留）；真正「無摘要依賴型」（移除 ticket ID 後僅剩「初始建立 / 落地」等通用詞）收斂為 7 處（皆為 changelog 式 footer 的初版行）。

**判準**：含變更摘要（如「規則 6 加入」「主文外移至 references/」）→ 歷史錨點型 → 保留；無摘要（如「— W-xxx 落地」「初始建立（W-xxx 落地）」）→ 依賴型 → 內聯變更摘要（補「改了什麼」，保留 ticket ID 作歷史錨點）。

**WRAP 三問**：
- W（Widen）：(a) PM 前台逐處判斷 + 內聯（採用——判準需語意判斷，代理人易誤判）；(b) 派發代理人清理。
