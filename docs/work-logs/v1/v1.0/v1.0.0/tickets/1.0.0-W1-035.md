### 問題根因

bash-edit-guard-hook.py 的裸 cd 偵測有五項盲區（W1-024 對抗驗證 + w1024-review-cd-guard.md 審計報告確認）：

1. **newline 多行漏報（medium）**：偵測 regex connector 集 `(^|&&|;|\|\||\()` 缺 `\n` 且無 re.MULTILINE，導致換行分隔多行命令的第二行行首裸 cd（如 `echo hi\ncd subdir`）完全不偵測，是 confabulation 觸發鏈防護的真實空窗。
2. **pushd 未涵蓋**：pushd 同樣改變持久 cwd 並觸發 chpwd ls 淹沒（IMP-056），但原 regex 只匹配 `cd`。
3. **子 shell FP 不一致**：`(cd a && cd b)` 中第二個 cd 的 connector 為 `&&` 非 `(`，被誤報；且 `&&` 與 `;` 形式行為不一致（原邏輯只檢查 match 自身 connector 是否為左括號，無括號平衡追蹤）。
4. **docstring 缺描述**：模組頂層 docstring 仍只描述原地編輯偵測，未提裸 cd/pushd warn。
5. **warn log 100 字截斷**：main() 的 `logger.info(..., command[:100])` 截斷長命令，妨礙 FP 診斷。

### 影響範圍

- `.claude/hooks/bash-edit-guard-hook.py`：偵測核心 `_detect_bare_cd` + main() 日誌 + 模組/函式 docstring
- `.claude/hooks/tests/test_bash_edit_guard_hook.py`：新增情境覆蓋

### 相關 Error Pattern

- IMP-056（chpwd hook 淹沒）：本 hook 偵測的反模式根因
- PC-166 / tool-output-trust-rules：裸 cd 為 confabulation 觸發鏈第 1 環
