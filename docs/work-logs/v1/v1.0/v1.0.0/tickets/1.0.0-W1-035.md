## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

bash-edit-guard-hook.py 的裸 cd 偵測有五項盲區（W1-024 對抗驗證 + w1024-review-cd-guard.md 審計報告確認）：

1. **newline 多行漏報（medium）**：偵測 regex connector 集 `(^|&&|;|\|\||\()` 缺 `\n` 且無 re.MULTILINE，導致換行分隔多行命令的第二行行首裸 cd（如 `echo hi\ncd subdir`）完全不偵測，是 confabulation 觸發鏈防護的真實空窗。
2. **pushd 未涵蓋**：pushd 同樣改變持久 cwd 並觸發 chpwd ls 淹沒（IMP-056），但原 regex 只匹配 `cd`。
3. **子 shell FP 不一致**：`(cd a && cd b)` 中第二個 cd 的 connector 為 `&&` 非 `(`，被誤報；且 `&&` 與 `;` 形式行為不一致（原邏輯只檢查 match 自身 connector 是否為左括號，無括號平衡追蹤）。
4. **docstring 缺描述**：模組頂層 docstring 仍只描述原地編輯偵測，未提裸 cd/pushd warn。
5. **warn log 100 字截斷**：main() 的 `logger.info(..., command[:100])` 截斷長命令，妨礙 FP 診斷。

### 影響範圍

- `.claude/hooks/bash-edit-guard-hook.py`：偵測核心 `_detect_bare_cd` + main() 日誌 + 模組/函式 docstring
- `.claude/hooks/tests/test_bash_edit_guard_hook.py`：新增情境覆蓋

### 相關 Error Pattern

- IMP-056（chpwd hook 淹沒）：本 hook 偵測的反模式根因
- PC-166 / tool-output-trust-rules：裸 cd 為 confabulation 觸發鏈第 1 環

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要（commit 8f84cb75）

| # | 缺陷 | 修復 |
|---|------|------|
| 1 | newline 多行漏報 | `_BARE_CD_PATTERN` connector 集加入 `\n`：`(^|\n|&&|;|\|\||\()`，多行第二行行首 cd 現命中 |
| 2 | pushd 未涵蓋 | 關鍵字 alternation 由 `cd` 改為 `(cd|pushd)`，並用 `\b` + `\s+` 排除 cdrom/pushder 等字面 |
| 3 | 子 shell FP 不一致 | 改用括號深度追蹤：計算 cd/pushd 關鍵字起點前的 `(` 減 `)` 淨深度，`depth > 0` 一律排除。不論 connector 為 `(`、`&&` 或 `;`，子 shell 內 cd 一致排除；`(cd a && cd b)` 兩 cd 皆排除，`cd outer && (cd inner)` 僅外層命中 |
| 4 | docstring 缺描述 | 模組頂層 docstring 補「檢測模式 B（裸 cd / pushd）」段；函式 docstring 重寫對齊括號深度邏輯 |
| 5 | warn log 截斷 | main() 移除 `command[:100]` 截斷，診斷日誌記錄完整命令 + 命中 target（`target=%s`）；UI 訊息仍截短顯示（`_bare_cd_warning_message` 內 `[:100]` 保留） |

### 設計決策

- 重構為 `_find_bare_cd_target`（回傳命中 target 或 None）+ `_detect_bare_cd`（bool 包裝），保持既有 28 測試對 `_detect_bare_cd` 的相容，同時讓 main() 可記錄完整 target 供診斷。
- 維持 warn-not-deny 設計定位（permission_decision=allow），未改為 deny。
- 括號平衡用字串 count 而非完整 shell parse，符合「PreToolUse 只見 raw command、務實邊界」原則；heredoc/quoted 內 cd 字面誤判仍為接受的 FP 邊界。
