## Context Bundle

### 來源
W1-023.4.1 shipped bash-edit-guard 裸 cd 偵測後，本 session PM 第三次裸 cd（`cd /Users/.../book_overview_v1/.claude/skills/ticket`）觸發 chpwd 淹沒但 hook **未警告**，暴露排除過寬。

### 確切程式碼位置
`.claude/hooks/bash-edit-guard-hook.py` `_detect_bare_cd`（約 line 101-114）：
```
for match in re.finditer(r'(^|&&|;|\|\||\()\s*\bcd\s+(\S+)', command):
    connector = match.group(1)
    ...
    if target.startswith('/'):   # ← line 110：過寬，排除所有絕對路徑
        continue
```

### 修復要點
1. line 110 排除條件由「target 以 `/` 開頭」收窄為「target 正規化後等於專案根」。專案根偵測：`os.environ.get('CLAUDE_PROJECT_DIR')`（PreToolUse hook 環境可用）；正規化去除 trailing slash 再比對。
2. 收窄後行為：`cd /Users/.../book_overview_v1`（= 專案根，污染後補救）→ 排除不 warn；`cd /Users/.../book_overview_v1/.claude/...`（絕對子目錄）→ 恢復 warn；`cd subdir`（相對）→ 維持 warn。
3. CLAUDE_PROJECT_DIR 不存在時的 fallback：保守採「不排除」（即絕對子目錄 warn），repo-root 還原少數情境多一行提示可接受（warn 不 deny）。

### 測試（擴充既有 test_bash_edit_guard_hook.py）
- `cd /<repo-root>` → 排除（不命中），以 monkeypatch 設 CLAUDE_PROJECT_DIR
- `cd /<repo-root>/subdir` → 命中 warn
- `cd /abs/other` → 命中 warn
- CLAUDE_PROJECT_DIR 未設 → 絕對子目錄仍命中
- 既有相對 cd / 子 shell / git -C / 原地編輯 regression 全保留

- 精確 git add：僅 `.claude/hooks/bash-edit-guard-hook.py` + `.claude/hooks/tests/test_bash_edit_guard_hook.py`（PC-092）
