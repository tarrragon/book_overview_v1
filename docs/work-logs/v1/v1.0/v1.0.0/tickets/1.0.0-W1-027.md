## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

同一業務不變量（5W1H + where.files + acceptance 必填）在三條建票路徑執法不一致：

| 路徑 | 驗證點 | 行為 |
|------|--------|------|
| `create.py` 命令層 | `_validate_create_checklist` (862) → `_enforce_create_checklist` (928) | 缺欄位且未 `--force` → `sys.exit(1)` 阻擋（W11-003.5 升級） |
| `bulk_create.py` | 無 | `_create_batch_tickets` (187-258) 直接 `save_ticket` (251)，零驗證 |
| `generate.py` → `ticket_generator.py` | 無 | `_save_tickets` (114) 直接 `save_ticket` (147)，零驗證 |

兩條繞過路徑的 config 預設值本身即觸發 checklist 全部缺失欄位：

- `bulk_create._create_ticket_config` (94-135)：`who="pending"`、`why=""`、`how_strategy=""`、`where_files=[]`、`when="待定義"`
- `ticket_generator.generate` (225-243)：`who="pending"`、`when="待定義"`、`how_strategy="待定義"`、`acceptance=None`

結果：可建出 why 空字串 / who=pending / 無 acceptance 的殘缺票，且無 hook 安全網接住（驗證僅綁在 create 命令層，非寫檔瓶頸）。

### 影響範圍

- `.claude/skills/ticket/ticket_system/commands/bulk_create.py`（batch-create 入口）
- `.claude/skills/ticket/ticket_system/commands/generate.py`（generate 命令層輸出）
- `.claude/skills/ticket/ticket_system/lib/ticket_generator.py`（generate 核心迴圈，flat config 階段）
- `.claude/skills/ticket/ticket_system/lib/ticket_builder.py`（驗證函式下沉目標 + `TicketConfig` 定義處）
- `.claude/skills/ticket/ticket_system/commands/create.py`（驗證函式來源，需改 import / 保留別名）
- `.claude/skills/ticket/ticket_system/tests/`（兩路徑 pytest 覆蓋；既有 `test_create_ux_merged_validation.py` import 向後相容）

### 相關 Error Pattern

- 設計缺陷類型：「前門裝鎖、側門敞開」——同一不變量在不同入口執法不一致（source W1-024 對抗性複審挑戰 4）。
- 架構教訓對齊 ARCH-020（同構驗證邏輯散落多處導致無法同步修復）：本修復採「下沉至共同層 + 各入口複用」根除散落。

### 關鍵架構觀察（決定驗證放置點）

`_validate_create_checklist` 讀取 **flat config key**（`where_files`/`who`/`why`/`how_strategy`/`when`/`acceptance`/`decision_tree_path`/`parent_id`）。

- bulk_create 的 `_create_batch_tickets` 迴圈、ticket_generator 的 `generate` 迴圈，config 都還是 flat（key 與驗證函式一致）→ 在此階段呼叫可直接複用，零 adapter。
- 反例：`generate.py:_save_tickets` 是從 content 字串 re-parse 出**巢狀** frontmatter（`who.current` dict、`where.files`），key 不符 → 不可在此驗證。

故驗證必須放在 **flat config 階段**，非 save 階段。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 策略：驗證下沉至共同層 + 各入口 flat-config 階段複用（warning 級）

依 how.strategy「warning 級起步，不阻擋既有模板流程」。分四步：

**步驟 1 — 下沉驗證函式至 `lib/ticket_builder.py`**
- 將 `_validate_create_checklist(config, ticket_type) -> List[str]` 移至 `ticket_builder.py`，公開為 `validate_create_checklist`。
- `ticket_builder.py` 補 `from ticket_system.lib.constants import DEFAULT_UNDEFINED_VALUE`（已 import 同模組 STATUS_PENDING）。
- `create.py` 改 `from ...ticket_builder import validate_create_checklist`；保留 `_validate_create_checklist = validate_create_checklist` 別名，向後相容既有測試 import（`test_create_ux_merged_validation.py`）與 `_enforce_create_checklist` 呼叫鏈（行為零變更，create 仍阻擋）。

**步驟 2 — bulk_create 接線（warning，不阻擋）**
- `_create_batch_tickets` 迴圈中，config 建立後、`save_ticket` 前呼叫 `validate_create_checklist(config, config["ticket_type"])`。
- 有缺失時 append `(ticket_id, "缺必填欄位: <逗號分隔列全>")` 至 `result.warned`（既有欄位）。
- ticket 照常建立（Never break userspace）。`_print_batch_result` (309-313) 已有印 warned 區塊，零新輸出基礎設施。

**步驟 3 — generate 接線（warning，不阻擋）**
- `GeneratedTicket` dataclass 加欄位 `missing_fields: List[str] = field(default_factory=list)`（lib 不 print，只回傳）。
- `ticket_generator.generate` 迴圈內 flat config 階段呼叫驗證，結果存入 `gen_ticket.missing_fields`。
- `generate.py` command 層（`_print_generation_summary` 或 `_save_tickets` 後）逐票印出 missing warning。
- tickets 照常生成。

**步驟 4 — pytest 覆蓋兩路徑**
- bulk_create：缺欄位模板 → 斷言 `result.warned` 列全欄位 + ticket 仍建立（dry-run 或 tmp dir）。
- generate：plan 輸入 → 斷言 `GeneratedTicket.missing_fields` 列全 + tickets 仍生成。
- 回歸：既有 create 測試（阻擋行為）不破。

### 設計取捨

| 維度 | 決定 | 理由 |
|------|------|------|
| 下沉 vs 各自複寫 | 下沉至 builder | DRY，避免 ARCH-020 散落漂移 |
| 阻擋 vs warning | warning（兩繞過路徑） | Never break userspace；既有模板流程多為 pending 待補；create 阻擋行為不動 |
| 驗證放置點 | flat config 階段 | 巢狀 frontmatter key 不符驗證函式（見 Problem Analysis 架構觀察） |
| 觀察後升級阻擋 | 暫不做（warning 起步） | 依策略「觀察一輪後再評估」；若日後升級需建 follow-up ticket（decision-trigger-binding 規則 2） |

### 不在本 ticket 範圍

- 升級 batch-create/generate 為阻擋級（warning 起步，升級需另開 ticket 並綁 trigger）。
- create.py 既有阻擋行為的任何變更（保持 W11-003.5 現狀）。
