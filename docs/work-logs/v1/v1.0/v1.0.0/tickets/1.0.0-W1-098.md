## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 變更 1：擴展 VERSION_FILE_CANDIDATES

將候選清單從 2 項擴展為 5 項，依 ANA 設計順序排列：

```python
VERSION_FILE_CANDIDATES = [
    ("pubspec.yaml", "yaml"),       # Flutter
    ("package.json", "json"),       # NPM / Chrome Extension
    ("manifest.json", "json"),      # Chrome Extension
    ("composer.json", "json"),      # PHP
    ("pyproject.toml", "toml"),     # Python
]
```

### 變更 2：新增 resolve_version_source()

依 ANA 設計的三層優先序：
1. config 指定 `version_source.primary` 且檔案存在 → 使用指定檔案
2. 無 config → 依 `VERSION_FILE_CANDIDATES` 順序掃描，第一個存在的即為版本源
3. 所有候選都不存在 → fallback 到 git-tag（Go 專案路徑：go.mod 存在即觸發）

回傳 `(file_path, parser_type)` tuple，`file_path` 為 None 表示 git-tag 策略。parser 未指定時由副檔名推斷。
