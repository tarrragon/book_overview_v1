### 修改摘要

`check_identity` 情境 2（PM 豁免）與情境 3（--as 與 who.current 相符）放行前各補 `_write_telemetry` 呼叫，result 列舉新增 `RESULT_EXEMPT = "exempt"` 與 `RESULT_PASS = "pass"`。四條判定路徑（warn/exempt/pass/deny）全落盤，使「--as 使用率」分子（exempt+pass+deny 即 has_as=true 筆數）與分母（全部紀錄數）皆可從 usage.log 計算，修復 W1-049.2 量測缺口（原設計完美遵循時 log 零增長，10+ 樣本 trigger 永不成立）。

### 設計決策

- **隱私設計維持**：不記錄 as_value 原文，僅記 `has_as` 布林（沿用 W1-057 設計，新增測試斷言固化）。
- **docstring 同步**：模組 telemetry 區塊註解、`_write_telemetry` 與 `check_identity` docstring 均更新為四值列舉描述，判定邏輯表標注各情境對應 result。
- **測試調整**：原 `test_pass_path_does_not_write_telemetry`（斷言 pass 不落盤）改寫為 `test_pass_path_writes_telemetry`（斷言 result=pass 落盤）；新增 `test_exempt_path_writes_telemetry`（含隱私斷言：PM agent 名稱不出現於 log 原文）；`test_telemetry_appends_multiple_records` 擴為四路徑連續觸發覆蓋。
