### 問題根因（thyme 實測診斷，2026-06-11）

`uv run --project . pytest` 在 fresh 環境 Failed to spawn 的直接原因：worktree pyproject 將 pytest/pytest-cov 宣告在 `[project.optional-dependencies] dev`（extras），而 `uv run --project .` **預設不安裝 extras**。本機目前能跑是兩層環境運氣遮蔽：

1. 既有 `.venv` 曾以 dev extra 安裝過 pytest（uv run 為 non-exact sync，不移除多餘套件）
2. PATH 上有全域 pytest 9.0.2（Python 3.13.7）——以 `UV_PROJECT_ENVIRONMENT=/tmp/fresh` 重現實驗證實：uv 建立的 fresh venv 是 Python 3.12.11 且只 Installed 1 package，但 pytest session 顯示 Python 3.13.7，即 uv 找不到 venv 內 pytest 後 fallback 到 PATH 全域 pytest。無全域 pytest 的機器即重現 ticket 描述的 Failed to spawn

正確慣例為 `.claude/hooks/pyproject.toml`（CLAUDE.md 記載的 PC-124 修復）使用的 PEP 735 `[dependency-groups] dev`——uv run 預設安裝 dev group。注意：派發指示參考的 `.claude/skills/ticket/pyproject.toml` 用的是 `[project.optional-dependencies] test`，同樣不會被 uv run 預設安裝，不是可比照的修復慣例。

### 影響範圍

- `.claude/skills/worktree/pyproject.toml`（唯一需修改檔案）
- `.claude/skills/worktree/uv.lock`（dependency-groups 變更後 uv run 會自動更新 lock，需一併 commit）
- pyyaml 不需加入：grep 證實 worktree skill 的 tests/、scripts/、hooks/ 無任何 yaml import（ticket why 提及 pyyaml 屬沿用 hooks 慣例的推測，實際無消費者）
- 無 `--extra dev` 消費者：SKILL.md 與 package.json 均無引用，extras → dependency-groups 遷移無破壞面

### 相關 Error Pattern

PC-124（UV ephemeral env 不拉 transitive deps / dev deps 未宣告於 project env）
