### 問題根因

兩個獨立根因（均在 `commands/create.py`）：

1. **A2 分批報錯**：`WHY_REQUIRED` 驗證位於 `_parse_cli_args_to_config`（舊 line 797-804），以 `sys.exit(1)` 提前退出；而 when/who/how_strategy 等其他必填欄位由 `_validate_create_checklist` → `_enforce_create_checklist`（`_persist_and_report` 步驟 1.5）一次蒐集列全。why 的提前退出使用戶必須先修 why、再跑一次才看到其餘缺漏（本 session 實證 3 次試錯）。且 checklist 內本來就有 why 檢查（雙重驗證），提前退出屬冗餘路徑。

2. **A3 縮寫歧義**：argparse prefix matching 使 `--how` 同時匹配 `--how-type` / `--how-strategy`，原生 ambiguous 英文訊息不說明兩旗標各自用途。

### 影響範圍

- `.claude/skills/ticket/ticket_system/commands/create.py`：`_parse_cli_args_to_config`（移除提前退出）+ `register`（新增 `--how` 攔截 action）
- `.claude/skills/ticket/ticket_system/tests/test_create_error_envelope.py`：測試 4 改寫（WHY_REQUIRED → CHECKLIST 合併斷言）
- `.claude/skills/ticket/ticket_system/tests/test_create_ux_merged_validation.py`：新增 6 測試
- `tests/test_error_channel_integration.py` 場景 4 原本即容忍 CHECKLIST_VALIDATION_FAILED 或 WHY_REQUIRED 任一，無需修改

### 相關 Error Pattern

- PC-018（why 必填驗證的原始動機）：本修復保留 why 必填語意，僅改變報錯時機（合併列全）
