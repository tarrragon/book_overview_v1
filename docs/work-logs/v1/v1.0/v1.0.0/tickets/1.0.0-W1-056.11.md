### AC-1 驗證記錄（2026-06-11 /clear session）

| 驗證項 | 固定值命令 | 結果 |
|--------|-----------|------|
| plugin list 無條目 | `claude plugin list \| grep -i "document-skills\|example-skills"` | exit=1（無命中） |
| installed_plugins.json 零殘留 | Python 遞迴掃描 key/value | scan done 零 FOUND |
| 本 session skills 清單 | 觀察 system 注入清單 | 仍出現 document-skills:* 與 example-skills:* 兩組 |

**關鍵發現**：磁碟層卸載真實生效，但 /clear 後 session 的 skills 清單仍顯示兩組 plugin skills。推論：/clear 只清對話 context，不重啟 CC process；plugin/skills 載入發生在 process 啟動時。AC-1 的「新 session」定義必須是 **process 重啟**（quit Claude Code 後重開），/clear 不構成有效觀測點。

**AC-1 狀態**：磁碟層已驗證；skills 清單消失需待下次 process 重啟後觀測（與 AC-3 實測可合併同一次重啟）。
### AC-3 文件查證結論（claude-code-guide 代理人，2026-06-11）

| 問題 | 答案 | 來源 |
|------|------|------|
| 停用範圍 | **僅 bundled skills**；plugin skills、`.claude/skills/`、`.claude/commands/` 明確不受影響 | code.claude.com/docs/en/settings.md（原文："Skills from plugins, .claude/skills/, and .claude/commands/ are unaffected"） |
| 配置層級 | user / project / local / managed 四層皆生效 | 同上 |
| 生效時機 | 文件未明確記載；通用規則推測 live reload，保險做法為重啟 | 同上 |
| 引入版本 | v2.1.169（2026-06-08），另有環境變數 CLAUDE_CODE_DISABLE_BUNDLED_SKILLS | code.claude.com/docs/en/changelog.md |

**對本票 why 假設的裁定**：「disableBundledSkills 若涵蓋 plugin skills 可作更持久的專案級替代」——官方文件明確否定。plugin skills 不在此設定作用範圍，**卸載是移除 document-skills / example-skills 的唯一手段**，已執行的卸載決策維持不變。

### AC-3 實測設計（與 AC-1 合併單次 process 重啟）

文件查證縮小範圍後，實測目的為驗證文件宣稱 + 完成 AC-1 觀測。步驟：

1. `.claude/settings.local.json`（local-only，不入 repo / 不被 sync）寫入 `"disableBundledSkills": true`
2. 用戶重啟 Claude Code process（quit 後重開，/clear 不構成有效觀測點——本 session 已實證）
3. 新 session 觀測三點：
   - (a) AC-1：document-skills:* / example-skills:* 是否從 skills 清單消失（預期：消失，因已卸載）
   - (b) AC-3：bundled skills（claude-api / loop / schedule / verify / run / init 等無前綴項）是否消失（預期：消失）
   - (c) AC-3 範圍判定：其餘 plugin skills（commit-commands:commit / feature-dev:* 等）是否仍在（預期：仍在，驗證「僅內建」結論）
4. 觀測後依結果決定 disableBundledSkills 保留或移除（bundled skills 含用戶可能使用的 /code-review、/loop；若用戶需要則移除設定）

### AC-1 + AC-3 三點觀測結果（2026-06-11 process 重啟後 session）

觀測點有效性：本 session 由 SessionStart source=startup 啟動（process 重啟，非 /clear），符合實測設計第 2 步的有效觀測點定義。

| 觀測項 | 預期 | 實際 | 判定 |
|--------|------|------|------|
| (a) AC-1：document-skills:* / example-skills:* | 消失 | skills 清單完全不存在 | PASS |
| (b) AC-3：bundled skills（loop / schedule / verify / run / init / claude-api 等無前綴項） | 消失 | skills 清單完全不存在 | PASS |
| (c) AC-3 範圍判定：其餘 plugin skills | 仍在 | commit-commands:* / feature-dev:* / code-review:* / frontend-design:* 皆存在 | PASS |

固定值交叉驗證（tool-output-trust 規則 3）：

| 驗證項 | 命令 | 結果 |
|--------|------|------|
| 設定存在 | grep -n disableBundledSkills .claude/settings.local.json | 第 2 行 "disableBundledSkills": true |
| plugin 零殘留 | claude plugin list 管線 grep -ci | count=0, exit=1（無命中） |
| impeccable 來源判別 | ls .claude/skills/ 管線 grep impeccable | 存在於專案層 skills，非 bundled，不影響觀測 (b) |

**AC-3 結論**：disableBundledSkills:true 實測結果與官方文件宣稱一致——僅停用 bundled skills，plugin skills 與 .claude/skills/ 專案 skills 完全不受影響。「以設定取代卸載」假設確定不成立，已執行的 plugin 卸載決策維持不變。

**副作用揭露**：bundled skills 停用後 /loop、/schedule、/verify 等內建命令不可用；/code-review 為 plugin skill（code-review:code-review）不受影響。設定保留與否待用戶決策。
