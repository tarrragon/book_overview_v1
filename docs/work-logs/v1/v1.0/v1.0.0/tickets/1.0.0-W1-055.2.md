### 問題根因

`agent-commit-verification-hook.py` 的 `get_uncommitted_files`（原 159 行）以 `result.stdout.strip().splitlines()` 解析 `git status --porcelain` 輸出。整體 `strip()` 剝除第一行的前導空白——porcelain 格式為「XY filename」（X=index、Y=worktree 兩個狀態字元），worktree-modified 的 X 為空白字元（如「 M path」）——導致後續 `line[3:]` 對第一行多切一個字元，路徑首字元遺失。

兩個連鎖症狀：

1. **顯示截斷**：警告中路徑顯示為 `ocs/work-logs/...`（掉 d）、`claude/...`（掉 .）
2. **豁免繞過**：截斷發生在 `EXCLUDED_PATH_PREFIXES` 比對之前，`claude/...` 不再匹配 `.claude/` 前綴、`ocs/...` 不匹配 `docs/`，豁免失效 → 乾淨範圍內檔案被誤報為未 commit，製造重播迴圈的非空輸出（與 W1-055.1 範圍的 additionalContext 自激迴圈共振）

確定性 bug：豁免路徑只要落在 porcelain 第一行且該行以空白開頭即必誤報（選擇效應解釋了警告恰好只列 1 個截斷形態檔案的現象）。

### 影響範圍

- `.claude/hooks/agent-commit-verification-hook.py`（get_uncommitted_files 單一函式）
- 不影響同檔其他 porcelain 解析（get_unmerged_worktrees 用 startswith 比對完整行、git log --oneline 無前導空白問題）
