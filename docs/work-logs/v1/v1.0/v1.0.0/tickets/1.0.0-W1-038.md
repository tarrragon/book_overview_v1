### 問題根因

create 驗證測試債兩項（1.0.0-W1-024 對抗複審識別）：

(a) **--force why 豁免零覆蓋**：1.0.0-W1-024.1 A2 將 why 從「不可豁免」改為
「可由 --force 跳過阻擋」（刻意決策）。但 `tests/test_create_checklist_validation.py`
既有測試僅測 `_validate_create_checklist`（回傳缺失清單），全部隱含 force=False，
從未驗證 `_enforce_create_checklist` 的 force=True 豁免分支。

(b) **整合測試死 errno 寬容斷言**：`tests/test_error_channel_integration.py`
場景 4 的 valid_errnos 含 WHY_REQUIRED（A2 已移除）與 WHEN_REQUIRED /
HOW_STRATEGY_REQUIRED（從未存在）。`any(e in combined for e in valid_errnos)`
寬容析取使「提前退出回歸」時測試仍綠燈，削弱 A2 修復的回歸訊號。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### (a) --force why 豁免測試覆蓋

`tests/test_create_checklist_validation.py`：
- import 新增 `_enforce_create_checklist`。
- 新增 `TestEnforceCreateChecklistForce` 群組 4 測試：
  - `test_no_missing_returns_without_action`：無缺失直接 return（不阻擋不警告）。
  - `test_missing_why_without_force_blocks`：缺 why + 未 force → `pytest.raises(SystemExit)` 且 code==1。
  - `test_missing_why_with_force_allows`：缺 why + force=True → 不 raise、印「已 --force 跳過阻擋」、列出 why、不含 CHECKLIST_VALIDATION_FAILED。
  - `test_multiple_missing_with_force_allows`：多欄位（含 why）+ force=True → 放行並列出全部缺失。
- 斷言錨定 src 實際 WARNING 文字「已 --force 跳過阻擋」（format_warning 不輸出字面 "WARNING"）。

### (b) 整合測試死 errno 收斂

`tests/test_error_channel_integration.py` 場景 4：
- 移除寬容析取 `valid_errnos = (CHECKLIST_VALIDATION_FAILED, WHY_REQUIRED, WHEN_REQUIRED, HOW_STRATEGY_REQUIRED)` + `any(...)`。
- 收斂為單一精確斷言 `assert "errno: CHECKLIST_VALIDATION_FAILED" in combined`，與 _render_envelope 輸出格式（`  errno: {errno}`）一致。
- 加註解說明 A2 後死 errno 已移除、收斂理由。

- 未動 ticket_system/ 任何 src（純測試清理）。
- 未觸碰 .claude/hooks/acceptance_checkers/（並行 agent 範圍）。
