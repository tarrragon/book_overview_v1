## Context Bundle

### 目標
讓 full-overlay 路徑不再復活「本地刻意刪除但上游仍存在」的檔（如 W10-049 刪的 root 孤兒）。根因：.sync-state.json 為 local-only（gitignored），fresh clone 缺 base SHA → 首次 sync 走 full-overlay → 復活上游孤兒。M2 用本地 git 史（隨 repo clone 而存在）作刪除 SSOT，survives fresh clone。詳見父 ANA 1.0.0-W8-036 與 1.0.0-W8-036.3 結論。

### 改動位置（檔：.claude/scripts/sync-claude-pull.py）
- `main()` full-overlay 分支 L1546-1595（`use_full_overlay` 為 True 時）。
- `sync_directory(src, dst, preserve, prefix)` L661：遞迴複製上游→本地，是復活發生點。
- `preview_overlay_changes(...)` L891：dry-run 預覽（目前只列 overwrite+delete，不列復活 → 須補）。
- 鏡像參考：既有 `cleanup_stale_files`（L809）已是 git-tracking-aware（用 `_is_git_tracked` 判定，tracked local-only 檔移 .sync-conflicts 不刪）。M2 在「複製方向」做對稱的 git-aware 判定。

### 設計：git-delete-aware 跳過復活
新增判定函式（語意）：給定 .claude/ 下相對路徑，判斷是否「刻意刪除」=
1. 該檔目前**不在本地磁碟**（absent）— 在本地的檔屬 overwrite 非復活，不適用。
2. 且本地 git 史**最後一次涉及該檔的事件為刪除（D）**，非後續 re-add。
判定為刻意刪除 → full-overlay **跳過複製**（不復活）；否則照舊複製。

實作要點：用 `git log -1 --diff-filter=D --format=%H -- <path>`（最近刪除 commit）與 `git log -1 --diff-filter=A --format=%H -- <path>`（最近新增 commit）比較先後（或 `git log -1 --name-status` 取最後事件 status）。project_root 為 .claude_dir.parent（git 判定基準，同 cleanup_stale_files）。

### 邊界（必測三情境）
1. **clean-delete**：本地刪除已 commit、磁碟 absent、上游有 → **跳過復活**。
2. **delete-then-readd**：刪後又 re-add 已 commit、磁碟 present（如現存 12 孤兒）→ **不跳過**，正常 overlay 覆蓋。
3. **never-existed-locally**：上游新檔、本地 git 史從無、磁碟 absent → **不跳過**（正常複製，是真新檔非刻意刪除）。此情境是關鍵——須區分「刻意刪除」與「上游新檔」。

### preview 補強
`preview_overlay_changes` 須新增「will_skip_resurrection」清單並在 main full-overlay 區印出（消除復活靜默，對齊 dry-run 可見性原則）。

### 測試
鏡像 `.claude/scripts/tests/test_sync_claude_pull_cleanup_git_aware.py` 結構，新增測試檔（如 test_sync_claude_pull_overlay_git_delete_aware.py），涵蓋上述三情境 + preview 顯示。測試以 tmp git repo fixture 建構（參考既有 cleanup_git_aware 測試的 fixture 模式）。
