## Problem Analysis

### 問題根因

`version_release.py` 缺少 `detect_project_type()` 函式，無法在 `.version-release.yaml` 不存在或缺少 `project_type` 欄位時自動判定專案類型。

### 影響範圍

- `.claude/skills/version-release/scripts/version_release.py` — 新增函式
- `.claude/skills/version-release/tests/test_detect_project_type.py` — 新增測試檔

### 相關 Error Pattern

無直接相關 error pattern。

---

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

在 `version_release.py` 的 `detect_version_files()` 與 `resolve_version_source()` 之間新增：

1. **7 個專案類型常數**：`PROJECT_TYPE_FLUTTER` / `GO` / `CHROME_EXT` / `PHP` / `NPM` / `PYTHON` / `MONOREPO` / `UNKNOWN`
2. **`_PROJECT_TYPE_MARKERS` 查找表**：根目錄檔案 → 專案類型的對應（pubspec.yaml→flutter, go.mod→go, composer.json→php, pyproject.toml→python）
3. **`detect_project_type(root)` 函式**：依優先序掃描根目錄，package.json+manifest.json→chrome-ext，package.json 單獨→npm，子目錄 depth=1 含版本檔→monorepo，全無→unknown
4. 所有偵測結果輸出至 stderr（`print(..., file=sys.stderr)`），不污染 stdout
5. monorepo 偵測跳過隱藏目錄（`.` 開頭）且僅掃描 depth=1
