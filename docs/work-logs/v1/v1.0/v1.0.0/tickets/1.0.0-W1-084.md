### 修改範圍

### 三段實作細節

**(1) 版本檔衝突自動採 upstream**：新增 `VERSION_FILES_TAKE_UPSTREAM = frozenset({"VERSION", "CHANGELOG.md"})` 白名單（以 claude_rel 完整相對路徑精確比對，巢狀同名檔不命中）。`apply_upstream_delta` 衝突分支內：先照原流程把含衝突標記的合併結果寫 `.sync-conflicts/` 作對照副本，白名單檔再以 `_atomic_write` 套用 upstream 內容（納入 rollback_log，失敗可回滾）+ stdout 黃色註記，不計入 conflicts 清單。非白名單檔走原 local-保留路徑，行為零變更（既有 `test_apply_delta_four_scenarios` 仍綠驗證）。

**(2) 殘留偵測**：新增 `detect_conflict_residue(claude_dir, before_time)`（rglob 掃 `.sync-conflicts/`，mtime 早於本次執行時間視為前次殘留；排除 `.gitignore` 與 symlink；mtime 讀取失敗寧可多警告列入殘留）與 `warn_conflict_residue`（stdout 警告列出 + 指向檢查清單，不阻擋）。`main()` 在 `_validate_environment` 後、clone 前呼叫，確保在本次新衝突寫入前列出，避免新舊混雜。

**(3) pull 後檢查清單**：`commands/sync-pull.md` 新增章節，三明示開頭（Why=ad-hoc 無 SOP + 殘留實證 / Consequence=殘留累積、衝突標記入正式檔、commit 邊界汙染 / Action=五步）。五步：delta 檢視（git status/diff）、衝突決策表（版本檔自動採 upstream 無需動作 + 非版本檔三選項：採 local/採 upstream/手動合併）、grep 衝突標記歸零驗證（排除 .sync-conflicts/）、commit 規範（單一 commit + 訊息格式 + 禁混入他務）、清理 .sync-conflicts/（清理 = 清單走完的訊號，與殘留偵測閉環）。

### 設計決策

- 對照副本沿用既有「含衝突標記的 merge-file 結果」格式（同時保留 local 與 upstream 雙方內容），與非版本檔衝突副本格式一致，最小行為分歧。
- 不改 `apply_upstream_delta` 回傳簽名 `(applied, conflicts)`，避免破壞既有測試與呼叫端；自動解決檔計入 applied。
- 殘留警告放 `main()` 而非 `_sync_with_backup`：偵測屬 pull 前置檢查語意，且需在任何新衝突寫入前執行。
