### 修復摘要

抽共用 helper `lib/ambiguous_prefix.py` 泛化原 `create.py:_AmbiguousHowAction`，對 4+1 個縮寫歧義 token 套用同款攔截，命中時輸出中文用途提示。

### helper 設計（lib/ambiguous_prefix.py）

- `AmbiguousPrefixAction(argparse.Action)`：基底類別，class 屬性 `hint` 攜帶提示，`__call__` 呼叫 `parser.error(self.hint)`。
- `make_ambiguous_action(hint)`：動態建立帶 hint 的子類別（argparse `action=` 需類別非實例）。
- `register_ambiguous_prefix(parser, option_string, hint)`：封裝 `add_argument(..., nargs="?", action=..., help=SUPPRESS)` 樣板，呼叫端只給 token + 中文提示。

機制：顯式 `nargs="?"` 註冊歧義 token 使 exact match 優先於 argparse 縮寫展開。`help=SUPPRESS` 不污染 --help。

### 套用 token（subparser scoped）

| token | subparser | 提示指向完整旗標 |
|-------|-----------|----------------|
| --decision-tree | create | --decision-tree-entry/-decision/-rationale |
| --how（既有） | create | --how-type/--how-strategy（改用 helper，DRY） |
| --ho（新增） | create | 同 --how（共用 _HOW_AMBIGUOUS_HINT 常數） |
| --from | handoff | --from-ticket-id/--from-worklog |
| --to | handoff | --to-parent/--to-child/--to-sibling |
| --all | **僅 set-acceptance** | --all-check/--all-uncheck |

### --ho 決策（約束 2 落地，decision-trigger-binding 狀態 a）

決策：攔截。--ho 為 --how 的更短前綴同類誤打，成本低（共用既有 _HOW_AMBIGUOUS_HINT 常數，零額外 hint 文字），故一併攔截而非文件化豁免。

### --all 作用域（約束 1）

helper 註冊於 p_set_acceptance（track.py），不全域。實測 list/stale-list/stuck-anas 的合法 --all 不受影響。
