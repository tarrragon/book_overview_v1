### 問題根因

dispatch-active 規則三方字面不一致（W9-007 修測試時發現）：
- pattern（GITIGNORE_REQUIRED_RULES line 33）：.claude/dispatch-active（無後綴）
- 實際要忽略的檔案：.claude/dispatch-active.json + .claude/dispatch-active.lock（兩運行時檔）
- missing 建議字面（動態 line 696 + _create_missing line 646）：.claude/dispatch-active.json
- _has_gitignore_rule 精確比對（line 614）：normalize 用 rstrip("/*") 只去斜線與星號，不去 .json 後綴

矛盾：使用者照 missing 建議寫 .claude/dispatch-active.json → normalize 後 .claude/dispatch-active.json != pattern .claude/dispatch-active → 永遠判缺失（建議不 actionable）。

### 修法方向（用戶選「改 missing 建議字面」，W9-008 acceptance 列選項之一）

匹配邏輯本身沒錯：pattern .claude/dispatch-active 配合 normalize rstrip("/*") 已認可使用者寫 .claude/dispatch-active*。真正 bug 是 missing 建議字面寫具體 .json 誤導使用者。改建議字面為 .claude/dispatch-active*：
- normalize rstrip("/*") → .claude/dispatch-active == pattern，通過 check
- git 實際忽略 .json + .lock 兩檔（runtime check-ignore 驗證命中）

### 衍生發現（建 W9-009 ANA 追蹤）

對本專案根目錄實測 check_gitignore_completeness：missing_rules=[.claude/hook-logs/, .claude/tool-results/, .claude/dispatch-active*]，三條被判缺：
- hook-logs：false positive（本專案用 **/hook-logs/ 萬用字元 line 72，精確比對不認）
- dispatch-active：false positive（本專案用具體 .json+.lock line 99-100，精確比對不認）
- tool-results：真缺（本專案 .gitignore 無此規則）

_has_gitignore_rule 精確匹配對合法 gitignore 變體系統性 false positive，建 1.0.0-W9-009(ANA) 分析正確修法（git check-ignore 語意理解）。
