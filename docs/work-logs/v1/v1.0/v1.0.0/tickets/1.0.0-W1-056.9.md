### 修復摘要

擴充 agent-ticket-validation-hook，在既有「ticket 存在性 + 決策樹欄位」驗證後新增「嵌套深度檢查」強制層（協議 v2 D3 / D2 條件 D-3 落地）。

### 設計決策

| 決策點 | 選擇 | 理由 |
|--------|------|------|
| 深度計算來源 | import 複用 `ticket_system.lib.depth.can_descend` / `compute_depth` + `constants.MAX_TICKET_DEPTH` | 禁止平行實作（ARCH-020 雙副本漂移）；can_descend 為唯一判準（depth < MAX_TICKET_DEPTH） |
| deny 判準 | `not can_descend(ticket_id)`（depth >= MAX_TICKET_DEPTH=3） | 被引用 ticket 已處最深層，不應再以其 descend 派發嵌套 Agent |
| 豁免型 agent | Explore/general-purpose/Plan 等在步驟 1 提前 return，完全不觸及深度檢查 | Never break userspace（唯讀探針不受影響，AC2） |
| import 失敗處理 | fail-open（DEPTH_AVAILABLE=False 時跳過深度檢查，既有派發行為不變） | 避免環境缺套件時破壞既有驗證；深度計算過程異常亦 fail-open |
| transitive deps | script header `dependencies` 加 `pyyaml>=5.0` | depth → parser → yaml；避免 uv ephemeral env 不拉 transitive deps 的 PC-124 模式（實測 dependencies=[] 會 ModuleNotFoundError: yaml） |
