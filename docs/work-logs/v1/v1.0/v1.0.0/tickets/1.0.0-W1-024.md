### A. ticket 系統 CLI 指令設計問題

| # | 問題 | 本 session 實證 | 疑似設計缺陷 |
|---|------|----------------|------------|
| A1 | `create` 命名分層不一致 | `ticket track create` 報 INVALID_CHOICE；create 是頂層 `ticket create`，但 claim/complete/append-log 是 `ticket track <op>` | 頂層命令 vs track 子命令分層不直觀；連 SKILL hook 都建議 `/ticket create` 而非 track create |
| A2 | ANA create 必填欄位**分批報錯** | 先報 WHY_REQUIRED → 補後再報 CHECKLIST_VALIDATION_FAILED（who/when/where/how_strategy）→ 試錯 4 次才建成 | 必填驗證未一次列全，逐項報錯導致多輪試錯 |
