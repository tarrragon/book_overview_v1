## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

acceptance-gate-hook 的 `execution_log_checker.py:check_execution_log_filled` 函式**僅檢查 Solution 和 Test Results 兩個章節**是否為空。「重現實驗結果」章節雖被列入 `_SCHEMA_SECTION_NAMES`（作為區段邊界辨識），但從未被任何 checker 函式檢查是否有實質內容。

兩層缺陷：
1. **主缺陷**：無任何 checker 呼叫 `_is_section_empty(content, '重現實驗結果')`
2. **次缺陷**：即使未來加入檢查，`_is_section_empty` 對含 H3 子標題（`### 實驗方法` 等）的 placeholder 會回傳 False（非空），因 H3 標題行本身在去除 HTML 註解和 `（待填寫）` 後仍存活

### 影響範圍

- 所有 ANA ticket 的「重現實驗結果」章節可為純 placeholder 通過 complete
- 受影響 ticket 實例：W7-003、W8-002（本次已回填）

### 相關 Error Pattern

- PC-110（空殼 complete 放行）— 同模式但不同章節

## 重現實驗結果
### 實驗方法

在 Python REPL 中直接呼叫 acceptance-gate-hook 的 `_is_section_empty` 和 `check_execution_log_filled` 函式，以 W7-003 原始 placeholder 內容（三個 H3 子標題 + `（必填：...）` 文字）作為輸入，驗證 hook 是否偵測到空殼。

### 實驗執行

```python
# 測試 1：_is_section_empty('重現實驗結果') 對 placeholder
result = _is_section_empty(placeholder_content, '重現實驗結果')
# 結果：False（H3 子標題行存活，被視為「有內容」）

# 測試 2：check_execution_log_filled 對含 placeholder reproduction + 實質 Solution
result = check_execution_log_filled(placeholder_content_with_solution, logger)
# 結果：False（表示「已有內容」），因為只檢查 Solution + Test Results
```

### 實驗發現

**已驗證事實**：
1. `check_execution_log_filled` 函式邏輯只含 `_is_section_empty(content, "Solution")` 和 `_is_section_empty(content, "Test Results")`，完全不觸碰「重現實驗結果」
2. `_is_section_empty` 的 placeholder 剝除邏輯只移除 `（待填寫...）` 格式，H3 標題（`### 實驗方法`）和 `（必填：...）` 格式文字存活 → 即使加入檢查也會 false negative
3. `multi_view_status: skipped` 僅影響 `multi_view_checker.py` 的獨立檢查，與 execution_log_checker 無關聯

**仍未驗證的假設**：無（三點均已程式碼驗證）

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 結論

**方向：hook 漏洞，需 spawn IMP 修復**

acceptance-gate-hook 對 ANA ticket「重現實驗結果」章節完全無偵測。這是 hook 偵測範圍缺口（非合法豁免），原因：

1. 「重現實驗結果」在 ticket-body-schema 中標為 ANA 必填（PC-063 防護措施），但 hook 實作僅覆蓋 Solution + Test Results
2. `multi_view_status: skipped` 不影響此缺口（兩者為獨立 checker，無連帶關係）
3. 評估型 ANA（如 W7-003）的 reproduction 應可合法填入「程式碼分析式驗證」而非傳統 bug 重現，但不應允許 placeholder 通過

### 建議修復（spawn IMP 範圍）

| # | 修復項 | 說明 |
|---|--------|------|
| 1 | `check_execution_log_filled` 加入「重現實驗結果」檢查 | ANA type ticket 時額外呼叫 `_is_section_empty(content, '重現實驗結果')` |
| 2 | `_is_section_empty` 增加 H3 子標題剝除 | `### 實驗方法` 等 H3-only 行應被視為結構骨架非實質內容 |
| 3 | placeholder 格式擴充 | `（必填：...）` 格式（圓括號+冒號）應與 `（待填寫...）` 同等剝除 |

### 對比分析：W8-001 vs W7-003/W8-002

- W8-001（同 saffron 執行）：reproduction 章節已實填（「代碼分析式驗證」）
- W7-003 / W8-002：reproduction 章節留 placeholder
- 差異原因：W8-001 prompt 未含「約束：ANA 只調查不修」等搶時效壓力指令，saffron 自然填寫；W7-003/W8-002 在快速產出壓力下跳過（非系統性 agent 瑕疵，是 hook 未攔截的結果）
