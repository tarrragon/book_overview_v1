## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 根因（精確定位，非 file_lock 本身）

垃圾不是 file_lock 對 raw mock 產生，而是 **command 在 call site 把 mock 轉成真實垃圾 Path**：

- command（`track_acceptance.py:305/491/617`、`track_relations.py`）：`lock_target = Path(get_ticket_path(version, tid))`
- 測試用 `patch("...get_ticket_path")`（無 return_value）→ get_ticket_path 回傳 MagicMock
- `Path(MagicMock)` 觸發 `os.fspath(mock)` → 合成字串 `"MagicMock/get_ticket_path()/<id>"` → 變成**真實 Path 物件**
- `file_lock(real_path)` → `open(lock_path,"w")` 在 cwd 寫出 `MagicMock/get_ticket_path()/<id>.lock` 真實垃圾

**關鍵**：file_lock 端的 isinstance(str/Path) guard 無效——mock 在進 file_lock 前已被 `Path()` 轉為合法 Path（曾試加 guard 後還原，file_lock.py 無淨變更）。

### 修復（測試 mock 衛生 + 防護）

| 項目 | 內容 |
|------|------|
| test_track_acceptance.py | import tempfile + 模組常數 `_LOCK_TARGET = Path(tempfile.gettempdir())/"w1018_test_lock_target.md"`；6 處 `patch(get_ticket_path)` 加 `return_value=_LOCK_TARGET` |
| test_track_relations_set_fields.py | 同上模式，5 處 patch |
| 效果 | lock 檔改落 /tmp（OS 清理），cwd 不再產生垃圾 |
| .gitignore | 加 `MagicMock/`（第二道防護，防誤 git add 復發；既有 `*.lock` 已涵蓋 .lock） |
| 本地清理 | rm `./MagicMock`（根）+ `.claude/skills/ticket/MagicMock`（632 垃圾） |

### 範圍邊界（與 W1-018.2/.3/W1-009 分工）

本 ticket 不碰遠端孤兒（W1-009 --clean 統一清，含這 632 的遠端副本）；不碰 should_exclude（W1-018.2）；不碰 gitlink（W1-018.3）。
