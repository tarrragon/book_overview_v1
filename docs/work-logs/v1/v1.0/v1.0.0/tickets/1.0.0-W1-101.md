## Problem Analysis

### 問題根因

`version_release.py` 僅有 `bump_json_version()`，無法處理 Flutter (`pubspec.yaml`) 和 Python (`pyproject.toml`) 專案的版本 bump。

### 影響範圍

`.claude/skills/version-release/scripts/version_release.py` — 新增兩個函式，不修改既有函式。

---

## Solution

在 `version_release.py` 的 `bump_json_version()` 之後新增兩個函式：

1. **`bump_yaml_version(file_path, new_version, dry_run)`** — regex 逐行替換 `version:` 欄位，保留檔案格式。支援 pubspec.yaml 的 `X.Y.Z+build` 格式（build number 自動遞增）。
2. **`bump_toml_version(file_path, new_version, dry_run)`** — regex 逐行替換 `version = "X.Y.Z"` 欄位，保留雙引號/單引號風格。

兩者介面與 `bump_json_version` 一致。
