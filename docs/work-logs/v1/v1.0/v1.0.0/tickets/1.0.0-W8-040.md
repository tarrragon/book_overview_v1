## Problem Analysis

### 問題根因

framework-meta-methodology 與 methodology-writing（方法論 + skill）三檔同步把方法論定義為「30 秒複習清單 / 電梯測試 / 不超過 1 頁 / ≤ 5 條要點」。此定義把兩件本質不同的事混為一談：(a) 不把操作流程放方法論（正確）、(b) 把判斷標準本身壓縮到 5 條要點（對 AI 開發判斷有害——壓縮遮蔽判準細節導致誤判）。

### 影響範圍

三檔定義源頭（framework-meta-methodology.md / methodology-writing-methodology.md 第一節 / skills/methodology-writing/SKILL.md）。下游約 20 個已瘦身方法論的回補判斷另由 W8-042 處理；標籤同步由 W8-041 處理。
