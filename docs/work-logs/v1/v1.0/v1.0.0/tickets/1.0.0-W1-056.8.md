## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要（H3 within Solution）

新增嵌套深度防護 CLI 強制層（協議 v2 D3 落地，S3 spawn）。

**深度計算（parent_id 鏈，linux F1 fatal 修正）**：
- 新增 `ticket_system/lib/depth.py`：`compute_depth(id)` 沿 `parent_id` 鏈回溯計算（根=depth 1），`can_descend(id)` = depth < MAX_TICKET_DEPTH。
- `_depth_via_resolver(id, parent_of)` 抽純邏輯供單元測試注入記憶體鏈，不依賴真實檔案。
- 刪除 ID 字串數點法：完整版本前綴 ID（`1.0.0-W1-056.5` 含 3 點）以 `count('.')+1` 會誤算 depth 4，parent_id 鏈正確算 depth 2。含環安全上限 `_MAX_CHAIN_WALK=100`。

**MAX_TICKET_DEPTH 常數（DRY，linux F2）**：
- 新增於 `ticket_system/constants.py`（值 3），唯一深度判準來源。

**create --parent 深度上限 warning（不硬擋）**：
- `create.py:_resolve_ticket_id_and_wave` 在 `--parent` 分支計算新子任務深度 = `compute_depth(parent)+1`，>= MAX_TICKET_DEPTH 時 emit `WarningMessages.DEPTH_LIMIT_REACHED`，仍回傳有效 ticket_id（留旁路）。

**ticket track depth 命令**：
- 新增 `commands/track_depth.py`（`register_depth` + `execute_depth`），輸出 depth/max_depth/can_descend，version-aware（version 由 ticket ID 自動提取）。
- track.py 註冊：handler dict + `register_depth(track_subparsers)`。

**SKILL.md 同步**（ticket-skill-sync-check：新增子命令 + create 行為變更）：
- 子命令總覽表新增 `track depth` 列。
- track 詳解新增 depth 命令說明 + create --parent 深度 warning 說明。

### 不在本 ticket 範圍
- S4 dispatch-gate hook 嵌套深度感知（另 ticket）。
- identity_guard.py、.claude/agents/（並行 session 範圍，未觸碰）。
