### 修復實作（framework issue #1 問題1+2）

### 變更檔案
- project_init/lib/env_detector.py：
  - 新增 _query_python_version(executable) helper：subprocess.run 查版本，Timeout/OSError 回 None，stdout 空時取 stderr。
  - detect_python 改 OS 感知多候選（Windows: python/python3/py；類 Unix: python3/python），全 miss 時 fallback shutil.which("uv") + uv python find。失敗統一回「未找到 Python（已嘗試 PATH 候選與 uv 管理的 Python）」。
- project_init/lib/messages.py：
  - 補 import platform / from typing import Optional。
  - get_python_install_steps(os_type=None) 改 OS 感知 + uv 優先：uv python install 為推薦，verify_cmd 依 Windows(python --version)/Unix(python3 --version) 切換，保留 python.org 為替代。caller check.py:151 無參呼叫相容（os_type 預設 platform.system()）。

### caller/test 同步（PC-135）
- test_env_detector.py：多候選邏輯使「不在 PATH／版本失敗／超時」統一為「全候選耗盡」，更新 3 個受影響斷言（test_python_not_found/version_command_failed/timeout 改驗「未找到 Python」）；新增 test_python_found_via_uv_fallback + test_python_windows_uses_python_exe_candidate 覆蓋新路徑。test_python_generic_error 不受影響（非 Timeout/OSError 冒泡至外層 handler）。
- test_messages.py：test_python_install_steps 加 uv 斷言；新增 test_python_install_steps_os_aware_verify_cmd 驗 OS 切換。
