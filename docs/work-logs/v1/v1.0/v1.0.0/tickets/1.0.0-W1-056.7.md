### 變更摘要

`.claude/references/agent-dispatch-template.md` 新增「嵌套派發（descend）派發端指引」章節（置於 Dispatch-Plan Template 之後，與嵌套欄位緊鄰），版本 bump 1.6.0 → 1.7.0。內容三件套：

| 子節 | 內容 | DRY 處理 |
|------|------|---------|
| descend 條件速查 | D-1~D-5 條件摘要 + 派發端動作對照表；ascend 優先於 descend 明示；`ticket track depth` 層級查詢指令範例 | 完整判定方式引用 AGENT_PRELOAD 規則 9.2，本章只列派發端動作 |
| dispatch-plan 嵌套欄位 | 既有七欄外每列補 `parent`（create --parent 建鏈）與 `depth / can_descend`（depth 命令回填）兩欄 + Why 說明 | parent_id 鏈 SSOT 語意引用 D3，不重複定義深度公式 |
| child prompt 範例 | 嵌套三段式骨架（與單層差異兩點：context 先入 child Problem Analysis、final message 僅指向 ticket）+ 上層回報義務（自身 ticket append-log 引用 child ID） | 五步自檢引用規則 9.2；血緣 vs 衍生引用 field-semantics.md |

### 設計決策

1. **MAX_TICKET_DEPTH 數字零出現**：本檔不出現深度上限數值；`ticket track depth` 指令範例輸出抽象化為「回傳 depth / max_depth / can_descend 三欄位」，明示「上限數值由 CLI 維護，prompt 與文件不重複硬編」。唯一定義點維持 AGENT_PRELOAD 規則 9.2。
2. **依據來源**：1.0.0-W1-056.5「### 修訂版協議（v2）」（v1 已取代未引用）+ AGENT_PRELOAD 規則 9（v1.11.0，.6 已落地）。S2 spawn 項落地。
3. **章節定位**：派發端視角（descend 前置確認 + dispatch-plan + child prompt），與 AGENT_PRELOAD 規則 9 的執行端視角（claim 後自檢、三階段表）分工互補，無平行定義。
