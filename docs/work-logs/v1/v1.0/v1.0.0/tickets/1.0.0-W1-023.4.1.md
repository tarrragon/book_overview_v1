## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要

擴充宿主 `bash-edit-guard-hook.py`（零新 hook，宿主已註冊 settings.json），新增裸 cd warn 偵測。

1. **`lib/hook_messages.py`**：新增 `ValidationMessages.BARE_CD_WARNING` 常數（集中字串管理，仿 `BASH_EDIT_DETAILED_WARNING` 範本），訊息提示改用 `git -C <abs>` 或子 shell，並引用 `bash-tool-usage-rules.md` 規則一。
2. **`bash-edit-guard-hook.py`**：
   - 新增 `_detect_bare_cd(command) -> bool`：以 `re.finditer(r'(^|&&|;|\|\||\()\s*\bcd\s+(\S+)')` 掃描所有 cd 出現點，逐一判斷。命中：行首 cd／`&& cd`／`; cd`／`|| cd`。排除：子 shell（connector 為左括號）、絕對路徑還原（target 以 `/` 開頭）；`git -C`／`uv -d` 因不含 cd 指令天然不命中。
   - 新增 `_bare_cd_warning_message(command)`：走 `ValidationMessages` 常數產生訊息。
   - `main()` 改為「兩偵測獨立」：原地編輯偵測與裸 cd 偵測各自蒐集警告至 `warnings` list，合併為單一 `emit_hook_output`，`permission_decision=allow`（warn 不 deny）。

### False-positive limitation（已知邊界）

PreToolUse 只見 raw command 字串，無完整 shell parse。heredoc／quoted 字串內的字面 cd（如 `git commit -m "改用 cd 取代..."`）可能誤判。warn 不 deny，誤判成本低（僅多一行提示）。本實作以 anchored 比對（connector 限 `^`／`&&`／`;`／`||`／`(`）+ `\bcd\b` 邊界降低常見誤判，不追求 100% 零誤判。

### 設計決策

- 兩偵測獨立而非合併條件：既有原地編輯偵測（regression 驗證通過）與裸 cd 偵測互不干擾，可同時命中並合併輸出。
- 不新增 settings.json 註冊（宿主已註冊，PC-148 避免雙重註冊）。
