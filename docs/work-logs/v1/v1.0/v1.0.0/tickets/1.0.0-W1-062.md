## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 三道修正落地

| 修正 | 實作 | 位置 |
|------|------|------|
| (1) 防 race | Stop 觸發時先查 `dispatch-active.json`，偵測到活躍（未超時）背景代理人即跳過代捕，讓代理人自行 ticket-referenced commit；無活躍代理人才兜底 | `has_active_background_agents` / `main` |
| (2) 訊息富化 | 代捕時掃描 in_progress ticket 的 `where.files`，與變更檔案路徑比對推斷 ticket ID 寫入訊息（`auto(<ids>): ...`）；無法推斷時附檔案摘要（`auto: ... (N files: ...)`） | `infer_ticket_ids` / `build_commit_message` |
| (3) add -A / --no-verify 取捨 | 顯性結論寫入 docstring：兩者皆保留 | module docstring |

### git add -A / --no-verify 顯性結論（acceptance 4）

- **git add -A 保留**：hook 本職為「防工作遺失」安全網，縮小搜刮範圍會讓 untracked 新檔在 worktree 清理時遺失。race 風險已由修正 (1) 從源頭消除（有活躍代理人即不搶先），故 add -A 在「無活躍代理人」前提下安全且必要。
- **--no-verify 保留**：保全性兜底 commit，若 pre-commit lint 失敗阻擋 commit 會導致 worktree 清理時工作遺失——「遺失工作」代價遠高於「一個未過 lint 的 auto-commit」。auto-commit 為暫存性質，後續正式 commit 仍經 lint。

### 原始威脅防護保留（acceptance 3）

「無活躍代理人時兜底代捕」路徑完整保留：worktree 真正終止（無共用代理人）時仍 `git add -A` + commit 保全所有未提交 / untracked 變更，防 CC 清理 worktree 致工作遺失。stale 派發記錄（代理人異常終止）先以 `cleanup_expired` + `_is_dispatch_stale` 清理，避免 ghost entry 永久癱瘓兜底安全網。

### index.lock 重試（不刪 lock）

`_run_git_with_lock_retry` 遇 `index.lock`（主線程並行 git 活動）等待數秒重試（預設 3 次 × 2s），非 lock 錯誤不重試。禁止刪除 lock 檔（測試 `test_does_not_delete_lock_file` 斷言無 `os.remove`/`Path.unlink` 呼叫）。

### Dogfooding 修正（cwd dispatch 來源）

首次 dogfood（worktree cwd）暴露 `find_project_root` 原以 git-common-dir（主 repo）為根，但 PM 寫 `dispatch-active.json` 於 cwd（worktree）的 `.claude/`，導致讀到主 repo 空檔案漏判活躍代理人——即本 ticket 要修的 race 本身。修正為來源優先序：(1) cwd `.claude/dispatch-active.json` 存在則優先；(2) fallback git-common-dir 主 repo root。修正後 dogfood 正確偵測本 session W1-062 派發並決定 SKIP。

### 不在本 ticket 範圍

- settings.json：hook 維持註冊於 Stop event（race 修復為內部 gating，無需改 event）；timeout 15s 涵蓋最壞重試（~8s）。
