### 問題根因（雙層）

**近因（4/20 事件本身，已被 W1-025 修復）**：commit e791a64d（2026-04-20 sync-pull）執行當時 `.claude/.sync-state.json` 在 repo 中不存在（`git show e791a64d:.claude/.sync-state.json` → fatal: not in tree；parent 亦無）。腳本 `should_use_full_overlay()` 在 `read_base_sha()` 回 None 時回傳 True → 走全量 overlay 路徑 → 把上游全部檔案覆蓋到本地，復活了 6ab3d762（W10-049，2026-04-15）刪除的 16 個 root .md 檔。當前腳本（1817 行，W1-025 schema）已具 base-SHA-aware 三方合併（`compute_upstream_delta(base, HEAD)` 只套用上游 delta），現況 `.sync-state.json` 含 `last_synced_base_sha=60132eb9`（= 當前 GitHub 上游 HEAD），故穩態下 delta 模式不會再復活。

**持久根因（尚未修復，本 ticket 真正標的）**：上游 `tarrragon/claude.git` 仍保留全部 16 孤兒檔（逐一 `git ls-files` 驗證皆 [1]）。W10-049 的刪除只發生在 book_overview_v1 本地，從未經 sync-push 傳播到上游。後果：任何 full-overlay fallback（新 clone / 遺失 .sync-state.json / base 不可達 / force-push / GC）都會從上游再次復活全部 16 檔。delta 模式雖防穩態復活，但 fallback 路徑仍是活的復活源。

### 影響範圍

- **book_overview_v1**：root 12/16 孤兒仍在（4 個已被 W8-023 處理：TECH-DEBT-REMINDER-DELIVERY-REPORT.md / code-smell-checklist.md〔已遷移 references/〕/ fix-progress-tracking.md / hook-system-reference.md）。
- **book_overview_app（companion Flutter，獨立 repo book_overview_app.git）**：root 16/16 孤兒全在 → 跨專案污染確認。
- **上游 tarrragon/claude.git**：16/16 孤兒仍在 root；另 7 個 session-start 報告的「未註冊 hook」（changelog-update / cli-failure-help-reminder / commit-handoff / post-commit-fetch / session-context-guard / skill-cli-error-feedback / test-timeout-post）亦同源於上游 hooks/（次要觀察，登記缺口可能另有原因，不混入本 ticket 主標的）。
- **本地上游工作 clone**：/Users/tarragon/Projects/claude（remote=tarrragon/claude.git，HEAD=faa3383d，領先 GitHub remote 60132eb9）為上游清理的操作點。

命中既有 memory「sync-pull full overlay 風險」（W1-003 ANA：.sync-state.json 缺 last_synced_base_sha → 退化 full overlay → 三類副作用：孤兒 hook 複製 / stale 誤刪本專案特有防護 / 可執行 .py 降權）。本 ticket 為該風險的具體爆發案例 + 跨專案 + 上游清理延伸。
