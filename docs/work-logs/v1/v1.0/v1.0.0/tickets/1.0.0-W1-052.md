## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因（PM 前台分析 2026-06-11）

`lib/ticket_generator.py` 約 218-225 行（generate / plan-to-ticket 路徑）配號邏輯：

```python
if wave not in wave_seq_map:
    wave_seq_map[wave] = get_next_seq(version, wave)
else:
    wave_seq_map[wave] += 1
seq = wave_seq_map[wave]
```

此即 W1-051 修復前 bulk_create 的舊 pattern：起始值依賴 get_next_seq（降級時雖有內聚 guard，但 guard 只保證起始值可用），後續 `+= 1` 純遞增無探測——稀疏佔用（起始 051 可用、052 已存在）時後續票經 `generate.py` `_save_tickets` → `get_ticket_path` + `save_ticket` 靜默連鎖覆寫。且 ticket_generator 是真正 per-task 多 wave 的呼叫端（`_assign_wave` 每 task 可不同 wave），wave_seq_map 多 key 並存，撞號面比單 wave 的 bulk_create 更廣。

### 修復基準（仿 W1-051 bulk_create 模式，commit f9cb36a0）

```python
if wave not in wave_seq_map:
    start = get_next_seq(version, wave)
else:
    start = wave_seq_map[wave] + 1
seq = resolve_available_seq(version, wave, start)
wave_seq_map[wave] = seq   # 存解析後值（關鍵：下一項起點 = 實配值 + 1）
```

### 附帶項（acceptance 對應）

1. **warning 措辭收斂**：`ticket_builder.py` get_next_seq 降級 warning「collision guard 已推進至可用序號」對 main-only 撞號屬輕微過度承諾——FS 探測看不到 main-only 檔，措辭應明示「僅保證本地檔案系統可用」。現有 stderr 斷言全鬆耦合（W1-051 coupling Q4 已驗證），改措辭不破壞測試。
2. **.yaml 掃描邊界註解**：get_next_seq 正常路徑掃描集合只認 .md（local glob `*-W{wave}-*.md` + main stems endswith .md），.yaml-only ticket 是「max+1 天然不撞」論證的洞。現實風險 ~0（repo 現存 0 個 .yaml ticket、系統只產 .md），不修碼，在 ticket_builder.py 註解顯性標註此邊界即可。

### 測試風險（WRAP P 預想）

generate 路徑既有測試可能 mock get_next_seq 固定值並斷言 ID 序列；引入 resolve_available_seq 真實探測後，未 mock 它的測試會對檔案系統探測（W1-050 autouse 已隔離到 tmp，通常安全），但若測試預建檔案到 tmp 可能改變配號預期。實作前先跑 generate 相關測試確認 baseline，必要時同步 mock 邊界。

## Solution

### 修復摘要

`lib/ticket_generator.py` generate/plan-to-ticket 路徑配號（約 218-225 行）由純
`wave_seq_map[wave] += 1` 遞增無探測，改為仿 W1-051 bulk_create 模式：

if wave not in wave_seq_map: start = get_next_seq(version, wave)
else: start = wave_seq_map[wave] + 1
seq = resolve_available_seq(version, wave, start)  # 探測 FS 跳過佔用序號
wave_seq_map[wave] = seq                            # 存解析後值（下一項起點 = 實配值 + 1）

resolve_available_seq 經 get_ticket_path(...).exists() 探測，同時覆蓋 .md/.yaml；
對 per-task 多 wave 的每個 wave_seq_map key 獨立生效。resolve_available_seq 從
ticket_builder.py import（既有 W1-051 內聚的共用 helper，不重複實作）。

### 附帶項（acceptance 2、3）

1. warning 措辭收斂（ticket_builder.py get_next_seq 降級分支）：原「collision guard
   已推進至可用序號」改為「已推進至本地檔案系統可用序號 N（僅保證本地 FS 可用，
   無法保證 main-only 票不撞）」。FS 探測看不到 main-only 檔，避免過度承諾。
2. .yaml 掃描邊界註解（ticket_builder.py get_next_seq 正常路徑掃描前）：顯性標註
   正常路徑「max+1 天然不撞」論證只認 .md，.yaml-only ticket 是邊界洞；現實風險 ~0
   不修碼，降級分支的 resolve_available_seq 經 get_ticket_path 探測為 .yaml 撞號實際防線。
