### 實作規劃（派發 thyme-python-developer 時依此執行）

**整合點**：`.claude/hooks/file-size-guardian-hook.py`（既有 SessionStart hook，已輸出單檔超標掃描）。在既有輸出尾端附加「Auto-load 預算」區塊，不另建新 hook（避免 SessionStart hook 數量膨脹）。

**量測邏輯**：

1. 集合解析：`CLAUDE.md` + glob `.claude/rules/core/*.md` + 解析 CLAUDE.md 內 `@` 引用（regex 匹配行首與行內 `@path` 形式，遞迴一層即可——目前 @ 鏈深度為 1）
2. Token 估算：合計字元數 / 3（中文為主的保守近似；與 /context 實測值校準一次後可調係數，校準結果寫入常數註解）
3. 預算常數：`AUTO_LOAD_BUDGET_TOKENS = 45_000`（集中常數管理，含來源註解指向本 ticket）
4. 輸出：低於預算 → 一行「Auto-load 集合 ~Nk / 45k」；超標 → WARNING + top 3 體量檔 + 與上次量測差值（state 存 `.claude/hook-logs/auto-load-budget-state.json`）
5. 失敗安全：量測異常時 stderr 警告 + 不阻擋 session（quality-baseline 規則 4 雙通道）

**測試**（`.claude/hooks/tests/`，pytest）：

- @ 引用解析正確（含行內引用、不存在路徑 graceful skip）
- 預算內 / 超標兩路徑輸出斷言
- state 檔首次不存在的初始化路徑
- 跑 `npm run test:hooks` 全綠

**最終驗收（acceptance 3 對應）**：.1-.6 全部完成後，開新 session 執行 `/context`，Memory files <= 45k tokens，數字記錄於本 ticket Test Results。
