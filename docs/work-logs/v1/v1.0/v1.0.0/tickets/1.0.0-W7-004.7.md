### 實作摘要

擴充 `.claude/hooks/file-size-guardian-hook.py`（既有 SessionStart hook，不另建新 hook），在既有單檔超標掃描輸出尾端附加「Auto-load 預算」區塊：

| 元件 | 實作 |
|------|------|
| 集合解析 | `collect_auto_load_files()`：CLAUDE.md + glob `.claude/rules/core/*.md` + `resolve_at_references()` 解析 CLAUDE.md 內 @ 引用（regex 匹配行首與行內 `@path.md`，遞迴一層 `AT_REF_MAX_DEPTH=1`，不存在路徑 graceful skip，set 去重） |
| Token 估算 | `measure_auto_load_budget()`：合計字元數 // 3（`CHARS_PER_TOKEN = 3`，中文為主保守近似，常數註解標明可依 /context 實測校準） |
| 預算常數 | `AUTO_LOAD_BUDGET_TOKENS = 45_000`（含來源註解指向本 ticket） |
| 輸出 | 低於預算 → 一行「[Auto-load 預算] 集合 ~Nk / 45k tokens」；超標 → WARNING + top 3 體量檔 + 與上次量測差值 |
| State | `.claude/hook-logs/auto-load-budget-state.json`（gitignored），首次不存在回傳 None 不顯示差值 |
| 失敗安全 | `report_auto_load_budget()` 捕捉所有量測異常 → stderr 警告 + logger.error 日誌，不阻擋 session（quality-baseline 規則 4 雙通道） |

既有掃描邏輯抽取為 `_report_oversized_files()`，main() 改為兩段式（掃描 + 預算量測），行為不變。

- 最大函式長度：`resolve_at_references` 約 20 行
- 最大巢狀深度：3 層

### 校準偏差發現（spawn follow-up）

hook 量測集合（CLAUDE.md + rules/core + @ 引用，不含 user-level ~/.claude/CLAUDE.md 1.1k 與 MEMORY.md 2k）對應的 /context 實測合計約 38.9k tokens，hook 自估僅 16.9k，偏差約 2.3 倍。根因：`CHARS_PER_TOKEN = 3` 假設每 token 3 字元，實測本集合（繁中為主混英文）約 1.3 字元/token。低估使預算守門靈敏度不足（實際已用 86% 預算，hook 顯示僅 38%）。係數校準依 Context Bundle 預留條款轉 follow-up ticket 執行（見 spawned_tickets）。
