## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

`validate_create_checklist`（lib/ticket_builder.py）對 `why` / `how_strategy` 的缺失判定，原本只檢查是否等於 `DEFAULT_UNDEFINED_VALUE`（「待定義」）：

```python
if ticket_type != "DOC" and config.get("why") == DEFAULT_UNDEFINED_VALUE:
    missing.append("why")
if config.get("how_strategy") == DEFAULT_UNDEFINED_VALUE:
    missing.append("how_strategy")
```

空字串 `""` 不等於 `"待定義"`，故漏判。`bulk_create._create_ticket_config` 預設 `why=template_defaults.get("why", "")`、`how_strategy=template_defaults.get("how_strategy", "")`——當模板未提供時即為 `""`，這兩欄殘缺不觸發 warning。W1-027 修復執法一致性時刻意保持既有判定語意（避免在「修一致性」時偷改規則），缺口另案（本 ticket）修補。

### 影響範圍

- `.claude/skills/ticket/ticket_system/lib/ticket_builder.py`（`validate_create_checklist` 兩處判定）
- `.claude/skills/ticket/ticket_system/tests/test_checklist_enforcement_consistency.py`（補空字串案例）
- 連帶（同函式）：create 阻擋路徑、bulk_create / generate warning 路徑判定同步修正

### 相關 Error Pattern

- source W1-027 Phase 4 評估發現（同函式三路徑共用，改一處三路徑同步）。
- 對齊 ARCH-020 精神：單一驗證函式為 SSOT，避免散落漂移。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復：判定改為 falsy 或等於待定義

`validate_create_checklist` 兩處判定：

```python
# why 非空且非「待定義」（DOC 類型豁免）
if ticket_type != "DOC":
    why_value = config.get("why")
    if not why_value or why_value == DEFAULT_UNDEFINED_VALUE:
        missing.append("why")

# how_strategy 非空且非「待定義」
how_strategy_value = config.get("how_strategy")
if not how_strategy_value or how_strategy_value == DEFAULT_UNDEFINED_VALUE:
    missing.append("how_strategy")
```

`not value` 涵蓋 `""` / `None`；`== DEFAULT_UNDEFINED_VALUE` 涵蓋「待定義」。兩者聯集即「未實質填寫」。

### 設計取捨

| 維度 | 決定 | 理由 |
|------|------|------|
| 改判定 vs 改各入口預設值 | 改判定（單點） | 同函式三路徑共用，改一處全涵蓋（DRY） |
| create 阻擋是否受影響 | 擴大涵蓋，語意一致 | create 預設 why=待定義（非 ""），現有測試不受影響；空字串本就應視為未填 |
| DOC 豁免 | 維持 | DOC 不檢查 why（既有設計），空字串對 DOC 仍豁免 |
