## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

現行 sync-pull target 永遠 = remote/main HEAD；push 不打 tag；consumer 無法 pin 特定版本。框架壞變更瞬時傳染全部 consumer，爆炸半徑無法收斂。

### 影響範圍

- .claude/scripts/sync-claude-push.py：push 流程加 git tag（每版打 tag）
- .claude/scripts/sync-claude-pull.py：加 pinned_version 讀取 + target ref 解析（tag vs HEAD）+ --bump 子命令

### 相關 Error Pattern

reference-stability-rules 規則 8（框架碼禁嵌 project-local ticket ID）；向後相容硬約束（不破既有 consumer）。

### WRAP 三問

- W（Widen）：方案 A tag-based pin（採；W1-088 設計定案，tag 給人類可讀 ref + remote 可見）；方案 B 直接在 .sync-state.json 存 SHA pin 無 tag（否決，無 remote 可見性）。
- A（機會成本）：擠壓其他 W1 ticket，但屬防禦縱深高價值。
- P（最可能失敗）：破壞既有 consumer 向後相容。防護：latest/未設路徑必須解析為 HEAD 與現行完全一致，由測試 (d) 覆蓋。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### push（sync-claude-push.py）

新增 create_and_push_version_tag(repo_dir, version)：push main 成功後對該 commit 打 annotated tag v<version> 並 push origin。冪等（tag 已存在不重建，僅補 push remote）；tag push 失敗僅警告不中止主流程（main 已成功推送）。於 main() push 成功後、寫 .sync-state.json 前呼叫。

### pull（sync-claude-pull.py）

- read_pinned_version(claude_dir)：讀 .sync-state.json pinned_version 欄；無檔/無欄/解析失敗回 None。
- resolve_target_ref(temp_dir, pinned_version)：None/"latest" → 上游 HEAD（向後相容）；特定版 → 正規化 v<X.Y.Z> 解析 tag commit；tag 不存在 fail-loud（sys.exit + stderr，規則 4）。
- update_pinned_version(claude_dir, version)：--bump 更新 pinned_version（None=latest）；保留既有欄位；無檔則建。
- _sync_with_backup：resolve 目標後，target != HEAD 時 git checkout --detach target_sha，使後續 diff/show/rev-parse HEAD 全對齊 pinned target。latest/未設不 checkout，行為與現行完全一致。
- main：--bump [版本] 先 update_pinned_version 再走正常同步。

### 向後相容

latest/未設 pin → resolve_target_ref 回 HEAD → 不 checkout → 與現行 HEAD 行為完全一致，不破既有 consumer（測試 d 覆蓋）。

### 框架約束

註解全用通用描述，無嵌 project-local ticket ID（reference-stability-rules 規則 8）。
