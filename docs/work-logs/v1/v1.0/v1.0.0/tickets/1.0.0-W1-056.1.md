## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

CC 2.1.170 發布 Fable 5（Mythos-class，能力聲稱高於 Opus），CC 2.1.166 新增 `fallbackModel` 設定（最多 3 個 fallback model，主 model API 過載時依序嘗試）。兩者共同影響代理人 model 配置策略：

1. **Fable 5 衝擊**：現行 agent frontmatter 以 Opus 為最高階模型，16/31 agent 硬編碼 `opus`、2 個硬編碼 `claude-opus-4-6[1m]`、9 個 `inherit`（跟隨 session，目前 session 為 opus 1m）。Fable 5 發布後 Opus 不再是能力天花板，策略需重評。
2. **fallbackModel 韌性缺口**：背景派發遇 API 過載時無 fallback，當前唯一恢復路徑是 agent-failure-sop 人工重派。

### 影響範圍

| settings.json | 1 | 無 fallbackModel 欄位（待新增） |
| agent-model-selection.md | 1 | model 決策流程文件需更新 |
| agent-dispatch-template.md | 1 | 派發時 model 參數指引可能需更新 |

### 實驗發現

現行策略有三個結構性問題：

1. **model 寫法不一致**：16 個用 `opus`（alias，不保證 1m），9 個用 `inherit`，2 個用硬編碼 `claude-opus-4-6[1m]`。W8-031 已統一 7 個過載風險 agent 為 inherit，但 16 個核心實作 agent 仍為 `opus` alias。
2. **無 fallback 韌性**：API 過載時無自動降級路徑。
3. **Fable 5 未納入策略**：agent-model-selection.md 決策流程仍以 Opus 為能力天花板。

### 一、Fable 5 重評實驗設計（沿用 W17-065 方法論）

#### 實驗前提

W17-065 方法論核心：每代理人 6 ticket（3 基準 model + 3 實驗 model），多視角分析比對。判準：錯字/風格違規可容忍（二次審查能修），邏輯/判斷/執行能力下降必回滾。

#### 實驗範圍（建議精簡版，非 W17-065 全量）

W17-065 原設計 16 代理人 × 7 ticket = 112 張。Fable 5 重評可精簡：

| 策略 | 代理人數 | ticket 數 | 理由 |
|------|---------|----------|------|
| 全量（W17-065 原版） | 16 | 112 | 完整覆蓋，成本極高 |
| **精簡版（建議）** | **4** | **28** | 選 4 個代表性角色覆蓋主要任務型態 |
| 最小可行 | 2 | 14 | 僅驗證寫碼 + 分析兩極端 |

**精簡版 4 個代表性代理人**：

| 代理人 | 任務型態 | 選擇理由 |
|--------|---------|---------|
| thyme-extension-engineer | 寫碼（JS/Chrome Extension） | 本專案核心實作角色，最高頻派發 |
| sage-test-architect | 測試設計 | 需精確的邏輯推理與規格理解 |
| linux | 多視角品質審查 | 深度分析與判斷力敏感 |
| lavender-interface-designer | 介面/規格設計 | 設計類任務對創意+嚴謹度雙需求 |

#### 實驗結構

每代理人 7 ticket：

| ticket | model | 用途 |
|--------|-------|------|
| X.1 ~ X.3 | 現行 model（opus/inherit） | 基準組 |
| X.4 ~ X.6 | fable-5（或 claude-fable-5[1m]） | 實驗組 |
| X.7 | 多視角分析 ANA | bay + linux 對比兩組產出 |

#### 實驗任務設計原則

| 原則 | 說明 |
|------|------|
| 可複現 | 6 個 ticket 使用完全相同的 prompt 和 Context Bundle |
| 貼近真實 | 從近期已完成 ticket 中選類似任務範本（非人造題） |
| 覆蓋判斷力 | 任務需含至少一個需判斷的分支（非純機械操作） |
| 可量化 | 產出有明確驗收條件（測試通過率、lint 零違規等） |

#### 實驗結果判定

| 維度 | Fable 5 可採用門檻 | 必回滾門檻 |
|------|-------------------|-----------|
| 邏輯正確性 | 實驗組 >= 基準組 | 實驗組有基準組無的邏輯錯誤 |
| 規格遵循 | 實驗組 >= 基準組 | acceptance 通過率下降 |

### 二、fallbackModel 採用建議

#### 設定語意（CC 2.1.166 release notes）

```json
{
  "fallbackModel": ["model-a", "model-b", "model-c"]
}
```

主 model API 過載（429/503）時，依陣列順序嘗試 fallback model。最多 3 個。

#### 建議設定值

**方案 A（建議採用）**：

```json
{
  "fallbackModel": ["claude-opus-4-6[1m]", "claude-sonnet-4-5-20250514[1m]"]
}
```

| 位置 | model | 理由 |
|------|-------|------|
| fallback 1 | claude-opus-4-6[1m] | 若 session 主 model 升為 Fable 5，Opus 作第一 fallback 能力損失最小 |
| fallback 2 | claude-sonnet-4-5-20250514[1m] | Sonnet 4.5 帶 1m 後綴，防過載降級時 context 不觸頂；最差情況仍能完成多數任務 |

**方案 B（若 session 維持 Opus 不升 Fable 5）**：

```json
{
  "fallbackModel": ["claude-sonnet-4-5-20250514[1m]"]
}
```

Opus 過載時降為 Sonnet 4.5 1m。單層 fallback，最簡設定。

**方案 C（最激進）**：

```json
{
  "fallbackModel": ["claude-fable-5[1m]", "claude-opus-4-6[1m]", "claude-sonnet-4-5-20250514[1m]"]
}
```

三層 fallback，但需確認 Fable 5 model ID 格式。此方案假設 session 主 model 為 Fable 5，三層 fallback 覆蓋所有能力等級。

#### fallbackModel 未驗證項

| 項目 | 風險 | 建議 |
|------|------|------|
| `inherit` agent 是否受 fallbackModel 影響 | 若 inherit 不觸發 fallback，9 個 inherit agent 無 fallback 保護 | 需實驗確認 |
| Fable 5 model ID 格式 | `claude-fable-5` / `fable-5` / `claude-fable-5[1m]` 未確認 | 執行 `claude /model` 查看可用 model 清單 |
| fallback 觸發條件 | 僅 429/503 或含其他錯誤碼 | 查 CC 原始碼或官方文件 |

### 三、model 策略綜合建議（交用戶決策）

#### 選項 A：先設 fallbackModel，Fable 5 待實驗

1. 立即：在 settings.json 加入 `"fallbackModel": ["claude-sonnet-4-5-20250514[1m]"]`（方案 B）
2. 近期：建 Fable 5 精簡實驗 ticket（32 張），依實驗結果決定是否升級
3. 同步：統一 16 個 `opus` 硬編碼 agent 為 `inherit`（與 W8-031 決策一致，減少維護成本）

**優點**：零風險立即提升韌性；Fable 5 有實證再採用。
**缺點**：延遲享受 Fable 5 潛在能力提升。

#### 選項 B：直接升級 Fable 5 + fallbackModel

1. 立即：PM session 切換至 Fable 5，settings.json 加入 `"fallbackModel": ["claude-opus-4-6[1m]", "claude-sonnet-4-5-20250514[1m]"]`（方案 A）
2. 統一所有 agent 為 `inherit`（跟隨 session 自動取 Fable 5）
3. 日常使用中觀察，2 週後回顧品質

**優點**：立即享受 Fable 5 能力提升；fallback 保底。
**缺點**：未經實驗驗證，品質風險未量化。

#### 選項 C：僅設 fallbackModel，Fable 5 不採用

1. 立即：settings.json 加入 `"fallbackModel": ["claude-sonnet-4-5-20250514[1m]"]`
2. 維持 Opus 為主 model，不評估 Fable 5

**優點**：最小變更。
**缺點**：放棄潛在能力提升。
