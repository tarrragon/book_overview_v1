## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

CC 2.1.170 發布 Fable 5（Mythos-class，能力聲稱高於 Opus），CC 2.1.166 新增 `fallbackModel` 設定（最多 3 個 fallback model，主 model API 過載時依序嘗試）。兩者共同影響代理人 model 配置策略：

1. **Fable 5 衝擊**：現行 agent frontmatter 以 Opus 為最高階模型，16/31 agent 硬編碼 `opus`、2 個硬編碼 `claude-opus-4-6[1m]`、9 個 `inherit`（跟隨 session，目前 session 為 opus 1m）。Fable 5 發布後 Opus 不再是能力天花板，策略需重評。
2. **fallbackModel 韌性缺口**：背景派發遇 API 過載時無 fallback，當前唯一恢復路徑是 agent-failure-sop 人工重派。

### 影響範圍

| 範圍 | 檔案數 | 說明 |
|------|--------|------|
| agent frontmatter model 欄位 | 31 個 .md | 全部 agent 定義（含 3 DEPRECATED/範本） |
| settings.json | 1 | 無 fallbackModel 欄位（待新增） |
| agent-model-selection.md | 1 | model 決策流程文件需更新 |
| agent-dispatch-template.md | 1 | 派發時 model 參數指引可能需更新 |

### 相關 Error Pattern

無直接相關 PC。間接關聯：
- agent-failure-sop（背景派發失敗處理，fallbackModel 可降低觸發頻率）
- W8-031（sonnet 1m 訂閱停用，建立現行 inherit 策略的前例）

## 重現實驗結果
### 實驗方法

本 ANA 為評估型分析，非 bug 重現。「重現實驗」對應為：盤點現行 model 配置事實 + 確認 Fable 5 / fallbackModel 的平台能力邊界。

### 實驗執行

**盤點結果**（已驗證事實）：

| # | 事實 | 驗證方式 |
|---|------|---------|
| 1 | 31 個 agent 定義，model 分佈：opus(16) / inherit(9) / claude-opus-4-6[1m](2) / haiku(3) / sonnet(1) | `grep -h 'model:' .claude/agents/*.md | sort | uniq -c` |
| 2 | settings.json 無 `fallbackModel` 欄位 | `grep fallbackModel .claude/settings.json`（零命中） |
| 3 | agent-model-selection.md 記載 W8-031 決策：過載風險 agent 用 inherit，核心審計用硬編碼 1m | Read 全文 |
| 4 | W17-065 實驗方法論：每代理人 6 ticket（3 基準 + 3 實驗）+ 多視角分析 | Read ticket 全文 |
| 5 | PM 主線程 session model = claude-opus-4-6[1m]（本次對話自報） | 系統 banner 確認 |

**仍未驗證的假設**：

| # | 假設 | 驗證所需 |
|---|------|---------|
| A | Fable 5 在本專案任務型態下確實優於 Opus | 需執行 W17-065 方法論實驗（本 ANA 僅設計不執行） |
| B | `fallbackModel` 對 agent frontmatter model 的行為（inherit agent 是否受 fallback 影響） | 需實驗或查 CC 原始碼確認 |
| C | `opus` 硬編碼在當前訂閱下實際取得 1M 或 200K | 需 `claude config` 或實驗確認 |

### 實驗發現

現行策略有三個結構性問題：

1. **model 寫法不一致**：16 個用 `opus`（alias，不保證 1m），9 個用 `inherit`，2 個用硬編碼 `claude-opus-4-6[1m]`。W8-031 已統一 7 個過載風險 agent 為 inherit，但 16 個核心實作 agent 仍為 `opus` alias。
2. **無 fallback 韌性**：API 過載時無自動降級路徑。
3. **Fable 5 未納入策略**：agent-model-selection.md 決策流程仍以 Opus 為能力天花板。
