## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

ana_spawn_consistency_checker 原有雙偵測策略皆要求嚴格樣式：
- row-per-spawn 正則 `_SPAWN_ROW_PATTERN` 要求表格行同行含 `\bP[0-3]\b`
- heading-based 正則 `_SPAWN_HEADING_PATTERN` 要求 H3 標題行同行含 IMP/DOC/ANA

W1-024 的「Spawn 落地確認」表格為 `| 項目 | 形態 | 狀態 |`：
- 形態欄為 `IMP（child）` / `IMP`（type 帶註記、無 P0-P3 欄）→ row-per-spawn N=0
- H3 為 `### Spawn 落地確認`（含 Spawn 但無 IMP/DOC/ANA 同行）→ heading-based N=0

雙策略 N=0 → checker 直接通過，acceptance-gate Step 2.5.2 對該票實質失效。
另「無需 spawn」系列豁免語彙不在 `_EXEMPTION_MARKERS` 清單，合法無 spawn 的 ANA 與漏建無法區分。

### 影響範圍

- .claude/hooks/acceptance_checkers/ana_spawn_consistency_checker.py（偵測邏輯）
- .claude/hooks/tests/test_ana_spawn_consistency_checker.py（回歸 fixture）

### 相關 Error Pattern

quality-baseline 規則 5（所有發現必須追蹤）；ANA spawn 落地確認 hook 強制層（W17-167/W17-168）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

新增第三偵測策略 type-annotated row（容忍 type 欄附註與無 P0-P3 欄），並擴充豁免語彙。

1. 豁免語彙擴充 `_EXEMPTION_MARKERS`：加入「無需 spawn / 無需spawn / 不需 spawn / 不需spawn / no spawn required」。

2. 新增 spawn 區段感知偵測：
   - `_SPAWN_SECTION_HEADING_PATTERN`：H3 含 Spawn 關鍵字（不要求同行含 IMP/DOC/ANA），涵蓋 `### Spawn 落地確認`
   - `_SPAWN_SECTION_HEADING_ZH`：純中文譯名（落地確認 / spawn 規劃 / 派生 / 衍生 ticket）
   - `_TYPE_ANNOTATED_CELL_PATTERN`：表格 cell 為 IMP/DOC/ANA，容忍 `IMP（child）` 註記
   - `_iter_spawn_section_bodies` + `_count_type_annotated_rows`：僅在 spawn 區段內計數 type-annotated 行，排除表頭分隔行

3. `_count_spawn_planning` 整合為三策略取 max（row-per-spawn / heading-based / type-annotated）。

### False positive 防護

type-annotated 偵測限縮在 spawn 區段（H3 含 Spawn 關鍵字或中文譯名）內。
一般說明表格（如風險評估表含 IMP/DOC 字樣但不在 Spawn H3 下）不被計數，
由 test_type_annotated_row_without_spawn_heading_not_triggered 守護。

### 設計決策

採「策略疊加取 max」而非改寫既有正則，向後相容既有 18 - 5 = 13 個既有測試零回歸，
新策略僅補強既有策略漏判的樣式（W1-024 變體），不放大既有命中表格的計數。
