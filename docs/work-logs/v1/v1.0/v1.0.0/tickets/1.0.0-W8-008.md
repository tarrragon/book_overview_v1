### 核心設計決策

1. **二軸定位法**：受眾軸（所有角色/PM/全體代理人/單一代理人/動作觸發者/僅本專案）先縮小候選，形態軸（禁令/流程/偏好/方法/理念/參考/案例）定位載體
2. **代理人定義權威判定**（用戶需求核心）：「人格與授權」（身份、三區塊、命名/技術手法/語氣偏好、分工路由）vs「可重複流程」（→ skill）；技術選型劃界——專案級選型在 CLAUDE.md、代理人層為手法傾向（依 framework-asset-separation §1，消除表面衝突）
3. **與既有規範關係**：定位為頂層地圖，framework-meta（三分法細分）/ framework-asset-separation（框架vs專案）/ auto-load-stub-conventions（stub 形態）/ agent-definition-standard（三區塊）均為下游路由，零取代零矛盾
