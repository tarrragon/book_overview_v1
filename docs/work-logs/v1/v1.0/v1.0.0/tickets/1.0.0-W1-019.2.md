## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W1-019 ANA 決策採 Model 1 來源前綴：新 error-pattern ID 一律 `<CAT>-<PROJ>-NNN`
（如 `PC-V1-001`）。但既有解析 error-pattern ID 的 regex 只認 flat `<CAT>-\d+`，
無法匹配含前綴段的新格式。若不先拓寬 regex，任何前綴 PC 一旦建立即被 hook/CLI
解析失敗（Solution §202 順序約束：regex 拓寬須在建立任何前綴 PC 前完成）。

### 影響範圍

掃描方法：`rg -g '*.py' 're\.compile\(.*<CAT>[-_].*\\d'` + 人工濾除文件引用雜訊
（多數 `PC-NNN` 出現於 docstring / log 訊息，非 regex 解析點）。

真實 category-ID 解析點 = **3 檔 3 regex**（接手者請獨立重驗，PC-162）：

| # | 檔案:行 | 變數 | 現值 | 處置 |
|---|---------|------|------|------|
| 1 | `.claude/scripts/sync-claude-pull.py:985` | `_PC_FILENAME_RE` | `^PC-(\d+)-(.+)\.md$` | 拓寬為 `<CAT>-([A-Z0-9]+-)?\d+` |
| 2 | `.claude/scripts/sync-claude-pull.py:987` | `_PROVENANCE_UPSTREAM_RE` | `編號為\s*PC-(\d+)` | 同步拓寬 |
| 3 | `.claude/hooks/acceptance_checkers/error_pattern_attribution.py:38` | `_PATTERN_ID_RE` | `\b(?:PC\|IMP\|ARCH\|ANA\|REF\|DOC\|CQ\|PROC\|TEST)-\d+\b` | 已全 category，補前綴段 |

排除（E5 Scope 1.1，凍結 base 協議字串）：
- `.claude/hooks/phase4-decision-enforcement-hook.py:85` `EXEMPT_MARKER_STRIP`
  （`PC-093-exempt` 固定協議字串，非通用 ID 解析，永不重編）

SSOT 收斂（E2 linux F3）：3 點 regex 收斂為單一 `PATTERN_ID_RE` util，散佈點改 import。
擺放挑戰：#1/#2 在 `scripts/`、#3 在 `hooks/acceptance_checkers/`，跨目錄 import 可達性
須於 phase 3b 驗證（`hooks/lib/` 為現成共用層）。

向後相容：拓寬 regex `<CAT>-([A-Z0-9]+-)?\d+` 中 `([A-Z0-9]+-)?` 為 optional group，
flat 既有格式（`PC-099`）仍匹配，前綴格式（`PC-V1-001`）亦匹配。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 範圍收斂決策（用戶 AUQ 確認）

初判「3 regex 全拓寬」於深入 capture group 用法後推翻（PC-162 描述是草稿）。
最終範圍 = **只拓寬 #3（識別用途）+ #1/#2 加確認註解（刻意只認 flat）**：

| # | 檔案 | 處置 | 理由 |
|---|------|------|------|
| 1 | `sync-claude-pull.py:_PC_FILENAME_RE` | 加註解，**不拓寬** | flat 整數撞號子系統（`int(group(1))`）；前綴格式天生不撞號，現值 regex 已正確排除前綴檔（回 None）。拓寬會 `int("V1-")` → ValueError |
| 2 | `sync-claude-pull.py:_PROVENANCE_UPSTREAM_RE` | 加註解，**不拓寬** | 溯源註記服務 flat 重編機制，前綴格式不重編 |
| 3 | `error_pattern_attribution.py:_PATTERN_ID_RE` | **拓寬** | 純「識別任意 ID 字串」歸屬用途，前綴 ID 須被識別 |

### 變更檔案

| 檔案 | 變更 |
|------|------|
| `.claude/hooks/lib/pattern_id.py` | **新建** SSOT util（`PATTERN_ID_RE` + `extract_pattern_id`），格式 `<CAT>-([A-Z0-9]+-)?\d+`，全 category + optional 前綴段（E2 linux F3 收斂單一權威） |
| `.claude/hooks/acceptance_checkers/error_pattern_attribution.py` | `_PATTERN_ID_RE` 改為 `from lib.pattern_id import PATTERN_ID_RE`（散佈點改 import SSOT），`_extract_pc_id` 等下游不變（alias 保留） |
| `.claude/scripts/sync-claude-pull.py` | #1/#2 加「刻意只認 flat」確認註解（防未來誤拓寬） |
| `.claude/hooks/tests/test_pattern_id.py` | **新建** 11 測試（flat / 前綴 / 全 category / 回溯正確性 / word-boundary / 大小寫正規化） |
| `.claude/scripts/tests/test_sync_claude_pull_pc_collision.py` | 加 `test_parse_pc_filename_prefixed_excluded_from_collision`（驗證前綴檔排除於 collision） |

### 設計依據

- **向後相容**：`(?:[A-Z0-9]+-)?` optional group，flat（`PC-099`）與前綴（`PC-V1-001`）皆匹配。
- **回溯正確性**：前綴段字元集含數字（專案代碼如 `V1`/`C2C` 含數字），flat 編號靠 regex 回溯不被誤吞，由 `test_flat_001_not_consumed_as_prefix` 鎖定。
- **word boundary 保留**：`\b` 邊界防 substring 誤判（PC-113 / PC-138 / PC-144 家族）。
- **用途邊界**：識別用途用 SSOT（含前綴）；flat collision 子系統刻意不用 SSOT（前綴不參與撞號）——兩用途 regex 本就不同，不強行 SSOT 化。

### 不在本 ticket 範圍

- `/error-pattern add` 前綴自動分配 + 拒絕 flat negative gate → W1-019.3
- flat base remediation（A+B+D）→ W1-019.4
- 跨專案 detect 腳本 + canonical 升格 → W1-019.5

### Phase 4 評估報告

- **重構需要**: 否
- **技術債務**: 無
- **程式碼品質**: A

**結論**：本 ticket 本身即為重構性質——將散佈的 ID regex 收斂為單一 SSOT util
（E2 linux F3），降低未來「再獵散佈點」的認知負擔。新建 `lib/pattern_id.py` 單一職責、
無重複、命名自說明（`PATTERN_ID_RE` / `extract_pattern_id`），認知負擔指數低。
#1/#2 的確認註解將「刻意不拓寬」的設計意圖顯性化，防止未來誤改（負向技術債預防）。
無需額外重構。
