### 設計規格

**目標**：建立 `skill-sync` Python CLI 工具（uv tool install），讓任何專案（有框架或無框架）都能 push/pull 單一 skill 到 tarrragon/claude-skills repo。

**三個子命令**：

| 命令 | 行為 |
|------|------|
| `skill-sync pull <name>` | 從 skills repo clone 單一 skill 到 `.claude/skills/<name>/` |
| `skill-sync push <name>` | 將本地 `.claude/skills/<name>/` 推送到 skills repo |
| `skill-sync list` | 列出 skills repo 所有可用 skills（名稱 + description 首行） |

**實作要點**：

1. **目錄結構**：在 `.claude/skills/skill-sync/` 下建立（與 ticket/doc 等 CLI 工具同層級）
   - `pyproject.toml`（uv tool 入口 `skill-sync`）
   - `skill_sync/` Python package
   - `SKILL.md`（skill 定義）

2. **Pull 實作**：
   - `git clone --depth 1 --filter=blob:none --sparse <repo>` 到暫存目錄
   - `git sparse-checkout set <name>/`
   - 複製到 `.claude/skills/<name>/`
   - 清理暫存目錄
   - 排除 `project-integration/` 子目錄（類型 E 內容不拉）

3. **Push 實作**：
   - clone skills repo 到暫存目錄
   - 複製本地 `.claude/skills/<name>/` 到暫存目錄（排除 project-integration/）
   - commit + push
   - 清理暫存目錄

4. **List 實作**：
   - `git ls-tree --name-only HEAD` 或 GitHub API 列出頂層目錄
   - 對每個 skill 讀取 SKILL.md 第一行 description

5. **配置**：
   - 預設 repo URL：`https://github.com/tarrragon/claude-skills.git`
   - 可透過環境變數 `SKILL_SYNC_REPO` 覆蓋
   - 預設目標目錄：`.claude/skills/`（相對 cwd）

6. **不依賴框架**：不 import sync_exclude_manifest 或其他框架模組
