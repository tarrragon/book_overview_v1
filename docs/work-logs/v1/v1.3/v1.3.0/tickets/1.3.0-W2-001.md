## Context Bundle

### 實作位置

`.claude/skills/version-release/scripts/version_release.py` 的 `update_docs()` 或 `release()` 函式。

### 現有邏輯

`release()` 流程：preflight check → update-docs（CHANGELOG 更新）→ git tag → push。`update-docs` 步驟目前不涉及 todolist.yaml。

### 需新增邏輯

在 release 成功（tag 建立）後：
1. 讀取 `docs/todolist.yaml`
2. 找到 `version: {current_version}` 的條目
3. 將 `status: active` 改為 `status: completed`
4. 寫回 todolist.yaml（保留 YAML 格式）
5. dry-run 時只顯示「todolist status 將改為 completed」不寫入

### todolist.yaml 格式參考

```yaml
versions:
  - version: "1.3.0"
    status: active
    description: "..."
```

### Commit Policy

同 release commit 或獨立 `chore(version-release): todolist status auto-update (W2-001)`
