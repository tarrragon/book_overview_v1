## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

兩個 lib 目錄共存（`.claude/hooks/lib/` 23 模組 + `.claude/lib/` 8 模組），造成 import 路徑歧義（IMP-V1-003 根因）。

### 現有兩種 import 模式

| 模式 | sys.path 指向 | import 語法 | 使用者數量 |
|------|-------------|-------------|-----------|
| A（多數） | `.claude/hooks/`（parent） | `from lib.xxx import yyy` | ~46 個 hook |
| B（少數） | `.claude/lib/`（parent.parent/"lib"） | `from xxx import yyy`（無 lib. 前綴） | ~7 個 hook |

### 名稱衝突

僅 `__init__.py` 重疊，無功能模組衝突。

### 影響範圍

- `.claude/hooks/*.py`：~70 個含 sys.path 設定
- `.claude/skills/*/hooks/*.py`：~12 個 skill hook
- `.claude/hooks/lib/` 子目錄：`parsers/`（3 模組）、`ticket_quality/`（4 模組）
- `.claude/lib/tests/`：6 個既有測試需確認不受影響

### 統一策略

合併後統一為：所有 hook 的 sys.path 指向 `.claude/`，import 統一用 `from lib.xxx import yyy`。

| hook 位置 | sys.path 寫法 |
|-----------|--------------|
| `.claude/hooks/*.py` | `Path(__file__).parent.parent` |
| `.claude/skills/*/hooks/*.py` | `Path(__file__).resolve().parents[3]` |

模式 B hook（已指向 `.claude/lib/`）需改為指向 `.claude/` 並加上 `lib.` 前綴。

### 相關 Error Pattern

IMP-V1-003（hook 搬移後 sys.path 指向錯誤 lib 目錄）
