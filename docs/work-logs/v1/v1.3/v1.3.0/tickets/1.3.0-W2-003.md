## Context Bundle

### 實作位置

TDD 紅綠燈計數邏輯散佈在：
- `.claude/skills/tdd/` 相關腳本
- PM 或代理人手動 grep test output 計數

### 需新增邏輯

建立統一的 test result parser：
1. pytest：`pytest --json-report` 或 `pytest --tb=no -q` 的結構化輸出
2. jest：`npx jest --json` 的結構化輸出
3. parser 讀取 JSON → 提取 passed/failed/skipped 三數字
4. 替換現有 grep 計數方式

### 注意事項

- 本專案用 jest（`npm run test:hooks` 底層是 pytest，但產品測試是 jest）
- pytest-json-report 不在 hooks/pyproject.toml 預設 deps，需評估是否新增
- 可能需新增一個 `.claude/lib/test_result_parser.py` 共用模組
