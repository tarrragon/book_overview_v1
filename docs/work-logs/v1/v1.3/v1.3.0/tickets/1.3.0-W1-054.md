### 設計規格

**目標**：在 sync-push/pull 腳本中支援 `.claude/sync-skills.yaml`，讓 consumer 專案可選擇性同步 skills。

**sync-skills.yaml schema**：

```yaml
# .claude/sync-skills.yaml
mode: all          # all | select | none
include:           # mode: select 時生效
  - wrap-decision
  - tdd
  - ticket
private:           # 任何 mode 下都排除推送
  - my-company-tool
```

**行為矩陣**：

| mode | Pull | Push |
|------|------|------|
| all（預設，無檔案時） | 拉全部 skills | 推全部（排除 private） |
| select | 只拉 include 清單 | 只推 include 清單（排除 private） |
| none | 不拉任何 skill | 不推任何 skill |

**private 清單**：不論 mode，private 清單的 skills 永遠不被 push 推送，且 pull 不覆蓋。

**改動檔案**：
1. `sync-claude-push.py` — push 前讀取 sync-skills.yaml，過濾 skills/
2. `sync-claude-pull.py` — pull 後依 sync-skills.yaml 過濾 skills/
3. 新建 `sync-skills.yaml`（local-only，已在 sync_exclude_manifest LOCAL_ONLY_PATTERNS）

**注意**：sync-skills.yaml 本身應加入 LOCAL_ONLY_PATTERNS（各專案設定不同，不跨專案同步）。

**commit policy**：在 feat/1.3.0-W1-054-sync-skills-yaml 分支 commit。
