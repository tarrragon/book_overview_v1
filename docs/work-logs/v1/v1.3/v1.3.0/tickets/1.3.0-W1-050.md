### 方向修正（2026-06-23）

原 ANA W1-002 的方案方向有誤：將 project-integration/ 加入 sync-preserve（防 pull 覆蓋）。

用戶指出判斷偏差：框架 repo 的 skill 是 SSOT（最新、最進步的版本），本地**應該接受 pull 覆蓋**。真正需要的是：

- **push 方向排除**：專案專屬的落地層內容不應被推到框架 repo
- 需要一個「專案專屬 skill 內容不推到框架」的機制（名單或標記）

### 正確方案

在 `sync_exclude_manifest.py` 或 push 腳本中加入 push-only 排除機制，讓 project-integration/ 類的專案落地層內容只存在於本專案，不隨 sync-push 推到框架 repo。

### AC 修正

原 AC 1（加入 sync-preserve）和 AC 3（防 pull 覆蓋）方向錯誤，需更新為：
- AC 1 → 在 push 排除機制中加入 project-integration/ 路徑
- AC 3 → 移除（pull 覆蓋是正確行為）

### 實作方案（PM 前台分析）

改動極小：在 `sync_exclude_manifest.py` 的 `LOCAL_ONLY_PATTERNS` 加入 `"project-integration"`。

`should_exclude` rule 4 會匹配路徑中任一目錄段，效果：
- 路徑如 `skills/wrap-decision/references/project-integration/README.md` → 排除
- Push 不推到框架 repo
- Pull 不覆蓋本地（框架 repo 本來就沒有，preserve 本地）
- Content hash 不納入

歸類為**類型 B - Local-only settings**：各專案落地層內容獨立管理，不應跨專案同步。

改動範圍：`sync_exclude_manifest.py` 一行。
