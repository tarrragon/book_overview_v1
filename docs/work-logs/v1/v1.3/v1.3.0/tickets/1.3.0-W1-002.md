## Solution
### F 案外移方案

### 目標

將 `.claude/skills/wrap-decision/references/project-integration/` 從 wrap-decision skill 主體分離，使 wrap-decision 通用層可獨立上架 Skill Market（對齊 skill-marketplace-standard.md §3 落地層架構）。

### 方案：保留在專案、不隨 sync-push 推送

**結論**：project-integration/ 是本專案的落地層，**不應外移到獨立 claude repo**，而應**保留在本專案但從 sync-push 排除**。

**Why**：
1. 獨立 claude repo（tarrragon/claude.git）是**跨專案共用框架**，project-integration/ 是**單一專案的落地整合**，方向相反
2. skill-marketplace-standard.md §3 明確定義：落地層（project-integration/）不隨 skill 上架——這意味著落地層應留在各自的專案，不是集中到框架 repo
3. 上架時排除 project-integration/ 即可，不需要物理搬移

### 搬移路徑

| 步驟 | 動作 | 原因 |
|------|------|------|
| 1 | 確認 sync-push 已排除 project-integration/ | sync-push 不應把專案落地層推到框架 repo |
| 2 | 確認 sync-pull 不會覆蓋本專案 project-integration/ | preserve 清單應包含 |
| 3 | Skill Market 上架時打包排除 project-integration/ | 對齊 skill-marketplace-standard.md |
