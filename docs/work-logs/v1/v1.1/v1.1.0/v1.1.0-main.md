# v1.1.0 版本工作日誌

**版本號**: v1.1.0
**開始日期**: 待定
**目標**: trigger-bound 監測任務集中錨點（自 v0.18 ~ v0.20 累積延後的監測票統一版本錨點重綁）
**狀態**: 待開始

---

## 前情提要

v1.1.0 作為**延後監測任務的版本錨點**。自 v0.18.0 起累積的一系列 trigger-bound / 時間 trigger / 外部依賴監測票，因 trigger 設計偏向「自然發生」，在低活躍領域長期未觸發，反覆跨版本延後（0.18 → 0.19 → 0.20）。

依 `.claude/rules/core/decision-trigger-binding.md` 規則 2，延後決策的合法 trigger 包含「版本錨點」。本版本將這些監測票統一遷入，以 v1.1.0 開版作為重評錨點，避免散落於活躍開發版本（0.20.0）造成 backlog 認知負擔。

## 遷移來源

自 0.20.0-W1-001 ~ W1-018（共 18 張）遷移，遷移日期 2026-06-05。各票 trigger 類型：

| 類別 | Ticket | trigger 條件 |
|------|--------|-------------|
| 外部 release / Issue | W1-001(Chrome 149), W1-002(zhtw-mcp), W1-003(Anthropic Issue), W1-005(cbm skip), W1-006(cbm namespace) | 等上游釋出 / 修復 |

| 觀察 / 3個月 | W1-009(三MCP使用率), W1-013(token節省), W1-014(Phase4觀察), W1-015(密度需求) | 等觀察結論 |

## 待執行

各 ticket 保持原 trigger 條件。v1.1.0 開版時 PM 應逐張重評 trigger 是否已滿足，滿足者認領執行，未滿足者再評估是否繼續延後或調整 trigger。
