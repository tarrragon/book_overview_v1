## Context Bundle

### 實作策略

在 sync-push.py 和 sync-pull.py 各自的成功路徑尾端加入 skill 版本比對邏輯。

#### 版本提取方式

遍歷 skills/*/SKILL.md，用 regex 提取版本號：
- 格式 1：`**Version**: x.y.z`（多數 skill）
- 格式 2：YAML frontmatter `version: x.y.z`（少數如 impeccable）

```python
import re
def extract_skill_version(skill_md_path):
    text = Path(skill_md_path).read_text()
    m = re.search(r'\*\*Version\*\*:\s*(\S+)', text)
    if not m:
        m = re.search(r'^version:\s*(\S+)', text, re.MULTILINE)
    return m.group(1) if m else None
```

#### Push 流程

1. push 前：掃描本地 .claude/skills/ 建 `{name: version}` dict → `before`
2. push 後：掃描遠端 repo 的 skills/ 建 `{name: version}` dict → `after`（但 push 完遠端就是本地，所以 before 和 after 都在本地取）

簡化：push 是把本地推到遠端。比對的是「遠端上次 push 的版本」vs「本次要推的版本」。可以用 sync-state.json 的 hash 判斷上次推送後哪些 skill 檔案變了，或直接比對 git diff。

**更簡化**：push 前用 git show 從遠端 HEAD 提取 skills/*/SKILL.md 版本，push 後用本地版本。兩者 diff 即為摘要。但 push 腳本用臨時目錄操作，可在 clone 時提取遠端版本。

#### Pull 流程

1. pull 前：掃描本地 .claude/skills/ 建 `before` dict
2. pull 後（overlay 完成）：掃描本地 .claude/skills/ 建 `after` dict
3. diff 兩 dict → 輸出摘要

Pull 比 push 簡單——前後都在本地。

#### 輸出格式

```
[Skill 變更摘要]
  新增: new-skill (1.0.0)
  更新: ticket 2.7.0 → 2.8.0, tdd 2.0.0 → 2.1.0
  移除: deprecated-skill (was 1.0.0)
```

無變更時不輸出。
