## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
於 `sync-claude-pull.py` 末尾加 post-sync runtime 驗證步驟（issue #10 斷裂 4）。

新增：
- 常數 `HOOK_IMPORT_TEST_RELPATH = "scripts/test-hook-imports.sh"`（複用 W2-007 產出）、`HOOK_IMPORT_TEST_TIMEOUT_SECONDS = 120`。
- 函式 `verify_hook_imports(project_root) -> int`：呼叫 test-hook-imports.sh 對所有 hook 做 import-level 煙霧測試；非零 returncode 時將失敗清單（腳本輸出於 stdout）轉發到 stderr 並回傳該 exit code；腳本不存在 / OSError / Timeout 時 graceful skip 回 0（不阻擋同步主流程，同步本體已完成）。
- `main()` 於 `_complete_sync` 後呼叫 verify_hook_imports；非零時 `sys.exit(import_rc)`。

設計理由：py_compile 只檢查語法不執行 import，無法偵測上游 lib 重構 / 新增 dependency 導致的 runtime import 斷裂。本驗證為後置安全網。
