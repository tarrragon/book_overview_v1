## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 調查發現（W2-008 拆出，PM 前台調查）

現況 import：
- `.claude/lib/common_functions.py:33` `from hook_utils import get_project_root`；唯一使用點 line 42 `project_dir = get_project_root()`。
- `.claude/lib/markdown_formatter.py:13` `from hook_utils import setup_hook_logging, read_json_from_stdin`；使用點 line 68-69（`setup_hook_logging("markdown-formatter")` + `read_json_from_stdin(logger)`）。

符號在 lib/ 的對應（關鍵：非一對一）：
- `get_project_root` → lib/ 定義在 `lib/git_utils.py:219`（簽章 `get_project_root(cwd=None)`），**不在** lib/hook_logging.py。注意 hook_utils 版是從 `hook_utils/hook_logging.py` 匯出，與 lib 版來源模組不同。
- `setup_hook_logging` → `lib/hook_logging.py:25` 有，可直接改。
- `read_json_from_stdin` → **lib/ 無此函式**。lib/hook_io.py 只有 `read_hook_input()`（無 logger 參數），簽章與 `read_json_from_stdin(logger)` 不同，不能直接替換。需評估：補一個 lib 等價 `read_json_from_stdin`，或改用 `read_hook_input()` 並調整呼叫端。

風險：
- `.claude/hooks/hook_utils/` 是 package（hook_base/hook_io/hook_logging/hook_ticket/__init__）；lib/ 有自己分歧版的 hook_logging.py（無 get_project_root）、hook_io.py（read_hook_input 而非 read_json_from_stdin）。盲目替換會 runtime 崩潰。
- lib/ → hooks/hook_utils/ 屬反向依賴（lib 應為較低層）。

建議 TDD：Phase 2 先寫 RED 驗證 (a) 改後 import 可解析 (b) get_project_root / read_json 行為等價；Phase 3b 補 lib 缺失符號後改 import；驗收含 hook 測試套件 + `test-hook-imports.sh`（W2-004 已接入 sync-pull）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
消除 lib → hooks/hook_utils 反向依賴，改依賴 lib 自身模組。

1. lib/hook_io.py：新增 read_json_from_stdin(logger)，複製 hook_utils 行為（空輸入→None、有效 JSON→dict、解析失敗→logger.info+None）；補 `import logging`。與既有 read_hook_input（回空 dict）並存，供需區分「無輸入 vs 空物件」者使用。lib/__init__.py re-export。
2. lib/common_functions.py:33：`from hook_utils import get_project_root` → `from .git_utils import get_project_root`（相對 import，package 模式；該檔僅以 lib.common_functions 載入，由 4 個 active hook 使用）。消除原靠 caller hook sys.path[0]=.claude/hooks/ 的隱式依賴。
3. lib/markdown_formatter.py:12-13：sys.path 由 .../hooks 改為 .../（lib 自身，script 模式 __main__）；`from hook_logging import setup_hook_logging` + `from hook_io import read_json_from_stdin`。

設計理由：lib 應為較低層，不應反向依賴 hooks/hook_utils；移除後 lib 自包含。
