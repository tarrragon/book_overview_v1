## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
根因：W1-058 將模組由 .claude/hooks/lib/ 遷移至 .claude/lib/ 後，測試 mock 的 patch 目標未同步。

test_git_utils.py 與 test_config_loader.py 的 top import 已正確（`from lib.git_utils import` / `from lib.config_loader import`），但 16 處 @patch 仍用 top-level 模組名：
- `@patch('git_utils.run_git_command')`（14 處）
- `patch('config_loader.load_config')`（2 處）

模組現以 lib.git_utils / lib.config_loader 載入，無 top-level git_utils/config_loader → patch 觸發 ModuleNotFoundError: No module named 'git_utils'。stash 對照（W2-010 執行中）確認 base 同樣 16 failed，與 W2-010 變更無關。
