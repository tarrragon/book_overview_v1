## Problem Analysis
W1-019 跑完整 ticket 測試套件時發現 test_track_dashboard.py::test_C1_no_subprocess_run / test_C2_no_subprocess_popen 2 紅燈。git stash 隔離驗證確認非 W1-019 引入（pre-existing）。

呼叫鏈：dashboard_main → _print_all_completed_warning(version) → check_version_all_completed(version) → lib.ticket_loader.list_tickets → get_tickets_dir → get_project_root → subprocess.run（git rev-parse）。

測試 _patch_loader 只 patch track_dashboard 命名空間的 list_tickets，未涵蓋 lib 層路徑，故 _print_all_completed_warning 的重新載入逃過 patch 觸發真實 subprocess。

## Solution
根因定位（W1-019 執行中發現的 pre-existing 失敗）：

test_track_dashboard.py::test_C1_no_subprocess_run / test_C2_no_subprocess_popen 紅燈。
dashboard_main line 306 已載入 all_tickets，但 line 351 _print_all_completed_warning
經 check_version_all_completed 透過 lib.ticket_loader.list_tickets（_patch_loader 只 patch
track_dashboard 命名空間的 list_tickets，未涵蓋此 lib 路徑）重新載入 → get_project_root →
subprocess.run（git rev-parse），違反測試 no-subprocess 斷言。

判定：真實碼缺陷（冗餘重載 + subprocess），非測試過嚴。

落地 IMP：1.4.0-W1-020.1（completed）— check_version_all_completed 加 optional tickets 參數，
dashboard 傳已載入 all_tickets，消除冗餘重載與 subprocess。完整套件 2519 passed 0 failed。
