### 三項 False Positive 根因分析

### Bug 1: Hook 配置檔搜尋路徑硬編碼
- **位置**: `onboard_checker.py:824` `check_hook_configurations()`
- **根因**: 函式硬編碼只搜 `.claude/config/` 目錄，但實際 `hook-exclude-list.json` 在 `.claude/hooks/`，`settings.json` 在 `.claude/`
- **修正方向**: 新增 fallback 搜尋路徑（`.claude/hooks/` 和 `.claude/`）

### Bug 2: CLAUDE.md 顯示邏輯子字串誤匹配
- **位置**: `onboard.py:489` `_print_claude_md_section()`
- **根因**: 使用 `_has_todo_item(result.todo_items, "CLAUDE.md")` 做子字串匹配。tech stack 的 todo item 描述 "CLAUDE.md 缺少技術選型 section" 也含 "CLAUDE.md"，導致 CLAUDE.md 存在但被誤報為不存在
- **修正方向**: 直接使用 `claude_md_info.exists` 判斷，不透過 todo items 間接判定

### Bug 3: 技術選型關鍵字過窄
- **位置**: `onboard_checker.py:417` `check_tech_stack_section()`
- **根因**: 只搜 `"技術選型"` 字面，但本專案用 `"語言特定規範"`、`"專案身份"`、`"開發語言"` 等等效表述
- **修正方向**: 擴充搜尋關鍵字清單，涵蓋多種等效 section 名稱
