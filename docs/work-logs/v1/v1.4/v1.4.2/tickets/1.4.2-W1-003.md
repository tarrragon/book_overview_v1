## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 用戶內測截圖根因線索

getBookElements 回傳 0 個元素：
- directLibraryItemCount: 0
- attrSelectorCount: 0
- readerLinkCount: 0
- readyState: interactive（頁面尚未完全載入）

可能根因：
1. DOM 選擇器 `.library-item` 在 Readmoo v1.55.1 失效（DOM 結構變更）
2. readyState=interactive 時過早偵測（SPA 尚未渲染書目元素）
3. WAIT_FOR_BOOK_ELEMENTS 的 timeoutMs=3000 可能不足

需實機驗證 Readmoo 當前 DOM 結構是否與 readmoo-adapter.js 記錄的選擇器一致。

### 同時觀察到的 Readmoo 網站自身問題（非本 extension 造成）
- CSP 違規警告（Google 身份驗證 iframe sandbox）
- Google 身份驗證庫棄用警告（gis-migration）
- accounts.google.com 400 錯誤
以上皆為 Readmoo 網站或 Google 服務的問題，不影響 extension 功能。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 根因

1. `getHealthStatus()`（content-modular.js L369）不回傳 `bookCount`，僅回傳模組狀態和效能資料
2. `checkCurrentTab()`（popup.js L951）PING 成功後硬設 `'檢測中...'`，不讀取 response 中的 bookCount

### 修復

1. **content-modular.js `getHealthStatus()`**：加入 `bookCount`（透過 `readmooAdapter.getBookElements().length` 同步取得）
2. **popup.js `checkCurrentTab()`**：PING 成功後讀取 `response.bookCount` 更新顯示，以 `response.bookCount !== undefined` 防護缺失欄位

### 修改檔案

- `src/content/content-modular.js`：getHealthStatus() 加入 bookCount
- `src/popup/popup.js`：checkCurrentTab() 讀取 response.bookCount
- `tests/unit/popup/popup-ping-bookcount.test.js`：新增 3 個測試案例
- `tests/unit/popup/popup-final-status-sticky.test.js`：更新 PING mock 含 bookCount
