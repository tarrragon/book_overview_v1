## Problem Analysis

### 用戶內測截圖根因線索

getBookElements 回傳 0 個元素：
- directLibraryItemCount: 0
- attrSelectorCount: 0
- readerLinkCount: 0
- readyState: interactive（頁面尚未完全載入）

可能根因：
1. DOM 選擇器 `.library-item` 在 Readmoo v1.55.1 失效（DOM 結構變更）
2. readyState=interactive 時過早偵測（SPA 尚未渲染書目元素）
3. WAIT_FOR_BOOK_ELEMENTS 的 timeoutMs=3000 可能不足

需實機驗證 Readmoo 當前 DOM 結構是否與 readmoo-adapter.js 記錄的選擇器一致。

### 同時觀察到的 Readmoo 網站自身問題（非本 extension 造成）
- CSP 違規警告（Google 身份驗證 iframe sandbox）
- Google 身份驗證庫棄用警告（gis-migration）
- accounts.google.com 400 錯誤
以上皆為 Readmoo 網站或 Google 服務的問題，不影響 extension 功能。

### 補充根因分析（PM 前台追蹤）

**完整資料流斷裂點**：

1. `periodicStatusUpdate`（每 3 秒）→ `checkCurrentTab()` → PING content script
2. PING handler（content-modular.js:314）呼叫 `getHealthStatus()`（L369）
3. `getHealthStatus()` **不含 bookCount**（只回傳模組狀態 + pageDetector.getPageStatus()）
4. popup.js L951 PING 成功後設 bookCount = '檢測中...'，**無後續請求書籍數量**
5. 含 bookCount 的 `getPageStatus()`（async，L339）從未在非提取場景被呼叫

**修復方案**：在 PING handler 的回應中加入 bookCount。最小修改兩處：
- A. content-modular.js `getHealthStatus()` 加入 `readmooAdapter.getBookElements().length`（同步呼叫，不阻塞 PING）
- B. popup.js `checkCurrentTab()` PING 成功後讀取 `response.bookCount` 更新 UI

不需新增 message type，複用 PING 即可。

## Context Bundle

### 規格

- PING 回應需包含 bookCount（書籍偵測數量）
- popup 在 PING 成功時讀取 bookCount 更新底部狀態列
- 無書籍時顯示 0（非 '檢測中...'）
- 不新增 message type，複用 PING

### 實作策略

1. **content-modular.js `getHealthStatus()` (L369)**：加入 bookCount
   ```js
   function getHealthStatus () {
     const pageStatus = pageDetector ? pageDetector.getPageStatus() : {}
     let bookCount = 0
     if (readmooAdapter && pageStatus.isReadmooPage) {
       bookCount = readmooAdapter.getBookElements().length
     }
     return {
       ...pageStatus,
       bookCount,
       modules: { ... },
       ready: contentScriptReady,
       performance: performanceStats
     }
   }
   ```

2. **popup.js `checkCurrentTab()` (L950-953)**：PING 成功後讀 bookCount
   ```js
   if (response && response.success) {
     const count = response.bookCount !== undefined ? response.bookCount : 0
     elements.bookCount.textContent = String(count)
     updateStatus('就緒', 'Content Script 連線正常', '可以開始提取書庫資料', STATUS_TYPES.READY)
     updateButtonState(false)
     return tab
   }
   ```

3. **測試**：
   - content-modular PING handler 測試：驗證回應含 bookCount
   - popup checkCurrentTab 測試：驗證 PING 成功時更新 bookCount 而非 '檢測中...'
