## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### PM 前台分析進度（2026-06-24）

#### 已確認事實

1. `readmoo-adapter.js:1071` 提取封面邏輯：`img.getAttribute('src') || ''`
2. 只讀 `src` 屬性，**未處理** `data-src` / `data-original` / `srcset` 等 lazy load 屬性
3. Readmoo 書庫是 React SPA + 虛擬捲動（`#react-container`）
4. adapter 有 `scrollAllIntoView`（L695）捲動觸發載入機制
5. 用戶 DOM 貼文顯示兩本書的 `<img class="cover-img">` 都有完整 `src`，但實機結果部分書封面缺失

#### 待驗證假設（需實機觀察）

- H1：React 虛擬捲動回收——捲動過的書被 React unmount 後 DOM 消失，重新渲染時 img src 延遲設定
- H2：CDN 圖片 src 延遲——img 先建空 DOM 再異步設 src，提取時 src 為空
- H3：部分書籍根本沒有 cover-img 元素（DOM 結構不同）
- H4：Readmoo 用 IntersectionObserver lazy load，viewport 外的 img src 尚未載入

### 相關 Error Pattern

無直接相關。

## 重現實驗結果

### 實驗方法

chrome-devtools-mcp（pipe 模式有頭 Chrome，用戶手動登入 Readmoo）實機觀察 `read.readmoo.com/#/library`，以 `evaluate_script` 注入腳本量測 `.library-item` / `.cover-img` 的 DOM 狀態。2026-06-24 執行。

### 實驗執行

實驗 1（初始 DOM 封面狀態統計，96 本）：
- `withSrc=18`（真實 CDN 封面）、`emptySrc=78`、`noImg=0`、`dataSrc=0`、`lazyAttr=0`
- 缺封面書的 `src` 並非空字串，而是 `/images/openbook.png`（佔位圖，`naturalWidth=140`，class=`cover-img opacity`）
- 無 `data-src` / `srcset` / `loading` 屬性

實驗 2（漸進整頁捲動，`window.scrollTo`）：
- 捲動前 real=18 / placeholder=78（total 96）
- 捲到底後 real=965 / placeholder=0（`.library-item` 從 96 暴增到 965）
- 捲回頂後 real=965 / placeholder=0（真圖保持，**無虛擬回收**）

實驗 3（容器辨識）：
- `#react-container` scrollHeight=4647 == clientHeight=4647 → **不可捲**
- 真正可捲者為整頁 `document.scrollingElement`（4647 > viewport 757）

實驗 4（初始狀態驗證，reload 後）：
- `.item-list-state` header：「擁有 981 本書，其中封存 15 本，借出 1 本」→ expectedTotal = 981-15-1 = **965**（與實驗 2 吻合）
- 「更多...」按鈕（`button.btn-outline-primary`）**存在且可見**

實驗 5（模擬 adapter 動作2，連點「更多...」按鈕 8 輪）：
- 每輪 items +96、real +96，但 **placeholder 恆定 ~90 本不減**
- init:96/r18/ph78 → r1:192/r102/ph90 → ... → r8:864/r774/ph90

實驗 6（新 build:dev + 實機 install_extension + popup 觸發完整提取，2026-06-24）：
- popup 初始「檢測到書籍: 96」→ 提取完成「檢測到書籍: **965**」
- 讀 `chrome.storage.local.readmoo_books.books` 統計 cover：total 965 / **httpCover 309 / openbook placeholder 656 / emptyCover 0**
- **F2 驗證為真**：新 build adapter 成功提取 965 本（W1-040 點更多按鈕修復已生效）；用戶先前僅 96 本確為舊 build（過期 416h）未含 W1-040。
- **F1 驗證且嚴重**：656/965（68%）cover 欄位為 `/images/openbook.png` placeholder。adapter 連點按鈕快速載入後立即提取，多數書未在 viewport 停留載入真圖。`UNSTABLE_COVER_IDS`（L187/L1592）僅用於 book-id 反查 fallback，不清空 cover 欄位，placeholder URL 直接寫入書庫資料。

### 實驗發現

**已驗證事實**：
1. 缺封面書的封面是佔位圖 `/images/openbook.png`，非空 src、非 data-src（H2/H4 標準 lazy-load 假設**否證**）。
2. 書庫採「更多...」按鈕分頁載入（每批 96，共 965 可見）+ React viewport 可見性驅動封面替換（viewport 外顯示 placeholder，進入 viewport 才換真實 CDN src）。
3. 封面真圖替換的充要條件 = 該 `.library-item` **進入過 viewport**（實驗 5 placeholder 恆 ~90：當前 scroll 位置下方那批永遠是 placeholder）。
4. 真圖一旦載入不回收（H1 虛擬回收**否證**）。
5. adapter `findScrollContainer` 命中的 `#react-container` 不可捲，`_scrollStep` 動作1（`container.scrollTop=scrollHeight`）完全無效；實際載入僅靠動作2 點「更多...」按鈕。

**根因**（兩機制疊加）：
- 即使按鈕點滿載入全部 965 本 item，提取當下停在某 scroll 位置，viewport 外的最後 ~90 本封面仍是 `/images/openbook.png` placeholder。
- 這批 placeholder 被 `UNSTABLE_COVER_IDS`（含 `openbook`，L187）過濾 → cover 變空/走 fallback → 用戶看到「部分書沒封面」。
- 用戶實機僅得 96 本，另一層問題：build/development 過期 416h，W1-040「每輪點更多按鈕」修復很可能未編譯進測試用 build（待驗證）。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復方向

| 編號 | 對應根因 | 方案 | 優先 |
|------|---------|------|------|
| F1 | 封面 placeholder（本 ticket 新發現） | `loadAllBooksLazy` item 全載入後，新增「完整捲過全部書籍」步驟（從頂漸進捲到底，讓每個 `.library-item` 進入過 viewport 觸發封面真圖替換），再進行提取 | 高 |
| F2 | build 過期 | 先 `npm run build:dev` 重建，用新 build 實機重測，確認 W1-040「每輪點更多按鈕」是否已生效、實際提取數是否達 965 | 高 |
| F3 | 容器策略防禦 | `findScrollContainer` 將整頁可捲（`document.documentElement` / `scrollingElement`）排在 `#react-container` 之前，或 `_scrollStep` 動作1 改用整頁捲動 `window.scrollTo`，避免依賴不可捲容器 | 中 |
| F4 | 提取期 placeholder 兜底 | 提取單本封面時若 src 命中 `openbook` placeholder，先 `scrollIntoView` 該元素 + 短等待 src 替換再讀取（F1 的 per-item 兜底） | 中 |

### 驗證要點

- F1 為核心：僅「載入 item」不足，必須「讓所有 item 進過 viewport」封面才完整（實驗 5 證 placeholder 恆 ~90）。
- 修復後 acceptance：實機提取 965 本且封面真圖數 == 965（placeholder 過濾後缺封面數為 0，或僅剩 Readmoo 端本身無封面者）。

2. **等書庫載入完成後，觀察 DOM**
   ```js
   // 統計所有 .library-item 中 cover-img 的 src 狀態
   mcp__chrome-devtools__evaluate_script({
     function: `() => {
       const items = document.querySelectorAll('.library-item')
       const result = { total: items.length, withSrc: 0, emptySrc: 0, noImg: 0, dataSrc: 0, samples: [] }
       items.forEach((item, i) => {
         const img = item.querySelector('.cover-img') || item.querySelector('img')
         if (!img) { result.noImg++; return }
         const src = img.getAttribute('src') || ''
         const dataSrc = img.getAttribute('data-src') || img.getAttribute('data-original') || ''
         if (src && src.startsWith('http')) result.withSrc++
         else result.emptySrc++
         if (dataSrc) result.dataSrc++
         if (i < 5 || !src) result.samples.push({ i, src: src.substring(0, 80), dataSrc: dataSrc.substring(0, 80), alt: img.alt })
       })
       return result
     }`
   })
   ```

3. **捲動後再次觀察**——確認虛擬捲動回收行為
   ```js
   // 捲到底再捲回頂，觀察第一批書的 img 是否被回收
   mcp__chrome-devtools__evaluate_script({
     function: `async () => {
       const container = document.querySelector('#react-container')
       container.scrollTop = container.scrollHeight
       await new Promise(r => setTimeout(r, 2000))
       container.scrollTop = 0
       await new Promise(r => setTimeout(r, 2000))
       const items = document.querySelectorAll('.library-item')
       const first5 = Array.from(items).slice(0, 5).map(item => {
         const img = item.querySelector('.cover-img')
         return { hasSrc: !!(img && img.src), src: img ? img.src.substring(0, 60) : 'NO_IMG' }
       })
       return { totalAfterScroll: items.length, first5 }
     }`
   })
   ```

4. **檢查 IntersectionObserver**
   ```js
   mcp__chrome-devtools__evaluate_script({
     function: `() => {
       const img = document.querySelector('.cover-img')
       return { loading: img?.loading, decoding: img?.decoding, hasLazyAttr: img?.hasAttribute('loading') }
     }`
   })
   ```

### 判定標準

| 觀察結果 | 對應假設 | 修復方向 |
|---------|---------|---------|
| emptySrc > 0 且 dataSrc > 0 | H4 lazy load | fallback 讀 data-src |
| emptySrc > 0 且 dataSrc = 0 | H1/H2 時序 | 提取前等 img.complete 或 MutationObserver |
| noImg > 0 | H3 DOM 差異 | 調查不同 DOM 結構的書 |
| emptySrc = 0 | 非 DOM 問題 | 檢查 adapter 後續處理是否遺失 cover |
