## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-004.1 basil 審查發現 popup 狀態 badge（`extensionStatus` 欄，經 updateStatus 第 1 參數寫入 textContent，使用者可見）詞彙體系不統一：同一「就緒」語意散用多種短語，且錯誤態用網路隱喻「離線」，造成使用者認知負擔。

### 影響範圍

使用者可見的狀態 badge 短語（updateStatus 第 1 參數 `status`）：
- src/popup/popup.js:692 「擴充功能就緒」
- src/popup/popup.js:746 「測試模式」
- src/popup/popup.js:758 「線上」
- src/popup/popup.js:893 「離線」
- src/popup/popup.js:956 「就緒」
- src/popup/popup.js:962 「載入中」
- src/popup/popup-event-controller.js:333 「線上」
- src/popup/popup-event-controller.js:345 「離線」
- src/popup/popup-event-controller.js:399 「就緒」
- src/popup/popup-event-controller.js:405 「載入中」

updateStatus 簽章（popup.js:392）：`updateStatus(status, text, info, type)`，status → elements.extensionStatus.textContent（badge，使用者可見）。

### 相關 Error Pattern

無直接相關 error pattern。

## 重現實驗結果
### 實驗方法

以 rg 盤點 src/popup/ 兩檔的 extensionStatus badge 短語與 generateErrorDiagnostic 呼叫鏈；以 Read 確認 updateStatus 簽章與欄位 → DOM 對應；以 rg 全專案（src + tests）追蹤 generateErrorDiagnostic 消費端。

### 實驗執行

1. `rg -n "線上|離線|就緒|載入中|extensionStatus" src/popup/popup.js src/popup/popup-event-controller.js` → 取得 badge 短語分佈（見 Problem Analysis）。
2. Read popup.js:392 updateStatus → 確認 arg1 status 寫入 elements.extensionStatus.textContent（DOM，使用者可見）。
3. `rg -n "generateErrorDiagnostic|userMessage|diagnosticInfo" src/popup/popup.js` → 僅定義（766）+ window 掛載（1759），無內部呼叫。
4. controller 檔 + 全專案（src/tests）grep generateErrorDiagnostic → 零消費端。

### 實驗發現

已驗證事實：
- badge「就緒」語意有 4 種寫法：線上 / 就緒 / 擴充功能就緒 / 測試模式；錯誤態用網路隱喻「離線」。詞彙不統一屬實。
- generateErrorDiagnostic 的 userMessage **目前不使用者可見**：popup.js 與 controller 均無呼叫端寫入 DOM，僅 window.generateErrorDiagnostic（1759）暴露（測試/除錯用），全專案無實際消費者。

結論：acceptance 1 詞彙不統一成立；acceptance 2 答案為「否，當前不可見」——userMessage 詞彙統一非當務之急，但函式為近乎無消費端的孤兒，另記為觀察項。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 統一詞彙表（badge / extensionStatus）

| 統一詞彙 | 語意 | 取代現況短語 | STATUS_TYPE |
|---------|------|------------|-------------|
| 就緒 | 擴充功能/連線可用 | 線上、擴充功能就緒（就緒保留） | READY |
| 載入中 | 初始化/連線進行中 | 載入中（保留） | LOADING |
| 未連線 | 連線失敗/重試用盡 | 離線 | ERROR |
| 測試模式 | 測試環境特殊態 | 測試模式（保留） | READY |

需替換的字面（4 處 + 2 處 controller）：
- popup.js:692 「擴充功能就緒」→「就緒」
- popup.js:758 「線上」→「就緒」
- popup.js:893 「離線」→「未連線」
- popup-event-controller.js:333 「線上」→「就緒」
- popup-event-controller.js:345 「離線」→「未連線」
- （popup.js:956/962、controller:399/405 已是「就緒/載入中」，無需改）

建議：badge 字面抽為常數（src/popup/constants/ui-text.js 既有 STATUS 分類或新增 EXTENSION_STATUS_TEXT），消除散落字面，後續擴充單點維護。

### WRAP — 詞彙軸選擇

- W（候選）：(a) 平實態 就緒/載入中/未連線（選用）；(b) 技術隱喻 線上/離線；(c) 維持現狀。
- A（機會成本）：低，純字面 + 常數抽取，不動控制流。
- P（選 a 理由 / 最可能失敗）：(b) 網路隱喻誤導——extension 就緒 ≠ 網路上線，使用者易誤解；(c) 違反 quality-baseline 規則 5（basil 發現須追蹤）。最可能失敗：測試對舊字面（線上/離線）有斷言 → IMP 須同步更新 tests 斷言（grep 確認）。

### generateErrorDiagnostic 可見性結論

userMessage 當前不可見（零消費端），詞彙統一**不納入本次 IMP**。函式為近乎孤兒（僅 window 暴露），另記為技術債觀察——若未來接線到可見 UI，再納入詞彙統一範圍。本 ANA 不為其單獨 spawn ticket（無使用者影響）。
