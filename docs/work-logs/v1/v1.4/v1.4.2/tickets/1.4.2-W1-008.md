## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因（定位後）

非 ticket CLI 缺陷。W1-006 subagent 自述「append-log --section Test Results/Exit Status 未寫入，CLI 回傳僅 echo heredoc」。關鍵線索「回傳僅 echo heredoc」表示：subagent 構造的 Bash 指令 malformed——heredoc 被 shell 自身 echo 出來，`ticket` CLI 根本未執行（無 `[OK] 已追加日誌` 回應），故 ticket md 無變更。subagent 將「自己的 shell echo 輸出」誤判為「CLI 的回傳行為」，屬 tool-output-trust 違反 + bash heredoc 使用瑕疵（PC-079 / bash-rule 5 鄰近模式）。

### 影響範圍

- 表象：W1-006.md Exit Status 區段仍留 placeholder 範本（subagent append-log 未生效）。
- 實際：ticket CLI append-log 對所有 section（含 Test Results / Exit Status）功能正常，PM 重現逐字持久化。
- 風險面：subagent 若 malformed heredoc 後誤歸因為「CLI bug」，會用 Edit 繞過或放棄收尾章節，造成 Exit Status 等可觀測性資訊靜默遺失。

### 相關 Error Pattern

PC-079（CLI args backtick/heredoc）、tool-output-trust-rules 規則 2（只信 raw stdout，帶旁白/非 CLI 格式視為自身雜訊）。

## 重現實驗結果
### 實驗方法

PM 環境直接呼叫 ticket CLI append-log 對 Test Results 與 Exit Status 兩 section 寫入唯一 marker，再以固定值 grep 驗證 ticket md 是否實際持久化（不信 CLI [OK] 回報，查世界平面）。對照組為 W1-006.md 的 Exit Status 區段實況。

### 實驗執行

1. claim 1.4.2-W1-008（in_progress）。
2. `append-log --section "Test Results" "REPRO_MARKER_TR_12345 ..."` → CLI 回 [OK]。
3. `append-log --section "Exit Status" "REPRO_MARKER_ES_67890 ..."` → CLI 回 [OK]。
4. `grep -c REPRO_MARKER_TR_12345 1.4.2-W1-008.md` = 1；`grep -c REPRO_MARKER_ES_67890` = 1，且 awk 確認落在正確 section。
5. 對照 W1-006.md：awk Exit Status 區段 → 僅 placeholder 範本，無 subagent 內容。

### 實驗發現

已驗證事實：
- ticket CLI append-log 對 Test Results / Exit Status **功能正常**，PM 呼叫逐字持久化（markers count=1，section 正確）。
- W1-006 subagent 的 Exit Status 確實未寫入（placeholder 殘留）→ 失敗發生在 subagent 端的指令執行，非 CLI 內部。
- 結合 subagent 自述「CLI 回傳僅 echo heredoc」→ 根因為 subagent malformed heredoc 指令（ticket CLI 未被執行）+ 將 shell echo 誤判為 CLI 行為。

未驗證假設：subagent 確切的 malformed 指令形態無法取得（transcript 不可讀），故 malformed heredoc 為最合理推斷而非直接觀測。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 結論

ticket CLI append-log **無缺陷**（PM 重現對 Test Results/Exit Status 逐字持久化）。W1-006 subagent 失敗根因為其端 malformed heredoc Bash 指令使 ticket CLI 未被執行，並將自身 shell echo 誤判為 CLI 回應。**不需修 CLI**。

### 預防（spawn 落地）

風險為可重現的 subagent 行為模式（heredoc 收尾誤用 + 誤歸因），既有規則（bash-rule 5 heredoc、tool-output-trust 規則 2、PC-079）涵蓋根因但未在「派發收尾指引」surface。spawn 1 個 P3 DOC child：在 `.claude/references/agent-dispatch-template.md` 收尾段補一行驗證準則——「append-log 收尾後以 grep 確認 section 已寫入；唯有 CLI 回 `[OK] 已追加日誌` 才算成功，shell echo 出 heredoc 內容代表指令 malformed、CLI 未執行」。詳見 children。
