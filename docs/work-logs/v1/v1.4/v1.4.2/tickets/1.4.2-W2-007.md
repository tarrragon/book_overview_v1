## Solution

### 修復內容（commit 05c254c0f，分支 feat/1.4.2-W2-007-overview-focus-reload）

1. manifest.json：permissions 加入 "tabs"（tabs.query 跨視窗以 url 比對 overview 所需）
2. src/popup/popup.js openLibraryOverview 改為 async：
   - overviewUrl = chrome.runtime.getURL('src/overview/overview.html')
   - chrome.tabs.query({}) 取全部分頁，再以 tab.url.startsWith(overviewUrl) 比對既有頁（涵蓋 hash 路由 overview.html#/library）
   - 既有：chrome.tabs.update(id,{active:true}) + chrome.windows.update(windowId,{focused:true}) + chrome.tabs.reload(id)
   - 不存在：chrome.tabs.create({url:overviewUrl})
   - 保留現有 try/catch（Logger.error + window.alert）
   - 暴露 window.openLibraryOverview 供測試
   - viewLibraryBtn 與 viewResultsBtn 共用同函式，一併受益
3. tests/mocks/chrome-mock-factory.js：補 runtime.openOptionsPage、tabs.reload、新增 windows 命名空間，匯出 createWindowsMock
4. tests/integration/chrome-extension/manifest.test.js：tabs 由危險權限清單移出並斷言其存在（本功能必需）

### 設計考量
- 用 query({}) 全量 + startsWith 前綴比對，而非 query({url}) filter，以穩定涵蓋 hash 路由
- promise-style 與 popup.js 既有 await chrome.tabs.query 用法一致（L931）

- overview URL：chrome-extension://<id>/src/overview/overview.html

### 實作指引

1. manifest.json permissions 加 "tabs"
2. popup.js openLibraryOverview 改寫：
   - const overviewUrl = chrome.runtime.getURL('src/overview/overview.html')
   - chrome.tabs.query({ url: overviewUrl }) 找既有（注意 url filter 需 tabs permission；若 overview 有 hash 路由用 overviewUrl + '*' 或 query 全部再 filter startsWith）
   - 存在：chrome.tabs.update(tab.id, { active: true }) + chrome.windows.update(tab.windowId, { focused: true }) + chrome.tabs.reload(tab.id)
   - 不存在：chrome.tabs.create({ url: overviewUrl })（或保留 openOptionsPage 作 fallback）
   - 保留 try/catch 錯誤處理（現有 Logger.error + alert）
3. viewResultsBtn 也綁同一函式（L1186），一併受益

### 測試
覆蓋兩路徑：mock chrome.tabs.query 回傳既有 tab → 驗證 update+reload+windows.update 呼叫；回傳空 → 驗證 tabs.create 呼叫。
