## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

Logger class (`src/core/logging/Logger.js:65`) constructor 預設 `level = 'INFO'`，不論 build mode。prod build 雖將 `process.env.NODE_ENV` 替換為 `'production'` 字串常量（`scripts/build.js:480`），但 Logger 不讀取此值來調整 level。

esbuild 配置（`scripts/build.js:456-483`）也沒有 `drop: ['console']` 或 `pure` 設定，因此所有 console.info/log 呼叫保留在 prod bundle。

結果：content-modular.js:276 的 `Logger.info('Content Script 收到訊息', { message })` 在每次 PING（每 3 秒）都輸出 console.info。

### 影響範圍

- `src/core/logging/Logger.js` — constructor 預設 level 需 prod-aware
- 所有使用 Logger.info/debug 的模組在 prod 都會洩漏日誌（非僅 content-modular.js）

### 相關 Error Pattern

無直接相關 error pattern。

### 修復策略

**方案 B（精確）**：Logger constructor 根據 `process.env.NODE_ENV === 'production'` 將預設 level 從 'INFO' 升為 'WARN'。

- prod：只輸出 WARN + ERROR（靜默 INFO/DEBUG）
- dev：維持 INFO（向後相容）
- 不用 esbuild `drop: ['console']`（太暴力，連 error 都移除）
- esbuild 已將 `process.env.NODE_ENV` 替換為字串常量，dead code elimination 會自動移除 dev-only 分支

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實際解法（外溢修復，PM 接手釐清 2026-06-25）

W1-002 已由 commit 8a65f03df「fix(logger): add PROD_LOG_FLOOR...」修復，但 ticket 未收尾、Problem Analysis 仍記舊方案 B（constructor 預設）。

### 真根因（修正 Problem Analysis 的不準確）
PING（content-modular.js Logger.info('Content Script 收到訊息')）走**靜態方法** Logger.info → _getDefaultInstance() → _defaultInstance = new Logger('App', 'DEBUG')（L472，level DEBUG）→ direct() 輸出。方案 B 改 constructor 預設對明確傳 'DEBUG' 的 _defaultInstance 無效——故未採用方案 B。

### 採用方案：PROD_LOG_FLOOR gate（更優）
Logger.js L54 `PROD_LOG_FLOOR = (process.env.NODE_ENV === 'production') ? LEVEL_PRIORITIES.WARN : LEVEL_PRIORITIES.DEBUG`；_shouldLog L132 `targetPriority >= currentPriority && targetPriority >= PROD_LOG_FLOOR`。在 _shouldLog 層強制 gate，涵蓋所有路徑（靜態/實例/明確傳 level），不受實例 level 影響。

### 實機驗證（npm run build:prod 2026-06-25）
build/production/src/content/content-modular.js：
- L688 `PROD_LOG_FLOOR = true ? LEVEL_PRIORITIES.WARN : LEVEL_PRIORITIES.DEBUG`（esbuild 把 NODE_ENV==='production' 求值為 true → WARN）
- L760 _shouldLog gate 含 PROD_LOG_FLOOR → INFO priority < WARN → PING 不輸出
- bundle 仍含 27 個 console 呼叫 + _consoleOutput（gate 不 strip，保留 console.error/warn 可觀測性）

用戶看到 PING 為測修復前舊 build/production。

### acceptance 1 措辭修正
原「不含 console 呼叫」基於 strip 假設，與實際 gate 方案不符。gate 優於 strip：保留 console.error/warn（quality-baseline 規則 4 異常可觀測性），strip 會連 error 移除。改為「prod 經 PROD_LOG_FLOOR gate 不輸出 INFO/DEBUG」。

### 實作策略

1. **`src/core/logging/Logger.js` constructor**（第 65 行）：
   將 `level = 'INFO'` 改為動態預設：
   ```js
   const DEFAULT_LOG_LEVEL = (typeof process !== 'undefined' && process.env && process.env.NODE_ENV === 'production') ? 'WARN' : 'INFO'
   ```

   constructor: `constructor (name = 'App', level = DEFAULT_LOG_LEVEL, messages = GlobalMessages)`

   esbuild 在 prod 會將 `process.env.NODE_ENV` 替換為 `'production'`，所以 `DEFAULT_LOG_LEVEL` 在 bundle 後變成常量 `'WARN'`。
