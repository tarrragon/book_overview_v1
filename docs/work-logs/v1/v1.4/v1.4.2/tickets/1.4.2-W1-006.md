## Context Bundle

### 既有資產（必先讀，勿重造）

- `scripts/package.js`（既有壓縮腳本，npm `package:zip` = `node scripts/package.js`）
- npm `package` = `npm run build:prod && node scripts/package.js`
- npm `build:prod` = `node scripts/build.js --prod`（輸出至 `build/`）
- npm `clean` = `rimraf build/ coverage/ test-results/`
- 版本號來源：`package.json` version / `manifest.json` version

**先 Read `scripts/package.js`** 了解現有壓縮邏輯與輸出位置，再決定是擴充它或新增薄包裝，避免與既有 packaging 重複。

### 目標（本 ticket）

新增 `npm run package:release`：
1. `build:prod` 產生生產版（移除 log）到 `build/`
2. 壓縮為含版本號 zip（如 `readmoo-book-extractor-v<version>.zip`，version 讀 package.json）
3. 輸出至根目錄 `release/`（重跑前清理舊 zip，不殘留）
4. 在 `release/` 產出安裝說明 `README.md`（Chrome `chrome://extensions` 開發者模式 → 載入未封裝/解壓後載入步驟、版本、已知注意事項）
5. `.gitignore` 忽略 `release/*.zip`（zip 不入版控）；`release/README.md` 可入版控

### Commit Policy

feature branch `feat/1.4.2-W1-006-package-release`，commit 不自行 merge，PM 驗證後合併。git add 精確路徑。

### 必讀

`docs/project-conventions.md`（寫 src 前）；build:prod 移除 log 為設計（CLAUDE.md 5 建置指令）。

### 驗證

實機跑 `npm run package:release`，確認 `release/` 產出版本化 zip + README，且 zip 解壓後 manifest 完整可在 Chrome unpacked 載入。
