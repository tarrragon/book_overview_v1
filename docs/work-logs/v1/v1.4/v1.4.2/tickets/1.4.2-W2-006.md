## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 根因（修正建 ticket 時的推測）

建 ticket 時推測為「bare specifier src/ module resolution 失敗」，實機驗證後推翻：

- jest moduleNameMapper（package.json L128-134）的 ^src/(.*)$ -> <rootDir>/src/$1 正常運作，眾多測試用 require('src/...') 皆通過。
- 實際錯誤：Cannot find module 'qrcode-generator' from 'src/sync/qr-encoder.js'（qr-encoder.js L11 import qrcode from 'qrcode-generator'）。
- qrcode-generator: ^2.0.4 已宣告於 package.json L275 且在 package-lock.json，但 node_modules 未實際安裝該套件——node_modules 與 package.json/lock 不同步（環境 stale）。

### 修復

npm install --legacy-peer-deps（CLAUDE.md 環境恢復標準）對齊 node_modules。package-lock.json 僅變動 2 行（lock 早已正確，純 node_modules 補裝）。無需改 jest config 或測試 import 寫法。
