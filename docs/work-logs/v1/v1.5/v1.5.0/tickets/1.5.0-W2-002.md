## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 勘查方法

用戶在正常瀏覽器登入博客來電子書櫃，取得完整 DOM 結構（HTML source）和 API JSON 回應，提供給 PM 分析。chrome-devtools MCP 因 Cloudflare Turnstile 人機驗證無法自動化存取。

### 勘查結果摘要

1. **技術架構**：jQuery 3.7.1 MPA（非 SPA），多個獨立 JS 模組（shelf/main.js、aBook.js 等）
2. **DOM 結構**：`.bookshelf__main` > `.bookshelf__book`，書名/作者/封面/進度皆有明確 CSS class
3. **載入機制**：「看更多」按鈕分頁載入（非 infinite scroll），API 支援 offset 分頁
4. **API 回應**：JSON 格式，欄位豐富（bookId/title/author/publisher/publishDate/coverUrl/readProgress/lastReadAt/purchaseDate/format/language 等）
5. **反爬**：Cloudflare Turnstile 在首頁/登入頁；書庫頁登入後無額外反爬
6. **瀏覽器限制**：UA 檢測僅支援 Chrome/Safari

### 完整分析已寫入

`docs/bookstores/books-com-tw.md`（含 DOM 選擇器對照表、API 欄位對 BookSchemaV2 映射、適配器策略建議）

## 重現實驗結果
### 實驗方法

用戶在正常 Chrome 瀏覽器手動登入博客來電子書櫃，取得完整頁面 HTML source 和 API JSON 回應。

### 已驗證事實

1. 書庫頁 URL 確認為 `https://viewer-ebook.books.com.tw/viewer/index.html?readlist=all`
2. DOM 結構穩定：`.bookshelf__main` > `.bookshelf__book` 為書籍容器/項目
3. 分頁載入使用「看更多」按鈕（`.bookshelf__load-more`），非 infinite scroll
4. API 回傳 JSON 含 `total_records`、`current_offset`、`records[]` 完整結構
5. API `item_info` 欄位覆蓋書名/作者/出版社/出版日期/封面/進度/購買時間等
6. Cloudflare Turnstile 阻擋自動化存取（chrome-devtools MCP 無法通過）
7. 技術架構為 jQuery 3.7.1 MPA（非 React/Vue/Angular SPA）
8. 書庫頁只有 1 本書（測試帳號），但 API 和 DOM 結構完整可推斷多書情境

### 仍未驗證假設

1. 大量書籍（100+ 本）時的分頁 API 行為（每頁筆數、offset 遞增值）
2. Session/Cookie 機制細節（逾時時間、重新認證流程）
3. API 端點 URL（需從 Network tab 擷取，本次未取得）
4. 不同分類（magazine/audiobook）的 DOM 結構是否一致

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 勘查結論

博客來電子書櫃已完成初步 DOM 勘查，核心發現已寫入 `docs/bookstores/books-com-tw.md`。

### 適配器開發策略

1. **API 優先**：博客來書庫頁載入時有 JSON API 回應，欄位比 DOM 更完整。適配器應優先攔截 XHR/Fetch 取得結構化資料
2. **DOM fallback**：API 不可用時從 `.bookshelf__book` 元素解析基本欄位
3. **分頁處理**：模擬點擊「看更多」按鈕或直接呼叫 API 的 offset 分頁
4. **反爬應對**：Cloudflare 僅在登入前攔截，Chrome Extension content script 在用戶已登入的頁面注入不受影響

### 未完成項目（需後續勘查）

| 項目 | 重要性 | 建議時機 |
|------|--------|---------|
| API 端點 URL 擷取 | 高 | 開發適配器時從 Network tab 取得 |
| 大量書籍分頁行為 | 中 | 需有較多書籍的帳號測試 |
| Session/Cookie 機制 | 低 | 實作登入偵測時再調查 |
| 雜誌/有聲書 DOM 差異 | 低 | v1.5.0 聚焦電子書，其他類型延後 |

### Spawn 規劃

無需額外 spawn。未完成項目可在博客來適配器實作 ticket 中一併處理。
