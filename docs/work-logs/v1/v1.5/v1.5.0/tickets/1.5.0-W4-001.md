## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 假設

agentType 指定框架 agent 時，subagent 是否載入該 agent 的 .claude/agents/*.md 定義（含 AGENT_PRELOAD.md 規則）。

### WRAP

W：(A) 寫 Workflow script 實機驗證，(B) 讀 CC 原始碼推斷
A：不擠壓產品開發
P：Workflow script 語法錯誤導致實驗失敗 → 防護：參照系統指令範例撰寫

### 實驗設計

寫 minimal Workflow script，使用 agentType='linux' 派發一個 agent，prompt 要求：
1. 報告自身看到的 system instructions 中是否包含 agent 定義內容
2. 使用 schema 回傳結構化結果

觀察指標：subagent 是否看到 linux agent 定義中的「Good Taste」「架構品味」等關鍵字

## 重現實驗結果
### 實驗方法

寫 Workflow script 派發兩個 probe agent（parallel）：
- probe A：`agentType: 'linux'`，使用 JSON schema 結構化輸出
- probe B：無 agentType（對照組），同 schema

prompt 要求 agent 報告自身 system instructions 中是否包含 agent 定義、AGENT_PRELOAD、CLAUDE.md 相關關鍵字。

### 實驗執行

Workflow 執行 20.4 秒，2 agent，111,210 subagent tokens。

### 實驗發現

**probe A（agentType: 'linux'）結果**：
- sees_agent_definition: **true**
- agent_identity_keywords: Linux, Code Quality Enforcement Specialist, Linus Torvalds, Good Taste, 架構品味, 品質把關者, Garbage, Acceptable, Never break userspace, 實用主義
- sees_agent_preload: **true**（透過 agent 定義的 @ 引用間接載入）
- sees_project_claude_md: **true**

**probe B（無 agentType，對照組）結果**：
- sees_agent_definition: **false**
- agent_identity_keywords: []（空）
- sees_agent_preload: **false**
- sees_project_claude_md: **true**

### 已驗證結論

1. **agentType 完整繼承 agent 定義**：Workflow 的 agentType 參數確實從 .claude/agents/ 載入對應 agent 定義
2. **AGENT_PRELOAD 透過 @ 引用間接繼承**：agent 定義中的 `@.claude/agents/AGENT_PRELOAD.md` 引用被解析，PRELOAD 規則生效
3. **CLAUDE.md + rules/core/ 無論有無 agentType 都載入**：這是 CC runtime 的基礎行為，不依賴 agentType
4. **無 agentType 時 = 匿名 agent**：不載入任何 agent 定義，只有 CLAUDE.md 和 rules/core/

### 對 W3-002 結論的影響

W3-002 AC3 agentType 橋接方案設計**完全可行**——agentType subagent 不僅繼承 agent 身份，還繼承 AGENT_PRELOAD 的全部規則（語言規範、ticket 操作、工具選擇、Git 限制等）。這意味著 Workflow 內的 agentType subagent 與 Agent 工具派發的 subagent 行為一致性高。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 結論

agentType 完整繼承框架 agent 定義和 AGENT_PRELOAD 規則。實驗以 A/B 對照設計驗證，結果確定性高。

### 繼承層級總結

| 層級 | agentType 指定時 | 無 agentType 時 |
|------|----------------|----------------|
| CLAUDE.md + rules/core/ | 繼承 | 繼承 |
| .claude/agents/<type>.md 定義 | 繼承 | 不繼承 |
| AGENT_PRELOAD.md（透過 @ 引用） | 繼承 | 不繼承 |

### 對框架 Workflow 融入的意義

1. agentType subagent 行為與 Agent 工具派發一致——語言規範、工具選擇、Git 限制全部生效
2. Workflow script 可放心使用 agentType 指定框架 agent，不需在 prompt 內重複規則
3. 無 agentType 的匿名 agent 仍受 CLAUDE.md 約束，但缺少 agent 身份和 PRELOAD 規則
