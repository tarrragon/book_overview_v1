## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

更名分兩類處理，**產品名稱替換、平台引用保留**：

**機器/英文識別符**（exact，全域）：
- `readmoo-book-extractor` → `book-overview`（package name、repo URL、ZIP 命名、source/sourceApp、clone dir）
- `Readmoo Book Extractor` → `Book Overview`（creator）

**中文產品名稱**（anchored `Readmoo ` 前綴，避免誤傷 emoji-test sample）：
- `Readmoo 書庫數據提取器 (模組化版本)` → `Book Overview — 電子書庫總覽`（manifest name）
- `Readmoo 書庫提取器 (模組化)` → `Book Overview`（manifest default_title）
- `Readmoo 書庫提取器` / `Readmoo 書庫數據提取器` / `Readmoo 書庫瀏覽器` → `Book Overview`
- `Readmoo 書庫管理器` → `電子書庫總覽管理器`

**保留（平台引用，非產品名）**：`Readmoo 書庫頁面`、`Readmoo 網站/網域/分頁`、`如 Readmoo`、host permission `*.readmoo.com`、`ReadmooAdapter`、bookstore displayName `Readmoo`、`Readmoo library 頁 DOM`、`Readmoo 書籍 ID`。

**Context Bundle 缺口（已補）**：原清單未涵蓋的執行期產品名稱 SSOT 也一併更名——`src/popup/constants/ui-text.js` TITLE、`src/background/i18n/locales/zh-tw.js` name、`background.js` 啟動 log、popup 說明標題、`src/deployment/chrome-store-readiness.js` mock name、多個 `scripts/*.js` 與檔頭註解，及對應測試斷言（manifest.test.js、popup-interface.test.js、background-local-dict.test.js、chrome-store-readiness e2e）。

**未改**：歷史 work-logs（凍結記錄）；`README.md` H1「多書城書庫管理器」（不含 readmoo，已過 acceptance，但與新品牌不一致，待品牌決策）。

**跨專案**：匯出 source/sourceApp 改為 `book-overview`、creator `Book Overview`、canonical fixture 同步更新；Flutter APP 端 spec 需建後續 ticket 同步。
