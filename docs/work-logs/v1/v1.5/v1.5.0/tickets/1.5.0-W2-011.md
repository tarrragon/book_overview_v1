## Context Bundle

### 現狀分析

page-detector.js 目前硬編碼 Readmoo 域名偵測：
- L55: `location.hostname.includes('readmoo.com')` — 硬編碼域名
- `detectReadmooPage()` 方法名本身即硬編碼書城
- `isReadmooPage` 狀態變數和同名方法都綁定 Readmoo
- L131: `isExtractablePage()` 依賴 `isReadmooPage`

PlatformRegistry 已存在（W2-006 完成）：`src/content/platform/platform-registry.js`
- `detect(url)` 回傳 `{ config, createAdapter }` 或 null
- `config.name` 為書城識別碼（如 'readmoo'）

### 改動策略

1. 新增 `const PlatformRegistry = require('../platform/platform-registry')`
2. `detectReadmooPage()` 改為 `detectPlatform()` — 用 `PlatformRegistry.detect(location.href)` 偵測
3. 內部狀態 `isReadmooPage` → `isSupportedPage`，新增 `platformName` 記錄偵測到的書城
4. `getPageStatus()` 回傳結構新增 `platformName`、`isSupportedPage`，保留 `isReadmooPage` 作向後相容（= `platformName === 'readmoo'`）
5. `isExtractablePage()` 改用 `isSupportedPage`
6. `isReadmooPage()` 方法保留但標記 deprecated，內部轉呼叫 `isSupportedPage()`

### 向後相容注意

- content-modular.js L71 引用 `pageStatus.isReadmooPage`（W2-010 會同步改）
- 測試可能直接檢查 `isReadmooPage` 屬性，需確保回傳結構保留此欄位
- `detectPageType()` 的 URL 模式（library/shelf/reader）是 Readmoo 專屬，其他書城頁面類型可能不同 — 暫時保留，各書城 adapter 可覆寫
