用戶在框架外的 blog 專案使用 Claude Code 原生 Workflow tool 處理 831 個 markdown lint warning（MD036/MD040/front-matter/POS/REF），體驗到 Workflow 的批量平行處理優勢後，要求評估是否可融入本框架。

### Workflow tool 核心能力

| 能力 | 說明 |
|------|------|
| `pipeline(items, stage1, stage2, ...)` | 每項獨立流經所有階段，無 barrier |
| `parallel(thunks)` | 並行執行，全部完成才返回（barrier） |
| `phase(title)` | 進度分組，`/workflows` 即時可視 |
| `agent(prompt, opts)` | 支援 `schema`（結構化輸出）、`agentType`（自定義代理人型別）、`isolation: 'worktree'` |
| `budget.remaining()` | Token 預算感知，動態控制深度 |
| Resume | 相同 script+args 可 resume，已完成 agent 走快取 |

關鍵橋接點：`opts.agentType` 可指定框架自定義 agent type（如 `mint-format-specialist`）。

### 初步比對（四機制）

| 維度 | Workflow | Agent 單派發 | bulk-evaluate | parallel-evaluation |
|------|---------|-------------|---------------|-------------------|
| 控制流 | 確定性腳本 | PM 模型驅動 | PM 模型驅動 | PM 模型驅動 |
| 結果回收 | 程式化 JSON schema | PM 讀 transcript | 寫入子 ticket | PM 彙整 |
| Ticket 追蹤 | 無 | 有 | 有 | 有 |
| Agent 型別 | 匿名/agentType | 框架定義 | 框架定義 | 固定三人組 |
| Resume | 有 | 無 | 無 | 無 |

### WRAP 三問

W（候選方案）：A 全面整合（PM 決策樹新增路由）、B 有限整合（特定 Skill 內部用）、C 不整合
