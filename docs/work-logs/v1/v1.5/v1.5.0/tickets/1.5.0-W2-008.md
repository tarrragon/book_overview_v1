### Context Bundle

**目標**：readmoo-adapter.js（2365 行）拆分為 Readmoo 專屬邏輯（<1000 行）+ 共用模組。

**前置完成**：
- W2-007 已擴充 PlatformAdapterInterface（新增 4 抽象方法 + 通用 scroll helpers）
- W2-001 ANA Solution 定義架構方向

**拆分策略**（三個新模組）：

1. `src/content/platform/adapter-utils.js` — 跨書城通用工具函式：
   - 文字處理：sanitizeText, cleanHtmlAndMaliciousContent, processUrlEncoding, normalizeTextContent, limitTextLength
   - URL 工具：isUnsafeUrl, extractFilenameFromUrl, extractDomainFromUrl
   - 型別工具：isNullOrUndefined, isStringType, requiresSpecialHandling, safeConvertToString, safeStringify
   - 錯誤處理：handleWithFallback（參數化 logger）

2. `src/content/platform/stable-id-generator.js` — 穩定書籍 ID 生成框架：
   - 核心：generateStableBookId, generateStableBookIdWithInfo, validateAndSanitizeInputs
   - 策略：applyIdGenerationStrategies/WithInfo, tryReaderStrategy, tryTitleStrategy, tryCoverStrategy, createFallbackId
   - 驗證：validateTitleInput, validateCoverUrlInput, generateTitleBasedId
   - Readmoo 專屬（extractCoverIdFromUrl, validateReadmooDomain, extractIdFromCoverPath, extractIdFromFilename）作為可配置的 coverIdExtractor 注入

3. `src/content/adapters/readmoo-scroll-loader.js` — Readmoo 捲動載入邏輯：
   - 容器偵測：findScrollContainer, findLoadMoreButton, findScrollableAncestor
   - 捲動控制：_scrollStep, _scrollWindowToBottom, _clickLoadMoreButton, _waitMs
   - 封面捲動：_countPlaceholderCovers, _waitForPlaceholderConvergence, _scrollThroughAllItemsForCovers, _readScrollY
   - 載入流程：loadAllBooksLazy, _measureBooks, parseLibraryTotal
   - Observer：waitForBookElements, waitForRenderSettle, _observeChildListOnce
   - describeStopReason

**readmoo-adapter.js 保留**（~930 行）：
- MessageDictionary + SELECTORS + 常數
- getBookElements（DOM 查詢 + fallback 策略）
- parseBookElement + extractAllBooks（批次提取）
- extractHrefFromElement, extractBookIdFromPrivacy, isPlaceholderUrl, extractCoverAndTitle, hasRequiredFields
- extractBookId, extractProgressFromContainer, extractBookTypeFromContainer
- getStats, parseDocument, findBookContainers, validatePage, getSupportedUrls, reset, getAdapterInfo
- logError（使用本地 logger）

**測試影響**：readmoo-adapter.test.js 等 7 個測試檔案透過 createReadmooAdapter() factory 測試，factory 返回的 object 介面不變（方法代理到新模組），測試不需修改。

**Commit Policy**：feature branch `feat/1.5.0-W2-008-refactor-readmoo-adapter`，完成後 commit。

**驗收**：
1. 通用捲動載入邏輯移至 PlatformAdapterInterface（W2-007 已完成）
2. Readmoo 專屬 DOM 選擇器和解析邏輯保留
3. readmoo-adapter.js 行數降至 1000 行以下
4. npm test 100% 通過（265 suites, 5864 tests）
