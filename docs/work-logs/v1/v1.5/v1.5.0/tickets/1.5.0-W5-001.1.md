## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

CC 2.1.195 修復「hook matchers with hyphenated identifiers accidentally substring-matching — they now exact-match.」（官方 CHANGELOG 原文，例：`code-reviewer`、`mcp__brave-search`）。**注意**：ticket 描述寫 2.1.194，實際落在 2.1.195；breaking change 範圍**僅限「含連字號（hyphen `-`）的 matcher」**——這類 matcher 過去以子串（substring）比對，修復後改為 exact-match。非連字號 matcher（如 `Edit`、`Bash`）的比對行為**未受此修復影響**。

> 修正父票 T1 與本票原始假設：原假設（來自 team-lead 訊息）認為 `Bash` 等一般 matcher 也可能曾意外子串匹配。官方 changelog 將 bug 範圍明確限縮於「hyphenated identifiers」，故 `Bash`/`Edit`/`Task` 等無連字號 matcher 不在此 breaking change 範圍內。

### 影響範圍

掃描範圍：`.claude/settings.json`（22 個 hook 群組）、`.claude/settings.local.json`（無 hooks 區塊）、`.claude/templates/settings-local-template.json`（hooks 僅註解無 matcher）、`.claude/skills/*/hooks/`（skill hooks 透過 settings.json 註冊，非自帶 matcher frontmatter）。

結論：**全部 22 個 matcher 皆使用底線（`__`）或 CamelCase，零個含連字號**，故此 breaking change **不影響本專案任何 hook**。

### 相關 Error Pattern

無直接對應 PC。屬平台版本相容性掃描，父票 1.5.0-W5-001 T1 張力項的落地驗證。

## 重現實驗結果
### 實驗方法

1. 取得 breaking change 精確語意：WebFetch 官方 CHANGELOG（anthropics/claude-code）取 2.1.195 條目原文，避免臆測（tool-output-trust 規則）。
2. 用固定值命令（Python json 解析）列出所有 hook matcher，非靠記憶（規則 3 交叉驗證）。
3. 對每個 matcher 值檢查是否含連字號（唯一觸發此 breaking change 的條件）。

### 實驗執行

- 官方 CHANGELOG 2.1.195 原文：「Fixed hook matchers with hyphenated identifiers (e.g. `code-reviewer`, `mcp__brave-search`) accidentally substring-matching — they now exact-match.」（2.1.191 無 matcher 相關條目，確認非該版）。
- `python3 json.load` 解析 `.claude/settings.json` → 22 個 hook 群組 matcher（PreToolUse 12、PostToolUse 5、其餘 SessionStart/UserPromptSubmit/SubagentStop/Stop/PostToolUse 為 5 個無 matcher 事件）。
- `.claude/settings.local.json`：無 hooks 區塊（0 群組）。
- template：hooks 僅含 `_COMMENT_HOOKS` 註解，無 matcher。
- 逐值檢查連字號：`'-' in matcher` → 22 個全部為 False。

### 實驗發現

**已驗證事實**：
1. breaking change 觸發條件 = matcher 字串含連字號（hyphen）。這是官方 changelog 明確限定範圍，非泛指所有 substring 行為。
2. 本專案全部 22 個 matcher 值：`Bash` / `WebFetch` / `WebSearch` / `mcp__dart__run_tests` / `mcp__serena__replace_content|...|mcp__serena__safe_delete_symbol`（pipe 全名 alternation）/ `Read` / `Edit` / `Write` / `Task` / `Agent` / `TodoWrite` / `AskUserQuestion`——皆為底線或 CamelCase，零連字號。
3. 故本專案不受 2.1.195 exact-match 修復影響。

**未驗證假設（不影響結論）**：無法在此 session 直接測試 CC 內部 matcher 引擎行為；結論建立於官方 changelog 語意 + matcher 字串靜態特徵（連字號有無），此判據充分且無歧義。
