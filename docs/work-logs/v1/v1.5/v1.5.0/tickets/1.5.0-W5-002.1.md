# Execution Log

## Task Summary

修 task-dispatch-readiness-check.py:204 與 askuserquestion-reminder-hook.py:66 的 tool_name != Task 早退守衛，改為 not in (Agent, Task)

---

## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

CC subagent 派發工具表面名已由 `Task` 改為 `Agent`，但 `task-dispatch-readiness-check.py:204` 與 `askuserquestion-reminder-hook.py:66` 內部以字面 `tool_name != "Task"` 過濾，導致每次真實 Agent 派發都在實質邏輯（甚至 log 寫入）之前早退，兩 hook 功能性死亡（詳見上游 ANA 1.5.0-W5-002）。

### 影響範圍

- `.claude/hooks/task-dispatch-readiness-check.py`（代理人分派正確性檢查，line 204 守衛）
- `.claude/hooks/askuserquestion-reminder-hook.py`（多任務派發提醒，line 66 守衛）
- 測試：`.claude/hooks/tests/test_agent_dispatch_check.py`（新增 main() 守衛回歸測試）、`.claude/hooks/tests/test_askuserquestion_reminder_hook.py`（新檔）

### 相關 Error Pattern

概念對應 quality-baseline 規則 1 邊界「測試綠燈不等於 Runtime 正確」；上游 ANA 1.5.0-W5-002 建議評估是否升級為新 error-pattern「hook 內部 tool_name 字面守衛與 matcher 投遞工具名漂移」，本票不重複建立，交由 PM 判斷。

---

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

比照 `agent-prompt-length-guard-hook.py` 已採用的雙名接受模式，將兩處守衛由 `!= "Task"` 改為 `not in ("Agent", "Task")`：

- `task-dispatch-readiness-check.py:204`：`if tool_name not in ("Agent", "Task"): sys.exit(0)`
- `askuserquestion-reminder-hook.py:66`：`if tool_name not in ("Agent", "Task"):`

新增 7 個回歸測試（3 個於既有 `test_agent_dispatch_check.py`，4 個於新建 `test_askuserquestion_reminder_hook.py`）驗證 Agent 通過、Task 不 regression、非 Agent/Task 仍早退。並以真實 hook 契約（非 pytest mock）直接呼叫兩支 hook 確認 hook-logs 出現實際執行紀錄（見 Test Results）。
