## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 改動摘要

1. **manifest.json**：content_scripts.matches 新增 `*://*.books.com.tw/*`
2. **scripts/build.js**：
   - 新增 `validateManifestMatchPatterns()` 純函式，比對 manifest content_scripts.matches 與 PlatformRegistry.getAllMatchPatterns()
   - build flow 開頭（esbuild check 後）執行一致性驗證，缺少 patterns 則 exit(1) 阻斷 build
   - 額外 patterns（如 `*://readmoo.com/*` 主網域）以 warn 提示不阻斷
   - 函式已匯出供測試使用

### 設計決策

- 驗證放 build.js 而非 validate-manifest.js：後者聚焦 build/production 靜態結構驗證，本驗證需 require PlatformRegistry 動態取 patterns，屬 build flow 前置 gate
- missing = error（阻斷）、extra = warn（不阻斷）：missing 代表書城 content script 不會注入是致命問題；extra 可能是歷史相容 patterns（如 readmoo.com 主網域）不需阻斷
