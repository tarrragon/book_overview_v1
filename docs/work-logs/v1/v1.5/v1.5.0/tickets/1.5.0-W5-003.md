## Problem Analysis
<!-- Schema[DOC/Problem Analysis]: 選填（若 DOC 起因於缺陷或盤點結論可填）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

`.claude/skills/chrome-extension-mcp-debug/SKILL.md` 撰寫時參照的 chrome-devtools MCP 工具名與現行工具集（session deferred tools 清單的 `mcp__chrome-devtools__*` 前綴）不一致：`getExtensions`/`getConsoleMessages`/`getNetworkRequests`/`takeScreenshot`/`snapshot` 分別對應現行 `list_extensions`/`list_console_messages`/`list_network_requests`/`take_screenshot`/`take_snapshot`；Workflow D-sw 引用的 `getExtensionLogs` 在現行工具集中已不存在。

### 影響範圍

`.claude/skills/chrome-extension-mcp-debug/SKILL.md` 全文（三個前置概念、Workflow A/B/C/D-popup/D-cs/D-sw/E-console/E-network、反模式速查）；照抄舊工具名呼叫會直接失敗。

### 相關 Error Pattern

未執行 /error-pattern query（本 ticket 為單純工具名漂移修正，非新錯誤模式）。

## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->

依 grep 全量掃描逐一替換 SKILL.md 內舊工具名為現行名稱：`getExtensions` → `list_extensions`、`getConsoleMessages` → `list_console_messages`、`getNetworkRequests` → `list_network_requests`、`takeScreenshot` → `take_screenshot`、`snapshot` → `take_snapshot`。

Workflow D-sw（Debug service worker）pipe 模式步驟改寫：原「`getExtensions` 取 SW URL → `getExtensionLogs --extensionId <id>` 取 SW console log」改為「`list_extensions` 取 extension ID/SW URL → `list_pages` 找到 SW target → `select_page` 切換 focus 至該 SW target → `list_console_messages` 讀取 SW console log」。三個前置概念表、Workflow E-console 對照表同步更新此流程說明。

版本footer（Last Updated / Version / Source）與 frontmatter `metadata.version` 同步更新為 1.1.0，記錄本次工具名漂移修正。
