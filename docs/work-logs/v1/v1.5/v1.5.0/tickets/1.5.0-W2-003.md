## Solution

### Phase 1 功能設計：博客來 PlatformAdapter 適配器

#### 1. 模組職責

`BooksComTwAdapter` 繼承 `PlatformAdapterInterface`，實作博客來電子書庫的資料提取。
策略：API 回應攔截優先，DOM 解析作 fallback。

#### 2. 實作方法清單

| 方法 | 實作策略 | 說明 |
|------|---------|------|
| `getPlatformName()` | 回傳 `'books-com-tw'` | 書城識別碼 |
| `getLibraryUrl()` | 回傳 `'https://viewer-ebook.books.com.tw/viewer/index.html?readlist=all'` | 書庫頁 URL |
| `requiresLogin()` | 回傳 `true` | 書庫頁需登入 |
| `getLoginCheckSelector()` | 回傳 `.bookshelf__main` | 書架容器存在表示已登入 |
| `isValidDomain(url)` | 檢查 hostname 含 `books.com.tw` | 域名驗證 |
| `getPageType()` | 從 URL query `readlist` 判斷 | library/unknown |
| `isExtractablePage()` | `getPageType() === 'library'` | 可提取判斷 |
| `checkPageReady()` | 等待 `.bookshelf__main` 出現 | 頁面就緒 |
| `findBookContainer()` | `document.querySelector('.bookshelf__main')` | 書籍容器 |
| `getBookElements()` | `document.querySelectorAll('.bookshelf__book')` | 書籍元素 |
| `getBookCount()` | `getBookElements().length` | 書籍數量 |
| `parseBookElement(el)` | 從 DOM 元素提取書名/作者/封面/進度 | DOM fallback 路徑 |
| `extractAllBooks()` | API 優先 + DOM fallback 雙路徑 | 核心提取方法 |
| `extractBookData(el)` | 委託 `parseBookElement` + `sanitizeData` | 完整資料提取 |
| `sanitizeData(data)` | URL/文字清理 | 資料安全性 |
| `getStats()` | 提取統計（成功/失敗/跳過數） | 統計資訊 |
| `reset()` | 重置內部狀態 | 狀態重置 |

#### 3. API 攔截策略

博客來書庫頁載入時會發起 XHR/Fetch 取得書籍 JSON。
Content Script 無法直接攔截 XHR（isolated world），需透過：

**方案 A（推薦）**：在 `extractAllBooks()` 中主動觸發分頁載入（點擊「看更多」按鈕），每次載入後從 DOM 解析新出現的書籍元素。此方案不需攔截 API，完全走 DOM 路徑。

**方案 B**：注入 page script 攔截 XHR — 複雜度高，需跨 world 通訊，不適合 v1.5.0。

**結論**：v1.5.0 先走 DOM 解析路徑（方案 A），未來若需 API 欄位再加 page script 攔截。

#### 4. 分頁載入策略

```
extractAllBooks():
  1. checkPageReady() — 等待容器出現
  2. parseCurrentPageBooks() — 解析當前頁面書籍
  3. while (hasLoadMoreButton()):
     a. clickLoadMore() — 點擊「看更多」
     b. waitForRenderSettle() — 等待新元素渲染
     c. parseNewBooks() — 解析新出現的書籍
  4. 回傳所有書籍陣列
```

| 書籍連結 | bookId | 從 URL 或 data attribute 提取 |
