## Problem Analysis

### 問題根因

CC 2.1.198 新增行為：從 `claude agents` dashboard 啟動的背景 session 在 worktree 完成程式碼工作後自動 commit、push 並開 draft PR，不再停下詢問。此行為與框架「PM 驗收後才 merge / 外部化動作需 PM gate」的預期存在潛在張力，需判定是否衝突。

### 影響範圍

- `.claude/rules/core/pm-role.md` 分工原則（代理人 commit 政策）
- `.claude/pm-rules/worktree-operations.md`（worktree 派發與清理流程）
- `.claude/pm-rules/parallel-dispatch.md`（dispatch-plan commit policy 欄位）
- 僅在「使用 claude agents dashboard 派發程式碼工作」情境下觸發；現行 Agent tool 派發模式不受影響

### 相關 Error Pattern

無直接對應 error pattern。相關規範：PC-036/PC-039（worktree 產出遺失，本項為其反向情境——產出被過早外部化而非遺失）。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### WRAP 分析

### W（擴增選項）
1. 判定不衝突（現行派發不經 claude agents dashboard）
2. 判定衝突並禁用背景 agent 派發
3. 補文件邊界註記（採用）
4. 增設 hook 攔截非 PM 發起的 push

### R（現實檢驗）
- 2.1.198 原文範圍：「Background agents launched from `claude agents` now commit, push, and open a draft PR when they finish code work in a worktree」——僅限 claude agents dashboard 啟動的背景 session
- 本框架派發模式：PM session 內 Agent tool 派發 subagent，不經 claude agents dashboard，新行為不觸發
- 框架 commit policy 本文：「代理人做 GREEN 實作與 git commit」（pm-role.md 分工原則）——代理人在 feature branch commit 本就合規
- 真正張力點：push + draft PR 是外部化動作（PM 驗收前不應對外發布），但 draft PR 可撤、commit 在隔離 branch，衝擊為中低

### A（拉開距離）
- 不動作的成本：若未來改用 claude agents dashboard 派發程式碼工作，draft PR 會未經 PM gate 出現在 GitHub。機率低（框架無此用法）、影響中（draft 可關閉、可逆）
- 過度反應的成本：禁用或 hook 攔截會增加維護負擔，防護一個不存在的路徑

### P（絆腳索）
- 絆腳索：GitHub 上出現 PM 未發起的 draft PR → 立即重新評估，屆時考慮 worktree.bgIsolation 或停用 dashboard 派發

### 結論

**不衝突（附邊界條件）**。現行框架派發模式（PM session 內 Agent tool）不觸發 2.1.198 自動 commit/push/PR 行為。已在 worktree-operations.md 補邊界註記：若未來採用 claude agents dashboard 派發程式碼工作，需先重新評估此項（絆腳索觸發條件同上）。
