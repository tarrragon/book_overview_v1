### 選擇位置的理由

- **核心規則在 parallel-dispatch.md**：此檔案是派發決策規則的主要入口，驗證類任務是一種派發決策
- **詳細 SOP 在 background-dispatch-rules.md（references）**：符合 progressive disclosure，常駐載入的 pm-rules 只放核心規則，細節供按需讀取
- **command-routing.md 增加一列**：確保 PM 接到命令後能從分工路由表直接看到「驗證類」應該怎麼處理
- **askuserquestion-rules.md 新增規則 5**：防止未來讀者誤用通用觸發原則（規則 1）對驗證類任務發動詢問
