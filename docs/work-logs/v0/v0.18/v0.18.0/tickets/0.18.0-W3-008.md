**修改設計建議**：

1. **核心規則位置** — `decision-tree.md` 新增一個識別層（或條件分支），在 PM 接收任務時判斷是否為「驗證類」：
   - 識別特徵：Ticket 要求「執行 X 並產出報告」「驗證 AC 實況」「跑測試/掃描/打包/建置」
   - 預設行動：直接建子 Ticket → 寫 Context Bundle → 背景派發，**不詢問用戶**
   - 例外：驗證結果會直接影響派發策略的根本決策

2. **詳細規範位置** — `background-dispatch-rules.md` 新增「驗證類任務自動派發」章節：
   - 驗證類任務清單（跑測試/掃描/建置/打包/AC 實況驗證）
   - 識別關鍵詞（從 Ticket What/How 判斷）
   - 流程 SOP：(a) PM 建子 Ticket (b) 寫 Context Bundle 到父 Ticket (c) 背景派發 (d) PM 立即切其他準備 (e) 收到通知再驗收

3. **decision-tree 分支更新**：
   - 找到「接收任務」路由，插入「是否為驗證類？」判斷節點
   - Yes → 自動派發流程
   - No → 既有流程
