## 實驗設計需求（用戶指示）

### 實驗架構

針對每個 W17-051 分類為「降級候選」的代理人，執行以下實驗流程：

| 步驟 | 動作 |
|------|------|
| 1 | 基於代理人職能設計 1 個實驗任務範本 |
| 2 | 建立 6 個內容一模一樣的子 ticket |
| 3 | 前 3 個 ticket 指派原 model 的代理人執行 |
| 4 | 後 3 個 ticket 指派降級 model 的代理人執行 |
| 5 | 6 個 ticket 全數完成後，派多視角代理人分析兩組產出差異 |

### 判準

| 發現類型 | 判定 | 動作 |
|---------|------|------|
| 錯字、風格不一致、語氣問題 | 不重要 | 不回滾，由二次審查層（basil-writing-critic）修正 |
| 邏輯錯誤、判斷錯誤、執行能力下降（如回合耗盡變頻繁、關鍵步驟遺漏） | 重要 | 回滾到原 model |
| 無顯著差異 | 通過 | 確認可降級 |

### 降級候選清單（來自 W17-051 linux 分類 + ginger 修正）

| 批次 | 目標 model | 代理人數 | 實施先後 |
|------|----------|---------|---------|
| Haiku | haiku | 3（mint、thyme-documentation、rosemary） | 先 |
| Sonnet | sonnet | 13（參見 W17-051 Solution） | 後 |
| 邊界 | sonnet（ginger 建議延長監測期） | 3（cinnamon、lavender、thyme-extension） | 最後 |

### 本 ANA 產出需求

1. 實驗任務範本設計原則（如何基於職能設計可複現任務）
2. 多視角分析規範（哪些視角、對比哪些維度）
3. 評分標準（量化錯字/邏輯/執行能力的差異判別門檻）
4. 下游衍生 ticket 結構（每代理人 6 IMP + 1 分析 ANA 的命名與依賴關係）

### 成本認知

用戶明確表示：此實驗會浪費相當多的額度（16 代理人 × 6 IMP + 16 分析 ANA ≈ 112 張 ticket），但可明確記錄每個代理人真實能力需求。
