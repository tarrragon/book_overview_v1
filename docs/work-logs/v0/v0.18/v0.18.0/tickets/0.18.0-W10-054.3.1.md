## Solution

### 實作方式

採用批次 patch 策略（PM 前台 Bash heredoc 執行 inline Python 腳本），理由：
- subagent 對 `.claude/` Edit 被 CC runtime 擋（ARCH-015）
- 37 處呼叫模式統一（`subprocess.run([...], capture_output=True, text=True, ...)`）
- 逐個 Edit 工具呼叫 37 次認知負擔過重，且易失誤

### 批次 patch 邏輯

1. 正則比對 `subprocess\.(run|Popen|check_output)\s*\(`
2. 找到匹配括號（支援字串轉義與 triple-quoted strings）
3. 檢查 call body 是否已含 `encoding=`，若有則跳過
4. 優先在 `text=True,` 後插入 `encoding="utf-8", errors="replace"`，次選 `capture_output=True,`
5. 依原始縮排決定同行或換行插入
6. Reverse-order 套用 patch（避免 offset 失效）
