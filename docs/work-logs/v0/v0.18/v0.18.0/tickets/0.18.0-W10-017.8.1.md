## Solution

### 變更內容

`.claude/skills/ticket/ticket_system/lib/checkpoint_state.py` 的 `checkpoint_state()` 主函式（Step 2 組半成品 state 處）：

- 移除 `unmerged_worktrees=list(worktrees)` → 改為 `unmerged_worktrees=worktrees`
- 移除 `in_progress_tickets=list(in_progress)` → 改為 `in_progress_tickets=in_progress`
- 保留 `data_sources=dict(errors)`（errors 後續被 `_write_metrics_log` 讀取，需隔離以保證 dataclass 欄位不受 log 端影響）

### 理由（linux L8）

`worktrees` 與 `in_progress` 來自 `SAFE_CALL` 返回的新 list（`_read_git_worktrees` / `_query_in_progress_tickets` 各自建構新 list，fallback 也是 `[]` 字面量）。函式內部未再修改這兩個區域變數，也不會洩漏至其他別名，因此 `list()` 複製為冗餘。

`errors` 則在 dataclass 建構後仍被 `_write_metrics_log` 同步使用，保留 `dict()` 防止語意耦合（dataclass snapshot 與 log 觀測點分離）。

### 變更行數

-2 / +1（diff = -2 行 list() 複製，其中兩行 `list(x)` 縮回為 `x`，語意等價）。實際 diff：

```
-        unmerged_worktrees=list(worktrees),
-        in_progress_tickets=list(in_progress),
+        unmerged_worktrees=worktrees,
+        in_progress_tickets=in_progress,
```
