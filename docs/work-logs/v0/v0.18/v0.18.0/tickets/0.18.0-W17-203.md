### 變更摘要

`.claude/pm-rules/askuserquestion-rules.md` 新增規則 7「多子任務情境必須含「平行派發」選項」，含 Why/Consequence/Action 三明示結構 + 判斷流程 + 可平行/不可平行範例 + 反模式表格 + 與其他規則銜接段落。

### 設計決策

| 決策點 | 選擇 | 理由 |
|--------|------|------|
| 規則位置 | 規則 6 後、題型判別前 | 與「決策品質防護」概念群組（規則 6 同類） |
| 觸發門檻 | 選項含 ≥ 2 pending 子任務 / 兄弟 | 單一 ticket 不存在平行決策；3+ 由 PC-137 限制處理 |
| 不可平行表達 | 顯式列「無可平行派發」選項並說明 | 對齊規則 1（禁止隱性決策） |
| 部分可平行 | 列具體子集 + 衝突項另列 | 避免 AB-test 被迫選錯誤組合 |
| Hook 偵測（acceptance 4） | 留至 W17-203.1 落地 | 規則層先穩定，CLI 強制層由子任務承接 |
