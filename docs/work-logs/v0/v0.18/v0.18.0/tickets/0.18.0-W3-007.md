## Problem Analysis

### Context Bundle

**背景**：IMP-055（PostToolUse Hook stdout JSON 格式）於 2026-04-12 再次發生，兩個 matcher=Agent 的 Hook 雖有 json.dumps 但結構不完整：
- active-dispatch-tracker-hook.py：缺外層 hookSpecificOutput 包裹
- agent-dispatch-logger-hook.py：DEFAULT_OUTPUT 缺 hookEventName 欄位

兩者皆觸發 `JSON validation failed` CLI error。已於 commit bac38ac4 修復。

**根因推測**：既有 Hook 開發規範未明確要求「半結構化 JSON 也會失敗」，開發者容易寫出缺欄位的半成品。既有其他 Hook 可能有相同問題但尚未觸發（取決於 matcher 觸發頻率）。

**掃描範圍**：
1. 所有 settings.json 中註冊的 PostToolUse Hooks（Bash/Edit/Write/Agent/mcp__dart__run_tests matcher）
2. 所有 PreToolUse Hooks
3. 重點檢查 stdout 輸出路徑：是否有 print(...) 純文字、json.dumps 結構是否完整

**驗證腳本設計提示**：
```bash
echo '{"tool_input":{},"tool_response":{}}' | python3 HOOK.py | python3 -c '
import sys, json
d = json.load(sys.stdin)
assert "hookSpecificOutput" in d, "missing hookSpecificOutput"
assert "hookEventName" in d["hookSpecificOutput"], "missing hookEventName"
print("OK")
'
```

**規範更新位置**：
- .claude/references/hooks/ 下的 Hook 開發指南（若有）
- .claude/rules/core/quality-baseline.md 規則 4（Hook 可觀測性）可能需延伸提到 JSON schema

**禁止事項**：
- 不要修改 Hook 的業務邏輯，只修 JSON 輸出結構
- 修復時保留既有 logger 行為（stderr 輸出不變）
- 不要碰 SessionStart Hook（它們的 stdout 不需要 JSON）

**參考資料**：
- IMP-055 錯誤模式：.claude/error-patterns/implementation/IMP-055-hook-stdout-plain-text-breaks-json-validation.md
- memory feedback_hook_stdout_must_be_json
