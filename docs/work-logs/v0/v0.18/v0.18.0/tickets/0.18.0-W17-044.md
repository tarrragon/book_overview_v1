### 問題根因

W17-035 進一步分析確認：PC-040 / PC-065 的共同 meta 根因是 PM 缺少可直接套用的短 prompt skeleton 與 dispatch-plan。現有 hook 能阻擋錯誤，但不提供正確派發形狀；現有 agent-dispatch-template 完整但偏長，容易與 30 行限制產生張力。

### 相關 Error Pattern

- PC-040：Context 存於 Prompt 而非 Ticket。
- PC-065：PM 並行派發多代理人時 prompt 模板遺漏 Ticket ID 格式。
- PC-092：並行 agent shared git index race，需在 dispatch-plan 保留 commit policy / precise staging。
- PC-104：Agent execution boundary misjudgment，支持不做 batch dispatch CLI。

## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->

### 變更摘要

- `.claude/references/agent-dispatch-template.md`
  - 新增 3 個短 prompt snippets：單任務、並行多任務、group coordinator。
  - 新增 dispatch-plan template，欄位包含 `ticket` / `agent` / `files` / `deps` / `context source` / `commit policy` / `run mode`。
  - 明確 dispatch-plan 是 orchestration description，不是 batch dispatch CLI。
- `.claude/pm-rules/parallel-dispatch.md`
  - 新增「Dispatch-Plan 先行」章節。
  - 明確區分 dispatch-plan 與 batch dispatch CLI，並禁止首輪新增 batch dispatch CLI。
- `.claude/pm-rules/dispatch-gate.md`
  - Context Bundle gate 補 PC-065 第一行 Ticket ID 檢查。
  - 多任務 / group / spawned follow-up 補 dispatch-plan 檢查。
- `.claude/skills/ticket/SKILL.md`
  - 補充 `batch-create` 邊界：只建立 tickets，不派發 agents。
  - 補充 group / children / spawned / 2+ agents 派發前先寫 dispatch-plan。
