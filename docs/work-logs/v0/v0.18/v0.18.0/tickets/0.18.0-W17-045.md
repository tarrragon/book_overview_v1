## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 根因分析

事件：2026-04-22，dispatch-active.json 和 hook-state/wrap-tripwire-state.json 被 sync-push 推送至共享 repo，ccsession 專案 sync-pull 後 session state 遭覆蓋汙染。

四層結構性缺口：
1. 分類知識缺失：無明文定義「哪些型別不該 sync」
2. 流程缺失：新增 .claude/ 機制時無 sync 評估 checklist
3. 兩端不對稱：push/pull 各自維護排除清單，容易只改一端
4. 無自動偵測：錯誤等到別的專案 pull 才發現

### 防護設計

三方向防護：
A. 文件規範（W17-045.1）：sync 腳本四類分類注解 + sync-exclusion-guide.md
B. error-pattern（W17-045.2）：PC-109 記錄失敗模式供未來對照
C. 自動偵測（W17-045.3）：SessionStart Hook 掃描未排除的 .json/.yaml

場景 #17 確認：PC-109 由 W17-045 建立並修復，不需額外 Ticket。
