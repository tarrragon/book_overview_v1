## Problem Analysis

### 問題根因

`_verify_handoff_dependencies` (handoff.py:174) 使用**當前 Ticket 的 version 參數**載入依賴：

```python
for dep_id in blocked_by:
    dep_ticket = load_ticket(version, dep_id)  # <- version 是當前 Ticket 的版本
    if not dep_ticket:
        incomplete_deps.append(f"{dep_id} (未找到)")
```

當依賴是跨版本 Ticket（例如 v0.18.0 的 Ticket 依賴 v0.17.4 的 Ticket），`load_ticket("0.18.0", "0.17.4-W1-002")` 會回傳 None（因為 v0.18.0 目錄下不存在此 Ticket 檔案），導致被誤判為「未找到」而阻擋 handoff。

### 影響範圍

- 檔案：.claude/skills/ticket/ticket_system/commands/handoff.py (_verify_handoff_dependencies, L147-193)
- 所有含跨版本 blockedBy 的 handoff 命令
- 跨版本依賴場景（舊版本殘留的依賴、多版本平行開發）

### 相關 Error Pattern

與 IMP-007/0.17.4-W1 類似：跨版本資料查找時需從 ID 萃取版本，不可假設所有依賴都在當前版本。

### Context Bundle

**修復策略**：

```python
from ticket_system.lib.ticket_validator import extract_version_from_ticket_id

for dep_id in blocked_by:
    # 從 dep_id 萃取版本（跨版本依賴支援）
    dep_version = extract_version_from_ticket_id(dep_id) or version
    dep_ticket = load_ticket(dep_version, dep_id)
    ...
```

**關鍵工具**：
- extract_version_from_ticket_id(id) 回傳版本字串或 None（ticket_validator.py:14）
- 格式錯誤 ID（無 "-W"）→ fallback 到當前 version（保持既有行為）

**既有測試位置**：tests/test_handoff_command.py::TestVerifyHandoffDependencies (L429+)

**fixture 限制**：既有 `temp_version_tickets` 只建立 v0.31.0 目錄，跨版本測試需在測試內額外建立另一版本目錄
