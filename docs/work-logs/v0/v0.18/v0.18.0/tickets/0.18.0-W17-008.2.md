## Problem Analysis

### 問題根因

`ticket track <subcommand>` 註冊清單缺 `show`：
- `commands/track.py` argparse subparsers 已有 `full`、`summary`、`query`，但無 `show`
- top-level `ticket show <id>` 已存在（`commands/show.py`，含渲染功能）
- 缺口：`ticket track show <id>` 反 Unix 慣例（git/docker/kubectl 慣有 show）

### 影響範圍

- `.claude/skills/ticket/ticket_system/commands/track.py` — 在 subparsers 加 `show` 子命令，handler 指向 `execute_full`
- `.claude/skills/ticket/references/track-command.md` — 列 show 為 full alias 並補範例

- 採 alias 而非分離 handler：`show` → `execute_full`，避免雙軌維護

- 保留 `full` 不刪除：向後相容（PC-105 防護同樣適用）
