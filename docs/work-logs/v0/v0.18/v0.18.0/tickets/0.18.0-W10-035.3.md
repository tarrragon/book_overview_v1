## WRAP 四階段分析（PM 前台）

### W（Widen）—— 降級機制候選

| 候選 | 機制 | 優點 | 缺點 |
|------|------|------|------|
| 1. 提醒改警告 | Hook 輸出從 stderr error 改為 info/debug | 低摩擦、易回退 | 可能漏判真實錯誤 |
| 2. 同步改非同步 | PreToolUse 改為 PostToolUse 或 Stop | 不阻擋流程 | 失去即時攔截能力 |
| 3. 頻率降級 | Hook 從每次觸發改為抽樣（每 N 次） | 大幅降低摩擦 | 需設計抽樣策略 |
| 4. 條件降級 | 依 Ticket type/phase 過濾（如 DOC 不觸發某 Hook） | 精準 | 需 Hook 感知 Ticket context |
| 5. 完全移除 | 連續 N 次無錯 Hook 直接廢除 | 徹底 | 不可回退、失去防護 |

**偏好組合**：候選 4（條件降級）+ 候選 1（提醒改警告）= 先收斂觸發條件，高誤報再降為 info
