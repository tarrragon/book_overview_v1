**情境 A：引號內引用（Quoted Reference）**

PM prompt 用引號包住禁止詞彙以引述 agent definition 或規則文件說明，無實際執行意圖。

- 範例 1：`請遵守 thyme-python-developer 定義中的『禁止實作程式碼』邊界`

- 範例 4：`說明：本 agent 不可『修改規格』，詳見 .md 第 X 節`

**情境 B：否定語意（Negation Context）**

PM prompt 明確用否定詞框架要 agent **不**做該動作，屬主動宣告邊界而非要求越界。

- 範例 1：`請勿實作程式碼，僅產出設計規格`

### 設計策略

在 `_detect_keyword_conflicts` 產出 conflicts 後、寫入 events.jsonl 前，加入白名單過濾層 `_filter_whitelisted_conflicts(prompt, conflicts) -> conflicts`。白名單規則以可擴充清單組織，每條規則為純函式 `Rule(prompt, match_start, match_end) -> (is_whitelisted: bool, reason: str)`。

### 白名單規則清單（初版）

**規則 A：引號包圍偵測 `_is_quoted_match`**

匹配片段外層被以下任一引號包圍視為引用：

- 中文單/雙引號：`『』`、`「」`
- 英文引號：`"..."`、`'...'`、反引號 `` `...` ``
- Markdown 粗體：`**...**`（僅當內容為禁止詞彙標籤時）

偵測：從 match_start 向前找最近的開引號字元，從 match_end 向後找對應閉引號字元，中間不能跨行。

**規則 B：否定前綴偵測 `_has_negation_prefix`**

匹配位置前 10 字元內（同句內）含否定詞視為主動宣告邊界：

- 中文否定詞：`不要` `請勿` `禁止` `避免` `不得` `不可` `切勿` `勿`
- 英文否定詞（case-insensitive）：`don't` `do not` `avoid` `forbid` `must not` `never`

偵測：`prompt[max(0, start-10):start]` 正規式匹配否定詞且不跨越句號/分號。

**規則 C：路徑/檔名上下文偵測 `_is_in_path_context`**

匹配位置落在檔案路徑內視為文件引用：

- 向前 30 字元內含 `.claude/` / `docs/` / `src/` / `tests/` 等路徑前綴
- 向後 15 字元內含 `.md` / `.py` / `.json` / `.yaml` 等副檔名
- 或匹配整體落在 ticket ID 模式內（`\d+\.\d+\.\d+-W\d+-\d+`）

**規則 D（延伸）：Meta-task 全域放寬 `_is_meta_task_prompt`**

若 prompt 前 100 字元含 `修改 .claude/rules/` / `編輯規則文件` / `更新 FORBIDDEN_KEYWORD_MAP` 等 meta-operation 宣告 → 整個 prompt 降級為 debug log，不寫 events.jsonl。
