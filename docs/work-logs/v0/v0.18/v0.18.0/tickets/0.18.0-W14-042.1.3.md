### 修改摘要

`.claude/skills/ticket/ticket_system/lib/parser.py`:

1. 將 `import fcntl` 改為 conditional import，設 `_HAS_FCNTL` 旗標；ImportError 時將 `fcntl` 設為 `None` 並標記旗標 False。
2. `_file_lock` context manager 入口加上 `_HAS_FCNTL` 檢查；False 時 raise `NotImplementedError`，訊息含遷移指引（portalocker / filelock library）與替代環境建議（WSL/macOS/Linux）。
3. macOS/Linux 路徑完全不變（_HAS_FCNTL=True 時直接進原本 flock 邏輯）。

### 設計決策

- **不使用 silent no-op fallback**：silent fallback 會在 Windows 上重新引入 W14-005 race condition 而無任何訊號，違反 quality-baseline 規則 4（失敗必須可見）。改用顯式 NotImplementedError 含遷移指引。
- **不使用 msvcrt.locking 替代**：semantics 與 fcntl.LOCK_EX 不同（msvcrt 對 byte range lock，需指定範圍），盲目替換有正確性風險。留給未來真正需要 Windows 支援時派專門 ticket 處理。
- **fcntl 設為 None 而非保留未綁定**：避免 type checker / linter 對下方 fcntl.flock 引用報未定義；運行時不會到達該分支（已有 _HAS_FCNTL guard）。

### 不在本 ticket 範圍

- Windows 實際 lock 實作（需評估 portalocker / filelock dep 引入成本）。
- CI matrix 加入 Windows runner。
