## Solution

### 重構內容

將 `checkpoint_state()` 主函式中 5 次重複的 `SAFE_CALL` 提煉為 `DATA_SOURCES` table，與既有的 `PRIORITIES` 形成平行 table-driven 結構。

**新增 DATA_SOURCES table**（位於 §"DATA_SOURCES table" 區塊）：
- 5 entries tuple：`(state_field, source_name, reader, fallback, extractor)`
- `extractor` 欄位處理回傳值正規化（如 `_read_dispatch_active` 回 `(count, dict)` 需取 `[0]`）
- 與 `PRIORITIES` 同構（皆為 list of tuple，主函式以 for loop 跑 table）

**主函式 Step 1 精簡**：
- 原：5 次獨立 `SAFE_CALL(...)` 呼叫 + 中介變數（24 行）
- 新：for loop 跑 `DATA_SOURCES` + `collected: Dict[str, Any]` 暫存（7 行）
- Step 2 組裝 `CheckpointState` 改為從 `collected[field_name]` 取值

### 變更統計

- 檔案：`.claude/skills/ticket/ticket_system/lib/checkpoint_state.py`
- 增減：+66 / -29（net +37，含 49 行 DATA_SOURCES table 聲明 + 文件）
- 主函式 Step 1 資料收集區塊：24 行 → 7 行（-17 行）
- `__all__` 新增導出 `DATA_SOURCES`

### 與 PRIORITIES 的平行結構

| 維度 | PRIORITIES | DATA_SOURCES |
|------|-----------|--------------|
| 型別 | `List[Tuple[...]]` | `List[Tuple[...]]` |
| 用途 | 決策優先級推導 | 資料來源收集 |
| 主函式 | `for ... in PRIORITIES: if predicate` | `for ... in DATA_SOURCES: SAFE_CALL` |
| 擴充 | 新增優先級只加一筆 | 新增資料來源只加一筆 |
