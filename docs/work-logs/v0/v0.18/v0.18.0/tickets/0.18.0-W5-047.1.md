## Problem Analysis

### 問題根因

PC-092：並行派發多個會執行 `git add` + `git commit` 的代理人時，代理人使用 `git add .` / `git add -A` 等廣域 staging 命令，導致後完成代理人把其他並行代理人已寫入但未 commit 的檔案一併 staged + commit，commit 訊息與實際邊界不對齊。

觸發案例：2026-04-18 W5-043.3 + W5-043.4 並行派發事件（thyme-python-developer），batch 3 的 6 個檔案被 batch 4 代理人併入同一 commit（8e5b05e4），訊息標 batch 4 但實際含 batch 3+4 共 12 檔案。

### 影響範圍

`.claude/pm-rules/parallel-dispatch.md`：需新增 PC-092 精準 staging 規則，並在並行安全檢查 checklist 同步增項。

### 相關 Error Pattern

- PC-092 — 並行代理人 git index 競爭導致 commit 邊界與訊息不對齊（`.claude/error-patterns/process-compliance/PC-092-parallel-agents-git-index-race.md`）
