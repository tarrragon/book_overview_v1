### 設計決策

1. **上限 50（非無上限）**：WRAP P 階段識別「多 worktree 並行 100+ files」失敗模式，保留 50 上限 + `git status` 提示作為 mitigation
2. **分組顯示**：staged / modified / untracked 三組獨立計數，避免單一上限把某組壓縮為 0（前 session 遺留多為 modified 與 untracked，必須各自可見）
3. **雙通道方式**：`hook_output` 保持 stdout（既有用戶可見路徑），新增 `print(..., file=sys.stderr)` 處理 stderr 通道，`logger.warning` 寫入 `.claude/hook-logs/branch-status-reminder/`（hook_utils 既有機制）
4. **logger 參數注入**：`_report_uncommitted_changes(logger)` 而非模組級 logger，遵循 basil-hook-architect AGENT_PRELOAD 規則（避免 IMP-003 作用域迴歸）
5. **情況 2/3/4 補列**：PC-076 W14-013 案例證明情況 2（主 repo + 開發分支）也會遺留；情況 3/4 為防禦性補強，邊界場景不靜默
