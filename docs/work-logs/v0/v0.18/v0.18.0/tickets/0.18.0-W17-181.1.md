### 問題根因

`lib/handoff_utils.py` 的 `is_ticket_completed` / `is_ticket_in_progress_or_completed` / `is_handoff_stale` 透過 `load_and_validate_ticket(version, ticket_id)` 載入 ticket，內部走 `paths.get_tickets_dir(version)` → `get_project_root()`。在 stop hook 子進程環境下若 `CLAUDE_PROJECT_DIR` 與 cwd 不一致或路徑解析時序與 hook 自身的 `find_ticket_file` 行為不同（後者支援三層階層 + 舊扁平結構雙重查找），`load_and_validate_ticket` 可能找不到檔案而回保守 False，導致 stale handoff 不被 GC（W17-179 補實驗）。

屬 ARCH-020 跨進程同構邏輯反模式：hook 已用 `find_ticket_file` SSOT，lib 仍持自己的 ticket 路徑解析，兩條路徑漂移。

### 影響範圍

- `lib/handoff_utils.py`：`is_ticket_completed` / `is_ticket_in_progress_or_completed` / `is_handoff_stale` 三函式
- 消費端：
  - `commands/resume.py` L179/L184/L202/L594（CLI，cwd 正確，行為向後相容）
  - `commands/handoff_gc.py` L64/L72（CLI，同上）
  - `hooks/handoff-auto-resume-stop-hook.py`（子進程，需 propagate project_root）
  - `hooks/handoff-reminder-hook.py`（同上）
- 測試：`tests/test_handoff_utils.py` / `test_handoff_stale_consistency.py` / `test_resume_command.py` / `test_handoff_gc.py` 內 mock 簽章需更新

### 修復摘要

`lib/handoff_utils.py` 真 SSOT delegate 至 `hook_utils.hook_ticket.find_ticket_file` + `parse_ticket_frontmatter`。lib 不再持有 ticket 路徑解析責任。

### 設計決策

1. **新 helper `_load_ticket_status(ticket_id, project_root)`**：集中路徑解析 + frontmatter status 讀取邏輯，呼叫 `hook_utils.hook_ticket.find_ticket_file` 與 `parse_ticket_frontmatter`，回傳 `Optional[str]` status。
2. **三函式新增 `project_root: Optional[Path] = None` 參數**：
   - `is_ticket_completed(ticket_id, project_root=None)`
   - `is_ticket_in_progress_or_completed(ticket_id, project_root=None)`
   - `is_handoff_stale(record, project_root=None)`
   None 時 fallback 至 `get_project_root()`，向後相容 CLI 場景（cwd 正確時行為不變）。
3. **lib 內 `sys.path` 注入 `.claude/hooks/`**：以 `Path(__file__).resolve().parents[4] / "hooks"` 計算，使 `from hook_utils.hook_ticket import ...` 可在任何呼叫時序下成功（含 lib 被 hook 子進程 import 的情境）。
4. **移除廢棄 import**：`load_and_validate_ticket` / `extract_version_from_ticket_id` 已不再使用，從 import 區塊移除。
5. **hook 端 propagate project_root**：
   - `handoff-auto-resume-stop-hook.py`：`should_preserve_pending_json` 加 `project_root` 參數；`is_handoff_stale(record, project_root)` / `should_preserve_pending_json(data, logger, project_root)` 兩處呼叫更新；fallback 簽章同步加 `project_root=None` 參數。
   - `handoff-reminder-hook.py`：`is_handoff_stale(data, project_root)` 呼叫更新；fallback 簽章同步。

### 不在本 ticket 範圍

- `commands/resume.py` 與 `commands/handoff_gc.py` 呼叫端維持原樣（傳 None），CLI 場景 cwd 正確，向後相容；後續 ticket 可顯性傳遞以對齊。
- W17-181.2 / .3（其他相關層級重構）。
