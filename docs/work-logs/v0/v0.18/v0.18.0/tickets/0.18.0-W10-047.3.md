### 觀察計畫摘要

建立 `.claude/methodologies/hook-downgrade-observation.md` 方法論文件，包含：

1. **觀察期啟動點**：P1 baseline (05f328b7) + P3 baseline (4a225bcf) 雙階段
2. **8 Hook 觸發頻率與 Action 比追蹤表**：每 Wave 結尾更新（baseline 列已填入 W10-035.3 ANA 數據）
3. **Rollback 觸發條件**：5 條（false-negative / Action 比異常 / 頻率異常 / 用戶體感劣化 / counter 異常）
4. **快速恢復 SOP**：3 場景（整批 git revert / 單一 hook 精準恢復 / 抽樣 N 值微調）
5. **觀察期結束評估標準**：3 判斷項（false-negative 件數 / Action 比變化 / 用戶體感）決定收斂或延長
6. **與 W10-047.4 銜接**：明示 W10-047.4 啟動條件（至少 1 Wave + 無 rollback 觸發 + hook log 充足）

### 設計決策

- **追蹤表格化**：避免散文敘述，直接用表格欄位讓每 Wave 收斂時無需思考格式
- **rollback 三場景分層**：完全 rollback / 單一 hook / 微調，依嚴重度匹配恢復成本
- **不引用具體 ticket ID 作 trigger**：rollback 觸發條件以行為訊號（false-negative 案例 / Action 比）為依據，符合 decision-trigger-binding 原則
