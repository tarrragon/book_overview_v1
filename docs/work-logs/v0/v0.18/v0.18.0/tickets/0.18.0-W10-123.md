## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

`ticket-quality-gate-hook` 的 c2 / c3 檢查邏輯設計目標為 Flutter Bloc 架構的實作 ticket（IMP），但 hook 未做 type-aware 過濾即套用於所有 ticket type。對於 ANA（分析）/ DOC（文件）類型 ticket：

- c2 incomplete check：要求至少 1 個 `test/` 前綴路徑（line 84），且建議文字寫死 `*.dart`（line 187）
- c3 ambiguous responsibility check：強制要求 `[Layer X]` 標示（line 548），缺即判 failed

ANA/DOC 屬分析與文件性質，無實作測試檔，無 Bloc 層級概念。套用此檢查產生大量 false positive。

### 影響範圍

- `.claude/hooks/ticket-quality-gate-hook.py:should_trigger_check`（觸發階段過濾）
- `.claude/hooks/lib/ticket_quality/detectors.py:_generate_incomplete_recommendations`（`.dart` 硬編碼建議）
- `.claude/hooks/tests/`（新增 type-aware 測試覆蓋）

### 相關 Error Pattern

- W10-118 ANA Case B：W10-113 ANA ticket 被誤判為 incomplete（首例觸發）
- 與 W10-100（路徑黑名單）同家族：兩者都是 hook 範圍過寬，需更精準過濾
## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復策略

採「觸發階段過濾」策略：在 `should_trigger_check` 第五道條件加入 type-aware 過濾，type=ANA/DOC 直接跳過 c2/c3 整體檢查。較「detector 層分支判斷」更乾淨，且避免 c2/c3 內部邏輯多型複雜化。

### 變更總覽

| 檔案 | 變更 |
|------|------|
| `hooks/ticket-quality-gate-hook.py:should_trigger_check` | 條件 5：parse frontmatter `type:` 欄位，命中 `type_excludes`（預設 ANA/DOC）即 return False |
| `hooks/lib/ticket_quality/detectors.py:_generate_incomplete_recommendations` | line 187：移除 `*.dart` 硬編碼，改通用「<語言對應檔>」描述 |
| `hooks/tests/test_ticket_quality_gate_type_aware.py`（新增） | 4 個 type-aware 測試：ANA skip / DOC skip / IMP 觸發 / 缺 type fallback 觸發 |

### 設計決策

| 議題 | 選項 | 採用 | 理由 |
|------|------|------|------|
| ANA/DOC 處理方式 | (a) 跳過 c2/c3；(b) 套用 type-specific detector | (a) 跳過 | acceptance「跳過 c2/c3 或套用對應 detector」二選一；(a) 實作簡單且解決首例觸發，未來如需 ANA 專屬檢查再加 detector |
| Acceptance 3「c3 Layer 標示對 ANA 改為選填」 | 滿足 | 採方案 (a) 自動覆蓋（ANA 整體跳過 c3 = Layer 對 ANA 不再要求） |
