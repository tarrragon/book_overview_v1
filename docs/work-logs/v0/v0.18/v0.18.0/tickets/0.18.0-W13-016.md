### 修復摘要

新建 `.claude/hooks/lib/hook_health.py`（純函式 lib，純 stdlib）：

| 函式 | 職責 | acceptance 對應 |
|------|------|---------------|
| `scan_logs(since, logs_root=None)` | 掃描 hook-logs/，依 mtime 過濾，回傳 `{hook: {total, per_day}}` | acceptance 1 |
| `classify_hook(name, settings)` | 2 類粗分（HIGH_FREQ_HOOK_PATTERNS 比對 → high_freq_ok，否則 low_freq_expected） | acceptance 2 |
| `evaluate(stats, hook_type, baseline)` | 回 Verdict（normal/warning/critical）；包含 3 天連續升級訊號 | acceptance 3, 6 |
| `read_session_marker(marker_path=None)` | 讀取 `.claude/state/last-session-start.marker` ISO timestamp | acceptance 4 |
| `_has_3_consecutive_over` | 私有 helper，連續 3 天超閾值偵測 | (支援 evaluate) |

Bootstrap 退化路徑：`baseline <= 0.0` 時改用絕對下界 `BOOTSTRAP_ABSOLUTE_THRESHOLD=100/day`，Verdict.bootstrap=True 顯示給呼叫端（acceptance 5）。

### 設計決策

- **常數抽出**：MULTIPLIER_BY_TYPE / HIGH_FREQ_HOOK_PATTERNS / BOOTSTRAP_ABSOLUTE_THRESHOLD 全部 module-level，未來調整不需翻邏輯。
- **無副作用**：lib 純讀取（檔案系統 stat + read_text），不寫檔不 subprocess，呼叫端決定 stderr / CLI 輸出形式（符合 W13-008 §觀察性原則）。
- **Verdict dataclass**：以 `status / recent / baseline / multiplier / bootstrap / reasons` 完整曝光診斷資訊，避免呼叫端反推。
- **Python 3.9 相容**：使用 `Optional[X]`、`Dict`、`List`、`from __future__ import annotations`；無 `match/case`、無 PEP 604 union 寫法。

### Phase 4 評估結論

- **重構需要**：否
- **技術債務**：無
- **程式碼品質**：A

評估依據：
- Cognitive Load 指數 ~6（變數 + 分支 + 巢狀），所有函式 < 20 行（私有 helper 25 行為連續日合理長度）
- 命名自說明（動詞函式 / 名詞資料 / `_` 前綴 helper）
- DRY 已透過 module-level 常數覆蓋
- 單一職責：scan_logs（IO 聚合）/ classify_hook（規則查表）/ evaluate（決策）/ read_session_marker（IO 解析）職責互不重疊

### WRAP 三問註腳（bootstrap 退化路徑）

- **W (Widen)**：bootstrap 是否可改用「複製其他 hook 平均值」替代魔數？否決：跨 hook 平均無語意（acceptance-gate 與 phase4 觸發次數量級差 10x+），會比魔數更失真。
- **R (Reality test)**：100/day 絕對下界憑何而定？採 W13-008 §Bootstrap 結論（保守魔數，僅 bootstrap 用），記錄滿 7 天後相對基線自動接管。
- **A (Attain distance)**：若數據顯示 100 仍偏低，調整成本？常數位於 module-level 一行，未來 quarterly review 可單行 PR 調整，不需重構邏輯。
