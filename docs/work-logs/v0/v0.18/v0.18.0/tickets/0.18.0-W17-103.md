### 二輪 WRAP 分析記錄

### 第一輪 WRAP（W17-098）— 錯誤推論

**第一輪結論**：「CC v2.1.126 runtime 層 hardcoded deny subagent .claude/ 寫入，不可被 hook/settings/permissionMode 覆蓋」

**證據基礎**：4 並行 thyme deny vs PM 同檔成功 + hook 全 allow + settings 無 deny 規則

**推論失誤**：
- 把「同 session 所見對照」當成「runtime 全域行為」過度泛化
- 沒注意到 4 並行只是 1 個觀察點，無控制其他變因
- W (Widen) 階段沒列出時間/PM 接觸史/Edit 複雜度/真 commit 等變因
- R (Reality test) 階段宣稱「成本/收益不對稱不執行」直接結束實驗

**錯誤的學習價值**：W17-098 結論被 W17-099 用相同條件不同時間點 success 直接推翻。第一輪 WRAP 在 W 階段不夠廣泛，導致 R 階段沒有正確的對照實驗。

### 二輪 WRAP（W17-099 推翻第一輪後）

**Widen**：列出 5 個可能變因（並行、Edit 複雜度、prompt 結構、commit 提及、PM 接觸史）

**Reality test**：設計 E1（單獨 + 複雜）+ E2（並行 + 簡單）+ Test G（PM 未碰過的檔）三組對照逐一隔離

**Attain distance**：跳出當下假設後，注意到 W17-099 改變了 5 個變因無法單一歸因

**Prepare to be wrong**：每個實驗預先列出「success / deny → 何種推論」的決策表

### 第三輪 WRAP（用戶質疑第二輪推論不合理後）

**新發現**：之前在矩陣裡用「PM 是否已 Edit 過該檔」（檔案層級），Test G 否證後跳到「不是 PM 接觸史」結論。但實際應該是「PM 是否已 Edit 過 prefix 內任一檔」（前綴層級）。**用錯粒度**。

### Hypothesis K（最新工作假設）

**主張**：CC v2.1.126 對 subagent .claude/ Edit 有「session 級 prefix-based 權限快取」：

- session 內**首次** subagent .claude/ Edit 觸發互動式 permission prompt
- 在 subagent 無 UI 環境下，prompt 自動 deny
- **PM 互動式 Edit 過 .claude/ 任一檔**會在 session 中快取「.claude/ 已授權」
- 之後 subagent .claude/ Edit 都繼承快取 → success（即使是不同檔，因為 prefix-based）

**矩陣對應**：

| 實驗 | PM 已 Edit .claude/ 任一檔（前綴層級）| 結果 | 對 Hypothesis K |
|-----|----|----|----|
| W17-097.1-.4 | 否 | DENY | 證實 |
| W17-099/100/101/102 | 是 | success | 證實 |

5 個實驗全部與 Hypothesis K 預測相符，零反例。
