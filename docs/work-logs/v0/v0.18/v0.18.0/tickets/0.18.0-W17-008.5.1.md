### PCB（PM 派發前注入）

#### 問題根因

`lifecycle.py:678` 的 f-string 提示輸出：

    ticket track append-log {ticket_id} --section "{section}" --content "內容"

但 `ticket track append-log --help` 顯示 `content` 為 positional argument，正確語法為：

    ticket track append-log <ticket_id> "內容" --section "..."

照錯誤提示執行會得到 `unrecognized arguments: --content`，使用者陷入「修錯一次再錯一次」。

#### 正確語法錨定

`append-log --help` 摘錄：

    usage: ticket track append-log [-h] --section SECTION [--version VERSION] ticket_id content
    positional arguments:
      ticket_id          Ticket ID
      content            日誌內容
    options:
      --section SECTION  日誌區段

完整等價語法（agent 修正建議在訊息中採用）：

    ticket track append-log <ticket_id> "內容" --section "<section>"

或多行 heredoc 版本（與 bash-tool-usage-details.md 規則 5 對齊）使用 `<<DELIM ... DELIM` 包裹的 `$(cat ...)`，把 cat 結果作為 positional content。
