## 實作摘要

### 新增函式：git_update_index_chmod(root)
直接操作 git index 設定 mode 100755，不依賴 filesystem 語意。對 `root/hooks/**/*.py` 每個檔案呼叫 `git update-index --chmod=+x <rel>`。

**與 restore_executable_bits 的關係**：
- `restore_executable_bits`：filesystem chmod +x（macOS/Linux 有效，Windows 無效但不失敗）
- `git_update_index_chmod`：直接寫 git index mode（跨平台一致，治本）
- 兩者並存：restore 處理 POSIX 本地狀態一致性（pull 後下游檔案行為），git_update 確保 push 上去的 git index 正確

### 呼叫時機
- `restore_executable_bits(temp_dir)`：line 543，copy_filtered 後、git add -A 前
- `git_update_index_chmod(temp_dir)`：line 602，git add -A 後、git commit 前（檔案需先 tracked）

### 整合測試（本 session 實測）

mini git repo 模擬 Windows 狀態：
1. 建立 `hooks/test-hook.py` 含 shebang
2. `chmod 644` 模擬 NTFS filesystem
3. `git add -A` → index 記為 `100644`（Windows 預設行為的再現）
4. 執行 `git_update_index_chmod` → index 變為 `100755`
5. 檔案 hash 未變（僅 mode 改變）

結果：PASS。治本方案可阻止 v1.36.2 類事件重現——即使在 Windows filesystem mode 仍是 644，git index 會被強制設為 100755，push 到 remote 的 mode 正確。

### 行為矩陣

| 場景 | restore_executable_bits | git_update_index_chmod | 整體效果 |
|------|------------------------|------------------------|---------|
| macOS / Linux 新檔 | chmod 755（filesystem） | 設 index 100755（已一致） | 正確 |
| Windows 新檔 | NTFS 無 exec bit, 無效 | 設 index 100755（關鍵修正） | 正確 |
| 本地既有 644 檔 | chmod 755（filesystem） | 設 index 100755 | 正確 |
| 本地既有 755 檔 | 跳過（mode 已正確） | 設 index 100755（no-op） | 正確 |

### 技術債
- TD-1: restore_executable_bits 與 git_update_index_chmod 雙份子目錄掃描邏輯重複；可抽共用 collect_executable_py_files(root) 函式
- TD-2: git_update_index_chmod 目前對每個檔案單獨呼叫 git，N 次 subprocess。可改用 batch update（git update-index --stdin）提升效能
- TD-3: pull.py 端未加對稱的 git index 操作（但 pull 時不做 git add 所以不適用）

### 下一步：round-trip 驗證
完成 .3 後執行一次 sync-push（把本次修改推到 remote）+ sync-pull（下游拉取確認 mode 正確）。預期：
- remote v1.36.3 新增 commit 的 hooks 檔案 mode 為 100755
- 本地 sync-pull 後 hooks mode 皆 100755
