## 問題現象

`ticket track complete` 對 ANA/IMP ticket 執行 body-check 時，將合法填寫的 Problem Analysis / Solution 章節誤判為「未填寫必填章節」，即使章節內有實質內容。

## Bug 位置

檔案：`.claude/skills/ticket/ticket_system/lib/ticket_validator.py`
函式：`validate_execution_log_by_type`
行號：L504-529

## 根因（已精準定位）

L522-526 的章節邊界判定：

```python
next_section_idx = len(body)
for marker in ["## ", "### "]:
    idx = body.find(marker, content_start)
    if idx != -1 and idx < next_section_idx:
        next_section_idx = idx
```

此邏輯同時把 `## ` 和 `### ` 當成章節結束標記，導致：

- 若章節內用 h3 子標題組織內容（例如 Problem Analysis 底下有「### 問題現象」、「### 根因」），第一個 `### ` 會被誤認為章節結束
- `section_content` 只剩章節標題與 Schema 註解（L60-61 之間）
- `_is_placeholder(section_content)` 剝除 HTML 註解後為空字串，回傳 True（placeholder）
- 誤擋合法 complete

## 觸發案例（已實測）

W17-046 ANA ticket 的 Problem Analysis 章節包含 `### 問題現象`、`### 問題本質`、`### 受影響 agent`、`### 根因分析`、`### 觸發案例`、`### 問題根因`、`### 影響範圍`、`### 相關 Error Pattern` 八個 h3 子標題。實際內容完整填寫，但 body-check 誤判未填，必須用 `--skip-body-check --yes` 逃生閥才能 complete。

## 修復方向（建議）

**方案 A（最小修改）**：只把 `## ` 視為章節邊界，移除 `"### "`：

```python
# 只用 h2 作為章節結束標記
idx = body.find("\n## ", content_start)
if idx != -1:
    next_section_idx = idx
```

注意需加 `\n` 前綴避免匹配段落開頭的 `## ` 字串。

**方案 B（更嚴謹）**：用 regex `^## ` 行首匹配，避免中段出現的 `## ` 字串（如 code block 內）誤匹配：

```python
import re
match = re.search(r"(?m)^## ", body[content_start:])
if match:
    next_section_idx = content_start + match.start()
```

建議採方案 B（更嚴謹，避免 edge case）。

## 測試要求

- **Regression 測試**：給定 body 含 `## Problem Analysis\n### 問題現象\n內容...\n## Solution\n...`，驗證 Problem Analysis 的 `section_content` 包含「內容...」字串，非 placeholder
- **既有測試保留**：確認原本 Problem Analysis 為空的情境仍被正確判為 placeholder
- **Edge case**：章節內含 code block 的 `## ` 字串（markdown 內文），不應被誤當作章節邊界
- **W17-046 重驗**：對 W17-046 ticket md 執行 `ticket track complete --dry-run`（若有此 flag），應通過而不需 --skip-body-check

### 修改範圍

- `.claude/skills/ticket/ticket_system/lib/ticket_validator.py`：`validate_execution_log_by_type` 的章節邊界判定改為 `re.search(r'^## ', body[content_start:], re.MULTILINE)`。
- `.claude/skills/ticket/ticket_system/tests/test_ticket_validator.py`：新增 `TestValidateExecutionLogByTypeH3SubheaderBoundary`，共 6 個 regression 測試。

### 核心差異

原實作：同時將 `## ` 與 `### ` 作為章節結束標記 → h3 子標題誤判章節終止。

新實作：只以 `^## ` 行首匹配作為章節邊界；h3 子標題保留在章節內，code block / 段落中間的行中 `## ` 字串不誤判。

### 範圍鎖定遵守

- 僅修 `validate_execution_log_by_type` 單一函式邏輯，未動 `_is_placeholder`，也未修 `validate_execution_log`（legacy 函式已有測試文件化其既有行為，不在本 ticket 修復範圍）。
- 不擴充 `_TYPE_REQUIRED_SECTIONS` 清單。
- 未新增共用 helper 抽象；保持 localized change 降低 blast radius。

### 失敗假設（P）

最可能失敗：程式碼範例（fenced code block）內部第一欄出現 `## ` 時會被 `re.MULTILINE` 誤判為行首。實測 regression 測試以段落中的「行中 `## `」驗證；極端 edge case（code block 內行首 `## ` 字串）保留為已知限制，暫不做更進一步的 markdown 結構解析（過度工程）。
