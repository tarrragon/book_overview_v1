## Solution

新增 `ticket_system/lib/staleness.py`：
- 常數：`STALE_INFO_DAYS=7`、`STALE_WARNING_DAYS=14`、`STALE_CRITICAL_DAYS=30`
- `calculate_stale_level(created, today=None)`：回傳 `info/warning/critical/None`，容忍缺值與格式錯誤
- `format_stale_warning(ticket)`：單一 Ticket 警告訊息（<=3 行），CRITICAL 級別明確提示「標記為 stale 或重新規劃」
- `format_stale_list_summary(tickets)`：list 命令的數量摘要（critical/warning/info 分級計數，<=2 行）

整合點：
- `execute_claim`（lifecycle.py）：claim 前呼叫 `format_stale_warning`，任一級別觸發都輸出；例外不阻擋 claim 主流程（寫 stderr）
- `execute_query`（track_query.py）：僅在 WARNING / CRITICAL 級別輸出（對應 AC「超過 14 天輸出 WARNING」）
- `_output_table`（track_query.py）：table 格式 list 輸出後附 stale 摘要行

設計選擇：
- 所有整合點使用 try/except 靜默失敗（stderr），不影響主功能
- today 參數可注入以利測試，省去 freeze_time 相依
- 輸出格式符合 `language-constraints.md`（禁 emoji，使用 `[INFO]`/`[WARNING]`/`[Stale]`）
