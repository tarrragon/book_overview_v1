### 修復摘要

抽取兩個型別防護 helper 至 `_parse_cli_args_to_config` 上方，相容舊 ticket 字串格式：

- `_inherit_parent_who(parent_ticket)` — 處理 parent.who 為 dict/str/None
- `_inherit_parent_where_layer(parent_ticket)` — 處理 parent.where 為 dict/str/None

修復點：

- L694（原）：`parent_ticket.get("who", {}).get("current") if parent_ticket else "pending"` → `_inherit_parent_who(parent_ticket)`
- L697（原）：`parent_ticket.get("where", {}).get("layer") if parent_ticket else DEFAULT_UNDEFINED_VALUE` → `_inherit_parent_where_layer(parent_ticket)`

### 設計決策

- **採 helper 函式而非 inline 三元**：型別分支邏輯（dict/str/None）超過單行可讀範圍；抽取後 inline 點剩單一函式呼叫，認知負擔顯著降低。
- **回傳語意**：str fallback 為 "pending"（who）/ `DEFAULT_UNDEFINED_VALUE`（where_layer），與既有預設一致。
- **空字串視為缺失**：legacy ticket 若 where 為 `""`，回傳 default（避免 layer 變成空字串造成下游 validator 失敗）。
- **L1373 / L1424 不在本 ticket 範圍**：屬 PM 另建 follow-up，不擴展 scope。

### Before / After 對照

修復前（L694, L697）：

```python
"who": args.who or (parent_ticket.get("who", {}).get("current") if parent_ticket else "pending"),
"where_layer": args.where_layer or (parent_ticket.get("where", {}).get("layer") if parent_ticket else DEFAULT_UNDEFINED_VALUE),
```

修復後：

```python
"who": args.who or _inherit_parent_who(parent_ticket),
"where_layer": args.where_layer or _inherit_parent_where_layer(parent_ticket),
```

新增 helper 位於 `_parse_cli_args_to_config` 上方，含 docstring 說明 W11-003.7 背景。
