## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

- W17-022 已確認 subagent commit 後不主動 complete 的根因：agent preload / agent definition standard 未把 `check-acceptance --all` + `complete` 定義為實作代理人收尾責任。
- 既有 error-pattern 只覆蓋 agent 執行邊界、文件引導缺失等相鄰問題，未收錄「commit 不等於 Ticket complete」此 lifecycle 漏洞。

### 影響範圍

- 受影響流程：subagent 實作任務收尾、PM handoff / runqueue / parent complete 判斷。
- 受影響案例：W17-020、W17-016.3。
- 新增檔案：`.claude/error-patterns/process-compliance/PC-108-subagent-commit-without-ticket-complete.md`。

### 相關 Error Pattern

- PC-104：agent 執行邊界誤判導致結果未落地。
- PC-105 feature implemented without doc integration：規則未寫入 preload / definition 時，流程不知道新責任。
- PC-075：spawned / children terminal status 檢查會被未 complete 的 ticket 放大。
- WRAP 三問簡記：候選方案為新增獨立 PC / 併入 PC-104；選擇獨立 PC，因本案的核心是 lifecycle terminal state，不是 agent 自稱權限不足。機會成本是暫不修 W17-033。主要失敗風險是編號衝突，已避開既有 PC-105 重號並採 PC-108。
