### 問題根因

W6-012.7.1 已建 `selectedBookIds` Set（overview-page-controller.js:107），但 book-exporter.js 透過 DI 取得 `getFilteredBooks` 始終回傳 `this.filteredBooks` 全量（行 118），未感知 selection state。所有 4 個匯出/copy 入口（CSV/JSON/copyText/未來其他）都仰賴 getFilteredBooks。

### 影響範圍

| 檔案 | 變更性質 |
|------|---------|
| `src/overview/overview-page-controller.js` | 修改 `getFilteredBooks` DI lambda 或新增 `getExportBooks()`；行 118 附近 |
| `src/overview/book-exporter.js` | 視策略可能不需改（若沿用 getFilteredBooks）或改為 `getExportBooks` |
| copyTextBtn 額外觀察 | HTML 列在 elementMap 行 167 但 setupEventListeners 無 handler — 屬 .7.1 漏網或另案 |

### 相關 Error Pattern

無。屬功能擴充。

### Context Bundle

**現況 API**：
- `book-exporter.js` constructor 接 `{ getFilteredBooks, document }`，所有 export 方法呼叫 `this.getFilteredBooks()`
- `controller.js:118` 注入：`getFilteredBooks: () => this.filteredBooks`
- `controller.js:107` 已有 `this.selectedBookIds = new Set()`（W6-012.7.1 落地）

**策略 A（推薦）**：修改 DI lambda 為「selection-aware」：
```js
getFilteredBooks: () => {
  if (this.selectedBookIds.size === 0) return this.filteredBooks
  return this.filteredBooks.filter(b => this.selectedBookIds.has(b.id))
}
```
- 優點：1 處變動，所有匯出/copy 自動尊重 selection；向後相容（無選取時行為不變）
- 風險：getFilteredBooks 語意被擴展，可能影響其他呼叫者

**策略 B**：新增 `getExportBooks()` 獨立方法
- 優點：語意清晰，匯出 vs 顯示分離
- 缺點：需改 book-exporter constructor 及 4 個方法

**範圍上限**：
- 不處理 copyTextBtn handler 缺失（屬 .7.1 後續或另案）
- 不改變 importJSON 行為（import 不應受 selection 影響）

### 簡化 WRAP

- W: (a) 策略 A DI lambda 改寫 (b) 策略 B 新方法 → 選 (a) 最小變動最大覆蓋
- A: 機會成本低；UI 互動完整性是 v1.0 內測前置
- P: 最可能失敗：getFilteredBooks 其他呼叫者（grep 確認）被意外影響 → 防護：grep 全專案 `getFilteredBooks` 呼叫點，僅在 book-exporter context 套用 selection 邏輯（必要時 wrapper）

### 策略選擇：A（DI lambda selection-aware）

**Grep 確認**：`grep -rn "getFilteredBooks" src/` 顯示僅 2 個檔案使用：
- `overview-page-controller.js:118`（DI provider，唯一注入點）
- `book-exporter.js`（唯一消費者，4 個方法呼叫：行 47/61/75/87）

無其他呼叫者（搜尋/排序/UI render 皆不依賴此 API），擴展 DI lambda 語意安全，採策略 A 最小變動。

### 變更摘要

`src/overview/overview-page-controller.js` 行 117-127：將 `getFilteredBooks` DI lambda 由 `() => this.filteredBooks` 改為 selection-aware：

```js
getFilteredBooks: () => {
  if (this.selectedBookIds.size === 0) return this.filteredBooks
  return this.filteredBooks.filter(b => this.selectedBookIds.has(b.id))
}
```

### 行為矩陣

| selectedBookIds 狀態 | 匯出/copy 範圍 |
|--------------------|---------------|
| size === 0（未選取） | 全量 filteredBooks（向後相容） |
| size > 0（已選取 N 本） | 僅 N 本（filter by id） |

涵蓋所有 4 個匯出入口（CSV / JSON / copy / 未來新增方法），單點修改全面覆蓋。

### 不在本 ticket 範圍

- **copyTextBtn handler 缺失（觀察）**：HTML 元素列在 `elementMap.buttons`（行 173）但 `setupEventListeners` 未綁定 click handler，故 UI 上點按 copy 按鈕無作用。屬 W6-012.7.1 漏網或另案。copy 邏輯層（`BookExporter.copyAsText` 等）已透過本 ticket 的 DI 改寫自動受益，待 UI button 串接後即可使用。
- **importJSON 行為**：未修改，import 不受 selection 影響（語意正確：匯入操作與當前選取無關）。

### 風險評估

低風險。DI lambda 為純函式，無副作用；selection 為空時退回原行為，向後相容。getFilteredBooks 無其他呼叫者，語意擴展不影響其他模組。
