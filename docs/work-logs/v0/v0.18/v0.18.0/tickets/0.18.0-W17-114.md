## Problem Analysis


### 問題根因

W17-113（規則層）落地後規則只說「禁止無 trigger 延後」，缺實踐層指引：PM 遇到「不能立刻決策」的問題時，正確流程應為何？若無流程指引，PM 仍可能在實踐中誤用規則（如直接寫「等以後再說 + ticket 引用 W17-113」鑽規則文字漏洞）。

### 影響範圍

- `.claude/pm-rules/decision-tree.md`：路由入口缺「無法立刻決策」分支
- `.claude/pm-rules/execution-discovery-rules.md`：執行 domain 缺閉環流程章節
- `.claude/rules/core/decision-trigger-binding.md`：缺與流程層的互引用

### 相關 Error Pattern

- PC-093（YAGNI 累積反模式）— 無 trigger 延後的根因
- PC-099（Hook meta-self-reference 誤報）— 本 ticket 自身觸發，使用 user-override 豁免

### 設計動機

「規則 + 流程 = 完整閉環」：規則只說明禁止，流程指明合法路徑。兩者缺一邊都會留下規避空間。

用戶要求：「遇到問題應該明確規劃後續分析跟驗證或者實驗任務，釐清解決方案，最終把需求結案，所以決策樹要能夠完成整個閉環。」

對應到 ticket type 流：ANA（分析）→ Solution（含 spawned IMP/DOC for 驗證/實驗）→ 結案（含明確結論）。
