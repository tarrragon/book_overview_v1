## Problem Analysis

### 問題根因

PM 切換 Ticket 時從 `ticket track list` / `runqueue --format=list` 的原始可執行清單挑選下一個要做的 Ticket，需自行執行以下推導：

1. 讀取當前 `in_progress` ticket 判斷所在 Wave
2. 掃描清單過濾同 Wave、過濾已 in_progress、過濾被阻塞
3. 依 priority 排序
4. 判斷「是否該收尾當前 Wave」vs「進下一 Wave」
5. 為決策找理由（為什麼選 A 不選 B）

這 5 步推導是機械性的——可由 CLI 輸出「top 1 + 3 備選 + 理由」取代，讓 PM 只做最終確認（父 ticket W10-017 Solution Phase 2 的原始動機）。

### API 規格

#### 命令介面

```
ticket track suggest-next [--wave N] [--version VERSION] [--format=text|json]
```

| 參數 | 必選 | 預設 | 說明 |
|------|-----|-----|------|
| `--wave N` | 否 | 自動偵測當前 in_progress 的 wave | 明示覆寫 Wave，用於 PM 想跨 Wave 查詢時 |
| `--version VERSION` | 否 | 自動偵測 | 沿用既有慣例 |
| `--format` | 否 | `text` | `text` 人類閱讀；`json` 供 Hook / Skill 組合（Phase 4 重構可加） |

**注意**：Phase 1 最小可行版本只需實作 `text` 格式；`json` 僅保留 API 擴展位。

#### 退出碼

| Exit code | 語意 | 使用場景 |
|-----------|-----|---------|
| 0 | 有 top 1 建議（含 Wave 收尾建議亦算 0） | 正常 |
| 1 | 無法偵測 Wave（無 in_progress 且未指定 --wave） | PM 需顯式指定 |
| 2 | 全被阻塞（有 pending 但全部被 blockedBy 卡住） | 需處理阻塞源 |
| 3 | 內部錯誤（ticket 讀取失敗等） | 沿用既有錯誤慣例 |

退出碼設計目的：讓 Hook / 未來 `runqueue` 組合使用時可用 exit code 快速判斷，不必 parse 文字。

|------|------|

### 排序演算法（虛擬碼）

```
def suggest_next(version, wave_override=None):
    all_tickets = list_tickets(version)
    in_progress = [t for t in all_tickets if t.status == "in_progress"]

    # 步驟 1：決定目標 Wave
    if wave_override is not None:
        target_wave = wave_override
    elif len(in_progress) >= 1:
        target_wave = in_progress[0].wave  # 若多個 in_progress，取第一個
    else:
        return exit_code=1, message="無 in_progress ticket，請指定 --wave N"

    # 步驟 2：篩選候選池
    same_wave = [t for t in all_tickets if t.wave == target_wave]
    same_wave_pending = [t for t in same_wave if t.status == "pending"]

    # 步驟 3：剔除被阻塞
    unblocked = [t for t in same_wave_pending if not t.blockedBy]
    blocked = [t for t in same_wave_pending if t.blockedBy]

    # 步驟 4：無候選處理
    if not same_wave_pending:
        return render_wave_wrapup(target_wave, same_wave)

    if not unblocked and blocked:
        return exit_code=2, render_all_blocked(blocked)

    # 步驟 5：排序 P0>P1>P2>P3，ID 升序 tiebreaker
    unblocked.sort(key=lambda t: (_priority_rank(t), t.id))

    # 步驟 6：切分 top 1 + 3 備選
    top = unblocked[0]
    alternatives = unblocked[1:4]

    return exit_code=0, render_suggestion(top, alternatives, target_wave)
```

**關鍵設計決策**：

1. **priority 未知值排最後**：沿用 runqueue 的 `_DEFAULT_PRIORITY_RANK = 99` 慣例
2. **多個 in_progress 時取第一個**：遵循 PM 通常只跑一個 focus ticket 的現況；若未來需支援多 focus，可在 Phase 4 重構擴充
3. **不主動偵測 `started_at` 新舊**：避免與 PC-078（並行 session）衝突；PM 可用 `--wave` 覆寫
4. **Wave 收尾建議的 parent 關係判讀**：僅列出同 Wave 內 parent 的 non-pending children，不跨 Wave 追查

### 理由生成規則（輸出「理由」文字邏輯）

為了讓 PM 只做最終確認，每項建議需附理由。理由組合規則：

| 位置 | 理由元素（必含） |
|------|----------------|
| top 1 | Wave 位置 + priority 排名 + 無阻塞確認 + type |
| 備選 | 與 top 1 的差異點（通常是 priority 較低 / id 較後） |

**差異點產生邏輯**：

```
def explain_alt(alt, top):
    if alt.priority > top.priority:  # P2 vs P1
        return f"priority 較低（{alt.priority} vs {top.priority}）"
    else:  # 同 priority，ID 較後
        return f"同 priority，id 較後（tiebreaker）"
```

```python
# track_suggest_next.py
from ticket_system.lib.ticket_loader import list_tickets
from ticket_system.commands.track_runqueue import _priority_rank  # 或自行局部定義


def detect_target_wave(tickets, wave_override):
    if wave_override is not None:
        return wave_override, None  # (wave, warning)
    in_progress = [t for t in tickets if t.get("status") == "in_progress"]
    if not in_progress:
        return None, "NO_IN_PROGRESS"
    if len(in_progress) > 1:
        return in_progress[0].get("wave"), "MULTI_IN_PROGRESS"
    return in_progress[0].get("wave"), None


def render_suggestion(args, version):
    tickets = list_tickets(version) or []
    wave, warning = detect_target_wave(tickets, getattr(args, "wave", None))
    if wave is None:
        return 1, "[ERROR] 無 in_progress ticket，無法自動偵測 Wave\n請使用 --wave N 明示"

    same_wave = [t for t in tickets if t.get("wave") == wave]
    same_wave_pending = [t for t in same_wave if t.get("status") == "pending"]
    unblocked = [t for t in same_wave_pending if not (t.get("blockedBy") or [])]
    blocked = [t for t in same_wave_pending if (t.get("blockedBy") or [])]

    if not same_wave_pending:
        return 0, _render_wave_wrapup(wave, same_wave)

    if not unblocked:
        return 2, _render_all_blocked(wave, blocked)

    unblocked.sort(key=lambda t: (_priority_rank(t), str(t.get("id", ""))))
    top = unblocked[0]
    alternatives = unblocked[1:4]
    return 0, _render_top_and_alternatives(wave, top, alternatives, warning)


def execute_suggest_next(args, version):
    code, text = render_suggestion(args, version)
    print(text)
    return code


def register_suggest_next(subparsers):
    p = subparsers.add_parser(
        "suggest-next",
        help="建議下一 Ticket：top 1 + 3 備選 + 理由（基於 Wave/priority/blockedBy）",
    )
    p.add_argument("--wave", type=int, default=None, help="明示 Wave（預設自動偵測 in_progress 的 wave）")
    p.add_argument("--version", help="指定版本號")
    p.add_argument("--format", choices=["text"], default="text", help="輸出格式（Phase 1 僅 text）")
    return p
```
