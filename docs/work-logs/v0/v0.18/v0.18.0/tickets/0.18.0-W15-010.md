## 終止條件設計（三層防護）

### Layer 1：質性自檢門檻（硬性，寫入方法論）

three-phase-reflection-methodology.md 新增「終止條件」章節：

```markdown
## 終止條件

反思流程可結束的判準（滿足 A + B 任一即可停止）：

### A. 深度門檻（Phase 1/2/3 全通過）
- Phase 1：能完整回答「若真因為 X，為何不是 B/C/D？」
- Phase 2：WRAP 四階段走完，tripwire 無觸發
- Phase 3：結論與 .claude/methodologies/ + .claude/rules/ 無衝突

### B. 用戶閘門
- 用戶明確表達「這個深度夠了」「可以停了」
- AskUserQuestion 確認繼續反思，用戶選擇「停止」

**禁止條件**：
- 不可以 PM 單方判斷「差不多了」而未過 A 或 B
- 不可連續 > 3 層 ANA spawn ANA 不停（Layer 3 警示）
```

### Layer 2：結構警示（軟性，由 ticket CLI 或 Hook 提供）

當 spawned_tickets chain 深度達 3：

- `ticket track deps <id>` 輸出顯示 `[WARNING] 反思鏈深度 = N`
- 新增 Hook：`reflection-chain-depth-warn-hook.py`，偵測 spawn 的 ANA type 祖鏈 ≥ 3 時 stderr 警示

觸發範例：`W15-005 → W15-008 → W15-009 → W15-010`（已達 3 層）→ 建立 W15-010 時應警示。

### Layer 3：用戶閘門（互動，PM 規則）

新增 PM 規則 `.claude/pm-rules/reflection-termination.md`（或併入 decision-tree.md）：

```markdown
## 反思終止閘門

PM 發現以下狀態時必須 AskUserQuestion 詢問終止：
- session 內連續 ≥ 3 個 ANA Ticket
- spawned_tickets 反思鏈深度 ≥ 3
- 單一主題 ANA 累計耗時 > 4 hr

AUQ 選項：
  - [Recommended] 終止反思，切換到實作線 / handoff
  - 繼續反思（用戶需明示「還不夠深」的理由）
  - /clear 結束 session
```
