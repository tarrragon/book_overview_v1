### Hook 行為驗證（19:33:49 同一時點）

5 個 PreToolUse Edit hook 全部 allow：
- main-thread-edit-restriction: "subagent 環境，跳過編輯限制"
- branch-verify: ".claude/ 豁免路徑，允許"
- file-type-permission: "code 檔案，允許"
- ticket-path-guard: exit 0
- ticket-file-access-guard: exit 0

settings.json + settings.local.json：無 deny 規則，無 defaultMode 限制。

agent frontmatter `permissionMode: bypassPermissions`：未生效。

### 結論

**CC v2.1.126 runtime 層對 subagent Edit/Write 主 repo 內 .claude/ 路徑有 hardcoded deny**，特性：
1. 不可被任何 PreToolUse hook 覆蓋（hook 全 allow 仍 deny）
2. 不可被 permissionMode: bypassPermissions 覆蓋
3. 不可被 settings.json permissions.allow 覆蓋
4. 無 hook stderr 訊息可診斷（runtime 在 hook 之後仍二次拒絕）
5. PM 主線程不受此限制

### 推論機制

CC runtime 對 subagent 有「敏感目錄保護」，將 .claude/ 視為框架資產防止 subagent 自我修改框架（reasoning：避免 subagent 任意改 hook/agent 定義繞過用戶授權）。permissionMode: bypassPermissions 只覆蓋 hook 層 + settings 層，不覆蓋 runtime hardcoded protection。

### 未驗證項（成本/收益不對稱，不執行）

- subagent Edit 主 repo 內非 .claude/（如 src/, docs/）：理論可行（main-thread-edit-restriction 對 subagent 豁免，branch-verify 對 docs/ 也豁免）
- subagent Edit 主 repo 外（/tmp/）：理論可行
- agent-dispatch-validation-hook 強制 thyme 走 worktree，要驗證需另設 agent type 或拆三 ticket 各自只觸碰一類路徑，成本大於收益

### PC-115 更新方向

- 補機制推論段（runtime hardcoded protection）
- 補「不可覆蓋」三層（hook/settings/permissionMode）
- 補診斷不可見性（無 stderr 訊息）
- 補對照證據（PM 同檔同分支可寫）
