## Solution

<!-- To be filled by executing agent -->

依 W17-016.1 盤點結論落地 type-aware body schema：

1. **pm-rules/ticket-body-schema.md**：定義 ANA/IMP/DOC × 各章節的必填/選填/免填對照表，含語義定義與歷史豁免條款。
2. **ticket_builder.py**：新增 _schema_marker(ticket_type, section)，在 create_ticket_body 產出時於每章節標題後插入 HTML 註解 Schema[TYPE/Section] 標示狀態與 pm-rules 路徑。未知 type 不產標註（向後相容）。
3. **SKILL.md**：新增「Ticket Body Schema（type-aware）」章節，含摘要表與 pm-rules 連結。
4. **測試**：test_ticket_builder.py 新增 TestCreateTicketBodySchemaMarkers 5 個案例（ANA/IMP/DOC 標註驗證、未知 type 相容、pm-rules 路徑引用）。全部 49 個 ticket_builder 測試 + 43 個 validation/generator 測試通過。

**設計決策**：保持既有四章節結構（Problem Analysis / Solution / Test Results / Completion Info）不變，以 HTML 註解附加 schema 標註而非刪減章節，維持 acceptance_auditor、ticket_validator 既有 consumer 相容。
