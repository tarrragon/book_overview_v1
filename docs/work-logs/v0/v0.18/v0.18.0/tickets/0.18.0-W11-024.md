### 問題根因

跨 session 誤報的根因是 `dispatch-active.json` 殘留派發記錄未被及時清理：

1. **GC 時機單一**：僅 `active-dispatch-tracker-hook.py` (PostToolUse Agent) 觸發 `cleanup_expired()`，新 session 啟動時沒有 GC 觸發點
2. **TTL 過長**：`cleanup_expired()` 預設 `max_age_hours=4`，4 小時內殘留會被反覆當作活躍派發提醒
3. **後果**：跨 session 看到「2 個代理人執行中」這類過時提醒（PC-076、PC-078 二度發生根因之一）

### 影響範圍

| 檔案 | 行 | 現況 | 變更 |
|------|-----|------|------|
| `.claude/hooks/lib/dispatch_tracker.py` | 325 | `max_age_hours: int = 4` | default 改 1 |
| `.claude/hooks/active-dispatch-tracker-hook.py` | 80 | `cleanup_expired(project_root)` | 沿用新 default（隱式 1h） |
| `.claude/hooks/session-start-sync-exclusion-check-hook.py` | (新增) | 未呼叫 | 新增 `cleanup_expired(project_root, max_age_hours=1)` |
| `.claude/hooks/tests/test_dispatch_tracker.py` | 132 | `max_age_hours=4` 測試 | 保留參數化測試，補 1h 案例 |

### 設計決策（W/A/P）

- **W（候選）**：(a) 改 default + session-start 顯式傳 1（推薦，語意最清）；(b) 不改 default 所有呼叫端顯式傳；(c) 引入 TTL_HOURS 常數模組級宣告
- **A（機會成本）**：P2 ticket，不執行則 dispatch-active 殘留持續累積（W11-002 27 筆 stale 同源），PM 認知負擔不可逆堆積
- **P（最可能失敗）**：(1) session-start hook `dependencies=[]` 限制 → 用 `sys.path.insert` 模式與 hook_utils 一致；(2) 修改 default 漏改測試 → 已 grep 全 caller，無遺漏；(3) 失敗降級 → session-start hook 必須 try/except 包覆，cleanup 失敗不阻塞 session 啟動
