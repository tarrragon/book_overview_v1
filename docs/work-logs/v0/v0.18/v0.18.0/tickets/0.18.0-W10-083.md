### 實作摘要

新增 `ticket track td-status <ticket_id>` 子命令，掃描 ticket body + git commit 訊息，將每個 TD 編號分類為三狀態：

- **已處理（done）**：body 中 TD 編號鄰近「已處理 / 已修正 / 已完成 / 已關閉 / 已落地 / 已實作」等關鍵字，或 commit 訊息（透過 `git log --grep=<ticket_id>`）引用該 TD 編號
- **無需處理（skipped）**：body 中 TD 編號鄰近「無需處理 / 無需 / 不需處理 / 豁免 / N/A」等關鍵字（優先序高於 done，避免「TD5 無需處理；雖然 TD5 已修正 stub」被誤判）
- **仍待處理（pending）**：body 與 commit 均無上述訊號

### 設計取捨

| 議題 | 取捨 |
|------|------|
| AC 4「Phase 演進中提示校準缺失」 | 採「呼叫時即提示」（pending 數量 > 0 時輸出 PC-094 校準提示），不接 hook。理由：保持單一職責；hook 整合應另建 ticket。 |
| 訊號來源排序 | body 訊號優先於 commit；body 內 skip 優先於 done。理由：作者顯式標註語意最強。 |
| TD 編號正則 | `\bTD(\d+)\b(?!D)`，負向 lookahead 避免命中 `TDD`。 |
| git log 失敗處理 | 任何 git 異常（含 timeout/no-repo）回 empty set，命令仍可用。 |

- 新增 `.claude/skills/ticket/ticket_system/commands/track_td_status.py`
