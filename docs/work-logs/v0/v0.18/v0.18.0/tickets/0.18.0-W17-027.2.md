### 修復摘要

`resume --list` 改採 runqueue 排序（priority + ticket_id），與 `ticket track runqueue --context=resume` 同序，落地 W17-027 group「runqueue 為單一排序入口」方向。

### 設計決策

- **方向 B（委派 runqueue）落地方式**：在 `resume.py` 新增兩個 helper（`_load_ticket_for_handoff` / `_apply_runqueue_ordering`），import 並複用 `track_runqueue._priority_rank`，避免 priority 排序邏輯 duplicate。
- **排序鍵與 runqueue 一致**：`(priority_rank, ticket_id)`；P0→P1→P2→P3→未知，同 priority 內字母序。與 `track_runqueue._render_list` 完全相同。
- **保留 stale 過濾**：`list_pending_handoffs()` 既有 stale handoff 過濾不變；新排序套用於過濾後的清單，acceptance #4 保留。
- **CLI 輸出格式不動**：仍輸出 idx / id / title / timestamp 三項，acceptance #5 相容。
- **scan_pending_handoffs 字母序 sort 保留**：sorted(glob) 對 file iteration 提供決定論，呼叫端再排序後不影響輸出順序；移除會牽動 handoff_gc，超出本 ticket 範圍。

### 檔案修改

- `.claude/skills/ticket/ticket_system/commands/resume.py`：
  - import `_priority_rank` from `track_runqueue`
  - 新增 `_load_ticket_for_handoff(ticket_id)`：從 handoff ticket_id 載入 ticket
  - 新增 `_apply_runqueue_ordering(handoffs)`：以 (priority_rank, ticket_id) 排序，未知排最後
  - `_execute_list` 取得 handoffs 後呼叫 `_apply_runqueue_ordering`
- `.claude/skills/ticket/tests/test_resume_command.py`：新增 `TestRunqueueOrdering` 4 項測試

### 不在本 ticket 範圍

- `scan_pending_handoffs` 內部 sorted(glob) 移除（無收益且影響 handoff_gc）
- runqueue context=resume 的其他渲染變更
