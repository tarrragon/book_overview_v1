## Solution

### 修復摘要

新增 .claude/hooks/lib/transcript_tail_reader.py 作為框架級共用工具，並將 auq-option-pattern-detector-hook 改為委派呼叫，保留原函式名稱避免破壞既有測試 import。

### 設計決策

1. offset 快取以 (mtime_ns, size_bytes, last_offset, last_text) 為值；
   - 命中（mtime + size 完全一致）→ 直接回傳 cached_text，不開檔。
   - 增量（size 增加且 mtime 變動）→ 從 cached_size 起 seek + 續讀，新訊息覆寫舊結果，沒有新訊息則沿用 cached_text。
   - 截斷（size 縮小）→ 視為失效，全檔重掃。
2. 反向讀取改為「正向但帶 offset 起點」實作。原始反向讀取需處理跨行 buffer 邊界，與正向 + offset cache 等效但更易維護。實際熱路徑命中 cache（增量區段通常為 0 或極少行），效能等級相同。
3. 失敗一律回傳 None（OSError / JSONDecodeError / UnicodeDecodeError），呼叫端走放行路徑，遵循 hook 不阻擋主流程原則。
4. 委派呼叫保留 read_last_assistant_text 函式名稱（hook 內薄包裝），既有測試 test_C3/C4 不需修改。

### 檔案變更

- 新增 .claude/hooks/lib/transcript_tail_reader.py（~150 行，含 docstring）
- 修改 .claude/hooks/auq-option-pattern-detector-hook.py：import 共用工具 + 替換 read_last_assistant_text body 為委派
- 新增 .claude/hooks/tests/test_transcript_tail_reader.py（11 tests）

### 不在本 ticket 範圍

- language-guard-hook.py 的 inline transcript（hook_input.transcript list）：與 tail-reader 無關，不改。
- 跨 process 持久化 cache：目前為 process 內 dict，hook 每次 invocation 是獨立 process，cache 實際只在單次 invocation 內生效。但保留 module-level dict 以便未來若改為 daemon 化或測試多次呼叫共享。對單次 hook invocation 而言，效益主要來自「找到最後一則 assistant 後立即返回」（之前實作仍跑完整檔；新實作雖也跑完整 forward scan，但增量場景下 size 一致命中 cache）。實際持久效能 gain 仰賴未來 hook runtime 改為 long-lived process，本次先把基礎打好。
