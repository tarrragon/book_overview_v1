### 實作摘要

為 6 個類別 A 中頻 strict-validator hook 加 effort 感知（v2.1.133+），複用 W14-034 已落地的 `hook_utils.get_effort_level` helper。預設行為：`effort=low` 短路放行（除 `wrap-decision-tripwire-hook` 外），`medium`/`high` 走完整驗證；各 hook 加 `logger.info` 留痕。

### 各 hook 改動

| Hook | 改動 | low effort 行為 |
|------|------|----------------|
| `5w1h-compliance-check-hook.py` | 加 `get_effort_level` import；`main()` 讀 stdin 後檢查 effort | 短路：emit `{"decision":"allow"}` JSON |
| `phase-completion-gate-hook.py` | 加 `get_effort_level` import；於 subagent 檢查前判 effort | 短路：emit PostToolUse hookSpecificOutput |
| `phase-contract-validator-hook.py` | CLI 模式（sys.argv）；加 `sys.path` 與 hook_utils import；於 args 解析後判 effort（僅 `$CLAUDE_EFFORT`） | 短路：印「effort=low 短路放行」訊息，exit 0 |
| `wrap-decision-tripwire-hook.py` | 加 `get_effort_level` import；`main()` pytest 檢查後僅記錄 effort | 不短路（advisory；PC-066/PC-093 事實判斷訊號 quality-baseline 規則 6 防護），維持完整偵測 |
| `sibling-blockedby-validator-hook.py` | 加 `get_effort_level` import；`main()` 讀 stdin 後判 effort | 短路：return 0 |
| `layer-boundary-validator-hook.py` | 加 `get_effort_level` import；於 subagent 檢查前判 effort | 短路：emit success JSON |

### 設計決策

- wrap-decision-tripwire 不短路：本 hook 為 PC-066（決策品質）/ PC-093（延後決策累積）核心訊號偵測，屬「事實判斷」型 advisory hook。依 ticket 要求（acceptance #2）low effort 下仍須提醒，避免 effort 機制變相關閉品質防護（quality-baseline 規則 6）。
- phase-contract-validator CLI 模式特例：本 hook 透過 `sys.argv` 而非 stdin JSON 接收輸入，effort 訊號僅來源於 `$CLAUDE_EFFORT` 環境變數（符合 `get_effort_level(payload=None)` 設計）。
- logger.info 留痕：每個 hook 在 effort 判斷後記錄 `effort=<level>` 與行為決策（短路 / 完整驗證），便於 hook-logs 追蹤 effort 機制實際觸發頻率。

### 不在本 ticket 範圍

- W14-037（其他類別 hook 的 effort 感知）由獨立 ticket 並行處理
- W14-038（acceptance-gate / ticket-quality-gate 之外的 strict validator 整合）blockedBy 本 ticket + W14-037
