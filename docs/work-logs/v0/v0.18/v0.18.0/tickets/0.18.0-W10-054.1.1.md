## Solution

### 實作內容

1. **建立專案根目錄 `.gitattributes`**
   - `*.py`/`*.sh`/`*.bash` 強制 `text eol=lf`（shebang 保護）
   - `*.yaml`/`*.yml`/`*.toml`/`*.json` 強制 LF（設定檔一致性）
   - `*.md`/`*.txt` 強制 LF（跨平台 diff 雜訊消除）
   - `*.png`/`*.jpg` 等明確標示 `binary`

2. **建立 `.claude/.gitattributes`**（框架同步傳播用）
   - 涵蓋 `.py`/`.sh`/`.md`/`.yaml`/`.yml`/`.json`/`.toml`/`.txt`
   - 隨框架 sync 機制傳播到所有使用 `.claude/` 的專案

3. **更新 hook allowlist**（.gitattributes 主線程可編輯）
   - `.claude/hooks/lib/path_permission.py`：新增 `.gitattributes` 與 `.claude/.gitattributes` 豁免
   - `.claude/hooks/branch-verify-hook.py`：新增 `.gitattributes` 保護分支豁免
   - 依 ARCH-018 原則：Hook 撞牆即修 allowlist 而非繞路

### Windows 使用者後續操作指南

```bash
git config core.autocrlf false
git rm --cached -r .
git reset --hard
```

確保既有檔案以 LF 重新 checkout，避免 shebang 污染。
