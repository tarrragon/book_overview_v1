## Solution

### 變更內容

新增 `CheckpointCaller` Literal 型別別名於 `checkpoint_state.py`，收斂 caller 欄位集合：

```python
CheckpointCaller = Literal["snapshot", "dispatch-check", "handoff-ready", "unknown"]
```

改動點（3 處函式簽名）：

| 位置 | 變更 |
|------|------|
| `_write_metrics_log` `caller` 參數 | `Optional[str]` → `Optional[CheckpointCaller]` |
| `checkpoint_state` `caller` 參數 | `Optional[str]` → `Optional[CheckpointCaller]` |
| import | 新增 `Literal` 於 typing |

### 變更統計

- 檔案：1 個（`.claude/skills/ticket/ticket_system/lib/checkpoint_state.py`）
- 行數：+7 -3（含 type alias 宣告與註解）
- Callsite 更新：0（生產端尚無呼叫，僅測試端使用任意字串；Python runtime 不強制 Literal，測試保持原樣）

### 設計決策

- 生產集合定義在 Literal（snapshot / dispatch-check / handoff-ready / unknown），未來新增 caller 需同步擴充
- Python runtime 不強制 Literal，但 mypy/pyright 會警告誤植字串
- 測試端使用的 "test"/"perf-test"/"call0" 等任意字串保持不變（測試關注 log 寫入行為而非 caller 值合法性）
