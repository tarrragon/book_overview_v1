### 問題根因

`phase4-decision-enforcement-hook.py` 偵測到 ticket body 含「Phase 4 評估」「Phase 5 再決定」字串時報告 PC-093 延後話術命中。但當這些字串出現在 Context Bundle 自動抽取的 `[ref]` 開頭行時（如 W10-116 line 186 命中 W10-113 ANA 的 Source Acceptance 引用），屬於規則引用而非本 ticket 的延後決策。

W10-122 已新增 `rule-quote` 豁免類別，要求 reason 含 `.claude/rules/` 或 `.claude/pm-rules/` 路徑。但此模式下：
1. 觸發行不是引用 rules 路徑，而是引用其他 ticket 的 acceptance（屬 ANA 引用，非 rule 引用）
2. Context Bundle 是自動抽取機制（ticket-loader），作者無法控制內容
3. 即使可加豁免標記，每張 source ANA 含此字串的 ticket 都需手動加 marker，違反 DRY

### 影響範圍

| 影響面 | 說明 |

| 命中模式 | 任何 ticket 之 source ANA acceptance 含「Phase 4 評估」「Phase 5 再決定」（屬規則對話框 ref，非延後決策）|
| 觸發位置 | `## Context Bundle` section 內 `### Source Acceptance (reference)` 區段，`- [ref]` 開頭行 |

W10-116 的 Context Bundle 自動抽取自 W10-113（source_ticket）的 acceptance：

`[ref]` 是 ticket-loader 自動抽取的標記，這行是 W10-113 自身的 acceptance（屬規則性陳述「Phase 4 評估結論必須明確」），不是 W10-116 的延後決策。
