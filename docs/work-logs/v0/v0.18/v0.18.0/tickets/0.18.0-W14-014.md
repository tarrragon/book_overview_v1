## Problem Analysis

### 問題根因

W12-002.5 ANA 首版 JAPANESE_ONLY 候選清單（65 字元）誤列「鑑」U+9451 為日文專屬，實為繁日共用字（繁中常用「鑑別」「借鑑」「鑑於」）。掃描 3 個正常繁體文件產生 11 處 false positive（PC-050 / PC-070 / pm-agent-observability.md）。若此清單直接用於 Hook 攔截，會在 3 個合法繁體文件上觸發假警報，打斷正常工作流。

### 影響範圍

- 清單設計層：所有 CJK 字元集清單設計，包括未來可能的 JAPANESE_SHINJITAI Hook 擴充
- 姊妹模式：PC-074（繁簡共用字誤判）為同構問題的另一維度
- 已知 false positive 字：鑑 U+9451（主案例）+ 繁中異體字 体/誉/豊/拝（保守列入不可）

### 相關 Error Pattern

- PC-072 AUQ payload 字元集污染（主目標）
- PC-074 繁簡共用字誤判（姊妹模式）

### 解決方案

建立 PC-084 `trad-jp-shared-char-false-positive.md` 記錄此模式，結構仿 PC-074 但聚焦繁日維度：

1. **症狀 / 根本原因 / 陷阱模式**：明確說明繁日共用字對清單設計的干擾機制
2. **已知繁日共用字表**：22 個候選字逐字驗證結果（含 17 個確認日專可入 + 1 繁日共用禁入 + 4 繁中異體保守不入），每字標註碼位與繁中正統字／簡中對應
3. **防護措施**：5 層（清單驗證 / 繁中回歸測試 / 程式註解 / 文件 / 交叉引用）
4. **擴充流程**：4 步驟（收集候選→查字典→三類判定→繁中回歸測試），含 Python 測試範例
5. **象限歸類**：C 象限（增加摩擦），與 PC-074 同理由
6. **交叉引用**：與 PC-072 主目標、PC-074 姊妹模式互相連結

### 同步更新

PC-074 相關文件區塊新增 PC-084 交叉引用，版本升級 1.0.0 → 1.1.0，確保設計 CJK 清單時雙方都會被查到。
