## Task Summary

分析 AI 諮詢視野狹窄偏誤系統性防護方案（觸發訊號 + 分級機制 + 顯性標記）

---

## Problem Analysis

## 重現實驗（Reality Test 結果）

**實驗場景**：2026-04-16 Session，用戶詢問羽球拍選擇建議（真實對話）。

**穩定重現的偏誤**：
- 第 1 輪：問「初階中階高階差別」→ AI 給客觀知識（合理）
- 第 2 輪：用戶說「雙打新手、十年斷層」→ AI 仍用平均值推薦，未問個人條件（**第一次疏失**）
- 第 3 輪：用戶問 4U/5U 差別 → AI 繼續用群體統計（**第二次疏失**）
- 第 4 輪：用戶問單雙打差別 → 同上（**第三次疏失**）
- 第 5 輪：用戶親自指出「你沒問我身高體重性別」→ AI 承認疏失

**已驗證事實**：
1. AI 在個人化建議場景會用平均值假設替代詢問當事人條件
2. 此行為在連續三輪穩定重現，未被任何機制打斷
3. AI 自身寫下的推薦表格明確列出「性別/年齡/舊傷影響建議」，卻從頭到尾沒問用戶這些變數（自相矛盾）

**未問卻關鍵的變數**（13 個）：身高、體重、性別、慣用手、年齡、舊傷史、手掌大小、握力、預算、所在地區、球友程度、每週頻率、是否上教練課。

**仍未驗證的假設**：
- 偏誤頻率在其他領域（健康/金錢/法律）是否一致
- 不同 prompt 風格對偏誤強度的影響

---

## W — Widen Options（擴增選項）

原始二元陷阱：「每次都問完整條件 vs 完全不問」。擴增後光譜：
- A. 完全不問（現狀偏誤）
- B. 每次問完整問卷（笨重）
- C. **漸進式詢問**（先問 2-3 個最關鍵，再決定是否深入）
- D. **顯性標記假設**（「基於 X 假設，若不符請告知」）
- E. 給多版本答案讓用戶對號入座
- F. 模式切換（查詢/諮詢前確認）
- G. 結構化分層問卷（告知為什麼問）
- H. 用戶設定互動深度
- I. **觸發式詢問**（關鍵字偵測強制停下）

跨領域借鑑：醫生必問病史、理專必問 KYC、教練必問身體狀況、律師必問事實、裁縫必量尺寸——**所有專業以「先問後答」為標準，AI 反其道而行屬文化偏誤非技術限制**。

---

## R — Reality Test（假設檢驗）

| AI 潛在假設 | 證據強度 | 反例 |
|-----------|---------|------|
| 「問問題破壞流暢度」 | 弱 | 用戶發現疏失更不滿 |
| 「用戶要速度 > 準確」 | 弱 | 諮詢場景願意花時間 |
| 「詢問代表無能」 | 錯誤 | 所有專業必問是專業象徵 |
| 「平均值可代表多數人」 | 部分錯 | 個人化場景準確率約 30-50% |
| 「用戶會主動補充關鍵資訊」 | 錯誤 | 用戶不知什麼是關鍵變數 |
| 「給完整答案比給對答案重要」 | 錯誤 | 完整但錯的答案更危險 |

**反向提問**：若付費諮詢場景會先問清楚，免費場景降低標準是錯誤的差別待遇。

---

## A — Attain Distance（抽離距離）

**10/10/10 檢驗**：
- 10 分鐘：對話當下看似流暢
- 10 個月：用戶可能買錯裝備、受傷、損失金錢
- 10 年：AI 諮詢信任度的基礎設施會被此選擇決定

**Core Priorities 反轉**：
| 優先級 | 當前 AI 行為 | 應有的行為 |
|-------|-----------|----------|
| 1 | 快速 | 準確 |
| 2 | 博學 | 誠實 |
| 3 | 完整 | 個人化 |
| 4 | 流暢 | 負責 |

**第三者視角**：資深人類顧問的核心技能不是「知道很多答案」，是「知道自己不知道什麼」「知道該問什麼」「知道何時該停下來」。AI 目前展現前者缺後三項。

---

## P — Prepare to be Wrong（準備出錯）

**不修正的 Premortem**：
- 用戶買錯裝備→受傷（實質傷害，高機率）
- 財務建議錯誤→損失（中機率）
- 健康建議錯誤→不可逆傷害（低機率但嚴重）
- 整體信任度崩壞（高機率累積效應）

**改採「先問後答」的失敗場景與對策**：
| 失敗場景 | 對策 |
|---------|------|
| 問太多用戶反感 | 分級詢問（低/中/高風險） |
| 問題不精準 | 用領域知識預判關鍵變數 |
| 用戶只想快速答案 | 提供「快速版（基於 X 假設）」vs「精準版（先問）」讓用戶選 |
| 簡單問題被過度複雜化 | 建立「查詢 vs 諮詢」判斷規則 |

## 三層機制設計

### 第一層：識別（Detection）—— 觸發訊號清單

**個人化關鍵字**（任一命中即進入檢查）：
- 第一人稱 + 需求動詞：「我要」「我該」「推薦給我」「我應該」「適合我」「給我建議」
- 個人化代名詞：「我的」「你覺得我」「像我這種」

**高風險領域關鍵字**：
| 領域 | 關鍵字 |
|------|-------|
| 健康 | 吃什麼、身體狀況、運動、減肥、受傷、症狀 |
| 運動裝備 | 球拍、跑鞋、健身器材、護具 |
| 金錢 | 投資、買入、預算、要不要花、理財、基金、股票 |
| 法律 | 合約、訴訟、可以告、權利、違法 |
| 安全 | 安全嗎、會不會出事、危險、風險 |

**對話輪次閾值**：
- 對話超過 3 輪仍未問過個人條件 → 強制回頭問

**強制檢查動作**：
偵測到觸發訊號 → 執行「個人化建議前 30 秒自查清單」：
- [ ] 我知道當事人是誰嗎？（年齡/性別/身體條件）
- [ ] 我知道當事人的目標嗎？
- [ ] 我知道當事人的限制嗎？（預算/時間/禁忌）
- [ ] 我知道當事人的環境嗎？（地區/可用資源）
- [ ] 我在用「平均值假設」替代「個人資料」嗎？
- [ ] 如果假設錯誤，後果會嚴重嗎？

若「後果嚴重且資料不足」→ 必須先問。

---

### 第二層：分級（Tiering）—— 查詢 / 諮詢 / 高風險

| 風險等級 | 判定條件 | 詢問策略 |
|---------|---------|---------|
| **低風險** | 客觀知識查詢（規格、定義、原理、歷史） | 直接答，**可選**標註假設 |
| **中風險** | 個人化建議（選裝備、選習慣、選工具、選方法） | **漸進式問 2-3 個關鍵變數**再答 |
| **高風險** | 健康/金錢/法律/安全決策；不可逆後果；超過用戶預算顯著影響 | **強制完整前置問卷**；答前建議諮詢專業人士 |

**判定優先順序**：
1. 先偵測高風險關鍵字 → 若命中直接走高風險策略
2. 再偵測個人化關鍵字 → 命中走中風險
3. 皆未命中 → 低風險

**中風險的 2-3 個關鍵變數選擇原則**：
- 預判「這些變數答案會根本性改變建議」
- 優先問「無法事後修正」的條件（如舊傷、禁忌）
- 用戶回答不了時提供假設選項讓其選擇

---

### 第三層：誠實（Transparency）—— 顯性標記假設

**標記格式**（強制包含三要素）：
```
此建議基於以下假設：[列出使用的假設]
若你實際是 [其他情境]，建議會改為 [替代答案]
請確認假設是否符合你的狀況。
```

**標記位置**：
- 必須出現在結論段落開頭或結尾
- 禁止隱藏在附註或中間段落
- 若為多假設，必須列點而非混在一段文字內

**觸發條件**：
| 風險 | 標記 |
|------|------|
| 低風險 | 可選（建議對非常識性知識標註） |
| 中風險 | **強制** |
| 高風險 | **強制 + 建議諮詢專業人士** |

## Tripwire（絆腳索）

本機制的預警線：
| 觸發訊號 | 行動 |
|---------|------|
| 用戶「我/我該/推薦給我」+ 個人化主題 | 停下來檢查資訊充足度 |
| 涉及健康/金錢/法律/安全決策 | 強制完整問卷 + 建議專業諮詢 |
| 要給具體品牌/型號/金額/劑量 | 檢查是否有個人化資料支持，否則強制標記假設 |
| 對話超過 3 輪仍未問個人條件 | 強制回頭問「為了更準確我需要了解 X」 |
| 用 WRAP 或其他決策框架時 | 納入「資料充足度」為 Step 0 強制項 |
| 用戶反饋「問太多了」 | 檢視分級機制是否把中風險誤判為高風險 |
