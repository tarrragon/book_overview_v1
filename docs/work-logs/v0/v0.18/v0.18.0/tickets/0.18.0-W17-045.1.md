## 變更摘要

### 1. sync-claude-push.py EXCLUDE_PATTERNS 分類注解強化

.claude/scripts/sync-claude-push.py 的 EXCLUDE_PATTERNS 區塊上方新增四類分類說明（類型 A Runtime state / B Local-only settings / C Session-bound log / D 敏感憑證），並在集合內部以注解分區重組項目順序，使每個項目的歸類一目了然。注解末尾引用 .claude/references/sync-exclusion-guide.md 作為新增機制時的決策權威。

### 2. sync-claude-pull.py LOCAL_ONLY 對稱更新

.claude/scripts/sync-claude-pull.py 的 LOCAL_ONLY 區塊採用對稱但簡化的三類說明（A/B/C，類型 D 不適用於 pull 端因遠端本無憑證）。注解強調「新增機制時須同步更新 push 端」的雙向對稱要求，防止未來發生單向漂移。

### 3. 新增 .claude/references/sync-exclusion-guide.md

新文件含：

- 四類分類定義（特徵、生命週期、範例、辨識問題）
- 4 項 checklist（分類確認 / push 端排除 / pull 端對稱 / 驗證與文件）
- 決策流程圖（Q1-Q4 路由）
- 反模式對照表（單向漂移、未分類、變體漏網、誤用 preserve）
- 與既有機制關係表（sync-preserve.yaml、EXCLUDE_NAME_PREFIXES、EXCLUDE_SUFFIXES）
