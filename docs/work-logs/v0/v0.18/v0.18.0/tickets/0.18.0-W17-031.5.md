### 修復摘要

在 `session-start-scheduler-hint-hook.py` 新增第三節 `## NeedsContext 警示`：

1. 新增常數：`NEEDS_CONTEXT_DISPLAY_LIMIT=3`、`HANDOFF_PENDING_RELPATH=".claude/handoff/pending"`、`EXIT_STATUS_NEEDS_CONTEXT="needs_context"`
2. 新增 `scan_needs_context_handoffs(project_root, logger)`：掃描 pending JSONs，找 `exit_status == "needs_context"`，回傳 ticket_id 清單。欄位缺失視為非 needs_context（schema 解耦）；目錄不存在 / 壞 JSON 皆靜默忽略。
3. 新增 `build_needs_context_section(ticket_ids)`：格式化 markdown，顯示總數 + 最多 3 個 ID + 省略提示 + 建議動作（`ticket track full <id>` 檢視）。
4. 新增 `fetch_needs_context_section(logger)`：fail-open 包裝，任何步驟失敗回 None。
5. `build_hook_output()` 擴充第三參數 `needs_context_section`，三段獨立顯示，皆無時 `suppressOutput=True`。
6. `main()` 串接新流程，logger.info 記錄三段是否輸出。

### 設計決策

- **Schema 解耦**：W17-031.2 尚未擴充 JSON，本 ticket 在無欄位時不報錯。確保 .5 可獨立先行（與 ticket when 欄位一致）。
- **Fail-open**：所有路徑失敗（讀檔錯、JSON parse 錯、root 取得錯）皆回 None / [] / 不阻擋 SessionStart。
- **顯示上限 3**：避免警示洗版；> 3 顯示「…其餘 N 筆省略」。
- **獨立小節**：與 `## 排程提示` / `## Spawned 推進清單` 並列，PM 一眼可區分。

### 不在本 ticket 範圍

- 擴充 handoff JSON schema（W17-031.2）
- 修改 runqueue 讀取 exit_status（W17-031.1）
- handoff readiness tag（W17-031.3）
