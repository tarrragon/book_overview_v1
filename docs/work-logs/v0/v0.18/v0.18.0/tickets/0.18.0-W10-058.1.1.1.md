### 核心邏輯

`.claude/hooks/wrap-decision-tripwire-hook.py` 的 `main()` 函式入口處新增早期豁免：

```python
import os
from pathlib import Path

def is_pytest_environment() -> bool:
    if os.environ.get("PYTEST_CURRENT_TEST"):
        return True
    cwd_str = str(Path.cwd())
    if "pytest-of-" in cwd_str:
        return True
    return False
```

在 `main()` 函式 `event = read_json_from_stdin(logger)` 之後、實際 detection 邏輯之前加入：

```python
if is_pytest_environment():
    logger.debug("pytest environment detected, skipping detection")
    return 0
```

### 實作摘要

於 `.claude/hooks/wrap-decision-tripwire-hook.py` 新增 `is_pytest_environment()` 函式並在 `main()` 入口（`read_json_from_stdin` 之後、`get_project_root` 之前）加入早期豁免：

- 偵測 `PYTEST_CURRENT_TEST` env var（pytest runtime 自動注入）
- 偵測 `Path.cwd()` 含 `pytest-of-`（pytest tmp_path fixture 慣例）
- 任一成立即 `logger.debug` 後 `return 0`，不執行 detection
- `Path.cwd()` 包 try/except 防 FileNotFoundError/OSError（cwd 已刪情境）

### 設計決策

1. 早期 return 位置：在 `read_json_from_stdin` 之後而非函式最開頭，保留 stdin 讀取以維持 hook input contract。
2. 函式提取為 module-level：方便測試以 monkeypatch 直接驗證偵測邏輯。
3. 不改動 detection 主邏輯：豁免只在 main() 入口，process_signals 等內部函式行為不變（讓既有單元測試直接呼叫 strategy.detect 仍能驗證 hit 行為）。

### 不在本 ticket 範圍

- 既有 hit 2 案例改為斷言不觸發：本檔現有測試直接呼叫 strategy.detect/process_signals 而非走 main()，未受 hit 2 fixture 污染影響；MVP 範圍僅實作 main() 早期 return。
- 既有 TestK_DocSync test_k2/k3 失敗為框架文件同步測試（與本 ticket 無關），已驗證 stash 前後皆失敗。
