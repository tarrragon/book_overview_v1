## Solution

### 修復摘要

新建 `.claude/skills/ticket/ticket_system/commands/track_dispatch_validate.py`，提供獨立子命令 `ticket track dispatch-validate <ticket_id>`，在 target ticket 的 Context Bundle 自動填料結果上做合理性檢查。實作 4 項硬/軟規則 + Exit code 三分（0=pass / 1=warn / 2=fail）。規則 5（LLM 審查）為保留 hook 點，不在本 ticket 範圍。

### 設計決策

| 決策 | 內容 | 理由 |
|------|------|------|
| 獨立子命令 | `dispatch-validate` 而非擴充 `dispatch-check` | 職責正交：dispatch-check 為活躍派發狀態查詢（W10-017.2），dispatch-validate 為 Context Bundle 內容合理性檢查（W17-003） |
| version-aware 註冊 | 加入 `_create_command_handlers()` 字典 | 需讀取 ticket md，必須 resolve version |
| 規則 1 硬性 / 規則 2-4 軟性 | exit 2 vs exit 1 | 規則 1 違反代表抽取機制完全失效；規則 2-4 違反代表內容品質不足但仍可派發 |
| 純函式拆分 | `check_section_present` / `check_content_length` / `check_where_files_exist` / `check_acceptance_count` 各自獨立 | 認知負擔低、易單元測試、未來可被其他 hook 重用 |
| where.files 空清單視為通過 | 規則 3 跳過 | DOC 類 ticket 常無 where.files；避免誤報 |
