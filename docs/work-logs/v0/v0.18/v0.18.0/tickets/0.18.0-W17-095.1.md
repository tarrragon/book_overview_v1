### 待新增 API 規格

```python
def is_handoff_stale(record: dict) -> tuple[bool, str]:
    """判斷 handoff record 是否為 stale。

    Args:
        record: 從 handoff/pending/*.json 載入的 dict（含 from_ticket / to_ticket / direction / from_status 等）。

    Returns:
        (is_stale, reason)
        is_stale=True 時 reason 為人類可讀說明（例：「任務鏈目標 W17-002 已 in_progress」）。
    """
```

### 4 個 stale 情境（acceptance 覆蓋）

| 情境 | 條件 | 預期 reason |
|------|------|------------|
| 1. 任務鏈目標啟動 | direction in {to-sibling/to-child/to-parent} AND target_id ticket status in {in_progress, completed} | 「任務鏈目標 {id} 已 {status}」 |
| 2. 非任務鏈 source 已完成 | direction not in 任務鏈 AND from_ticket status == completed | 「來源 ticket {id} 已 completed」 |
| 3. from_status 已 completed | record["from_status"] == "completed" AND 非任務鏈 | 「from_status 標記 completed」 |
| 4. 未完成（不 stale） | 上述皆不滿足 | "" |

### 實作摘要

於 `.claude/skills/ticket/ticket_system/lib/handoff_utils.py` 新增 is_handoff_stale(record) -> (bool, str)：

- 收斂三套消費者（reminder-hook / stop-hook / handoff_gc）對 stale 的判斷規則
- 依序檢查 4 情境：任務鏈目標已啟動 → 非任務鏈來源 completed → from_status 標 completed → 否則非 stale
- 引用既有 helper：is_task_chain_direction / extract_direction_target_id / is_ticket_in_progress_or_completed / is_ticket_completed / load_and_validate_ticket
- 任務鏈情境額外讀取 target 實際 status 以填入 reason（in_progress / completed）
- 向後相容：from_ticket 缺失時 fallback 到 ticket_id

### 設計決策

- 不改動 handoff_gc._collect_stale_handoffs：避免 .1 改動引發 .2/.3 漂移（Context Bundle 邊界明示）
- reason 字串遵循 Context Bundle 4 情境表的「預期 reason」格式
- 例外處理：load_and_validate_ticket 失敗時 status fallback 為 in_progress（保守策略）

### 不在本 ticket 範圍

- handoff_gc / reminder-hook / stop-hook 接線改造留 .2/.3
