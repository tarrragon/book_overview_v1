## Solution

### 方案選擇：選項 A（獨立新檔）

**決策依據**：

`writing-prompts.md` 原始 541 行中，全部內容均屬 prompt 寫作情境（ai-communication-rules.md 詳細版庫）。「Skill 品質規範」職責僅存在於：
1. 檔案定位聲明（第 5-17 行）的角色 2 宣告
2. 閱讀決策表格中「作為 skill 新增 reference 的撰寫範本 | 本檔整體架構」一行

沒有任何獨立的品質規範章節——整份文件都是 prompt 寫作。因此：
- 抽出後 writing-prompts.md 殘留 540 行（遠超 200 行 WRAP P 防護門檻），選項 A 成立
- 需要創建真正有實質內容的 reference-authoring-standards.md，而非只是移除宣告

### 執行步驟

1. **新建 `reference-authoring-standards.md`**：
   - 五項必要要素（情境聲明、職責單一性、自包含性、結構一致性、範例具體性）
   - 新增 reference 的決策流程（四問）
   - 驗收清單（23 項）
   - 與 meta-metrics.md 的分工說明

2. **編輯 `writing-prompts.md`**：
   - 移除角色 2 宣告（「compositional-writing skill 的產出品質規範」）
   - 更新閱讀決策表格：移除「作為 skill 新增 reference 的撰寫範本」行，新增指向 `reference-authoring-standards.md` 的行

3. **編輯 `SKILL.md`**：
   - 觸發路由表新增：「要新增或修改一份 Skill reference」→ `references/reference-authoring-standards.md`
   - Directory Index 新增 `reference-authoring-standards.md` 條目

4. **編輯 `ai-communication-rules.md`**：
   - 相關文件中 writing-prompts.md 描述移除「skill 產出品質規範」字樣
