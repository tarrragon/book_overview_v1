## 實作摘要

新建 `.claude/hooks/proposal-evaluation-gate-hook.py`（PreToolUse Hook，事件 Write/Edit），落地 proposal-evaluation-gate 規則 4 + 規則 5 第三層強制。

### 設計決策

| 決策點 | 選擇 | 理由 |
|--------|------|------|
| Hook 事件 | PreToolUse + matcher Write/Edit | 寫入前阻擋，避免不合規 PROP 落盤 |
| 阻擋方式 | hookSpecificOutput permissionDecision=deny + stderr 同步輸出 | 規則 4 雙通道（用戶可見 + 日誌） |
| 解析方式 | YAML frontmatter（PyYAML） + body 關鍵字章節匹配 | 章節寫法多樣，採關鍵字陣列 OR 匹配 |
| 微調豁免 | Edit/MultiEdit 字元差 < 30 視為格式調整 | 拼字/排版類調整不應觸發完整評估 |
| ANA-only 偵測 | 採「ticket_refs 非空」第一階段 | 避免本 hook 過度耦合 ticket type 解析；ANA-only 需另查 ticket md，留待 follow-up |
| 異常處理 | 解析失敗 → allow + warning 日誌 | 避免半成品 yaml 寫入過程被卡死（IMP-049：明確分支非吞掉異常） |

### 攔截邏輯

1. 命中目標檔案？（docs/proposals/PROP-*.md 或 docs/proposals-tracking.yaml）→ 否則 return 0
2. 微調豁免？（Edit/MultiEdit 且字元差 < 30）→ 是則 return 0
3. 取得寫入後完整內容（Write: tool_input.content；Edit: 讀現檔 + 套用 edits）
4. 分派檢查：
   - PROP-*.md：frontmatter 必含 evaluation_level；standard/heavy 必含對應章節
   - tracking.yaml：confirmed/approved 狀態必有非空 ticket_refs
5. 違規 → deny + stderr；通過 → return 0

### 註冊位置

.claude/settings.json PreToolUse 區塊，加入 Edit matcher 與 Write matcher 兩處（與既有 ticket-path-guard / framework-rule-edit-skill-trigger 並列）。

### 既有 PROP 影響

既有 12 個 PROP-*.md 皆無 evaluation_level 欄位，本 hook 上線後對這些檔案的下次 Edit 將被阻擋（除非微調 < 30 字元）。屬預期行為（規則 1 補完缺口），需後續 DOC ticket 為既有 PROP 補欄位。本 ticket 範圍內不處理。

### Schema 補強

docs/proposals-tracking.yaml 既有 schema 已含 ticket_refs 欄位，無需額外修改 yaml schema。AC 第 3 項以本 hook 的執行驗證取代額外 schema 檔。
