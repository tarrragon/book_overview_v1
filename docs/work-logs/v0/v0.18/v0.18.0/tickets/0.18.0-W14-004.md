## Problem Analysis

### 背景
W12-005 lifecycle.py 新增 `_SPAWNED_TERMINAL_STATUSES` 時於註解明示："與 .claude/hooks/acceptance_checkers/children_checker.py 對齊"。語義一致但雙邊各自宣告，違反 DRY 原則。

### 現況盤點
- children_checker.py:29 `TERMINAL_STATUSES: Set[str] = {"completed", "closed"}`
- ana_spawned_checker.py:20 `from acceptance_checkers.children_checker import TERMINAL_STATUSES`（已正確複用）
- lifecycle.py:1086 `_SPAWNED_TERMINAL_STATUSES = (STATUS_COMPLETED, STATUS_CLOSED)`（類型不同：tuple vs set）
- ticket_system/lib/constants.py 有 STATUS_COMPLETED / STATUS_CLOSED 但缺 TERMINAL_STATUSES 聚合常數

### WRAP 三問
- **W (Widen)**：四個候選方案
  1. 集中到 ticket_system/lib/constants.py，hook 透過 sys.path 引入（選中，對齊 ticket 預設 files 清單）
  2. 集中到 hooks/lib/constants.py，skill 引入（倒置依賴，不推薦）
  3. 新建 .claude/shared/ 模組（過度設計，P3 不值得）
  4. 保留雙宣告 + 同步測試（違反 AC #2 同處 import）
- **A (Attain distance)**：機會成本 = 引入 hook→skill 跨套件 import 的脆弱性；但 acceptance_checkers 已有 sys.path 注入模式（_hooks_dir），ticket-reinstall-hook.py 也有類似跨模組模式，成熟度足夠
- **P (Prepare to be wrong)**：最可能失敗 = acceptance-gate-hook 從非標準位置執行時 sys.path 無法解析 ticket_system 位置；防護 = 保留路徑解析為相對 _hooks_dir.parent/skills/ticket，加測試驗證

### 實作範圍
- 在 constants.py 新增 `TERMINAL_STATUSES = frozenset({STATUS_COMPLETED, STATUS_CLOSED})`
- lifecycle.py 改用 `from ticket_system.lib.constants import TERMINAL_STATUSES`
- children_checker.py 改用 sys.path 注入 + 從 ticket_system 引入（保留本模組 re-export 供 ana_spawned_checker 續用）
- 更新 PC-075 文件中「權威來源」的指向

### 預計影響
- 運行時行為不變（frozenset 支援 `in` 運算等同 set/tuple）
- 向下相容：ana_spawned_checker 維持原本 `from .children_checker import TERMINAL_STATUSES`

### 變更摘要

1. **constants.py** 新增單一來源：
   ```python
   TERMINAL_STATUSES: frozenset = frozenset({STATUS_COMPLETED, STATUS_CLOSED})
   ```
   frozenset 保證不可變，支援 `in` 運算，與原本 set/tuple 呼叫端完全相容。

2. **lifecycle.py** 改為從 constants.py import：
   - 移除本地 `_SPAWNED_TERMINAL_STATUSES = (STATUS_COMPLETED, STATUS_CLOSED)`
   - import 新增 `TERMINAL_STATUSES`
   - `_collect_non_terminal_spawned` 改用共用常數

3. **children_checker.py** 透過 sys.path 注入 ticket_system，改為 re-export：
   - 移除本地 `TERMINAL_STATUSES: Set[str] = {"completed", "closed"}`
   - 注入 `.claude/skills/ticket` 至 sys.path（仿照既有 _hooks_dir 注入模式）
   - 從 ticket_system.lib.constants import 後 re-export
   - ana_spawned_checker.py 無需修改（透過 children_checker re-export 向後相容）

4. **PC-075** 更新權威來源指向 constants.py（本次整併結果）。

### 設計決策
- **方案選中**：集中到 ticket_system/lib/constants.py（ticket 預設 files 清單第三項）
- **方案 A 否決**：不採用 hooks/lib/constants.py 為權威源（倒置依賴）
- **方案 B 否決**：不新建 .claude/shared/（過度設計）
- **方案 C 否決**：不保留雙宣告 + 同步測試（違反 AC #2）

### 跨套件 import 風險緩解
- 路徑解析採相對 _hooks_dir.parent/skills/ticket，不依賴 cwd 或環境變數
- 保留 hook 端的 sys.path 注入模式與現有 _hooks_dir 邏輯一致
- ticket_system 作為 ticket CLI 核心套件，必然存在（否則 hook 也無從被觸發）
