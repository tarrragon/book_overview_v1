### 削減比結論

實測 **57.8%** vs 預估 **85%**，差距 27.2 百分點（> 20% 閾值）。

### 差距根因

1. **降級 ≠ 移除**：8 hook 中僅 parallel-dispatch-verification 從 settings.json 完全移除（83.2% 削減）。其餘 7 hook 仍觸發，每次 spawn Python 進程 + 寫 log，只是內部邏輯改為 fast-path exit 0 或抽樣。
2. **高頻 hook 削減不足**：bash-edit-guard（佔 pre 總量 26.6%）和 acceptance-gate（26.5%）各僅削減 46.2%，拉低整體。
3. **預估假設偏差**：W10-035.3 的 ~5500/3d 預估可能假設這些 hook 的觸發會被消除，實際僅是處理降級。

### 建議 PM 行動

1. 建立檢討 ticket（差距 > 20%）：評估是否需進一步降級（移除 bash-edit-guard / acceptance-gate 註冊，或縮減 matcher）
2. file-type-permission Action 比上升（0.7% -> 5.5%）納入追蹤
3. 延長觀察至完整 2 Wave 後更新追蹤表
4. 修正預估方法：區分「觸發消除」vs「處理降級」兩種機制的不同削減效果
