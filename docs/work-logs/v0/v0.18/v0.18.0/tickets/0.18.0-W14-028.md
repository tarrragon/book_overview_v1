## Phase A IMP-1 實作指引（依 W14-024 結論方案 E 主路徑）

### 目標
新建 PostToolUse Hook，偵測 subagent 或 PM 直接 Edit/Write ticket .md 檔案時 YAML frontmatter 的合規性，違規時 stderr 警告 + hook-logs 記錄（不阻擋 Edit）。

### 檔案位置
- 新建：`.claude/hooks/ticket-frontmatter-validator-hook.py`
- 修改：`.claude/settings.json` 註冊 PostToolUse

### Hook 行為規範

**觸發條件**：
- PostToolUse 事件
- 工具為 Edit 或 Write
- 目標路徑符合：`docs/work-logs/**/tickets/*.md`（glob 模式）

**驗證邏輯**：解析 YAML frontmatter，檢查以下欄位：

| 欄位 | 合規規則 | 違規範例 |
|------|---------|---------|
| `status` | ∈ {pending, in_progress, completed, blocked, closed} | `complete`（非 completed）|
| `completed_at` | ISO 8601 格式 `YYYY-MM-DDTHH:MM:SS`（或 null）；HH ∈ [00, 23] | `T24:00:00`（無效小時）|
| `acceptance` | 必為 YAML list，每項為獨立字串，不含 `,[ ]` 或 `,[x]` 分隔 | 4 條擠單行 `'[x] A,[ ] B,[ ] C'` |
| `who` | 為 object 含 `current` 欄位（非 plain string） | `who: thyme-documentation-integrator`（plain string）|

**違規處理**：
- 寫 stderr：`[ticket-frontmatter-validator] WARNING: {ticket_id} {field} 違規：{detail}`
- 寫 hook-logs（檔案路徑同既有 hook 慣例）
- **不阻擋** Edit（exit 0）
- 可選：建議修復命令（如 AC 違規時建議 `ticket track check-acceptance <id> --all`）
