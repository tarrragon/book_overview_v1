## 實作摘要

**根因 RCA**（原動機修訂）：
之前派發 thyme 被擋的真實原因**不是** hook 規則設計錯誤，而是**短 prompt 未包含 .claude/ 路徑線索**。Hook 邏輯本身正確（line 1019-1026 已有全主 repo .claude/ 豁免），但依賴 PM 在 prompt 顯式寫路徑。當 PM 寫短 prompt（「Ticket: X，Read md 依規格實作」），hook 無從分類，fallback 到 worktree 強制 → 與 ARCH-015 衝突。

**修法**：在 `_classify_prompt_paths` 不明時（全 False），從 prompt 擷取 ticket ID，讀 ticket md frontmatter `where.files`，按路徑補分類。

**變更**：
- `.claude/hooks/agent-dispatch-validation-hook.py`
  - 新增 `_TICKET_ID_PATTERN`（完整 ID 格式 X.Y.Z-WN-NNN[.N]）
  - 新增 `_extract_ticket_ids(prompt)` 擷取 ID
  - 新增 `_load_ticket_where_files(ticket_id)` 解析 frontmatter yaml
  - 新增 `_classify_paths(paths, root)` 通用路徑分類
  - `main()` 在 `_classify_prompt_paths` 結果全 False 時，fallback 讀 ticket where.files
- `.claude/rules/core/pm-role.md` 註記 W17-018 補強機制

**測試**：`.claude/hooks/tests/test_agent_dispatch_validation_hook.py` 加 5 項 W17-018 測試；全 186/186 passed（181 既有 + 5 新）。

**驗收對照**：
- hook 豁免全 .claude/ ticket：`test_hook_fallback_to_ticket_when_prompt_has_no_paths`
- 混合路徑擋並提示：現有 test_classify_cross_paths + worktree 檢查鏈
- 無 where.files 維持現行：既有邏輯不變
- 測試覆蓋：5 新 test + 181 既有
- pm-role.md 同步：已註記
