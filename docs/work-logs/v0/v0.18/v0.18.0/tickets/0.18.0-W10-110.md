## Problem Analysis

### 問題根因

W10-099 多視角審查仲裁結論（方案 D）分兩步落地：

- **IMP-A（W10-109）**：修補 hook，新增 `status=draft` 豁免分支（豁免優先序 P2）。
- **IMP-B（本 ticket）**：解除舊有 grandfather 妥協，讓 PROP-001/003 的 `evaluation_level` 反映實際工程強度。

根因：PROP-001/003 初建時一律標 `evaluation_level: light` 作為 grandfather 妥協（避免 hook 阻擋未完整的 draft），而非反映提案的真實設計強度。IMP-A 落地後，`status=draft` 本身即豁免章節檢查，grandfather 妥協的歷史原因消除，level 應還原為實際值。

| `docs/proposals/PROP-003-backup-recovery.md` | `evaluation_level: light → standard`（單版本功能 v0.16.3 備份恢復） |

# 豁免優先序 P2：status=draft 探索期豁免章節檢查
