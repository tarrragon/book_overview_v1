### 實作摘要（PM 前台執行）

**Hook 修改**: `.claude/hooks/agent-dispatch-validation-hook.py`

新增邏輯：
1. `_classify_prompt_paths(prompt)` 函式：以正則偵測 prompt 中 `.claude/` 與常見專案路徑（src/、tests/、lib/、docs/、app/、assets/、scripts/、public/、bin/、cmd/）
2. `CROSS_PATH_BLOCK_MESSAGE` 常數：跨路徑情境的指引訊息（引用 ARCH-015）
3. main() 內三分類決策：
   - 僅 `.claude/` → 放行（exit 0，ARCH-015 豁免）
   - 跨路徑 → 阻擋（exit 2，含拆分指引）
   - 僅非 `.claude/` 或空 prompt → 原強制邏輯（exit 2）

**關鍵設計**：`_NON_CLAUDE_PATH_PATTERN` 使用 `(?<![./\w])` lookbehind 排除 `.claude/docs/` 這類巢狀路徑被誤判為 docs/，落實 ARCH-015 `.claude/` 下任何巢狀路徑都視為 .claude/ 內部變更的規則。

**新測試檔**: `.claude/hooks/tests/test_agent_dispatch_validation_hook.py`

測試結構（25 個測試）：
- **_classify_prompt_paths 單元測試**（7 + parametrize 11）= 16 個
  - 空 prompt、僅 .claude/、僅 src/、各常見路徑前綴、跨路徑、巢狀 .claude/docs/、無路徑
- **main() 整合測試**（8 個）
  - 非實作代理人放行、worktree isolation 放行、僅 .claude/ 豁免、僅非 .claude/ 阻擋、跨路徑阻擋、空 prompt 回退、缺 subagent_type 放行、非 Agent 工具放行

**對 W10-041 衝突的解除**:
- 本次 W10-039.3 和 W10-042 皆需修改 .claude/，依新邏輯 thyme 派發不需 worktree（但本次仍 PM 前台執行因 Hook 變更前的衝突仍存在，下次派發可驗證新邏輯）
