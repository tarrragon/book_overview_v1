## Task Summary

ticket CLI create 目前僅有 --parent 建立父子關係；ANA 衍生 IMP 的場景實際需要 spawned/source 衍生關係，但缺 CLI 參數導致 PM 只能先用 --parent 再手動 Edit 5+ 檔案修正。新增 --source-ticket 參數讓 spawned 關係一步到位

---

## Problem Analysis

### Context Bundle（PC-040：PM 派發前分析輸入）

#### 問題根因（來自 PC-073）

PM 執行 ANA 類型 Ticket 完成後，為調查結論建立後續防護 IMP 時：
1. 只能用 `ticket create --parent {ANA-ID}`（CLI 唯一關聯參數）
2. CLI 自動設定為 ANA 的 `children`（父子關係）而非 `spawned_tickets`（衍生關係）
3. PM 執行 `ticket track complete {ANA-ID}` 時被 acceptance-gate-hook 擋下
4. 必須手動 Edit 5+ 個檔案清除 parent_id/children、填 source_ticket 才能 complete

#### 概念模型區分

- 父子關係（children/parent_id）：必須一起交付；子未完成父無法 complete
- 衍生關係（spawned_tickets/source_ticket）：獨立交付時機；衍生項不阻擋 source complete

#### 既有現況

`ticket create --help` 目前關聯參數僅有三個：
- `--parent`（父子）
- `--blocked-by`（阻塞）
- `--related-to`（一般關聯）

缺口：無法在建立時直接設定 source_ticket 欄位。

#### 姊妹 Ticket 現場（W12-002 / W12-005）

- W12-002（ANA，已完成）：實測 `--parent W12-002` 建 .1~.4 時被 CLI 誤設為 children，PM 手動 Edit 5 檔案修復
- W12-005（IMP，已完成）：在 CLI complete 層加 blocking confirmation 處理既有 spawned 非 terminal 場景（防後果）
- W12-006（本 Ticket）：從源頭修 CLI 能力缺口（防前因）

#### 相關檔案（預期修改範圍）

- `.claude/skills/ticket/ticket_system/commands/create.py`（CLI 參數定義與轉發）
- `.claude/skills/ticket/ticket_system/lib/ticket_builder.py`（建立邏輯）
- `.claude/skills/ticket/ticket_system/lib/ticket_loader.py`（讀寫 source ticket 的 spawned_tickets）
- `.claude/skills/ticket/tests/`（CLI 測試 + builder 測試）

#### 驗收條件（自 ticket frontmatter）

1. CLI 支援 --source-ticket 參數
2. 建立時自動雙向關聯（source_ticket 與 spawned_tickets）
3. ticket CLI 測試覆蓋新參數

#### Phase 1 規劃重點（lavender 需回答）

1. CLI 參數語法：`--source-ticket <ID>` 單值？互斥驗證（不可與 --parent 同用）？
2. 雙向關聯的失敗處理：source ticket 不存在時？source ticket 已 completed 時？
3. CLI 輸出：建立成功後是否顯示雙向關聯確認訊息？
4. 引導訊息：PM 用 --parent 於 ANA source ticket 時，是否提示考慮 --source-ticket？（可選，超出本 Ticket 範圍）
