### 修復摘要

對照 commands/create.py 業務錯誤路徑，將 16 處 `print(format_error(...))` 與 1 處 ad-hoc `[ERROR]` + `sys.exit` 全改寫為 `format_error(ErrorEnvelope(...))` 結構化呼叫，輸出統一含 `__error_envelope_v1__` 版本標記與 component/action/errno/hint 四欄位。

### 變更範圍對照

| 站點 | 函式 | errno |
|------|------|-------|
| L105 | `_validate_blocked_by_references` | BLOCKED_BY_NOT_FOUND |
| L119 | `_validate_blocked_by_references` (cycle) | BLOCKED_BY_CYCLE |
| L146 | `_validate_decision_tree_params` | DECISION_TREE_EMPTY_VALUE |
| L199 | `_build_decision_tree_path` | DECISION_TREE_MISSING_ALL |
| L214 | `_build_decision_tree_path` (exempted partial) | EXEMPTED_PARTIAL_PARAMS |
| L223 | `_build_decision_tree_path` (non-exempt partial) | DECISION_TREE_MISSING_PARTIAL |
| L571 | `_resolve_ticket_id_and_wave` | MISSING_WAVE_PARAMETER |
| L579 | `_resolve_ticket_id_and_wave` | INVALID_TICKET_ID_FORMAT |
| L647-649 | `_parse_cli_args_to_config` (PC-018) | WHY_REQUIRED |
| L804 | `_enforce_create_checklist` | CHECKLIST_VALIDATION_FAILED |
| L1028 | `_validate_source_ticket_arg` | SOURCE_PARENT_MUTUALLY_EXCLUSIVE |
| L1033 | `_validate_source_ticket_arg` | INVALID_TICKET_ID_FORMAT |
| L1042 | `_validate_source_ticket_arg` (no version) | SOURCE_TICKET_NOT_FOUND |
| L1049 | `_validate_source_ticket_arg` (load failed) | SOURCE_TICKET_NOT_FOUND |
| L1114 | `execute` | VERSION_NOT_DETECTED |
| L1121 | `execute` (validate_version_registered) | VERSION_NOT_REGISTERED |

共 16 個結構化錯誤封包；ticket 描述「10 處」為下界估計，實際盤點後全數納入。

### 設計決策

- **errno 命名**：採大寫 SNAKE_CASE，沿襲 ErrorEnvelope 範例與 .5.2 文件章節風格
- **action 命名**：用內部函式語意動詞（resolve_version / validate_blocked_by 等），便於 grep 定位呼叫源
- **hint 內容**：保留原 message 中的修復建議與動態插值（如 ticket_id / missing_fields），確保使用者可獲得相同等級的修復指引
- **WHY_REQUIRED 走 stderr**：保留原 `sys.stderr.write` + `sys.exit(1)` 行為，僅改 message 為 envelope 字串
- **舊 CreateMessages / ErrorMessages template constants 不刪除**：messages.py 為共用模組，其他 commands 仍引用；W17-008.5.4 處理 messages.py 整理

### 範圍邊界（避免 scope creep）

- 不改 messages.py（.5.4 範圍）
- 不改其他 commands/*.py
- 不改 hook 補充邏輯（.5.5 範圍）
- 保留 CreateMessages/ErrorMessages 既有 template constants（仍被其他 commands 引用）
