## Solution



### 變更摘要

依 WRAP 完整模式分析（D + A + E 三層落地）執行：

#### A. 新增規則 `.claude/rules/core/decision-trigger-binding.md`

- 4 條核心規則：(1) 兩種合法狀態（已決策 / 綁 ticket trigger 延後）；(2) Trigger 限 ticket ID only；(3) 寫法替換對照表；(4) 違規偵測時機
- 反模式範例：合法 vs 違規對照、長期研究的處理（不設探索性例外）
- 與 PC-093 / quality-baseline / document-writing-style 邊界章節
- 撰寫前檢查清單
- 體量：約 110 行（auto-load context 增量可控）

#### D. 修正 W11-017 case 2 框架

- 原措辭「結論詞 vs 延後詞語意對立分類」保留了「延後詞」合法性，框架錯誤
- 修正為「明確決策 vs 未綁 ticket trigger 延後」三狀態：合法已決策 / 合法綁 trigger 延後 / 違規無 trigger 延後
- Hook 修正方向更新：偵測自由文字含 ticket ID 格式 + frontmatter spawned_tickets/blockedBy 連結，兩者皆無才觸發警告

#### E. Memory feedback + index

- `feedback_no_deferred_decisions.md`：原則 + Why + How to apply
- `MEMORY.md`：新增 index 條目

- Source 段移除具體 ticket 引用，改為原則性陳述

對照 memory `feedback_pm_rule_writing_ticket_id_reflex.md`（PC-098）反射模式，本次 hook 防護有效。
