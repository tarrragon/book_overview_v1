### 落地方案 D（自動 add + stdout 提示）

**修改檔案**：
- `.claude/skills/ticket/ticket_system/commands/lifecycle.py`
  - 新增 `import subprocess`
  - 從 `worklog_appender` 額外 import `_build_worklog_path`，包裝成 `_build_worklog_path_for_stage(version) -> str`（測試替身錨點）
  - 新增 `_auto_stage_git_add(paths)` 薄封裝 `subprocess.run(["git","add",*paths], capture_output=True, check=False)`，作為 patch 錨點避免污染全域 `subprocess.run`（會影響 `get_project_root` 等 git 呼叫）
  - 新增 `_auto_stage_completion_files(ticket_id, modified_paths)`：去重 + degrade gracefully + stdout 提示 `chore(<id>): metadata sync post-completion`
  - `_post_complete_cascade(...)` signature 由 `-> None` 改為 `-> List[Dict[str, Any]]`，回傳 `_cascade_unblock_children` 的 unblocked list 供 caller 收集 children 路徑
  - `complete()` 新增 `no_stage: bool = False` 參數；在 return 0 前收集 modified_paths（ticket_path + worklog_path + cascade children paths）並呼叫 `_auto_stage_completion_files`
  - `execute_complete()` 讀取 `args.no_stage` 並傳入

- `.claude/skills/ticket/ticket_system/commands/track.py`
  - `p_complete` 新增 `--no-stage` flag（store_true, dest=no_stage）

- `.claude/skills/ticket/tests/test_complete_auto_stage.py`（新建）
  - 5 個測試覆蓋：正常 complete / cascade unblock / --no-stage / stdout commit 提示 / 路徑精確性（無 `.`、`-A`、`--all`，全為 `.md`）

### 設計取捨

1. **`_auto_stage_git_add` 包裝層**：直接 patch `lifecycle.subprocess.run` 會污染全域 subprocess（lifecycle import subprocess module 後與 paths.py 共用同一 module 物件），導致 `get_project_root()` 的 `git rev-parse` 也被攔截回傳 bytes，下游 Path() 失敗。包裝薄函式提供精確 patch 邊界。

2. **`_build_worklog_path_for_stage` 包裝層**：同理隔離測試替身；複用既有 `_build_worklog_path(version) -> Path`，轉 str 後給 git add。

3. **`_post_complete_cascade` 改回傳 unblocked list**：原本 `-> None` 無法讓 caller 取得 children 路徑。改為回傳已 unblocked 的 children dict（與 `_cascade_unblock_children` 第一回傳值同型），保持向後相容（caller 可忽略回傳）。

4. **Degrade gracefully**：subprocess 失敗、worklog 路徑解析失敗、child 路徑解析失敗均寫 stderr 警告且不中斷 complete return 0；符合 ticket Edge case 處理要求（non-git directory / git not found / index.lock 競爭 PC-092）。

5. **精確路徑 add（無 `.` / `-A`）**：測試 5 強制驗證，確保不夾帶 WIP。
