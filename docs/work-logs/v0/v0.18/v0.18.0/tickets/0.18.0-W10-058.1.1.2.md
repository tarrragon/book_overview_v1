#### 2. hook 實作 context check

於 `RestrictiveKeywordsStrategy.detect()` 在 keyword match 後加入 context 檢查：

```python
def _check_context_blacklist(self, prompt: str, keyword: str, window: int, blacklist: list) -> bool:
    """檢查觸發詞前後 window 字內是否含 blacklist 詞。回傳 True 表示應 suppress。"""
    idx = prompt.find(keyword)
    if idx < 0:
        return False
    start = max(0, idx - window)
    end = min(len(prompt), idx + len(keyword) + window)
    context_excerpt = prompt[start:end]
    return any(bl in context_excerpt for bl in blacklist)
```

於 detect() match 成功後：

```python
if matched_keyword and config.context_blacklist:
    if self._check_context_blacklist(prompt, matched_keyword, config.context_blacklist.window, config.context_blacklist.words):
        logger.debug("context blacklist matched; suppressing signal")
        return DetectResult(hit=False, should_warn=False, signal_id=sd.id)
```

### 修復摘要

1. **新增 `ContextBlacklist` dataclass**（hook 檔）：含 `window: int = 20` 與 `words: List[str]` 兩欄位，承載 context-aware filter 黑名單配置。
2. **`SignalDef` 新增 `context_blacklist: Optional[ContextBlacklist]` 欄位**：None 表示未啟用 filter（向後相容預設）。
3. **`_parse_signals` 解析 YAML 新欄位**：當 `context_blacklist.words` 為非空 list 時建立 ContextBlacklist 物件；window 解析失敗時 fallback 20。
4. **`RestrictiveKeywordsStrategy.detect` 加入 context check**：keyword match 後若 `sd.context_blacklist` 非 None，呼叫 `_check_context_blacklist(prompt, keyword, window, words, case_sensitive)`；命中時 return DetectResult(hit=False, log_reason="context_blacklist")。
5. **`_check_context_blacklist` 為 staticmethod**：以 keyword 在 prompt 的位置為中心，取 ±window 字 excerpt，檢查 blacklist 任一詞是否出現；大小寫敏感性與 keyword 匹配邏輯一致。
6. **YAML schema 擴充**：restrictive_keywords 新增 `context_blacklist`，window=20，words=[hook, 訊號, 偵測, 事件, 機制, 觸發, fire]。

### 設計決策

- **保守向後相容**：未配置 context_blacklist 時行為與原始一致（test_no_blacklist_config_keeps_original_behavior 驗證）。
- **suppress 採 log_reason 標記**：沿用 S3 `wrap_section_written` 既有 pattern，便於日後觀測誤報率。
- **大小寫敏感性**：blacklist 比對與 keyword 比對採同一 `case_sensitive` 旗標，避免不一致行為。
