### 問題根因（自 W11-037 ANA 結論摘要，2026-05-19 已驗證）

**核心模式**：`ticket` CLI 透過 `uv tool install` 安裝為獨立工具，source（`.claude/skills/ticket/ticket_system/`）與 installed（`/Users/tarragon/.local/share/uv/tools/ticket-system/lib/python3.12/site-packages/ticket_system/`）是**兩份獨立 Python package**。修改 source 後若未執行 `uv tool install --reinstall`，CLI binary 仍使用 stale installed 版本——新增的函式會 **AttributeError**，但 caller 端如 `_auto_stage_completion_files` 透過 hasattr/try-except 包裝時表現為**靜默不觸發**（非 error，更難察覺）。

**證據鏈**：

| 項目 | source lifecycle.py | installed lifecycle.py |
|------|---------------------|----------------------|
| 行數 | 2105 | 2004（差 101 行） |
| `_auto_stage_completion_files` | 有（line 1681） | 無（count=0） |
| 修改時間 | 2026-05-19 00:14 | 2026-05-18 22:44 |

`uv tool install --reinstall` 後驗證 W11-040 complete 正常觸發 auto-stage。

### 影響範圍

- 所有 `uv tool install` 安裝的 skill：`ticket` / `worktree` / `project-init` / `version-release` / `branch-worktree-guardian` / `doc` / `mermaid-ascii`
- 本 session 啟動時 `project-init-env-check-hook` 已報告 `project-init [OUTDATED]` 訊號，證明此 hook 已實作 source vs installed mtime 比對邏輯（可作為設計範本複用）

### 相關 Error Pattern

PC-135（source pytest 通過 vs hook subprocess 失敗）為同源——「source 測試通過 ≠ installed binary 行為一致」。PC-145（修改 ticket_system 源碼後撞 validator 矛盾必先 reinstall）為 PM 已知 workaround，但屬人工 fallback，本 ticket 目標是建立自動偵測機制。

### 用戶決策（2026-05-19）

**推進方向**：方向 A+B（文件防護 + 自動偵測），由 lavender Phase 1 規格設計決定具體權重。

**Hook 機制偏好**：SessionStart hook 比對 mtime（複用 `project-init-env-check-hook` 模式擴展至全部 uv tool install 安裝的 skill）。理由：不打斷工作流、與既有 `[OUTDATED]` 訊號機制一致、覆蓋所有 skill 而非僅 ticket。

**排除選項**：post-commit hook（每次 commit 都跑、負擔重）、complete-time self-check（邏輯耦合度高，每個 CLI 命令都要加自檢）。

#### Hook 介面規格

**Hook Event**：SessionStart（不重複觸發於其他事件以降低成本）

**Input**：標準 SessionStart hook stdin JSON（無需特殊欄位）

**Output**（stdout JSON）：

```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "<格式化訊息>"
  },
  "suppressOutput": false
}
```

**additionalContext 訊息格式**：

| 情境 | 訊息範例 |
|------|---------|
| 全部同步 | `[UV Tool Staleness] 全部 7 個 uv tool skill 已同步` |
| 部分 stale | 多行，每個 stale skill 一行：`[UV Tool Staleness] ticket-system [OUTDATED] → 修復: cd .claude/skills/ticket && uv tool install . --force --reinstall` |
| 部分未安裝 | `[UV Tool Staleness] doc-system [MISSING] → 安裝: cd .claude/skills/doc && uv tool install .` |
| Hook 內部錯誤 | `suppressOutput: true`，錯誤寫入 hook log（規則 4：失敗可見） |

**Exit Code 政策**：

| 情境 | Exit Code |
|------|-----------|
| 全部同步 | 0 |
| 偵測到 stale 或 missing | 0（提示不阻塊） |
| Hook 內部錯誤 / 超時 | 0（降級不阻塊） |

**設計理由**：與 ticket-reinstall-hook 不同，本 hook **不自動 reinstall**。原因：(1) 7 個 skill 同時 reinstall 可能耗時 30 秒以上，嚴重拖累 session 啟動；(2) reinstall 失敗時自動修復會造成靜默退化；(3) 用戶決策已偏好「不打斷工作流的提示」。ticket-reinstall-hook 保留現狀（單 skill 自動修復），本 hook 作為其他 6 skill 的提示層補充。

#### Hook 在 SessionStart 鏈中的位置

註冊於 `.claude/settings.json` SessionStart 鏈，**插入位置**：`project-init-env-check-hook.py` 之後、`version-consistency-guard-hook.py` 之前。

理由：邏輯上接續 project-init 環境檢查（兩者都檢查 uv tool 安裝健康度）；不與 ticket-reinstall-hook 重複（後者只處理 ticket 一個 skill 且自動修復，本 hook 處理另外 6 個 + 提示模式覆蓋 ticket 作雙重保險）。

#### 偵測演算法（細節）

```
for skill in SKILLS:
    source_dir = repo_root / skill.source_path
    installed_dir = find_installed_module_dir(skill.package_name)
    if installed_dir is None:
        report MISSING
        continue
    source_hashes = compute_file_hashes(source_dir)   # {rel_path: sha256}
    installed_hashes = compute_file_hashes(installed_dir)
    if source_hashes != installed_hashes:
        report OUTDATED (含 source_path 與修復指令)
    else:
        accumulator.ok_count += 1
```

**Hash 比對範圍**：所有 `.py` 檔（遞迴），排除 `__pycache__/`、`.pyc`、`tests/`（測試碼通常不影響 installed 行為，且測試檔可能差異大）。

**安裝位置定位**：使用 `find_ticket_module_path()` 同樣的 `which <cli>` → readlink → site-packages 推導模式，抽至共用函式 `find_installed_module_dir(cli_name, package_dir_name)`。

#### 邊界條件

| 條件 | 預期行為 |
|------|---------|
| `which <cli>` 失敗 | 視為 MISSING |

#### 3. SKILLS 常數設計

**結論**：採 **module-level frozen list of dataclass**，放於 hook 檔案頂部（非 lib，因為清單是本 hook 特定 policy）。

```
@dataclass(frozen=True)
class SkillEntry:
    source_subpath: str        # ".claude/skills/ticket"
    module_subpath: str        # ".claude/skills/ticket/ticket_system"（hash 計算來源）
    package_name: str          # "ticket-system"（uv tool 安裝名）
    package_dir_name: str      # "ticket_system"（site-packages 子目錄）
    cli_name: str              # "ticket"

SKILLS: tuple[SkillEntry, ...] = (
    SkillEntry(".claude/skills/ticket", ".claude/skills/ticket/ticket_system",
               "ticket-system", "ticket_system", "ticket"),
    SkillEntry(".claude/skills/doc", ".claude/skills/doc/doc_system",
               "doc-system", "doc_system", "doc"),
    # ... 共 7 entry
)
```

**Why dataclass(frozen) over dict**：T02 測試斷言 `hasattr(entry, "package_name")` 比 dict key 檢查更明確；frozen 防止 runtime 篡改；型別友善。
**Why tuple not list**：immutable 語意符合「常數」；len(SKILLS) == 7 斷言可讀性高。
**放置位置**：hook 檔案頂部（緊接 import 之後），不放 lib——因 SKILL 清單是本 hook policy，未來增減 skill 屬本 hook 變更，與通用工具職責分離。

#### 4. 主流程虛擬碼

```
main():
    logger = setup_hook_logging("uv-tool-staleness-check-hook")
    try:
        project_root = get_project_root()
        results = []  # list of (skill_entry, status, detail)
                       # status ∈ {"OK", "OUTDATED", "MISSING", "ERROR"}

        for skill in SKILLS:
            source_dir = project_root / skill.module_subpath
            if not source_dir.exists():
                # source 不存在：repo 結構異常，跳過該 skill
                logger.warning(f"source missing: {skill.module_subpath}")
                results.append((skill, "ERROR", "source dir missing"))
                continue

            installed_dir = find_installed_module_dir(
                skill.cli_name, skill.package_dir_name, logger
            )
            if installed_dir is None or not has_any_py(installed_dir):
                results.append((skill, "MISSING", None))
                continue

            try:
                src_hashes = compute_file_hashes(source_dir, EXCLUDE_DIRS)
                inst_hashes = compute_file_hashes(installed_dir, EXCLUDE_DIRS)
            except Exception as e:
                # 規則 4：失敗可見
                logger.error(f"hash computation failed for {skill.cli_name}: {e}")
                results.append((skill, "ERROR", str(e)))
                continue

            is_identical, diff = compare_hash_sets(src_hashes, inst_hashes)
            if is_identical:
                results.append((skill, "OK", None))
            else:
                results.append((skill, "OUTDATED", diff))

        message = format_results(results)
        print(json.dumps({
            "hookSpecificOutput": {
                "hookEventName": "SessionStart",
                "additionalContext": message
            },
            "suppressOutput": False
        }, ensure_ascii=False, indent=2))
        return 0

    except Exception as e:
        logger.critical(f"hook failed: {e}", exc_info=True)
        print(json.dumps({"suppressOutput": True}, ensure_ascii=False, indent=2))
        return 0  # 永不阻塊
```

#### 5. format_results 訊息組裝

```
format_results(results):
    ok = [r for r in results if r.status == "OK"]
    outdated = [r for r in results if r.status == "OUTDATED"]
    missing = [r for r in results if r.status == "MISSING"]
    errors = [r for r in results if r.status == "ERROR"]

    # 全部同步：簡潔訊息（AC5）
    if len(ok) == len(SKILLS):
        return "[UV Tool Staleness] 全部 7 個 uv tool skill 已同步"

    lines = []
    for r in outdated:
        lines.append(
            f"[UV Tool Staleness] {r.skill.package_name} [OUTDATED] → "
            f"修復: cd {r.skill.source_subpath} && uv tool install . --force --reinstall"
        )
    for r in missing:
        lines.append(
            f"[UV Tool Staleness] {r.skill.package_name} [MISSING] → "
            f"安裝: cd {r.skill.source_subpath} && uv tool install ."
        )
    for r in errors:
        lines.append(f"[UV Tool Staleness] {r.skill.package_name} [ERROR] {r.detail}")
    if ok:
        lines.append(f"[UV Tool Staleness] 其他 {len(ok)} skill 已同步")
    return "\n".join(lines)
```
