### 修改摘要

| 檔案 | 變更 |
|------|------|
| `.claude/skills/ticket/ticket_system/lib/handoff_utils.py` | 新增 `is_ticket_terminal(ticket_id, project_root)`：lib SSOT 的 terminal 狀態判定（completed + closed），共用既有 `_load_ticket_status` 路徑解析。引用 `TERMINAL_STATUSES` 常數（從 `ticket_system.constants` 來）。 |
| `.claude/hooks/handoff-auto-resume-stop-hook.py` | (1) 新增 `from handoff_utils import is_ticket_terminal as _lib_is_ticket_terminal`；(2) `is_ticket_completed` 函式體改 thin wrapper delegate 至 `_lib_is_ticket_terminal`，刪除自定義的 `find_ticket_file + parse_ticket_frontmatter` 邏輯（L361 區段）；(3) 移除未使用的本地 `TERMINAL_STATUSES = {"completed", "closed"}` 常數，留說明註解指向 lib SSOT。 |
| `.claude/hooks/handoff-prompt-reminder-hook.py` | (1) 新增 sys.path bootstrap + `from handoff_utils import is_ticket_completed as _lib_is_ticket_completed`；(2) `is_ticket_completed` 函式體改 thin wrapper delegate 至 `_lib_is_ticket_completed`，刪除自定義的 `find_ticket_file + parse_ticket_frontmatter` 邏輯（L89 區段）。 |

### 設計決策

| 決策 | 理由 |
|------|------|
| 保留 hook 端 thin wrapper（簽名 `(project_root, ticket_id, logger)`） | 維持既有呼叫端相容，不影響 stop hook L563 / prompt-reminder L227 等行 |
| 新增 `is_ticket_terminal` 而非改 `is_ticket_completed` 語義 | lib `is_ticket_completed` 仍被 `resume.py` / `track_runqueue.py` / `handoff_gc.py` 等 CLI 場景使用，這些場景僅關心 `completed` 嚴格語義；stop hook 的 terminal 判定（含 `closed`）需獨立函式以避免改動 CLI 行為 |
| import path bootstrap 沿用 stop hook 既有模式（手動 sys.path.insert + try/except 防禦） | 與 W17-181.1 / W17-095.2 已落地的模式一致 |

### 不在本 ticket 範圍

- lib `is_ticket_completed` 的 CLI 呼叫端是否應改用 `is_ticket_terminal`：屬於 ARCH-020 後續觀測題目，不在本 ticket 範圍。
