### 修改摘要

1. **新增 `find_target_repo(file_path)` minimal helper**（W17-148 完成後可遷移至 git_utils.py）
   - 從檔案路徑往上找最近的 `.git` 目錄；非絕對路徑直接回傳 None。

2. **`is_exempt_path_on_protected_branch` 新增 `target_repo` 參數**
   - same repo（target_repo == host_root 或 None）：維持本專案豁免清單（`.claude/`、`docs/`、`scripts/experiments/`、`CLAUDE.md`、`README.md` 等）
   - cross repo：退化為通用清單 `GENERIC_EXEMPT_EXACT = [README.md, CHANGELOG.md, .gitignore, .gitattributes]`

3. **新增 `build_cross_repo_deny_message`**
   - 訊息含目標 repo 路徑、目標 branch、可複製貼上的 `cd <repo>` + `git status` + `git checkout -b <branch>` 指令。

4. **`main()` 整合**：偵測 `is_cross_repo` 後分流至跨專案豁免邏輯與專屬 deny 訊息。

### 測試

新增 `.claude/hooks/tests/test_branch_verify_cross_repo.py`（16 測試）：
- find_target_repo（nested file / relative path / no .git ancestor）
- same-repo 豁免（.claude/、docs/、非豁免檔）
- cross-repo 豁免（README.md/CHANGELOG.md/.gitignore/.gitattributes 放行；.claude/、docs/、任意檔案不放行）
- deny 訊息含目標 repo 路徑、branch、cd + checkout 指令、自訂 suggested_branch
- GENERIC_EXEMPT_EXACT 完整性（含通用文件、不含本專案約定）

### 設計決策

- **未自動建分支**（沿用 ANA W17-147 結論）：PreToolUse 純決策慣例 + race condition + 未提交變更風險 + 用戶授權邊界不明，改為提供準備好的 `git checkout -b` 指令。
- **W17-148 並行考量**：W17-148 預計新增 `find_target_repo` 至 `git_utils.py`，本 ticket 先以 hook 內 minimal helper 解耦；W17-148 merge 後可重構合併。

### 不在本 ticket 範圍

- main-thread-edit-restriction-hook 的 cross-repo 豁免（W17-148 處理）
- git_utils.py 集中化 helper（W17-148 後續重構）

- 0.18.0-W17-147 why: 從 session 案例觸發：在 book_overview_v1 主線程嘗試編輯 /Users/mac-eric/project/blog/content/skills/compositional-writing/skill.md，被兩個 hook 同時阻擋：

1. main-thread-edit-restriction-hook.py：以本專案 .claude/、docs/ 等白名單判斷，目標路徑因不在「本專案」白名單而被誤擋。但目標路徑根本不在本專案內，這個檢查應該不適用。

2. branch-verify-hook.py：以本專案 cwd 的 git branch 判斷保護分支（main），目標檔案在另一個 repo（blog），用本專案 branch 判斷無意義。

正確設計應該是：
- 識別目標檔案所屬 repo（往上找 .git 目錄）
- 用該 repo 的白名單與 branch 狀態判斷
- 若該 repo 在保護分支，自動為該 repo 建 feature branch 後再放行
- 路徑允許清單按目標 repo 的設定判斷（每個專案可有自己的 .claude/ 規則）
