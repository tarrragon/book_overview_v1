### 4 擴充點落地

1. **SessionStart marker 寫入**：新增 `write_session_marker(project_root, now=None) -> Path`；早於既有檢查執行；marker 寫入失敗時 stderr WARN 但不阻擋 main 回 0。
2. **觸發頻率掃描（全部 hooks scope）**：新增 `run_frequency_scan(project_root, logger, now=None, logs_root=None)`；呼叫 W13-016 `hook_health.scan_logs(since=7d)` 取 per_day → `classify_hook` → `evaluate`；scope 不限於 SessionStart hooks，掃 `hook-logs/` 下全部子目錄。
3. **stderr 摘要（不自動建 ticket）**：格式 `[hook-health] LVL: <hook> 今日觸發 N 次 > 基線 B × n`（符合 W13-008 Solution §自動建 Ticket 流程範例）；bootstrap 路徑單列；尾段附「建議手動建 ANA」提示。critical / warning 才輸出；normal 沉默。
4. **既有 SessionStart 健康檢查零回歸**：原 `check_hook_logs` + `_output_health_report` + 失敗 hook stderr 廣播邏輯保留；frequency scan 包在 try/except，例外時 stderr WARN 不影響既有流程。

### 設計決策

- **baseline = sum(per_day) / 7**（固定除數）：W13-008 量化標準；不足 7 天時不膨脹基線（保守，under-trigger 優於 over-trigger）。
- **recent = today_str 在 per_day 的計數**：與 W13-008 §觸發判定一致。
- **classification 用 W13-016 lib**：避免重複定義 high_freq_ok 名單，DRY。
- **marker 與 frequency scan 各自 try/except**：監測本身禁止讓既有健康檢查降級（observability paradox 防護）。
