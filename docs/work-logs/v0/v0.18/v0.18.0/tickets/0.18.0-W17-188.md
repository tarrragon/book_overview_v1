### 修法摘要

對 5 hook 一律以 `hook_utils` 共用 helper 取代硬編碼路徑組裝：

| Hook | 修改函式 | 引入 helper | 變更性質 |
|------|---------|------------|---------|
| commit-handoff-hook.py | `scan_wave_tickets` | `scan_ticket_files_by_version` | 函式內部實作改用 helper 取得檔案清單 |
| tech-debt-reminder.py | `find_tickets_directory` → `find_ticket_files_for_version` | `scan_ticket_files_by_version` | 改回傳 `List[Path]`；同步調整 `scan_tech_debt_tickets` 簽名（接受 `List[Path]` 取代 `Path` dir）；更新 main 流程呼叫；module docstring 補述雙結構 |
| post-git-commit-hook.py | `_scan_wave_tickets` | `scan_ticket_files_by_version` | 函式內部實作改用 helper |
| prompt-submit-hook.py | `resolve_ticket_path` | `find_ticket_file` | 函式內部改委派至 helper；同步更新呼叫處檢查語義（`is None` 取代 `not exists()`） |
| file-ownership-guard-hook.py | `get_active_tickets` | `scan_ticket_files_by_version` | 函式內部實作改用 helper |

### 設計決策

1. **共用 helper 已存在，不新建**：W17-180 ANA 已確認 `hook_utils/hook_ticket.py` 的 `scan_ticket_files_by_version` 與 `find_ticket_file` 都支援雙結構（flat + hierarchical），且 5 hook 已 import `hook_utils`，零依賴新增。

2. **`tech-debt-reminder` 函式簽名變更**：原 `find_tickets_directory` 回傳 `Optional[Path]` 但下游 `scan_tech_debt_tickets` 用 `dir.glob()`；改用 helper 後直接拿 `List[Path]` 更自然，避免「先拿 dir 再 glob」的多餘抽象。函式重命名為 `find_ticket_files_for_version` 反映新語義。

3. **`prompt-submit-hook` 存在性檢查語義同步**：`find_ticket_file` 既已驗證存在性，原 `if ticket_path and not ticket_path.exists(): continue` 改為 `if ticket_path is None: continue`，語義對齊。

### 不在本 ticket 範圍

- `tech-debt-reminder.py` 內 `extract_frontmatter` 函式未 import `yaml` 即呼叫 `yaml.safe_load` 屬另一既存 bug（被 try/except 吞掉），屬不同根因，不在本 IMP 修復範圍。後續需另開 ticket 追蹤。
