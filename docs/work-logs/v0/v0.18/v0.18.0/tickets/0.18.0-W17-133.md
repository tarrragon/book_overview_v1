### 問題根因

hook-completeness-check.py 在 SessionStart 偵測 `.py` 檔缺少 exec bit 時會自動 chmod +x（IMP-054），但修正後產生的 mode-only 變更殘留在工作樹，跨 session 容易讓用戶誤以為有未提交變更（PC-076 同類）。

### 影響範圍

- `.claude/hooks/hook-completeness-check.py`：新增 `_attempt_auto_commit` helper 與其呼叫點
- `.claude/hooks/tests/test_hook_completeness_check_autocommit.py`：新增三情境測試

### 相關 Error Pattern

PC-076（cross-session uncommitted legacy）為動機；本 ticket 修補的是「框架自動修改後不留尾」的特定路徑。

### 修復摘要

於 `hook-completeness-check.py` 的 `_check_and_fix_permissions` 後追加 `_attempt_auto_commit(fixed_files, hooks_dir, project_root, logger)`：

1. 計算被 chmod 的檔案相對 repo 路徑集合 `fixed_rel_set`
2. `git status --porcelain` 取得當前工作樹變更集合 `changed_paths`；若 `changed_paths != fixed_rel_set` → print 警告並跳過
3. `git diff -- <fixed files>` 內容若含 `@@` hunk → 非 mode-only → 跳過
4. 條件全部成立 → `git add` + `git commit -m "chore: auto-fix executable permissions for hook files (IMP-054)"`
5. 任一 git 步驟失敗皆 print 到 stderr + logger.warning，不靜默
6. 整段以 `try/except Exception` 包覆，保證 hook 不會讓 session 啟動失敗
7. main 分支沒有額外限制，符合驗收條件

成功時 print「可用 git revert HEAD 撤銷」提示。

### 設計決策

- `git diff` 偵測 mode-only 採用「不含 `@@` hunk 標記」啟發式：mode-only 變更只會輸出 `old mode` / `new mode` header，不會有 hunk
- commit message 不附 Co-Authored-By（依 ticket 要求）
- pre-commit hook 失敗以 returncode != 0 路徑處理，不 crash

### 不在本 ticket 範圍

- PC-076 全面治理（涵蓋其他自動修改源）；本 ticket 只處理 hook-completeness-check 觸發的 chmod 案例
