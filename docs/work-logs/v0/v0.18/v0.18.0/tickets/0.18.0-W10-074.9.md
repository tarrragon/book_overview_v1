### methodology-writing-methodology.md 保留/刪除評估

**核心能力盤點**（該檔兩大章節 125 行）：

1. **章節一「方法論撰寫」**（9-35 行，共 27 行）：
   - 讀者定位（專家找回記憶，非教導）
   - 30 秒電梯理論
   - SKILL vs 方法論職責分工
   - 撰寫/改寫檢查清單
2. **章節二「經驗分享文章撰寫」**（38-113 行，共 76 行）：
   - 六項準則（語氣、陳述事實、功能描述、敘事角度、完整脈絡、去產品名稱）
   - 經驗分享文章檢查清單

**與新 compositional-writing Skill 的職責重疊度分析**：

| 內容主題 | 新 Skill 規劃涵蓋？ | 重疊度 |
|---------|-------------------|--------|
| 讀者定位（專家找回記憶） | 主題 2（索引建立）+ 主題 3（不隱藏意圖） | 部分重疊 |
| 30 秒電梯理論 | 主題 1（原子化原則） | 高度重疊 |
| SKILL vs 方法論分工 | 不在六大主題範圍 | **不重疊**（跨框架管理層的分類原則） |
| 撰寫/改寫檢查清單 | 不在六大主題範圍 | **不重疊**（方法論元層管理） |
| 經驗分享六準則（語氣、去產品名） | 主題 3（不隱藏意圖）+ 主題 4（商業邏輯陳述） | 部分重疊 |
| 經驗分享敘事脈絡（發現→找因→修復） | 不在六大主題範圍 | **不重疊**（是 retrospective 體裁特性） |

**保留/刪除建議**：**部分保留**（轉型為「框架元層」方法論，保留不重疊部分）

**理由**：

1. **重疊的內容可安心遷移**：30 秒電梯理論、六項準則中關於「語氣」「功能描述」「去特定產品名」的部分，與新 Skill 主題 1/3/4 高度對齊，應搬遷到對應 references
2. **不重疊的內容必須保留**：
   - SKILL vs 方法論分工原則（是**框架元層的職責分類**，決定何時建 SKILL、何時建 methodology，不是「寫作本身」的方法）
   - 方法論撰寫/改寫檢查清單（是**方法論元層管理工具**）
   - 經驗分享敘事脈絡（**retrospective 體裁特性**，新 Skill 焦點是 AI 可查詢性和認知負擔，不涵蓋敘事結構）
3. **新 Skill 定位是「可獨立發佈到 marketplace」，若把「方法論管理層原則」併入，會違反 marketplace 可攜性**（外部使用者不需要本框架的 SKILL/methodology 分類制度）
4. **具體建議**：
   - **遷移至新 Skill**：30 秒電梯理論 → 原子化原則 reference；六項準則中語氣/功能描述/去產品名 → 不隱藏意圖 / 商業邏輯陳述 references
   - **原檔保留並改名**：剩餘內容（SKILL vs 方法論分工、撰寫檢查清單、敘事脈絡準則）重組為更精確名稱，如 `framework-meta-methodology.md`（框架元層管理方法論）或 `retrospective-writing-methodology.md`（經驗分享敘事體裁），或直接併入既有 `methodology-rewriting-methodology.md`
   - **若選擇全刪**：必須先將「SKILL vs 方法論分工原則」升級至 `.claude/rules/` 或合併到 `methodology-rewriting-methodology.md`，避免框架元層管理原則消失

**關鍵警告**：三個目標方法論中，`methodology-writing-methodology.md` 是**唯一不能直接全刪**的，其職責與「寫作方法論 Skill」不是同層概念——它是「管理框架中其他方法論如何撰寫」的元層文件，屬於框架治理而非寫作本身。決策時必須明確區分。
