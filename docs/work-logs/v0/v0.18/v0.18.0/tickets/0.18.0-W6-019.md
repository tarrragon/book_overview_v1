### jest-chrome 版本狀況調查

jest-chrome npm registry 最新版本為 0.8.0（最終版，無更新計畫），其 peerDependencies 約束 jest@26||27，但本專案使用 jest@29。

peerDependencies: { "jest": "^26.0.1 || ^27.0.0" }

npm install 無 --legacy-peer-deps flag 時報錯：Conflicting peer dependency: jest@27.5.1

### jest-chrome 使用點盤查

使用範圍（grep jest-chrome tests/ src/）：
- tests/test-setup.js（全域 setup：require jest-chrome + global.chrome 賦值）
- tests/unit/content/modular/content-modular.test.js
- tests/unit/storage/chrome-storage-tag-operations.test.js
- tests/integration/chrome-extension/background-event-system.test.js
- tests/integration/chrome-extension/content-script-extractor.test.js
- tests/integration/chrome-extension/data-flow-end-to-end.test.js
- tests/utils/chrome-extension-mocks-enhanced.js
- tests/e2e/validation/simple-validation.test.js（requiredDevDeps 清單驗證）
- src/: 無 jest-chrome 直接引用

### 關鍵發現

jest-chrome 本質是靜態 Chrome API 定義物件加 jest mock 配置，lib/index.cjs.js 為純 CommonJS bundle。不直接呼叫任何 jest 版本限制的 API，peerDep 衝突是「宣告層」問題，非「運行層」問題。現有測試在 jest@29 環境已正常執行（unit: 4098 pass；integration: 629 pass）。

### 方案評估

- 升級 jest-chrome：無更新版本（最新 0.8.0，2022 年停更），不可行
- 替換 jest-mock-extended：jest-chrome 提供完整 chrome API schema，替換成本高且需重寫所有 mock 使用點
- 自訂 chrome API mock：需自行維護大量 Chrome API 定義，長期維護成本高
- npm overrides（選用）：宣告 jest-chrome 的 jest peerDep 由頂層 jest 版本滿足，最小改動，無測試風險

### 選用方案：npm overrides

在 package.json 頂層加入 overrides 欄位：

```json
"overrides": {
  "jest-chrome": {
    "jest": "$jest"
  }
}
```

`$jest` 語法表示引用頂層 devDependencies 的 jest 版本（^29.7.0），讓 npm 以頂層版本滿足 jest-chrome 的 peerDep 約束，無需修改任何測試程式碼或 mock 邏輯。

### 選擇理由

- 最小改動原則：只改 package.json，不動測試程式碼
- 零測試風險：jest-chrome 在 jest@29 環境已正常運作（peerDep 是宣告問題非運行問題）
- 可維護性：overrides 欄位明示「jest-chrome peerDep 衝突已知且已處理」，後人可直接追蹤原因
- 替換方案成本過高：替換 jest-chrome 需重寫 8 個測試檔案的 mock 邏輯，維護自訂 Chrome API schema，風險遠大於 overrides

### 驗證結果

npm install（純粹，無 --legacy-peer-deps）：成功，無 peer dep 衝突警告。

### Phase 4 評估

- 重構需要：否
- 技術債：jest-chrome 長期停滯（2022 年停更），若未來需要新的 Chrome API mock，可評估替換為 webextension-polyfill 或自訂 mock factory。已存在的 chrome-mock-factory.js 已具備擴展基礎。
- 程式碼品質：A+（單一欄位修改）
