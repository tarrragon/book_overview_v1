## 實作摘要

依 W17-016.2 schema（.claude/pm-rules/ticket-body-schema.md）擴充 complete hook 從 soft-check 升級為 type-aware hard-block + 逃生閥。

### 變更點

1. ticket_validator.py::_is_placeholder：新增中文佔位符「（待填寫：...）」/「（必填：...）」偵測（template 預設內容）；並加「整段只由中文佔位符組成」判定，避免 template 區段整體被誤判為已填寫。

2. 新增 validate_execution_log_by_type(ticket_type, body)：對照 schema _TYPE_REQUIRED_SECTIONS：
   - ANA: Problem Analysis + Solution 必填
   - IMP: Test Results 必填
   - DOC: 無強制 body 章節
   - 未知 type（TST/ADJ/RES/INV）：回退通用 validate_execution_log

3. lifecycle.py::TicketLifecycle.complete Step 3.5：
   - 新增 skip_body_check 參數（逃生閥）
   - 未啟用逃生閥時，呼叫 validate_execution_log_by_type 並於未填寫時 return 1（hard block）
   - 錯誤訊息指引：列出未填寫章節、append-log 指令範本、逃生閥用法
   - 啟用逃生閥時仍跑原 soft check 印警告（可見但不阻擋）

4. track.py + execute_complete：新增 --skip-body-check CLI 旗標，dest skip_body_check，沿用 getattr 傳入 lifecycle。

### 與既有機制的關係

- 原 validate_execution_log（通用三章節 soft check）保留，新版於 type 未定義時 fallback 使用，不破壞向後相容。
- W17-016.4 的 sync_completion_body_fields 於 hard-block 通過後才執行，與本 ticket 無衝突。
