### Bug 1 修復：id_parser.calculate_chain_info 保留前導 0

檔案：\`.claude/skills/ticket/ticket_system/lib/id_parser.py\`（L244-268）

改動：
- 原做法：使用 parse_sequence 將序號字串 int 化，再 format_sequence 組回字串，導致前導 0 遺失
- 新做法：從 extract_id_components 取得原始序號字串，直接 split(".") 並以原字串片段組回 root_id/parent_id；sequence_list 仍維持 int list 供 API 相容

影響：
- W10-036.1 的 parent 正確為 "0.18.0-W10-036"（原 "0.18.0-W10-36"）
- 所有前導 0 序號（001、007、036 等）都受此修復保護
- 單位數根 ID（如 W3-001）原本也有 typo，一併修復

### Bug 2 修復：_update_cross_references 精確匹配 new_id

檔案：\`.claude/skills/ticket/ticket_system/commands/migrate.py\`（L104-114）

改動：
- 原做法：\`ticket_file.stem.startswith(new_id)\` 誤跳過所有以 new_id 為前綴的檔案（含子任務）
- 新做法：使用 extract_core_ticket_id 取得核心 ID，精確比較 \`core_id == new_id\`
- 新增 import：extract_core_ticket_id

影響：
- 子任務 file stem 以 new_id 為前綴時（如 migrate .4 → .2，且存在 .2.1 子任務），其引用也會被正確更新
- 帶描述後綴的檔名（如 \`{id}-description.md\`）也能被 extract_core_ticket_id 正確識別

- \`tests/test_migrate_cross_references.py\`：5 個測試，覆蓋 blockedBy/relatedTo/children/parent_id 更新 + 子任務邊界情境
