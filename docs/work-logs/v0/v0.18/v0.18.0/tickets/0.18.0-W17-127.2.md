### 設計決策

1. cache 設計：file-based per-session（`.claude/hook-logs/skill-trigger-cache-{session_id}.json`），session_id 嚴格隔離跨 session；in-memory 不可行（hook 每次新 process）
2. transcript 掃描：純字串預過濾 + 行級 JSON parse，禁 LLM（對齊 W17-122 ginger 警示「hook 用純字串 match」）
3. lifecycle.py 整合：lazy import + sys.path 注入，避免 ticket_system 純單元測試環境強依賴 hooks/lib；fallback 路徑覆蓋 W17-125 既有 6 個前綴
4. 訊息語氣（acceptance #4 + #8 重點）：WARN_MESSAGE 與 STRICT_MESSAGE 皆排除「禁止 / 必須 / 不可」絕對主義詞，改用「建議」「成本較高的捷徑」「豁免條件」機會成本語氣，並由單元測試強制保護
5. 失敗策略：所有 I/O / YAML / transcript 解析失敗皆走放行路徑（exit 0）
