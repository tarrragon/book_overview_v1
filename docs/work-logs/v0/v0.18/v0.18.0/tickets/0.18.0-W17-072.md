## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W17-070 ANA 雙根因分析之根因 B：agent 行為層缺乏「禁止自定義 H2」明示規則。配合 W17-071 已修的 validator 漏判分隔符（根因 A），本 IMP 堵住源頭，避免 agent 再寫 `## 實作摘要` / `## 驗證指令與結果` 等自定義 H2 把 Schema section 切斷。

- **源頭**：`.claude/rules/core/agent-definition-standard.md` v1.1.0 僅規範「必填章節」但未明示「禁止自定義 H2」，agent 按業界 TDD 慣例自發寫摘要/驗證章節視為「合規產出」。
- **觸發鏈**：agent 自定義 H2 → validator section 擷取被截斷 → Schema section 剩 schema note + `---` → W17-071 前被誤判為已填寫 → complete 放行空殼。
- **防護分層**：根因 A（W17-071）堵住 validator 漏擋；根因 B（本 IMP）堵住 agent 行為源頭；兩者組成 defense-in-depth。

### 影響範圍

- `.claude/rules/core/agent-definition-standard.md` — 新增「章節結構規則（W17-072）」條款 + 「禁止行為」表新增 row
- `.claude/hooks/acceptance_checkers/custom_h2_checker.py` — 新檔，偵測非 Schema H2
- `.claude/hooks/acceptance_checkers/__init__.py` — export 新 checker
- `.claude/hooks/acceptance-gate-hook.py` — 整合 checker，新增 warning 訊息 + 清單項目 7
- `.claude/error-patterns/process-compliance/PC-110-body-check-false-negative-via-schema-separator.md` — 防護章節補完 + 狀態升為 Protected
- `.claude/hooks/tests/test_custom_h2_checker.py` — 13 單元測試

### 修復摘要（對照 4 條 AC）

四處檔案同步修改，組成根因 B 防護：

#### AC1：agent-definition-standard.md 禁止自定義 H2 條款

`.claude/rules/core/agent-definition-standard.md` v1.2.0 修改：

- 「禁止行為」表新增一列：`寫自定義 H2（## 實作摘要 / ## 驗證指令與結果 等非 Schema 章節）` + 原因說明
- 新增「章節結構規則（W17-072）」章節，採 **Why / Consequence / Action** 三明示結構，含允許/禁止對照表與 Schema 章節清單
- 「違規偵測」表重構為兩列：必填 Schema 章節缺失（阻擋） vs 自定義 H2 章節（警告）

#### AC2：custom_h2_checker.py 偵測 hook + 整合到 acceptance-gate

新增 `.claude/hooks/acceptance_checkers/custom_h2_checker.py`：

- `_strip_frontmatter(content)`：剝除 YAML frontmatter 避免 `---` 誤判
- `find_custom_h2_sections(content)`：掃描 body H2，回傳非 Schema 章節名清單
- `check_custom_h2_sections(content, logger)`：wrapper 加 logger 輸出
- `_SCHEMA_SECTION_NAMES` 與 `execution_log_checker` / `ticket_validator` 同步（9 個章節）

整合 `.claude/hooks/acceptance-gate-hook.py`：

- `AcceptanceCheckResult` 新增 `custom_h2_sections: List[str] = []` 欄位
- `check_acceptance_status` 步驟 7 呼叫 checker
- 清單新增項目 7：`[x] body 僅使用 Schema 章節` 或 `[WARNING] 偵測到 N 個非 Schema H2 章節`
- `generate_hook_output` 在優先級 2.6 附加 `CUSTOM_H2_WARNING` 訊息（不阻擋）

#### AC3：PC-110 防護章節補完

`.claude/error-patterns/process-compliance/PC-110-body-check-false-negative-via-schema-separator.md` 修改：

- 狀態：`Active (防護設計待 W17-070 ANA 完成後補完)` → `Protected (雙根因防護已落地 — W17-071 症狀修復 + W17-072 根因防護)`
- 「防護」章節重寫為雙根因防護落地說明（W17-071 根因 A / W17-072 根因 B）
- 「相關事件」表新增 W17-071 / W17-072 / W17-074 三列
- Footer 升版 0.1.0 → 1.0.0

#### AC4：Hook 實測

Integration test 以 `W17-056` ticket（含 8 個自定義 H2）驗證：Hook 正確輸出 `[WARNING] 7. 偵測到 8 個非 Schema H2 章節` + 詳細警告訊息列出每個違規章節。

### 設計決策

| 決策點 | 選項 | 採用理由 |
|--------|------|---------|
| Warning vs Block | Warning（不阻擋） | 根因 A（W17-071）已堵住空殼 complete 漏擋路徑，本 IMP 的 H2 違規已無共振風險；warning 是對 PM/agent 的可見訊號（quality-baseline 規則 6 失敗案例學習），留彈性給 ANA/DOC 的特殊結構 |
| Schema 章節清單位置 | 獨立 `_SCHEMA_SECTION_NAMES` in checker | 與 `execution_log_checker._SCHEMA_SECTION_NAMES` 對齊；三處同步定義（validator + hook + checker）列入 ARCH-020 後續 refactor，本 IMP 不擴大範圍 |
| Checker 位置 | `acceptance_checkers/` 子目錄（與 execution_log_checker 同源） | 與既有 checker 體系一致；`acceptance-gate-hook.py` 作為 orchestrator 協調所有 checker |
| Frontmatter 剝除策略 | 純文字匹配（`---` 開頭 + 尋找第二個 `---`） | 避免依賴 PyYAML（hook 為 uv script `dependencies = []`）；符合既有 `parse_ticket_frontmatter` 慣例 |

### 不在本 IMP 範圍

- **W17-074 範圍**：validator 內部 substring matching bug（W17-071 發現但不在該 AC 範圍）
- **ARCH-020 範圍**：`_SCHEMA_SECTION_NAMES` 三處同步定義的 DRY refactor
