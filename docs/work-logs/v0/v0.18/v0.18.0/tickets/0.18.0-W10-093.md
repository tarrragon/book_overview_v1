### 修復摘要

依 W10-092 範本執行 worktree skill 專用 hook 遷移：

1. **git mv 7 個 hook**：保留 blame 歷史
   - 6 個 `worktree-*-hook.py` + 1 個 `session-start-merged-worktree-audit-hook.py`
   - 目的地：`.claude/skills/worktree/hooks/`

2. **路徑常數更新（8 處）**：
   - 6 個 hook 的 `sys.path.insert(0, str(Path(__file__).parent))` → `parents[3] / "hooks"`
   - 1 個 hook（worktree-branch-check）額外更新 lib 路徑 → `parents[3] / "hooks" / "lib"`

3. **settings.json 更新（7 處）**：所有 `.claude/hooks/worktree-*` 與 `session-start-merged-worktree-*` entry 改指向新路徑；settings.local.json 無 path entry，不需更新。

4. **測試檔 HOOK_PATH 更新（3 處）**：
   - test_session_start_merged_worktree_audit_hook.py
   - test_worktree_merge_reminder_hook.py
   - test_worktree_zombie_cleanup_hook.py
   - 統一改為 `Path(__file__).resolve().parents[2] / "skills" / "worktree" / "hooks" / <name>`

- **ANA 估算偏差**：W10-016 估 6 個（`worktree-*` glob），實測有 `session-start-merged-worktree-audit-hook.py` 不符 glob 但屬 worktree skill。依 PC-007 獨立驗證後一併遷移。
