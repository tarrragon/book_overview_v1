## Solution

### 實作內容

於 `.claude/methodologies/hook-system-methodology.md` 末尾新增「跨平台部署規範」一級章節，含以下結構：

1. **斷層點總覽**（三大 Windows 斷層對照表）
   - Python 環境斷層（MS Store Python Stub）
   - Shebang 污染斷層（CRLF）
   - Console 編碼斷層（cp950/cp936）

2. **規範 1：Windows Python 環境要求**
   - 真實 Python 3.12+ 安裝要求
   - 關閉 Microsoft Store App 執行別名
   - PowerShell 驗證 stub 的標準指令

3. **規範 2：Shebang 與換行符防護**
   - 專案根目錄與 `.claude/` 的 `.gitattributes` 強制 LF
   - Windows 使用者 clone 後 `git config core.autocrlf false`
   - `git check-attr eol` 驗證方式

4. **規範 3：UTF-8 I/O 強制**
   - 機制 A：Hook 入口呼叫 `ensure_utf8_io()`（含完整實作範例）
   - 機制 B：PEP 723 搭配 `PYTHONUTF8=1` 環境變數
   - 機制 C：subprocess 呼叫強制 `encoding="utf-8", errors="replace"`

5. **規範 4：路徑分隔符**
   - settings.json 的 hook command 路徑必須使用 forward slash

6. **規範 5：Windows 測試要求**
   - shebang LF 驗證
   - UTF-8 I/O 驗證
   - subprocess encoding 驗證

7. **Hook 作者檢查清單**（6 項必做項目）

### 未納入本 Ticket 的項目

- `hook-architect-technical-reference.md` UV 單檔模板更新：本次優先補 methodology（全框架適用），tech-reference 的模板更新延後處理。主要 UTF-8 強制範例已在 methodology 規範 3 機制 A 提供。
