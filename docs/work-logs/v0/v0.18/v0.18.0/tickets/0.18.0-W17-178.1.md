### 修復摘要

新增 heading-based spawn 偵測策略，與 row-per-spawn 雙策略整合（取 max），涵蓋 W17-176 key-value 表格格式漏判場景。

### 設計決策

1. **新增正則 `_SPAWN_HEADING_PATTERN`**：匹配 `^### .*\bSpawn(?:ed)?\b.*\b(IMP|DOC|ANA)\b.*$`，IGNORECASE + MULTILINE
   - 命中：`### Spawned IMP 規劃` / `### Spawn 規劃 (DOC)` / `### Spawned DOC/ANA 清單`
   - 不命中：`### 根因分析` / `### Implementation Plan`（無 Spawn 關鍵字）
2. **新增 `_count_spawn_heading_rows(section)`**：每個命中 H3 視為 1 項 spawn 規劃
3. **新增 `_count_spawn_planning(section)`**：`max(row, heading)` 整合，避免重複計數同時不漏判
4. **`check_ana_spawn_consistency` 改用整合函式**：替換原 `_count_spawn_planning_rows` 呼叫

### 為何取 max 而非 sum

兩策略適用不同表格樣式：row-per-spawn 對應「多項目同表」，heading-based 對應「H3+key-value 單項表」。同一規劃不會同時被兩者命中（key-value 表無同行 type+priority）。取 max 是最小變更原則下避免漏判的安全選擇。

### 不在本 ticket 範圍

- 進一步降低正則誤判（如 H3 含「Spawn」但語境為討論其他 ANA 的 spawn）：目前測試已覆蓋常見 false positive，若日後出現可再迭代
- W17-176 本身的 spawn 落地檢視：已 complete，children 含 0.18.0-W17-176.1
