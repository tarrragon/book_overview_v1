## Context Bundle（來自 W17-046 WRAP 分析）

### Hook 檔案與現況

路徑：`.claude/hooks/agent-ticket-validation-hook.py`

現況（L61-65）：

```python
TICKET_EXEMPT_AGENT_TYPES = [
    "Explore",  # 探索代理人：用於蒐集資訊以建立 Ticket
]
```

根因：白名單僅含 Explore，未區分「情報蒐集 agent」vs「執行任務 agent」。

### 修改要求

**1. 白名單擴充（TICKET_EXEMPT_AGENT_TYPES）**

新增四個情報蒐集類 agent：

```python
TICKET_EXEMPT_AGENT_TYPES = [
    "Explore",                      # codebase 探索（既有）
    "claude-code-guide",            # Claude Code / SDK / API 文件查詢
    "general-purpose",              # 複雜問題多步驟研究
    "Plan",                         # 架構規劃、實作計畫（唯讀）
    "feature-dev:code-explorer",    # 既有功能深度分析
]
```

**2. Hook 註解補充**

在 `TICKET_EXEMPT_AGENT_TYPES` 常數上方補充：

- 可豁免：agent 工具僅含唯讀（Read/Grep/Glob/WebFetch/WebSearch/Bash 查詢類），用途為情報蒐集、文件查詢、分析規劃
- 不可豁免：agent 可 Edit/Write/git commit，用途為實作/重構/測試執行
- 升級路徑：白名單 > 10 或誤用率升高 → 升級為「讀 agent definition tools 自動分類」（來源：W17-046 ANA）

**3. 測試檔案（新建）**

路徑：`.claude/hooks/tests/test_agent_ticket_validation_hook.py`

覆蓋：
- 正面：每個白名單 agent type（5 個）派發時豁免，即使無 Ticket ID 也允許
- 負面：非白名單 agent type（如 `thyme-python-developer`）派發仍要求 Ticket ID
- 既有行為保留：有 Ticket ID 時正常驗證（有效允許、無效拒絕）
- Handoff 恢復模式豁免邏輯不受影響

參考風格：`.claude/hooks/tests/test_agent_dispatch_validation_hook.py`

執行：`cd .claude/hooks/tests && python3 -m pytest test_agent_ticket_validation_hook.py -v`
