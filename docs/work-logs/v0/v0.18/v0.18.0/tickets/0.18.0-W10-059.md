## Problem Analysis

### Reality Test (PC-063 強制要求)

**根因假設**: PostToolUse(Agent) Hook 在 `run_in_background=true` 派發時於代理人啟動時觸發，Hook 邏輯誤把此時點當作「代理人完成」進行清理和廣播。

**最小重現指令**（已實證兩次）:
```
1. Agent(subagent_type=X, run_in_background=true, prompt=Y)
2. 立即收到 PostToolUse:Agent 訊息「已清理派發記錄 | 所有代理人已完成」
3. ToolSearch + TaskOutput(task_id, block=false, timeout=3000)
4. 觀察 <status> 標籤
```

**實驗結果（2 次觸發）**:

| 案例 | 派發時間 | Hook 廣播 | TaskOutput status | 真完成時間 |
|------|---------|---------|-----------------|---------|
| W10-056.2（thyme） | 15:16 | 15:16 廣播「完成」 | running | 15:23（7 分鐘後） |
| W10-056.3（thyme） | 15:32 | <1 分鐘廣播「完成」 | running | 仍在執行 |

**已驗證事實**:
- Hook 於代理人啟動時即廣播「完成」，與代理人真完成狀態不同步
- 兩個案例均發生於 `run_in_background=true`
- TaskOutput `<status>` 標籤是可靠的 runtime 狀態查詢源
- `<status>` 值：running = 代理人仍在工作；completed = 真完成；error = 錯誤中止

**仍未驗證**: foreground 模式（`run_in_background=false` 或省略）是否有同樣時機問題（需額外實驗，但實務上 PM 不太用 foreground 代理人）。

### 根因定位

`active-dispatch-tracker-hook.py` L60-L93（PostToolUse matcher="Agent"）:
- L61: `clear_dispatch(project_root, agent_description)` — 清除 dispatch-active.json 記錄
- L84: `get_active_dispatches` 查剩餘
- L85-93: 無剩餘 → 廣播「[OK] 所有代理人已完成」

PostToolUse(Agent) 在 background 模式下 = Agent tool call 返回時刻 = **代理人啟動完成**（非工作完成）。Hook 邏輯把此時點當作代理人工作完成，觸發過早清除與廣播。

**W (Widen) 候選修正方案**:

| # | 方案 | 優點 | 缺點 |
