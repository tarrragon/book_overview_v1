## Context Bundle

### 既有實作（行號 + 介面瓶頸）

```python
# .claude/hooks/handoff-auto-resume-stop-hook.py:387-415
def should_preserve_pending_json(direction: str, logger) -> bool:
    chain_directions = {"to-sibling", "to-parent", "to-child"}
    direction_type = direction.split(":")[0]
    if direction_type in chain_directions:
        logger.debug(f"handoff 類型 '{direction}' 應保留已完成 Ticket 的 pending JSON")
        return True
    return False
```

**介面瓶頸**：只接受 `direction: str`，無法存取 record 其他欄位（target_id / from_ticket / from_status）→ 無法呼叫 `is_handoff_stale(record)`。

### 兩個介面方案（thyme 自行擇優）

| 方案 | 改動範圍 | 優點 | 缺點 |
|------|---------|------|------|
| A. 改 should_preserve_pending_json 簽章 | 從 `(direction, logger)` 改為 `(record, logger)` | 內聚高，邏輯集中於同一函式 | 需修改所有 caller |
| B. 在 caller 處先呼叫 is_handoff_stale | 保留 should_preserve_pending_json 簽章；caller 在呼叫前先 `is_stale, _ = is_handoff_stale(record)` | 不破壞既有 API | 邏輯分散於 caller 與 callee |

**建議方案 A**：should_preserve_pending_json 唯一 caller 應在 `scan_pending_handoff_tasks`（從 line 418 開始），介面改動成本可控；且原函式名「is preserve」與 `is_handoff_stale` 是反向關係，自然演進為 `return not is_handoff_stale(record)[0]`。
