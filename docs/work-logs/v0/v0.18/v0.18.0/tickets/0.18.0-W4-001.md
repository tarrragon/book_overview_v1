## Problem Analysis

### 背景資訊（WRAP 分析必讀）

**問題發現經過**：
2026-04-11 派發 basil-hook-architect 實作 error-pattern 衝突檢查時，terminal 顯示大量 `JSON validation failed: Hook JSON output validation failed`。調查發現 worktree-merge-reminder-hook（PostToolUse:Bash）在代理人每次 Bash 呼叫都觸發，共 46 次。

**根本問題**：
PostToolUse:Bash matcher 範圍太廣，所有 Bash 工具呼叫都會觸發所有掛載在此 matcher 的 Hook。worktree-merge-reminder-hook 實際只需要在 `ticket track complete` 命令時執行。

**已知影響**：
- 代理人每次 Bash 呼叫都觸發 N 個不必要的 Hook
- Hook 執行消耗時間和資源
- Hook error（如 IMP-055 的 JSON validation failed）會干擾代理人

**WRAP 分析方向**：
- W：有哪些方式可以限制 Hook 觸發範圍？（command 過濾、Hook 內部 early return、拆分 matcher）
- R：Claude Code CLI 的 matcher 機制支援什麼級別的過濾？
- A：修改 settings.json 結構 vs 修改 Hook 內部邏輯，哪個風險更低？
- P：修改 matcher 機制可能影響哪些現有 Hook 的正常觸發？

**參考資料**：
- .claude/hooks/worktree-merge-reminder-hook.py — 已有 command 檢查邏輯但仍被觸發
- .claude/settings.json — PostToolUse:Bash matcher 的所有 Hook
- IMP-055 — Hook stdout JSON 格式問題（已修復，但觸發頻率問題仍在）
