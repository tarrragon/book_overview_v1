### 修復摘要

新建 PostToolUse Hook `ana-ticket-metadata-validation-hook.py`，於 ticket md 寫入時驗證 ANA 來源 ticket 的 metadata。

### 設計決策

| 決策 | 理由 |
|------|------|
| 新建 Hook 而非擴充 ticket-creation-validation | 職責單一；PC-058 邏輯（CLAUDE.md 對照、acceptance 拆解、tdd_phase 啟發式）足以獨立模組 |
| PostToolUse Write/Edit matcher | ANA 代理人完成後產出 ticket md，寫檔即觸發；不依賴 ticket CLI 是否被呼叫 |
| 警告而非阻擋（exit 0 + stderr） | 符合 ticket AC 與 PC-058 短期防護建議；PM 仍可派發前修正 |
| 僅檢 ANA 來源 ticket | 過濾條件：who.current 含 saffron/sage 或 source_ticket 非空，避免誤報一般 IMP |

### 驗證項目

1. who.current vs CLAUDE.md `實作代理人`（限 IMP/FEAT/BUG type）
2. acceptance：每項 ≤ 100 字元、無 `;` `；` ` and ` `&&` 多條件分隔符
3. tdd_phase：DOC type 不應有 phase；tdd_stage 全 4 phase + what 極短時提示

### 不在本 ticket 範圍

- where.files 寫入目標驗證（PC-058 提及，但 acceptance 未列；後續可擴充）
- ANA 代理人 prompt 強化（屬 PM 派發改善，非 Hook）

### 註冊位置

`.claude/settings.json` PostToolUse > Write matcher，timeout 5000ms。
