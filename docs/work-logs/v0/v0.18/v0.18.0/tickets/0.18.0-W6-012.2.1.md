### 真根因（已 100% 確認）

`src/content/adapters/readmoo-adapter.js:871-894` `applyIdGenerationStrategiesWithInfo()` 策略順序為 cover → title → reader-link，導致 privacy ID 即使取到（Phase 2 實驗 5/5 樣本 privacyHit）也被 cover-XXX 覆寫為主 ID。

實機 chrome-devtools-mcp 證據：
- 96 個 `[id^="privacy-"]` 與 `.library-item` 1:1 對應
- DOM path: `.library-item > .cover-outer > .desktop-overlay > .openbook-overlay > div.privacy[id="privacy-{8位數字}"]`
- 5/5 樣本 `effectiveUrl` 已正確重建為 `/api/reader/{privacyBookId}`，僅 `bookData.id` 主欄位被 cover 策略覆寫

### 具體實作改動（從 W6-012.2 Solution 複製）

#### Change 1：`tryReaderStrategy` 返回值（line 882-888）

當前：
```js

```

改為：
```js

```

#### Change 2：`applyIdGenerationStrategiesWithInfo` 順序重排（line 871-894）

當前：cover → title → reader-link → fallback
改為：**reader-link → cover → title → fallback**

```js
applyIdGenerationStrategiesWithInfo (inputs) {
  const readerResult = this.tryReaderStrategy(inputs)
  if (readerResult) {
    return { id: `reader-${inputs.readerId}`, strategy: 'reader-link' }
  }
  const coverResult = this.tryCoverStrategy(inputs)
  if (coverResult) {
    return { id: coverResult, strategy: 'cover' }
  }
  const titleResult = this.tryTitleStrategy(inputs)
  if (titleResult) {
    return { id: titleResult, strategy: 'title' }
  }
  return { id: this.createFallbackId(), strategy: 'fallback' }
}
```

#### Change 3：`tryCoverStrategy` 過濾 known-unstable（line 902-906）

新增常數（建議放檔案頂部 SELECTORS 附近）：
```js
const UNSTABLE_COVER_IDS = new Set(['openbook', 'undefined', 'placeholder', 'default'])
```

修改：
```js
tryCoverStrategy ({ cover }) {
  if (!cover || !cover.trim()) return null
  const coverId = this.extractCoverIdFromUrl(cover)
  if (!coverId) return null
  if (UNSTABLE_COVER_IDS.has(coverId)) return null
  return `cover-${coverId}`
}
```
