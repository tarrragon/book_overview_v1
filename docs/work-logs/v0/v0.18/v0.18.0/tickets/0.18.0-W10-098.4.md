### 變更摘要

修改檔案：`docs/proposals/PROP-003-backup-recovery.md`

在 YAML frontmatter 末尾（`supersedes: null` 之後，`---` 之前）加入兩個欄位：

```yaml
# draft 過期暫設 light；promote 至 confirmed/approved 時需重評 standard/heavy
evaluation_level: light
promotion_review_required: true
```

### 設計決策

- `evaluation_level: light`：PROP-003 status=draft（目標版本 v0.16.3 已過期），依 W10-098 WRAP B+D 結論，draft 過期提案暫設 light，hook 自動跳過章節完整性檢查，避免阻擋整個 PROP 目錄的正常 Edit。
- `promotion_review_required: true`：明示本欄位為暫設值，待 PROP-003 promote 至 confirmed/approved 時，評估者須重新評估是否升級至 standard 或 heavy，不可沿用 light。
- 不補章節、不修 body 內容（PROP-003 body 反映過期規格，修改超出本 ticket 範圍）。

### Hook 通過理由

proposal-evaluation-gate-hook 對 `evaluation_level: light` 的提案自動跳過章節完整性驗證，故 Edit 不受阻擋。
