### 問題根因

兩個 hook 從 stdin 讀取 Claude Code 提供的事件欄位時使用 camelCase（hookEventName / toolName / toolInput），但官方 spec（.claude/hook-specs/claude-code-hooks-official-standards.md）規定為 snake_case（hook_event_name / tool_name / tool_input）。導致 `input_data.get("hookEventName")` 永遠回傳預設值，hook 始終 fall-through 到「未知 Hook 事件」分支，等同失效。

### 影響範圍

- `.claude/hooks/dispatch-count-guard-hook.py` line 331：路由分發邏輯
- `.claude/hooks/file-ownership-guard-hook.py` lines 256, 293-294：is_valid_trigger 與 tool_input 提取
- ticket 原 spec 為 lines 293-294；同步發現 line 256 同類 bug（toolInput → tool_input），不修則 hook 拿不到 tool_input.target_id 仍無法工作

### 相關 Error Pattern

W10-048 ANA：M-3 降級計畫第二批 P2 hook 失效診斷

### 修復摘要

將兩 hook 從 stdin 讀取的欄位從 camelCase 改為 snake_case，對齊 Claude Code 官方規範。

### 修改清單

1. `.claude/hooks/dispatch-count-guard-hook.py` line 331
   - `input_data.get("hookEventName", "")` → `input_data.get("hook_event_name", "")`

2. `.claude/hooks/file-ownership-guard-hook.py` lines 293-294（is_valid_trigger）
   - `input_data.get("hookEventName")` → `input_data.get("hook_event_name")`
   - `input_data.get("toolName")` → `input_data.get("tool_name")`

3. `.claude/hooks/file-ownership-guard-hook.py` line 256（同類 bug，順帶修復）
   - `input_data.get("toolInput", {})` → `input_data.get("tool_input", {})`

### 測試新增

- 新增 `.claude/hooks/tests/test_dispatch_count_guard_hook.py`：6 個測試（含回歸防護：camelCase 不命中 handler）
- 更新 `.claude/hooks/tests/test_file_ownership_guard_hook.py`：將既有 stdin fixture 從 camelCase 替換為 snake_case（既有測試之前在驗證 bug 行為）

### 範圍外（在 ticket scope 之外但已順帶處理）

line 256 toolInput → tool_input 不在原 ticket spec（spec 為 lines 293-294），但屬同類 bug 且不修則 hook 仍失效；以單一 commit 涵蓋避免遺留半修狀態。
