### 修復摘要

在 `_execute_auto_handoff` 構造 handoff JSON 時新增 `context_bundle` 欄位，值來自新 helper `_extract_context_bundle_for_handoff(ticket)`：

1. import `extract_context_bundle`（lazy import 於 helper 內，避免冷啟動成本）
2. 呼叫後以 `dataclasses.asdict(result)` 序列化 ExtractResult 為 JSON 可寫的 dict
3. import 失敗 / 抽取例外 → 印 stderr warning（`format_warning`）+ 回傳 None，handoff 主流程不阻擋

### 設計決策

- **欄位永遠存在**（`context_bundle` 鍵總在 JSON 中，no_source 時為 ExtractResult 空殼，例外時為 None）：消費端（hook / agent prompt 組裝）可用單一檢查路徑
- **使用 `dataclasses.asdict`**：ExtractResult 已是 @dataclass，asdict 遞迴處理 ExtractedField / SkipRecord，輸出純 dict 結構，避免自寫序列化
- **降級 stderr 而非檔案 log**：handoff 為 CLI 工具，stderr 警告對用戶可見（規則 4 / IMP-013）
- **lazy import**：避免 handoff 路徑冷啟動載入 extractor 增加耗時

### 不在本 ticket 範圍

- `_execute_handoff`（互動模式）整合 context_bundle：W17-031.2 範圍僅 `--auto`，互動模式維持現狀
- handoff-reminder-hook.py 消費 context_bundle：屬 W17-031.5 / 後續 ticket
