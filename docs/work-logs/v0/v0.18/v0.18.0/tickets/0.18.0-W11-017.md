### 修復摘要

針對 `phase-completion-gate-hook.py::is_phase_completion_report` 加入三層 guard，
過濾「ticket md 含 Phase 4 文本引用」的誤判（PC-142 同家族）。

### 設計決策

新增 module-level helper：

| Helper | 用途 |
|--------|------|
| `_parse_frontmatter(content)` | 從 ticket md frontmatter 提取 `status` 與 `id`；非 ticket md 回傳 (None, None) |
| `_has_phase_section_heading(content)` | regex 偵測 `## Phase 3b/4` 或 `## Phase 完成` 等 H2/H3 標題行 |

`is_phase_completion_report` 重構為四個判定階段：

1. **檔名識別**（既有行為，最高優先）— 檔名含 phase 關鍵字直接回傳 True
2. **Layer 3 ticket frontmatter id guard** — 解析 frontmatter；含 `id:` 視為 ticket md，走嚴格判定
3. **Layer 1 狀態 guard** — `status ∈ {pending, in_progress, blocked, draft}` 時內文 phase 關鍵字視為文本引用；但若 Layer 2 命中章節標題仍識別
4. **Layer 2 章節位置 guard** — 對 ticket md 一律要求 `## Phase X` 標題行才識別，純段落 keyword 不算
5. **非 ticket md** — 維持原寬鬆 keyword + title pattern 識別（向後相容獨立 phase 報告檔）

### 對齊規則

- `decision-trigger-binding.md` 規則 4「違規偵測精度」— 文本引用 vs 真實章節必須區分
- PC-142（phase4-hook rule quote misjudgment）同家族修補策略
- PC-146（exempt marker proximity）— 採「位置 guard」而非「字面豁免清單」實作

### 不在本 ticket 範圍
