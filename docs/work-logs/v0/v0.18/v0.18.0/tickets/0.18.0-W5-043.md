# Agent Definition 結構標準

每個 .claude/agents/*.md 的主文（description 之後）必須含以下三區塊：

## 允許產出
- 明列此 agent 可產生的產出類型（檔案類別、操作類型）

## 禁止行為
- 明列此 agent 不可做的事，包含哪種檔案、哪種操作

## 適用情境
- TDD Phase 標註（Phase 0/1/2/3a/3b/4）或獨立任務類型
- 觸發條件描述

## 豁免類別
- 元文件（如 AGENT_PRELOAD.md）
- 已 DEPRECATED 的 agent（明示 DEPRECATED 標記）
\`\`\`

### 驗收條件

- AC-1: 新建 .claude/rules/core/agent-definition-standard.md 規範三區塊格式
- AC-2: 批量 patch 25 個已有「禁止行為」的 agent 補「允許產出」+「適用情境」
- AC-3: patch 2 個完全缺的 agent（incident-responder, thyme-python-developer）補全三區塊
- AC-4: grep 驗證 27 個 agent 各含三區塊（每個區塊至少出現 1 次）

### 禁止行為

- 禁止修改 description（只加新區塊，不動既有 frontmatter）

- 每個 agent 的「允許產出」「適用情境」內容需根據其現有 description 和 Tools 欄位推斷，禁止憑空捏造
