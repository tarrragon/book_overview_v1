### 實作要點

在 `.claude/skills/worktree/SKILL.md` 適當位置（建議「核心功能」之後）新增章節：

```markdown
## Agent isolation worktree（cc 自動建 worktree-agent-*）

### 機制
- Claude Code Agent tool 設 `isolation: "worktree"` 時，cc runtime 自動建 `.claude/worktrees/agent-XXXXXXXX` worktree
- 建立同時加 git lock（含 cc CLI process PID），防止 git GC

### 殭屍問題
- cc 在 agent 結束 / process 死亡時**不會自動清** worktree
- PID 死後 lock 變殭屍，累積污染 statusline 與 disk

### 專案對策（W17-119.1 SessionStart hook GC）
- `.claude/hooks/worktree-zombie-cleanup-hook.py` 在 SessionStart 觸發
- 邏輯：列舉 → PID 死活檢測 → dead 則 unlock + remove
- 防護：dirty != 0 警告不清；排除 < 30 min 建的；env var 開關

### 手動清理指令

`​`​`bash
# 列出殭屍（PID 死的）
git worktree list --porcelain | grep "^locked" | grep -oE "pid [0-9]+" | awk '{print $2}' | while read p; do ps -p $p > /dev/null || echo "$p dead"; done

# 強制清所有 agent worktree（謹慎使用）
git worktree list --porcelain | grep "^worktree .claude/worktrees/agent-" | awk '{print $2}' | xargs -I{} git worktree remove --force {}
`​`​`

### 與人工 /worktree create 的區別

| 維度 | cc Agent isolation:worktree | 人工 /worktree create |
|------|----------------------------|----------------------|
| 觸發者 | cc runtime（Agent tool） | 使用者 |
| 路徑 | `.claude/worktrees/agent-XXX` | `../ccsession-<ticket-id>` |
| 分支命名 | `worktree-agent-XXX` | `feat/<ticket-id>` |
| Lock | 自動加 lock（含 PID） | 不加 lock |
| 清理 | cc 不清，需 W17-119.1 hook | 使用者 `/worktree remove` |
```
