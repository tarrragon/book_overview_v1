### 實作摘要

新建 `.claude/hooks/homoglyph-guard-hook.py`（UV single-file mode, exec bit 已設 PC-086），PreToolUse 攔截 `git commit` 命令，掃描 `git diff --cached -U0` 是否存在已知形似字混淆對的「替換」模式。

### 混淆對清單（可擴充）

定義於 hook 內 `HOMOGLYPH_PAIRS` dict：

| 誤改字 | 正確候選字 | 來源 |
|--------|----------|------|
| 汲 (U+6C72) | 汙 (U+6C59) / 污 (U+6C61) | PC-150 / W11-028 |

擴充方式：新增 entry 至 `HOMOGLYPH_PAIRS`，格式 `{"誤改字": {"候選正確字集合"}}`。

### 偵測策略

避免一般正當使用（如「汲取」）誤判：僅當同一 hunk 內存在 `-line` 含正確候選字（汙/污） 且 `+line` 含誤改字（汲），才視為誤替換並阻擋。新增檔案內含「汲取」、純新增內容含汲均不觸發。

### 行為

- 命中：exit 2 + stderr 列檔案/hunk/原始-修改後示例 + 修正建議（符合 quality-baseline 規則 4 stderr 可見性）
- 通過：exit 0
- 例外（diff 取得失敗、JSON 解析失敗）：exit 0（避免誤擋）

### 豁免

- commit 命令含 `[skip homoglyph]` 標記
- 非 `git commit -m` 形式（amend / log / show / diff / status 排除）

### 註冊

`.claude/settings.json` PreToolUse `Bash` matcher 區尾端新增 hook entry，與既有 phase4-decision-enforcement-hook 同層級。

### 自測場景與結果

於 `/tmp/hg-test` 建立隔離 git repo，四個場景：

| # | 場景 | 輸入 | 預期 | 實測 |

| 1 | 汙→汲 replacement（同 hunk 內 - 含 汙、+ 含 汲） | `git commit -m test` | exit 2 + stderr 報告 | exit 2，stderr 列 file/hunk/diff |
