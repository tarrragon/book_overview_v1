## 重現實驗結果（2026-04-13 PM 前台執行）

**實驗方法**：清理 docs/work-logs/v0/v0.31/ 後逐檔案跑 pytest。

**污染源（2 個檔案）**：

1. `.claude/skills/ticket/tests/test_track_batch.py`
2. `.claude/skills/ticket/tests/test_track_relations.py`

**根因 — 測試 mock 漏/錯路徑**：

`track_relations.py` 和 `track_batch.py` 使用 `from ...lib.ticket_loader import save_ticket`，import 綁定在 commands 模組的 namespace。

| 反模式 | 範例位置 | 錯誤 |
|-------|---------|-----|
| 完全不 mock save_ticket | test_track_batch.py L142-L176（test_batch_complete_with_incomplete_criteria_failure）、L181-L209（test_batch_complete_nonexistent_ticket）、L250+ | 真實 save_ticket 執行 → 寫入 repo |
| Mock 錯誤 import 路徑 | test_track_relations.py L159、L187、L248（`patch('ticket_system.lib.ticket_loader.save_ticket')`） | patch 不生效（command 已 `from ... import save_ticket`），真實 save_ticket 仍執行 |
| 未 mock save_ticket | test_track_relations.py L132（test_add_child_already_exists）、L187（test_phase_set_success 用錯路徑） | 同上 |

**正確 mock 路徑**：
- `ticket_system.commands.track_relations.save_ticket`
- `ticket_system.commands.track_batch.save_ticket`

**次要根因**：
1. 測試未使用 `tmp_path` 或 `CLAUDE_PROJECT_DIR` env 沙箱化——即使未 mock，若 env 指向 tmp 目錄也不會污染真實 repo
2. 缺少 pytest session level 污染偵測（autouse fixture 掃描專案根目錄新增的版本目錄）

## 問題根因（最終結論）

**直接原因**：數個測試呼叫會觸發 `save_ticket` 的函式（`execute_add_child`/`execute_batch_complete`/`execute_phase`）時，沒有 mock `save_ticket` 或 mock 了錯誤的 import path。

**間接原因**：測試未沙箱化（無 tmp_path 隔離、無 env 重導向），且缺少 session level 污染偵測。

## 影響範圍

- 測試檔案：2 個（test_track_batch.py + test_track_relations.py）
- 實際寫入的 call：每次跑測試在 repo 根生成 `docs/work-logs/v0/v0.31/v0.31.0/tickets/0.31.0-W4-001*.md`
- 無功能影響（測試仍 pass，只是 untracked file 污染 git status）
