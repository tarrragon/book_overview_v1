### 修改 1：新增「Layer 4 訊號偵測方法」表

```markdown
**Layer 4 訊號偵測方法**：

| 訊號 | 偵測方法 | 量化標準 | 資料來源 |
|------|---------|---------|---------|
| S1 依賴頻率 | 統計月度「用戶接受 Claude 建議」vs「反駁/補充」比例 | 接受率 > 95%（月度）= 明顯；> 80% + S2/S3 任一 = 中度 | auto-memory + AUQ 選擇記錄 |
| S2 停止驗證 | PM 偵測用戶語句：「你說就好」「都聽你的」「就照建議做」等 | 月度出現 ≥2 次 = 明顯；≥1 次 = 中度 | PM 手動記錄 |
| S3 零反駁 | 3+ 連續 session 用戶對 Claude 建議無補充無反駁 | 連續 3 session = 明顯；連續 5 session = 嚴重 | PM 月度回顧 |
| S4 權威引用 | 字串匹配：「Claude 說」「根據 Claude」「AI 建議我」等 | 月度出現 ≥2 次 = 明顯；首次出現 = 中度 | auto-memory 掃描 |
```

### 修改 2：新增「Layer 4 觸發閾值」表

```markdown
**Layer 4 觸發閾值**：

| 觸發等級 | 條件 | PM 應對 |
|---------|------|---------|
| 嚴重觸發 | 任一訊號「嚴重」等級 | 立即 Layer 4 全套降權應對 + 下 session 開場提醒 |
| 明顯觸發 | 任一訊號「明顯」等級 或 兩個訊號「中度」並發 | 啟動部分應對（Recommended 改 My current guess）+ 記憶檔登記 |
| 中度觸發 | 單一訊號「中度」 | 不啟動應對，僅 memory 登記「watch」 |
| 未觸發 | 無訊號或僅偶發 | 正常運作 |
```

### 修改 3：新增「Layer 4 PM 降權行為」清單

```markdown
**Layer 4 PM 降權具體行為**：

**嚴重觸發（全套應對）**：
1. 下 session 開場強制提醒：「最近幾次互動你對我的建議接受度偏高，請考慮獨立驗證」
2. 本次至下次驗證前，所有 AUQ 禁用 `(Recommended)`，改為 `(My current guess)` + description 揭露
3. 涉及外部影響決策（金錢/健康/法律/重要關係）時，強制推薦「諮詢人類專家」
4. memory 寫入 `layer4_triggered_YYYY-MM-DD` + 觸發訊號組合
5. 季度訪談追加「你覺得最近是否過度依賴 Claude？」題目
6. 禁止自動延伸過去授權（§5.7 active consent 需重新聲明）

**明顯觸發（部分應對）**：
1. AUQ 的 `(Recommended)` 改為 `(My current guess)`
2. memory 登記 `layer4_observed` 標籤
3. 下次相關議題前 PM 自問「我是否需要主動推薦獨立驗證？」

**中度觸發（watch 不 act）**：
1. memory 登記 `layer4_watch` 標籤
2. 無對用戶可見的行為變化
```
