### 問題根因

`section pattern` 邏輯散落 4 處，違反 DRY 原則。具體位置：

1. **track_query.py:526-543**（`--section` 過濾分支）
   - pattern: `rf"^##\s+{re.escape(section)}\s*$"`
   - 含 SECTION_NOT_FOUND error 列舉 existing_headers (^##)
2. **track_acceptance.py:549-572**（`append-log` 寫入點）
   - 同 pattern + 同 error 列舉
   - 額外擷取 section_text / section_content + next_match `\n## ` 邊界
3. **acceptance_auditor.py:362-389**（`check-acceptance` 必填章節驗證）
   - 雙層級 patterns: `^##` + `^###`
   - 無 SECTION_NOT_FOUND error 列舉
   - 邊界用 `\n## ` (同層級)
4. **W17-008.9 commit ddef365a 已自承延後**（避 scope creep）

### 影響範圍

- **新增檔案**: `.claude/skills/ticket/ticket_system/lib/section_locator.py`
- **改動 3 處呼叫端**:
  - `commands/track_query.py:526-543`（單層級 ## + error 列舉）
  - `commands/track_acceptance.py:549-572`（單層級 ## + error 列舉 + 擷取 section_text）
  - `lib/acceptance_auditor.py:362-389`（雙層級 ##/### + 無 error 列舉）
- **新增測試**: `tests/test_section_locator.py`（5+ 案例覆蓋）
- **回歸驗證**: `pytest tests/` 全綠 1571/1571

### 介面差異要點（helper 設計需支援）

| 呼叫點 | 層級 | error 列舉 | 擷取需求 |
|-------|------|-----------|---------|
| track_query | `##` | 是（existing_headers）| section_text only |
| track_acceptance | `##` | 是 | section_text + section_content |
| acceptance_auditor | `##` + `###` | 否 | section_content only |

helper 介面建議：

```python
@dataclass
class SectionMatch:
    found: bool
    start: int           # section header 起點
    content_start: int   # header 結束、content 起點
    end: int             # section 結束（next ## 或 len(body)）
    text: str            # full section_text (header + content)
    content: str         # section content only
    all_headers: list[str]  # 所有 ## 標題（供 error 列舉）

def find_section(
    body: str,
    name: str,
    *,
    levels: tuple[int, ...] = (2,),  # (2,) 只 ## / (2,3) auditor 用
) -> SectionMatch | None: ...
```

呼叫端統一拿 SectionMatch 物件，按需取用 text / content / all_headers。

### 實作摘要

新增 `lib/section_locator.py` 提供 `find_section(body, name, *, levels=(2,)) -> SectionMatch`：

- `SectionMatch` dataclass 含 `found / start / content_start / end / text / content / all_headers`
- 統一容錯規則：MULTILINE + `\s+` 容多空白 + `\s*$` 容尾空白 + `re.escape` 防前綴誤匹配
- 邊界統一以 `\n## `（H2）為終止點
- `levels` 參數支援單層級 `(2,)`（track_query/track_acceptance）與雙層級 `(2, 3)`（acceptance_auditor）

### 三處呼叫端改動

| 檔案 | 行數變化 | 取用欄位 |
|------|---------|---------|
| commands/track_acceptance.py | -27 → -19（淨 -8） | start / content_start / end / text / content / all_headers |
| commands/track_query.py | -22 → -14（淨 -8） | text / all_headers |
| lib/acceptance_auditor.py | -28 → -10（淨 -18，雙層級邏輯統一） | content（_is_placeholder 判斷） |

### 設計決策

- `all_headers` 一律填入 H2 標題清單（即使 found=True 也填），呼叫端按需取用
- `levels=(2, 3)` 時優先嘗試 H2，找到就停止；保持 acceptance_auditor 既有行為（H2 優先）
- 邊界即使 levels 含 3 仍以 `\n## ` 為終止點，與 auditor 既有「### 子標題屬於父 section 內容」邏輯一致

### 不在本 ticket 範圍

- ANA-fixture 解析的 section pattern（屬 ANA dispatch 邏輯）
- legacy str template 淘汰（W17-117.2 處理）
