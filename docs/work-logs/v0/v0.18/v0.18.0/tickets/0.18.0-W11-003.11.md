### 問題根因

W11-003.10 完成 test_migrate_reverse_refs.py 三項共識重構後，Phase 4 多視角審查（thyme/ginger/linux）保留三項細節改善：

1. **`_update_ticket_id_references` 行為缺口**：`migrate.py:58` 處理 ticket 自我引用時僅覆蓋 `blockedBy / children(dict-form only) / source_ticket` 三欄位，相較 `_update_cross_references` (L82) 處理六欄位（blockedBy/relatedTo/children/source_ticket/parent_id/spawned_tickets），缺：
   - `relatedTo`
   - `parent_id`
   - `spawned_tickets`
   - `children` 純 string list 形式（目前只處理 dict 形式）
2. **散彈式 monkeypatch**：fixture `project_with_tickets` 在三處對 `get_project_root` 設 monkeypatch（L90 migrate_mod, L91 loader_mod, L97 paths_mod），thyme + ginger F4 共識應收斂單一 patch 點
3. **Local import 重複**：實測 17 處 local import（grep 確認），以下散彈分布應評估移至 module level：
   - `_update_cross_references` 8 次（L118/141/167/264/285）
   - `_batch_migrate` 6 次（L337/362/394/439/526）
   - `_migrate_single_ticket` 1 次 (L194)
   - `_update_ticket_id_references` 1 次 (L575)
   - `yaml` 1 次 (L319，W11-003.10 重構新增)
   - 其餘 fixture 內 with try/except 保護的需保留

### 變更摘要

**(a) `_update_ticket_id_references` 補齊六欄位**（migrate.py:58-）

從 3 欄位（blockedBy / children dict-only / source_ticket）擴充至 6 欄位，與 `_update_cross_references` 對齊：

| 欄位 | 形式 | 變更 |
|------|------|------|
| blockedBy | list of string | 既有 |
| relatedTo | list of string | 新增 |
| spawned_tickets | list of string | 新增 |
| children | string list + dict 混合 | 擴充支援 string list（原僅支援 dict 形式） |
| source_ticket | scalar | 既有 |
| parent_id | scalar | 新增 |

children 處理重構為單一 loop，依 isinstance 分派 string / dict，自然保留兩種形式並覆蓋混合情境。

**(b) Monkeypatch 散彈收斂為 helper function**

新增 `_patch_get_project_root(monkeypatch, tmp_path)` 統一處理 migrate / ticket_loader / paths（try/except）三模組 patch。fixture `project_with_tickets` 簡化為單一 helper 呼叫。

paths 模組的 try/except ImportError 保留於 helper 內（並非所有版本暴露 `paths.get_project_root`），符合 ticket 設計決策中「跨版本容錯」要求。

**(c) Local import 17 → 3 (≤ 5)**

將以下 import 移至 module level：
- `_update_cross_references`（原 5 處）
- `_migrate_single_ticket`（原 1 處）
- `_batch_migrate`（原 5 處）
- `_update_ticket_id_references`（原 1 處）
- `yaml`（原 fixture 內 1 處 + module-level 1 處去重）
- `parse_frontmatter`（原 1 處）

實測剩餘縮排 import 為 3 處，全在 `_patch_get_project_root` 內（migrate_mod / loader_mod / paths_mod with try/except），屬「跨版本容錯防護」設計性保留，符合 ≤ 5 閾值。

**連帶測試補強**：`TestTD2_UpdateTicketIdReferencesSelfLoop` 從 1 個測試擴充至 10 個，覆蓋：
- 既有 source_ticket self-loop（保留）
- 新支援的 blockedBy / relatedTo / parent_id / spawned_tickets / children string list
- 既有 children dict 形式行為保留
- children string + dict 混合形式
- 六欄位一次性同時更新
- 不引用 old_id 時欄位不變

### 設計決策（W/A/P）

- **W**：children 處理改為單一 loop + isinstance 分派；helper function 集中 patch；paths import 以 try/except 保留容錯
- **A**：對齊兩個函式行為差異，消除 latent bug；test 維護性提升
- **P**：無破壞既有 16 GREEN，且補 9 個新測試全綠
