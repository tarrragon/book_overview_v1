### 設計決策

**選擇預設 pipe 模式而非 attach 模式**：

- 開發中 fresh clone 即可用，無需先啟動帶 `--remote-debugging-port=9222` 的 Chrome
- attach 是進階情境（debug installed extension），需求頻率較低，使用者手改 `.mcp.json` 可接受
- 兩個 server 名稱並存（pipe + attach）的方案因 tool 命名空間衝突風險未採納

**選擇 `--isolated`**：

- 避免 chrome-devtools-mcp 啟動的瀏覽器污染使用者主 Chrome profile
- 每次啟動使用臨時 user-data-dir，與既有 Chrome session 隔離

**選擇 `build/development` 相對路徑而非絕對**：

- 相對於 cwd（專案根），fresh clone 即可工作
- MCP server 由 Claude Code 在專案根啟動，cwd 一致

### 已知限制

| 限制 | 影響 | 追蹤 |
|------|------|------|
| `--categoryExtensions` 不支援 attach 模式 | attach 既有 Chrome 看 installed extension 時 `getExtensions` / `getExtensionLogs` 不可用，但基礎工具（console / network / screenshot）皆可用 | W6-009（Chrome 149 釋出後重評） |
| AC 3「Claude Code 重啟後工具可呼叫」需手動驗證 | 本 session 無法重啟 Claude Code 自身驗證；設定檔已就位 | 使用者下次重啟 Claude Code 後自然觸發 |
