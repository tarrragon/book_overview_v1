### Rationale Chain
- 0.18.0-W17-147 why: 從 session 案例觸發：在 book_overview_v1 主線程嘗試編輯 /Users/mac-eric/project/blog/content/skills/compositional-writing/skill.md，被兩個 hook 同時阻擋：

1. main-thread-edit-restriction-hook.py：以本專案 .claude/、docs/ 等白名單判斷，目標路徑因不在「本專案」白名單而被誤擋。但目標路徑根本不在本專案內，這個檢查應該不適用。

2. branch-verify-hook.py：以本專案 cwd 的 git branch 判斷保護分支（main），目標檔案在另一個 repo（blog），用本專案 branch 判斷無意義。

兩者共同根因：用「本專案 cwd context」判斷「外部專案目標路徑」，邏輯維度錯位。

正確設計應該是：
- 識別目標檔案所屬 repo（往上找 .git 目錄）
- 用該 repo 的白名單與 branch 狀態判斷
- 若該 repo 在保護分支，自動為該 repo 建 feature branch 後再放行
- 路徑允許清單按目標 repo 的設定判斷（每個專案可有自己的 .claude/ 規則）
