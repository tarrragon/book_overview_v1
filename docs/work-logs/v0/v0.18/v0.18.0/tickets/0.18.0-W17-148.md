### 修改摘要

**新增 helper（`.claude/lib/git_utils.py`）**：
- `find_target_repo(file_path) -> Optional[str]`：從檔案路徑往上走訪父目錄，遇 `.git`（目錄或檔案）即視為 repo 根並回傳絕對路徑；找不到則 None
- 純檔案系統查找，不依賴 `git` CLI（效能 + 離線可用）
- 同時處理一般 repo（`.git` 為目錄）與 worktree（`.git` 為檔案，內含 gitdir 指標）

**修改 hook（`.claude/hooks/main-thread-edit-restriction-hook.py`）**：
- import 增加 `find_target_repo`
- 在 `check_file_permission` 之前，若 `file_path` 為絕對路徑：
  - 取 `target_repo = find_target_repo(file_path)`
  - 取 `current_root = Path(get_project_root()).resolve()`
  - 若 `target_repo and target_repo != current_root` → log + `_allow_and_exit(...)` 直接放行
- 保留所有既有路徑（subagent 跳過、開發分支跳過、相對路徑與本專案路徑）行為不變

### 設計決策

- **跨專案放行原因**：`path_permission` ALLOWED/BLOCKED 模式是本專案 `.claude/`、`docs/`、`tests/` 等相對路徑 regex，套用到外部 repo 絕對路徑必然全 miss → 預設拒絕。對 path_permission 來說，外部 repo 不在其語意設計範圍內。
- **branch-verify 職責不重疊**：本 hook 只 skip path_permission，不影響 branch-verify-hook（W17-149 並行 ticket 處理跨 repo 分支判斷）。
- **失敗保守**：`find_target_repo` 回傳 None 時（target 不在任何 git repo 內）走原 path_permission 邏輯，不過度放行。

### 不在本 ticket 範圍

- branch-verify-hook 跨 repo 分支判斷 → 由 0.18.0-W17-149 處理
- path_permission 規則重新設計（讓外部 repo 也適用某種白名單）→ ANA W17-147 已評估為過度設計，不採
