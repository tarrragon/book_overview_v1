在 .claude/rules/core/pm-role.md 行為循環章節（派發後行為段落之後）新增「對話列選項時：必用 AskUserQuestion（強制）」段落，包含以下要素：

1. 觸發條件：明確指出 PM 在行為循環任一階段（聆聽/拆分/分析/派發/收取/驗收）回覆呈現需用戶決策時必用 AUQ
2. 觸發訊號表（4 行）：對應 askuserquestion-rules 規則 1（多選/二元確認/等待回應）和規則 3（純文字提問）
3. 反模式表（4 行）：含 Markdown 列表、純文字問句、替用戶選擇、合理化跳過——其中第三項「替用戶選擇後再告訴用戶」即驗收條件 4 要求的反模式
4. SOP 4 步驟：從自問到使用 AUQ 工具的完整流程
5. 適用範圍註記：對無 Ticket 場景同樣適用（規則 4）
6. 來源段落：完整引用 askuserquestion-rules.md（規則 1、規則 3）與 PC-064 error-pattern 路徑
