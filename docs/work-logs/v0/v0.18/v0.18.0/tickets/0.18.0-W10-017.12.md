Phase 4b P1 重構完成（3 檔異動 + 1 新增測試檔）：

AC1 _compute_blockers 上移 lib 層
- 新增 lib/checkpoint_view.Blocker（frozen dataclass）+ compute_blockers(state, ticket_id)
- 從 commands/track_handoff_ready.py 移除 _Blocker / _compute_blockers
- commands 層僅 consume lib 結果決定 exit code

AC2 PRIORITIES 改 NamedTuple
- 新增 PriorityRule NamedTuple（predicate/phase/label/action_fn/commands_fn）
- 新增 FallbackRule NamedTuple（phase/label/action_fn/commands_fn）
- PRIORITIES 與 FALLBACK 改用具名欄位建構，消除匿名 5-tuple unpack
- 匯出 PriorityRule / FallbackRule 至 __all__

AC3 合併 _derive_checkpoint + get_suggested_commands
- 新增 _resolve_rule(state) 回傳 (phase, label, action, commands) 單次 PRIORITIES loop
- _derive_checkpoint 改為 _resolve_rule 薄包裝（保留 3-tuple 簽章向後相容）
- get_suggested_commands 改為委派 _resolve_rule（不再獨立掃描 PRIORITIES）

AC4 _print_go 複用 render_ready_check
- 刪除 _print_go 手刻 checklist（四行 active_agents/uncommitted/worktree/handoff）
- 改呼叫 render_ready_check(state, caller="handoff-ready") 渲染統一 checklist

AC5 分層契約回歸測試
- 新增 tests/test_checkpoint_layering_contract.py（8 個 test）
- C1 契約：compute_blockers 在 lib 層；commands 禁止定義 _compute_blockers / _Blocker
- C2 契約：PRIORITIES / FALLBACK 元素為 NamedTuple 而非匿名 tuple
- C3 契約：_resolve_rule 為單源；_derive_checkpoint 與 get_suggested_commands 皆委派；禁止 in PRIORITIES 再度獨立掃描

邊界遵守：僅動 Context Bundle 列 3 檔 + 1 新增 test；未改對外 CLI 介面；017.13 未動。
