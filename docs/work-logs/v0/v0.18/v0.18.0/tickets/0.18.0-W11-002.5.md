## Problem Analysis

### 問題根因

ANA 0.18.0-W5-022 驗證 W11-002.1 cascade 重構時發現：既有 cascade 測試（TC-A1 ~ TC-C2）共用 `complete_env` Mock 框架，parent 與 children 的 dict 在記憶體中為同一引用。即使 `_post_complete_cascade` 改成在呼叫者尚未 `save_ticket(parent)` 前就執行（順序顛倒），測試仍會綠燈，因為：
- Mock 的 `list_tickets.return_value` 直接回傳 in-memory `all_tickets`，不經 disk
- parent dict 在 `execute_complete` 內被 in-place 改為 `status=completed`，cascade 透過 ticket_map 看到的就是已 completed 的 parent dict（即使尚未落盤）

→ 真實 disk 路徑（fallback 走 `list_tickets(version)` 重新從 disk 讀）會在 parent 未落盤時讀到舊 status，child 就不會被解鎖；但 Mock 測試永遠綠燈，看不到此 regression。

### 修復摘要

補強 cascade 測試契約（ANA W5-022 防護）：避免 Mock 共用 in-memory dict 矇混「parent 必須先落盤再 cascade」的順序契約。

### 變更內容

1. **`ticket_system/commands/lifecycle.py` `_cascade_unblock_children` docstring**
   - 新增「呼叫契約（W11-002.5 / ANA W5-022）」段：
     - caller 必須在呼叫前已將 parent 落盤為 completed（含 fallback 路徑解釋）
     - ticket_map 中 child dict 會被原地 mutate（status → pending），caller 不應重用
     - save 失敗不 fail-fast（單一 child 失敗只列入 warnings）
   - 重構此函式呼叫順序前必須同步更新測試契約。

2. **`tests/test_track_lifecycle.py` 新增 3 個契約測試**
   - **TC-D1（`TestCompleteCascadeSaveOrderContract`）**：用 spy 記錄 `save_ticket` / `list_tickets` 呼叫順序，斷言 `save_ticket(parent)` 必先於 `list_tickets()` 第一次呼叫。
   - **TC-D2（`TestCascadeRealDiskRoundtrip`）**：tmp_path 真實往返。寫 parent (completed) + child (blocked, blockedBy=[parent]) 到 disk，呼叫 `_post_complete_cascade` 不傳 ticket_map（走 list_tickets fallback），清快取後重新 `load_ticket(child)` 驗證 disk 上 status 為 pending。
   - **TC-D3（負向對照）**：disk 上 parent 仍 in_progress、in-memory parent dict 標 completed（模擬 caller 改了狀態但沒 save），驗證 cascade 從 disk 讀真實 status 並保留 child 為 blocked。此測試是 TC-D2 的對照組，若 cascade 改成依賴 in-memory dict 而非 disk，本測試會抓到 regression。

### 設計決策

- **TC-D2/D3 直接呼叫 `_post_complete_cascade` 而非走 `execute_complete`**：避免在 real-disk 路徑上構造 IMP body schema 全章節（type-aware body validation 會擋）。契約焦點是 cascade 對 disk parent status 的依賴，不是整個 complete pipeline。
- **TC-D1 走 Mock 框架**：順序斷言不需真實 disk，重點是 execute_complete 內部呼叫順序，用 spy 即可。
- **快取清理**：每個 real-disk 測試在 load 前清 `_ticket_cache` + `_chain_index_cache`，避免 cross-test 殘留干擾。

### 不在本 ticket 範圍

- 重構 cascade 順序本身（W11-002.1 已完成抽取 `_post_complete_cascade`，本 ticket 只補測試契約）。
- `_can_cascade_unblock` 邏輯改動。
