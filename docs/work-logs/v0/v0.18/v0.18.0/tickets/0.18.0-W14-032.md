## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

Claude Code v2.1.133+ 引入 `effort.level` payload 與 `$CLAUDE_EFFORT` 環境變數，hook 可依當前 effort 動態調整驗證深度。本專案 107 個 hook 全部以 medium 行為運作，未利用 effort 訊號；low effort 互動下嚴 validator 仍全執行，造成不必要的 token 消耗與互動阻塞。

### 影響範圍

`.claude/hooks/*.py` 全部 107 個檔案。重點影響面：類別 A 嚴 validator（≈16 個）在 low effort 高頻互動下持續阻塞。

### 相關 Error Pattern

- PC-145（CLI install 過期）— 提醒 effort 感知改動需 reinstall ticket CLI 才生效（若改動觸及 ticket_system）
- PC-093（無 trigger 延後）— effort 感知不可削弱 PC-093 偵測，phase4 hook 必須在 low effort 下仍阻擋
- PC-082（regression fix）— 移除原驗證前必檢全域規則
## 重現實驗結果

### 實驗方法

本 ANA 為純規劃分析，無需重現實驗。實證來源：(1) `ls .claude/hooks/*.py` 列出 107 個檔案；(2) 依檔名語意 + 既有 hook 知識（agent prompt 已內化所有 hook 名稱與職責）分類；(3) 參照 `.claude/references/hook-architect-technical-reference.md` §環境變數與 effort 感知章節確認 API 規格。

### 實驗執行

- 執行 `ls /Users/mac-eric/project/book_overview_v1/.claude/hooks/*.py | wc -l` → 107
- 扣除 4 個非 hook 工具（dispatch_stats / eval_pyyaml / hook-completeness-check / hook-health-monitor），實質 hook ≈103
