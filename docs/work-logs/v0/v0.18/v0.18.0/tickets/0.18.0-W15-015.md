## 路徑選擇

**選 Path A（enhanced retrospective）**，放棄 Path B/C。理由：

1. E3 已證明 find_files rate 0.412 > 0.3，問題從「是否存在偏誤」轉為「偏誤的機制」
2. Path B prospective A/B 的致命缺陷：control/treatment 都載入同套 .claude/rules/，測的是「prompt 工程」不是「預設啟發式」；且 single-session single-model 樣本不足以支持 causal claim
3. Path A 的分層分析可直接區分 H1（步驟估算偏誤）/ H2（工具等價性）/ H3（情境依賴）三假設，成本為 0

| Mixed session 可替代率 | 0.609 |
| Session profile | mixed=9, glob-only=12, long-only=16, neither=13 |

判讀：H1（步驟估算偏誤）觀察性強證據支持。

- 16 個 long-only session 顯示偏誤最強訊號
