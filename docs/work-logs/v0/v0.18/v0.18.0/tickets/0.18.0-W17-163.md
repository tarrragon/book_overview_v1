### 問題根因

W17-162 ANA 識別三層耦合根因：

**Root Cause 1：direction 概念模型缺統一分類（L1-B）**
- `constants.py` 定義 TASK_CHAIN_DIRECTION_TYPES + NON_CHAIN_DIRECTION_TYPES 兩組
- handoff.py:1228 的 `_VALID_AUTO_DIRECTIONS` 額外接受 `to-source`，但 to-source 不在任一常數組
- 結果：to-source 是合法 auto direction，但對 GC / stale / is_valid_direction 都是「未知類型」

**Root Cause 2：GC 非任務鏈分支邏輯漂移（L1-A）**
- handoff_gc.py:78-81 獨立重寫 stale 判定，與 handoff_utils.is_handoff_stale 不一致
- 舊 GC 對非任務鏈：`from_status != "completed"` 才標 stale
- is_handoff_stale 對非任務鏈：`is_ticket_completed(from_ticket)` 直接標 stale（情境 2 優先）
- ARCH-020 同構：跨進程（GC + reminder-hook + stop-hook）重複實作 stale 判定

**Root Cause 3：terminal ticket 仍可建非任務鏈 handoff（L1-C）**
- terminal status（completed/closed）的 ticket 建 context-refresh handoff 後 SessionStart/Stop hook 視為孤兒
- 入口未 guard，孤兒 JSON 在 GC 清理前持續被誤報為「待恢復任務」

### 修復摘要

對齊 W17-162 ANA L1 三項修復，採跨進程同步修復豁免條款（≤2 進程 / ≤4 檔 / 同根因群）。

#### L1-A: handoff_gc 改 delegate 至 is_handoff_stale

`_collect_stale_handoffs` 從獨立判定改為呼叫 `handoff_utils.is_handoff_stale(record.data)`，
單一 stale 判定來源，消除 ARCH-020 同構。Markdown 格式因無 direction 資訊沿用原行為
（is_ticket_completed → True 即 stale）。

```python
# 舊：JSON 分支獨立判定（任務鏈/非任務鏈兩組規則 vs handoff_utils 不同步）
# 新：
is_stale, reason = is_handoff_stale(record.data)
if is_stale:
    stale.append((record.file_path, ticket_id, reason))
```

#### L1-B: 移除 to-source

- `_VALID_AUTO_DIRECTIONS` 從五項改四項（移除 to-source）
- argparse help text 同步移除 to-source
- 改動原因：(1) 不在 constants 任一 direction 組；(2) chain_analyzer 不產生此方向；
  (3) 語意矛盾（from_ticket completed 時 source 已無可操作性）；(4) 「回溯 source」需求
  改由 source_ticket 欄位 + runqueue --context=resume 路由處理

#### L1-C: terminal handoff 防護

`_execute_auto_handoff` 入口在 ticket 載入後加 guard：

```python
ticket_status = ticket.get("status")
direction_type = direction.split(":", 1)[0]
if (
    ticket_status in TERMINAL_STATUSES
    and direction_type not in TASK_CHAIN_DIRECTION_TYPES
):
    raise ValueError(...)
```

任務鏈方向（to-sibling/to-parent/to-child）允許，因 chain handoff 仍有任務鏈下游目標。

### 設計決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| L1-A 移除舊邏輯 vs 保留作 fallback | 完全移除 | 漂移本身是 bug 來源；保留 fallback 等於保留 ARCH-020 |
| L1-C 用 ValueError vs return 1 | ValueError | 對齊 ANA strategy 字面要求；表達「不該發生的非法輸入」語意 |
| 既有測試 context_refresh + from_status==completed not_stale → 翻為 is_stale | 翻轉 | 對齊 is_handoff_stale 統一規則（情境 2：is_ticket_completed 優先於 from_status 快照）|
