```
function dashboard_main(args, version):
    if version is None:
        stderr_print("No active version detected")
        return 1
    try:
        all_tickets <- list_tickets(version)   // 單次載入（C3 斷言點）
    catch loader_error:
        stderr_print("Failed to load ticket index: {root cause}")
        return 1

    scoped <- _filter_by_wave(all_tickets, args.wave)
    handoff_info <- _get_pending_handoff_info()

    in_progress <- load_in_progress(scoped)
    ready <- load_top_ready(scoped, top=args.top, handoff_info)
    stale <- load_stale_warning(scoped, threshold_min=args.stale_threshold) if not args.no_stale else NONE_SENTINEL

    if args.format == "json":
        print(render_json(version, args.wave, in_progress, ready, stale,
                          stale_threshold=args.stale_threshold,
                          stale_disabled=args.no_stale))
    else:
        print(render_text(version, args.wave, in_progress, ready, stale,
                          top=args.top,
                          stale_threshold=args.stale_threshold,
                          stale_disabled=args.no_stale))
    return 0
```

##### 2.2 load_in_progress

```
function load_in_progress(tickets):
    result <- []
    for t in tickets:
        if t.status == "in_progress":
            result.append({id, title, started_at, agent: t.who.current})
    sort result by started_at ascending  // 較早開始的排前面
    return result
```

##### 2.3 load_top_ready

```
function load_top_ready(tickets, top, handoff_info):
    candidates <- []
    for t in tickets:
        if _is_unblocked_pending(t):
            readiness <- _compute_readiness(t, handoff_info)
            if readiness == "ready":
                candidates.append((t, readiness))
    sort candidates by (_priority_rank(t), t.id)  // id 為穩定 tiebreaker
    limited <- candidates[0:top] if top > 0 else []
    return [{id, title, priority, readiness} for (t, readiness) in limited]
    // 不在資料層注入 index；index 由渲染層 enumerate 產生
```

##### 2.4 load_stale_warning

```
function load_stale_warning(tickets, threshold_min):
    now <- current_time()
    result <- []
    for t in tickets:
        if t.status == "in_progress":
            stale_min <- compute_stale_minutes(t, now)
            if stale_min >= threshold_min:
                result.append({id, stale_minutes: stale_min, status, agent})
    sort result by stale_minutes descending
    return result
```

##### 2.5 render_text（含 [N] 編號注入）

```
function render_text(version, wave, in_progress, ready, stale, top, stale_threshold, stale_disabled):
    lines <- []
    lines.append("=== Dashboard (wave={wave}, version={version}) ===")
    lines.append("")

    // --- In Progress 區塊（不編號）---
    lines.append("[In Progress] {len(in_progress)} ticket(s)")
    if in_progress is empty:
        lines.append("  (none)")
    else:
        for item in in_progress:
            lines.append("  - {id}  {title}  (started_at: {ts}, agent: {agent})")
    lines.append("")

    // --- Ready Top N 區塊（唯一注入 [N] 編號處）---
    lines.append("[Ready Top {top}]  priority 排序，可直接 claim")
    if ready is empty:
        lines.append("  (none)")
    else:
        for index, item in enumerate(ready, start=1):
            lines.append("  [{index}] [{priority}] [{readiness}] {id}  {title}")
    lines.append("")

    // --- Stale 區塊（可被 --no-stale 完全略過）---
    if not stale_disabled:
        lines.append("[Stale Warning] {len(stale)} ticket(s) over {stale_threshold}min")
        if stale is empty:
            lines.append("  (none)")
        else:
            for item in stale:
                lines.append("  - {id}  stale={stale_minutes}m  status={status}  agent={agent}")
        lines.append("")

    lines.append("Hint: ticket track claim <id>")
    return join(lines, newline)
```

##### 2.6 render_json

```
function render_json(version, wave, in_progress, ready, stale, stale_threshold, stale_disabled):
    payload <- {
        "version": version,
        "wave": wave,
        "in_progress": [list of dict 原樣],
        "ready": [
            {"index": i+1, "id":..., "priority":..., "readiness":..., "title":...}
            for i, item in enumerate(ready)
        ],
        "stale": None if stale_disabled else [list of dict 原樣],
        "stale_threshold_minutes": stale_threshold
    }
    return json.dumps(payload, ensure_ascii=False, indent=2)
```

| `--no-stale` | flag | False（預設顯示 stale 章節） | 隱藏 stale warning 區塊 |
| `--stale-threshold` | int (分鐘) | `60` | stale 判定門檻；複用 `stale-list` 既有預設值 |
| `--format` | enum | `text` | `text`（PM 預設）或 `json`（機器消費，給 hook / 自動化） |

#### 1.3 輸出格式（text，三章節結構）

```
=== Dashboard (wave={W}, version={V}) ===

[In Progress] {count} ticket(s)
  - {id}  {title}  (started_at: {ts}, agent: {who.current})
  - ...

[Ready Top {N}]  priority 排序，可直接 claim
  [1] [P0] [ready] {id}  {title}
  [2] [P1] [ready] {id}  {title}
  ...

[Stale Warning] {count} ticket(s) over {threshold}min   (用 --no-stale 隱藏)
  - {id}  stale={mins}m  status={status}  agent={who.current}
  - ...

Hint: ticket track claim <id>     # 從上方 [N] 編號直接取 id
```

#### 1.4 `[N]` 編號規則

| 規則 | 說明 |
|------|------|
| 編號範圍 | 僅 **Ready Top N** 章節使用 `[1]..[N]`，從 1 起算 |
| 編號穩定性 | 同次命令輸出內穩定；不跨命令保持（每次重新從 1 編） |
| In Progress / Stale 不編號 | 該兩章節項目用 `-` bullet，不混淆 claim 語意 |
| 編號與 id 對應 | PM 必須以 `id` 而非 `[N]` 傳給 `claim`（編號僅是視覺輔助，避免 PM 拼錯複雜 id） |
| 空章節處理 | 三章節各自於 count=0 時印「(none)」一行，保留章節標題（讓輸出結構穩定） |

### 3. 與既有命令的關係（複用策略）

| 既有命令 | 複用方式 | 不複用的部分 |
|---------|---------|-------------|
| `track_runqueue.py` | 直接 import `_priority_rank`、`_is_unblocked_pending`、`_filter_by_wave`、`_compute_readiness`、`_apply_context_resume` 等內部函式；複用 ready 篩選 + 排序邏輯 | 不複用 `_render_list` / `_render_dag` / `_render_critical_path`（dashboard 自有輸出格式） |
| `track.py`（list） | 不直接呼叫子進程；複用其 ticket loading 入口（共用 ticket index loader） | 不複用 `--status` filter 流程；dashboard 自行對 in_progress / pending 分流 |
| `track_stale_list.py` | 直接 import stale 判定函式（`_is_stale` / `_compute_stale_minutes`）+ threshold 預設值 | 不複用 stdout 渲染；dashboard 自有 stale 區塊格式 |

**關鍵原則**：dashboard 是 **聚合 view**，內部以 Python function call 複用，**禁止以 subprocess 呼叫既有 CLI**。理由：(1) 避免重複載入 Python interpreter；(2) 避免重複解析 ticket YAML；(3) 單一 ticket index 載入即可完成三章節計算。

```text
策略：函式層 monkeypatch，非 process 層攔截
```

| 步驟 | 動作 |
|------|------|
| 1 | `monkeypatch.setattr("subprocess.run", spy_run)` 其中 `spy_run = MagicMock(side_effect=AssertionError("subprocess.run called"))` |
| 2 | 同步替換 `subprocess.Popen` / `subprocess.check_output` / `subprocess.check_call` 全部為 raise spy |
| 3 | 呼叫 dashboard 主入口（CliRunner 或直接 invoke function） |
| 4 | 若 spy 被觸發 → 測試 fail with `subprocess.run called`；未觸發 → 通過 |
| 5 | 另注入 `ticket_loader_spy = MagicMock(wraps=real_loader)`，斷言 `call_count == 1`（C3） |

**替代方案考量**：不採用 `subprocess._mswindows`-level patch 或 `os.fork` hook，函式層 spy 已覆蓋 Python stdlib 所有 subprocess 入口，符合 sociable unit test 原則（只 mock 外部依賴）。
