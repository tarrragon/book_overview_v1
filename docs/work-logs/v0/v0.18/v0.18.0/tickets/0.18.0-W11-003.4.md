# Execution Log

## Task Summary

修復 hook_utils YAML parser 頂層列表解析（children/blockedBy/spawned_tickets）

---

## Problem Analysis

### WRAP 三問

- W（Widen）：A 修 _parse_yaml_lines 支援頂層 list（本方案）；B 改用 PyYAML 依賴（加重依賴，否決）；C 每個 checker 各自實作 list parser（重複，否決）。
- A（Attain distance）：機會成本為延遲 W11-002.x 重構，但 parser bug 影響多 hooks 正確性，根因性優先。
- P（Prepare to be wrong）：最可能失敗 = flow-style `children: [a, b]` 與 block-style `children:\n- a` 兩種 YAML 格式處理不全。防護：AC4 明定兩種格式測試。
### 問題根因

`_parse_yaml_lines` 僅處理「2 空白縮排」路徑的嵌套行，且該路徑中的列表項目邏輯（`_parse_nested_line` 路徑 2）要求 `current_nested_key` 已設定——此條件只有在先解析過嵌套 dict 鍵時才成立。因此兩種頂層列表格式都漏接：
1. 無縮排 block-style（`children:\n- id`）：直接跳過所有未縮排、無冒號的列表行
2. 2 空白縮排 block-style（`children:\n  - id`）：`_parse_nested_line` 路徑 2 因 `current_nested_key is None` 而 fallthrough
3. Flow-style（`children: [a, b]`）：`_parse_top_level_pair` 只 strip 引號，方括號被保留為字串

三種情況都導致 `result['children']` 為空字串 `''`。

### 影響範圍

- `.claude/hooks/hook_utils/hook_ticket.py`（parser 本體）
- `.claude/hooks/acceptance_checkers/children_checker.py`（可移除 fallback）
- 所有讀取 `children/blockedBy/relatedTo/spawned_tickets` 的 hook，之前只有 children_checker 有 fallback，其他欄位實質失效

### 相關 Error Pattern

無直接對應 error-pattern；根因是 parser 設計時未考慮頂層列表，屬於功能缺漏而非行為錯誤。

## Solution

**修改檔案**：
1. `.claude/hooks/hook_utils/hook_ticket.py`
   - `_parse_yaml_lines`：新增「頂層 block-style 列表項目」處理分支（無縮排 `- item` 與 2 空白縮排 `  - item` 兩種），累積到 `result[current_key]` 為 `list[str]`。2 空白縮排分支加守衛：僅在 `current_nested_key is None` 且 `result[current_key]` 非 dict 時觸發，避免吃掉嵌套 dict 內的列表。
   - `_parse_top_level_pair`：新增 flow-style 列表處理（`key: [a, b]` / `key: []`），切割逗號並去除引號後回傳 `list[str]`。
2. `.claude/hooks/acceptance_checkers/children_checker.py`
   - 移除 `_read_children_from_file` 與 `re` import。
   - `_extract_children_robust` 簡化為直接呼叫 `extract_children_from_frontmatter`，保留函式簽名供呼叫端向後相容。
3. `.claude/hooks/tests/test_yaml_toplevel_list.py`（新增）
   - 12 個測試覆蓋 block-style（無縮排/2 空白縮排）、flow-style（含空列表、帶引號）、混合欄位（children/blockedBy/relatedTo/spawned_tickets）、列表後接嵌套 dict、端到端整合。

**設計決策**：
- 保留「空值 `key:` 初始化為空字串」的舊行為，僅在後續偵測到 block-style 列表時升級為 list，避免破壞既有呼叫端對「無值 key」的字串假設。
- `extract_children_from_frontmatter` 已支援 list/string 雙型別輸入，因此 parser 回傳 list 不破壞既有消費端。
