### 實作摘要（PM 前台執行）

**背景**: 派發 thyme 時 agent-dispatch-validation-hook 要求 worktree isolation，但 ARCH-015 禁止 .claude/ 在 worktree 編輯，兩條規則衝突。用戶決議 PM 前台執行（測試屬規格定義，pm-role.md 允許）。

**新增 5 個測試定義（parametrize 展開後共 8 個測試）**:

1. `test_child_with_non_terminal_status_should_block[blocked|failed|unknown]` — parametrize 覆蓋 3 種非終止狀態
2. `test_circular_reference_does_not_infinite_loop` — A→B→A 循環，visited 機制阻止重訪
3. `test_circular_reference_with_incomplete_descendant_still_reports` — 循環中未完成節點仍被報告
4. `test_multi_branch_tree_reports_correct_pending_descendant` — 多分支只報孫而非兄弟本身
5. `test_fallback_parser_handles_block_style_children_list` — block-style YAML 列表走 fallback
6. `test_fallback_parser_empty_children_returns_no_descendants` — 無 children 欄位 fallback 返回空 → pass

**設計取捨**:
- 新增 `_write_raw_ticket` helper（僅限新測試使用），繞過原 `_write_ticket` 的 inline `[]` 限制，同時保留原 helper 介面供既有 7 測試不變
- parametrize 合併 blocked/failed/unknown 為單一測試定義，3 倍覆蓋僅需 1 個函式

**Hook/ARCH-015 衝突**: 需後續建新 Ticket 追蹤 agent-dispatch-validation-hook 應對 .claude/ 路徑豁免 worktree 要求（規則 5 + 規則 6：框架優先）。
