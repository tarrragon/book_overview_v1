## Problem Analysis

### 問題根因

既有 Hook 文件停留在較早事件模型，主要涵蓋 `PreToolUse`、`PostToolUse`、`Notification`、`Stop` 等既有流程，未同步官方 hooks 文件中已出現的 session、permission、subagent、task、config/file/worktree、compact、elicitation 與 failure lifecycle 事件。

文件也缺少 handler schema 的治理規則，尤其是 HTTP handler、`if` 條件欄位與 prompt/agent handler 的使用邊界。若 PM 或 hook architect 只依舊文件設計 hook，會把事件時機、handler 類型與條件過濾混在腳本內處理。

### 影響範圍

- `.claude/hook-system-reference.md`
- `.claude/pm-rules/hook-governance.md`
- `.claude/methodologies/hook-system-methodology.md`
- `.claude/references/hook-architect-technical-reference.md`

### 相關 Error Pattern

已檢查既有 error-pattern，這次不新增新 pattern：

- `ARCH-019`：代理人完成相關 hook 必須用 `SubagentStop`，避免用 `PostToolUse(Agent)` 造成時機錯位。本次在 reference 內保留並擴充事件索引。
- `DOC-010` 類型教訓：清單型文件必須逐項核對，不可只補概述。本次以官方 event list 對照補齊事件與觸發時機。
- 任務草稿內提到的 `Setup` 未出現在官方 hook event list；本次改以「名稱校準」記錄，要求依實際需求使用 `SessionStart`、`InstructionsLoaded`、`ConfigChange`、`CwdChanged` 或 `FileChanged`。

## Solution

### WRAP 決策

**Widen options**：

- A. 在既有 4 份 Hook 文件內補齊事件、handler schema、HTTP 與 `if` 條件。
- B. 另建一份新文件集中描述新能力，再從既有文件連結。

**Reality-test assumptions**：

- 官方 hooks 文件是 event name 與 handler schema 的來源；任務描述中的版本清單只作為提示，不作為權威來源。
- `Setup` 不應直接寫成官方事件，避免框架文件記錄不存在的 event name。
- 通用框架文件不得留下本 ticket ID 或 worklog 路徑；ticket 內容只留在工作日誌。

**Attain distance**：

採 A。Hook reference、governance、methodology、technical reference 已各有不同讀者與用途，直接補在現有文件可降低讀者跳轉成本，也避免多一份獨立文件成為第二來源。

**Prepare to be wrong**：

補上「名稱校準」與外部規格來源，後續若官方 hooks 文件更新，可以先從 `.claude/hook-system-reference.md` 的 event overview 與 `.claude/references/hook-architect-technical-reference.md` 的 schema section 對照更新。

### 實作內容

- 更新 `.claude/hook-system-reference.md`：加入外部規格來源、Hook 能力導覽、完整事件總覽、matcher/`if` 規則與 `Setup` 名稱校準。
- 更新 `.claude/pm-rules/hook-governance.md`：加入 handler type 治理、HTTP hook 規則、`if` 條件規則與三個條件範例。
- 更新 `.claude/methodologies/hook-system-methodology.md`：擴充 lifecycle 圖、event 選擇流程、handler 選擇與 `if` 使用方法。
- 更新 `.claude/references/hook-architect-technical-reference.md`：加入事件快速索引、新事件技術說明、handler schema、HTTP POST JSON 範例與 prompt/agent handler 邊界。
- 移除通用 Hook 文件中的具體 worklog 路徑範例，改用 `<worklog-path>` placeholder。
