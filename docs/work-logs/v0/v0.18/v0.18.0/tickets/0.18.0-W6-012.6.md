### 問題根因

CSV 匯出僅 5 欄是 **設計選擇 + 規格未對齊 + 架構分裂** 三重因素共振：

1. **架構分裂（核心根因）**：專案存在兩條 CSV 路徑：
   - `src/overview/book-exporter.js`（Overview UI 實際使用，5 欄精簡版）
   - `src/export/book-data-exporter.js`（含 v2 field presets COMPLETE_V2，但**未接線到 Overview 按鈕**）
   - `overview-page-controller.js:241` 點擊 exportCSVBtn 走的是前者，後者形同孤立

2. **設計選擇（部分合理）**：5 欄裁減符合 UC-03「CSV 適合 Excel 處理」的人類可讀導向（書名/書城來源/進度/狀態/封面URL 是 spreadsheet 友善摘要）

3. **規格未對齊（v2.0+ 缺口）**：use-cases.md L180-182 明訂 v2.0+ CSV 必須含 tag 欄位、6 種閱讀狀態，但 overview/book-exporter.js CSV_HEADERS 未更新

### 影響範圍

- `src/overview/book-exporter.js` L17 CSV_HEADERS / L182-190 _bookToCSVRow（生產端）
- `src/export/book-data-exporter.js`（孤立的完整版實作，未接線）
- `src/overview/book-file-importer.js` L92（**只接受 application/json，CSV 無法 round-trip 匯入** — AC3 由 importer 設計阻斷，與本 ticket 範圍正交）
- `docs/use-cases.md` UC-03（v2.0+ 規格與 v0.18 實作落差）

### Schema 完整欄位（來源：readmoo-adapter.js:541-557）

每本 book 物件含：`id, title, authors, cover, progress, readingStatus, type, tagIds, tags, identifiers{privacyBookId,...}, progressInfo{...}, isManualStatus, extractedAt, updatedAt, source`

當前 CSV 缺：`id, tagIds/tags, identifiers, progressInfo, authors, type, extractedAt`

### 相關 Error Pattern

- 模組組裝遺漏模式（feedback_module_assembly_check）：book-data-exporter.js 完整實作但未接線到 UI
- 規格未對齊：v2.0+ 規格寫了但 v0.18 實作未跟進（v0.17 tag-based model 重構的尾巴）
