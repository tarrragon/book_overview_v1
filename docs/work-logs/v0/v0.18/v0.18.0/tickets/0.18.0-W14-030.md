## Phase B IMP-3 實作指引（依 W14-024 結論長期補強）

### 目標
新增 2 個 ticket CLI 子命令，提供 subagent 正確路徑，長期消除直接 Edit frontmatter 的動機。

### 子命令規格

**1. `ticket track set-acceptance`**
```bash
ticket track set-acceptance <ticket-id> --check <index> [<index>...]
ticket track set-acceptance <ticket-id> --uncheck <index> [<index>...]
ticket track set-acceptance <ticket-id> --all-check
ticket track set-acceptance <ticket-id> --all-uncheck
```

與既有 `check-acceptance` 差異：
- 更明確語意（check / uncheck 雙向）
- 可考慮 deprecate `check-acceptance` 或別名化

**2. `ticket track validate <ticket-id>`**
呼叫與 W14-028 hook 相同的驗證邏輯（共用模組），回傳：
- 合規欄位清單
- 違規欄位與詳情
- 建議修復命令

**共用驗證模組**（與 W14-028 blocker）：
- 建議將驗證邏輯抽到 `ticket_system/validators/frontmatter_validator.py`
- Hook 和 CLI 都 import 使用
- 測試覆蓋 4 欄位的合規/違規案例

### 實作步驟

1. 等 W14-028 完成（blocker）→ 取得共用 validator 模組
2. 新增 `ticket_system/commands/track_set_acceptance.py`
3. 新增 `ticket_system/commands/track_validate.py`
4. 註冊到 `ticket_system/cli.py`
5. 更新 SKILL.md 命令清單 + references/track-command.md
6. 執行 `uv tool install .claude/skills/ticket --reinstall`
7. 手動驗證：建立測試 ticket → set-acceptance --check 1 → validate → 檢查結果

### 驗收條件對應
- AC 1-2: 兩個子命令 → 新增命令檔
- AC 3-4: 文件同步 → SKILL.md + references
- AC 5: uv install → 全局可用
