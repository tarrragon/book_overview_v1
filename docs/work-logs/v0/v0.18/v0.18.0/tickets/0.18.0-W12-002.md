### 污染源調查結果（按優先級）

#### Top 1：`.claude/hooks/` 19 個 Hook 檔案含 emoji — 主要系統性污染源

- **影響範圍**：Hook 執行時 stdout/stderr 直接注入 PM context
- **觸發頻率**：幾乎每個 tool call 都會觸發 1+ Hook
- **累積效應**：長 session 下 PM context 充滿 emoji，token 分佈偏向含 emoji 樣式
- **命中檔案**：worklog-format-check、ticket-quality-gate-hook、ticket-id-validator-hook、tech-debt-reminder、l10n-sync-verification-hook、dispatch-count-guard-hook、comment-qa-hook、command-entrance-gate-hook 等 19 個 + archived/ 6 個
- **因果鏈**：Hook 用 `[OK] ✅` `[WARN] ⚠️` `[FAIL] ❌` 作輸出標記 → 每 session 累積數十次 → PM 生成 AUQ payload 時 token pool 已被 emoji 樣式污染 → ⚡ 等相鄰 emoji 誤植

#### Top 2：`.claude/rules/core/language-constraints.md` 含 6 個 emoji（L43-49）

- **影響範圍**：自動載入規則區，每 session 起點即進 context
- **角色**：作為「禁用範例」列在 table 的「禁止」column
- **諷刺點**：教導禁 emoji 的規則本身含 emoji
- **命中 emoji**：🎯 ✅ ❌ 🔴 🟢

#### Top 3：`.claude/hooks/command-entrance-gate-hook.py` L138 含簡體「没错」

- **影響範圍**：Hook 常數列表（SHORT_ANSWER_PATTERNS）
- **角色**：刻意納入簡體變體以容錯用戶輸入
- **判定**：設計需求，不列為污染源修復目標（但需在防護規則中標記）

#### Top 4：`.claude/skills/` 多個 Skill 的 stdout/範例含 emoji

- `.claude/skills/version-release/` 7 個檔（QUICK_START/BUILD_SUMMARY/TECH_DEBT_GUIDE 等）
- `.claude/skills/ticket/ticket_system/lib/` py 檔（constants, command_tracking_messages）
- `.claude/skills/compositional-writing/references/writing-documents.md`
- **影響**：僅在對應 Skill 活動時污染，頻率低於 Hook

#### Top 5：`.claude/templates/` 範本含 emoji

- work-log-template.md 和 ticket-log-template.md
- **影響**：只在模板複製時

### 根因結論

**主根因**：Hook 系統普遍使用 emoji 作為 stdout 狀態標記，每個 tool call 後 Hook 輸出注入 PM context，累積下來污染 token pool，導致 AUQ payload 生成時相鄰 emoji（如 ⚡）被誤選。

**次根因**：自動載入區（language-constraints.md）含 emoji 範例，session 啟動即進 context baseline。

**無法修復的根因**：Claude base model 的 zh-CN/zh-TW token space 相鄰（用戶的簡體字污染主要來源於此，command-entrance-gate-hook 的貢獻很小）。

### 已驗證排除的假設

- CLAUDE.md：無 emoji（Grep 0 命中）
- auto-memory（除本 session 新增的 PC-072 配對檔）：無簡體字明顯污染
- `.claude/rules/` 其他檔案：無 emoji（除 language-constraints.md 外）

### 防護策略

1. **Top 1 修復（IMP P1）**：清洗 .claude/hooks/ 19 個檔案的 emoji stdout，改用 ASCII `[OK]`/`[WARN]`/`[FAIL]`
2. **Top 2 修復（IMP P1）**：清洗 language-constraints.md 的 6 個 emoji 範例，改為 unicode escape 或文字描述
3. **Top 3 不修**：command-entrance-gate-hook 的「没错」屬設計需求
4. **Top 4/5 修復（IMP P2）**：清洗 skills/ 和 templates/ 的 emoji
5. **Hook 攔截層（IMP P0）**：PreToolUse Hook 掃描 AskUserQuestion payload，偵測簡體字與 emoji 時 block 並提示 PM 修正
6. **PM 自檢（已落實）**：PC-072 檢查清單已涵蓋
