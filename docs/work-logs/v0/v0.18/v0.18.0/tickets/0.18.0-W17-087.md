## Problem Analysis

### 問題根因

W17-050 ANA 設計 basil-writing-critic 時，因平台限制「代理人無法載入 SKILL」這一前提（W17-050 §一設計脈絡），W17-056 / W17-067 落地時將 `document-writing-style.md` 三明示原則、隱含表達 6 句型表、compositional-writing 五大原則等規範**手抄成文字摘要**寫入 agent 主文（共約 270 行內嵌規則）。

但實際 CC platform 對 agent 主文的 `@path` markdown import 是支援的（如既有 `@.claude/agents/AGENT_PRELOAD.md` 的 auto-load 機制），W17-050 設計時錯估了平台能力。手抄方式造成：

1. DRY 違反：規範更新時 agent 需手動同步（compositional-writing 已有 12 份 references，none 被 agent 引用）
2. 規範深度不足：agent 看到的是「v1.2.0 摘要」這層，無法用 writing-articles.md 的「層次意識」「五輪 review checklist」等深度規則
3. 維護摩擦：規範與執行者脫鉤，後續對 document-writing-style 的修改不會自動反映到 basil

### 影響範圍

- `.claude/agents/basil-writing-critic.md`（手抄規則約 270 行）
- 間接影響：未來新增類似常駐審查代理人時會延續錯誤模式

### 相關 Error Pattern

無直接 PC，但屬 DRY 違反（compositional-writing SKILL 與 agent 內容重複）。

## Solution

### 變更摘要

`.claude/agents/basil-writing-critic.md` 從 311 行降至 236 行；移除手抄規則摘要（隱含表達 6 句型表、三大職責詳細執行步驟），改用 7 個 `@-import` auto-load 規範來源：

| @-import 路徑 | 提供內容 |
|--------------|---------|
| `@.claude/agents/AGENT_PRELOAD.md` | 既有共用 preamble |
| `@.claude/rules/core/document-writing-style.md` | 三明示原則、隱含表達 6 句型、二次審查清單 |
| `@.claude/rules/core/language-constraints.md` | 禁用詞、字元集規範 |
| `@.claude/skills/compositional-writing/SKILL.md` | 五大原則速查 + references 索引 |
| `@.claude/skills/compositional-writing/references/writing-documents.md` | 文件審查情境特化指南 |
| `@.claude/skills/compositional-writing/references/writing-articles.md` | 文章審查情境特化指南（含層次意識、五輪 review） |
| `@.claude/skills/compositional-writing/references/writing-code-comments.md` | 註解審查情境特化指南 |

### 保留內容

- frontmatter（name / description / tools / color / model / effort）
- 角色定位 + 命名決策說明（basil 前綴群組）
- 三區塊（允許產出 / 禁止行為 / 適用情境）
- 核心職責摘要（從 detailed steps 改為三大職責對照表，細節指向 @-import 規範）
- Hook 層化（L1/L2/L3）
- 審查報告輸出格式（agent 專屬）
- 與其他代理人的邊界（agent 專屬）
- 升級機制 / 成功指標 / 二次審查紀錄

### 設計決策

1. agent 主文不重複規範細節，只定義「行為邊界 + 輸出格式」
2. 規範細節由 `@-import` 在 agent runtime 自動展開
3. 規則更新只需改規範來源（document-writing-style.md / SKILL.md），agent 自動同步
4. 三大職責章節保留對照表（讓 agent 一眼看到審查重心），但詳細執行步驟移至規範來源
