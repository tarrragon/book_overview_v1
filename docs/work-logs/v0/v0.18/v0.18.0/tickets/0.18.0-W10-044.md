## Problem Analysis

### 問題根因

- Hook test 檔案可以用 PEP 723 metadata 宣告 dependencies，但 `pytest` collection 不會自動讀取每個測試檔的 inline script metadata。
- PM / agent 若直接執行裸 `pytest` 或只用 `uv run --with pytest python -m pytest ...`，會漏掉 `pyyaml` 等檔頭依賴，產生 false-positive `ModuleNotFoundError`。

### 影響範圍

- `.claude/hooks/tests/*.py` 的測試執行方式。
- PM / agent 在處理 hook 相關 ticket 時的 Test Results 判讀。
- Codex 專案入口對 Python hook test 的測試命令選擇。

### 相關 Error Pattern

- 相關：PC-005 CLI failure assumption attribution。測試命令失敗時，不能直接歸因為測試壞掉；需先檢查執行環境與依賴模型。
- WRAP 三問簡記：候選方案為建立 README SOP / 實作 conftest 自動解析 PEP 723。選擇 README SOP，因 pytest collection 讀取任意測試檔 metadata 會引入非標準機制且不必要。機會成本是暫不做自動化依賴聚合。失敗風險是文件不可見，已同步更新 CODEX.md、hooks README 與 methodology。

## Solution

- 掃描 `.claude/hooks/tests/*.py`，確認目前含 PEP 723 dependencies 的檔案：
  - `test_agent_dispatch_check.py`：`pytest`, `pyyaml`
  - `test_frontmatter_parser.py`：`pytest>=7.0`, `pyyaml>=6.0`
  - `test_ticket_tracker.py`：`pytest`
- 新增 `.claude/hooks/tests/README.md`，定義：
  - 偵測 inline dependencies 的 `rg` 指令。
  - PEP 723 單檔測試：`uv run .claude/hooks/tests/<test-file>.py`
  - 普通 pytest 檔：`uv run --with pytest python -m pytest <test-file> -v`
  - 針對 PEP 723 檔做 node selection 時，需把 dependencies 轉成 `--with` 參數。
  - mixed full suite 無單一自動合併 metadata 命令；需直接跑 PEP 723 檔或明確傳 union dependencies。
- 更新 `CODEX.md`、`.claude/hooks/README.md`、`.claude/methodologies/hook-system-methodology.md`，指向新 SOP。
- conftest 自動解析 PEP 723 的結論：暫不做。這會偏離標準 pytest/uv 模型，且 full suite 可以用 union dependencies 或分批執行處理。
