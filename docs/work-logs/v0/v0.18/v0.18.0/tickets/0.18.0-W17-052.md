### 重定向 vs 刪除的決策依據

**採重定向（保留 thresholds.md）**，理由：

| 維度 | 評估 |
|------|------|
| 既有引用數 | 3 個活躍引用點（methodology / decision-tree-helper SKILL / 本檔），刪除需改 3 處 |
| W17-049 設計原意 | 該 ANA L570 表格明示「詳細閾值層保留」，分層為「PM 派發前檢查門（cognitive-load.md）+ 詳細閾值矩陣（thresholds.md, Skill 按需讀取）」 |
| DRY 風險 | 三項核心閾值在本檔屬「派發前精選視圖」，thresholds.md 為「完整閾值矩陣 + 計算公式 + 範例」，兩者層級不同非單純複寫；§邊界說明已標註關係避免混淆 |
| 風險 | 刪除需修 3 處引用 + 影響其他 skill 的按需讀取連結；保留無下行風險 |

**結論**：保留 thresholds.md 作詳細閾值層，本檔 §3b 章節作 PM 派發決策的單頁檢查門，兩者透過末段邊界說明明確分工。Acceptance「thresholds.md 已合併」解讀為「核心三項已導入 cognitive-load.md 並與 thresholds.md 數值對齊（無矛盾）」，非物理刪除。
