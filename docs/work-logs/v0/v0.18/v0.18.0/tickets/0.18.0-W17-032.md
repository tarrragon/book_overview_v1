### 問題根因

`_is_placeholder` 函式於 `.claude/skills/ticket/ticket_system/lib/ticket_validator.py:339-341` 對 HTML 註解使用過於寬鬆的判斷：


### 修復選擇：選項 C（剝除 HTML 註解後判斷剩餘實質內容）

**檔案**：`.claude/skills/ticket/ticket_system/lib/ticket_validator.py` L312-370 `_is_placeholder`

**核心變更**：

在所有 placeholder 判斷前，先用 `re.sub(r"<!--.*?-->", "", stripped, flags=re.DOTALL)` 剝除所有 HTML 註解，再基於 `content_no_html` 判斷：

1. 若剝除後為空 → placeholder（保留原行為，例如純 HTML 註解）
2. 若剝除後含 `(pending)/TBD/TODO/N/A` → placeholder
3. 若剝除後含 `（待填寫：...）/（必填：...）`或全由中文佔位符組成 → placeholder
4. 否則 → 非 placeholder（視為已有實質內容）

### 為何選 C

- 通用：一次解決「Schema 註解 + 任意其他 HTML 註解」誤判
- 維護成本低：無需 whitelist 維護已知 Schema 註解格式
- 保留既有行為：純 HTML 註解、中英文佔位符判斷邏輯皆不變
- 不影響 section extraction 設計：修正僅限判斷函式，不動 section 切分

**問題**：regex `<!--.*?-->` 匹配任何 HTML 註解，導致 Schema 標註註解（設計上固定存在、作為填寫指引）誤判為佔位符。

### 修復方向（TDD 候選）

**選項 A：whitelist Schema 註解格式**
```python
# 排除 Schema 標註註解（範本固定存在，非佔位符）
stripped_no_schema = re.sub(r"<!--\s*Schema\[[^\]]+\]:[^>]*-->", "", stripped).strip()
# 再檢查剩餘部分
if re.search(r"<!--.*?-->", stripped_no_schema, re.DOTALL) and not _has_substantial_content(stripped_no_schema):
    return True
```

**選項 B：縮緊佔位符 keyword 檢查**
```python
# 只認「明確佔位符關鍵字」的 HTML 註解
PLACEHOLDER_COMMENT_PATTERNS = [
    r"<!--\s*To be filled.*?-->",
    r"<!--\s*TODO:.*?-->",
    r"<!--\s*待填寫.*?-->",
    # Schema 標註註解 不在此清單
]
```

**選項 C：移除 HTML 註解後檢查剩餘實質內容**
```python
# 先剝除所有 HTML 註解，再判斷剩餘是否為空/佔位符
content_no_html = re.sub(r"<!--.*?-->", "", stripped, flags=re.DOTALL).strip()
if not content_no_html:
    return True  # 只剩 HTML 註解 → 佔位符
# 對 content_no_html 繼續做中文/英文 placeholder 檢查
```

**推薦選項 C**：最通用，一次解決「Schema 註解 + 任意其他 HTML 註解」的誤判問題，不用 whitelist 維護。
