## Problem Analysis

### 衝突根因分析

**Hook 設計**（agent-dispatch-validation-hook.py line 24-31）：
```python
IMPLEMENTATION_AGENTS = frozenset({
    "parsley-flutter-developer", "fennel-go-developer",
    "thyme-python-developer", "cinnamon-refactor-owl",
    "pepper-test-implementer", "mint-format-specialist",
})
```
6 個實作代理人一律強制 `isolation: "worktree"`，目的是防止 .git/HEAD 污染。**沒有**路徑檢測邏輯。

**ARCH-015 規則**（subagent .claude/ 寫入 hardcoded 保護）：
- CC runtime 硬編碼拒絕 worktree subagent Edit `.claude/` 路徑
- `permissionMode`、`additionalDirectories` 均無法繞過
- 規範要求：`.claude/` 修改必須在主 repo cwd 派發（PM 前台或主 repo subagent）

**衝突矩陣**：

| 情境 | Hook 要求 | ARCH-015 要求 | 結果 |
|------|----------|--------------|------|
| thyme 改 `.claude/hooks/` | 必用 worktree | 禁用 worktree | **硬衝突** |
| thyme 改 worktree 內 src/ | 必用 worktree | 無限制 | OK |
| parsley 改 src/ widget | 必用 worktree | 無限制 | OK |

**本專案影響放大**：專案是 Chrome Extension（JavaScript），thyme 幾乎**只**處理 `.claude/hooks/` 的 Python 腳本。衝突頻率 ≈ 100% 的 thyme 派發。

### 三方案 Trade-off

| 方案 | 摩擦力 | 維護成本 | 擴展性 | .git/HEAD 保護 |
|------|-------|---------|-------|--------------|
| **A: Hook 路徑檢測豁免** | 低（自動） | 中（pattern 維護） | 高（新情境自動適配） | 保留（非 `.claude/` 任務仍強制） |
| B: thyme 移出強制清單 | 無 | 低（刪一行） | 低（只救 thyme） | 失去（thyme 全域失去保護） |
| C: 新建 `.claude-thyme` agent | 中（派發選擇） | 高（雙份定義） | 低 | 保留 |

### 方案 A 詳細設計（推薦）

在 agent-dispatch-validation-hook.py 加入 prompt 掃描邏輯：

```python
def _classify_prompt_paths(prompt: str) -> tuple[bool, bool]:
    """回傳 (has_claude_path, has_other_path)。"""
    has_claude = bool(re.search(r'\.claude/', prompt))
    # 粗略偵測：非 .claude/ 的 Edit/Write 目標路徑
    other_pattern = re.search(r'(?:Edit|Write)[^\n]{0,50}?(?:src/|tests/|docs/|lib/)', prompt)
    has_other = bool(other_pattern)
    return has_claude, has_other

# 在 main() 內（實作代理人分支內）：
has_claude, has_other = _classify_prompt_paths(prompt)
if has_claude and not has_other:
    logger.info("放行：%s 僅修改 .claude/（ARCH-015 豁免）", subagent_type)
    return 0
if has_claude and has_other:
    print("錯誤：prompt 跨 .claude/ 與非 .claude/ 路徑，請拆分為兩次派發（ARCH-015）",
          file=sys.stderr)
    return 2
# 僅非 .claude/ → 原強制 worktree 邏輯
```

**決策表**：

| Prompt 路徑分類 | Hook 行為 | 派發位置 |
|---------------|----------|---------|
| 僅 `.claude/` | 放行（豁免 worktree） | 主 repo cwd |
| 僅非 `.claude/` | 強制 worktree | worktree |
| 跨路徑 | 阻擋並要求拆分 | - |

### 建議

**採方案 A**。理由：
1. 符合 ARCH-015 既有派發決策表（worktree-operations.md line 131-133）
2. 保留 Hook 對非 `.claude/` 任務的 .git/HEAD 保護
3. 自動處理未來新增的實作代理人（pattern 共用）
4. 零代理人定義變動

**後續 IMP Ticket**：
- W10-042（待建）：實作方案 A — 擴充 agent-dispatch-validation-hook 加入 prompt 路徑分類和豁免邏輯
- 驗收：thyme 派發 prompt 含 .claude/ 時放行；跨路徑時阻擋；非 .claude/ 仍強制 worktree
- 測試：新增 Hook 單元測試覆蓋三分類決策
