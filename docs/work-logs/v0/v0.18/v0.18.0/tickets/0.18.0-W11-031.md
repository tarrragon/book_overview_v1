## 修復摘要

兩個 path-mismatch bug 串接造成 skill 完全失效：

### Bug 1：裸 import 與 entry point 不一致
- 位置：`scripts/worktree_manager.py:20-31`
- 症狀：`uv run worktree` 拋 `ModuleNotFoundError: No module named 'constants'`
- 根因：pyproject.toml 入口點 `scripts.worktree_manager:main` 將 scripts 視為 package，但模組內用 `from constants import` 裸 import；測試端用 `sys.path.insert(scripts_dir)` + 裸 import 又是另一情境
- 修復：採 try/except dual import（先 relative `.constants/.messages`，fallback 至裸 import），兩種情境都相容，不需動測試

### Bug 2：project_root 推導少算一層
- 位置：`scripts/worktree_manager.py:49`
- 症狀：`get_project_root()` 永遠走 fallback 退化為 `os.getcwd()`，導致從 skill 目錄執行時 worktree 路徑變成 `.claude/skills/worktree-<id>` 而非正確的 `<project_root>-<id>`
- 根因：`Path(__file__).resolve().parent.parent.parent.parent` 取到 `.claude/` 不是 project root；少算一層
- 修復：補一個 `.parent`，加註層級註解防止再犯

### 連帶修復：測試 patch 目標錯誤
- 位置：`tests/test_git_utils_extensions.py:58,67`
- 症狀：Bug 1 修好後 `test_worktree_list_empty` / `test_worktree_list_malformed_output` 失敗
- 根因：原本 git_utils 永遠 import 失敗走 fallback，`get_worktree_list` 在 `worktree_manager` 模組內定義；修復後變成 `from git_utils import get_worktree_list`，patch `worktree_manager.run_git_command` 已碰不到 git_utils 內部
- 修復：patch 目標改為 `git_utils.run_git_command`

## 驗證
