## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 實作摘要

為類別 A 剩餘 6 hook 加入 effort 感知（v2.1.133+，複用 W14-034 hook_utils.get_effort_level helper）。

### 各 hook 改動

| Hook | 行為策略 | 理由 |
|------|---------|------|
| creation-acceptance-gate-hook | low effort 短路放行（emit allow JSON） | 一般 strict-validator |
| ana-ticket-metadata-validation-hook | low effort 短路放行 | 一般 strict-validator |
| ticket-creation-validation-hook | low effort 短路放行（首次接入 hook_utils） | 原未用 hook_utils，新增條件 import 並 fallback 防護 |
| comment-qa-hook | low effort 短路放行 | 一般 strict-validator |
| askuserquestion-charset-guard-hook | **不短路**，僅記錄 effort | PC-074/PC-131 字元集偵測為事實判斷，必擋邏輯與 effort 無關 |
| auq-option-pattern-detector-hook | **不短路**，僅記錄 effort | PC-064 AUQ pattern 偵測為事實判斷 |

### 設計決策

- 字元集 / AUQ pattern 偵測屬「事實判斷」防護，PC-074/PC-131/PC-064 防護完整保留，low effort 不削弱
- 一般 strict-validator 在 low effort 時短路後仍輸出對應 hook event 的 allow JSON（不破壞 hook 契約）
- 各 hook 加 `logger.info` 記錄 effort 與行為決策，便於後續觀測

### 不在本 ticket 範圍

- `ticket-creation-validation-hook` 整體 hook_utils 重構（僅做最小侵入式 effort 感知接入）
- effort=high 時的加嚴邏輯（W14-032 ANA 規劃為後續 ticket）
