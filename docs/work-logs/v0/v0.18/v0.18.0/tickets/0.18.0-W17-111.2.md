## Context Bundle

### 既有 schema 範例（行 33-45）

```yaml
example_reviewed: |
  multi_view_status: reviewed
  reviewers: [linux-reviewer, security-reviewer]
  conclusion: 三層防護 DRY 違反獲確認，建議治本方案 D

example_skipped: |
  multi_view_status: skipped
  reason: 本 ANA 僅彙整既有資料，無新設計決策

example_na: |
  multi_view_status: n_a
  reason: 純文件調查，無程式碼或架構影響
```

### 待加入段落（建議放在 examples 之後）

```yaml
  # 禁用格式（PC-117 / W17-111）
  # PM 易誤用 nested YAML 結構導致 hook regex 跨行誤判：
  forbidden_format: |
    # 錯誤：nested YAML 結構
    multi_view_status:
      status: skipped       # hook 會跨行抓「status: skipped」字串並判定值非法
      reason: ...

    # 正確：flat YAML 結構（與 examples 一致）
    multi_view_status: skipped
    reason: ...

  # 案例：W17-095 收尾踩坑 → PC-117 落地 → W17-111 ANA 評估改善方向
  # 詳見：.claude/error-patterns/process-compliance/PC-117-ana-solution-multi-view-status-nested-yaml-misparse.md
```

### 邊界（嚴守）

- 允許：`.claude/config/ana-solution-schema.yaml` 加 forbidden_format 段落 + 註解
- 禁止：改 examples 既有內容；改 allowed_values / required_subfields；改 hook py（W17-111.1 範圍）
- 禁止：git add . / git add -A
- 驗證：`grep -nE "forbidden_format|PC-117" .claude/config/ana-solution-schema.yaml` 應有 hit
