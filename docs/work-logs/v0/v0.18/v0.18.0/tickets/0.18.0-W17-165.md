### L2-B：SessionStart 提示語改寫

session-start-scheduler-hint-hook.py：
- scheduler 區塊標題從「## 排程提示（scheduler hint）」改為「## 下 session 建議項目（排程提示 / scheduler hint）」。「下 session 建議項目」為主標，「排程提示 / scheduler hint」作副標保留可追蹤性。
- docstring 同步補註 W17-165 L2-B 設計意圖（建議取代恢復，避免暗示中斷）。

### L2-C：Stop hook terminal 過濾

handoff-auto-resume-stop-hook.py：
- 新增模組級 TERMINAL_STATUSES = {"completed", "closed"}（與 ticket_system.constants 對齊）。
- is_ticket_completed 擴展為 `status in TERMINAL_STATUSES`（原僅 `status == 'completed'`），函式名保留以維持向後相容；docstring 補註行為變更。

stop-worklog-handoff-sync-check-hook.py：
- _scan_pending_dir 從純檔名掃描升級為 JSON 解析 + from_ticket terminal 過濾。fail-open：JSON 解析失敗保留原 ID（不誤刪）。
- 既有 TERMINAL_STATUSES 常數複用。

### 設計取捨

- 為何不改 `is_ticket_completed` 函式名為 `is_ticket_terminal`：呼叫端僅關心「該 ticket 是否屬 GC/過濾路徑」，重命名牽動測試與其他文件，超出本 ticket 範圍。
- 為何 _scan_pending_dir 改為讀 JSON：N 通常 < 20，stop event 觸發時的額外讀檔成本可忽略；對齊 ANA 中「即時過濾」需求。
- 向後相容：所有變更為純文字標題 + 過濾擴展（completed → terminal 集合），無 schema 變更，無新欄位。

### 不在本 ticket 範圍

- ticket_system.commands.resume 與 lib.messages 仍有「待恢復任務」字樣（runqueue/CLI 訊息），屬 W17-164 L2-A 或獨立 ticket 範圍。
- handoff-auto-resume-stop-hook.py 內 stderr 訊息（「[Handoff] 偵測到未完成的 handoff 任務」）未涉「待恢復」字面，無需改寫。
