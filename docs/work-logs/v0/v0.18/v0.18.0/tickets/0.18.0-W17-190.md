### 問題根因

`.claude/hooks/lib/frontmatter_parser.py:28` 直接 `import yaml`，並透過 `yaml.safe_load`（line 133、解析 frontmatter）+ `yaml.dump`（line 284、寫回 frontmatter）使用。該模組是 hook 系統共用 YAML 解析入口（15+ hook 直接或間接依賴）。

PC-124 根因：`frontmatter_parser.py` UV script header（line 4-7）宣告 `pyyaml>=6.0` 僅在以 `uv run --script` 直接執行時生效；當其他 hook / 測試 import 它為 module，UV ephemeral env 不會拉 transitive deps，造成 `ModuleNotFoundError: yaml`。

### 影響範圍

直接受影響的 9 個 hook 測試檔（W17-189 follow-up 點名 10 檔，含 test_analytics 但該檔有獨立 stale module bug，與 yaml 無關）：

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 三方案優劣對照

| 方案 | 範圍 | 認知負擔 | 風險 | 同步策略契合度 | 可行性 |
|------|------|---------|------|--------------|------|
| A. 中央化 deps（pyproject.toml + wrapper） | 1 新建檔案 + 開發者執行指南 | 低 | 低 | 高（單一 SSOT） | 高 |
| B. 移除 frontmatter_parser yaml 依賴 | lib + 15 個 hook 使用者 | 中 | 中-高（W39-002 已知 1 差異） | 中（避免 transitive deps） | 中 |
| C. 測試檔 UV script header 補 deps | 不適用（技術不可行） | 不適用 | 不適用 | 不適用 | 不可行 |

**方案 A 細節**：

- A1（pyproject.toml）：新建 `.claude/hooks/pyproject.toml` 宣告 dev deps（pytest, pyyaml）；測試入口改 `uv run --project .claude/hooks pytest tests/`
- A2（wrapper script）：新建 `.claude/hooks/tests/run_tests.sh` 統一注入 `--with pyyaml --with pytest`；更新開發者指南
- A3（package.json npm script）：在 root package.json 新增 `test:hooks` script 包裝 uv 指令；保持與既有 jest scripts 同層級

**方案 B 細節**：

- 改造 `lib.frontmatter_parser.py` 改用手寫 YAML 解析器（W39-002 評估報告已寫初版）
- 同步修復 ARCH-020：13 個 production hook 全部驗證行為不變
- 風險：W39-002 報告所述 1 個功能差異需先界定影響面

**方案 C 不可行原因**：

UV script header（`# /// script ... ///`）僅在 `uv run --script foo.py` 直接執行時生效；pytest 不解析此 header，測試檔被 pytest 載入時 header 完全無效。此方案技術上不適用測試場景，應從候選清單剔除。

### 推薦方案：A1 + A3 組合（高優先）

**主路徑**：A1（建立 `.claude/hooks/pyproject.toml`）作為 SSOT 宣告 dev deps，配合 A3（package.json `test:hooks` script）作為統一執行入口。

**Why**：

- 解決 PC-124 transitive deps gap 的根因——pytest 執行情境本就需要明確 venv 而非 ephemeral env
- 認知負擔最低：1 新建檔案 + 1 行 npm script，無需改任何測試檔或 lib
- 同步策略契合：所有 hook 測試共用同一 deps 宣告，未來新增測試零摩擦
- 風險低於方案 B：不觸碰生產 hook 的 yaml 路徑

**Consequence（若不採用）**：

- 開發者每次跑測試都要手動記得 `--with pyyaml`，違反 PC-124 教訓
- ARCH-020 同步漂移風險：每個測試檔自行宣告 deps 時跨檔不一致
- 9 個測試持續無法在預設環境執行，等同 CI 失能

**Action**：

1. **IMP spawn**：建立 IMP ticket 實作 A1 + A3 組合
   - 新建 `.claude/hooks/pyproject.toml` 含 `[project]` + `[dependency-groups.dev]` 段落（pytest + pyyaml）
   - root `package.json` 新增 `"test:hooks": "uv run --project .claude/hooks pytest tests/"` script
   - 更新 CLAUDE.md §5 開發指令章節補 `npm run test:hooks` 說明
   - 修復 test_analytics.py stale module reference（順便處理，獨立 commit）
2. **方案 B 觀察期**：方案 A 落地 30 天後，若仍因 hook 生產端 transitive deps 撞牆（例如其他 hook 之間互相 import 拉不到 yaml），再啟動 W39-002 評估的方案 B
