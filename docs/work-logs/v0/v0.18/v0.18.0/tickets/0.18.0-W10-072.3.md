### 問題根因

父 ANA（W10-072）Reality Test 事實 4 確認：ANA children 全完成阻擋 complete 是合理設計，非 hook bug。根本原因是方法論文件未區分「研究性 ANA」與「防護性 ANA」兩種子類型的差異：

- **研究性 ANA**：分析結論落地為設計文件/規則/error-pattern，不建 children → 可直接 complete
- **防護性 ANA**：建 children 追蹤後續實作 → 等 children 全 completed/closed 後才可 complete

在此區別未明示前，PM 與代理人容易誤認「ANA 完成分析報告即可 complete」，違反 W17-120.2 / PC-091 路線設計意圖。

### 影響範圍

- `.claude/pm-rules/ticket-lifecycle.md`（已有「ANA Ticket 落地下游血緣選擇」章節，需在旁補充子分類定義）
- 所有撰寫或執行 ANA 類型 ticket 的角色（PM 與代理人）

### 相關 Error Pattern

- PC-091：ANA 落地下游血緣選擇（tickets 建為兄弟而非 children）
- W17-120.2：ANA 路線多視角審查共識，統一 ANA 衍生用 children（--parent）
