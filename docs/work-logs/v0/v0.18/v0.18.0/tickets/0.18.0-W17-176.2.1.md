### 問題根因

`handoff-prompt-reminder-hook.py` 的 `is_ticket_completed()` (L89-123) 與 `resolve_ticket_path()` (L125-154) 使用硬編碼扁平路徑 `docs/work-logs/v{version}/tickets/{id}.md`，未涵蓋現行三層階層 `docs/work-logs/v{major}/v{minor}/v{version}/tickets/{id}.md`。已完成 ticket 的路徑解析永遠回傳不存在的扁平路徑 → `is_ticket_completed()` 回傳 False → reminder 每 turn 重複觸發。

### 影響範圍

- 直接修改：`.claude/hooks/handoff-prompt-reminder-hook.py` 兩個函式
- ARCH-020 跨進程同步：`handoff-auto-resume-stop-hook.py` 已於 W17-165 改用 `find_ticket_file()`，本 ticket 為其同步補修

### 相關 Error Pattern

ARCH-020（跨進程同步修復遺漏）：stop hook 升級路徑解析時未同步 reminder hook。

### 修復摘要

1. `import` 增加 `find_ticket_file`（from `hook_utils`）
2. `is_ticket_completed()`：移除自製 `parts = ticket_id.split('-')` 路徑組裝邏輯，改呼叫 `find_ticket_file(ticket_id, project_root, logger)` 取得實際路徑；後續 frontmatter 解析與 status 比對不變
3. `resolve_ticket_path()`：完全委派給 `find_ticket_file()`，回傳型別改為 `Optional[Path]`（找不到時回傳 None，與既有呼叫端 `if not ticket_path or not ticket_path.exists()` 相容）

### 設計決策

- 不刪除 `resolve_ticket_path()` wrapper，保留作為 hook 內部 API 介面（呼叫端 `scan_handoff_pending_directory` L228 使用），降低 diff 噪音
- 保留 `parse_ticket_frontmatter` import（仍用於 `is_ticket_completed`）
