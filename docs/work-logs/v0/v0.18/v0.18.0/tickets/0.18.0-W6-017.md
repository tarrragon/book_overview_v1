### 動機案例（直接從 W6-012.1 教訓繼承）

W6-012.1 事件：

- build/development/ 停在 2025-09-24（8 個月前）
- src/overview/overview-page-controller.js 自 2025-09 起至少 5 次 commit
- Chrome Extension 載入舊版 source，bug 看似源自 src 但實為 stale build
- 用戶花費大量時間診斷「假 bug」前才發現需重 build

### Hook 設計需求

| 觸發點選項 | 評估 |
|----------|------|
| SessionStart hook | 簡單，每 session 開始檢查；不會主動阻擋開發 |
| PreToolUse on Edit/Write src/ | 寫 src 前提示「記得 build:dev」；可能過度干擾 |
| Post-commit hook for src/ changes | commit 後若改了 src 提示重 build；訊號清晰 |

**推薦**：SessionStart hook（最小干擾且高訊號）。

### 偵測邏輯

```python
src_mtime = max(mtime of src/**/*)
build_mtime = mtime of build/development/manifest.json (or oldest if dir)
if src_mtime > build_mtime + threshold:
    print warning("build/development/ outdated: src 更新於 X，build 停於 Y")
```

### Hook 規範

- 位置：`.claude/hooks/build-staleness-check-hook.py`
- 註冊：`.claude/settings.json` SessionStart events
- 訊息含 `npm run build:dev` 重建指令
- 不阻擋（warning only）
- 閾值：可設 1 小時或 24 小時（防止剛 build 完誤判）

- AC 1 正確偵測 build 過期 → 比對 src vs build mtime

- AC 3 不阻擋工作流 → warning level，無 exit code 1
- AC 4 Hook 註冊於 settings.json → 通過 hook-completeness-check

### 修復摘要

新增 `.claude/hooks/build-staleness-check-hook.py`（UV single-file Python），SessionStart 觸發，偵測 `src/**/*` 最新 mtime vs `build/development/manifest.json` mtime；落差 > 1 小時時輸出 warning 含 `npm run build:dev` 指令；manifest 不存在時提示尚未 build。不阻擋（exit 0）。

### 設計決策

- **純函式 `check_build_staleness(project_root, threshold_seconds)`** 與 main 分離，便於單測；回傳 `(status, message)` 四種狀態：`stale` / `missing` / `fresh` / `no_src`。
- **閾值 1 小時**（`STALE_THRESHOLD_SECONDS = 3600`）避免剛 build 完誤判（src 寫入時間略晚於 build 是正常 race）。
- **src 不存在回 `no_src` 並靜默**：避免在非 Chrome Extension 倉誤觸發。
- **不阻擋**：訊息走 stdout，return 0；SessionStart 屬 informational，不應 fail。
- **遵循 hook_utils 規範**：`setup_hook_logging` + `run_hook_safely` + import 失敗 graceful fallback。

### 測試覆蓋

8 tests，涵蓋三主要場景 + 兩邊界：
- stale（src 比 build 新 8 個月）
- fresh（build 較新）
- fresh（差距 10 分鐘 < 閾值）
- missing（manifest 不存在）
- no_src（src 目錄不存在 → 靜默）
- main 三場景（stale / fresh / missing）驗證 exit code 0 與 stdout

### 註冊

`.claude/settings.json` SessionStart events 插入 `project-init-env-check-hook.py` 之前。
