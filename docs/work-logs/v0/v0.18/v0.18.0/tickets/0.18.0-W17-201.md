**根因**：`commit-msg-layer2-marker-check-hook.py`（W17-126 / commit 91fcdd7a 引入）shebang 為 `#!/usr/bin/env python3`，無 PEP723 script block 宣告 dependencies。Hook import 鏈：

```
commit-msg-layer2-marker-check-hook.py
  └─ from lib.framework_paths import is_framework_path
       └─ import yaml   # ← 系統 python3 無 pyyaml
```

**症狀**：PostToolUse:Bash hook 每次 Bash 工具呼叫後觸發 `ModuleNotFoundError: No module named 'yaml'`，traceback 流到 stderr，hook system health check 顯示 "PostToolUse:Bash hook error / Failed with non-blocking status code: Traceback"。

**對比正常設定**（同樣 import framework_paths 的 hook）：

| Hook | shebang | PEP723 | 狀態 |
|------|---------|--------|------|
| layer-boundary-validator-hook.py | uv run script | 有 pyyaml | OK |
| framework-rule-edit-skill-trigger-hook.py | uv run script | 有 pyyaml | OK |
| agent-dispatch-validation-hook.py | python3（但 PreToolUse 場景未觸發 framework_paths import） | 有 pyyaml | OK |
| **commit-msg-layer2-marker-check-hook.py** | **python3 + 無 PEP723** | **缺** | **FAIL** |

**對應 PC-135**：「子代理人 pytest 通過 ≠ hook subprocess 環境通過 / 修共用 lib 後新 dep 必須同步至所有 caller pep723」——本案為同模式新案例（W17-126 caller 漏同步）。
## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

**修法**：對齊 layer-boundary-validator-hook 既有設定。

Diff:

```diff
-#!/usr/bin/env python3
+#!/usr/bin/env -S uv run --quiet --script
+# /// script
+# requires-python = ">=3.11"
+# dependencies = ["pyyaml"]
+# ///
 """commit-msg Layer 2 marker check hook（W17-126）。
```

**驗證**：對 10 個 PostToolUse Bash hooks 逐一以 mock tool_response JSON stdin 餵入並執行，全部 exit 0；commit-msg-layer2 修前 exit 1 traceback yaml ModuleNotFoundError，修後 exit 0。
