## 架構

## 章節結構

1. **為什麼 Prompt 寫作需要獨立指引** — 對比表格說明 prompt 與一般文件的關鍵差異（主要讀者、冗餘成本、結構化標記必要性）
2. **五大原則在 Prompt 情境的應用**（5 章節完整對應）：
   - 原子化 × Prompt：一個 prompt 一個任務目標，含判斷標準表格 + 反例/正例對比
   - 索引建立 × Prompt：結構化標記工具箱（## / ### / 表格 / 列表 / XML / 程式碼圍欄）+ 建議章節骨架
   - 意圖顯性與商業邏輯 × Prompt：開頭即表達意圖、明確輸出格式、告訴 AI 為什麼（3 個子節，含反例/正例）
   - 可查詢性 × Prompt：穩定關鍵字、變數佔位符命名（snake_case 規範）、區塊邊界標記
   - 欄位設計 × Prompt：標準欄位骨架（任務/輸入/約束/驗收/背景/參考）+ 反例/正例
3. **Token 節省策略**（5 種技巧，全部含 before/after）：
   - 策略 1：用符號取代連接詞（節省 ~40%）
   - 策略 2：表格取代重複句型（節省 ~35%）
   - 策略 3：引用路徑取代完整內容貼入（節省 > 90%，含適用判斷表）
   - 策略 4：刪除客套與鋪陳（節省 ~70%）
   - 策略 5：用預設約定取代顯式枚舉（節省 ~80%，含適用原則）
4. **結構化標記範例**（3 類）：
   - 範例 A：Agent Prompt（派發代理人）— 完整骨架 + 設計要點表格
   - 範例 B：Ticket Context Bundle — YAML frontmatter 區塊 + 欄位角度對照 + DRY 原則
   - 範例 C：小型內聯 Prompt（對話中）— 壓縮版三層訊號
5. **自檢清單** — 五大原則 + Token 節省 共 15+ 項勾選條目
6. **反模式速查** — 10 種常見反模式 + 症狀 + 正確做法
7. **延伸閱讀** — Anthropic 官方 prompt engineering、Zettelkasten 原子化、結構化 output 應用
