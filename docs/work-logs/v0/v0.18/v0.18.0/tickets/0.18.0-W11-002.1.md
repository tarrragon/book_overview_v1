### 重構摘要

依 AC 三項完成 lifecycle.py 重構：

1. `_cascade_unblock_children` signature 新增 `ticket_map: Optional[Dict[str, Any]] = None`，None 時 fallback 走 `list_tickets(version)` 維持向後相容；docstring 補上「child dict 會被原地 mutate」契約。
2. `complete()` 頂層於 line 752 區塊統一載入 `all_tickets` 並建立 `ticket_map = {t.get("id"): t for t in all_tickets}`，下傳給 `_analyze_next_steps`（透過 all_tickets）與 cascade 流程（透過 ticket_map）。
3. 抽出 `_post_complete_cascade(parent_ticket, version, ticket_map)` helper（位於 `_cascade_unblock_children` 之上），封裝原 line 758-764 的 children 檢查 + cascade 呼叫 + 兩個 print helper。`complete()` 主體變為單行 `_post_complete_cascade(ticket, self.version, ticket_map)` 編排。

### 設計決策

- `_cascade_unblock_children` 保留 `ticket_map=None` fallback：避免破壞潛在外部 caller。
- `_post_complete_cascade` 的 `ticket_map` 為必填（非 Optional）：唯一 caller 是 `complete()`，已強制建立 map，無需重複 fallback。
