### 修復摘要

新增 `ticket track hook-health` 子命令，PM 手動觸發 Hook 觸發頻率掃描與評估（W13-008 IMP-3 落地，補強 W13-017 SessionStart 被動掃描的主動性）。

### 變更檔案

- `.claude/skills/ticket/ticket_system/commands/track_hook_health.py`（新建，222 行）
- `.claude/skills/ticket/ticket_system/commands/track.py`（註冊 import / handler / register；3 處新增 8 行）
- `.claude/skills/ticket/tests/test_track_hook_health.py`（新建，9 條測試覆蓋 5 acceptance）

### 設計決策

1. **路徑修正**：原 ticket `where.files` 指 `cli.py` 過時；實際採模組化 `commands/track_*.py` + `commands/__init__.py` 註冊。實作沿用 W17-200 / W10-114 / W17-008.15 既有風格。
2. **Lib 載入策略**：採三層 fallback 定位 `.claude/hooks/lib/hook_health.py`：(1) `$CLAUDE_PROJECT_DIR` env、(2) cwd 向上找、(3) `Path(__file__).parents[4]` dev fallback。原因：全局 `uv tool install` 後本檔位於 site-packages，`parents[4]` 不再指向 `.claude/`；首次以 module-level eager load 觸發 `ModuleNotFoundError` 被 entry script 的 wrapper 吞為「套件未正確安裝」，故改為 lazy + 多策略。
3. **共用 collector**：`scan_logs` / `classify_hook` / `evaluate` 透過 thin wrapper 暴露，便於測試 patch；與 W13-017 `hook-health-monitor.py` 共用同一 lib。baseline 公式 `sum(per_day) / window_days` 與 monitor 一致（W13-008 量化標準）。
4. **--dry-run 為契約點**：本命令本即無副作用（不建 ticket、不寫檔），旗標明示「禁止任何副作用」並在 stdout 顯式標記，作為對外契約。
5. **輸出格式**：table（PM 預設視圖，含 reasons 行顯示觸發理由）、json（自動化消費，含 `flagged_count` 摘要）。

### 不在本 ticket 範圍

- 自動建 ticket 流程：W13-008 Solution 已明示「stderr 提醒，不自動建 ticket」；本命令保留同設計
- ML pattern detector：W13-008 WRAP 已排除（過度設計）
- `hook-health-monitor.run_frequency_scan` 重構：該函式 hook 內部用，混合 stderr + ticket 建議；本命令需 stdout 結構化輸出，不複用該渲染層（避免耦合 hook side effects）
