## Context Bundle（來自 W17-048 WRAP 方案 B）

### Hook 檔案現況

路徑：`.claude/hooks/agent-prompt-length-guard-hook.py` (87 行)

當前邏輯：
- 只檢查 `line_count > PROMPT_LINE_LIMIT(30)` 時擋
- 符合則 return 2 + stderr 輸出 `BLOCK_MESSAGE_TEMPLATE`
- 不符則 return 0 直接放行

### 擴充目標

**新增「軟提示」層**（不擋，只提示）：在 prompt 介於 **10-30 行**且**缺關鍵字**時輸出提示訊息到 stderr，讓 PM 看到但仍放行（exit 0）。

### 判別邏輯

```
if line_count > 30:
    [現行行為] BLOCK 返回 exit 2
elif line_count > 10 and not has_template_keywords(prompt):
    [新增] stderr 輸出提示 + 返回 exit 0（allow）
else:
    放行
```

### 模板關鍵字

PM 使用派發模板時，prompt 通常會含以下任一關鍵字（至少一個）：

- `讀取 Ticket` / `讀取 ticket` / `讀取 ticket 的`
- `ticket track full`
- `ticket track query`
- `依 Context Bundle` / `Context Bundle` / `依 ticket`

實作時定義常數：

```python
TEMPLATE_KEYWORDS = [
    "讀取 Ticket",
    "讀取 ticket",
    "ticket track full",
    "ticket track query",
    "Context Bundle",
    "依 ticket",
]
```

判斷函式：

```python
def has_template_keywords(prompt: str) -> bool:
    """檢查 prompt 是否包含模板關鍵字（大小寫敏感，因中文關鍵字有差異）"""
    return any(keyword in prompt for keyword in TEMPLATE_KEYWORDS)
```

### 提示訊息範本

```python
SOFT_HINT_TEMPLATE = """提示：Agent prompt 為 {actual} 行，但未偵測到模板關鍵字（W17-048 方案 B）

建議結構：
  Ticket: {{ticket_id}}

  ## 任務
  {{簡短動作描述}}

  讀取 ticket：`ticket track full {{ticket_id}}`
  依 Context Bundle 執行流程。

模板：.claude/references/agent-dispatch-template.md
"""
```

注意：提示不擋（return 0），只是在 stderr 輸出讓 PM 看到。

## 實作摘要

在 `.claude/hooks/agent-prompt-length-guard-hook.py` 加入軟提示層（W17-048 方案 B），保持 30 行硬上限（PC-040）不變。

### 檔案變更

- `.claude/hooks/agent-prompt-length-guard-hook.py`（延伸既有 Hook）
- `.claude/hooks/tests/test_agent_prompt_length_guard_hook.py`（新建測試檔）

### 關鍵實作

1. 新增常數：
   - `SOFT_HINT_THRESHOLD = 10`
   - `TEMPLATE_KEYWORDS = ["讀取 Ticket", "讀取 ticket", "ticket track full", "ticket track query", "Context Bundle", "依 ticket"]`
   - `SOFT_HINT_TEMPLATE`（stderr 訊息範本，引導 PM 參考 `.claude/references/agent-dispatch-template.md`）
2. 新增 `has_template_keywords(prompt)` helper：任一關鍵字命中即視為已使用模板。
3. `main()` 判別邏輯分兩層：
   - Layer 1 硬上限：`line_count > 30` → exit 2 + BLOCK 訊息（行為未變）
   - Layer 2 軟提示：`10 < line_count <= 30` 且缺關鍵字 → stderr 輸出 SOFT_HINT + exit 0
   - 其他：exit 0，無輸出

### 行為保留

- 非 Agent/Task 工具豁免（tool_name 早退出）
- tool_input JSON 字串或 dict 皆可處理
- 空 prompt 放行
- hook_utils.setup_hook_logging + run_hook_safely 統一日誌 / crash 處理
