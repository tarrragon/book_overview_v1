### 問題根因

PC-093 hook (`phase4-decision-enforcement-hook.py`) 的 `format_warn_info_message`（line 479-514）在 invalid exempt markers 輸出時，僅顯示原始 err code（`ticket-tracked-need-id` / `format-error` 等），未對用戶提供：
1. err code 的人類可讀說明（這個錯誤代表什麼）
2. 修復範例（合法格式是什麼樣）

W10-017 session 中 PM 必須讀 hook 原始碼才推論出 `ticket-tracked` category 需要 `W{wave}-{seq}` 格式 ID，違反 quality-baseline 規則 4「Hook 失敗必須可見」。

### 影響範圍

- `.claude/hooks/phase4-decision-enforcement-hook.py`（單一模組）
- `.claude/hooks/tests/test_phase4_decision_enforcement_hook.py`（新增 2 個測試）

### 程式碼錨點

`.claude/hooks/phase4-decision-enforcement-hook.py`:

- **line 117-121** `ExemptRef` dataclass：欄位 `err: Optional[str]` 帶有失敗原因 code
- **line 262-275** `validate_exempt_fields`：產出 4 種 err code（`category-whitelist` / `reason-too-short` / `baseline-need-number` / `ticket-tracked-need-id`）
- **line 282-298** `collect_exempt_markers`：format 不符時產出 `format-error`
- **line 479-514** `format_warn_info_message`：當前 invalid_exempts 輸出邏輯（line 508-510）

### 實作策略

**2. 改寫 invalid_exempts 輸出（line 508-513）**

### 變更摘要

1. **ERR_MESSAGE_MAP 對照表**（hook 檔頂層常數，line 91-115）：覆蓋全部 5 種 err code（format-error / category-whitelist / reason-too-short / baseline-need-number / ticket-tracked-need-id），每項為 `(humanized 描述, 修復範例/提示)` tuple。
2. **format_warn_info_message 改寫**（line 508-516）：將 invalid_exempts 輸出從單行 `[INVALID: <err_code>]` 擴充為兩行（含 `[INVALID: <err_code>] <humanized title>` 與 `修復提示: <hint>`）。保留 `[INVALID: <err_code>]` 字串作 grep 訊號（向後相容）。
3. **匯入清理**：順手移除既有未使用的 `os` 與 `dataclasses.field` import 以滿足 ruff 0 issues；新增 `Dict` 至 typing import。
4. **新增測試** 2 個：
   - `test_format_warn_info_humanizes_ticket_tracked_need_id`：建構無 ticket ID 的 ticket-tracked marker，斷言 output 同時含 `ticket-tracked-need-id` grep 訊號、`[INVALID:` 標籤、`W{wave}-{seq}` hint 關鍵字、「修復提示」字樣。
   - `test_format_warn_info_humanizes_format_error`：建構 `<!-- PC-093-exempt -->`（缺 cat:reason），斷言 output 含 `format-error` 訊號、`<!-- PC-093-exempt:` 範例字樣、「修復提示」字樣。

### 設計決策

- **保留原 err code 作 grep 訊號**：向後相容既有測試（如 ex_n2/n3/n4），且支援 PM 在 stderr/log 用 grep 快速分類。
- **修復範例使用合法格式而非僅描述**：用戶可直接複製貼上修正，最大化 quality-baseline 規則 4「Hook 失敗必須可見」價值。
- **不修改 validate_exempt_fields 回傳結構**：避免擴散影響至其他 caller / 測試，限縮 ticket 範圍。
