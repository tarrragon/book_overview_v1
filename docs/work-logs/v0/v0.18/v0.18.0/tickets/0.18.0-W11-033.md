## 範圍擴充：併入 ticket track complete metadata 孤兒（2026-05-17 補）

### 第二缺口

`ticket track complete <id>` 修改 ticket md frontmatter（status / acceptance / completed_at）+ body Completion Time，**但不自動 commit metadata**。Session 邊界處易遺失：
- 案例：W6-022 上 session (2026-05-17T15:07-15:27) 實作 commit `0043b3e0` 後 ticket complete，但 session 結束前未 commit metadata
- 本 session 啟動時 `git status` 看到 M `0.18.0-W6-022.md`（孤兒）
- commit `03cbf286` 補回

### 與第一缺口（worktree cleanup）的關係

兩者皆「ticket complete 後系統性缺口」家族：
- 第一缺口：合併 branch 後 worktree 目錄殘留（disk / 視圖污染）
- 第二缺口：complete 後 metadata 改動未 commit（state inconsistency / 接手 session 看到 M）

共同根因：`ticket track complete` 為 metadata-only 操作，無 git 副作用；PM 完成 ticket 後習慣直接進下個任務或結束 session，少有「回頭 commit metadata」習慣。

### 擴充修復方向

| 修復 | 適用缺口 |
|------|---------|
| `ticket track complete` 結束時自動 `git add <ticket-md>` + 提示 commit（或直接自動 commit） | 第二缺口 |
| 擴充 worktree-merge-reminder-hook PostToolUse 階段檢查合併 worktree | 第一缺口 |
| SessionStart 統一 audit：列出 (a) ahead=0 user worktree (b) 已 complete 但 ticket md 未 commit 的 metadata 孤兒 | 兩者 |
| ticket-lifecycle.md 補步驟：complete 後 cleanup checklist（含 worktree + metadata commit）| 兩者 |

### 補強 acceptance

原 4 條 acceptance 不變，追加：
- ticket track complete 自動處理 metadata commit（自動 add + 提示，或自動 commit）
- SessionStart 列出 metadata orphan ticket（已 complete 但 md 仍 M）
