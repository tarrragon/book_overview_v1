## Problem Analysis

### 問題根因

PC-096 揭露 CLI 退出碼設計常採「成功/失敗」二分思維（exit 0 / exit 1），將兩類本質不同的非成功狀態擠進同一個 exit 1：

1. **程式錯誤（internal error）**：例外、檔案讀不到、依賴崩潰——應通知人類介入
2. **業務拒絕（business reject）**：邏輯正常運作、依規則回答「不可放行」——應由呼叫方依拒絕原因處理

PM/Hook/shell pipeline 拿到 exit 1 無法決定「該 retry、abort 還是顯示拒絕原因」，必須 parse stdout 才能決策，違反 Unix exit code 「機器可讀決策訊號」設計初衷。
