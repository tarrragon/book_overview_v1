### 問題根因

W11-003.7 落地後，同檔案 `commands/create.py` 仍有兩處 `.get("where", {}).get(...)` 假設 `where` 為 dict。當 ticket 為舊字串格式時 `.get("files")` 同樣 AttributeError。

實際當前行號（W11-003.7 helper 加入後 +32 偏移）：

- **L1405**: `child_info.get("where", {}).get("files", [])` 在 `_print_parallel_analysis_result()`
- **L1456**: `new_ticket.get("where", {}).get("files") or []` 在另一處 main flow

兩者都取 `where.files`（list），與 W11-003.7 修的 `where.layer`（str）不同欄位。

### 影響範圍

- 檔案：`.claude/skills/ticket/ticket_system/commands/create.py`
- 函式：`_print_parallel_analysis_result`（L1369-）+ 另一處 main flow 區段
- 觸發條件：建立子任務時，parent / new_ticket / child sibling 為舊字串格式
- 風險等級：與 W11-003.7 同等（AttributeError 阻斷 ticket create）

### Helper 策略（與 W11-003.7 對齊）

既有 helper（W11-003.7 落地）：
- `_inherit_parent_who(parent_ticket)` → str（dict/str/None）
- `_inherit_parent_where_layer(parent_ticket)` → str（dict/str/None）

本 ticket 需求差異：
- 取的是 `where.files`（list 非 str）
- 場景是 child_info / new_ticket（非 parent_ticket，但 dict 結構相同）

建議實作：新增 `_extract_where_files(ticket_data: Dict) -> List[str]`，遵循同模式（isinstance dict → .get('files', []), isinstance str → []（舊格式無 files 概念）, None → []）。L1405 / L1456 統一改用此 helper，避免重複型別判斷。

### 修復摘要

新增 `_extract_where_files(ticket_data: Optional[Dict]) -> List[str]` helper（位於 `commands/create.py` _inherit_parent_where_layer 之後），對 ticket dict 取 where.files 時進行型別防護：

- where 為 dict → `.get("files")`（檢查為 list，否則回 `[]`）
- where 為 str（legacy）/ None / 缺失 / ticket_data 為 None → 回 `[]`

替換兩處原 unsafe 寫法：

| 位置 | 原寫法 | 新寫法 |
|------|--------|--------|
| `_print_parallel_analysis_result` (L1405) | `child_info.get("where", {}).get("files", [])` | `_extract_where_files(child_info)` |
| `_print_cognitive_load_assessment` (L1456) | `new_ticket.get("where", {}).get("files") or []` | `_extract_where_files(new_ticket)` |

### 設計決策

- 與 W11-003.7 既有 `_inherit_parent_who` / `_inherit_parent_where_layer` 同模式（dict / str / None 三分支）
- helper 不限於 parent_ticket，可重用於 child_info / new_ticket 任何 where 結構 dict
- 對 dict.files 額外檢查 `isinstance(list)`，防護 `files: None` 等異常 schema

### 不在本 ticket 範圍

- W11-003.7 與本 ticket 修復同根問題（where dict/str 不對稱），可後續升級為「型別假設未防護」error-pattern（ticket md why 已記錄此規劃）
