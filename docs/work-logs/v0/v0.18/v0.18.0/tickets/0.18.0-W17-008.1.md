### 修改摘要

- `commands/track.py:549`：`p_add_spawned.add_argument("value", nargs="+", ...)` 開放多 ID positional
- `commands/fields.py:execute_add_spawned`：將 `args.value` 視為 list 迴圈處理；新增 `added` / `skipped` 兩清單，輸出區分「新增」「已存在略過」

### 設計決策

- 對齊 Unix 慣例（rm a b c / ln a b c）：採 nargs='+' 而非 --batch FILE 或 wrapper（WRAP A1）
- 向後相容：`isinstance(args.value, list)` 防護，理論上 argparse 必傳 list，但允許程式呼叫傳 str
- 去重邏輯保留：重複 ID 不重複加入 spawned，但獨立列出讓使用者知曉

### 不在本 ticket 範圍

- remove-spawned CLI（目前無此命令；如需另開 ticket）
- spawned ID 格式驗證（W17-004 其他落差項）
