### 5 個候選假設（含粒度自檢）

#### 候選 5（ROI #1）：prompt 內容觸發特殊 audit
- **假設**：W17-097.1 prompt 結構（含 import 修改 + 8 替換 + git commit/pytest 提及）觸發 CC runtime 的某種內容 audit 機制
- **粒度層**：內容粒度（修改類型）vs 語意粒度（pytest/commit 提及）vs 結構粒度（替換數量）
- **區辨度**：高 — 與 W17-097.1 完整 prompt 風格直接對照
- **成本**：低（30 min，單派 thyme）
- **執行條件**：必須冷 session（PM 從未 Edit .claude/）
- **子 ticket**: W17-110.1

#### 候選 2（ROI #2）：並行 + 複雜 Edit 組合
- **假設**：4 並行 + 複雜內容的組合（單獨任一變因都不觸發）導致 deny
- **粒度層**：並行粒度（同時啟動 vs 分批）vs 複雜度粒度（簡單 vs import+多替換）
- **區辨度**：中 — 需 4 並行對照 W17-101 簡單 Edit 結果
- **成本**：中（1 hr，4 並行派發 + 結果對照）
- **執行條件**：冷 session
- **子 ticket**: W17-110.2

#### 候選 3（ROI #3）：session 累積狀態
- **假設**：session 累積到一定狀態（多 commit + pytest 後）觸發 deny
- **粒度層**：累積粒度（commit 數 / pytest 數 / tool call 數 / context 估計）
- **區辨度**：中 — 需先做累積後測試
- **成本**：高（2 hr，需先累積再測試）
- **執行條件**：新 session，刻意累積後測試
- **子 ticket**: W17-110.3

#### 候選 1（ROI #4）：transient runtime（多 session base rate）
- **假設**：W17-097.1-.4 那次是 CC runtime 暫態故障
- **粒度層**：時間粒度（同 session 不同時段 / 跨 session）vs 觸發粒度（無條件 vs 特定觸發）
- **區辨度**：低 — 需多 session 重複觀察才能拿到 base rate
- **成本**：高（5+ hr，跨 3-5 個 session）
- **執行條件**：連續多 session 獨立執行 Phase A
- **子 ticket**: W17-110.4

#### 候選 4（觸發條件，不獨立 ticket）：CC 版本內部行為波動
- **假設**：同版號 v2.1.126 不同 build 行為不同
- **粒度層**：版本粒度（major / minor / patch / build hash）
- **區辨度**：低 — 需 CC 升級才有對照組
- **成本**：不可控（依賴外部 CC 升級）
- **觸發條件**：CC 升級後在 PC-115 補新版本下的 Phase A 重測
