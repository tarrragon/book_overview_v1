## 設計決策（Phase 1）

### Signal: 並行場景判定
- 資料來源：.claude/dispatch-active.json 的 dispatches[] 陣列
- 判定：len(dispatches) >= 2 視為並行（既存條目僅在 SubagentStop 後清理，>=2 代表有前序派發仍在跑）
- 理由：簡單可測、無需時間窗；既有 active-dispatch-tracker-hook 負責清理

### Wide staging 偵測
- 正則：\bgit\s+add\s+(?:\.|-A|--all)\b
- 排除：git add -- 合法、具體路徑不匹配

### 決策：警告非阻擋
- 並行 + 廣域 staging → stderr 印警告（含 PC-092 引用）+ 不改變 return code
- 單一場景或無廣域 staging → 不觸發

### 整合點
- main() 內確認 subagent_type 為實作代理人後、path classification 前執行偵測
- 警告與 block/allow 決策解耦：先印警告再跑既有邏輯

### 新增函式
- _count_active_dispatches() -> int：讀 dispatch-active.json（錯誤 → 0）
- _has_wide_staging(prompt) -> bool
- _emit_wide_staging_warning_if_parallel(prompt, logger) -> None
