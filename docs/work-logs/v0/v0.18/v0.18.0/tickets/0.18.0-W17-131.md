### 修正方向（建議交 IMP follow-up）

| 選項 | 內容 | trade-off |
|-----|------|----------|
| A | `framework_paths` 加 `.claude/hooks/` | 簡單；副作用：PreToolUse 對所有 hook Edit 觸發警告，依賴 cache + dry-run 觀察 |

| C | lifecycle.py 額外 inline 判 `.claude/hooks/`（不動 SSOT） | 違反 SSOT 原則，不推薦 |

### 推薦方案

**B（拆 strict/broad 兩 list）**：兩 list 在 SSOT 內由 framework-paths.yaml 統一管理，`framework_paths_broad = framework_paths_strict + .claude/hooks/`；消費端依用途取對應 list。理由：
1. PreToolUse skill-trigger-hook 對所有 hook Edit 觸發警告會與 cache 機制競爭（hook refactor 頻繁時 cache invalidation 失準）
2. lifecycle.py S 問是 claim-time 提示，frequency 低（每 ticket 一次），broad 觸發無 PreToolUse 過度誤擋風險
3. 兩 list 用途分離後，未來新增類別（例如 .claude/config/ 配置文字）可獨立加 broad 不影響 strict
