## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W17-088 派發 thyme-documentation-integrator 失敗的直接根因為兩層疊加：

1. **PC-088 LLM 工具選擇偏誤**：thyme 面對「修改 .md 檔」任務時，自選 mcp__serena__replace_content（單步高精度工具），忽略 Edit 工具（同等能力但更輕量）。根因是 LLM 對總步驟數估算錯誤，誤以為 serena MCP 是更短路徑。
2. **無 Fallback 規則導致 early stop**：serena 寫入工具不在 settings.local.json allow list → 被拒 → thyme 錯誤泛化為「所有寫入工具都被拒」，提前停止，未嘗試 Edit 工具。

### 影響範圍

- AGENT_PRELOAD.md（所有代理人都載入）：缺乏工具選擇優先序規則，任何代理人都可能犯同樣錯誤
- thyme-documentation-integrator.md：已有「工具使用策略」表格，但缺 Fallback 規則
- 潛在影響：其他有 serena MCP 全域配置的代理人（若收到文字修改任務也可能自選 serena）

### 相關 Error Pattern

- PC-088（LLM 工具選擇偏誤）：LLM 單步敏感、總步驟盲，誤估長路徑為短路徑
- W17-088 實驗 A/C 證明：純 Edit 對 docs/ 與 .claude/ 都成功

### W（Widen）— 候選方案

1. **規則層（本方案）**：AGENT_PRELOAD + thyme 主文加工具選擇規則 + Fallback 規則 → 治本，一次修補覆蓋所有代理人
2. **每次 prompt 顯式限定工具**：PM 派發時明示「禁用 serena MCP 寫入工具」→ 需要每次維護 prompt，治標不治本

### A（Attain distance）— 機會成本

佔用 thyme 本次派發配額（~20 tool calls）；但修補後避免後續重工，機會成本為正。

### P（Prepare to be wrong）— 最可能失敗原因

Edit 工具被 AGENT_PRELOAD 某些情境誤解為「只能改文字，不能改結構」→ 防護：明確說明 Edit 適用範圍包含任何 markdown 文字內容修改。
