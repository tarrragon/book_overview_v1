## Phase A IMP-2 實作指引（依 W14-024 結論輔助路徑）

### 目標
將 ticket frontmatter YAML 格式規則固化到代理人定義或 context-bundle-spec.md，避免 PM 每次派發手動提醒。

### 實作選擇（建議方案 B：集中定義）

**方案 A：分散定義**：各代理人 .md 獨立寫入
- 成本：每個代理人加段，維護 4 份
- 風險：規則分散不一致

**方案 B：集中定義 + 引用**（推薦）：
- 在 `.claude/pm-rules/context-bundle-spec.md` 或新建 `.claude/references/ticket-frontmatter-yaml-rules.md` 集中寫入規則
- 各代理人 .md（thyme/parsley/fennel/cinnamon）加一行引用

### 集中規則內容

```markdown
## Ticket Frontmatter YAML 格式要求（強制）

實作型代理人修改 ticket frontmatter 時，必須遵守以下 YAML 格式：

| 欄位 | 格式 | 違規範例 | 正確範例 |
|------|------|---------|---------|
| `status` | enum: pending/in_progress/completed/blocked/closed | `complete` | `completed` |
| `completed_at` | ISO 8601: YYYY-MM-DDTHH:MM:SS（HH ∈ [00,23]）或 null | `T24:00:00` | `T23:55:57` |
| `acceptance` | YAML list，每項獨立字串 | 4 條擠單行用 `,[ ]` 分隔 | 4 個獨立 list items |
| `who` | object 含 current 欄位 | `who: thyme-xxx` | `who:\n  current: thyme-xxx` |

**正確路徑（優先用 CLI，非直接 Edit）**：
- 標記 AC 完成：`ticket track check-acceptance <id> --all` 或 `... <id> 1 2 3`
- 設定 status：`ticket track claim/complete`
- 完成時間：CLI 自動寫入

**Edit frontmatter 合法情境**（需 PM 手動修復）：
- CLI 無法涵蓋的欄位修正（如 migration、schema 變更）
- PM 修復歷史瑕疵
```

### 代理人引用段落
