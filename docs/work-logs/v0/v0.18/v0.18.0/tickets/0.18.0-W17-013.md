## Problem Analysis

## Context Bundle

### Source Chain
- W17-004（落差 12）→ W17-008.10 → 本 ticket（方案 A）

### W17-008.10 產出核心
- `version_str` (positional) - 實際使用中
- `--version` → `version_param` (optional flag) - **dead code**（grep 全專案確認未使用點）
- ARG_VERSION_PARAM 說明「自動偵測」與實作不符

### 修改
- `.claude/skills/ticket/cli/track.py` line 395-397 區：刪除 `--version` argparse 定義（保留 positional version_str）
- `.claude/skills/ticket/references/track-command.md`：移除 `--version` flag 相關範例

### 驗證
- 回歸：`ticket track version <X>` positional 仍可用
- 回歸：所有使用 `ticket track version` 的文件 / Hook 不受影響
- pytest 綠燈

### 風險
- 極低。dead code 本就無呼叫；若有隱藏呼叫 pytest/實測會捕捉
### 問題根因

`ticket track version` 已有必填 positional `version_str`，但 parser 仍額外註冊 `--version` optional flag 到 `version_param`。`execute_version()` 實際只讀 `args.version_str`，`version_param` 不參與行為，形成死參數與錯誤介面暗示。

### 影響範圍

- `.claude/skills/ticket/ticket_system/commands/track.py`：`version` 子命令 argparse 定義。
- `.claude/skills/ticket/ticket_system/commands/track_query.py`：移除 `version_param` 舊註解。
- `.claude/skills/ticket/ticket_system/lib/command_tracking_messages.py`：移除未使用 help 常數。
- `.claude/skills/ticket/tests/test_track_query.py`：測試 fixture 移除不存在欄位。

### 相關 Error Pattern

無新增 error pattern。這是 W17-008.10 識別的 CLI 介面 dead code 清理，風險低且不改變 `ticket track version <X>` 正常路徑。

## Solution

完成 dead code 清理：

1. 從 `track.py` 的 `version` 子命令移除 `p_version.add_argument("--version", dest="version_param", ...)`。
2. 從 `command_tracking_messages.py` 移除 `ARG_VERSION_PARAM` 未使用常數。
3. 從 `track_query.py` 移除 `args.version_param` fallback 註解，保留單一路徑 `args.version_str`。
4. 從 `test_track_query.py` 移除測試 mock 上不再存在的 `version_param` 欄位。

`track-command.md` 檢查結果：文件只保留正確 positional 用法 `/ticket track version 0.31.0`，未找到 `ticket track version ... --version` 類範例，因此無需修改文件內容；「相關範例移除」在本次實作中等價為確認無殘留錯誤範例。
