### 修改摘要

- 刪除 `.claude/hooks/branch-verify-hook.py` 的 inline `find_target_repo`（22 行）
- 在既有 `from git_utils import (...)` block 加入 `find_target_repo` import
- 移除多餘空行

### 兩份實作差異分析

git_utils 版本對 inline 版本是嚴格超集：

| 維度 | inline (W17-149) | git_utils (W17-148) |
|------|-----------------|---------------------|
| 絕對路徑 | 必須 `startswith("/")` | 接受任意路徑（Path.resolve） |
| 相對路徑 | 直接回傳 None | 解析為絕對路徑後搜尋 |
| 終止條件 | `str(check) != "/"` 字串比對 | `current.parent == current` 物件比對（更穩健） |
| 例外處理 | 無 | 捕捉 OSError/RuntimeError |
| .git 為 worktree 檔案 | 支援（`.exists()`） | 支援（`.exists()`） |

呼叫端 `find_target_repo(file_path) if file_path and file_path.startswith("/") else None`
已先行守門絕對路徑，故 hook 實際運行時行為一致。

### 測試調整

`test_find_target_repo_returns_none_for_relative_path` 因 git_utils 對相對路徑會 resolve，
改名為 `test_find_target_repo_returns_none_for_empty_path` 並加註說明。

### 測試結果

- `.claude/hooks/tests/test_branch_verify_cross_repo.py`: 16/16 通過
- `.claude/lib/tests/test_git_utils.py`: 27/27 通過（含 5 個 TestFindTargetRepo）
- `.claude/hooks/tests/test_main_thread_cross_repo.py`: 3/3 通過
