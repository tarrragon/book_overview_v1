### 五 Hook 改動摘要

採 W10-035.3 標示之降級候選方案，N 值起步選擇 10（高/中頻 Hook 一致），settings.json 註冊區塊不動。

| Hook | 候選 | 改動 | 預估削減 |
|------|------|------|---------|
| worklog-format-check | 候選 3 抽樣 N=10 | 新增 should_sample_run + persistent counter（.claude/hook-logs/_sampling/<hook>.count）；非 worklog file 仍提早 return；抽樣命中時才走完整 emoji pattern 檢查 | ~90% |
| utf8-integrity-check-hook | 候選 3 抽樣 N=10 | 同上機制；在 file_paths 解析後、scan 前判斷；命中前 emit 空 hookSpecificOutput | ~90% |
| language-guard-hook | 候選 3 抽樣 N=10 | 同上機制；在 hook_input None 檢查後判斷，避免空輸入也計數失準 | ~90% |
| comment-qa-hook | 候選 4 matcher 限定 | should_process_file 新增 skip_prefixes（tests/ test/ integration_test/ docs/ .claude/）+ 測試命名 pattern + 純文件副檔名（.md/.yaml/.yml/.json/.toml/.txt） | 視變更分布，估 ~60-80% |
| file-type-permission-hook | 候選 1 提醒級別降級 | _print_permission_prompt → _log_permission_prompt（debug log），不再 stderr write；保留 allow JSON output 維持審計鏈 | 100% UI 摩擦削減（仍有 logger.debug 可追溯） |

### 設計決策

- 抽樣機制統一用 file-based persistent counter，重啟 session 後仍延續計數，避免抽樣偏差；計數讀寫失敗時保守執行（return True）符合「降級不犧牲安全網」原則。
- counter 檔案統一存於 `.claude/hook-logs/_sampling/`，不污染既有 hook log 目錄結構。
- comment-qa-hook 的 skip 條件放在 should_process_file 開頭，早於 ParserFactory 載入；既有 `exclude_patterns` 配置仍生效（疊加而非取代）。
- file-type-permission-hook 保留 logger.debug 可追溯，符合「靜默不等於消失」原則。

### 累計削減估算（W10-047.1 + W10-047.2）

W10-047.1（三 Hook，~50%）+ W10-047.2（五 Hook，加權平均 ~80%）合計預估削減 Phase 3b Hook 觸發摩擦約 85%（與 ticket prompt 預估值一致）。觀察期啟動點以本 commit SHA 起算（W10-047.3 acceptance 第 5 條）。
