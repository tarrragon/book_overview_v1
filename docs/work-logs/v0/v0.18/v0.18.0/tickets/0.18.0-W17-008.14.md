## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

PM 在執行中發現需新增 ticket 時，預設行為是「建立同 wave 兄弟 ticket」（如 W17-XXX 系列），即使新議題明確衍生自某 group ticket（如 W17-008）。導致：

- 追蹤鏈斷裂：新 ticket 與 source group 沒有 parent_id / spawned_tickets 關聯
- 雖然 source_ticket 欄位可彌補，但 PM 心智習慣是「建獨立 ticket」而非「建子任務」
- 實例：本 session 三個議題若建為 W17-XXX（如 W17-085/086/087）而非 W17-008.12/.13/.14，會與 source group 失去 children 鏈接

### 影響範圍

- PM 行為規則：`.claude/pm-rules/plan-to-ticket-flow.md`、`ticket-lifecycle.md`
- ticket CLI：`ticket create` 預設行為 / 防呆檢查
- 文件：`.claude/skills/ticket/SKILL.md` 創建決策樹

### 相關 Error Pattern

- 與 ARCH-017 兄弟任務獨立性原則互補（兄弟需獨立，但歸屬位置決策本規則處理）
- 與 PC-073 ANA spawned 區分相關（衍生關係的另一面）

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### WRAP 結論：方案 D（規則文件 + CLI 互動防呆）

#### W. Widen Options

| 選項 | 做法 | 風險 |
|------|------|------|
| A | 純規則文件強化（plan-to-ticket-flow.md 加決策樹） | 依賴 PM 自律 |
| B | ticket create 強制要求 --parent 或 --source-ticket（無則拒絕） | 過嚴，獨立議題受阻 |
| C | ticket create 加互動 prompt：「此 ticket 是否衍生自其他 ticket？」 | 影響 batch-create 流程 |
| **D** | **規則文件 + CLI 偵測：當 wave 同時有 in_progress group ticket 時，create 後輸出提示「→ 是否該建為 <group_id> 子任務？」** | **低（提示不阻擋）** |
| E | 不做：依賴 PM 直覺 | 已被本 session 否決 |

**Consider Opposite**：「同 wave 平行 ticket 也可有效追蹤」？反例：W17-008.10/.11 若無 parent_id，盤點 stuck ANAs 會更難（無法用 children 篩選）。

#### R. Reality-Test

- **基本率**：Linear 的 sub-issue / GitHub 的 task-list / Jira 的 sub-task 皆鼓勵 hierarchy 而非 flat。flat 結構在 ticket 數 > 50 時可讀性陡降
- **Steelman A**：純文件規則在 W17-072（agent body H2 規則）成功落地，證明文件 + Hook 雙通道有效
- **Ooch**：D 的提示行可在 lifecycle.py create handler 加 ~20 LOC，prototype 5 分鐘

#### A. Distance

- **機會成本**：D 占用 CLI 資源最少；C 影響 batch flow
- **10/10/10**：D 短期低感、長期累積追蹤完整度；A 短期高感（PM 讀文件）、長期淡忘
- **優先順序**：P2，與 008.13 同 group，可一併實作

#### P. Prepare

| 方案 | Premortem 主因 | 回退 |
|------|--------------|------|
| D | 提示行被 PM 忽略 → 補：規則 + memory 雙重落地 | 低 |

### 推薦：方案 D（規則 + 提示行 + memory feedback）

**設計**：

1. **規則文件**：
   - `.claude/pm-rules/plan-to-ticket-flow.md` 新增「Ticket 創建位置決策樹」：
     ```
     新 ticket 衍生自既有 ticket？
       是 → 衍生關係類型？
         父子（落地子任務）→ ticket create --parent <id>
         並行獨立議題 → ticket create --source-ticket <id>
       否 → 全新議題 → ticket create（同 wave 兄弟）
     禁止：建為已知 group ticket 的「父任務的兄弟」（追蹤斷裂）
     ```

2. **CLI 偵測**：
   - `ticket create` 不帶 --parent 時，掃描當前 wave 所有 in_progress 且有 children 的 ticket
   - 若存在，輸出單行提示：
     `→ 偵測到 in_progress group：W17-008（11 children）。本 ticket 是否衍生於此？建議 --parent W17-008`
   - 不阻擋（避免誤擋獨立議題）

3. **Memory feedback**：寫入 user memory 「新 ticket 預設考慮 --parent」（跨 session 強化）

**ROI**：規則文件 ~15 LOC + CLI 偵測 ~30 LOC + memory ~5 LOC。
