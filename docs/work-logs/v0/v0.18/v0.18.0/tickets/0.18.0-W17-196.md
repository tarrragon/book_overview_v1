### 修法摘要

兩檔皆為 **B 類（測試陳舊重寫對齊現行 API）**，0 個 D 類，不需衍生 ticket。

#### Group 2: test_stop_hook.py（10 failures → 0）

**根因**：W17-095.2 改造 `should_preserve_pending_json` 簽名與語意：
- 舊：`should_preserve_pending_json(direction: str, logger) -> bool`，僅任務鏈方向回 True
- 新：`should_preserve_pending_json(record: dict, logger, project_root=None) -> bool`，委派 `handoff_utils.is_handoff_stale` 判定，非 stale 即回 True

**修法**：
1. `TestShouldPreservePendingJson` 9 個 case：傳入 dict 而非 str；保留語意對齊新 stale 規則
   - 任務鏈方向（to-sibling/to-parent/to-child）目標未啟動 → 保留
   - 非任務鏈（context-refresh/continuation）來源未 completed → 保留（語意改變，新增測試說明）
   - 新增測試 `test_do_not_preserve_when_from_status_completed` 覆蓋 stale 路徑
   - 移除 `test_do_not_preserve_other_formats`（語意不再適用）
2. `test_auto_direction_completed_ticket_gc`：新代碼先呼叫 lib `is_handoff_stale`，需額外 patch `hook_module.is_handoff_stale` 模擬「來源 completed」狀態觸發 GC

#### Group 3: test_stop_worklog_handoff_sync_check_hook.py（8 failures → 0）

**根因**：W17-176 引入兩道豁免（_is_blocked_this_session / _has_in_progress_ticket）。8 個失敗測試 fixture 用 `in_progress` status，觸發 `_has_in_progress_ticket` → drift 檢查直接 return None。

**修法**：擴充 `hook_mod` fixture，monkeypatch 兩道豁免函式回 False，讓 detect_sync_drift 主路徑可測。本套件聚焦 worklog↔pending 比對邏輯；in_progress 豁免應由獨立測試覆蓋（fixture docstring 已說明）。
