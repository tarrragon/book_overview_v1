### 修復摘要

新增 copyTextBtn 點擊處理器，將 Overview 頁當前顯示/選取的書本以 tab-separated 純文字格式複製到剪貼簿。

### 變更檔案

1. `src/overview/book-exporter.js`
   - EXPORT_CONSTANTS 新增 `NO_DATA_COPY` / `COPY_SUCCESS` / `COPY_FAILED` / `COPY_TEXT_HEADERS`
   - 新增 `async handleCopyText()` 方法：取 getFilteredBooks → 生成文字 → navigator.clipboard.writeText → alert 回饋
   - 新增 `generateCopyText()` 方法：header + 每書一行，欄位 `\t` 分隔，便於貼入 Excel/Sheets

2. `src/overview/overview-page-controller.js`
   - `setupEventListeners()` 在 exportJSONBtn 之後新增 `copyTextBtn.addEventListener('click', ...)` 區塊
   - 透過 W6-012.7.2 的 selection-aware getFilteredBooks DI 自動尊重 selectedBookIds

### 設計決策

- **格式選擇 tab-separated 而非 CSV**：複製到剪貼簿後最常見場景是貼入 Excel/Sheets/Notion 表格，TSV 是這些工具的原生欄位分隔符
- **錯誤處理**：try/catch 包覆 navigator.clipboard.writeText 失敗（HTTPS 限制 / 權限拒絕 / 瀏覽器不支援），給予使用者明確 alert 回饋
- **selection-aware 自動繼承**：不重新實作選取邏輯，直接透過 bookExporter.getFilteredBooks DI（W6-012.7.2 已注入）取得正確範圍

### 不在本 ticket 範圍

- 視覺回饋從 alert 升級為 toast（沿用 exportCSV/exportJSON 既有 alert 行為，維持一致性）
- 自動偵測選取數量顯示「複製 N 本」按鈕文字（屬 UI/UX 增強，非必要）
