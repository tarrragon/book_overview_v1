### 規則本體（最終版）

> **`.claude/` 變更不在 worktree 進行**。subagent 對 worktree 內 `.claude/` 路徑的 Edit/Write 會被 CC runtime hardcoded 拒絕，`additionalDirectories` 無法繞過。詳見 ARCH-015。

派發位置決策：

| Prompt 內容 | 派發位置 |
|------------|---------|
| 含 `.claude/` 路徑 Edit/Write | 主 repo cwd |
| 僅含非 `.claude/` 路徑 | worktree 或主 repo |
| 跨 `.claude/` 與其他路徑 | 拆分為兩次派發 |

### Hook 擴充評估

`agent-dispatch-validation-hook.py` **不擴充**。理由：

1. 此 hook 當前職責是「強制實作代理人使用 worktree 隔離」，與 `.claude/` 路徑檢查職責互相衝突（實作代理人若需改 `.claude/` 反而不該用 worktree）
2. CC runtime 對 `.claude/` 寫入有 hardcoded 拒絕 — subagent 嘗試 Edit 會立即失敗，PM 可即時看到並修正
3. 規則文件 + 派發決策表已落地，PM 派發前查表可避免誤派
4. 若未來統計顯示誤派頻率高，再評估擴充或建立新 hook

### 順手發現

`agent-dispatch-decision.md` 第 17/29/38/50 行對 `additionalDirectories` 在 `src/` 路徑 + acceptEdits 下「即時生效」的描述，與本次 W10-018 的 H4-Option4 實驗結果（即時測試失敗）有部分張力。但本次實驗目標是 `.claude/` 不是 `src/`，且舊描述針對的是 `permissionMode: acceptEdits`（非 default）的場景，未直接衝突。**不擴大本 ticket 範圍處理**，記錄為未來驗證點。
