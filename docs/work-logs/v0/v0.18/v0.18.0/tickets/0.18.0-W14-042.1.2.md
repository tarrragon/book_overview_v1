### 修復摘要

1. **新增 `.claude/skills/ticket/ticket_system/lib/file_lock.py`**
   - 將 `_file_lock` 從 `parser.py` 搬出並 rename 為 public `file_lock`
   - 加上完整模組 docstring（Why / POSIX-only 限制 / 設計決策）
   - 加上 `file_lock()` 巢狀禁止警示 docstring（同 process 同 fd 對同 file 二次 LOCK_EX 會 self-block）
   - 保留 `_HAS_FCNTL` 條件 import + `NotImplementedError` Windows 指引

2. **`parser.py` 縮減**
   - 移除 fcntl import、`_HAS_FCNTL`、`_file_lock` 區段
   - 移除不再需要的 `contextmanager` / `Iterator` import
   - 新增 backward-compat alias：`from .file_lock import file_lock as _file_lock`（noqa: F401），避免破壞既有依賴 `parser._file_lock` 的測試與外部 import

3. **`ticket_builder.py` 抽 `_append_unique_to_list_field` helper**
   - import 從 `parser._file_lock` 改為 `file_lock.file_lock`
   - 新增 helper：統一 load → 防禦正規化 → append-unique → save 序列
   - `update_parent_children` / `update_source_spawned_tickets` 縮減為 4 行：解析 version + guard None + 呼叫 helper
   - 統一 try/except 行為（兩者都統一在 helper 內 try/except IOError/OSError 回傳 False）

### 設計決策

- **保留 `_file_lock` alias 而非全面 rename**：parser 對外仍可 `import _file_lock`，零破壞既有測試與外部使用者；新 caller 應改用 `file_lock.file_lock`
- **helper 命名為 `_append_unique_to_list_field`**：底線前綴標示為 module-internal（兩個 public update_* 才是 API）
- **helper 接受已解析 version 而非 ticket_id**：把版本解析責任留在兩個 public wrapper，避免 helper 多扛一層 guard
- **value 已存在時 early return True**：與原行為一致（兩個原函式都是 `if x not in list:` 之後才 append），避免無謂的 save_ticket I/O

### 不在本 ticket 範圍

- fcntl Windows fallback（已由 W14-042.1.3 處理）
- `_normalize_children` 改名（pepper Phase 3a §2 決策保留現名，避免大量改動）
