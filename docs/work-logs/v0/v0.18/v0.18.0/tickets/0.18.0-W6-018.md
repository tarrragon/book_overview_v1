### 問題根因

Readmoo `read.readmoo.com` 為 hash-based SPA：路由變更只更新 `location.hash`（如 `#/library`），`pathname` 始終為 `/`。popup 程式碼用 `new URL(tab.url).pathname` 取得頁面標識時，書庫頁顯示為「Readmoo (/)」，看起來與首頁無異，誤導用戶與後續 debug。

### 影響範圍

- `src/popup/popup.js:702` — popup 主流程 label 顯示
- `src/popup/popup-event-controller.js:386` — popup event controller 同邏輯

兩處皆為 label 顯示，無 functional 行為依賴（按鈕啟用條件、訊息分派等不依賴此值）。

### 相關 Error Pattern

無直接對應 PC 模式；此屬 SPA URL 解析誤用，跨專案通用知識（hash-based vs path-based routing）。

### 重現

| URL | 修復前顯示 | 修復後顯示 |
|-----|----------|----------|
| `https://read.readmoo.com/#/library` | `Readmoo (/)` | `Readmoo (/#/library)` |
| `https://readmoo.com/` | `Readmoo (/)` | `Readmoo (/)` |
| `https://member.readmoo.com/account` | `Readmoo (/account)` | `Readmoo (/account)` |

修復後對 hash 路由可正確區辨；對傳統 path（hash 為空字串）顯示效果不變。
