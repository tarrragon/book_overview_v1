### 修復摘要

於 process-skip-guard-hook.py 新增 active-dispatch guard：

1. **新增 has_active_dispatch() helper**（process-skip-guard-hook.py:160-187）
   - 透過 _find_repo_root() 定位 repo 根目錄
   - 讀取 .claude/dispatch-active.json
   - 支援新格式 {"dispatches": [...]} 與舊格式 bare list（防退）
   - JSON 損毀 / 檔案不存在 / repo 找不到 → 回傳 False（不阻擋 hook）

2. **main() 流程套用 guard**（process-skip-guard-hook.py:307-316）
   - 偵測到 skip_type 後（cold path：無 skip 不查），先呼叫 has_active_dispatch()
   - 若有 active dispatch → 輸出基本 JSON 並 return EXIT_SUCCESS（靜音任何 skip 類型）
   - 例外時 logger.warning 記錄並回退原行為（規則 4 雙通道：file log + warning）

3. **覆蓋範圍**：套用於 main() 內所有 skip_type，不僅 SKIP_ACCEPTANCE/SKIP_SA_REVIEW；理由是代理人執行中的用戶輸入對任何 skip 類別都易誤判。

### 設計決策

- **為何放在 type/phase guard 之前**：active-dispatch 是更上位的執行狀態訊號，先過濾再做 type/phase 細分判斷。
- **為何 cold path**：dispatch-active.json 讀取頻繁會增 hook 延遲；只在已偵測到 skip intent 時查詢。
- **為何兼容舊 bare list 格式**：避免歷史檔案在升級期失效。

### 不在本 ticket 範圍

- dispatch-active.json 寫入端格式變動（屬 dispatch-record-hook 範疇）
- 其他 hook 套用 active-dispatch guard（如有需要另開 ticket）
