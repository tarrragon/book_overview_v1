### 結論：確認 ticket-quality-gate-hook 誤觸發，需 spawn IMP 修復配置

#### 修復方向（候選排序）

**方向 1（首選）：路徑黑名單擴展 + keyword 強化**

修改 `.claude/config/quality_rules.yaml`：

1. 新增 `ticket_path_excludes`（黑名單先於 keywords 檢查）：
   ```yaml
   ticket_path_excludes:
     - ".claude/error-patterns/"
     - ".claude/methodologies/"
     - ".claude/rules/"
     - ".claude/references/"
     - ".claude/pm-rules/"
     - ".claude/skills/"
     - ".claude/hooks/"
     - ".claude/agents/"
   ```

2. 強化 `ticket_path_keywords`：將含路徑分隔符的 substring 改為 anchored 路徑：
   - 移除 `-ticket-`（過於寬泛）
   - 改為 `tickets/` + `task-`（限定父目錄）

3. Hook 程式碼 line 144 前插入 exclude 檢查：
   ```python
   ticket_path_excludes = trigger_config.get("ticket_path_excludes", [])
   if any(exclude in file_path for exclude in ticket_path_excludes):
       logger.info(f"檔案路徑命中排除規則: {file_path}")
       return False
   ```

**ROI**：
- 影響範圍：`.claude/` 下 50+ 含 `-ticket-`/`-task-` 子字串檔案
- 成本：~30-45 分鐘（hook + config 修改 + 既有測試補強）
- 收益：消除 PM/agent 撰寫框架文件的誤觸發雜訊
- 風險：移除 `-ticket-` keyword 可能漏接非標準路徑的真實 ticket（風險低，現代 ticket 都在 `docs/work-logs/v*/tickets/`）

**方向 2（備選）：frontmatter 識別**

Hook 入口判斷檔案是否含 ticket frontmatter（`id: X.Y.Z-WN-NNN` 格式 + `acceptance:` 欄位）才視為 ticket。

**ROI**：成本 ~45-60 分鐘，但邏輯較精確，未來新增資產類型不需更新黑名單。

**方向 3（暫不採納）：完全移除 substring 匹配**

改要求所有 ticket path 必須含 `tickets/` 父目錄，但風險較高（影響既有測試與外部引用）。

#### 後續處理決策

- **Spawn IMP**：是（修復方向 1）
- **不併入 W10-072.1 範圍**：W10-072.1 處理 phase-completion-gate-hook，本 ticket 確認不同 hook（ticket-quality-gate），分開派發更聚焦
- **不關閉本 ANA**：實驗結論已產出可執行修復方向，等 spawned IMP 完成後 complete
