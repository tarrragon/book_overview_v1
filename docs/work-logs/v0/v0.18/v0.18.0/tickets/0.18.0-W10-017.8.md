### 3. checkpoint_state() 函式簽名

位置：ticket_system/lib/checkpoint_state.py

簽名：
  def checkpoint_state(
      ticket_id: Optional[str] = None,
      *,
      use_cache: bool = False,
      log_metrics: bool = True,
  ) -> CheckpointState

語意：
- ticket_id=None 使用當前 in_progress 推導
- use_cache: 60 秒內快取（Phase 1 僅留接口）
- log_metrics: 是否寫入 pm-automation-metrics.jsonl
- Raises: CheckpointStateError（關鍵來源失敗且無 fallback）

資料來源優先順序（fail-open）：

| 順序 | 資料來源 | 失敗 fallback |
|------|---------|-------------|
| 1 | git status --porcelain | uncommitted_files=None + 記 log（區分於 0 = clean） |
| 2 | .claude/state/dispatch-active.json | 缺檔時 active_agents=0 |
| 3 | .claude/handoffs/pending/*.json | 缺目錄時 active_handoff=None |
| 4 | ticket CLI query（當前版本 in_progress） | 錯誤時空清單 |
| 5 | git worktree list --porcelain | 錯誤時空清單 |

單一來源失敗不阻斷整體；data_sources 欄位記錄錯誤，pending_checks 以 auto_detectable=False 標記。

### 4. Checkpoint 推導邏輯（核心決策）

「從高到低第一個命中」的優先級：

| 優先級 | 判定條件 | current_phase | next_action |
|-------|---------|---------------|------------|
| 1 | active_agents > 0 | "1.85" | 等待代理人完成或 ticket track agent-status |
| 2 | unmerged_worktrees != [] | "1.9" | 合併 worktree 並清理 |
| 3 | uncommitted_files > 0 | "1" | git add + git commit |
| 4 | active_handoff is not None | "2" | ready for /clear（有待恢復 handoff） |
| 5 | in_progress_tickets 非空且 ticket_id 指定 | "0.5" | ticket track append-log 記錄階段 |
| 6 | 其他 | "3" | ready for /clear 或選下個 Ticket |

ready_for_clear = (current_phase in {"2","3"}) AND pending_checks 全為 auto_detectable=False

#### 1.2 checkpoint_state() 主流程虛擬碼

```
FUNCTION checkpoint_state(ticket_id=None, *, use_cache=False, log_metrics=True, caller=None):
  start_timer = perf_counter()
  errors = {}
  pending = []

  # Step 1: 5 層資料收集（fail-open）
  uncommitted    = SAFE_CALL(_read_git_status,           errors, pending, 'git-status',      fallback=None)
  agents, raw    = SAFE_CALL(_read_dispatch_active,      errors, pending, 'dispatch-active', fallback=(0, {}))
  active_handoff = SAFE_CALL(_read_handoff_pending,      errors, pending, 'handoff-pending', fallback=None)
  in_progress    = SAFE_CALL(_query_in_progress_tickets, errors, pending, 'ticket-query',    fallback=[])
  worktrees      = SAFE_CALL(_read_git_worktrees,        errors, pending, 'git-worktree',    fallback=[])

  # Step 2: Checkpoint 推導（高→低，第一個命中）
  phase, label, action = _derive_checkpoint(agents, worktrees, uncommitted, active_handoff, in_progress, ticket_id)

  # Step 3: ready_for_clear
  ready = (phase IN {'2','3'}) AND ALL(c.auto_detectable == False for c in pending)

  # Step 4: 建構 state
  state = CheckpointState(..., data_sources=materialize(errors), computed_at=utc_now_iso())

  duration_ms = (perf_counter() - start_timer) * 1000

  # Step 5: 觀測 log（fail-open；規則 4 stderr+log 雙通道）
  IF log_metrics:
    TRY _write_metrics_log(state, caller, duration_ms, errors)
    EXCEPT Exception AS e: stderr.warning(f'metrics log failed: {e}')  # NOT raise

  RETURN state
```

#### 1.3 SAFE_CALL 輔助（fail-open 核心）

Evidence + linux 審查：縮窄 except whitelist 明列 I/O 類型，避免吞掉程式 bug（MemoryError / AttributeError / ImportError 等應上拋）。對應 `.claude/error-patterns/implementation/IMP-025` 靜默例外必須文件化要求。

```
# I/O 類例外白名單（走 fallback）
IO_ERRORS = (
  subprocess.CalledProcessError,   # git / ticket CLI 非零退出
  subprocess.TimeoutExpired,       # subprocess 超時
  FileNotFoundError,               # 檔案或目錄不存在
  PermissionError,                 # 權限拒絕
  json.JSONDecodeError,            # JSON 解析失敗（資料源毀損）
  OSError,                         # 其他 I/O（含 NotADirectoryError / IsADirectoryError）
)

FUNCTION SAFE_CALL(fn, errors, pending, source, fallback):
  TRY:
    RETURN fn()
  EXCEPT IO_ERRORS AS e:                      # 僅 I/O 類走 fallback
    errors[source] = f'{type(e).__name__}: {str(e)[:100]}'
    pending.APPEND(PendingCheck(
      check_id=f'data_source_{source}',
      reason=f'{source} unavailable: {e}',
      blocker=False,
      auto_detectable=False))
    RETURN fallback
  # 其他例外（MemoryError / AttributeError / ImportError / KeyError 等）
  # 視為程式 bug，不在此捕獲 → 上拋到 checkpoint_state() → 依規則 4 stderr + log 雙通道
```

捕獲判準（文件化）：
- 捕獲（走 fallback）：資料來源不可用、權限拒絕、JSON 毀損、subprocess 失敗、超時
- 不捕獲（上拋）：型別錯誤（AttributeError / TypeError）、記憶體不足、匯入失敗、鍵值錯誤（KeyError / IndexError）、中斷信號（KeyboardInterrupt / SystemExit 屬 BaseException 本就不會被 `except Exception` 捕獲）

#### 1.4 _derive_checkpoint（資料驅動 priority table）

linux Good Taste 審查建議：用 priority table 取代 if/elif 鏈，新增優先級只動資料不改控制流。

```
# 模組層級常數（純資料結構）
PRIORITIES = [
  # (predicate, phase, label, action_template)
  (LAMBDA s: s.active_agents > 0,
   '1.85', 'C1.85 代理人運行中',
   LAMBDA s: f'等待 {s.active_agents} 個代理人或 ticket track agent-status'),

  (LAMBDA s: LEN(s.unmerged_worktrees) > 0,
   '1.9', 'C1.9 worktree 待合併',
   LAMBDA s: f'合併 {LEN(s.unmerged_worktrees)} 個 worktree 並清理'),

  (LAMBDA s: s.uncommitted_files IS NOT None AND s.uncommitted_files > 0,
   '1', 'C1 未提交變更',
   LAMBDA s: f'git add + git commit ({s.uncommitted_files} 檔)'),

  (LAMBDA s: s.active_handoff IS NOT None,
   '2', 'C2 handoff 就緒',
   LAMBDA s: f'ready for /clear (handoff pending: {s.active_handoff})'),

  (LAMBDA s: LEN(s.in_progress_tickets) > 0 AND s._ticket_id IS NOT None,
   '0.5', 'C0.5 階段進行中',
   LAMBDA s: 'ticket track append-log 記錄階段進展'),
]
FALLBACK = ('3', 'C3 流程完成', LAMBDA s: 'ready for /clear 或選下個 Ticket')

FUNCTION _derive_checkpoint(state):
  # state 含 active_agents / unmerged_worktrees / uncommitted_files / active_handoff / in_progress_tickets / _ticket_id
  FOR predicate, phase, label, action_fn IN PRIORITIES:
    IF predicate(state):
      RETURN (phase, label, action_fn(state))
  # 全部未命中 → fallback
  phase, label, action_fn = FALLBACK
  RETURN (phase, label, action_fn(state))
```

Linus Good Taste 效益：
- 新增優先級（例 Checkpoint 0.8）：只需在 PRIORITIES append 一筆，無需修改 _derive_checkpoint 控制流
- 優先級重排：純資料順序調整，不動函式
- 測試 C1-C8 仍然有效（斷言輸入 state 推導結果）
- uncommitted_files 採 Optional[int] 後，predicate 以 `IS NOT None AND > 0` 明確區分「資料源失敗」與「無未提交變更」

```
FUNCTION _write_metrics_log(state, caller, duration_ms, errors):
  log_path = .claude/logs/pm-automation-metrics.jsonl
  ENSURE log_path.parent EXISTS  # mkdir -p, exist_ok=True

  # Rotate（預寫檢查）
  IF log_path EXISTS AND log_path.size > 10_MB:
    rotate = log_path.with_suffix('.1.jsonl')
    IF rotate EXISTS: rotate.DELETE     # 僅保留 1 份歷史
    log_path.RENAME_TO(rotate)

  entry = {
    'ts': state.computed_at, 'event': 'checkpoint_state',
    'caller': caller or 'unknown', 'ticket_id': ticket_id or '',
    'current_phase': state.current_phase, 'ready_for_clear': state.ready_for_clear,
    'active_agents': state.active_agents, 'uncommitted_files': state.uncommitted_files,
    'duration_ms': ROUND(duration_ms, 2),
    'data_source_errors': [k for k in errors if errors[k] != 'ok']
  }

  WITH log_path.open('a', encoding='utf-8') AS f:
    f.WRITE(json.dumps(entry, ensure_ascii=False) + '\n')
```

## TDD Phase 2 測試設計（sage-test-architect, 2026-04-18）

### 1. 測試策略總覽

採用 Sociable Unit Tests 原則（Kent Beck）：
- Unit = checkpoint_state module（checkpoint_state.py 全部公開 API）
- Isolation = 只隔離外部世界：git subprocess / 檔案系統 / ticket CLI subprocess
- 不 Mock 內層 dataclass（CheckpointState / PendingCheck），直接用真實物件斷言

分層：
- 純單元測試（85%）：mock 5 個資料來源 → 驗證 dataclass 建構、推導邏輯、log 契約
- 整合測試（10%）：真實 git repo 與 tmp 檔案系統下跑 checkpoint_state() 一次端到端
- 效能測試（5%）：time.perf_counter() 包呼叫，中位數 < 100ms

### 3. GWT 場景群組（行為鏈式推演）

#### Group A: CheckpointState / PendingCheck dataclass 建構

功能職責：dataclass 欄位契約 / frozen 性 / 預設值。跨群組依賴：無。

- A1（正常）：Given 合法 12 欄位 + 空 pending_checks，When 建構 CheckpointState，Then 所有欄位可讀 + computed_at 為 ISO 格式
- A2（正常）：Given 3 個 PendingCheck（auto_detectable True/False 混合），When 建構，Then pending_checks 長度=3 + frozen（dataclasses.FrozenInstanceError on write）
- A3（邊界）：Given 全空 list 欄位，When 建構，Then ready_for_clear 可推導
- A4（邊界）：Given dataclasses.fields() 計數，Then >= 12（AC1 判定）
- A5（異常）：Given 必填欄位缺失，When 建構，Then 拋 TypeError

#### Group B: 5 層 fail-open 資料來源

功能職責：subprocess / filesystem 隔離。跨群組依賴：conftest 提供 fixture。

對每個資料來源（git-status / dispatch-active / handoff-pending / ticket-query / git-worktree）設計四維場景：

- B{N}.1（正常）：資料來源回傳合法內容 → CheckpointState 對應欄位正確
- B{N}.2（異常）：資料來源拋 subprocess.CalledProcessError 或 FileNotFoundError
  - 前置驗證：mock 確認被呼叫一次
  - Then：對應欄位走 fallback（uncommitted_files=None / active_agents=0 / active_handoff=None / 空清單）
  - Then：data_sources[source_name] 欄位記錄錯誤訊息
  - Then：pending_checks 追加 auto_detectable=False 標記
  - Then：整體函式不拋例外（fail-open）
- B{N}.3（邊界）：空回傳（空 json / porcelain 空字串 / 空目錄）→ 對應欄位預設值
- B{N}.4（中斷）：多個資料來源同時失敗 → 所有 fallback 串聯成立，函式仍回傳
- B{N}.5（邊界：目錄不存在）：.claude/state/ 或 .claude/handoffs/ 整個目錄不存在（非僅檔案缺失），Path.iterdir 拋 FileNotFoundError → SAFE_CALL whitelist 捕獲 → 走 fallback（active_agents=0 / active_handoff=None），pending_checks 追加 auto_detectable=False 標記

優先級對照：對應 Phase 1 §3 資料來源表。

#### Group C: 6 優先級 Checkpoint 推導（核心決策）

功能職責：決策函式。跨群組依賴：Group A 真實 CheckpointState。

前置驗證原則：每個 case 先 assert 輸入 mock state 的預期值（active_agents / uncommitted_files 等），再斷言推導結果。

- C1（優先級 1）：Given active_agents=2 + 其他條件全觸發，When checkpoint_state() 執行，Then current_phase="1.85" + next_action 提示等待代理人
- C2（優先級 2）：Given active_agents=0 + unmerged_worktrees=["wt1","wt2"] + 其他觸發，Then current_phase="1.9" + next_action 提示合併 worktree
- C3（優先級 3）：Given active_agents=0 + unmerged=[] + uncommitted_files=3，Then current_phase="1" + next_action 提示 git commit
- C4（優先級 4）：Given 前三項全 clean + active_handoff="W10-017.8"，Then current_phase="2" + ready for /clear
- C5（優先級 5）：Given 前四項全 clean + in_progress_tickets=["X"] + ticket_id 指定，Then current_phase="0.5" + next_action 提示 append-log
- C6（優先級 6 fallback）：Given 全部 clean + 無 in_progress，Then current_phase="3" + next_action 提示選下個 Ticket
- C7（邊界：優先級順序驗證）：Given 優先級 1 與 3 同時觸發（active_agents=2 + uncommitted=3），Then 命中優先級 1（"1.85"），不被優先級 3 覆蓋
- C8（邊界：ready_for_clear 推導）：Given current_phase="3" + pending_checks 全 auto_detectable=False，Then ready_for_clear=True；若任一 auto_detectable=True 未通過，Then ready_for_clear=False

AC5 判定：C1-C8 共 8 個 case，滿足「>= 8 test case」。

#### Group D: 觀測 log JSONL 寫入契約

功能職責：橫切關注點。跨群組依賴：mock checkpoint_state 內部呼叫。

- D1（正常）：Given log_metrics=True + 合法 state，When 呼叫完成，Then .claude/logs/pm-automation-metrics.jsonl 末尾追加一行 JSON，含 ts/event/caller/ticket_id/current_phase/ready_for_clear/active_agents/uncommitted_files/duration_ms/data_source_errors 10 欄位
- D2（正常）：Given 連續 3 次呼叫，Then 檔案行數 += 3（append 非覆蓋）
- D3（邊界）：Given log_metrics=False，Then 檔案未被寫入（stat mtime 不變）
- D4（異常）：Given 日誌檔寫入失敗（chmod 444 或磁碟滿 mock），When 呼叫，Then 主流程不拋例外（fail-open 規則 4）+ stderr 有 warning
- D5（邊界：rotate）：Given 既有 log > 10MB，When 呼叫，Then 自動 rename 為 .1.jsonl + 新檔從 0 開始 + 保留一份歷史
- D6（契約）：Given caller 參數="snapshot"，Then log 行 caller 欄位="snapshot"（驗證呼叫端識別）
- D7（契約）：Given data_source_errors 非空，Then JSON 欄位為 list of string，非 truncate

#### Group E: 效能測試（AC6）

- E1：Given 真實 tmp git repo（含 10 未提交檔 + 模擬 dispatch-active 含 2 active + 3 in_progress tickets），When 執行 checkpoint_state() 100 次，Then 中位數 duration_ms < 100 + p95 < 150
- E2：Given use_cache=False vs use_cache=True（Phase 1 noop），Then 行為一致（use_cache 留接口）

#### Group F: 模組邊界測試（防循環依賴）

- F1：Given import checkpoint_state 模組，When 檢查 sys.modules，Then 不含 track_snapshot / track_query（防止循環）
- F2：Given 模組原始碼 AST 解析，Then import 清單僅含 paths / constants / handoff_utils / subprocess / dataclasses / datetime / json / pathlib / typing / time / sys
- F3：Given 反向：三投影 CLI（snapshot/handoff-ready/checkpoint-status）import，Then 可 import checkpoint_state
