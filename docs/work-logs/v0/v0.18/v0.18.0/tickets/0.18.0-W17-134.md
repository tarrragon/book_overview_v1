## 變更摘要

擴充 `.claude/rules/core/decision-trigger-binding.md` v1.0.0 → v1.1.0：

1. **規則 1 禁止詞表**：加入「將來」
2. **新增規則 1.5「適用邊界：程式碼/文件 vs worklog/ticket」**：
   - 表格區分四種載體（程式碼註解 / 規則方法論 / Worklog ticket / Phase 4 結論）的嚴格度
   - 三明示完整：Why 引摩擦力方法論（所有任務都有排程權重、難題優先、時間非權重單位）；Consequence 時間 trigger 在繁忙期系統性失效；Action 區分排程問題 vs 決策問題
3. **規則 2 替代方案表**：新增兩列非時間 trigger（ticket 計數 / 版本錨點 ticket）
4. **規則 4 違規偵測表**：加入「將來」「下週」「下個月」字面
5. **與其他規則的邊界**：新增 friction-management-methodology.md 引用列
6. **檢查清單**：同步詞表 + 新增規則 1.5 載體判別項

## 設計決策

- 採規則 1.5 編號：與規則 1「兩種合法狀態」緊鄰，邏輯延伸關係明確
- 載體分類四列：區分「設計脈絡傳遞」與「執行追蹤」兩種文件性質
- 時間單位禁令直接引摩擦力方法論：所有任務皆有排程權重，時間非合法權重單位
