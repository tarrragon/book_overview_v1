## 前提失效確認（claim 前獨立驗證）

**原計畫前提**：W9-002/003/004 三 ANA 的共通元結構「原則層 → Skill 層 → Hook 層三層防護」可提煉為跨領域方法論。

**否決證據**（PC-066 line 131-133，2026-04-15 多視角審查後修訂）：

> 三 ANA 暴露同樣的結構問題（原則正確但執行落差），但**防護方式因領域差異而不同**：PC-060 是工具發現規則化、PC-061 是 memory 升級評估規則化、PC-066 是決策觸發 Hook 化。**三者沒有可重用的「三層元架構」**，將其抽象為通用結構是錯誤的概念升維。

**加強證據**（PC-067 line 32-40，W10-008 活案例）：

> W9-004 規劃「規則層 + Skill 層 + 持續層」三層防護元架構。W10-008 PM 直接執行此規劃：過程中無人質疑「三層複寫同概念」是 DRY 違反。linux 多視角審查評分 Acceptable 偏 Garbage，揭示「元架構是空洞包裝」「WRAPCheck 是儀式稅」。

**元結構失效的根因**（W9-002/003/004 映射到實際 PC）：

| ANA | 個別 PC | 防護方式 | 抽象可行性 |
|-----|--------|---------|----------|
| W9-002 | PC-060 | 工具發現 rule + tool-discovery SKILL | 領域獨立 |
| W9-003 | PC-061 | memory 升級評估 rule + continuous-learning SKILL | 領域獨立 |
| W9-004 | PC-066 | Hook + CLI claim 強制（單點） | 領域獨立 |

三者已各自最佳化，**沒有跨領域可重用層**。強加三層抽象會重複定義觸發條件（DRY 違反，PC-066 W10-008 已驗證）。
