## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 實作摘要

1. **create.py wire-in**
   - 新增 `_auto_extract_context_bundle_post_create(version, ticket_id, quiet, verbose)` helper
   - 於 `execute()` 中 `_persist_and_report` 回傳 0 後呼叫
   - 僅當 target ticket 具 `source_ticket` / `blocked_by` / `related_to`（含 camelCase 變體）才觸發
   - 異常以 `traceback.format_exc()` 寫入 stderr，退出碼保 0（§v2.3 Non-raising / §S20）
   - register 新增 `--quiet` / `--verbose` argparse flag

2. **lifecycle.py wire-in**
   - 新增 `_auto_extract_context_bundle_post_claim(version, ticket_id, quiet, verbose)` helper
   - 於 `execute_claim` 中 `lifecycle.claim_with_verification` 回傳 0 後呼叫
   - 幂等性由 `merge_auto_extracted_block` 的 sources 主鍵保證（no_change_idempotent）
   - 同樣異常降級

3. **track.py**：claim subparser 新增 `--quiet` / `--verbose` flag

4. **模組架構**：不修改 `lib/context_bundle_extractor.py`（cinnamon 並行審查中）；wire-in 純 delegate 到 extractor 既有公開 API。

### 觸發侵入點

| 檔案 | 位置 | 行為 |
|------|------|------|
| commands/create.py | execute() 末端（rc==0 時） | post-persist enhancement |
| commands/lifecycle.py | execute_claim 末端（rc==0 時） | post-claim enhancement |

### 設計取捨

- **為何不用 execute_append_log 的 replace_section flag？** W17-002 Phase 1 BLK-1 PC-093 移除此追蹤；`extract_and_write_context_bundle` 已內部完成 render+merge+save，wire-in 只需 delegate，不需再動 append_log 路徑。
- **為何 claim 不先檢查 section 是否已填？** Idempotent merge 已自動處理；若 sources 未變 → `no_change_idempotent` 直接 return，無額外 I/O。
