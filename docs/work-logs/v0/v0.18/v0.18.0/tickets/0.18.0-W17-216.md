### 問題根因

memory `feedback_worktree_agent_issues.md`（2026-04-13 標記 partial upgrade）含三個子項：

| 子項 | 內容 | 升級狀態 |
|------|------|---------|
| 1 | `.claude/` 路徑在 worktree 被 Hook 攔截 | 已升級至 ARCH-015 |
| 2 | 代理人超出 Ticket scope（重試擴大範圍） | 已升級至 `.claude/rules/core/pm-role.md` 重試守則 |
| 3 | Worktree 不自動合併依賴分支（blockedBy 需手動 merge） | **保留於 memory，未升級為 ticket trigger** |

子項 3 描述：`worktree create` 只從 main 建立分支，不解析 Ticket 的 `blockedBy` 欄位；派發前需在 prompt 中手動加入 `git merge feat/{dependency}` 前置步驟。

### 與 W17-215 的關聯

Claude Code v2.1.143 新增 `worktree.bgIsolation: "none"`，啟用後背景 session（`claude --bg` / `claude agents`）**完全不建 worktree**，直接編輯工作副本。

W17-215 已在本專案啟用此設定（commit 83fc350e）。

### 待釐清問題

`bgIsolation:none` 啟用後，子項 3「worktree 不自動合併依賴分支」議題：

- 是否已被「規避」（背景不建 worktree → 沒有依賴合併需求）？
- 是否仍存在於「前景 `claude -w <name>` 手動建立 worktree」場景？
- memory 子項 3 是否應標記為 resolved 或重新定義範圍？

### 影響範圍

- memory `feedback_worktree_agent_issues.md` 子項 3 狀態欄位
- 若仍存在，需 spawn IMP ticket 追蹤剩餘範圍（避免 PC-093 無 trigger 延後）
- 若已解，可從 memory 移除子項 3 或標記 resolved

### 相關 Error Pattern

- PC-093: YAGNI 延後決策累積（子項 3 屬「保留」狀態 1+ 年屬高風險）
- ARCH-015: subagent `.claude/` 寫入保護（子項 1 已升級對照）
