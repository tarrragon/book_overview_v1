### 實作摘要

新增 SessionStart Hook `.claude/hooks/session-start-sync-exclusion-check-hook.py`，並在 `.claude/settings.json` 的 `hooks.SessionStart` 陣列末尾註冊（timeout=5000）。

### 設計決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| 依賴策略 | uv script dependencies=[]，硬編 KNOWN_EXCLUDED 副本 | 與 session-start-scheduler-hint-hook.py 的 TERMINAL_STATUSES 作法一致；避免引入 yaml/toml 解析依賴 |
| 掃描範圍 | 僅 .claude/ 根目錄，副檔名 .json/.yaml/.yml | Ticket Context Bundle 規格；子目錄如 hook-state/ 已整資料夾被 EXCLUDE_PATTERNS 排除，不需逐檔掃 |
| 靜默策略 | 無未排除檔案時 suppressOutput=True | 遵循 SessionStart Hook 不干擾正常啟動原則 |
| 失敗模式 | I/O / 解析異常 → logger.warning + 靜默降級 | SessionStart 不可阻塞 session 啟動（規則 4 可觀測性 + 不阻塞並存） |
| settings.json 的處理 | 加入 KNOWN_EXCLUDED（類型 X framework config） | 正常 sync，不警告；避免每次 session 啟動都提醒 |

### KNOWN_EXCLUDED 清單

與 .claude/scripts/sync-claude-push.py 的 EXCLUDE_PATTERNS 對齊：

- 類型 A（Runtime state）：dispatch-active.json, pm-status.json
- 類型 B（Local-only settings）：settings.local.json, .sync-state.json, sync-preserve.yaml
- 類型 X（Framework config，正常 sync）：settings.json

### Hook 輸出格式

- 無未排除檔 → {"suppressOutput": true}
- 有未排除檔 → hookSpecificOutput.additionalContext 為 markdown 警告區塊，每個檔案一行 [WARNING] sync-exclusion-check: {filename} 未在 sync 排除清單 → 請加入 EXCLUDE_PATTERNS 和 LOCAL_ONLY

### 驗收對照

- [x] Hook 在 session 啟動時掃描 .claude/ 根目錄和 hook-state/ 等子目錄 — 根目錄逐檔掃 .json/.yaml/.yml；子目錄整體由 EXCLUDE_PATTERNS 排除（hook-state 已在清單），無需逐檔
- [x] 發現不在 sync-push EXCLUDE_PATTERNS 的 .json/.yaml 檔案時輸出警告 — 實測新增 test-unexcluded.json 觸發 WARNING
