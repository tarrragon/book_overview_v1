### 問題根因

`.claude/hooks/tech-debt-reminder.py:113` 呼叫 `yaml.safe_load(frontmatter_text)`，但模組頂部 imports 完全沒有 `import yaml`。Python 執行到該行拋出 `NameError: name 'yaml' is not defined`，被外層 try/except 吞掉（line 116-118），對所有 ticket 都靜默 return None。

整個 tech-debt-reminder hook 的掃描功能（scan_tech_debt_tickets → extract_frontmatter）實質失效：無論 ticket 真實狀態如何，都無法解析 frontmatter，技術債務提醒永遠不會觸發。

### 影響範圍

- `.claude/hooks/tech-debt-reminder.py` 單檔案
- 功能影響：SessionStart Hook 載入 tech-debt-reminder 時，所有 TD ticket 掃描靜默失敗
- 唯一 caller：`scan_tech_debt_tickets`（同檔內），caller 寫法 `if not frontmatter: continue` 對 None 和 {} 都語意正確，故修復後不需改動 caller

### 方案決策

採方案 B（改用 hook_utils.parse_ticket_frontmatter），原因：

1. **DRY 對齊**：hook_utils.hook_ticket 已有 public `parse_ticket_frontmatter(content_or_path, logger) -> dict`，無需重新發明
2. **避免 ARCH-020 共振**：方案 A（加 import yaml）會讓本 hook 維護獨立 YAML 解析路徑，與 hook_utils 形成跨進程同邏輯重複，未來修法漂移風險高
3. **延續 W17-188 方向**：source_ticket W17-188 已將此 hook 改用 hook_utils 共用 helper，繼續沿用方向一致
4. **無需暴露新 API**：`parse_ticket_frontmatter` 已是 public（hook_utils/__init__.py line 37, 71 export），不需要新增 helper

### 變更摘要

`.claude/hooks/tech-debt-reminder.py`：

1. `from hook_utils import (...)` 增加 `parse_ticket_frontmatter`
2. 移除未使用的 `import re`（原 frontmatter regex 不再需要）
3. `extract_frontmatter` 改為薄包裝：呼叫 `parse_ticket_frontmatter(file_path, logger)`，保留原 `Optional[Dict]` API 契約（空 dict → None 轉換，讓 caller 的 `if not frontmatter` 行為不變）
4. 加上 W17-189 修復註解說明來龍去脈

### API 契約保持

- 原 `extract_frontmatter`：`(file_path, logger) -> Optional[Dict]`，失敗回 None
- 新 `extract_frontmatter`：相同簽章，相同回傳契約（內部呼叫 parse_ticket_frontmatter 後將空 dict 轉 None）
- 唯一 caller `scan_tech_debt_tickets` 用 `if not frontmatter: continue`，None 和 {} 行為一致，無需改動
