### ARCH-020 三次重爆共因（已驗證）

每次修復只修「最近發現的 caller」，未掃所有實作位置。修復者心智模型「我修了觸發 bug 的檔案」≠「我修了所有使用此 predicate 的檔案」。

| 重爆 | 修了什麼 | 漏了什麼 |
|------|---------|---------|
| W17-165 | stop hook 自身 is_ticket_completed | prompt-reminder hook + lib 層 |
| W17-176.2.1 | prompt-reminder hook | lib 層 is_ticket_completed / is_ticket_in_progress_or_completed |
| W17-181 | 識別完整消費者清單（8 處） | 待 W17-181.1/.2 執行 |

### W17-179 false negative 根因

ANA 驗證了 stop hook 自身函式（已修），但未驗證 is_handoff_stale 呼叫的是 lib 版而非 hook 版。**ANA 驗證範圍只到「直接被問的函式」，未追蹤 callees 呼叫鏈**。
