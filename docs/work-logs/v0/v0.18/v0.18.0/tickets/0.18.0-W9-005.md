## Problem Analysis

## 問題根因

**歷史誤解**：將「代理人失敗」統一歸因為「context 不足」，統一升級所有代理人 model 至 opus 1m。

**實際根因**（經 W9-004 釐清）：代理人失敗有 3 個獨立原因，被誤混為一：
1. **回合限制**（tool call ~20）— 任務拆分 / cognitive load 問題
2. **PM 決策品質下降**（context 累積降低判斷力）— W9-004 處理
3. **model 配置不當**（本 ticket 處理）— 資源浪費問題

**結果**：24/26 代理人統一為 opus 1m，其中多數任務特性不需要：
- 結構化規則驗證（acceptance-auditor, project-compliance 等）用 opus 1m 浪費
- 機械性格式修復（mint-format-specialist）用 opus 1m 極度浪費

## 影響範圍

- 26 個活躍代理人（排除 2 個 DEPRECATED）
- 影響層面：派發成本、執行速度、回合限制下的決策品質

## agent-model-matrix（26 個代理人）

| 類別 | model | 數量 | 代表代理人 |

| D (1m) | claude-opus-4-6[1m] | 2 | saffron-system-analyst, bay-quality-auditor |

| B (sonnet) | sonnet | 7 | acceptance-auditor, coriander, oregano, project-compliance, sassafras, sumac, thyme-doc |
| A (haiku) | haiku | 1 | mint-format-specialist |
| inherit | inherit | 1 | rosemary-project-manager |

## 分類邏輯（4 維度）

| 維度 | 等級 |
|------|------|
| 閱讀量 | S(單檔) / M(數檔) / L(整模組) / XL(跨模組) |
| 決策複雜度 | 低(機械) / 中(規則推理) / 高(設計判斷) / 極高(架構決策) |
| 輸出量 | 短 / 中 / 長 |
| 對話深度 | 單輪 / 2-3 輪 / 多輪 |

- **D 條件**：閱讀量 = XL（系統級審查）
- **C 條件**：決策複雜度 ≥ 高，或實作型代理人
- **B 條件**：決策複雜度 = 中（規則推理），結構化產出
- **A 條件**：決策複雜度 = 低（純機械執行）
