## 實作完成

### saffron-system-analyst.md 變更

#### 核心職責新增第 5 項「系統衝突檢查（強制 checklist）」

三項強制 checklist，每項必須明確判斷「無衝突 / 有衝突（並列出）」：

1. **跨模組依賴衝突**
   - 新功能跨模組依賴盤點
   - 模組介面變更對呼叫者的影響
   - 依賴方向違反架構規則
   - 跨模組事件流衝突

2. **既有 UC 變動影響**
   - 新功能修改既有 UC 行為或驗收條件
   - 受影響 UC 清單完整性
   - 既有 UC 測試案例更新需求
   - 使用者心智模型一致性

3. **跨版本相容性**
   - 資料結構變更對既有版本的破壞性
   - Migration 路徑設計
   - 多平台同步變更（Chrome Extension / APP / CLI）
   - 向後相容 fallback 需求

#### 核心職責新增第 6 項「Proposal 評估支援」

配合 proposal-evaluation-gate.md 強制機制，saffron 在以下場景提供審查：
- 新建 standard / heavy 分級 PROP
- 既有 PROP 流轉至 confirmed / approved

審查項目：
- 替代方案跨模組視角覆蓋
- 跨版本/跨專案 PROP 的本端 Reality Test 充分性
- Framework 類 PROP candidate 數 >= 3
- confirmed 狀態綁定 ticket_refs

#### 審查報告模板新增強制章節

模板新增「系統衝突檢查」強制章節：
- 跨模組依賴衝突（勾選項 + 嚴重度：critical/warning/info）
- 既有 UC 變動影響（勾選項 + UC 清單 + 嚴重度）
- 跨版本相容性（勾選項 + Migration 路徑 + 嚴重度）
- 衝突檢查結論：無 critical 才能進入 Phase 1

#### Phase 0 完成驗收條件強化

「下一步」條件從原本的「可以進入 TDD Phase 1」升級為：「可以進入 TDD Phase 1（前提：衝突檢查結論為『無 critical』）」
