### 變更摘要

**目標檔案**：`docs/proposals/PROP-011-auto-navigation-and-library-scraping.md`

#### 1. frontmatter 新增 `evaluation_level: standard`

在 frontmatter 末尾加入 `evaluation_level: standard`，使 proposal-evaluation-gate-hook 能正確識別並驗證必填章節。

#### 2. 新增 `## 替代方案` 章節

整合已有「候選方案評估」各子決策的選擇依據，以符合 hook 關鍵字匹配（`替代方案`）。包含：
- 子決策 A（自動導航）：A1 採用原因、A2/A3 未採用原因
- 子決策 B（自動滾動）：B1 採用原因、B2 未採用原因
- 選擇依據摘要

#### 3. 新增 `## 失敗防護` 章節

整合已有「風險與應變」各風險點，改寫為「失敗情境 + 前兆 + 防護」格式，符合 hook 關鍵字匹配（`失敗防護`）。涵蓋：
- 失敗情境 1：autoStart flag 在 SW idle 時遺失
- 失敗情境 2：DOM 結構變更致選擇器失效
- 失敗情境 3：安全閥失效致無限迴圈
- 失敗情境 4：manifest 新增權限被審核拒絕

#### 4. 新增 `## Reality Test` 章節

新增 3 個假設驗證（符合 hook 關鍵字 `Reality Test` / `假設驗證`）：
- 假設 1：storage.session flag 傳遞可靠性（ANA-A 前置驗證）
- 假設 2：scrollTo + 2s 等待足夠觸發無限滾動（ANA-A 實地勘查）
- 假設 3：A1+B1 涵蓋 > 95% 使用者需求（Phase 1 觀察後重評條件）

加上觸發案例說明本提案的起源背景。
