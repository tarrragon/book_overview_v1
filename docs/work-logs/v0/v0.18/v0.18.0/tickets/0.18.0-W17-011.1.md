## Context Bundle：track runqueue CLI 規格（W17-009 多視角收斂版）

### 背景
本 ticket 為 W17-009 ANA 落地實作，經三視角審查（Evidence/Alternatives/linux）一致收斂：原 next/schedule/resume-hint 三命令合併為單一 `track runqueue`。

### CLI Schema（精確規格）

```
ticket track runqueue [--format FORMAT] [--top N] [--context CONTEXT] [--wave N] [--status STATUS]
```

**參數定義**：

| 參數 | 值域 | 預設 | 語意 |
|------|------|------|------|
| `--format` | `list` / `dag` / `critical-path` | `list` | 輸出視圖 |
| `--top` | int ≥ 1 | 無限 | 返回前 N 筆（僅 list / critical-path 有效） |
| `--context` | `resume` / 未指定 | 未指定 | resume 模式：掃描 .claude/handoff/pending/ 交集 runqueue |
| `--wave` | int | 全部 | 過濾 wave 範圍 |
| `--status` | `pending` | `pending` | 目前僅支援 pending（排除已完成/blocked） |

### --format 三視圖語意

1. **list**（預設）：純可執行清單，`blockedBy == []` 且 `status == pending`，按 priority P0→P3 + creation_date 排序
2. **dag**：拓撲排序 + 層級分組，整個 DAG（含 blocked 者），關鍵路徑高亮
3. **critical-path**：僅返回關鍵路徑節點（slack = 0）

### 復用既有 lib（禁止重寫）

| 需求 | 復用對象 | 檔案 |
|------|---------|------|
| DAG 拓撲排序 + CPM | `CriticalPathAnalyzer` | .claude/skills/ticket/lib/critical_path.py |
| 環檢測 | `CycleDetector` | .claude/skills/ticket/lib/cycle_detector.py |
| Ticket 載入 | 既有 loader | .claude/skills/ticket/lib/（沿用 track_query 模式）|
