## Context Bundle：track runqueue CLI 規格（W17-009 多視角收斂版）

### 背景
本 ticket 為 W17-009 ANA 落地實作，經三視角審查（Evidence/Alternatives/linux）一致收斂：原 next/schedule/resume-hint 三命令合併為單一 `track runqueue`。

### CLI Schema（精確規格）

```
ticket track runqueue [--format FORMAT] [--top N] [--context CONTEXT] [--wave N] [--status STATUS]
```

**參數定義**：

| 參數 | 值域 | 預設 | 語意 |
|------|------|------|------|
| `--format` | `list` / `dag` / `critical-path` | `list` | 輸出視圖 |
| `--top` | int ≥ 1 | 無限 | 返回前 N 筆（僅 list / critical-path 有效） |
| `--context` | `resume` / 未指定 | 未指定 | resume 模式：掃描 .claude/handoff/pending/ 交集 runqueue |
| `--wave` | int | 全部 | 過濾 wave 範圍 |
| `--status` | `pending` | `pending` | 目前僅支援 pending（排除已完成/blocked） |

### --format 三視圖語意

1. **list**（預設）：純可執行清單，`blockedBy == []` 且 `status == pending`，按 priority P0→P3 + creation_date 排序
2. **dag**：拓撲排序 + 層級分組，整個 DAG（含 blocked 者），關鍵路徑高亮
3. **critical-path**：僅返回關鍵路徑節點（slack = 0）

### 復用既有 lib（禁止重寫）

| 需求 | 復用對象 | 檔案 |
|------|---------|------|
| DAG 拓撲排序 + CPM | `CriticalPathAnalyzer` | .claude/skills/ticket/lib/critical_path.py |
| 環檢測 | `CycleDetector` | .claude/skills/ticket/lib/cycle_detector.py |
| Ticket 載入 | 既有 loader | .claude/skills/ticket/lib/（沿用 track_query 模式）|

**禁止重新實作**：Kahn 算法、拓撲排序、關鍵路徑、環檢測。若 critical_path.py API 不敷使用，先派 ANA 評估改 API，不直接複製邏輯。

## 實作摘要

新增 `.claude/skills/ticket/ticket_system/commands/track_runqueue.py` 實作統一 scheduler CLI。

### 架構

- `execute_runqueue(args, version)`：主入口，依 `--format` 分派三個渲染函式
- `register_runqueue(subparsers)`：argparse 子命令註冊
- 註冊路徑：`track.py` 的 `_create_command_handlers()` 字典新增 `"runqueue": execute_runqueue`，並於 `_register_all_subcommands` 呼叫 `register_runqueue`。**不新增 `execute()` 特殊分支**（對齊 W17-008.11 結論，避免 snapshot/dispatch-check 雙軌）

### 視圖實作

| format | 規則 | 排序 |
|--------|------|------|
| list（預設）| `status=pending` 且 `blockedBy=[]` | priority P0→P3，id 次序 |
| dag | 完整 DAG（含 blocked），以 CPM forward-pass 的 ES 作為層級 | 層級內 priority + id |
| critical-path | `result.critical_path`（slack=0） | analyzer 回傳序 |

### 復用既有 lib

- `CriticalPathAnalyzer.analyze`：拓撲/CPM/層級（ES）、關鍵路徑
- `CycleDetector`：經由 analyzer 間接使用（`is_valid=False` 時 fallback + WARN 輸出）
- `ticket_loader.list_tickets`：ticket 載入
- `paths.get_project_root`：定位 `.claude/handoff/pending/`

### 參數行為

- `--top N`：list / critical-path 裁切；dag 忽略（呈現完整 DAG）
- `--context=resume`：掃描 `.claude/handoff/pending/*.json`，交集 `ticket_id`
- `--wave N`：wave 過濾
- `--status`：目前預設 pending（保留未來擴充）
- **未新增 --nice 參數**（對齊 linux 審查結論）

- critical-path：線性鏈、CriticalPathAnalyzer spy（2）
