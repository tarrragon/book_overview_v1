### 問題敘述

W17-060「規則 6 估時禁令條款」落地過程暴露兩個結構性流程缺口，這兩缺口非單次失誤而是 PM 改框架規則時的常態：

1. **事前未觸發 compositional-writing skill**：PM Edit framework 規則檔（rules/core/）前未引用 SKILL，導致規則 6 條款違反原則 3（機會成本語氣、不用絕對主義）
2. **事後缺 Layer 2 委員獨立審查**：PM 自做 Layer 1 自檢（掃 document-writing-style 清單）+ Layer 4 PM 收斂為同主體，失去獨立審查值；W17-051 IMP-C 設計的「多輪驗收流程」未活演練

兩個缺口的後果在 W17-060 規則 6 內容上具體呈現：標題「禁止以估時為決策依據」+ 條款內 N 處「禁止 / 不可」屬絕對主義詞，違反 compositional-writing SKILL 原則 3 + 原則 6 第 3 輪 review（絕對主義詞翻 trade-off）。

### 元層教訓（dogfooding 失敗暴露的議題）

本 ANA 設計「強制觸發 SKILL 防止絕對主義」的同時，自身文字使用絕對主義（template / hook 訊息 / claim 提示）。此元層級違規顯示：

- 設計新防護的條款（規範性文字）必須以機會成本示範，否則防護機制違反它要保護的原則
- 事實陳述（描述歷史違規）可保留絕對語氣
- 兩者在同一文件中需明確分區，避免讀者混淆規範性 vs 描述性

此教訓建議升級為 PC error-pattern：「規範性文字 dogfooding 違規」（spawned 後續評估）。

### 隱藏成本清單

1. **Hook transcript 掃描成本**：每次 PreToolUse 需讀 session transcript 找 `Skill compositional-writing` 呼叫紀錄。長 session 下 transcript 可達 500K+ token，hook 內若用 grep/regex 還好；若 LLM 解析則爆炸性成本。**設計時需明確 hook 用純字串 match，禁 LLM 介入**。
2. **誤擋連鎖**：Layer A 強制模式下 PM 改規則 typo 也被擋 → PM 用 `--skip` flag → flag 累積 → hook 形同虛設（PC-066 mode）。
3. **路徑清單維護成本**：5 個 glob 模式（rules/pm-rules/skills/methodologies/agents），新增類別（如 references/）需同步更新；無更新則漏擋。
4. **SKILL 改版時 hook 失準**：compositional-writing SKILL 主文若改版（v1.x → v2.x），hook 偵測「已讀 SKILL」邏輯需同步；否則「假已讀」放行。
5. **Layer 2 委員派發成本**：Layer C 規範「派 basil-writing-critic」每次 ~3-5 min + 5-10K token，每月 framework edit ×30 → **150K+ token + 90-150 min 等待**。此為 Layer C 最大隱藏成本，未在原 Solution 量化。
