## 問題敘述

W17-060「規則 6 估時禁令條款」落地過程暴露兩個結構性流程缺口，這兩缺口非單次失誤而是 PM 改框架規則時的常態：

1. **事前未觸發 compositional-writing skill**：PM Edit framework 規則檔（rules/core/）前未引用 SKILL，導致規則 6 條款違反原則 3（機會成本語氣、不用絕對主義）
2. **事後缺 Layer 2 委員獨立審查**：PM 自做 Layer 1 自檢（掃 document-writing-style 清單）+ Layer 4 PM 收斂為同主體，失去獨立審查值；W17-051 IMP-C 設計的「多輪驗收流程」未活演練

兩個缺口的後果在 W17-060 規則 6 內容上具體呈現：標題「禁止以估時為決策依據」+ 條款內 N 處「禁止 / 不可」屬絕對主義詞，違反 compositional-writing SKILL 原則 3 + 原則 6 第 3 輪 review（絕對主義詞翻 trade-off）。

## 三層防護方案

對應因果鏈「How（缺口維持機制）」四個動因，設計三層防護（Hook / Claim / Template）相互強化：

### Layer A：Hook 自動偵測層（強制）

| 屬性 | 內容 |
|------|------|
| Hook 名稱 | `framework-rule-edit-skill-trigger-hook.py` |
| 觸發點 | PreToolUse: Edit / Write |
| 觸發條件 | 目標檔案路徑符合 `.claude/rules/**/*.md` / `.claude/pm-rules/**/*.md` / `.claude/skills/**/SKILL.md` / `.claude/methodologies/**/*.md` / `.claude/agents/**/*.md` |
| 行為 | 偵測本 session transcript 是否已呼叫 `Skill compositional-writing`（或對應 reference）；若未呼叫，依配置輸出 exit 2 阻擋或 exit 0 警告 |
| 配置 | 預設 exit 0 警告（避免大規模誤擋）；強制模式可由 `.claude/config/skill-trigger-strict.yaml` 切換 |
| 預期攔截 | 本次 W17-060：PM 直接 Edit ai-communication-rules.md 前無 SKILL 呼叫 → Hook 警告「規則層編輯需先讀 compositional-writing SKILL」 |
| 落地 IMP | spawned W17-122.1 |

### Layer B：Claim WRAP 三問擴增（提示）

| 屬性 | 內容 |
|------|------|
| 修改位置 | `.claude/skills/ticket/ticket_system/commands/lifecycle.py` claim 命令的 WRAP 提示區塊 |
| 觸發條件 | claim ticket 時 type = IMP 且 where.files 含 framework 路徑（同 Layer A 路徑清單） |
| 行為 | 既有 W/A/P 三問後新增第四問「S（SKILL trigger）—— 本次 Edit 涉及 framework 規則層，是否已 Read compositional-writing SKILL（或相關 reference）？若無，請暫停 claim 並先讀 SKILL」 |
| 預期攔截 | claim W17-060 時：S 問提示「框架規則 Edit 場景需先讀 SKILL」 |
| 落地 IMP | spawned W17-122.2 |

### Layer C：Agent dispatch template + PM 自做 template 規範

| 屬性 | 內容 |
|------|------|
| 修改位置 | `.claude/references/agent-dispatch-template.md` 新增章節「PM 自做 framework 規則編輯流程」 |
| 內容 | 必經步驟：(1) Read SKILL → (2) 撰寫 → (3) 派 Layer 2 委員 → (4) 收報告 → (5) 修正 → (6) commit；禁止：跳過 SKILL 直接 Edit、自做 Layer 1 + Layer 4 同主體 |
| 對應 commit msg 規範 | commit msg 必含「Layer 2 by [agent-name]」或「N/A by [理由]」標記 |
| 預期攔截 | PM 改規則前查 template 看到完整流程 |
| 落地 IMP | spawned W17-122.3 |

### 三層協同效果

| 缺口 | Layer A | Layer B | Layer C |
|------|---------|---------|---------|
| 事前未觸發 SKILL | 強制 hook 攔截 | claim S 問提示 | template 必經步驟 |
| 事後缺 Layer 2 | — | — | template + commit msg 規範 |

Layer A / B 解決「事前」缺口；Layer C 解決「事前 + 事後」缺口。三層相互備援：Layer A 失效時 Layer B 補；Layer B 失效時 Layer C 補。
