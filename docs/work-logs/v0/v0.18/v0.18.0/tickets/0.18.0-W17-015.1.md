## Problem Analysis

### 問題根因

`ticket track full` 輸出純文字約 110 行，終端閱讀困難；用戶需要 ticket 內容美化渲染入口。現況 CLI 無 `show` 子命令，用戶必須用 `cat` 或編輯器打開 md，失去 CLI 流程連貫性。

### 影響範圍

- `.claude/skills/ticket/ticket_system/scripts/ticket.py`：主 argparse 需加 `show` 子命令分支
- `.claude/skills/ticket/ticket_system/commands/show.py`：新模組，實作渲染邏輯
- `.claude/skills/ticket/SKILL.md`：文件需補 `show` 使用範例
- `.claude/skills/ticket/tests/test_show.py`：新測試檔

### 相關 Error Pattern

無直接相關；設計上對齊 git show/kubectl describe 慣例，避免 CLI 語意漂移。

---

## Solution

### 架構

`show.py` 模組承擔三責任：

1. **ID 解析**：引用 track 既有 ID resolver，支援短 ID（`W17-015.1`）自動補當前版本
2. **渲染器選擇**：`shutil.which` 偵測順序 `glow > mdcat > bat`，缺失則 fallback raw
3. **輸出管線**：TTY 偵測（`sys.stdout.isatty()`）→ 渲染器 subprocess → 可選 pager（`less -R`）

### 關鍵決策

- **短 flag 採用 Unix 慣例**：`-r`（raw）、`-R`（renderer）、`-p`（pager）、`-P`（no-pager）
- **渲染器參數**：`bat -l md --style=plain`、`glow -`、`mdcat`（stdin 讀入 markdown）
- **pipe 自動降級**：非 TTY 時強制 raw，避免 ANSI 污染下游處理
- **錯誤處理**：指定 renderer 未安裝時明確報錯（不 fallback），全部缺失才 fallback raw
