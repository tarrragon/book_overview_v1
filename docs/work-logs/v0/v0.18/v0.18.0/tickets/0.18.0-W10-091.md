### 問題根因

W10-016 ANA 結論：Skill 私有 hook 應遷移至 `.claude/skills/<skill>/hooks/` 雙層架構。前置條件是 `hook-completeness-check.py` 必須能掃描兩層 hook 目錄，否則遷移後私有 hook 註冊狀態無法驗證。

### 影響範圍

- `.claude/hooks/hook-completeness-check.py` (316 行) — 主掃描器入口
- `.claude/skills/project-init/project_init/lib/hook_checker.py` — `scan_hooks_directory()` 共用 lib（line 90-108，目前只支援單一目錄非遞迴 glob `*.py`）
- 既有測試（位於 `.claude/skills/project-init/` 或 `.claude/hooks/tests/`，需確認）

### 當前狀態

- 目前僅 `.claude/skills/test-async-guardian/hooks/pre-test-scan.py` 一個 skill 私有 hook
- 主掃描器只掃 `.claude/hooks/*.py`（hook_utils.py 等共用模組已在 exclude pattern）
- settings.json 已支援 skill hook 路徑（`${CLAUDE_PLUGIN_ROOT}` / 相對路徑），但本掃描器尚未列入比對

### 實作策略

**Step 1**: 擴充 `scan_hooks_directory()` 支援遞迴模式
- 在 `hook_checker.py` 加 `recursive: bool = False` 參數，True 時改用 `rglob("*.py")`
- 或新增獨立函式 `scan_skill_hooks(skills_dir: Path)` 掃 `<skill>/hooks/*.py`（推薦：保留現有 lib 語意，新增專用函式更清晰）

**Step 2**: 主掃描器合併兩層結果
- 在 `hook-completeness-check.py` 主流程加入 skill hooks 掃描
- 結果合併成單一清單，但區分顯示來源（標註 `[skill:<name>]`）

**Step 3**: settings.json 註冊比對
- 比對邏輯需識別 skill hook 在 settings.json 中的路徑形式（如 `.claude/skills/<skill>/hooks/<file>.py`）
- exclude 規則保留（`__init__.py`、`hook_utils.py` 等）

**Step 4**: 測試補強
- 既有測試確認不破壞（pytest 全綠）
- 新增 skill-hook 註冊測試：
  - case: skill hook 已註冊 → pass
  - case: skill hook 存在但未註冊 → warning
  - case: 兩層皆有同名 hook → 各自獨立報告（路徑區分）

### 預期變更檔案

- `.claude/skills/project-init/project_init/lib/hook_checker.py` (+10~20 行)
- `.claude/hooks/hook-completeness-check.py` (+20~30 行)
- `.claude/hooks/tests/test_hook_completeness.py` 或對應測試（新增測試 ~30 行）

### 風險與防護

- 風險 A：rglob 誤掃 `__pycache__/.venv` → 沿用 `_check_and_fix_permissions` 的 skip 邏輯
- 風險 B：settings.json 路徑格式不一致 → 比對時統一規範化為相對 `.claude/` 的路徑
- 風險 C：W10-092/093/094 尚未執行，目前只 1 個 skill hook → 測試使用 fixture 模擬多 skill 情境
### 實作摘要

擴充 hook 完整性檢查器至雙層 Hook 架構（`.claude/hooks/` 主層 + `.claude/skills/<skill>/hooks/` skill 私有層），為 W10-092/093/094 私有 hook 遷移建立前置基礎建設。

### 變更檔案

| 檔案 | 變更 |
|------|------|
| `.claude/skills/project-init/project_init/lib/hook_checker.py` | 新增 `scan_skill_hooks()` 與 `extract_registered_skill_hooks()` 兩個 API（共 +60 行）。沿用既有 `should_exclude_file` / `get_exclude_patterns` 規則。 |
| `.claude/hooks/hook-completeness-check.py` | 主流程加入 skill hooks 掃描與比對；報告中新增獨立 Skill Hooks 區塊（+25 行） |
| `.claude/hooks/tests/test_hook_checker_skill_hooks.py` | 新增 13 個測試覆蓋 scan / extract / unregistered detection / dual-namespace 獨立性 |

### 設計決策

- **獨立函式而非 recursive flag**：`scan_skill_hooks` 為獨立函式，回傳值用相對路徑 `<skill>/hooks/<file>.py`（POSIX）而非裸檔名，避免與主層命名空間衝突
- **路徑識別不收 skill 根目錄檔案**：`extract_registered_skill_hooks` 僅收 `.claude/skills/<skill>/hooks/<file>.py` 形式；如 `strategic-compact/suggest-compact.py`（非 hooks/ 子目錄）不算 skill hook，保持與雙層 Hook 架構語意一致
- **exclude 規則沿用**：`hook_utils.py` / `*-backup.py` 等規則同步套用於 skill hooks，避免遷移後分歧
