### 遷移摘要

將 `.claude/settings.json` 與 `.claude/settings.local.json` 所有 hook `command` 字串形式遷移至 v2.1.139+ 的 `args` 陣列 exec 形式（直接 spawn 不經 shell，消除 PC-079 同源 shell 解析風險）。

### 遷移統計

| 檔案 | 已遷移 | 保留 command | 說明 |
|------|--------|------------|------|
| `.claude/settings.json` | 121 | 0 | 全數遷移；含 6 大 event（PreToolUse/PostToolUse/SessionStart/UserPromptSubmit/SubagentStop/Stop） |
| `.claude/settings.local.json` | 2 | 0 | 包含 PreToolUse(Agent) 的 `python3 $CLAUDE_PROJECT_DIR/...` 條目轉成 `["python3", "$CLAUDE_PROJECT_DIR/.claude/hooks/agent-dispatch-validation-hook.py"]` |

### 遷移規則（一次性 Python 腳本，shlex.split 保留 $VAR）

1. 偵測 shell 操作符（`|`/`>`/`<`/`&&`/`||`/`;`/`$(`/backtick）：本次掃描全部 123 條目皆無，不需保留 command。
2. `style-guardian-hook.py "$CLAUDE_FILE_PATH"` 類條目正確拆為兩個 args，`$CLAUDE_FILE_PATH` 變數展開仍由 Claude Code runtime 處理（與 command 字串形式行為一致）。
3. JSON 欄位順序保持 `type` → `args` → 其他（timeout 等），維持可讀性。

### 配套修正

`acceptance` 第 4 條「hook-architect-technical-reference.md 標註遷移完成」：在 `.claude/references/hook-architect-technical-reference.md` 「args exec 形式範例」段落後追加「本專案遷移狀態」note，紀錄 121+2 條已遷移。

### 測試碼適配

3 個測試檔案的舊式 `h.get("command", "")` 比對改為 `h.get("command", "") + " ".join(h.get("args", []))`：

- `.claude/hooks/tests/test_phase4_decision_enforcement_hook.py`（3 處 + 1 處）
- `.claude/hooks/tests/test_wrap_decision_tripwire_hook.py`（4 處 replace_all）

### 不在本 ticket 範圍

- 既有 hook 內部如何讀取 args 不變（args 由 Claude Code spawn 時轉為 argv，hook script 仍從 stdin 讀 JSON event payload）。
- `style-guardian-hook.py` 等接收 argv 的 hook 行為不變（拆分後仍是同樣的 argv）。
