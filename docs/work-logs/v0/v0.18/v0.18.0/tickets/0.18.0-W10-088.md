## 問題根因

W10-086 在 main 修復了 set-* dict flatten bug（新增 `DICT_FIELD_SUBKEY` 映射與 `_build_dict_field` helper），但 `test_track_fields.py` 中的 3 支 set-*_success 測試仍沿用舊 string 語義（mock_ticket["who"]="sage-test-architect"；assert saved["who"]=="parsley-..."），在 W10-086 修復後實際 saved["who"] 為 dict {"current":"parsley-...", "history":{}}，導致 assertion 失敗。

根因：修復程式碼時未同步更新對應的既有測試語義。測試設計前提（欄位為 string）已被修復取代（欄位為 dict）。

## 修正策略

因本 worktree branch 基於較舊 commit（無 W10-086），採以下做法：
1. 將 W10-086 的 fields.py 修復 port 到本 worktree（DICT_FIELD_SUBKEY/_build_dict_field/execute_set_field dict 分流）
2. 修正 3 支既有測試 mock_ticket 改為 dict 結構、assertion 改為 isinstance+subkey 檢查
3. 新增 TestDictFieldFlattenRegression 類（5 個 test）覆蓋 AC4：
   - test_set_who_preserves_history_when_dict（dict 正常）
   - test_set_who_recovers_dict_when_flattened_string（降級復原）
   - test_set_where_preserves_files（保留 files 陣列）
   - test_set_how_preserves_task_type（保留 task_type）
   - test_set_what_scalar_field_unchanged（非 dict 欄位迴歸保護）
