### 改動摘要

依 PC-091 路線（W17-120.1 規則層落地）在 hook 層收斂 ANA complete 的雙路徑判斷：移除 `ana_spawned_checker` 對 ANA 的阻擋邏輯，ANA complete 阻擋統一由 `children_checker` 判斷。

### 改動清單

| # | 檔案 | 動作 |
|---|------|------|
| 1 | `.claude/hooks/acceptance-gate-hook.py` | 移除 `ana_spawned_checker` 的 import（line 81-85 區段重寫為 deprecated 註解）；移除 Step 2.5.1（235-252）改為 deprecated stub；checklist 項目 5 改寫為「ANA 後續 ticket missing warning」三狀態變兩狀態；優先級 1.5 區塊保留向後相容輸出但 orchestrator 已不 set 該欄位 |
| 2 | `.claude/hooks/acceptance_checkers/ana_spawned_checker.py` | 模組頭加 DEPRECATED 標註，指向 W17-120 / W17-120.2 / PC-091；保留 `check_ana_has_spawned_tickets`（warning 層仍由 orchestrator 呼叫）與 `extract_spawned_tickets_from_frontmatter` 向後相容 |
| 3 | `.claude/hooks/lib/hook_messages.py` | `ANA_MISSING_SPAWNED_TICKETS_WARNING` 訊息更新：明示「ANA 落地請用 `--parent <ANA-ID>` 建 children（PC-091 路線）」；`CHILDREN_INCOMPLETE_ERROR` 加同義 hint |
| 4 | `.claude/hooks/tests/test_ana_spawned_checker.py` | 整檔加 DEPRECATED 標註與 `pytestmark = pytest.mark.skip(...)`；11 個 testcase 全 skip |
| 5 | `.claude/hooks/tests/test_acceptance_gate_hook.py` | 新增情境 9：4 個 ANA case 回歸測試（含 case (d) 行為翻轉核心驗證——ANA 有 spawned 無 children 不阻擋） |

### 設計決策

- **保留 `check_ana_has_spawned_tickets`**：此函式產生「ANA 缺後續 ticket」warning（不阻擋），與本次退場的「blocking 層」（`check_spawned_tickets_blocking`）職責不同。退場目標精準限縮在阻擋層，warning 層保留並更新訊息文案引導用 `--parent` 建 children。
- **保留檔案不刪除**：`ana_spawned_checker.py` 與 `test_ana_spawned_checker.py` 仍保留檔案（加 deprecated 標註），便於追蹤歷史並避免外部 import 突然斷裂。後續若 grep 確認無外部引用，可在獨立 ticket 物理刪檔。
- **新測試走 orchestrator**：4 個 ANA case 透過 `check_acceptance_status` 主協調函式驗證，而非直接呼叫單一 checker——這精準錨定「行為翻轉」發生在 orchestrator 層，符合本 ticket 收斂雙路徑的核心意圖。
