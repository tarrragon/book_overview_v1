## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

對 Context Bundle 抽取模組套用 10 項 acceptance，一次性提升風格、可觀測性與擴充性。測試 65/65 green（原 39 + 新增 26 於 TestGroupF），全套件 1484 passed 無回歸。

### 各 acceptance 對應變更

1. **quiet 預設單行**：`format_cli_summary(quiet=True)` 為預設；verbose=True 隱含關閉 quiet；所有非 success 狀態（no_source/opt_out/self_reference/all_sources_missing）在 quiet 模式皆有對應單行訊息。
2. **MAX_TOTAL_CHARS rationale**：值 2000 移至 `ticket_system/constants.py::CONTEXT_BUNDLE_MAX_TOTAL_CHARS`，附三段 rationale 註解（token 對照 / PM 掃描時間錨點 / 超量處理原則）。
3. **Literal 源自 lib/constants.py**：新增 `CONTEXT_BUNDLE_SOURCE_KINDS` / `CONTEXT_BUNDLE_EXTRACT_STATUSES` / `CONTEXT_BUNDLE_SKIP_REASONS` 至 constants.py 並加入 `__all__`；extractor 內 Literal 類型重新用這些常數為單一真實來源。
4. **SkipRecord dataclass**：已確認並加型別標註（`ExtractResult.skipped: List[SkipRecord]`）；docstring 補註歷史脈絡取代 list[dict]。
5. **專屬 Exception**：新增 `ContextBundleExtractionError`；`extract_and_write_context_bundle` 對 target load 失敗用 `raise ... from exc` 統一抛出，保留原因鏈；source load 失敗仍記入 warnings（non-raising 契約）。
6. **ac_parser 整合 acceptance 過濾**：新增 `use_ac_parser=True` 規則欄位與 `_filter_acceptance_with_ac_parser` helper，使用共用 `checkbox_utils.strip_checkbox_prefix` 過濾 [x] 項；全項已勾選時該欄位不寫入 extracted（降噪）。
7. **--json 結構化輸出**：新增 `format_cli_summary_json(result)`，schema 含 target_ticket_id / status / sources_declared / sources_ok / total_chars_estimate / extracted[] / skipped[] / warnings[]；CLI create/track claim 同步註冊 `--json` flag 並 wire-in。
8. **metric event 埋點**：新增 `set_metric_sink()` / `_emit()` 模組級注入；每次 `extract_context_bundle` 結束前發送 `context_bundle.extract` 事件，payload 固定 8 欄（status/sources_declared/sources_ok/fields_extracted/total_chars/truncated_count/warnings_count/skipped_count），sink 失敗會吞例外只寫 stderr 不影響主流程（支援 Tripwire T1 量測）。
9. **opt-out 標記**：新增 `_detect_opt_out()`，支援 `context_bundle: manual` 與 `context-bundle: manual` 兩種 YAML key 形式；status=`opt_out` 短路、render 回傳空、CLI 摘要對應訊息。
10. **placeholder 擴為 list**：`CONTEXT_BUNDLE_PLACEHOLDER_VALUES = (DEFAULT_UNDEFINED_VALUE, "TBD", "TODO", "待填寫", ...)` 放 constants.py；`_is_placeholder` 以 membership 判定，保留原空字串 / None / 空 list 分支。

### cinnamon 補充項

`_count_field_chars` helper 抽出，`_estimate_chars` 與 `_apply_total_chars_limit` 改用該 helper，消除雙處字元計算邏輯重疊。

### 設計取捨

- ac_parser 整合選擇「沿用 checkbox_utils」而非直接呼叫 `ac_parser.parse_ac(ticket_id)`，避免重複 load_ticket I/O（source 已在記憶體中）。
- metric sink 採注入模式而非 import，保持 lib 純度；預設 no-op 不影響既有呼叫者。
- Literal 類型別名 `SourceKind` / `ExtractStatus` 等保留於 extractor（型別標註用），值集合改引自 constants（單一真實來源）。
