## Problem Analysis

### 問題根因（本 session 實測觀察）

執行 `ticket track chain 0.18.0-W14-024` 時 CLI 回應：

```
[Warning] Ticket 0.18.0-W14-024 無 chain.root 資訊
      嘗試將本身視為 root 展開
0.18.0-W14-024 [已完成] - ... （僅顯示本身）
```

相較之下 `chain 0.18.0-W14-019`（有 children）能正確展開 019 + 019.1-.6 七層樹狀結構。

差異根因：
- W14-019 有 `children` 陣列（019.1-.6 透過 `parent_id` 關聯）→ chain 可追溯
- W14-024 有 `spawned_tickets` 衍生 W14-028/029/030（透過 `source_ticket` 關聯）→ chain 無法追溯

現行 chain/tree 命令僅實作「血緣關係」（parent/children），**忽略「衍生關係」**（spawned/source），導致 ANA→IMP 任務鏈無法完整可視化。

### 影響範圍

| 層級 | 受影響內容 |
|------|----------|
| CLI 命令 | `ticket track chain`, `ticket track tree` |
| PM 工作流 | 無法透過單一命令查看 ANA 的所有 spawned IMP 狀態 |
| Handoff | next-session handoff 需手動整理衍生關係 |
| 驗收流程 | ANA complete 前需手動確認 spawned IMP 是否全完成 |

- PC-075: spawned children 狀態檢查不對稱 — chain 命令應同步支援
