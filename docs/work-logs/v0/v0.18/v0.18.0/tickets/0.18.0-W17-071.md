### 問題根因

W17-070 ANA 雙根因分析：validator 與 hook 兩處獨立實作同時漏擋 schema note + markdown 分隔符的空殼結構（root cause A + B）。

- **Root cause A（分隔符未剝除）**：`_is_placeholder` 與 hook 端 `_is_section_empty` 剝除 HTML 註解後，剩下 markdown 分隔符 `---` 被誤判為實質內容。
- **Root cause B（section 邊界被任意 H2 切斷）**：`validate_execution_log` / `_is_section_empty` 以「任意 `##`」作為下一個 section 起點，agent 寫自定義 H2（如 `## 實作摘要`）會把 schema section 切成空殼。

兩根因共振：W17-056 事件中 agent 把實作內容寫在 `## 實作摘要` / `## 驗證指令與結果`，Schema 章節 `Solution` / `Test Results` 只剩 schema note + 分隔符 → 兩處 validator 都誤判為「已填寫」，complete 放行。

### 修復摘要

三處檔案同步修改，遵循 AC 定義範圍（禁止擴大到 IMP-2）。

#### (1) `ticket_validator._is_placeholder`：新增 markdown 分隔符剝除

在 HTML 註解剝除後、佔位符檢查前，以 `re.sub(r'^[ \t]*-{3,}[ \t]*$', '', content, flags=re.MULTILINE)` 剝除獨立一行的 `---+`。若剝除後內容為空即視為 placeholder。

保留既有行為：分隔符 + 實質內容仍判為 non-placeholder；段落中非獨立行的 `a---b` 不被剝除。

#### (2) `ticket_validator.validate_execution_log`：section 邊界限定 Schema 章節

抽取常數 `_SCHEMA_SECTION_NAMES` 含 9 個 schema 定義章節（Task Summary / Problem Analysis / Solution / Test Results / Completion Info / NeedsContext / Exit Status / 重現實驗結果 / Context Bundle）。

新增輔助函式 `_find_next_schema_section_boundary(body, content_start)`：以 `^#{2,3} SectionName\b` regex 行首匹配尋找下一個 schema 章節起點；自定義 H2（`## 實作摘要`）不算邊界。

section 標題定位亦改用 `^### Section\b` / `^## Section\b` 行首 regex 匹配，避免段落內相似字串誤判。

#### (3) `execution_log_checker._is_section_empty`：hook 端同步修復

- 同步定義 `_SCHEMA_SECTION_NAMES`（與 validator 對齊）；抽取 `_find_schema_section_boundary_regex()` 產生 regex alternation。
- 原 `(?=^## |\Z)` 改為 `(?=^## (?:SchemaA|SchemaB|...)\b|\Z)`。
- HTML 註解剝除後新增分隔符剝除（`re.sub(r'^[ \t]*-{3,}[ \t]*$', '', ..., flags=re.MULTILINE)`）。

### 設計決策

- **為何不呼叫 `_is_placeholder`（AC bonus 選項）**：hook 是獨立 uv script（`dependencies = []`），而 ticket_validator 使用相對 import（`from .constants import ...`）。跨環境 import 需做 sys.path 調整且跨 venv 風險高。選擇在兩處保留獨立實作 + 同步修復，並在 ARCH-020 記錄後續 refactor。
- **Schema 章節清單硬編碼 vs 從規則檔動態載入**：此 IMP 症狀修復不做抽象化。未來 IMP-2 可能引入統一 schema 定義，屆時兩處都改為共用。

### 不在本 ticket 範圍（IMP-2）

- agent-definition-standard.md 「禁止自定義 H2」條款
- pre-complete hook 偵測 body 中非 Schema H2 章節
- ARCH-020 refactor：validator 與 hook 共用 placeholder 邏輯
