### 問題根因

`agent-authoring-guide.md` 僅涵蓋 `permissionMode` 一個 frontmatter 欄位，缺少 CC 2.1.x 引入的 `initialPrompt`、`memory`（user/project/local）、`hooks`（PreToolUse/PostToolUse/Stop）、`background`、`effort`、`maxTurns`、`disallowedTools` 共 7 個欄位說明。`agent-template.md` frontmatter 範本同樣缺少這些欄位占位符，導致新代理人撰寫者無法得知完整可用欄位集。

- `.claude/references/agent-authoring-guide.md` — 欄位文件不完整

### 修改摘要

**1. `.claude/references/agent-authoring-guide.md`**

在文件開頭新增「完整 Frontmatter 欄位總覽」表格，統整所有 12 個可用欄位（必填/條件必填/建議/選填）。

接著新增「新增欄位詳細說明」章節，逐一說明 8 個新欄位：

- `initialPrompt` — 自動首輪 Prompt，含 3 個使用場景範例
- `memory` — 記憶範圍控制（user/project/local），含安全審查代理人範例
- `permissionMode` — 確認並重申現有規範（整合至新章節結構）
- `hooks` — 代理人生命週期 Hook（PreToolUse/PostToolUse/Stop），含 Stop Hook 自動 complete 範例
- `background` — 強制背景執行，含 4 種場景判斷表
- `effort` — 推理深度（low/medium/high），含 model vs effort 獨立控制說明
- `maxTurns` — 最大輪數限制，含各任務類型建議值表
- `disallowedTools` — 明確禁用工具，含 3 種適用場景及優先序說明

在文件末尾新增「代理人升級建議清單」章節，包含：

- 13 個代理人的升級建議表（欄位：代理人/建議新增欄位/理由/預期效益）
- 升級優先序分 3 級（高/中/低）
- 高優先：5 個 `allowed-tools` 非標準格式需修正的代理人
- 中優先：4 個需加 `maxTurns` 的高頻代理人
- 低優先：3 個優化但非急迫的代理人

**2. `.claude/templates/agent-template.md`**

- frontmatter 範本補入 8 個選填欄位佔位符（以 `# 註解` 形式保留，避免強迫填寫）
- 「必要章節說明 > Frontmatter」拆為「必填欄位表格」+「選填欄位表格」雙層結構
- 「範本檢查清單 > Frontmatter 檢查」補入 `permissionMode`、`effort`、`maxTurns`、`background`、`disallowedTools` 五個新檢查項

### 設計決策

- 升級建議表只記錄分析結論，不直接修改 `.claude/agents/` 任何代理人（本 ticket 範疇）
- `memory` 欄位說明偏向「避免 user 範圍過廣」為主，未驗證 CC 實際是否強制 scope，故僅作說明不加入 template 必填欄位
- 升級建議中 `rosemary-project-manager` 列為低優先，因主線程 `maxTurns` 設定可能影響 session 主 loop，需謹慎驗證
