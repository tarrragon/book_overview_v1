### 實作摘要

L2-A handoff --next CLI 與 target_ticket_id 欄位落地：

#### 變更清單

| 檔案 | 變更 |
|------|------|
| `.claude/skills/ticket/ticket_system/lib/handoff_utils.py` | 新增 `resolve_target(record) -> Optional[str]` helper：優先讀 `target_ticket_id` 欄位，fallback 至 `extract_direction_target_id(direction)` |
| `.claude/skills/ticket/ticket_system/commands/handoff.py` | (1) 新增 `--next <target-id>` CLI flag (W17-164 / L2-A)；(2) `_execute_next_handoff()` 寫入 `target_ticket_id`、`direction="context-refresh"`、`auto_generated=False`；(3) `execute()` 入口先檢 --next/--auto 互斥；(4) `_execute_auto_handoff` JSON schema 同步加入 `target_ticket_id`（從 direction 後綴提取） |
| `.claude/skills/ticket/tests/test_handoff_next.py` | 新增 11 個 RED 測試：resolve_target 6 case + --next/--auto 互斥 1 case + --next 寫欄位 2 case + 向後相容 2 case |

#### 設計決策

1. **--next 模式 direction 選擇**：使用既有 `context-refresh`（NON_CHAIN）而非新增 direction 類型。理由：避免 schema 連鎖變更（constants.py + handoff_utils._KNOWN_DIRECTION_VALUES + GC + hooks），且 target_ticket_id 欄位已承擔絕對指向語意，direction 退為「跨 session 對焦」描述符。
2. **resolve_target 優先序**：顯式欄位 > direction 後綴。空字串視為未提供（fallback），避免被誤填空值阻斷既有 direction 解析路徑。
3. **--auto 同步寫 target_ticket_id**：從 direction 後綴提取（如 `to-child:X` → `X`），讓所有新生 JSON schema 一致。舊 JSON 無此欄位仍可由 resolve_target 透過 direction fallback 解析（向後相容）。
4. **--next 與 --auto 互斥**：在 `execute()` 入口攔截，明確錯誤訊息「請擇一使用」。

#### 向後相容驗證

- 既有 handoff JSON（無 target_ticket_id）透過 `resolve_target()` fallback 至 direction 後綴解析（測試 `TestBackwardCompatibility` 兩個 case）
- 所有既有 handoff 相關測試（auto + utils + gc + stale_consistency + command + target_validation + exceptions + context_refresh + exit_status 共 162 個）全綠
- 三個 Hook 測試（reminder / auto_resume_stop / prompt_reminder 共 25 個）全綠
