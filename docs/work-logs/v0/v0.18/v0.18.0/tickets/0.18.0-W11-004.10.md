## Problem Analysis

### 問題根因

### 修復摘要

1. AC1 - 移除雙重問號判定：BINARY_QUESTION_KEYWORDS 中「嗎？」「嗎?」合併為純文字「嗎」；問號條件改由 is_binary_question 透過 QUESTION_MARKS 獨立檢查（單一責任）。
2. AC2 - 重疊關鍵字註解：在 OPTION_QUESTION_KEYWORDS 與 BINARY_QUESTION_KEYWORDS 加註解說明「要不要」「需要做」重疊為刻意保留——OPTION 路徑需搭配選項標記、BINARY 路徑需搭配問號，兩條路徑語意分離且豁免邏輯獨立。
3. AC3 - 合併 tail 掃描：has_question_ending 三段判斷統一使用單一 tail 變數；tail.rstrip().split 取代 text.rstrip().split（語意等價，避免重複切字串）；加 docstring 列出三條件。
4. AC4 - 具名常數：新增 QUESTION_MARKS = ('?', '？')，has_question_ending 與 is_binary_question 共用。

### 設計決策

- 不抽 helper 函式：三段判斷已扁平化為三行 any/endswith，認知負擔低於 helper 抽取（cognitive-load 原則）。
- endswith(QUESTION_MARKS) 利用 Python tuple 多後綴支援，比 endswith(a) or endswith(b) 更精簡。
