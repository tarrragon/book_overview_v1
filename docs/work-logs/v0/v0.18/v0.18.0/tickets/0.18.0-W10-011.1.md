## 實作摘要

建立 `.claude/rules/core/document-writing-style.md`（163 行），定義跨書面文字的論述明示性元規則。

## 核心設計

### 三明示原則
每段論述必須回答：
1. **Why** — 此原則的依據
2. **Consequence** — 違反後果
3. **Action** — 讀者下一步動作

### 適用範圍（明示邊界）
- 適用：規則檔、方法論、PC error-pattern、Skill/Agent description + 主文、長段註解（> 3 行）
- 不適用：程式碼短註解（註解專項規則處理）、inline TODO、外部文件

### 與其他規則的分工（避免 DRY 違反）
- ai-communication-rules：對話 / prompt 即時互動品質
- document-format-rules：Markdown 語法格式
- comment-writing-methodology：註解業務情境 + 抽象層級（註解專項）
- language-constraints：字元選擇

### 4 對正反範例
規則條文、PC 錯誤模式論述、方法論原則、Skill description，每對附判別依據。

### 反模式列表
6 個常見隱含表達句型（希望、按理應、自然、通常、規則寫得好卻沒人用、假設讀者注意），每項附重寫方向。
