# Execution Log

## Task Summary

調查 ticket CLI 建立/migrate 時 frontmatter 模板重複插入 Solution/Test Results 標題

---

## Problem Analysis


### 問題根因

ticket md 出現相同 Schema H2（如 `## Test Results`）重複的真實根因：

1. 手動 Edit ticket md 時誤插入第二份預設模板（Solution / Test Results placeholder）。
2. 自定義 H2（例 `## 簡化 WRAP 三問`）切斷 append-log 的 section regex 匹配範圍，使 append-log 改在末尾另起一份預設模板。
3. CLI 本身的 create / migrate 並未重複插入；migrate.py 僅改 ID 不改 body，create_ticket_body 只在 frontmatter 建立時呼叫一次。

### 影響範圍

- W10-060：原 line 228 重複 Test Results placeholder。
- W10-063：原 line 162 重複 Test Results placeholder。
- W10-062：實際無重複（acceptance-auditor 報告為概略歸類）。
- 影響 acceptance-auditor 對 Solution / Test Results 是否填寫的判讀，造成誤報。

### 相關 Error Pattern

PC-110（H2 自定義切斷 Schema section 擷取邊界）為共振來源；W17-072 已堵 agent 行為層，本 ticket 補 CLI 寫回層的 idempotent dedupe 形成雙層防護。

---

## Solution

於 ticket CLI 寫回層新增 idempotent `dedupe_schema_sections` helper，並在 append-log / complete 兩處整合呼叫，配合清理三張既有殘留 ticket 形成雙層防護（agent 行為層 W17-072 + CLI 寫回層本 ticket）。

### 三層方案落地

**Layer 1 — Idempotent dedupe helper**

新增 `dedupe_schema_sections(body: str) -> str` 於 `.claude/skills/ticket/ticket_system/lib/ticket_builder.py`：

- Schema 章節清單對齊 `agent-definition-standard.md` §章節結構規則：Task Summary / Problem Analysis / 重現實驗結果 / Solution / Test Results / Context Bundle / NeedsContext / Exit Status / Completion Info。
- 對每個重複出現的 Schema H2：
  - 全為 placeholder：保留首次出現位置，移除後續重複段。
  - 一份有實質內容、其餘 placeholder：保留有內容那份的位置，移除 placeholder。
  - 多份均有實質內容：保留首次出現位置與內容，將後續同名段內容附加到首段尾端。
- 非 Schema 章節（自定義 H2、ANA 重現實驗子段等）維持原樣不動。
- Idempotent：對同一輸入呼叫多次回傳值相同。

**Layer 2 — 整合到 commands 層**

在以下兩個寫回 ticket body 的入口呼叫 `dedupe_schema_sections`：

1. `commands/track_acceptance.py::execute_append_log` — append-log 寫回前 dedupe，避免歷史殘留 placeholder 與當前內容並存。
2. `commands/lifecycle.py::TicketLifecycleManager.complete_ticket` — `complete` 時 `sync_completion_body_fields` 之後 dedupe，作為最後守門。

兩處皆以 try/except 包裹，dedupe 失敗時不阻斷寫入並把例外訊息送 stderr（quality-baseline 規則 4 雙通道可觀測性）。

**Layer 3 — 清理既有殘留**

- W10-060：移除 line 228 重複 `## Test Results` 空白模板（保留 line 194 的實質內容）。
- W10-063：移除 line 162 重複 `## Test Results` 空白模板（保留 line 159 的實質內容）。
- W10-062：實際無重複，無需處理。

### 設計決策

- **不重寫 append-log 的 section regex**：規範化邊界的責任歸 W17-072（agent 行為層）+ W17-071 validator；本 ticket 的 dedupe 是針對「歷史 ticket 與寫回時殘留」的 idempotent 修補。
- **placeholder 判定標準**：純 frontmatter 模板殘留（含 HTML 註解 `To be filled` 字樣、中文「待填寫」括弧標記、`### 問題根因/影響範圍/相關 Error Pattern` 子標題、HTML 註解區塊）。任何實質文字、表格、列表、程式碼皆視為 substantive。
