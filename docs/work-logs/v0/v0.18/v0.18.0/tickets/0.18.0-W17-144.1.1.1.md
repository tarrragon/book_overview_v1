## 重現實驗結果

### 實驗方法

對 23 字測試集（10 簡體 + 7 台灣標準字 + 3 繁體標準字 + 3 共用字）逐一套用 Method 1/2/3，計算 accuracy。Method 6 改從 hook log 提取既存的 PC-085 self-check 警示資料。

### 實驗執行

執行於 `/tmp/test_methods.py`（PEP 723 inline metadata，uv run --script）：

```
=== Method 1: OpenCC s2t ===
  [FAIL] '台' (TAIWAN_STD): 判斷簡體=True, 期望=False
  [FAIL] '群' (TAIWAN_STD): 判斷簡體=True, 期望=False
  [FAIL] '干' (TAIWAN_STD): 判斷簡體=True, 期望=False
  Accuracy: 87% (20/23)

=== Method 2: OpenCC t2s + s2t 雙向 ===
  Accuracy: 87% (20/23) [同 Method 1 誤判]

=== Method 3: 純 codepoint 黑名單（PC-072 已知）===
  Accuracy: 100% (23/23)
```

Method 6 直接從 `.claude/hook-logs/askuserquestion-charset-guard/*.log` 提取 PC-085 self-check 警示，得到 12 個明確簡體字（详见 Solution 區塊）。

### 實驗發現

| 已驗證事實 | 證據 |
|----------|------|
| OpenCC s2t / t2s+s2t 雙向同樣 87% accuracy（誤判台/群/干） | Method 1/2 實測 |
| 純 codepoint 黑名單對本測試集 100% accuracy | Method 3 實測 |
| Hook 自身已有 PC-085 self-check 警示，可作為高 confidence 候選來源 | Method 6 hook log 直接讀取 |
| W17-144.1 detector 既有架構（OpenCC + ANCHORS 動態建構）已涵蓋 Method 1+3 雙重保險 | self-test 5 層全 PASS |

仍未驗證的假設：Method 5（教育部標準字反查）的 11151 字精度。需獨立 IMP 解析公開字表後實測。

| 6 | **Hook log 反推（PC-085 self-check 警示提取）**| `.claude/hook-logs/askuserquestion-charset-guard/*.log` | **極低** | **直接給出 12 字** | hook 自身警示 SIMPLIFIED_CHARS 漏網字 |

### Method 6 重大發現：Hook 自身已產出反推資料

`askuserquestion-charset-guard-hook.py` 的 PC-085 self-check 機制每次執行時警示 12 個明確漏網字，直接可用：

```
PC-085 self-check 漏網提示 12 處：
- 圖 / 图 / 図 — 簡 U+56FE
- 兩 / 两 / 両 — 簡 U+4E24
- 譯 / 译 / 訳 — 簡 U+8BD1
- 驛 / 驿 / 駅 — 簡 U+9A7F
- 氣 / 气 / 気 — 簡 U+6C14
- 樂 / 乐 / 楽 — 簡 U+4E50
- 觀 / 观 / 観 — 簡 U+89C2
- 檢 / 检 / 検 — 簡 U+68C0
- 權 / 权 / 権 — 簡 U+6743
- 鐵 / 铁 / 鉄 — 簡 U+94C1
- 轉 / 转 / 転 — 簡 U+8F6C
- 廣 / 广 / 広 — 簡 U+5E7F
```

這 12 字是「**已知必為簡體**」的高 confidence 候選，可直接擴充 detector.py 的 `_ANCHORS_SEED`。

### 結論：採行 A4 + Method 6 補強（落地）

| 候選 | 是否採行 | 理由 |
|------|---------|------|
| 採新方法替換 OpenCC | **否** | OpenCC 87% 對 ANCHORS 種子驗證仍有效（W17-144.1.1 已論證） |
| **補強 OpenCC 為多源** | **是（Method 6 落地）** | hook log 反推 12 字直接擴充 _ANCHORS_SEED；零新增依賴 |
| 維持 A4 不變 | 否 | 用戶質疑「還有沒有別的方法」表示應主動擴充 |
| Method 4 (Unicode CJK block) | 否 | 工程量高且簡繁共用範圍 |
| Method 5 (教育部標準字) | **追蹤但本 ticket 不開** | 11151 字反向過濾需獨立 IMP；非緊急 |

### 落地動作（W17-144.1 detector 補強）

`_ANCHORS_SEED` 加入 hook log 反推的 12 個簡體字：「圖 兩 譯 驛 氣 樂 觀 檢 權 鐵 轉 廣」對應簡體「图 两 译 驿 气 乐 观 检 权 铁 转 广」。OpenCC 動態建構時會自動驗證為簡體（self-test 第四層 `s2t(X) != X`）。
