## Problem Analysis

## Solution

<!-- To be filled by executing agent -->

---

拆兩層清單 + 統一簽名 + 消除 wrapper：

1. **簽名統一**：`_has_negation_prefix(prompt, start, end=None)` 新增 `end` 參數（忽略不用），與 per-match 規則 A/C 統一為 `(prompt, start, end) -> (bool, str)`。
2. **兩層清單**：
   - `PER_MATCH_WHITELIST_RULES: List[Callable[[str,int,int], Tuple[bool,str]]]` = [規則 A/B/C]
   - `PROMPT_LEVEL_WHITELIST_RULES: List[Callable[[str], Tuple[bool,str]]]` = [規則 D]
3. **Backward-compat alias**：`WHITELIST_RULES = [*PER_MATCH_..., *PROMPT_LEVEL_...]`，保留原 3 個 registry 測試。
4. **消除 _rule_b_wrapper**：簽名統一後不再需要。
5. **_detect_keyword_conflicts 清單驅動**：prompt-level 迴圈收集 reason；per-match 迴圈任一規則回 True 即跳過；meta-task verb 降級邏輯由 prompt_level_reason 驅動（不綁死 _is_meta_task_prompt）。

**擴充即生效**：新增規則 append 到對應清單無需改 _detect_keyword_conflicts。AC5 RED 測試 test_adding_per_match_rule_takes_effect 鎖死此契約。
### Context Bundle（Phase 3b 派發用）

**實作檔案**：`.claude/hooks/agent-dispatch-validation-hook.py`（主 repo，豁免 worktree）

**現狀**（commit 3905e0a7 + 8e39071d）：
- `WHITELIST_RULES: List[Callable]` 定義但未使用
- `_detect_keyword_conflicts` 硬編碼 `_is_quoted_match` / `_has_negation_prefix` / `_is_in_path_context` 三次
- `_rule_b_wrapper` 存在僅為簽名統一
- `_is_meta_task_prompt` 已由 .3.2 改為 per-match 標記方式

**修復策略**：拆兩層清單，統一簽名。

**步驟 1**：定義兩個清單
```python
# per-match 層：(prompt, start, end) -> (bool, reason)
PER_MATCH_WHITELIST_RULES: List[Callable[[str, int, int], Tuple[bool, str]]] = [
    _is_quoted_match,
    _has_negation_prefix_by_position,  # 重命名統一簽名（吃 end 但忽略）
    _is_in_path_context,
]

# prompt-level 層：(prompt) -> (bool, reason)
PROMPT_LEVEL_WHITELIST_RULES: List[Callable[[str], Tuple[bool, str]]] = [
    _is_meta_task_prompt,
]
```

**步驟 2**：修改 `_has_negation_prefix(prompt, start)` → `_has_negation_prefix(prompt, start, end=None)`，忽略 end 但簽名統一。消除 `_rule_b_wrapper`。

**步驟 3**：`_detect_keyword_conflicts` 改用迭代：
```python
for keyword, patterns in FORBIDDEN_KEYWORD_MAP.items():
    ...
    for m in pattern.finditer(prompt):
        # per-match 白名單過濾
        for rule in PER_MATCH_WHITELIST_RULES:
            is_wl, reason = rule(prompt, m.start(), m.end())
            if is_wl:
                whitelist_reason = reason
                break
        ...
```

prompt-level 規則在 `_emit_keyword_conflict_warning_if_any` 或 `_detect_keyword_conflicts` 開頭整體跑一次，結果附加到每個 match 的 `whitelist_reason`。

**步驟 4**：保留 `WHITELIST_RULES` 作 backward-compat alias，值為 `PER_MATCH_WHITELIST_RULES + PROMPT_LEVEL_WHITELIST_RULES`（或直接移除，改以兩個清單為權威）。

- AC1（迭代而非硬編碼）→ 步驟 3
