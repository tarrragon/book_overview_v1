### 問題根因

`phase-completion-gate-hook.is_worklog_write_operation()` 僅做路徑前綴匹配（`docs/work-logs/v[\d.]+`、`docs/work-logs/`）即視為 worklog 寫入並進入 phase 完成識別流程，導致：

1. ticket md（`docs/work-logs/v.../tickets/*.md`）內容含 Phase 字樣（如 ticket Problem Analysis 提及「Phase 4」「實作執行完成」）→ 被當 phase 報告 → 後續 `## Solution` 仍是 placeholder → 警告「缺 Solution 實際內容」
2. worklog 主檔（`docs/work-logs/v0.18.0/v0.18.0.md`）進度索引行含「W10-XXX Phase 4 評估」→ 同樣誤判

父 ANA W10-072 即時實證段已驗證本 ticket 撰寫過程中 PostToolUse Edit 多次觸發此警告。

### 影響範圍

- `.claude/hooks/phase-completion-gate-hook.py`（新增黑名單常數 + `is_worklog_write_operation` 排除邏輯）
- `.claude/hooks/tests/test_phase_completion_gate_hook.py`（新建測試）

### 修復摘要

依父 ANA W10-072 修復方向 1（路徑黑名單擴展），在 `is_worklog_write_operation()` 加入兩層排除：

1. **目錄黑名單**：路徑含 `/tickets/` → 排除（ticket md 不是 phase 報告）
2. **basename regex**：`v\d+\.\d+\.\d+\.md` 命中 → 排除（patch 級 worklog 主檔）

### 設計決策

- 新增模組常數 `WORKLOG_EXCLUSION_DIR_PATTERNS` 與 `WORKLOG_MAIN_FILE_REGEX`，與既有 `WORKLOG_PATTERNS` 並列，職責邊界清晰。
- 排除邏輯放在 `WORKLOG_PATTERNS` 命中後的二次過濾，避免為非 worklog 路徑做多餘檢查。
- 採用 `os.path.basename` 取檔名比對 regex（不依賴完整路徑），相容絕對與相對路徑。
- 排除時 logger.info 記錄具體原因，方便日後排查。
