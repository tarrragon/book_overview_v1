### 根因

mint-format-specialist 的 agent definition 不含 normalize 任務的目標字白名單或完成後自驗步驟（PC-150 系統性原因 1）。haiku model + effort: low 設定下，水部字族（污/汙/汲）token embedding 距離過近，導致 W11-028 案例出現 33% 錯誤率。

- 所有由 mint 執行的字元 normalize 任務均無自驗機制

### 整合位置

在 `mint-format-specialist.md` 的「核心職責」下新增「§4. 字元 Normalize 任務」子章節，位於「文件格式標準化」（§3）之後、「禁止行為」之前。

### 新增內容

1. **目標字白名單**：含「污→汙」（U+6C61→U+6C59）和「只读→唯讀」兩個初始字對
2. **混淆字警示表**：標註 汲 (U+6C72) 和 汚 (U+6C5A) 兩個禁止目標字及原因
3. **執行步驟**（四步驟）：確認目標字 → 執行替換 → grep 自驗 → 發現混淆字時立即修正
4. **grep 自驗指令**：
   - `git diff --cached | grep -E "[汲汚]"` — 偵測混淆字（預期：無輸出）
   - `git diff --cached | grep "汙"` — 確認目標字已寫入

- Why/Consequence/Action 三明示結構符合 document-writing-style.md 要求
- 整合策略：完整整合（內容 < 200 行，與 agent 核心職責強耦合）
- 白名單可擴充：後續發現新字對可直接追加表格列
