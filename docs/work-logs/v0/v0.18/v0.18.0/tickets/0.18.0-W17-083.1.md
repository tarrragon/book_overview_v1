## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 實作摘要

依 W17-083 Phase 2 sage 12 RED 測試契約 + Phase 3a pepper API 設計實作 6 公開 API：

| API | 用途 |
|-----|------|
| `HANDOFF_KEYWORDS` | 14 關鍵字常數（標題式 6 / 建議式 6 / 續行式 2）|
| `detect_handoff_keywords(content)` | 任一關鍵字命中即 True |
| `extract_ticket_ids(content, active_version)` | 完整 ID 優先 + 短 ID 用 active_version 補全 + 去重保序 |
| `extract_recent_content(worklog_path, since_mtime)` | mtime 過濾，不 raise（hook 靜默退出契約） |
| `extract_handoff_section(content)` | 從首個關鍵字命中切到下一 H1/H2（額外 API） |
| `find_worklog_path(version)` | 公開包裝既有 `_build_worklog_path` shim |

### 設計決策

- **完整 ID 優先策略**：避免短 pattern 重複命中已被完整 pattern 涵蓋的 ID；用 `_preceded_by_version_prefix()` helper 檢測短 match 是否實為某完整 ID 的 wave 部分
- **code block 不過濾**（S1-T12）：方案 D 接受誤報邊界，簡化邏輯
- **SOT 雙寫策略**（ARCH-020）：S3 hook 因 PEP 723 隔離環境無法 import 本模組，需照搬 HANDOFF_KEYWORDS / regex；docstring 已標註 SOT 維護責任

### 不在本 ticket 範圍

- S3 Stop hook `stop-worklog-handoff-sync-check-hook.py` 實作（W17-083.2）
