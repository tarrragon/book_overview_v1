### 問題根因

`.claude/hooks/process-skip-guard-hook.py` line 271 `def main()` 含 8 個 emit 點：
- 7 處重複格式：`print(json.dumps(generate_hook_output("UserPromptSubmit"), ensure_ascii=False, indent=2)) + return EXIT_SUCCESS`（行 291-292, 295-296, 302-303, 309-310, 320-321, 339-340, 355-357）

### 重構方案選擇

採方案 1（`_emit_basic()` helper），原因：

| 維度 | 方案 1（helper） | 方案 2（單一出口） |
|------|----------------|-----------------|
| 對既有結構衝擊 | 低（純抽 helper） | 中（reminder 變數需貫穿全函式） |
| 7 處重複消除 | 達成 | 達成 |
| 風險（語意改變） | 極低 | 中（多分支 reminder 預設值需設計） |
| diff 集中度 | 高 | 低 |

方案 1 在達成 F2/F4 acceptance 同時保留可讀性與低風險，符合 quality-baseline 規則 2 的 Phase 4「無需大改」結論型重構。

### 具體變更

1. 新增 `_emit_basic()` helper（line 271-279）：封裝 `print(json.dumps(generate_hook_output("UserPromptSubmit"), ...)) + return EXIT_SUCCESS` 共 2 行邏輯。
2. `main()` 內 7 處重複 emit 改為 `return _emit_basic()`：
   - line 291-292 (JSON 解析失敗)
   - line 294-295 (input_data None)
   - line 302-303 (subagent 環境)
   - line 309-310 (空 prompt)
   - line 320-321 (active dispatch 靜音)
   - line 339-340 (SKIP_SA_REVIEW type/phase 靜音)
   - line 355 (未偵測到 skip intent，原 else 分支)
3. line 348 特殊 reminder emit 保留原樣（含 reminder payload，不經 helper）並附註解說明保留理由。
4. `else` 分支扁平化為函式底部 fallback，移除巢狀。

### 不在本 ticket 範圍

- `generate_hook_output` 簽名變更（方案 2 才需要）
- 其他 hook 同型重複 emit 收斂（屬獨立 ticket）
