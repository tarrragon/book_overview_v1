## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復內容

在 `.gitignore` 既有 `.claude/hook-state/` 規則之後新增：

```
# Claude session state markers (runtime state, written by session-start hooks)
.claude/state/
```

### 設計決策

1. **位置選擇**：插入到 `.claude/hook-state/` 之後，因兩者性質相似（runtime state, auto-managed by hooks），維持相關規則的視覺鄰近
2. **規則粒度**：用 `.claude/state/` 整目錄忽略，而非單檔（`last-session-start.marker`），因為該目錄定位為 session state markers 容器，未來可能加入其他 marker（如 last-task-completed、last-prompt-token 等），預先涵蓋
3. **註解風格**：對齊既有規則註解格式（"runtime state, auto-managed by hooks"），明示來源與性質

### 為何不需要 git rm --cached

`git ls-files .claude/state/` 為空，確認該目錄下沒有任何檔案曾被 git tracking。session-start hook 每次寫入 marker 都只是「新建 untracked file」狀態，從未被 add。因此 acceptance 第 4 項自動滿足（無需 git rm --cached）。
