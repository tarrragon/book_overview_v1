## 實作摘要

在 `ticket_system/commands/handoff.py` 新增 `_execute_auto_handoff()` 與三個 CLI 參數（`--auto` / `--from-ticket-id` / `--direction`），並在 `execute()` 最前端路由。

### 設計決策
- **不共用 `_create_handoff_file_internal`**：auto 模式跳過 status / blockedBy / duplicate 驗證（scheduler 用途），JSON 多一個 `auto_generated=true` 欄位。
- **Chain 透傳而非重解析**：`ticket.get("chain", {})` 直接取 ticket yaml 已填好的 root/parent/depth，符合「不自行重解析」要求。
- **Direction 白名單**：to-parent / to-child / to-sibling / to-source / context-refresh，支援 `:TARGET_ID` 後綴。
- **覆寫語義**：scheduler 重生成時直接覆寫（不呼叫 duplicate 檢查）。

### JSON Schema（對齊 handoff-reminder-hook.py）
ticket_id / direction / timestamp / from_status / title / what / chain{root,parent,depth} / resumed_at / auto_generated

### 不破壞既有行為
execute() 先檢查 --auto 旗標才分流；手動模式（--to-parent/--to-child/--to-sibling/--context-refresh/--status/--gc）流程未動。
