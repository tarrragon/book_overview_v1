### 3. 行為規格

#### 3.1 Hook 寫入 event

1. 偵測 N >= 1 衝突
2. 組裝 event dict
3. 序列化為單行 JSON + `\n`（**必須單次 `f.write(line)`** 避免多次 write 交錯）
4. 寫入前 `fcntl.flock(f, LOCK_EX)`（並發安全）
5. 寫失敗 → logger.warning + stderr（規則 4 雙通道），**不 raise**

#### 3.2 annotations.json 寫入（atomic rename）

1. 讀 annotations.json（不存在視為空 dict）
2. 修改 dict
3. 寫 tmp 檔
4. `os.replace(tmp, annotations.json)`

#### 3.3 邊界

| 情境 | 行為 |
|------|------|
| events.jsonl 不存在/空 | list/stats 輸出「尚無事件」，exit 0 |
| 父目錄不存在 | Hook 寫入時自動建立 |
| 格式錯誤行 | skip + stderr「第 N 行格式錯誤」；stats 輸出 `malformed_lines: N` |
| 重複 event_id | list 顯示首筆；不主動去重 |
| annotations 指向不存在 event_id | list 顯示 orphan；stats 不計入 |
| TP + FP = 0 | 誤報率顯示 `N/A`，禁 ZeroDivisionError |

### 1. 資料模型

#### 1.1 JSONL Event Schema（`.claude/hook-logs/agent-dispatch-validation/events.jsonl`）

每行一個 JSON object，對應一次「Hook 偵測到關鍵字衝突」的事件：

| 欄位 | 型別 | 說明 | 範例 |
|------|------|------|------|
| event_id | string | 唯一 ID，格式 {timestamp}-{prompt_hash[:8]} | 20260419T053012Z-a1b2c3d4 |
| timestamp | string (ISO 8601, UTC) | 事件發生時間 | 2026-04-19T05:30:12Z |
| hook_version | string | 寫出此 event 的 Hook 版本，便於 schema 演進 | w11-004.1.1 |
| kind | string | "event" 或 "annotation"（tombstone 用） | event |
| subagent_type | string | 派發目標 agent 名稱 | sage-test-architect |
| conflicts | array<object> | 衝突清單（下表） | 見下 |
| prompt_hash | string | prompt SHA-256 前 16 字元（去重 + 不存原文） | a1b2c3d4e5f67890 |
| prompt_length | int | prompt 原文長度（字元數） | 1523 |
| annotation | string \| null | true_positive / false_positive / unknown / null | null |
| annotation_note | string \| null | 標註備註（可選） | "prompt 引用 agent 定義作為說明" |
| annotated_at | string \| null | 標註時間（ISO 8601） | null |

**conflicts 子物件欄位**：

| 欄位 | 型別 | 說明 |
|------|------|------|
| action | string | agent 禁止行為標籤（如「實作程式碼」） |
| keyword | string | FORBIDDEN_KEYWORD_MAP 的 key（如「實作」） |
| matched_pattern | string | 命中的 regex pattern 原文（pattern.pattern） |
| prompt_excerpt | string | 命中片段前後各 20 字元上下文（換行替換為空白） |

#### 1.2 Annotation 存放策略

**選定：JSONL append-only + tombstone（LWW merge）**

- 標註時不改寫舊行（避免競爭寫入損壞 JSONL）
- 新增一行 tombstone：`{"event_id":"...","kind":"annotation","annotation":"true_positive","annotated_at":"...","annotation_note":"..."}`
- 讀取時以「同 event_id 最後一筆 kind=event 為事件本體 + 最後一筆 annotation 為標註」
- 優點：純 append、崩潰安全；缺點：長期膨脹（v2 做 compaction，另開 Ticket）

#### 1.3 向後相容舊格式

- 既有 `.log` 檔（INFO 純文字，約 10 筆）保留不轉換
- 新事件一律寫 `events.jsonl`
- stats 輸出 `legacy_unstructured_count`（glob 計數 .log），不納入 TP/FP 計算
- Hook 改造後雙寫過渡期，移除舊寫入另開 Ticket

### 3. 行為規格（Happy Path + 邊界）

#### 3.1 Hook 寫入 event（happy path）
1. Hook 偵測 N 個衝突（N >= 1）
2. 組裝 event dict，計算 prompt_hash 與每筆 prompt_excerpt
3. `open(events.jsonl, 'a', encoding='utf-8')` append 一行 JSON + `\n`
4. 寫失敗（OSError / 唯讀 / 磁碟滿）→ logger.warning + stderr（遵 quality-baseline 規則 4 雙通道），**不 raise**（維持 Hook 非阻擋）

#### 3.2 邊界：無 log 檔 / 空檔
- events.jsonl 不存在或空：list / stats 輸出「尚無事件」，退出碼 0
- 父目錄不存在：Hook 寫入時自動建立；CLI 讀取時視為空

#### 3.3 邊界：格式錯誤行
- 非 JSON / 缺必填欄位 → skip + stderr 警告「第 N 行格式錯誤」
- 統計繼續；stats 輸出 `malformed_lines: N`

#### 3.4 邊界：重複 event_id
- 同秒同 prompt 可能產生同 event_id
- 讀取時以「同 event_id 最後一筆 kind=event 為事件本體」
- 不主動去重（保留觀測痕跡）

#### 3.5 邊界：annotation tombstone 指向不存在 event_id
- 記錄 warning，不納入統計

#### 3.6 邊界：TP+FP 皆為 0
- 誤報率顯示 `N/A`，**禁止 ZeroDivisionError**
