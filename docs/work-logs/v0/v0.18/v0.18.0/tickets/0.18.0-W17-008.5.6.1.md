1. **invalid-section**：`ticket track append-log <id> --section "NotAValidSection" "..."` → CLI 輸出 ErrorEnvelope（含 `__error_envelope_v1__` + errno=`INVALID_SECTION` + 列出 valid sections）

每個場景至少 assert：
- exit code != 0
- stderr 或 stdout 含 `__error_envelope_v1__`
- 含預期 errno（`component:` `action:` `errno:` `hint:` 結構欄位）

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 實作摘要（W17-008.5.6.1）

新增 `.claude/skills/ticket/tests/test_error_channel_integration.py`（共 7 個測試案例）覆蓋 5 個錯誤通道整合場景 + 2 個補強檢查。純測試 ticket，未修改任何產品程式碼。

### 場景設計與路徑分類

| # | 場景 | 路徑 | 觸發指令 | 預期 errno / 訊號 |
|---|------|------|---------|------------------|
| 1 | invalid-section | legacy str（W17-008.5.2 雙路徑保留向後相容） | `track append-log <id> --section "NotAValidSection"` | INVALID_SECTION 訊息 + 有效值清單 |
| 2 | unrecognized-args | argparse 預設 POSIX | `track list --not-a-real-flag` | `unrecognized arguments` + 不含 envelope 標記 |
| 3 | section-not-found | legacy str（in-process + monkeypatch 構造缺 section body） | `execute_append_log(args)` 對缺 Solution section 的 fake body | SECTION_NOT_FOUND 訊息 + 現有 H2 標題列舉 |
| 4 | missing-required-field | ErrorEnvelope（W17-008.5.3） | `create --type IMP --action ... --target ...`（缺 when/why/how_strategy） | envelope 標記 + 必填欄位類 errno（CHECKLIST_VALIDATION_FAILED / WHY_REQUIRED 等任一） |
| 5 | wrong-value-type | ErrorEnvelope（W17-008.5.4 ArgparseFormatErrorParser） | `track list --format INVALID_FORMAT_VALUE` | envelope 標記 + errno=INVALID_CHOICE + hint 含 argparse 原訊息 |

補強：
- TestEnvelopeMarkerConsistency::test_marker_value_is_v1 — 防止 v2 升級時 messages.py 與 hook 標記未同改
- TestEnvelopeMarkerConsistency::test_envelope_output_format_is_stable — 驗證 _render_envelope 行序契約（marker → component → action → errno）

### 設計決策

- **subprocess.run 模式優先**（場景 1/2/4/5）：取得真正 CLI 端到端輸出，含 argparse + envelope rendering 完整路徑；subprocess 捕捉的 stdout/stderr 不受 outer shell chpwd hook 干擾。
- **in-process 例外**（場景 3）：構造缺特定 section 的真實 ticket md 成本高且易污染 repo（受 conftest._assert_no_repo_pollution 攔截）；改用 monkeypatch load_ticket 注入 fake body，仍走 execute_append_log → section_locator.find_section 完整路徑。
- **import ERROR_ENVELOPE_VERSION_MARKER**：避免硬編碼 `__error_envelope_v1__`，未來升級至 v2 時測試自動跟隨。
- **Marker negative assertion**（場景 2）：明確斷言 `ERROR_ENVELOPE_VERSION_MARKER not in combined`，驗證 W17-008.5.4「保留純語法錯誤 POSIX 預設路徑」設計界線。
- **Errno 容忍清單**（場景 4）：missing-required-field 在 create.py 內可能由不同必填驗證器先觸發（WHY_REQUIRED / WHEN_REQUIRED / HOW_STRATEGY_REQUIRED / CHECKLIST_VALIDATION_FAILED），任一皆屬合法分類；測試以集合斷言避免實作細節耦合。

### 不在本 ticket 範圍

- linux 視角審查（W17-008.5.6.2 處理）
- track_acceptance.py / track_query.py 從 legacy str 遷移至 ErrorEnvelope（屬後續 ticket；本場景 1/3 驗證的是當前 W17-008.5 group 落地的混合狀態）
