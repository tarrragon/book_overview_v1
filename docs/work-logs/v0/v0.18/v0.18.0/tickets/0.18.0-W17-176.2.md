### 問題根因

**根因分類：GC 條件未涵蓋此情境（reminder hook 路徑解析 bug）**

`handoff-prompt-reminder-hook.py`（UserPromptSubmit 事件，每 turn 觸發）的 `is_ticket_completed()` 和 `resolve_ticket_path()` 使用硬編碼的扁平路徑模式：

```python
# L108 / L146
ticket_path = project_root / "docs" / "work-logs" / f"v{version}" / "tickets" / f"{ticket_id}.md"
```

實際專案目錄結構為三層階層：
```
docs/work-logs/v0/v0.18/v0.18.0/tickets/0.18.0-W17-174.md
```

因此 `ticket_path.exists()` 永遠回傳 `False`，`is_ticket_completed()` 回傳 `False`，已完成的 ticket 不被跳過，每 turn 持續提醒。

**對比**：`handoff-auto-resume-stop-hook.py`（Stop 事件）的同名函式已在 W17-165 L2-C 改用 `find_ticket_file()`（`hook_utils/hook_ticket.py`），支援三層階層結構，GC 正常運作。

### 影響範圍

- 檔案：`.claude/hooks/handoff-prompt-reminder-hook.py` L89-123（`is_ticket_completed`）、L125-154（`resolve_ticket_path`）
- 影響：所有已完成 ticket 的 handoff pending JSON 在 reminder hook 中無法被正確跳過，持續產生無效提醒

### 相關 Error Pattern

無直接相關已知 error-pattern。屬 ARCH-020 類型（跨進程同步修復遺漏）：stop hook 升級了路徑解析，reminder hook 未同步。

### 根因分類

**GC 條件未涵蓋此情境**（更精確：reminder hook 路徑解析未升級，導致 completed ticket 過濾失效）

### 修復方案

將 `handoff-prompt-reminder-hook.py` 的 `is_ticket_completed()` 和 `resolve_ticket_path()` 改為使用 `hook_utils/hook_ticket.py` 的 `find_ticket_file()`，與 stop hook 對齊。

修改範圍：
1. 移除 L89-154 的兩個函式（`is_ticket_completed` + `resolve_ticket_path`）
2. 改 import `find_ticket_file` 和 `parse_ticket_frontmatter` from `hook_utils`
3. `is_ticket_completed` 改呼叫 `find_ticket_file()` 取路徑
4. `resolve_ticket_path` 改為 wrapper 呼叫 `find_ticket_file()`
