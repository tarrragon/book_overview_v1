### 實作摘要

在 `.claude/pm-rules/ticket-lifecycle.md` 新增「ANA 衍生 Ticket Priority 繼承（強制）」章節（位於「ANA Ticket 落地下游血緣選擇」與「ANA Ticket 完成階段衍生 Ticket 檢查」之間）。

### 章節內容

1. **預設繼承規則表**：區分防護性 ANA（繼承 source priority）vs 純研究性 ANA（獨立指定）
2. **Why 段落**：說明語意延續（分析急迫性 → 落地急迫性），防止 runqueue 排序矛盾
3. **降級條件表**：三種合法降級情境（非阻塞性優化、已有替代防護、跨版本延後），每項含必要條件
4. **降級明示要求**：必須在 ticket body 明示理由；禁止無理由降級、「看起來不急」降級、未引用替代防護
5. **檢查清單**：PM 建立 ANA 衍生 Ticket 時四項自檢
6. **CLI 行為提醒**：目前 `--source-ticket` 不自動繼承 priority，本規則作為未來 CLI 實作依據
