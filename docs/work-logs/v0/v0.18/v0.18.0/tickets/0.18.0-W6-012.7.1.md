### 實作摘要

採取「Controller 為實際渲染主體」路線，發現 `src/ui/book-grid-renderer.js` 未被 `src/overview/` 引用（grep 確認），實際 row 渲染在 `overview-page-controller.js#createBookRow`。故將 checkbox UI 與 selection state 落地於 controller，不修改 dead-code 的 book-grid-renderer.js。

### 變更檔案

| 檔案 | 變更性質 |
|------|---------|
| `src/overview/overview-page-controller.js` | 新增 `selectedBookIds: Set` state + selectAll handler + per-row checkbox handler + 渲染 checkbox 欄位 + header checkbox 同步邏輯 |
| `src/overview/overview.html` | 表格 header 新增 checkbox column（`#selectAllHeaderCheckbox`） |
| `src/overview/overview.css` | 新增 `.select-col` / `.row-checkbox` / `.row-selected` 樣式 |

### 設計決策

1. **State source of truth**：`this.selectedBookIds = new Set<bookId>`，UI checkbox.checked 與 row.row-selected class 一律從 Set 計算（WRAP 選項 a，維護性優於每次掃 DOM）
2. **bookId fallback**：`_getBookId(book)` 採 `book.id || book.bookId || null`，無 id 的書本不渲染 checkbox（避免 selection 對應不到實體）
3. **selectAll toggle 邏輯**：若 displayedBooks 全已選 → 清空該範圍；否則 → 加入該範圍。保留 displayed 範圍外的已選（雖當前無 UI 路徑暴露此狀態，但 Set 語義一致）
4. **Per-row toggle 不 re-render 全表**：個別 checkbox 點擊只 `_updateRowSelectedClass(bookId)` toggle 單列 class，避免使用者操作時整表抖動。selectAll 才整表 re-render（必要因可能影響多 row）
5. **selectAllBtn + header checkbox 並存**：兩者綁同一 handler `handleSelectAllToggle`，header checkbox 在 render 後透過 `_syncHeaderCheckboxState` 同步勾選狀態
6. **CONSTANTS.TABLE.COLUMNS**：從 6 改為 7（新增 select-col），empty state colspan 同步調整
7. **事件冒泡防護**：row checkbox `e.stopPropagation()` 避免將來 row click handler（W6-012.7.2+）與 checkbox 衝突

### 範圍上限遵守

未修改 `src/overview/book-exporter.js`（屬 W6-012.7.2 範圍）；selectedBookIds 目前僅供 UI 視覺反饋，未接 export 邏輯，符合本 ticket 邊界。

### 不在本 ticket 範圍

- 批次操作（匯出選取 / 刪除選取）→ W6-012.7.2
- selectedBookIds 跨 filter 變更時的清理策略（目前 filter 切換後選取狀態自然保留 Set 內所有 ID，displayed 範圍外的不可見但仍存在）→ 待 W6-012.7.2 評估
