## Solution

### 改善方案表（2026-05-05）

#### 修復方向 1：phase-completion-gate-hook 路徑排除規則

**問題**：worklog 索引行 + ticket md 含 Phase 關鍵字均誤觸發。本 session 同檔 Edit 兩次連續觸發確認雜訊累積。

**修復方向**（候選排序）：
1. **路徑黑名單擴展（首選）**：在 `is_worklog_write_operation()` 增加排除規則：
   - 排除 `tickets/` 子目錄（ticket md 不是 phase 報告）
   - 排除 worklog 主檔（`v\d+\.\d+\.\d+\.md` 對應 `v[major][minor][patch].md` 自身）
2. **內容啟發式補強**：含 PHASE_COMPLETION_KEYWORDS **但同時** YAML frontmatter 含 `id:` 欄位 → 視為 ticket md 不是 phase 報告
3. **檔案類型啟發式**：phase 報告通常是 `phaseN-*.md` 命名 → 限定觸發

**ROI**：
- 影響範圍：所有 PM/agent 寫入 worklog 主檔/ticket md
- 成本：~30-60 分鐘（單檔修改 + 既有測試補強）
- 收益：消除每 session 多次（5+）誤觸發雜訊
