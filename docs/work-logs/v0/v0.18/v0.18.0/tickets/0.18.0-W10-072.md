## 新增觸發案例（2026-04-18，來自 W10-017 審查後 complete 嘗試）

### 案例：ANA Ticket 無法在 session 內 complete

**場景**：
- W10-017（ANA 類型）WRAP 四階段分析 + parallel-evaluation 審查完成
- 分析結論衍生 10 個 children（其中 3 個 closed 為 superseded，7 個 pending 作為後續實作）
- 執行 `ticket track complete 0.18.0-W10-017` 時被 acceptance-gate-hook block
- Hook 要求所有 7 個 pending children 完成才能 complete parent

### 問題本質

| 類型 | children 語意 | complete 觸發時機 |
|------|-------------|---------------|
| IMP | 拆分的子任務（parent 的實作步驟） | children 全部完成 |
| ANA | **分析結論衍生的實作清單（parent 的後續工作）** | **分析本身完成即可 complete** |

acceptance-gate-hook 對兩類一視同仁，導致 ANA Ticket 必須等所有衍生實作完成才能 complete——可能跨多個 Wave 或版本，實際上永遠無法 complete。

### 建議改善方向（補充 W10-072 原範圍）

1. Hook 判斷 Ticket type：若為 ANA，豁免 children 全完成檢查
2. 改為警告：「ANA 的 N 個 children pending，確認已建立追蹤再 complete？」
3. 或新增 `--allow-ana-incomplete-children` flag

### 替代解法（本 session 採用）

- W10-017 保持 in_progress
- 下個 session 開始做 017.8（core state 函式）
- 所有 children 實作完成後才 complete W10-017
- 符合 Hook 當前邏輯，但實際職責已完成
