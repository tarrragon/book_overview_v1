### 問題根因

`ticket-quality-gate-hook.py` `should_trigger_check()` line 144 使用 substring 匹配判斷檔案是否屬 ticket：`if not any(keyword in file_path for keyword in ticket_path_keywords)`。配置 `ticket_path_keywords` 含 `-ticket-` 與 `-task-` 過寬子字串，會誤命中 `.claude/error-patterns/` 等框架資產下含「ticket」字眼的檔名（如 `PC-098-...current-ticket-id.md` 命中 `current-ticket-id` 中段的 `-ticket-`）。一旦命中，檔案內容若含 Layer N / `## 實作步驟` 等結構標記（描述其他 hook 行為時常見），即觸發 C2/C3 檢測必然失敗 → exit 2 BLOCK。

額外發現：hook 雖 `try: from config_loader import load_quality_rules`，但只將 `.claude/hooks/` 加入 sys.path，而 `config_loader` 位於 `.claude/lib/`，故 import 一直靜默失敗，fallback 為 `{}`，配置實際值由 `should_trigger_check()` 內部 inline default 提供。本 ticket 同時修復 sys.path 與 inline default。

### 影響範圍

- `.claude/hooks/ticket-quality-gate-hook.py`：`should_trigger_check()` + sys.path
- `.claude/config/quality_rules.yaml`：`ticket_path_keywords` 縮緊 + 新增 `ticket_path_excludes`
- 受惠：`.claude/error-patterns/`、`.claude/methodologies/`、`.claude/rules/`、`.claude/references/`、`.claude/pm-rules/`、`.claude/skills/`、`.claude/hooks/`、`.claude/agents/` 下所有 md 檔（50+ 含 `-ticket-`/`-task-` 子字串者）

- 父 ANA: 0.18.0-W10-072.4（重現實驗）

### 修復摘要

採 W10-072.4 ANA 修復方向 1（路徑黑名單 + keyword 縮緊），三層落地：

1. **`.claude/config/quality_rules.yaml`**
   - `ticket_path_keywords` 移除過寬的 `-ticket-` / `-task-`，改為含父目錄錨點的 `docs/work-logs/` / `docs/tickets/` / `/tickets/`
   - 新增 `ticket_path_excludes` 黑名單，含 `.claude/error-patterns/` / `methodologies/` / `rules/` / `references/` / `pm-rules/` / `skills/` / `hooks/` / `agents/` 共 8 個目錄
   - 配置註解明示 W10-100 來源與設計理由

2. **`.claude/hooks/ticket-quality-gate-hook.py`**
   - `sys.path` 補加 `.claude/lib/`，修復 `config_loader` import 一直靜默失敗的既有 bug
   - `should_trigger_check()` 條件 3a 新增黑名單檢查（命中即 return False），優先於 keyword 檢查
   - inline default 同步縮緊（移除 `-ticket-` / `-task-`），保證 yaml 缺失時 fallback 行為與 yaml 一致

3. **`.claude/hooks/tests/test_ticket_quality_gate_path_filter.py`**（新建）
   - 5 案例：(1) 黑名單 error-patterns + ticket substring；(2) 黑名單 methodologies；(3) 真實 ticket md 仍觸發；(4) 縮緊後純 `-ticket-` substring 不再觸發；(5) `/tickets/` 父目錄錨點觸發
   - 動態載入 hook 模組（檔名含連字號）；fixture 清除 config_loader 快取

### 設計決策

- 黑名單優先於 keyword：即使檔名或內容意外命中 keyword/structure marker，命中黑名單即 return False
- keyword 改用父目錄錨點而非完全移除：保留 `docs/work-logs/` / `docs/tickets/` / `/tickets/` 三模式，涵蓋本專案 + 跨專案 sync 後的常見 ticket 路徑
