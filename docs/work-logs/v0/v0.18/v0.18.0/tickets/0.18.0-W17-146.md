### 問題根因

`_apply_context_resume()` 只取出 handoff JSON 的 `ticket_id` 欄位（source ticket_id）作為過濾依據，未解析 `direction` 欄位取出真正的 target ticket id。

**bug 邏輯路徑**（`commands/track_runqueue.py:247-256`）：

```python
def _apply_context_resume(tickets, context):
    if context != "resume":
        return tickets
    pending_ids = _get_pending_handoff_ticket_ids()  # 取 handoff JSON 的 ticket_id（source）
    if not pending_ids:
        return []
    return [t for t in tickets if t.get("id") in pending_ids]
```

`_get_pending_handoff_ticket_ids()` 走 `_get_pending_handoff_info()`，僅讀 `data.get("ticket_id")`（source），完全忽略 `data.get("direction")` 中包含的 target。

### 重現實驗（已完成）

當前 `.claude/handoff/pending/` 狀態：

```text
0.18.0-W17-110.1.json: ticket_id=110.1 (completed), direction=to-sibling:110.2
0.18.0-W17-110.2.json: ticket_id=110.2 (completed), direction=to-sibling:110.3
```

110.3 是 pending（真正應接手的目標）。

執行：

```bash
ticket track runqueue --context=resume --top 3
# 實際輸出：（無 resume 候選；當前無 handoff pending ticket）
# 期望輸出：列出 W17-110.3 為接手候選
```

**根因**：`pending_ids = {110.1, 110.2}`，過濾後 scoped=[110.1, 110.2]，再經 `_render_list` 內部 `_is_listable` 過濾掉 completed → 0 結果。target 110.3 從未進入 pending_ids。

### 修復策略（evidence-driven-bugfix）

**最小修復**：改寫 `commands/track_runqueue.py` 的 `_apply_context_resume()`，從 `_get_pending_handoff_info()` 取出每個 handoff 的 `direction`，按下表分流：

| direction 類型 | 取值 | 範例 |
|---------------|------|------|
| `to-sibling:X` / `to-parent:X` / `to-child:X`（task-chain 含 target） | `X`（target_id） | `to-sibling:0.18.0-W17-110.3` → 110.3 |
| `to-sibling` / `to-parent` / `to-child`（task-chain 無 target） | source ticket_id（fallback） | 沒有 `:` 後綴退回 source |
| `context-refresh` / `next-wave` / `auto`（非 task-chain） | source ticket_id | 自身就是接手目標 |
| 空字串 / 未知格式 | source ticket_id（fallback） | degrade 不 crash |

### 復用現有工具（避免 ARCH-020）

`lib/handoff_utils.py` 已提供：
- `is_task_chain_direction(direction) -> bool`
- `extract_direction_target_id(direction) -> Optional[str]`

修復應 import 這兩個函式，**不要**在 `track_runqueue.py` 重寫一份。

### 修復後的目標邏輯（虛擬碼）

```python
def _apply_context_resume(tickets, context):
    if context != "resume":
        return tickets
    handoff_info = _get_pending_handoff_info()
    if not handoff_info:
        return []
    candidate_ids: Set[str] = set()
    for source_id, info in handoff_info.items():
        direction = (info or {}).get("direction", "") or ""
        if is_task_chain_direction(direction):
            target = extract_direction_target_id(direction)
            candidate_ids.add(target if target else source_id)
        else:
            candidate_ids.add(source_id)
    return [t for t in tickets if t.get("id") in candidate_ids]
```

注意：保留現有 `_get_pending_handoff_ticket_ids()` thin wrapper（向後相容，可能仍被其他模組使用）；不要刪除。
