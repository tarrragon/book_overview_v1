## 方案 C 落地：雙通道輸出

### 修改檔案
- .claude/hooks/askuserquestion-charset-guard-hook.py：main() 攔截路徑改為 emit_hook_output + exit 0
- .claude/hooks/tests/test_askuserquestion_charset_guard.py：新增 TestDualChannelOutput 類別（6 測試）

### 核心變更
1. 新增 from hook_utils.hook_io import emit_hook_output
2. main() 攔截分支改為：
   - stderr：極簡摘要（錯誤標題 + 違規清單 + PC-072 連結），3-5 行，不含修復方式 1-4 步
   - stdout：emit_hook_output(PreToolUse, permission_decision=deny, permission_decision_reason=完整 BLOCK_MESSAGE_TEMPLATE)
   - exit code：return 0（讓 JSON 生效，exit 2 會忽略 JSON）
3. logger.warning 保留（file log 可觀測性，規則 4）

### BLOCK_MESSAGE_TEMPLATE 保留不動
既有內容（修復方式 1-4 步、PC-072 連結）轉為 permissionDecisionReason 主體，給 Claude 讀取。
