### 修復摘要

於 `.claude/hooks/tests/test_agent_dispatch_validation_hook.py` 既有 W11-004.7 測試區塊後新增 W11-004.7.3 區塊，含 4 個邊界測試：

1. `test_resolve_l3_not_triggered_when_ticket_scope_has_external_claude` — ticket scope 同時含 `.claude/` + 外部 `.claude/` + `src/`，驗證 L3 inner check 因 `ticket_e=True` 失敗 → 保留混合分類
2. `test_resolve_l3_not_triggered_when_prompt_has_external_claude_marker` — L1 prompt 已偵測 external（`e=True`），L3 outer guard `not e` 直接擋住覆蓋
3. `test_resolve_symlink_path_not_expanded_by_regex` — 建立真實 symlink，驗證 regex 純字串匹配（不展開）；含 case A（外部絕對路徑）+ case B（相對 `.claude/` 開頭）
4. `test_resolve_symlink_in_ticket_where_files_not_expanded` — `_classify_paths` 對「看似 symlink 的外部絕對路徑」依字串前綴比對，不展開 filesystem

### 設計決策

- 不修改 hook 主檔（W11-004.7.1 已重構完，本 ticket 純補測）
- Symlink 案例 B 改為純相對路徑開頭（`.claude/foo.py`）以對齊既有 regex 設計（`_RELATIVE_CLAUDE_PATTERN` 要求 `.claude/` 前無 `/` 或 word char）；`./some_link/.claude/...` 並非當前 regex 接受形式，避免測試與實作意圖矛盾
