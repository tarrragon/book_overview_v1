  變 100644），導致 Stop/SessionStart hook 執行時回傳 Permission denied 錯誤。本次 sync-pull 實測需手動對
  40 個檔案（頂層 30 + acceptance_checkers/ 子目錄 10）執行 chmod +x 才能恢復。


  歷史先例：commit ac2dcbaa (chore: add executable bit to acceptance checker hook (auto-fix))
  曾修過同類問題，說明此缺陷反覆出現，屬系統性而非偶發。


  根本原因：sync-claude-pull.py 腳本同步檔案時未保留來源 repo 的 file mode（推測使用 shutil.copy 而非 shutil.copy2，或
  git checkout 時 core.filemode 設定未生效）。


  影響：

  (1) Stop hook 無法執行 → 結束時缺少 session-experience-persistence、worktree-auto-commit 等提醒

  (2) 每次 sync-pull 都需要 PM 手動 chmod，增加認知負擔

  (3) 違反規則 4「Hook 失敗必須可見」的精神（失敗雖可見，但根因未治本）'
