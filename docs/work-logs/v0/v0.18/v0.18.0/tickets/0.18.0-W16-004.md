## Problem Analysis

## Context Bundle

### 現象確認
本次 /sync-pull 後觀察到 Stop hook 回傳 Permission denied：
- session-experience-persistence-reminder-hook.py
- worktree-auto-commit-hook.py

`ls -l` 顯示檔案為 `-rw-r--r--`（未帶 +x），git diff --summary 顯示 40 個檔案 mode 從 100755 降為 100644。

### 根因定位（實證）
執行 `git clone --depth 1 https://github.com/tarrragon/claude.git` 到 temp 後對遠端 repo 檢查 `git ls-files --stage`：
- 遠端總計 1226 個 100644 + 147 個 100755
- hooks/acceptance_checkers/*.py 遠端記錄為 100644（本地 .git HEAD 是 100755）

**結論**：上游 tarrragon/claude.git 的 mode 本身已損壞。shutil.copy2 雖保留 source mode，但 source（clone 下來的 temp_dir）本身 mode 就錯，任憑如何保留都是錯的。

### 惡性循環根源
1. 某次 push 沒保留 mode → 遠端 repo 變 644
2. 後續 pull 從 644 拿 mode → 本地也變 644
3. 再次 push 把 644 推回去 → 惡化循環

### 解法邊界
- Pull 端 safety net：pull 後對 `.claude/hooks/**/*.py` 強制 chmod +x（convention-based）
- Push 端 safety net：push 前對 temp_dir/hooks/**/*.py 強制 chmod +x，確保 git add 記錄 100755
- 範圍限制：僅 hooks/ 目錄下 .py；scripts/ 有 644/755 混合（sync 腳本本身 644），不盲目處理
