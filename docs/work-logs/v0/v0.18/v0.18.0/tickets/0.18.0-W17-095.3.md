### 問題根因

`scan_handoff_pending_directory()`（行 88-117）只用 `resumed_at != null` 過濾，**完全不檢查 stale**。導致 SessionStart additionalContext 顯示 6 個待恢復，但 CLI `resume --list` 只顯示 0 個（過濾後 3 stale 被排除），三方資料源不一致。

### 影響範圍

- `.claude/hooks/handoff-reminder-hook.py`：`scan_handoff_pending_directory()` + reminder 訊息產生
- `.claude/hooks/tests/test_handoff_reminder_hook.py`：補過濾與訊息測試

### 相關 Error Pattern

W17-095 ANA 根因表行 122-126；無直接 PC error-pattern。

### 既有實作（行號）

```python
# .claude/hooks/handoff-reminder-hook.py:88-117 摘錄
for file_path in sorted(handoff_dir.glob("*.json")):
    try:
        with open(file_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        resumed_at = data.get("resumed_at")
        if resumed_at is not None:
            continue  # 跳過已接手
        ticket_id = data.get("ticket_id", file_path.stem)
        # ... 直接 append 進 pending_tasks
        pending_tasks.append({
            "ticket_id": ticket_id,
            "title": data.get("title", "無標題"),
            "direction": data.get("direction", "unknown"),
            ...
        })
```

**改動點**：在 `if resumed_at is not None: continue` 之後、`pending_tasks.append` 之前，加 `is_handoff_stale(data)` 過濾並計數。

### 改造方向

```python
stale_count = 0
for file_path in sorted(handoff_dir.glob("*.json")):
    # ... 既有 resumed_at 檢查
    is_stale, reason = is_handoff_stale(data)
    if is_stale:
        stale_count += 1
        logger.debug(f"跳過 stale handoff: {file_path.stem} ({reason})")
        continue
    pending_tasks.append({...})

if stale_count > 0:
    logger.info(f"已過濾 {stale_count} 個 stale handoff")
```

### Reminder 訊息調整

reminder 輸出（用戶可見的 SessionStart additionalContext）需顯示「已過濾 N 個 stale」與 CLI 一致。具體訊息位置由 thyme 自行定位（搜尋 `additionalContext` 或 reminder 主訊息產生點）。
