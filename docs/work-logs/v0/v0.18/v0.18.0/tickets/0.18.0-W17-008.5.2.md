### 任務本質

W17-008.5（統一 CLI 錯誤訊息雙軌）拆分後第 1 棒。本 ticket 建立統一 helper 基礎設施，讓 .5.3-.5.5 可改寫呼叫。

### 現況（PM 前台盤點，引自父 ticket .5 Problem Analysis）

- `lib/messages.py` 已有 `format_error(template, **kwargs)` 文字插值版本
- `class ErrorMessages` 已有部分 template constants
- 10 個 Messages class 散落（TrackMessages / LifecycleMessages / ...）

### 設計目標

`format_error` 從文字插值升級為**結構化格式**：
- 輸入：component（CLI 子命令） / action（操作動詞） / errno（錯誤分類） / hint（修復建議）
- 輸出：統一 markdown 結構，便於 hook 偵測標記跳過重複補充
- 保留向後相容：既有 `format_error(template, **kwargs)` 路徑不變

### 設計決策

- 採 dataclass `ErrorEnvelope`（dataclasses 內建，無外部依賴）
- 加入版本標記（如 `__error_envelope_v1__` 前綴）讓 hook 可偵測
- 雙路徑：legacy str template / structured envelope
- isinstance 分發

### 實作範圍

- `.claude/skills/ticket/ticket_system/lib/messages.py` 新增 ErrorEnvelope + 升級 format_error 雙路徑
- `.claude/skills/ticket/references/track-command.md` 補新格式用法

### WRAP 三問

- W：A1 dataclass + envelope / A2 dict + schema / A3 直接寫死 markdown template；選 A1（型別清楚 + 向後相容 + 標記 hookable）
- A：機會成本低，1h；不做則 .5.3-.5.5 無共用 helper 基礎
- P：最可能失敗為向後相容測試遺漏；防護=unit test 涵蓋 5+ 既有 ErrorMessages 呼叫路徑

### 修復摘要

實作雙路徑 `format_error()`：保留 legacy str template 路徑（向後相容），新增 `ErrorEnvelope` dataclass 結構化路徑。

### 變更檔案

- `.claude/skills/ticket/ticket_system/lib/messages.py`
  - 新增 `ERROR_ENVELOPE_VERSION_MARKER = "__error_envelope_v1__"` 模組常數（hook 偵測錨點）
  - 新增 `@dataclass(frozen=True) ErrorEnvelope`（component / action / errno / hint，hint 可選）
  - 新增 `_render_envelope()` 內部 helper 將 envelope 渲染為含版本標記字串
  - 升級 `format_error(template, **kwargs)` 簽名為 `Union[str, ErrorEnvelope]`，isinstance 分發；非預期型別拋 TypeError
- `.claude/skills/ticket/ticket_system/tests/test_messages_format_error.py`（新增）
  - 9 個測試：4 個 legacy 路徑（單參、無參、多參、缺參 KeyError）+ 3 個 envelope 路徑（完整欄位、無 hint、frozen 防護）+ 2 個分發/錯誤（TypeError、envelope 路徑忽略 kwargs）
- `.claude/skills/ticket/references/track-command.md`
  - 文件末尾新增「統一錯誤訊息格式（W17-008.5.2）」章節，含雙路徑用法、版本標記說明、grep 範例

### 設計決策

- **dataclass(frozen=True)**：避免封包被意外突變；`hint` 為 `Optional[str] = None` 預設，無 hint 時不渲染該行
- **isinstance 分發**：明確區分兩條路徑，envelope 路徑忽略 kwargs（不拋錯，純粹忽略以降低呼叫端心智負擔）
- **版本標記置於 [Error] 同行**：方便 grep 與 hook regex 同時偵測 `[Error]` 與 `__error_envelope_v1__`

### 不在本 ticket 範圍

- commands/*.py 呼叫端改寫（W17-008.5.3 / .5.4 處理）
- 10 個 Messages class 重構（獨立 ticket）
- Hook 補充邏輯實作（W17-008.5.5 處理）
