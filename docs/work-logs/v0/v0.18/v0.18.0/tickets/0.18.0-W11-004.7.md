### 3. API 介面定義：`_resolve_path_classification`

#### 函式簽名

```python
def _resolve_path_classification(
    prompt: str,
    ticket_ids: list[str],
    *,
    logger: logging.Logger | None = None,
) -> tuple[bool, bool, bool]:
    """統一決議 prompt 的路徑分類結果。

    依序套用三層規則並回傳合併後的 (has_main_repo_claude, has_external_claude, has_other)：

    L1. 從 prompt 抽取分類（沿用 _classify_prompt_paths）
    L2. W17-018 fallback：若 L1 全 False 且有 ticket_ids → 從 where.files 補分類
    L3. W11-004.7 覆蓋：若 L1 has_other=True 且 ticket scope 純 .claude/
        → 將 has_other 降為 False、has_main_repo_claude 升為 True
    """
```

#### 輸入規格

| 參數 | 類型 | 必填 | 說明 |
|------|------|------|------|
| prompt | str | 是 | 派發 prompt 全文（非空字串；空字串視為無分類訊號） |
| ticket_ids | list[str] | 是 | 已用 `_extract_ticket_ids(prompt)` 抽出的 ticket ID 清單（可為空 list） |
| logger | logging.Logger 或 None | 否 | 用於記錄 L2/L3 規則觸發；None 時不輸出日誌 |

#### 輸出規格

| 欄位 | 類型 | 說明 |
|------|------|------|
| has_main_repo_claude | bool | prompt 或 ticket scope 涉及主 repo `.claude/` |
| has_external_claude | bool | prompt 或 ticket scope 涉及外部 worktree 內 `.claude/`（runtime 必拒） |
| has_other | bool | prompt 或 ticket scope 涉及非 `.claude/` 實作路徑 |

#### 邊界條件

| 條件 | 行為 |
|------|------|
| prompt 為空字串 | L1 回 (False, False, False)；L2 若有 ticket_ids 則套用；L3 不觸發 |
| ticket_ids 為空 list | L1 結果直接回傳；L2/L3 不觸發 |
| ticket_id 無法載入 where.files | 該 ticket 跳過，繼續處理下一個；全失敗時等同 ticket_ids 為空 |
| 多個 ticket_id：部分純 `.claude/`、部分混合 | OR 合併：任一 ticket 含 `o=True` → has_other=True → L3 不觸發（保守） |
| has_external_claude=True | L3 不觸發（外部 .claude/ 在主流程 (1) 立即阻擋） |

#### 副作用

- 唯讀檔案系統存取：透過 `_load_ticket_where_files()` 讀取 ticket md
- 唯讀全域路徑：透過 `get_project_root()` 取得專案根
- logger.info 日誌（L2/L3 觸發時）；無 stderr / 阻擋輸出

### 4. L1/L2/L3 整合演算法

```
def _resolve_path_classification(prompt, ticket_ids, *, logger=None):
    # L1: prompt 直接分類
    m, e, o = _classify_prompt_paths(prompt)

    # L2: W17-018 fallback（prompt 全空 → 讀 ticket scope）
    if not m and not e and not o and ticket_ids:
        m, e, o = _merge_ticket_scope(ticket_ids)
        if logger and (m or o):
            logger.info("W17-018 fallback 觸發：ticket scope 補分類 m=%s o=%s", m, o)

    # L3: W11-004.7 覆蓋（has_other 為 True 但 ticket scope 純 .claude/）
    elif o and ticket_ids:  # elif 確保 L2/L3 互斥
        ticket_m, ticket_e, ticket_o = _merge_ticket_scope(ticket_ids)
        if ticket_m and not ticket_e and not ticket_o:
            if logger:
                logger.info(
                    "W11-004.7 覆蓋：ticket scope 純 .claude/，"
                    "prompt 內 src/tests/lib/ token 視為 .claude/ 巢狀路徑"
                )
            o = False
            m = True

    return m, e, o


def _merge_ticket_scope(ticket_ids):
    """合併多個 ticket 的 where.files 分類為 (m, e, o) tuple（OR 合併）。"""
    try:
        project_root_str = str(get_project_root().resolve())
    except Exception:
        project_root_str = ""

    m = e = o = False
    for tid in ticket_ids:
        files = _load_ticket_where_files(tid)
        if not files:
            continue
        tm, te, to = _classify_paths(files, project_root_str)
        m, e, o = m or tm, e or te, o or to
    return m, e, o
```
