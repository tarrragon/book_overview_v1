## Problem Analysis

### 問題根因

`agent-dispatch-validation-hook.py` 的 `_classify_prompt_paths()` 將 prompt 文字內 `tests/` / `src/` 等 token 一律歸為 `has_other=True`，**無上下文意識**：

- 即使 prompt 主要修改目標在 `.claude/skills/X/`，只要任何位置（包含 prose 引用、shell 命令、pytest 參數）出現 `tests/` token，分類即觸發 `has_other`
- 走 hook 主流程 (4) 「其餘情況 → 阻擋強制 worktree」，與 ARCH-015（CC runtime 拒絕外部 worktree 內 `.claude/` Edit）形成**雙向阻塞**

**實證案例**（W11-003.5 派發 2026-04-29）：

prompt 含 `(cd .claude/skills/ticket && uv run pytest tests/test_create_checklist_validation.py)`，hook 偵測到 `tests/` 並判 `has_other=True`，阻擋首次派發。第二次派發強制加 `isolation: worktree` 才通過。

### 影響範圍

- 主檔案：`.claude/hooks/agent-dispatch-validation-hook.py`
  - `_classify_prompt_paths()` line 236-280
  - `_NON_CLAUDE_PATH_PATTERN` line 61-63（正則表達式）
  - 主流程 line 1100-1166（分類後路由邏輯）
  - W17-018 既有 fallback line 1106-1127（prompt 路徑不明時讀 ticket where.files）
- 測試檔案：`.claude/hooks/tests/test_agent_dispatch_validation_hook.py`（待補測試）

### 相關 Error Pattern

無新增 error pattern。本 ticket 是 ARCH-015（CC runtime hardcoded 拒絕）與 hook 強制 worktree 雙向阻塞的修補。延伸 W17-018（prompt 路徑不明時 fallback 讀 ticket where.files）的設計哲學。

## Context Bundle

### 當前狀態（PM 驗證 2026-04-29）

**Hook 已有部分豁免邏輯**（line 1143-1150）：

```python
# (2) 僅主 repo .claude/ 且無其他路徑 → 放行（ARCH-015 豁免 worktree）
if has_main_repo_claude and not has_other:
    return 0
```

**但 has_other 過於敏感**：`_NON_CLAUDE_PATH_PATTERN = re.compile(r"(?<![./\w])(?:src|tests?|lib|app|assets|scripts|public|bin|cmd)/")` 不識別 shell 上下文（`(cd .claude/X && pytest tests/Y)` 中 `tests/` 實為 `.claude/X/tests/`）。

**W17-018 既有 fallback**（line 1106-1127）：prompt 路徑分類**全 False** 時才讀 ticket where.files 補分類。本 ticket 的場景中分類**部分為 True**（has_main_repo_claude=True），W17-018 fallback 不觸發。

### WRAP 結論：採用方案 F

**Trust ticket where.files 為 source of truth**：當 ticket where.files 純 `.claude/` 時，prompt 內 `tests/`/`src/`/`lib/` 等 token 視為 `.claude/` 內路徑，豁免 worktree 強制。

**設計哲學**：對齊 PC-040「context 在 ticket 不在 prompt」，prompt 內路徑提及僅作補充訊號，不應覆蓋 ticket 已聲明的 scope。

### 待 Phase 1 lavender 設計的開放問題

| 問題 | 候選答案 |
|------|---------|
| 何時觸發「ticket where.files 純 .claude/」優先邏輯？ | (a) 永遠先讀 ticket（取代 prompt 分類為主） (b) prompt 含 `.claude/` 時讀 ticket 補強 (c) 維持 W17-018 「prompt 全空才讀」並擴增「prompt 含 .claude/ 時也讀」 |
| ticket where.files 純 `.claude/` 的判定？ | 全部檔案開頭為 `.claude/` 或 `^/.../.claude/` → 純 `.claude/` |
| 與 W17-018 fallback 的整合？ | 合併為單一 `_resolve_path_classification(prompt, ticket_ids)` helper |
| has_other 仍為 True 的 .claude/ ticket 怎處理？ | 例：ticket where.files 含 `src/` 也含 `.claude/X/` → 維持原強制 worktree |
| 反向風險：純非 .claude/ ticket 的 prompt 引用 .claude/ 規則文件 | 應該強制 worktree（不能因 prompt 提一個 `.claude/rules/X.md` 而豁免） |
