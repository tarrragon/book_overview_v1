# Execution Log

## Task Summary

將 ac_parser.py 與 acceptance_auditor.py 的 checkbox 解析邏輯抽取為共用函式（位於 parser.py 或新增 checkbox_utils.py）

---

## Problem Analysis

### 問題根因

`ac_parser.py` 與 `acceptance_auditor.py` 皆實作 checkbox 前綴剝除邏輯，但策略不同：
- `ac_parser._parse_single_item`：嚴格匹配 `[x]/[X]/[ ]` + 切片 3 字元
- `acceptance_auditor.detect_vague_acceptance`：寬鬆 `split("]", 1)`，任何 `[...]` 前綴都會被剝除

雙寫導致行為微妙不一致，且一處修改需雙邊同步。

### 影響範圍

- `.claude/skills/ticket/ticket_system/lib/ac_parser.py`（移除本地常數與前綴剝除邏輯）
- `.claude/skills/ticket/ticket_system/lib/acceptance_auditor.py`（`detect_vague_acceptance` 改呼叫共用函式）
- 新增 `.claude/skills/ticket/ticket_system/lib/checkbox_utils.py`

### 相關 Error Pattern

無（純重構，DRY 落實）。

---

## Solution

新增 `lib/checkbox_utils.py` 作為 checkbox 前綴處理的單一權威實作：

- **公開 API**：`strip_checkbox_prefix(text: str) -> tuple[bool, str]`
  - 支援 `[x]` / `[X]` / `[ ]` 三種前綴
  - 回傳 `(checked, stripped_text)`；無前綴時回傳 `(False, text.strip())`
- **常數**：`CHECKBOX_CHECKED_LOWER`、`CHECKBOX_CHECKED_UPPER`、`CHECKBOX_UNCHECKED`、`CHECKBOX_PREFIX_LEN`（公開供呼叫方引用）

兩處呼叫點均改為使用共用函式：

1. `ac_parser._parse_single_item`：原本的 if/elif/else 三分支 → 單行 `checked, text = checkbox_utils.strip_checkbox_prefix(raw_str)`
2. `acceptance_auditor.detect_vague_acceptance`：原本的 `split("]", 1)` 寬鬆邏輯 → `_, cleaned = strip_checkbox_prefix(condition)`（收斂為嚴格 checkbox 匹配，符合註解「移除 [ ] 或 [x] 前綴」的原意）

### 設計選擇

- **位置**：新建 `checkbox_utils.py` 而非併入 `parser.py`。`parser.py` 專責 YAML frontmatter 解析，checkbox 前綴屬獨立 domain，分離可降低 `parser.py` 的認知負擔。
- **命名**：公開 API 去除底線前綴（`_strip_checkbox_prefix` → `strip_checkbox_prefix`），因需跨模組呼叫。
- **語義收斂**：`acceptance_auditor` 先前會剝除任何 `[...]` 前綴（如 `[重要] xxx`），現在僅剝除 checkbox。此變更符合原始註解意圖，且 `VAGUE_ACCEPTANCE_WORDS` 檢測對少量多出的 `[xxx]` 字元不敏感（模糊詞匹配仍有效）。
