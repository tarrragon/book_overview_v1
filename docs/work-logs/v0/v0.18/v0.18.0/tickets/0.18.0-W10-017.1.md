**參數規格（修正 6：argparse flag 取代位置參數）**：

ticket track checkpoint-status [--ticket-id TICKET_ID]

#### 1.2 「時間」欄位語意（修正 2）

三命令輸出的「時間: ...」一律取自 `state.computed_at`（lib 已產出 UTC ISO8601 字串），透過 lib/checkpoint_view.py 新增的 `format_local_time(state) -> str` helper 轉本地時區字串。

**消除 git status 雙重 I/O**：snapshot 既有實作另呼叫 `git status` 列印未提交檔案計數；v2 改為從 `state.uncommitted_files` 取值，移除既有獨立呼叫。後果：snapshot "Git 狀態" 區塊與 "Ready Check" 來自同一資料源，數字必然一致；Phase 3b 須驗證既有 snapshot 測試（若有斷言 git status 區塊內容）仍通過。

#### 1.3 退出碼規範（修正 1：對齊 thyme T1）

| 命令 | exit 0 | exit 1 | exit 2 |
|------|--------|--------|--------|
| snapshot | 永遠 | 內部錯誤（IO_ERRORS 以外例外） | — |
| handoff-ready | GO（state ready_for_clear=True 且通過所有阻擋項判定） | 內部錯誤（IO_ERRORS 以外例外） | NO-GO（業務拒絕：有阻擋項） |
| checkpoint-status | 永遠（純報告型） | 內部錯誤（IO_ERRORS 以外例外） | — |

### 2. /clear Ready Check checklist 完整定義

| Checklist 項目 | 判定條件 | 訊號來源（CheckpointState 欄位） | 修復建議文案 |
|---------------|---------|--------------------------------|-------------|
| 無活躍代理人 | active_agents == 0 | state.active_agents | ticket track agent-status 查看; 等待代理人完成或合併產出 |
| 無未提交變更 | uncommitted_files == 0（且非 None） | state.uncommitted_files | git add + git commit ({n} 檔) |
| 無未合併 worktree | len(unmerged_worktrees) == 0 | state.unmerged_worktrees | 合併 {n} 個 worktree 並清理 (paths: {list}) |
| 無 pending handoff | snapshot: active_handoff is None；handoff-ready: active_handoff is not None | state.active_handoff | snapshot 情境: 先處理 pending handoff: {ticket_id}；handoff-ready 情境: 視為通過項 |

**特殊處理**：

- uncommitted_files is None（資料源失敗）時，顯示為「[?] 未提交變更（資料源不可用）」並列入「需人工確認」清單，**不阻擋** Go 結論（fail-open）但提示警告。
- active_handoff 在 snapshot 與 handoff-ready 中語意相反：
  - snapshot：偵測「是否還有未處理的 handoff」→ 有則阻擋
  - handoff-ready：偵測「handoff 是否已建立」→ 有則為 Go 訊號之一
- 推導邏輯**全部來自 lib/checkpoint_state.py 既有欄位**，CLI 層不新增判定。
