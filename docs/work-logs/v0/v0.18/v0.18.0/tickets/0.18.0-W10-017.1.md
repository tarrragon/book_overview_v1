```
def execute_handoff_ready(args):
    try:
        state = checkpoint_state(
            ticket_id=args.ticket_id, caller="handoff-ready", log_metrics=True
        )
    except IO_ERRORS as e:
        # handoff-ready 無 fail-open 路徑（業務語意必須明確）；
        # 但 IO_ERRORS 仍視為「無法判定」，保守回 exit 2 + stderr
        sys.stderr.write(f"data source unavailable: {e}\n")
        return 2  # 保守視為 NO-GO（無法確認 GO）
    except Exception as e:
        sys.stderr.write(f"internal error: {e}\n")
        return 1

    # 使用 handoff_status_for + state 欄位逐項判定
    blockers = _compute_blockers(state, ticket_id=args.ticket_id)
    if blockers:
        print("結論: NO-GO ...")
        for b in blockers: print(f"  [ ] {b.label} → {b.fix}")
        return 2
    print("結論: GO  ready for /clear")
    return 0


def execute_checkpoint_status(args):

        state = checkpoint_state(
```

```
    ├─ format_phase_label(state)             → checkpoint label
    ├─ format_next_action(state)             → 「下一步:」行
    ├─ handoff_status_for(state, caller)     → handoff 行 (ok, msg)
    ├─ get_suggested_commands(state)         → PRIORITIES[*][4](state)
    ├─ render_current_suggestion(state)      → 「--- 當前建議 ---」區塊
    └─ render_ready_check(state, caller)     → 「--- /clear Ready Check ---」區塊
    │
    ▼
print(...) → exit code (snapshot=0, handoff-ready=0/1/2, checkpoint-status=0/1)
```

### 1. 三命令 CLI 介面定義（v2）

#### 1.1 命令架構與參數

| 命令 | caller 值 | ticket-id 參數 |
|------|----------|---------------|
| ticket track snapshot | "snapshot" | 不接受（全域視角） |
| ticket track handoff-ready | "handoff-ready" | 可選 --ticket-id TICKET_ID flag |
| ticket track checkpoint-status | "checkpoint-status" | 可選 --ticket-id TICKET_ID flag |

**參數規格（修正 6：argparse flag 取代位置參數）**：

```
ticket track handoff-ready [--ticket-id TICKET_ID]
ticket track checkpoint-status [--ticket-id TICKET_ID]
```

理由：位置參數 `[TICKET_ID]` 與既有 ticket CLI 慣例不一致（既有命令如 append-log/claim 等的 positional ticket_id 語意必填；此處可選且影響 lib 內部判定，flag 更明確）。Phase 3b 實作時 argparse 用 `--ticket-id` 並支援短形式 `-t`（保留實作彈性）。

#### 1.2 「時間」欄位語意（修正 2）

三命令輸出的「時間: ...」一律取自 `state.computed_at`（lib 已產出 UTC ISO8601 字串），透過 lib/checkpoint_view.py 新增的 `format_local_time(state) -> str` helper 轉本地時區字串。

**消除 git status 雙重 I/O**：snapshot 既有實作另呼叫 `git status` 列印未提交檔案計數；v2 改為從 `state.uncommitted_files` 取值，移除既有獨立呼叫。後果：snapshot "Git 狀態" 區塊與 "Ready Check" 來自同一資料源，數字必然一致；Phase 3b 須驗證既有 snapshot 測試（若有斷言 git status 區塊內容）仍通過。

#### 1.3 退出碼規範（修正 1：對齊 thyme T1）

| 命令 | exit 0 | exit 1 | exit 2 |
|------|--------|--------|--------|
| snapshot | 永遠 | 內部錯誤（IO_ERRORS 以外例外） | — |
| handoff-ready | GO（state ready_for_clear=True 且通過所有阻擋項判定） | 內部錯誤（IO_ERRORS 以外例外） | NO-GO（業務拒絕：有阻擋項） |
| checkpoint-status | 永遠（純報告型） | 內部錯誤（IO_ERRORS 以外例外） | — |

**設計原則**：exit 1 專屬「程式本身錯誤」（unexpected exception），exit 2 專屬「業務語意拒絕」（NO-GO）。Shell pipeline 可用 `if ! ticket track handoff-ready --ticket-id X; then` 區分兩者：`$?==1` 須 alert，`$?==2` 屬正常 NO-GO 流程。
