## 重現實驗結果

### 實驗方法

驗證兩件事：(1) 本專案 runtime 是否存在 `mcp__dart__run_tests` 工具；(2) Chrome Extension 專案中此 hook 的實際執行行為（是否有 W10-043.2 audit 中提到的「每次 PreToolUse 開銷」）。

### 實驗執行

```bash
# 1. .mcp.json 配置檢查
ls .mcp.json → 不存在（本專案無 MCP server 配置）

# 2. settings.json 中 dart 相關 hook 註冊
grep -n "mcp__dart" .claude/settings.json
→ line 70: matcher: mcp__dart__run_tests + mcp-run-tests-validator.py
→ line 391: matcher: mcp__dart__run_tests + 另一 hook
→ line 87: matcher: mcp__serena__* (不相關)

# 3. agent 定義中 mcp__dart 引用範圍
grep -rn "mcp__dart" .claude/agents/
→ cinnamon, sage, pepper, mint, bay, linux 等 agent 列出 mcp__dart__* 工具
→ 框架共享設計（為 Flutter/Dart 專案準備）

# 4. PreToolUse 觸發機制
hook 註冊 matcher: "mcp__dart__run_tests"，僅在 LLM 呼叫該工具時觸發
本專案 runtime 不存在此工具 → 不會被呼叫 → hook 從不執行
```

### 實驗發現

**已驗證的事實**：

1. `.mcp.json` 不存在於本專案 → 無 MCP server 註冊 → `mcp__dart__*` 工具不可用
2. settings.json line 70/391 註冊了 `mcp-run-tests-validator.py` 在 matcher `mcp__dart__run_tests` 下
3. PreToolUse hook 採 matcher 模式，**只在工具實際被呼叫時觸發**，並非每次任何 PreToolUse 都跑
4. 工具不存在 → LLM 不會呼叫 → matcher 不命中 → hook 永不執行 → **零執行開銷**
5. `.claude/agents/` 中多個 agent 仍引用 `mcp__dart__*`（屬框架共享資產）

**仍未驗證的假設**：

- 無

**重要修正（W10-043.2 audit 推論偏差）**：

W10-043.2 Batch B audit 報告 Hook 3 標註「死碼 hook 增加每次 PreToolUse 的 hook 執行開銷（啟動 Python + 讀 stdin + 早期 return）」。此推論**錯誤**——PreToolUse 採 per-matcher 機制，不會因 hook 註冊就每次執行。實際情況：hook 處於 **dormant（休眠）** 狀態而非 dead code，本專案 runtime 中從未觸發。

### 候選方案重新校準

原 W10-043.5 設定的三選項（移除註冊 / 刪除檔案 / 加 exclude list）皆建立在「死碼造成開銷」前提上。Reality Test 推翻前提，需新增第四選項：

- **(d) 保留現狀**（新增）：hook 已自然休眠，無成本，無需行動
- (a) settings.json 移除註冊：可選但無實際收益
- (b) 刪除檔案：傷害跨專案重用（Flutter 專案需要）
- (c) hook-exclude-list：機制適合「需排除但不能刪」場景，本案不適用

### 結論

**採方案 (d) 保留現狀**，無需移除/刪除/排除任何配置。

### 理由

1. **無實際成本**：hook 採 per-matcher 觸發機制，工具不存在 → 永不觸發 → 零執行開銷
2. **跨專案資產**：`.claude/hooks/mcp-run-tests-validator.py` 為框架共享 hook，刪除會傷害 Flutter/Dart 專案的測試守護機制
3. **agent 定義一致性**：多個共享 agent（cinnamon/sage/pepper/mint/bay/linux）的 tools 欄位仍列 `mcp__dart__*`，移除 hook 但保留 agent 引用會造成不對稱
4. **YAGNI 原則**：尚無實際痛點需修復；W10-043.2 audit 推論的「每次 PreToolUse 開銷」已被 Reality Test 證偽
