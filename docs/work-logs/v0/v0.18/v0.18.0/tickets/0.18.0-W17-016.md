## Problem Analysis

### 問題根因

W17-015.1 暴露 completed ticket body 仍殘留空白或 placeholder 的系統性缺口。既有流程只要求 frontmatter 狀態與 acceptance 勾選，沒有依 ticket type 定義哪些 body section 必填，也沒有在 complete 前檢查「待填寫」佔位符。

這造成 agent 完成實作後可能只回報 commit 或測試結果，後續接手者缺少 Problem Analysis、Solution、Test Results 等可追溯內容。W17-001 已處理 PM -> agent PCB 填料與 agent -> PM exit status，但 agent -> 後人 body 填料仍無獨立機制。

### 影響範圍

- Ticket 規則與模板：`.claude/pm-rules/ticket-body-schema.md`、ticket body template、`SKILL.md`
- Complete lifecycle：`ticket_validator.py`、`lifecycle.py`、`track.py`
- Agent 責任文件：`.claude/rules/core/agent-definition-standard.md` 與實作類 agent 定義
- W17 ticket body 可讀性與 completion metadata 同步

### 相關 Error Pattern

- PC-063：ANA 類 ticket 需有重現/實驗事實，避免空泛分析。
- PC-087：禁止用暫存檔迴避 append-log heredoc 等正式記錄流程。
- W17-016.3 新增 complete hard-block 後，未填必填 body section 或殘留中文佔位符會被阻擋。

## Solution

W17-016 group 以 5 個子任務完成 body 完整性機制閉環：

1. `W17-016.1` 完成樣本盤點，產出 ANA/IMP/DOC 的 required/optional section 初步對照。
2. `W17-016.2` 建立 `.claude/pm-rules/ticket-body-schema.md`，並讓 ticket template 依 type 插入 `Schema[TYPE/Section]` 註解；`SKILL.md` 同步新增 schema 使用說明。
3. `W17-016.3` 擴充 complete hook：依 ticket type 驗證必填章節、偵測中文 placeholder，未通過時 hard-block；同時提供 `--skip-body-check` 逃生閥。
4. `W17-016.4` 新增 `sync_completion_body_fields`，complete 時把 frontmatter `completed_at` 同步回 body `Completion Time`，並填入 executing agent placeholder。
5. `W17-016.5` 更新 `agent-definition-standard.md` 與 thyme/parsley/fennel 實作 agent，明示 ticket body 填寫是 agent 收尾責任。

此 group 的四項 acceptance 已由 children 與 repo 內容核對完成，W17-016 可收尾。
