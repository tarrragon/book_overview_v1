### 影響範圍

- W17-111.1 偵測分支（value 含冒號）對多行/同行皆有效，無需擴充邏輯

### 相關 Error Pattern

- PC-117（與本 ANA 同源，描述驗證為正確）

## 重現實驗結果

### 實驗方法

從 git 取出 449da97c（W17-095 觸發 invalid 警告當下的 commit）的 ticket md Solution 區段，直接用 python 跑 `_parse_field` 的 regex 驗證匹配行為：

```bash
git show 449da97c:docs/work-logs/v0/v0.18/v0.18.0/tickets/0.18.0-W17-095.md | sed -n '/^## Solution/,/^## Test Results/p'
python3 -c "import re; ..."  # 三種形式對照
```

### 實驗執行

三種形式並行測試 regex `^\s*multi_view_status\s*:\s*(.+?)\s*$` with `re.MULTILINE | re.IGNORECASE`：

```python
# 形式 1：多行 nested + yaml code fence（W17-095 實際情境）
section1 = '''
```yaml
multi_view_status:
  status: skipped
  reason: "..."
```
'''

# 形式 2：純多行 nested（無 code fence）
section2 = '''
multi_view_status:
  status: skipped
'''

# 形式 3：同行形式（thyme 測試 fixture）
section3 = 'multi_view_status: status: skipped'
```

執行結果：

| 形式 | python re.search 結果 | 取得 value |
|------|----------------------|-----------|
| 1. 多行 nested + code fence | match | `'status: skipped'` |
| 2. 純多行 nested | match | `'status: skipped'` |
| 3. 同行形式 | match | `'status: skipped'` |

### 實驗發現

**已驗證的事實**：

| 事實 | 證據 |
|------|------|
| 多行 nested **確實** 觸發 invalid（regex 跨行抓 status: skipped）| python 重現三形式皆 match |
| `\s*` 跨行吞 newline + 縮排是 Python re 標準行為 | `\s` 含 `\n`，`re.MULTILINE` 不影響 `\s` |
| W17-095 觸發 invalid 是 nested 結構直接導致（無第三變因）| 449da97c content 與 python 重現一致 |
| thyme caveat「多行 nested → missing」聲稱錯誤 | 矛盾於 python 重現結果 |

**真因**：thyme 的 testing gap——`test_invalid_value_with_colon_includes_nested_hint` 用同行形式做 fixture（避開可能的 fixture 設計問題），但**未驗證**多行 nested 也走 invalid 分支。caveat 的「missing 非 invalid」推論基於某種 fixture 邊界（可能是測試裡 Solution 區段擷取邊界、或某段 mock 影響），非 hook 主流程行為。

**可能的 thyme fixture 邊界假設**：thyme 可能用了類似這樣的 fixture：

```python
content = '''---
type: ANA
---
## Solution

multi_view_status:
  status: skipped
'''  # 末尾無 ## 或 \Z 邊界，可能影響 _extract_solution_section 的擷取
```

但這仍不應導致 missing——`_extract_solution_section` 會匹配到 `\Z`（字串尾），section 內含 nested 內容，`_parse_field` 會匹配。

需 W17-112.1 子任務補多行 nested 測試實際驗證 thyme 假設邊界。

### 結論

**PC-117 / W17-111 forbidden_format 描述本就正確**，不需修正。
**W17-111.1 邏輯正確**（value 含冒號偵測對多行/同行皆有效），僅 testing 需補 case。
