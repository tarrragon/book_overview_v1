### 問題根因

`_ABSOLUTE_CLAUDE_PATTERN = re.compile(r"(/[^\s]+?)/\.claude/")` 用 `finditer()` 對含巢狀 `.claude/` 的絕對路徑會產生多重匹配。第二次以後的 match 只截取後半段（如 `/foo/.claude/`），丟失絕對路徑前綴，導致與 project_root_str 比對時誤判為 external。

### 重現（PM 已驗證）

prompt = `修改 {project_root}/.claude/skills/foo/.claude/bar.py`

- 預期：`(main=True, external=False, other=False)`
- 實際：`(main=True, external=True, other=False)` ← bug

finditer 機制：
- match 1: `{project_root}/.claude/` → starts with root → main=True
- match 2: `/foo/.claude/`（從 match 1 結束位置往後找到的下一個絕對路徑樣式）→ 不 starts with root → external=True

### 影響範圍

- `.claude/hooks/agent-dispatch-validation-hook.py:245` `_classify_prompt_paths` 對 prompt 內路徑分類
- `.claude/hooks/agent-dispatch-validation-hook.py:203` `_classify_paths` 對 ticket where.files 分類（同樣 bug）
- 觸發條件：prompt 或 where.files 含 absolute path 且 path 中含 `.claude/` 子段（如 worktree absolute path 指向主 repo `.claude/`，或專案根目錄路徑本身含 `.claude/`）

### 相關 Error Pattern

無直接對應，本身屬 regex 設計缺陷。

### 修法方向

加 lookbehind 限定 match 起點為「絕對路徑開頭」（前面為空白 / 引號 / 行首等非路徑字元）：

```python
_ABSOLUTE_CLAUDE_PATTERN = re.compile(r"(?<![\w./-])(/[^\s]+?)/\.claude/")
```

驗證：
- Case A `/Users/foo/.claude/projects/p/.claude/hooks/bar.py`：match 1 取 `/Users/foo/.claude/`（前綴空白）；match 2 候選 `/.claude/` 前面是字母 `p`（`\w`）→ lookbehind 阻擋 → 只 1 個 match
- Case C `{root}/.claude/skills/foo/.claude/bar.py`：同理只 1 個 match，正確分類為 main

### 修法

在 `_ABSOLUTE_CLAUDE_PATTERN` 加 negative lookbehind `(?<![\w./-])`，限定 match 起點必須位於絕對路徑開頭（前面不能是 word char / `.` / `/` / `-`），避免 finditer 對含巢狀 `.claude/` 的路徑產生多重匹配。

修改前：
```python
_ABSOLUTE_CLAUDE_PATTERN = re.compile(r"(/[^\s]+?)/\.claude/")
```

修改後：
```python
_ABSOLUTE_CLAUDE_PATTERN = re.compile(r"(?<![\w./-])(/[^\s]+?)/\.claude/")
```

### 受影響的呼叫端

- `_classify_prompt_paths` (line 245)：對 prompt 內路徑分類
- `_classify_paths` (line 203)：對 ticket where.files 分類

兩者共用同一個 module-level pattern，單一 regex 修改即同時修正兩處。

### 邊界驗證

| 案例 | match 數 | 分類 |
|------|---------|------|
| `{root}/.claude/skills/foo/.claude/bar.py` | 1（lookbehind 阻擋第二次） | main=True, external=False |
| `/Users/foo/.claude/projects/p/.claude/hooks/bar.py` | 1 | main=False, external=True |
| 既有 13 個測試案例 | 不變 | 全部維持原預期 |
