### 問題根因

**發現 1（resume.py 行號漂移）**：`test_no_bare_status_completed_filter.py` allowlist 登記 `commands/resume.py:193`，但 W10-107 修改 resume.py 後 `!= "completed"` 漂移至第 195 行。兩個測試同時失敗：
- `test_no_bare_status_completed_negation_filter`：第 195 行被偵測為新增反模式（不在 allowlist）
- `test_allowlist_sites_still_exist`：第 193 行已不含目標 pattern（allowlist 過時）

根因：allowlist 以 `檔名:行號` 為 key，上游修改插入/刪除行後行號漂移，allowlist 未同步更新。

**發現 2（Validator N/A keyword 誤判）**：`_is_placeholder` 對非表格情境的 `N/A` 仍以 `\bN/A\b` regex 判定為 placeholder。W10-125 已修表格情境（剝除表格行後再做 keyword 檢查），但非表格的合法 N/A 使用（如 hook PostToolUse 建議標記語法）仍被誤判。

根因：`_is_placeholder` 的 keyword 檢查是字面匹配，無法區分「作者寫 N/A 表達不適用」（合法）vs「佔位符未填」（需攔截）。PC-138 / PC-144 的表格治本已完成，但非表格情境的 N/A 語意歧義仍存在。

### 影響範圍

- `.claude/skills/ticket/tests/test_no_bare_status_completed_filter.py`（2 failures）
- `.claude/skills/ticket/ticket_system/lib/ticket_validator.py`（`_is_placeholder` 函式）
- `.claude/skills/ticket/ticket_system/commands/resume.py`（行號對應）

### 相關 Error Pattern

- PC-138：Validator 表格 N/A 誤判（已修）
