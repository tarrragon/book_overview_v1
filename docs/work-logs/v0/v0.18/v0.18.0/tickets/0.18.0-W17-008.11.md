## 重現實驗結果

### 實驗方法

直接讀 `track.py:160-236` 與 5 個 `track_*.py` 模組的 execute 函式 signature，比對 register dict 的 handler signature 契約。

### 實驗執行

1. `_create_command_handlers()` (track.py:160-217) 回傳 `dict[str, Callable]`，所有 handler 隱式 signature 為 `(args, version: str) -> int`（line 271 `handlers[operation](args, version)`）
2. 5 個特殊 handler signature 皆為 `execute_*(args)`（不收 version）：snapshot / agent-status / handoff-ready / checkpoint-status / dispatch-check
3. 註冊路徑亦不對稱：dict-based 命令統一在 `_register_*_commands` 群組函式（track.py:747-756）；5 特殊命令各有獨立 `register_*` 函式（track.py:757-760）

### 實驗發現

**已驗證事實**：
- 雙軌根因為 handler signature 不一致（version-aware vs version-agnostic）
- 路徑不對稱導致新增類似命令必須記得改兩處（與 PC-090「重複實作跨流程未同步」同構風險）
- 5 個命令本質都是「全域狀態查詢」，無 version 概念合理

**已驗證假設**：
- 屬偶然複雜度（accidental complexity）：signature 強耦合 version 是設計選擇而非必要
- 統一可行：將 signature 改為 `(args, version: Optional[str])` 或拆兩 dict 即可消除特殊路徑

## Solution

### 結論：偶然複雜度，建議統一（方案 B 雙 dict）

**三方案比較**：

| 方案 | 實作 | 風險 | LOC | 維護收益 |
|------|------|------|-----|---------|
| A. 統一 signature | `(args, version: Optional[str])`，version-agnostic 收 None | 改動所有 handler signature；高 | ~50 | 高 |
| B. 雙 dict（推薦） | `version_aware_handlers` + `version_agnostic_handlers` 兩個 dict，execute() 先查 agnostic | 最小，不動既有 handler | ~25 | 高 |
| C. 文件化雙軌 | 加註解 + 新增命令 checklist | 零 | ~10 | 低（依賴自律） |

### 推薦方案 B 理由

1. **顯性化分類**：兩 dict 名稱即說明「version-aware vs agnostic」，新貢獻者一眼可辨
2. **不動既有 handler signature**：避開 A 的批量改動風險
3. **消除 if-elif 鏈**：dispatch 邏輯統一為 `agnostic.get(op) or aware.get(op)`
4. **註冊路徑可同步收斂**：在 `_register_all_subcommands` 中將 5 個 `register_*` 呼叫包到 `_register_global_state_commands` 群組函式

### 實作骨架（供 IMP 子任務）

```python
def _create_version_agnostic_handlers() -> dict:
    return {
        "snapshot": execute_snapshot,
        "agent-status": execute_agent_status,
        "handoff-ready": execute_handoff_ready,
        "checkpoint-status": execute_checkpoint_status,
        "dispatch-check": execute_dispatch_check,
    }

def execute(args):
    operation = args.operation
    if operation == "version":
        return execute_version(args, None)
    agnostic = _create_version_agnostic_handlers()
    if operation in agnostic:
        return agnostic[operation](args)
    # ... 既有 version-aware 流程不變
```
