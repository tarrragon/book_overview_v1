### 實作摘要

依 W6-012.2.2 Solution 5 案例合併規則表交付 `cover-XXX → reader-XXX` migration step：

**新增檔案**

  - 純函式 `transformBooks(books, logger)` 套用 5 案例合併規則
  - 主入口 `migrateCoverToReader(storage, logger)` 包裝 backup → transform → write → rollback 完整流程
  - 備份 key：`migration_backup_v3_1`（與 v1-to-v2 的 `migration_backup` 隔離，避免互相覆蓋）
  - 目標 schema_version：`'3.1.0'`

**修改檔案**

- `src/background/domains/data-management/services/migration-service.js`
  - 新增 `coverToReaderApplies` 函式（與 v1-to-v2 同採寬鬆策略，由 step 自身依 storage.schema_version 判斷）
  - `createDefaultSteps()` 追加第二個 step `{ id: 'cover-to-reader', applies, run }`
  - 使用既有 step registry 機制（與 `registerStep` API 等價路徑）

**5 案例處理對照**

| Case | 條件 | 處理結果 |
|------|------|---------|
| 1 正常遷移 | cover-XXX + identifiers.privacyBookId 存在 | id → `reader-{privacyBookId}`，保留 tagIds/readingStatus/coverUrl 等自訂欄位，新增 `legacy_cover_id` 追溯 |
| 2 privacyBookId 為空 | cover-XXX + privacyBookId='' 或 identifiers 缺失 | 保留舊 id，標記 `requires_manual_review=true` |
| 3 cover-openbook 共用 | id=`cover-openbook` 群組 | 以 `title__extractedAt` secondary key 去重，標記 `legacy_duplicate=true` + `requires_manual_review=true` |
| 4 同 reader-id 多筆 | 遷移後同 reader-{pbk} 兩條（手動匯入舊備份等） | 取 `updatedAt > extractedAt > addedAt` 較新者為主體，tagIds 採並集（保序、去重） |
| 5 跨裝置 sync 衝突 | sync push 來另一裝置 cover-XXX | 範圍外，本 module 不處理（屬 follow-up） |

**遷移日誌記錄**

`migrateCoverToReader` 完成時 info-log 受影響書籍數：

```
cover-to-reader 完成：migrated=N1 manualReview=N2 duplicates=N3 unchanged=N4
```

其中 `manualReview + duplicates` 即「待重新提取」總數。

### 設計決策

1. **不在 BookSchemaV2.SCHEMA_VERSION 常數中升至 3.1.0**：v3.0.0 屬於 schema 結構基線（Tag-based Book Model），3.1.0 僅是 id 命名規則演進，未變更欄位定義；保持 `BookSchemaV2.SCHEMA_VERSION='3.0.0'` 不變，cover-to-reader 內部以 `TARGET_SCHEMA_VERSION='3.1.0'` 私有常數寫入 storage。
2. **備份 key 隔離**：`migration_backup_v3_1` 與 v1-to-v2 的 `migration_backup` 分開，避免 v1-to-v2 失敗回滾意外覆蓋本 step 的 backup（或反之）。
3. **applies 寬鬆策略**：與 v1-to-v2 一致，由 step 自身依 storage 真實狀態（schema_version === '3.1.0' 即跳過）判斷，避免在 MigrationService 路由層重複實作冪等性檢查。
4. **空書庫快速路徑**：書庫為空時直接升 schema_version，跳過備份/還原（無資料可備）。
