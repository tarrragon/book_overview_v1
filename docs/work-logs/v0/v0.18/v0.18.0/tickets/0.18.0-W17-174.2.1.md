### 設計決策

1. **E6 適用範圍限定**：僅在「只有」表格路徑命中時生效（`hit_table_path and not (hit_option_path or hit_binary_path)`）。若同時命中 option/binary 路徑，純資料表豁免不適用，避免漏報合法選項提醒。
2. **E3 文件層約束**：規格層註明 E3 僅適用 §3.2，但程式碼層 E3 尚未實作，本次未引入新 E3 邏輯，僅在 spec 加文件約束以防未來實作時違規。
3. **header_count 計算**：`min(1, separator_count)` 確保僅當有分隔列時才扣 1 行作標題，純資料列無分隔的退化情況不會誤扣。
4. **TABLE_SEPARATOR_RE 含 `|`**：分隔列允許含內部 `|`（多欄分隔），避免被誤判為資料列。
