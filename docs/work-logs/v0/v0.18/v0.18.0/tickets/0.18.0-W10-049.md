#### 分析方法（每檔逐一執行）

1. **內容摘要**（Read 前 50 行）：標題、版本/日期/用途、是否有「已遷移」「Deprecated」註記、主題
2. **引用分析**（Grep 檔名）：分類引用來源
   - 現行框架：`.claude/rules/`、`pm-rules/`、`references/`、`methodologies/`、`skills/`、`agents/`、`CLAUDE.md`
   - 舊檔互相引用：根目錄其他 md 檔
   - 其他：docs/、error-patterns/
3. **比對現行分類**：主題是否已有更新版本？（例：`tdd-collaboration-flow.md` vs `pm-rules/tdd-flow.md`）
4. **決策分類**（每檔一個）：
   - **REMOVE**：內容完全過時且已有更新版本，現行未使用
   - **ARCHIVE_TO_X**：有歷史參考價值，移至 `.claude/analyses/archived/` 或 references/ 子目錄
   - **MIGRATE_TO_X**：內容有效但位置錯誤，搬至正確分類
   - **KEEP**：確實應留根目錄（須說明理由）
   - **SPLIT**：大檔需拆分分別歸檔/遷移
5. **影響引用清單**：若 REMOVE/ARCHIVE/MIGRATE，列出需同步更新的引用檔案
