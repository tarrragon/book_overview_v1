## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

W10-017 父 ANA complete 過程觸碰兩個非顯而易見的 hook 規範，造成 PM 額外摩擦：

### 問題 1：PC-093 ticket-tracked 豁免要求 W{wave}-{seq} 完整 ID 格式

**症狀**：
- 用戶寫 `<!-- PC-093-exempt: ticket-tracked:Phase 1b 已由 017.9 完成評估 ... -->` <!-- PC-093-exempt: ticket-tracked:本行引用 W10-017.9 為 W10-017 任務鏈衍生 ticket（已 completed），用以說明 hook 拒絕的具體案例 -->
- complete 被擋，但 hook 用戶端輸出未顯示原因；只在程式內部標 `valid=False, err='ticket-tracked-need-id'`
- 必須直接讀 hook 程式碼或 dry-run 才知道需求

**根因**：
- `phase4-decision-enforcement-hook.py:66` `TICKET_ID_PATTERN = re.compile(r"\bW\d+-\d+")` 強制 `W` 前綴
- 專案內部慣用簡寫 `017.9` / `017.1` 不符合
- 驗證失敗的 err 字串 `ticket-tracked-need-id` 未在 hook stderr 輸出 humanized 訊息

**用戶端影響**：
- 5 處豁免有 3 處被拒（235/270 用簡寫；743/758 含 W17-011/W10-014 全格式通過）
- PM 重試多次才推論出格式要求

### 問題 2：multi_view_status H2 vs H3 標題級別衝突

**症狀**：
- ana-solution-schema.yaml 提供 `multi_view_status: reviewed` 純 key-value 範例（line 34-46）
- 用戶寫 `## multi_view_status` 滿足 schema 偵測（field_key 前綴匹配）
- 但 W17-072 hook（agent-definition-standard.md「禁止自定義 H2」）標 `## multi_view_status` 為非 Schema H2 警告
- 必須改 `### multi_view_status` 才同時滿足兩端

**根因**：
- ana-solution-schema 未指定標題層級，留給用戶自由發揮
- W17-072 hook Schema H2 白名單（Task Summary / Problem Analysis / Solution / Test Results / Completion Info / Context Bundle / NeedsContext / Exit Status）未包含 multi_view_status
- 兩端規則設計時未對齊：一端把 multi_view_status 視為「Solution 子標註」，另一端視為獨立 H2 章節

**用戶端影響**：
- W10-017.17 標註後遭 W17-072 警告
- 必須再 commit 改 H3

### 改善方向（兩個獨立 IMP，已建立為 W17-085 / W17-086）

#### IMP A：PC-093 hook 用戶端訊息可見性

**症狀修復**：當 marker 因 `ticket-tracked-need-id` 失敗時，在 hook stderr 輸出 humanized 訊息，例如：
> `[PC-093] line 235 marker 失效：ticket-tracked 類別要求 reason 含 W{wave}-{seq} 格式 ID（如 W10-017.9），偵測到的 reason 為「短 ID 引用未含 W 前綴」` <!-- PC-093-exempt: ticket-tracked:本 ANA 引用 W10-017.9 為 W10-017 任務鏈衍生 ticket（已 completed），用以說明 hook 拒絕的具體案例 -->

**根因防護**（可選）：放寬 `TICKET_ID_PATTERN` 接受專案內慣用的短格式（如 `\b\d+\.\d+`），或在 PC-093 error-pattern 文件首段顯式列出格式要求 + 反例。

**目標檔案**：`.claude/hooks/phase4-decision-enforcement-hook.py:66, 273-274`

#### IMP B：multi_view_status 與 W17-072 規則對齊

**選項 B1（推薦）**：W17-072 hook 加 multi_view_status 到 Schema H2 白名單。理由：ana-solution-schema 已將其定為 ANA Solution 區段必填標註，等同 Schema 規範一部分。

**選項 B2**：ana-solution-schema 範例改為 H3 形式（`### multi_view_status`），明示是 Solution 子節，配合 W17-072 邏輯。

**選項 B3**：兩規則文件互相連結，但仍需用戶記住 H3 形式（不解決 UX）。

**目標檔案**：
- `.claude/config/ana-solution-schema.yaml`（範例與描述）
- `.claude/rules/core/agent-definition-standard.md`（H2 白名單）
- 對應 W17-072 hook 實作
