## 實作摘要（H3 子段）

### 檔案清單
- 新增：`.claude/hooks/worktree-zombie-cleanup-hook.py`（核心 hook）
- 新增：`.claude/hooks/tests/test_worktree_zombie_cleanup_hook.py`（27 unit tests）
- 修改：`.claude/settings.json`（SessionStart 註冊，timeout 15s）

### 核心邏輯
1. 列舉 `.claude/worktrees/agent-*`
2. 從 `.git/worktrees/<name>/locked` 解析 PID（regex `(pid \d+)`）
3. `ps -p <pid> -o comm=` 確認 PID 死活；進程名含 claude/cc 才視 alive（PID 重用防護）
4. dead + clean → `git worktree unlock` + `git worktree remove --force`
5. dead + dirty → 列 warning，不清
6. alive / recent (< 30 min) / 無 PID → 跳過

### 防護機制
- env var `WORKTREE_ZOMBIE_CLEANUP_DISABLE` 非空整體停用
- 排除建立時間 < 30 min 的 worktree（避免清掉啟動中代理人）
- subprocess 失敗時保守（is_worktree_dirty 視為 dirty / check_pid_alive 視為 alive）
- 任何例外皆 suppressOutput 不阻擋 SessionStart

### 可觀測性
- stderr 印摘要（cleaned / warn-dirty / failed 數量與名稱）
- 檔案日誌：`.claude/hook-logs/worktree-zombie-cleanup/*.log`

### 不在本 ticket 範圍
- 實際清理 24 個現存殭屍 worktree（屬手動操作，PM 後續執行）
