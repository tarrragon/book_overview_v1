Phase 4b P0 重構完成，修改 3 檔 + 新增 2 測試檔：

AC 1/2（caller Literal 單源一致性）：
- checkpoint_state.CheckpointCaller：移除未使用的 "dispatch-check"，加入實際 CLI 第三值 "checkpoint-status"；
  Literal 現為 ("snapshot", "handoff-ready", "checkpoint-status", "unknown"，unknown 為 log fallback)。
- checkpoint_view._VALID_CALLERS：改為 `tuple(c for c in get_args(CheckpointCaller) if c != "unknown")`，
  從 Literal 派生，消除雙源漂移。新增 CLI caller 時只需擴 Literal 一處。

AC 3（docstring 校正）：
- checkpoint_view.format_local_time docstring 補齊主路徑（datetime.fromisoformat + astimezone）
  與 fallback 路徑（time.localtime）雙描述，對齊實作行為。
- checkpoint_state.checkpoint_state docstring caller 範例由 "snapshot/dispatch-check" 更正為
  "snapshot/handoff-ready/checkpoint-status"。

AC 4（_render_degraded_snapshot DRY）：
- 新增 _build_degraded_state(error) 構造 fallback CheckpointState（uncommitted_files=None 觸發
  Ready Check [?] 分支；其他資料源空值落入 PRIORITIES FALLBACK → phase "3"）。
- _render_degraded_snapshot 改為呼叫 render_current_suggestion(state) + render_ready_check(state,
  caller="snapshot")，刪除獨立 print 字串；Git 狀態行合併錯誤訊息保留可觀測性。

AC 5（regression 測試）：
- tests/test_checkpoint_state/test_caller_consistency.py（7 test）：Literal↔CLI caller↔_VALID_CALLERS
  三向對齊 + 明確防護 "dispatch-check" 不復出現 + handoff_status_for 接受/拒絕行為。
- tests/test_checkpoint_state/test_degraded_snapshot_dry.py（5 test）：_build_degraded_state 欄位 +
  _render_degraded_snapshot 輸出必含 render_current_suggestion / render_ready_check 兩 block 字串 +
  錯誤訊息可見性。

邊界遵守：僅動 Context Bundle 列出 3 檔（checkpoint_state.py / checkpoint_view.py / track_snapshot.py）
+ 新增 2 測試檔；未動其他模組，未改 CLI 對外介面（snapshot / handoff-ready / checkpoint-status 輸出保持）。
