### 問題根因

`checkpoint_state.py:331` 將 `_DISPATCH_ACTIVE_RELPATH` 硬編碼為 `.claude/state/dispatch-active.json`，但專案實際慣例（11 個 hook + `track_dispatch_check.py`）一律使用 `.claude/dispatch-active.json`，且 `.claude/state/` 目錄根本不存在。

`_read_dispatch_active()` 永遠拋 `FileNotFoundError`，被 SAFE_CALL 捕獲走 fail-open fallback（`active_agents=0`），導致 `process-skip-guard` 等 caller 永遠讀不到真實 active dispatch 數，靜默退化為「無法偵測 active dispatch」。

### 影響範圍

- 直接：`.claude/skills/ticket/ticket_system/lib/checkpoint_state.py` (lines 331, 402, 408, 414)
- 測試：`.claude/skills/ticket/tests/test_checkpoint_state/test_observability_log.py:262` 斷言訊息含舊路徑
- 下游 caller：`process-skip-guard-hook` 等讀 `_read_dispatch_active` 的模組，全部已 fail-open 走 fallback（W10-017.2 實作時發現）
