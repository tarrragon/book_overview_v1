### 變更摘要

W14-045 落地 commands/ 下 4 檔的 file_lock 包圍，沿用 W14-042.1.2 共用 helper（lib/file_lock.py）。

| 檔案 | 加 lock 點數 | 說明 |
|------|-------------|------|
| fields.py | 5 | execute_set_field / execute_add_acceptance / execute_remove_acceptance / execute_add_spawned / execute_set_decision_tree |
| track_acceptance.py | 3 | execute_check_acceptance（含 single/batch helper save 在 lock 內）/ execute_accept_creation / execute_append_log（抽 helper 維持 schema） |
| track_relations.py | 3 | _execute_set_relation_field / execute_add_child（雙 lock：parent→child 固定順序）/ execute_phase |
| track_set_acceptance.py | 1 | execute_set_acceptance |

### 設計決策

- Lock target 統一用 `get_ticket_path(version, ticket_id)`（不依賴 load 後的 _path），與 lifecycle.py / ticket_builder 一致。
- execute_check_acceptance 將 load + dispatch 整體置於 lock 內；helper 內部的 save_ticket 仍在同一 lock 區塊（同 path 不會 self-block，因為 helper 不重複加 lock）。
- execute_append_log 抽出 _execute_append_log_locked 維持 schema：避免在原函式內加大量縮排破壞既有 PC-093 / W17-208 邏輯可讀性。
- execute_add_child 需保護兩個 ticket file（parent + child）；採固定順序嵌套 lock（parent → child）避免將來 caller 反向加鎖造成 deadlock；兩 path 不同無 self-block。
- 被引用 ticket 的 validate_ticket_exists 為 read-only，與其分離 lock 範圍會讓邏輯複雜；統一在 lock 內執行（read 其他 ticket 安全）。

### 不在本 ticket 範圍

- list/show/query 類 read-only command（無 save 不需 lock）
- track_relations.execute_agent / list_tickets（純讀）
