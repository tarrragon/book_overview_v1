### Context Bundle（W10-064 衍生洞察清單）

**洞察 1：Hook event 時機與名稱的語意陷阱**
- 現象：PostToolUse(Agent) 字面像「Agent 結束後」，但實測 background 派發**啟動時**就觸發
- 證據：W10-024/060/061 三輪修補皆繞道
- 升級層：basil-hook-architect agent + hook-development-methodology

**洞察 2：「代理人完成」相關 Hook 必須掛 SubagentStop**
- 現象：本專案 75+ hooks 中至少 7 個服務「代理人完成」語意但全掛 PostToolUse(Agent)
- 證據：active-dispatch-tracker / agent-commit-verification / agent-completion-broadcast / handoff-prompt-reminder 等
- 升級層：規則或 agent 強制提示
- 落地：W10-066/067 為兩個典型實作

**洞察 3：agent_id 才是 source of truth**
- 現象：dispatch-active.json 用 agent_description 字串匹配，碰同名派發失準
- 證據：當前 session 殘留 7 筆未清記錄，agent_description 重複
- 升級層：basil-hook-architect agent 設計指引

**洞察 4：「啟動 vs 完成」職責必須分掛兩個 event**
- 規則：啟動邏輯（註冊、驗證 prompt）→ PreToolUse(Agent)；完成邏輯（清理、驗證 commit、廣播）→ SubagentStop
- 反模式：混掛同一 event 是時機混亂的源頭
- 升級層：basil-hook-architect agent + ARCH-019 error-pattern

**洞察 5：LLM 清單類答案的幻覺模式**
- 現象：claude-code-guide agent 列「18+ events」「SubagentStart」「TaskCompleted」全幻覺，但 SubagentStop 細節正確
- 規律：清單最易幻覺（模型「補齊」合理項目），單項細節較可信
- 防護：清單必須對 source（本機 spec / Context7）逐項核對
- 升級層：agent guideline 或 methodology（已部分覆蓋於 PC-063 Reality Test，但「清單幻覺」是新角度）
