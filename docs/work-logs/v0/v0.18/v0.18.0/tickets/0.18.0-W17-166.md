### 三選結論：不拆分

**結論狀態 (a) 已決策：不拆分 handoff schema**。

### 收益分析（假設拆分）

| 預期收益 | 實際評估 | 結論 |
|---------|---------|------|
| 減少 schema 混淆 bug | W17-162 根因已明確為 direction 常數不一致 + GC 重複邏輯（ARCH-020），非兩種 handoff 混用同一 schema 造成。L1 已修復根因 | 收益為零：混淆 bug 的根因已消除 |
| 降低讀取端邏輯複雜度 | W17-164 的 resolve_target() 已統一讀取邏輯為單一 helper，讀取端不需區分 chain vs session | 收益為零：複雜度已由統一 helper 吸收 |
| 語意更清晰（chain 和 session 各有專屬欄位） | target_ticket_id 欄位已讓兩種場景的指向語意統一為「絕對 target」，direction 降為輔助分類標籤 | 收益極低：語意清晰度已由 target_ticket_id 解決 |

**收益總結**：三項預期收益均已被 L1（根因修復）+ L2-A（target_ticket_id 統一指向）覆蓋，拆分的增量收益趨近於零。

### 成本分析（假設拆分）

| 成本項 | 量化 |
|-------|------|
| 雙 schema 定義與維護 | 新增一組欄位定義 + 驗證邏輯，持續維護兩份 schema 的一致性 |
| 既有 JSON 遷移 | pending/ 和 archive/ 下所有 handoff JSON 需分類遷移，需寫遷移腳本 + 向後相容 fallback |
| 讀取端分流邏輯 | resolve_target() 需拆為兩個函式或加分流判斷；GC / SessionStart / Stop hooks 各自新增分流 |
| 測試維護 | 現有 11 個 target_ticket_id 測試需拆分或複製為兩組 |

**成本總結**：非零且持續（雙 schema 維護為長期負擔），無對應收益抵消。

### 成本 vs 收益量化對照

| 維度 | 拆分收益 | 拆分成本 | 淨效益 |
|------|---------|---------|--------|
| Bug 減少 | 0（根因已修） | 遷移 + 分流引入新 bug 風險 | 負 |
| 複雜度降低 | 0（resolve_target 已統一） | 雙 schema + 分流邏輯增加複雜度 | 負 |
| 語意清晰 | 極低（target_ticket_id 已解決） | 新概念學習成本 | 負 |
| 維護成本 | 0 | 雙 schema 持續維護 | 負 |

**所有維度淨效益為負或零。拆分不具合理性。**

### 資料缺口聲明

本分析基於理論推理與 W17-164 schema 設計分析，未使用 W18+W19 運行數據（因尚未進入 W18）。此缺口不影響結論：收益評估基於架構分析（L1/L2-A 已消除拆分動機），非使用率統計。即使運行數據顯示 target_ticket_id 使用率低，也不改變「拆分無收益」的結論——使用率低只代表 --next 較少被使用，不代表需要拆分 schema。

### DOC 化決策（AC4：避免後人重複提問）

本決策應記錄於 handoff 機制設計文件中，核心結論：

> **Handoff schema 不拆分 chain/session 兩種類型。**
> 理由：(1) W17-162 L1 已修復混淆根因（direction 常數不一致 + GC 重複邏輯）；(2) W17-164 L2-A target_ticket_id + resolve_target() 已統一指向語意；(3) 拆分在所有維度淨效益為負。
> 決策日期：2026-05-08。Source：0.18.0-W17-166 ANA。
