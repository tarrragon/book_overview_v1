# Execution Log

## Task Summary

建立一個可獨立發佈到 Skill marketplace 的寫作方法論 Skill。

Skill 內含**六大原子主題**：

1. **卡片盒筆記法：原子化原則** — 一張卡片一個概念，原子性如何判斷
2. **卡片盒筆記法：索引建立** — MOC（Map of Content）、index notes、標籤層級
3. **不隱藏意圖的寫作語氣** — 如何讓讀者立刻看懂意圖，避免繞彎、模糊
4. **商業邏輯的陳述方式** — 如何把 business context 寫入註解/文件，什麼該寫什麼不該寫
5. **AI 可查詢性** — grep/find/regex 友善的命名、分隔、結構設計
6. **欄位設計精神** — 同一文件/ticket/schema 內不同欄位應有不同表述角度，避免用混淆方式撰寫同一欄位

這六大主題**適用於跨媒介**：註解 / 文件 / log / prompt / ticket 欄位。跨媒介本身不是第七個主題，而是上述六個主題的應用範圍。

**兩大量化目標**：
- 降低閱讀的認知負擔
- 降低 AI 查詢或閱讀所需的 token

## Problem Analysis

### 問題根因

當前寫作相關的規範散落在多個位置：
- `comment-writing-methodology.md`（註解）
- `worklog-writing-methodology.md`（工作日誌）
- `methodology-writing-methodology.md`（方法論撰寫）
- `quality-common.md` 1.6 節（註解標準）
- `document-format-rules.md`（交接文件格式）
- `ticket-design-dispatch-methodology.md`（ticket 欄位）

缺少統一的「寫作方法論骨幹」來回答以下問題：
1. **為什麼這樣寫**（語氣、意圖、商業邏輯陳述）
2. **如何原子化**（卡片盒筆記法精神）
3. **AI 如何查詢**（grep/find/regex 適配）
4. **跨媒介如何一致**（註解/文件/log/prompt/ticket 欄位）

### 影響範圍

**直接影響**：新建 `.claude/skills/(待定)/` 目錄
**間接影響**：
- `.claude/methodologies/comment-writing-methodology.md`（可能成為 Skill reference 的來源）
- `.claude/methodologies/worklog-writing-methodology.md`（同上）
- `.claude/rules/core/document-format-rules.md`（可能保留框架特定規則，內容往 Skill 遷移）
- `0.18.0-W10-011`（註解原則擴充，併入 Skill 前置子任務）
- 未來所有新文件、註解、ticket 欄位設計將套用此 Skill

### 相關 Error Pattern

- 尚無直接 error pattern，但本 Skill 的目標是預防以下類別問題：
  - 文件重複（同概念多處定義）
  - 隱藏意圖（註解或描述繞過關鍵商業邏輯）
  - AI 查詢困難（命名/格式缺乏 grep 適配）
  - 跨媒介風格不一（同一團隊的註解、log、ticket 風格各異）

**語氣規範**（從該 prompt 提煉）：

| 代號 | 規範 | 通用度 |
|------|------|-------|

**內容規範**：

| 代號 | 規範 | 通用度 |
|------|------|-------|
| R5 | 多重可能並陳，不強加結論 | 中（分析類適用，決策類可能需要推薦） |
| R6 | 以原文為準，不替使用者圓說 | 高 |
| R7 | 並列多方視角（不服務單一敘事） | 高 |
| R8 | 描述位置/次數/頻率，取代推論心理動機 | 高 |
| R9 | 指出轉述與原文的落差 | 高 |

**結構規範**：

| 代號 | 規範 | 通用度 |
|------|------|-------|
| R10 | 並列選項不排序推薦 | 中（分析類適用） |
| R11 | 標示代價與取捨 | 高 |
| R12 | 不替人做行為決定 | 低（諮商特定） |
| R13 | 資訊不足時承認不足 | 高 |

**核心精神**：描述優先於評價（Description over Judgment）、多元並陳（Plural Framings）、拒絕敘事偏袒（Reject Biased Narrative）
