### 修改摘要

採方案 A：最小改動擴充 `WORKLOG_MAIN_FILE_REGEX`，並補強註解說明四種主檔變體。

### 變更檔案

| 檔案 | 變更 |
|------|------|
| `.claude/hooks/phase-completion-gate-hook.py` | regex 從 `^v\d+\.\d+\.\d+\.md$` 擴充為兩個 alternation 涵蓋 -main / -work-log suffix；同步更新註解列出四種命名模式 |
| `.claude/hooks/tests/test_phase_completion_gate_hook.py` | 新增 6 測試案例（3 正例 + 3 反例） |

### regex 設計

```python
WORKLOG_MAIN_FILE_REGEX = re.compile(
    r"^v\d+\.\d+\.\d+(-(main|work-log))?\.md$"
    r"|^v\d+\.\d+-(main|work-log)\.md$"
)
```

雙 alternation 結構：

- alt 1（3-component 版本，suffix 可選）：`v0.18.0.md` / `v0.18.0-main.md` / `v0.18.0-work-log.md`
- alt 2（2-component 版本，suffix 必須）：`v0.18-main.md`

設計理由：若採單一 `^v\d+(\.\d+)*(-(main|work-log))?\.md$` 會誤命中 `v0.18.md`（既有 test_minor_version_worklog_not_excluded_as_main_file 要求其仍走檢查）。

### 反例維持不命中

`v0.18.0-phase-completion.md` / `v0.18.0-W6-007.md` / `v0.18.0-refactor.md` 均不符合任一 alt（suffix 限定 main/work-log），保持原行為。
