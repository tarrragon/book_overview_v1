### 問題根因

W17-009 ANA → W17-011 group 完成 scheduler CLI 基礎設施（`ticket track runqueue` 支援 priority + blockedBy + chain + `--context=resume` handoff 交集），並整合了 `session-start-scheduler-hint-hook`（W17-011.4）與 `handoff --auto`（W17-011.5）。**但用戶最常接觸的 4 個入口未同步改線**：

| # | 入口 | 現況路徑 | 應接線至 |
|---|------|---------|---------|
| 1 | `/ticket` 裸指令（SKILL.md v2.3.0 §58） | 執行 `ticket resume --list`（檔名字母序） | `ticket track runqueue --context=resume` |
| 2 | `resume --list` 內部邏輯 | `scan_pending_handoffs()` → `sorted(glob('*.json'))` 檔名字母序（`handoff_utils.py:206`） | 套用 priority + chain 排序，或直接委派給 runqueue |

## 4 入口逐一改用方式

### 入口 1：SKILL.md 第58行（W17-027.1）

現況：SKILL.md:58 寫 ticket resume --list
改法：改為 ticket track runqueue --context=resume --top 3
複雜度：低（純文字替換 1 行 + Change Log bump v2.4.0）
Scope：僅 SKILL.md 第58行流程步驟 1
非-Scope：不改第175-185行的 runqueue 說明段（已正確）；不改其他 SKILL 章節

### 入口 2：resume --list 內部邏輯（W17-027.2）

現況：resume.py:433 _execute_list() 透過 list_pending_handoffs() 透過 scan_pending_handoffs() 走 sorted(glob) 檔名字母序排列，無 priority/chain 資訊

改法建議（策略 B：排序注入）：
- 保留 _execute_list 輸出格式，但 list_pending_handoffs() 回傳前依 priority 排序
- 排序 key：_priority_rank() 邏輯從 track_runqueue.py 提取為 shared utility（放 lib/scheduler_utils.py）
- scan_pending_handoffs() 保留為底層 IO helper，不改動

策略 A（轉發 execute_runqueue）不建議：需構造 argparse mock、stdout 格式 breaking change

Scope：resume.py list_pending_handoffs 加 priority 排序；可選新增 lib/scheduler_utils.py；新增 2-3 個測試
非-Scope：不刪 scan_pending_handoffs；不改 stale/error 邏輯；不改 resume 其他子命令

### 入口 3：handoff-prompt-reminder-hook（W17-027.3）

現況：handoff-prompt-reminder-hook.py:297 輸出 /ticket resume --list 查看完整清單
改法：替換為 /ticket 或 ticket track runqueue --context=resume --top 3
複雜度：低（1 行字串替換）
Scope：僅 line 297 訊息模板
非-Scope：不改 hook 偵測邏輯；不改觸發條件

### 入口 4：handoff --auto 追加 runqueue 提示（W17-027.4）

現況：handoff.py:1176-1180 _execute_auto_handoff 成功後只印 handoff 檔案已建立，無下一步建議
改法：成功（return 0）前追加 stderr 提示「下一個可接手 ticket：ticket track runqueue --context=resume --top 3」
設計決策：提示走 stderr（不汙染 stdout，因 --auto 的 stdout 可能被 pipe 消費）
Scope：_execute_auto_handoff return 0 前追加 2-3 行 stderr
非-Scope：不改 JSON 寫入；不改互動式 handoff；不自動執行 runqueue
