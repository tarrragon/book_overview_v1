## Layer 2 落地：deps 指令反思鏈深度警示

### 實作摘要

**寫入檔案**：
- `.claude/skills/ticket/ticket_system/commands/track_query.py`（+43 行）
  - 新增 `REFLECTION_CHAIN_WARN_THRESHOLD = 3` 常數
  - 新增 `_compute_reflection_chain_depth(ticket, version)` 輔助函式
  - `execute_deps` 整合反思鏈深度顯示與 WARNING 輸出
- `.claude/skills/ticket/tests/test_reflection_chain_depth.py`（8 個測試）

### 反思鏈定義

從當前 Ticket 出發沿 `source_ticket` 祖鏈回溯，連續 ANA type Ticket 的數量（含當前）。遇到非 ANA type / 無 source_ticket / 循環引用即停止。

### 輸出格式

未觸發（depth < 3）：`Reflection Chain Depth: N`

觸發（depth >= 3）：
```
Reflection Chain Depth: 3
[WARNING] 反思鏈深度 = 3（ANA spawn ANA 連續 3 層）
          鏈: A -> B -> C
          建議：評估是否繼續反思或終止（參見 W15-010 Layer 2 終止條件）
```

### Hook 評估（AC 選擇性）

決策：不實作 Hook。理由：
1. CLI 警示已覆蓋 PM 派發前查看 context 的主要時機
2. Layer 3 反思終止閘門規則（W15-022）已由 PM 端規則覆蓋同情境
3. 減少 Hook 數量降低維護負擔（參考 IMP-048/049/051）

若未來發現覆蓋不足，可低成本追加 Hook。

### 測試結果

`tests/test_reflection_chain_depth.py` 8 passed in 0.12s。涵蓋：
1. 單一 ANA 無 source（depth=1）
2. 非 ANA root（depth=0）
3. 連續 3 層 ANA 觸發閾值
4. 非 ANA 祖先中斷鏈
5. 循環引用防護
6. source_ticket 指向不存在 Ticket
7. execute_deps 閾值達標輸出 WARNING
8. execute_deps IMP type 不輸出 WARNING
