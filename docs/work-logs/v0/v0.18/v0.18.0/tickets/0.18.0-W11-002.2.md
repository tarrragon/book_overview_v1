### 重構摘要

_cascade_unblock_children 主迴圈從 5 個 if 分支 / 巢狀 3 層改寫為 classify → dispatch → collect 三步資料流：

1. classify（純函式）：新增 _classify_child(child_id, child, ticket_map) -> ChildOutcome，依 Phase 1 §2.2 規則決策每個 child 的 outcome。無副作用、可單獨測試。
2. dispatch：對 kind == "unblock" 的 outcome 執行 child["status"] = STATUS_PENDING + save_ticket；成功歸 unblocked，失敗歸 warnings（status="save_failed"）。
3. collect：unblocked 與 warnings 兩個清單以 list comprehension + 簡單 for 構造，巢狀深度 1 層。

### 新引入的資料結構

- ChildOutcomeKind: Literal["unblock", "blocked_pending", "in_progress_warning", "skip"]
- @dataclass class ChildOutcome: id / title / kind / status_for_warning

### 行為差異（acceptance §4 save 失敗顯示於警告清單）

之前：save 失敗只透過 format_warning(...) 印一行 stderr-style 訊息，不計入 unblocked 也不計入 pending，從 _print_children_warnings 統一警告區塊中消失。

現在：保留原 format_warning 即時提示（兼容 test_cascade_save_failure_non_fail_fast 對 "C1" in out 的斷言），同時把 child 加入 warnings 清單（status="save_failed"），由 _print_children_warnings 顯示於統一警告區塊，使 save 失敗與 blocked_pending、in_progress 三類問題在 caller 處可見一致。

### 公開介面保持不變

- _cascade_unblock_children(parent_ticket, version, ticket_map=None) -> Tuple[List, List] 簽名與回傳結構不變（warnings dict 仍含 id/title/status）。
- _post_complete_cascade 完全未動。
- ticket_map 中 child dict 的 in-place mutation 契約保留（dispatch 階段執行），docstring 已更新說明 mutation 發生在 dispatch unblock 階段。

### 不在本 ticket 範圍

- _print_children_warnings 對 status="save_failed" 的顯示文字未調整（沿用 [save_failed] 格式），若需更友善訊息可另起 ticket。
