## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

`ticket_validator._is_placeholder` 預處理流程在剝除 HTML 註解 + markdown 分隔符後，直接對殘餘內容做 `\bN/A\b|\bTODO\b|\bTBD\b|\(pending\)` 全段 substring 搜尋。表格 cell 中的 N/A（合法「不適用」標示）、TODO（描述程式碼 TODO 註解的內容）、TBD（待定項目參考）皆會命中，整章節被誤判為 placeholder 阻擋 complete。

### 影響範圍

- `.claude/skills/ticket/ticket_system/lib/ticket_validator.py:_is_placeholder`（主要修復點）
- `.claude/skills/ticket/ticket_system/lib/acceptance_auditor.py:_is_placeholder`（平行簡化版本，ARCH-020 漂移點，本 ticket 順帶 consolidate）
- 既有測試：`tests/test_ticket_validator.py` + `ticket_system/tests/test_ticket_validator.py` + `tests/test_acceptance_auditor.py`

### 相關 Error Pattern

- PC-138 trade-off 表格 N/A cell 誤判（首例）
- PC-144 Solution 中 TODO trigger 註解描述誤判（同家族延伸，本 session 撞牆觸發）
- ARCH-020 跨進程 validator 邏輯漂移（acceptance_auditor 與 ticket_validator 兩處平行實作）
## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復策略

**ticket_validator._is_placeholder**（主修復）：在 keyword 檢查前新增「剝除 markdown 表格行 + 表格存在性判定」前置步驟。

1. 偵測表格存在：`has_table = re.search(r"^\s*\|.*\|\s*$", content, MULTILINE)`
2. 剝除表格行：`content_no_tables = re.sub(r"^\s*\|.*\|\s*$", "", content, MULTILINE).strip()`
3. 分支判定：
   - 剝除後為空且原本有表格 → 作者寫表格即實質內容，回 `False`
   - 否則對 `content_no_tables`（或無表格時對 `content_no_separator`）做 keyword 檢查

**acceptance_auditor._is_placeholder**（consolidate）：原 acceptance_auditor 內有簡化版本（HTML 註解命中即 True、無 word boundary、無表格豁免），與 ticket_validator 版本邏輯漂移（ARCH-020 模式）。本次改為 delegate 到 ticket_validator._is_placeholder，獲得完整修復家族（W17-032 / W17-071 / W17-094 / W10-125）。
