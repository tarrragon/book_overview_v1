**核心原則**（語言無關）：

2. **不抽新 fixture**——既有三個工具函式對 15 測試足夠，引入額外 helper 反增認知負擔

#### 共通虛擬碼骨架（GWT 三段式）

每個測試體採用統一結構，phase3b 依此貼樣即可:

```pseudocode
def test_xxx(self, project_with_tickets):
    tmp_path, tickets_dir = project_with_tickets

    # Given: 寫入初始 ticket disk 狀態
    _write_ticket(tickets_dir, "<id_1>", {"<field>": <value>})
    _write_ticket(tickets_dir, "<id_2>", {"<field>": <value>})

    # When: 觸發目標函式
    from ticket_system.commands.migrate import _update_cross_references
    result = _update_cross_references(old_id, new_id)

    # Then: 讀回 disk 驗證
    fm = _read_frontmatter(tickets_dir / "<id>.md")
    assert fm["<field>"] == <expected>
```

**關鍵呼叫慣例**:

| 函式 | 簽名 | 備註 |
|------|------|------|
| `_update_cross_references(old_id, new_id)` | 直接呼叫 | 回傳 int（updated_count），AC2 多欄位測試會用到 |
| `_migrate_single_ticket(version, source_id, target_id, dry_run=False, backup=False)` | 端到端 | AC1 第 4 測試 / AC4 / AC3 內部 |
| `_batch_migrate(version, config_file, dry_run=False, backup=False)` | 批量入口 | AC3 / AC4，需先寫 migrations.yaml |
