### 根因假設（非 migrate.py bug，而是歷史/測試缺口）

- (A) **時間差**：父 ticket 在 child migrate 完成後才手動建立並把舊 ID 列入 children，未觸發 cross_references 掃描

- (C) **batch 遷移路徑**：批量遷移每筆都呼叫 _migrate_single_ticket → _update_cross_references，理論上正常
