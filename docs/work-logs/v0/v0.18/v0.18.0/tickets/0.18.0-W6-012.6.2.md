### 修復摘要

在 `src/overview/book-file-importer.js` 補入 CSV 反序列化路徑，對齊 W6-012.6.1 `book-exporter.js` 輸出格式：

1. **檔案類型偵測**：新增 `_isCSVFile`（識別 `.csv` 副檔名與 `text/csv` MIME，含 exporter 實際使用的 `text/csv;charset=utf-8;`），`_validateFileBasics` 同時接受 JSON 與 CSV
2. **格式分派**：`_readFileWithReader` → `_handleReaderSuccess` → `_handleFileContent` 新增 `fileFormat` 參數，CSV 走 `_parseCSVContent`、JSON 維持原 `_parseJSONContent`
3. **CSV parser**（`_parseCSVRows`）：state machine 處理雙引號包覆欄位、`""` 跳脫雙引號、`\n` / `\r\n` 換行
4. **欄位順序容錯**（`_csvRowToBook` + `CSV_HEADER_TO_FIELD`）：依 header 名稱對應 field，非依固定位置；未知 header 欄位忽略（前向相容）
5. **反序列化策略**（`_deserializeCSVField`）：
   - `authors` 以 `", "` split → string[]
   - `tagIds` 以 `"; "` split → string[]
   - `progress` → number
   - 其餘欄位保留 string
6. **走既有過濾流程**：CSV parser 回傳的 book 陣列進入 `_processBookData` → `_filterValidBooks`（_isValidBook 要求 id/title/cover 為非空字串），id 衝突由消費端既有 dedupe 流程處理

### 設計決策

- **分隔符對齊 exporter**：authors `", "` / tagIds `"; "` 與 W6-012.6.1 `_bookToCSVRow` 一致，而非 spec 規劃的 pipe `|`。理由：當前 exporter 已用 comma/semicolon 落地，importer 對稱必須對齊實際輸出；spec 的 pipe 策略屬 v2.0+ 完整集規劃，本 ticket 範圍為 v0.18 最小集 round-trip
- **必要 header 驗證**：缺 `id` 或 `書名` header 時拋 `PARSE_ERROR`，避免無 id 資料進入 dedupe 流程造成靜默重複
- **空欄位處理**：authors / tagIds 空字串 → `[]`（與 exporter `_bookToCSVRow` 對 `Array.isArray=false` 輸出空字串對稱）
- **錯誤碼分類**：CSV 解析失敗使用 `ErrorCodes.PARSE_ERROR`（與 JSON `_parseJSONContent` 一致）

### 不在本 ticket 範圍

- identifiers / progressInfo JSON-in-cell 序列化（spec 已定義但屬 .6.3 落地，當前 exporter 未輸出）
- v2.0+ 完整 14 欄集合（屬 COMPLETE_V2 preset 接線範圍）
- pipe `|` 分隔策略（spec 規劃，需配合 exporter 同步切換才能 round-trip）
