## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

三個 hook 各自實作 where.files 解析（約 100 行重複邏輯）：

- agent-dispatch-validation-hook.py:153-211 _load_ticket_where_files：手刻 regex 解析 frontmatter（不規範化）
- file-ownership-guard-hook.py:152-181 _parse_ticket_files：吃已解析 frontmatter dict（規範化）
- parallel-dispatch-verification-hook.py:213-261 read_ticket_where_files：用 find_ticket_file + parse_ticket_frontmatter（規範化）

行為三種不同：(1) 用 ad-hoc regex；(2)(3) 用 hook_utils 既有 helper；(2)(3) 額外做 normalize_path。

### 影響範圍

四個檔案：hook_utils/hook_ticket.py（新增 helper）+ 三個 hook（改用 helper）。

### 設計決策

新增兩個 helper：

1. extract_where_files_from_frontmatter(frontmatter) — 從已解析 frontmatter 取出 where.files，相容 dict / list / 換行字串 / 空值。
2. extract_where_files(ticket_id, project_root, logger) — 完整鏈路：定位 ticket md（find_ticket_file 支援 flat + hierarchical）+ 解析 frontmatter + 委派 helper 1。

回傳「原始 strip 後字串」、不做 normalize。需要規範化的呼叫端（file-ownership / parallel-dispatch）自行套用 normalize_path，避免破壞 agent-dispatch-validation 的絕對路徑大小寫敏感比對。

### 為何分兩個 helper

file-ownership-guard 在同一流程已先呼叫 parse_ticket_frontmatter 取 status / parent_id 等欄位，再次呼叫 extract_where_files(ticket_id) 會造成重複 I/O。提供 frontmatter 版本避免冗餘解析。

### 修復摘要

1. 新增 hook_utils/hook_ticket.py 兩個 helper：
   - extract_where_files_from_frontmatter(frontmatter) — 純解析（無 I/O）
   - extract_where_files(ticket_id, project_root, logger) — 完整鏈路（含 find_ticket_file）
2. 在 hook_utils/__init__.py re-export 兩個 helper。
3. agent-dispatch-validation-hook.py：_load_ticket_where_files 從 60 行收斂為 1 行（return extract_where_files(ticket_id)）。
4. file-ownership-guard-hook.py：_parse_ticket_files 委派 extract_where_files_from_frontmatter，保留 normalize_path 套用。
5. parallel-dispatch-verification-hook.py：read_ticket_where_files 委派 extract_where_files，保留 normalize_path 套用。

### 不在本 ticket 範圍

- normalize_path 兩份實作的合併（file-ownership 版本含 ".." 防護，parallel-dispatch 版本較簡）— 屬另一個 DRY 點，本 ticket 聚焦 where.files 抽取。
- _ensure_file_list 在 file-ownership 已成 dead code，但保留以避免擴大 scope。

### 行為一致性

agent-dispatch-validation：原 ad-hoc regex 僅支援 hierarchical 路徑（v0/v0.18/v0.18.0/...），新 helper 透過 find_ticket_file 同時支援 flat（v0.18.0/tickets/）+ hierarchical，等同於擴充覆蓋（無回歸）。原回傳格式保持「raw strip 字串」，下游 _classify_paths 不變。

file-ownership-guard / parallel-dispatch-verification：normalize_path 套用流程保持，回傳結果與重構前相同。
