### Option D 可行性結論（15:18）— PM workflow 不可行

**結論：Option D 對 PM workflow 結構性不可行，無需實測 .claude/ 寫入結果**

#### 實證
PM 用 ToolSearch 搜尋以下 query：
- `SlashCommand` → 無結果
- `slash command execute` → 無相關工具
- `add directory permission` → 無相關工具
- `workspace directory scope` → 無相關工具
- `+add +dir` → 無結果

CC runtime 未提供任何 deferred tool 讓 AI 動態擴展 `additionalDirectories`。`/add-dir` 是用戶端 interactive slash command。

#### 為何「請用戶執行 /add-dir」不算驗證 Option D

Option D 的價值前提是「PM 能在派發代理人前自動擴展 scope」。若每次都要用戶手動執行 `/add-dir`：
- 不是「PM workflow」，是「人工 workflow」
- 每個 worktree 都要記得跑，跨 session 不持久
- 用戶體驗比 Option E（白名單規則）差

#### 結論

Option D 從 PM workflow 角度結構性不可行。即使 `/add-dir` 能突破 .claude/ hardcoded 保護（這也只是推測），PM 無法呼叫它，無法構成自動化解法。
