### 問題根因（靜態分析）

**初步矛盾**：靜態程式碼讀取顯示自動載入路徑「已存在」，但實際表現為 Overview 顯示 0 本書。需 browser 實機驗證確認根因。

**已驗證的程式碼路徑**：

| 階段 | 位置 | 狀態 |
|------|------|------|
| 進入點 | `src/overview/overview.js:222` | 已呼叫 `overviewController.loadBooksFromChromeStorage()` |
| 載入實作 | `src/overview/overview-page-controller.js:297-346` | 正確讀 `chrome.storage.local.get(['readmoo_books'])` |
| 結構檢查 | controller line 313 | 檢查 `result.readmoo_books && result.readmoo_books.books` |
| 資料更新 | controller line 829 `_updateBooksData` | `this.currentBooks = books; this.filteredBooks = [...books]` |
| 顯示 | controller line 519 `updateDisplay` → `renderBooksTable(this.filteredBooks)` | 直接渲染 filteredBooks |
| 儲存端對應 | `src/background/events/event-coordinator.js:553-562` | 儲存結構 `{ books, extractionTimestamp, extractionCount, ... }` |

儲存端寫入結構與載入端讀取結構一致（`readmoo_books.books`），靜態檢查未發現結構不匹配。

### AC 1 驗證：Overview 開啟自動顯示 storage 書目

**測試方法**：跳過 Readmoo 登入流程，直接 inject 5 本 mock 書到 storage：

```javascript

```

| 項目 | 預期 | 實際 |
|------|------|------|
| Console log | 載入完成 log | ✓ `📅 提取時間: 2026/5/12 上午10:03:22` |

| DOM displayedBooks | "5" | ✓ "5" |
| DOM tableBody rows | 5 | ✓ 5 |
| First row 內容 | 含書名 | ✓ "測試書籍 1readmoo-reading未分類" |
| controller.currentBooks | length=5 | ✓ length=5 |

**結論**：AC 1 已通過。自動載入路徑 `overview.js:222 → loadBooksFromChromeStorage() → _updateBooksData() → updateDisplay() → renderBooksTable()` 全鏈正確運作。

### AC 2 驗證：無 storage 資料時保留手動載入 UI

**測試方法**：清空 storage 後 reload overview：

```javascript
await chrome.storage.local.remove(['readmoo_books']);
```

**結果**：

| 項目 | 預期 | 實際 |
|------|------|------|
| Console log | 空資料 log | ✓ `📂 Chrome Storage 中沒有書籍資料` |
| DOM totalBooks | "0" | ✓ "0" |

| 空狀態提示 | 顯示 | ✓ "📚 目前沒有書籍資料" |
| 錯誤容器 | 隱藏 | ✓ `display: none` |
| Loading indicator | 隱藏 | ✓ `display: none` |

**結論**：AC 2 已通過。空 storage 時降級回手動載入流程，未顯示錯誤、未隱藏匯入按鈕。
