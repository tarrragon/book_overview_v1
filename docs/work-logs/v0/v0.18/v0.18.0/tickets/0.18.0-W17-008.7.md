# check-acceptance 語法歧義分析（Explore 代理人產出，PM 代寫入）

> 註：Explore agent 誤判「只讀模式」無法執行 ticket CLI，實際 Explore 有 Bash 工具可跑。此為 agent 執行邊界誤判——記為 PC-101 變種觀察。PM 代寫入以免結果流失。

## 語法組合矩陣（8+ 組合）

### check-acceptance 實際支援

| 組合 | 指令 | 行為 |
|------|------|------|
| 1 | `check-acceptance <id> <index>` | 勾選單一項 |
| 2 | `check-acceptance <id> <index> --uncheck` | 取消單一項 |
| 3 | `check-acceptance <id> --all` | 批量勾選全部 |
| 4 | `check-acceptance <id> --all --uncheck` | 批量取消全部 |

### set-acceptance 平行支援

| 組合 | 指令 | 行為 |
|------|------|------|
| 5 | `set-acceptance <id> <index> --check` | 語意明確勾選 |
| 6 | `set-acceptance <id> <index> --uncheck` | 語意明確取消 |
| 7 | `set-acceptance <id> --all-check` | 全部明確勾選 |
| 8 | `set-acceptance <id> --all-uncheck` | 全部明確取消 |

## 歧義點識別

### 歧義 A（P0）：--uncheck 無 index 時語意不明

- 文件只示範 `check-acceptance <id> 1 --uncheck`
- 用戶可能嘗試 `check-acceptance <id> --uncheck`（缺 index），行為未定義
- CLI 實際行為：拒絕或靜默失敗（需實測確認）

### 歧義 B（P1）：check-acceptance vs set-acceptance 雙軌並存

- W14-030 引入 set-acceptance 標記「推薦」，但 check-acceptance 未標為 deprecated
- 兩套命令並行，文件未清楚說明何時用哪套
- 與 W17-008.8（決策指引）高度重疊——應交叉引用

### 歧義 C（P0）：index 三種格式文件未全列舉

- 代碼支援：1-based 整數 / 0-based / 文字搜尋
- 文件只示範整數（1, 2, 3）
- 用戶嘗試文字搜尋（如 `check-acceptance <id> "完整對照表"`）可能意外成功但不穩定

### 歧義 D（P1）：--all + index 互斥規則未明

- `check-acceptance <id> 1 --all` 是否合法？代碼 argparse 行為需確認
- 文件未列錯誤組合範例
