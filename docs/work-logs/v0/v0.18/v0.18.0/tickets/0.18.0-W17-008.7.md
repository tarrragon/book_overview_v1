## Problem Analysis

### 問題根因

`check-acceptance` 同時支援 `--all` 和 positional index 兩種語法路徑，且 `--uncheck` 與兩者皆可組合，但 `references/track-command.md` 只示範部分組合，導致用戶對「`--uncheck` 無 index 時的行為」「文字搜尋 index 是否支援」「`--all` 與 index 互斥規則」皆無從判斷。同期 W14-030 引入語意更明確的 `set-acceptance` 但未標 `check-acceptance` 為 deprecated，雙軌並存放大歧義。

### 影響範圍

- `.claude/skills/ticket/cli/track.py` check-acceptance argparse（line 591-607）
- `.claude/skills/ticket/references/track-command.md`（語法表 line 72-74）
- 與 W17-008.8（set vs check 決策指引）文件補完位置完全相同 → 應合併實作

## Solution

### 語法組合矩陣（8 組合）

**check-acceptance**：

| # | 指令 | 行為 |
|---|------|------|
| 1 | `check-acceptance <id> <index>` | 勾選單一項 |
| 2 | `check-acceptance <id> <index> --uncheck` | 取消單一項 |
| 3 | `check-acceptance <id> --all` | 批量勾選全部 |
| 4 | `check-acceptance <id> --all --uncheck` | 批量取消全部 |

**set-acceptance**（語意明確版）：

| # | 指令 | 行為 |
|---|------|------|
| 5 | `set-acceptance <id> <index> --check` | 明確勾選 |
| 6 | `set-acceptance <id> <index> --uncheck` | 明確取消 |
| 7 | `set-acceptance <id> --all-check` | 全部勾選 |
| 8 | `set-acceptance <id> --all-uncheck` | 全部取消 |

### 歧義點清單

| ID | Pri | 歧義 |
|----|-----|------|
| A | P0 | `--uncheck` 無 index 時行為未定義（文件未說明） |
| B | P1 | check-acceptance vs set-acceptance 雙軌並存，未標 deprecated |
| C | P0 | index 三種格式（1-based / 0-based / 文字搜尋）文件未全列舉 |
| D | P1 | `--all` 與 index 互斥規則未明 |
