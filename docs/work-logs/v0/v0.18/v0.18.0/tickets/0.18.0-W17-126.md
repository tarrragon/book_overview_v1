## 實作摘要

新增 `.claude/hooks/commit-msg-layer2-marker-check-hook.py`（PostToolUse, Bash matcher）：

### 核心邏輯
1. 偵測 `git commit` 命令且 commit 成功
2. 取得 HEAD changed files（`git diff-tree --name-only -r HEAD`）
3. 用 `lib.framework_paths.is_framework_path()`（W17-127.1 SSOT）篩 framework 檔案
4. 取 commit msg（`git log -1 --format=%B HEAD`），檢查 Layer 2 標記
5. 缺則輸出機會成本語氣警告（exit 0 不阻擋）

### 合法 Layer 2 標記
- `Layer 2 by <agent-name>` — 已派審查
- `Layer 2 N/A by <理由 ≥ 10 字元>` — 明示豁免並附理由

### 豁免情境
- merge commit（msg 開頭 `Merge ` 或多 parent）
- revert commit（msg 開頭 `Revert "..."`）
- commit msg 含 `[skip layer2]` 標記
- commit msg 含 `nothing to commit` / `Aborting`（commit 未成功）

### 警告訊息設計（機會成本語氣）
建議用詞：「建議補做」「成本較高的捷徑，建議事後補做」「豁免條件」；避免：「禁止 / 必須 / 不可」

### 註冊位置
`.claude/settings.json` PostToolUse Bash matcher，timeout 5000ms，緊接 post-git-commit-hook 後

### 設計依據
W17-122 ANA linux 視角警示：原 Layer A/B/C 三層防護對「事後 Layer 2 維度」實質僅 Layer C 紙本約束，本 hook 補強為 Layer A（事後自動偵測）形成完整防護鏈
