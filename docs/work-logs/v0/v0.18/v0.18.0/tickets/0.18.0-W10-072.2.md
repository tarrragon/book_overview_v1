### 問題根因

acceptance-gate-hook 的 `verify_acceptance_record`（acceptance_checkers/acceptance_checker.py）對所有 IMP 類型一視同仁，當 acceptance 未勾選且 body 無關鍵字時統一輸出 `ACCEPTANCE_RECORD_MISSING_WARNING`，建議「派發 acceptance-auditor 執行驗收」。對純文件 IMP（如 `.claude/rules/`、`.claude/methodologies/`、`docs/` 修改）此建議過頭——純文件編輯無程式碼測試需求，PM 確認 spec 一致性即可手動勾選 acceptance items。

### 影響範圍

- `.claude/hooks/acceptance_checkers/acceptance_checker.py`：`verify_acceptance_record` 加 doc-only 識別
- `.claude/hooks/lib/hook_messages.py`：新增 `ACCEPTANCE_RECORD_DOC_ONLY_HINT` 常數

### 相關 Error Pattern

父 ANA W10-072 Reality Test 事實 3（已驗證觸發鏈）。Cry wolf 累積會降低 PM 對 hook 警告的信任度（quality-baseline 規則 4）。

### 修復摘要

1. `lib/hook_messages.py`：新增 `GateMessages.ACCEPTANCE_RECORD_DOC_ONLY_HINT` 常數，文字改為「建議手動驗收」並明示「不需派發 acceptance-auditor（純文件編輯無程式碼測試需求）」。
2. `acceptance_checkers/acceptance_checker.py`：
   - 新增 `_DOC_PATH_PREFIXES` 常數列舉純文件路徑前綴（.claude/rules/、methodologies/、pm-rules/、references/、error-patterns/、agents/、hook-specs/、handoff/、skills/、docs/）。
   - 新增 `_DOC_ONLY_THRESHOLD = 0.8`。
   - 新增 `_is_doc_path()`：判定單一路徑是否屬純文件（前綴命中或 .md 副檔名）。
   - 新增 `is_doc_only_imp(frontmatter)`：依 `where.files` 計算純文件路徑佔比，≥ 80% 回傳 True。
   - `verify_acceptance_record`：當 should_check_acceptance=True 且 has_accept=False 時，先呼叫 `is_doc_only_imp` 決定使用 `ACCEPTANCE_RECORD_DOC_ONLY_HINT`（純文件 IMP）或 `ACCEPTANCE_RECORD_MISSING_WARNING`（一般 IMP）。
3. 新增測試檔 `tests/test_acceptance_checker_doc_only.py`（11 個案例）：
   - 整合測試 5 案例：純文件 100%、80% 邊界、60% 不足、純程式碼、無 where.files
   - 單元測試 6 案例：is_doc_only_imp 各種輸入

### 設計決策

- 路徑識別採前綴 + .md 副檔名雙判別：既可精準命中框架路徑類別，也能涵蓋 README.md / CHANGELOG.md 等 repo 根層文件。
- 80% 門檻而非 100%：允許純文件 IMP 附帶少量 helper script（如 lint 檢查腳本），不誤判為一般 IMP。
- `where.files` 缺失時保守 fallback 至一般訊息（資訊不足無法判斷）。
