### 三層防護完成

**Layer 1 — Framework Rule（人類可讀 + auto-load）**

新增 `.claude/rules/core/tool-selection.md`（v1.0.0）：
- 規則 1：依檔案類型選工具（對照表 + Why/Consequence/Action 三明示）
- 規則 2：Hook 被拒時的 Fallback（禁 self-imposed early stop）
- 與 tool-discovery.md / AGENT_PRELOAD 規則 7 / bash-tool-usage-rules.md 邊界章節
- 違規偵測表：規則文件層 + Hook 強制層雙層

**Layer 2 — Hook Hard Enforcement**

新增 `.claude/hooks/mcp-write-tool-on-text-file-guard-hook.py`：
- 偵測條件 1：tool_name 屬 SERENA_WRITE_TOOLS frozenset（5 個寫入工具）
- 偵測條件 2：file_path / relative_path 副檔名屬 NON_CODE_EXTENSIONS（.md/.txt/.yaml/.yml/.json/.toml）
- 大小寫不敏感（.lower() 後比對）
- 多副檔名取最後一個（Path.suffix）
- 模組化函式：is_serena_write_tool / classify_extension / is_non_code_file / extract_file_path / build_deny_message
- 採用 hook_utils 統一日誌（setup_hook_logging + run_hook_safely + read_json_from_stdin + emit_hook_output）
- exit 0 allow / exit 2 deny；輸入為空或無路徑時預設 allow（fail-safe）

**Layer 3 — Settings 註冊**

`.claude/settings.json` PreToolUse 新增 matcher entry：
- matcher: `mcp__serena__replace_content|mcp__serena__replace_symbol_body|mcp__serena__insert_after_symbol|mcp__serena__insert_before_symbol|mcp__serena__safe_delete_symbol`
- hooks: 1 entry 指向新 hook
- JSON 結構驗證通過

### 設計決策

1. **副檔名清單採白名單反向（限制非程式碼）**：對未知副檔名預設允許，降低誤擋風險。新增類型只需加入 NON_CODE_EXTENSIONS frozenset。
2. **Read-only serena 工具不在範圍**：find_symbol / get_symbols_overview 不影響檔案，本 hook 只 match 寫入工具集合。
3. **多副檔名取最後一個**：foo.test.md 視為 .md，符合直覺與 Path.suffix 行為。
4. **deny message 引用 PC-112 與規則路徑**：讓代理人 / PM 看到拒絕時能直接定位修正方向。
