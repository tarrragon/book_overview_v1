### 變更摘要

針對 `src/content/adapters/readmoo-adapter.js` 四個 logger key 調整級別：

| Key | Before | After | 行號 |
|-----|--------|-------|------|
| FALLBACK_SELECTOR_ATTEMPT | `logger.warn` | `logger.debug` | 157 |
| LAST_RESORT_STRATEGY | `logger.warn` | `logger.info` | 171 |
| SELECTOR_PARADOX | `logger.warn` | `logger.warn`（保留） | 195 |
| WAIT_FOR_BOOK_ELEMENTS_TIMEOUT | `logger.warn` | 條件式：finalCount > 0 → `logger.info`；finalCount === 0 → `logger.warn` | 323-332 |

### 設計決策

- 條件式級別採 `if/else` 分支呼叫 `logger.info` / `logger.warn`，避免將方法解構為變數（`const fn = logger.info`）造成 `this` 綁定遺失（首次嘗試此寫法時所有 timeout 路徑的測試都 30s 卡死）。
- Logger 方法內部使用 `this._shouldLog` 與 `this._output`，呼叫端必須維持 `logger.xxx()` 形式。

### 不在本 ticket 範圍

- 動態 selector 偵測重構（W6-012.9 ANA 方案 B）
- 其他 adapter 的 logger 級別調整
