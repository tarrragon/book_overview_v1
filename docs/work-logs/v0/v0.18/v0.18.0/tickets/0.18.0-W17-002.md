### 1. 抽取規則 YAML Schema

#### 1.1 規則配置檔位置

`.claude/skills/ticket/ticket_system/config/context_bundle_rules.yaml`（新建；為單一事實來源，避免常數散落各處）。

#### 1.2 Schema 定義

```yaml
# Context Bundle 自動抽取規則
version: 1

# 1.2.1 來源優先序（高→低；第一個命中即停止對同一欄位的進一步抽取）
source_priority:
  - source_ticket     # 最高：顯式指定的根因/提案 ticket
  - blocked_by        # 次之：依賴前置完成的 ticket
  - related_to        # 最低：相關脈絡參考

# 1.2.2 可抽取欄位（key 為 source ticket 的欄位，value 為目標 Context Bundle 子節）
extractable_fields:
  - source_field: what
    target_subsection: "Task Reference"
    format: "- {source_id} what: {value}"
    type: string
  - source_field: why
    target_subsection: "Rationale Chain"
    format: "- {source_id} why: {value}"
    type: string
  - source_field: where.files
    target_subsection: "Related Files"
    format: "- {value}  # from {source_id}"
    type: list[string]
    dedup: true              # 與本 ticket 既有 where.files 合併去重
  - source_field: acceptance
    target_subsection: "Source Acceptance (reference)"
    format: "- [ref] {value}  # from {source_id}"
    type: list[string]
    include_completed: false # 僅帶未完成項供參考；completed 項略過避免噪音

# 1.2.3 衝突解決規則
conflict_resolution:
  rule: "pm_wins"              # PM 手寫 > 自動抽取
  detect_placeholder: true     # 將 DEFAULT_UNDEFINED_VALUE（「待定義」）視為未填，可被抽取覆寫
  never_overwrite_sections:    # 這些 section 一旦非空，抽取器不得寫入
    - "Problem Analysis"
    - "Solution"
    - "Test Results"

# 1.2.4 循環引用與深度
traversal:
  max_depth: 1                 # Phase 1 僅抽取直接來源，不遞迴 source of source（避免指數放大）
  cycle_detection: true        # A→B→A 檢出後停止並警告
  skip_completed_source: false # completed source 允許抽取（rationale 仍有效）
  skip_missing_source: true    # source 不存在→警告但不中止

# 1.2.5 規模限制（token 預算防護）
limits:
  max_total_chars: 2000        # 抽取結果總字元數上限；超過截斷並註記
  max_items_per_field: 5       # 單一欄位最多保留 N 項（如 where.files 只取前 5）
  truncate_indicator: "... (truncated, see source ticket)"
```

#### 2.1 模組位置

#### 2.2 資料型別

```python
from dataclasses import dataclass, field
from typing import Literal, Optional

SourceKind = Literal["source_ticket", "blocked_by", "related_to"]
ExtractStatus = Literal["success", "partial", "no_source", "source_missing", "cycle_detected"]

@dataclass
class ExtractedField:
    source_id: str
    source_kind: SourceKind
    source_field: str         # e.g. "what", "where.files", "acceptance"
    target_subsection: str    # e.g. "Rationale Chain"
    rendered_lines: list[str] # 已套用 format 模板的輸出行
    truncated: bool = False

@dataclass
class ExtractResult:
    status: ExtractStatus
    target_ticket_id: str
    extracted: list[ExtractedField] = field(default_factory=list)
    skipped: list[dict] = field(default_factory=list)  # [{source_id, reason}]
    warnings: list[str] = field(default_factory=list)
    context_bundle_markdown: str = ""                   # 可直接寫入 Context Bundle section 的完整 markdown
    total_chars: int = 0
```

#### 2.4 錯誤情境契約

| 情境 | 處理 | ExtractResult.status | 警告訊息 |
|------|------|----------------------|---------|
| 無任何 source/blocked_by/related_to | 提早返回 | `no_source` | 「本 ticket 無可抽取來源，Context Bundle 需 PM 手填」 |
| source_ticket 檔案不存在 | skip 該項，繼續其他 | `partial` 或 `source_missing` | 「source {id} 不存在於 {version}，略過」 |
| source 的欄位為 DEFAULT_UNDEFINED_VALUE | 跳過該欄位 | 視整體而定 | 「source {id} 的 {field} 未填，無內容可抽取」 |
| 循環引用（A source=B, B source=A） | 檢測後停止遞迴 | `cycle_detected` | 「偵測循環引用 {path}，停止遞迴」 |

| 寫入後 `ticket track full {id}` 的 Context Bundle section 含抽取內容 | markdown heading `## Context Bundle` 可見 | 是 |
| 二次呼叫不重複寫入（idempotency） | 若 Context Bundle section 已含「已從 X 抽取」標記，跳過或合併 | 是 |
| PM 手寫在先，自動抽取在後 | 依 conflict_resolution.pm_wins 保留 PM 內容，抽取內容附加於子節但不覆蓋 PM 子節 | 是 |
| section 寫入觸發 frontmatter updated 欄位更新 | `updated` 日期刷新 | 是 |

#### 4.3 幂等性策略

在 Context Bundle section 開頭寫入隱藏標記：

```markdown
<!-- auto-extracted: v1 | sources: W17-001,W17-005,W17-004 | chars: 1243 -->
## Context Bundle
...
```

抽取器在寫入前先檢查是否有此標記：
- 無標記 → 直接 append
- 有標記且 sources 相同 → 跳過（顯示「已是最新抽取結果」）
- 有標記但 sources 不同 → 更新標記並重新渲染（PM 手寫子節保留）
