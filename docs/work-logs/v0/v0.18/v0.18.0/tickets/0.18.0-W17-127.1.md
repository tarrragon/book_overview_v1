### SSOT 設計

新建 `.claude/config/framework-paths.yaml` 含四個欄位：
- `categories`：framework 類別名清單（rules/pm-rules/references/skills/methodologies/agents/hooks/error-patterns），供 dispatch-validation 動態組裝 alternation regex
- `framework_paths`：廣義 framework 路徑前綴清單（含 `.claude/` 前綴），供 W17-127.2 skill-trigger-hook 與 W17-126 commit-msg hook 使用
- `layer1_paths`：Layer 1 narrower scope（與既有 `LAYER1_PATTERNS` 完全等價的 6 條路徑）
- `exempt_paths`：豁免 glob 清單（`.claude/skills/**/tests/`、`.claude/hooks/tests/` 等測試與暫存檔）

### Lib 模組

新建 `.claude/hooks/lib/framework_paths.py` 提供共用 API：
- `get_categories() / get_framework_paths() / get_layer1_paths() / get_exempt_paths()`：載入 SSOT 並回傳對應清單
- `is_framework_path(path) -> bool`：廣義 framework 判定（含豁免優先檢查）
- `is_layer1_path(path) -> bool`：Layer 1 narrower 判定（保留既有 substring + .md 結尾語意）
- `reset_cache()`：測試專用，清空 `_load_config` 的 lru_cache

效能設計：`_load_config` 套用 `@lru_cache(maxsize=1)`，同 process 只讀 YAML 一次（對齊 ginger 視角 cache 警示）。Python 3.9 相容（用 `typing.List/Optional`，未用 PEP 604 union）。

### 漸進替換策略（保證行為等價）

對齊 PM WRAP P「重構策略採漸進替換」防護：

1. **layer-boundary-validator-hook.py**：
   - `LAYER1_PATTERNS = get_layer1_paths()` — 別名指向 SSOT，向後相容
   - `is_layer1_file()` 內部委派 `lib.framework_paths.is_layer1_path()`，logger.debug 行為保留
2. **agent-dispatch-validation-hook.py**：
   - 新增 `_build_meta_task_patterns()` 函式：從 SSOT `categories` 動態組裝 alternation regex
   - SSOT 載入失敗時保守降級為原 hardcoded 預設集（避免重構引發 regression）
   - `_META_TASK_PATTERNS` 仍為 module 級 tuple，呼叫端零修改
