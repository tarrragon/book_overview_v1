### 設計決策

**選擇單一 SKILL.md 不拆 references/**：

- 流程性內容（每 workflow 都是「前置 → 步驟 → 預期輸出」三段式），讀者一次看完最完整
- 各 workflow 主體 < 30 行，總 SKILL.md ~300 行，符合 progressive disclosure 上限
- 未來若新增 workflow 過多再拆 references/

**選擇 9 個 workflow 而非用戶要求的 5 個**：

- 用戶 5 情境合併為 9 workflow：debug 情境細分為 3 個（popup / content script / service worker），因 Manifest V3 三者 console 隔離，操作流程差異大
- console / network 雖屬同類但操作差異大（network 多了「篩選」維度），拆兩個 workflow

**選擇跨專案通用而非 Readmoo 專屬**：

- 用戶選項明確：「跨專案通用 Chrome Extension debug skill」
- 通用化路徑：路徑用 `<unpacked-extension-path>`、host page 用 `<target-host>`、extension ID 用 `<id>` 等 placeholder
- 專案範例延伸到 docs/chrome-devtools-mcp-usage.md（W6-008 已建）

### 與既有資源邊界

| 資源 | 定位 | 與本 SKILL 邊界 |
|------|------|---------------|
| 本 SKILL | 跨專案程序化 debug 流程 | 觸發 → 進對應 workflow → 呼叫 chrome-devtools-mcp 工具 |
| `docs/chrome-devtools-mcp-usage.md` | 專案級設定與範例 | 設定細節、attach 模式切換、專案路徑 |
| `.mcp.json` | 專案級設定檔 | MCP server 註冊 |
| `frontend-with-playwright` SKILL | 自動化 E2E + DOM/CSS 設計協議 | Playwright 而非 chrome-devtools-mcp；不處理 extension 專屬情境 |
