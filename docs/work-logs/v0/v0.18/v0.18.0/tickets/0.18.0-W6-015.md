**已驗證事實**：
1. 當前 npm audit 僅 4 個 low severity（非 ticket 標題的 19 個）
2. 4 個 vulnerability 全部位於 jest-environment-jsdom 依賴鏈內（jsdom → http-proxy-agent → @tootallnate/once）

## Solution

### 分類表（runtime vs dev × direct vs transitive × severity）

| Package | severity | direct? | runtime/dev | 修復路徑 |
|---------|---------|---------|-------------|---------|
| jest-environment-jsdom | low | direct | dev | 升級至 v30.4.1（**BREAKING**，semver major） |
| jsdom | low | transitive | dev | 隨 jest-environment-jsdom 升級 |
| http-proxy-agent | low | transitive | dev | 隨 jest-environment-jsdom 升級 |
| @tootallnate/once | low | transitive | dev | 隨 jest-environment-jsdom 升級 |

統計：runtime/high = **0**，runtime 任何 severity = **0**，dev/low = 4。

### 執行時暴露評估

**結論：當前無任何 Chrome Extension runtime 暴露的安全漏洞。**

理由：
1. package.json `dependencies` 為空陣列，所有 npm 套件均為 devDependencies
2. Chrome Extension 透過 build.js / babel 將 `src/` 打包至 `build/`，devDeps 僅參與建置 / 測試流程，不進入發布產物
3. 4 個 low vulnerability 全在 jest 測試工具鏈（jest-environment-jsdom 子樹），僅在 `npm test` 期間於開發者本機載入
4. CVE GHSA-vpq2-c234-7xj6（@tootallnate/once ReDoS）需特定 input pattern 才能觸發，jest 測試環境內無外部攻擊面

### 修復路徑分類

| 類型 | 數量 | 說明 |
|------|------|------|
| `npm audit fix` 可自動修復 | 0 | 所有修復標記為 `isSemVerMajor: true` |
| 需 major upgrade（semver-breaking） | 4 | 全數隨 `jest-environment-jsdom@30.4.1` 升級解決 |
| 需替換 lib | 0 | 不需 |

實際操作只需單一指令：`npm install --save-dev jest-environment-jsdom@^30.4.1`（須驗證 jest 主版本相容性，可能連動 `jest@30.x` 升級）。
