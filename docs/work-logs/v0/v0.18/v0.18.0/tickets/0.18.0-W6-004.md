## Problem Analysis

### 問題根因

`.claude/pm-rules/worktree-operations.md` 已有舊版手動 worktree SOP、合併檢查與 `.claude/` 路徑限制，但尚未把 CC 新增/現行使用語彙整合成派發入口規則。PM 在建立人工 session、派發 subagent、限制 sparse checkout、觀測 Hook event、清理 stale worktree 時，需要跨多份 error-pattern 與歷史 ticket 推論，容易重犯 worktree agent 產出不可見、基底過舊或 `.claude/` 編輯被擋的問題。

### 影響範圍

- `.claude/pm-rules/worktree-operations.md`：主 SOP，補齊能力入口、範例與清理流程。
- `.claude/pm-rules/command-routing.md`：作為決策樹命令分支的實際路由入口，補上 worktree SOP 指引。
- `docs/work-logs/v0/v0.18/v0.18.0/tickets/0.18.0-W6-004.md`：更新執行紀錄與驗收狀態。

### 相關 Error Pattern

- PC-019：worktree 合併流程中 Ticket 狀態遺失。
- PC-036：worktree 分支基底過舊導致代理人工作無效。
- PC-039：代理人回報完成但主倉庫看不到變更。
- ARCH-015：subagent 對 worktree 內 `.claude/` Edit/Write 的 CC runtime hardcoded 保護。
- IMP-066：subagent 在 `isolation: worktree` 下透過 ticket CLI 看不到主 repo 新建 ticket。
- W9-003 盤點：`feedback_worktree_agent_issues` 仍有部分僅存 memory，需升級到規則層。

## Solution

採用「擴充既有 SOP」而非新增 `worktree-guide.md`。原因是 `worktree-operations.md` 已是 worktree 操作與風險的集中入口；新增獨立文件會讓 PM 在派發前多一個查找點，反而增加漏讀風險。

### WRAP 三問

W（Widen）候選方案：
- 方案 A：擴充 `.claude/pm-rules/worktree-operations.md`，並在命令路由補入口。
- 方案 B：新增獨立 `worktree-guide.md`，主 SOP 僅保留舊流程。

選擇方案 A，因為既有文件已承載 PC-019、PC-039、ARCH-015，集中更新能降低規則分散。

A（Attain distance）機會成本：
- 本 ticket 是 DOC 修正，會擠壓實作任務時間；但 worktree 是並行代理人的核心安全邊界，優先補文件能降低後續派發和合併成本。

P（Prepare to be wrong）最可能失敗原因：
- `--worktree` / `-w`、`worktree.sparsePaths` 等能力若只寫概念而無路由與清理檢查，PM 仍可能不知道何時使用。防護：在 `command-routing.md` 補 Worktree 路由，並在 SOP checklists 加入派發前、合併時、清理後檢查項。

### 實作內容

- 盤點結果記錄於本 ticket Problem Analysis；規則文件本身不放執行報告，只保留可執行規則。
- 在 `worktree-operations.md` 新增「適用範圍與強制規則」與「Claude Code Worktree 能力入口」：
  - `claude --worktree <path>` / `claude -w <path>` 人工 session 範例與使用規則。
  - `isolation: worktree` subagent frontmatter 範例與 prompt 範本。
  - `worktree.sparsePaths` YAML 範例與範圍設定規則。
  - `WorktreeCreate` / `WorktreeRemove` event 用途、不可用途與最小紀錄欄位。
- 新增「Stale Worktree 清理」：補基底過舊、未提交、未合併 commit、可保留片段與強制刪除前提。
- 擴充派發前、合併時、清理後 checklist。
- 相關文件補連 PC-036、IMP-066、W9-003 memory 盤點。
- 在 `command-routing.md` 補 Worktree 路由，作為決策樹命令分支的實際入口。
