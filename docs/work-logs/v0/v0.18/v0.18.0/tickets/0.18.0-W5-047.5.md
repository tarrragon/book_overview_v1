## Phase 1 設計（PM 前台產出）

### 新分類函式

將 \`_classify_prompt_paths\` 回傳從 \`(has_claude, has_other)\` 改為三元：

\`\`\`
(has_main_repo_claude, has_external_claude, has_other)
\`\`\`

- **has_main_repo_claude**：相對路徑 \`.claude/\` 或絕對路徑 \`.claude/\` 落在主 repo 樹內
- **has_external_claude**：絕對路徑 \`.claude/\` 不落在主 repo 樹內（例 \`/tmp/xxx/.claude/\`）
- **has_other**：常見專案路徑（src/, tests/, lib/, app/...）

### 偵測實作要點

1. 用 \`hook_utils.get_project_root()\` 取得主 repo 根（已支援 worktree + CLAUDE_PROJECT_DIR）
2. 新增絕對路徑 regex：\`_ABSOLUTE_CLAUDE_PATTERN = re.compile(r"(?<![.\w])(/[^\s]+/\.claude/)")\`
3. 逐一比對 absolute match 是否以 \`str(project_root) + "/"\` 為前綴 → 主 repo 內；否則 → 外部
4. 相對路徑 \`.claude/\`（現有 pattern）預設視為 has_main_repo_claude=True（符合 PM 派發主流慣例）

### 新決策邏輯（優先順序判斷）

條件層疊，前者優先：

| 順序 | 條件 | 行為 |
|-----|------|------|
| 1 | has_external_claude=True | 阻擋：指示搬檔到主 repo 或 PM 前台處理 |
| 2 | has_other=True 且 isolation != "worktree" | 阻擋：原強制 worktree（防 .git/HEAD 污染） |
| 3 | 其他情況 | 放行 |

### 決策矩陣對照（新 vs 舊）

| has_main_repo | has_external | has_other | isolation=worktree | 舊行為 | 新行為 |
|-----------|-----------|-------|------------|-------|-------|
| True | False | False | 任何 | 放行 | 放行 |
| False | False | True | True | 放行 | 放行 |
| False | False | True | False | 阻擋（強制 wt） | 阻擋（強制 wt） |
| True | False | True | True | **阻擋（跨路徑）** | **放行（W5-050 新發現）** |
| True | False | True | False | **阻擋（跨路徑）** | 阻擋（強制 wt，訊息換） |
| False | True | False | 任何 | 放行（舊邏輯沒辨別） | **阻擋（runtime 必擋）** |
| True | True | 任何 | 任何 | 阻擋 | 阻擋 |

### 新錯誤訊息

**EXTERNAL_CLAUDE_BLOCK_MESSAGE**（新增）：

\`\`\`
錯誤：實作代理人 {agent} 的 prompt 指向外部 .claude/ 路徑

為什麼阻止：
  ARCH-015（2026-04-18 重驗）：CC runtime 對外部 worktree 內 .claude/ Write/Edit
  硬編碼拒絕（如 /tmp/ 下的 worktree）。subagent 即使在對的 cwd 下也會被擋。
  主 repo 樹內的 .claude/ 可派發成功，但外部 .claude/ 無法。

修復方式：
  1. 若目標 .claude/ 檔案應在主 repo，改用相對路徑或主 repo 絕對路徑
  2. 若目標是外部 .claude/，改由 PM 前台處理，或搬檔到主 repo 再派發

詳見: .claude/error-patterns/architecture/ARCH-015-subagent-claude-dir-hardcoded-protection.md
\`\`\`

保留 **BLOCK_MESSAGE_TEMPLATE**（僅非 .claude/ 無 worktree 時用）。
移除 **CROSS_PATH_BLOCK_MESSAGE**（跨路徑在主 repo 情境下已放行，不需此訊息；若 has_other 且無 worktree 由 BLOCK_MESSAGE 覆蓋）。
