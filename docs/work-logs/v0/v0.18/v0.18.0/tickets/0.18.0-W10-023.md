## 合法檢查 vs 焦慮性檢查（供 W10-025 使用）

**合法檢查**（放行）：
- 派發前的前置檢查（branch-status、dispatch-active 清點）
- task-notification 到達後的 dispatch-active 清點
- commit 前的 git status/diff
- 驗收類子任務 SOP 明確要求的中間檢查

**焦慮性檢查**（禁止）：
- task-notification 未到達前主動 polling agent transcript
- 同一派發週期內重複執行 git status/grep/find 無具體目的
- dispatch-active 空且無通知時主動搜尋代理人產出（可能落入 PC-050 模式 D）
