### 問題根因

Phase 4 多視角審查發現三項可獨立改善的細節（W11-004.2 父 ticket）：

- (a) bay 視角：`SKIP_PARALLEL_EVAL` 的 pair 列表中 `("不需要","評估")` 與 `("不用","評估")` 屬語意重複（同為「否定詞 + 評估」結構），命中條件等價但維護成本翻倍
- (b) thyme 視角：`pattern_info` 在 `detect_skip_intent` 與 `generate_skip_reminder` 簽名以裸 `dict` 流通，IDE/型別檢查無法捕捉欄位拼寫錯誤
- (c) linux 視角：W11-004.2 原始 spec 使用 `started_at` 字串排序（lexicographic）。雖然合法 ISO 8601 字串字典序與時序大致一致，但非 ISO 格式（舊資料、手動編輯錯誤、缺欄位）會給出未定義排序結果

### 影響範圍

- `.claude/hooks/process-skip-guard-hook.py`（單檔）
- 既有 46 個測試（test_process_skip_guard_hook.py / test_process_skip_guard_type_phase_guard.py / test_process_skip_guard_active_dispatch.py）必須持續 GREEN

### 相關 Error Pattern

### 修復摘要

單檔三項改善，外部介面（`detect_skip_intent` 返回 `(skip_type, pattern_info)` 結構 / `get_active_in_progress_ticket` 返回 `{type, current_phase}` 投影）完全不變：

1. **SKIP_PARALLEL_EVAL pair 去冗餘**（acceptance [1]）
   - 移除 `("不用", "評估")`，保留 `("不需要", "評估")`
   - 兩者皆屬「否定詞 + 評估」語意，命中條件等價
   - 加註釋說明移除原因（對齊 W11-004.2 同類 pair 移除註解風格）

2. **SkipPatternInfo TypedDict**（acceptance [2]）
   - 新增 `SkipPatternInfo(TypedDict)`，欄位 `pairs: List[Tuple[str, str]]` / `description: str` / `full_process: str`
   - `SKIP_PATTERNS` 型別標註改為 `dict[str, SkipPatternInfo]`
   - `detect_skip_intent` 返回型別 `Tuple[Optional[str], Optional[SkipPatternInfo]]`
   - `generate_skip_reminder` 參數型別 `pattern_info: SkipPatternInfo`
   - runtime 結構零變化（TypedDict 純型別層註記）

3. **started_at fromisoformat fallback**（acceptance [3]）
   - 新增 `_started_at_sort_key(value) -> Tuple[int, object]` 防禦性 helper
   - 可解析 ISO 8601 → `(0, datetime)` tier；不可解析/空/None → `(1, str)` tier
   - tuple 首元素保證「可解析項永遠優於不可解析項」，同 tier 內仍可比較
   - 在 `get_active_in_progress_ticket()` 中對返回 fm 的 `started_at` 套用驗證（不改返回值，僅 exercise helper 並保留異常資料偵測點）
   - 設計考量：W11-021 統一入口採 mtime 排序，hot path 不需 started_at 排序；本 helper 為未來 tie-break / 校驗保留通用工具

### 設計決策

- **為何不在 `find_active_in_progress_ticket` 改 mtime → started_at 排序**：違反 W11-021 統一契約與 ticket files 限定範圍（僅 `.claude/hooks/process-skip-guard-hook.py`）。本 ticket 僅在 hook 本地新增 helper，不污染共用模組
- **TypedDict vs dataclass**：TypedDict 對既有 dict literal 零侵入；dataclass 需改寫所有 `SKIP_PATTERNS` 字典字面值，違反「不改變 runtime 結構」約束

### 不在本 ticket 範圍

- W11-021 mtime 排序語意是否替換為 started_at（屬跨 hook 統一決策，需另開 ticket）
- 將 `_started_at_sort_key` 提升至 hook_utils 共用（待有第二個消費者時再抽取，避免 ARCH-020 重現）
