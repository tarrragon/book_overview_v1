### 修復摘要

在 `handoff.py` 新增 `_extract_exit_status_for_handoff(ticket)` helper，於兩個 write 路徑（manual `_create_handoff_file_internal` 與 auto `_execute_auto_handoff`）皆注入 `exit_status` 欄位。

### 設計決策

1. **抽取邏輯**：以 regex `## Exit Status\s*\n(.*?)(?=^## |\Z)` 抽 body H2 section（與 `context_bundle_extractor._read_section_body` 同模式），剝除 HTML 註解（避免誤抽 schema 樣板示例），優先取 fenced ```yaml 區塊，否則整段視為 YAML。

2. **status 枚舉驗證**：限定 success/needs_context/blocked/partial_success/failed 五值（與 W17-010 schema + W17-031.1 `_TAGGED_EXIT_STATUSES` 對齊），非合法 status 回 None。

3. **Fail-open**：缺段 / YAML 解析失敗 / status 非合法 → 回 None + stderr warning，不阻擋 handoff 主流程。對齊既有 `_extract_context_bundle_for_handoff` 風格。

4. **回傳整個 dict**（非單純 status 字串）：W17-031.1 `_get_exit_status_tag` 已預期 `isinstance(exit_status, dict)`，下游可讀 reason / confidence 等子欄位供未來 hook 摘要使用。

### 不在本 ticket 範圍

- ticket body 寫入 `## Exit Status` YAML 的代理人責任：屬 agent 行為層（已由 W17-010 schema 定義）
- runqueue / hook 顯示邏輯：屬 W17-031.1 / .5 範圍
