## 實作摘要（依 Problem Analysis 四項）

### 1. ticket track stuck-anas CLI 命令

新建 `.claude/skills/ticket/ticket_system/commands/track_stuck_anas.py`：

- 過濾條件：type=ANA AND status=in_progress AND spawned_tickets 非空 AND 全部 spawned 已 completed
- 保守設計：spawned ticket 不存在時不視為已完成（避免誤報「卡住」）
- 註冊為 version-agnostic handler（_create_version_agnostic_handlers 字典 + register_stuck_anas）
- 參數：--wave N（過濾 wave）、--all（跨所有 active 版本）、--version（覆寫自動偵測）
- 輸出格式對齊 runqueue 視覺風格（分隔線 + 編號 + complete 建議）

### 2. IMP complete 後 source ANA 提示行

lifecycle.py 新增 _print_source_ana_complete_hint：

- 在 complete 末尾、_auto_handoff_if_needed 之前呼叫
- 觸發條件：completed ticket type=IMP AND source_ticket 非空 AND source 為 type=ANA AND status=in_progress AND 全部 spawned completed
- 輸出至 stdout（非阻擋）：→ Source ANA <id> spawned 全 completed，可考慮 ticket track complete <id>

### 3. ticket create 偵測 in_progress group 提示行

create.py 新增 _detect_in_progress_groups 與 _print_in_progress_group_hint：

- 在 _persist_and_report 末尾、僅當 args.parent 為空時呼叫
- 觸發條件：當前 wave 內存在 status=in_progress 且 children 非空的 ticket
- 噪音抑制：若新 ticket ID 已是該 group 的子（前綴匹配 gid + .）→ 跳過提示
- 輸出至 stdout（非阻擋）：→ 偵測到 in_progress group：<id> (N children)。是否該 --parent <id>？

### 4. plan-to-ticket-flow.md 新增 Ticket 創建位置決策樹

新增章節「Ticket 創建位置決策樹（W17-008.15）」：

- ASCII 決策樹涵蓋三條路徑：children（--parent）/ spawned（--source-ticket）/ siblings（無 flag）
- 三種關係速查表（CLI 旗標、適用情境、違反成本）
- 創建後輔助提示對照表（指向本 ticket 三項 CLI 落地）
- 禁止項表（含「禁止建為已知 group ticket 的『父任務的兄弟』」）

版本號 bump 至 v3.4.0。

### 設計決策

- source ANA 提示限定 IMP completes：非 IMP（如 DOC、ANA 自身）completes 不觸發，避免噪音
- group 提示前綴匹配抑制：避免 0.18.0-W17-008.15 自身建立時又提示 0.18.0-W17-008
- stuck-anas 保守判定：spawned ticket 找不到時不視為「已 completed」，避免誤勸 PM 完成 source ANA

### 不在本 ticket 範圍

- 將提示行從 stdout 升級為 Hook 強制檢查（屬 W17-008 後續任務）
