### 變更摘要

依 W17-003 linux 審查 4.1/4.3/4.4 三項建議改善：

1. **4.1 docstring 誠實性**：規則 5 註解從「（保留）LLM 審查 — 本 ticket 不實作，預留 hook 點」改為「未實作（不在 W17-003 範圍；未來如需加入須另起 ticket）」，移除「保留 hook 點」誇大用語。
2. **4.3 find_section cache**：`check_section_present` / `check_content_length` 新增可選 `match: SectionMatch | None` 參數；`execute_dispatch_validate` 在外層呼叫一次 `find_section` 並將結果傳入兩函式，規則 1+2 共用一次解析。
3. **4.4 where.files 空時 INFO 提示**：`execute_dispatch_validate` 在 `where.files` 為空時於 stdout 多印一行 `[INFO] where.files 為空：規則 3 自動通過。若本 ticket 應修改檔案，請確認 frontmatter where.files 已正確填寫（DOC 類 ticket 可忽略）`，避免靜默通過遮蔽 W17-002 抽取漏掉。

### 設計決策

- `match` 為 keyword-only 可選參數：保持既有單元測試 / 外部呼叫端 API 向後相容。
- INFO 提示不改變 exit code 語意（仍為 pass）：屬「靜默 → 顯性」純資訊性改善，避免破壞既有自動化判讀。

### 不在本 ticket 範圍

- linux 4.2「規則 5 LLM 審查 hook 點」實裝（保留為未來 ticket）
- find_section 的 cache 抽取為 ticket_system.lib 公用 helper（YAGNI，目前只兩處受惠）
