### 問題根因

`.claude/hooks/` 集中所有 hook（80+ 個），其中 19 個與 ticket skill 強耦合：依賴 ticket schema、ID 規範、生命週期事件。Skill 跨專案 `sync-push` / `sync-pull` 時這 19 個 hook 不會跟著走，導致下游專案使用 ticket skill 時 hook 缺失，行為退化。

W10-016 ANA 已分析雙層 Hook 架構並決議遷移 ticket / worktree / wrap-decision 等 skill 專屬 hook 至各 skill 目錄。W10-091 已落地 `hook-completeness-check.py` 雙層掃描支援。本 ticket 為 ticket skill 的具體遷移實作。

- `.claude/hooks/` → `.claude/skills/ticket/hooks/`（20 個 hook 檔案）

- `.claude/settings.local.json`（1 處 handoff-auto-resume-stop-hook 路徑）
- `.claude/hooks/tests/`（12 個測試檔案路徑更新，混合 `hooks_path` / `_HOOKS_DIR` / `HOOKS_DIR` / `_hooks_dir` / `HOOK_PATH` 等變數命名）
- `.claude/skills/ticket/tests/`（2 個測試檔案路徑更新）

### 設計決策

1. **git mv 保留歷史**：使用 `git mv` 批次遷移以保留 git blame 歷史；隨後對遷移後檔案執行 path 常數修正。
2. **共享資源 `hook_utils` 不複製**：所有 hook 仍透過 `parents[3] / "hooks"` sys.path 引用 `.claude/hooks/hook_utils.py`，避免雙寫導致漂移（ARCH-020 同構防護）。`lib.hook_messages` / `lib.ticket_quality.*` 沿用相同路徑解析。
3. **不複製避免維護負擔**：選擇「集中 utils + 分散 hook」而非「skill 自帶完整 utils 拷貝」，本 ticket 為遷移 hook 主體；跨專案 sync 時 skill 帶 hook 但仍需 `.claude/hooks/hook_utils.py` 同步（屬框架層基線資產，已由 hook-completeness-check 涵蓋）。
4. **`META_SELF_PATH` 常數同步更新**：`ticket-skill-sync-check-hook.py` 內部用於識別「meta self-reference 豁免」的字面常數，由 `.claude/hooks/ticket-skill-sync-check-hook.py` 更新為 `.claude/skills/ticket/hooks/ticket-skill-sync-check-hook.py`，避免 PC-099 同類 false positive。
5. **不在本 ticket 註冊 post-ticket-complete-checkpoint-hook.py**：該 hook 移轉前即未在 settings.json 註冊（pre-existing 狀態），維持原狀；屬獨立 follow-up 議題，不在本 ticket 範圍。
