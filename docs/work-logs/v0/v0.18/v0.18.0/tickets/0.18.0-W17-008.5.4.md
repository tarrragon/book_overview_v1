### 修復摘要

於 `lib/messages.py` 新增 `ArgparseFormatErrorParser`（`argparse.ArgumentParser` subclass），覆寫 `error()` 方法依訊息分類：業務錯誤（invalid choice / invalid type value）改走 `format_error(ErrorEnvelope)` 結構化輸出（含 `__error_envelope_v1__` 版本標記），純語法錯誤（unrecognized args / missing required positional）委回父類保留 argparse 預設 POSIX 風格。`commands/track.py` 的 `register()` 對 track parser 綁定客製 `error()` 方法，並對 `add_subparsers` 傳 `parser_class=ArgparseFormatErrorParser`，讓所有操作子 parser 繼承業務錯誤客製。

### 變更檔案

- `.claude/skills/ticket/ticket_system/lib/messages.py`
  - 新增 `import argparse / re / sys`
  - 新增 `_ARGPARSE_BUSINESS_ERROR_PATTERNS`（兩個 regex：`invalid choice` / `invalid \S+ value`）
  - 新增 `_classify_argparse_error(message) -> Optional[str]`（回 `INVALID_CHOICE` / `INVALID_VALUE` / `None`）
  - 新增 `class ArgparseFormatErrorParser(argparse.ArgumentParser)`：覆寫 `error()`，業務錯誤渲染 `ErrorEnvelope` 寫 stderr + `sys.exit(2)`，純語法錯誤 `super().error(message)` 保留預設
- `.claude/skills/ticket/ticket_system/commands/track.py`
  - import 加入 `ArgparseFormatErrorParser`
  - `register()` 對 track parser 用 method-binding 套用客製 `error()`（範圍邊界：頂層 parser 屬 scripts/ticket.py，本 ticket 不動）
  - `add_subparsers(parser_class=ArgparseFormatErrorParser)` 讓所有操作子 parser 繼承
- `.claude/skills/ticket/ticket_system/tests/test_argparse_format_error_parser.py`（新增）
  - 8 個測試：2 業務錯誤路徑（invalid choice / invalid int value）+ 2 純語法錯誤路徑（unrecognized / missing required）+ 4 分類器邊界
- `.claude/skills/ticket/references/track-command.md`
  - 新增「CLI 錯誤分類（W17-008.5.4）」章節：分類表 / 範例 / 設計理由 / 範圍邊界

### 設計決策

- **regex pattern 而非例外攔截**：argparse 內部不拋自訂例外，唯一鉤點是 `error(message)` 字串入口；以 keyword regex（`invalid choice` / `invalid \S+ value`）分類穩定且維護成本低
- **method-binding 套用 track parser**：scripts/ticket.py 頂層 parser 不在本 ticket 範圍，避免 scope creep；用 `ArgparseFormatErrorParser.error.__get__(parser, type(parser))` 將 unbound method 綁到既有 instance，達到等效效果
- **errno 命名 `INVALID_CHOICE` / `INVALID_VALUE`**：對應 argparse 兩類業務錯誤的概念名，後續 .5.5 hook 可依 errno 分流

### 不在本 ticket 範圍

- `commands/create.py` 的 argparse 客製（W17-008.5.3 處理）
- `scripts/ticket.py` 頂層 parser 改造（會影響所有命令，超出 .5.4 範圍）
- Hook 補充邏輯實作（W17-008.5.5 處理）
