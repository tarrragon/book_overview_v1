## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### Context Bundle

**目標**：找出所有「spawned IMP 全 completed 但 source ANA 仍 in_progress」的 stuck 案例。

**步驟**：
1. `ticket track list --status in_progress`，過濾 type=ANA（讀 frontmatter）
2. 對每個 in_progress ANA：讀 frontmatter `spawned_tickets`，逐一查詢狀態
3. 分類：A(全 completed=stuck) / B(部分=進行中) / C(無 spawned=獨立)
4. W17 group 優先全列，其他 W 摘要

**產出位置**：
- Solution: 表格(ticket_id|spawned/completed|days|classification) + 統計 + multi_view_status=single_view_only
- 重現實驗結果: 用 Edit 工具編輯 ticket md（CLI 不支援該 section）

**限制**：只盤點不改 status，不建新 ticket，完成後 commit + complete。
### 問題根因

ANA ticket 衍生 IMP（spawned_tickets）後，缺少自動 cascade 機制：當所有 spawned IMP 完成時，source ANA 不會自動進入 close 狀態，須人工觸發。實際盤點顯示僅 2 張 stuck，規模有限但仍會造成 in_progress 列表認知雜訊。

### 影響範圍

- ticket frontmatter 欄位：`spawned_tickets` / `source_ticket`
- ticket lifecycle：ANA close 時機判定
- PC-075（spawned children status asymmetric）相關：本盤點為 PC-075 防護鏈的實證面

### 相關 Error Pattern

- PC-073（ANA spawned not children）：禁用 `--parent` 改用 `source_ticket/spawned_tickets`
- PC-075（spawned children status asymmetric）：acceptance-gate-hook 對 spawned 僅檢存在性
