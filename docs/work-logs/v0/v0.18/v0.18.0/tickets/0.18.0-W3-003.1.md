## 既有實作的次要觀察（非阻擋，供後續改進參考）

1. **grep `-E` 正規表達式風險**：關鍵詞直接用 `|` 串接送進 grep `-E`，若檔名包含 regex 特殊字元（如 `.`、`+`、`?`、`(`、`)`、`*`）會被當 regex 解讀。當前檔名以英文為主風險低，但應該在之後加 `re.escape()` 或改用 `-F` 固定字串模式。
2. **關鍵詞寬鬆匹配**：`path` / `hooks` / `plugins` 這類常見英文詞會產生偽陽性。例如修改 `plugins/extractor.py` 會匹配 47 個含 plugin 一詞的 pattern 檔。當前 WARNING 設計為「參考提示」可接受此寬鬆度，但若日後升級為 BLOCK 需更嚴謹。
3. **短檔名/短目錄被跳過**：`e.js`、`c/d/` 這類路徑段會完全無關鍵詞可用，導致檢查 silent skip（無 WARNING、無衝突）。實務上本專案不太會出現這種檔名，但屬於覆蓋漏洞。
4. **路徑 `parts[-3:-1]` 語義**：對 5 層路徑只取倒數 2-3 層目錄（跳過最頂層和直接父目錄的祖父目錄），邏輯較特殊，建議在程式碼加註解說明設計意圖。

以上觀察**不影響 W3-003 AC 達成**，可獨立開 IMP Ticket 追蹤。
