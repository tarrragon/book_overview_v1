---

## Problem Analysis

### 問題根因

`ticket track log` 子命令缺 `--section` 參數：
- `commands/track.py:454-456` 只有 `ticket_id` 與 `--version`
- `commands/track_query.py:506` `execute_log` 輸出整個 Execution Log
- 介面不對稱：`append-log` 已支援 `--section`，`log` 卻沒有對應過濾

### 影響範圍

- `.claude/skills/ticket/ticket_system/commands/track.py` line 454-456 附近：argparse 加 `--section`
- `.claude/skills/ticket/ticket_system/commands/track_query.py` line 506 附近：`execute_log` 加過濾分支
- `.claude/skills/ticket/references/track-command.md`：補 log --section 範例

### 設計決策

- section pattern 沿用 W17-008.9 的容錯規則：`rf"^##\s+{re.escape(section)}\s*$"` 多行模式
- 無 `--section` 時保持原行為（輸出全部 Execution Log）
- SECTION_NOT_FOUND 錯誤訊息與 W17-008.9 風格一致：列出該 ticket md 所有 `^## ` 標題

### 範圍邊界（避免 scope creep）

- 不抽 helper（同 W17-008.9 邊界）；section parsing 邏輯複製/同步即可
- 若用戶之後要重構共用 helper，由獨立 ticket 承擔

- W：A1 加 --section / A2 新增 ticket track section CLI / A3 wrapper script；選 A1 對齊 append-log 介面對稱原則
- A：機會成本低，P1 短任務
- P：最可能失敗為 section pattern 與 W17-008.9 不一致導致兩處 drift；防護=測試覆蓋同樣 5 案（標準/末尾空白/雙空白/前綴相同 false positive 兩種）

### 修復摘要

- `commands/track.py:454-457`：`p_log` argparse 加 `--section`（選填，預設 None），共用 `TrackMessages.ARG_SECTION` 常數
- `commands/track_query.py:execute_log`：在 Execution Log 全量輸出之前插入 `--section` 分支
  - section pattern：`rf"^##\s+{re.escape(section)}\s*$"` + `re.MULTILINE`（與 `track_acceptance.execute_append_log` line 549 同步，沿用 W17-008.9 容錯規則）
  - 範圍擷取：`header.start()` 到下一個 `\n## ` 或文件結尾
  - SECTION_NOT_FOUND：沿用 `ErrorMessages.SECTION_NOT_FOUND`，並列出該 ticket md 所有 `^## ` 標題引導
- `references/track-command.md:33`：補 `--section` 範例與可用 section 清單，PC-105 grep 可命中

### 設計決策

- 不抽共用 helper：與 W17-008.9 邊界一致，section parsing 邏輯在 `execute_log` / `execute_append_log` 兩處複製即可；之後若有重構需求由獨立 ticket 承擔
- 介面對稱性：`append-log --section` 已存在，`log --section` 對齊降低 PM 認知負擔（W17-004 第 17 項 live 落差來源）
- `getattr(args, "section", None)` 防呼叫端未設 attribute（測試 mock 友善）

### 不在本 ticket 範圍

- 共用 section helper 抽取
- 其他子命令（query / full / show）支援 --section
