## Task Summary

在 ticket track complete 實作父子 cascade 邏輯：父 complete 後自動解鎖 blocked children + 有未完成 children 時顯示警告但不阻止

---

## Problem Analysis

### Context Bundle (from W5-015 ANA)

**根因**（W5-015 分析結論）：ticket CLI 父子生命週期完全解耦。
- `add-child`（track_relations.py:242-312）：只寫 parent_id/children/chain，不觸 status 或 blockedBy
- `complete`（lifecycle.py:374-489）：不檢查 children 狀態，父可越過未完成子 complete
- `release`（lifecycle.py:491-519）：status=blocked 唯一來源，無自動解鎖機制
- `_check_unblocked_tickets`（lifecycle.py:618-640）：只處理 blockedBy 欄位，對 parent_id 無效

**採用方案**（WRAP 推薦 B+E 組合）：
- **方案 B（cascade 解鎖）**：父 complete 成功後，對所有 status=blocked 且 blockedBy 不含其他未完成 ticket 的 children 自動轉為 pending
- **方案 E（children 警告）**：父 complete 前若 children 存在未完成（status != completed），輸出警告列出 children 狀態但不阻止 complete

**修改對象**：
- `.claude/skills/ticket/ticket_system/commands/lifecycle.py`（complete 函式 374-489 + 新增 cascade helper）

**不改範圍**：
- add-child 邏輯（保持單一職責，不觸發生命週期變化）
- release 邏輯
- 既有 blockedBy 解鎖機制（_check_unblocked_tickets）

**遺留問題**：W5-014 目前 status=blocked 需手動遷移（方案：本實作完成後，manually claim W5-014 或將 status 改為 pending）

**相關 Ticket**：W5-015（ANA，已完成）、W5-014（待遷移的 blocked 子任務）、W5-013（已完成的父）
