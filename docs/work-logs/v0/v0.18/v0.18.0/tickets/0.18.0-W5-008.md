## Problem Analysis

## WRAP 三問（IMP 簡化版）

### W（Widen）候選方案
- 方案 A：Hook 校驗兩表象限一致（非侵入但仍雙寫）
- 方案 B：pm-rules 表移除「象限」欄，references 為單一來源（真 DRY）
- 方案 C：兩表合併（違反 domain 分層）

### 關鍵觀察
`.claude/references/friction-management-decision-points.md` 第 50-68 行已存在「AskUserQuestion 18 場景覆蓋對照」表，明確列出每個場景對應象限。此表已是事實單一來源；`pm-rules/askuserquestion-rules.md` 第 214-233 行的場景表中「象限」欄位是事後補的冗餘。

### A（Attain distance）機會成本
投入 30m-2h 修復 maintenance smell。不做代價：未來象限調整時可能遺漏同步導致 PM 決策時兩處語意分歧。此風險概率低但影響範圍廣（所有 PM session）。

### P（Prepare to be wrong）失敗預想
失敗情境：移除「象限」欄後，PM 讀 pm-rules 時需跳 reference 才能取象限，增加認知跳躍。
防護：在表標題註解明確寫「象限見 references/friction-management-decision-points.md 的對照表」（現已有類似引用，強化即可）。

### 決策：選 B
references 既有對照表即單一來源；pm-rules 的象限欄屬便利重複。
