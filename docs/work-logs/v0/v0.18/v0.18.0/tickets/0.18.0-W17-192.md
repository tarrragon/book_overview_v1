### 處理策略選擇

W17-192 acceptance 列兩選項：A 修 import path / B 整檔 skip。實作採 B（整檔 pytest.skip）：

| 比較項目 | A 修 import 指向 archived/ | B 整檔 pytest.skip |
|---------|------------------------|-------------------|
| 工程 cost | 改 sys.path + import | 加 pytestmark + try/except 包 import |
| 復活 archived 測試 | 是（需驗證 7 測試類仍可跑） | 否（測試保留但 skip） |
| 是否合理 | 不（archived 模組通常無人維護） | 是（明示「已 archive」狀態） |
| 維護負擔 | 高（archived 模組 bug 仍需修） | 低（skip 後不影響） |
| 恢復路徑 | 模組搬回後仍工作 | 移除 pytestmark + 改 import 即可 |

選 B 理由：archived 模組通常為 deprecation 證據，resurrect tests 反而增加維護負擔；clear skip marker + 註解說明恢復條件，符合「先停損再評估」原則。
