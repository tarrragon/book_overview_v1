## Solution

### 已實作項目

1. **新增 `ensure_utf8_io()` 於 `.claude/hooks/hook_utils/hook_base.py`**
   - 使用 Python 3.7+ `stream.reconfigure(encoding="utf-8", errors="replace")` API
   - 套用於 stdin、stdout、stderr 三個 stream
   - 靜默處理 ValueError/AttributeError（不阻斷 Hook 執行）

2. **`setup_hook_logging` 自動呼叫 `ensure_utf8_io`**
   - 於 `.claude/hooks/hook_utils/hook_logging.py` 入口加入
   - 所有使用 `setup_hook_logging()` 的 Hook 自動獲得 UTF-8 保護
   - 零侵入：既有 Hook 無需修改 import 或呼叫

3. **`hook_utils/__init__.py` 匯出 `ensure_utf8_io`**
   - 加入 `__all__` 列表，允許個別 Hook 直接 import

4. **修復 `hook_base.py` 內的 git subprocess 呼叫**
   - 為 `git rev-parse --show-toplevel` 加上 `encoding="utf-8", errors="replace"`
