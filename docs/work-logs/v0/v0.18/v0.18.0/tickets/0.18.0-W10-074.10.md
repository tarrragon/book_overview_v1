## Problem Analysis

### Scope 調整（基於 W10-074.9 掃描結果，2026-04-16）

**原決策（074.1 決策 3）**: 全取代三個方法論檔案，引用全部遷移到新 Skill

**074.9 掃描發現**:
- `comment-writing-methodology.md`: 11 個引用點跨 9 檔案（含 comment-qa-hook.py 3 處）→ 可全遷移到 writing-code-comments.md
- `worklog-writing-methodology.md`: 4 個引用點跨 3 檔案（含 worklog-format-check.py 2 處、ask_user_question_reminders.py）→ 可全遷移到 writing-documents.md
- `methodology-writing-methodology.md`: 3 個引用點 + 5 個同名 SKILL 引用（需分辨）→ **建議部分保留**

**methodology-writing 的 scope 調整**:
原因：本檔案承載「SKILL vs 方法論職責分工」（框架元層治理）和「方法論管理檢查清單」「經驗分享敘事脈絡」——這些不屬於 compositional-writing Skill 的「寫內容」範疇。

**新處理方式**（三段式）:
1. **重疊內容**（30 秒電梯理論、六準則語氣/功能描述/去產品名）→ 遷移至新 Skill 的 writing-documents.md 或 designing-fields.md
2. **不重疊內容**（SKILL vs 方法論分工、撰寫檢查清單、retrospective 敘事）→ 重組改名為 `.claude/methodologies/framework-meta-methodology.md`
3. **引用更新**（3 個 methodology 引用點）→ 依重疊/不重疊性質，分別指向新 Skill 或新改名檔案

**Hook 字串引用特別注意**（runtime 輸出影響）:
- `comment-qa-hook.py` 3 處字串引用 → 必須先確認新 Skill 路徑已建立才能更新
- `worklog-format-check.py` 2 處字串引用 → 同上
- `ask_user_question_reminders.py` 1 處字串引用 → 同上
- `dart_parser.py:263` docstring 內描述性提及 → 無路徑依賴，可彈性處理
