## 介面最終定版（2026-04-20）

所有功能統一於 `ticket show` 下，短 flag 為主、長 flag 為輔，細節交由 `ticket show --help` 說明。

### 完整 CLI 規格

```
ticket show <id> [-r] [-R RENDERER] [-p|-P]
```

| 短 flag | 長 flag | 行為 |
|--------|--------|------|
| （無） | （預設） | TTY auto render（glow>mdcat>bat>raw），非 TTY 自動 raw |
| `-r` | `--raw` | 強制純文字輸出 |
| `-R X` | `--renderer X` | 指定渲染器：glow/mdcat/bat/auto |
| `-p` | `--pager` | 強制開分頁（less -R） |
| `-P` | `--no-pager` | 停用分頁 |

### 使用範例

```bash
ticket show W17-015              # 預設：TTY 渲染 + auto pager
ticket show W17-015 -r           # raw
ticket show W17-015 -R bat       # 指定 bat
ticket show W17-015 | cat        # 自動降 raw（TTY 偵測）
```

### 設計原則

1. **單一入口**：只有 `ticket show`，不在 `track full` 重複
2. **短 flag 優先**：常用操作（-r/-R/-p/-P）全部單字母
3. **細節靠 help**：`ticket show --help` 完整列舉；SKILL.md 只放典型範例
4. **保留 `track full`**：仍可用，作為 raw-only 的明確語意別名

### 落地 IMP 合併為單一任務

原 IMP-A 與 IMP-B 合併，不再分 Tier：

**IMP（下一步建）**：實作 `ticket show` subcommand
- argparse 主入口加 show 分支
- 短 ID 解析（引用 track 的 ID resolver）
- TTY 偵測 + 渲染器 fallback 鏈
- 四個短 flag（-r/-R/-p/-P）+ 長 flag
- `--help` 完整文件
- 驗收：6 項
  1. TTY 下預設彩色渲染
  2. pipe 至 cat 自動 raw
  3. `-r` 強制 raw
  4. `-R X` 指定渲染器，缺失時明確報錯
  5. `-p`/`-P` 分頁控制
  6. 所有渲染器皆缺失時 fallback raw 不 crash

- 支援短 ID（`W17-015` 自動補版本 0.18.0）
- `--raw` flag 供 pipe/script 使用

## 建議方案：三層漸進

採「先驗證需求再升級」路線，避免過度工程。

### Tier 0（立即，零成本）— 必做
用 `bat` 包裝 `ticket track full`：
```bash

```
**落地**：SKILL.md 加使用範例 + 建議 shell alias（如 `tshow`）。

### Tier 1（低成本，驗證後做）— 條件式
`ticket track full` 新增 `--render` flag：
- TTY 偵測（`sys.stdout.isatty()`），非 TTY 或 `--raw` 強制純文字
- 渲染器優先序：`glow` > `mdcat` > `bat -l md` > raw fallback
- 不強制安裝任何外部工具，找不到就 fallback

**觸發條件**：Tier 0 用 1-2 週後用戶仍覺不足才做。

### Tier 2（高成本，暫不做）— 擱置
rich (Python native) 整合。擱置理由：依賴膨脹、啟動成本、Tier 0/1 已覆蓋 80% 情境。

## CLI Flag 介面設計（Tier 1）

```
ticket track full <id> [--render|--raw] [--renderer glow|mdcat|bat|auto]
```

| Flag | 行為 |
|------|------|
| 預設（無 flag） | TTY 時 auto render，非 TTY 時 raw（向後相容 pipe 使用者） |
| `--render` | 強制渲染（即使非 TTY） |
| `--raw` | 強制純文字（腳本/pipe 友善） |
| `--renderer X` | 指定渲染器，缺失時報錯 |

---

## 向後相容性

**風險**：若預設開啟渲染，現有 pipe 使用者（如 hook/script 解析輸出）會被 ANSI 污染。

**防護**：
1. TTY 偵測為核心防線
2. 所有自動化腳本強制加 `--raw`（可用 `NO_COLOR` 環境變數作備援）
3. Tier 0 期間不改 `track full` 預設行為，僅補文件
