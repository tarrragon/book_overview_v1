### 問題根因

代理人在 IMP/ANA/DOC 任務中缺乏結構化的自檢機制（Layer 1）：
- 文字品質違規（三明示缺失、資訊優先序顛倒、單向禁令缺正向錨點）靠外部委員審查才能發現
- 禁用字（簡體、emoji、日文漢字）在代理人完成後才被 Layer 2 委員發現
- Schema 章節填寫問題（自定義 H2、佔位符未替換）需 Hook 攔截，但 Hook 屬末端防護

document-writing-style v1.2.0 已落地「二次審查強制執行」，但僅針對文件撰寫；代理人工作流缺對應機制。

### 影響範圍

- 所有 IMP/ANA/DOC ticket 的完成流程（代理人 complete 前缺自檢輪）
- .claude/references/agent-dispatch-template.md（需加 Layer 1 觸發指引）
- .claude/pm-rules/context-bundle-spec.md（需加 Layer 1 觸發時機說明）
- .claude/references/agent-self-check-template.md（新建）

**關鍵設計決策**：
1. **自檢不替代首次品質**：模板首段聲明避免「依賴自檢」反模式（AC 4 防護）
2. **Layer 1 寫 Solution / Layer 2 寫 Test Results**：避免覆寫衝突（AC 5 介面同步）
3. **prompt 末段插入**：自檢是完成後動作，放末段符合「生成 → 審查」認知流程
4. **精簡版備用**：純文件類 DOC 可用精簡版（掃禁用字 + emoji），降低觸發成本

**反模式防護（AC 4）**：
- 模板「首段聲明」明確：「自檢不替代首次品質要求；首次提交前已符合規範的項目無需重複掃描」
- 零違規時可省略記錄，避免強制流程化的虛工

**Layer 1 / Layer 2 介面同步（AC 5）**：
- Layer 1（自律）：代理人執行，結果寫 `## Solution → ### 自檢結果`
- Layer 2（外部委員）：basil-writing-critic 執行，報告寫 `## Test Results`
- 兩者 H2 章節不重疊，PM 合併 ticket 時無衝突
