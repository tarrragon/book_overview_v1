### 根因確認：H4 版本未生效

**驗證結果**：H4 confirmed。其餘假設 H1/H2/H3/H5 排除。

**根因**：`ticket` CLI 透過 `uv tool install` 安裝為獨立工具，安裝後的 Python package 位於 `/Users/tarragon/.local/share/uv/tools/ticket-system/lib/python3.12/site-packages/`。W11-035 commit (bba99b08) 新增了 `_auto_stage_completion_files` 和 `_auto_stage_git_add` 兩個函式到 source `lifecycle.py`，但未執行 `uv tool install --reinstall` 同步安裝版本。

**證據鏈**：

| 項目 | 來源 lifecycle.py | 安裝版 lifecycle.py |
|------|-----------------|-------------------|
| 行數 | 2105 | 2004（差 101 行） |
| 檔案大小 | 76k | 73k |
| `_auto_stage_completion_files` | 有（line 1681） | 無（grep count = 0） |
| 修改時間 | May 19 00:14 | May 18 22:44 |

**thyme W11-035 self-validation 為何成功**：thyme 的測試（`test_complete_auto_stage.py`）使用 `sys.path` 直接指向 source 目錄，繞過了安裝版本。測試通過 = source 正確，但 CLI binary 使用的是 stale 安裝版。

**修復方式**：執行 `uv tool install /path/to/.claude/skills/ticket --force --reinstall` 後驗證 auto-stage 正常運作（已驗證成功）。

### 防護建議

此為已知模式：修改 ticket skill source 後忘記同步 `uv tool install`。建議：

1. W11-035 的 Solution 或 SKILL.md 應註明「修改 lifecycle.py 後須執行 `uv tool install --reinstall`」
2. 或建立 post-commit hook 自動偵測 `.claude/skills/ticket/ticket_system/` 變更時提醒重新安裝

### 結論

根因：環境差異（installed package stale）。非 source bug，非操作差異。

W11-035 的 auto-stage 實作本身正確，但 commit 流程未含 `uv tool install --reinstall`，導致 CLI binary 使用 stale installed 版本。thyme self-validation 通過是因為測試直接使用 source（sys.path），不經 installed package。
