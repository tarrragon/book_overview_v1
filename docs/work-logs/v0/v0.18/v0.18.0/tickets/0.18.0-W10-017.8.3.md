## Solution

實作 metrics log 多份 rotate（TD3）：

**檔案變更**（`checkpoint_state.py`，+26 行 / -4 行）：
- 新增常數 `_METRICS_LOG_ROTATE_KEEP = 3`（可配置保留份數）
- 新增 `_rotate_metrics_log(log_path, keep)` 函式：從最舊往新滾動（i = keep → 2），避免覆蓋未搬移檔
  - `.{keep-1}.jsonl` → `.{keep}.jsonl`（舊 `.{keep}` 被覆蓋丟棄）
  - ...
  - `.1.jsonl` → `.2.jsonl`
  - 原檔 → `.1.jsonl`
- `_write_metrics_log` 調用 `_rotate_metrics_log(log_path, keep=_METRICS_LOG_ROTATE_KEEP)` 取代原 inline 邏輯
- Docstring 更新：描述多份保留語意

**測試變更**（`test_observability_log.py`，+28 行 / -9 行）：
- `test_D5b_rotate_replaces_old_backup` 改名 `test_D5b_rotate_keeps_multiple_backups`
- 驗證 4 次連續 rotate 後：`.1`=gen4、`.2`=gen3、`.3`=gen2、`.4` 不存在

**設計要點**：
- 抽出 `_rotate_metrics_log` 讓主函式認知負擔不上升
- 從最舊往新滾動（reverse order）確保不覆蓋尚未搬移的檔
- `keep` 參數化便於未來調整或測試注入
