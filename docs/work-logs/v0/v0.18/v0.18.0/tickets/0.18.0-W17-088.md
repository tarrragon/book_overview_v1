## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題描述

W17-087 將 basil-writing-critic 從手抄規則改為 @-import 載入 7 份規範，動機是 DRY。檢視後發現每次派發 auto-load 達 2230 行，其中 1587 行為情境特化 references（writing-documents 424 / writing-articles 704 / writing-code-comments 459），一次審查只用得到一份。

### 問題根因

DRY 設計被誤解為「全載入」。compositional-writing/SKILL.md 自身設計為 progressive disclosure 入口（含 references 索引），由使用者依情境讀對應 reference。v3 將三份 references 直接 @-import 違反此設計，把 SKILL 的索引機制空轉。

### 影響範圍

- .claude/agents/basil-writing-critic.md 每次派發 token 預算被佔用約 1500 行無關內容
- 違反 SKILL.md 自身設計，未來其他 agent 若仿效會擴大誤用
### 問題根因

basil v3（W17-087）將 7 份規範改用 @-import 全載入：AGENT_PRELOAD + document-writing-style + language-constraints + compositional-writing/SKILL.md + writing-documents + writing-articles + writing-code-comments。三份情境特化 references（writing-documents 424 行 / writing-articles ~530 行 / writing-code-comments 459 行）每次審查任務只用其中一份，卻全部預載。

### 影響範圍

- `.claude/agents/basil-writing-critic.md`：7 份 @-import 導致 auto-load 約 2230 行，超出 compositional-writing/SKILL.md 自身 progressive disclosure 設計原則（SKILL.md 觸發路由明示：審查前依文件類型讀對應 reference，非全部預載）。

### 修正方向

移除三份情境特化 references 的 @-import，改在「核心職責」段落加入「文件類型 → reference Read 指令」對照表，讓 agent 執行時依任務類型按需 Read 對應 reference。保留 4 份必要 @-import：AGENT_PRELOAD / document-writing-style / language-constraints / compositional-writing/SKILL.md。
