### 問題根因

`framework-rule-edit-skill-trigger-hook` 在 strict / warn 兩個分支（line 295-302）僅 log 路徑，無編輯體量資訊。W17-127.3 ANA 發現現有 log 無法區分三類誤報：

1. **typo 修正**（< 5 行 / size delta < 50 bytes）— 通常不需重讀 SKILL
2. **格式調整**（< 20 行 / size delta < 500 bytes）— 視情況
3. **語意修訂**（≥ 20 行 / size delta ≥ 500 bytes）— 應重讀 SKILL

無 size / line count 資料時，後續 strict mode 重評（W17-198）無法量化誤報分類。

### 影響範圍

- `.claude/hooks/framework-rule-edit-skill-trigger-hook.py`（單一模組，line 295-302 兩處 log）
- `.claude/hooks/tests/test_framework_rule_edit_skill_trigger_hook.py`（新增測試覆蓋 size / diff 計算）

### 實作策略

**1. 新增 helper `_compute_edit_metrics(tool_name, tool_input, file_path)`**

回傳 `(file_size_before, file_size_after, diff_line_count)` 三元組：

- **Edit**：
  - `file_size_before` = `os.path.getsize(file_path)` 若檔存在，否則 0
  - `old = tool_input["old_string"]`、`new = tool_input["new_string"]`
  - `file_size_after` = `file_size_before - len(old.encode('utf-8')) + len(new.encode('utf-8'))`（估算）
  - `diff_line_count` = `abs(new.count("\n") - old.count("\n"))`，並 `+1` 修正單行修改情況（若 old != new 但行數相同，至少算 1 行）

- **Write**：
  - 若檔存在：`file_size_before` = 既有檔 size，讀取舊內容算 `old_lines = old_content.count("\n") + 1`
  - 若新檔：`file_size_before` = 0、`old_lines` = 0
  - `content = tool_input["content"]`
  - `file_size_after` = `len(content.encode('utf-8'))`
  - `diff_line_count` = `abs(content.count("\n") + 1 - old_lines)`

- 任何 IO/解析錯誤一律回傳 `(0, 0, 0)`，不破壞主流程

**2. 在 strict / warn 分支補 log（line 297 / 301）**

```python
metrics = _compute_edit_metrics(tool_name, tool_input, file_path)
logger.info(
    "edit metrics: path=%s tool=%s file_size_before=%d file_size_after=%d diff_line_count=%d",
    rel_path, tool_name, *metrics,
)
```

兩處（strict 與 warn）共用同一行 log，緊鄰原本的 `logger.info("strict/警告 ...")`。

**3. 測試新增**

於 `.claude/hooks/tests/test_framework_rule_edit_skill_trigger_hook.py` 新增：

- `test_edit_metrics_small_typo`：Edit 工具，old="foo"、new="fo0"，斷言 diff_line_count=1、size delta=0
- `test_edit_metrics_multiline_revision`：Edit 工具，多行替換，斷言 diff_line_count > 1
- `test_write_metrics_new_file`：Write 工具新檔，斷言 file_size_before=0、file_size_after=len(content)
- `test_write_metrics_overwrite`：Write 工具覆寫既有檔，斷言 file_size_before > 0、file_size_after = len(new content)
- `test_metrics_io_error_returns_zero`：構造不存在路徑，斷言三元組為 (0, 0, 0) 不拋例外

- **不修改** WARN_MESSAGE / STRICT_MESSAGE 輸出格式（用戶端訊息不變）

### 實作摘要

新增 `_compute_edit_metrics(tool_name, tool_input, file_path)` helper，於 strict / warn 兩分支同步 emit 一行 INFO `edit metrics: ...` log（含 path / tool / file_size_before / file_size_after / diff_line_count），協助 W17-198 dry-run 期間誤報分類（typo / 格式 / 語意修訂）。

### 設計決策

- Edit 分支：`file_size_before` 透過 `os.path.getsize` 取得；`file_size_after` 以 `before - len(old.utf8) + len(new.utf8)` 估算；`diff_line_count` 為換行數絕對差，單行同行數修改保底 1 行。
- Write 分支：新檔 `before=0`；覆寫時讀取既有檔計算 size_before 與 old_lines；`after` 以 `len(content.utf8)` 計算。
- 錯誤策略：任何 IO / 解析錯誤回傳 `(0, 0, 0)`，metrics 不可破壞主流程（與既有 hook fail-open 哲學一致）。

- trigger 條件、WARN / STRICT 訊息文字、exit code 行為均未變更
- framework 規則文件未動（純 hook 改善）
