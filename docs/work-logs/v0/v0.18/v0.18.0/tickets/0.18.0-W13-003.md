**修改檔案**：
1. .claude/hooks/askuserquestion-charset-guard-hook.py

   - 14 個測試，涵蓋 SIMPLIFIED_CHARS 清單、find_violations 行為、scan_payload 整合

### 設計決策

- **僅加 2 字元**：本 Ticket 嚴守範圍，只處理本 session 實證的『隶/遗』
- **遵循 PC-074**：確認兩字元在現代繁體中極罕用（繁體用『隸/遺』）；未從其他常見簡體詞拆字
- **Hook 邏輯不動**：EMOJI_RANGES、main() 流程、stderr 格式維持原狀
- **測試涵蓋**：
  - W13-003 新字元確實在清單（positive）
  - 繁體對應字不在清單（負向保障）
  - 既有字元與 emoji 回歸保障
  - 空陣列 / 畸形資料不崩潰

### 後續升級路徑（非本 Ticket 範圍）

本方案屬清單式防護，PC-072 v1.1.0 已標註「清單無法窮盡」為最可能失敗模式。長期升級路徑為「通用 CJK 簡繁體偵測（opencc 或字典比對）」，待本方案驗證 2+ session 後評估升級必要性。
