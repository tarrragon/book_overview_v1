## Solution

依父 Ticket `0.18.0-W11-001.1.md` Phase 3a 虛擬碼實作執行層 5 函式於
`.claude/skills/ticket/ticket_system/commands/claim_verification.py`：

1. **`apply_parse_strategy(strategy, stdout, stderr, exit_code, ac_text)`**
   - 4 種策略：`exit_code` / `tail_lines` / `coverage_number` / `manual`。
   - `tail_lines`：掃描 stdout 最後 5 行，regex `(\d+)\s+failed` 命中且 N>0 → failed。
   - `coverage_number`：AC 文字抽 `(\d+)\s*%` 閾值（fallback 80），stdout 抽實測；
     實測 >= 閾值 → passed，未抽取到實測 → failed。
   - `manual` → unverifiable；未知策略防禦性回 failed。

2. **`execute_verification(ac, vc, cwd)`**
   - `Popen(shell=True, cwd=cwd, start_new_session=True, stdout/stderr=PIPE)`，
     `env` 省略 → 繼承父 process（I3 驗證）。
   - `communicate(timeout=vc.timeout_sec)` 捕 `TimeoutExpired` → `_kill_process_group` → status='timeout'。
   - 捕 `KeyboardInterrupt` → killpg 後 re-raise（E5 驗證）。
   - 捕 `OSError`（含 FileNotFoundError）→ status='env_error'（E6 驗證）。
   - stdout/stderr bytes 透過 `_decode_safe(errors='replace')` 解碼（J2 驗證）。

3. **`_kill_process_group(pid)`**
   - 查 `os.getpgid(pid)` → `SIGTERM` → `time.sleep(1)` → `SIGKILL`；
     任一階段遇 `ProcessLookupError` 靜默略過。E4a 驗證 SIGTERM/SIGKILL 順序。

4. **`run_all_verifications(pairs, cwd)`**
   - 三路分流：`vc is None` → no_template（E2）；`is_verifiable=False` → unverifiable（E3）；
     其餘進 `execute_verification`。進度訊息寫 stderr 避免污染 stdout 摘要。

5. **`collect_ac_verifications(ticket_id)`**
   - 委派 `parse_ac` + `match_template`，產生 `(AC, VC|None)` 配對。
     YAML 損毀 / 無效 ticket_id 時 ValueError 由 parse_ac 自然傳播。

**命名常數化**：tail_lines 掃描窗口（`_TAIL_SCAN_LINES=5`）、coverage fallback
（`_COVERAGE_FALLBACK_THRESHOLD=80`）、SIGTERM→SIGKILL 等待（`_SIGTERM_SIGKILL_DELAY_SEC=1`）
皆為模組常數，消除魔法數字。
