## 實作摘要

建立 `scripts/portability-check.sh`，掃描指定 Skill 目錄中所有檔案（.md/.yaml/.yml/.json/.txt），偵測 7 類禁用模式：

| 模式名稱 | 偵測目標 |
|---------|---------|
| framework-rules-path | `.claude/rules/` 路徑引用 |
| framework-methodologies-path | `.claude/methodologies/` 路徑引用 |
| framework-hooks-path | `.claude/hooks/` 路徑引用 |
| project-ticket-id | `0.17.x-W*` / `0.18.x-W*` / PROP-* / ANA-* / IMP-* / PC-* / ARCH-* |
| project-commit-hash | 7-40 hex（含 Unicode codepoint / 版本號二次過濾） |
| project-worklog-path | `docs/work-logs/` |
| framework-hook-name | skill-description-length-check / doc-sync-check / branch-verify-hook 等 |

## 設計要點

1. **退出碼語義**：0 = 通過，1 = 有違規，2 = 目錄不存在
2. **違規報告格式**：`file:lineno:pattern_name: matched_content`，雙通道輸出（違規送 stderr、成功訊息送 stdout）
3. **誤判防護**：commit-hash 正則做二次過濾（排除 `U+XXXX` Unicode codepoint、排除明顯版本號 `X.Y.Z` 上下文）
4. **chmod +x** 已設定（PC-086）
