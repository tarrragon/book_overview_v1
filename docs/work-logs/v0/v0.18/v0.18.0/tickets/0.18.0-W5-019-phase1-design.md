# Phase 1 功能規格設計：父子 cascade 與 children 警告

## 1. 功能目的

補齊 ticket CLI 父子生命週期的語義完整性：當父 Ticket `complete` 時，
（A）對 blocked 且僅因本父阻塞的 children 自動解鎖為 pending；
（B）對尚未 completed 的 children 輸出結構化警告，但**不阻止** complete 執行。

解決 W5-015 ANA 結論識別的「子孤兒 blocked 無人解鎖」缺陷。

---

## 2. 功能邊界

### 2.1 觸發條件

cascade/警告邏輯在 `complete(ticket_id)` 流程中**所有既有驗證通過後**（狀態可完成 + acceptance 全勾 + 狀態與 `completed_at` 已寫入並 save 完成後）觸發。

即：本功能是 `complete` 成功後的「後置副作用」，不影響 complete 成功/失敗判定。

### 2.2 作用範圍

| 物件 | 處理 |
|------|------|
| 當前 complete 的父 ticket（`P`） | 不變 |
| `P.children` 中 status=blocked 且 `blockedBy` 不含其他未完成 ticket 的子 | cascade 解鎖（status → pending） |
| `P.children` 中 status=blocked 但 `blockedBy` 尚有其他未完成項 | 保留 blocked，但在警告列表中列出 |
| `P.children` 中 status=pending / in_progress | 不改動狀態，列入警告 |
| `P.children` 中 status=completed / closed | 不改動，不警告 |
| 孫子（children 的 children） | 不遞迴處理（本版本不支援多層 cascade，見 §6.1） |

### 2.3 不觸發情境

- 當前 ticket 無 children（`children=[]` 或缺欄位）→ 跳過 cascade 與警告
- complete 因驗證失敗而 return 1 時 → 不執行 cascade/警告
- `close` 命令（與 complete 不同路徑）→ 本版本不處理

---

## 3. 輸入/輸出契約

### 3.1 輸入

- `ticket_id`: 被 complete 的父 ticket ID
- 隱含輸入：當前版本所有 tickets（透過 `list_tickets(self.version)` 載入，用於查詢 children 與 blockedBy 狀態）

### 3.2 輸出 — 標準情境

complete 原有輸出（`[INFO] TICKET_COMPLETED`、worklog 追加、驗收提示、next-steps）保持不變。

在「完成操作」與「驗收提示」之間新增 cascade/警告區塊（若有可報告內容）。

### 3.3 輸出 — Cascade 成功訊息

當有 children 被自動解鎖時，輸出：

```
[Cascade] 以下子 Ticket 已自動解鎖（blocked → pending）：
   - {child_id}: {child_title}
   - {child_id}: {child_title}
```

若無子被解鎖，則不輸出此區塊。

### 3.4 輸出 — Children 警告訊息

當有 children 未完成（含 pending/in_progress/blocked-with-other-deps）時，輸出：

```
[Warning] 父 Ticket 完成時尚有未完成的子 Ticket：
   - {child_id} [{status}]: {child_title}
   - {child_id} [{status}]: {child_title}
   （提示：父 complete 不阻止，但建議確認子任務是否需要獨立推進或 close）
```

若無未完成 children，則不輸出此區塊。

注意：已被 cascade 解鎖的 children **不**同時列入警告（避免資訊衝突，解鎖後狀態為 pending，但對本次 complete 語義上已處理）。

### 3.5 返回值

`complete` 的返回值**不因 cascade/警告改變**：驗證通過則永遠回傳 0。cascade 過程內部錯誤（如 save 失敗）以 warning 輸出，不影響父 complete 的成功結論。

## 4. 使用場景（Given-When-Then）

### 場景 1：正常 cascade — 唯一 blocked 子自動解鎖

```
Given:
  - 父 P0（status=in_progress, children=[C1]）
  - 子 C1（status=blocked, blockedBy=[P0], parent_id=P0）
  - P0 的 acceptance 全部勾選

When:
  執行 `ticket track complete P0`

Then:
  - P0.status = completed, completed_at 有值
  - C1.status = pending（自動解鎖）
  - stdout 含 "[Cascade] 以下子 Ticket 已自動解鎖"
  - stdout 含 "C1" 條目
  - stdout 不含 "[Warning] 父 Ticket 完成時尚有未完成的子 Ticket"（C1 已解鎖）
  - 返回 0
```

### 場景 2：警告但不阻止 — 有 pending/in_progress children

```
Given:
  - 父 P0（children=[C1, C2]）
  - C1（status=pending）
  - C2（status=in_progress）
  - P0 的 acceptance 全部勾選

When:
  執行 `ticket track complete P0`

Then:
  - P0.status = completed
  - C1.status = pending（不變）
  - C2.status = in_progress（不變）
  - stdout 含 "[Warning] 父 Ticket 完成時尚有未完成的子 Ticket"
  - 警告列表含 "C1 [pending]" 和 "C2 [in_progress]"
  - 不含 "[Cascade]" 區塊
  - 返回 0
```

### 場景 3：其他依賴保留 blocked — blockedBy 尚有未完成項

```
Given:
  - 父 P0（children=[C1]）
  - 另一 Ticket X（status=in_progress, 非 P0 的 child）
  - C1（status=blocked, blockedBy=[P0, X], parent_id=P0）

When:
  執行 `ticket track complete P0`

Then:
  - P0.status = completed
  - C1.status = blocked（保留，因 X 仍未完成）
  - stdout 含 "[Warning]" 區塊
  - 警告列表含 "C1 [blocked]"（附提示可檢視 blockedBy）
  - 不含 "[Cascade]" 區塊
  - 返回 0
```

## 6. 邊界情境

### 6.1 孫子孤兒（多層 parent）

**情境**：P0 → C1（blocked）→ GC1（blocked, parent_id=C1）。P0 complete 後 C1 解鎖為 pending，但 GC1 仍 blocked。

**本版本處理**：不遞迴。GC1 將在 C1 自己 complete 時透過同樣邏輯解鎖。

**理由**：cascade 只發生在「當前被 complete 的 ticket 的直接 children」層級，避免跨層級的副作用擴散。

### 6.2 多層 parent_id（間接父子）

**情境**：若未來出現 `C1.parent_id=P0` 但 C1 不在 `P0.children` 中（資料不一致），本邏輯**以 `P0.children` 為權威來源**，不掃描全版本 ticket 的 parent_id 欄位。

**理由**：避免雙向資料來源衝突，維持單一真實來源原則。

### 6.3 children 為空

`children=[]` 或缺欄位 → 直接跳過 cascade/警告區塊，無任何 stdout 新增。

### 6.4 全 completed 的 children

所有 children 都已 completed → 無 cascade 對象、無警告對象，無任何 stdout 新增。

### 6.5 children 中有找不到的 ID

若 `ticket_map.get(child_id) is None`（資料不一致）→ 跳過該項，不報錯（與既有 `_check_pending_children` 行為一致）。

### 6.6 children 含 closed 狀態

closed 視同 completed，不警告、不 cascade。

### 6.7 cascade 儲存失敗

若解鎖子 ticket 的 save_ticket 拋例外：
- 以 `[Warning]` 輸出失敗訊息與 child_id
- 不影響父 complete 的 return 0
- 其他 children 繼續嘗試處理（不 fail-fast）

### 6.8 blockedBy 語義

判斷「是否僅因本父阻塞」：
- child.blockedBy 為空 → 可解鎖（若 status=blocked，視為異常但仍解鎖）
- child.blockedBy == [P0] → 可解鎖
- child.blockedBy == [P0, X] 且 X.status != completed → 保留 blocked
- child.blockedBy == [P0, X] 且 X.status == completed → 可解鎖

## 7. 介面契約（供 Phase 3b 參考）

### 7.1 complete() 流程增補位置

在 `lifecycle.py` 第 489 行 `return 0` 之前、第 487 行 `_auto_handoff_if_needed(...)` 之後，新增 cascade 與警告處理。

或置於 `_print_next_steps(analysis)`（484）之後、`_auto_handoff_if_needed`（487）之前，以保持「狀態副作用優先於 handoff」的語義順序。**推薦放在 `_auto_handoff_if_needed` 之前**，因為 cascade 可能影響 handoff 的建議內容。

### 7.2 建議新增 helper 函式

```python
def _cascade_unblock_children(
    parent_ticket: Dict[str, Any],
    version: str,
) -> Tuple[List[Dict[str, Any]], List[Dict[str, Any]]]:
    """
    對 parent 的 blocked children 執行 cascade 解鎖，並收集未完成 children 清單。

    Returns:
        (unblocked_children, pending_children)
        - unblocked_children: 被解鎖的 child 資料（含 id, title）
        - pending_children: 未完成但未解鎖的 child 資料（含 id, title, status）
    """
```

職責：
1. 載入當前版本所有 ticket，建立 ticket_map
2. 遍歷 parent.children
3. 對每個 child 判斷（依 §6.8 規則）是否 cascade
4. 對可 cascade 的 child：更新 status=pending，save_ticket
5. 回傳兩份清單供 complete 格式化輸出

### 7.3 輸出格式化

建議兩個印出函式或直接 inline：
- `_print_cascade_unblocked(unblocked_list)` — 對應 §3.3
- `_print_children_warnings(pending_list)` — 對應 §3.4

## 8. 設計決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| cascade 時機 | complete 成功後（非事前檢查） | complete 語義是「我已完成」，事後解鎖是結果不是前置條件 |
| 警告等級 | Warning 非 Error | WRAP 選擇 B+E，警告不阻止，尊重使用者判斷 |
| 遞迴深度 | 只處理直接 children | 避免跨層副作用，每層 complete 自己觸發 |
| blockedBy 語義 | 視為 AND（全 completed 才解鎖） | 與既有 `_check_unblocked_tickets` 一致 |
| cascade 失敗處理 | 記錄 warning 不中斷 | 父 complete 已成功，不因子 save 失敗回滾 |
| 資料權威來源 | parent.children 欄位 | 單一來源，不掃描 parent_id 反查 |

## 9. 潛在風險

| 風險 | 影響 | 緩解 |
|------|------|------|
| cascade 解鎖後 child 被其他機制再次 block | 狀態震盪 | 本版本不處理，依賴人工監控；未來可加冪等檢查 |
| children 清單很大（>50） | 輸出過長 | 若 >10 項，截斷顯示前 10 + "... 共 N 項" |
| 版本內 ticket 載入效能 | complete 變慢 | 既有 `_analyze_next_steps` 已載入 all_tickets，可複用 |
| 與 close 命令不一致 | 語義不完整 | 本 Ticket 範圍僅 complete，close 未來另開 Ticket |
| W5-014 回溯 | 需手動觸發 | 實作完成後 re-complete W5-013 或手動調 W5-014 status |
