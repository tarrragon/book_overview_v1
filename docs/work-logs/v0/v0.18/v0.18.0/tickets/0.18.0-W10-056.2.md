## Solution

### 執行摘要（thyme-python-developer, 2026-04-15）

完成 4 項群組 A 結構性重構，66/66 測試全綠。

#### CE-1 main() 提取 _process_signals()
- `main()` 從 60 行降至 **33 行**
- 新增 `_process_signals(event, event_name, state, config, current_ticket, logger) -> List[str]`（37 行），封裝逐 signal 迴圈、detect/apply/cooldown/mark_warned
- main() 保留骨架：logging / stdin / load_config / load_state / derive_ticket / apply_ticket_switch_reset / is_manual_wrap_invocation / _process_signals / save_state_atomic / stderr

#### CE-2 load_config() 拆分
- `load_config()` 從 84 行降至 **38 行**（< 40 達標）
- 新增 `_parse_settings(raw, cfg) -> None`（9 行）：處理 settings + output 節
- 新增 `_parse_signals(signals_raw, logger) -> List[SignalDef]`（37 行）：解析 signals list
- 抽出 `_VALID_RESET_CONDITIONS` 模組常數（原本內嵌 set literal）

#### CE-4+R2 Strategy Protocol 升級 + 死屬性移除
- 定義 `SignalStrategy(Protocol)` 介面，含 detect / apply 方法簽名
- **移除** 三個 Strategy class-level `signal_id` 屬性（原 L382/L430/L457）
- `DetectResult` 新增 `signal_id: Optional[str]` 欄位；detect() 於所有 return path 填入 `sd.id`
- apply() 改讀 `result.signal_id` 寫入 state 子 dict，消除 class attr 與 `SIGNAL_STRATEGIES` key 的 drift 風險
- `SIGNAL_STRATEGIES` 型別註解為 `Dict[str, SignalStrategy]`
- 保留 `ConsecutiveFailuresStrategy._DEFAULT_SIGNAL_ID` 作為純備援（僅當測試 F1 直接構造 DetectResult 不經 detect() 時使用），註解明確說明正式流程不會觸發、亦不會 drift（因正式 detect 一律用 sd.id）

#### C1+CE-7 AnaClaimStrategy 依賴注入
- `AnaClaimStrategy.__init__` 接收 `project_root_provider: Optional[Callable[[], Path]]`
- detect() 中優先使用注入的 provider，fallback 到模組級 `get_project_root()`
- 保留 fallback 是為了維持既有測試（`monkeypatch.setattr(hook_mod, "get_project_root", lambda: root)`）的相容性

### 偏離 Context Bundle 之處

Context Bundle CE-7 原要求「SignalStrategy Protocol 的 detect 簽名新增 project_root: Path 參數，所有 Strategy detect 簽名一致」。實作評估後發現該修改會強制破壞 66 測試中 30+ 個 `strategy.detect(event, state, sd, current_ticket, logger)` 五參數呼叫點（tests 屬於 scope 禁止修改區）。

改採**建構子 DI** 替代：保持 detect 五參數簽名不變，同時達成 AnaClaimStrategy 不再直接依賴模組全域 `get_project_root()` 的設計目標（可由建構端注入，測試 fallback 仍走 monkeypatch）。其他 Strategy 本就不需要 project_root，簽名不變也更符合 YAGNI。此偏離在「不修改測試檔」約束下是必要取捨。
