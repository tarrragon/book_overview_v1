## Solution

### 實作摘要（2026-04-14）

於 `.claude/hooks/acceptance_checkers/children_checker.py` 擴充父 Ticket complete
前置檢查邏輯，並新建 `.claude/hooks/tests/test_acceptance_gate_hook.py` 覆蓋 AC 5
要求的四情境（含遞迴孫層案例）。

### 核心變更

1. **終止狀態含 closed（AC 2）**：新增 `TERMINAL_STATUSES = {"completed", "closed"}`
   常數，取代原本只比對 `"completed"` 的硬判斷，使 `closed` 狀態的子 Ticket 視為
   有效終態。

2. **遞迴檢查所有後代（AC 1）**：新增 `_collect_incomplete_descendants()`，
   對每一層 child 無論自身狀態均往下展開，避免「子 completed、孫 pending」
   漏檢情境。使用 `visited` 集合防止循環引用造成無限遞迴。

3. **錯誤訊息列出未完成後代（AC 3）**：保留既有 `GateMessages.CHILDREN_INCOMPLETE_ERROR`
   訊息格式，將所有層級的 incomplete 後代合併列出（`  - id: title (status: xxx)`）。

4. **Robust children 解析（防禦 parser 缺陷）**：新增 `_extract_children_robust()`
   和 `_read_children_from_file()`。背景：hook_utils 內建 YAML parser 無法解析
   頂層列表 `children:\n- id1`（回傳空字串）；新增的 fallback 直接從 frontmatter
   原始文字掃描，同時支援 flow-style `[a, b]` 和 block-style `- a / - b` 兩種
   格式。

5. **保留 warning 邏輯（AC 4）**：未變更 `verify_acceptance_record` 等既有
   warning checker；block 邏輯僅補齊，不取代 warning 層。

### 父子責任傳遞的程式化體現

| 哲學原則 | 程式行為 |
|---------|---------|
| 父責任是「問題被解決」非「報告寫完」 | 遞迴檢查孫層確保實作/驗證鏈完整 |
| 子完成是父責任履行的證據 | 任一後代非 completed/closed → block |
| block 訊息須指引行動 | 錯誤訊息列出所有未完成後代 ID + status |
