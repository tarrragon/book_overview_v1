### 問題根因

`get_tickets_dir()` 在 `.claude/skills/ticket/ticket_system/lib/paths.py` 行 99-104 使用存在性檢查決定回傳三層或 flat 路徑。對未存在的主版本（v1+, v2+），三層目錄不存在導致 fallback 到 flat 結構。

### 影響範圍

- `.claude/skills/ticket/ticket_system/lib/paths.py`：`get_tickets_dir()` 函式
- 所有呼叫 `get_tickets_dir()` 的寫入操作（migrate, create 等）在新主版本首次使用時受影響
- 讀取端不受影響（已存在的目錄結構讀取正常）

### 修復策略

移除行 99-104 的存在性檢查。當 `len(parts) >= 2` 時一律回傳三層階層路徑，不論目錄是否已存在。flat 結構僅作為版本號無法解析時的最終 fallback。

### 測試案例

- v2.0.0 → `v2/v2.0/v2.0.0/tickets/`
- v10.3.1 → `v10/v10.3/v10.3.1/tickets/`
- v0.18.0 → `v0/v0.18/v0.18.0/tickets/`（既有行為不變）
- 單段版本號 "1" → flat fallback（邊界）

### 來源 ANA

0.18.0-W14-051 Solution 章節含完整根因分析與修復方案。

### 修復摘要

1. **paths.py get_tickets_dir()**：移除存在性檢查（原行 99-104），`len(parts) >= 2` 時一律回傳三層階層路徑；flat 結構僅作為版本字串無法解析 major.minor 時的最終 fallback。
2. **migrate.py _update_cross_references()**：原 glob 模式 `v*/tickets` 僅匹配 flat 結構，新增 `v*/v*/v*/tickets` glob 並合併 set 去重，同時支援 flat 與三層佈局。
