## Problem Analysis

Chrome 載入擴充功能後出現兩個錯誤：
1. Service worker registration failed. Status code: 15 — background.js 無法註冊為 Service Worker
2. Uncaught SyntaxError: Cannot use import statement outside a module — ES module import 語法在非 module context 中執行

Status code 15 = MANIFEST_ERROR_PAGE_IN_CONTENT_SCRIPTS_ONLY，通常表示 manifest 配置問題。
import 錯誤表示 background.js 使用了 ES module 語法但未在 manifest 中宣告 type: module。
[2026-03-28 14:03 更新] 根本原因：專案模組系統不一致。Logger.js 等核心模組使用 CommonJS (module.exports)，但 background.js 使用 ES module (import)。Chrome Extension Service Worker 以 type:module 載入時，無法解析 CJS export。需要統一模組系統或在 build 階段轉換。
[2026-03-28 14:20 更新] 決策：採用方案 B — build bundling。根因：build script 只做檔案複製不做打包，CJS require 在 Chrome Extension 環境不可用。需在 build 階段用 esbuild/rollup 打包 background.js 及其依賴鏈。同時需評估 content script 和 popup 是否也有同樣問題。

## Solution

<!-- To be filled by executing agent -->

---

分 4 個子任務解決 Chrome Extension 載入失敗：
.1: manifest.json 新增 type: module (534ef02)
.2: background.js import 改為相對路徑 (137a376)
.3: Logger.js/MessageDictionary.js 新增 ESM export (ab9dabe)
.4: build script 加入 esbuild bundling 三入口點 (e196b3b)

最終方案：esbuild 在 build 階段將三個入口點 (background.js, content-modular.js, popup.js) 及其 CJS 依賴鏈打包為 IIFE。Service Worker 成功註冊，但仍有 runtime 錯誤（storage API、常數未定義等）留給後續版本。
