# v0.15.0 版本工作日誌

**版本號**: v0.15.0
**開始日期**: 2026-03-26
**目標**: 專案重啟與品質整理（暫停半年後恢復開發）
**狀態**: 已完成

---

## 前情提要

專案自 2025-09（v0.14.2）暫停約半年。重啟後需要全面評估專案現況、修復累積的測試問題、清理技術債務，為後續功能開發建立健康基礎。

---

## 開發日誌

### 2026-03-26：版本開始 — 專案重啟評估

- W1-001 對 ErrorCodes 系統進行設計分析，評估半年未維護的影響
- W1-002 全面修復測試錯誤，恢復編譯與測試通過
- W1-003 推進版本號至 v0.15.0

### 2026-03-26：W2 — 深度分析與環境清理

- W2-001 多視角分析測試失敗模式與系統設計檢討
- W2-002/003 清理 __pycache__ 從 git 追蹤並加入 .gitignore

### 2026-03-27：W3 — 核心重構（12 個 Ticket）

這是本版本最大的一波，進行了四項核心重構：
- Logger 統一為實例模式（W3-001, W3-003）
- 46 個 toThrow ESLint 違規轉換（W3-002）
- ErrorCodes 分層重構：316 個常數精簡為核心層+域層，按域分檔（W3-005, W3-009, W3-010）
- StandardError 遺留體系清理，移除 14 個 UC 適配器（W3-006）
- manifest/package.json 版本同步機制（W3-007）
- 集中 Chrome Mock 工廠建立（W3-004）

### 2026-03-27：W4 — 效能與文件

- W4-002 效能測試閾值配置化，支援 CI/local 環境分級
- W4-003 撰寫 API 風格指南文件（class vs factory、錯誤處理、Logger、命名規範）

### 2026-03-28：W5 — 收尾修復

- W5-001 修復全量測試 flaky test 測試間干擾問題
- W5-002 修復 ticket CLI _extract_who_name NoneType bug
- W5-003 修復 UC02ErrorAdapter 效能測試 flaky test
- 191 個 skip 測試重啟任務遷移至 v0.16.0

---

## 關鍵決策

- **Logger 統一**：從 3 套 API 統一為 1 套實例模式，降低認知負擔
- **ErrorCodes 分層**：核心層+域層設計，取代過去 316 個扁平常數
- **skip 測試遷移**：191 個 skip 測試不在本版處理，遷移到 v0.16.0 專門處理
