# Execution Log

## Task Summary

修復 Agent worktree 完成後主線程 Shell 工作目錄被污染到 worktree 路徑

---

## Problem Analysis

### 問題根因

**Claude Code 平台行為**：當使用 `Agent(isolation: "worktree")` 派發代理人時，Claude Code 內部會透過 `EnterWorktree` 將主線程的 Bash Shell 工作目錄切換到 worktree 路徑（如 `.claude/worktrees/agent-a0ddc09d/`）。代理人完成後，`ExitWorktree` 未能可靠地將 Shell cwd 還原到專案根目錄。

**關鍵排除**：所有 hooks 都是 Python 子程序（subprocess），`os.chdir()` 只影響子程序自身，無法改變父 shell 的 cwd。因此根因不在 hooks，而是 Claude Code 平台的 worktree isolation 實作。

**調查結果**：
1. `prompt-submit-hook.py` 第 452 行有 `os.chdir(project_root)`，但這是子程序內部的操作，不影響父 shell
2. `agent-commit-verification-hook.py` 使用 `cwd=project_root` 參數執行 git，不會改變 shell cwd
3. `worktree-auto-commit-hook.py` 使用 `Path.cwd()` 和預設 cwd，但同樣是子程序
4. 所有 PostToolUse (Bash) hooks 都是獨立子程序，無法污染父 shell

### 影響範圍

- 主線程在 worktree 代理人完成後的所有 Bash 操作（git status/add/commit 等）
- 會導致 `git add` 找不到檔案（pathspec did not match）
- `git status` 顯示在 worktree 分支而非 main

### 相關 Error Pattern

- IMP-008（Bash 工作目錄污染）— 類似模式但根因不同（IMP-008 是 hook 內 cd，此問題是平台行為）

---

## Solution

### 緩解方案（已實作）

由於根因是 Claude Code 平台行為，無法從 hooks 層面直接修復。採取**PostToolUse (Agent) hook 提醒**方案：

**修改檔案**：`.claude/hooks/agent-commit-verification-hook.py`

**修改內容**：
1. 在 Agent 完成後的 `additionalContext` 中，一律附加工作目錄還原提醒
2. 提醒訊息包含具體的 `cd` 指令和專案根目錄路徑
3. 使用 `get_project_root()`（優先讀取 `CLAUDE_PROJECT_DIR` 環境變數）確保路徑正確

**提醒訊息格式**：
```
[Agent 完成 - 工作目錄還原提醒]
Worktree 代理人完成後，主線程的 Bash Shell 工作目錄可能被污染到 worktree 路徑。
[強制] 執行下一個 Bash 命令前，先確認並還原工作目錄：
  cd /path/to/project && pwd && git branch --show-current
```

### 為何不能根本修復

- `EnterWorktree`/`ExitWorktree` 是 Claude Code 內建工具，非本專案可控
- Hooks 作為子程序無法改變父 shell 的 cwd
- `additionalContext` 提醒是目前最可行的防護措施

## Test Results

手動測試：使用 mock JSON 輸入驗證 hook 輸出正確包含 cwd 還原提醒。
Hook 輸出的 `additionalContext` 包含兩部分：
1. 未 commit 檔案警告（如有）
2. 工作目錄還原提醒（一律附加）

路徑解析正確：`get_project_root()` 透過 `CLAUDE_PROJECT_DIR` 環境變數正確解析為主專案路徑。
