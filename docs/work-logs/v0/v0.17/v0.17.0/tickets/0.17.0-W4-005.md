## 架構決策：TDD references 目錄重構

### 決策
每個 Phase 獨立子資料夾，rules 和 cases 分離。

### 結構
```
references/
  phase1/
    rules.md        ← decision questions + 指引（精簡）
    cases/           ← 真實案例（持續擴充）
      v0.17.0-spec-gaps.md
  phase2/
    rules.md
    cases/
      v0.17.0-test-blind-spots.md
  phase3/
    rules.md
    cases/
  phase4/
    rules.md         ← 含流程改善回饋環 workflow
    cases/
      v0.17.0-review-findings.md
```

### 執行要求
- W4-005.1/.2/.3 執行時需同步完成目錄重構
- 現有 phase{N}-*.md 內容拆分為 rules.md（指引）+ cases/（案例）
- SKILL.md 的路由引用更新為新路徑
- 代理人派發指令改為：Read references/phase{N}/rules.md

### 理由
- rules 保持精簡，代理人只讀規則不讀案例
- cases 無限擴充不影響 rules 篇幅
- Phase 4 檢討時追加案例只需在 cases/ 新增檔案
- 符合 progressive disclosure 和 supporting files 模式
