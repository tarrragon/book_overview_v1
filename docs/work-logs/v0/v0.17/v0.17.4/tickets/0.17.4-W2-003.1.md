## Problem Analysis

### Context Bundle

**問題**：Pre/PostToolUse Hook 的 stdout 必須是 JSON 格式（hookSpecificOutput），bare print() 導致 CLI 顯示 JSON validation failed。

**修復目標（4 個 Hook，7 處違規行）**：
1. bash-edit-guard-hook.py:99 — print(warning) → JSON stdout
2. pre-test-hook.py:122 — print(warning_text) → JSON stdout
3. version-release-guard-hook.py:171,176 — print(warning_msg) → JSON stdout
4. mcp-run-tests-validator.py:156,163,177 — print(error_message) → JSON stdout

**正確 JSON 格式範本**：
output = {"hookSpecificOutput": {"hookEventName": "PreToolUse", "additionalContext": message}}
print(json.dumps(output, ensure_ascii=False))

**或移至 stderr**（純 debug 訊息）：
print(message, file=sys.stderr)

**注意**：
- 需 import json（如未引入）
- hookEventName 依 Hook 類型使用 PreToolUse 或 PostToolUse
- 不要改變 Hook 的邏輯、exit code、或 stderr 輸出
- 環境有 chpwd zsh hook，禁止使用裸 cd 命令
