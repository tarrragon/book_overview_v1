### Context Bundle

**問題**：Pre/PostToolUse Hook 的 stdout 必須是 JSON 格式（hookSpecificOutput），bare print() 導致 CLI 顯示 JSON validation failed。

**修復目標（4 個 Hook，7 處違規行）**：
1. l10n-sync-verification-hook.py:240,243,259,272 — 4 處 bare print() → JSON stdout 或 stderr
2. file-type-permission-hook.py:83 — print(prompt) → JSON stdout
3. external-query-guide-hook.py:92 — print(guide_message) → JSON stdout
4. task-dispatch-readiness-check.py:242 — print(WARNING) → JSON stdout 或 stderr

**正確 JSON 格式範本**：
output = {"hookSpecificOutput": {"hookEventName": "PostToolUse", "additionalContext": message}}
print(json.dumps(output, ensure_ascii=False))

**或移至 stderr**（純 debug 訊息）：
print(message, file=sys.stderr)

**注意**：
- 需 import json（如未引入）
- hookEventName 依 Hook 類型使用 PreToolUse 或 PostToolUse
- 不要改變 Hook 的邏輯、exit code、或 stderr 輸出
- 環境有 chpwd zsh hook，禁止使用裸 cd 命令
