## Problem Analysis

### Context Bundle

**現有 Hook 架構**：version-consistency-guard-hook.py（429 行）
- SessionStart 時執行，非阻塞（exit 0）
- 現有兩個檢查：(1) 舊版本有未完成 Ticket (2) 版本號不匹配
- 使用 hook_utils 的 setup_hook_logging, run_hook_safely, get_project_root, parse_ticket_frontmatter, scan_ticket_files_by_version
- 無 yaml 依賴（dependencies = []），用 regex 解析 todolist.yaml

**新增檢查邏輯**：
1. 新增 get_registered_versions(project_root, logger) 函式
   - 用 regex 從 todolist.yaml 提取所有 version 欄位值
   - 回傳 set[str]（如 {'0.17.3', '0.17.4'}）

2. 新增 check_unregistered_worklog_versions(project_root, registered_versions, logger) 函式
   - 掃描 docs/work-logs/ 下所有版本目錄（支援階層結構 v{major}/v{major}.{minor}/v{version}/）
   - 比對每個目錄版本是否在 registered_versions 中
   - 回傳未註冊的版本列表

3. 新增 print_unregistered_warning(unregistered_versions) 函式
   - 印出 WARNING 訊息，列出未註冊版本
   - 提示用戶執行 /version-release start 或 /doc-flow worklog init

4. 在 main() 中呼叫（在 current_version 檢查之前，因為即使 current_version 無法讀取也應該檢查目錄）

**重要限制**：
- Hook 的 dependencies = []，不能用 pyyaml，必須用 regex 解析
- 支援階層目錄結構（v0/v0.17/v0.17.4/）和舊式平行結構（v0.17.4/）
- 輸出用 print()（SessionStart hook stdout 直接顯示）
- 不要用 cd 切換目錄（chpwd hook 干擾）
