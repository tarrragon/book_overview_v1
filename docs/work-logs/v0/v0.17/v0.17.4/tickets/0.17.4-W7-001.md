## Problem Analysis

### Context Bundle

**漏洞位置**：ticket_system/lib/version.py resolve_version() 第 193-226 行
- resolve_version() 呼叫 get_current_version()，後者有 Fallback 目錄掃描
- 不驗證版本是否在 todolist.yaml 中註冊

**修改計畫**：
1. version.py 新增 validate_version_registered(version: str) -> bool
   - 讀取 docs/todolist.yaml
   - 檢查版本是否存在於 versions 列表中
   - 檢查版本狀態是否為 active
   - todolist.yaml 不存在時 → 回傳 True（向後相容）

2. create.py execute() 第 869-872 行之間加入驗證呼叫
   - resolve_version 成功後呼叫 validate_version_registered
   - 驗證失敗 → 印出錯誤訊息提示用戶執行 /version-release start 或 /doc-flow worklog init
   - 錯誤訊息加入 ErrorMessages 常數

**測試要求**：
- 測試未註冊版本被拒絕
- 測試已註冊 active 版本通過
- 測試已註冊非 active 版本被拒絕
- 測試 todolist.yaml 不存在時向後相容（允許）
- 現有測試全部通過

**重要**：不要用 cd 切換目錄，用子 shell (cd ... && ...) 或 uv -d。此環境 zsh chpwd hook 會觸發 ls 輸出。
