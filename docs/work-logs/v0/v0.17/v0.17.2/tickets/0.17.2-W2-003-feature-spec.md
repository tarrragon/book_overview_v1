# 0.17.2-W2-003 功能規格：Tag Resolver 統一模組

## 1. Purpose（目的）

多個下游模組（FilterEngine、SearchEngine、BookDataExporter、Overview Page）都需要將 `tagId` 解析為 tag 名稱和 category 資訊。目前無統一解析模組，各消費端將自行實作重複邏輯。Tag Resolver 提供單一工廠函式 `createTagResolver`，注入 tag/category 查詢能力，回傳可被所有消費端共用的 resolver 物件。

**框架內建機制驗證（ARCH-010）**：JavaScript 無內建 tag 解析機制，Map 查找是標準做法，無需額外框架。本模組僅封裝 Map 查找邏輯和邊界處理，複雜度適當。

---

## 2. API Signatures（介面定義）

### 2.1 工廠函式：`createTagResolver`

```javascript
/**
 * 建立 Tag Resolver 實例
 *
 * @param {Object} deps - 依賴注入
 * @param {Function} deps.getTagById - (tagId: string) => Tag | null
 *   Tag 物件結構: { id: string, name: string, categoryId: string, isSystem: boolean }
 * @param {Function} deps.getCategoryById - (categoryId: string) => TagCategory | null
 *   TagCategory 物件結構: { id: string, name: string, color: string, isSystem: boolean }
 * @returns {TagResolver}
 */
function createTagResolver({ getTagById, getCategoryById })
```

**回傳的 TagResolver 物件介面**：

```javascript
/**
 * @typedef {Object} TagResolver
 * @property {Function} resolveTag - 單一 tagId 解析
 * @property {Function} resolveTagsForDisplay - 批量解析為顯示用資料
 * @property {Function} resolveTagName - 單一 tagId → tag 名稱（便捷方法）
 * @property {Function} resolveCategoryTagIds - categoryId → 該分類下所有 tagId[]
 */
```

### 2.2 `resolveTag(tagId)`

```javascript
/**
 * 解析單一 tagId 為完整 tag + category 資訊
 *
 * @param {string} tagId
 * @returns {{ tagId: string, tagName: string, categoryId: string, categoryName: string, categoryColor: string } | null}
 *   - 回傳 null：tagId 無效或對應 tag 已刪除
 *   - categoryName/categoryColor 為 fallback 值：tag 存在但 category 已刪除
 */
function resolveTag(tagId)
```

### 2.3 `resolveTagsForDisplay(tagIds)`

```javascript
/**
 * 批量解析 tagIds 為顯示用資料陣列
 *
 * @param {string[]} tagIds - 書籍的 tagIds 陣列
 * @returns {Array<{ tagId: string, tagName: string, categoryId: string, categoryName: string, categoryColor: string }>}
 *   - 過濾掉無法解析的 tagId（已刪除 tag），不報錯
 *   - 回傳順序與輸入順序一致（已過濾項除外）
 *   - 空輸入回傳空陣列
 */
function resolveTagsForDisplay(tagIds)
```

### 2.4 `resolveTagName(tagId)`

```javascript
/**
 * 便捷方法：單一 tagId → tag 名稱字串
 * 供 SearchEngine 文字搜尋使用（只需名稱，不需 category 資訊）
 *
 * @param {string} tagId
 * @returns {string | null} tag 名稱，或 null（tag 不存在）
 */
function resolveTagName(tagId)
```

### 2.5 `resolveCategoryTagIds(categoryId, allTagIds)`（暫不實作）

> 此方法在 search-engine-tag-query-spec.md 的 `_applyTagCategoryFilter` 中被引用，但 W2-003 範圍僅涵蓋 tag 解析，category→tagIds 展開需要遍歷所有 tags，屬 FilterEngine 整合層面。**本模組不包含此方法**，由 W2-004 FilterEngine 自行實作或在整合時擴展。

## 3. GWT Scenarios（行為場景）

### 場景 1: 正常解析 - 單一 tag

```
Given: getTagById('tag-01') 回傳 { id: 'tag-01', name: '科幻', categoryId: 'cat-01' }
       getCategoryById('cat-01') 回傳 { id: 'cat-01', name: '類別', color: '#808080' }
When: 呼叫 resolveTag('tag-01')
Then: 回傳 { tagId: 'tag-01', tagName: '科幻', categoryId: 'cat-01', categoryName: '類別', categoryColor: '#808080' }
```

### 場景 2: 正常批量解析

```
Given: 2 個有效 tag（tag-01, tag-02）皆可正常解析
When: 呼叫 resolveTagsForDisplay(['tag-01', 'tag-02'])
Then: 回傳包含 2 個解析結果的陣列，順序與輸入一致
```

### 場景 3: 無效 tagId（tag 已刪除）

```
Given: getTagById('tag-deleted') 回傳 null
When: 呼叫 resolveTag('tag-deleted')
Then: 回傳 null
```

### 場景 4: tag 存在但 category 已刪除

```
Given: getTagById('tag-01') 回傳 { id: 'tag-01', name: '科幻', categoryId: 'cat-deleted' }
       getCategoryById('cat-deleted') 回傳 null
When: 呼叫 resolveTag('tag-01')
Then: 回傳 { tagId: 'tag-01', tagName: '科幻', categoryId: 'cat-deleted', categoryName: '未分類', categoryColor: '#808080' }
     （使用 fallback 值）
```

### 場景 5: 批量解析含已刪除 tag

```
Given: tag-01 可解析，tag-deleted 無法解析
When: 呼叫 resolveTagsForDisplay(['tag-01', 'tag-deleted', 'tag-02'])
Then: 回傳陣列僅含 tag-01 和 tag-02 的解析結果，tag-deleted 被過濾
```

### 場景 6: 空 tagIds 陣列

```
Given: 任意 resolver
When: 呼叫 resolveTagsForDisplay([])
Then: 回傳空陣列 []
```

### 場景 7: tagIds 為 null/undefined

```
Given: 任意 resolver
When: 呼叫 resolveTagsForDisplay(null) 或 resolveTagsForDisplay(undefined)
Then: 回傳空陣列 []（不拋錯）
```

### 場景 8: tagIds 含非字串元素

```
Given: 任意 resolver
When: 呼叫 resolveTagsForDisplay(['tag-01', 123, null, 'tag-02'])
Then: 忽略非字串元素，僅解析 'tag-01' 和 'tag-02'
```

### 場景 9: resolveTagName 便捷方法

```
Given: getTagById('tag-01') 回傳 { id: 'tag-01', name: '科幻', categoryId: 'cat-01' }
When: 呼叫 resolveTagName('tag-01')
Then: 回傳 '科幻'
```

### 場景 10: resolveTagName - tag 不存在

```
Given: getTagById('tag-gone') 回傳 null
When: 呼叫 resolveTagName('tag-gone')
Then: 回傳 null
```

## 4. Error Handling（錯誤處理）

### 4.1 工廠函式參數驗證

| 情境 | 處理方式 |
|------|---------|
| `deps` 為 null/undefined | 拋出 `TypeError('createTagResolver requires deps object')` |
| `deps.getTagById` 不是函式 | 拋出 `TypeError('deps.getTagById must be a function')` |
| `deps.getCategoryById` 不是函式 | 拋出 `TypeError('deps.getCategoryById must be a function')` |

**設計理由**：工廠函式的依賴注入錯誤是程式設計錯誤（bug），應立即拋錯讓開發者發現，而非靜默回傳有缺陷的 resolver。

### 4.2 resolver 方法的寬容處理

| 情境 | 處理方式 | 理由 |
|------|---------|------|
| tagId 為空字串 | `resolveTag` 回傳 null | 寬容處理，非致命錯誤 |
| getTagById 拋出異常 | 不攔截，讓異常冒泡 | 儲存層錯誤應由上層處理 |
| tagIds 含重複 ID | 正常處理，不去重 | 去重是呼叫端職責 |

### 4.3 Fallback 常數

```javascript
const TAG_RESOLVER_DEFAULTS = {
  FALLBACK_CATEGORY_NAME: '未分類',
  FALLBACK_CATEGORY_COLOR: '#808080'  // 與 TagStorageAdapter DEFAULT_CATEGORY_COLOR 一致
}
```

## 5. Dependencies（依賴）

### 5.1 注入依賴

| 依賴 | 提供者 | 說明 |
|------|--------|------|
| `getTagById` | 呼叫端從 TagStorageAdapter 載入後建構 | `(tagId) => tagMap.get(tagId) \|\| null` |
| `getCategoryById` | 呼叫端從 TagStorageAdapter 載入後建構 | `(categoryId) => categoryMap.get(categoryId) \|\| null` |

### 5.2 典型使用方式（消費端範例）

```javascript
// Overview Page Controller 初始化時
const tags = await loadFromStorage('tags') || []
const categories = await loadFromStorage('tag_categories') || []

const tagMap = new Map(tags.map(t => [t.id, t]))
const categoryMap = new Map(categories.map(c => [c.id, c]))

const resolver = createTagResolver({
  getTagById: (id) => tagMap.get(id) || null,
  getCategoryById: (id) => categoryMap.get(id) || null
})

// 供 Overview 表格渲染
const displayTags = resolver.resolveTagsForDisplay(book.tagIds)

// 供 SearchEngine 注入
const searchEngine = new SearchEngine({
  tagResolver: resolver.resolveTagName  // 便捷方法直接傳入
})
```

### 5.3 檔案位置

- **模組位置**：`src/ui/search/tag-resolver.js`
- **測試位置**：`tests/unit/ui/search/tag-resolver.test.js`
- **匯出方式**：`module.exports = { createTagResolver, TAG_RESOLVER_DEFAULTS }`

### 5.4 下游消費者整合介面

| 消費者 | 使用的 API | Ticket |
|--------|-----------|--------|
| FilterEngine | `resolver.resolveTagName`（文字篩選輔助） | W2-004 |
| SearchEngine | `resolver.resolveTagName`（tag 名稱文字搜尋） | W2-005 |
| BookDataExporter | `resolver.resolveTagsForDisplay`（匯出時 tag 名稱展開） | W2-006 |
| Overview Page | `resolver.resolveTagsForDisplay`（表格 tag 欄位渲染） | W2-007 |

## 6. Acceptance（驗收條件）

- [ ] `createTagResolver` 接收 `{ getTagById, getCategoryById }` 回傳 resolver 物件
- [ ] `createTagResolver` 缺少必要依賴時拋出 TypeError
- [ ] `resolveTag` 正常解析含完整 category 資訊
- [ ] `resolveTag` 對已刪除 tag 回傳 null
- [ ] `resolveTag` 對已刪除 category 使用 fallback 值
- [ ] `resolveTagsForDisplay` 批量解析正確，順序與輸入一致
- [ ] `resolveTagsForDisplay` 過濾無法解析的 tagId
- [ ] `resolveTagsForDisplay` 對 null/undefined/空陣列回傳空陣列
- [ ] `resolveTagsForDisplay` 忽略非字串元素
- [ ] `resolveTagName` 回傳 tag 名稱或 null
