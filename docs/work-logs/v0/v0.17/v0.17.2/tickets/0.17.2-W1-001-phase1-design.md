# Phase 1 功能規格設計：Interchange Format v2

## Purpose

PROP-007 tag-based book model 重構後，匯出格式需從 v1 升級至 v2，支援 tag 結構、6 種閱讀狀態、以及對 v1 格式的匯入相容。

## 規格產出

規格文件位於：`docs/spec/export-interchange-format-v2.md`

### 涵蓋內容

1. **v2 JSON Schema**：metadata + tagCategories + tags + books 四區段結構
2. **v2 CSV 欄位對照表**：14 個欄位，tag 以 `; ` 分隔的扁平字串表達
3. **tag_tree 序列化格式**：JSON 採扁平化三陣列（與 Storage 一致），CSV 採分號分隔名稱
4. **版本偵測策略**：JSON 依 metadata.formatVersion 和 readingStatus 欄位；CSV 依標題行欄位名
5. **v1 匯入相容策略**：isNew/isFinished → readingStatus 轉換、category → tag 自動建立
6. **readingStatus 6 狀態映射表**：含 v1 對映規則和異常組合處理
7. **Field Presets v2**：BASIC_V2、EXTENDED_V2、COMPLETE_V2 定義

### 設計決策摘要

| 決策 | 選擇 | 理由 |
|------|------|------|
| JSON tag 結構 | 扁平化三陣列 | 與 Chrome Storage 一致，無需轉換 |
| CSV tag 表達 | 名稱而非 id | CSV 主要供人工閱讀 |
| v1 category 匯入 | 自動建立 tag | 保留使用者原有分類資訊 |
| tag 合併策略 | 聯集 | 不刪除既有 tag，安全保守 |
