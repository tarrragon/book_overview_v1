## 1. 測試策略

### 1.1 分層決策

| 層級 | 模組 | 測試類型 | 理由 |
|------|------|---------|------|
| Layer 5 (Domain) | createTagResolver 工廠 + resolver 方法 | 單元測試 | 純邏輯模組，無 UI/Storage 依賴，Map 查找 + 邊界處理 |

### 1.2 Sociable Unit Tests 應用

| 依賴 | 類型 | Mock 策略 | 理由 |
|------|------|----------|------|
| `deps.getTagById` | 注入函式（外部查詢） | Mock | 外部依賴，隔離 Storage 層 |
| `deps.getCategoryById` | 注入函式（外部查詢） | Mock | 外部依賴，隔離 Storage 層 |

本模組無 Domain Entity 依賴，所有輸入輸出皆為 plain object，完全符合 Sociable Unit Tests 原則。

### 1.3 測試檔案

- `tests/unit/ui/search/tag-resolver.test.js`

## 2. Mock 設計

### 2.1 測試資料

```
// Tag 物件
TAG_SCI_FI   = { id: 'tag-01', name: '科幻',   categoryId: 'cat-01', isSystem: false }
TAG_ROMANCE  = { id: 'tag-02', name: '言情',   categoryId: 'cat-01', isSystem: false }
TAG_ORPHAN   = { id: 'tag-03', name: '歷史',   categoryId: 'cat-deleted', isSystem: false }

// Category 物件
CAT_GENRE    = { id: 'cat-01', name: '類別', color: '#1976d2', isSystem: false }
```

### 2.2 Mock 函式建構策略

使用 Map 建構 `getTagById` 和 `getCategoryById`，模擬 Storage 查詢行為：

```
tagMap = new Map([['tag-01', TAG_SCI_FI], ['tag-02', TAG_ROMANCE], ['tag-03', TAG_ORPHAN]])
categoryMap = new Map([['cat-01', CAT_GENRE]])

mockGetTagById = jest.fn((id) => tagMap.get(id) || null)
mockGetCategoryById = jest.fn((id) => categoryMap.get(id) || null)
```

### 2.3 測試清理

每個測試前重置 jest.fn() 呼叫記錄。無外部狀態需清理。

## 3. 測試案例設計（GWT 格式）

### 群組 A：createTagResolver 工廠函式驗證

**功能職責**：依賴注入驗證，確保工廠函式正確拒絕無效依賴。

**跨群組依賴**：無。群組 B/C/D 依賴群組 A 建立的 resolver，但透過各自的 beforeEach 獨立建構，無共享 mutable 狀態。

---

#### TC-A1：正常建立 resolver（正常流程）

```
Given: 提供有效的 { getTagById: Function, getCategoryById: Function }
Then（前置驗證）: getTagById 是函式
Then（前置驗證）: getCategoryById 是函式
When: 呼叫 createTagResolver({ getTagById, getCategoryById })
Then: 回傳物件包含 resolveTag, resolveTagsForDisplay, resolveTagName 三個方法
Then: 每個屬性皆為 function 型別
```

#### TC-A2：deps 為 null（異常流程）

```
Given: deps = null
When: 呼叫 createTagResolver(null)
Then: 拋出 TypeError，訊息包含 'createTagResolver requires deps object'
```

#### TC-A3：deps 為 undefined（異常流程）

```
Given: deps = undefined
When: 呼叫 createTagResolver(undefined)
Then: 拋出 TypeError，訊息包含 'createTagResolver requires deps object'
```

#### TC-A4：deps.getTagById 不是函式（異常流程）

```
Given: deps = { getTagById: 'not-a-function', getCategoryById: jest.fn() }
When: 呼叫 createTagResolver(deps)
Then: 拋出 TypeError，訊息包含 'deps.getTagById must be a function'
```

#### TC-A5：deps.getCategoryById 不是函式（異常流程）

```
Given: deps = { getTagById: jest.fn(), getCategoryById: 123 }
When: 呼叫 createTagResolver(deps)
Then: 拋出 TypeError，訊息包含 'deps.getCategoryById must be a function'
```

#### TC-A6：deps 為空物件（邊界條件）

```
Given: deps = {}
When: 呼叫 createTagResolver({})
Then: 拋出 TypeError（getTagById 為 undefined，不是函式）
```

---

### 群組 B：resolveTag 單一解析

**功能職責**：單一 tagId 解析為完整 tag + category 資訊。

**跨群組依賴**：無。各測試獨立建構 resolver。

---

#### TC-B1：正常解析含完整 category（正常流程）

```
Given: resolver 已建立，tag-01 和 cat-01 皆存在於 Mock 資料
Then（前置驗證）: mockGetTagById('tag-01') 回傳非 null 的 tag 物件
Then（前置驗證）: mockGetCategoryById('cat-01') 回傳非 null 的 category 物件
When: 呼叫 resolver.resolveTag('tag-01')
Then: 回傳 { tagId: 'tag-01', tagName: '科幻', categoryId: 'cat-01', categoryName: '類別', categoryColor: '#1976d2' }
```

#### TC-B2：tag 不存在（異常流程）

```
Given: resolver 已建立，mockGetTagById('tag-nonexist') 回傳 null
Then（前置驗證）: mockGetTagById('tag-nonexist') === null
When: 呼叫 resolver.resolveTag('tag-nonexist')
Then: 回傳 null
Then: mockGetCategoryById 未被呼叫（tag 不存在時不需查詢 category）
```

#### TC-B3：tag 存在但 category 已刪除 - fallback（異常流程）

```
Given: resolver 已建立，tag-03 存在但 categoryId='cat-deleted' 無對應 category
Then（前置驗證）: mockGetTagById('tag-03') 回傳非 null
Then（前置驗證）: mockGetCategoryById('cat-deleted') === null
When: 呼叫 resolver.resolveTag('tag-03')
Then: 回傳 { tagId: 'tag-03', tagName: '歷史', categoryId: 'cat-deleted', categoryName: '未分類', categoryColor: '#808080' }
Then: categoryName 和 categoryColor 使用 TAG_RESOLVER_DEFAULTS 的 fallback 值
```

#### TC-B4：tagId 為空字串（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTag('')
Then: 回傳 null（空字串視為無效 tagId）
```

#### TC-B5：tagId 為 null（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTag(null)
Then: 回傳 null（不拋錯）
```

#### TC-B6：tagId 為 undefined（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTag(undefined)
Then: 回傳 null（不拋錯）
```

---

### 群組 C：resolveTagsForDisplay 批量解析

**功能職責**：批量 tagIds 解析為顯示用資料陣列，過濾無效項。

**跨群組依賴**：無。內部呼叫 resolveTag，但透過 resolver 物件自包含。

---

#### TC-C1：正常批量解析 2 個有效 tag（正常流程）

```
Given: resolver 已建立，tag-01 和 tag-02 皆可正常解析
Then（前置驗證）: mockGetTagById('tag-01') 回傳非 null
Then（前置驗證）: mockGetTagById('tag-02') 回傳非 null
When: 呼叫 resolver.resolveTagsForDisplay(['tag-01', 'tag-02'])
Then: 回傳長度為 2 的陣列
Then: 第一個元素 tagName === '科幻'，第二個元素 tagName === '言情'
Then: 順序與輸入一致
```

#### TC-C2：含已刪除 tag 的批量解析 - 自動過濾（異常流程）

```
Given: resolver 已建立，tag-01 可解析，tag-deleted 不可解析，tag-02 可解析
Then（前置驗證）: mockGetTagById('tag-deleted') === null
When: 呼叫 resolver.resolveTagsForDisplay(['tag-01', 'tag-deleted', 'tag-02'])
Then: 回傳長度為 2 的陣列（tag-deleted 被過濾）
Then: 結果僅含 tag-01 和 tag-02 的解析結果
```

#### TC-C3：空陣列輸入（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTagsForDisplay([])
Then: 回傳空陣列 []
Then: mockGetTagById 未被呼叫
```

#### TC-C4：null 輸入（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTagsForDisplay(null)
Then: 回傳空陣列 []（不拋錯）
```

#### TC-C5：undefined 輸入（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTagsForDisplay(undefined)
Then: 回傳空陣列 []（不拋錯）
```

#### TC-C6：含非字串元素 - 自動忽略（邊界條件）

```
Given: resolver 已建立，tag-01 和 tag-02 可解析
When: 呼叫 resolver.resolveTagsForDisplay(['tag-01', 123, null, undefined, 'tag-02'])
Then: 回傳長度為 2 的陣列
Then: 僅解析字串型別的 'tag-01' 和 'tag-02'
Then: 非字串元素（123, null, undefined）被忽略
```

#### TC-C7：所有 tagId 皆無效（邊界條件）

```
Given: resolver 已建立，mockGetTagById 對所有輸入皆回傳 null
When: 呼叫 resolver.resolveTagsForDisplay(['tag-gone-1', 'tag-gone-2'])
Then: 回傳空陣列 []
```

#### TC-C8：含重複 tagId - 不去重（邊界條件）

```
Given: resolver 已建立，tag-01 可正常解析
When: 呼叫 resolver.resolveTagsForDisplay(['tag-01', 'tag-01'])
Then: 回傳長度為 2 的陣列（重複項各自解析，不去重）
Then: 兩個元素的 tagName 皆為 '科幻'
```

### 群組 D：resolveTagName 便捷方法

**功能職責**：tagId → tag 名稱字串的快捷查詢，供 SearchEngine 使用。

**跨群組依賴**：無。

---

#### TC-D1：正常回傳 tag 名稱（正常流程）

```

Then: 回傳 '科幻'（字串型別）
```

#### TC-D2：tag 不存在回傳 null（異常流程）

```
Given: resolver 已建立，mockGetTagById('tag-gone') 回傳 null
When: 呼叫 resolver.resolveTagName('tag-gone')
Then: 回傳 null
```

#### TC-D3：tagId 為 null（邊界條件）

```
Given: resolver 已建立
When: 呼叫 resolver.resolveTagName(null)
Then: 回傳 null（不拋錯）
```
