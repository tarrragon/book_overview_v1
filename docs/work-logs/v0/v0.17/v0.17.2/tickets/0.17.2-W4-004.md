## Problem Analysis

5 個 Hook 在 main() 中直接呼叫 json.load(sys.stdin) 無 try-except 保護。當 tool_output 含控制字元（ANSI codes、raw newlines）時 JSONDecodeError 被 run_hook_safely 捕獲，_log_exception 寫入 stderr 觸發 UI 顯示 hook error。確認的崩潰 hook: test-timeout-post（20:39:46）、changelog-update-hook（20:39:47）。hook-logs 中有完整 traceback。

## Solution

<!-- To be filled by executing agent -->

---

在 5 個 hook 的 json.load(sys.stdin) 外加 try-except (json.JSONDecodeError, ValueError) 保護，解析失敗時 return 0 靜默跳過（符合既有 hook 慣例）。修改檔案: test-timeout-post.py, changelog-update-hook.py, post-commit-fetch-hook.py, pre-test-hook.py, test-timeout-pre.py
第二輪修復：根因是 hook_logging.py 的 StreamHandler 指向 stderr 且 level=WARNING，導致 logger.error() 的 JSON 解析錯誤訊息寫入 stderr 觸發 hook error。修復：將 STREAM_HANDLER_LEVEL_NORMAL 從 WARNING 提升為 CRITICAL，只有 _log_exception 的 logger.critical() 才會寫入 stderr。影響 8 個 hook（handoff-cleanup, commit-handoff, worktree-merge-reminder, cli-failure-help-reminder, skill-cli-error-feedback, session-context-guard, post-ticket-complete-checkpoint, parallel-dispatch-verification）。
