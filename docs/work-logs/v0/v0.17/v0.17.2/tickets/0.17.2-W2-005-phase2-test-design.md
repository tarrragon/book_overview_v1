## 1. 測試策略

### 1.1 分層決策

| 層級 | 測試類型 | 理由 |
|------|---------|------|
| `_matchesSearchCriteria` (Layer 3, UseCase 層行為) | BDD 測試 (Given-When-Then) | 核心業務邏輯：tag 文字搜尋匹配 |
| SearchEngine 建構函式 (Layer 2) | 單元測試 | 配置接受和儲存 |

### 1.2 Sociable Unit Tests 原則應用

| 依賴 | Mock 策略 | 理由 |
|------|----------|------|
| tagResolver (外部函式) | Mock | 外部依賴，隔離 tag 儲存層 |
| IndexManager | Mock | 外部依賴（現有測試已 mock） |
| EventBus | Mock | 外部依賴（現有測試已 mock） |
| Logger | Mock | 外部依賴（現有測試已 mock） |
| Book 物件 | 不 Mock | Domain Entity，直接使用真實物件 |

### 1.3 測試檔案

- **檔案**：`tests/unit/ui/search/core/search-engine.test.js`
- **方式**：在現有檔案追加新 `describe` 區塊（第 9 節），不修改現有測試
- **預期狀態**：所有新測試 RED（實作尚未遷移）

## 3. GWT 測試案例設計

### 群組 A：建構函式 tagResolver 接受（功能職責：初始化）

**跨群組依賴**：無，獨立可執行。

#### A-1: 建構時接受 tagResolver 選項

```
Given: 提供包含 tagResolver 函式的 options
When: 建構 SearchEngine 實例
Then: 實例的 _tagResolver 屬性為傳入的函式
```

#### A-2: 建構時未提供 tagResolver

```
Given: options 中不含 tagResolver
When: 建構 SearchEngine 實例
Then: 實例的 _tagResolver 為 undefined 或 null
Then: 建構不報錯（tagResolver 為選填）
```

---

### 群組 B：新邏輯 - tagIds + tagResolver 文字匹配（功能職責：tag 名稱搜尋）

**跨群組依賴**：依賴群組 A 的建構函式能接受 tagResolver。

#### B-1: tagIds + tagResolver 精確包含匹配 - 正常流程

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookWithTagIds 有 tagIds ['tag-sf', 'tag-novel']
Then（前置驗證）: bookWithTagIds.tagIds 存在且為陣列
Then（前置驗證）: mockTagResolver('tag-sf') 回傳 { name: '科幻小說' }
When: 以查詢 '科幻' 搜尋 [bookWithTagIds]
Then: bookWithTagIds 出現在搜尋結果中（tag 名稱 '科幻小說' 包含 '科幻'）
```

#### B-2: tagIds + tagResolver 不匹配

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookWithTagIds 有 tagIds ['tag-sf', 'tag-novel']
When: 以查詢 '歷史' 搜尋 [bookWithTagIds]
Then: bookWithTagIds 不出現在搜尋結果中（無 tag 名稱包含 '歷史'）
```

#### B-3: tagIds + tagResolver 大小寫不敏感

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookWithTagIds 有 tagIds ['tag-prog']
Then（前置驗證）: mockTagResolver('tag-prog') 回傳 { name: '程式設計' }
When: 以查詢 '程式設計' (全小寫已正規化) 搜尋
Then: 匹配成功（不區分大小寫）
```

#### B-4: tagIds + tagResolver 模糊匹配 - 正常流程

```
Given: SearchEngine 已建構，注入 mockTagResolver，enableFuzzySearch = true
Given: bookWithTagIds 有 tagIds ['tag-sf']
Then（前置驗證）: mockTagResolver('tag-sf') 回傳 { name: '科幻小說' }
When: 以查詢 '科幻小説'（「說」寫成「説」，模糊拼寫）搜尋
Then: bookWithTagIds 出現在搜尋結果中（模糊匹配命中）
```

---

### 群組 C：tagResolver 回傳 null / 邊界條件（功能職責：容錯處理）

**跨群組依賴**：依賴群組 A 的建構函式。

#### C-1: tagResolver 回傳 null（已刪除 tag）

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookDeletedTag 有 tagIds ['tag-deleted', 'tag-sf']
Then（前置驗證）: mockTagResolver('tag-deleted') 回傳 null
Then（前置驗證）: mockTagResolver('tag-sf') 回傳 { name: '科幻小說' }
When: 以查詢 '科幻' 搜尋 [bookDeletedTag]
Then: bookDeletedTag 出現在結果中（tag-deleted 被跳過，tag-sf 匹配成功）
```

#### C-2: book.tagIds 為 undefined

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookNoTags 無 tagIds 屬性
When: 以查詢 '科幻' 搜尋 [bookNoTags]（書名和作者也不含 '科幻'）
Then: bookNoTags 不出現在結果中
```

#### C-3: book.tagIds 為空陣列

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookEmptyTagIds 有 tagIds = []
When: 以查詢 '科幻' 搜尋 [bookEmptyTagIds]
Then: bookEmptyTagIds 不出現在結果中（無 tag 可匹配）
```

#### C-4: tagIds 含非字串元素

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookInvalidTagIds 有 tagIds = ['tag-sf', 123, null, 'tag-prog']
When: 以查詢 '科幻' 搜尋 [bookInvalidTagIds]
Then: 搜尋不報錯
Then: bookInvalidTagIds 出現在結果中（'tag-sf' 解析為 '科幻小說'，非字串元素被忽略）
```

---

### 群組 D：向下相容（功能職責：遷移相容）

**跨群組依賴**：無，獨立可執行。

#### D-1: 無 tagResolver + 有 book.tags — 沿用舊邏輯

```
Given: SearchEngine 已建構，未注入 tagResolver
Given: bookWithOldTags 有 tags = ['科幻小說', '冒險']，無 tagIds
Then（前置驗證）: searchEngine._tagResolver 為 undefined
When: 以查詢 '科幻' 搜尋 [bookWithOldTags]
Then: bookWithOldTags 出現在結果中（沿用舊 book.tags 字串匹配）
```

#### D-2: 有 tagResolver + 有 tagIds + 有 tags — 優先使用 tagIds

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: bookMixed 有 tagIds = ['tag-sf']，tags = ['舊標籤']
When: 以查詢 '舊標籤' 搜尋 [bookMixed]
Then: bookMixed 不出現在結果中（有 tagResolver 時不走 book.tags 舊邏輯）
```

#### D-3: 有 tagResolver + 無 tagIds + 有 tags — tag 搜尋不匹配

```
Given: SearchEngine 已建構，注入 mockTagResolver
Given: book 只有 tags = ['科幻小說']，無 tagIds
When: 以查詢 '科幻' 搜尋（書名和作者不含 '科幻'）
Then: 該書不出現在結果中（有 tagResolver 時跳過舊 tags，新 tagIds 不存在）
```

## 4. 行為鏈式推演完成性驗證

### 4.1 四維度行為分支窮舉

| 維度 | 覆蓋案例 |
|------|---------|
| 正常流程 | B-1, B-3, B-4 |
| 異常流程 | C-1 (已刪除 tag), C-4 (非字串元素) |
| 邊界條件 | C-2 (undefined tagIds), C-3 (空陣列), B-2 (不匹配) |
| 中斷操作 | 不適用（同步匹配函式，無可中斷點） |

### 4.2 向下相容分支

| 維度 | 覆蓋案例 |
|------|---------|
| 正常流程 | D-1 (舊邏輯仍運作) |
| 優先級衝突 | D-2 (tagIds 優先於 tags) |
| 邊界條件 | D-3 (有 resolver 但無 tagIds) |

### 4.3 完成判斷

無法再提出新的「如果...那麼...」組合。所有 Then 都是可直接觀察的搜尋結果（書籍是否在結果中）。
