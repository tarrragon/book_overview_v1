## 1. 測試策略

### 1.1 分層決策

| 層級 | 模組 | 測試類型 | 理由 |
|------|------|---------|------|
| Layer 1 (UI) | FilterEngine tag 篩選擴展 | 單元測試 | 透過公開 API `applyFilters()` 測試，驗證篩選行為 |

### 1.2 Sociable Unit Tests 應用

| 依賴 | 類型 | Mock 策略 | 理由 |
|------|------|----------|------|
| `options.eventBus` | 外部依賴 | Mock | 現有模式，隔離事件系統 |
| `options.logger` | 外部依賴 | Mock | 現有模式，隔離日誌系統 |
| `options.tagResolver` | 注入函式（optional） | Mock | 外部依賴，optional 注入 |
| `options.categoryResolver` | 注入函式（optional） | Mock | 外部依賴，將 categoryId 展開為 tagIds |

tag 篩選核心邏輯（AND/OR 比對）是 FilterEngine 內部邏輯，不需要 Mock。`book.tagIds` 是 plain data，直接使用真實值。

### 1.3 測試檔案

- `tests/unit/ui/search/filter/filter-engine.test.js`（追加新的 describe 區塊）

### 1.4 測試風格對齊

沿用現有模式：
- `filterEngine = new FilterEngine({ eventBus, logger, ... })`
- `const result = await filterEngine.applyFilters(books, filters)`
- 斷言 `result.filteredBooks`、`result.totalCount`
- `eslint-disable-next-line no-unused-vars` 用於 result 變數（現有慣例）

## 3. 測試案例設計（GWT 格式）

### Group A：建構函式 tagResolver/categoryResolver 注入（3 案例）

**功能職責**：FilterEngine 建構函式 optional 參數擴展
**跨群組依賴**：無

#### A-1：接受 optional tagResolver

```
Given: 提供 mockTagResolver 函式作為 options.tagResolver
When: 建構 FilterEngine
Then: 實例建立成功，為 FilterEngine 實例
```

#### A-2：接受 optional categoryResolver

```
Given: 提供 mockCategoryResolver 函式作為 options.categoryResolver
When: 建構 FilterEngine
Then: 實例建立成功，為 FilterEngine 實例
```

#### A-3：不提供 tagResolver/categoryResolver 時向下相容

```
Given: 僅提供 eventBus 和 logger（無 tagResolver/categoryResolver）
When: 建構 FilterEngine
Then: 實例建立成功，不報錯
```

---

### Group B：_applyTagFilter OR 邏輯（2 案例）

**功能職責**：tag 篩選 OR（聯集）邏輯
**跨群組依賴**：依賴 Group A 的建構函式成功

#### B-1：OR 邏輯 — 多 tag 聯集（預設 operator）

```
Given: tagBooks 已載入（5 筆），其中 tb1 有 [tag-sf, tag-novel]，tb2 有 [tag-sf]，tb3 有 [tag-history]
Then（前置驗證）: tagBooks 長度為 5
When: applyFilters(tagBooks, { tagIds: ['tag-sf', 'tag-history'] })
Then: 回傳 3 筆（tb1, tb2, tb3），含任一指定 tag 即匹配
```

#### B-2：OR 邏輯 — 明確指定 tagOperator='OR'

```
Given: tagBooks 已載入
When: applyFilters(tagBooks, { tagIds: ['tag-sf'], tagOperator: 'OR' })
Then: 回傳 2 筆（tb1, tb2）
```

---

### Group C：_applyTagFilter AND 邏輯（2 案例）

**功能職責**：tag 篩選 AND（交集）邏輯
**跨群組依賴**：依賴 Group A 的建構函式成功

#### C-1：AND 邏輯 — 書籍同時含所有 tag

```
Given: tagBooks 已載入，tb1 有 [tag-sf, tag-novel]
When: applyFilters(tagBooks, { tagIds: ['tag-sf', 'tag-novel'], tagOperator: 'AND' })
Then: 回傳 1 筆（僅 tb1）
```

#### C-2：AND 邏輯 — 無書同時匹配所有 tag

```
Given: tagBooks 已載入
When: applyFilters(tagBooks, { tagIds: ['tag-sf', 'tag-history'], tagOperator: 'AND' })
Then: 回傳 0 筆（無書同時含兩者）
```

---

### Group D：邊界條件（6 案例）

**功能職責**：tag 篩選的無效/邊界輸入處理
**跨群組依賴**：無

#### D-1：空 tagIds 陣列跳過篩選

```
Given: tagBooks 已載入（5 筆）
When: applyFilters(tagBooks, { tagIds: [] })
Then: 回傳全部 5 筆（跳過 tag 篩選）
```

#### D-2：null tagIds 跳過篩選

```
Given: tagBooks 已載入（5 筆）
When: applyFilters(tagBooks, { tagIds: null })
Then: 回傳全部 5 筆（跳過 tag 篩選）
```

#### D-3：書籍無 tagIds 欄位（undefined）不匹配

```
Given: tagBooks 含 tb5（tagIds 為 undefined）
When: applyFilters(tagBooks, { tagIds: ['tag-sf'] })
Then: tb4（空陣列）和 tb5（undefined）不在結果中
```

#### D-4：不存在的 tagId 不報錯

```
Given: tagBooks 已載入
When: applyFilters(tagBooks, { tagIds: ['tag-nonexistent'] })
Then: 回傳 0 筆，不拋出錯誤
```

#### D-5：無效 tagOperator 回退 OR

```
Given: tagBooks 已載入
When: applyFilters(tagBooks, { tagIds: ['tag-sf', 'tag-history'], tagOperator: 'INVALID' })
Then: 使用 OR 邏輯，回傳 3 筆（tb1, tb2, tb3）
```

#### D-6：tagIds 含非字串元素時忽略非字串

```
Given: tagBooks 已載入
When: applyFilters(tagBooks, { tagIds: ['tag-sf', 123, null, undefined, 'tag-history'] })
Then: 僅處理字串元素，等同 ['tag-sf', 'tag-history'] OR，回傳 3 筆
```

---

### Group E：與現有篩選的組合（2 案例）

**功能職責**：tag 篩選與 status/category 篩選的管線組合
**跨群組依賴**：依賴 Group B/C 的 tag 篩選和現有 status/category 篩選

#### E-1：tag + status 組合

```
Given: tagBooks 已載入
Then（前置驗證）: status='reading' 的書有 tb1, tb4, tb5；tagIds 含 'tag-sf' 的書有 tb1, tb2
When: applyFilters(tagBooks, { status: 'reading', tagIds: ['tag-sf'] })
Then: 回傳 1 筆（tb1，為兩者交集）
```

#### E-2：tag + category 組合

```
Given: tagBooks 已載入
Then（前置驗證）: category='programming' 的書有 tb1, tb2；tagIds 含 'tag-sf' 的書有 tb1, tb2
When: applyFilters(tagBooks, { category: 'programming', tagIds: ['tag-sf'], tagOperator: 'OR' })
Then: 回傳 2 筆（tb1, tb2）
```

---

### Group F：tagCategory 篩選（4 案例）

**功能職責**：`_applyTagCategoryFilter` 分類展開為 tagIds 後篩選
**跨群組依賴**：依賴 Group A 的 categoryResolver 注入

#### F-1：單一 tagCategoryId 展開篩選

```
Given: catBooks 已載入，categoryResolver 將 'cat-fiction' 展開為 ['tag-sf', 'tag-novel', 'tag-mystery']
When: applyFilters(catBooks, { tagCategoryIds: ['cat-fiction'] })
Then: 回傳 2 筆（tc1, tc2），OR 匹配展開後的 tagIds
```

#### F-2：多個 tagCategoryIds

```
Given: catBooks 已載入，categoryResolver 同時展開 cat-fiction 和 cat-nonfiction
When: applyFilters(catBooks, { tagCategoryIds: ['cat-fiction', 'cat-nonfiction'] })
Then: 回傳 3 筆（全部）
```

#### F-3：無 categoryResolver 時跳過篩選

```
Given: FilterEngine 建構時未提供 categoryResolver
When: applyFilters(catBooks, { tagCategoryIds: ['cat-fiction'] })
Then: 跳過 tagCategory 篩選，回傳全部 3 筆
```

#### F-4：分類展開為空時無匹配

```
Given: categoryResolver 對 'cat-unknown' 回傳空陣列
When: applyFilters(catBooks, { tagCategoryIds: ['cat-unknown'] })
Then: 回傳 0 筆
```

## 4. 行為鏈式推演

### 4.1 起點：使用者選擇 tag 篩選

行為鏈：使用者選擇 tag -> FilterEngine 接收 filters.tagIds -> _applyFilterLogic 管線中 tag 步驟觸發 -> _applyTagFilter 執行 AND/OR -> 回傳結果

### 4.2 前置條件驗證

| 步驟 | 前置條件 | 驗證方式 |
|------|---------|---------|
| applyFilters 呼叫 | FilterEngine 未 destroyed | `expect(filterEngine._isDestroyed).toBe(false)` |
| tag 篩選觸發 | filters.tagIds 存在且非空 | `expect(tagIds).toBeDefined(); expect(tagIds.length).toBeGreaterThan(0)` |
| tagCategory 篩選觸發 | categoryResolver 已注入 | 建構時確認注入 |
| 書籍資料有效 | tagBooks 有 title 欄位 | `_isValidBook` 過濾（現有邏輯） |

### 4.3 四維度行為分支窮舉

| 維度 | 涵蓋案例 |
|------|---------|
| 正常流程 | B-1, B-2, C-1, E-1, E-2, F-1, F-2 |
| 異常流程 | D-4（不存在 tagId）, D-5（無效 operator）, D-6（非字串元素） |
| 邊界條件 | D-1（空陣列）, D-2（null）, D-3（undefined tagIds 欄位）, F-4（空展開） |
| 中斷操作 | A-3（無 resolver 時向下相容）, F-3（無 categoryResolver 跳過） |

### 4.4 完成判斷

- 四個維度皆有至少一個測試案例：是
- 無法再提出新的「如果...那麼...」組合：是
- 所有 Then 都是使用者可觀察的狀態（filteredBooks 長度和內容）：是

## 5. knownFilters 更新提醒

現有 `_handleUnknownFilters` 的 `knownFilters` 陣列只包含 `['status', 'category', 'progressRange', 'lastReadAfter', 'lastReadBefore']`。實作時必須新增 `'tagIds'`、`'tagOperator'`、`'tagCategoryIds'` 到 knownFilters，否則這些欄位會觸發「未知的篩選條件」warning log。
