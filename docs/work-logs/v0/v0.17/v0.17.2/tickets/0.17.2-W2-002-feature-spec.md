# 0.17.2-W2-002 功能規格

## 1. Purpose（目的）

匯入功能需要辨識使用者提供的資料是 v1（category-based）還是 v2（tag-based）格式，並將 v1 格式自動轉換為 v2 結構，讓後續匯入流程只需處理單一格式。本模組為純轉換邏輯，不涉及 Storage 寫入。

---

## 2. API Signatures（介面定義）

### 2.1 模組位置

```
src/export/format-version-detector.js   — 版本偵測
src/export/v1-to-v2-converter.js        — v1 -> v2 轉換
```

### 2.2 detectFormatVersion(data)

偵測匯入資料的格式版本。

```javascript
/**
 * @param {*} data - 解析後的 JSON 資料（物件或陣列）
 * @returns {'v1' | 'v2' | null} 格式版本，null 表示無法辨識
 */
function detectFormatVersion(data)
```

**偵測規則（依優先序）**：

| 優先序 | 條件 | 回傳 |
|--------|------|------|
| 1 | `data?.metadata?.formatVersion` 以 `"2."` 開頭 | `'v2'` |
| 2 | `data?.metadata` 存在且 `data.books` 陣列中任一物件含 `readingStatus` 欄位 | `'v2'` |
| 3 | `data` 本身是陣列（`Array.isArray(data)`） | `'v1'` |
| 4 | `data?.books` 是陣列但無 `metadata.formatVersion` | `'v1'` |
| 5 | 以上皆不符合 | `null` |

### 2.3 detectCsvFormatVersion(headers)

偵測 CSV 匯入資料的格式版本。

```javascript
/**
 * @param {string[]} headers - CSV 標題行欄位名稱陣列
 * @returns {'v1' | 'v2'} 格式版本（CSV 不回傳 null，預設降級為 v1）
 */
function detectCsvFormatVersion(headers)
```

**偵測規則**：

| 條件 | 回傳 |
|------|------|
| headers 含 `'readingStatus'` | `'v2'` |
| headers 含 `'isNew'` 或 `'isFinished'` | `'v1'` |
| 以上皆無 | `'v1'`（降級處理） |

### 2.4 convertV1ToV2Book(v1Book)

將單本 v1 書籍轉換為 v2 格式。

```javascript
/**
 * @param {Object} v1Book - v1 格式書籍物件
 * @param {string} [importTimestamp] - 匯入時間 ISO 字串，預設為呼叫時的 new Date().toISOString()
 * @returns {Object} v2 格式書籍物件
 */
function convertV1ToV2Book(v1Book, importTimestamp)
```

**欄位轉換規則**：

| v1 欄位 | v2 欄位 | 轉換邏輯 |
|---------|---------|---------|
| `id` | `id` | 直接保留 |
| `title` | `title` | 直接保留 |
| `author` (string) | `authors` (string[]) | 空字串/undefined -> `[]`；非空 -> `[author]` |
| `publisher` | `publisher` | 直接保留，undefined -> `''` |
| `isNew`, `isFinished`, `progress` | `readingStatus` | **重用** `BookSchemaV2.mapV1StatusToV2(v1Book)` |
| `progress` | `progress` | **重用** `BookSchemaV2.normalizeV1Progress(v1Book.progress)` + clamp(0, 100) |
| `type` | `type` | 直接保留，undefined -> `''` |
| `cover` | `cover` | 直接保留，undefined -> `''` |
| （無） | `tagIds` | `[]` |
| （無） | `isManualStatus` | `false` |
| `extractedAt` | `extractedAt` | 保留（若存在） |
| （無） | `updatedAt` | `importTimestamp` |
| （無） | `source` | `'readmoo'` |

**移除欄位**：轉換後的物件不包含 `isNew`、`isFinished`、`category`。

**`category` 欄位處理**：`convertV1ToV2Book` 不處理 category -> tag 轉換。category 值保留在回傳物件的 `_v1Category` 暫存欄位中（若原始資料有 category），供上層 `convertV1CategoryToTag` 使用。

### 2.5 convertV1CategoryToTag(categories)

將 v1 的 category 字串集合轉換為 tag 結構。純轉換邏輯，輸出資料結構，不寫入 Storage。

```javascript
/**
 * @param {string[]} categories - 不重複的 category 名稱陣列
 * @param {string} [timestamp] - 建立時間 ISO 字串，預設為呼叫時的 new Date().toISOString()
 * @returns {{ tagCategory: Object, tags: Object[], categoryToTagIdMap: Map<string, string> }}
 *   - tagCategory: "匯入分類" TagCategory 物件（符合 TagSchema.TAG_CATEGORY_SCHEMA）
 *   - tags: 每個 category 對應的 Tag 物件陣列（符合 TagSchema.TAG_SCHEMA）
 *   - categoryToTagIdMap: category 名稱 -> tag id 的對映，供上層將 tagId 寫入書籍的 tagIds
 */
function convertV1CategoryToTag(categories, timestamp)
```

**行為規則**：

| 輸入 | 行為 |
|------|------|
| `categories` 包含 `""` 或 `"未分類"` | 跳過，不建立 tag |
| `categories` 為空陣列 | 回傳 `{ tagCategory: null, tags: [], categoryToTagIdMap: new Map() }` |
| 正常 category 名稱 | 建立 `"匯入分類"` TagCategory（id: `cat_imported`）+ 每個名稱一個 Tag |

**Tag id 生成規則**：`tag_{timestamp_ms}-{隨機6碼}`，與現有 TagSchema 一致。

### 2.6 convertV1ToV2Data(v1Data)

完整的 v1 -> v2 資料轉換（組合上述函式的頂層 API）。

```javascript
/**
 * @param {Object|Array} v1Data - v1 格式的匯入資料（純陣列或含 metadata 的物件）
 * @returns {Object} v2 interchange format 結構（含 metadata, tagCategories, tags, books）
 */
function convertV1ToV2Data(v1Data)
```

**處理流程**：

1. 提取書籍陣列：純陣列用 `v1Data`，物件用 `v1Data.books`
2. 逐本呼叫 `convertV1ToV2Book`
3. 收集所有書籍的 `_v1Category` 值（去重、過濾空值和 `"未分類"`）
4. 呼叫 `convertV1CategoryToTag` 建立 tag 結構
5. 根據 `categoryToTagIdMap` 將對應 tagId 寫入每本書的 `tagIds`
6. 移除 `_v1Category` 暫存欄位
7. 組裝 v2 interchange format 結構回傳

**回傳結構**：

```javascript
{
  metadata: {
    formatVersion: '2.0.0',
    exportDate: importTimestamp,
    source: 'readmoo-book-extractor',
    schemaVersion: BookSchemaV2.SCHEMA_VERSION,
    totalBooks: books.length,
    totalTags: tags.length,
    totalTagCategories: tagCategories.length
  },
  tagCategories: [...],  // 預設 2 個系統分類 + 匯入分類（若有 category）
  tags: [...],
  books: [...]
}
```

## 3. GWT Scenarios（行為場景）

### JSON 版本偵測

場景 1: v2 明確版本號偵測
- **Given**: JSON 資料含 `metadata.formatVersion: "2.0.0"`
- **When**: 呼叫 `detectFormatVersion(data)`
- **Then**: 回傳 `'v2'`

場景 2: v2 隱含偵測（有 metadata + readingStatus）
- **Given**: JSON 資料含 `metadata` 但無 `formatVersion`，books 中有 `readingStatus` 欄位
- **When**: 呼叫 `detectFormatVersion(data)`
- **Then**: 回傳 `'v2'`

場景 3: v1 純陣列格式
- **Given**: JSON 資料為純陣列（無 metadata 包裝）
- **When**: 呼叫 `detectFormatVersion(data)`
- **Then**: 回傳 `'v1'`

場景 4: v1 含 metadata 包裝但無 formatVersion
- **Given**: JSON 資料含 `books` 屬性但無 `metadata.formatVersion`
- **When**: 呼叫 `detectFormatVersion(data)`
- **Then**: 回傳 `'v1'`

場景 5: 無法辨識的格式
- **Given**: JSON 資料為字串、數字、或不含 books 的物件
- **When**: 呼叫 `detectFormatVersion(data)`
- **Then**: 回傳 `null`

### CSV 版本偵測

場景 6: CSV v2 偵測
- **Given**: CSV 標題行含 `readingStatus` 欄位
- **When**: 呼叫 `detectCsvFormatVersion(headers)`
- **Then**: 回傳 `'v2'`

場景 7: CSV v1 偵測
- **Given**: CSV 標題行含 `isFinished` 但不含 `readingStatus`
- **When**: 呼叫 `detectCsvFormatVersion(headers)`
- **Then**: 回傳 `'v1'`

場景 8: CSV 降級為 v1
- **Given**: CSV 標題行不含 `readingStatus`、`isNew`、`isFinished`
- **When**: 呼叫 `detectCsvFormatVersion(headers)`
- **Then**: 回傳 `'v1'`

### 書籍轉換

場景 9: 標準 v1 書籍轉換
- **Given**: v1 書籍 `{ id: 'b1', title: '三體', author: '劉慈欣', isFinished: true, progress: 100 }`
- **When**: 呼叫 `convertV1ToV2Book(v1Book)`
- **Then**: 回傳含 `authors: ['劉慈欣']`, `readingStatus: 'finished'`, `tagIds: []`, `isManualStatus: false`

場景 10: v1 書籍 author 為空
- **Given**: v1 書籍 author 為 `''` 或 `undefined`
- **When**: 呼叫 `convertV1ToV2Book(v1Book)`
- **Then**: `authors` 為 `[]`

場景 11: v1 書籍 progress 為異常值
- **Given**: v1 書籍 progress 為 `"75"` (字串) 或 `150` (超過100) 或 `null`
- **When**: 呼叫 `convertV1ToV2Book(v1Book)`
- **Then**: progress 分別正規化為 `75`、`100`、`0`

場景 12: v1 書籍含 category 欄位
- **Given**: v1 書籍含 `category: '科幻小說'`
- **When**: 呼叫 `convertV1ToV2Book(v1Book)`
- **Then**: 回傳物件含 `_v1Category: '科幻小說'`，不含 `category` 欄位

### Category 轉換

場景 13: 正常 category 轉換
- **Given**: categories 為 `['科幻', '文學']`
- **When**: 呼叫 `convertV1CategoryToTag(categories)`
- **Then**: 建立 1 個 TagCategory（id: `cat_imported`，name: `"匯入分類"`）和 2 個 Tag，categoryToTagIdMap 含 2 筆對映

場景 14: category 含空值和「未分類」
- **Given**: categories 為 `['科幻', '', '未分類', '文學']`
- **When**: 呼叫 `convertV1CategoryToTag(categories)`
- **Then**: 只建立 2 個 Tag（科幻、文學），空字串和「未分類」被跳過

場景 15: 空 category 陣列
- **Given**: categories 為 `[]`
- **When**: 呼叫 `convertV1CategoryToTag(categories)`
- **Then**: `tagCategory` 為 `null`，`tags` 為 `[]`，`categoryToTagIdMap` 為空 Map

## 4. Error Handling（錯誤處理）

| 情境 | 處理策略 | 回傳值 |
|------|---------|--------|
| `detectFormatVersion(null)` | 安全處理 | `null` |
| `detectFormatVersion(undefined)` | 安全處理 | `null` |
| `detectFormatVersion(123)` | 非物件/非陣列 | `null` |
| `detectCsvFormatVersion([])` | 空標題行 | `'v1'`（降級） |
| `detectCsvFormatVersion(null)` | 無效輸入 | `'v1'`（降級） |
| `convertV1ToV2Book(null)` | 拋出 `BookValidationError` | 輸入必須為物件 |
| `convertV1ToV2Book({})` | 缺少 id/title | 拋出 `BookValidationError`，列出缺失的必填欄位 |
| `convertV1ToV2Book` 單一選填欄位無效 | 降級為預設值 | 不中斷轉換，記錄 warning |
| `convertV1ToV2Data` 書籍陣列為空 | 正常處理 | 回傳含空 books 的 v2 結構 |
| `convertV1ToV2Data` 單本書轉換失敗 | 跳過該書，記錄 error | 其他書籍繼續轉換 |

**錯誤類別**：使用專案現有 `BookValidationError`（`src/core/errors/BookValidationError.js`）。

## 5. Dependencies（依賴）

| 依賴模組 | 使用的函式/常數 | 用途 |
|---------|---------------|------|
| `BookSchemaV2` | `mapV1StatusToV2(v1Book)` | readingStatus 轉換（重用，非重寫） |
| `BookSchemaV2` | `normalizeV1Progress(value)` | progress 正規化（重用，非重寫） |
| `BookSchemaV2` | `SCHEMA_VERSION` | metadata 中的 schemaVersion |
| `BookSchemaV2` | `READING_STATUS` | 狀態列舉值 |
| `BookValidationError` | 錯誤類別 | 驗證失敗時拋出 |
| `TagSchema`（若已存在） | `TAG_CATEGORY_SCHEMA`, `TAG_SCHEMA` | tag 結構定義參考 |

**明確排除的依賴**：

- 不依賴 Storage 模組（不讀寫 Chrome Storage）
- 不依賴 `src/data-management/migration/v1-to-v2.js`（那是 Storage Schema 遷移，職責不同）
