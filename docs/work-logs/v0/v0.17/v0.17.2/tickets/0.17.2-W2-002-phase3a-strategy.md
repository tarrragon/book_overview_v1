## 1. 實作策略設計

本模組分為兩個檔案、五個函式，職責清晰分離：

- **format-version-detector.js** — 純偵測，無副作用，回傳版本字串
- **v1-to-v2-converter.js** — 純轉換，無 Storage 寫入，回傳新結構

### 實作優先序

**第一階段**（無依賴，可獨立實作）：
1. `detectFormatVersion` — 純條件判斷，零依賴
2. `detectCsvFormatVersion` — 純條件判斷，零依賴

**第二階段**（依賴 BookSchemaV2、TagSchema）：
3. `convertV1ToV2Book` — 依賴 BookSchemaV2.mapV1StatusToV2 + normalizeV1Progress
4. `convertV1CategoryToTag` — 依賴 TagSchema 結構定義

**第三階段**（組合上述函式）：
5. `convertV1ToV2Data` — 組合 3 + 4，頂層 API

## 3. 虛擬碼

### 3.1 detectFormatVersion(data)

```
FUNCTION detectFormatVersion(data):
  // 規則 1: 明確版本號
  IF data?.metadata?.formatVersion 是字串 AND 以 "2." 開頭:
    RETURN 'v2'

  // 規則 2: 隱含 v2（有 metadata + readingStatus）
  IF data?.metadata 存在 AND data?.books 是陣列:
    FOR EACH book IN data.books:
      IF book 含 'readingStatus' 屬性:
        RETURN 'v2'

  // 規則 3: 純陣列 = v1
  IF data 是陣列:
    RETURN 'v1'

  // 規則 4: 有 books 但無 formatVersion = v1
  IF data?.books 是陣列:
    RETURN 'v1'

  // 規則 5: 無法辨識
  RETURN null
```

### 3.2 detectCsvFormatVersion(headers)

```
FUNCTION detectCsvFormatVersion(headers):
  // 防禦性：無效輸入降級
  IF headers 非陣列 OR 為空:
    RETURN 'v1'

  IF headers 含 'readingStatus':
    RETURN 'v2'

  IF headers 含 'isNew' OR headers 含 'isFinished':
    RETURN 'v1'

  // 降級
  RETURN 'v1'
```

### 3.3 convertV1ToV2Book(v1Book, importTimestamp)

```
FUNCTION convertV1ToV2Book(v1Book, importTimestamp):
  // 驗證
  IF v1Book 為 null/undefined 或非物件:
    THROW BookValidationError("輸入必須為物件")

  IF v1Book 缺少 id 或 title:
    THROW BookValidationError("缺少必填欄位: " + 缺失欄位列表)

  timestamp = importTimestamp OR 當前 ISO 時間字串

  // 欄位轉換
  v2Book = {
    id:             v1Book.id,
    title:          v1Book.title,
    authors:        (v1Book.author 為空/undefined) ? [] : [v1Book.author],
    publisher:      v1Book.publisher OR '',
    readingStatus:  BookSchemaV2.mapV1StatusToV2(v1Book),          // 重用
    progress:       clamp(BookSchemaV2.normalizeV1Progress(v1Book.progress), 0, 100),  // 重用 + clamp
    type:           v1Book.type OR '',
    cover:          v1Book.cover OR '',
    tagIds:         [],
    isManualStatus: false,
    extractedAt:    v1Book.extractedAt OR undefined,  // 有才保留
    updatedAt:      timestamp,
    source:         'readmoo'
  }

  // category 暫存
  IF v1Book.category 存在且非空:
    v2Book._v1Category = v1Book.category

  RETURN v2Book
```

**clamp 輔助邏輯**：`MAX(0, MIN(100, value))`

### 3.4 convertV1CategoryToTag(categories, timestamp)

```
FUNCTION convertV1CategoryToTag(categories, timestamp):
  // 過濾無效值
  validCategories = categories 過濾掉 '' 和 '未分類'

  IF validCategories 為空:
    RETURN { tagCategory: null, tags: [], categoryToTagIdMap: new Map() }

  ts = timestamp OR 當前 ISO 時間字串

  // 建立 TagCategory
  tagCategory = {
    id:          'cat_imported',
    name:        '匯入分類',
    description: '',
    color:       '#808080',    // TagSchema DEFAULT_COLOR
    isSystem:    false,
    sortOrder:   0,
    createdAt:   ts,
    updatedAt:   ts
  }

  tags = []
  categoryToTagIdMap = new Map()

  FOR EACH categoryName IN validCategories:
    tagId = "tag_" + 時間戳毫秒 + "-" + 隨機6碼
    tag = {
      id:         tagId,
      name:       categoryName,
      categoryId: 'cat_imported',
      isSystem:   false,
      sortOrder:  目前 index,
      createdAt:  ts,
      updatedAt:  ts
    }
    tags.PUSH(tag)
    categoryToTagIdMap.SET(categoryName, tagId)

  RETURN { tagCategory, tags, categoryToTagIdMap }
```

### 3.5 convertV1ToV2Data(v1Data)

```
FUNCTION convertV1ToV2Data(v1Data):
  timestamp = 當前 ISO 時間字串

  // 步驟 1: 提取書籍陣列
  IF v1Data 是陣列:
    rawBooks = v1Data
  ELSE:
    rawBooks = v1Data.books OR []

  // 步驟 2: 逐本轉換（容錯）
  convertedBooks = []
  failedBooks = []
  FOR EACH book IN rawBooks:
    TRY:
      v2Book = convertV1ToV2Book(book, timestamp)
      convertedBooks.PUSH(v2Book)
    CATCH error:
      記錄 warning（書籍 id/index + 錯誤訊息）
      failedBooks.PUSH({ index, error })

  // 步驟 3: 收集 category（去重）
  categorySet = new Set()
  FOR EACH book IN convertedBooks:
    IF book._v1Category 存在:
      categorySet.ADD(book._v1Category)

  // 步驟 4: 轉換 category -> tag
  { tagCategory, tags, categoryToTagIdMap } =
    convertV1CategoryToTag(Array.from(categorySet), timestamp)

  // 步驟 5: 寫入 tagIds + 移除暫存欄位
  FOR EACH book IN convertedBooks:
    IF book._v1Category 存在:
      tagId = categoryToTagIdMap.GET(book._v1Category)
      IF tagId 存在:
        book.tagIds = [tagId]
    DELETE book._v1Category

  // 步驟 6: 組裝 tagCategories（系統預設 + 匯入分類）
  tagCategories = []
  IF tagCategory 非 null:
    tagCategories.PUSH(tagCategory)

  // 步驟 7: 回傳 v2 interchange format
  RETURN {
    metadata: {
      formatVersion:    '2.0.0',
      exportDate:       timestamp,
      source:           'readmoo-book-extractor',
      schemaVersion:    BookSchemaV2.SCHEMA_VERSION,  // '3.0.0'
      totalBooks:       convertedBooks.length,
      totalTags:        tags.length,
      totalTagCategories: tagCategories.length
    },
    tagCategories: tagCategories,
    tags:          tags,
    books:         convertedBooks
  }
```

## 4. 控制流程圖

### 整體資料流

```
匯入資料 (JSON/CSV)
    |
    v
detectFormatVersion / detectCsvFormatVersion
    |
    +-- 'v2' --> 直接進入 v2 匯入流程（本模組不處理）
    +-- 'v1' --> convertV1ToV2Data
    +-- null --> 報錯給使用者
    |
    v
convertV1ToV2Data
    |
    +-- 提取 rawBooks
    +-- FOR EACH: convertV1ToV2Book (容錯，失敗跳過)
    +-- 收集 _v1Category (Set 去重)
    +-- convertV1CategoryToTag
    +-- 寫入 tagIds, 移除 _v1Category
    +-- 組裝 v2 interchange format
    |
    v
v2 結構 --> 交給匯入流程
```

### convertV1ToV2Book 內部流程

```
v1Book 輸入
    |
    +-- null/非物件? --> THROW BookValidationError
    +-- 缺 id/title? --> THROW BookValidationError
    |
    v
欄位逐一轉換
    +-- author: 空 -> [], 非空 -> [author]
    +-- readingStatus: 委派 BookSchemaV2.mapV1StatusToV2
    +-- progress: 委派 normalizeV1Progress + clamp(0,100)
    +-- 其他: 直接複製或給預設值
    +-- category: 移至 _v1Category
    |
    v
回傳 v2Book（不含 isNew/isFinished/category）
```

## 6. 權宜方案與技術債務

### 最小可行實作

- `convertV1ToV2Data` 的 metadata 中 `tagCategories` 只含匯入分類，不含系統預設分類（規格提到「預設 2 個系統分類」，但本模組為純轉換，系統分類應由匯入流程處理）
- tag id 生成使用 `Date.now() + 隨機碼`，非加密級隨機，足夠唯一但不保證分散

### 已知限制

- `normalizeV1Progress` 不做 clamp，本模組需自行 clamp(0, 100)
- `convertV1ToV2Data` 容錯模式下失敗書籍的錯誤資訊只記錄不回傳（符合規格但限制了上層的錯誤報告能力）

### 技術債務標記

| 項目 | 說明 | Phase 4 建議 |
|------|------|-------------|
| TD-001 | tag id 生成邏輯與 TagSchema 可能重複 | 評估是否提取共用 id 生成工具 |
| TD-002 | `_v1Category` 暫存欄位是隱式契約 | 評估是否改為明確的中間結構 |
| TD-003 | 失敗書籍資訊未回傳 | 評估是否在回傳結構加 `warnings` 欄位 |

## 7. 語言特定實作注意事項

**JavaScript / Chrome Extension 考量**:

- `BookSchemaV2` 和 `TagSchema` 使用 CJS (`module.exports`)，新模組也應使用 CJS 保持一致
- `Date.now()` 在 Service Worker 和 Content Script 中行為一致，無平台差異
- `BookValidationError` 的引入路徑需對齊現有慣例（`require('../../core/errors/BookValidationError')`）
- Jest mock 策略：使用 `jest.mock('../../../src/data-management/BookSchemaV2')` 隔離依賴
- `Map` 在所有目標 Chrome 版本中原生支援，無需 polyfill
