## 1. 實作策略設計

### 核心結構

本模組是一個**工廠函式**，接收兩個查詢函式作為依賴，回傳包含三個方法的物件。無狀態、無快取、純函式式設計。

### 虛擬碼

```
CONSTANT TAG_RESOLVER_DEFAULTS:
  FALLBACK_CATEGORY_NAME = "未分類"
  FALLBACK_CATEGORY_COLOR = "#808080"

FUNCTION createTagResolver(deps):
  // --- 參數驗證（快速失敗）---
  IF deps 為 null 或 undefined:
    THROW TypeError("createTagResolver requires deps object")
  IF deps.getTagById 不是函式:
    THROW TypeError("deps.getTagById must be a function")
  IF deps.getCategoryById 不是函式:
    THROW TypeError("deps.getCategoryById must be a function")

  // --- 從 deps 取出依賴 ---
  getTagById = deps.getTagById
  getCategoryById = deps.getCategoryById

  // --- 內部方法定義 ---

  FUNCTION resolveTag(tagId):
    tag = getTagById(tagId)
    IF tag 為 null:
      RETURN null

    category = getCategoryById(tag.categoryId)

    IF category 不為 null:
      categoryName = category.name
      categoryColor = category.color
    ELSE:
      categoryName = TAG_RESOLVER_DEFAULTS.FALLBACK_CATEGORY_NAME
      categoryColor = TAG_RESOLVER_DEFAULTS.FALLBACK_CATEGORY_COLOR

    RETURN {
      tagId: tagId,
      tagName: tag.name,
      categoryId: tag.categoryId,
      categoryName: categoryName,
      categoryColor: categoryColor
    }

  FUNCTION resolveTagsForDisplay(tagIds):
    IF tagIds 不是陣列:
      RETURN []

    results = []
    FOR EACH item IN tagIds:
      IF item 不是字串:
        CONTINUE
      resolved = resolveTag(item)
      IF resolved 不為 null:
        APPEND resolved TO results
    RETURN results

  FUNCTION resolveTagName(tagId):
    tag = getTagById(tagId)
    IF tag 為 null:
      RETURN null
    RETURN tag.name

  // --- 回傳 resolver 物件 ---
  RETURN {
    resolveTag: resolveTag,
    resolveTagsForDisplay: resolveTagsForDisplay,
    resolveTagName: resolveTagName
  }
```

## 2. 資料結構選擇

| 結構 | 用途 | 理由 |
|------|------|------|
| 物件字面值（回傳的 resolver） | 封裝三個方法 | 無需原型鏈，工廠函式回傳即可 |
| 閉包 | 持有 getTagById / getCategoryById 引用 | 避免暴露依賴，消費端只看到方法 |
| 常數物件 | TAG_RESOLVER_DEFAULTS | fallback 值集中管理，下游可引用 |

不需要 Map、Set 或其他複雜資料結構。本模組**不持有資料**，只是查詢委派和結果組裝。

## 3. 資料流程圖

```
createTagResolver
  輸入: { getTagById, getCategoryById }
  輸出: { resolveTag, resolveTagsForDisplay, resolveTagName }

resolveTag(tagId):
  tagId --> getTagById(tagId) --> tag
                                   |
                            null? --+--> RETURN null
                                   |
                            tag.categoryId --> getCategoryById --> category
                                                                    |
                                                             null? --+--> 使用 DEFAULTS
                                                                    |
                                                              組裝結果物件 --> RETURN

resolveTagsForDisplay(tagIds):
  tagIds --> 型別檢查（非陣列 → []）
         --> 逐元素過濾（非字串 → 跳過）
         --> 逐元素呼叫 resolveTag
         --> 過濾 null 結果
         --> RETURN 結果陣列

resolveTagName(tagId):
  tagId --> getTagById(tagId) --> tag
                                   |
                            null? --+--> RETURN null
                                   |
                            RETURN tag.name
```

```
  |

  +-- 非陣列? --> RETURN []

       +-- 非字串? --> SKIP
       +-- resolveTag(item)
            +-- null? --> SKIP
            +-- 有值? --> APPEND
  |
  +-- RETURN results
```

## 6. 權宜方案與技術債務

- **最小可行實作**：本模組邏輯簡單，無需權宜方案。工廠函式 + 閉包 + 陣列過濾即完成所有需求。
- **已知限制**：無快取機制（每次呼叫都查詢依賴函式）。對於目前規模（數百筆 tag）不需要快取。
- **技術債務標記**：無。本模組複雜度低，不預期產生技術債務。

## 7. JavaScript 特定實作注意事項

- **型別檢查方式**：`typeof x === 'function'` 檢查依賴、`Array.isArray()` 檢查陣列、`typeof x === 'string'` 檢查字串元素
- **模組匯出**：使用 `module.exports = { createTagResolver, TAG_RESOLVER_DEFAULTS }` (CJS)，因專案 build 工具處理模組格式
- **注意 nullish 判斷**：`getTagById` 回傳 null 時用 `=== null` 或 `== null`（含 undefined），需與依賴提供者約定一致
- **不使用 class**：工廠函式 + 物件字面值即可，無需 class 語法
