## Problem Analysis

### Context Bundle

**目標**: 提取 tag 顯示方法到 tag-cell-renderer.js，同時消除 DRY 違規（W5-005）和修復 CSS injection（W5-003）

**新檔案: src/overview/tag-cell-renderer.js**
搬遷方法（3 個）:
- _createTagCell(tagIds, tagResolver, document) — 建立 td 元素
- _appendTagChips(container, resolvedTags, document) — 填充 chips + +N 摺疊
- _createTagChipElement(tag, document) — 建立單一 chip span

**W5-005 DRY 修復**:
- 移除 Controller 的 resolveTagsForDisplay 方法
- tag-cell-renderer 改用 tag-resolver.js 的 createTagResolver
- API: createTagResolver({ getTagById: id => tagMap.get(id), getCategoryById: id => categoryMap.get(id) })
- 已有 resolveTagsForDisplay(tagIds) 回傳 [{tagId, tagName, categoryName, categoryColor}]

**W5-003 CSS injection 修復**:
- _createTagChipElement 中 categoryColor 加格式驗證
- 僅接受 #RRGGBB 格式（/^#[0-9a-fA-F]{6}$/.test(color)）
- 不符合時 fallback 為 TAG_RESOLVER_DEFAULTS.FALLBACK_CATEGORY_COLOR (#808080)

**搬遷常數**: TAG_DISPLAY（MAX_VISIBLE: 3, EMPTY_LABEL: '未分類'）

**Controller 修改**:
- require tag-cell-renderer
- constructor 建立 tagResolver（使用 createTagResolver）和 tagCellRenderer
- createBookRow 改為呼叫 tagCellRenderer
- 移除 resolveTagsForDisplay, _createTagCell, _appendTagChips, _createTagChipElement

**注意**: 不要修改測試檔案。測試中 controller.resolveTagsForDisplay 有被直接呼叫，需保留 proxy。
