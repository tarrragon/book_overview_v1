## 前情提要

v0.17.2 W2-007.2 Tag 顯示元件實作中發現：測試硬編碼色彩值，spec/CSS/test 各層各自定義色彩，缺乏 Single Source of Truth。調查後發現 89+ 硬編碼色值散落 10+ 檔案。

APP 專案（Flutter）已有完整的 8 子系統 Design System（UIColors/UISpacing/UIBorderRadius 等）。本版本將 APP 的設計規格值移植到 Chrome Extension，建立統一 Design System 基礎。

### 來源文件

| 文件 | 角色 |
|------|------|
| PROP-008 | 統一 UI Design System 提案（confirmed） |
| SPEC-DESIGN-SYSTEM | 正式規格（docs/spec/design-system-spec.md） |
| APP `lib/core/ui/ui_config.dart` | 配色/間距/圓角/陰影基準 |
| APP `lib/core/ui/flat_design_config.dart` | 語意化按鈕規格 |

### 核心設計決策（PROP-008 討論確認）

| 決策 | 結論 |
|------|------|
| 配色系統 | 三色系統：藍 90% / 綠 5% / 橘 5%，對齊 APP |
| ReadingStatus 6 狀態 | 全部映射到三色系統，廢棄紫色/青色 |
| reading 狀態色 | 藍色 #2196F3（非橘色） |
| reference 撞色修復 | 改用 primaryDark #1976D2（APP 需同步） |
| Badge 樣式 | 淡色底 15% alpha + 深色字 |
| 品牌梯度 | 紫色 → 藍色系 |
