# Execution Log

## Task Summary

設計 Ticket Context Checklist — 認領時強制補充背景資訊機制

---

## Problem Analysis

### 問題根因

W12-008 WRAP 壓測發現 W4-001 的目標檔案 `reading_status_badge.dart` 不存在於當前 codebase（Flutter APP 範疇），暴露認領檢查清單缺少「前提驗證」機制。Ticket 建立時的描述常是草稿品質，到認領時可能已過時。

### 影響範圍

- `command_lifecycle_messages.py` — 新增 3 個 checklist 常數
- `lifecycle.py` — `_print_claim_checklist()` 插入新項目
- `context-bundle-spec.md` — 補充認領時 Context 驗證流程

---

## Solution

新增 3 項 Context 驗證檢查項到認領檢查清單（所有 Ticket 類型適用）：

1. **已驗證目標檔案/模組存在** — glob/ls 確認 where.files 路徑
2. **已確認前提假設仍成立** — 架構/依賴/API 未變更
3. **已檢查跨專案/版本關聯性** — 範疇正確、無重疊 Ticket

設計決策：
- 放在「已理解驗收條件」之後、「開發環境就緒」之前
- 適用所有 Ticket 類型（非僅 IMP/ADJ），因任何類型都可能有過時前提
- 同步更新 context-bundle-spec.md，將 Context 驗證定為 Context Bundle 填寫的前置步驟
