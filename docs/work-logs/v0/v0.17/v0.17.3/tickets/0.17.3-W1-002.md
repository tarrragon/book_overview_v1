## Problem Analysis

### Context Bundle（含 W1-003 + W1-004 合併執行）

**問題 1 (W1-002)**: index.js 的 CJS 區塊為 dead code
- 頂層 `export { } from` 是純 ESM 語法，CJS 引擎在解析第 6 行就會 SyntaxError
- 第 11-16 行的 `require()` 永遠不會執行
- 對照 `src/core/enums/index.js` 和 `src/core/errors/index.js`：兩者均採純 CJS

**問題 2 (W1-002)**: 常數物件缺 Object.freeze()
- 所有 7 個常數物件可被外部 `COLORS.primary = 'xxx'` 修改
- Chrome Extension content script 與 popup 共享常數，意外修改會影響全局

**問題 3 (W1-003)**: GRADIENT 硬編碼重複 COLORS 色值
- `GRADIENT.start` = `'#2196F3'` 重複 `COLORS.primary`
- `GRADIENT.end` = `'#1976D2'` 重複 `COLORS.primaryDark`
- `GRADIENT.css` 字串也重複這兩個色值

**問題 4 (W1-004)**: 子模組 CJS shim 冗餘
- colors.js/spacing.js/typography.js 底部的 `if (typeof module !== 'undefined')` 區塊
- esbuild 以 ESM IIFE 打包，CJS shim 為 dead code
- 對照 enums/errors：均為純 CJS（module.exports），或由 esbuild 處理

**修正策略**:
1. 所有 4 個檔案統一為純 CJS（module.exports），移除 ESM export — 與 enums/errors 一致
2. 所有常數物件套 Object.freeze()
3. GRADIENT 引用 COLORS 而非硬編碼
4. index.js 改為純 CJS require + re-export
