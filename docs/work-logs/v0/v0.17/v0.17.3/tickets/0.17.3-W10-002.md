# Execution Log

## Task Summary

修復 Hook exit code 問題 — run_hook_safely 異常返回 1 導致 CLI 顯示 hook error

---

## Problem Analysis

### 問題根因

Claude Code CLI exit code 語義：exit 0 = 成功、exit 2 = 阻止操作、exit 1 = hook error。
`run_hook_safely` 異常時返回 `EXIT_ERROR=1`，CLI 把它當作 Hook 本身壞掉，吞掉所有 stdout 訊息。
此外，多個 Hook 的 ImportError 處理也使用 `sys.exit(1)`，同樣觸發 hook error。

### 影響範圍

- `hook_utils/hook_logging.py` — `run_hook_safely` 核心安全包裝
- `parallel-suggestion-hook.py` — ImportError 處理
- `lib/markdown_formatter.py` — 頂層 try-except
- `lib/` 下 6 個檔案的 `__main__` 區塊
- 3 個其他 Hook 的 ImportError 處理（hook-health-monitor, lsp-environment-check, eval_pyyaml）
- `task-dispatch-readiness-check.py` — 輸入錯誤處理
- `phase-contract-validator-hook.py` — `__main__` 區塊

---

## Solution

1. **`run_hook_safely` 異常處理**：返回 0 + JSON `additionalContext` 錯誤訊息 + stderr 雙通道
2. **ImportError 處理**：Hook 執行路徑的 ImportError 改為 `sys.exit(0)` + stderr
3. **docstring 更新**：`common_functions.py` 範例模板更新

保留的改善（Hook 執行路徑）：
- `parallel-suggestion-hook.py` — ImportError exit(0)
- `hook-health-monitor.py` — ImportError exit(0)
- `lsp-environment-check.py` — ImportError exit(0)
- `eval_pyyaml.py` — ImportError exit(0)
- `task-dispatch-readiness-check.py` — 輸入錯誤 exit(0)
- `lib/markdown_formatter.py` — 頂層 exit(0)
- `lib/common_functions.py` — docstring 範例

### WRAP 決策修正（第二次 commit）

WRAP 分析發現 IMP-049 記錄了相同修改的 2 次失敗，且規範存在 3 個矛盾：
1. `run_hook_safely` 不應繞過 CLI bug（IMP-049 教訓）
2. Hook stderr 輸出違反 methodology 禁止規則（IMP-048 教訓）
3. `__main__` CLI 工具的 `sys.exit(1)` 語義正確，不應改為 0

回退的改動：
- `hook_utils/hook_logging.py` — 恢復 `return EXIT_ERROR`（不繞過 CLI bug）
- `lib/frontmatter_parser.py` — 恢復 `__main__` 的 `sys.exit(1)`
- `lib/hook_messages.py` — 恢復 `__main__` 的 `sys.exit(1)`
- `lib/ask_user_question_reminders.py` — 恢復 `__main__` 的 `sys.exit(1)`
- `lib/parse-test-json.py` — 恢復 `sys.exit(1)`
- `phase-contract-validator-hook.py` — 恢復 `__main__` 的 `sys.exit(1)`

規範更新：
- `hook-architect-technical-reference.md` — 修正 exit code 表 + 區分 Hook 路徑 vs `__main__`
- `hook-system-methodology.md` — stderr 規範加入 `__main__` 例外 + 與 quality-baseline 關係說明
