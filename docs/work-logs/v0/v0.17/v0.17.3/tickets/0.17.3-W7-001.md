## Problem Analysis

## 背景分析

### 問題根因
v0.17.2/v0.17.3 因 Claude Code subagent 回合限制（~20 tool calls），PM 工作流程大幅調整，新增 Context Bundle、二階段分工、tool call 預算模型等規則。決策分支隨之增多，主檔案 decision-tree.md（311 行）包含 8.5 層決策邏輯。

### 現狀評估
1. 主檔案 311 行，PM 每次接收訊息需讀全檔案，但實際只走一個分支
2. 代理人只需路徑表（15 行），卻要讀 311 行
3. 多個檔案引用錯誤路徑 rules/core/decision-tree.md（實際在 pm-rules/）
4. 91 個檔案引用決策樹，引用密度高

### 探索過程（3 個並行 Explore agent）
- Agent 1：讀取決策樹完整內容、分析層級結構、引用關係
- Agent 2：調查 v0.17.2/v0.17.3 PM 工作流程調整、Context Bundle、二階段分工
- Agent 3：分析代理人如何使用決策樹、發現路徑引用錯誤

### 決策
按「決策路徑」而非「DDD domain」進行二元化拆分：
- 主檔案精簡為純路由索引（~90 行）
- 按二元分支建立 5 個子檔案
- 代理人路徑表獨立為 Source of Truth 檔案
- 同時修復所有已知的錯誤引用路徑
