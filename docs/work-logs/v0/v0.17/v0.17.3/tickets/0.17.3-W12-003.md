## Problem Analysis

### 用戶設計方向（待補充）

**核心概念：擴增選項（Widen Your Options）**

引用：青少年做選擇時幾乎是盲目的，心智聚光燈只照到眼前的一個選項（「要不要去派對？」），不會去想其他選項。開竅的做法是讓聚光燈動一動，看到更多可能性。

**對應 PM 的問題**：
- PM 遇到「CLI 沒有 API」→ 聚光燈只照到「直接查詢 API」這一個選項
- 預設只有「能做 / 不能做」的二元判斷
- 沒有擴增選項：Hook 推送？檔案追蹤？流程繞過？既有 API 接線？

**設計方向**：
- 思考的第一階段：擴增選項
- 不要預設只有一種做法
- 不要只問「要不要做」「能不能做」
- 要問「還有什麼其他方式可以達成目標？」
