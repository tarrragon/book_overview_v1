## Problem Analysis

### Context Bundle

#### 問題根因
3 個 Service 使用 iOS 原生色系硬編碼，與 design-system Material Design 色系不一致。用戶決策：統一為 Material Design 色系。

#### 修改範圍（4 個檔案）

**1. src/core/design-system/colors.js（擴充）**
需新增：
- `error: '#F44336'`（Material Design Red 500）
- `errorLight: '#FFCDD2'`（用於 dark theme error 對應）
- dark theme 色板不在此新增（theme-management-service 直接引用 COLORS 即可）
- `border: '#E0E0E0'` 和 `borderDark: '#424242'`（邊框色）

**2. theme-management-service.js（L64-93）**
light theme 映射：
- primary: COLORS.primary (#2196F3)
- background: COLORS.surface (#FFFFFF)
- surface: COLORS.background (#FAFAFA)
- text: COLORS.onBackground (#212121)
- textSecondary: COLORS.onSurfaceMuted (#757575)
- border: COLORS.border (新增)
- success: COLORS.positive (#4CAF50)
- warning: COLORS.negative (#FF9800)
- error: COLORS.error (新增)

dark theme 映射：
- primary: COLORS.primaryMedium (#64B5F6)
- background: '#121212' (Material dark bg)
- surface: COLORS.onBackground (#212121)
- text: '#FFFFFF'
- textSecondary: COLORS.onSurfaceMuted
- border: COLORS.borderDark (新增)
- success: COLORS.positiveDark (#388E3C)
- warning: COLORS.negativeDark (#F57C00)
- error: COLORS.error

**3. notification-service.js（L60-78）**
- info color: COLORS.primary
- success color: COLORS.positive
- warning color: COLORS.negative
- error color: COLORS.error

**4. system-error-handler.js（L402-407）**
- HIGH: COLORS.error
- MEDIUM: COLORS.negative
- LOW: COLORS.positive
- fallback: COLORS.onSurfaceMuted
