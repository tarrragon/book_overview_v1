- 凡 IMP ticket 修改 src/ 字串輸出字面（log 前綴、錯誤訊息、UI 文案），acceptance 必含 `npm test`（或對應測試套件），不可只驗 `npm run build:dev`
- Why：tests/ 斷言期待值（toHaveBeenCalledWith/toContain）與 src 字串同源，build 綠燈不覆蓋斷言一致性
