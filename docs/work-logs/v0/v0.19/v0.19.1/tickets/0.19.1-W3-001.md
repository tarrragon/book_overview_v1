## Solution

### Wave B 視覺重塑方向（D5 配色策略 + 字級層次 + 卡片權重，saffron 定案）

用戶已授權配色自主定（不需確認）。以下決策基於 impeccable 產品 register heuristic + WCAG AA（全部已 Python 計算驗證）+ design-system token 架構相容性。落地路徑 = 改 colors.js（+ 必要時 spacing/typography 微調）→ 重新產生 design-system.css → overview.css / popup.html 視覺套用（D1 廉價操作）。

### a. 配色策略決策（commitment axis = Restrained）

**選定 Restrained**（impeccable 產品 register 明示「Product defaults to Restrained ... Restrained is the floor」）：tinted neutrals + 單一 accent，accent 僅用於 primary action / 當前選取 / 狀態指示，不作裝飾。

**跳脫 category-reflex**：原 #2196F3 是「書庫工具 → Material 飽和天藍」的第一序訓練反射。重塑改用更深、略降飽和、偏 slate-indigo 的 **primary #1A56DB**——對資料工具仍是可信賴的藍（不換 hue family 避免二序反射「避開藍就一定用綠/紫」），但離開飽和天藍 lane，配合中性 surface 系統使藍不再大面積氾濫（藍只出現在 primary 按鈕 + 統計數字 + 當前選取，面積 < 10%）。

**核心結構轉變**：氾濫的根因不是「藍不好」，是「藍貼到所有元素」。Restrained 的解法是建立中性 surface 層承載 90% 的面積（canvas / surface / panel 三級灰），藍退回 accent 角色。

### b. 字級層次（type scale，相容 typography.js）

detector 命中 popup flat-type-hierarchy（11/12/14/18 步階過近）。產品 register 用 fixed rem scale + 1.125~1.2 比例。typography.js FONT_SIZES 既有值大致合理，問題在「使用端只用了 11/12/14/18 四個近接級別」，未拉開對比。

**type scale 套用規範（不改 typography.js 數值，改使用端對應）**：

| 層級 | token | px | weight | 用途 |
|------|-------|----|--------|------|
| headline | --font-size-headline3 | 24 | bold(700) | overview 頁標題 / 統計大數字 |
| title | --font-size-title-medium | 18 | semiBold(600) | popup h1 / 卡片標題 / 區塊標題 |
| body-large | --font-size-body-large | 16 | regular(400) | 主要 body / 按鈕（primary）|
| body | --font-size-body-medium | 14 | regular/medium | 一般 body / 次要按鈕 / 表格 |
| caption | --font-size-caption | 12 | regular(400) | 標籤 / 輔助說明 / info-text |
| overline | --font-size-overline | 10 | medium(500) | 罕用微標籤 |

**對比改善**：明確用「24 / 18 / 16 / 14 / 12」五階（headline:title = 24:18 = 1.33、title:body = 18:14 ≈ 1.29），跳過 popup 現用的 11px（升為 12 caption 統一）。weight 對比同步拉開：標題用 600~700，body 用 400，避免「全 14px 不同灰」的扁平感。

### c. 卡片視覺權重（陰影/間距/尺寸差異化，取用 shadows token）

根因：overview 統計/表格/上傳區同階；popup 4 張 status-card 同白同陰影同 radius。impeccable absolute ban「Identical card grids」。

**權重三級（用既有 shadows token sm 0.1 / md 0.25 + 間距 + 邊框差異化）**：

| 權重 | 視覺處方 | 套用對象 |
|------|---------|---------|
| 主（primary surface） | surface 白底 + box-shadow 0 2px 8px var(--shadow-color-sm) + padding lg(24) + radius md(12) | overview 書籍表格（核心內容）/ popup 主操作卡 |
| 次（secondary panel） | panel 底 #EDF1F7 + 無陰影或 1px border var(--color-border) + padding md(16) + radius sm(8) | overview 上傳區 / 搜尋區 / popup 狀態卡 / 頁面資訊卡 |
| 平（inline/flat） | 無卡片容器，直接置於 canvas，僅靠間距分組 | overview 統計區（見下）/ popup 版本資訊 |

**overview 統計區去 hero-metric 化**：移除藍 slab，改為 canvas 上的 inline 數字組——大數字用 --color-primary（accent，5.76:1 on canvas）+ caption 標籤 muted，數字間用 spacing-xl 分隔，無背景色塊。消除「藍 slab + 大數字」模板，同時讓藍只出現在數字（accent 角色）。

**overview 表格權重提升為主元素**：表頭由 --color-primary 藍改 --color-panel 中性底 + ink 文字（th-text 14.03:1），表格本體保留唯一較重 shadow（0 2px 8px sm），使表格成為頁面視覺重心（資料工具的核心是資料，非按鈕）。

**popup body 去藍**：body background 由 --color-primary 改 --color-background（canvas #F5F7FA），白卡不再漂浮藍場，版本文字改 muted on canvas（5.51:1，修復對比失敗）。主操作卡（含 primary 按鈕）給 sm shadow 提升為主，其餘卡降為 panel 底次階。

**按鈕主次三級（語意化，非全同色）**：

| 級別 | 樣式 | 用途 |
|------|------|------|
| primary | --color-primary 底 + 白字 + radius sm(8，非 pill) | 每頁唯一主動作（overview「匯出 CSV」或無主動作時留空；popup「開始提取書庫資料」）|
| secondary | --color-panel 底 + ink 字 + 1px border | 一般動作（匯出 JSON / 匯入 JSON / 設定 / 說明 / 檢視書庫）|
| danger | --color-error 底 + 白字 | 破壞性（清空書庫等二次確認；現 modal-btn-danger 已具雛形）|

**radius 去 pill 化**：export-btn / searchBox 由 radius-xl(20，pill) 改 radius-sm(8)，對齊類別最佳工具（Linear/Notion/Stripe 約 6-8px），pill 對資料工具顯 toy-like。此為 CSS 套用端改 var()，token 值不變。

**error-card 按鈕收斂**：popup error-card 現 6 按鈕（達 cognitive load 上限），降為 1 primary（重試）+ 1 secondary，其餘收進「更多」或移除（spawn IMP 處理）。
