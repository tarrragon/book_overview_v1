### W1-014 延伸評估補充：併入本 ANA base 設計範圍的缺陷（K/L/M）

W1-014 complete 後的延伸評估發現以下 3 項與 base snapshot 設計直接相關，併入本 ANA 範圍一併設計（PM 預填 context，接手者應獨立驗證行號）：

- **K（push 預設只增不刪）**：`clean_stale_files` 僅 `--clean` 模式呼叫（push.py:879-882），預設 push 不移除上游已不存在於本地的檔 → 本地刪除不傳上游、上游累積 cruft。base 模型下 push 應以 git 追蹤態 delta 表達刪除（與 C1 共構）。
- **L（pull 無原子套用/回滾）**：pull backup→overwrite→delete→chmod 中途失敗無自動回滾（pull.py:516-572），半同步狀態僅手動還原。三方合併設計需含失敗回滾策略。
- **M（.sync-state.json 只記 push）**：sync-state 僅 push 成功後寫（push.py:963-974），pull 從不記錄 → 無 pull 側 base 錨點。base snapshot（A3/B3）的前提即 pull 也必須記錄上次同步的上游 commit SHA，本項為結構前提。
