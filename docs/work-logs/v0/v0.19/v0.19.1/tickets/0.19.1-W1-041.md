新增 IMP-079：批次替換工具誤傷偵測目標字面

路徑：.claude/error-patterns/implementation/IMP-079-batch-replace-tool-corrupts-detection-target-literals.md

主要內容：
- 症狀：批次替換工具（scripts/remove-emoji.js）對 tests/ 全量執行時，tests/unit/background/background-local-dict.test.js L508-518 antiPatterns 陣列中 10 個 regex 字面內嵌的 emoji 偵測目標被盲目轉換，造成兩種失效模式：(a) 映射內字元替換為 [KEYWORD] 後字元類別 [...] 誤匹配英文字母；(b) 映射外字元移除後 regex 塌縮過寬
- 根因：工具設計無語境感知，無法區分「被替換的業務字面」與「用於偵測目標的 regex 字元字面」
- 案例：W1-005.3，以 codepoint 描述 emoji 字元（遵守 language-constraints 規則 3 禁止 emoji 字元）
- 防護三層：(1) 工具排除清單（根本解）；(2) tests/ 批次替換前 diff 審查 regex 字面；(3) 警示註解（tests/unit/background/background-local-dict.test.js L499-500 已有既有警示可引用）
- README 索引：IMP-079 條目已加入

關聯：PC-165（上位 pattern），PC-158（弱相關）
