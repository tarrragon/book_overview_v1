## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

push 舊流程以 copy_filtered 從**磁碟** walk `.claude/`，產生兩個根因問題（W1-018 flaw C）：
1. **無 git 真相源**：磁碟內容包含 untracked / gitignored 檔，排除僅靠檔名黑名單 → gitignored 機密檔（檔名不在黑名單，如 my-api-token.txt）會被靜默推上公開 repo（W1-019 secret-leak，靠 detect_secret_leak_risk interim 防護擋）。
2. **無刪除傳播**：磁碟上仍存在的檔不會反映 git rm；遠端殘留需 --clean 比對磁碟。

### 影響範圍

- .claude/scripts/sync-claude-push.py（主要重寫：copy_filtered → git archive；移除 detect_secret_leak_risk 死碼）
- .claude/hooks/lib/sync_exclude_manifest.py（LOCAL_ONLY 加 .sync-conflicts）
- .claude/scripts/README-subtree-sync.md（commit-first 行為 + git-archive 來源說明）
- 測試：新增 test_sync_claude_push_git_archive.py；移除 test_sync_claude_push_secret_leak.py

### 相關 Error Pattern

W1-019（secret-leak interim 防護）已由 C1 從架構層消滅。ARCH-020（排除邏輯 SSOT manifest）持續沿用 W1-027 should_exclude，不重寫。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要（C1 + K + M1 + M2 + M5 + B3）

- **C1（git archive）**：新增 `stage_tracked_tree(project_root, staging_dir)` 以 `git archive --format=tar HEAD -- .claude` 取 git tracked 樹，tarfile 解壓 strip `.claude/` 前綴到 staging。取代 copy_filtered 磁碟 walk。
- **M1（過濾保留）**：新增 `copy_filtered_from_staging(src, dst)` 從 staging 複製到遠端 temp_dir 時施加 `should_exclude`（相對路徑），tracked 但屬 local-only / 憑證者（settings.local.json / .sync-state.json / 憑證）仍被擋。git archive 只換來源不換過濾。
- **K（刪除傳播）**：git rm 的檔不在 archive → 不入 staging；`clean_stale_files` 改以 staging（tracked 樹）為 reference（原為本地磁碟），並對 should_exclude 命中者跳過刪除（避免誤刪其他專案內容）。
- **M2（移除死碼）**：刪除 `detect_secret_leak_risk` 函式與 main step 2.2 abort 區塊。C1 從架構層消滅 untracked/gitignored 洩漏（不在 tracked 樹即不可能被推），interim 防護無存在必要。
- **M5（commit-first）**：新增 `ensure_committed(project_root)` 以 `git diff --quiet -- .claude` + `git diff --cached --quiet -- .claude` 雙維度確認已 commit；main step 2 改用之，未 commit 即 abort。docstring + README 補述 commit-first 行為差異與根因（push 取 HEAD tracked 樹）。
- **last_synced_base_sha（W1-025 schema）**：新增 `read_base_sha` / `write_base_sha`（與 pull 對稱，寫同一單一欄位，禁雙欄位）。push 成功後 `git rev-parse HEAD`（遠端 clone）作 base SHA 寫入；既有 last_push_hash/version/time 改為 update 保留（原為整檔覆蓋會遺失 base SHA）。
- **.sync-conflicts EXCLUDE**：manifest LOCAL_ONLY_PATTERNS 加入 .sync-conflicts（push/pull 皆不同步）。
- **B3（VERSION 純鏡像）**：docstring 標明本地 .claude/VERSION 永不手改、純鏡像上游，base 錨點由 .sync-state.json SHA 承擔；push 流程仍在 clone 上 bump 遠端 VERSION（既有邏輯不變）。

### 向後相容

無 base 時 `read_base_sha` 回 None（首次推送 / 無 .sync-state.json / 解析失敗）；push 流程不依賴 base 進行 archive（archive 永遠取 HEAD 全量 tracked 樹），故無 base 時行為一致。

### W1-016 B2 覆蓋確認

B3「本地永不手改 VERSION」即 W1-016 的 B2。本 ticket 在 push docstring 顯性標明此約束（VERSION 段落），但**未**在 push 流程加入「拒絕本地 VERSION 手改」的強制檢查——push 取 tracked 樹會把本地 VERSION 一併帶入 staging，隨後在 clone 上以 bump 後值覆寫遠端 VERSION，故本地 VERSION 手改值不會污染遠端。W1-016 若需強制阻擋本地手改 VERSION（如 commit hook 層），屬獨立範圍，本 ticket 僅以文件約束涵蓋 B2 的「規範層」，強制層留待 W1-016 評估。
