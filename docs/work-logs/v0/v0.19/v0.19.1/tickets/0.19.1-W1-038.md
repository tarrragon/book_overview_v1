### 問題根因

`tests/test_monorepo_version_sync.py` 是過時測試，非待實作功能。

- 該檔 import `get_monorepo_version` / `check_monorepo_version_sync`（line 16-26），但兩函式已於 commit `caa4618f`（W4-002「修正 version-release Skill 路徑/專案類型/CLI 不一致」）移除，重構為 `check_version_sync_dual`（`git show caa4618f` 確認 `-def get_monorepo_version` / `-def check_monorepo_version_sync` / `+def check_version_sync_dual`）。

- 測試未隨 W4-002 重構同步更新，遺留 import error 造成 pytest collection 失敗，全套件需 `--ignore` 排除。
