### 變更摘要

複用 W1-005.2 產出的 `scripts/remove-emoji.js`（同源 emoji→[KEYWORD] 映射），apply 到 tests/ 的 127 個 active 測試檔（112 檔實際修改），同時完成「修斷言 emoji」與「清裝飾 emoji」。斷言期待值因此自動對齊 src 新格式，消除 W1-005.2 引入的隱性回歸。

### 三項主要變更

1. **tests/ emoji 對齊 src（113 檔修改）**
   - 範圍：rg 列舉 active 測試檔，glob 排除 `*.backup-*` / `*.bak` / `*.tmp` / `*.deprecated` + `tests/helpers/*-data-generator.js` + `tests/fixtures/*.json`
   - 語意前綴 emoji（✅→[OK]、❌→[FAIL]、⚠→[WARN] 等）轉 [KEYWORD]，裝飾 emoji 純移除，與 src 同源映射一致
   - 12 檔約 48+ 處斷言內 emoji（toHaveBeenCalledWith/toContain）對齊 src 的 [KEYWORD] log 輸出

2. **122 垃圾備份檔刪除**
   - `git rm` 122 個 git-tracked 備份檔（110 `.backup-N` + 11 `.bak` + 1 `.tmp`）
   - 新增 `.gitignore` 規則 `*.backup-*` / `*.bak` / `*.tmp` 防再生

3. **test-data/fixtures 書名 emoji 保留**
   - data-generator（書名含 📚🚀🔖✨）、fixtures、以及 overview-import-function.test.js 的書名 round-trip 斷言（測試書籍 🔥 / 🌟Unicode測試 🇹🇼）均保留——這些 emoji 不在 remove-emoji.js 映射表，腳本正確區分 log 前綴 emoji（轉換）vs 資料 emoji（保留）

### 關鍵發現：meta-test regex 字面被腳本誤轉（手動修復）

`tests/unit/background/background-local-dict.test.js` 含「PC-165 反模式 grep」meta-test，其 antiPatterns 陣列在 **regex 字面內** 嵌入原始 emoji 作為偵測目標（如 `/log\.error\(`?❌\s*\[緊急模式\]/`）。remove-emoji.js 無法區分「程式碼/字串字面內的 emoji」與「regex 偵測目標 emoji」，盲目轉換造成兩類破壞：

- 映射內 emoji（❌→[FAIL]、🚨→[ALERT] 等）→ 變成 regex **字元類別** `[FAIL]`（匹配 F/A/I/L 任一字元），誤匹配合法 log call → false positive
- 映射外 emoji（🏁🎉💥📦📨）→ 純移除後 regex 塌縮為過寬的 `/log\.(info|error|warn)\(['"`]/`，匹配任意 log call

**修復**：手動還原 antiPatterns 為原始 emoji 字面（偵測目標應為「src 殘留 emoji 前綴 log」），並加註解標明此處 emoji 為偵測目標、禁經 remove-emoji 轉換。line 428 的 `EMERGENCY_ERROR: '[FAIL] [緊急模式]...'` 因 src dict 實際已是 [FAIL] 前綴，保留轉換結果（與 src 對齊正確）。
