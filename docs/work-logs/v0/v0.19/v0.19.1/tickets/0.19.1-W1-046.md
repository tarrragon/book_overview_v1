## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W17-159 / W17-160 當時 CC runtime 的 SubagentStop event schema 不允許 `hookSpecificOutput.additionalContext`，導致 2 個 SubagentStop hook 被迫改用 top-level `systemMessage` 作為 workaround。CC 2.1.163 #4 已解除此限制，systemMessage workaround 成為過時技術債（且 systemMessage 注入 PM context 的可靠性不如 additionalContext）。

### 影響範圍

- `.claude/hooks/subagent-stop-dispatch-cleanup-hook.py`（dispatch 清理 + [OK]/[WAIT] 廣播輸出）
- `.claude/hooks/agent-commit-verification-hook.py`（commit/merge 驗證警告輸出）
- `.claude/hooks/hook_utils/hook_io.py`（新增共用版本偵測 + 輸出 helper）
- 對應 2 個 schema 測試檔 + 新增 helper 測試檔

### 相關 Error Pattern

- quality-baseline 規則 4（Hook 失敗必須可見）— additionalContext 為正確注入機制
- PC-104（agent execution boundary）— commit 驗證結果注入 PM context

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 變更摘要

1. 新增共用 helper（`hook_utils/hook_io.py`，並 re-export 至套件 `__init__.py`）：
   - `_parse_version_tuple(raw)`：解析 `claude --version` 輸出為 `(major, minor, patch)`。
   - `get_claude_code_version(logger)`：subprocess 呼叫 `claude --version`，進程內快取；失敗回 None。
   - `supports_subagent_stop_additional_context(logger)`：版本 >= 2.1.163 回 True；偵測失敗採樂觀預設 True。
   - `build_subagent_stop_output(context, logger)`：依版本回傳對應 shape。
2. `subagent-stop-dispatch-cleanup-hook.py:107` → 改用 `build_subagent_stop_output(context, logger)`。
3. `agent-commit-verification-hook.py:651` → 改用 `build_subagent_stop_output(combined_message, logger)`。
4. 3 檔 header（2 hook + DEFAULT_OUTPUT 區段註解）過時的「SubagentStop schema 不允許 additionalContext（W17-159/W17-160）」改記「CC 2.1.163 #4 已解禁，改回 additionalContext + graceful fallback」。
5. 2 個 schema 測試檔重寫斷言為 additionalContext shape + 新增 legacy fallback case；新增 `test_subagent_stop_output_helper.py` 覆蓋版本偵測。

### graceful fallback 取捨（acceptance #2）

- 採 graceful fallback（非強制 `requiredMinimumVersion`）：runtime 版本以 `claude --version` 偵測，>= 2.1.163 用 additionalContext，否則降級 systemMessage。
- 偵測失敗（binary 不在 PATH / subprocess 逾時 / sandbox）採「樂觀預設 additionalContext」——2.1.163 為向後新基線，避免長期沿用過時 workaround；舊版實際拒絕僅退化為一次非致命 hook error。
- 不採強制 `requiredMinimumVersion` 的理由：本框架跨專案 sync，硬鎖最低版本會在尚未升級的專案造成 hook 全面失效（高破壞性），graceful fallback 風險可控。

### runtime pilot 受限說明

本 subagent 的 SubagentStop 完成時這 2 hook 會觸發，但 subagent 無法觀測自身輸出是否到達 PM 主線程 context。已透過直接 stdin 灌入 dummy input 做 live 煙霧測試（見 Test Results），確認 CC 2.1.163 下實際輸出 additionalContext shape 且 exit 0 無 hook error。**完整 runtime pilot（內容是否進入 PM context）由 PM 在本 subagent 回報後觀測**，acceptance #3 標為 PM 待觀測。

### 不在本 ticket 範圍

- Stop event hook（`stop-worklog-handoff-sync-check-hook.py` 等）：W1-044 ANA 判定 Stop 有 unwanted turn continuation 風險、不採用；其 header「Stop event schema 不允許 additionalContext」為 Stop event 正確陳述，未修改。
