### 實作摘要

session-start-sync-exclusion-check-hook.py 新增 gitignore↔manifest 交叉驗證層（W1-024 方案 C / 多視角 M2）。

1. **manifest 新增具名集合 `GITIGNORE_EXPECTED`**（`.claude/hooks/lib/sync_exclude_manifest.py`）= `LOCAL_ONLY_PATTERNS`。不含 CREDENTIAL_PATTERNS（憑證涵蓋由 .env/secret 專用 glob 處理，且部分為目錄層級，語意不同）。hook import 此集合，不重寫分類（ARCH-020）。

2. **gitignore 解析正規化（M2 避免 false positive）**：`_normalize_gitignore_entry()` 統一 glob / 目錄 / 檔名三形式——去前後空白、忽略註解（#）與空行、strip 前綴否定符 `!`、strip `**/` glob 前綴、strip `.claude/` 前綴、strip 尾端 `/`，收斂為裸名比對。

3. **漂移偵測**：`find_gitignore_drift()` 回傳 `GITIGNORE_EXPECTED - covered` 排序清單；.gitignore 不存在/讀取失敗時保守視為全部缺項。

4. **warning 輸出**：`build_gitignore_drift_section()` 列缺項；`build_hook_output()` 改為可並存「未排除檔案」+「gitignore 漂移」兩警告區塊；皆無異常時靜默（suppressOutput=True）。`main()` 以 try/except 包裹交叉驗證，失敗降級不阻塞 session。

### 真實漂移發現（true positive）

對真實 .gitignore 跑交叉驗證偵測到缺項 `handoff`：manifest 將 `handoff` 列為完整 local-only，但 .gitignore 僅 ignore 子路徑 `.claude/handoff/archive/`，且 `.claude/handoff/archive/*.json` 已被 git track（早於 gitignore 規則 commit）。此為交叉驗證設計目標所要攔截的真實漂移（非 false positive），hook 以非阻塞 warning 正確列出。子路徑 ignore 不等於頂層 ignore，故不收斂為涵蓋。

### 不在本 ticket 範圍

`handoff` 漂移的根本對齊（補 `.claude/handoff/` 規則或 git rm tracked archive）屬 .gitignore/git 狀態調整，與本 ticket（hook 交叉驗證實作）邊界不同，應另建 ticket 追蹤（quality-baseline 規則 5）。
