### 問題根因

overview 與 popup UI 散落 emoji，違反 language-constraints 規則 3（禁止所有文字輸出使用 emoji），造成跨平台渲染不一致。

**發現 1：原 where.files 與實際 emoji 分布嚴重不符**

掃描指令：`rg -P -c '[\x{1F000}-\x{1FFFF}\x{2600}-\x{27BF}]' src/overview src/popup`（ticket 明定範圍）

| 目錄 | 實際有 emoji 的檔（命中數） |
|------|--------------------------|
| src/overview | overview.html(10), overview-page-controller.js(4) |
| src/popup | popup.js(30), popup.html(19), popup-initialization-tracker.js(7), popup-event-controller.js(4), popup-error-handler.js(3), utils/event-manager.js(2), popup-diagnostic-enhancer.js(1), popup-controller.js(1) |

原 where.files 列的 `overview.js`/`overview.css`/`popup-ui-components.js` 實際 **0 emoji**；多個有 emoji 的檔未列入。acceptance「目錄全域 grep=0」故真實範圍為目錄級（10 檔 ~81 個）。

**發現 2：emoji 分兩類，第二類觸及契約**

- UI-facing emoji（overview.html / popup.html 標題、按鈕）→ 安全移除，無爭議。
- console.log message 前綴 emoji（popup.js 等，如 `'✅ Popup Script 載入完成'`、`'❌ 提取錯誤詳情'`）→ 屬 `docs/spec/extraction/e2e-contract.md` §3.4「結構化 Console Log 前綴規則」契約字面。

**發現 3：e2e-contract §3.4 契約衝突**

契約明定「結構化前綴必須**完全相等**（含 emoji、空格、冒號）才視為合法 E2E 觀察點」「修改前綴文字時必須同步更新本表 + 所有相關 E2E 觀察測試」。契約含 `❌ <名詞>失敗:` 等多處 console.error 前綴。

**發現 4：E2E 測試實際依賴 popup emoji 字串**

`'Popup Script 載入完成'`、`'Popup Interface 初始化完成'` 各被 1 個測試檔引用 → 移除 emoji 可能破壞 acceptance 4「npm test 全綠」。

## Context Bundle

### 選定機制（用戶決策 2026-06-05）

採 **方案 A：共用偵測腳本 + pre-commit + CI 雙防線**。

選擇理由：原始 emoji 違規大量在 `.html`（overview.html 10、popup.html 19），但專案 ESLint 體系（lint-staged + lint.yml）**只掃 JS**，ESLint 自訂規則會漏 html/css。獨立偵測腳本可掃 js/html/css 全檔型，且接入 pre-commit（本機第一道）+ CI（安全網），符合 CLAUDE.md §5 雙防線哲學。

### 檔案 1：scripts/check-no-emoji.js（新增，CLI + 可測函式）

仿 `scripts/remove-emoji.js` 單檔模式，但**必須加 `require.main === module` 守衛**（remove-emoji.js 無條件呼叫 `main()`，import 會觸發 CLI exit；本檔須匯出偵測函式供測試 import 而不執行 CLI，參考 IMP-074 dual import mode 教訓）。

匯出：`containsEmoji(text) -> boolean`、`findEmoji(text) -> [{line, col, char, codepoint}]`（供精準報告與測試）。

CLI 行為：
- `node scripts/check-no-emoji.js <file...>`：檢查指定檔（lint-staged 傳 staged 路徑用）
- `node scripts/check-no-emoji.js`（無參數）：掃預設 glob `src/**/*.{js,jsx,ts,mjs,cjs,html,css}`（CI 全量掃描用）
- 發現 emoji → 印 `file:line:col 命中 U+XXXX` + exit 1；乾淨 → exit 0

### 偵測 pattern（關鍵正確性，務必精確）

偵測集 = 兩個範圍 + 9 個特定字 + FE0F：
- Range `U+1F000-1FFFF`
- Range `U+2600-27BF`（已涵蓋 SEMANTIC/DECORATION 落在此段者：U+2705 U+274C U+26A0 U+26A1 U+2753 U+2699 U+2728 U+267F U+2696 U+2695）
- **9 個特定字**（不可用整段 U+2139-25B6 範圍）：U+2139 U+2328 U+23ED U+23F0 U+23F1 U+23F3 U+23F8 U+23F9 U+25B6
- `U+FE0F` variation selector（孤立殘留也算違規）

**為何不可用整段 U+2139-25B6**：該整段含箭頭（U+2190-21FF）、數學運算子（U+2200-22FF）、雜項技術符號（U+2300-23FF）、圈號字母（U+2460-24FF）等大量**非 emoji 合法字元**，整段納入會誤判。權威字元清單對照 `scripts/remove-emoji.js`（SEMANTIC + DECORATION + RETRY_EMOJI）。

### 檔案 2：tests/unit/scripts/check-no-emoji.test.js（新增）

import check-no-emoji.js 匯出的函式（不觸發 CLI）。

正例（必須被偵測）：各範圍代表字（U+1F680 火箭、U+2705、U+26A0、U+25B6、U+2139、U+2328）、含 FE0F 的序列、孤立 FE0F。

反例（必須通過，**防誤判是本測試重點**）：
- ASCII 替換標記 `[OK]` `[FAIL]` `[WARN]` `[CHECK]` `[START]`
- 繁體中文段落（U+4E00-9FFF 不在偵測集）
- 一般標點：em-dash U+2014、省略號 U+2026、箭頭 U+2192、數學 U+2264（皆在 U+2139-25B6 整段內但非 9 特定字 → 驗證未用整段範圍）

### 範圍決策（為何只掃 src/）
