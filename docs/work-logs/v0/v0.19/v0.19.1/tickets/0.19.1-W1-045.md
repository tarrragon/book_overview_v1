## Problem Analysis
<!-- Schema[DOC/Problem Analysis]: 選填（若 DOC 起因於缺陷或盤點結論可填）（.claude/pm-rules/ticket-body-schema.md） -->
### 背景

source: 0.19.1-W1-043 採用項 #9。CC 2.1.163 #9 修復 2.1.154 引入的 regression——$TMPDIR 被覆寫為 /tmp/claude-{uid}（套用於所有 bash 命令，而非僅 sandboxed）。

### 待答問題

PC-157（chrome-devtools-mcp install_extension 必須用 $TMPDIR 規避 workspace roots 拒 /tmp）的 workaround 是否受 #9 regression 影響？修復後是否仍有效或可簡化？

### 關鍵機制

- PC-157 workaround = 解壓 extension 到 $TMPDIR（macOS=/var/folders/.../T/）而非 /tmp，因 MCP workspace roots 含 $TMPDIR（名為 "temp"）但不含 /tmp。
- MCP workspace root "temp" 由 host（Claude Code）設定 = 真實系統 $TMPDIR（/var/folders/.../T），不受 bash $TMPDIR 覆寫影響。
- 2.1.154 regression：bash 命令的 $TMPDIR 被覆寫為 /tmp/claude-{uid}。

## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->
### 實測結果

| 項目 | 結果 |
|------|------|
| 本 session CC 版本 | 2.1.163（#9 已含） |
| `echo "$TMPDIR"` | `/var/folders/g3/hlvjn2px3p74gdcq4l_2_wz80000gn/T/`（真實 macOS 系統 temp） |
| 與 PC-157 記載 workspace root "temp" 比對 | 一致（PC-157 錯誤訊息列 `file:///var/folders/.../T` name=temp） |
| 完整 install_extension live test | 未重跑（需 build:dev + Chrome；deductive chain 已完整：$TMPDIR=workspace-root-temp → install_extension 接受該路徑） |

### 結論：workaround 仍有效，不可簡化

1. **仍有效**：2.1.163 #9 修復後 bash `$TMPDIR` 解析回真實系統 temp（= MCP workspace root "temp"），PC-157 的「用 $TMPDIR 而非 /tmp」workaround 正確。
2. **不可簡化**：仍須用 $TMPDIR（不能改用 /tmp，workspace roots 不含 /tmp）。
3. **新增 regression window 風險記錄**：2.1.154–2.1.162 期間 bash `echo $TMPDIR` 返回被覆寫的 `/tmp/claude-{uid}`（在 /tmp 下），會破壞 PC-157 Action「echo $TMPDIR 取路徑」步驟 → install_extension 仍被拒。版本無關的可靠 fallback 為 PC-157 既有指引「讀錯誤訊息的 workspace roots 清單，從中挑路徑」。

### PC-157 更新

於 PC-157 Action 表「取得正確 TMPDIR 路徑」列新增版本註記：CC ≥ 2.1.163 `echo $TMPDIR` 可靠；2.1.154–2.1.162 因 regression 返回被覆寫路徑，此窗口期改以錯誤訊息的 workspace roots 清單為權威來源。
