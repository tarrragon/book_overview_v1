### WRAP 三問（認領 Checkpoint）

- **W（Widen）**：候選方案 (1) 將 scripts/ 全部 console 改為 logger 並逐項修；(2) 為 scripts/** 加 eslintConfig override 關閉 no-console（CLI 工具本職輸出），manual 修真實錯誤。選 (2)：scripts/ 為 Node CLI/build 工具，console 為其正當輸出，與既有 config 中 `tests/e2e/performance/** → no-console: off` override 同精神。
- **A（Attain distance）**：機會成本 = W2 其他清理任務；本 ticket 為 quality-baseline 規則 5 技術債，優先處理避免違規再累積。
- **P（Prepare to be wrong）**：最可能失敗原因 = scripts/ 違規量遠超 ticket 估計（實測 2330 problems vs 估 150），可能漏修或 --max-warnings=0 因 no-console warn 無法歸零。防護 = 對 scripts/** 加 no-console override 從根本消除 console warning，再逐項 manual 修剩餘真實錯誤。

### 問題根因

scripts/ 目錄從未納入 `npm run lint`（範圍僅 `src/ tests/`），累積大量 ESLint 違規無人攔截。W2-001.1 執行時被迫 --no-verify commit。

### 影響範圍（實測）

- `package.json` lint / lint:fix script 範圍（src/ tests/ → 加 scripts/）
- `package.json` eslintConfig（需加 scripts/** override）
- scripts/ 下 ~50 個 .js 檔（含 build.js、validate-*.js、migration 工具、fix-*/mass-*/batch-* ad-hoc 腳本、experiments/w4-006-* spike 腳本）

**初始違規**：2330 problems（1593 errors / 737 warnings），遠超 ticket 估計的 150（ticket 僅檢視 build.js）。
- `--fix` 自動修復後降至 744（11 errors / 733 warnings）
- 733 warnings 中 673 為 no-console（CLI 工具正當輸出）→ override 關閉
- 11 errors 集中於 experiments/w4-006-* spike（camelcase 6 / no-async-promise-executor 2）+ fix-relative-paths-comprehensive.js（no-useless-escape 1）+ run-auto-migration.js（no-undef 2）
- 剩餘 warnings：no-unused-vars 18、no-case-declarations 5、param-names 37（全在 experiments/）

### 相關 Error Pattern

CLAUDE.md pre-commit 防線設計（W4-009 違規累積根因）：scripts/ 缺 lint 把關即此模式延伸。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

**1. lint 範圍擴充（package.json scripts）**
- `lint`: `eslint src/ tests/` → `eslint src/ tests/ scripts/`
- `lint:fix`: 同步加入 `scripts/`

**2. eslintConfig 新增兩個 override（package.json）**
- `scripts/**` → `no-console: off`（Node CLI/build 工具 console 為正當輸出，同既有 `tests/e2e/performance/** → no-console: off` 精神）
- `scripts/experiments/**` → `camelcase: off` / `no-async-promise-executor: off` / `promise/param-names: off`（w4-006-popup-launch-spike-v2..v8 為已完成 W4-006 調查的凍結 spike 產物，probeX_ 命名為刻意 spike 標記，不強制改寫）

**3. 自動修復（npm run lint:fix）**
- 27 個 scripts/.js 檔的 formatting 違規（semicolon、quote、trailing-spaces、space-before-function-paren、eol-last、padded-blocks 等），含 build.js reformatting（本 ticket 範圍）

**4. Manual 修復真實錯誤/警告（非 experiments，production 工具）**
- run-auto-migration.js: `options` 提升至 try 外宣告（no-undef，scope bug 修正 — catch 區塊存取 options.enableLogging）
- fix-relative-paths-comprehensive.js: 移除 `[/\\]` 字元類內多餘 `\/` 跳脫（no-useless-escape）+ 移除死變數 pathsFixedInFile
- migration-progress-manager.js: switch case 加 block scope `{}`（no-case-declarations ×5）+ 移除未用 import MIGRATION_ITEM_TYPE/VALIDATION_TYPE + 移除未用 const progress
- auto-fix-console.js / fix-no-console-151-300.js / mass-console-fix.js / run-unification-test.js / master-final-warnings-fix.js: 移除未用 require('path')
- batch-fix-console.js: 移除未用 execSync
- batch-path-fixer.js: 移除死函式 generatePathMappings（未呼叫、不在 exports）
- check-missing-references.js: 移除未用 const EXTENSIONS_TO_CHECK / IGNORE_PATTERNS
- fix-console-warnings.js: 移除死變數 nextLine
- fix-require-paths.js: 移除未用 const testDir
- test-dual-error-system.js: bridgeError 移除未用 binding（保留呼叫的驗證副作用）+ 將孤兒函式 testSpecificScenarios 接線進 runDualSystemTests 主流程（恢復快取/錯誤鏈/記憶體洩漏情境測試覆蓋）
- validate-monitoring-system.js: monitorErrorCreation 移除未用 binding（保留呼叫記錄監控數據的副作用）

### 設計決策

- console warning（673 個）以 override 關閉而非逐項改 logger：scripts/ 為 Node CLI/build 工具，console 為其本職輸出
- experiments spike 違規以 override 豁免而非改寫：凍結產物，改寫無收益且破壞 spike 與文件對應
- 副作用呼叫（bridgeError / monitorErrorCreation）保留呼叫只移除 binding，不誤刪行為

### 不在本 ticket 範圍
