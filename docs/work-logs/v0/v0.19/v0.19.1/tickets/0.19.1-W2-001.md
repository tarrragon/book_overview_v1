## 重現實驗結果
### 實驗方法

使用 WCAG 2.1 相對亮度公式計算對比度：
1. sRGB → linear RGB（gamma 解碼）
2. 相對亮度 L = 0.2126*R + 0.7152*G + 0.0722*B
3. 對比度 = (L_lighter + 0.05) / (L_darker + 0.05)
4. WCAG AA 標準：普通文字 >= 4.5:1，大文字 >= 3:1

讀取 `src/core/design-system/colors.js` STATUS_COLORS.unread 定義：
- fg（文字色）= `#E3F2FD`（primaryLightest）
- bg = `rgba(#E3F2FD, 0.15)` 合成於白色表面

### 實驗執行

Python 計算結果：
- unread fg `#E3F2FD` 亮度 L=0.8693
- unread bg 合成後 `#FAFDFE` 亮度 L=0.9773
- **對比度 = 1.12:1**（WCAG AA 要求 4.5:1）
- fg 直接放白底上：1.14:1

spec 3.3 規定「深色字規則」但 STATUS_COLORS.unread.fg 卻是 `#E3F2FD`（淺藍近白色）——規定與實作自相矛盾。

### 實驗發現

**已驗證事實**：
1. unread 徽章對比度 1.12:1，遠低於 WCAG AA 4.5:1 標準，幾乎不可見
2. spec 3.3 宣稱「深色字」但 colors.js 給出淺色字，自相矛盾
3. 現有 STATUS_COLORS 設計為「彩色文字 + 15% alpha 背景」，對淺色（unread/queued）天然低對比

**修正方向（D3 已定案）**：改為「深色文字 + 不透明淺色背景」，全 6 狀態均達 WCAG AA。具體配色值見 Solution。
