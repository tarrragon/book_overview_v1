### 變更摘要

本 DOC ticket 在兩個框架檔案補強 ANA 全量 grep/regex 範圍驗證完整性規範。

#### 修改檔案 1：`.claude/pm-rules/tdd-flow.md`（v2.18.0）

**新增章節**：`## ANA 全量 grep/regex 範圍驗證完整性規範`，置於 Phase 0 章節前。

三層強制要求：

1. **驗證方法涵蓋完整性聲明**（必填）：說明三種驗證方法（區段列舉法 / 全字元集掃描法 / rg regex 法）的適用場景與已知盲區，並提供 Python 全字元集掃描 + rg 交叉驗證的組合使用範本。

2. **覆蓋完整性聲明格式**：四欄必填格式（範圍驗證方法 / 驗證指令 / 已知盲區 / 盲區影響評估）+ 強制觸發條件句型表（數量斷言、零依賴斷言、路徑範圍斷言、字元集覆蓋斷言任一出現即觸發）。

3. **AC 設計連動要求**：ANA 覆蓋完整性聲明必須直接體現在後續 IMP acceptance 設計中，禁止 IMP acceptance 僅驗 build/lint 通過而不重新驗證 ANA 所聲明的範圍。

附觸發案例（W1-005 AC-4 二度誤判）及相關 Error Pattern 引用（PC-161，互補不重複）。

#### 修改檔案 2：`.claude/agents/saffron-system-analyst.md`（v1.2.0）

**新增核心職責第 6 項**：`### 6. ANA 全量 grep/regex 範圍判定自檢（強制）`

內容包含：
- 觸發句型自檢表（4 種句型）
- 必填聲明格式（與 tdd-flow.md 一致）
- 禁止行為表（手工區段列舉替代全量掃描 / 單一關鍵字零依賴斷言 / AC 不回鏈驗證指令）
- 三明示（Why/Consequence/Action）
- 引用指向 tdd-flow.md 完整規範

#### 規範要點（三明示）

**Why**：W1-005 ANA 以手工列舉 Unicode 區段（U+1F000-1FFFF + U+2600-27BF）作為全量依據，漏 U+2139-25B6 區段 9 種 + FE0F 336 處；AC-4 以單一關鍵字 rg 驗證「斷言依賴=0」，實際 12 檔 48+ 處被遺漏。

**Consequence**：IMP acceptance 建立在錯誤覆蓋假設 → W1-005.2 隱性回歸 12 檔 48+ 處斷言不一致 → 需兩個 patch ticket 修復。

**Action**：ANA Solution 涉全量 grep/regex 範圍判定時，必須附完整性聲明（驗證方法 + 指令 + 盲區 + 盲區評估）；IMP acceptance 必須回鏈 ANA 驗證指令重新驗證。
