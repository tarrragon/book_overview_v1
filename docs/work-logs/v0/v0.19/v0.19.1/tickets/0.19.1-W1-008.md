#### #18 subagent 卡 running — 取證

1. Read pm-agent-observability.md：TaskOutput status 標籤（running/completed/error）是唯一可信狀態來源；已有 SubagentStop event 作為可信完成訊號

4. 關鍵發現：#18 修復「完成 subagent finalize 出錯卡 running」——此前 PM 看到 running 但無 output 的困惑可部分歸因於此 bug

#### #4 並行 Bash 失敗隔離 — 取證

1. `rg "parallel\|batch\|cancel\|取消"` 在 parallel-dispatch.md 和 bash-tool-usage-rules.md 均無結果
2. bash-tool-usage-rules.md 規則三：禁止串接多 git 寫入，但原因是 index.lock 競爭，非 batch cancellation
3. parallel-dispatch.md 聚焦代理人層級並行（worktree 隔離、檔案衝突），非 tool call 層級並行
4. 關鍵發現：本專案框架未針對「同批並行 Bash 一失敗全取消」制定任何保守限制或繞道。此修復自動受益，無既有規則需調整
