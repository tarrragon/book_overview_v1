## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因
W1-018 統一根因：同步機制缺 base/共同祖先概念，push/pull 皆為無狀態全量 overlay。
.sync-state.json schema 僅記 last_push_hash，無 base snapshot 欄位，無法支撐後續三方合併。

多視角 H1：禁雙欄位（push/pull 各一）+ 禁對 SHA 用 max()。commit SHA 為字典序字串，
max(SHA) 會選錯共同祖先，故必須用單一 last_synced_base_sha，push/pull 成功皆覆寫同一欄位。

### 影響範圍
- .claude/.sync-state.json（schema 擴充）
- .claude/scripts/sync-claude-status.py（讀取 + 顯示 base SHA）
缺陷 N（status/push hash EXCLUDE 不一致）已由 W1-027 manifest consolidate 解除（status 改用 sync_exclude_manifest.compute_content_hash，與 push 同一 SSOT）。

### 相關 Error Pattern
ARCH-020（跨進程重複定義漂移）—— 缺陷 N 即此模式，已由 W1-027 manifest import 統一。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### Schema 擴充
.claude/.sync-state.json 新增單一 `last_synced_base_sha` 欄位（初值 ""，由 push/pull 填）。
非雙欄位、不對 SHA 做 max()（H1 防護）。

### status 適配
sync-claude-status.py：
- 新增 BASE_SHA_FIELD / NO_RECORD 常數與 `resolve_base_sha_display(sync_state)` helper：
  只認單一鍵 last_synced_base_sha，缺欄位 / 空值回退「（無記錄）」；雙欄位 schema 不被支援。
- main() 讀取並在輸出新增「同步 base SHA: <sha>」行。
- compute_content_hash 沿用 W1-027 sync_exclude_manifest（與 push 同一 EXCLUDE 集合，缺陷 N 已解，本票不重複處理）。

### 不在本 ticket 範圍
push/pull 實際寫入 base SHA 與三方合併邏輯屬後續 IMP（本票為共用前置 schema + status 顯示）。
