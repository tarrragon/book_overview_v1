### 問題根因

### 相關 Error Pattern

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 設計

在 sync-claude-pull.py 新增純函式（易測）：

- parse_pc_filename(repo_rel) → (num, slug) | None：僅匹配 error-patterns/ 下 PC-<NNN>-<slug>.md
- build_local_pc_index(claude_dir) → {numbers: {號→slug}, provenance: {(上游號,slug)→本地號}}；provenance 由本地檔內「編號溯源」註記解析
- resolve_pc_collision(repo_rel, content, index) → (new_rel, new_content, action)
  - action=none：無衝突 / 同號同 slug（同一檔），原樣交三方合併
  - action=dedup_skip：上游帶回先前已重編的同一 pattern（provenance 命中）→ 跳過不重複匯入
  - action=renumber：同號不同 slug → 重編為本地下一可用號 + 注入溯源註記 + 更新 frontmatter id
- _renumber_pc_content：僅改 frontmatter id 與插入溯源行，不全域替換 PC-<號> 字面（避免破壞合法引用）
- _next_available_pc_number：基於本地已佔用號找空號

### 整合進 W1-028 三方合併

apply_upstream_delta 迴圈開頭一次性 build_local_pc_index；對 A/M 的 PC 檔先呼叫 resolve_pc_collision：dedup_skip→跳過；renumber→_apply_renumbered_pc 原子寫新檔並回寫索引（含 provenance），使同批後續同 pattern 也能 dedup；none→落入原三方合併路徑。D（上游刪除）不改名。

### 不在本 ticket 範圍

- 跨號重編後對原 PC 內文其他位置 PC-<上游號> 引用的更新（保留以免破壞合法引用）
- IMP/TEST/ARCH 等非 PC 前綴 error-pattern 撞號
