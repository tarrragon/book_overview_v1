## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

local-only 檔案分類分散於三份獨立維護清單（.gitignore、sync push/pull/status 各自的 EXCLUDE_PATTERNS/LOCAL_ONLY、session-start hook 的 KNOWN_EXCLUDED），無單一權威來源（Single Source of Truth），新增機制時需手動同步 3-4 個位置。歷史已實證漂移必然發生（W1-014 缺陷 T/U/E）。

### 影響範圍

| 檔案 | 角色 |
|------|------|
| `.gitignore` | git 追蹤排除（影響 porcelain 可見性） |
| `.claude/scripts/sync-claude-push.py` EXCLUDE_PATTERNS | sync push 複製過濾 |
| `.claude/scripts/sync-claude-pull.py` LOCAL_ONLY | sync pull 跳過覆蓋 |
| `.claude/scripts/sync-claude-status.py` EXCLUDE_PATTERNS | sync status 差異比較過濾 |
| `.claude/hooks/session-start-sync-exclusion-check-hook.py` KNOWN_EXCLUDED | session 啟動掃描漏網偵測 |
| `.claude/references/sync-exclusion-guide.md` | 人工決策指南（文件） |

### 相關 Error Pattern

無直接 PC 編號，但根因與 ARCH-020（validator 平行實作漂移）同構：多處獨立維護同語意清單。

**push vs pull 差異（設計合理）**：push 多出 10 項敏感憑證類（`.env*`/`credentials.json`/`secrets*`/`private`/`.keys`），pull 不需這些因為遠端 repo 本不應有。

**hook KNOWN_EXCLUDED 差異（設計合理）**：hook 只掃 `.claude/` 根的 `.json/.yaml/.yml`，目錄類（handoff/hook-logs 等）和 `__pycache__` 不在其掃描範圍內。

**push clean-check 缺陷機制（缺陷 T 根因）**：push.py 步驟 2 用 `git status --porcelain .claude` 檢查乾淨；步驟 2.2 用 `git ls-files --others --ignored` + `should_exclude` 偵測洩漏風險。`.zhtw-mcp-skip` 在 EXCLUDE 中所以步驟 2.2 不警告，但若該檔未在 .gitignore 且未 git add，porcelain 會將其列為 untracked → 步驟 2 abort「有未提交變更」。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 設計：單一分類來源（local-only manifest）

#### 方案選擇

| 方案 | 描述 | 優點 | 缺點 |
|------|------|------|------|
| A. 共享 Python module | `.claude/scripts/sync_exclude_manifest.py` 定義權威清單，push/pull/status import 它 | 零冗餘、型別安全、改一處生效 | .gitignore 仍需手動同步（gitignore 是 git 原生格式無法 import Python） |
| B. YAML manifest + 生成腳本 | `.claude/config/local-only-manifest.yaml` 定義分類，build 腳本生成 .gitignore 片段 | 完全自動化 | 增加 build 步驟複雜度；.gitignore 非人工維護降低可讀性 |
| C. 共享 Python module + gitignore 交叉驗證 hook | 方案 A + hook 在 session-start 或 commit 時驗證 .gitignore 涵蓋 manifest | 改一處 + 自動偵測 gitignore 漂移 | 仍需人工修 gitignore，但漂移會被攔截 |

**建議採用方案 C**：共享 Python module 作為 SSOT + session-start hook 交叉驗證 .gitignore 覆蓋率。

**Why**：方案 A 解決 sync 三腳本漂移但不解 gitignore 漂移；方案 B 過度工程化（.gitignore 是開發者常手動查看的檔案，自動生成降低透明度）；方案 C 用最小侵入性解決所有漂移路徑。

**Consequence**：不採方案 C 則 gitignore 漂移仍依賴人工記憶同步，W1-014 缺陷 T 類問題會再發。

**Action**：見下方 spawn IMP 規劃。

#### 方案 C 詳細設計

**1. 新建 `.claude/scripts/sync_exclude_manifest.py`**

```python
"""local-only 檔案分類權威來源（Single Source of Truth）。

所有 sync 腳本（push/pull/status）和 session-start hook 從此 import。
新增 local-only 項目只需改此檔 + 對應更新 .gitignore（hook 會驗證覆蓋率）。
"""

# 類型 A - Runtime state（本 session 執行期狀態）
RUNTIME_STATE = frozenset({
    "dispatch-active.json",
    "dispatch-active.lock",
    "pm-status.json",
    "hook-state",
    "state",
    "scheduled_tasks.lock",
})

# 類型 B - Local-only settings（各專案個別設定）
LOCAL_SETTINGS = frozenset({
    "sync-preserve.yaml",
    ".sync-state.json",
    "settings.local.json",
    ".zhtw-mcp-skip",
    ".version-release.yaml",
})

# 類型 C - Session-bound log / temp
SESSION_TEMP = frozenset({
    "handoff",
    "hook-logs",
    "PM_INTERVENTION_REQUIRED",
    "ARCHITECTURE_REVIEW_REQUIRED",
    "TASK_AVOIDANCE_FIX_MODE",
    "worktrees",
    "migration-backups",
})

# 類型 D - 敏感憑證（push 專用，防止推到公開 repo）
SENSITIVE_CREDENTIALS = frozenset({
    ".env", ".env.local", ".env.production",
    "credentials.json", "secrets.yaml", "secrets.json",
    ".secrets", "secrets", "private", ".keys",
})

# 工具產物（無跨專案共用價值）
TOOL_ARTIFACTS = frozenset({
    "__pycache__", ".pytest_cache", ".venv",
})

# 組合集合（供各腳本 import）
SYNC_EXCLUDE_ALL = RUNTIME_STATE | LOCAL_SETTINGS | SESSION_TEMP | TOOL_ARTIFACTS
PUSH_EXCLUDE = SYNC_EXCLUDE_ALL | SENSITIVE_CREDENTIALS
GITIGNORE_EXPECTED = RUNTIME_STATE | LOCAL_SETTINGS | SESSION_TEMP  # gitignore 應涵蓋的項目（不含工具產物和憑證，那些有各自 gitignore 規則）
```

**2. push/pull/status 改為 import manifest**

各腳本移除內嵌 EXCLUDE_PATTERNS/LOCAL_ONLY，改為：
```python
from sync_exclude_manifest import PUSH_EXCLUDE, SYNC_EXCLUDE_ALL
```
（pull 用 SYNC_EXCLUDE_ALL 作為 LOCAL_ONLY 的替代）

**3. session-start hook 改為 import manifest**

hook 的 KNOWN_EXCLUDED 改為從 manifest import LOCAL_SETTINGS | RUNTIME_STATE（hook 只掃根目錄 .json/.yaml/.yml，但 import 後自動同步）。

**4. gitignore 交叉驗證**

在既有 session-start-sync-exclusion-check-hook.py 中增加一段：import GITIGNORE_EXPECTED，解析 .gitignore 中 `.claude/` 相關條目，比對缺漏項並輸出 WARNING。這比新建 hook 更輕量。

#### 缺陷 T 具體修復方案

**根因**：`.zhtw-mcp-skip` 在 sync EXCLUDE 但不在 .gitignore → 若該檔存在於磁碟，`git status --porcelain` 將其列為 untracked → push 步驟 2 abort。

**修復**：
1. 在 `.gitignore` 新增 `.claude/.zhtw-mcp-skip`
2. 在 `.gitignore` 新增 `.claude/.version-release.yaml`
3. 在 `.gitignore` 新增 `.claude/sync-preserve.yaml`、`.claude/settings.local.json`
4. 在 manifest 中確保以上項目都在 LOCAL_SETTINGS 分類

**補充修復（gitignore 有 sync 無的項目）**：
`TASK_AVOIDANCE_FIX_MODE`、`state/`、`worktrees/`、`migration-backups/` 需加入 manifest 的 SESSION_TEMP 分類。`scheduled_tasks.lock` 和 `dispatch-active.lock` 加入 RUNTIME_STATE。這些目前 gitignore 有但 sync 無，漂移方向為 E 型（gitignore 有 EXCLUDE 無 → 可能洩漏至 sync 目標 repo）。
