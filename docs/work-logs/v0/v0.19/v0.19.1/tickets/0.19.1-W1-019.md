## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

sync-claude-push.py 步驟 2 以 `git status --porcelain .claude` 判定 .claude/ 是否乾淨，但 gitignored 檔不顯示於 porcelain 輸出（這正是設計瑕疵的核心）。而真正執行推送內容的 copy_filtered() 從「磁碟」遍歷 .claude/（非從 git index），排除機制僅靠 EXCLUDE_PATTERNS / EXCLUDE_SUFFIXES / EXCLUDE_NAME_PREFIXES 檔名黑名單。

因此：一個 gitignored 或 untracked 的機密檔，只要檔名不在黑名單內（例如 my-api-token.txt、leaked-creds.conf），會通過 porcelain 乾淨檢查，再被 copy_filtered 從磁碟複製，最終靜默推上公開 repo github.com/tarrragon/claude.git。一旦發生即安全事故。

### 影響範圍

- .claude/scripts/sync-claude-push.py（push 入口，唯一受影響檔）
- 任何在 .claude/ 下被 .gitignore 命中、或尚未 git add、且檔名不符黑名單的機密檔

### 相關 Error Pattern

W1-014 延伸評估缺陷 E（push 從磁碟讀 + porcelain 不顯示 gitignored），本票為其 interim 防護落地。根本解 C1（push 改從 git ls-files 讀取內容）屬 W1-018，不在本票範圍。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

新增 detect_secret_leak_risk(project_root) 函式並接入 main() 步驟 2.2（porcelain 檢查之後、clone 之前）。

- 函式以 `git ls-files -z --others`（untracked）與 `git ls-files -z --others --ignored`（gitignored）枚舉 .claude/ 下 git 不追蹤的檔案。
- 對每個結果轉成相對 .claude/ 的路徑，套用既有 should_exclude() 過濾掉 copy_filtered 本就會排除的黑名單檔（如 .env），避免誤報。
- 剩餘者即「會被推上公開 repo 但 git 不追蹤」的洩漏風險，回傳排序去重的清單。
- main() 偵測到非空清單即 print 紅字 ABORT + 列出每個風險檔 + 處置建議（刪除 / 移出 .claude/ / 加入 EXCLUDE_PATTERNS），並 sys.exit(1)。

### 設計決策

- 同時涵蓋 ignored 與 untracked：ignored 是 porcelain 完全看不到的核心瑕疵；untracked 雖 porcelain 看得到但原 abort 訊息未顯式點名機密外洩，一併納入更安全。
- 用 -z（NUL 分隔）解析 ls-files 輸出，避免檔名含空白 / 特殊字元解析錯誤。
- 復用 should_exclude 確保「黑名單已涵蓋」的檔不誤報，正常乾淨 push 不受影響。

### 不在本 ticket 範圍

根本解 C1（push 內容改從 git ls-files 讀取而非磁碟）於 W1-018 處理；本票為 interim abort 防護。
