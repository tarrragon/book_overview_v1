## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`restore_executable_bits`（pull 與 push 兩端）與 `git_update_index_chmod`（push 端）皆以
`EXECUTABLE_PY_SUBDIRS = ("hooks",)` 配合 `root / subdir` 單層查找，只覆蓋頂層
`.claude/hooks/`。W10-092 將部分 hook 遷移至 `skills/<name>/hooks/` 後，這些路徑不被
單層查找命中，pull 後 .py 可能無 +x（Permission denied）、push 後 git index mode 記為 100644。

### 影響範圍

- `.claude/scripts/sync-claude-pull.py`：`restore_executable_bits`
- `.claude/scripts/sync-claude-push.py`：`restore_executable_bits` + `git_update_index_chmod`
- 受影響執行體：所有遷移至 `skills/*/hooks/` 的 hook（pull 端 +x 還原 + push 端 index mode）

### 相關 Error Pattern

IMP-067（Windows git mode loss）同家族 — exec-bit / git index mode 覆蓋範圍正確性。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

將單層查找 `root / subdir` 改為遞迴目錄名比對：

- 新增 `EXECUTABLE_HOOK_DIR_NAMES = ("hooks",)` 取代 `EXECUTABLE_PY_SUBDIRS`（語意由「子目錄」改為「目錄名」）
- 新增 `iter_executable_hook_dirs(root)` helper：以 `os.walk`（followlinks=False）遞迴 yield 所有名為 `hooks` 的目錄，涵蓋頂層 `hooks/` 與 `skills/<name>/hooks/`
- pull 端 `restore_executable_bits`、push 端 `restore_executable_bits` 與 `git_update_index_chmod` 三處統一改用此 helper，維持 pull/push 對稱

### 設計決策

- 採「遞迴掃描目錄名」而非「擴充清單列舉每個 skill」：skill 動態增減時免維護清單，零漏網。
- `os.walk` 預設 `followlinks=False`，自然避免 symlink 循環。
- 非 `hooks/` 目錄下的 .py（如 `skills/*/scripts/`）不受影響，符合既有行為。

### 與新 push 流程相容性（W1-029 git-archive）

W1-029 push 改以 `git archive HEAD` 解 tracked 樹至 staging，目錄結構完整保留，
`skills/*/hooks/` 仍存在於 temp_dir。`git_update_index_chmod` 在 `git add -A` 後執行，
遞迴覆蓋後對每個 hook .py 執行 `git update-index --chmod=+x`，確保 index mode 100755，
與 git-archive 流程無衝突。
