## Problem Analysis

### 問題根因

W1-031 交叉驗證 hook（`session-start-sync-exclusion-check-hook.py`）上線後首航偵測到真實漂移：

- sync manifest（`.claude/hooks/lib/sync_exclude_manifest.py`）的 `LOCAL_ONLY_PATTERNS` 將 `handoff` 列為類型 C（session-bound log）完整 local-only，`GITIGNORE_EXPECTED = LOCAL_ONLY_PATTERNS` 因此期望 `.gitignore` 涵蓋裸名 `handoff`。
- `.gitignore` 原僅 ignore 子路徑 `.claude/handoff/archive/`，hook 正規化後得 `handoff/archive`（非 `handoff`），故交叉驗證回報 `DRIFT: ['handoff']`。
- 進一步盤點 `git ls-files .claude/handoff/`：頂層目錄下有 112 個檔案被 git track，包含 `.claude/handoff/archive/*.json`（111 個歷史 handoff 快照）與內層 `.claude/handoff/.gitignore`。git 追蹤態（tracked）與 manifest local-only 分類（不該版控、不該同步）不一致。

### 影響範圍

- `.gitignore`（補 `.claude/handoff/` 全目錄規則）
- 既有 tracked handoff 檔（`git rm --cached` untrack，保留磁碟）
- 驗證對象：`session-start-sync-exclusion-check-hook.py` 交叉驗證、`.claude/scripts/tests/`、`.claude/hooks/tests/` sync 相關測試

### 相關 Error Pattern

無直接 error pattern。屬 W1-024 sync 排除一致性系列（W1-031 交叉驗證層偵測 → 本 ticket 對齊）。

## Solution

### 採用方案：原則解（補 gitignore + untrack tracked 檔）

二擇一中選擇「補 gitignore `.claude/handoff/` 規則 + untrack 既有 tracked 檔」，而非「調整 manifest 排除 handoff」。

**理由**：handoff 是 session-bound 的交付/恢復暫存檔（類型 C），其內容與單一 session 綁定，無跨專案共用價值，亦不該進版控（歷史快照膨脹 repo、無 review 價值）。manifest 將其分類為 local-only 是正確分類，應讓 git 追蹤態與 `.gitignore` 對齊此既有正確分類，而非反向放鬆 manifest。

### 變更內容

1. `.gitignore`：將 `.claude/handoff/archive/`（normalize→`handoff/archive`）改為 `.claude/handoff/`（normalize→`handoff`），與 `GITIGNORE_EXPECTED` 的裸名 `handoff` 一致。
2. `git rm -r --cached .claude/handoff/`：untrack 112 個 tracked 檔（111 個 `archive/*.json` + 內層 `.gitignore`），保留磁碟（168 個檔案實際在 disk 上，含未曾 track 者）。git 追蹤態歸零，與 local-only 分類一致。
