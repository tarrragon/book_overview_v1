## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

缺陷 T 根因：push clean-check（main step 2 `ensure_committed`）舊版用
`git diff --quiet -- .claude` + `git diff --cached --quiet -- .claude`，只檢查
tracked 檔的 unstaged/staged 變更，完全忽略 untracked。此設計同時有兩個反向問題：

1. 缺陷 T（誤 abort）：W1-024 記錄的原始故障是「未追蹤 local-only 檔讓 push 乾淨
   檢查跳錯而 abort」。若 clean-check 改看 untracked 但不過濾，未進 .gitignore 的
   local-only untracked 檔（如 .zhtw-mcp-skip）會被誤判為未提交變更而 abort——但
   這類檔本就不被 push（git archive 只取 tracked 樹），不應阻擋。
2. 漏判（靜默放行）：真正新增的 untracked 框架檔（如新 rule.md 尚未 git add）被
   git diff 靜默漏過，使 push 推出與工作區不一致內容。

### 影響範圍

- .claude/scripts/sync-claude-push.py：ensure_committed + main step 2 註解 + 模組 docstring
- .gitignore：local-only 漂移補齊

### 相關 Error Pattern

ARCH-020（多檔重複定義漂移）家族；本 ticket 沿用 W1-027 manifest should_exclude 不重寫。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### M1 根因解（clean-check 改 should_exclude 過濾 porcelain）

`ensure_committed` 重寫：以 `git status --porcelain --untracked=all -- .claude`
取工作區全狀態（含 untracked），逐行解析 path（處理 rename `orig -> new` 取末段、
strip `.claude/` 前綴），以 manifest `should_exclude(Path(rel))` 過濾 local-only /
憑證。過濾後仍存在任何條目即回 False（要求先 commit）；過濾後乾淨回 True。
git status 非零回傳保守視為不乾淨。沿用 W1-027 manifest should_exclude 不重寫
（acceptance 4）。

效果：
- untracked local-only 檔（.zhtw-mcp-skip / settings.local.json）不再誤 abort（缺陷 T 根因解）
- 真正未 commit 的 tracked 變更仍攔截
- 非 local-only 的 untracked 框架檔（舊 git diff 漏判）改為被攔截

同步更新 main step 2 註解與模組 docstring 的 commit-first 行為描述。

### M3/L3 .gitignore 補齊漂移項

於 .claude sync state 區塊後新增 local-only 漂移補齊段，與 sync_exclude_manifest
LOCAL_ONLY_PATTERNS 對齊：.zhtw-mcp-skip / .version-release.yaml / sync-preserve.yaml /
settings.local.json / .venv/ / .pytest_cache/ / .sync-conflicts/，加 nested skill
變體 `**/.zhtw-mcp-skip`、`**/.pytest_cache/`。已 git tracked 者（.version-release.yaml /
settings.local.json / sync-preserve.yaml）由 push should_exclude 過濾，gitignore 為
縱深防禦，防未來新增同類 untracked 變體污染工作區。

### Phase 4 評估結論

無需重構。本 ticket 復用 W1-027 manifest 的 should_exclude（DRY 無新漂移面），
ensure_committed 單一函式認知負擔低（一個迴圈 + guard），無技術債。程式碼品質 A。
