## Problem Analysis
<!-- Schema[DOC/Problem Analysis]: 選填（若 DOC 起因於缺陷或盤點結論可填）（.claude/pm-rules/ticket-body-schema.md） -->

### 發現脈絡

本 session 執行「呼叫 mcp__codegraph__codegraph_status 確認 codegraph 連線並建索引」時，發現該工具名在 codegraph v0.16.6 **不存在**（ToolSearch 精確選取 + 關鍵字兩輪皆未命中）。改用實際暴露的 `codegraph_reindex_workspace`（force=true）成功建立索引（750 檔），確認 server 已連線。

### 問題根因

跨 codegraph 版本的工具改名/移除。SKILL.md:433 的範例回應 `{"index":"up to date","nodes":24432,"edges":52772}` 形似舊版 `codegraph_status` 的真實輸出，顯示該工具曾存在於早期版本，於 v0.16.6 被移除/改名，但框架提示文字未同步。

### 影響範圍（grep -rn "codegraph_status" .claude/，接手者請獨立重驗 PC-162）

| # | 檔案:行 | 內容性質 |
|---|---------|---------|
| 1 | `.claude/skills/project-init/project_init/lib/messages.py:260` | `CodegraphMessages.get_codegraph_reindex_steps()` 末步驗證提示字串 |
| 2 | `.claude/skills/search-tools-guide/SKILL.md:433` | 工具對照表行，含虛構範例回應 `{"index":"up to date","nodes":24432,"edges":52772}` |
| 3 | `.claude/skills/search-tools-guide/SKILL.md:458` | `ToolSearch(query="select:codegraph_status,...")` 範例 |

`PC-159-install-command...` 內亦有 codegraph_status 字面，但屬歷史案例引用（acceptance 條件描述），保留為歷史脈絡，不在修正範圍。

### codegraph v0.16.6 實際暴露的狀態/索引類工具

`codegraph_reindex_workspace`（回傳 files_indexed / status）、`codegraph_get_module_summary`（files/functions/classes/lines）、`codegraph_memory_stats`、`codegraph_index_directory` / `codegraph_index_files`。無專屬「status」工具回傳 nodes/edges。

### 修正方向建議

優先採「版本無關描述」（acceptance #5），避免硬編碼另一個可能再漂移的工具名。例如將驗證提示改為「在 session 內呼叫 codegraph 的 reindex/summary 工具，回報 files_indexed 或 module summary 即代表索引就緒」。

### 注意事項

- messages.py 屬 `project-init` skill（uv tool 安裝），修改後需 `uv tool install .claude/skills/project-init --force --reinstall`，否則 SessionStart hook 仍輸出 stale 提示。
- 兩檔皆在 `.claude/` sync 範圍（框架共用），修正內容須 project-type-generic（memory: framework_content_project_type_generic）。

### 相關 Error Pattern

無直接對應 PC；性質近似「提示/文件與實機行為漂移」，與 memory feedback_wrapper_arg_injection_diagnostic（不可盡信 wrapper/腳本提示，須以實機暴露為準）同源。
