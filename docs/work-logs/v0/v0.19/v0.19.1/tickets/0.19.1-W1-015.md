## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

`ticket track list`（及任何走 priority 排序的命令）在 ticket 集合的 `created` 欄位型別不一致時拋：

```
TypeError: '<' not supported between instances of 'datetime.date' and 'str'
```

根因位於 `.claude/skills/ticket/ticket_system/commands/track_query.py`，函式 `_sort_tickets_by_priority` 的 `_key()`（已實機確認，行號為當前狀態）：

```python
# track_query.py:628-634（當前）
def _key(t: Dict[str, Any]):
    p_norm = _normalize_priority(t.get("priority"))
    created = t.get("created") or ""          # <- line 630 問題點
    tid = t.get("id") or t.get("ticket_id") or ""
    return (p_norm, created, tid)             # <- sort tuple 第 2 元素混型

return sorted(tickets, key=_key)
```

YAML 將未加引號的 `created: 2026-06-03` 解析為 `datetime.date`；加引號或缺欄位則為 `str`（含 `or ""` fallback 的空字串）。當同一 `p_norm` 下需比較 tuple 第 2 元素時，`date` 與 `str` 無法比較 → TypeError，整個 list 命令中斷。

### 影響範圍

- 直接：`track_query.py` `_sort_tickets_by_priority`（被 list / board / aggregated runqueue 三處呼叫，見 line 702 / 751 / 792）。
- 觸發條件：專案內跨 ticket 的 `created` 型別不一致（部分 unquoted date、部分 quoted str、部分缺欄位）。

### 修復樣板（建議）

```python
import datetime  # 檔頭補 import（若未有）

def _key(t: Dict[str, Any]):
    p_norm = _normalize_priority(t.get("priority"))
    raw = t.get("created")
    if isinstance(raw, datetime.date):        # datetime.datetime 為 date 子類，一併涵蓋
        created = raw.isoformat()
    else:
        created = str(raw) if raw else ""
    tid = t.get("id") or t.get("ticket_id") or ""
    return (p_norm, created, tid)
```

### 邊界（勿越界）

- 同檔 `track_runqueue._priority_rank` 為 int 變體，**本次只修 list 排序的 `_key()`，勿順手改動其他命令語意**。
- 全專案可順手 grep 其他對 `created`/日期欄位直接排序處，若發現相同 date vs str 隱患，建 follow-up ticket 追蹤（勿在本 ticket 擴張範圍）。

### 收尾必做

修復後必須 reinstall CLI（否則 stale installed 版仍報錯，IMP-023 / PC-145）：


### 相關 Error Pattern

- YAML unquoted scalar 型別陷阱家族（參考 PC-079 backtick、YAML hash truncation memory）；本案為 unquoted date 自動轉 datetime.date。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

`.claude/skills/ticket/ticket_system/commands/track_query.py`：

1. 檔頭新增 `import datetime`。
2. `_sort_tickets_by_priority._key()` 將 `created` 正規化：`isinstance(raw, datetime.date)` 時轉 `raw.isoformat()`（datetime.datetime 為 date 子類一併涵蓋），其餘 `str(raw) if raw else ""`。消除 sort tuple 第 2 元素 date vs str 混型的 TypeError。

### 設計決策

- 只改 list 排序的 `_key()`，未動 `track_runqueue._priority_rank`（int 變體）等其他命令語意，遵守 ticket 邊界。
- isoformat 與 str date 字面對齊（皆 YYYY-MM-DD），混型下排序語意一致。

### 不在本 ticket 範圍

- 未發現其他對 created/日期欄位直接排序的隱患；如後續發現另建 follow-up ticket。
