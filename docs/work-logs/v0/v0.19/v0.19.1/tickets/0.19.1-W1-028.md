<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-018 統一根因：同步機制無 base/共同祖先概念，pull 採全量 overlay 直接覆蓋本地 .claude/。
本地刪除（W10-092 模組遷移後刪除的檔案）會被上游重新帶回；本地 runtime state（LOCAL_ONLY）
與上游 delta 無法區分。缺少 base snapshot 使三方合併不可能，瑕疵 A（pull 覆蓋本地刪除）無法收斂。

### 影響範圍

- .claude/scripts/sync-claude-pull.py（主要重寫，新增三方合併核心 + 原子套用 + base SHA 讀寫）
- 依賴 W1-027 manifest 的 should_exclude（import 不重寫）
- 依賴 W1-025 schema 的 last_synced_base_sha 欄位（status 端已讀，pull 端負責寫）

### 相關 Error Pattern

無直接對應；屬同步機制架構缺陷（W1-014 → W1-018 ANA 鏈）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 三方合併改造（A3+L+M）

新增核心函式（sync-claude-pull.py）：
- read_base_sha / write_base_sha：讀寫 .sync-state.json 的 last_synced_base_sha（W1-025 schema，保留既有欄位，H1 單欄位禁雙欄位 max）
- is_base_reachable（H4）：git cat-file -e SHA^{commit} 驗可達；不可達降級全量 overlay + warning
- compute_upstream_delta（H3）：git diff --name-status --no-renames base HEAD；--no-renames 使 rename 退化為 D+A；split('\t') + 欄位數斷言（== 2）防空白路徑誤判
- three_way_merge_file：4 情境（add / take-upstream / conflict / preserve-local-delete）；本地刪除優先回 None；衝突走 git merge-file
- apply_upstream_delta（H2/M3/M4）：逐檔三方合併，merge 前先 should_exclude + preserve 過濾（M4）；衝突檔寫 .sync-conflicts/ 保留本地原檔（M3）
- _atomic_write / _atomic_remove / _rollback（H2）：原子置換（os.replace，跨 fs fallback copy）+ 失敗自動回滾；只搬 delta 涉及檔案保留 LOCAL_ONLY / runtime state
- _ensure_conflicts_dir（M3）：.sync-conflicts/ + .gitignore（內容 *）
- should_use_full_overlay（向後相容）：無 base SHA 或不可達 → 全量 overlay fallback

### clone 改造（L1）

clone_repo 改 --filter=blob:none partial clone（保留 commit graph 供 base 可達 + git show，blob lazy fetch），
partial 不支援時降級 full clone；timeout 120s → 300s。

### 整合

_sync_with_backup 依 should_use_full_overlay 分支：可達走三方合併，否則全量 overlay。
兩路徑成功後皆 git rev-parse HEAD 寫 last_synced_base_sha。

### 設計決策

上游獨立 repo 的 root 直接對應本地 .claude/（無 .claude/ 前綴）；_rel_under_claude 跳過 REMOTE_ONLY（project-templates/.git）。
should_exclude import 自 W1-027 manifest（ARCH-020 不重寫）。
