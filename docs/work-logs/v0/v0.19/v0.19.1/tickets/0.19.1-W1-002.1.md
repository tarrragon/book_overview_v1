## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

**根因確認**：overview 表格欄位數三來源不一致——thead 7 th（缺標籤欄）vs createBookRow 8 td（含 tagCell）vs CONSTANTS.TABLE.COLUMNS=8。tag 欄加入列渲染時表頭漏同步。

### 變更檔案（2 檔）

1. `src/overview/overview.html`：thead 的 `<th>狀態</th>` 後新增 `<th>標籤</th>`，使表頭 8 th 對齊列 8 td。表頭順序：select / 封面 / 書名 / 作者 / 書城來源 / 進度 / 狀態 / 標籤，與 createBookRow td 渲染順序完全一致。

2. `tests/unit/overview/overview-column-consistency.test.js`（新增）：欄位數一致性守衛，斷言三來源皆為 8：
   - 來源 1（thead th 數）：以 fs 讀取真實 `src/overview/overview.html`，JSDOM 解析 `#booksTable thead th`。
   - 來源 2（CONSTANTS.TABLE.COLUMNS）：因 CONSTANTS 為 module-private 未匯出，改以 `renderBooksTable([])` 渲染 empty-state row 後讀取 `td.colSpan` 間接取得。此設計驗證強度更高——若常數與實際 colspan 脫鉤亦會被捕捉。
   - 來源 3（createBookRow td 數）：實例化 OverviewPageController，呼叫 createBookRow(sampleBook)，計 `td` 數。
   - 三者任一漂移即測試紅燈。

### 設計決策

- **方案 B（補表頭 + 測試守衛）**：依 parent ANA 0.19.1-W1-002 用戶決策（方向 = 補表頭顯示標籤欄，非移除 tagCell；依據 v0.20.x Tag 管理里程碑 + tagCellRenderer 已渲染）。
- **COLUMNS 經 runtime 行為讀取**：不 import 私有常數，改讀 empty-state colspan，將「常數值」與「實際渲染效果」綁定驗證，避免守衛本身與 runtime 脫鉤。

- GREEN：補 `<th>標籤</th>` 後三來源皆 8，測試通過。
