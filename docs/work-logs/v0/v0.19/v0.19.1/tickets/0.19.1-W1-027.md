## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

local-only / 敏感憑證分類三件套（EXCLUDE_PATTERNS + EXCLUDE_SUFFIXES + EXCLUDE_NAME_PREFIXES）與 should_exclude() / compute_content_hash() 在 sync-claude-push.py 與 sync-claude-status.py 各自獨立定義（ARCH-020 重複實作漂移）。

實證缺陷 N：status 的 EXCLUDE_PATTERNS 漏列 push 已有的 dispatch-active.json / hook-state / settings.local.json / .zhtw-mcp-skip，導致兩端 compute_content_hash 對同一 .claude/ 內容算出不同指紋，status「內容狀態」判斷不可信。

### 影響範圍

- .claude/scripts/sync-claude-push.py（should_exclude + _compute_content_hash + 三件套）
- .claude/scripts/sync-claude-status.py（should_exclude + compute_content_hash + 漂移的 EXCLUDE_PATTERNS，缺陷 N 源頭）
- .claude/hooks/session-start-sync-exclusion-check-hook.py（KNOWN_EXCLUDED 硬編副本）

### 相關 Error Pattern

ARCH-020（跨進程同構邏輯獨立重寫導致 bug 無法同步修復）。本 ticket 為其根治：建單一 SSOT manifest。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

1. 新建 SSOT manifest：.claude/hooks/lib/sync_exclude_manifest.py
   - 原始分類 frozenset 對外：LOCAL_ONLY_PATTERNS / CREDENTIAL_PATTERNS / EXCLUDE_SUFFIXES / EXCLUDE_NAME_PREFIXES
   - 組合集合具名常數 + 單行理由：PUSH_EXCLUDE = LOCAL_ONLY | CREDENTIAL；SYNC_EXCLUDE_ALL 同義別名
   - should_exclude(path)：唯一實作；docstring 明訂「path 必須相對 claude_dir」並 assert not path.is_absolute()
   - compute_content_hash(claude_dir)：唯一實作
   - 語意完全等同 push 原版（pull 的 LOCAL_ONLY 對應 = PUSH_EXCLUDE 去掉憑證項，故拆為兩 frozenset 保持各端語意不變）

2. push / status import manifest，移除各自內嵌定義
   - 兩腳本 sys.path.insert 指向 ../hooks/lib，from sync_exclude_manifest import should_exclude, compute_content_hash
   - push 保留 _compute_content_hash 別名（import as），維持既有呼叫端與測試相容

3. hook import manifest
   - session-start-sync-exclusion-check-hook.py 的 KNOWN_EXCLUDED 改 = LOCAL_ONLY_PATTERNS | {settings.json}
   - settings.json 為 framework-config allowlist，不屬 local-only 故顯式併入

### 缺陷 N 修復

status 舊 EXCLUDE_PATTERNS 漏列 4 項，改 import 後與 push 共用同一 should_exclude，兩端指紋必然一致。新增回歸測試 test_push_and_status_hash_equal 對同 fixture 斷言相等。

### 不在本 ticket 範圍

pull 的 LOCAL_ONLY（name-only 機制，無 suffix/prefix/should_exclude）語意與 push/status 的 should_exclude 不同；強制遷移會改變 pull 行為（開始排除 .pem 等），屬行為變更，不納入本 consolidation。manifest 已預留 LOCAL_ONLY_PATTERNS 供 pull 未來對齊（W1-024 cluster 後續）。
