## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

長背景任務（如 pytest 跑全套件約 9 分鐘）在非 TTY 環境中使用 `-q` + `| tail` 雙層緩衝，導致輸出檔全程 0 行。根因兩層：
1. `| tail`：tail 等 EOF 才吐最後 N 行，憋住全部輸出到結束。
2. `-q` + 非 TTY：pytest 偵測 stdout 非終端機時不即時 flush（fully-buffered）。

### 影響範圍

`.claude/rules/core/bash-tool-usage-rules.md`——規則條缺失，PM/Claude 每次憑記憶，長任務反覆黑箱化。

### 相關 Error Pattern

來源：0.19.0-W3-086 ANA（buffered 全程 0 行 vs PYTHONUNBUFFERED=1 逐行成長 spike 實證）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修改摘要

在 `.claude/rules/core/bash-tool-usage-rules.md` 規則五之後新增「規則六：長背景任務需即時可觀察時使用 PYTHONUNBUFFERED + tee」。

### 規則結構

- Why：非 TTY fully-buffered + `| tail` 等 EOF 雙層緩衝機制說明
- Consequence：長任務黑箱化，失敗早現能力喪失
- Action：場景對照表（長任務觀察 vs 只要最終結果）+ 三慣例速查表
- 調和說明：規則二（`tail/head` 防淹沒）vs 規則六（`tee` 即時可觀測）的取捨表
- 識別特徵：輸出檔全程 0 行 → 確認是否 `-q | tail` 雙層緩衝

### 設計決策

- `PYTHONUNBUFFERED=1` 單一環境變數即足（不需 stdbuf，macOS LD_PRELOAD 可靠性存疑）
- pytest 用 `-v` 而非 `-q`（-q 非 TTY 不即時 flush）
- 規則編號選擇規則六（接續規則五後，邏輯分區清晰）
- 調和而非取代規則二：兩者服務不同需求，在說明表中並列

### 統一檢查清單更新

新增三條：
- 長背景任務需即時觀察 → tee streamable
- pytest 長任務 → `-v` 非 `-q`
- 背景任務輸出全程 0 行 → 確認 `-q | tail` 觸發條件
