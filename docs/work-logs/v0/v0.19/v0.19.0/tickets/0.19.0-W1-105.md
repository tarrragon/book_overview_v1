## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`MessageDictionary` constructor 簽章為 `constructor(language = 'zh-TW')`，僅接受字串型 language 參數。三處 caller 以 `new MessageDictionary({...customMessages})` 形式呼叫，期望傳入初始 messages 物件並批次註冊，實際發生：

1. 物件被當作 `language` 參數，賦值給 `this.language`（型別污染：language 成為 object）
2. 自訂 messages 完全未進入 `this.messages`（只有 `_loadDefaultMessages()` 產出的預設值）
3. 後續呼叫 `dict.get(customKey)` 必回 `[Missing: KEY]` 佔位符（src/core/messages/MessageDictionary.js:172）

關鍵觀察：類別已有 `addMessages(messages)` 方法（src/core/messages/MessageDictionary.js:246），完整含 cache size check 與 type validation，可重用作為 union signature 的 object 路徑實作。

### 影響範圍

| Caller | 行數 | 傳入 keys 數 | 受影響功能 |
|--------|------|------------|-----------|
| src/popup/popup.js | :79 | 20+ | popup logger（已部分由 W1-004 透過 GlobalMessages 補救 5 個 key） |
| src/ui/book-search-filter-integrated.js | :37 | 8 | 搜尋/篩選元件 logger |
| src/platform/readmoo-platform-migration-validator.js | :40 | 10+ | Readmoo 平台遷移驗證器 logger |

所有自訂 messages 全部靜默丟棄 → 對應 `dict.get(key)` 全部回傳 `[Missing: KEY]`，不影響功能但污染 log 可讀性。

### 相關 Error Pattern

- W1-004（已完成）：為表層外科修復，將 popup 5 個關鍵 key 加進 `_loadDefaultMessages()` GlobalMessages（POPUP_INTERFACE_LOADED 等），但 constructor 根因未動。
- 本 ticket：根因修復；W1-004 加進 GlobalMessages 的 5 個 key 可後續評估是否回歸到 popup local dict（屬可選清理，列在 how.strategy 第 3 點，本 ticket 不強制）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復策略：Constructor Union Signature

改寫 `src/core/messages/MessageDictionary.js` constructor 為 union signature，依參數型別分流：

```js
constructor (languageOrMessages = 'zh-TW') {
  // 先初始化 cache 欄位（addMessages 內部需要）
  this._cacheSize = 0
  this._maxCacheSize = 100 * 1024
  this.messages = this._loadDefaultMessages()
  this._updateCacheSize()

  if (typeof languageOrMessages === 'object' && languageOrMessages !== null) {
    // Object 路徑：批次註冊自訂 messages，沿用預設 language
    this.language = 'zh-TW'
    this.addMessages(languageOrMessages)  // 重用既有方法，含 cache size 檢查與型別驗證
  } else {
    // String 路徑或 undefined（向後相容）
    this.language = languageOrMessages || 'zh-TW'
  }
}
```

### 設計決策

1. **重用 `addMessages()`**：避免重複實作 cache size check 與 type validation；維持原有 ErrorCodes.VALIDATION_ERROR 拋出行為。
2. **語言處理**：傳入 object 時 language 預設 zh-TW（caller 目前無傳 language 場景，未來如需可擴 `new MessageDictionary({messages, language})` schema，本 ticket 不前置實作）。
3. **不改 3 個 caller**：caller 端 `new MessageDictionary({...messages})` 寫法保持不變，constructor 改寫後即可正確註冊。
4. **不動 W1-004 補救 keys**：how.strategy 第 3 點「popup 5 key 回歸 local dict」屬可選清理，本 ticket 範圍只處理 constructor bug；保留 GlobalMessages 中的 5 個 key 不影響功能。
