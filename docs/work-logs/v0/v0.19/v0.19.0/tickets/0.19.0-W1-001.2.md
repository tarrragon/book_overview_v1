| popup 初始化狀態 | 矛盾：同時顯示「Content Script 連線正常／可以開始提取」與「Background Service Worker 連線失敗」 |

**根因（已定位）**：src/content/detectors/page-detector.js 的 detectPageType() 檢查 pathname.includes('/shelf')（含前導斜線），當前書櫃路徑 /me/bookshelf/total 的 bookshelf 不含 /shelf 子字串 -> 回傳 unknown -> book-data-extractor.js:161 isExtractableReadmooPage() 為 false -> 拋「不支援的頁面類型」。對照 readmoo-adapter.js:1317 用的是 pathname.includes('bookshelf')，兩層偵測邏輯不一致。

**結論**：Readmoo 已變更網站 URL 結構（readmoo.com/library 移除、書櫃移至 share.readmoo.com）。extension 的頁面偵測、content-script-coordinator、UC01ErrorFactory 多處硬編碼 readmoo.com/library 已過時。UC-02/03/04/06 在 extension 修復頁面偵測前無法驗證——本 ticket 再次中斷，性質與前次 model gap 中斷相同。
