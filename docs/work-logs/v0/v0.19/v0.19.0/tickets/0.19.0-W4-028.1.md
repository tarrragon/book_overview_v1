### 變更摘要

檔案：`.claude/skills/worktree/SKILL.md`（三處編輯）

1. **「殭屍問題」段落**：將「不會自動清理」修正為「不會自動 remove」，新增「Lock 行為（CC v2.1.157 起的變化）」對照表——正常結束自動 unlock（可直接 remove/prune）；異常死亡仍殘留 lock（保留 unlock 前置）。
2. **「手動清理指令」**：新增路徑 1 `git worktree prune`（v2.1.157+ 首選，清正常結束已 unlock 的殘留，免 unlock 前置）；保留路徑 2（unlock + remove --force）對異常死亡殘留 lock。附絆腳索：prune 仍遇 lock 則改走路徑 2。
3. **新增「EnterWorktree mid-session 切換（CC v2.1.157）」小節**：說明 mid-session 切換能力 + 三時機強制查核表（派發後 / 接 notification 後 / 主動切換後均查 `git branch --show-current` + `pwd`）。

### 其他 where.files 查證後不改的理由

- `.claude/skills/branch-worktree-guardian/SKILL.md`：其清理段落針對人工 worktree（`../ccsession-`），建立時不加 lock（見 worktree SKILL 對照表），`git worktree remove` 本就無需 unlock，無 v2.1.157 影響，不改。
- `.claude/rules/core/pm-role.md`：cwd 查核要求已於 worktree SKILL（按需讀取）詳述；pm-role 為自動載入檔（token 敏感），避免重複膨脹，不改。
