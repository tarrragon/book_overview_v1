**根因**：`window.addEventListener('unhandledrejection', ...)` 捕捉到：

```
ReferenceError: global is not defined
    at FileReaderFactory.createReader (src/overview/overview.js:2419:11)
    at BookFileImporter._createFileReader (src/overview/overview.js:2621:36)
    at BookFileImporter._readFileWithReader (src/overview/overview.js:2608:18)
    at BookFileImporter.read (src/overview/overview.js:2515:23)
    at OverviewPageController2.handleFileLoad (src/overview/overview.js:6277:60)
```

**原始程式碼**（`src/utils/file-reader-factory.js:24,37`）：

```javascript
static createReader () {
  if (global.FileReader) return new global.FileReader()    // L24 — ReferenceError
  if (typeof FileReader !== 'undefined') return new FileReader()
  ...
}

static isAvailable () {
  return Boolean(global.FileReader || typeof FileReader !== 'undefined')  // L37
}
```

`global` 為 Node.js 全域變數，Chrome Extension（無論 SW / Content Script / Popup / Overview）皆為瀏覽器環境，無 `global` 變數，存取即拋 `ReferenceError`。CLAUDE.md §7（Chrome Extension 開發規範）明列「禁用 `global`，必須使用 `globalThis`」——本檔違反此規則。

由於 `if (global.FileReader)` 在 short-circuit 評估前就需先解析 `global`，第一行即 throw。第二行 fallback `typeof FileReader !== 'undefined'`（CE 環境合法且有 FileReader）**永遠不會執行**。

**影響範圍**：UC-04 JSON 匯入功能在 Chrome Extension 環境下 100% 失敗——modal 因為自身 click handler 同步關閉而看起來正常，但 `bookFileImporter.read(file)` 立即 reject，handleFileLoad 之後的 `replaceAllData / mergeAllData` 永不執行。錯誤訊息也不顯示給使用者（`handleFileLoad` 未對 await 失敗做 try-catch）。

**附帶發現**（不阻塞本 ticket）：`src/export/book-data-exporter.js:934,945` 同樣使用 `global.URL.createObjectURL` / `global.URL.revokeObjectURL`，匯出 download 功能在 CE 環境疑似亦有相同問題（W1-001.2 UC-03 匯出驗證 commit 0ffe31a1 通過，可能是測試方式以 evaluate_script 抽取資料而非觸發實際下載按鈕——需後續驗證）。
