## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`overview-import-function.test.js` 的「應該正確處理表格顯示更新」測試以 `td:nth-child(2..5)` 斷言 title/source/progress/status，但產品 `OverviewPageController.createBookRow`（src/overview/overview-page-controller.js:723-792）的實際欄位順序為：

| nth-child | 欄位 |
|-----------|------|
| 1 | select-checkbox（W6-012.7.1 新增） |
| 2 | cover |
| 3 | title |
| 4 | source |
| 5 | progress |
| 6 | status |
| 7 | tag（tagCellRenderer 新增） |

測試斷言 `td:nth-child(2)` 命中的是 cover 欄（內含 `<img>`，textContent 為空字串），故 `expect("").toContain("完整資訊書籍")` 失敗。此為測試斷言 off-by-one：測試在 select-col 與 tag-col 兩欄加入產品前撰寫，未隨產品欄位演進同步更新。

判定：測試 stale，非產品 render bug。產品 `_formatBookSource` / `_formatBookRowData` 對測試資料（progress:75, status:'閱讀中', source:'readmoo'）渲染正確，7 欄結構完整。

### 影響範圍

- tests/unit/overview/overview-import-function.test.js（單一測試案例，4 行斷言）
- 無產品程式碼受影響

### 相關 Error Pattern

無直接對應 error-pattern。屬「產品結構演進後測試斷言未同步」的 stale test 類型，與 source ticket 0.19.0-W1-011 清查的 pre-existing UI 測試 tech debt 一致。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

修正 tests/unit/overview/overview-import-function.test.js「應該正確處理表格顯示更新」測試的 `td:nth-child` 斷言偏移：`nth-child(2..5)` -> `nth-child(3..6)`，對齊產品 `createBookRow` 的實際 7 欄結構（select / cover / title / source / progress / status / tag）。

同時於斷言前補上註解，說明欄位順序與 select-col（W6-012.7.1）、tag-col 兩欄的來源，避免後人再次誤判。

### 設計決策

- 僅改測試斷言，不動產品 `src/`：產品 render 邏輯正確，問題純為測試斷言未隨產品欄位演進同步。
- 採偏移修正而非改寫測試 DOM：測試斷言走 `row.querySelector('td:nth-child(N)')` 對實際渲染的 `<tbody>` row 查詢，與測試 JSDOM 的 `<thead>` 宣告無關。

### Phase 4 評估

- 重構需要：否
- 技術債務：發現 1 項（非本 ticket 範圍）
- 程式碼品質：A

結論：本次修復為單一測試案例 4 行斷言偏移 + 說明註解，無產品程式碼變更，無新增技術債，無需重構。

額外發現（quality-baseline 規則 5）：測試 `beforeEach` 的 JSDOM `<thead>` 僅宣告 5 個 `<th>`（封面/書名/來源/進度/狀態），與產品實際渲染的 7 欄 `<tbody>` 不一致。此 thead 不影響任何斷言（無測試查詢 thead），屬既存 cosmetic 不一致，不阻擋本 ticket。已評估：屬 source ticket 0.19.0-W1-011 清查的 pre-existing UI 測試 tech debt 同一類別，優先級低，不另建 ticket（避免 ticket 膨脹），改在本 Solution 顯性記錄供後續 UI 測試整體整理時一併處理。
