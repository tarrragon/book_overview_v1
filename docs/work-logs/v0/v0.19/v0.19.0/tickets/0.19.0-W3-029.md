## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

Claude Code v2.1.144 將 /extra-usage 命令重新命名為 /usage-credits（舊名仍可用但屬 deprecated path）。需盤點專案內是否有引用舊名的規則、Skill、Hook 或文件。

### 影響範圍

經全專案 grep 確認：無任何檔案引用 extra-usage 或 usage-credits（僅本 ticket 自身）。影響範圍為零。

### 相關 Error Pattern

無相關已知錯誤模式。
## 重現實驗結果

### 實驗方法

使用 ripgrep 對全專案（排除 node_modules/dist/.git）執行 fixed-string 搜尋，分別搜尋 extra-usage、/extra-usage、usage-credits 三個關鍵字。額外對 .claude/ 和 docs/ 目錄做二次確認搜尋。

### 實驗執行

1. rg -rn --fixed-strings 'extra-usage' 全專案
2. rg -rn --fixed-strings 'usage-credits' 全專案
3. rg -rn --fixed-strings '/extra' .claude/ — 排除 false positive（extractors 路徑）
4. 對 .claude/ 單獨搜尋排除 ticket 自身後確認無命中

### 實驗發現

已驗證事實：extra-usage 和 usage-credits 在全專案中僅出現在本 ticket 自身（0.19.0-W3-029.md）。.claude/rules/、.claude/skills/、.claude/hooks/、.claude/pm-rules/、.claude/references/、docs/、CLAUDE.md 等所有位置均無引用。

結論：專案內不存在需要更新的引用點。/extra-usage 命令從未被寫入專案的規則、Skill、Hook 或文件中。
## Solution

### WRAP 分析

#### W（Widen）擴增選項

| 選項 | 說明 |
|------|------|
| A. 立即批次更新 | 將所有 /extra-usage 引用更新為 /usage-credits |
| B. 漸進替換 | 遇到時順手更新，不主動掃描 |
| C. 不更新 | 舊名仍可用，維持現狀 |
| D. 無需動作 | 專案內無引用，無更新標的 |

#### R（Reality-Test）確認事實

grep 全專案結果：extra-usage 和 usage-credits 僅出現在本 ticket 自身。.claude/rules/、.claude/skills/、.claude/hooks/、.claude/pm-rules/、.claude/references/、docs/、CLAUDE.md 均無引用。

#### A（Attain Distance）評估

選項 A/B/C 的前提（專案內存在需更新的引用）不成立。唯一合理選項為 D。

#### P（Prepare to be Wrong）

若未來有新 Skill 或文件引用此命令，應直接使用新名 /usage-credits。本 ticket 無需 spawn DOC。
