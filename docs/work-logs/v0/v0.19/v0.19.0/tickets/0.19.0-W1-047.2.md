## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

IMP-A（W1-047.1）已讓 `BookFileImporter` 接入版本偵測與 v1→v2 轉換，但回傳值仍為純 `books` 陣列。`_extractBooksFromData` 在 v1 路徑只取 `convertV1ToV2Data(data).books`、在 v2 路徑完全忽略頂層 `tagCategories`/`tags`（程式碼註解明示「tagCategories/tags 暫時丟棄，回傳介面變更屬 IMP-B」）。tag 資料在匯入第一站即遺失，後續 IMP-C 的 storage 持久化無資料可寫。

### 影響範圍

| 檔案 | 影響 |
|------|------|
| `src/overview/book-file-importer.js` | `_extractBooksFromData` 新增 tag 區段提取；`_processBookData` / `_handleFileContent` / `_readFileWithReader` / `handleFileLoad` 回傳結構由陣列改為 `{ books, tagCategories, tags }` |
| `src/overview/overview-page-controller.js` | `handleFileLoad` 與 `_handleFileContent` 代理方法解構 `books` 後傳給 `_updateUIWithBooks`；`_updateBooksData` 不變 |

### 相關 Error Pattern

無直接對應的 PC error pattern。此為介面契約擴充（IMP-A 已預留銜接點），非流程或實作錯誤。

### Phase 1 功能規格

完整功能規格見 `0.19.0-W1-047.2-feature-spec.md`（Full 模式，6 區段 + 10 個 GWT 場景 + 4 條細化 AC）。spec validate 迭代 2 次通過，無未回答問題。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### Phase 1 設計摘要（lavender-interface-designer）

**核心介面變更**：定義 `ImportResult = { books: Book[], tagCategories: TagCategory[], tags: Tag[] }`，三欄位型別恆為陣列。

**回傳鏈變更**：`_extractBooksFromData` → `_processBookData` → `_handleFileContent` → `_readFileWithReader` → `handleFileLoad` 五處回傳由 `Book[]` 改為 `ImportResult`。

**各路徑 tag 來源**：
- v1 JSON：取 `convertV1ToV2Data(data)` 的 `.tagCategories`/`.tags`（converter 回傳已含三區段，IMP-A 僅取 `.books`，本 ticket 補取其餘兩段）。
- v2 JSON：從根物件讀 `data.tagCategories`/`data.tags`，缺失或非陣列降級為 `[]`（SPEC §2.3）。
- CSV / 空物件：`tagCategories: []`、`tags: []`。

**controller 適配**：`handleFileLoad` 與 `_handleFileContent` 解構 `books` 傳給 `_updateUIWithBooks`；`tagCategories`/`tags` 本 ticket 暫不流入 UI 層（IMP-C 接手 storage 持久化）。`_updateBooksData` 簽名與非匯入呼叫端不變。

**範圍邊界**：僅 v2 結構提取 + 回傳介面變更；覆蓋/合併模式、storage 持久化、tag id 重映射、模式選擇 UI 屬 IMP-C~E。

**錯誤處理原則**：tag 區段「寬鬆降級」（缺失/型別錯 → `[]`，不拋錯）；books 提取「維持既有嚴格契約」（無法辨識結構仍拋 `VALIDATION_ERROR`）。本 ticket 不新增拋錯路徑。

詳見 `0.19.0-W1-047.2-feature-spec.md`。Phase 2（sage 測試設計）依該規格 10 個場景與 AC-1~AC-4 設計 RED 測試。
