## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W3-031.3 ANA 結論：3 個可 exit 2 的 PostToolUse hook（phase-completion-gate / ticket-id-validator / ticket-quality-gate）值得啟用 continueOnBlock，讓 Claude 阻擋後可收到回饋訊息自行修正，減少 PM 介入。

### 影響範圍

`.claude/settings.json`：4 處 hook 註冊（非 ANA 原寫的 3 處）

### ANA 假設校準（執行中事實發現）

ANA W3-031.3 標記 phase-completion-gate-hook (Bash) 但實際註冊位置為：
| 行號 | Hook | 實際 matcher |
|------|------|-------------|
| L446 | phase-completion-gate-hook | Edit (非 Bash) |
| L489 | ticket-id-validator-hook | Write |
| L493 | phase-completion-gate-hook | Write |
| L515 | ticket-quality-gate-hook | Write |

phase-completion-gate-hook 註冊在 Edit + Write 各一處，共 4 處需修改而非 3 處。ANA 的「(Bash)」標記為事實錯誤，本 IMP 依實際 settings.json 結構執行，校準範圍為 4 處註冊。

### 相關 Error Pattern

無直接相關 PC。屬「ANA 假設與實際結構不符」的執行中校準，依 quality-baseline 規則 3「設計問題立即修正」處理。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

在 `.claude/settings.json` PostToolUse 4 處 hook 註冊新增 `"continueOnBlock": true`：

| 行號 | Matcher | Hook | 動作 |
|------|---------|------|------|
| L446-447 | Edit (PostToolUse) | phase-completion-gate-hook | 新增 |
| L489-490 | Write (PostToolUse) | ticket-id-validator-hook | 新增 |
| L493-494 | Write (PostToolUse) | phase-completion-gate-hook | 新增 |
| L515-516 | Write (PostToolUse) | ticket-quality-gate-hook | 新增 |

phase4-decision-enforcement-hook 依 ANA 結論刻意保持現狀（強制決策閘門，不應讓 Claude 自行繞過）。

### 行為效果

- Hook exit 2（block）時 Claude 仍會收到 additionalContext / stderr 回饋
- Claude 可依錯誤訊息自行修正（如補完 ticket body 章節、修 ticket ID 格式、補完 phase 條件）後重試
- 減少 PM 手動介入「Claude 撞 hook 後靜默放棄」場景

### 風險防護

| 風險 | 緩解 |
|------|------|
| Claude 無限重試 | CC runtime 內建重試上限 |
| 修正方向錯誤 | hook 訊息已含明確修正指引（既有設計） |
| phase4 閘門被繞過 | 此 hook 不加 continueOnBlock，維持硬阻擋 |

### 設計決策

- 採用 ANA W3-031.3「整合 3 個 hook」建議，實際對應 4 處註冊（ANA Bash 標記為事實錯誤，本 IMP 校準）
- 既有 l10n-sync-verification-hook (Edit/Write) 已設 continueOnBlock，無需改動
- 24 個純 advisory (exit 0) hook 不在範圍（continueOnBlock 對 exit 0 無行為影響）
