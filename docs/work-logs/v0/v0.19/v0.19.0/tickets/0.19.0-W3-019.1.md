## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 重構摘要

**目標**：將 `find_violations` 從三分支 flag pattern 改為 CATEGORY_MAP 單一 dict.get lookup，降低 TD-4/TD-5 擴充摩擦。

### 改動內容

1. **新增 `_build_category_map()` + 模組層級 `CATEGORY_MAP`**（`.claude/hooks/askuserquestion-charset-guard-hook.py`）
   - 建表順序：簡體字 → 日文漢字 → emoji（`setdefault` 確保先入者贏）
   - 模組層級單次建構，記憶體實測 `len(CATEGORY_MAP) == 2010 entries`（低於 ANA 估算 290 KB）
   - 保留原始常數 `SIMPLIFIED_CHARS` / `JAPANESE_ONLY` / `EMOJI_RANGES`（CONFUSABLE_PAIRS self-check 仍依賴）

2. **重寫 `find_violations`**
   - 移除 `matched_emoji` flag 變數
   - 移除 A/B/C 三分支 if-continue
   - 改為單一 `category = CATEGORY_MAP.get(char)` lookup

3. **新增 4 個測試**（`.claude/hooks/tests/test_askuserquestion_charset_guard.py` `TestCategoryMapConstruction`）
   - `test_category_map_contains_all_simplified_chars` — 覆蓋 SIMPLIFIED_CHARS 全量
   - `test_category_map_contains_all_japanese_only` — 覆蓋 JAPANESE_ONLY 全量
   - `test_category_map_contains_emoji_range_samples` — 每 range 驗證起始/結束/中點 codepoint
   - `test_category_map_returns_none_for_traditional_chinese` — 純繁體字應回傳 None

### 前後對比

**重構前**（三分支 flag pattern）：
```python
for idx, char in enumerate(text):
    code = ord(char)
    if char in SIMPLIFIED_CHARS:
        violations.append(...); continue
    matched_emoji = False
    for range_start, range_end in EMOJI_RANGES:
        if range_start <= code <= range_end:
            violations.append(...); matched_emoji = True; break
    if matched_emoji: continue
    if char in JAPANESE_ONLY:
        violations.append(...); continue
```

**重構後**（單一 lookup）：
```python
for char in text:
    category = CATEGORY_MAP.get(char)
    if category:
        violations.append((field_path, char, ord(char), category))
```

### TD-4/TD-5 擴充路徑

未來新增類別只需在 `_build_category_map()` 內擴充 `mapping.setdefault(char, "新類別")`，`find_violations` 本體不變。

### 設計決策

- **方案 A 採用**：純 CATEGORY_MAP dict lookup（ANA W3-019 結論）
- **未選方案 B**：保留 EMOJI_RANGES range check + 簡日/dict——維持兩路徑反而增加認知負擔
- **`setdefault` 而非 `[]=`**：先入者贏的語意更顯式；若未來簡體/日文清單與 emoji range 有 codepoint 衝突，第一類別優先（簡體最敏感）
