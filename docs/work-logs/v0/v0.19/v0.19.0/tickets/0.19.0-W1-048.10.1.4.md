### Phase 2 測試設計（2026-05-24，sage-test-architect）

#### 測試策略決策

| 項目 | 決策 | 理由 |
|------|------|------|
| 測試類型 | 整合測試（importer + 注入 stub helper） | importer 為 thin orchestrator，測試重點在 DI 與 delegate 行為 |
| 測試檔案異動 | 改寫 3 既有檔（import-mode-modal.test.js / overview-page-controller.test.js / overview-import-private-methods.test.js）+ 既有 importer 測試檔（book-file-importer.test.js 等）需確認仍綠 | 8 處 jest.fn 直接賦值替換為 stub helper 注入 |
| Mock 策略 | 統一使用 createStubHelpers() 工廠注入 validator/reader/parser stub | 取代 `importer._method = jest.fn()` 反模式 |

#### 共用 stub helper 規格（必須跨檔對齊）

新增 `tests/helpers/import-stub-helpers.js`：

```js
function createStubValidator (overrides = {}) {
  return {
    validate: jest.fn(),
    detectFormat: jest.fn(() => 'json'),
    ...overrides
  }
}
function createStubReader (overrides = {}) {
  return {
    read: jest.fn().mockResolvedValue({ books: [], tagCategories: [], tags: [] }),
    ...overrides
  }
}
function createStubParser (overrides = {}) {
  return {
    parse: jest.fn().mockReturnValue({ books: [], tagCategories: [], tags: [] }),
    ...overrides
  }
}
function createStubHelpers (overrides = {}) {
  return {
    validator: createStubValidator(overrides.validator),
    reader: createStubReader(overrides.reader),
    parser: createStubParser(overrides.parser)
  }
}
```

8 處遷移後 pattern：

```js
// Before（反模式）
controller.bookFileImporter._readFileWithReader = jest.fn().mockResolvedValue(...)

// After（注入 stub helper）
const { validator, reader, parser } = createStubHelpers({
  reader: { read: jest.fn().mockResolvedValue({ books: [...], ... }) }
})
const importer = new BookFileImporter({ document, showError, validator, reader, parser })
controller.bookFileImporter = importer  // 或 controller 建構時注入
```

#### DI Constructor 簽章測試

新增獨立 describe `BookFileImporter constructor (DI)`：

| Scenario | Given | When | Then |
|---------|-------|------|------|
| 預設自建 helpers | 僅傳 {document, showError} | new BookFileImporter | importer.validator instanceof FileValidator；importer.reader instanceof FileContentReader；importer.parser instanceof ContentParser |
| 注入 validator | 傳入 stubValidator | new BookFileImporter | importer.validator === stubValidator |
| 注入 reader | 傳入 stubReader | new BookFileImporter | importer.reader === stubReader |
| 注入 parser | 傳入 stubParser | new BookFileImporter | importer.parser === stubParser |
| reader 預設用內部 parser | 僅傳 stubParser | new BookFileImporter | 預設 reader.parser === stubParser（生產情境鏈一致） |

#### 4 Public Method Thin Wrapper 向後相容測試

每個 public method 一組 GWT，驗證 delegate 行為而非實作細節：

| Method | GWT |
|--------|-----|
| `validate(file)` | Given stubValidator + importer，When validate(file)，Then stubValidator.validate 被呼叫 1 次參數 file |
| `read(file)` | Given stubReader + importer，When read(file)，Then stubReader.read 被呼叫 1 次參數 file；回傳值 === stub resolved value |
| `parseContent(content, fileFormat)` | Given stubParser + importer，When parseContent(c, 'json')，Then stubParser.parse 被呼叫 1 次參數 (c, 'json')；回傳值 === stub return |
| `handleFileLoad(file)` | Given stubValidator + stubReader + importer，When handleFileLoad(file)，Then validate 先被呼叫（順序斷言），read 後被呼叫，回傳 reader 結果 |
| handleFileLoad 失敗短路 | stubValidator.validate throw，When handleFileLoad，Then stubReader.read 不被呼叫 |
