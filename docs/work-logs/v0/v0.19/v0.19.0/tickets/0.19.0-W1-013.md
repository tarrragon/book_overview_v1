## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-010 已使 `extractCoverAndTitle` 保留相對路徑封面（移除誤判清空），但 `extractCoverIdFromUrl` 呼叫鏈仍有雙重阻擋：

1. `validateCoverUrlInput:1689` 呼叫 `this.isUnsafeUrl(trimmed)` 未傳 base → 相對路徑單參數 `new URL()` throw → 判定 unsafe → 回傳 `null`
2. `validateReadmooDomain:1701` 呼叫 `new URL(url)` 未傳 base → 相對路徑 throw → `handleWithFallback` 回傳 `false`

兩處任一返回 null/false，`extractCoverIdFromUrl` 即回傳 `null` → `tryCoverStrategy` 失效 → coverId 無法作為書籍 ID 來源。

### 影響範圍（Phase 1 釐清）

呼叫鏈 grep 結果（限定 `src/` + `tests/`）：

| 函式 | 定義位置 | 呼叫者 |
|------|---------|--------|
| `extractCoverIdFromUrl` | `readmoo-adapter.js:1734` | `hasRequiredFields:1108`、`parseBookElement identifiers.coverId:1195`、`tryCoverStrategy:1590` |
| `validateCoverUrlInput` | `readmoo-adapter.js:1686` | 僅 `extractCoverIdFromUrl:1735` |
| `validateReadmooDomain` | `readmoo-adapter.js:1697` | 僅 `extractCoverIdFromUrl:1738` |

**結論**：
- 所有呼叫鏈節點集中於 `src/content/adapters/readmoo-adapter.js`，無跨檔案影響
- `validateCoverUrlInput` / `validateReadmooDomain` 為 `extractCoverIdFromUrl` 私有 helper，重構安全
- 範圍未超預期，不觸發 NeedsContext / Exit Status needs_context

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

修改 `src/content/adapters/readmoo-adapter.js` 兩處 URL 解析以支援相對路徑：

| 位置 | 修改前 | 修改後 |
|------|--------|--------|
| `validateCoverUrlInput:1689` | `this.isUnsafeUrl(trimmed)` | `this.isUnsafeUrl(trimmed, getLocation().origin)` |
| `validateReadmooDomain:1697-1706` | `new URL(url)` 嚴格 hostname 檢查 | 區分 isRelative 分支 + 接受相對 `/cover/` 路徑 |

### validateReadmooDomain 設計決策

絕對 vs 相對路徑採不同信任模型：

| URL 類型 | 驗證規則 | 理由 |
|---------|---------|------|
| 絕對 URL | `hostname === 'cdn.readmoo.com'` AND `pathname.includes('/cover/')` | 保留原嚴格性，防止 evil.com 偽裝 |
| 相對 URL（startsWith `/` 且非 `//`） | `pathname.includes('/cover/')` | 相對 URL 來自 Readmoo SPA 本身，隱含 readmoo-domain；已過上游 `isUnsafeUrl` 安全閘 |

排除 protocol-relative URL（`//evil.com/cover/...`）：採 `startsWith('/') && !startsWith('//')` 判斷，避免被誤判為合法相對路徑。

### 安全保證（測試覆蓋）

下列攻擊向量在「相對路徑支援」加入後仍會被攔截：

| 攻擊向量 | 攔截層 |
|---------|--------|
| `javascript:alert(1)` | `validateCoverUrlInput` → `isUnsafeUrl` 協議檢查 |
| `data:image/png;base64,...` | 同上 |
| `/cover/../../../etc/passwd` | `isUnsafeUrl` 路徑遍歷原始字串檢查（W1-010 強化） |
| `/cover/%2e%2e/...` | 同上（編碼遍歷檢查） |
| `https://evil.com/cover/xyz.jpg` | `validateReadmooDomain` 絕對 URL hostname 嚴格檢查 |
| `//evil.com/cover/xyz.jpg` | protocol-relative 不視為相對路徑，hostname 檢查擋下 |
| `/other/abc/xyz.jpg`（非 `/cover/`） | `validateReadmooDomain` 路徑特徵檢查 |

### 不在本 ticket 範圍

- W1-010 cover URL `isUnsafeUrl` base 修復已完成（`extractCoverAndTitle:1082`）
- W1-012 reader link `extractHrefFromElement` base 修復已完成（`:1004`）
- 本 ticket 完成後，相對 cover URL 從 DOM 提取 → 保留 → 通過 isUnsafe 檢查 → 通過 readmoo domain 檢查 → 成功提取 coverId 之端到端鏈路修復完備
