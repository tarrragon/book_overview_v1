|------|------|

**關鍵契約**：

1. **schema_version**：'3.0.0'（BookSchemaV2.SCHEMA_VERSION）與 §2 對齊
2. **readingStatus 6 enum**：unread / reading / finished（自動可追蹤）+ queued / abandoned / reference（手動專用）
3. **自動轉換 2 path**：unread→reading（0→>0）/ reading→finished（=100）；isManualStatus=true 鎖定
4. **手動覆寫規則**：設手動專用狀態 → isManualStatus=true；設自動狀態 → isManualStatus=false（恢復追蹤）
5. **State machine 不變式 5 條**：初始 unread / 手動 lock / 手動恢復 / 自動 bound / backward block
6. **v1→v2 優先級 4 條**：isFinished 最優先 → progress=100 → progress>0 → 預設 unread
7. **Progress 正規化**：null/NaN/字串/object 各情境的歸 0 規則
8. **E2E 對照常數**：READING_STATUS_ENUM (6) / DERIVABLE_STATUS (3) / PROGRESS_TO_STATUS map

### 關鍵設計決策

1. **手動狀態鎖定機制**：avoid 用戶手動標 abandoned 後 progress 變化又被自動轉回 reading 的衝突。isManualStatus 旗標將兩個 sub-state machine（自動 / 手動）明確分離。
2. **自動 backward 不轉換**：reading→unread / finished→reading 不自動觸發，避免下游同步衝突時自動回退用戶閱讀進度。需 backward 必須用戶手動操作。
3. **首次提取限制**：DERIVABLE_STATUS 三種為首次提取結果上界（queued/abandoned/reference 僅手動設定可達），E2E 測試據此斷言。
