session-start `project-init check` 對 OUTDATED 自製套件僅輸出 informational 行（`{name} ({version}) [OUTDATED]` + 縮排 reinstall 指令），與其他常規區段視覺上無區別。PM 看到後不會主動 reinstall，導致 W6-001.2 設計的 MCP 檢查功能與最新原始碼脫鉤，對 session 完全沉默失效（PC-164 第二層遮蔽機制）。

- `.claude/skills/project-init/project_init/commands/check.py` — `_print_check_result` summary 區塊
- `.claude/skills/project-init/tests/test_messages.py` — 對應測試
- 下游：所有 session-start 時觀察 `project-init-env-check-hook` 輸出的 PM / agent

### 派發執行指引（PM Context）

**工作目錄**: main repo `/Users/mac-eric/project/book_overview_v1`（不開 worktree，ARCH-015）

**根因（從 W1-101 + W1-102 + PC-164 提煉）**：

session-start project-init check 對「OUTDATED 套件」僅輸出 informational 行（範例）：

```
project-init (1.1.0) [OUTDATED]
    → 原始碼已更新，需重新安裝: uv tool install . --force --reinstall
```

訊息淹沒在常規輸出中，PM 看到後不會主動 reinstall，導致 W6-001.2 設計的 MCP 檢查功能對 session 完全沉默失效（PC-164 第二層遮蔽機制）。

**修改範圍**:

主要檔案：`.claude/hooks/project-init-env-check-hook.py`（或 `.claude/skills/project-init/project_init/lib/` 對應輸出邏輯）

具體方向（agent 評估後選一）:

1. **方向 A（顯眼前綴）**: OUTDATED 訊息加入顯眼前綴（如 `[WARNING-STALE]` 或方框框起來），與其他 informational 視覺區隔
2. **方向 B（stderr 雙通道）**: 額外將 OUTDATED 訊息寫入 stderr（依 quality-baseline 規則 4「雙通道可觀測性」），確保 PM session terminal 醒目顯示
3. **方向 C（summary 警示）**: 在 project-init check 報告底部「總結」前加入 `[WARNING] N 個套件 OUTDATED，請執行 reinstall` 一行，無 OUTDATED 時不顯示

**推薦**: 方向 A + 方向 C 組合（顯眼前綴 + summary 警示），不採方向 B 因為 stderr 可能被部分 terminal 配置遮蔽。

**Acceptance 對齊**:

- [ ] project-init-env-check-hook.py 對 OUTDATED 套件升級為強制提醒（stderr + 顯眼前綴）
- [ ] 提醒含具體 reinstall 指令（已存在於原 informational 訊息中，保留即可）
- [ ] 不阻擋 session 啟動但確保 PM 注意（exit code 仍為 0）

**驗證**:

1. 模擬 OUTDATED：將 project-init 套件移除（`uv tool uninstall project-init`）或保持當前 OUTDATED 狀態
2. 執行 `project-init check` 確認輸出含顯眼提醒
3. 重新 trigger session-start hook（透過建立新 terminal session 或 mock 方式）確認顯示
4. reinstall 後（`uv tool install . --force --reinstall`）確認警示消失

### 修改摘要

**`.claude/skills/project-init/project_init/lib/messages.py`**

- `PackageMessages.OUTDATED` 從 `"{name} ({version}) [OUTDATED]"` 改為 `">>> [STALE-CLI] {name} ({version}) [OUTDATED] — 必須 reinstall <<<"`，加入顯眼 `>>> ... <<<` 框與 `[STALE-CLI]` 前綴。
- 新增 `PackageMessages.OUTDATED_SUMMARY_WARNING`，含套件數量 + 完整 reinstall 指令。

**`.claude/skills/project-init/project_init/commands/check.py`**

- 新增 `_count_outdated_packages(result)` helper，掃描「自製套件」section details 計數 `[OUTDATED]` 出現。
- `_print_check_result` 在 summary 行前判定 `outdated_count > 0` 時印出 `OUTDATED_SUMMARY_WARNING`（無 OUTDATED 時不顯示，避免雜訊）。

**`.claude/skills/project-init/tests/test_messages.py`**

- 既有 `test_outdated_format` 保留（`[OUTDATED]` substring 斷言仍通過）。
- 新增 `test_outdated_eyecatching_prefix` 斷言含 `[STALE-CLI]` 與「必須 reinstall」。
- 新增 `test_outdated_summary_warning_format` 斷言含 `[WARNING]`、count、reinstall 指令。

### 設計決策

- 顯眼前綴用純 ASCII（`>>>` / `<<<` / `[STALE-CLI]`）而非 emoji，遵循 language-constraints.md 規則 3 + document-format-rules.md。
- summary 警示僅在 outdated_count > 0 時顯示，避免污染正常狀態輸出（PC-080 框架測試：「能一句話描述觸發條件」）。
- 不修改 hook 本體；hook 透過 additionalContext 已是適當推送通道，僅需強化字面訊息顯眼度。
