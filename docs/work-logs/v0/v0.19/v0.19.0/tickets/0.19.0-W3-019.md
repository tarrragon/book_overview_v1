## Problem Analysis
### 問題根因

`find_violations` 函式（L226-261）採三分支 if-continue 線性堆疊：

1. **分支 A**（L237）：`char in SIMPLIFIED_CHARS` → frozenset O(1) lookup
2. **分支 B**（L241-246）：`for range_start, range_end in EMOJI_RANGES` → 6 tuple 線性掃描 + `matched_emoji` flag 變數
3. **分支 C**（L249）：`char in JAPANESE_ONLY` → frozenset O(1) lookup

**Good-taste 問題**：
- 分支 B 使用 flag 變數 `matched_emoji` 控制 continue，是典型 special-case 補丁
- 每新增類別（平假名/片假名/韓文/Write-Edit 工具覆蓋）需複製一段 if-continue pattern
- 優先序由程式碼順序隱含表達，非顯性資料結構

### 影響範圍

- 檔案：`.claude/hooks/askuserquestion-charset-guard-hook.py`（408 行）
- 函式：`find_violations`（35 行）、`scan_payload`（呼叫者）
- 測試：`.claude/hooks/tests/test_askuserquestion_charset_guard.py`（43 tests）
- 後續擴充受影響：TD-4（平假名/片假名）、TD-5（韓文）、W3-051 Write-Edit 工具覆蓋

## Solution
### Analysis Result：CATEGORY_MAP 重構方案

### 方案評估

#### 方案 A：純 CATEGORY_MAP dict（每字元 → 類別名稱）

```python
# 將所有字元（含 emoji range 展開）統一建為 dict
CATEGORY_MAP: dict[str, str] = {}
for ch in SIMPLIFIED_CHARS:
    CATEGORY_MAP[ch] = "簡體字"
for ch in JAPANESE_ONLY:
    CATEGORY_MAP[ch] = "日文漢字"
for range_start, range_end in EMOJI_RANGES:
    for cp in range(range_start, range_end + 1):
        CATEGORY_MAP[chr(cp)] = "emoji"
```

**優點**：find_violations 簡化為 1 行 lookup；新增類別 = 1 行 dict update
**缺點**：emoji 6 ranges 展開約 2,816 entries（非 9MB，實測遠小於預期），但仍是靜態浪費

**記憶體估算**（修正）：
- SIMPLIFIED_CHARS：66 字元
- JAPANESE_ONLY：17 字元
- EMOJI_RANGES 展開：(0x27BF-0x2600+1) + (0x1F5FF-0x1F300+1) + ... = 2,816 entries
- 總計約 2,899 entries × ~100 bytes/entry ≈ 290 KB（非 9MB，原 ticket 估算有誤）

**結論**：記憶體完全可接受。

#### 方案 B：混合策略（dict for 字元型 + 保留 range for emoji）

```python
CHAR_CATEGORY_MAP: dict[str, str] = {}
for ch in SIMPLIFIED_CHARS:
    CHAR_CATEGORY_MAP[ch] = "簡體字"
for ch in JAPANESE_ONLY:
    CHAR_CATEGORY_MAP[ch] = "日文漢字"
# emoji 保留 range check
```

**優點**：emoji range 不展開，記憶體更小
**缺點**：find_violations 仍需兩段邏輯（dict lookup + range check），未完全消除分支

### 推薦方案：方案 A（純 CATEGORY_MAP）

**理由**：
1. 記憶體 290 KB 完全可接受（hook 啟動一次性成本，非持續佔用）
2. `find_violations` 簡化為：
   ```python
   def find_violations(text: str, field_path: str) -> list:
       violations = []
       for idx, char in enumerate(text):
           category = CATEGORY_MAP.get(char)
           if category:
               violations.append((field_path, char, ord(char), category))
       return violations
   ```
3. `matched_emoji` flag 變數消失
4. 新增類別（平假名/片假名/韓文）= 一行 `for ch in NEW_SET: CATEGORY_MAP[ch] = "類別名"`
5. 優先序由建表順序天然表達（先入者贏），語意清晰

### 重構步驟

| 步驟 | 內容 | 風險 |
|------|------|------|
| 1 | 在模組層級建立 `CATEGORY_MAP: dict[str, str]`，依序注入三類別 | 低（純資料結構） |
| 2 | 重寫 `find_violations` 為單一 dict.get lookup | 低（邏輯等價） |
| 3 | 移除 `matched_emoji` flag 變數 | 無 |
| 4 | 保留 `SIMPLIFIED_CHARS` / `JAPANESE_ONLY` / `EMOJI_RANGES` 原始常數（CONFUSABLE_PAIRS self-check 依賴） | 關鍵：不可刪除原始常數 |
| 5 | 既有 43 tests 全數通過 | 驗收門檻 |
| 6 | 新增 CATEGORY_MAP 建表正確性測試（覆蓋三類別各至少 1 字元） | 必要 |

### 測試覆蓋要求
