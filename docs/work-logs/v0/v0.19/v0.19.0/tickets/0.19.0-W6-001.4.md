### 三 MCP 實證紀錄（PM 前台 cbm 部分，acceptance 1 + 6）

**cbm CLI 模式驗證**（MCP 工具未在 ToolSearch deferred 曝光，改用 `codebase-memory-mcp cli <tool> <json>` 等效呼叫）：

1. `list_projects '{}' ` → `{"projects":[],"hint":"No projects indexed."}` — index 完全空白
2. `index_repository '{"repo_path": "/Users/tarragon/Projects/book_overview_v1", "include_globs": [".claude/**/*.md", ".claude/**/*.py", "docs/**/*.md"]}'` → indexed nodes=63155 edges=90919 elapsed_ms=3283
3. `search_code '{"project": "Users-tarragon-Projects-book_overview_v1", "pattern": "AGENT_PRELOAD"}'` → 10 results，**全部 docs/，無任何 .claude/**
4. 對照 `grep -rl "mcp__serena__find_symbol" .claude/` → 11 個 .claude/ 檔案（含 .claude/agents/AGENT_PRELOAD.md）
5. 重 index 加 `include_hidden: true` + `force_reindex: true` + 空 `exclude_globs: []` → 仍無 .claude/ 結果
6. `delete_project` 後 clean reindex 同條件 → 依舊不 index .claude/

**結論**：cbm v0.6.1 對 `.claude/` 目錄完全不索引，且 `include_globs` / `include_hidden` 無法覆寫。相同條件下 `.codex/` 可索引（前置實驗 search "lsp-first" 命中 .codex/skill-trigger-map.md），代表非統一 hidden dir blacklist，而是 .claude/ 特定的 hardcoded skip。
