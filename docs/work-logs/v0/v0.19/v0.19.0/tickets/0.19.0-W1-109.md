### 問題根因（PM 補強，W1-108 完成後現況）

GlobalMessages `_loadDefaultMessages()` 含 83 keys 但僅 ~20 個屬真正跨模組共用。其餘 ~60 個為 module-specific 污染，所有 caller 已有對應 local dict（議題 A 邊界違反），可安全清理。

### Caller Local Dict 覆蓋狀態（清理前提）

| 模組 | local dict 狀態 | 應移除 key 數 | 風險評估 |
|------|---------------|--------------|---------|
| search-filter | searchUIMessages 已存在 | 8 | 安全 |
| validator | validatorMessages 已存在 | 15 | 安全 |
| extractor / readmoo-adapter | W1-108 已建 readmooAdapterMessages (36 keys) | 18 | 安全（W1-108 c5c5f21a 落地） |
| Logger | 無 local dict，但 caller 不直接傳 Logger key（DEBUG_MESSAGE 等）作 messageKey；Logger 內部用 `this.messages.get(key)` 動態 dispatch | 4 | 安全（grep 確認無 caller） |
| Chrome ext (POPUP_OPENED 等) | 無引用 | 4 | 安全可直接移除 |
| 中文 key 遺留 | 無引用 | 3 | 安全 |
| 書庫相關 | 多數僅 1 caller 引用 | 5 | 評估後處理 |
| popup-error-handler | 已有 local dict | (W1-108 涵蓋確認) | 安全 |
| 重疊 key（search-filter + validator local dict 已覆蓋） | local dict 已 addMessages 覆寫 | 25 | 安全 |

### 影響範圍

| 檔案 | 變更 |

| `src/core/messages/MessageDictionary.js` | `_loadDefaultMessages()` 移除 ~60 個 module-specific key |

### 相關 Error Pattern

PC-165 False Positive 修復鏈（W1-114 撰寫）— 清理後須含 runtime 驗證，避免 mock 綠燈但 runtime 失效。建議：除 unit test 外，integration 層斷言關鍵 caller（如 readmoo-adapter）渲染後訊息文字仍正確。

### 簡化 WRAP 三問

**W（Widen）候選方案**：
- 方案 1（本案）：一次性移除 ~60 keys
- 方案 2：分批移除（先 ext + 中文 + 書庫 = 12 keys 試水，再 search-filter + validator + extractor = 41 keys，最後 Logger = 4 keys）— 安全但 ticket 拆 3 個過細
- 方案 3：只移除「無引用」keys（~7 個：POPUP_OPENED 等），保留「重複定義」keys — 違反議題 A 邊界規範

**A（Attain distance）機會成本**：W1-109 完成解鎖 W1-110（freeze GlobalMessages）；不做則 freeze 後 GlobalMessages 仍含 60+ 污染 key

**P（Prepare to be wrong）失敗預想**：
- 風險 1：某 caller 仍直接引用被移除 key（grep 漏網）→ 對應防護：acceptance 3 強制 grep 驗證
- 風險 2：local dict 未 addMessages 該 key 而 fallthrough 到 GlobalMessages → 對應防護：跑 integration test 驗證真實渲染文字（PC-165）
