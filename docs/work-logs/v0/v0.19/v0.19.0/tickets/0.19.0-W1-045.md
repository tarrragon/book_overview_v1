## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

src/overview/book-exporter.js 的 EXPORT_CONSTANTS 定義（修復前 L14-16）僅含 NO_DATA_EXPORT 一個 key，但 copyText 相關方法引用了 4 個未定義的 key：

| 引用位置 | 引用的 key | 後果 |
|---------|-----------|------|
| handleCopyText L48 | NO_DATA_COPY | alert(undefined)，提示文字遺失 |
| handleCopyText L57 | COPY_SUCCESS | alert(undefined)，提示文字遺失 |
| handleCopyText L59/L65 | COPY_FAILED | alert(undefined)，提示文字遺失 |
| generateCopyText L78 | COPY_TEXT_HEADERS | undefined.join() → TypeError，copyText 觸發即崩潰 |

最嚴重的是 COPY_TEXT_HEADERS：generateCopyText 對 undefined 呼叫 .join() 直接丟 TypeError，使「複製為文字」按鈕（overview-page-controller.js L273-277 綁定）功能完全不可用。

### 影響範圍

- 受影響檔案：src/overview/book-exporter.js（唯一檔案）
- 受影響功能：overview 頁面「複製為文字」按鈕（copyTextBtn）
- 此為 pre-existing 缺陷：W1-042.2 清理 v1 JSON/CSV 匯出方法時，EXPORT_CONSTANTS 只保留了 NO_DATA_EXPORT，但 copyText 方法保留下來仍引用已被一併清除的 4 個 key。非 W1-042.2 清理造成，而是清理過程暴露此缺陷。

### 相關 Error Pattern

屬「常數定義與引用不同步」類缺陷——重構移除常數時未同步檢查所有引用點。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

在 src/overview/book-exporter.js 的 EXPORT_CONSTANTS 補齊 4 個缺漏 key：

| key | 值 | 用途 |
|-----|-----|------|
| NO_DATA_COPY | '沒有資料可以複製' | handleCopyText 無資料早退提示 |
| COPY_SUCCESS | '已複製到剪貼簿' | 複製成功提示 |
| COPY_FAILED | '複製失敗，請確認瀏覽器支援剪貼簿功能' | 剪貼簿 API 不支援或拋例外提示 |
| COPY_TEXT_HEADERS | ['書名', '書城', '閱讀進度', '狀態'] | 複製純文字的 header 列 |

### 設計決策

COPY_TEXT_HEADERS 欄位由 generateCopyText 實際組裝邏輯反推決定，須與資料列順序一致：

generateCopyText 每本書組裝 [book.title, _formatBookSource(book), String(book.progress||0), book.status] 共 4 欄，因此 header 對應為「書名 / 書城 / 閱讀進度 / 狀態」。_formatBookSource 回傳書城來源（tags / tag / store / source 多層 fallback），故第二欄命名為「書城」。

提示字串採繁體中文，與 NO_DATA_EXPORT 既有風格一致。

### 變更檔案

- src/overview/book-exporter.js：EXPORT_CONSTANTS 補 4 key（含註解說明各 key 用途與欄位順序對應）
- tests/unit/overview/book-exporter-copy-text.test.js：新增 copyText 觸發測試（7 個 test case）
