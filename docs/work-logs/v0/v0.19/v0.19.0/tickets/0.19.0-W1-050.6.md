## Problem Analysis

### 問題根因

W1-050.3 PM 前置 grep 發現 5 處 W1-050 ANA 範圍外的 process.* 違規。雖然 popup/content 兩處已有 typeof guard（CE runtime 不會崩潰），但仍出現在 build/development bundle，違反 W1-050 「process.* 全清」AC 標準；migration 3 檔的 process.cwd() 無 guard 且 build 包進 bundle，CE runtime 若實例化這些類別會 ReferenceError。

PM 驗證（已完成）：build/development/src/core/migration/ 確認含 MigrationProgressTracker / MigrationValidator / StandardErrorMigrationAnalyzer 三檔，bundle 內有 process.cwd 殘留。

### 影響範圍（5 檔 5 hits）

| 檔案 | 行號 | 當前 | 既有 guard | 處理方式 |
|------|------|------|----------|---------|
| `src/popup/diagnostic-module.js` | L1053 | `: process.platform` | navigator typeof | 移除 process.platform fallback，改 `'unknown'` literal |
| `src/content/detectors/page-detector.js` | L64-65 | `process.stderr.write(...)` | typeof process | 移除整塊（Jest debug 改 console.warn 或刪除） |
| `src/core/migration/MigrationProgressTracker.js` | L43 | `\|\| process.cwd()` | 無 | 加 typeof process guard fallback `'/'` |
| `src/core/migration/MigrationValidator.js` | L41 | `\|\| process.cwd()` | 無 | 同上 |
| `src/core/migration/StandardErrorMigrationAnalyzer.js` | L61 | `\|\| process.cwd()` | 無 | 同上 |

### 轉換規則細節

**A. popup/diagnostic-module.js L1053**
```js
// 改前：platform: typeof navigator !== 'undefined' ? navigator.platform : process.platform,
// 改後：platform: typeof navigator !== 'undefined' ? navigator.platform : 'unknown',
```

**B. content/page-detector L64-65**
```js
// 改前（保留 typeof guard 但 Jest debug 已過時）：
// if (typeof process !== 'undefined' && process.stderr) {
//   process.stderr.write(`PageDetector Debug: ${JSON.stringify(locationInfo)}\n`)
// }
// 改後：直接移除整塊。Jest 環境 debug 可用 console.warn 或保留為註解
```

**C. migration 3 檔 process.cwd()**
```js
// 改前：options.projectRoot || process.cwd()
// 改後：options.projectRoot || (typeof process !== 'undefined' && process.cwd ? process.cwd() : '/')
```

註：migration 類別在 CE runtime 不會主動實例化（屬 dev tool 路徑），typeof guard 是保險帶。Bundle 內仍有 `process.cwd` 字串殘留但被 guard 包裹，不執行。

### 修復摘要

依 Problem Analysis 5 hits 表執行，5 檔變更如下：

| 檔案 | 行號 | 修改前 | 修改後 |

### 設計決策

| 決策 | 採用 | 拒絕 | 理由 |
|------|------|------|------|
| popup platform fallback | `'unknown'` literal | `navigator.userAgentData` 或保留 process.platform | navigator typeof guard 已涵蓋 95% 路徑；fallback 僅在 CE 之外（never happens in CE runtime）才被觸發，literal 最簡且不引新依賴 |
| content stderr 處理 | 移除整塊（含 locationInfo） | 改 console.warn 或 AppLogger | Jest debug 已過時（測試框架已不需要這層 stderr 強制輸出）；locationInfo 物件在移除 stderr 後成 dead code，一併刪除 |
| migration cwd 處理 | typeof guard fallback `/` | 改 location.href 或保留 unguarded | migration 類別在 CE runtime 不會主動實例化（屬 dev tool 路徑），加 guard 是保險帶；fallback `/` 為 POSIX root，邏輯安全 |
