### W1-001.2 實機驗證發現（PM chrome-devtools MCP，2026-05-21）

本 ANA 由 W1-001.2 實機驗證中斷觸發。已確認事實：

**1. Readmoo URL 結構變更**

| 測試 URL | 結果 |
|---------|------|
| readmoo.com/library | 404（程式碼預期的提取頁已失效） |
| share.readmoo.com/me/bookshelf/total | 真實書櫃，tarragonstop 919 本書，DOM 含 300+ a[href*=/book/]、208 [class*=book] 元素 |
| read.readmoo.com/（new-read SSO 後）| reader app SPA，pathname=/ |

**2. 頁面偵測根因（已定位）**

- page-detector.js detectPageType()（約 line 85-102）：檢查 pathname.includes('/shelf')（含前導斜線），/me/bookshelf/total 的 bookshelf 不含 /shelf 子字串 -> 回傳 unknown。
- 對照 readmoo-adapter.js:1315-1319 用 pathname.includes('bookshelf') -> 兩層偵測邏輯不一致。
- page-detector.js isExtractablePage()（約 line 144）只認 ['library','shelf']。
- book-data-extractor.js:161 isExtractableReadmooPage() false -> 拋「不支援的頁面類型: unknown」。

**3. readmoo.com/library 硬編碼引用位置（grep 結果）**

- src/content/detectors/page-detector.js（detectPageType /library 判斷）
- src/content/adapters/readmoo-adapter.js:1333-1336（getSupportedUrls）
- src/background/domains/page/services/content-script-coordinator-service.js:187,298
- src/background/domains/page/services/tab-state-tracking-service.js:643
- src/core/errors/UC01ErrorFactory.js:73

**4. popup 矛盾初始化狀態**

popup 開啟時同時顯示「Content Script 連線正常／可以開始提取」與「Background Service Worker 連線失敗（init 背景服務檢查步驟失敗，15ms）」。疑為 Manifest V3 SW 休眠時序問題，但矛盾顯示本身是 UX 缺陷。

### 分析任務重點

1. 確認 canonical 提取頁：share.readmoo.com/me/bookshelf/total 是否為唯一目標？閱讀進度資料是否只在 reader app？
2. DOM 相容性：share.readmoo.com 書櫃 DOM 與既有 readmoo-adapter.js extractor selectors 是否相容——此項需 chrome-devtools 實測 probe（saffron 無瀏覽器，請在 spawned IMP 規劃實測任務）。
3. 引用清理範圍：上述 5 處 readmoo.com/library 引用。
4. spawned IMP 清單必須落地為實際 ticket（quality-baseline 規則 5）。
