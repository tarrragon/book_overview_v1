## Problem Analysis

### 問題根因

W11-018 Phase 4 多視角評估（linux/cinnamon/ginger）對 `phase4-decision-enforcement-hook.py` 提出三項共識性觀察條款，本 ANA 作為觀察容器追蹤觸發條件是否成立。

三項觀察條款：
- **(a) TD#4 抽 lib**：`compute_fenced_block_lines` 與 `compute_schema_placeholder_lines` 結構同構（接收 `List[str]` 回傳行號 `set`，逐行掃描含 open/close 狀態機），YAGNI 原則下不立即抽取，但當第 3 個同類 region 計算器出現時觸發重構。
- **(b) audit warning**：ginger 建議在 fenced/total 行數比 > 50% 時輸出 audit warning，防止格式擴大導致大面積豁免。
- **(c) inline backtick**：linux 指出規格選擇排除 inline backtick（僅 fenced code block），蒐集 false positive > 5 例後評估是否需擴展。

### 影響範圍

- `.claude/hooks/phase4-decision-enforcement-hook.py`（L295-382：兩個 region 計算函式；L385-420：`scan_lines_for_phrases` 呼叫處）

### 相關 Error Pattern

- PC-093（YAGNI 累積反模式）：hook 本身即為 PC-093 防護工具

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 觀察結論

三項條款截至 2026-05-26 均**未觸發**。本 ANA 作為觀察容器，定義各項的觸發判斷標準與後續 spawn 規劃。

### 監測項 (a)：TD#4 抽 lib — region 計算器同構重構

- **觸發條件**：`.claude/hooks/` 內出現第 3 個接收 `List[str]` 回傳行號 `set` 且含 open/close 狀態機的 region 計算函式
- **觸發判斷方法**：`grep -rn "def compute_.*_lines\|def _compute_.*_lines" .claude/hooks/*.py` 結果 >= 3
- **觸發後 spawn**：IMP ticket — 抽取共用 `RegionLineComputer` 或 `compute_region_lines(lines, start_pattern, end_pattern)` 通用函式至 `hook_utils/`
- **當前狀態**：2 個同構函式（未達觸發門檻）
- **無需建 ticket：YAGNI 原則，2 個同構不足以證明抽象的收益大於成本**

### 監測項 (b)：fenced/total > 50% audit warning

- **觸發條件**：實際使用中觀察到 ticket 的 fenced block 行數占比 > 50%，且該豁免導致 MUST-block 漏偵測
- **觸發判斷方法**：Phase 4 hook 掃描結果中出現「大量行被 fenced 豁免但實際含延後話術」的案例
- **觸發後 spawn**：IMP ticket — 在 `format_warn_info_message` 或 `main` 中新增 `fenced_ratio = len(fenced_lines) / len(lines)` 計算，> 50% 時輸出 INFO 級 audit warning
- **當前狀態**：無已知漏偵測案例
- **無需建 ticket：無實際漏偵測案例支撐，屬預防性建議，等實際案例出現再評估**

### 監測項 (c)：inline backtick false positive 蒐集

- **觸發條件**：蒐集到 > 5 例 inline backtick（如 `` `Phase 4 再決定` ``）內的 PC-093 誤報案例
- **觸發判斷方法**：ticket 或規則文件中 inline backtick 包裹的延後話術被 hook 命中且非實際延後決策
- **觸發後 spawn**：IMP ticket — 擴展 `scan_lines_for_phrases` 加入 inline backtick 範圍排除邏輯（類似 fenced block 但為行內 span）
- **當前狀態**：0 例已知誤報
- **無需建 ticket：尚無案例蒐集，inline backtick 內延後話術被命中的頻率極低**

### Spawn 落地確認

三項條款均未觸發，無需建 follow-up ticket。本 ANA 可作為長期觀察容器，未來任一條款觸發時重開或 spawn 對應 IMP。
