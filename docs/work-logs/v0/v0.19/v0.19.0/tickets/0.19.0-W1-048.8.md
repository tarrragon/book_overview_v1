## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### F13 _parseCSVRows 狀態機抽 helper

**動機**：原 inline 函式認知指數 12（變數 4 + 分支 6 + 巢狀 2），單一函式處理 6 個字元分支（quoted-`""`、quoted-`"`、quoted-other、unquoted-`"`、unquoted-`,`、unquoted-`\r\n`、unquoted-`\r|\n`、default char）。

**設計決策**：採方案 1（cinnamon C1+C2+R1 共識）— state 物件 + per-char helper 分流，主迴圈降為單一 dispatch + EOF flush。

**helper 命名與職責**：

| Helper | 職責 | 認知指數 |
|--------|------|---------|
| `_parseCSVRows` | 主迴圈：依 `state.inQuotes` 分流，呼叫 helper 取得 advance 步數 | 5（var 2 + branch 2 + nest 1） |
| `_consumeQuotedChar(state, content, i)` | 引號狀態內：處理 `""` 跳脫 / 單 `"` 結束 / 其他累加 | 5（var 1 + branch 2 + nest 2） |
| `_consumeUnquotedChar(state, content, i)` | 非引號狀態：處理 `"` / `,` / `\r\n` / `\r|\n` / default | 7（var 1 + branch 5 + nest 1） |
| `_commitRow(state)` | flush currentField + currentRow 進 rows，重置兩者 | 3 |
| `_flushPendingRow(state)` | EOF 收尾：補入未提交的 field/row | 3 |

**state 物件設計**：`{ rows, currentRow, currentField, inQuotes }` 由 reference 傳遞，避免 5 個 helper 各自 destructure + return 多值的複雜度。Per-call 物件分配為單次（主迴圈一次建立），無 hot path 效能影響。

**advance return 設計**：每個 helper 回傳前進步數（1 或 2），主迴圈以 `i +=` 推進。取代原 `continue` + `i += N` 散落模式，控制流線性化。

### F15 _loadFileReaderFactory 死碼分支移除

**動機**：原 `window.FileReaderFactory` fallback 為純死碼——全 repo `grep -rn 'window.FileReaderFactory'` 僅命中此處讀取，零賦值點；Chrome Extension 環境 bundle 後 `__require` 永遠定義（esbuild IIFE module shim），分支永遠成立；Node/Jest 環境 `require` 為全域。

**處置決策**：移除 `window.FileReaderFactory` fallback，失敗路徑改 `throw FILE_ERROR` 取代靜默回傳 undefined。

**為何保留 typeof require 檢查**：

| 環境 | 檢查結果 | 行為 |
|------|---------|------|
| Node / Jest | typeof require === function | 走 CommonJS 載入 |
| Chrome Extension bundle | esbuild 重命名為 __require | bundle 階段 esbuild 已 inline，throw 分支不執行 |
| 未來假設 bundler 切換 | typeof require === undefined 觸發明確 throw FILE_ERROR | 取代原 return undefined → .createReader() 神祕 TypeError |

**錯誤契約改善**：原邏輯 `return window.FileReaderFactory` 在 fallback 失敗時靜默回傳 undefined，下游 `FileReaderFactory.createReader()` 觸發 `Cannot read properties of undefined (reading createReader)`，錯誤訊息與根因脫節。改為 throw FILE_ERROR 後，錯誤訊息直指 root cause。
