##### 衝突場景設計

| 場景 | 設置 | 期望結果 |
|------|------|---------|
| A: tag 聯集 | A 有書 id=B001 tagIds=[t1,t2]；B 有書 id=B001 tagIds=[t2,t3] | 合併後 B001.tagIds 為 union = [t1,t2,t3]；對應 tag_categories 也須完整存在於兩端 |
| B: isManualStatus 保留 | A 有書 id=B002 readingStatus=abandoned, isManualStatus=true, progress=0.5；B 有書 id=B002 readingStatus=reading, isManualStatus=false, progress=0.7 | 匯入 A→B 合併模式後，B002 保留 abandoned + isManualStatus=true（手動狀態優先於自動進度） |
| C: v1→v2 跨版本 | A 匯出 v1 格式（schema 含 isNew/isFinished 而非 readingStatus/isManualStatus）；B 端 import 接收 | format-version-detector 偵測為 v1，v1-to-v2-converter 自動轉換：isNew=true→unread, isFinished=true→finished, 兩者皆 false→reading (預設) |
| D: category 重建 | A 端 tag_categories 缺 cat-x（已刪除）；B 端有書引用 cat-x 內 tagId | 匯入後 B 端 cat-x 自動重建（或退回未分類）並保留書籍-tag 關聯不丟失 |

```
Step 1: 建立「裝置 A」storage（含場景特定資料）

Step 4: 呼叫 ContentParser.parse + TagStorageAdapter.mergeAllData 匯入到 B

```
