## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

> **PM 已蒐集 context（2026-06-02，以實作碼為準）；接手代理人應獨立複驗源檔行為（PC-162）。**

### 問題根因

現有匯入 detector（`src/export/format-version-detector.js`，70 行）只辨識 2 種格式：

| detector 規則 | 判定 | 缺口 |
|--------------|------|------|
| `metadata.formatVersion` 以 `2.` 開頭 | v2（內部 tag-based） | — |
| `metadata` + `books[]` 含 `readingStatus` | v2 | — |
| 純陣列 / `{books:[]}` 無版本標記 | v1（flat，跑 v1-to-v2 converter） | — |

**資料損壞破口（W4-031 假設#1，本 ticket「止血」目標）**：APP（book_overview_app）匯出格式為 `{backup_info|export_info, books:[...]}`，**無 `metadata.formatVersion`**。匯入時命中 detector Rule 4（有 books 無版本標記）→ 誤判為 **v1** → 跑 `v1-to-v2-converter`（`convertV1ToV2Book` 只取 id/title/author(單)/publisher/cover/category）→ **丟失 APP 的 tags、readingStatus、多值欄位**，違反 canonical C1（雙向無損）。

detector 同樣不認得新 canonical `format: "book-interchange-v1"` 標記，無法正確分流 v3 everything-as-tags 格式。

### 影響範圍

| 檔案 | 現況 | 本 ticket 變更方向 |
|------|------|------------------|
| `src/export/format-version-detector.js` | 2 來源辨識（v1/v2） | 擴充為 canonical 四來源優先序（spec §8）：canonical → 內部 v2 → APP legacy(backup_info) → flat v1 |
| `src/export/v1-to-v2-converter.js` | flat v1 → 內部 v2 | 新增 canonical adapter（read/write v3）；APP legacy 無損匯入分支（不重推 readingStatus、coverImageUrl→cover、多值入 tag） |
| `src/export/book-data-exporter.js`（1364 行） | 匯出內部格式 | write canonical v3.0.0（format 標記 + metadata + everything-as-tags tags 物件 + tagTree 階層） |
| `src/overview/import/content-parser.js`（555 行，DI 注入 detector+converter） | 預設用 src/export/* | 接線新 detector/adapter（透過既有 constructor DI，向後相容） |

### 相關 Error Pattern

### 實作策略（已在 frontmatter how.strategy，重點摘要）

1. 匯出 write canonical v3（format 標記 + metadata + tags 物件 + tagTree）
2. detector 四來源優先序（spec §8）：canonical(`format===`) → 內部 v2(`formatVersion 2.`) → APP legacy(`backup_info`/`export_info` wrapper) → flat v1
3. V1 固定欄位↔tag 雙向映射（authors→tags.author、source→tags.platform、readingStatus enum→單選 tag、tagIds 圖→tags.custom）
4. APP legacy 無損匯入（readingStatus 不重推、coverImageUrl→cover、多值入對應 tag、importance/ccl carry+display）
5. cover 多尺寸相容單字串、progress 物件相容單數字、crossPlatformId/dataFingerprint carry
