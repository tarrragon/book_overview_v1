### 1. 功能需求分析

**核心問題**：現有匯入 detector 只認 v1/v2 兩來源，無法辨識新 canonical `format: "book-interchange-v1"`，且 APP legacy（`backup_info`/`export_info` wrapper，無 `formatVersion`）會誤命中 flat-v1 規則 → 跑 lossy converter → 丟失 tags/readingStatus/多值（違反 spec C1 雙向無損）。本 foundation 提供兩個純函式核心供 .2.1（匯出 write）/ .2.2（匯入接線 + APP legacy 止血）消費。

**使用者角色與場景**：
- Extension 匯入流程（ContentParser）— 需正確分流四來源
- Extension 匯出流程（book-data-exporter）— 需 V1 固定欄位 → canonical tags（write 方向）
- 跨專案同步（APP ↔ Extension）— 需雙向可逆無損

**範圍邊界**：僅 foundation 純函式核心。匯出 write 接線（.2.1）/ 匯入接線 + APP legacy 止血 + passthrough（.2.2）不在本 ticket。

### 2. API 介面定義

#### 2.1 detector：四來源辨識（擴充 format-version-detector.js）

```
detectInterchangeSource(data) -> SourceTag | null
```

| 項目 | 規格 |
|------|------|
| 輸入 | `data`: 解析後 JSON（any；非 object 一律 null） |
| 輸出 | enum SourceTag: `'canonical'` \| `'v2'` \| `'app-legacy'` \| `'v1'` \| `null` |
| 優先序（spec §8，高→低） | 1. `data.format === 'book-interchange-v1'` → `'canonical'`；2. `data.metadata.formatVersion` 以 `'2.'` 開頭（且無 `format`）→ `'v2'`；3. 含 `backup_info` 或 `export_info` wrapper + `Array.isArray(books)` → `'app-legacy'`；4. 純陣列 或 `{books:[]}` 無版本標記 → `'v1'`；5. 皆不符 → `null` |
| 錯誤契約 | 純判定函式，不 throw；無法辨識回 `null`（與既有 `detectFormatVersion` 一致） |
| 向後相容 | 既有 `detectFormatVersion(data)` 回傳 `'v1'/'v2'/null` 簽名與行為**不變**（ContentParser 既有 DI 仍可用）。新增 `detectInterchangeSource` 為獨立 export，供四來源消費端注入 |

> 設計理由：不破壞既有 `detectFormatVersion` 簽名（避免 ContentParser 既有 DI 回歸），以新函式承載四來源語意。canonical 命中後由消費端讀 `formatVersion` 分流 1.x/2.x/3.x（§8 優先序 1 內）。

#### 2.2 V1↔canonical 雙向映射（新增模組，建議檔名 `src/export/book-interchange-v1-adapter.js`）

write 方向（V1 內部 → canonical，供 .2.1 匯出消費）：

```
mapV1BookToCanonical(v1Book) -> CanonicalBook
```

| 項目 | 規格 |
|------|------|
| 輸入 | `v1Book`: V1 內部 v2 book（id/title/authors[]/publisher/source/identifiers/readingStatus/tagIds 等） |
| 輸出 | CanonicalBook（§4：固定欄位 + `tags` 物件 12 類別 + `_passthrough`） |
| 固定欄位映射 | `id→id`（保留）、`title→title`、`cover(str)→{original}`、`progress(num)→{percentage}`、`crossPlatformId/dataFingerprint→carry`、`createdAt/updatedAt→carry` |
| tag 映射（固定欄位→單元素 tag） | `authors[]→tags.author[]`（各成 tag）、`source→tags.platform[]`、`publisher→tags.publisher[]`、`identifiers.isbn→tags.isbn[]`、`readingStatus(enum)→tags.readingStatus[單選]`（§7 正規化）、`tagIds 圖→tags.custom[]` |
| 錯誤契約 | `v1Book` 非 object 或缺 `id`/`title` → throw `BookValidationError`（對齊既有 `convertV1ToV2Book`） |

read 方向（canonical → V1 內部，供 .2.2 匯入消費）：

```
mapCanonicalToV1Book(canonicalBook) -> V1Book
```

| 項目 | 規格 |
|------|------|
| 輸入 | CanonicalBook |
| 輸出 | V1 內部 v2 book |
| tag→固定欄位還原 | `tags.author[]→authors[]`、`tags.platform[0]→source`、`tags.publisher[0]→publisher`、`tags.isbn→identifiers.isbn`、`tags.readingStatus[0]→readingStatus`（§7 逆正規化 `not_started→unread`）、`tags.custom[]→tagIds 圖` |
| 固定欄位還原 | `cover.original→cover(str)`、`progress.percentage→progress(num)` |
| 不還原（V1 carry+display） | `tags.importance/ccl/description/series/language/alias`、`activeLoan` → 進 `_passthrough` 或 carry（不重推、不丟失，C1） |
| 錯誤契約 | canonical 非 object 或缺 `id`/`title` → throw `BookValidationError` |

readingStatus 正規化子函式（§7 對照表，雙向）：

```
normalizeReadingStatusToCanonical(v1Status) -> canonicalName | _passthrough
normalizeReadingStatusFromCanonical(canonicalName) -> v1Status
```

| 項目 | 規格 |
|------|------|
| §7 對照 | `unread↔not_started`、`queued↔queued`、`reading↔reading`、`finished↔finished`、`abandoned↔abandoned`、`reference↔reference` |
| 缺對應態 | 記 `_passthrough.readingStatusRaw` 保留原值（C1，禁丟失） |
| 錯誤契約 | 不 throw；未知值回 `null` + 由 caller 記 passthrough |

### 3. 行為單元清單（含邊界條件）

**detector（detectInterchangeSource）**
- U1 canonical：`{format:'book-interchange-v1',...}` → `'canonical'`
- U2 v2：`{metadata:{formatVersion:'2.0.0'},books:[]}` 無 format → `'v2'`
- U3 app-legacy：`{backup_info:{...},books:[]}` → `'app-legacy'`；`{export_info:{...},books:[]}` → `'app-legacy'`
- U4 v1（純陣列）：`[{...}]` → `'v1'`
- U5 v1（wrapper 無版本）：`{books:[...]}` 無 format/formatVersion/backup_info → `'v1'`
- 邊界 B1 null/undefined/非物件（字串/數字）→ `null`
- 邊界 B2 空物件 `{}` → `null`
- 邊界 B3 優先序衝突：同時有 `format` 與 `formatVersion:'2.'` → `'canonical'`（優先序 1 勝）
- 邊界 B4 同時有 `backup_info` 與 `format:'book-interchange-v1'` → `'canonical'`（優先序 1 勝）
- 回歸 B5 既有 `detectFormatVersion` 對 U2/U4/U5 仍回 `'v2'/'v1'/'v1'`（簽名不變）

**write 映射（mapV1BookToCanonical）**
- U6 多 author：`authors:['村上春樹','Haruki Murakami']` → `tags.author` 兩元素
- U7 source→platform、publisher→tags.publisher、isbn→tags.isbn
- U8 readingStatus enum→單選 tag（`unread`→`not_started`）
- U9 tagIds 圖→tags.custom
- U10 cover 單字串→`{original}`、progress 單數字→`{percentage}`
- 邊界 B6 `v1Book` 非 object → throw BookValidationError
- 邊界 B7 缺 `id` 或 `title` → throw BookValidationError（含 missing 欄位明細）
- 邊界 B8 空 authors `[]` → `tags.author = []`（不報錯）
- 邊界 B9 缺 source/publisher/isbn（undefined）→ 對應 tag 類別為 `[]`
- 邊界 B10 readingStatus 缺值 → tags.readingStatus `[]`（不臆造）

**read 映射（mapCanonicalToV1Book）**
- U11 tags.author→authors[]、platform[0]→source、publisher[0]→publisher
- U12 tags.readingStatus `not_started`→`unread`（逆正規化）
- U13 tags.custom→tagIds 圖
- U14 cover.original→cover(str)、progress.percentage→progress(num)
- U15 importance/ccl/description/activeLoan → 進 _passthrough/carry（不丟失）
- 邊界 B11 canonical 非 object / 缺 id/title → throw BookValidationError
- 邊界 B12 tags 類別缺漏（無 author key）→ authors `[]`
- 邊界 B13 readingStatus 未知 canonical name → 記 _passthrough.readingStatusRaw

**雙向可逆（map round-trip）**
- U16 V1 → canonical → V1：固定欄位 + tag 映射欄位完整還原（authors/source/publisher/isbn/readingStatus/tagIds）
- 邊界 B14 §7 正規化可逆：`unread`→`not_started`→`unread`
- 邊界 B15 缺對應 readingStatus 態經 _passthrough 保留後可還原原值
