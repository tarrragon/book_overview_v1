### 1. 功能需求分析

**核心問題**：現有匯入 detector 只認 v1/v2 兩來源，無法辨識新 canonical `format: "book-interchange-v1"`，且 APP legacy（`backup_info`/`export_info` wrapper，無 `formatVersion`）會誤命中 flat-v1 規則 → 跑 lossy converter → 丟失 tags/readingStatus/多值（違反 spec C1 雙向無損）。本 foundation 提供兩個純函式核心供 .2.1（匯出 write）/ .2.2（匯入接線 + APP legacy 止血）消費。

**使用者角色與場景**：
- Extension 匯入流程（ContentParser）— 需正確分流四來源
- Extension 匯出流程（book-data-exporter）— 需 V1 固定欄位 → canonical tags（write 方向）
- 跨專案同步（APP ↔ Extension）— 需雙向可逆無損

**範圍邊界（顯式聲明，C 補強）**：本 foundation 僅含 **book 級雙向映射 + detector**（視下方 §2.3 決定含 `buildTagTree`）。以下明文不在 foundation：

| 項目 | 歸屬 | 理由 |
|------|------|------|
| app-legacy → canonical 映射 | **.2.2** | foundation 僅由 detector 辨識回傳 `'app-legacy'`；實際 legacy adapter 轉換（不重推 readingStatus、coverImageUrl→cover、多值入 tag）在 .2.2 接線時消費 |
| dedup（spec §8：id 主鍵 + crossPlatformId/dataFingerprint 軟連結）| **.2.2** | dedup 是匯入端跨書集合操作，非單書映射；foundation 只保證單書 id 保留禁重生 |
| root 組裝（`format`/`metadata`/`totalBooks` 驗證）| **.2.1** | root-level 結構組裝與交叉驗證屬匯出 write，非 book 級映射 |
| 匯出 write 接線（book-data-exporter）| .2.1 | 消費 foundation `mapV1BookToCanonical` |
| 匯入接線（ContentParser）+ passthrough 平鋪 | .2.2 | 消費 foundation detector + `mapCanonicalToV1Book` |

> 「V1」一詞雙義澄清（D）：本文件「V1 內部 model」指 **Extension 內部 v2 book model**（`authors[]` 陣列、`readingStatus` enum、`tagIds[]`），即 `mapV1BookToCanonical` 的輸入型別。與 detector 回傳值 `'v1'`（legacy flat 格式，`author` 單字串、無 tagIds）**非同一概念**——後者由 .2.2 的 legacy converter 先升為內部 v2 後才進映射。下文 §2.2 函式名沿用 `mapV1Book*` 但語意鎖定為「Extension 內部 v2 model」。

### 2. API 介面定義

#### 2.1 detector：四來源辨識（擴充 format-version-detector.js）

```
detectInterchangeSource(data) -> SourceTag | null
```

| 項目 | 規格 |
|------|------|
| 輸入 | `data`: 解析後 JSON（any；非 object 一律 null） |
| 輸出 | enum SourceTag: `'canonical'` \| `'v2'` \| `'app-legacy'` \| `'v1'` \| `null` |
| 優先序（spec §8，高→低） | 1. `data.format === 'book-interchange-v1'` → `'canonical'`；2. `data.metadata.formatVersion` 以 `'2.'` 開頭（且無 `format`）→ `'v2'`；3. 含 `backup_info` 或 `export_info` wrapper + `Array.isArray(books)` → `'app-legacy'`；4. 純陣列 或 `{books:[]}` 無版本標記 → `'v1'`；5. 皆不符 → `null` |
| 錯誤契約 | 純判定函式，不 throw；無法辨識回 `null`（與既有 `detectFormatVersion` 一致） |
| 向後相容 | 既有 `detectFormatVersion(data)` 回傳 `'v1'/'v2'/null` 簽名與行為**不變**（ContentParser 既有 DI 仍可用）。新增 `detectInterchangeSource` 為獨立 export，供四來源消費端注入 |

> 設計理由：不破壞既有 `detectFormatVersion` 簽名（避免 ContentParser 既有 DI 回歸），以新函式承載四來源語意。canonical 命中後由消費端讀 `formatVersion` 分流 1.x/2.x/3.x（§8 優先序 1 內）。

#### 2.2 V1↔canonical 雙向映射（新增模組，建議檔名 `src/export/book-interchange-v1-adapter.js`）

write 方向（V1 內部 → canonical，供 .2.1 匯出消費）：

```
mapV1BookToCanonical(v1Book) -> CanonicalBook
```

| 項目 | 規格 |
|------|------|
| 輸入 | `v1Book`: **Extension 內部 v2 book model**（id/title/authors[]/publisher/source/identifiers/readingStatus/tagIds 等；非 detector `'v1'` legacy flat，見 §1 雙義澄清） |
| 輸出 | CanonicalBook（§4：固定欄位 + `tags` 物件 12 類別 + `_passthrough`） |
| 固定欄位映射 | `id→id`（保留）、`title→title`、`cover(str)→{original}`、`progress(num)→{percentage}`、`createdAt/updatedAt→carry` |
| crossPlatformId/dataFingerprint（G，對齊 spec §11）| V1 內部 model **無此來源** → write 時不輸出（`undefined`，不臆造空值）。對應 spec §11「V1 端入 `_passthrough`」：僅當 read 方向收到 canonical 帶值時保留於 `_passthrough`，非 V1 固定欄位 carry |
| tag 映射（固定欄位→單元素 tag） | `authors[]→tags.author[]`（各成 tag）、`source→tags.platform[]`、`publisher→tags.publisher[]`、`identifiers.isbn→tags.isbn[]`、`readingStatus(enum)→tags.readingStatus[單選]`（§7 正規化）、`tagIds 圖→tags.custom[]` |
| 錯誤契約 | `v1Book` 非 object 或缺 `id`/`title` → throw `BookValidationError`（對齊既有 `convertV1ToV2Book`） |

read 方向（canonical → V1 內部，供 .2.2 匯入消費）：

```
mapCanonicalToV1Book(canonicalBook) -> V1Book
```

| 項目 | 規格 |
|------|------|
| 輸入 | CanonicalBook |
| 輸出 | Extension 內部 v2 book model |
| tag→固定欄位還原 | `tags.author[]→authors[]`（多值 author 對齊 V1 `authors[]`，全保留）、`tags.platform[0]→source`、`tags.publisher[0]→publisher`、`tags.isbn[0]→identifiers.isbn`、`tags.readingStatus[0]→readingStatus`（§7 逆正規化 `not_started→unread`）、`tags.custom[]→tagIds 圖` |
| 固定欄位還原 | `cover.original→cover(str)`、`progress.percentage→progress(num)` |
| 不還原（V1 carry+display） | `tags.importance/ccl/description/series/language/alias`、`activeLoan`、`crossPlatformId/dataFingerprint`、`extensions.book_overview_app` → 進 `_passthrough`（不重推、不丟失，C1） |
| 錯誤契約 | canonical 非 object 或缺 `id`/`title` → throw `BookValidationError` |

**多值收斂通用規則（A，C1 critical — 保 APP→Ext→APP round-trip）**：

canonical 多值 tag 類別還原至 V1 單值固定欄位時，**首元素入固定欄位，其餘元素 + 原始完整 tag 陣列存入 `_passthrough`**，使 write 回 canonical 時可完整還原：

| canonical 多值來源 | V1 單值欄位 | 收斂規則 |
|-------------------|-------------|---------|
| `tags.platform[]`（多平台）| `source`（單值）| `[0].name→source`；`_passthrough.tags.platform = 完整陣列` |
| `tags.publisher[]`（多版本）| `publisher`（單值）| `[0].name→publisher`；`_passthrough.tags.publisher = 完整陣列` |
| `tags.isbn[]`（多版本）| `identifiers.isbn`（單值）| `[0].name→isbn`；`_passthrough.tags.isbn = 完整陣列` |
| `tags.author[]`（多作者）| `authors[]`（V1 為陣列，**不收斂**）| 全保留至 `authors[]`，無需 passthrough |
| `cover{thumbnail,medium,original}` | `cover`（單字串）| `original→cover`；`_passthrough.cover = {thumbnail, medium}`（其餘尺寸）|
| `progress{percentage,currentPage,totalPages,lastReadAt}` | `progress`（單數字）| `percentage→progress`；`currentPage/totalPages/lastReadAt → V1 progressInfo`（既有欄位）或 `_passthrough.progress` |

> 設計原則：read 端**禁止丟棄** canonical 多值/多欄位資訊。V1 單值欄位取首元素為「主顯示值」，其餘進 `_passthrough`，由 write 端優先讀 `_passthrough` 重建完整多值 tag（達成 APP 出發的 round-trip 無損，C1）。`tags.author` 因 V1 內部即 `authors[]` 陣列故無收斂損失。

readingStatus 正規化子函式（§7 對照表，雙向）：

```
normalizeReadingStatusToCanonical(v1Status) -> canonicalName | null
normalizeReadingStatusFromCanonical(canonicalName) -> v1Status | null
```

| 項目 | 規格 |
|------|------|
| §7 對照 | `unread↔not_started`、`queued↔queued`、`reading↔reading`、`finished↔finished`、`abandoned↔abandoned`、`reference↔reference` |
| 回傳型別（E，統一）| 純查表函式，回 `canonicalName \| null`（或 `v1Status \| null`）。**不承擔 passthrough 副作用**——未知值僅回 `null` |
| 錯誤契約 | 不 throw；未知值回 `null` |
| passthrough 責任歸屬（E）| 記 `_passthrough.readingStatusRaw` 的責任在 **caller（`mapV1BookToCanonical` / `mapCanonicalToV1Book`）**：caller 收到子函式 `null` 時，將原值寫入 `_passthrough.readingStatusRaw`（C1，禁丟失）。子函式保持純查表無副作用 |

#### 2.3 tagTree 聚合（B，歸屬決策：屬本 foundation）

spec §6 要求 root `tagTree.{ccl,custom}` 為扁平樹節點陣列（`{id,name,parentId,locked}`），匯入端據此重建對方 tag 樹。聚合函式歸屬決策如下：

```
buildTagTree(books) -> { ccl: TreeNode[], custom: TreeNode[] }
```

| 項目 | 規格 |
|------|------|
| 歸屬決策（B）| **屬本 foundation**（非外移 .2.1）|
| 決策理由 | (1) tagTree 聚合是純函式（無 IO、無 state），與 book 級映射同性質，內聚於 adapter 模組；(2) read 與 write 兩端皆需（write 由 book.tags.{ccl,custom} 聚合產生 tagTree；read 由 tagTree 重建 path），若外移 .2.1 則 .2.2 read 端需重複實作，違反 DRY（ARCH-020）；(3) 置於 foundation 使 .2.1/.2.2 共用單一實作 |
| 輸入 | `books`: CanonicalBook[]（各 book 的 `tags.ccl[]` / `tags.custom[]` 含 `{id,name,path}`）|
| 輸出 | `{ ccl: TreeNode[], custom: TreeNode[] }`；TreeNode = `{id, name, parentId, locked}`（spec §6）|
| 聚合規則 | 收集所有 book 的 ccl/custom tag node，依 `path` 拆解階層推導 `parentId`，去重（同 id 合併）；ccl 節點 `locked:true`、custom `locked:false`（spec §6）|
| 錯誤契約 | 不 throw；`books` 非陣列回 `{ccl:[], custom:[]}` |

> 邊界聲明：`buildTagTree` 為 book 級 tag node 的**樹結構聚合**，仍屬 book/tag 級純函式範疇；root 組裝（將 tagTree 嵌入 `{format,metadata,books,tagTree}`）仍屬 .2.1（見 §1 範圍邊界表）。

### 3. 行為單元清單（含邊界條件）

**detector（detectInterchangeSource）**
- U1 canonical：`{format:'book-interchange-v1',...}` → `'canonical'`
- U2 v2：`{metadata:{formatVersion:'2.0.0'},books:[]}` 無 format → `'v2'`
- U3 app-legacy：`{backup_info:{...},books:[]}` → `'app-legacy'`；`{export_info:{...},books:[]}` → `'app-legacy'`
- U4 v1（純陣列）：`[{...}]` → `'v1'`
- U5 v1（wrapper 無版本）：`{books:[...]}` 無 format/formatVersion/backup_info → `'v1'`
- 邊界 B1 null/undefined/非物件（字串/數字）→ `null`
- 邊界 B2 空物件 `{}` → `null`
- 邊界 B3 優先序衝突：同時有 `format` 與 `formatVersion:'2.'` → `'canonical'`（優先序 1 勝）
- 邊界 B4 同時有 `backup_info` 與 `format:'book-interchange-v1'` → `'canonical'`（優先序 1 勝）
- 回歸 B5 既有 `detectFormatVersion` 對 U2/U4/U5 仍回 `'v2'/'v1'/'v1'`（簽名不變）

**write 映射（mapV1BookToCanonical）**
- U6 多 author：`authors:['村上春樹','Haruki Murakami']` → `tags.author` 兩元素
- U7 source→platform、publisher→tags.publisher、isbn→tags.isbn
- U8 readingStatus enum→單選 tag（`unread`→`not_started`）
- U9 tagIds 圖→tags.custom
- U10 cover 單字串→`{original}`、progress 單數字→`{percentage}`
- 邊界 B6 `v1Book` 非 object → throw BookValidationError
- 邊界 B7 缺 `id` 或 `title` → throw BookValidationError（含 missing 欄位明細）
- 邊界 B8 空 authors `[]` → `tags.author = []`（不報錯）
- 邊界 B9 缺 source/publisher/isbn（undefined）→ 對應 tag 類別為 `[]`
- 邊界 B10 readingStatus 缺值 → tags.readingStatus `[]`（不臆造）

**read 映射（mapCanonicalToV1Book）**
- U11 tags.author→authors[]、platform[0]→source、publisher[0]→publisher
- U12 tags.readingStatus `not_started`→`unread`（逆正規化）
- U13 tags.custom→tagIds 圖
- U14 cover.original→cover(str)、progress.percentage→progress(num)
- U15 importance/ccl/description/activeLoan/crossPlatformId/dataFingerprint → 進 _passthrough（不丟失）
- U18 多值收斂（A）：`tags.platform[2]`/`tags.publisher[2]`/`tags.isbn[2]` → 首元素入 source/publisher/isbn，**其餘 + 完整陣列入 `_passthrough.tags.*`**
- U19 cover 多尺寸（A）：`cover{thumbnail,medium,original}` → `original→cover(str)`，`{thumbnail,medium}→_passthrough.cover`

- 邊界 B11 canonical 非 object / 缺 id/title → throw BookValidationError
- 邊界 B12 tags 類別缺漏（無 author key）→ authors `[]`
- 邊界 B13（F，重定位）read 收到**非 V1 來源**寫入的未知 canonical readingStatus name（如 APP 未來新增態）→ 子函式回 `null`，caller 記 `_passthrough.readingStatusRaw`。**註**：此非 V1↔canonical 必經路徑（V1 六態 §7 全對應，正向不可達未知）；本單元為「read 端對任意 canonical 輸入的防禦」，真正的 app-legacy 接線在 .2.2
- 邊界 B16（A）read 端不丟棄多值/多欄位：`_passthrough` 含被收斂的 tag 陣列與 cover 尺寸（為 U17 round-trip 前提）

**雙向可逆（map round-trip）**
- U16 **V1 出發** round-trip（V1→canonical→V1）：完整還原下列欄位 — 固定欄位 `id/title/cover/progress/createdAt/updatedAt`；tag 映射 `authors[]/source(platform)/publisher/isbn/readingStatus/tagIds(custom)`；carry 類 `importance/ccl/activeLoan/extensions` 經 _passthrough 還原不丟失
- U17（A，新增）**canonical 出發** round-trip（canonical→V1→canonical，APP→Ext→APP 真實路徑）：含多值 tag（platform/publisher/isbn 各 2+ 元素）、cover 多尺寸、progress 多欄位、importance/ccl 的 canonical book，經 read 收斂 + write 重建後，多值 tag 完整還原（首元素 + _passthrough 其餘），無損（C1）
- 邊界 B14 §7 正規化可逆：`unread`→`not_started`→`unread`
- 邊界 B15（F，重定位）非 V1 來源未知 readingStatus 經 `_passthrough.readingStatusRaw` 保留後 write 可還原原值（與 B13 同屬非 V1 必經路徑的防禦，非正向 round-trip）

**tagTree 聚合（buildTagTree，B）**
- U21 多 book 的 ccl/custom tag node 聚合為 `{ccl[],custom[]}`，依 path 推導 parentId，同 id 去重
- U22 ccl 節點 `locked:true`、custom 節點 `locked:false`（spec §6）
- 邊界 B17 `books` 非陣列 → `{ccl:[],custom:[]}`（不 throw）
- 邊界 B18 book 無 ccl/custom tag → 對應樹為 `[]`

#### 2. detectInterchangeSource 策略（四來源優先序，spec §8）

**資料流**：`data (any)` → 型別守衛 → 依優先序逐規則短路 return → `SourceTag | null`

**控制流虛擬碼**（H2 已採納：Rule 2 移除冗餘 `!format` 死條件，靠 return 順序保證互斥）：

```
FUNCTION detectInterchangeSource(data):
    # 型別守衛：非物件且非陣列一律 null（與既有 detectFormatVersion 一致）
    IF data IS null OR undefined OR (NOT object AND NOT array):
        RETURN null

    # 優先序 1（最高）：canonical 字面標記 → 凌駕 formatVersion / backup_info（B3/B4）
    IF data.format EQUALS 'book-interchange-v1':
        RETURN 'canonical'

    # 優先序 2：內部 v2（formatVersion 以 '2.' 開頭）
    #   到此處 format 必不等於 canonical 字面（Rule 1 已短路），故 !format 為死條件，移除（H2）
    IF typeof data.metadata.formatVersion IS string AND startsWith('2.'):
        RETURN 'v2'

    # 優先序 3：APP legacy wrapper（止血關鍵）→ 不再誤命中 flat v1
    IF (data HAS backup_info OR data HAS export_info) AND isArray(data.books):
        RETURN 'app-legacy'

    # 優先序 4：flat v1（純陣列 或 {books:[]} 無版本標記）
    IF isArray(data):
        RETURN 'v1'
    IF isArray(data.books):
        RETURN 'v1'

    # 優先序 5：皆不符
    RETURN null
```

**優先序短路即互斥**：規則由高至低逐一 return，命中即出；故 B3（format + formatVersion 同在）→ Rule 1 勝、B4（format + backup_info 同在）→ Rule 1 勝，無需額外排他條件。

**detectFormatVersion 委派收斂（H1，ADR-2）**：

```
FUNCTION detectFormatVersion(data):   # 既有簽名不變，回 'v1'|'v2'|null（B5）
    source = detectInterchangeSource(data)
    SWITCH source:
        'v2'                  -> RETURN 'v2'
        'v1'                  -> RETURN 'v1'
        'canonical'           -> RETURN null   # 四來源新語意，既有二值消費端不消費 canonical（由新 detector 分流）
        'app-legacy'          -> RETURN null   # 同上，app-legacy 接線在 .2.2 走新 detector
        null                  -> RETURN null
```

> **收斂映射的 Phase 3b 確認點**：canonical/app-legacy 在既有二值語意下回 `null`（而非 'v1'/'v2'）是刻意設計——既有 ContentParser DI 消費 `detectFormatVersion` 僅辨識 v1/v2，canonical/app-legacy 的接線改走 .2.2 注入的 `detectInterchangeSource`，故二值函式對新來源回 null 不影響既有 B5 回歸（既有測試僅驗 v1/v2/null 三類 fixture，不含 canonical/app-legacy）。Phase 3b 須先跑既有 `format-version-detector.test.js` 確認全綠再動。**技術債標記 TD-1**：若未來既有消費端需區分 canonical，需重新評估收斂語意（屬 .2.2/後續，非本 foundation）。

#### 3. mapV1BookToCanonical 策略（write 方向，V1 內部 v2 → canonical）

**資料流**：`v1Book` → 必填守衛 → 固定欄位物件化 → 各固定欄位包單元素 tag → readingStatus 正規化 → 組裝 CanonicalBook（含初始 `_passthrough`）

```
FUNCTION mapV1BookToCanonical(v1Book):
    # 錯誤契約：對齊既有 convertV1ToV2Book
    IF v1Book IS NOT object:
        THROW BookValidationError(v1Book, [{field:'book', message:'Input must be an object'}])
    missing = []
    IF NOT v1Book.id:    missing.push({field:'id', message:'Required field missing'})
    IF NOT v1Book.title: missing.push({field:'title', message:'Required field missing'})
    IF missing NOT empty:
        THROW BookValidationError(v1Book, missing)

    canonical = {}
    # --- 固定欄位 ---
    canonical.id    = v1Book.id          # 保留禁重生（C4）
    canonical.title = v1Book.title
    canonical.cover    = objectifyCover(v1Book)      # 見子流程 3a
    canonical.progress = objectifyProgress(v1Book)   # 見子流程 3b
    carryIfPresent(canonical, v1Book, ['createdAt','updatedAt'])
    # crossPlatformId/dataFingerprint：V1 內部無來源 → 不輸出（不臆造空值，§G/spec §11）

    # --- 固定欄位 → 單元素 tag（everything-as-tags）---
    canonical.tags = {}
    canonical.tags.author    = mapEach(v1Book.authors, name -> {id:genTagId('a',name), name})   # 多值全保留（B8 空→[]）
    canonical.tags.platform  = wrapSingle(v1Book.source, 'platform')        # B9 undefined→[]
    canonical.tags.publisher = wrapSingle(v1Book.publisher, 'publisher')    # B9
    canonical.tags.isbn      = wrapSingle(v1Book.identifiers?.isbn, 'isbn') # B9
    canonical.tags.custom    = mapTagIdsToCustom(v1Book.tagIds)             # 圖→custom（U9，tagId→name 來源見 TD-2）

    # readingStatus 單選 tag（§7 正規化），passthrough 責任在此 caller（§E）
    rsName = normalizeReadingStatusToCanonical(v1Book.readingStatus)
    IF v1Book.readingStatus IS undefined/null:
        canonical.tags.readingStatus = []           # B10 不臆造
    ELSE IF rsName IS null:                          # 未知值（V1 正向不可達；防禦）
        canonical.tags.readingStatus = []
        canonical._passthrough.readingStatusRaw = v1Book.readingStatus   # C1 禁丟失
    ELSE:
        canonical.tags.readingStatus = [{id:'rs-'+rsName, name:rsName}]

    # --- _passthrough：V1 出發時初始為 caller 已寫入項（readingStatusRaw 等），其餘空 ---
    canonical._passthrough = canonical._passthrough OR {}

    # --- H3 / ADR-5：write 端多值重建（canonical 出發 round-trip U17 的 write 半段）---
    # 若 v1Book 帶有 read 階段存入的 _passthrough（V1 出發路徑無此鍵，退化跳過）：
    rebuildMultiValueFromPassthrough(canonical, v1Book._passthrough)   # 見子流程 3c

    RETURN canonical
```

**子流程 3a — objectifyCover**：`cover(str) → {original: str}`；若 v1Book._passthrough.cover 帶其餘尺寸 → 併入 `{thumbnail,medium,original}`（H3 重建）。

**子流程 3b — objectifyProgress**：`progress(num) → {percentage: num}`；若有 progressInfo（currentPage/totalPages/lastReadAt）或 _passthrough.progress → 併入多欄位。

**子流程 3c — rebuildMultiValueFromPassthrough（H3 / ADR-5 核心）**：

```
FUNCTION rebuildMultiValueFromPassthrough(canonical, pt):
    IF pt IS undefined: RETURN          # V1 出發路徑：無 passthrough，保留單元素 tag（已組裝）
    # 多值 tag：優先用 passthrough 完整陣列覆蓋單元素版本
    FOR category IN ['platform','publisher','isbn']:
        IF pt.tags?.[category] IS array:
            canonical.tags[category] = pt.tags[category]    # 完整還原（U17/U18）
    # cover 多尺寸：pt.cover 含 {thumbnail,medium} → 與 original 合併（U19）
    IF pt.cover: canonical.cover = merge(canonical.cover, pt.cover)
    # importance/ccl/description/series/language/alias/activeLoan/extensions：read 端存於 pt → write 還原
    FOR category IN ['importance','ccl','description','series','language','alias']:
        IF pt.tags?.[category]: canonical.tags[category] = pt.tags[category]
    carryIfPresent(canonical, pt, ['activeLoan','crossPlatformId','dataFingerprint','extensions'])
    # readingStatusRaw：未知態還原（B15/D B15）
    IF pt.readingStatusRaw: canonical.tags.readingStatus = [{id:'rs-'+pt.readingStatusRaw, name:pt.readingStatusRaw}]
```

> **設計原則**：write 端對「同一資訊」有兩個來源——V1 固定欄位（單值）與 `_passthrough`（read 階段保留的完整多值）。**passthrough 優先**：有完整陣列用完整陣列，無則由固定欄位包單元素。此即 H3 統一兩路徑（V1 出發 / canonical 出發）的關鍵。

#### 4. mapCanonicalToV1Book 策略（read 方向，canonical → V1 內部 v2）

**資料流**：`canonicalBook` → 必填守衛 → tag → 固定欄位還原（多值收斂，首元素入欄位）→ 被收斂的其餘 + carry 類入 `_passthrough` → 組裝 V1Book

```
FUNCTION mapCanonicalToV1Book(canonical):
    IF canonical IS NOT object:  THROW BookValidationError(...)
    IF NOT canonical.id OR NOT canonical.title:  THROW BookValidationError(... missing ...)

    v1 = {}
    v1.id    = canonical.id        # 保留禁重生（C4）
    v1.title = canonical.title
    v1._passthrough = {}

    # --- tag → 固定欄位還原 ---
    t = canonical.tags OR {}
    v1.authors = mapEach(t.author, node -> node.name)     # 多值全保留（不收斂，§A author 例外，U17/C-U17）；B12 缺→[]

    # 多值收斂（§A / ADR-5）：首元素入固定欄位，其餘 + 完整陣列入 _passthrough（C1）
    convergeMultiValue(v1, 'source',           t.platform,  'platform')    # U18
    convergeMultiValue(v1, 'publisher',        t.publisher, 'publisher')
    convergeMultiValueNested(v1, 'identifiers','isbn', t.isbn, 'isbn')     # isbn 入 identifiers.isbn

    # readingStatus 逆正規化（§7），passthrough 責任在此 caller（§E）
    IF t.readingStatus IS non-empty array:
        rsName = t.readingStatus[0].name
        v1Status = normalizeReadingStatusFromCanonical(rsName)
        IF v1Status IS null:                                 # 未知態（B13/F 防禦）
            v1._passthrough.readingStatusRaw = rsName        # C1 禁丟失
            # v1.readingStatus 不設（undefined，不臆造）
        ELSE:
            v1.readingStatus = v1Status

    # custom → tagIds 圖（U13）
    v1.tagIds = mapCustomToTagIds(t.custom)

    # --- 固定欄位還原 + 多尺寸收斂 ---
    convergeCover(v1, canonical.cover)        # original→cover(str)，其餘尺寸→_passthrough.cover（U19）
    convergeProgress(v1, canonical.progress)  # percentage→progress(num)，其餘→progressInfo 或 _passthrough.progress（U20）
    carryIfPresent(v1, canonical, ['createdAt','updatedAt'])

    # --- 不還原（V1 carry+display）→ 進 _passthrough（不重推、不丟失，C1，U15）---
    FOR category IN ['importance','ccl','description','series','language','alias']:
        IF t[category] non-empty:  v1._passthrough.tags ||= {}; v1._passthrough.tags[category] = t[category]
    carryIfPresent(v1._passthrough, canonical, ['activeLoan','crossPlatformId','dataFingerprint','extensions'])

    RETURN v1
```

**子流程 4a — convergeMultiValue（§A 多值收斂通用規則）**：

```
FUNCTION convergeMultiValue(target, fieldName, tagArray, passthroughKey):
    IF tagArray IS empty/undefined: RETURN          # B12 安全降級，不設欄位
    target[fieldName] = tagArray[0].name            # 首元素為主顯示值
    IF tagArray.length > 1:
        target._passthrough.tags ||= {}
        target._passthrough.tags[passthroughKey] = tagArray   # 完整陣列存 passthrough（C1，B16）
    # 設計取捨：是否「即使單元素也存 passthrough」由 Phase 3b 確認；
    # 推薦「length>1 才存」以最小化 passthrough 體積，單元素 round-trip 由固定欄位重建（H3 退化路徑覆蓋）
```

> **Phase 3b 確認點（U17 round-trip 正確性關鍵）**：`convergeMultiValue` 僅在 `length > 1` 存完整陣列至 passthrough。U17（canonical 出發，多值 length>=2）→ write 階段 `rebuildMultiValueFromPassthrough` 讀完整陣列還原，正確。單元素 tag（length=1）→ 不存 passthrough → write 由固定欄位包單元素，亦正確。兩路徑統一（H3）。**驗證方法**：Phase 3b 跑 D-U17（toEqual 多值陣列）+ D-U16（V1 出發單值）雙向綠燈即確認。

#### 6. buildTagTree 策略（樹結構聚合，spec §6，ADR-4）

**資料流**：`books[]` → 收集所有 book 的 ccl/custom tag node → 依 path 拆階層推導 parentId → 同 id 去重 → `{ccl, custom}`

```
FUNCTION buildTagTree(books):
    IF NOT isArray(books):  RETURN { ccl:[], custom:[] }    # B17 安全降級

    cclMap    = Map()   # id -> TreeNode，去重用（B18 無 ccl→留空）
    customMap = Map()

    FOR book IN books:
        IF NOT book.tags: CONTINUE
        accumulateTree(cclMap,    book.tags.ccl,    locked=true)    # ccl locked:true（U22）
        accumulateTree(customMap, book.tags.custom, locked=false)  # custom locked:false（U22）

    RETURN { ccl: values(cclMap), custom: values(customMap) }


FUNCTION accumulateTree(treeMap, nodes, locked):
    IF NOT isArray(nodes): RETURN
    FOR node IN nodes:                          # node = {id, name, path}
        # 依 path 拆解階層：'語言文學/東方文學/日本文學' → 三層
        segments = node.path ? split(node.path, '/') : [node.name]
        # 為每層推導 parentId（前一段對應節點 id），葉節點用 node.id；中間層需穩定 id 推導
        FOR i, seg IN segments:
            nodeId   = (i EQUALS segments.length-1) ? node.id : deriveStableId(seg, segments[0..i])
            parentId = (i EQUALS 0) ? null : deriveStableId(segments[i-1], segments[0..i-1])
            IF NOT treeMap.has(nodeId):         # 同 id 去重（U21，ccl-861 多 book 只一節點）
                treeMap.set(nodeId, { id:nodeId, name:seg, parentId, locked })
```

> **Phase 3b 確認點（中間層 id 推導）**：葉節點 id 來自 tag node.id（穩定）；path 中間層（如「語言文學」「東方文學」）無顯式 id，需 `deriveStableId` 由 path 前綴推導穩定 id（建議 hash(pathPrefix) 或 slug 化），確保多 book 同路徑前綴去重一致。**技術債標記 TD-3**：中間層 id 推導策略影響跨匯入穩定性；若 APP 端提供完整 tagTree（spec §3 root tagTree），匯入時應優先採用對方 tagTree 而非由 path 重建（屬 .2.2 接線決策）。本 foundation 僅提供「由 book.tags 聚合產生 tagTree」的 write 能力。
