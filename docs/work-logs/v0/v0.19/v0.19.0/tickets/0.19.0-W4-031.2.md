### 1. 功能需求分析

**核心問題**：現有匯入 detector 只認 v1/v2 兩來源，無法辨識新 canonical `format: "book-interchange-v1"`，且 APP legacy（`backup_info`/`export_info` wrapper，無 `formatVersion`）會誤命中 flat-v1 規則 → 跑 lossy converter → 丟失 tags/readingStatus/多值（違反 spec C1 雙向無損）。本 foundation 提供兩個純函式核心供 .2.1（匯出 write）/ .2.2（匯入接線 + APP legacy 止血）消費。

**使用者角色與場景**：
- Extension 匯入流程（ContentParser）— 需正確分流四來源
- Extension 匯出流程（book-data-exporter）— 需 V1 固定欄位 → canonical tags（write 方向）
- 跨專案同步（APP ↔ Extension）— 需雙向可逆無損

**範圍邊界（顯式聲明，C 補強）**：本 foundation 僅含 **book 級雙向映射 + detector**（視下方 §2.3 決定含 `buildTagTree`）。以下明文不在 foundation：

| 項目 | 歸屬 | 理由 |
|------|------|------|
| app-legacy → canonical 映射 | **.2.2** | foundation 僅由 detector 辨識回傳 `'app-legacy'`；實際 legacy adapter 轉換（不重推 readingStatus、coverImageUrl→cover、多值入 tag）在 .2.2 接線時消費 |
| dedup（spec §8：id 主鍵 + crossPlatformId/dataFingerprint 軟連結）| **.2.2** | dedup 是匯入端跨書集合操作，非單書映射；foundation 只保證單書 id 保留禁重生 |
| root 組裝（`format`/`metadata`/`totalBooks` 驗證）| **.2.1** | root-level 結構組裝與交叉驗證屬匯出 write，非 book 級映射 |
| 匯出 write 接線（book-data-exporter）| .2.1 | 消費 foundation `mapV1BookToCanonical` |
| 匯入接線（ContentParser）+ passthrough 平鋪 | .2.2 | 消費 foundation detector + `mapCanonicalToV1Book` |

> 「V1」一詞雙義澄清（D）：本文件「V1 內部 model」指 **Extension 內部 v2 book model**（`authors[]` 陣列、`readingStatus` enum、`tagIds[]`），即 `mapV1BookToCanonical` 的輸入型別。與 detector 回傳值 `'v1'`（legacy flat 格式，`author` 單字串、無 tagIds）**非同一概念**——後者由 .2.2 的 legacy converter 先升為內部 v2 後才進映射。下文 §2.2 函式名沿用 `mapV1Book*` 但語意鎖定為「Extension 內部 v2 model」。

### 2. API 介面定義

#### 2.1 detector：四來源辨識（擴充 format-version-detector.js）

```
detectInterchangeSource(data) -> SourceTag | null
```

| 項目 | 規格 |
|------|------|
| 輸入 | `data`: 解析後 JSON（any；非 object 一律 null） |
| 輸出 | enum SourceTag: `'canonical'` \| `'v2'` \| `'app-legacy'` \| `'v1'` \| `null` |
| 優先序（spec §8，高→低） | 1. `data.format === 'book-interchange-v1'` → `'canonical'`；2. `data.metadata.formatVersion` 以 `'2.'` 開頭（且無 `format`）→ `'v2'`；3. 含 `backup_info` 或 `export_info` wrapper + `Array.isArray(books)` → `'app-legacy'`；4. 純陣列 或 `{books:[]}` 無版本標記 → `'v1'`；5. 皆不符 → `null` |
| 錯誤契約 | 純判定函式，不 throw；無法辨識回 `null`（與既有 `detectFormatVersion` 一致） |
| 向後相容 | 既有 `detectFormatVersion(data)` 回傳 `'v1'/'v2'/null` 簽名與行為**不變**（ContentParser 既有 DI 仍可用）。新增 `detectInterchangeSource` 為獨立 export，供四來源消費端注入 |

> 設計理由：不破壞既有 `detectFormatVersion` 簽名（避免 ContentParser 既有 DI 回歸），以新函式承載四來源語意。canonical 命中後由消費端讀 `formatVersion` 分流 1.x/2.x/3.x（§8 優先序 1 內）。

#### 2.2 V1↔canonical 雙向映射（新增模組，建議檔名 `src/export/book-interchange-v1-adapter.js`）

write 方向（V1 內部 → canonical，供 .2.1 匯出消費）：

```
mapV1BookToCanonical(v1Book) -> CanonicalBook
```

| 項目 | 規格 |
|------|------|
| 輸入 | `v1Book`: **Extension 內部 v2 book model**（id/title/authors[]/publisher/source/identifiers/readingStatus/tagIds 等；非 detector `'v1'` legacy flat，見 §1 雙義澄清） |
| 輸出 | CanonicalBook（§4：固定欄位 + `tags` 物件 12 類別 + `_passthrough`） |
| 固定欄位映射 | `id→id`（保留）、`title→title`、`cover(str)→{original}`、`progress(num)→{percentage}`、`createdAt/updatedAt→carry` |
| crossPlatformId/dataFingerprint（G，對齊 spec §11）| V1 內部 model **無此來源** → write 時不輸出（`undefined`，不臆造空值）。對應 spec §11「V1 端入 `_passthrough`」：僅當 read 方向收到 canonical 帶值時保留於 `_passthrough`，非 V1 固定欄位 carry |
| tag 映射（固定欄位→單元素 tag） | `authors[]→tags.author[]`（各成 tag）、`source→tags.platform[]`、`publisher→tags.publisher[]`、`identifiers.isbn→tags.isbn[]`、`readingStatus(enum)→tags.readingStatus[單選]`（§7 正規化）、`tagIds 圖→tags.custom[]` |
| 錯誤契約 | `v1Book` 非 object 或缺 `id`/`title` → throw `BookValidationError`（對齊既有 `convertV1ToV2Book`） |

read 方向（canonical → V1 內部，供 .2.2 匯入消費）：

```
mapCanonicalToV1Book(canonicalBook) -> V1Book
```

| 項目 | 規格 |
|------|------|
| 輸入 | CanonicalBook |
| 輸出 | Extension 內部 v2 book model |
| tag→固定欄位還原 | `tags.author[]→authors[]`（多值 author 對齊 V1 `authors[]`，全保留）、`tags.platform[0]→source`、`tags.publisher[0]→publisher`、`tags.isbn[0]→identifiers.isbn`、`tags.readingStatus[0]→readingStatus`（§7 逆正規化 `not_started→unread`）、`tags.custom[]→tagIds 圖` |
| 固定欄位還原 | `cover.original→cover(str)`、`progress.percentage→progress(num)` |
| 不還原（V1 carry+display） | `tags.importance/ccl/description/series/language/alias`、`activeLoan`、`crossPlatformId/dataFingerprint`、`extensions.book_overview_app` → 進 `_passthrough`（不重推、不丟失，C1） |
| 錯誤契約 | canonical 非 object 或缺 `id`/`title` → throw `BookValidationError` |

**多值收斂通用規則（A，C1 critical — 保 APP→Ext→APP round-trip）**：

canonical 多值 tag 類別還原至 V1 單值固定欄位時，**首元素入固定欄位，其餘元素 + 原始完整 tag 陣列存入 `_passthrough`**，使 write 回 canonical 時可完整還原：

| canonical 多值來源 | V1 單值欄位 | 收斂規則 |
|-------------------|-------------|---------|
| `tags.platform[]`（多平台）| `source`（單值）| `[0].name→source`；`_passthrough.tags.platform = 完整陣列` |
| `tags.publisher[]`（多版本）| `publisher`（單值）| `[0].name→publisher`；`_passthrough.tags.publisher = 完整陣列` |
| `tags.isbn[]`（多版本）| `identifiers.isbn`（單值）| `[0].name→isbn`；`_passthrough.tags.isbn = 完整陣列` |
| `tags.author[]`（多作者）| `authors[]`（V1 為陣列，**不收斂**）| 全保留至 `authors[]`，無需 passthrough |
| `cover{thumbnail,medium,original}` | `cover`（單字串）| `original→cover`；`_passthrough.cover = {thumbnail, medium}`（其餘尺寸）|
| `progress{percentage,currentPage,totalPages,lastReadAt}` | `progress`（單數字）| `percentage→progress`；`currentPage/totalPages/lastReadAt → V1 progressInfo`（既有欄位）或 `_passthrough.progress` |

> 設計原則：read 端**禁止丟棄** canonical 多值/多欄位資訊。V1 單值欄位取首元素為「主顯示值」，其餘進 `_passthrough`，由 write 端優先讀 `_passthrough` 重建完整多值 tag（達成 APP 出發的 round-trip 無損，C1）。`tags.author` 因 V1 內部即 `authors[]` 陣列故無收斂損失。

readingStatus 正規化子函式（§7 對照表，雙向）：

```
normalizeReadingStatusToCanonical(v1Status) -> canonicalName | null
normalizeReadingStatusFromCanonical(canonicalName) -> v1Status | null
```

| 項目 | 規格 |
|------|------|
| §7 對照 | `unread↔not_started`、`queued↔queued`、`reading↔reading`、`finished↔finished`、`abandoned↔abandoned`、`reference↔reference` |
| 回傳型別（E，統一）| 純查表函式，回 `canonicalName \| null`（或 `v1Status \| null`）。**不承擔 passthrough 副作用**——未知值僅回 `null` |
| 錯誤契約 | 不 throw；未知值回 `null` |
| passthrough 責任歸屬（E）| 記 `_passthrough.readingStatusRaw` 的責任在 **caller（`mapV1BookToCanonical` / `mapCanonicalToV1Book`）**：caller 收到子函式 `null` 時，將原值寫入 `_passthrough.readingStatusRaw`（C1，禁丟失）。子函式保持純查表無副作用 |

#### 2.3 tagTree 聚合（B，歸屬決策：屬本 foundation）

spec §6 要求 root `tagTree.{ccl,custom}` 為扁平樹節點陣列（`{id,name,parentId,locked}`），匯入端據此重建對方 tag 樹。聚合函式歸屬決策如下：

```
buildTagTree(books) -> { ccl: TreeNode[], custom: TreeNode[] }
```

| 項目 | 規格 |
|------|------|
| 歸屬決策（B）| **屬本 foundation**（非外移 .2.1）|
| 決策理由 | (1) tagTree 聚合是純函式（無 IO、無 state），與 book 級映射同性質，內聚於 adapter 模組；(2) read 與 write 兩端皆需（write 由 book.tags.{ccl,custom} 聚合產生 tagTree；read 由 tagTree 重建 path），若外移 .2.1 則 .2.2 read 端需重複實作，違反 DRY（ARCH-020）；(3) 置於 foundation 使 .2.1/.2.2 共用單一實作 |
| 輸入 | `books`: CanonicalBook[]（各 book 的 `tags.ccl[]` / `tags.custom[]` 含 `{id,name,path}`）|
| 輸出 | `{ ccl: TreeNode[], custom: TreeNode[] }`；TreeNode = `{id, name, parentId, locked}`（spec §6）|
| 聚合規則 | 收集所有 book 的 ccl/custom tag node，依 `path` 拆解階層推導 `parentId`，去重（同 id 合併）；ccl 節點 `locked:true`、custom `locked:false`（spec §6）|
| 錯誤契約 | 不 throw；`books` 非陣列回 `{ccl:[], custom:[]}` |

> 邊界聲明：`buildTagTree` 為 book 級 tag node 的**樹結構聚合**，仍屬 book/tag 級純函式範疇；root 組裝（將 tagTree 嵌入 `{format,metadata,books,tagTree}`）仍屬 .2.1（見 §1 範圍邊界表）。

### 3. 行為單元清單（含邊界條件）

**detector（detectInterchangeSource）**
- U1 canonical：`{format:'book-interchange-v1',...}` → `'canonical'`

- 邊界 B2 空物件 `{}` → `null`
- 邊界 B3 優先序衝突：同時有 `format` 與 `formatVersion:'2.'` → `'canonical'`（優先序 1 勝）
- 邊界 B4 同時有 `backup_info` 與 `format:'book-interchange-v1'` → `'canonical'`（優先序 1 勝）

**write 映射（mapV1BookToCanonical）**

**雙向可逆（map round-trip）**
- U16 **V1 出發** round-trip（V1→canonical→V1）：完整還原下列欄位 — 固定欄位 `id/title/cover/progress/createdAt/updatedAt`；tag 映射 `authors[]/source(platform)/publisher/isbn/readingStatus/tagIds(custom)`；carry 類 `importance/ccl/activeLoan/extensions` 經 _passthrough 還原不丟失

- 邊界 B14 §7 正規化可逆：`unread`→`not_started`→`unread`
- 邊界 B15（F，重定位）非 V1 來源未知 readingStatus 經 `_passthrough.readingStatusRaw` 保留後 write 可還原原值（與 B13 同屬非 V1 必經路徑的防禦，非正向 round-trip）

**tagTree 聚合（buildTagTree，B）**
- U21 多 book 的 ccl/custom tag node 聚合為 `{ccl[],custom[]}`，依 path 推導 parentId，同 id 去重
- U22 ccl 節點 `locked:true`、custom 節點 `locked:false`（spec §6）

- 邊界 B18 book 無 ccl/custom tag → 對應樹為 `[]`
