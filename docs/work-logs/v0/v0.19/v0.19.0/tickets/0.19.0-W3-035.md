### 缺陷定位（PM 前台分析）

**主邏輯位置**：`.claude/skills/ticket/ticket_system/commands/track_acceptance.py:614`

```python
# 追加內容（當前實作）
updated_section = section_text + new_entry
```

`section_text` 含原始 placeholder（如 `<!-- To be filled by executing agent -->` / `（待填寫：問題發生的直接原因？）`），直接 `+ new_entry` 導致 placeholder 殘留。body-schema-checker 後續偵測 placeholder 為 false positive 阻擋 complete，4/9 W3 ANA saffron 都用 `--skip-body-check` 繞過。

### 既有可重用 utilities

- `constants.py:239-241`：`待填寫 / (待填寫) / （待填寫）` placeholder 清單（並含 `(pending)` / `TBD` / `TODO` / `N/A`）
- `ticket_validator._is_placeholder()`：placeholder 偵測函式（test_ticket_validator.py 已有完整測試）
- `lib/section_locator.find_section()`：section 擷取（已在 line 583 引用）

### 修復方向（建議 thyme 確認）

1. 抽出 `_replace_or_append_section_content(section_content, new_entry, schema_comment_pattern)` helper
2. 邏輯：若 section_content 在排除 Schema HTML 註解（`<!-- Schema[...]: ... -->`）後僅剩 placeholder，則用 new_entry 替換 placeholder；否則正常 append
3. 保留 Schema 註解（這是 type-aware schema 標記，不應移除）

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

在 `track_acceptance.py` 新增 helper `_replace_or_append_section_content`，於 append-log 寫入非 Execution Log 章節時偵測 section 是否為 placeholder-only（透過 `ticket_validator._is_placeholder`）。是 placeholder 時保留 H2 header + Schema HTML 註解、丟棄 `（待填寫：...）` 與 `<!-- To be filled by executing agent -->` 等 placeholder 文字，再 append new_entry；否則沿用既有 append 行為。Execution Log 維持純 append（每筆 log 是新事件）。

### 設計決策

- 重用既有 `_is_placeholder` 與 `find_section`，與 body-schema-checker 共用語意，避免雙份偵測規則漂移
- Schema 註解保留採行級 regex（`<!--\s*Schema\[[^\]]+\]:`），支援單行與多行 Schema 註解
- 不動 Execution Log 行為，避免影響 W14-045 race regression test 語意

### 受影響 ticket 清理範圍

W3-028 / W3-030 / W3-032 / W3-033 四份 ANA ticket：
- 移除 Problem Analysis 章節重複的 `### 問題根因 / 影響範圍 / 相關 Error Pattern` placeholder 樣板（含「調查過程記錄」多行 HTML 註解），保留 PM 前台寫入的實質內容
- 將 `## Solution` 章節的 `<!-- To be filled by executing agent -->` 替換為「詳見下方 WRAP 框架分析 / Spawn 規劃」等引導文字（指向 saffron 寫入但放在自定義 H2 的歷史結構，保留語意而不重寫）
- 保留 `## Test Results` 的 `<!-- To be filled by executing agent -->`（ANA Schema 標註為選填，無實驗可留 placeholder）
- 保留表格 cell 內的 `N/A`（PC-138 / PC-144 治本：表格內為合法不適用標示）
