### PM 前台分析（2026-05-23）

W1-042.1 已將 overview 匯出按鈕改接至 BookDataExporter v2 路徑（commit fc4abd42）。本 ticket 清理隨之成為死碼的 v1 JSON/CSV 匯出鏈。

**book-exporter.js 公開方法盤點**（src/overview/book-exporter.js）：

| 方法 | 類別 | 處置 |
|------|------|------|
| handleExportCSV() | v1 CSV 匯出 | 移除 |
| handleExportJSON() | v1 JSON 匯出 | 移除 |
| generateCSVContent() | v1 CSV 內容組裝 | 移除 |
| generateJSONContent() | v1 JSON 內容組裝 | 移除 |
| downloadCSVFile() | v1 下載輔助 | 移除（確認 copyText 流程未共用） |
| downloadJSONFile() | v1 下載輔助 | 移除（確認 copyText 流程未共用） |
| handleCopyText() | 複製文字（非匯出） | 保留 |
| generateCopyText() | copyText 輔助 | 保留 |

**結論**：book-exporter.js 仍有 handleCopyText/generateCopyText 用途，採「保留檔案、移除 v1 JSON/CSV 方法」策略（對應 W1-042 ANA Solution「保留該檔但移除 JSON/CSV 方法」分支）。getFilteredBooks DI 須保留（copyText 也依賴 filtered books）。

**overview-page-controller.js 連帶死碼**：
- L260/266：匯出按鈕已呼叫 handleExportCSVv2() / handleExportJSONv2()（W1-042.1 新增，v2 路徑，保留）。
- L275：this.bookExporter.handleCopyText() — copyText 仍走 BookExporter（保留）。
- L870-927「匯出方法（委派至 BookExporter）」區塊：generateCSVContent/generateJSONContent/downloadCSVFile/downloadJSONFile 委派方法呼叫 book-exporter.js v1 方法。匯出按鈕已不經此路徑，須確認無其他呼叫者後一併移除。
- L33 import BookExporter、L134 new BookExporter：copyText 仍需，保留。

**測試檔影響**：
- tests/unit/overview/book-exporter.test.js — 含 v1 JSON/CSV 方法測試，須同步移除對應 describe，保留 copyText 測試。
- tests/unit/overview/overview-export-v2.test.js — v2 匯出測試，不受影響保留。
- tests/unit/overview/book-file-importer.test.js — importer，僅註解提及 book-exporter，無程式碼依賴。

**驗收要點**：移除後執行 npm test 確認 100% 通過；確認 overview 匯出按鈕（v2 路徑）與 copyText 功能不受影響。
