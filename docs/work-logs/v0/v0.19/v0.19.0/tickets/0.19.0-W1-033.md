## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### R1-R6 逐項處理摘要

| # | 處理 | 對外行為 |
|---|------|---------|
| R1 | loadAllBooksLazy 主迴圈雙計數器 stableCount + ineffectiveScrollCount 合併為單一 noProgressRounds。原 ineffectiveScrollCount 條件（scrollUnchanged AND count 未變）是 stableCount 條件（count 未變）的子集且共用同一 stableRounds 上限，完全冗餘。loadedCount 未增加即視為無進展。 | 不變（仍觸發 count_stable） |
| R2 | (a) waitForRenderSettle 與 waitForBookElements 重複的 MutationObserver observe 模式抽為共用 helper _observeChildListOnce。(b) waitForRenderSettle 監聽目標由傳入 scrollContainer 改為 document.body——load-more-button 策略下 container 是按鈕，掛 observer 永不觸發；新書 .library-item 渲染於 body 子樹，監聽 body 對所有策略皆有效。 | 不變（修正後混合等待對 load-more 策略恢復提早結束效益） |
| R3 | _scrollStep document 策略移除多餘的 window.scrollTo——document 策略下 container 即 documentElement，設定 scrollTop 已是整頁捲動標準做法，雙寫在 Mobile Chrome 可能不一致。改為僅 container.scrollTop，與其他策略一致。 | 不變 |
| R4 | MAX_ANCESTOR_DEPTH 原於 getBookElements LAST_RESORT(10) 與 findScrollableAncestor(12) 兩處 local 定義且值不一致，提為模組級共用常數，取較大值 12。 | getBookElements LAST_RESORT 祖先搜尋深度 10→12（更寬鬆，向下相容） |
| R5 | 隨 R1 收斂：scrollTop try/catch 重複 4 處抽為 readScrollTop helper（捲動有效性探測移除後實際剩 1 處 originalScrollTop 讀取，仍以 helper 統一容錯）。maxIterations 三處判定（while 條件 + 迴圈內二次判定 + 收尾補設）收斂為 while(true) 單一出口 + 迴圈內 (c) 判定。loadAllBooksLazy doc 註解的停止條件說明同步更新與實作一致。 | 不變 |
| R6 | parseLibraryTotal 負數處理由 Math.max(0, N-X-Y) 夾 0 改為回 null（對齊 Phase 3a 待釐清 2 決議）。夾 0 會讓 loadAllBooksLazy 誤判 already_complete（0 本書庫），回 null 才正確退回穩定條件停止。TC-7.6 期望值同步由 toBe(0) 改為 toBeNull()。 | R6 為對齊已決策規格的刻意行為變更 |

### 不在本 ticket 範圍（Phase 4a 僅記錄不重構，未動）

linux F5（finalize 閉包多狀態）、ce F2（_unsafeUrlCount 計數器污染）、ce F5/F7（進度回報時序、涵蓋率可觀測性）。

### 設計決策

- R1 等價性論證：原 stableCount 必先於或同時於 ineffectiveScrollCount 達上限（兩者同 stableRounds 限制，前者條件更寬），故合併後 noProgressRounds 行為等同原 stableCount，ineffectiveScrollCount 為純冗餘。
- R2 監聽 body 而非按鈕：經測試確認無測試斷言 waitForRenderSettle 內部監聽目標，tests 全部 spy _scrollStep / _measureBooks，斷言對象為 stopReason 枚舉（耦合行為而非實作結構）。
- 測試穩定性檢查：R1/R3/R4/R5 純內部重構，4 個 adapter 測試檔（121 測試）零修改全綠 → 測試耦合到行為（正確）。僅 R6 因屬刻意行為變更（對齊規格）同步更新 TC-7.6。
