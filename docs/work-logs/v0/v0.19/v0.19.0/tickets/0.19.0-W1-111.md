### W1-107 Solution Extract（供 thyme 撰寫文件用，免跨 ticket Read）

#### 議題 A：GlobalMessages 納入標準（3 條件）

| 條件 | 說明 |
|------|------|
| 被 2+ 獨立模組引用 | 同一 `src/` 子目錄視為同一模組 |
| 屬通用詞彙 | 錯誤碼（NETWORK_ERROR）、操作狀態（SUCCESS/FAILED）、系統生命週期（SYSTEM_READY） |
| 無模組前綴 | key 名稱不含 POPUP_/SEARCH_/VALIDATOR_ 等模組識別詞 |

#### 合格 GlobalMessages key 清單（W1-107 建議：約 20 個）

| 分類 | key 範例 |
|------|---------|
| 錯誤類 | `VALIDATION_FAILED`, `NETWORK_ERROR`, `STORAGE_ERROR`, `PERMISSION_DENIED`, `UNKNOWN_ERROR` |
| 操作類 | `OPERATION_START`, `OPERATION_COMPLETE`, `OPERATION_CANCELLED`, `OPERATION_TIMEOUT`, `OPERATION_RETRY` |
| 系統類 | `SYSTEM_READY`, `SYSTEM_SHUTDOWN`, `LOADING`, `PROCESSING` |
| 通用 UI | `SUCCESS`, `FAILED`, `RETRY`, `CANCEL`, `CONFIRM` |
| 測試 | `TEST_MESSAGE`, `TEST_WITH_PARAMS` |

#### 應移除 key 分類（W1-107 統計：約 60+ 個）

| 模組 | 數量 | 處理方式 |
|------|------|---------|
| search-filter 專屬 | 8 個 | 移至已有 local dict（searchUIMessages） |
| validator 專屬 | 15 個 | 移至已有 local dict（validatorMessages） |
| extractor/content 專屬 | ~18 個 | 需建立 local dict |
| Chrome Extension 專屬（`POPUP_OPENED` 等） | 4 個 | 移至 popup local dict 或刪除（未引用） |
| 日誌模板（`DEBUG_MESSAGE` 等） | 4 個 | 移至 Logger 模組 local dict |
| 書庫相關 | 5 個 | 移至對應 local dict |
| 中文 key 遺留 | 3 個 | 移除或移至對應 local dict |

#### 議題 B 方案 1：命名前綴規範（W1-111 屬此方案落地）

- 含模組前綴的 key（如 `POPUP_*`、`SEARCH_*`、`VALIDATOR_*`、`EXTRACTOR_*`）**禁止**出現在 `MessageDictionary.js` 的 `_loadDefaultMessages()` 中
- 模組前綴 key 必須由對應模組的 local dict（`new MessageDictionary({...})`）註冊
- 命名前綴規範作為方案 3（freeze GlobalMessages，由 W1-110 處理）的輔助文件，提供 code review 判定標準
