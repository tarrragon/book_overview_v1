#### 0. 框架內建機制驗證（ARCH-010 防護）

| 步驟 | 結論 |
|------|------|
| 框架內建（瀏覽器 `confirm()`）是否足夠？ | 否——CSP 與一致 UX 要求自製 modal；既有 Modal A 已設立 modal 慣例，confirm() 阻塞且樣式不一致 |
| 既有 Modal A 是否可重用？ | 否——Modal A 職責是「mode 選擇」（資訊輸入），Modal B 是「destructive 操作確認」（警告 + 二次確認），混用破壞單一職責並使既有測試/邏輯複雜化 |
| 本地狀態是否足夠？ | 否——需 DOM 渲染 + 焦點管理 + Promise 包裝事件，與 Modal A 為對稱結構 |
| 結論 | 新建獨立 Modal B，沿用 Modal A 的 Promise 包裝 + settle 統一收斂模式 |

**Consequence（混用既有 Modal A 的代價）**：若強行擴充 Modal A 承擔「mode 選擇 + destructive 確認」雙職責，將直接觸發既有 26 TC（import-mode-modal.test.js）regression——按鈕集合 / focus trap 集合 / settle 出口分支 / 文案斷言全數改變；同時破壞 single-responsibility，未來任一職責變更皆需重測雙場景。獨立 Modal B 為唯一保留既有測試 baseline 的設計選擇。

##### 2.3 Modal A 與 Modal B closure 順序保證

| 順序保證 | 機制 |
|---------|------|
| Modal A 完全 closed -> Modal B 才開啟 | execute 為 async function，await Modal A.promptImportMode() resolve 時 settle() 已執行完畢（移除監聽器 + 隱藏 + 焦點還原 + 清 pending 旗標）。Modal B 在後續 await 點之後才呼叫，DOM 與 listener 層面不重疊 |

#### 4. 與既有 Modal A 隔離設計

##### 4.1 DOM 隔離

| 維度 | Modal A | Modal B |
|------|---------|---------|
| Overlay id | importModeOverlay | emptyFileConfirmOverlay |
| Modal id | importModeModal | emptyFileConfirmModal |
| 按鈕 id | importMode{Overwrite,Merge,Cancel}Btn | emptyFileConfirm{Proceed,Cancel}Btn |
| aria-labelledby 指向 | importModeTitle | emptyFileConfirmTitle |

兩 modal DOM 完全獨立，無 id 衝突，無共用節點。

##### 4.2 Event listener 隔離

| 維度 | 機制 |
|------|------|
| 各自獨立 settle 閉包 | Modal A 的 settle 只 removeEventListener Modal A 的 handlers；Modal B 同理 |
| 不共用 keydown handler | Modal A 的 Esc handler bind 在 importModeModal，Modal B bind 在 emptyFileConfirmModal，互不觸發 |
| 不共用 overlay click handler | 同上，bind 在各自 overlay |
| 時間軸序列化 | execute 流程保證 Modal A settle 完成（含 removeEventListener）後才開啟 Modal B，物理上不重疊 |

##### 4.3 Focus 隔離

| 機制 | 說明 |
|------|------|
| 獨立 previousFocus 變數 | _importModePreviousFocus（Modal A）vs _emptyConfirmPreviousFocus（Modal B）獨立儲存 |
| 還原鏈正確 | Modal A 還原至 loadFileBtn；Modal B 開啟時 previousFocus 為 Modal A settle 後的 activeElement（通常為 loadFileBtn 或 body）。Modal B 開啟時的 previousFocus 不會是 Modal A 的按鈕（Modal A 已 hide），故無焦點丟失問題 |
| Focus trap 集合獨立 | Modal A: [overwrite, merge, cancel]；Modal B: [cancel, proceed] |

##### 4.4 Pending Promise 隔離

| 變數 | 用途 |
|------|------|
| _importModePending | Modal A 單一實例保護（既有） |
| _emptyConfirmPending | Modal B 單一實例保護（新增，同設計模式） |

兩變數獨立，重複觸發各自 modal 不會互相干擾。
