## Problem Analysis

UC-01 主要流程 step 6 要求「Popup 顯示提取結果：成功提取 X 本書籍」。W1-001.2 實機驗證發現提取成功後 popup 恆顯「檢測中...」，缺少正向回饋。W1-007 已修復「提取失敗/未知錯誤」誤報，但未加正向成功訊息。

---

## Solution
### 修復摘要

在 popup.js 新增 `setupExtractionCompletionListener()` 函式，註冊 `chrome.storage.onChanged` 監聽器於 `startExtraction()` 觸發 START_EXTRACTION 之前。當 background 異步寫入 `chrome.storage.local.readmoo_books` 時，listener 讀取 newValue.books.length 並更新 popup UI 顯示「提取成功 N 本書籍」與 bookCount=N，然後一次性自我移除。

### 設計決策

- **採 chrome.storage.onChanged 而非 chrome.runtime.onMessage**：background 既有資料流終點為 `chrome.storage.local.set({ readmoo_books })`（event-coordinator.js:586），無 popup 推送訊息通道。新增 storage 監聽是「最小耦合 + 範圍鎖定 src/popup/」的合規實作，符合 ticket scope。
- **監聽一次性自我移除**：popup 生命週期短，提取結果顯示完即可解除，避免重複觸發或 memory 累積。
- **STORAGE area 限定 local**：與 background 寫入位置一致；sync / managed 區域變更不誤觸發。
- **newValue 與 books 陣列防禦**：缺失（移除事件）或非陣列（資料異常）時保留既有狀態，避免錯誤回饋。

### 失敗情境處理

- chrome.storage 不存在時 early return（測試環境 / Service Worker 重啟邊界）。
- removeListener 失敗時記錄 warn log（observability-rules 規則 1）。
- 「提取失敗」路徑透過 catch updateStatus('失敗', '提取失敗', ...) 不受影響。
