| 項目 | 內容 |
|------|------|
| 根因分類 | 過時測試（測試前提被新增的 status gate 攔截，斷言對象從未產生） |
| 直接原因 | execute_append_log 在 W3-044 新增 require_in_progress precondition（track_acceptance.py:631-642），早於 section-not-found 邏輯。test 的 fake_ticket 只有 id/_body、無 status，被 status gate 判為 status=pending，寫 stderr + return 2，根本到不了 SECTION_NOT_FOUND 程式碼。stdout 為空 → assert "Solution" in output 失敗（訊息實際在 stderr）。 |
| 引入 commit | 4946d028 feat(0.19.0-W3-044): S3 wire append-log status precondition（新增 status gate） |

| 修復路徑 | 直接修測試（讓 fake_ticket 帶 status: in_progress，使其通過 precondition 進到 section-not-found 邏輯）。屬測試前提過時，非產品 bug。 |

| 引入 commit | b1781659 feat(0.18.0-W13-018): 實作 ticket track hook-health CLI 子命令（test 與實作同 commit，fixture 自始與 evaluate 閾值公式不匹配） |
| 與 W3-082 關係 | 無關。W13-018 遠早於 W3-082。pre-existing 確認成立。 |
| 修復路徑 | 直接修測試 fixture：將 _fake_stats() 的 per_day 值拉高到穩定超過 high_freq threshold（如連 3 日 >baselinex3），或注入 now 使 recent 命中 today。附加脆弱性：fixture per_day 用寫死日期，若改走 recent 路徑會時間依賴；建議用相對日期或固定 now 注入 _evaluate_all。 |
