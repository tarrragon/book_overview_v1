## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 執行摘要

PM 自行執行（thyme-extension-engineer 為規劃代理人，無 Edit/Write/chrome-devtools-mcp 不適合實機驗證）。流程：

1. 跑 `npm run package` 產出 `dist/readmoo-book-extractor-v0.18.0.zip`（1.27 MB）
2. 解壓至 `$TMPDIR/readmoo-ext-w1-002-2/`（chrome-devtools-mcp workspace roots 限制 /tmp，必須用 TMPDIR 系統 temp）
3. `mcp__chrome-devtools__install_extension` 載入解壓後 unpacked extension，ID: `phhicihcocdiopiaimejpapbkdammnjn`
4. `list_pages` 確認 SW 自動啟動（Manifest V3 idle unload 後可由事件觸發喚醒）
5. `evaluate_script` 在 SW context 確認 listener 註冊與 chrome API 可用
6. `new_page` 到 `read.readmoo.com/#/library`（被 Readmoo 重定向至 SSO 登入頁，符合未登入預期）
7. `list_console_messages` 確認 content script 注入並回報 ready

### WRAP 三問

- **W (Widen)**：候選方案 (a) PM 自行執行 + chrome-devtools-mcp [採用] (b) 派 thyme-extension-engineer（無 Edit/Write/mcp 不可行）(c) 用戶手動本機驗證
- **A (Attain distance)**：機會成本 = 未接 W1-002.3 installation-guide；本任務優先因為 W1-003.x 都依賴 ZIP 驗證可用
- **P (Prepare to be wrong)**：預想失敗點 = chrome-devtools-mcp install_extension 對 Manifest V3 SW idle unload 處理（實際遇到 sw-1 unload 後改 evaluate sw-2，已避免）

### 設計決策

- **選擇 TMPDIR 路徑**：chrome-devtools-mcp 拒絕 /tmp（workspace roots 限制），改用 macOS `/var/folders/.../T/` 系統 temp
- **不勾選未登入流程驗證**：acceptance 只要求安裝成功 + SW 0 error + content script 注入；未要求 OAuth 完整登入後的書庫提取（屬 W1-001 / W1-003 範圍）
- **SW 驗證採 evaluate_script 而非 console**：list_console_messages 只能看 selected page，SW 需用 evaluate_script with serviceWorkerId 在 SW context 內驗證 listener 與 API
