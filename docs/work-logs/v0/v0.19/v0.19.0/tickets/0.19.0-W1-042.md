分析 UC-03 要求的 Interchange Format v2 匯出職責歸屬與合規缺口：overview 匯出 JSON 產出 {books:[...]} 無 version 無 tag_categories，book 欄位為 id/title/progress/status/cover/tags 非 v2 schema

## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

Overview 匯出按鈕（`src/overview/overview-page-controller.js` L243/L249）直接呼叫 `src/overview/book-exporter.js` 的 `handleExportJSON()` / `handleExportCSV()`。此模組是早期 v1 時代的簡易匯出器，完全不知道 Interchange Format v2 規格的存在：

1. **JSON 匯出**（`generateJSONContent` L144-157）：手動組裝 `{ books: [...] }`，每本書只取 `{id, title, progress, status, cover, tags}`。無 `metadata`、無 `tagCategories`、無 `tags` 頂層區段、無 `readingStatus`（用 `status`）、無 `tagIds`（用 `tags`）。
2. **CSV 匯出**（`generateCSVContent` L132-139）：固定 8 欄 headers `['書名','書城來源','進度','狀態','封面URL','id','authors','tagIds']`，欄位名稱為中文混英文，無 `readingStatus`、無衍生欄位 `tagNames`/`tagCategories`。

同時，`src/export/book-data-exporter.js` 已有完整的 v2 匯出實作（`_exportToJSONv2` L470-521、`_exportToCSVv2` L291-342），含 EXTENDED_V2/COMPLETE_V2 preset、metadata 區段、tagCategories/tags 頂層區段、衍生欄位 resolve。但 overview 按鈕**完全未接線到此模組**。

根因：overview 匯出按鈕接線的是 v1 簡易匯出器（`book-exporter.js`），而非已實作的 v2 完整匯出器（`book-data-exporter.js`）。

### 影響範圍

- `src/overview/book-exporter.js` — v1 簡易匯出器（JSON/CSV 均不符 v2）
- `src/overview/overview-page-controller.js` — 匯出按鈕接線（L243/L249）
- `src/export/book-data-exporter.js` — v2 匯出器（已實作但未接線至 overview UI）

### 相關 Error Pattern

無直接對應的已知 error-pattern。此為「設計選擇 + 規格未對齊」共振（csv-export-spec.md 背景段已記載）。

## 重現實驗結果
### 實驗方法

靜態程式碼分析：逐行比對 `src/overview/book-exporter.js` 的 JSON/CSV 產出結構 vs `docs/spec/export-interchange-format-v2.md` 的 v2 schema 要求。

### 實驗執行

**JSON 匯出（book-exporter.js L144-157）產出結構**：
```json
{
  "books": [
    { "id": "", "title": "", "progress": 0, "status": "", "cover": "", "tags": [] }
  ]
}
```

**v2 規格要求結構**（spec S3.1）：
```json
{
  "metadata": { "formatVersion": "2.0.0", "exportDate": "...", "source": "readmoo-book-extractor", "schemaVersion": "3.0.0", "totalBooks": N, "totalTags": N, "totalTagCategories": N },
  "tagCategories": [...],
  "tags": [...],
  "books": [
    { "id": "", "title": "", "readingStatus": "...", "authors": [], "publisher": "", "progress": 0, "type": "", "cover": "", "tagIds": [], "isManualStatus": false, "extractedAt": "...", "updatedAt": "...", "source": "readmoo" }
  ]
}
```

**CSV 匯出（book-exporter.js L17/L132-139）產出 headers**：
`書名,書城來源,進度,狀態,封面URL,id,authors,tagIds`

**v2 CSV 規格要求 headers**（spec S5.1 EXTENDED_V2 + 衍生欄位）：
`id,title,authors,publisher,progress,readingStatus,type,tagIds,tagNames,tagCategories`

### 實驗發現

**已驗證事實**：

| # | 缺口 | book-exporter.js 現況 | v2 規格要求 | 嚴重度 |
|---|------|----------------------|------------|--------|
| 1 | 無 metadata 區段 | root = `{books}` | root = `{metadata, tagCategories, tags, books}` | Critical |
| 2 | 無 tagCategories 頂層 | 缺失 | 必須含完整 tagCategory 陣列 | Critical |
| 3 | 無 tags 頂層 | 缺失 | 必須含完整 tag 陣列 | Critical |
| 4 | 書籍用 `status` 非 `readingStatus` | `status` (自由字串) | `readingStatus` (6 種 enum) | Critical |
| 5 | 書籍用 `tags` 非 `tagIds` | `tags` (混合型) | `tagIds` (string[]) | Critical |
| 6 | 缺 authors/publisher/type/isManualStatus/extractedAt/updatedAt/source | 僅 id/title/progress/status/cover/tags | 至少 EXTENDED_V2 全欄位 | High |
| 7 | CSV headers 中文混英文 | `書名,書城來源,...` | `id,title,authors,...` | High |
| 8 | CSV 無 readingStatus 欄位 | `狀態` (中文) | `readingStatus` (英文 enum) | Critical |
| 9 | CSV 無衍生欄位 tagNames/tagCategories | 缺失 | spec S4.2 要求 | Medium |
| 10 | JSON version=null | 無 formatVersion | `"2.0.0"` | Critical |

**結論**：`book-exporter.js` 的 JSON 和 CSV 匯出均與 v2 規格完全不相容。修復方向不是修補 `book-exporter.js`，而是將 overview 匯出按鈕改接已有的 `src/export/book-data-exporter.js` v2 路徑。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### Analysis Result

#### 職責歸屬結論

v2 匯出職責歸屬 `src/export/book-data-exporter.js`（已有完整 v2 JSON/CSV 實作）。`src/overview/book-exporter.js` 是 v1 遺留簡易匯出器，不應升級——應改接線。

#### 修復策略

Overview 匯出按鈕（`overview-page-controller.js`）應改為呼叫 `BookDataExporter` 的 v2 路徑：
- JSON：`exportToJSON({ formatVersion: '2.0.0', fieldPreset: 'EXTENDED_V2', tags, tagCategories })`
- CSV：`exportToCSV({ formatVersion: '2.0.0', fieldPreset: 'EXTENDED_V2', tags, tagCategories })`

接線時需解決：
1. `BookDataExporter` 需要 `tags` 和 `tagCategories` 資料——overview controller 需從 Chrome Storage 讀取（透過既有 `TagStorageAdapter` 或直接 `chrome.storage.local.get`）
2. `BookDataExporter` 構造函式接收 `books` 陣列和 `options`，與 `BookExporter` 的 DI 介面（`getFilteredBooks`）不同——需適配
3. 下載觸發邏輯（`_triggerFileDownload`）可保留 `book-exporter.js` 的實作或改用 `BookDataExporter.downloadFile`

#### 缺口清單（JSON + CSV vs v2）

見「重現實驗結果」章節的 10 項缺口表。
