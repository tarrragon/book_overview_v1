#### 測試策略決策

| 項目 | 決策 | 理由 |
|------|------|------|
| 測試類型 | 單元測試（Sociable Unit） | ContentParser 為純函式 class，無 IO 無 state |
| 測試檔案 | `tests/unit/overview/import/content-parser.test.js`（新建） | 隔離於 importer 既有測試外 |
| Mock 策略 | detectFormatVersion / convertV1ToV2Data 預設用真實模組（內層 Domain 邏輯，依 Sociable 原則不 Mock）；測試僅在版本邊界 case 注入 stub | 對齊 sociable unit 原則：只 Mock 外部依賴 |
| 測試框架 | Jest | 既有 |

#### 既有測試案例遷移清單

來源：
- `tests/unit/overview/book-file-importer.test.js`
- `tests/unit/overview/book-file-importer-import-result.test.js`
- `tests/unit/overview/book-file-importer-version-detection.test.js`
- `tests/unit/overview/overview-import-private-methods.test.js`（_handleFileContent 區段）

| 測試類別 | 遷移後 GWT 簡述 |
|---------|---------------|
| JSON parse 成功 | Given 合法 JSON content + fileFormat='json'，When parse，Then 回傳 ImportResult {books, tagCategories, tags} 三欄位皆陣列 |
| CSV parse 成功 | Given 含 header 的 CSV + fileFormat='csv'，When parse，Then books 含對應 entries |
| v1 → v2 轉換 | Given v1 schema JSON，When parse，Then convertV1ToV2Data 被執行（透過結果結構驗證） |
| v2 直接解析 | Given v2 schema JSON，When parse，Then 不經 v1 converter，直接結構化 |
| 三區段格式（books/tagCategories/tags） | Given metadata 包裝 JSON，When parse，Then 三欄位分別填充 |
| INV-1 不變式 | Given 任意合法 input，When parse，Then books/tagCategories/tags 皆為陣列（即使空） |
| BOM 移除 | Given content with `﻿` 前綴，When parse，Then 解析成功（_removeBOM 等價） |
| 大型資料集警告 | Given books 數量 > 1000，When parse，Then 警告 metadata 存在或 log（依既有契約） |

#### 新增測試案例（API 簽章 + 邊界）

| Scenario | Given | When | Then |
|---------|-------|------|------|
| API 簽章存在 | ContentParser import | `new ContentParser()` | instance.parse 為 function |
| fileFormat 缺漏 | content 合法 | `parse(content, undefined)` | throw TypeError（fileFormat 必填） |
| fileFormat 無效 | content 合法 | `parse(content, 'xml')` | throw Error code=PARSE_ERROR |
| 空 content | content='' | `parse('', 'json')` | throw Error code=VALIDATION_ERROR |
| null content | content=null | `parse(null, 'json')` | throw Error code=VALIDATION_ERROR |
| 損毀 JSON | `'{ broken'` | `parse(_, 'json')` | throw Error code=PARSE_ERROR，details.cause 含原 SyntaxError |
| CSV 缺必要欄位 | header 無 'title' | parse(_, 'csv') | throw Error code=PARSE_ERROR |
| 無法辨識的 JSON 結構 | `'{}'` | parse(_, 'json') | throw Error code=VALIDATION_ERROR |
| INV-1：空 books 但結構合法 | `'{"books":[]}'` | parse | books=[], tagCategories=[], tags=[]（陣列非 undefined） |
| 自訂 largeDatasetThreshold | deps.largeDatasetThreshold=10 + 11 books | parse | 觸發警告路徑 |
| detectFormatVersion 注入 stub | deps.detectFormatVersion=jest.fn().mockReturnValue('v2') | parse(any json) | stub 被呼叫 1 次 |
| convertV1ToV2Data 注入 stub | deps.convertV1ToV2Data=jest.fn(d=>d) | parse(v1 json) | stub 被呼叫 1 次 |

#### 反向斷言

- instance 不暴露舊底線方法（_handleFileContent / _parseJSONContent / _parseCSVContent / _processBookData / _extractBooksFromData 等核心 helper 皆 undefined）

#### 行為鏈推演

parse(content, fileFormat) 前置鏈：
1. fileFormat 必填（TypeError）
2. fileFormat 合法（PARSE_ERROR）
3. content 非空（VALIDATION_ERROR）
4. BOM 移除
5. format-specific parse（JSON SyntaxError → PARSE_ERROR；CSV 欄位 → PARSE_ERROR）
6. 結構辨識（v1/v2/三區段 / 未知 → VALIDATION_ERROR）
7. INV-1 後處理（三欄位陣列化）

每個前置失敗都用獨立 case 驗證短路。

#### Mock 與測試資料

- 真實 detectFormatVersion / convertV1ToV2Data 載入（內層）；僅版本邊界 case 注入 stub
- JSON / CSV fixture 直接 inline 在測試檔（既有 fixture 可從 importer.test.js 複製對應字串常數）
- 無外部 IO

#### 預期測試案例數：~35-40 case（既有 ~25 + 新增 ~12）
