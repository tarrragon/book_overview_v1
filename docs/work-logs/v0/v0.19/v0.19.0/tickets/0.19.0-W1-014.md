### 修復摘要

W6-012.2.1 將 `generateStableBookId` 的 ID 策略優先級由 `cover > title > reader` 改為 `reader > cover > title`。本 ticket 逐一更新 28 個測試（`stable-id-generation.test.js` 27 + `content-modular.test.js` 1）的期望值對齊新邏輯，未改動 `src/` 產品程式碼。

逐測試處理（非無腦 find-replace，期望值含 `toBe('cover-X')` / `toBe('reader-X')` / `toMatch(/regex/)` 三種形式）：

- 測試名稱明示 reader 策略、或策略無關（唯一性 / 冪等性 / 效能）者：直接修正 expected。一併修正 4 處過時的 `unstable-` 前綴（TC007/008/012/020/029）——實際 reader 策略與 `createFallbackId` 輸出皆為 `reader-` 前綴。
- 測試名稱明示須驗證 cover / title 策略者（封面 ID 優先、標題正規化、XSS/SQL 過濾等）：將 `readerId` 輸入改為 `null`，讓目標策略真正生效，保留測試原意（否則新優先級下 reader 策略恆先，測試名稱失真）。
- 每處變更附 `W6-012.2.1` 註解。
