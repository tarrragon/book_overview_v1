### Phase 1 功能規格摘要（lavender-interface-designer）

完整規格：`docs/work-logs/v0/v0.19/v0.19.0/tickets/0.19.0-W1-047.4-feature-spec.md`（Full 模式 6 區段，/spec validate 2 次迭代收斂、0 未回答問題）。

#### 範圍

實作 SPEC-EXPORT-V2 §8.2 合併模式與 §8.3 tag/category id 重映射。與 IMP-C 覆蓋模式對稱：storage 層新增 mergeAllData（合併），與既有 replaceAllData（覆蓋）介面對稱。模式選擇 UI 屬 IMP-E，不在本 ticket——本 ticket 僅提供可被 IMP-E 呼叫的合併 API。

#### 架構決策

- storage 層新增 mergeAllData({ books, tags, tagCategories })：與 replaceAllData 對稱。合併「讀本地三 key → 計算 → 原子寫回」全程在 operationLock 內，複用 withAtomicRollback / checkQuotaLevel。回傳 { success, error?, counts?, remap? }。
- 內部純函式 computeMergeResult(local, incoming)：合併計算與 storage I/O 解耦，使 books 聯集 / 同名比對 / id 重映射可獨立於 chrome.storage mock 單元測試。新建 id 透過注入的 id 產生器提供（純函式不可內呼 Date.now/Math.random）。
- id 重映射兩階段：先 category（建 categoryIdMap）後 tag（以「重映射後本地 categoryId + name.toLowerCase」為鍵）。多個匯入 category 收斂至同一本地 category 時其下同名 tag 亦收斂。
- 合併不修復本地破損：本地既有孤兒 tagId 在聯集後原樣保留；引用完整性修復屬既有 checkReferentialIntegrity 職責。

#### 核心行為（14 GWT 場景）

books 同 id tagIds 聯集去重 / 新 id 加入 / tag·category 同名（大小寫不敏感）保留本地、不存在則新建 / id 重映射貫穿 category→tag→book 三層 / 原子持久化 / 配額攔截 / 寫入失敗回滾 / 空匯入合法 / 兩階段收斂 / 不修復本地孤兒。

#### 行為鏈推演說明

`computeMergeResult` 為核心。行為鏈：`category 同名比對 → 建 categoryIdMap`（前置：incoming categories 已知）→ `tag 同名比對（鍵 = 重映射後 categoryId + name.toLowerCase）→ 建 tagIdMap`（前置：categoryIdMap 已完成）→ `book tagIds 透過 tagIdMap 重映射 + 本地聯集`（前置：tagIdMap 已完成）。每層測試前先斷言上一階段映射表已正確建立，再驗證本層輸出。`mergeAllData` 行為鏈：`配額前置攔截 → 讀本地三 key → 呼叫 computeMergeResult → withAtomicRollback 原子寫回`。

#### 1. 實作策略設計

新增兩個函式於 `tag-storage-adapter.js`：純函式 `computeMergeResult`（合併計算）+ 公開 API `mergeAllData`（storage I/O 編排）。`mergeAllData` 與既有 `replaceAllData` 對稱：`operationLock.run` 內「配額前置攔截 → 讀本地三 key → 呼叫純函式 → withAtomicRollback 原子寫回」。

**虛擬碼 A：`mergeAllData`（公開 API）**

```
函式 mergeAllData(books, tags, tagCategories):
  回傳 operationLock.run(async ():
    # 步驟 A：配額前置攔截（與 replaceAllData 對稱）
    quota ← await checkQuotaLevel()
    若 quota.level == 'blocked':
      console.error('[tag-storage-adapter] mergeAllData blocked: quota exceeded')
      回傳 { success: false, error: 'quota_exceeded' }

    # 步驟 B：讀本地三 key 現況
    previousBooks ← await loadBooks()
    previousTags ← await loadTags()
    previousCategories ← await loadCategories()

    # 步驟 C：計算合併結果（純函式，注入正式 id 產生器）
    idGenerators ← {
      nextCategoryId: () => `cat_${Date.now()}-${randSuffix()}`,
      nextTagId:      () => `tag_${Date.now()}-${randSuffix()}`
    }
    merged ← computeMergeResult(
      { books: previousBooks, tags: previousTags, tagCategories: previousCategories },
      { books, tags, tagCategories },
      idGenerators
    )

    # 步驟 D：原子寫回（快照 key 名對齊 SNAPSHOT_KEY_TO_STORAGE_KEY：books/tags/categories）
    result ← await withAtomicRollback(
      { books: previousBooks, tags: previousTags, categories: previousCategories },
      async ():
        await saveBooksWrapper(merged.books)
        await saveToStorage({ [TAGS]: merged.tags })
        await saveToStorage({ [TAG_CATEGORIES]: merged.tagCategories })
        回傳 { success: true }
      , 'mergeAllData'
    )

    # 步驟 E：判定終點
    若 result.success == true:
      回傳 {
        success: true,
        counts: { books: merged.books.length,
                  tags: merged.tags.length,
                  tagCategories: merged.tagCategories.length },
        remap: { categories: merged.remap.categoryIdMap.size,
                 tags:       merged.remap.tagIdMap.size }
      }
    console.error('[tag-storage-adapter] mergeAllData failed after rollback')
    回傳 { success: false, error: 'storage_error' }
  )
```

**虛擬碼 B：`computeMergeResult`（純函式）— 四階段管線**

```
函式 computeMergeResult(local, incoming, idGenerators):
  # === 階段 1：category 同名比對與重映射 ===
  categoryIdMap ← 空 Map        # 匯入 category id → 本地 category id
  resultCategories ← local.tagCategories 的淺拷貝陣列
  # 本地同名索引：name.toLowerCase() → 本地 category 物件
  localCatByName ← Map(resultCategories 每筆: c.name.toLowerCase() → c)

  對每個 impCat in incoming.tagCategories:
    name ← 截斷至 50 字元(impCat.name)
    key ← name.toLowerCase()
    若 localCatByName 含 key:
      # 同名（含「先前 incoming 已收斂建立」的）→ 映射至既有本地 id
      categoryIdMap.set(impCat.id, localCatByName.get(key).id)
    否則:
      # 新建本地 category
      newId ← idGenerators.nextCategoryId()
      newCat ← { ...impCat, id: newId, name: name }
      resultCategories.push(newCat)
      localCatByName.set(key, newCat)        # 後續同名 incoming 收斂至此
      categoryIdMap.set(impCat.id, newId)

  # === 階段 2：tag 同名比對與重映射 ===
  tagIdMap ← 空 Map             # 匯入 tag id → 本地 tag id
  resultTags ← local.tags 的淺拷貝陣列
  # 本地同名索引：複合鍵 `${categoryId} ${name.toLowerCase()}` → 本地 tag 物件
  localTagByKey ← Map(resultTags 每筆: tagKey(t.categoryId, t.name) → t)

  對每個 impTag in incoming.tags:
    name ← 截斷至 50 字元(impTag.name)
    # 重映射 categoryId；找不到對應 → 歸入「未分類」
    localCatId ← categoryIdMap.get(impTag.categoryId)
    若 localCatId 未定義:
      localCatId ← ensureUncategorized()   # 見階段 2a
    key ← `${localCatId} ${name.toLowerCase()}`
    若 localTagByKey 含 key:
      tagIdMap.set(impTag.id, localTagByKey.get(key).id)
    否則:
      newId ← idGenerators.nextTagId()
      newTag ← { ...impTag, id: newId, name: name, categoryId: localCatId }
      resultTags.push(newTag)
      localTagByKey.set(key, newTag)
      tagIdMap.set(impTag.id, newId)

  # === 階段 3：book tagIds 重映射 + 本地聯集 ===
  resultBooks ← local.books 的淺拷貝（依 id 建索引 localBookById）
  validLocalTagIds ← (略；本地側不過濾，僅匯入側過濾)

  對每個 impBook in incoming.books:
    若 impBook.id 缺失:
      console.warn('[tag-storage-adapter] merge: book 缺 id 跳過')
      continue
    # 重映射匯入 tagIds，過濾無法重映射者
    remappedTagIds ← []
    對每個 tid in (impBook.tagIds || []):
      mapped ← tagIdMap.get(tid)
      若 mapped 未定義:
        console.warn('[tag-storage-adapter] merge: 匯入 tagId 無法重映射，過濾 ' + tid)
        continue
      remappedTagIds.push(mapped)
    若 localBookById 含 impBook.id:
      localBook ← localBookById.get(impBook.id)
      # 同 id：其餘欄位以匯入覆蓋，tagIds 取聯集（本地在前，新增在後，去重）
      unionTagIds ← dedup([...（localBook.tagIds || []）, ...remappedTagIds])
      若 unionTagIds.length > 100:
        console.warn('[tag-storage-adapter] merge: tagIds 聯集超 100 截斷')
        unionTagIds ← unionTagIds.slice(0, 100)
      mergedBook ← { ...localBook, ...impBook, tagIds: unionTagIds }
      localBookById.set(impBook.id, mergedBook)
    否則:
      # 新 id 直接加入
      localBookById.set(impBook.id, { ...impBook, tagIds: remappedTagIds })

  resultBooks ← localBookById 的 values 陣列

  回傳 {
    books: resultBooks, tags: resultTags, tagCategories: resultCategories,
    remap: { categoryIdMap, tagIdMap }
  }
```

**階段 2a：`ensureUncategorized` 輔助邏輯**

當匯入 tag 的 `categoryId` 在 `categoryIdMap` 無對應（incoming 檔案自身不一致）時，需歸入名為 `"未分類"` 的 category。此「未分類」同樣參與同名比對：

```
函式 ensureUncategorized():
  key ← '未分類'.toLowerCase()
  若 localCatByName 含 key:
    回傳 localCatByName.get(key).id      # 本地已有「未分類」→ 用既有
  newId ← idGenerators.nextCategoryId()
  newCat ← { id: newId, name: '未分類', ... 預設欄位 }
  resultCategories.push(newCat)
  localCatByName.set(key, newCat)
  回傳 newId
```

惰性建立（首次需要時才建），避免無孤兒 tag 時憑空多出「未分類」。

#### 2. 資料結構選擇

| 結構 | 選擇 | 理由 |
|------|------|------|
| `categoryIdMap` / `tagIdMap` | `Map` | 鍵為任意字串 id，`Map` 語意明確、`.size` 直接供 remap 統計、避免 `Object` 原型污染風險 |
| `localCatByName`（同名索引） | `Map<string, Object>` | O(1) 同名查找；值存物件參考以便收斂時同一筆被後續 incoming 命中 |
| `localTagByKey`（tag 複合鍵索引） | `Map<string, Object>`，鍵 = `categoryId + ' ' + name.toLowerCase()` | 同名比對鍵須含 categoryId（TC-C12：不同 category 同名 tag 不誤判）；用 NUL 字元 ` ` 作分隔避免 id/name 內含分隔字元造成鍵碰撞 |
| `localBookById` | `Map<string, Object>` | 同 id 合併需 O(1) 查找；最終以 `.values()` 還原陣列，保留本地書在前、新書在後的插入順序 |
| 聯集去重 | `[...new Set([...localTagIds, ...remappedTagIds])]` | `Set` 保留插入順序（本地在前），自動去重，對齊 TC-C1 預期「本地在前、新增在後」 |
