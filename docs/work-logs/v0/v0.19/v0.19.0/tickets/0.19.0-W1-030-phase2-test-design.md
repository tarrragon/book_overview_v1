## 1. 測試策略總覽

### 1.1 分層測試決策

| 待測對象 | 層級 | 測試類型 | 理由 |
|---------|------|---------|------|
| `loadAllBooksLazy` 停止邏輯（4 條件） | 演算法核心 | 單元測試（注入 mock 捲動/量測函式） | 規格 §spec validate「可測性」R5：核心與 DOM 分離，注入序列驅動停止邏輯 |
| `findScrollContainer` 5 層 fallback | DOM 操作 | 單元測試（jsdom DOM fixture） | fallback 順序為純結構判定，jsdom 可建構 DOM 樹驗證 |
| `parseLibraryTotal` header 解析 | 純函式 | 單元測試（字串 fixture） | 純文字 regex 解析，無 DOM layout 依賴 |
| `loadedCount` 累積 unique Set 去重 | 演算法核心 | 單元測試（mock 書籍 ID 序列） | §7 D1 行為，注入重複/新增 ID 序列驗證 |
| 涵蓋率回報日誌契約 | 行為驗證 | 單元測試（mock Logger，斷言呼叫參數） | §4 日誌契約，驗證 log 事件名稱與欄位 |
| `extractAllBooks` 整合（呼叫順序、型別不變） | 整合測試 | 單元測試（mock `loadAllBooksLazy` / `waitForBookElements`） | acceptance #1、#11 整合點 |
| renderWaitMs / overallTimeoutMs 計時 | 演算法核心 | 單元測試 + fake timer | 規格 Q3，`jest.useFakeTimers()` 驅動計時 |

### 1.2 Sociable Unit Tests 與 Mock 策略

**Unit = readmoo-adapter 模組**。測試行為（停止邏輯、降級路徑、日誌契約），不測內部結構。

| 依賴 | Mock 策略 | 理由 |
|------|----------|------|
| 注入的捲動函式（`scrollStep`） | Mock（可注入） | 外部世界（DOM 捲動），R5 要求可注入 |
| 注入的量測函式（`measureBooks` / 回傳當輪 DOM book ID 陣列） | Mock（可注入） | 外部世界（DOM 查詢），以注入序列驅動停止邏輯 |
| `Logger` | Mock（spy 呼叫參數） | 外部依賴；驗證日誌契約需斷言呼叫 |
| `MutationObserver` | Mock / fake（控制觸發時機） | 瀏覽器內建；以 fake 控制「新項目渲染」事件 |
| `window` / `document` / `scrollTop` / `scrollHeight` | jsdom 提供 + 手動設定 | `findScrollContainer` DOM 結構測試需真實 DOM 樹 |
| `setTimeout` / `Date.now` 計時 | `jest.useFakeTimers()` | 計時斷言以 fake timer 推進，不依賴真實時鐘 |
| book ID 解析邏輯（adapter 既有 `extractBookId` / `parseBookElement`） | 不 Mock（內層邏輯，使用真實） | §7 D1 沿用既有解析邏輯，屬模組內行為 |

**測試耦合目標**：重構 `loadAllBooksLazy` 內部實作（如改變迴圈寫法）時，本批測試應保持綠燈——測試只斷言「給定注入序列 → 產出 LoadAllBooksResult」的行為，不斷言內部呼叫順序細節（整合測試的呼叫順序斷言除外，該斷言本身即行為契約 acceptance #1）。

### 1.3 計時斷言設計約束（test-assertion-design-rules）

| 規則 | 套用 |
|------|------|
| 規則 1：npm test 主套件禁絕對計時硬門檻 | 本 ticket 所有計時驗證以 fake timer 推進虛擬時間，斷言「`durationMs` 等於注入的虛擬耗時」「逾時於第 N 輪觸發」——斷言對象為 fake timer 控制值，非真實 `Date.now()` 差值。屬規則 1 mock 豁免（mock 控制值不受環境影響），可放主套件 |
| 規則 2：效能測試放 tests/perf/ | 本 ticket 無效能 SLA 測試需求。捲動載入為一次性操作，無「N 輪須於 X ms 內完成」類斷言。**不新增 tests/perf/ 檔案** |
| 規則 3：toBeCloseTo 精度 ≤ 2 | `durationMs` 為整數累加（fake timer 推進量總和），用 `toBe` 精確比對，不涉及浮點 |
| 規則 4：快取驗證 | 不適用（無快取邏輯） |

**fake timer 斷言範例（合規）**：
```javascript
jest.useFakeTimers()
// 注入：每輪 renderWait 推進 800ms 虛擬時間
const result = loadAllBooksLazy({ overallTimeoutMs: 5000, ... })
jest.advanceTimersByTime(5000)
const r = await result
expect(r.stopReason).toBe('timeout')   // 斷言邏輯結果，非真實計時
expect(r.durationMs).toBeGreaterThanOrEqual(5000)  // durationMs 來自 fake timer 累加
```

## 2. 行為鏈式推演

### 2.1 核心行為鏈

```
extractAllBooks 觸發
  │  [前置] adapter 已建立、頁面為 Readmoo library
  ▼
loadAllBooksLazy 啟動
  │  ├─ parseLibraryTotal()  → expectedTotal（可見書目數 | null）
  │  ├─ findScrollContainer() → { container, strategy }
  │  ├─ 記錄起始 scrollTop（D5 還原用）
  │  └─ 首次量測 → loadedCount（unique Set 大小）
  ▼
首次量測達標判定（停止條件 a 於 iteration=0）
  ├─ loadedCount >= expectedTotal → already_complete（iterations=0）─┐
  └─ 否 → 進入捲動迴圈                                                │
       ▼                                                              │
   每輪：scrollStep → renderWait（MutationObserver | renderWaitMs）   │
        → 量測（解析 DOM book ID 加入 Set）→ 捲動有效性探測          │
        → 檢查 4 停止條件                                            │
       ├─ (a) loadedCount >= expectedTotal → reached_total           │
       ├─ (b) 連續 stableRounds 不變      → count_stable             │
       ├─ (c) iterations >= maxIterations → max_iterations           │
       └─ (d) durationMs >= overallTimeoutMs → timeout               │
       ▼                                                              │
   迴圈結束 ◄──────────────────────────────────────────────────────┘
       │  finally: best-effort 還原 scrollTop
       ▼
   輸出 SCROLL_LOAD_COMPLETED (info)
   coverageComplete=false → 額外輸出 COVERAGE_INCOMPLETE (warn)
       ▼
extractAllBooks → waitForBookElements（降級安全網）→ 提取
```

### 2.2 前置條件清單（每個 When 的前提）

| 行為（When） | 前置條件 | 前置驗證斷言設計 |
|-------------|---------|----------------|
| `loadAllBooksLazy` 進入捲動迴圈 | `findScrollContainer` 回傳 container 非 null | 注入的 `findScrollContainer` mock 回傳 `strategy !== 'none'` |
| 停止條件 (a) 可判定 | `expectedTotal` 非 null | 注入 `parseLibraryTotal` 回傳 `total` 為數字 |
| 停止條件 (b) count_stable 觸發 | 已執行 ≥ `stableRounds` 輪 | 注入量測序列長度 ≥ stableRounds 且後段值相同 |
| `MutationObserver` 提早結束輪內等待 | observer 已掛載於 container | mock observer 提供 `triggerMutation()` 觸發函式 |
| 捲動位置還原（D5） | 起始已記錄 scrollTop | 設定 container.scrollTop 初值，斷言結束後還原 |
| `extractAllBooks` 呼叫 `waitForBookElements` | `loadAllBooksLazy` 已 resolve | spy `loadAllBooksLazy` resolve 後才 spy 到 `waitForBookElements` 呼叫 |

### 2.3 四維度行為分支窮舉

| 維度 | 涵蓋場景 | 對應 TG |
|------|---------|---------|
| 正常流程 | 完整載入達標、首批即完整、連續穩定停止 | TG-1（場景 1/2/3） |
| 異常流程 | 容器找不到降級、捲動例外、總數解析失敗 | TG-3、TG-4（場景 6/7/8） |
| 邊界條件 | maxIterations 上限、overallTimeoutMs 逾時、空書庫、可見書目 < 96 | TG-2、TG-5（場景 4/5/9） |
| 中斷操作 | 捲動有效性探測失敗早期降級、單輪 hang 逾時中斷迴圈 | TG-2、TG-6（§4 捲動有效性探測） |

## 3. 測試案例設計

### TG-1 正常流程停止邏輯（對應場景 1/2/3，acceptance #2/#4/#9）

#### TC-1.1 大型書庫完整捲動載入至達標（場景 1）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=928`；注入量測序列每輪累積 unique book ID 數遞增 `[96, 200, 400, 650, 850, 928]`；`maxIterations=30`
- **Then（前置驗證）**：`findScrollContainer` mock 回傳 `strategy='selector'`、container 非 null
- **Then（前置驗證）**：首次量測 `loadedCount=96 < 928`，未觸發 already_complete
- **When**：`loadAllBooksLazy` 執行至累積 unique 數達 928
- **Then**：`stopReason='reached_total'`、`coverageComplete=true`、`loadedCount=928`、`expectedTotal=928`、`missingCount=0`、`iterations>=1`

#### TC-1.2 小型書庫首批即完整略過迴圈（場景 2，acceptance #9）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=50`；首次量測回傳 50 個 unique book ID
- **Then（前置驗證）**：首次量測 `loadedCount=50 >= expectedTotal=50`
- **When**：`loadAllBooksLazy` 執行
- **Then**：`stopReason='already_complete'`、`iterations=0`、`coverageComplete=true`；注入的 `scrollStep` mock **未被呼叫**（驗證略過迴圈）

#### TC-1.3 連續穩定後停止（場景 3）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=928`；量測序列 `[96, 300, 500, 500, 500, 500]`（第 3 輪起不變）；`stableRounds=3`
- **Then（前置驗證）**：序列後 3 輪 `loadedCount` 相同（500），且 `500 < 928`
- **When**：連續 3 輪 `loadedCount` 不變
- **Then**：`stopReason='count_stable'`、`loadedCount=500`、`coverageComplete=false`、`missingCount=428`

#### TC-1.4 reached_total 與 already_complete 語義區分（§7 D3）
- **Given**：兩組注入——(A) 首次即達標、(B) 捲動後達標
- **When**：分別執行
- **Then**：(A) `stopReason='already_complete'` 且 `iterations=0`；(B) `stopReason='reached_total'` 且 `iterations>=1`；兩者 stopReason 不相等（驗證未被合併）

### TG-2 邊界條件停止（對應場景 4/5/9，acceptance #3/#10）

#### TC-2.1 達 maxIterations 上限（場景 4）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=928`；量測序列每輪 +5（持續微增永不達標、永不穩定）；`maxIterations=30`
- **Then（前置驗證）**：每輪 `loadedCount` 皆增加（不觸發 count_stable）、皆 `< 928`（不觸發 reached_total）
- **When**：執行第 30 輪完成
- **Then**：`stopReason='max_iterations'`、`iterations=30`、回傳當下 `loadedCount`、`missingCount>0`

#### TC-2.2 整體逾時（場景 5）— fake timer
- **Given**：`jest.useFakeTimers()`；注入 `parseLibraryTotal=928`；每輪 renderWait 推進 800ms；`overallTimeoutMs=5000`；量測序列持續微增不達標
- **Then（前置驗證）**：累計 `durationMs` 於前幾輪 `< 5000`
- **When**：`jest.advanceTimersByTime` 推進至累計 `>= 5000`
- **Then**：`stopReason='timeout'`、`durationMs>=5000`、回傳當下 `loadedCount`、`missingCount>0`

#### TC-2.3 單輪 hang 由 overallTimeoutMs 兜底（§7 D2，中斷維度）
- **Given**：`jest.useFakeTimers()`；mock `MutationObserver` **永不觸發**；`renderWaitMs=800`、`overallTimeoutMs=5000`
- **Then（前置驗證）**：observer mock 的 `triggerMutation` 未被呼叫（模擬 observer 失效）
- **When**：推進虛擬時間至 5000ms
- **Then**：`stopReason='timeout'`（驗證 maxIterations 無法兜底單輪 hang，overallTimeoutMs 為唯一防線）

#### TC-2.4 空書庫 parseLibraryTotal=0（場景 9 路徑 A）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=0`；首次量測 `loadedCount=0`
- **Then（前置驗證）**：`loadedCount=0 >= expectedTotal=0`
- **When**：`loadAllBooksLazy` 執行
- **Then**：`stopReason='already_complete'`、`iterations=0`；`scrollStep` mock 未被呼叫

#### TC-2.5 空書庫 parseLibraryTotal=null（場景 9 路徑 B）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=null`；量測序列恆為 0 個 book ID
- **Then（前置驗證）**：`expectedTotal=null`，停止條件 (a) 不可判定
- **When**：連續 `stableRounds` 輪 `loadedCount=0` 不變
- **Then**：`stopReason='count_stable'`、`loadedCount=0`、`coverageComplete=false`、`missingCount=0`；不無限捲動

#### TC-2.6 四停止條件獨立性矩陣（acceptance #3）
- **Given**：4 組注入，各組僅讓單一停止條件成立、其餘不成立
- **When**：分別執行
- **Then**：各組 `stopReason` 分別為 `reached_total` / `count_stable` / `max_iterations` / `timeout`；驗證 DOM 永不穩定（量測序列無限微增）時仍由 (c)(d) 保證終止——測試本身在有限步內 resolve 即證明終止性

### TG-3 總數解析失敗路徑（對應場景 6，acceptance #8）

#### TC-3.1 expectedTotal=null 改依穩定條件停止（場景 6）
- **Given**：注入 `parseLibraryTotal` 回傳 `total=null`；量測序列 `[96, 300, 500, 500, 500]`；`stableRounds=3`
- **Then（前置驗證）**：`expectedTotal=null`、`coverageComplete` 恆為 false
- **When**：連續 3 輪穩定
- **Then**：`stopReason='count_stable'`、`expectedTotal=null`、`coverageComplete=false`、`missingCount=0`；捲動迴圈確實執行（`scrollStep` mock 被呼叫 ≥1 次）

### TG-4 降級與例外路徑（對應場景 7/8，acceptance #5/#7）

#### TC-4.1 捲動容器找不到降級（場景 7）
- **Given**：注入 `findScrollContainer` 回傳 `{ container: null, strategy: 'none' }`；當下 DOM 解析出 96 個 book ID
- **Then（前置驗證）**：`strategy='none'`
- **When**：`loadAllBooksLazy` 執行
- **Then**：`stopReason='container_not_found'`、`iterations=0`、`loadedCount=96`（當下 DOM 解析數）；`scrollStep` mock 未被呼叫；輸出 `warn` 級 `SCROLL_CONTAINER_NOT_FOUND` log；**Promise resolve 不 reject**

#### TC-4.2 捲動過程拋出例外（場景 8）
- **Given**：注入 `scrollStep` mock 於第 3 輪 `throw new Error('DOM detached')`；前 2 輪正常累積至 400 unique ID
- **Then（前置驗證）**：前 2 輪 `loadedCount` 正常累積
- **When**：第 3 輪 `scrollStep` 拋例外，於 `loadAllBooksLazy` 內被捕捉
- **Then**：`stopReason='error'`、`loadedCount=400`（拋例外前已累積）；輸出 `error` 級 log（含錯誤訊息與元件名稱 `ReadmooAdapter`）；**Promise resolve 不 reject**

#### TC-4.3 量測函式拋例外亦被捕捉（場景 8 變體）
- **Given**：注入 `measureBooks` mock 於某輪拋例外
- **When**：例外發生
- **Then**：`stopReason='error'`、回傳已累積 `loadedCount`；resolve 不 reject

#### TC-4.4 loadAllBooksLazy 永遠 resolve（acceptance #5 總驗證）
- **Given**：遍歷所有 stopReason 路徑（7 枚舉）的注入組合
- **When**：分別執行
- **Then**：每組皆 resolve，無任一組 reject（以 `await expect(promise).resolves` 斷言）

### TG-5 loadedCount 去重與日誌契約（對應 §7 D1，acceptance #2/#4/#6）

#### TC-5.1 累積 unique book ID Set 去重（§7 D1，acceptance #6）
- **Given**：注入量測序列每輪回傳的 book ID 陣列含跨輪重複——輪1 `[a,b,c]`、輪2 `[b,c,d,e]`、輪3 `[d,e,f]`
- **Then（前置驗證）**：跨輪有重複 ID（b、c、d、e）
- **When**：三輪量測加入累積 Set
- **Then**：`loadedCount=6`（unique：a,b,c,d,e,f），非 `3+4+3=10`；重複 ID 不灌大 count

#### TC-5.2 同輪 DOM 內重複 ID 去重
- **Given**：某輪量測回傳含同輪內重複 ID `[a, a, b]`
- **When**：加入 Set
- **Then**：該輪貢獻 2 個 unique（a、b）

#### TC-5.3 累積 Set 對 DOM 回收免疫（§7 D1 robustness）
- **Given**：模擬回收式虛擬捲動——量測序列 輪1 `[a,b,c]`、輪2 `[d,e,f]`（前批 a,b,c 已被 DOM 回收，當輪 DOM 僅 d,e,f）
- **Then（前置驗證）**：輪2 量測不含輪1 的 ID
- **When**：兩輪加入累積 Set
- **Then**：`loadedCount=6`（累積保留 a-f），非「當下 DOM count=3」；驗證累積 Set 對回收式免疫

#### TC-5.4 SCROLL_LOAD_COMPLETED 日誌契約（acceptance #2）
- **Given**：mock Logger spy；任一完整執行（如 TC-1.1）
- **When**：`loadAllBooksLazy` 結束
- **Then**：Logger 收到 `info` 級 `SCROLL_LOAD_COMPLETED`，payload 含全部 6 欄位 `loadedCount` / `expectedTotal` / `coverageComplete` / `stopReason` / `iterations` / `durationMs`

#### TC-5.5 COVERAGE_INCOMPLETE 日誌契約（acceptance #2/#3）
- **Given**：mock Logger spy；`coverageComplete=false` 的執行（如 TC-1.3 count_stable 未達標）
- **When**：`loadAllBooksLazy` 結束
- **Then**：Logger 額外收到 `warn` 級 `COVERAGE_INCOMPLETE`，payload 含 `missingCount` 與停止原因可讀說明

#### TC-5.6 coverageComplete=true 時不輸出 COVERAGE_INCOMPLETE
- **Given**：mock Logger spy；`coverageComplete=true` 的執行（如 TC-1.1）
- **When**：執行結束
- **Then**：Logger **未**收到 `COVERAGE_INCOMPLETE`（驗證只在不一致時輸出）

#### TC-5.7 expectedTotal=null 時 coverageComplete 恆 false 且輸出 COVERAGE_INCOMPLETE
- **Given**：`parseLibraryTotal` 回傳 null
- **When**：執行結束
- **Then**：`coverageComplete=false`、輸出 `COVERAGE_INCOMPLETE`（missingCount=0，原因為「無法取得可見書目數」）

### TG-6 findScrollContainer 5 層 fallback 與捲動有效性探測（對應場景 7、§4/§5）

#### TC-6.1 優先級 1 selector 命中
- **Given**：jsdom DOM 含 Readmoo library 捲動容器 class
- **When**：`findScrollContainer` 執行
- **Then**：回傳 `strategy='selector'`、container 為該元素

#### TC-6.2 優先級 2 load-more-button 命中
- **Given**：DOM 無 selector 容器，但有「更多...」按鈕
- **When**：執行
- **Then**：回傳 `strategy='load-more-button'`

#### TC-6.3 優先級 3 scrollable-ancestor 命中

## 4. 測試環境設置規劃

### 4.1 Mock 物件清單

| Mock | 用途 | 設置方式 |
|------|------|---------|
| `scrollStep`（注入捲動函式） | 控制每輪捲動行為、可拋例外 | `jest.fn()`，依 TC 設定回傳/throw |
| `measureBooks`（注入量測函式） | 控制每輪回傳的 DOM book ID 陣列 | `jest.fn()` + `mockReturnValueOnce` 序列 |
| `findScrollContainer` | 控制容器辨識結果 | `jest.spyOn(adapter, 'findScrollContainer')` |
| `parseLibraryTotal` | 控制 expectedTotal | `jest.spyOn(adapter, 'parseLibraryTotal')` |
| `Logger`（spy） | 驗證日誌契約 | mock Logger，spy `info` / `warn` / `error` / `debug` |
| `MutationObserver` | 控制「新項目渲染」事件 | fake class，提供 `triggerMutation()` |
| `waitForBookElements` | 整合測試降級安全網驗證 | `jest.spyOn(adapter, 'waitForBookElements')` |
| `reportProgress` | book-data-extractor 進度回報驗證 | `jest.fn()` |
| fake timer | 計時驗證 | `jest.useFakeTimers()` / `advanceTimersByTime` |

### 4.2 測試資料

| 資料 | 內容 |
|------|------|
| 量測序列 fixture | 預定義 book ID 陣列序列（遞增 / 穩定 / 微增 / 含重複 / 回收式） |
| header 文字 fixture | 標準格式、無子句、0 本、格式變更、異常負數 5 組字串 |
| DOM fixture | 5 種捲動容器結構（selector 容器 / load-more 按鈕 / 可捲動祖先 / 純 document / 空） |
| scrollOptions 預設 | `{ maxIterations:30, stableRounds:3, renderWaitMs:800, overallTimeoutMs:60000 }`；各 TC 視需要覆寫小值加速 |

### 4.3 測試清理與隔離

| 項目 | 方式 |
|------|------|
| 模組重載 | `beforeEach` `jest.resetModules()`（沿用既有檔案慣例） |
| timer 還原 | 用 fake timer 的 TC 於 `afterEach` `jest.useRealTimers()` |
| mock 清理 | `afterEach` `jest.clearAllMocks()` |
| DOM 清理 | jsdom DOM fixture 於 `afterEach` 重設 `document.body.innerHTML=''` |
| 測試獨立性 | 每 TC 自備注入序列，不共享 mutable 狀態；量測序列以 `mockReturnValueOnce` 鏈，不跨 TC 殘留 |
