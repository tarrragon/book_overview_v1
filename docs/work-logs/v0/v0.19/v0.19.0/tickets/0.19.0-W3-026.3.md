### 實作策略：mirror sibling W3-026.1 pattern

依 parent W3-026 ANA Solution 表格指引（S3）「有背景任務時跳過或降級為 INFO」。選擇「跳過」（return None）以與既有 in_progress ticket 防護一致。設計為 additive：欄位不存在 / 空 list / 非 list / 非 dict 全部 fallback 到現行兩道防護邏輯（stop flag + in_progress ticket）。

#### 三道防護順序（detect_sync_drift 內）

依現行為：(1) `_is_blocked_this_session`、(2) `_has_in_progress_ticket`、(3) [本任務新增] `_has_background_agents`、(4) 主檢查邏輯。

順序選擇：放在 in_progress 偵測「之後」，因為：
1. 三道防護皆 short-circuit return None，順序對性能無實質差異
2. 既有兩道為「無/弱依賴外部訊號」的本地推斷；新增的 background_tasks 是 v2.1.145 stdin 直接訊號
3. 放最後可讓 fixture 維持向後相容（`hook_mod` fixture 已 monkeypatch 既有兩道；新測試需另加 monkeypatch）

#### 函式新增（mirror sibling line 532-562）

```python
def _has_background_agents(input_data: dict, logger) -> bool:
    """
    W3-026.3：mirror sibling W3-026.1 has_background_agents 的弱依賴策略。
    只判斷 list 非空，不依賴 task object 內部 schema。
    SOT: .claude/skills/ticket/hooks/handoff-auto-resume-stop-hook.py:has_background_agents
    任一處更新需同步另一處（ARCH-020 同構雙寫）。
    """
    if not isinstance(input_data, dict):
        return False
    bg_tasks = input_data.get("background_tasks")
    if not isinstance(bg_tasks, list):
        return False
    has_active = len(bg_tasks) > 0
    if has_active:
        logger.info(
            f"input_data.background_tasks 非空（{len(bg_tasks)} 項），"
            f"跳過 worklog-handoff sync check"
        )
    return has_active
```

#### main() stdin 改造（mirror sibling line 906-922）

現有 main()（line 493-522）已讀 stdin 但只取 `session_start_timestamp`。改造方向：
1. 加入 isatty 防護（避免互動環境誤讀）
2. 加入 `if not isinstance(parsed, dict): event = {}` non-dict fallback
3. 保留現有解析失敗 fallback
4. 將 event 透傳給 detect_sync_drift 作為 input_data

實作示意：
```python
def main():
    logger = setup_hook_logging("stop-worklog-handoff-sync-check")
    try:
        event: dict = {}
        if not sys.stdin.isatty():
            try:
                parsed = json.loads(sys.stdin.read() or "{}")
                if isinstance(parsed, dict):
                    event = parsed
                else:
                    logger.warning(
                        f"stdin JSON 非 dict 型別（{type(parsed).__name__}），fallback"
                    )
            except (json.JSONDecodeError, ValueError) as e:
                logger.warning(f"解析 stdin 失敗，fallback: {e}")

        session_start = float(event.get("session_start_timestamp", 0) or 0)
        project_root = get_project_root()
        warning = detect_sync_drift(project_root, session_start, logger, input_data=event)
        # ... 原本輸出邏輯不變
```
