## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-061 ANA 實機驗證確認 Readmoo library 頁 DOM 不提供作者欄位（96 樣本所有 `[class*="author"]` selector 0 命中）。屬 **source data limitation**，非 extractor 邏輯遺漏。當前 overview UI 連 author 欄位都未顯示（COLUMNS=7：select/cover/title/source/progress/status/tag），CSV/JSON 匯出僅保留 `authors:[]` 空值但無註記，造成用戶誤判 extractor 漏抓。

### 影響範圍

- `src/overview/overview.html`：書籍表格未含 author 欄位
- `src/overview/overview-page-controller.js`：`createBookRow` / `_formatBookRowData` / `CONSTANTS.TABLE.COLUMNS` 需擴充
- `src/overview/overview.css`：可能需 author cell 樣式
- `src/export/book-data-exporter.js`：`_exportToCSVv2` / `exportToJSON` 需加 source limitation 註記能力
- 新增測試：`tests/unit/overview/` 與 `tests/unit/export/` 對應 authors:[] 場景

### 相關 Error Pattern

無直接相關 PC。本任務屬規格落地，非錯誤防護。

### WRAP 三問

**W（Widen）**：列出三個方案
1. UI 新增 author 欄位顯示 placeholder + CSV 前置註解行 + JSON metadata 註記（採用）
2. 不改 table，只 tooltip 提示（隱藏，違反「明示」原則）
3. 只改 CSV/JSON 不動 UI（acceptance 第一條未滿足）

**A（Attain distance）**：機會成本
- table COLUMNS 從 7 → 8，需檢視 empty row colspan、CSS、其他相依測試
- 範圍鎖在 src/overview/ + src/export/ 與對應 tests，避免擴散

**P（Prepare to be wrong）**：最可能失敗
- CSV 前置 `#` 註解行不符 RFC 4180，部分 reader 報錯
- 對策：以 `metadataComments` 為可選參數（預設關閉），預設仍為純 CSV；overview 預設啟用註解輸出。並在 JSON metadata 新增 `sourceLimitations` 欄位作雙保險

### 設計決策

**UI 設計**：
- table 加入「作者」欄位（位置：書名 → 作者 → 書城來源），COLUMNS 從 7 增為 8
- authors=[] / undefined / null 顯示 placeholder「— 待補」+ tooltip「Readmoo 來源頁不提供作者欄位，可手動編輯（v0.20.0 將支援標籤化編輯）」
- authors 含值時以「、」分隔多作者
- authorCell 標註 data-source-limited="authors" 供樣式或未來編輯流程使用

**CSV 設計（雙保險）**：
- BookDataExporter._exportToCSVv2 新增 includeSourceLimitations 參數（預設 false，保持 RFC 4180 純度）
- overview-page-controller.handleExportCSVv2 顯式傳 includeSourceLimitations: true
- 啟用時前置一行 `# source-limited: authors (Readmoo library 頁 DOM 不提供作者欄位，空白為來源未提供，可手動編輯)`
- 註解列在 headers 之前；多數 CSV reader（Excel/LibreOffice/pandas）容忍 # 註解列
- 純函式庫消費者（如 v1 相容測試）使用預設值不受影響

**JSON 設計**：
- metadata.sourceLimitations 為陣列，每項描述一個欄位的來源限制
- 結構：`{ field, reason, recommendation, sourceTicket }`
- 預設啟用（純擴充欄位不破壞既有 consumer）；preset 含 authors 時必輸出

### 決策對應 P 預想

WRAP P（最可能失敗）= CSV 註解列違反 RFC 4180 的相容性風險。實際採取的對策：
- 預設 false（純函式庫使用者不受影響）
- caller 顯式啟用（overview 等需要明示 source limitation 的場景）
- JSON metadata.sourceLimitations 作為「結構化雙保險」：即使有讀者剝離 CSV 註解列，JSON 仍保留資訊

### 不在本 ticket 範圍

- 手動補作者 UI 編輯流程 → v0.20.0 tag 管理整合
- 探索 Readmoo reader API 取得作者 metadata → 後續 ANA spike
- 第三方 metadata enrichment（Google Books / Open Library）→ v2.0+ 多書城支援後
