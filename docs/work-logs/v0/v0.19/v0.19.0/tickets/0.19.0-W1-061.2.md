## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-061 ANA 實機驗證確認 Readmoo library 頁 DOM 不提供作者欄位（96 樣本所有 `[class*="author"]` selector 0 命中）。屬 **source data limitation**，非 extractor 邏輯遺漏。當前 overview UI 連 author 欄位都未顯示（COLUMNS=7：select/cover/title/source/progress/status/tag），CSV/JSON 匯出僅保留 `authors:[]` 空值但無註記，造成用戶誤判 extractor 漏抓。

### 影響範圍

- `src/overview/overview.html`：書籍表格未含 author 欄位
- `src/overview/overview-page-controller.js`：`createBookRow` / `_formatBookRowData` / `CONSTANTS.TABLE.COLUMNS` 需擴充
- `src/overview/overview.css`：可能需 author cell 樣式
- `src/export/book-data-exporter.js`：`_exportToCSVv2` / `exportToJSON` 需加 source limitation 註記能力
- 新增測試：`tests/unit/overview/` 與 `tests/unit/export/` 對應 authors:[] 場景

### 相關 Error Pattern

無直接相關 PC。本任務屬規格落地，非錯誤防護。

### WRAP 三問

**W（Widen）**：列出三個方案
1. UI 新增 author 欄位顯示 placeholder + CSV 前置註解行 + JSON metadata 註記（採用）
2. 不改 table，只 tooltip 提示（隱藏，違反「明示」原則）
3. 只改 CSV/JSON 不動 UI（acceptance 第一條未滿足）

**A（Attain distance）**：機會成本
- table COLUMNS 從 7 → 8，需檢視 empty row colspan、CSS、其他相依測試
- 範圍鎖在 src/overview/ + src/export/ 與對應 tests，避免擴散

**P（Prepare to be wrong）**：最可能失敗
- CSV 前置 `#` 註解行不符 RFC 4180，部分 reader 報錯
- 對策：以 `metadataComments` 為可選參數（預設關閉），預設仍為純 CSV；overview 預設啟用註解輸出。並在 JSON metadata 新增 `sourceLimitations` 欄位作雙保險
