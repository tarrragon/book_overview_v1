### Phase 3b-B 實作摘要（thyme-extension-engineer，2026-06-02）

新建 `src/export/book-interchange-v1-adapter.js`（純函式核心），依 Phase 3a 虛擬碼 ADR-1~5 實作 5 函式：

| 函式 | 方向 | 說明 |
|------|------|------|
| `mapV1BookToCanonical` | write | V1 內部 v2 → canonical；固定欄位物件化 + 包單元素 tag；H3/ADR-5 由 `_passthrough` 重建多值/carry |
| `mapCanonicalToV1Book` | read | canonical → V1 內部 v2；多值收斂（首元素入固定欄位，其餘+完整陣列入 `_passthrough`，C1） |
| `normalizeReadingStatusToCanonical` | 查表 | §7 六態 V1→canonical，純查表回 `name｜null`，無副作用（ADR-3） |
| `normalizeReadingStatusFromCanonical` | 查表 | §7 逆向，回 `v1Status｜null` |
| `buildTagTree` | 聚合 | 多 book ccl/custom tag node → 扁平樹（path 推 parentId + 同 id 去重 + locked 標記，§6，ADR-4） |

### 關鍵設計落地

- **多值收斂（§A / ADR-5）**：`convergeMultiValue`/`convergeIsbn`/`convergeCover`/`convergeProgress` 首元素入固定欄位、`length>1` 存完整陣列至 `_passthrough.tags.*`（TD-4 採推薦最小體積策略）；write 端 `rebuildFromPassthrough` 優先讀完整陣列重建，統一 V1 出發（退化單元素）與 canonical 出發兩路徑（H3）。
- **穩定 id**：tag id 由 name 衍生（`genTagId`）、buildTagTree 中間層由 path 前綴衍生（`deriveStableId`），避免 timestamp race flaky（呼應 W1-057/W1-078 同源原則）；本模組不重用 `convertV1CategoryToTag`，未破壞其 timestamp 邏輯。
- **錯誤契約**：`mapV1BookToCanonical`/`mapCanonicalToV1Book` 缺 id/title throw `BookValidationError(book, [{field,message}])`（對齊既有 `convertV1ToV2Book`）；detector/normalize/buildTagTree 不 throw。

### 設計補強（3b 實作期發現，對齊 C1）

D-U17（canonical 出發 round-trip）要求 author tag node 的 **id** 無損還原。設計 §A 原註「author 無需 passthrough」僅就「authors[] 不丟 name」成立，未涵蓋 tag id。因 V1 `authors[]` 僅承載 name，read 時將原始 author tag 陣列存入 `_passthrough.tags.author`，write 端優先讀以重建 id，達成 D-U17 無損（C1）。對應調整 C-U17 測試斷言：主行為（authors[] 全保留）不變，passthrough 斷言由「author 不入 passthrough」改為「author tag 陣列入 passthrough 以保 id」。屬 quality-baseline 規則 3（設計問題立即修正），C1 雙向無損為較高優先契約。

### 不在本 ticket 範圍

detector 四來源（3b-A，`format-version-detector.js`）、匯出 write 接線（.2.1）、ContentParser/app-legacy 接線（.2.2）。本 foundation 僅產出 adapter 純函式 + 測試。
