## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

Overview 匯出按鈕（`overview-page-controller.js` 原 L243/L249）的 click handler 直接呼叫 `this.bookExporter.handleExportCSV()` / `handleExportJSON()`，而 `bookExporter` 是 v1 簡易匯出器 `src/overview/book-exporter.js` 的實例。

`book-exporter.js` 完全不知道 Interchange Format v2 規格：
- JSON 手動組裝 `{ books: [...] }`，每本書僅 `{id,title,progress,status,cover,tags}`，無 metadata/tagCategories/tags 頂層，書籍欄位用 `status`（非 `readingStatus`）、`tags`（非 `tagIds`）。
- CSV headers 為中文混英文 `書名,書城來源,進度,狀態,封面URL,id,authors,tagIds`，無 `readingStatus`、無衍生欄位 `tagNames`/`tagCategories`。

同時 `src/export/book-data-exporter.js` 已有完整 v2 實作（`_exportToJSONv2` / `_exportToCSVv2`，含 EXTENDED_V2/COMPLETE_V2 preset、metadata/tagCategories/tags 區段、衍生欄位 resolve），但從未接線到 overview UI。

根因：按鈕接線指向 v1 簡易匯出器而非已實作的 v2 完整匯出器。

### 影響範圍

- `src/overview/overview-page-controller.js` — 匯出按鈕接線（本 ticket 修改）
- `src/overview/book-exporter.js` — v1 簡易匯出器（本 ticket 不修改；仍由 copyText 按鈕使用，且既有測試 spy 其委派方法）
- `src/export/book-data-exporter.js` — v2 匯出器（本 ticket 不修改，既有 v2 實作直接複用）

### 相關 Error Pattern

無直接對應 error-pattern。屬「設計選擇 + 規格未對齊」共振（W1-042 ANA 已記載 10 項缺口）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

將 overview 匯出 CSV/JSON 按鈕的 click handler 從 v1 `bookExporter` 改接至 `BookDataExporter` 的 v2 路徑。修改僅在 `src/overview/overview-page-controller.js`，`book-data-exporter.js` 既有 v2 實作直接複用，未修改。

### 變更內容（overview-page-controller.js）

1. 新增 import：`BookDataExporter`（`src/export/book-data-exporter`，default export）、`TagStorageAdapter`（`src/storage/adapters/tag-storage-adapter`）。
2. 新增 `CONSTANTS.EXPORT_V2`：FORMAT_VERSION `2.0.0`、FIELD_PRESET `COMPLETE_V2`（涵蓋全部 v2 書籍欄位，最大化規格合規）、MIME、檔名前綴。
3. 按鈕 click handler 改接：`exportCSVBtn` → `handleExportCSVv2()`、`exportJSONBtn` → `handleExportJSONv2()`。
4. 新增方法：
   - `handleExportJSONv2()` / `handleExportCSVv2()`（async）：取書籍 → 載 tag 資料 → 呼叫 `BookDataExporter.exportToJSON/exportToCSV({ formatVersion:'2.0.0', fieldPreset:'COMPLETE_V2', tags, tagCategories })` → 觸發下載。匯出失敗以 `console.error` + `showError` 雙通道回報（quality-baseline 規則 4）。
   - `_getBooksForExport()`：selection-aware，與 BookExporter 的 getFilteredBooks DI 行為一致（有選取僅匯選取項，否則全量 filteredBooks）。
   - `_loadTagData()`：透過 `TagStorageAdapter.getAllTags()` / `getAllTagCategories()` 讀取 tags/tagCategories；非 Chrome 環境退回空陣列。
   - `_triggerExportDownload()`：以 `this.document` 建立隱藏 anchor 觸發下載（不用 `BookDataExporter.downloadFile`，因其依賴 `global.document`）。

### 不在本 ticket 範圍

- `book-exporter.js` 未修改：`copyTextBtn` 仍使用 `bookExporter.handleCopyText()`，既有測試 spy 其委派方法。v1 匯出方法清理屬 W1-042 ANA spawn 規劃的可選 IMP（W1-042.2），不在本 ticket。
- Controller 既有 `handleExportCSV/handleExportJSON/generateCSVContent` 等委派方法保留不動（向後相容，既有測試直接呼叫）。
- 既有 2 個 pre-existing lint 錯誤（overview-page-controller.js no-multiple-empty-lines）與 `ErrorCodes` unused warning：經 git stash 驗證為改動前既存，且位於本 ticket 範圍外的私有 helper 區塊，依 ticket 約束「僅改 src/overview/、src/export/ 匯出相關」不予修改。

### 設計決策

- 選 `COMPLETE_V2` 而非 `EXTENDED_V2`：COMPLETE_V2 含 readingStatus/cover/isManualStatus/extractedAt/updatedAt/source 全欄位，最大化 Interchange Format v2 合規（UC-03 acceptance 要求完整 v2）。
- 自建 `_triggerExportDownload` 而非用 `BookDataExporter.downloadFile`：後者用 `global.document`/`global.URL`，在 overview 頁面 `window.document` 才是可靠來源，與既有 `book-exporter.js._triggerFileDownload` 一致。
- `_loadTagData` 在非 Chrome 環境退回空陣列：匯出仍可進行，書籍 tagIds 保留但 tagNames/tagCategories 衍生欄位為空（測試環境與優雅降級）。
