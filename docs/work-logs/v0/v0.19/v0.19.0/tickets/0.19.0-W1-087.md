### 問題根因

`diagnostic-module-enhancement.test.js` 行 47 含一個 `toBeCloseTo(initTime, 0)` 斷言，比較兩個獨立的 `performance.now()` 差值：

1. **測試端** (行 35/44)：`const initTime = performance.now() - startTime`（包含 `initializeWithPerformanceTracking` 呼叫前後的完整區間）
2. **模組端** (src 行 410)：`const initTime = performance.now() - this._performanceStartTime`（僅包含 `initialize()` 呼叫後的區間）

兩者量測區間不完全重疊（測試端包含函式進入到 `_performanceStartTime` 賦值之間的額外時間），在高負載環境下差值可能超過 ±0.5ms（precision 0 的容忍範圍），導致偶發失敗。

**分類**：依 test-assertion-design-rules.md 規則 1，此斷言屬「真實計時量測比較」（兩個獨立 `performance.now()` 差值），非 mock 固定回傳值，不適用豁免。
