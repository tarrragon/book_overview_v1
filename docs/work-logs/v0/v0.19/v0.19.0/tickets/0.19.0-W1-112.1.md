## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-108 PM 前台分析發現 Logger constructor 為兩參數 `(name, level)` 並 hardcoded `this.messages = GlobalMessages`，但 popup.js 等 3 個 caller 以 `new Logger(name, level, localMessages)` 傳第三參數，JS 靜默 ignore。W1-106 移除 GlobalMessages 5 個 popup key 後，local dict 機制無法生效，預期 popup runtime 回歸 [Missing: KEY]。本 ticket 為實機驗證實驗。

### 影響範圍

- src/popup/popup.js（讀取 5 個 popup-specific messageKey）
- src/core/logging/Logger.js（constructor 兩參數規格）
- src/core/messages/MessageDictionary.js（W1-105 修復的 union signature constructor）
- build/development/（chrome-devtools-mcp 載入目標）
- 受影響的 5 個 key：POPUP_INTERFACE_LOADED / POPUP_SCRIPT_LOADED / POPUP_INIT_START / POPUP_INIT_COMPLETE / INITIALIZATION_COMPLETE

### 相關 Error Pattern

- 「測試綠燈但 CE runtime 崩潰」（MEMORY 條目）：W1-105 41/41 測試通過但 runtime 假設不成立
- 「Logger 雙模式」相關 v0.15.0~v0.15.2 經驗：CE 環境（jsdom 測試 vs 真實 Chrome）差異常見
