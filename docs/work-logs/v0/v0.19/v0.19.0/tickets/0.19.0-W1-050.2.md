### 問題根因

W1-050 ANA 在 src/ 全面盤點 CE 環境不相容 anti-pattern 後，發現 P7-P16 共 13 處 `process.env.NODE_ENV` 為致命級違規：

- Chrome Extension runtime（SW / content script / popup / overview）無 Node.js `process` 物件
- esbuild 配置 `scripts/build.js` 未配置 `define`，bundle 後 `process.env.NODE_ENV` 保留原樣
- 任何 entry point 依賴鏈中的程式碼觸及 `process.env.NODE_ENV` 即會 `ReferenceError: process is not defined`，整個 CE runtime 崩潰

### 影響範圍

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

在 `scripts/build.js` 的 `bundleEntryPoints()` 函式內的 `esbuild.build()` 配置中加入 `define` 選項：

```javascript
define: {
  'process.env.NODE_ENV': JSON.stringify(MODE)
}
```

其中 `MODE` 是檔案頂層既有的常數（依 `--prod` flag 決定為 `'development'` 或 `'production'`）。esbuild 在 bundle 階段會將所有 `process.env.NODE_ENV` 字面替換為 JSON 字串常量（如 `"development"` 或 `"production"`），CE runtime 執行期不再觸及 `process` 物件。

### 設計決策

| 決策點 | 選擇 | 理由 |
|--------|------|------|
| define 位置 | esbuild 配置 | 單點修復，13 處 process.env.NODE_ENV 自動解決，無需逐一修改 src/ |
| mode 來源 | 既有 MODE 變數 | 已有 `process.argv.includes('--prod')` 判斷邏輯，重用避免重複 |
| 值序列化 | `JSON.stringify(MODE)` | esbuild define 文件指定的標準寫法，確保字串常量正確帶引號 |
| 條件語意 | 保留 | `if (process.env.NODE_ENV === 'production')` 等條件在 bundle 後變成 `if ("development" === 'production')`，esbuild 進一步做 dead code elimination，語意完整保留 |
