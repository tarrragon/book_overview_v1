### 修改策略

**Phase 1：hook 移除 light enum**

- 找出 `light` enum 定義位置（如 `evaluation_levels = ['light', 'standard', 'heavy']`）
- 移除 `light` 保留 `['standard', 'heavy']`
- 驗證邏輯：若有 `if level == 'light':` 分支，整段移除
- 錯誤訊息：若有「light/standard/heavy」描述改為「standard/heavy」

**Phase 2：規則文件移除 light 級別描述**

- `.claude/pm-rules/proposal-evaluation-gate.md` 11 處 light 出現點逐項判斷：

  - 內文「light/standard/heavy」三分制描述 → 改為二分制
  - light 觸發條件描述 → 移除
  - 範例中 light 引用 → 更新或移除
