## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### WRAP 分析：分類模型決策

### W — Widen options（擴增選項）

| 方案 | 描述 |
|------|------|
| A. everything-as-tags | 採 PROP-007 完整：author/publisher/isbn/readingStatus/importance 全變 tag 類別，CCL 為 is_locked tag |
| B. 結構化 classification | 採現有 D2=C：固定欄位 + 單一 classification 物件（tagIds + importanceLevel/category/genre） |
| C. 折衷（結構化外殼 + 正規化 tag 內裡 + CCL first-class） | 固定核心欄位 + classification 物件，內含「結構化分類欄位（importanceLevel/genre/ccl）+ 正規化 tag 圖（tagIds 指向 root tag pool，pool 支援 system categories 與 custom）」 |

### R — Reality-test（用證據檢驗假設）

| 假設 | 檢驗結果 |
|------|---------|
| PROP-007 是 APP 現行方向 | **否證**：APP 完全未實作 PROP-007，現況 fixed-field（重現實驗已證） |
| everything-as-tags 全面較優 | 部分成立：多版本/多平台書的 author/isbn/platform 多值有真實價值；但 readingStatus/importance 本是 enum，tag 化（單選 tag）反增複雜度 |
| CCL 必須是 tag pool 成員 | 否證：CCL 是 is_locked 受控階層分類，PROP-007 自己也特殊化（is_locked，獨立於 custom）；作為結構化欄位 classification.ccl={code,path,locked} 語意更清晰、Extension 唯讀展示更易 |
| C2（Ext 唯讀展示 APP 分類） | 任何方案可成立；結構化 classification 對 Ext「直接讀 classification.ccl.path」最易 |
| C1（無損） | 任何方案 + pass-through 皆可達成 |

### A — Attain distance（拉開距離看長期）

| 里程碑 | 分類需求 |
|--------|---------|
| v1.0 內測 | 核心 = Ext 自身 round-trip + 基本跨專案匯入匯出；不需 everything-as-tags 全部複雜性 |
| v0.20.x（V1） | 中文圖書分類法 + Tag 管理 → CCL 需 first-class |
| v2.0（兩端） | 多書城 + Drive 同步 → 多值 author/isbn/platform 需求才真正浮現 |

「大石頭先放」：分類模型是大石頭，但「立即全採 everything-as-tags」非唯一大石頭方案。可設計**可擴展 canonical**：v1.0 用結構化核心 + 正規化 tag pool，為 v2.0 multi-value tag 預留擴展（加 system tag categories 是擴展非重構）。

### P — Prepare to be wrong（可逆性與絆腳索）

- formatVersion 版本協商 + pass-through，保護未來模型演進（含全面 tag 化）向後相容
- canonical 標 draft，待兩端 adapter 實作驗證後升 approved
- 絆腳索：v2.0 多書城開發若發現 author/isbn 多值需求頻繁，升版 canonical 將其 tag 化（pass-through 已保護舊資料）

### 決策結論：採方案 C（折衷）

**理由**（依證據排序）：
1. 貼合兩端現況（APP fixed-field + Ext 結構化 classification 都更接近 C），降低 v1.0 落地成本，不需兩端立即大改為 everything-as-tags
2. CCL first-class（classification.ccl={code,path,locked}）滿足 V1 v0.20 里程碑，比 tag pool 成員更易表達「受控階層 + 鎖定」+ Ext 唯讀展示
3. 保留 everything-as-tags 核心優點：正規化 tag pool（root tags[] + tagCategories[]）支援自訂 tag tree + 系統 tag；多值欄位（authors[]/identifiers）以陣列承載，未來可平滑 tag 化
4. 可擴展性：formatVersion + pass-through 為 v2.0 全面 tag 化預留路徑，現在不過度設計
5. 吸收 PROP-007 具體改進：多尺寸 cover、progress 物件、cross_platform_id/data_fingerprint、authors 物件陣列
