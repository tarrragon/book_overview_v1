### 問題根因（具體位置）
`.claude/hooks/post-test-hook.py` line 98：

```python
ANALYZER_WARNING_PATTERNS = [
    (r"info\s*-.*?unused", "未使用的警告"),
    (r"warning\s*-", "lint 警告"),            # <-- 過寬
    (r"deprecated\s+(?:function|class|method)", "已棄用 API"),
]
```

`r"warning\s*-"` 在 jest 測試輸出（console.warn 訊息含 `warning - ...` 字串、測試名稱含 "warning" 等）情境下會誤報。
W1-048.8 實證：npm test 4962 passed / 0 failed + eslint exit 0 完全無警告，仍誤報 lint 警告 3 次。

### 影響範圍
- 檔案：`.claude/hooks/post-test-hook.py`（line 96-100 + 相關 _detect_errors）
- 測試：`.claude/hooks/tests/test_post_test_hook.py`（新增 jest console.warn / eslint warning 雙樣態驗證）
- 影響者：所有派發 npm test / eslint 的 agent（thyme / parsley 等）

### 簡化 WRAP 三問

**W（Widen）— 候選方案**：
1. 縮窄 regex 為 eslint stylish formatter 完整格式（`^\s*\d+:\d+\s+warning\s+`，需 row:col 起頭）
2. 拆 patterns 為 jest_patterns + eslint_patterns 分組，按 command 路由
3. 完全移除 lint warning detection（過於 noisy）

選方案 1：最小改動 + 仍保留 lint warning 偵測能力。方案 2 過度設計，方案 3 失去 lint warning 訊號。

**A（Attain distance）— 機會成本**：
P2 hook 修復佔用 thyme-python-developer ~1 派發回合。v0.19.x 主線是 E2E + 打包，本 ticket 不擋內測；現處理可解除 thyme 系列任務的 hook 噪音。

**P（Prepare to be wrong）— 失敗預想**：
最可能失敗：jest console.warn 樣態多樣（含 `\n  warning - ...`、indent 變化），新 regex 仍可能漏。
防護：測試樣本至少含 3 種 jest console.warn 樣態（一般 / 縮排 / 跨多行）+ 2 種真實 eslint 樣態（含 / 不含 rule name）。

### 預期改動
- post-test-hook.py: 縮窄 line 98 regex
- tests/test_post_test_hook.py: 新增 ≥ 5 個測試案例（jest false-positive + eslint true-positive）
