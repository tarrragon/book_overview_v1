### W1-001.2 實機重現證據（PM chrome-devtools MCP 採集 2026-05-21）

**重現步驟**：dev build 載入 extension -> 登入 Readmoo 導航 read.readmoo.com/#/library -> popup 觸發「開始提取書庫資料」。

**提取結果（成功）**：
- content script：`EXTRACTION_COMPLETED extracted:96 total:96 successful:96 failed:0 duration:19.40ms`
- SW storage `readmoo_books`：96 本書，topLevelKeys = [books, extractionCount, extractionDuration, extractionTimestamp, source]

**欄位覆蓋率（96 本書）**：
| 欄位 | 覆蓋 | 問題 |
|------|------|------|
| id / title | 96/96 | 正常 |
| progress | 96/96 | 正常（數字，如 45 / 100） |
| readingStatus | 0/96 | 全部 undefined — 6 種閱讀狀態完全未套用 |
| tagIds | 0/96 | 無此欄位；book 物件僅有 tags:["readmoo"]（書城來源名，非 tag 系統） |
| cover | 18/96 | 78 個被 UNSAFE_COVER_URL_FILTERED 過濾 |

**提取的 book 物件實際 schema（sample）**：
`{ id:"reader-15931298", title, progress:45, cover, coverInfo, identifiers:{coverId,privacyBookId,readerLinkId,titleBased,primarySource}, progressInfo:{hasProgress,progress,progressStyle,progressText}, source:"readmoo", tags:["readmoo"], type:"版式", url, extractedAt, extractedFrom }`
— 無 readingStatus、無 tagIds、無 tag_categories、無 schema_version。

**Overview 顯示**：96 本書全部列出，但「狀態」欄 96/96 顯示「未知」、「分類」欄 96/96 顯示「未分類」（Overview 未做讀取時 model 轉換）。

**分析切入點**：v1.1 model（readingStatus 6 狀態可由 progress derive：0->unread / 0<p<100->reading / 100->finished）的轉換層應在 content script 提取 / ReadmooAdapter / SW save / Book Model 哪一層；釐清 v0.17.x PROP-007 重構為何未涵蓋此 runtime 路徑（193 單元測試通過但實機未驗證）。
