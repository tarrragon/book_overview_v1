## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`.mcp.json` 中 codegraph MCP server 配置指向不存在的 binary `codegraph-mcp`。實際 binary 為 `codegraph`（位於 `/Users/mac-eric/.nvm/versions/node/v22.18.0/bin/codegraph`），MCP 模式透過 `codegraph serve --mcp` 子命令啟動。

舊配置 `["-w", "."]` 是早期 codegraph CLI 介面（watch 模式 + 路徑參數），現行版本已改為 `serve --mcp` 統一入口。

### 影響範圍

- 檔案：`.mcp.json`（單一檔案，1 處 command + 1 處 args，共 4 行變更）
- 功能影響：session 啟動時 codegraph MCP server 因 command 不存在而 fail，導致所有 `codegraph_*` tools（search/callers/callees/trace/impact/node/context/explore/files/status）全部未註冊，無法執行結構化程式碼查詢。

### 相關 Error Pattern

無直接對應 PC；屬「外部工具升級後配置未同步」類型，與 PC-145（stale CLI install diagnosis）為同源——CLI 版本演進後本機配置失準。
