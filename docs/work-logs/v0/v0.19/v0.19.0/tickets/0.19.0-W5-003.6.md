### 撰寫完成摘要

**新增章節**：`docs/spec/extraction/e2e-contract.md` §6 DOM 提取選擇器契約

**章節結構**（5 項契約結構標準）：

| 項目 | 內容 |
|------|------|
| Name | .library-item DOM 結構 + selector + ID 策略 + cover 過濾 + lazy load |
| Source of Truth | 16 個引用點（readmoo-adapter.js 多處函式 / docs/bookstores/readmoo.md / W1-061 ANA / e2e fixture） |
| 契約定義 | 8 子節：6.1 SELECTORS 完整清單 (12 個) / 6.2 .library-item DOM 結構樹 v1.3.0 / 6.3 欄位 selector 對應表 (8 欄位) / 6.4 Book ID 4 級策略 + 不可變更順序契約 / 6.5 UNSTABLE_COVER_IDS 黑名單與過濾 4 步驟 / 6.6 作者欄位 source limitation (W1-061 6 selector 0 命中證據) / 6.7 Lazy load scroll-to-end 演算法 + 已知缺口 / 6.8 getBookElements 三層 fallback |
| 變更影響 | 6 種變更類型對下游契約的影響 |
| Grep 驗證 | 7 條 grep 指令全部命中（含 §6.6 作者 selector 不存在的反向驗證） |

**關鍵契約**：

1. **Book ID 策略順序**：reader-link (1) → cover (2) → title (3) → fallback (4)，順序不可變更（W6-012.2.1 確立）
2. **真實 book ID 來源**：`[id^="privacy-"]` 提供 `privacy-{8位數字}`，`a.reader-link[href]` 是 SPA 佔位 URL 不可用
3. **Cover 過濾黑名單**：UNSTABLE_COVER_IDS = ['openbook', 'undefined', 'placeholder', 'default']
4. **作者 source limitation**：6 個 selector 對 96 樣本 0 命中（W1-061 證據），extractor 永遠產出 authors=[]
5. **Lazy load 演算法**：scroll → 找「更多」按鈕 → 點擊 → 等 800ms → count；3 輪 plateau OR 30 輪上限終止
6. **Header 文字 regex**：可見書目 = 擁有 N 本 - 封存 X 本 - 借出 Y 本
7. **getBookElements 三層 fallback**：bookContainer → alternativeContainers → readerLink 反查（最多 12 層祖先）
8. **已知缺口**：extractAllBooks 缺 scroll-to-end loop，僅提取首批 96/944（W6-021 ANA 結論）

### Grep 驗證結果

7 條 grep 全部命中：
- SELECTORS 12 個 selector 定義位置確認（L60-94）
- ID 策略 applyIdGenerationStrategiesWithInfo + try*Strategy 4 函式命中
- UNSTABLE_COVER_IDS L112 + L1513 命中
- 5 個 PATTERN 常數 + MAX_ANCESTOR_DEPTH 命中
- 作者 selector grep 0 命中（反向驗證 §6.6 W1-061 結論）

### 設計決策

1. **ID 策略順序鎖定**：reader → cover → title → fallback 順序由 W6-012.2.1 確立，本契約明示「不可變更」；變更會破壞下游 §2.5 migration 對 id 格式的假設
2. **作者欄位 fail-fast**：與其加 selector 假裝有結果，明示 source limitation；UI 提示用戶手動編輯
3. **Lazy load 已知缺口記錄**：W6-021 ANA 結論已含 follow-up IMP 規劃，本契約不重複落地，僅記錄缺口讓讀者知曉
