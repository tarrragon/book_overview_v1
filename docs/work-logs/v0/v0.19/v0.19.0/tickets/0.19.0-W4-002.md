## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

> PM 前台調查（2026-05-22）。接手者應獨立重新驗證數量/範圍（PC-007）。

### 問題根因

Overview 書名 `<td>` 由 `src/overview/overview-page-controller.js` 的 `createBookRow()` 動態建立（line 772-774），只設 `titleCell.textContent`，未設 `class`。E2E 與單元測試因此被迫使用脆弱的位置選擇器 `td:nth-child(3)` 定位書名欄，一旦欄位順序變動即無聲失效。

### 影響範圍

| 檔案 | 變更內容 |
|------|---------|
| `src/overview/overview-page-controller.js` | `createBookRow()` line 772-774 titleCell 加 `className`（同檔 line 731 `selectCell.className = 'select-col'` 已有 class 慣例可參照） |
| `tests/e2e/browser/extraction-pipeline.e2e.test.js` | line 210-213 將 `#tableBody tr td:nth-child(3)` 改用語意 selector，並移除「TD-1：無語意化 class」標記 |

**ticket frontmatter `where.files` 修正**：原列 `src/overview/overview.html` 不準確 — `overview.html` 僅含靜態 `<thead>`（`<th>書名</th>`），`<tbody id="tableBody">` 為空，書名 `<td>` 不在此檔。實際實作檔為 `overview-page-controller.js`。`where.files` 已由 PM 透過 `set-where` 修正。

### 相關 Error Pattern

無直接對應 PC/IMP。本質為可測性技術債（脆弱位置選擇器），對應 E2E 測試內既有 `TD-1` 標記。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 變更摘要

| 檔案 | 變更 |
|------|------|
| `src/overview/overview-page-controller.js` | `createBookRow()` line 773 新增 `titleCell.className = 'book-title-cell'`，跟隨同檔 line 731 `selectCell.className = 'select-col'` 字串字面慣例 |
| `tests/e2e/browser/extraction-pipeline.e2e.test.js` | G5-1 測試 selector 由 `#tableBody tr td:nth-child(3)` 改為 `#tableBody tr td.book-title-cell`，並移除「TD-1：無語意化 class」註解，改寫為語意化說明 |

### 設計決策

- **class 常數管理機制查核**：`overview-page-controller.js` 的 `CONSTANTS.TABLE` 物件僅集中數值/字串配置（COLUMNS、COVER_SIZE、DEFAULT_COVER），不含 CSS class 名稱。同檔既有 class 設定（line 731 `select-col`、line 735 `row-checkbox`）皆以字串字面直接賦值。故本次依現有慣例採字串字面 `'book-title-cell'`，未引入新的常數集中管理機制（避免為單一 class 過度設計）。
- **selector 替換驗證**：`npm run build:dev` 後產物 `overview-page-controller.js:773` 與 bundle `overview.js:3252` 均含 `book-title-cell`；`npm run test:e2e:browser` G5-1 在真實 Chrome 環境以 `td.book-title-cell` 成功匹配 5 本書書名，確認 selector 真實有效。

### 不在本 ticket 範圍

- `tests/unit/overview/overview-import-function.test.js` 的 `nth-child` 為刻意對齊多欄佈局，未動。
- `overview.html` 靜態 `<th>` 未動。
- 執行 `npm run test:unit` 期間發現既有 flaky 計時斷言（`tests/unit/core/errors/UC04ErrorAdapter.test.js:410` `toBeLessThan(10)`，違反 test-assertion-design-rules.md 規則 1），已依 quality-baseline 規則 5 建 follow-up ticket `0.19.0-W4-010` 追蹤，不在本 ticket 修復。

### Phase 4 評估

- **重構需要**：否。本 ticket 為 1 行 class 賦值 + 1 行 selector 字串替換，跟隨既有慣例，無新增抽象、無認知負擔上升。
- **技術債務**：無（執行期間發現的既有 flaky 斷言已轉 W4-010 追蹤）。
- **程式碼品質**：A。
- **結論**：無需重構。
