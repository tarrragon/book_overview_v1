## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因（PM 前台修正）

**原 ticket 描述方向偏離**：原 strategy「補 pyproject.toml 加 pyproject_scanner dep」假設 pyproject_scanner 是 PyPI 包，**實際上它是本地模組** `.claude/lib/pyproject_scanner.py`（PyPI 無此包）。

**真正根因**：`package_manager.py:18-20` 用 sys.path hack 動態載入：

```python
_lib_path = Path(__file__).parent.parent.parent.parent.parent / 'lib'
sys.path.insert(0, str(_lib_path))
from pyproject_scanner import ...
```

uv tool install 後 `__file__` 指向 `~/.local/share/uv/tools/project-init/lib/python.../site-packages/project_init/lib/package_manager.py`，向上 5 層**不會**到達專案的 `.claude/lib/` — 此即 ModuleNotFoundError 真因，非 dep 缺失。

### 影響範圍

| 檔案 | 影響 |
|------|------|
| `.claude/lib/pyproject_scanner.py` | SSOT 模組（含 5 個 export API） |
| `.claude/skills/project-init/project_init/lib/package_manager.py` | sys.path hack 失效根因；CLI entry point 阻斷 |
| `.claude/hooks/package-version-sync-hook.py` | 同樣 import pyproject_scanner，但 hook 以 main repo cwd 執行（sys.path 可解析），未踩同根因 |
| `.claude/lib/tests/test_pyproject_scanner.py` | 既有測試 |

### 相關 Error Pattern

| Pattern | 關聯 |
|---------|------|
| PC-007 接手者獨立驗證 | 原 ticket strategy「補 dep」誤判 pyproject_scanner 為 PyPI 包，**PM 前台已驗證並修正方向** |
| PC-159 安裝指令 fresh shell 驗證 | 本 ticket 是 PC-159 案例（換電腦才暴露 sys.path hack 問題） |
| W11-037 stale install 模式 | uv tool install --reinstall 必要，但本 ticket 根因比 stale 更深（是 sys.path 設計問題） |

### 修復方向評估（三方案）

| 方案 | 描述 | 優點 | 缺點 |
|------|------|------|------|
| A | 複製 `pyproject_scanner.py` 進 `project-init/project_init/lib/`，移除 sys.path hack | 最簡單；entry point 立刻可用 | SSOT 破壞（hook + CLI 兩處同名模組），未來 pyproject_scanner 改動需同步 2 處 |
| B | 把 `.claude/lib/` 包成 internal pkg（如 `claude-shared-lib`），hook + CLI 皆加 dep | 長期解；SSOT 保留 | 範圍最大（影響 hook 與多 skill）；需設計 internal pkg 打包流程 |
| C | 修 sys.path hack 用 `PROJECT_ROOT` 環境變數或 `git rev-parse` 動態解析 | 不破壞 SSOT；改動最少 | installed 環境下用戶 cd 到其他 repo 跑 CLI 會失效（脆弱） |
| D | pyproject.toml 用 local path dependency `pyproject_scanner @ file://../../lib`，wheel 內含模組 | 不破壞 SSOT；entry point 自動找到模組 | uv tool install 對 file:// 本地路徑支援有限；wheel build 路徑變動需測試 |

**PM 傾向**：方案 A（最簡可靠），複製 SSOT 破壞屬可接受（pyproject_scanner.py 含 5 個 API，每年變動頻率低，可加 sync hook 防漂移）。但需用戶確認方向。
