## Problem Analysis

### 問題根因

W6-001.2 設計 `detect_codegraph()` 時誤將 `codegraph-mcp` 設為主 binary、`--info` 設為版本 flag。實際系統安裝為 `@colbymchenry/codegraph` 提供 `codegraph` binary（支援 `--version`，不支援 `--info`），與 .mcp.json 同源誤判（W1-101 已修正 .mcp.json）。即使用戶執行 `npm install -g`，`detect_codegraph()` 主路徑找不到 `codegraph-mcp`、fallback 用 `--info` 也失敗，導致 `project-init check` 永遠回報 codegraph MISSING。

### 影響範圍

- `.claude/skills/project-init/project_init/lib/mcp_detector.py:102-132` `detect_codegraph()` 主路徑與 fallback 路徑顛倒
- `.claude/skills/project-init/tests/test_mcp_detector.py` 5 個 codegraph tests 全部 mock `codegraph-mcp` 為主、`codegraph` 為 fallback，需對調
- 影響 `project-init check` 報告完整性（W6-001.2 設計但未生效的功能）

### 相關 Error Pattern

W1-101 同源：.mcp.json 與 mcp_detector 同時誤用 `codegraph-mcp` binary 名稱。屬「規格描述未驗證假設」家族（PC-143 / PC-145）。
