## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-107 議題 A（GlobalMessages 作用域邊界）收斂時，3 個中文字面 keys 因 caller 數量少（各 1 個）暫保留於 GlobalMessages._loadDefaultMessages，等待後續為 caller 建 local dict。本 ticket 為這個收尾任務。

### 影響範圍

**3 個中文 keys 位置**：`src/core/messages/MessageDictionary.js:107-109`
```js
未知的篩選條件: '未知的篩選條件',
'索引搜尋失敗，回退到線性搜尋': '索引搜尋失敗，回退到線性搜尋',
'Event listener registration failed': '事件監聽器註冊失敗'
```

**3 個 caller 使用點**：

| Caller | 位置 | 使用方式 |
|--------|------|---------|
| filter-engine | src/ui/search/filter/filter-engine.js:421 | this.logger.warn('未知的篩選條件', {...}) |
| search-engine | src/ui/search/core/search-engine.js:202 | this.logger.warn('索引搜尋失敗，回退到線性搜尋', { error: error.message }) |
| platform-detection | src/background/domains/platform/services/platform-detection-service.js:72 | this.logger.warn('Event listener registration failed', { error: listenerError.message }) |

### 相關 Error Pattern

依賴 W1-115（Logger constructor 第三參數 messages 注入）已修復。仿 W1-108 readmoo-adapter local dict 模式 + popup local dict 模式。

### 違反規範

docs/project-conventions.md Messages 系統規範 — GlobalMessages 納入標準 3 條件 AND（被 2+ 獨立模組引用 + 屬通用詞彙 + 無模組前綴）：

- 每個中文 key 各只被 1 個 caller 引用 → 違反「2+ 獨立模組引用」
- 屬具體業務訊息（不是通用詞彙如 NETWORK_ERROR / SUCCESS） → 違反「屬通用詞彙」
- 中文字面當 key 不是模組前綴，但屬 module-specific 語意

### 設計決策

1. 為 3 個 caller 各建 local MessageDictionary
2. 命名建議：filterEngineMessages / searchEngineMessages / platformDetectionMessages
3. Key 命名改 UPPER_SNAKE_CASE（如 FILTER_UNKNOWN_CONDITION / SEARCH_INDEX_FALLBACK_TO_LINEAR / EVENT_LISTENER_REGISTRATION_FAILED）
4. caller 透過 Logger constructor 第三參數注入 local dict
5. caller 改用 UPPER_SNAKE_CASE key 呼叫 this.logger.warn(KEY, params)
6. 最後從 MessageDictionary._loadDefaultMessages 移除 3 個中文 keys + L65 註解
7. npm test 驗證無回歸

### 仿照模式

W1-108 readmoo-adapter pattern（src/content/adapters/readmoo-adapter.js）：
```js
const readmooAdapterMessages = new MessageDictionary({
  KEY_NAME: '中文訊息',
  ...
})
this.logger = new Logger('ReadmooAdapter', 'INFO', readmooAdapterMessages)
```

### Test 策略

1. 為各 caller 建 unit test 確認新 messageKey 可正確 lookup
2. 確認 Logger.warn 渲染文字正確（依 PC-165 防護避免測試斷言只看 key 不看渲染）
3. npm test 全綠（5263 → 5263+ passed）
