### 問題根因

`commands/lifecycle.py` 的 `release()`（line 848-888）在通過前置檢查後，無條件執行：
- line 879: `ticket["status"] = STATUS_BLOCKED`

### 修復摘要

`commands/lifecycle.py` `release()`（line 848-)：

1. import 新增 `STATUS_LABELS`（來自 `ticket_system.lib.constants` shim，re-export `ticket_system.constants`）。
2. 計算目標狀態：`blocked_by = ticket.get("blockedBy") or []`；空 → `STATUS_PENDING`，非空 → `STATUS_BLOCKED`，取代原 line 879 寫死的 `STATUS_BLOCKED`。
3. stdout 改用 `STATUS_LABELS.get(target_status, target_status)` 動態查顯示名，取代原 line 887 寫死的 `print("   狀態: 被阻塞")`。
4. docstring 同步說明 W3-082 語意（blocked = 被其他 ticket 擋著；無 blocker 應退回 pending 休眠態）。

### 狀態顯示名集中管理

grep 確認專案已有集中映射 `STATUS_LABELS`（`ticket_system/constants.py:113`，status → 中文）。沿用此既有常數，未新增散落字面，符合專案規範。`StatusMessages.STATUS_CHANGE_BLOCKED`（messages.py:160）為舊寫死字面，本次改用 `STATUS_LABELS` 後 release 不再依賴它。

### SKILL.md / reference 同步（ticket-skill-sync-check 規則）

行為變更（release 目標狀態語意改變）已同步至 `references/track-command.md` 的 release 命令說明（補 blockedBy 兩分支目標狀態註解）。SKILL.md 主文僅列 release 於操作清單，無具體狀態語意敘述，無需改動。完成後執行 `uv tool install .claude/skills/ticket --reinstall`（IMP-023）。
