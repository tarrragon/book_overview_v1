# Execution Log

## Task Summary

修復 ticket migrate 在更新反向交叉引用（spawned_tickets/source_ticket）時，正確改寫被引用 ticket 的 frontmatter 但刪除其整個 body（Execution Log）的 bug

---

## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因（PM 前台分析，已對照原始碼確認）

資料損壞鏈：

1. `commands/migrate.py:_load_ticket_from_path`（L227）：`ticket, _ = parse_frontmatter(content)` — **丟棄 body**，未設 `ticket["_body"]`。
2. `commands/migrate.py:_update_cross_references`（L196-198）：更新 frontmatter 後呼叫 `save_ticket(ticket, ticket_file)`。
3. `lib/parser.py:save_ticket`（L317）：`body = ticket.pop("_body", "")` — `_body` 不存在時預設空字串 → 寫入時 body 為空 → 整個 Execution Log 被截斷。

對照組：標準 loader（`lib/parser.py:246`）正確設 `frontmatter["_body"] = body`；migrate 私有 loader 漏此步（兩個 loader 慣例漂移，ARCH-020 家族）。

### 影響範圍

`_update_cross_references` 對「被引用 ticket」（反向引用）的所有寫回路徑。本次遷移 5 檔（W1-056/W3-002/W3-072/W3-095/W6-001.4）共約 1027 行 body 全毀，已 `git restore` + 手動修 frontmatter 復原（commit e091e603）。被遷移 ticket 自身（正向引用）走不同路徑，未受影響。

### 相關 Error Pattern

IMP-061（migrate parent_id typo + 依賴未同步，同檔案家族）、ARCH-020（跨進程/跨函式同構邏輯漂移）、PC-152（migrate collision 靜默覆寫）、PC-166（git diff 為事實基準，攔截本次損壞）。

### 最小修復方向

`_load_ticket_from_path` 改為保留 body：

```python
ticket, body = parse_frontmatter(content)
if ticket:
    ticket["_body"] = body
return ticket if ticket else None
```

### 回歸測試

`tests/test_migrate_reverse_refs.py` 新增案例：被引用 ticket 含非空 body → 觸發反向引用更新 → 斷言 (a) 引用 ID 已更新 (b) body 完整保留（逐字比對）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復摘要

走 evidence-driven-bugfix（RED → GREEN → 回歸）：

1. **RED**：`tests/test_migrate_reverse_refs.py` 新增 `TestW4_025_BodyPreservedOnReverseRefUpdate`，被引用 ticket 含非空多行 body，觸發 `_update_cross_references(old_id → new_id)` 後斷言 (a) `spawned_tickets` 已更新、(b) body 完整保留。修復前讀回 body 為空字串 `''`，FAIL 確認。
2. **GREEN**：`commands/migrate.py:_load_ticket_from_path` 改為保留 body：
   ```python
   ticket, body = parse_frontmatter(content)
   if ticket:
       ticket["_body"] = body
   ```
   對齊標準 loader（`lib/parser.py:246`）慣例，使 `save_ticket`（L317 `body = ticket.pop("_body", "")`）取得正確 body 不再截斷。

### 設計決策

- body 斷言比對 `body.strip()`：`parse_frontmatter`（parser.py:155）對 body 執行 `.strip()` 為既有 round-trip 契約，非本 bug；strip 後比對仍可明確攔截「body → 空字串」的資料損壞（與 `''` 比對差異顯著）。

### 不在本 ticket 範圍

- 兩個 loader 慣例漂移（ARCH-020 家族）的根本 consolidate 不在本次最小修復範圍，僅修正 migrate 私有 loader 漏設 `_body` 一步。
