## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

overview-page-controller.js（1500 行）承載 >=5 個不同職責（表格渲染、搜尋篩選、匯出、匯入流程、事件處理），其中匯入相關方法佔 ~200 行（handleFileLoad 40 行 + promptImportMode 96 行 + modal show/hide 14 行 + _handleDuplicateBooks 3 行 + 匯入專用狀態 _importModePending/_importModePreviousFocus），違反 SRP。

具體問題：
- handleFileLoad 7 步驟線性流程（驗證 → modal → 分流 → 讀檔 → 持久化 → UI 更新）承擔驗證協調、UI modal 管理、持久化策略分流三個職責
- promptImportMode 96 行內聯 modal 邏輯（含 focus trap、Esc 處理、overlay click、settle 收斂）完全與 controller 核心職責無關
- IMPORT_MODE_MODAL_MISSING 哨兵值、_importModePending/_importModePreviousFocus 狀態汙染 controller 命名空間
- _handleDuplicateBooks 為死碼（grep 確認 src/ 無任何呼叫點，僅測試直接呼叫）

### 影響範圍

- src/overview/overview-page-controller.js — 匯入相關行（1056-1255, 1472-1488）
- src/overview/book-file-importer.js — 被 ImportFlowController 依賴（不改動，僅改消費者）
- tests/unit/overview/overview-import-duplicate-handling.test.js — 直接呼叫 controller._handleDuplicateBooks，需遷移

### 相關 Error Pattern

無直接相關 PC。SRP 違反屬 cognitive-load 設計原則範疇（rules/core/cognitive-load.md）。

### WRAP 分析

**W (Widen)**：
- 方案 A：抽出 ImportFlowController 為獨立模組（三視角共識建議）
- 方案 B：僅抽出 promptImportMode 為獨立 ImportModeModal 類別，handleFileLoad 留 controller
- 方案 C：不抽出，僅清理死碼（_handleDuplicateBooks）

**R (Reality-test)**：
- 方案 A：移出 ~200 行，controller 降至 ~1300 行；匯入流程有獨立測試面；DI 介面清晰
- 方案 B：僅移出 ~110 行，controller 仍 ~1390 行；handleFileLoad 仍綁 TagStorageAdapter 靜態呼叫
- 方案 C：移出 3 行；不解決 SRP 問題

**A (Attain distance)**：
- 方案 B 只治標（modal 行數），不治本（handleFileLoad 仍混合持久化分流 + UI 更新）
- 方案 C 不符合 W1-048 Phase 4 三視角共識

**P (Prepare to be wrong)**：
- 方案 A 風險：若 controller 後續功能需要直接呼叫匯入子步驟（如 validate-only），介面需保持足夠彈性
- 緩解：execute(file) 為主入口，但保留 BookFileImporter 的 validate/read/parseContent 公開 API 不變

**結論**：採方案 A。

### 檔案位置

### constructor 簽名

```javascript
/**
 * @typedef {Object} ImportFlowDeps
 * @property {BookFileImporter} bookFileImporter - 檔案驗證/讀取模組
 * @property {Object} elements - DOM 元素引用（importModeOverlay/Modal/Buttons）
 * @property {Document} document - DOM 文檔（焦點管理用）
 * @property {Object} tagStorageAdapter - { replaceAllData, mergeAllData }
 * @property {Function} showError - 顯示錯誤訊息
 * @property {Function} showLoading - 顯示載入狀態
 * @property {Function} onImportSuccess - 匯入成功後的 UI 更新 callback(books)
 */
constructor(deps: ImportFlowDeps)
```

### execute 簽名

```javascript
/**
 * 執行完整匯入流程：驗證 → modal → 讀檔 → 持久化 → callback
 *
 * @param {File} file - 使用者選取的檔案
 * @returns {Promise<ImportFlowResult>}
 *
 * @typedef {Object} ImportFlowResult
 * @property {boolean} success - 是否完成（含取消也算 success:false）
 * @property {'completed'|'cancelled'|'modal_missing'|'storage_error'} reason
 * @property {Array<Object>|null} books - 成功時的書籍陣列
 */
async execute(file): Promise<ImportFlowResult>
```

### 模式常數

```javascript
const IMPORT_MODE = {
  OVERWRITE: 'overwrite',
  MERGE: 'merge'
}
```

### 依賴方向圖

```
overview-page-controller.js
    |
    |--- new ImportFlowController(deps)
    |        |
    |        |--- BookFileImporter (DI, 既有模組)
    |        |--- TagStorageAdapter (DI, 不再靜態呼叫)
    |        |--- elements (DI, DOM refs)
    |        |--- showError / showLoading / onImportSuccess (DI callbacks)
    |
    |--- BookExporter (既有, 不變)
    |--- DuplicateBookMerger (既有, 不變; _handleDuplicateBooks proxy 刪除)
    |--- TagStorageAdapter (匯出用, 保留)
```

依賴方向：controller → ImportFlowController → BookFileImporter / TagStorageAdapter
無環：ImportFlowController 不依賴 controller，透過 callback 反向通知。

### controller 整合方式

```javascript
// constructor 中：
this.importFlowController = new ImportFlowController({
  bookFileImporter: this.bookFileImporter,
  elements: {
    importModeOverlay: this.elements.importModeOverlay,
    importModeModal: this.elements.importModeModal,
    importModeOverwriteBtn: this.elements.importModeOverwriteBtn,
    importModeMergeBtn: this.elements.importModeMergeBtn,
    importModeCancelBtn: this.elements.importModeCancelBtn
  },
  document: this.document,
  tagStorageAdapter: TagStorageAdapter,
  showError: (msg) => this.showError(msg),
  showLoading: (msg) => this.showLoading(msg),
  onImportSuccess: (books) => this._updateUIWithBooks(books)
})

// handleFileLoad 簡化為：
async handleFileLoad(file) {
  await this.importFlowController.execute(file)
}
```
