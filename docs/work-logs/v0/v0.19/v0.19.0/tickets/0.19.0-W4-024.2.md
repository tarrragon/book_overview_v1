### 設計決策

#### 為何選擇 local MessageDictionary 而非 plain console wrapper

依 project-conventions.md Messages 系統規範：模組專屬 key 必須透過 new MessageDictionary({...}) 註冊為 local dict，禁止加進 GlobalMessages。tag-storage-adapter 屬 src/storage/ 模組，所有 10 個訊息含 [tag-storage-adapter] 模組前綴語意，必須走 local dict 路徑。

#### 為何顯式引用 messages/index 而非 ../../core/messages

src/core/ 同時存在 messages.js（legacy 基礎字典）和 messages/ 目錄，Node 模組解析優先選 file 而拉到 legacy 版（無 MessageDictionary export）。改用 ../../core/messages/index 顯式指向新版字典系統，避免歧義。此細節以行內註解標註於 import 行上方。
