**直接根因**：Logger constructor 簽章 constructor(name = 'App', level = 'INFO') 僅接受 2 個參數，this.messages = GlobalMessages hardcoded 於 constructor body（第 68 行）。3 個 caller 傳入第三參數 localMessages 被 JavaScript 靜默忽略。

### PC Error-Pattern 草稿

**ID**: PC-XXX（待 PM 分配）
**名稱**: False Positive 修復鏈——測試綠燈不等於 Runtime 正確

**症狀**：
- 修復 ticket 測試全通過（綠燈），但 runtime 行為未改變或回歸
- 後續 ticket 基於「修復已生效」假設執行進一步操作，造成連鎖破壞

**根因**：
- 測試斷言不覆蓋 runtime 實際行為路徑（只檢查 DOM/結構，不檢查訊息內容）

- W1-105 修 MessageDictionary constructor → 41/41 通過 → Logger 仍 hardcoded GlobalMessages → runtime 無效

**防護**：
1. 修復「訊息系統」「日誌系統」「多模組整合」類 bug 時，acceptance 必須含 runtime 層級驗證（chrome-devtools-mcp 實機或 integration test 斷言訊息文字）

3. JavaScript 專案：函式簽章變更時 grep 所有 caller 確認參數數量匹配
