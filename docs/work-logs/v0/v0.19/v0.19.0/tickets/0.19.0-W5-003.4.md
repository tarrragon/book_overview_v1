**章節結構**（5 項契約結構標準）：

| 項目 | 內容 |
|------|------|
| Name | SW 啟動 → CS 注入 → 用戶觸發提取 → Storage 寫入 → Overview 顯示 → SW 關閉 完整 lifecycle |
| Source of Truth | 15 個引用點（background.js / lifecycle/*.js / content-modular.js / message-router / popup.js / overview-page-controller / manifest.json） |
| 契約定義 | 8 子節：4.1 完整 Mermaid sequenceDiagram (6 Phase) / 4.2 Phase 1 SW 啟動 / 4.3 Phase 2 CS 注入九步驟 / 4.4 Phase 3 用戶觸發提取 10 step / 4.5 Phase 4 Storage 寫入廣播 / 4.6 Phase 5 Overview 顯示 / 4.7 Phase 6 SW 關閉 / 4.8 並發與時序約束 |
| 變更影響 | 6 種變更類型對下游契約的影響 |
| Grep 驗證 | 8 條 grep 指令，全部命中 |

**關鍵契約**：

1. **Mermaid sequenceDiagram** 完整繪製 6 Phase / 用戶 + Browser + SW + CS + Popup + Overview + Storage 7 個 participant
2. **CS 注入時機**：`run_at: document_idle` （DOM 載入完成後）
3. **CS 九步驟順序約束**：Step 6 globalThis 設定必須在 Step 7-8 之前；Step 9 reportReadyStatus 必須在所有初始化完成後
4. **提取訊息流**：popup → SW (chrome.runtime) → CS (chrome.tabs) → bookDataExtractor.startExtractionFlow → contentEventBus emit → contentChromeBridge forward → SW event-coordinator → tag-storage-adapter.saveBooksWrapper
5. **CS → SW 轉發 5 種事件**：EXTRACTION.STARTED / PROGRESS / COMPLETED / ERROR / CANCELLED
6. **並發約束**：Migration 必須在 SW 啟動完成前；storage.set 必須在 EXTRACTION.COMPLETED emit 之前（避免 race）；SYSTEM.SHUTDOWN 必須在 SW 真正回收前
