## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

依 W1-050 ANA Solution 致命級表 F1/F2/F3/F5/F9/F10 逐項替換 `global.*` 裸用為 `globalThis.*` 或直接 API：

| ID | 檔案 | 修復方式 |
|----|------|---------|
| F1 | src/utils/file-reader-factory.js | `global.FileReader` → `globalThis.FileReader`（typeof guard 保留測試環境相容） |
| F2 | src/export/book-data-exporter.js (downloadFile) | `global.URL` → `URL`、`global.document` → `document`（瀏覽器原生 API） |
| F3 | src/export/book-data-exporter.js (copyToClipboard) | `global.navigator` → `navigator`，加 `typeof navigator !== 'undefined'` guard |
| F5 | src/error-handling/event-performance-monitor.js | `global.gc` → `globalThis.gc` + `typeof === 'function'` guard（CE runtime 無 GC API，Node `--expose-gc` 才有） |
| F9 | src/core/performance/performance-anomaly-detector.js | 同 F5 |
| F10 | src/content/content-modular.js | `global.*` 6 處 → `globalThis.*`（CE content + Node 測試環境共用） |

### 設計決策

1. **globalThis 而非 self/window**：CE 四個 runtime（SW / content / popup / overview）的全域物件不同（SW 用 self，其餘用 window），但 `globalThis` 是 ES2020 標準在所有 runtime 都可用，最一致。
2. **典型物件直接使用而非透過 globalThis**：`URL` / `document` / `navigator` 在 overview / popup / content 環境是頂層 API，直接使用比 `globalThis.URL` 更清楚且符合 W3C 慣例。SW context 不適用（SW 無 document），但 F2/F3 僅在 overview 觸發。
3. **global.gc 移除而非完全刪除**：保留 `globalThis.gc` typeof guard，便於 Node `--expose-gc` 測試環境保留手動 GC 路徑，CE runtime 該分支永遠 false（無功能影響）。
