### 修改摘要

在 `.claude/references/pm-agent-observability.md` 文件末尾新增「附錄：外部觀察工具速查」章節，記錄 `claude agents --json` (v2.1.145+) 的 schema 與適用 / 不適用情境。

依 W3-027 ANA 方案 D 結論（不整合 + 文件補充）：明示「補充非取代既有五工具」立場，避免讀者誤以為第六工具。

### 段落內容（5 個子結構）

1. **Why 補充非取代**：粒度為 session 級非 agent/dispatch 級
2. **輸出 schema 表**：7 個欄位（pid / cwd / kind / startedAt / sessionId / name / status）
3. **適用情境表**：3 項（session 存活性 / orphan 偵測 / 跨 session 並行觀察）
4. **不適用情境表**：4 項（明示為什麼不能取代既有五工具）
5. **未來升級重評觸發條件**：明示何時需重評整合決策

### 設計決策

| 維度 | 選擇 | 理由 |
|------|------|------|
| 段落位置 | 文件末尾「附錄」 | 避免擾亂既有「五工具分工」主軸結構，明示補充性質 |
| 章節標題 | 「附錄：外部觀察工具速查（補充，非取代既有五工具）」 | 標題直接表達 W3-027 ANA 結論 |
| 適用 / 不適用對稱表 | 兩個表並列 | 防止讀者選擇性閱讀只看適用情境，引發誤用 |
| 與五工具對應欄 | 「不適用情境表」明示替代工具 | 引導讀者回到既有五工具體系，避免重覆評估 |
