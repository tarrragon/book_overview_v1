## Problem Analysis

### 問題根因

APP（book_overview_app）匯出格式為 `{backup_info|export_info, books:[]}`，**無 `metadata.formatVersion`**。匯入時舊 detector 命中 flat-v1 規則 → 誤判 `'v1'` → 跑 lossy `convertV1ToV2Book`（只取 id/title/author 單值/publisher/cover/category）→ **丟失 APP 的 tags、readingStatus、多值欄位、importance/ccl**，違反 canonical C1（雙向無損）。

foundation（W4-031.2）已交付 `detectInterchangeSource`（四來源）+ `mapCanonicalToV1Book`（read 映射），但匯入端（ContentParser）尚未接線，且 v1-to-v2-converter 無 app-legacy 無損分支。

### 影響範圍

| 檔案 | 變更 |
|------|------|
| `src/overview/import/content-parser.js` | 注入四來源 detector + canonical/app-legacy/dedup 依賴；`_extractBooksFromData` JSON 路徑改四來源分流；新增 `_extractCanonicalBooks` |
| `src/export/v1-to-v2-converter.js` | 新增 `convertAppLegacyToV2Book` / `convertAppLegacyToV2Data` / `dedupBooks`（app-legacy 無損 + id 主鍵/軟連結去重） |

### 相關 Error Pattern

- PC-162（stale spec）：APP fixed-field 形狀以 spec §4/§11 + 既有 detector test fixture（backup_info wrapper）為準，未臆造未驗證欄位。
- 既有 W1-057/W1-078 timestamp 同源修復（`convertV1CategoryToTag`）未被觸碰，回歸保護維持。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### ContentParser 四來源接線（`content-parser.js`）

- constructor 新增 DI：`detectInterchangeSource` / `convertAppLegacyToV2Data` / `mapCanonicalToV1Book` / `dedupBooks`（皆預設指向 src/export/* foundation，向後相容）。
- `_extractBooksFromData` JSON 路徑改為四來源分流（spec §8）：
  - `canonical` → `_extractCanonicalBooks`：逐本 `mapCanonicalToV1Book` read 映射（首元素入固定欄位，多值/carry/未知欄位入 `_passthrough`），失敗書籍跳過，最後 `dedupBooks`。
  - `app-legacy` → `convertAppLegacyToV2Data` 無損止血。
  - `v1`/`v2`/null → 委派既有 `_detectFormatVersion`，保留 v2 隱含偵測啟發式（B5 回歸不變）。
- 防禦降級：`_detectInterchangeSource` 缺失時（legacy 測試 partial-module mock 僅保留 `detectFormatVersion`）降級走既有二值流程，避免破壞 book-file-importer 既有 path 鎖定測試（TC-19/20/22/23/25）。

### APP legacy 無損匯入（`v1-to-v2-converter.js`）

- `convertAppLegacyToV2Book`：
  - **readingStatus 不重推**：carry APP 原值；canonical 命名（not_started 等）逆正規化為 V1 命名（`normalizeReadingStatusFromCanonical`），V1 命名直用，未知態記 `_passthrough.readingStatusRaw`（禁臆造，C1）。
  - `coverImageUrl → cover`（spec §11）。
  - 多值欄位（publisher/platform/isbn）首值入固定欄位，完整陣列入 `_passthrough.tags.*` 供 write 端重建（C1）。
  - `importanceLevel(1-7) → imp-N` tag、ccl/description/series/language/alias carry+display 入 `_passthrough.tags`。
  - 未知頂層欄位（白名單外）+ APP 自身 `_passthrough` 保留（spec §9，禁 strip）；id 保留禁重生（§8 D5）。
- `convertAppLegacyToV2Data`：wrapper 提取 + 逐本轉換（失敗跳過）+ `dedupBooks`。
- `dedupBooks`：以 id 為主鍵，`crossPlatformId`/`dataFingerprint` 為軟連結輔助（不取代 id，spec §8 C5），保序保留首見。

### 既有修復保護

`convertV1ToV2Book` / `convertV1CategoryToTag`（W1-057/W1-078 timestamp 同源）未變更，原 v1 匯入回歸維持綠燈。

### 不在本 ticket 範圍

- 匯出 write / root 組裝 → `.2.1`（`book-data-exporter.js` 未碰）。
- foundation 純函式本體（detector / read 映射 / buildTagTree）→ 已於 `.2.3`/`.2.4` 交付。
