### 共用 context 來源（必讀，DRY 不重抄）

- **父 foundation `0.19.0-W4-031.2` 的 Context Bundle + Solution（Phase 1 設計）**：含 SSOT spec 引用、§1 範圍邊界表、API 契約、行為單元清單（含 read 方向多值收斂規則 A、§7 逆正規化）。先 `ticket track full 0.19.0-W4-031.2` 通讀。

### 問題根因（止血核心）

APP（book_overview_app）匯出格式為 `{backup_info|export_info, books:[]}`，**無 `metadata.formatVersion`** → 現有 detector 誤判為 flat **v1** → 跑 lossy `convertV1ToV2Book`（只取 id/title/author 單值/publisher/cover/category）→ **丟失 APP 的 tags、readingStatus、多值欄位**，違反 canonical C1（雙向無損）。本 ticket 接線新 detector 並補 app-legacy 無損匯入分支。

### Foundation 已交付（本 ticket 直接 require，不可重寫）

| 檔案 | 可用 export | 本 ticket 用途 |
|------|-------------|---------------|
| `src/export/format-version-detector.js` | `detectInterchangeSource(data)` → `'canonical'|'v2'|'app-legacy'|'v1'|null`（spec §8 優先序） | ContentParser 分流四來源 |
| `src/export/book-interchange-v1-adapter.js` | `mapCanonicalToV1Book(canonical)`、`normalizeReadingStatusFromCanonical` | canonical → 內部 v2（read 方向，含多值收斂 A、§7 逆正規化、passthrough） |

> 既有 `detectFormatVersion` 簽名不變（v1/v2/null），ContentParser 既有 DI 向後相容。

### 本 ticket 範圍（§1 範圍邊界表「.2.2」列）

1. **ContentParser 接線**（`src/overview/import/content-parser.js`，constructor DI @ line 66-71）：注入 `detectInterchangeSource`，依四來源分流；canonical → `mapCanonicalToV1Book`；passthrough 平鋪。
2. **APP legacy 無損止血**（`src/export/v1-to-v2-converter.js`）：新增 app-legacy → canonical（或內部 v2）分支——不重推 readingStatus、`coverImageUrl→cover`、多值入對應 tag、importance/ccl carry+display。**勿破壞** `convertV1ToV2Book` 既有 W1-057/W1-078 timestamp 同源修復。
3. **dedup**（spec §8）：id 主鍵 + crossPlatformId/dataFingerprint 軟連結，跨書集合去重。
4. **passthrough**（spec §9）：`extensions` + `_passthrough` 保留不丟失。
