### 設計分析（PM 前台，2026-05-28）

**動機**：另一 session 遇 PC-019 死循環——commit 後 AUQ 觸發 hook 寫入 runtime state（pm-status.json/dispatch-active.json/hook-state/logs），因檔案被 git tracked → 阻擋 worktree fork（dirty tree）→ 治本是 .gitignore 補齊 + `git rm --cached`。需 session-start 自動檢查避免跨專案重蹈。

**本專案 .gitignore 現況**（line 77/81/84/68 已部分覆蓋）：

| 已覆蓋 | 未覆蓋（本專案需補） |
|--------|-------------------|
| `.claude/dispatch-active.json` line 77 | `.claude/pm-status.json` |
| `.claude/hook-state/` line 81 | `.claude/ARCHITECTURE_REVIEW_REQUIRED` |
| `.claude/state/` line 84 | `.claude/TASK_AVOIDANCE_FIX_MODE` |
| `**/hook-logs/` line 68 | `.claude/PM_INTERVENTION_REQUIRED`（symmetry with LOCAL_ONLY） |

**必要 entry 清單**（hook 內 const 定義）：
```python
REQUIRED_GITIGNORE_ENTRIES = {
    # Runtime state markers
    ".claude/ARCHITECTURE_REVIEW_REQUIRED",
    ".claude/TASK_AVOIDANCE_FIX_MODE",
    ".claude/PM_INTERVENTION_REQUIRED",
    # Runtime state files
    ".claude/pm-status.json",
    ".claude/dispatch-active.json",
    ".claude/dispatch-active.lock",
    # Runtime state dirs
    ".claude/hook-state/",
    ".claude/state/",
    # logs/coverage（也許已被 broader pattern 覆蓋）
    ".claude/logs/",
    "coverage/",
}
```

**等效匹配考量**：git pattern `logs/`（不以 / 開頭、結尾斜線）會匹配任何層級的 `logs/` 目錄，等同 `.claude/logs/`。hook 應接受「精確字串 OR 等效 broader pattern」雙判定。

**等效規則表**：

| 必要 entry | 等效 broader pattern |
|-----------|--------------------|
| `.claude/logs/` | `logs/`、`**/logs/` |
| `.claude/hook-logs/` | `**/hook-logs/`、`hook-logs/` |
| `coverage/` | （無需等效，已是 broader） |
| `.claude/*.lock` 類 | `*.lock`（line 24 已含） |

**Hook 檢查邏輯**：
1. 讀 `<project_root>/.gitignore`，parse 成非空非註解行 set
2. 對 REQUIRED 逐項：若不在 set 且無等效 broader pattern → 缺失清單 append
3. 用 `git ls-files` 列出實際被 tracked 的檔案，過濾出符合 .claude/runtime-state-pattern 的 → tracked-but-should-ignore 清單
4. 缺失或 tracked 不為空 → WARN 輸出（仿 session-start-sync-exclusion-check 格式），含修復建議：
   - 補 .gitignore：列出需附加的 entry
   - 已 tracked：列出 `git rm --cached <file>` 命令
5. .gitignore 不存在 → WARN「未偵測到 .gitignore，所有 runtime state 將被 git tracked」

**註冊到 settings.json SessionStart**：
- 順序：放在現有 session-start-sync-exclusion-check-hook 之後（同類型 + 共用 logger pattern）
- 非阻擋（不 exit 2），WARN 等級

**測試策略**（pytest，新增 `.claude/hooks/tests/test_session_start_gitignore_check.py`）：
- `test_complete_gitignore_passes` — 完整 entries → 0 缺失 + 0 tracked
- `test_missing_entry_warns` — 缺 pm-status.json → 缺失 list 含該項
- `test_equivalent_broader_pattern_passes` — 用 `logs/` 取代 `.claude/logs/` → 通過（等效匹配）
- `test_tracked_runtime_state_warns` — git tracked pm-status.json → tracked list 含該項 + 提示 cmd
- `test_no_gitignore_file_warns` — 無 .gitignore → WARN 訊息含「all tracked」
- `test_comment_lines_ignored` — # 開頭與空行不影響 parse
