## Problem Analysis
<!-- Schema[DOC/Problem Analysis]: 選填（若 DOC 起因於缺陷或盤點結論可填）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W3-032 ANA 評估 Claude Code v2.1.139 `/goal` 指令與 ticket acceptance 機制的整合可行性，結論方案 D（不整合，平行存在）。

ANA 識別 nice-to-have 文件補充：agent-dispatch-template.md 缺少明確說明 `/goal`（session-level）vs `acceptance`（ticket-level）兩者抽象層級差異的段落，未來 PM 或代理人可能誤用 `/goal` 取代 acceptance 驗收，造成驗收鏈路失效。

### 影響範圍

- `.claude/references/agent-dispatch-template.md`（新增章節）

### 相關 Error Pattern

間接相關：PC-055（AC 漂移）— 若 `/goal` 被誤用為 acceptance 替代品，acceptance 狀態可能失去同步。

## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->

### 變更摘要

新增章節「與 /goal 的邊界」至 `.claude/references/agent-dispatch-template.md`（「相關文件」章節之前）。

**新增內容**：
- 章節說明設計依據（W3-032 方案 D）
- 層級對照表：7 個維度（層級、持久性、定義者、驗證者、核心用途、可追蹤性、多條件支援）
- 不可互相取代的原因（3 點：evaluator 無法客製、死鎖風險、acceptance 鏈路完整性）
- 允許的搭配使用方式（含程式碼範例）

**Last Updated** 同步更新為 2026-05-26，版本號未 bump（版本管理由 PM 統一處理）。
