## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W1-093 complete 時，phase4-hook 的 PreToolUse 掃描命中 ticket body 的 Context Bundle 區塊（auto-extracted 逐字引用 source ANA W1-089 的 why 欄位，內含 source ticket 的階段評估歷史字面），誤判為延後話術，需臨時加 history exempt marker 繞過。

根本原因：Context Bundle 是 auto-extracted 結構化元資料（機器逐字引用 source ticket why / what），與 frontmatter 同類——非人類撰寫的決策論述。W1-092 方案 b 的 `compute_frontmatter_lines` 僅跳過 frontmatter YAML 區塊，未涵蓋 Context Bundle。任何 spawned ticket 只要 source why 含階段名稱歷史字面都會復發此 false positive（PC-142 case 5 同根因，不同結構位置）。

### 影響範圍

- `.claude/hooks/phase4-decision-enforcement-hook.py`（新增 `compute_context_bundle_lines` + 兩處整合）
- `.claude/hooks/tests/test_phase4_decision_enforcement_hook.py`（新增測試）

### 相關 Error Pattern

- PC-142 case 5（frontmatter 同根因，W1-092 已修 frontmatter 面向）
- PC-093（hook 設計原意，本案為其 false positive 子模式）

### 設計決策：錨定 auto-extracted marker（非整個 section）

採「錨定 `<!-- auto-extracted: ... -->` marker」而非「跳過整個 `## Context Bundle` section」。

- **Why**：marker 是「機器產生引用」的 SSOT。若 Context Bundle 為人工撰寫（無 marker），則不跳過——人工延後論述仍應被攔截。
- **邊界**：起點 = 含 auto-extracted marker 的行（含該行）；終點 = 下個 H2（`## `）或 `---`（不含邊界行），與 `compute_schema_placeholder_lines` 完全同構。

### 實作規格（mirror W1-092 / compute_schema_placeholder_lines）

1. 新增常數（near line 100，SCHEMA_PLACEHOLDER 常數旁）：
   `CONTEXT_BUNDLE_START = re.compile(r"<!--\s*auto-extracted\s*:")`
   終點複用既有 `SCHEMA_PLACEHOLDER_END_H2`（`^\s*##\s`）與 `SCHEMA_PLACEHOLDER_END_HR`（`^\s*---\s*$`）。
2. 新增函式 `compute_context_bundle_lines(lines)`（仿 `compute_schema_placeholder_lines` 約 L323-345 結構，`in_block` 線性掃描，回傳區塊行號集合）。
3. 整合兩處：
   - `scan_lines_for_phrases`（約 L425 後，frontmatter/placeholder/fenced 呼叫旁）：`context_bundle_lines = compute_context_bundle_lines(lines)`，迴圈內 `if idx in context_bundle_lines: continue`
   - `collect_exempt_markers`（約 L506 後）：同樣呼叫 + short-circuit
4. 注意：H3（`### `）不可被誤判為終點（`^\s*##\s` 第三字元需空白，"###" 天然不匹配，無需特判）。

### 測試需求（mirror W1-092 的 7 測試，adapt 為 Context Bundle）

| # | 測試情境 | 期望 |
|---|---------|------|
| 1 | Context Bundle auto-extracted 區塊含階段名稱字面 | 不命中（跳過） |
| 2 | 一般 body 內文含階段名稱字面（非 Context Bundle） | 仍命中（regression 防護） |
| 3 | 人工撰寫 Context Bundle（無 auto-extracted marker） | 不跳過（人工延後論述仍攔截） |

| 7 | 區塊內 H3 子標題 | 不誤判為終點邊界 |

撰寫測試資料時用 fixture 字串，避免在測試檔本身寫入觸發 hook 的字面。
