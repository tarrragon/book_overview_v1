**實際採用 means**：手動 Edit package.json + manifest.json + CHANGELOG.md

**為什麼 means 偏離**：

W3-042 修復後 version-release CLI 可用（install + path discovery），但執行 `start --version 0.19.0 --dry-run` 在 Step 2「重複檢查」失敗：
```
[FAIL] 版本 0.19.0 已存在於 todolist.yaml（狀態: active）
```

`start` 命令設計為「啟動新版本」，前提是 todolist 無該版本條目。本場景為「v0.19.0 已先啟動但漏 bump package.json/manifest.json」——歷史遺漏修正情境，不符 start 命令前提。

CLI 無 `--force` flag 跳過此檢查；其他子命令（release / check / update-docs）皆非 bump-only 功能。

**結論**：原 AC1 means 對「歷史遺漏 bump」場景不適用，但其 outcome（package.json/manifest.json bump 為 0.19.0）由手動 Edit 等效達成，且 outcome AC2-5 全部 PASS。AC1 spirit 達成、means 偏離已顯性記錄。
