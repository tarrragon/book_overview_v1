**症狀時序**：
1. W3-082 完成：修 release()、reinstall、runtime 驗證 W3-021 release → 「狀態: 待處理」（修復版 STATUS_LABELS 才印得出此字串）✓
2. 稍後 release W3-086 → 「狀態: 被阻塞」（舊行為復現）✗
3. 比對 installed vs source release()：installed 為舊版（寫死 STATUS_BLOCKED），source 為修復版（依 blockedBy + STATUS_LABELS）

**關鍵證據**（`uv tool install .claude/skills/ticket --reinstall` 輸出）：
```
- ticket-system==1.0.0 (from file:///Users/mac-eric/project/screen_clock/.claude/skills/ticket)
+ ticket-system==1.0.0 (from file:///Users/mac-eric/project/book_overview_v1/.claude/skills/ticket)
```
installed 來源是 **screen_clock 專案**（另一專案的 ticket skill 副本，無 W3-082 修復），非本專案。

**結論**：`ticket` 全域 uv tool 被 screen_clock 專案的並行 session / hook reinstall 覆蓋，蓋掉本專案版本。reinstall 回本專案版本後，release W3-086 → 「狀態: 待處理」恢復正常。

**受影響面**：所有跨專案共用名稱的全域 uv tool skill CLI（ticket / doc / worktree / mermaid-ascii / project-init / version-release）。

**既有防護的缺口**：uv-tool-staleness-check-hook 僅 SessionStart 比對且僅提示，無法防 mid-session 被他專案覆蓋。
