## Problem Analysis

### 問題根因

W1-050 ANA 確認 process.memoryUsage 在 background SW runtime 觸發崩潰；esbuild platform:browser 不 polyfill `process` 物件，SW 執行到任一 process.memoryUsage 即 ReferenceError。本 ticket 為 W1-050.3 拆分 C2 範圍（W1-050.3 Problem Analysis 含完整 8 檔 23 hits 表與拆分決策）。

C2 特性：模式單一（全部 process.memoryUsage 變體），但檔案多（5 檔），測試覆蓋廣。

### 轉換規則（W1-050.3.1 共用，本 ticket 只涉及 C 規則）

**`process.memoryUsage()` → `performance.memory` 含 guard**

- `performance.memory` Chrome-only（V8 / Chromium），需 typeof guard 防呼叫崩潰
- 對應欄位：
  - `process.memoryUsage().heapUsed` → `performance.memory.usedJSHeapSize`
  - `process.memoryUsage().heapTotal` → `performance.memory.totalJSHeapSize`
  - `process.memoryUsage().rss` → 無對應，回傳 0 或 omit
- 共用 helper 範本（建議放 `src/background/domains/data-management/services/` 或共用 util，避免 5 檔重複）：
  ```js
  function getMemoryUsage () {
    if (typeof performance !== 'undefined' && performance.memory) {
      return {
        heapUsed: performance.memory.usedJSHeapSize,
        heapTotal: performance.memory.totalJSHeapSize,
        unavailable: false
      }
    }
    return { heapUsed: 0, heapTotal: 0, unavailable: true }
  }
  ```
- 各檔內 `process.memoryUsage().heapUsed` 替換為 `getMemoryUsage().heapUsed`

### 設計權衡

5 檔重複呼叫相同模式 → 建議抽 helper（DRY 原則）。可選位置：
- 新建 `src/background/domains/data-management/services/memory-usage-helper.js`（最小變更）
- 或放到既有 util（需先 grep 確認既有 util 位置）

若 helper 抽出，須加 RED 測試覆蓋兩條路徑（performance.memory 存在 / 不存在）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要

依 Problem Analysis 規則 C 將 5 檔 10 處 `process.memoryUsage` 全部替換為 `performance.memory` 含 typeof guard 的 inline pattern。未抽共用 helper（與 sibling W1-050.3.1 相同決策），避免新增模組依賴與 esbuild bundle 複雜度。

### 統一轉換 Pattern

所有 hit 點採用一致的 inline typeof guard：

```js
// 單欄位 (heapUsed) 取值
const heapUsed = (typeof performance !== 'undefined' && performance.memory)
  ? performance.memory.usedJSHeapSize
  : 0

// 多欄位物件取值（保留 external 欄位但回傳 0，performance.memory 無對應）
let memoryUsage
if (typeof performance !== 'undefined' && performance.memory) {
  memoryUsage = {
    heapUsed: performance.memory.usedJSHeapSize,
    heapTotal: performance.memory.totalJSHeapSize,
    external: 0
  }
} else {
  memoryUsage = { heapUsed: 0, heapTotal: 0, external: 0 }
}
```

### 各檔變更明細

| 檔案 | hit | 變更後 |
|------|-----|-------|
| cache-management-service.js | L476 `getMemoryUsage()` | 改 typeof guard + 回 `{ heapUsed, heapTotal }`，fallback 全 0 |
| data-validation-service.js | L1209 performance.memoryUsage | inline guard 取 `heapUsed`，fallback 0 |
| ValidationServiceCoordinator.js | L796 `_getCurrentResourceUtilization` | inline guard，fallback 0 |
| ValidationServiceCoordinator.js | L1018 `_calculateResourceUtilization` | inline guard，fallback 0 |
| ValidationServiceCoordinator.js | L1103 `_calculateOverallResourceUtilization` | 多欄位 if/else，保留 `external: 0`，fallback 全 0 |
| ValidationServiceCoordinator.js | L1135 `_calculateResourceUtilizationSummary` | inline guard，fallback 0 |
| conflict-resolution-service.js | L631-632 `enableMemoryOptimization` 區塊 | 提一個 heapUsed 變數，peak/final 共用，fallback 0 |
| conflict-resolution-service.js | L1058 `getResourceUsage` | inline guard，fallback 0 |
| batch-validation-processor.js | L692 `isBatchProcessorHealthy` | 多欄位 if/else，保留 `external: 0`，fallback 全 0 |

### 註解策略

每處轉換點上方加註：
- `performance.memory 為 Chrome-only API`
- `typeof guard 保留：跨環境安全、Firefox / Safari / Jest jsdom 無此 API`
- 多欄位處額外註：`heapUsed = usedJSHeapSize，heapTotal = totalJSHeapSize，external 無對應回傳 0`

### 與 sibling W1-050.3.1 (C1) 對齊

C1 的 `event-performance-monitor.js _measureMemory()` 採 performance.memory primary + Node `process.memoryUsage` typeof-guard fallback（保留 Node 測試環境支援）。本 ticket 採純 performance.memory + 數值 0 fallback（無 process fallback），原因：

1. background SW runtime 永遠無 `process`，保留 process fallback 會引入 `no-undef` ESLint 例外 + esbuild platform:browser 不 polyfill
2. 5 檔的調用點都是 metric / monitoring 性質，fallback 為 0 不影響業務邏輯正確性
3. Jest jsdom 測試環境也走 performance.memory 不存在的 fallback，回 0 安全

### 不在本 ticket 範圍

- `src/background/domains/data-management/services/cross-device-sync-service.js` 等其他檔（無 process.memoryUsage 命中）
- 共用 helper 抽取（5 處 inline pattern 重複度可接受，避免新增依賴模組）
- C1 範圍（export+error-handling 3 檔已 merged）
