## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W3-071 session 期間 PM 嘗試 `ticket track set-where 0.19.0-W3-071 --layer 'Framework Rules'` 失敗（`unrecognized arguments: --layer`）。現有 `skill-cli-error-feedback-hook` 僅將此類錯誤分類為「SKILL.md 文檔缺失」並建議補文檔，但實際根因為 CLI 缺少 dict 欄位子欄位寫入路徑（系統功能缺口），補文檔無法解決使用者意圖。

### 影響範圍

- `.claude/hooks/skill-cli-error-feedback-hook.py`：新增 classify 邏輯與系統功能缺失輸出路徑
- `.claude/hooks/tests/test_skill_cli_error_feedback_hook.py`：新增 8 個測試覆蓋三類分類與主流程整合

### 相關 Error Pattern

無既有 PC 直接對應；W3-072 ANA 評估 set-where 子欄位 flag 缺口為對應系統功能側分析。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

升級 `skill-cli-error-feedback-hook.py` 加入三類分類邏輯：

1. **新增資料結構**：
   - `DICT_FIELD_SUBFIELDS`：對應 ticket frontmatter dict 欄位（where/who/how）的已知子欄位清單
   - `CLASSIFICATION_*` 常數：USER_TYPO / SYSTEM_GAP / SKILL_DOC 三類

2. **新增純函式**：
   - `detect_system_gap(command, stderr, stdout)`：偵測 `set-<dict-field>` + 子欄位 flag 拒絕的訊號組合，返回 `{subcommand, flag, known_subfields}` 或 None
   - `classify_error(command, stderr, stdout)`：三類分類入口（排除 → 系統缺口 → SKILL 文檔）

3. **新增輸出範本**：`SYSTEM_GAP_FEEDBACK_TEMPLATE` 含 `ticket track create --type ANA` 命令骨架與 W3-072 reference case

4. **主流程接入**：在既有 `detect_skill_error_type` 路徑前插入系統功能缺失偵測，命中即輸出 ANA 引導；未命中則 fallback 回既有 SKILL 文檔回饋路徑（向後相容）

### 設計決策

- **優先序**：system gap 偵測優先於 SKILL doc 分類，避免「補文檔誤導使用者」
- **訊號嚴格性**：必須同時滿足 `unrecognized arguments` 錯誤 + 已知 dict 子命令 + 已知子欄位 flag，避免泛化誤判
- **向後相容**：非 dict 子欄位的 `unrecognized arguments`（如 `--bogus-flag`）仍走原 SKILL 文檔路徑

### 不在本 ticket 範圍

- CLI 端是否真正補 set-where --layer flag → W3-072 ANA 評估後另行 spawn IMP
- `who` / `how` dict 子欄位的 CLI 實作 → 同上由 W3-072 評估
