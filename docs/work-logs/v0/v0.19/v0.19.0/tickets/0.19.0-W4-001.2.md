## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要

新增獨立 E2E 測試檔 `tests/e2e/browser/uc04-import.e2e.test.js`（10 test，2 group），覆蓋 UC-04 匯入鏈完整真實瀏覽器路徑。重用既有框架（ExtensionTestSetup / storage-reader helpers / timeouts），未改既有 3 測試檔（含 uc03），未重建基礎設施，未改任何 src。

### 真實鏈路設計（檔案注入而非繞過）

採 Puppeteer `elementHandle.uploadFile` 注入既有 fixture 至 `#jsonFileInput`，完整經過真實 click `#loadFileBtn` -> `OverviewPageController.handleFileLoad` -> `ImportFlowController.execute`（validate -> 模式 modal -> read -> 持久化）-> `TagStorageAdapter.replaceAllData / mergeAllData` -> `chrome.storage.local` 全鏈。僅替換「OS 檔案對話框選檔」這一步以注入 fixture，不繞過任何匯入業務邏輯。模式 modal（promptImportMode）以真實 click `#importModeOverwriteBtn` / `#importModeMergeBtn` resolve，完整觸發模式分流。

持久化為 async，採 polling 等待 storage 書數穩定（禁止固定 sleep，遵循 test-assertion-design-rules）。

### 測試覆蓋

G1 UC-04 v1 格式匯入自動轉換（acceptance #1，覆蓋模式）：
- G1-1 [SETUP] v1 匯入後 storage 寫入 2 本書
- G1-2 [V1] 每本書 readingStatus 為 6 狀態 enum 合法值，無 isNew/isFinished 殘留
- G1-3 [V1] isFinished=true -> finished、進行中 (progress=30) -> reading（BookSchemaV2.mapV1StatusToV2 驗證）
- G1-4 [V1] 每本書 tagIds 恆為陣列（INV-1）
- G1-5 [V1] v1 category 轉為 tag，書 tagIds 指向有效 tag（引用完整性）

G2 UC-04 v2 合併模式 tag 聯集 + 去重（acceptance #2）：
- G2-1 [SETUP] base 覆蓋載入 + merge 合併載入前置完成
- G2-2 [MERGE] 合併後書數恆 2（同 id 去重不複製、本地書原樣保留）
- G2-3 [MERGE] 同 id 書 (uc04-book-x) tagIds 取聯集（base 科幻 + merge 推理 = 2 個，去重無重複）
- G2-4 [MERGE] 未涉合併本地書 (uc04-book-y) tagIds 原樣保留（維持 1 個 tag「文學」）
- G2-5 [MERGE] tags 總數為 3（科幻 + 文學 base，推理為合併新建）

### 框架慣例調整（不改既有 helper）

- storage-reader.js 的 readBooksFromStorage 僅讀 readmoo_books；UC-04 另需讀 tags / tag_categories 驗證合併結果，故在本測試檔內自帶通用 readStorageKey helper（重用既有 getServiceWorker），不修改既有 storage-reader（遵守「不改既有」約束）。
- 另在測試檔內提供 clearAllImportStorage（清 books + tags + tag_categories 三 key），確保 v1 / v2 案例從乾淨狀態開始；clearBooksStorage 僅清 readmoo_books 不足。

### spike 修復（W4-001.2 過程記錄）

首次執行 G2 合併匯入 beforeAll 報 "Node is either not clickable or not an Element"。根因：`#importJSONBtn` 監聽器為 toggle `#fileUploader` 顯示（display none <-> block），G1 已 click 一次使 uploader 顯示，G2 再 click 把它切回隱藏導致 `#loadFileBtn` not clickable。修正：importFixture 改以 evaluate 直接設定 `#fileUploader` display='block'（冪等可見），不依賴 toggle 擺盪。此操作僅還原 UI 可見性，真實匯入仍經 `#loadFileBtn` click -> handleFileLoad 全鏈。
