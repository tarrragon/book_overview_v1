### UC-04 匯入入口（src/overview/）

- HTML 按鈕：`#importJSONBtn`（📥 匯入 JSON，overview.html L32）/ `#loadFileBtn`（📂 載入檔案 L54）
- 匯入模式 modal：`#importModeOverlay`（overview.html L90，預設隱藏）→ `#importModeOverwriteBtn`（覆蓋）+ 合併按鈕（由 `promptImportMode` 控制顯示）
- 匯入邏輯：`import-flow-controller.js` → `book-file-importer.js` → `import/file-reader.js` + `import/file-validator.js`

### UC-04 規格要點（docs/use-cases.md L189-260）

- v1 格式匯入自動轉換：isNew/isFinished → 6 種閱讀狀態、無 tag → 空 tag 集合
- v2 格式匯入：直接載入，tag 依模式處理（合併=聯集 / 覆蓋=完全替換）
- 合併模式 tag 採聯集策略（匯入 tag 加入既有，不刪除）+ 去重
- 成功標準：格式驗證準確率 100%（v1/v2 皆可）、載入完整性 100%、去重 > 99%

### E2E 設計策略

1. beforeAll：`setup.setup()` → `clearBooksStorage` → 開 overview 頁
2. 檔案注入：E2E 環境無真實檔案選擇對話框，採 `page.evaluate` 直呼 import 入口 API 傳入 fixture JSON 字串，或 Puppeteer `elementHandle.uploadFile` 注入 `<input type=file>`（擇可行者，參考既有 import-flow-controller 介面）
3. v1 案例：匯入 sample-v1.json，斷言 storage 書物件 readingStatus 為 6 狀態之一、tagIds 為陣列
4. v2 merge 案例：先載入 sample-v2-base，再以合併模式載入 sample-v2-merge，斷言 tag 聯集 + 去重後總數正確
5. 新增獨立 test 檔，不改既有測試檔

### 執行前置與指令

- 前置：`npm run build:dev`
- 執行：`npm run test:e2e:browser`（須全綠，含既有 uc03 + 新增 uc04）

### 邊界與防護

- 若 import 入口無法在 E2E 環境直接觸發（檔案對話框限制）→ 採 evaluate 直呼 importer API；若連 API 都無法繞過 → 填 NeedsContext 回報，勿臆測
- 若發現匯入實作不符 v1/v2 規格 → 屬 regression 發現，建 follow-up（規則 5），**勿擅改 src**
- 接手者應獨立重新驗證入口行號/欄位/fixture 結構（PC-162）
