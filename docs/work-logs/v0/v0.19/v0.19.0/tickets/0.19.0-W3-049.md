## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 根因結論

本案為 **PC-105（subagent commit 後未 complete）** 在 haiku low-effort model 的重現，加上 **commit policy 違反**（直接在 main 而非 worktree）。兩個疏失共振的根因：

1. **Model 能力限制**：haiku + effort: low 對 AGENT_PRELOAD 長文規則（規則 2.4 收尾責任，第 70-90 行）的 attention 不足，導致收尾步驟被跳過
2. **Agent 定義缺口**：mint-format-specialist.md 無收尾責任段落，完全依賴 AGENT_PRELOAD 傳遞收尾規範；haiku model 對共享 preamble 的落地率低於 sonnet/opus
3. **Commit Policy 傳遞失敗**：worktree 指示寫在 ticket Problem Analysis（Commit Policy 段落），而非 prompt 第一行；haiku 對 ticket 內嵌長文的 attention 不足以提取此指示

### 模式判定

本案屬 **已知模式（PC-105）在新 model 層級的重現**，非新模式。但暴露一個 model-specific 的落地率缺口：

- sonnet/opus model 對 AGENT_PRELOAD 規則 2.4 的落地率尚可（既有 PC-105 案例多為偶發）
- haiku model 對 AGENT_PRELOAD 的落地率系統性偏低（本案為 100% 未落地）

### 防護建議（三層）

| 層級 | 措施 | 成本 | 優先級 |
|------|------|------|--------|
| Agent 定義層 | 在 mint-format-specialist.md 新增「收尾責任」段落，重複 AGENT_PRELOAD 規則 2.4 核心指令（check-acceptance + complete） | 低 | P1 |
| 派發 prompt 層 | 對 haiku model 派發時，prompt 末段顯式加入 worktree + 收尾指令（而非僅依賴 ticket 內嵌 Commit Policy） | 低 | P1 |
| Hook 層 | branch-verify-hook 已存在但本案未觸發（mint 非 worktree 派發）；考慮對 .claude/ 檔案的 commit 增加 branch 檢查 | 中 | P2 |
