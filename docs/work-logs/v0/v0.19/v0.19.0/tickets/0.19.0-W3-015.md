### (b) 規則內容骨架（三明示）

#### `.claude/` 修改派發決策矩陣

**Why**：`.claude/` 目錄受 CC runtime hardcoded 保護（ARCH-015），worktree 內 `.claude/` 的 Edit/Write 被 runtime 拒絕。主 repo cwd 的 subagent 可成功 Edit `.claude/`（PC-115 實證 18/18），但受並行數限制（PC-137 <= 2）。PM 跨 session 易誤判為「完全擋死」或「加 worktree 就行」。

**Consequence**：誤判導致三種浪費：(1) 可派發但 PM 自己做（低估 subagent 能力）；(2) 用 worktree 派發後被 runtime 擋（高估 worktree）；(3) 跨 `.claude/` + `src/` 未拆分（subagent 只完成一半）。

**Action**：派發前依以下矩陣判斷：

| 修改目標 | 派發方式 | isolation 參數 | 並行限制 |
|---------|---------|---------------|---------|
| 僅 `.claude/` | subagent 主 repo cwd | 不加 worktree | <= 2 並行（PC-137） |
| 僅 `src/` 或非 `.claude/` | subagent worktree | isolation: worktree | 無特殊限制 |
| `.claude/` + `src/` 混合 | 拆分為兩個 ticket | 各依上述規則 | 分別計數 |
| `.claude/` 且緊急 / 並行已滿 | PM 前台 | N/A | N/A |
| 僅讀取 `.claude/`（不修改） | 任何方式皆可 | worktree 或主 repo 皆可 | 無限制 |
