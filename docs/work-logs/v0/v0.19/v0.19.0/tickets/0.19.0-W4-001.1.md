### 問題根因

### 影響範圍

- 新增：`tests/e2e/browser/uc03-export.e2e.test.js`（獨立測試檔，不改既有 2 檔）
- 受測 src（只讀，禁改）：`src/overview/book-exporter.js`、`src/overview/overview.html`（`#exportJSONBtn` L31 / `#exportCSVBtn` L30）

### 框架慣例（重用，禁重建基礎設施）

| 模組 | 路徑 | 提供 |

| mock 頁面 | `tests/e2e/fixtures/readmoo-mock-page.html` | 5 本書 fixture |

### UC-03 規格要點（docs/use-cases.md L139-187）

- JSON 匯出符合 Interchange Format v2：root 含 `books` + `version` + `tags`（按類別）；book 無 tag 時匯出空陣列非省略欄位
- CSV 匯出含 tag 欄位（分隔符序列化）+ 6 種閱讀狀態欄（詳 `docs/spec/csv-export-spec.md`）
- 成功標準：匯出資料完整性 100%、JSON 符合 v2、CSV 可開啟

### E2E 設計策略

1. beforeAll：`setup.setup()` → `clearBooksStorage` → `navigateToFixture` → 觸發提取（FIXTURE_BOOK_COUNT 本）→ 開 overview 頁（`chrome-extension://<id>/src/overview/overview.html`，event-driven 自動讀 storage）
2. 匯出觸發：overview 頁 click `#exportJSONBtn`，Puppeteer 攔截下載 blob；若 isolated profile 下載定位困難（W1-001.2 曾遇），改用 `page.evaluate` 直呼 book-exporter API 取輸出字串斷言
3. 斷言：解析匯出 JSON 斷言 root 結構（books/version/tags）；CSV 斷言 header 含 tag/狀態欄

### 執行前置與指令

- 前置：`npm run build:dev`（產出 build/development，extension 載入需要）
- 執行：`npm run test:e2e:browser`（注意：`npm run test:e2e` 以 testPathIgnorePatterns 排除 browser/，新增測試須用 `test:e2e:browser`）

### 邊界與防護

- 若發現匯出實作不符 v2 規格 → 屬 regression 發現，建 follow-up ticket（quality-baseline 規則 5），**勿擅改 src**
- 接手者應獨立重新驗證入口行號/欄位（PC-162）
