### 問題根因

phase4-decision-enforcement-hook 對 ticket md frontmatter (YAML 區塊) 內 source ticket 歷史引用做 phrase 偵測。但 frontmatter 是結構化元資料 (title / why / strategy 欄位儲存的歷史脈絡引用)，非人類撰寫的決策論述，依規則應整段跳過 (PC-142 case 5)。

### 影響範圍

- `.claude/hooks/phase4-decision-enforcement-hook.py` (scan_lines_for_phrases / collect_exempt_markers)
- `.claude/hooks/tests/test_phase4_decision_enforcement_hook.py` (新增 W1-092 區段 7 個測試)

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

採 W1-089 ANA 方案 (b)：在 phase4-decision-enforcement-hook 新增 `compute_frontmatter_lines(lines)` 函式，於 `scan_lines_for_phrases` 與 `collect_exempt_markers` 內整段跳過 frontmatter 區塊。

### 設計決策

1. **Frontmatter 邊界匹配限「行首僅有 `---` 三字元」**：PM WRAP P 防護，避免內文水平分隔符 `---` 誤判為 frontmatter 結束。
2. **未閉合 frontmatter 視為無 frontmatter (容錯)**：回傳空集合，避免整檔被跳過。
3. **第一行非 `---` 視為無 frontmatter**：與 YAML frontmatter spec 一致。
4. **行級 short-circuit 與既有 fenced / placeholder 跳過邏輯並列**：類比 W10-130 / W11-018 設計。
