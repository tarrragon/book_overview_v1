## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W5-003 parallel-evaluation 三人組審查（basil-writing-critic）發現 Critical-1 設計層衝突：

`docs/spec/extraction/e2e-contract.md` §3.4 Console Log 前綴規則表內含 4 處 emoji 字面引用（U+1F4CD 圖釘符 / U+274C 叉號）。這些 emoji 是 `src/content/detectors/page-detector.js` L63 和 `console.error` 多處的 log 前綴字面，受 grep 對齊機制保護（修改字面即破壞契約驗證）。

但 `.claude/rules/core/language-constraints.md` 規則 3「所有文字輸出禁止使用 emoji 符號」屬絕對禁令，**無豁免條款**。產生設計層衝突：刪除 emoji 破壞 grep 對齊機制；保留 emoji 違反規則 3。

W5-003.3 §3.4 已加短期豁免聲明作局部防護，但屬「規格內局部豁免」未升級為框架層規則，違反 DRY；後續新規格文件遇相同情境須各自決策。

### 影響範圍

- `.claude/rules/core/language-constraints.md` 規則 3（規則層需補豁免條款）
- `docs/spec/extraction/e2e-contract.md` §3.4（短期豁免聲明需引用收斂）
- 未來新規格文件（跨專案推廣性）
- hook / 審查者對規格內 emoji 的判定（避免誤判為違規）

### 相關 Error Pattern

- PC-072 askuserquestion-payload-charset-contamination（emoji 污染 PM token pool 案例）
- W12-002.3 設計原則（規則檔內用「裝飾類別 + Unicode codepoint」描述避免自動載入污染）
- PC-080 自動載入字元集污染累積鏈

## Solution

### 盤點階段

#### 規格類文件 emoji 引用分布

| 檔案 | emoji 引用數 | 用途 |
|------|------------|------|
| `docs/spec/extraction/e2e-contract.md` | 5 處（page-detector log ×4 + console.error 失敗前綴 ×1） | 機械引用 src code log 字面 |
| `.claude/rules/` | 0 處 | 已執行 W12-002.3 設計原則：用「裝飾類別 + Unicode codepoint」描述 |
| `.claude/error-patterns/PC-072` | 2 處列舉 | 展示禁用 emoji 違規型態 |
| `docs/bookstores/`, `docs/use-cases.md`, `docs/proposals/`, `docs/work-logs/` | 0 處 | 已遵循禁令 |

#### 當前 language-constraints 規則 3 狀態

- L39「所有文字輸出禁止使用 emoji 符號」絕對禁令
- L41-49 禁令表已採 W12-002.3 設計原則（用 codepoint 描述）
- 規則 3 無豁免條款

### 三方案利弊評估

#### 方案 A：language-constraints 規則層補豁免條款

操作：規則 3 後新增豁免段，明示 `docs/spec/**/*.md` 對 src code log 字面的機械引用允許 emoji，限定「結構化前綴對應表」+ 字面完全一致兩條件。

| 維度 | 評估 |
|------|------|
| Grep 對齊機制 | 保留（規格與 src 用相同字元） |
| 跨專案推廣性 | 高（適用所有專案的 spec 文件） |
| W12-002.3 自我一致性 | 需精心措辭：豁免條款描述本身須用 codepoint 而非 emoji |
| 邊界明確性 | 限定 docs/spec/ + 結構化前綴對應表 + 機械引用三條件 |
| 落地成本 | 低（單檔 Edit） |

#### 方案 B：改用 Unicode codepoint 標記

操作：規格內 emoji 改為 `U+1F4CD` 等 codepoint 標記。

| 維度 | 評估 |
|------|------|
| Grep 對齊機制 | 破壞（src 用字元，規格用 codepoint） |
| 跨專案推廣性 | 可行但增加維護成本 |
| W12-002.3 自我一致性 | 完全符合 |
| 邊界明確性 | 規則無例外 |
| 落地成本 | 中（規格 + 工具鏈轉換層） |

#### 方案 C：規格文件全數移除 emoji

操作：所有 spec 內 emoji 刪除，僅保留文字描述。

| 維度 | 評估 |
|------|------|
| Grep 對齊機制 | 完全破壞（無字面可 grep） |
| 跨專案推廣性 | 違反「契約字面 = src 字面」spec 本職 |
| W12-002.3 自我一致性 | 完全符合 |
| 邊界明確性 | 規則無例外 |
| 落地成本 | 低，但破壞契約驗證機制故不採 |

### 結論

**採方案 A 變體**：language-constraints 補豁免條款，描述本身用 codepoint 標記避免 W12-002.3 違反。

**Why**：方案 C 破壞 spec 本職與 grep 對齊不採；方案 B 破壞 grep 對齊增加維護成本；方案 A 唯一同時保留 grep 對齊 + 維持 language-constraints 規則自身一致性。

**Consequence（不採 A）**：§3.4 短期豁免聲明屬規格內局部，未升級為框架規則時新規格文件須各自決策違反 DRY；hook / 審查者可能誤判規格內 emoji 為違規。

**Action**：spawn IMP ticket 落地 language-constraints.md 規則 3 補豁免條款。
