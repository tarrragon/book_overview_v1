### 三明示落實

每段論述含「Why / Consequence / Action」三層：
- 觸發識別：Why=描述被寫時為真，環境會演進；Consequence=誤派或關錯 ticket；Action=claim 後第一 tool call 即驗證
- 考古 5 指令：Why=5 維度覆蓋；Consequence=跳過部分產生位置誤判；Action=記錄每指令輸出
- 三問：Why=三維度獨立判斷；Consequence=逆向修復；Action=各寫一行進 Problem Analysis
- 需求重評：Why=三種完全不同動作；Consequence=PC-162 案例 A 再現；Action=明示判定結果
- 落地：Why=必須留在 ticket md；Consequence=下次重做考古；Action=不可省略
- 強制四問：Why=ticket 描述 vs PM 自身判斷互補；Consequence=stale memory 靜默誤判；Action=印在認領後第一步驟
