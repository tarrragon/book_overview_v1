### 問題根因

來源 ANA `0.19.0-W3-038` 重現實驗證實 `handoff-auto-resume-stop-hook.py`（939 行）承載 3 大 domain（session 管理 / ticket-handoff 判定 / scan-決策-output），超過 cognitive-load.md「單檔 domain 數 > 2 必須拆分」閾值。本 IMP 執行 ANA 建議的方案 2「部分拆分」，將 session 管理 domain 抽出為獨立模組，使原檔降至 domain 數 2。

### 影響範圍

| 項目 | 內容 |
|------|------|
| 待抽出函式（session 管理 domain，原檔約 142-229 行，約 88 行） | `get_session_stop_flag` / `get_session_state_file` / `has_been_triggered_this_session` / `mark_triggered_this_session` / `read_session_state` |
| 待移除 dead code | `scan_in_progress_tickets`（DEPRECATED，原檔約 447-464 行，約 18 行，空實作 `return []`） |
| 新建模組 | `.claude/skills/ticket/hooks/handoff_session_mgmt.py`（底線命名，可正常 import） |
| 原檔 | `.claude/skills/ticket/hooks/handoff-auto-resume-stop-hook.py`，import 新模組取代內嵌定義 |
| 測試檔 | `.claude/hooks/tests/test_handoff_auto_resume_stop_hook.py`（40 個 test）；若測試直接呼叫被抽出函式，import 路徑需同步更新 |
| cache 不受影響 | 被抽出的 5 個函式均屬 session-flag 讀寫，不涉及 `frontmatter_cache`（ANA cache 作用域分析已確認） |

### 設計風險（接手者注意）

- 原檔 `handoff-auto-resume-stop-hook.py` 為連字號檔名 hook script，含 `sys.path` 操作（原檔約 83-109 行）。新模組與原檔同目錄，原檔 import 新模組需確認 hook 執行時該目錄在 `sys.path` 上。
- 測試檔以 `importlib` 載入連字號 hook 為 `hook` 模組；新模組為合法識別符可直接 import，但測試需確認 path 設定涵蓋新模組。
- 被抽出 5 函式為呼叫圖葉節點（由 `generate_hook_output` 呼叫，彼此間無交叉依賴），抽出後行為應完全不變。

### 相關 Error Pattern

無直接對應 error-pattern。本 IMP 為 W3-038 ANA 的 spawn 落地，屬結構重構非缺陷修復；既有 40 個測試為行為不變的回歸防線。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

執行 ANA 0.19.0-W3-038 建議的方案 2「部分拆分」，將 session 管理 domain 從 handoff-auto-resume-stop-hook.py 抽出為獨立模組。

### 變更內容

| 項目 | 內容 |
|------|------|
| 新建模組 | `.claude/skills/ticket/hooks/handoff_session_mgmt.py`（147 行，底線命名可直接 import）|
| 移入函式 | get_session_stop_flag / get_session_state_file / has_been_triggered_this_session / mark_triggered_this_session / read_session_state |
| 移入常數 | STOP_FLAG_FILE / STOP_FLAG_EXPIRY_SECONDS / STATE_FILE_TEMPLATE（隨 session domain 一併遷移）|
| 原檔變更 | import 新模組取代內嵌定義；移除 DEPRECATED scan_in_progress_tickets（空實作 return []）；移除不再使用的 `import os` |
| 測試同步 | test_stop_hook.py:56 STOP_FLAG_FILE 引用改指向新模組（新增 load_session_mgmt_module helper）|

### 設計決策

- import path：原檔上方 `sys.path.insert(.../"hooks")` 已涵蓋 handoff_session_mgmt 的 hook_utils 依賴；額外補加 `sys.path.insert(__file__ 父目錄)`，因 hook 以絕對路徑執行時自身目錄不在 sys.path 上，確保連字號檔名 hook 可 import 同目錄的合法識別符模組。
- 影響範圍分析（IMP-003 防護）：grep 全 .claude/ 確認 5 個移入函式與 3 個常數無 hook 檔以外的引用；唯一外部引用為測試檔 test_stop_hook.py 對 STOP_FLAG_FILE 的引用，已同步更新。
- os import：移入的 get_session_state_file 是原檔唯一 os 使用點，抽出後原檔 `import os` 成為 dead import，一併移除。

### 行為不變驗證

5 個移入函式為呼叫圖葉節點（由 generate_hook_output 呼叫，彼此無交叉依賴），抽出後行為完全不變。frontmatter_cache（PC-097）不涉及本模組，cache 作用域未受影響。

### Phase 4 評估結論

本 ticket 即為結構重構任務（ANA W3-038 規劃的拆分落地）。重構後：原檔 domain 數從 3 降至 2（ticket-handoff 判定 + scan-決策-output），達 cognitive-load.md 閾值邊界；新模組 147 行單一 domain（session 管理），認知負擔低。新模組函式無巢狀超 3 層、無魔法數字。無需額外重構。
