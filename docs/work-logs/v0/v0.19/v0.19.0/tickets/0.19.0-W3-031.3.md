## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W3-031 parent ANA 識別 PostToolUse `continueOnBlock` 為 v2.1.139 新能力：當 PostToolUse hook exit 2（block）時，設定 `continueOnBlock: true` 可讓 Claude 仍收到 hook 回饋訊息並自行修正，而非靜默停止。專案有 6 個 PostToolUse matcher group，需逐一評估整合價值。

### 影響範圍

6 個 PostToolUse matcher group，共 38 個 hook command（29 個唯一 .py 檔案）：
- Bash: 10 hooks
- mcp__dart__run_tests: 1 hook
- Edit: 9 hooks
- Write: 14 hooks
- Agent: 3 hooks
- catch-all: 1 hook

### 現況盤點

**已使用 continueOnBlock 的 hook**（1 個，2 處註冊）：
- `l10n-sync-verification-hook.py`：Edit + Write 各一處，已設 `continueOnBlock: true`

**可 exit 2（block）的 hook**（4 個）：
1. `phase-completion-gate-hook.py`（Bash matcher）— EXIT_BLOCK = 2
2. `phase4-decision-enforcement-hook.py`（Bash matcher）— 多處 exit 2（MUST-block 等級）
3. `ticket-id-validator-hook.py`（Write matcher）— 1 處 exit 2
4. `ticket-quality-gate-hook.py`（Write matcher）— 3 處 exit 2

**純 advisory（exit 0）的 hook**（24 個）：
- 全部其餘 hook。`continueOnBlock` 對 exit 0 hook 無行為影響。

### 相關 Error Pattern

無直接相關 error pattern。IMP-048（hook stderr triggers error）為間接相關背景。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### continueOnBlock 語意說明

`continueOnBlock: true` 僅在 hook exit 2 時有效。效果：Claude 收到 hook 的阻擋訊息（additionalContext / stderr），可據此自行修正再重試，而非靜默放棄。對 exit 0（advisory）的 hook 無行為影響。

### 逐 hook 整合機會評估

#### 1. 可 exit 2 的 hook（4 個）— continueOnBlock 有實質意義

| Hook | Matcher | 阻擋場景 | continueOnBlock 整合建議 | 理由 |
|------|---------|---------|------------------------|------|
| phase-completion-gate-hook | Bash | ticket phase 轉換條件不滿足 | 建議整合 | Claude 收到「缺少 X 條件」後可自行補完再重試，減少 PM 介入 |
| phase4-decision-enforcement-hook | Bash | ticket 含延後話術（MUST-block） | 保留現狀（不整合） | 此 hook 的 block 是強制決策閘門，Claude 自行修改延後話術可能規避審查意圖 |
| ticket-id-validator-hook | Write | ticket ID 格式錯誤 | 建議整合 | Claude 收到格式錯誤訊息後可自行修正 ID 格式再寫入 |
| ticket-quality-gate-hook | Write | ticket body 品質不達標 | 建議整合 | Claude 收到品質缺口清單後可自行補完 body 章節再重試 |

#### 2. 純 advisory hook（24 個）— continueOnBlock 無效果

| 分類 | 代表 hook | 說明 |
|------|----------|------|
| 測試回饋 | post-test-hook, cli-error-feedback-hook | 永遠 exit 0，已透過 additionalContext 回饋 Claude |
| 格式檢查 | style-guardian, worklog-format-check, comment-qa | exit 0 + 建議訊息，Claude 已能讀取 |
| 追蹤/記錄 | agent-dispatch-logger, active-dispatch-tracker, post-git-commit | 純記錄，無回饋需求 |
| 提醒類 | worktree-merge-reminder, memory-upgrade-reminder, wrap-decision-tripwire | exit 0 advisory |
| 偵測類 | deletion-residual-reference, utf8-integrity-check, ticket-skill-sync-check | exit 0 |
| 生命週期 | handoff-cleanup, post-ticket-lifecycle, needs-context-listener | exit 0 |
| 驗證類 | ticket-frontmatter-validator, ticket-creation-validation, ana-ticket-metadata-validation | exit 0（警告不阻擋） |
| 閘門類 | layer-boundary-validator, worklog-path-guard, phase-completion-gate(Edit/Write context) | exit 0 |
| 已整合 | l10n-sync-verification-hook | 已設 continueOnBlock: true |

#### 3. 整合建議總結

| 建議 | Hook | 動作 |
|------|------|------|
| 整合 | phase-completion-gate-hook (Bash) | 在 settings.json 對應註冊加 `"continueOnBlock": true` |
| 整合 | ticket-id-validator-hook (Write) | 同上 |
| 整合 | ticket-quality-gate-hook (Write) | 同上 |
| 保留現狀 | phase4-decision-enforcement-hook (Bash) | 強制決策閘門，不應讓 Claude 自行繞過 |
| 不適用 | 其餘 24 個 advisory hook | exit 0，continueOnBlock 無效果 |
| 已完成 | l10n-sync-verification-hook (Edit/Write) | 已設定 |

### 風險評估

| 風險 | 說明 | 緩解 |
|------|------|------|
| Claude 無限重試 | continueOnBlock 讓 Claude 看到錯誤訊息後可能反覆重試 | CC runtime 已有內建重試上限；hook 訊息應包含明確修正指引 |
| 閘門失效 | phase4-decision-enforcement 若加 continueOnBlock，Claude 可能自行刪除延後話術 | 不對此 hook 整合，保持硬阻擋 |
| 行為變更 | 從「靜默阻擋」變為「阻擋 + 回饋」 | 正面變更，降低 PM 手動介入頻率 |

### Spawn 規劃

| 類型 | 標題 | 優先級 | 理由 |
|------|------|--------|------|
| IMP | 為 3 個 PostToolUse hook 啟用 continueOnBlock（phase-completion-gate / ticket-id-validator / ticket-quality-gate） | P2 | 整合機會落地：修改 settings.json 3 處註冊 |

無需建 ticket 的豁免項：
- phase4-decision-enforcement-hook：評估結論為不整合（強制決策閘門）
- 24 個 advisory hook：continueOnBlock 對 exit 0 無效果，無需動作
