### 影響範圍

`build/development/` 建置產物、`manifest.json` 三入口（background SW / content script / popup）、chrome-devtools MCP 實機驗證環境。

無直接相關 error-pattern。實機驗證流程依 `chrome-extension-mcp-debug` skill Workflow A/C/D 與 `docs/bookstores/readmoo.md` MCP E2E checklist。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 執行摘要

由主線程 PM 透過 chrome-devtools MCP（pipe + `--isolated`）驅動實機驗證，依 `chrome-extension-mcp-debug` skill Workflow A（install）+ C（verify）+ D（debug popup/SW/CS）。無產品程式碼變更——本 ticket 僅重建 dev build 並驗證實機環境。

### 執行步驟

1. `npm run build:dev` 重建 `build/development/`。
2. chrome-devtools MCP `install_extension` 載入 unpacked extension（絕對路徑）。
3. `list_extensions` 確認 Enabled；`chrome://extensions` 開發人員模式檢視無錯誤按鈕。
4. SW `evaluate_script` 確認 runtime / storage 初始化。
5. 導航 `readmoo.com` 首頁 + `read.readmoo.com/#/library`（已登入）驗證 content script 注入。
6. 開啟 `popup.html` 驗證 UI 渲染。
