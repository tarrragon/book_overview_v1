## Problem Analysis

### 問題根因

W3-046（v0.19.0 W3）將 `claim` 預設改為「不執行 AC verification」，並把 `--skip-verify` 旗標降為向後相容的 no-op，配合 deprecation warning 引導用戶停止使用。後續 W3-055 / W3-092 / W4-015 三輪觀察期共同收集 24hr / 75 commits / 5 ticket cycles 樣本，量化驗證以下兩條件：

1. 外部 / 用戶腳本對 `--skip-verify` 的依賴 = **0**（22 個程式碼引用全部位於 `.claude/skills/ticket/` 內部測試與實作）
2. 真實 `[Deprecation]` runtime 觸發案例 = **0**（hook-logs 內 `--skip-verify` 字串 match 皆屬 worktree-branch-check 記錄 PM 指令字面，非 deprecation 警告觸發）

W4-015 ANA 結論 trigger 達成，可移除 flag + dual conditional branch，並另拆 `ticket track verify` 子命令補足查詢缺口（W3-046 後 PM 想單獨跑 AC 驗證卻不想 claim 的場景）。

### 影響範圍

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### Stage 1: 移除 --skip-verify flag + dual conditional branch

**commands/track.py**：

- 移除 `p_claim.add_argument("--skip-verify", ...)`（W3-046 / W3-055 deprecated 旗標）
- 移除 deprecation warning 與向後相容引用，改寫為 W4-019 註解（保留設計脈絡）

**commands/lifecycle.py**：

- `execute_claim`：移除 `skip_verify = bool(getattr(args, "skip_verify", False))`，移除 `[Deprecation]` warning 區塊；條件簡化為 `if verify_opt_in: ... else: ...`（dual conditional → single conditional）
- `claim_with_verification`：移除 `skip_verify: bool = False` 參數、移除 `skip_verify + auto_yes` 衝突 warning、移除 `if skip_verify: ... return self.claim(...)` 分支；非 tty 警告訊息從「`use --yes or --skip-verify`」改為「`use --yes`」
- docstring 補上 W4-019 移除 trigger 達成註記，引用 `ticket track verify` 子命令

**commands/claim_verification.py**：

- `_NON_TTY_WARNING` 從「`use --yes or --skip-verify to override`」改為「`use --yes to override`」

### Stage 2: 拆 ticket track verify 子命令

**commands/track.py**：

- 新增 `verify` 子 parser（與 `release` 並列），參數 `ticket_id` + `--version`
- 新增 `_execute_verify` handler：建立 `TicketLifecycle(version)` 並呼叫 `lifecycle.verify_only(ticket_id)`
- 註冊到 `_create_command_handlers()` dict

**commands/lifecycle.py**：

- 新增 `TicketLifecycle.verify_only(ticket_id)` 方法（位於 `claim_with_verification` 與 `complete` 之間）
- 行為：呼叫 `collect_ac_verifications` → `run_all_verifications` → `summarize_results` → `render_results` 印出結果，**不變更 ticket 狀態**
- Exit code：0（正常完成查詢，不論 pass / fail）/ 1（AC 解析失敗）/ 130（Ctrl-C 中斷）
- 與 `claim --verify` 解耦：PM 想單獨巡檢 AC 漂移而不 claim 時使用

### Stage 3: 更新規則文件

**.claude/pm-rules/ticket-lifecycle.md**：

- 「Consequence」段：移除「PM 必須以 `--skip-verify` 繞過」陳述，改述「W4-019 拆出 `ticket track verify <id>` 子命令補足查詢缺口」
- 「豁免條件」表：將 `--skip-verify` 列改為 `ticket track verify <id>` 子命令列

### Stage 4: regression test 補強

**tests/test_claim_no_verify_by_default.py**：

- 移除 4 個 backward-compat 測試（test 3 / 3b / conflict / no-noise）：基於 `--skip-verify` 旗標存在的前提已不成立
- `_make_args` defaults 去除 `skip_verify=False`
- `fake_claim_with_verification` 簽名移除 `skip_verify=False`
- 新增 `test_skip_verify_flag_removed_from_argparse`：直接驗證 argparse parser 對 `claim --skip-verify` 拋 SystemExit（移除回歸防護）

**tests/test_claim_with_verification.py**：

- 移除 H7 (`test_h7_s5_skip_verify_bypasses`) / H10 (`test_h10_flag1_skip_plus_yes_warns`) / H11 (`test_h11_flag2_only_skip_no_warning`)
- H12 (`test_h12_flag3_only_yes_no_warning`)：移除 `skip_verify=False` kwarg
- K3 (`test_k3_non_tty_without_flags_rejects`)：移除 `skip_verify=False` kwarg，更新 docstring

**tests/test_ac_drift_regression.py**：

- `_claim_args` 移除 `skip_verify=False`，補 W4-019 註記

**tests/test_track_lifecycle.py**：

- 4 個 TestClaim 測試新增 `args.verify = False`（必要：`args = Mock()` 預設 attr 為 truthy Mock object，會被 `bool(getattr(args, "verify", False))` 視為 True 而誤觸 `claim_with_verification` 路徑）
