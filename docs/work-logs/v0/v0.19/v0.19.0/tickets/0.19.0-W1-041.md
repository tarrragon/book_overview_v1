# Execution Log

## Task Summary

修復 build overview 入口接線：shipped overview.html 引用 legacy assets/js/main.js（檔案式不讀 storage），新版 src/overview event-driven overview（自動讀 chrome.storage）為孤兒，需讓 extension overview 使用新版

## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

shipped extension 存在兩套 overview 入口，build 同時打包：

| 入口 | 檔案 | 行為 |
|------|------|------|
| legacy 孤兒 | 根目錄 `overview.html` → `assets/js/main.js` | 10 個檔案式函式，完全不讀 `chrome.storage`；需手動上傳 JSON 才有資料 |
| 新版 event-driven | `src/overview/overview.html` → `src/overview/overview.js`（esbuild bundle，含 `overview-page-controller.js`） | 自動 `loadBooksFromChromeStorage()` 載入 storage |

W1-001.2 實機驗證者直接導航根 `overview.html` → 總書籍數 0（legacy 不讀 storage）；導航 `src/overview/overview.html` → 顯示 928，行為正常。

### 影響範圍

- `scripts/build.js` `filesToCopy` 將根 `overview.html` 複製進 build 產物
- 根 `overview.html`、`assets/js/main.js`、`assets/css/style.css` 為 legacy 孤兒檔

### 接線現況核查（重要）

`manifest.json` `options_page` 已指向 `src/overview/overview.html`（新版）；`src/popup/popup.js:840` `openLibraryOverview()` 使用 `chrome.runtime.openOptionsPage()` → 開啟 `options_page` → 即新版 event-driven overview。**正式入口（popup「檢視書庫」按鈕）本來就到新版**，無須改 manifest / popup。問題僅在於 build 產物殘留可被直接導航的 legacy 孤兒入口。

### 相關 Error Pattern

feedback_verify_spa_hash_routes_before_concluding_broken（W1-001.2）：驗證者從錯誤 URL 斷定故障——本案是「兩套入口、驗證者導航到 legacy 那套」的具體案例。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 接線方案決策

候選方案：
- A：讓根 `overview.html` 改引用 bundle → 否決，會永久保留重複入口
- B：manifest `options_page` 指向 `src/overview/overview.html` → 已是現況，無須改動
- C：移除 legacy 孤兒檔 + build 不再複製根 `overview.html` → 採用，單一真實入口

採方案 C：新版入口已完整接線（manifest `options_page` + popup `openOptionsPage()`），唯一缺陷是 build 產物殘留可被直接導航的 legacy 孤兒。移除孤兒即達成「extension overview 入口 = event-driven 新版」。

### 變更摘要

| 檔案 | 變更 |
|------|------|
| `scripts/build.js` | `filesToCopy` 移除 `'overview.html'`；補註解說明 overview 入口由 `src/overview/overview.html` 提供 |
| `overview.html`（根） | 刪除（legacy 孤兒，無任何 manifest/popup/test 引用） |
| `assets/js/main.js` | 刪除（legacy 孤兒，僅根 `overview.html` 引用） |
| `assets/css/style.css` | 刪除（legacy 孤兒，僅根 `overview.html` 引用）；`assets/js/`、`assets/css/` 空目錄一併移除 |

### 未改動項目與理由

- `manifest.json`：`options_page` 已為 `src/overview/overview.html`，無須改
- `src/popup/`：`openLibraryOverview()` 已用 `openOptionsPage()`，無須改
- `assets/icons/`：manifest icons 依賴，保留

### 孤兒驗證

`grep -rn "assets/js/main\|assets/css/style"` 於 `src/ scripts/ manifest.json tests/` 全無命中（除被刪的根 `overview.html` 自身）；`assets/icons/` 為 `assets/` 唯一使用部分。
