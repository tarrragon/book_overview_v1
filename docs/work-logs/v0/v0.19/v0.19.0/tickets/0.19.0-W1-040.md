## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要（2026-05-23）

根因：W1-030 的 `loadAllBooksLazy` / `_scrollStep` 依 `findScrollContainer` 回傳的 strategy 二擇一觸發 lazy load——`load-more-button` 策略點按鈕、其餘策略捲動容器。但實機 `read.readmoo.com/#/library` 的 `findScrollContainer` 恆先命中 `selector` 策略（`#react-container` 為頁面根容器），`load-more-button` 分支永不執行，「更多...」按鈕從未被點擊。Readmoo library 首批僅 render 96 本，必須先點「更多...」按鈕才展開至 192，之後虛擬捲動才接手——只捲動 `#react-container` 對首批 96 完全無效，3 輪 count 不變即誤觸 `count_stable` 放棄。

修復（`src/content/adapters/readmoo-adapter.js`）：`_scrollStep` 改為每輪同時執行兩動作（不再二擇一）——(1) 捲動容器至底部、(2) 呼叫新增的 `_clickLoadMoreButton` 點擊「更多...」按鈕。按鈕載入完畢後消失，`findLoadMoreButton` 回 null 時靜默略過（正常終止）。

測試：`tests/unit/adapters/readmoo-adapter-scroll-load.test.js` 補強（jest.spyOn 注入驗證點擊行為）。

整合：agent worktree commit 9d0556e1 → main merge commit。
