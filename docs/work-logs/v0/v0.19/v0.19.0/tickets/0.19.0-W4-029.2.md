## Task Summary

評估調和方向 CC 減少自動提問傾向 vs 專案強制 AUQ 規則的對齊

---

## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 兩維度框架

AUQ 涉及兩個正交維度，混淆會誤判 CC v2.1.154 與專案規則衝突：

| 維度 | 問題 | 來源 |
|------|------|------|
| A：該不該問？ | 此決策 PM 有足夠 context 自己決定，還是真正需要用戶？ | CC v2.1.154「reserves multiple-choice for decisions it genuinely cannot make itself」 |
| B：要問就怎麼問？ | 決定要問時，用 AUQ 工具還是純文字列選項？ | 專案 PC-064（純文字繞過）+ 通用觸發原則三問 |

### 現有規則缺口

`askuserquestion-rules.md` 通用觸發原則三問（「我的回覆是否包含選擇/確認？」）**全屬維度 B**——預設「PM 已決定要問」，規範問的形式。**無維度 A 前置閘門**。

### 相容性結論

CC v2.1.154 與專案規則**不衝突，互補**：
- PC-064 根因是維度 B（純文字繞過），hook 強制層防的也是維度 B。
- CC 新傾向是維度 A（問的時機/頻率）。
- 現狀缺維度 A，可能導致「為了用 AUQ 而過度提問」（能自己決定的也拿來問）。

### 風險（敏感點）

新增維度 A「先判斷該不該問」若判準不嚴，可能淪為 PC-014（合理化跳過）藉口——PM 把「該問的」也判為「能自己決定」而不問。故維度 A 必須附**嚴格的「必須問」硬性類別**，不是放寬而是精準化。

## 重現實驗結果
機制分析（非程式實驗）：
- 讀 `askuserquestion-rules.md` 通用觸發原則（行 7-25）：三問判斷方法皆為「我的回覆是否在問選擇/確認」→ 規範問的形式（維度 B），預設 PM 已決定要問。
- 讀 CC v2.1.154 release note：行為改變在「decisions it genuinely cannot make itself」→ 決定要不要問（維度 A）。
- 交叉確認：現有 18 場景 + S1-S6 訊號皆針對「已在問」的形式偵測，無「該不該問」前置判斷。
- 結論：現有規則完整覆蓋維度 B，完全未覆蓋維度 A。CC 傾向填補維度 A 缺口，不推翻維度 B。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 調和方案：新增維度 A 前置閘門（不放寬維度 B）

在 `askuserquestion-rules.md` 通用觸發原則**之前**新增前置閘門：

```
前置自問（維度 A）：此決策我有足夠 context 自己決定嗎？
  能自己決定 → 直接決定 + 在回應顯性說明理由，不問
  屬下列「必須問」硬性類別 → 進入通用觸發原則（維度 B），用 AUQ

「必須問」硬性類別（防止維度 A 淪為跳過藉口）：
  1. 不可逆 / 高成本後果（刪除、發布、覆寫、外部副作用）
  2. 用戶價值觀 / 偏好（風格、優先序、取捨無客觀對錯）
  3. 需用戶授權（規則 / 流程 / 架構的方向性改動）
  4. 規格缺口（需求不明確，無合理預設可覆蓋）
```

現有通用觸發原則三問 + S1-S6 訊號 + hook 強制層（維度 B）**完全保留不變**。

### 設計要點

- 不是放寬 PC-064，是在其**之前**加「該不該問」判斷，避免過度提問。
- 「必須問」硬性類別確保敏感決策（含本 ANA 這類規則改動）仍強制問。
- 自我驗證：本 ANA「是否採納規則改動」正屬硬性類別 3（需用戶授權），故下一步必須 AUQ 問用戶——dogfooding 證明方案自洽。

### WRAP

- **W**：採用粒度——(a) 完整新增維度 A 閘門 / (b) 僅在規則加註 CC 傾向不改結構 / (c) 維持現狀。
- **R**：(a) 填補真實缺口且防過度提問；(b) 不痛不癢；(c) 過度提問問題持續。傾向 (a)。
- **A**：機會成本——改互動規則影響每次 PM↔用戶對話，價值高但敏感，須用戶授權。
- **P 絆腳索**：若維度 A 上線後 PM 重現「該問不問」（PC-014 變體）→ 收緊硬性類別或回退；若過度提問未改善 → 強化前置閘門措辭。
