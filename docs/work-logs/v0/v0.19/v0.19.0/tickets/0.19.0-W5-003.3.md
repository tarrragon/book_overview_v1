### 撰寫完成摘要

**新增章節**：`docs/spec/extraction/e2e-contract.md` §3 Console 訊息與事件格式契約

**章節結構**（5 項契約結構標準）：

| 項目 | 內容 |
|------|------|
| Name | 跨進程通訊 envelope + v2.0 命名 + 核心 message types + log 前綴 |
| Source of Truth | 13 個引用點（event-type-definitions / message-router / content-modular / page-detector / log 前綴來源） |
| 契約定義 | 5 子節：3.1 v2.0 命名規範 / 3.2 核心 E2E message types (9 種) / 3.3 envelope JSON Schema (request + response) / 3.4 結構化 log 前綴 (12 類) / 3.5 status string 集合 |
| 變更影響 | 5 種變更類型對下游契約的影響 |
| Grep 驗證 | 6 條 grep 指令，全部命中 |

**關鍵契約**：

1. **v2.0 命名規範**：DOMAIN.PLATFORM.ACTION.STATE 四層大寫，regex 驗證 `/^[A-Z][A-Z_]*\.[A-Z][A-Z_]*\.[A-Z][A-Z_]*\.[A-Z][A-Z_]*$/`
2. **遺留命名**：`START_EXTRACTION` / `CONTENT.STATUS.READY` / `SYSTEM.SHUTDOWN` 等保留，新事件必用 v2.0
3. **核心 message types**：PING / HEALTH_CHECK / GET_STATUS / EVENT.EMIT / EVENT.STATS / EVENT_SYSTEM_STATUS_CHECK / START_EXTRACTION / CANCEL_EXTRACTION / SYSTEM.SHUTDOWN
4. **Envelope schema**：Request `{ type, data?, source?, timestamp? }`；Response `{ success, data?, error?, messageType, timestamp }`
5. **結構化 log 前綴**：12 類（含 emoji 與冒號精確匹配），E2E 觀察依賴
6. **Source routing**：content-script / popup / background 三種來源；未匹配回傳「未知訊息來源」
