## Problem Analysis

### 問題根因

W3-041（pre-fix-eval hook 對 ticket body 寫入誤判修復）接手考古發現 5 項衍生缺陷，依 quality-baseline 規則 5 合成處理：

1. **`_is_test_command` substring 比對缺陷**：`TEST_COMMAND_KEYWORDS = ["flutter test", "dart test", "npm test"]` 配合 `any(kw in command for kw in ...)`，導致 `echo "npm test"` 等回顯關鍵字的非測試命令觸發 hook。同時 `npm run test:hooks` 因不含 `"npm test"` 連續子字串反而 **未觸發**（雙向誤判）。

2. **`TICKET_WRITE_COMMAND_KEYWORDS` 過時 + 漏列**：
   - 含已不存在的 `ticket track create` / `ticket track update`（ticket CLI 實際 write 命令為 top-level `ticket create`，無 track 前綴）
   - 漏列 `ticket create` / `ticket batch-create` / `ticket show` → 本 ticket 建立過程觸發 false positive

3. **`__pycache__` 殘留**：`pre-fix-evaluation-hook.py` 已於 W10-001 合併至 post-test-hook.py 並刪除原檔，但 `.claude/hooks/__pycache__/pre-fix-evaluation-hook.cpython-311.pyc` 仍存在。

4. **`sync-preserve.yaml` 過時保留項**：`hooks/pre-fix-evaluation-hook.py` 已刪除卻仍列在 preserve 清單，sync-pull 邏輯對不存在路徑會浪費判斷成本。

5. **Ticket schema 模板 PC 引用錯誤**：`ticket_builder.py:758` 與 `command_lifecycle_messages.py:240` 引用「接手者驗證 (PC-007)」，但 PC-007 實際標題為「Command 引導與腳本實作行為不符」，與接手者驗證語意完全不對應。應引用 PC-162（ticket-description-stale-environment-and-schema-citation-mismatch，已建）。

## Solution

### 子任務 1 — `_is_test_command` 改 word boundary regex

- 移除 `TEST_COMMAND_KEYWORDS` 列表常數
- 新增 `TEST_COMMAND_PATTERN` regex：
  ```python
  TEST_COMMAND_PATTERN = re.compile(
      r"(?:^|[;|&(\n]|&&|\|\|)\s*"          # statement 邊界
      r"(?:[A-Za-z_][A-Za-z0-9_]*=\S*\s+)*" # 可選環境變數前綴
      r"(?:flutter\s+test|dart\s+test|npm\s+(?:test|run\s+test))\b"
  )
  ```
- `_is_test_command` 改用 `TEST_COMMAND_PATTERN.search(command)`
- 同時修復 `npm run test:hooks` 等舊版未觸發的 substring 漏判場景

### 設計決策（WRAP 三問）

- **W（Widen）候選方案**：
  - (a) regex word boundary（採用）：statement 邊界 + word boundary，精準且維護成本低
  - (b) shlex 解析 + token 比對：理論最精準但 quoted string 仍需特殊處理，複雜度高於收益
  - (c) substring + echo 白名單：白名單持續膨脹的反模式
- **A（Attain distance）機會成本**：W3-068（接手考古 SOP 規則檔）；本 ticket 4 子任務合計約 10 tool calls 不擠壓 W3-068。
- **P（Prepare to be wrong）失敗風險**：regex 未覆蓋某 shell 用法（如 `xargs npm test`、`exec npm test`）→ 已加 11 個 true case + 7 個 false case 涵蓋 statement 邊界（含 `cd && npm test`、`(...)`、env prefix）。
