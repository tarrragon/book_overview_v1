## Problem Analysis

### 問題根因

W4-010 執行 npm run test:unit 第 2 次連跑時發現 operation-result.test.js:414 `expect(durationMs).toBeLessThan(2)` 在完整套件下失敗（單獨執行 467 tests 全過），屬 test-assertion-design-rules.md 規則 1 違反（npm test 主套件禁止絕對計時門檻當 pass-fail 斷言）。同檔 line 392 `toBeLessThan(0.5)` 為同家族 latent flaky。

**直接根因**：jsdom 環境 `process.hrtime.bigint()` 計時受全套件並行負載影響——單獨執行時 CPU/GC 壓力低、計時穩定；完整套件下 195 suites 並行 + JIT 暖機差異 + GC 觸發隨機性 → 計時值飄移超越硬門檻。

**修復軌跡**：W1-091 ANA spawn 6 個批次清理 ticket（W1-095/W1-096/W1-097/W1-099 等）統一掃描 timing assertion 違規。本檔被 W1-095 commit ad438102（test: 修復 UC Error 20 檔 26 行 timing assertion 違規）覆蓋——line 382-384 與 395-397 既有 `toBeLessThan` 已替換為 `// W1-095: 移除 process.hrtime.bigint() 差值斷言（規則 1 違規）；大幅退化防護改由 npm run test:perf 提供。保留功能正確性驗證。` 註解 + 功能斷言（`expect(result.success).toBe(true)` / `expect(!result.success).toBe(true)`）保留於原位。

### 影響範圍

- 單檔影響：tests/unit/core/errors/operation-result.test.js（line 382-384、395-397 兩處）
- 同 W1-095 commit 涵蓋 UC Error 系列共 20 檔 26 行修復
