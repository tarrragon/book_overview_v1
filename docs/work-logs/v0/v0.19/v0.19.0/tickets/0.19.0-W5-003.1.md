### 撰寫完成摘要

**新增章節**：`docs/spec/extraction/e2e-contract.md` §1 URL 與 SPA 路由契約

**章節結構**（符合本檔契約結構標準 5 項目）：

| 項目 | 內容 |
|------|------|
| Name | 定義 CS 注入範圍 / page detector / SPA hash / chrome-extension URL / E2E fixture |
| Source of Truth | 11 個引用點（manifest.json / page-detector.js / extraction-flow.js / docs/bookstores/readmoo.md / e2e test） |
| 契約定義 | 6 子節：1.1 Readmoo URL 表 / 1.2 manifest matches / 1.3 page detector 規則 / 1.4 SPA hash 路由 / 1.5 chrome-extension URL / 1.6 fixture 差異 |
| 變更影響 | 5 種變更類型對下游契約的影響 |
| Grep 驗證 | 6 條 grep 指令，已執行確認契約與 src code 100% 對齊 |

**關鍵契約**：

1. **CS 注入範圍**：`*://*.readmoo.com/*` + `*://readmoo.com/*` 兩條 matches
2. **Page type 分類**：library / shelf / reader / unknown 四類，皆用 `url.includes()` OR `pathname.includes()` 雙重檢查（W1-001.2 SPA hash 路由誤判教訓的契約化）
3. **可提取頁面**：`isReadmooPage && pageType in [library, shelf]`
4. **SPA hash log 契約字面**：`URL 變更檢測:` （page-detector.js L178）
5. **Overview URL**：`chrome-extension://{id}/src/overview/overview.html`
6. **E2E fixture URL 與真實 URL 差異**：fixture 用 path route（`readmoo.com/library`），真實用 hash route（`read.readmoo.com/#/library`）；E2E 無法直接驗證 SPA hash 偵測邏輯

### Grep 驗證結果

- manifest matches: 2 條 matches + 1 條 web_accessible_resources matches

- URL 變更檢測 log: L178 命中
- options_page + overview URL: manifest 與 e2e test 引用一致
- FIXTURE_URL: extraction-flow.js L34 + 2 處內部使用
