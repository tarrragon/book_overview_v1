#### 測試策略決策

| 項目 | 決策 | 理由 |

| 測試類型 | 單元測試 + IO 邊界整合 | FileContentReader 包裹 FileReader API（IO 邊界），需 mock FileReader |
| 測試檔案 | `tests/unit/overview/import/file-reader.test.js`（新建） | 隔離 |
| Mock 策略 | parser（注入 stub，對應 10.1.2 ContentParser 介面）；readerFactory（注入 stub FileReader）；detectFormat（注入 stub 函式）；showError（jest.fn） | FileReader 為外部依賴（DOM API），須 Mock 控制事件觸發 |
| 測試框架 | Jest | 既有 |

#### 關鍵 Mock：FakeFileReader stub

設計共用 helper：

```js
function createFakeFileReader () {
  const fr = {
    readAsText: jest.fn(),
    onload: null,
    onerror: null,
    result: null
  }
  fr._triggerLoad = (content) => { fr.result = content; fr.onload?.({ target: fr }) }
  fr._triggerError = (err) => { fr.onerror?.(err ?? new Error('read failed')) }
  return fr
}
function createStubReaderFactory () {
  const reader = createFakeFileReader()
  const factory = jest.fn(() => reader)
  return { factory, reader }
}
```

供 deps.readerFactory 注入，控制 onload / onerror 觸發時機。

#### 既有測試案例遷移清單

來源：`tests/unit/overview/overview-import-private-methods.test.js`（_readFileWithReader 區段，L380-L530）

#### 反向斷言

- instance 不暴露舊底線方法（_readFileWithReader / _createFileReader / _setupReaderHandlers / _handleReaderSuccess / _handleReaderError / _handleFileProcessError 皆 undefined）

#### 行為鏈推演

read(file) 前置鏈：
1. parser 存在（建構時驗證）
2. 建立 FileReader（readerFactory or globalThis.FileReader）
3. 註冊 onload/onerror
4. readAsText(file)
5. onload 觸發 → detectFormat(file) → parser.parse(content, format) → resolve
6. onerror 觸發 → UNKNOWN_ERROR reject + showError

每節點獨立測試（onload 不觸發 onerror，反之亦然）。

#### 與 10.1.2 stub 整合

本 IMP 測試使用 parser stub 而非真實 ContentParser instance；介面契約：
- `parser.parse(content: string, format: 'json'|'csv'): ImportResult`
- 失敗時 throw（同步），FileContentReader 須在 onload 內 try/catch 並透 reject
