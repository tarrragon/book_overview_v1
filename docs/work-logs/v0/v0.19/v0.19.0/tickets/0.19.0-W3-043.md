### 問題根因

W3-012 本 session 處理過程中發生兩次外部寫入導致 ticket 變更遺失。經 git reflog + 行為時序分析確認三層共振根因：

L1 觸發層：`ticket track claim --yes` 首次呼叫在背景模式下執行時未持久化 status 變更（status 仍 pending，但 CLI 顯示成功）。已知 bug 範圍見 W1-054。

L2 協議違反層：PM 在 status=pending 狀態下執行了 3 次 `append-log` 操作（13:21）。協議應為「claim 持久化驗證 → 才能 body 操作」，但實際流程跳過驗證直接操作。

L3 並行偵測層：parallel session（另一 terminal/Claude 實例）偵測到「status=pending 但 body 有修改」異常，誤判為「殘留 junk」，執行 `git reset --hard HEAD`（13:39:35 reflog 記錄）清理我的 W3-012 working tree 變更。

三層共振：L1 是 CLI bug、L2 是協議違反、L3 是並行 session 過度防衛。任一層阻斷即可避免事件。
