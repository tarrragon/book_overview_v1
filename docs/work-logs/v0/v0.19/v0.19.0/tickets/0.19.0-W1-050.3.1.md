### 轉換規則細節（必讀）

**A. `process.hrtime.bigint()` → `performance.now()`**
- Node `hrtime.bigint()` 回傳 nanoseconds（BigInt）；`performance.now()` 回傳 milliseconds（Number）
- 差值用法：`(end - start)` ns 需 `/ 1_000_000n` 才轉 ms → 改寫後直接 `end - start` ms
- 若程式碼有與 BigInt 比較或運算，需同步調整為 Number
- 注意 event-performance-monitor.js 內可能有以 ns 為基準的閾值常數，需轉換

**B. `process.nextTick(fn)` → `queueMicrotask(fn)`**
- 語意對齊：兩者都在當前 microtask 後執行
- 替代方案：`setTimeout(fn, 0)`（macrotask，順序延後）— 僅當 export-manager 邏輯不依賴 microtask 順序時可用
- 若 fn 需傳參：`queueMicrotask` 不支援 args，需用 closure 包裝

**C. `process.memoryUsage()` → `performance.memory` 含 guard**
- `performance.memory` Chrome-only，需 `typeof performance.memory !== 'undefined'` guard
- 對應欄位：
  - `process.memoryUsage().heapUsed` → `performance.memory.usedJSHeapSize`
  - `process.memoryUsage().heapTotal` → `performance.memory.totalJSHeapSize`
  - `process.memoryUsage().rss` → 無對應，fallback 0 或 undefined
- 範本：
  ```js
  const memoryUsage = typeof performance !== 'undefined' && performance.memory
    ? { heapUsed: performance.memory.usedJSHeapSize, heapTotal: performance.memory.totalJSHeapSize }
    : { heapUsed: 0, heapTotal: 0, unavailable: true }
  ```

**D. `process.on('uncaughtException'|'unhandledRejection')` → `self.addEventListener`**
- SW context 用 `self`；overview/content context 用 `window`（CE Manifest V3 用 `self` 通用安全）
- Event payload 差異：
  - `process.on('uncaughtException', (error) => {})` → `self.addEventListener('error', (event) => { const error = event.error })`
  - `process.on('unhandledRejection', (reason, promise) => {})` → `self.addEventListener('unhandledrejection', (event) => { const reason = event.reason; const promise = event.promise })`
- Logger handler 內若用到 `error.stack` 等屬性，注意 SW context 的 ErrorEvent 行為一致

### 修改摘要

依 Problem Analysis 轉換規則 A/B/C/D，替換 3 檔共 13 hits 的 process.* Node-only API 為 CE runtime 相容的等效 API。

### 檔案層級變更

| 檔案 | 修改範圍 | 轉換規則 |
|------|---------|---------|
| src/error-handling/event-performance-monitor.js | L832/L839/L884/L892 process.hrtime.bigint() → performance.now()；L915-934 _measureMemory 重排：performance.memory 為 primary path（CE runtime），process.memoryUsage typeof-guard fallback（Node 測試環境） | A + C |
| src/export/export-manager.js | L397/L437/L544/L660 process.nextTick(fn) → queueMicrotask(fn) x 4；L899 process.memoryUsage() → performance.memory typeof-guard，fallback 回 { heapUsed: 0, heapTotal: 0, external: 0, unavailable: true } | B + C |
| src/error-handling/error-system-init.js | L168/L172 process.on → self.addEventListener；條件改為 typeof self !== undefined AND typeof window === undefined，避免 SW context 與 window listener 重複註冊（window 環境下 self === window） | D |

### 設計決策

1. performance.now() 單位轉換消除：原 Number(endTime - startTime) / 1000000 將 hrtime ns 轉 ms；改用 performance.now() 後直接 endTime - startTime 即為 ms，移除 / 1000000 與 Number(BigInt) 轉型。

2. _measureMemory 優先序反轉：原版 process.memoryUsage 為 primary、performance.memory 為 fallback；CE runtime 永遠無 process，反轉後 CE 環境直接走 performance.memory 路徑，效能與正確性同時改善。process.memoryUsage 區塊以 typeof process !== undefined guard 保留作 Node 測試環境 fallback（acceptance 允許）。

3. queueMicrotask 取代 process.nextTick：兩者語意對齊（皆於當前 microtask 後執行）。export-manager.js 內 4 處皆無 args 傳遞需求，直接以 arrow function 包裝，無 closure 額外處理。

4. error-system-init.js self.addEventListener 註冊條件：原 process.on 區塊註解為「Node.js 環境錯誤處理（用於測試）」，但 CE runtime SW context 既無 process 也無 window；改用 typeof self !== undefined AND typeof window === undefined 條件，僅在 SW context 註冊（避免 window 環境下與既有 window.addEventListener 重複註冊）。Event payload 對應：ErrorEvent.error / PromiseRejectionEvent.reason+promise 已在註解中標註。
