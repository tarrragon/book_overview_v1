## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W1-029 稽核發現 src/ 殘留過時硬編碼 `readmoo.com/library`（已 404）。Readmoo 書庫實際頁面為 `https://read.readmoo.com/#/library`（Vue SPA hash route），此字串**不含**子字串 `readmoo.com/library`（中間夾 `/#/`），導致頁面偵測與 content script 相關邏輯對真實書庫頁失效。

### 影響範圍

PM 前台調查確認 6 處引用（4 檔；W1-029 稽核標「5 處」，readmoo-adapter 兩行屬同一 pattern 陣列，執行代理人應獨立複驗）：

| # | 檔案:行 | 內容 | 類型 |
|---|---------|------|------|
| 1 | src/content/adapters/readmoo-adapter.js:1873 | `'https://readmoo.com/library*'` | match pattern |
| 2 | src/content/adapters/readmoo-adapter.js:1875 | `'https://*.readmoo.com/library*'` | match pattern |
| 3 | src/core/errors/UC01ErrorFactory.js:73 | expectedPatterns 預設 `['readmoo.com/library','readmoo.com/shelf']` | 錯誤訊息文字 |
| 4 | src/background/domains/page/services/tab-state-tracking-service.js:643 | `url.includes('readmoo.com/library')` | JS 子字串偵測 |
| 5 | src/background/domains/page/services/content-script-coordinator-service.js:187 | `matches: ['*://readmoo.com/library*']` | match pattern |
| 6 | src/background/domains/page/services/content-script-coordinator-service.js:298 | `url.includes('readmoo.com/library')` | JS 子字串偵測 |

#### 關鍵技術事實 — 同一錯字串、兩類修法

1. **Chrome match pattern 不含 URL fragment**。`#/library` 是 fragment，match pattern 的 path 段只能比對 `/`。第 1/2/5 處無法編碼 `#/library`，只能放寬為 `*://*.readmoo.com/*`（或 `*://read.readmoo.com/*`）。
2. **JS `url.includes` 拿得到完整含 fragment 字串**，但 `'readmoo.com/library'` 對 `read.readmoo.com/#/library` 回傳 false → active bug。第 4/6 處改為偵測 `read.readmoo.com` + `#/library`。
3. 第 3 處 expectedPatterns 僅用於錯誤訊息文字，更新為 `read.readmoo.com/#/library`。

#### 不在本 ticket 範圍

`manifest.json` content_scripts.matches 已是 `*://*.readmoo.com/*`，正確涵蓋 `read.readmoo.com`，不需修改。

### 相關 Error Pattern

無直接對應 PC/IMP error-pattern。執行代理人可 `/error-pattern query` 複查。

### 參考文件

- docs/bookstores/readmoo.md（URL、SPA 路由、content script 注入點）

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 獨立複驗結果（6 處）

逐一讀取並以 RED 測試獨立複驗 PM 標列的 6 處引用：

| # | 檔案:行 | 複驗結論 | 修法 |
|---|---------|---------|------|
| 1 | readmoo-adapter.js:1873 `'https://readmoo.com/library*'` | 過時死字串（`getSupportedUrls` 回傳資訊性陣列，非 runtime 注入比對）；字串本身無效 | match pattern 放寬 |
| 2 | readmoo-adapter.js:1875 `'https://*.readmoo.com/library*'` | 同上；`*.readmoo.com` 可涵蓋 host 但 `/library` path 段無法比對 `#/library` fragment | match pattern 放寬 |
| 3 | UC01ErrorFactory.js:73 expectedPatterns 預設值 | 僅用於錯誤訊息文字，非比對邏輯 | 文字更新 |
| 4 | tab-state-tracking-service.js:643 `url.includes('readmoo.com/library')` | Active bug：`'https://read.readmoo.com/#/library'.includes('readmoo.com/library')` 為 false，書庫頁誤判為 `readmoo_main` | url.includes 改寫 |
| 5 | content-script-coordinator-service.js:187 `matches: ['*://readmoo.com/library*']` | Active bug：經 `shouldProcessTab` regex `.*://readmoo.com/library.*` 不匹配 `read.readmoo.com`（要求 `://` 後緊接字面 `readmoo.com`），書庫分頁不被處理 | match pattern 放寬 |
| 6 | content-script-coordinator-service.js:298 `url.includes('readmoo.com/library')` | Active bug：同 #4 子字串問題，`detectPageType` 誤判 | url.includes 改寫 |

確認 3 處 active bug（#4/#5/#6），3 處過時資料（#1/#2/#3）。

### 修復摘要

依 PM 標註的兩類修法（Chrome match pattern 不含 fragment）：

1. match pattern（#1/#2/#5）：放寬為 host 層級。
   - `readmoo-adapter.js getSupportedUrls()`：4 條 path-encoded 模式收斂為單一 `'*://*.readmoo.com/*'`。
   - `content-script-coordinator-service.js readmoo_library_extractor.matches`：`'*://readmoo.com/library*'` 改為 `'*://*.readmoo.com/*'`（與 `manifest.json` content_scripts.matches 一致）。
   - 細部頁面類型判斷交由 `detectPageType` 負責，不依賴 match pattern 編碼 fragment。
2. url.includes（#4/#6）：`detectPageType` 改為 `(url.includes('read.readmoo.com') && url.includes('#/library')) || url.includes('readmoo.com/library')`，新偵測 Vue SPA hash route，同時保留舊版 path 形式（`member.readmoo.com/library`）的向後相容。
3. expectedPatterns（#3）：預設文字 `['readmoo.com/library','readmoo.com/shelf']` 改為 `['read.readmoo.com/#/library','read.readmoo.com/#/shelf']`。

### 不在本 ticket 範圍

- `manifest.json` content_scripts.matches 已是 `*://*.readmoo.com/*`，未修改。
- 其他 `detectPageType`（`page-detection-service.js`、`page-detector.js`）使用 `location.hostname` 或不同偵測路徑，未在 PM 標列 6 處內，未動。

### Phase 4 重構評估

- 重構需要：否
- 技術債務：無新增。發現一處可選收斂點 — `tab-state-tracking-service.js` 與 `content-script-coordinator-service.js` 兩個 `detectPageType` 邏輯近乎相同（含本次修法），屬既存重複，非本 ticket 引入；本 ticket 範圍受限「只改 4 檔對應行」，不擴大。
- 程式碼品質：A。修改為最小化字串與條件調整，加註 W1-029.1 業務情境註解說明 Chrome match pattern fragment 限制與向後相容意圖。
- 結論：無需重構。本 ticket 為過時 URL pattern 的最小修正，修改範圍小、語意清晰、有對應回歸測試覆蓋；既存的 `detectPageType` 重複若需收斂應由獨立 ticket 處理（範圍涉及 4 檔以上，超出本 ticket 認知負擔閾值）。
