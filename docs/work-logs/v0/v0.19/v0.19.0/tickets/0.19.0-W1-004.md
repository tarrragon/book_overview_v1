## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

**表層症狀**：popup 載入時 console 出現 5 個 `[Missing: KEY]` 佔位符（W1-001.1 實機驗證發現）：
- POPUP_INTERFACE_LOADED
- POPUP_SCRIPT_LOADED
- POPUP_INIT_START
- POPUP_INIT_COMPLETE
- INITIALIZATION_COMPLETE

**直接原因**：
1. 前 4 個 key 被 `src/popup/popup.js:79` 透過 `new MessageDictionary({...})` 嘗試註冊，但 `MessageDictionary` constructor 簽章為 `constructor (language = 'zh-TW')`，第一個參數被當作 `language` 而非初始訊息，註冊靜默失敗
2. `INITIALIZATION_COMPLETE` 由 `src/popup/popup-error-handler.js` 使用 `new Logger('PopupErrorHandler')`（無 messages dict），fallback 至 `GlobalMessages` 但該 key 未定義

### 影響範圍

- 表層（本 ticket）：5 個 key 必須有對應訊息字串
- 系統性（另開 follow-up ticket）：`MessageDictionary` constructor 簽章 bug 影響 3 處呼叫：
  - src/popup/popup.js:79（popupMessages，影響 21 個 key）
  - src/ui/book-search-filter-integrated.js:37（searchUIMessages）
  - src/platform/readmoo-platform-migration-validator.js:40（validatorMessages）

### 相關 Error Pattern

無既有 error pattern；popup logger 雙來源（local dict broken + GlobalMessages fallback missing）為新發現模式。

### 修復路徑決策（AskUserQuestion）

用戶選擇 **Path A：外科修復 + 建根因 follow-up ticket**：
- 本 ticket 將 5 個 key 加入 `MessageDictionary._loadDefaultMessages()`（GlobalMessages）解 AC
- 另建 IMP follow-up ticket 追蹤 constructor 系統性 bug（遵 quality-baseline 規則 5）

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作策略

修改 `src/core/messages/MessageDictionary.js` 的 `_loadDefaultMessages()`，在「Chrome Extension 相關」分組內補上 5 個 messageKey：

```javascript
POPUP_INTERFACE_LOADED: 'Popup 介面已載入',
POPUP_SCRIPT_LOADED: 'Popup 腳本載入完成',
POPUP_INIT_START: '開始初始化 Popup 介面',
POPUP_INIT_COMPLETE: 'Popup 介面初始化完成',
INITIALIZATION_COMPLETE: '初始化流程完成',
```

### 設計決策

- **訊息文字無 emoji**：對齊 `_loadDefaultMessages()` 既有訊息風格（既有 EXTENSION_READY / POPUP_OPENED 等皆無 emoji）；popup.js 原本 dict 含 emoji 是因預期走 local dict（已知 broken），既然落到 global 就應對齊 global 規範
- **不修 popup.js / popup-error-handler.js**：constructor 系統性 bug 屬另一 ticket 範圍，本 ticket 不擴大 scope
- **不刪 popup.js 既有 dict 定義**：保留作為「原始設計意圖」證據，由 follow-up ticket 一併處理（避免本次提早改動造成 root-cause ticket 失去重現點）

### 不在本 ticket 範圍

- `MessageDictionary` constructor union signature 修復（→ follow-up ticket）
- popup.js / book-search-filter / validator 三處呼叫端的清理（→ follow-up ticket）
- popup.js 其他 16 個自訂 key（VERSION_ERROR / EXTRACTION_ERROR 等）目前未觸發 `[Missing:]` 因 INFO level 載入流程未呼叫，但本質同樣 broken（→ follow-up ticket）
