## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

version-release CLI 從他專案 sync 而來，三處硬編碼假設與本專案 all-on-main 工作流不符：

1. **tag 命名硬編碼**：`git_merge_and_push()`（version_release.py:1966）寫死 `tag_name = f"v{version}-final"`，但本專案既有 tag（v0.18.0 / v0.17.4）皆為 plain `v{version}` 無 -final 後綴。
2. **feature-branch merge 硬編碼**：同函式寫死 `feature_branch = f"feature/v{major_minor}"` 並無條件執行 merge（Step 3.4）與分支清理（Step 3.7）；本專案無 feature 分支，CLI 直接執行會 merge 不存在分支。
3. **Step 1 worklog 路徑扁平假設**：`check_worklog_completed()` 查 `docs/work-logs/v{version}/`（扁平），本專案 worklog 在巢狀路徑 `docs/work-logs/v0/v0.19/v0.19.0/`，導致 false FAIL「找不到工作日誌檔案」。

### 影響範圍

- `.claude/skills/version-release/scripts/version_release.py`：`git_merge_and_push`、`check_worklog_completed`、`DEFAULT_VERSION_RELEASE_CONFIG`、`load_version_release_config`。
- 新增 helper `resolve_worklog_dir`。
- 新增 repo 配置 `.claude/.version-release.yaml`。

### 相關 Error Pattern

無直接對應 PC；config 化解法符合 framework-asset-separation（專案設定與工具邏輯分離）原則。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### config 設計（DEFAULT_VERSION_RELEASE_CONFIG 新增三 top-level 鍵）

| 鍵 | 預設值 | 語意 |
|----|--------|------|
| `release_workflow` | `trunk` | `trunk`=all-on-main，跳過 feature-branch merge 與分支清理；`feature-branch`=維持原行為 |
| `tag_format` | `v{version}` | tag 命名範本（plain，無 -final）；保留 -final 需顯式設 `v{version}-final` |
| `worklog_path_pattern` | `docs/work-logs/v{major}/v{major_minor}/v{version}` | worklog 目錄範本，支援 {version}/{major_minor}/{major} 佔位符；扁平結構設 `docs/work-logs/v{version}` |

`load_version_release_config()` 的缺漏欄位 merge 清單同步加入三鍵，確保部分配置檔仍取得預設值。

### 實作變更

1. **tag_format**：`git_merge_and_push` 改 `tag_name = tag_format.format(version=..., major_minor=...)`，移除寫死 `-final`。
2. **release_workflow**：以 `use_feature_branch = (release_workflow == "feature-branch")` 包裹 Step 3.4 merge 與 Step 3.7 branch 清理；trunk 模式印 `[SKIP]` 不報錯。
3. **worklog_path_pattern**：新增 helper `resolve_worklog_dir(root, version, pattern)` 解析範本；`check_worklog_completed` 改用 config 解析 version_subdir，取代寫死 `worklog_dir / f"v{version}"`。

### 配置檔位置決策（.claude/ fallback）

repo root `.version-release.yaml` 受 branch-verify 保護分支 hook 阻擋（豁免清單僅 .claude/ / docs/ / CLAUDE.md 等）。為使 all-on-main 工作流的配置可直接 commit 到 main，`load_version_release_config()` 改為依序查找 `<root>/.version-release.yaml`（優先，向後相容）與 `<root>/.claude/.version-release.yaml`（fallback，豁免路徑）。本專案配置置於 `.claude/.version-release.yaml`。

### 不在本 ticket 範圍

- `check_technical_debt_status()` 仍印扁平路徑 `docs/work-logs/v{version}/tickets`（graceful no-op，非 FAIL），未納入本次 config 化（acceptance 僅要求 Step 1 worklog 路徑）。
- 實際 release（tag/push）由 PM 後續處理。
