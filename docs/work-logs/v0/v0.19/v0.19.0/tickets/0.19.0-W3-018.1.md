### 修復摘要

在 `.claude/skills/ticket/ticket_system/commands/resume.py` 的 `_find_handoff_file` fallback 迴圈新增第三層匹配：`target_ticket_id` 頂層欄位比對。

位置：direction 反向匹配之後、`ticket_id` 欄位匹配之前（resume.py:124-128）。

```python
# target_ticket_id 頂層欄位匹配（支援 handoff --next 模式：
# direction="context-refresh" 無後綴，target 存於 target_ticket_id 欄位）
target_tid = data.get("target_ticket_id")
if target_tid and target_tid == ticket_id:
    return (json_candidate, "json")
```

### 設計決策

- **匹配順序**：direction-suffix → target_ticket_id → ticket_id 欄位。direction-suffix 優先以保持 `--auto` 模式既有行為一致；target_ticket_id 與 ticket_id 兩者都在 fallback 尾段，順序對結果無影響（兩個欄位語意互斥：source 用 ticket_id，target 用 target_ticket_id）。
- **archive 隔離**：target_ticket_id 匹配與其他 fallback 一致，僅在 `subdir == HANDOFF_PENDING_SUBDIR` 時啟用。

### 不在本 ticket 範圍

- gc --force 旗標（W3-018.2，P2）：由 parent ANA 規劃為獨立 IMP。
- handoff JSON schema 變更：`target_ticket_id` 欄位已存在於 handoff.py:1564/1478，本 ticket 僅在 resume 端讀取。
