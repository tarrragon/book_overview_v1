## Problem Analysis

### 問題根因

W1-112 ANA WRAP 分析確認：`src/core/logging/Logger.js` L65 constructor 簽章為 `(name = 'App', level = 'INFO')`，L68 hardcoded `this.messages = GlobalMessages`。3 處 caller（popup.js:103 / book-search-filter-integrated.js:54 / readmoo-platform-migration-validator.js）以 `new Logger('Name', 'INFO', localMessages)` 形式傳遞第三參數 `localMessages`，但 JS 對未宣告參數靜默 ignore，導致整個 popup/search-filter/validator local dict 機制從未生效。

W1-105 雖修 MessageDictionary constructor 為 union signature 通過 41/41 測試，但僅完成 local dict 註冊機制，未修 Logger 取用環節──runtime 上 `logger.info('KEY')` 仍走 `GlobalMessages.get()`，local dict 完全旁路。

### 影響範圍

- `src/core/logging/Logger.js`（本 ticket 唯一程式碼修改）

行為影響（無需改 caller，因 caller 早已傳第三參數）：
- `src/popup/popup.js` 22 個自訂 messageKey 預期生效
- `src/ui/book-search-filter-integrated.js` 自訂 messageKey 預期生效
- `src/platform/readmoo-platform-migration-validator.js` 自訂 messageKey 預期生效

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### 修復摘要

`src/core/logging/Logger.js` L65-71 constructor 簽章擴充第三參數 `messages = GlobalMessages`，L68 改為 `this.messages = messages`（取自參數而非 hardcoded 引用）。完全向後相容：未傳第三參數的 caller 仍取得 `GlobalMessages` 預設行為；傳第三參數的 caller（popup / search-filter / validator）的 local dict 從本次修復起開始生效。

### 程式碼變更

```diff
 class Logger {
   /**
    * 建立 Logger 實例
+   *
+   * 業務情境：caller 可注入自訂 MessageDictionary（local dict）以覆寫 GlobalMessages
+   * 預設行為。W1-112 ANA 根因：原 constructor 僅 (name, level)，hardcoded
+   * this.messages = GlobalMessages，導致 popup/search-filter/validator 等 3 處
+   * `new Logger('Name', 'INFO', localMessages)` 的第三參數被 JS 靜默 ignore，
+   * 整個 local dict 機制從未生效（W1-105 修 MessageDictionary union signature
+   * 雖通過測試但 runtime 無效用）。
+   *
    * @param {string} name - Logger 名稱
    * @param {string} level - 日誌等級（預設: 'INFO'）
+   * @param {Object} messages - 訊息字典（預設: GlobalMessages，可注入 local dict）
    */
-  constructor (name = 'App', level = 'INFO') {
+  constructor (name = 'App', level = 'INFO', messages = GlobalMessages) {
     this.name = name
     this.level = level
-    this.messages = GlobalMessages
+    this.messages = messages
     this._buffer = []
     this._maxBufferSize = 100
   }
```

### 設計決策

| 決策 | 採用方案 | 替代方案及拒絕原因 |
|------|---------|------------------|
| 參數位置 | 第三參數 `messages = GlobalMessages` | 拒絕「options 物件」：3 處 caller 已用位置參數呼叫，改 options 破壞既有 caller |
| 預設值 | `GlobalMessages` 引用 | 拒絕「null + 內部 fallback」：增加分支邏輯，預設參數更直觀 |
| 內部成員 | `this.messages = messages` | 拒絕「保留 hardcoded」：W1-112 根因即此 hardcoded，必須改為取自參數 |
| caller 變更 | 零變更（caller 早已傳第三參數） | N/A |
