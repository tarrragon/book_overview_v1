### 範圍擴充說明（執行階段發現）

W3-042 原定範圍：修 pyproject.toml setuptools 配置缺失（install 失敗）。
執行階段發現 install fix 後 runtime 仍不可用：

**第二個 bug**：`scripts/version_release.py:184` `get_project_root()` 寫死 `Path(__file__).parent.parent.parent.parent.parent`，此邏輯假設 source tree 結構（`.claude/skills/version-release/scripts/version_release.py` → parent x5 = project root）。`uv tool install` 後 `__file__` 變為 `~/.local/share/uv/tools/version-release/lib/python3.X/site-packages/version_release.py`，parent x5 進入 `~/.local/share/` 上層，找不到 `docs/todolist.yaml`。

**為什麼納入 W3-042 而非另開**：
- 兩個 bug 同屬「讓 version-release skill installed 後可用」這個 single coherent goal
- AC3「version-release start --version 0.19.0 --dry-run 可預覽（為 W1-073 鋪路）」原本就涵蓋 runtime 可用性
- 拆 ticket 反而增加 PM 追蹤成本與 context 切換

**修正策略**：
- 改 `get_project_root()` 採三層 fallback：(1) cwd 含 .git 或 package.json → 用 cwd (2) cwd parents 向上找 .git → 用該 ancestor (3) dev mode 相對路徑作最後 fallback
- 加單元測試覆蓋 installed / dev / cwd-from-subdir 三情境

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 兩個 Bug 的修復

**Bug 1: pyproject.toml setuptools 配置缺失** （install 失敗）

修法：改用 hatchling backend（與 ticket skill pattern 對齊）+ 明示 packages = [] + only-include single-file module。最終 pyproject.toml:

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "version-release"
version = "1.0.0"
description = "Version Release Tool for Monorepo"
readme = "SKILL.md"
requires-python = ">=3.10"
dependencies = ["pyyaml>=5.0"]
license = {text = "MIT"}

[project.scripts]
version-release = "version_release:main"

[tool.hatch.build.targets.wheel]
sources = ["scripts"]
only-include = ["scripts/version_release.py"]
```

**Bug 2: get_project_root() installed mode 失效** （runtime path discovery）

修法：scripts/version_release.py:184 改為三層 fallback：
1. cwd 含 .git 或 package.json → 用 cwd（installed mode 主路徑）
2. cwd parents 向上找 .git → 用該 ancestor（從子目錄執行的情境）
3. dev fallback：__file__ 上溯（向後相容 source tree 直接執行）
4. 最後 fallback：cwd（後續檔案存取會 fail 並回報明確錯誤）

加 `Why:` docstring 說明 installed mode 失效的根因（site-packages parent x5 進入 ~/.local/share/ 上層）。
