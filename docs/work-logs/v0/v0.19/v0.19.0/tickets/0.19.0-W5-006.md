### 問題根因

v0.19.0（W1-042/W1-045 期間）為 README 補寫「權限與隱私聲明」段落並新增 PRIVACY.md，內容依當下 manifest.json 實際 permission 撰寫。發布流程原無任何步驟保證版本發布或推進時檢查權限宣告變更並同步更新權限說明，權限說明會隨 manifest.json 變更而 drift。

### 影響範圍

version-release skill 發布前檢查清單、pm-rules/version-progression.md。屬流程缺口正規化，不涉及程式碼。

### 相關 Error Pattern

無直接對應 error-pattern。本 ticket 為發布流程缺口的正規化（quality-baseline 規則 5：發現即追蹤）。

## Solution

### 變更摘要

1. `.claude/skills/version-release/SKILL.md`：
   - 新增「權限需求變更檢查（Chrome Extension）」章節：比對 manifest.json 的 permissions/host_permissions 與上一發佈版本差異；有變更則同步更新 README「權限與隱私聲明」、PRIVACY.md、Chrome Web Store 後台權限說明；無變更則於發佈紀錄標註。
   - 「使用流程檢查清單」新增「權限需求變更檢查已完成」項。

   - 新增「權限需求變更檢查（Chrome Extension 專案）」提醒章節，交叉引用 version-release skill 的完整步驟，不重複定義（避免 DRY 違規）。
