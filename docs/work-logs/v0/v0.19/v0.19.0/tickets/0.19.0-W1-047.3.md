## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

UC-04 匯入功能 `overview-page-controller.handleFileLoad` 只將解析後書籍寫入記憶體（`currentBooks` / `filteredBooks`）並更新 UI，不寫入 `chrome.storage.local`。IMP-A/B 已讓 `BookFileImporter` 回傳 `ImportResult { books, tagCategories, tags }`，但 `tagCategories` / `tags` 在 controller 邊界被丟棄（見 `handleFileLoad` 既有註解「IMP-C 接手 storage 持久化」）。結果：匯入後重整 overview 頁面，匯入的書籍與標籤全部遺失。

### 影響範圍

| 檔案 | 影響 |
|------|------|
| `src/overview/overview-page-controller.js` | `handleFileLoad` 接入覆蓋模式持久化，成功才更新 UI |
| `src/storage/adapters/tag-storage-adapter.js` | 需新增 `replaceAllData` 公開 API（既有無整批覆寫 API） |

**關鍵設計發現**：`tag-storage-adapter.js` 目前 `module.exports` **未匯出**任何整批覆寫 API。`saveToStorage` / `saveBooksWrapper` / `loadBooks` 皆為私有函式。覆蓋模式（清空後載入）無法用逐項 CRUD 表達（`createTag` 含 duplicate_name 檢查、會與既有資料疊加），故規格定義新增 `replaceAllData`。

### 相關 Error Pattern

無直接對應 PC error pattern。屬功能缺口（IMP-B 已預留接線點）。

### Phase 1 產出

功能規格：`docs/work-logs/v0/v0.19/v0.19.0/tickets/0.19.0-W1-047.3-feature-spec.md`（Full 模式 6 區段，/spec validate 2 次迭代收斂通過）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### Phase 1 功能規格摘要

完整規格見 `0.19.0-W1-047.3-feature-spec.md`。實作要點：

1. **tag-storage-adapter 新增 `replaceAllData({ books, tags, tagCategories })`**：覆蓋模式整批寫入，三 key（`readmoo_books` / `tags` / `tag_categories`）一併取代；經 `withAtomicRollback` 保證原子性、`operationLock` 序列化、寫入前 `checkQuotaLevel` 攔截配額不足。回傳 `{ success, error?, counts? }`。
2. **overview-page-controller `handleFileLoad` 接線**：解構 `ImportResult` 後呼叫 `replaceAllData`，success=true 才呼叫 `_updateUIWithBooks`；失敗呼叫 `showError` 並中止 UI 更新。
3. **覆蓋語意**：空 tags / tagCategories 也覆蓋為 `[]`（清空後載入）；空書庫不視為錯誤。
4. **範圍邊界**：本 ticket 預設一律覆蓋模式，不含模式選擇 UI（IMP-E）與合併邏輯（IMP-D）。

### 行為場景（GWT，移交 Phase 2）

6 場景：覆蓋成功 / 僅含 books / 空書庫 / 配額不足 / 寫入中途失敗回滾 / 檔案驗證失敗（既有不變）。詳見 feature-spec §3。

### 驗收條件

AC-1 ~ AC-8（覆蓋寫入 / 持久化 API / controller 接線 / 空集合覆蓋 / 原子性 / 配額攔截 / 失敗不更新 UI / 測試 100%）。詳見 feature-spec §6。
