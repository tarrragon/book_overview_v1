## Problem Analysis
<!-- Schema[DOC/Problem Analysis]: 選填（若 DOC 起因於缺陷或盤點結論可填）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`.claude/hooks/branch-verify-hook.py` 的 `exempt_prefixes = [.claude/, docs/, scripts/experiments/]`，tests/ 未列入豁免。原始設計意圖為防止 tests/ 紅燈直接進 main（tests/ 與 src/ 緊耦合，獨立 commit 易破壞測試通過率 100% 承諾）。

### 影響範圍

- 每次派發涉及 tests/ Edit 的代理人在 main branch cwd 都會被 hook deny
- 代理人若未正確 fallback 會 self-imposed early stop（PC-112 同精神）
- 浪費代理人回合預算（PC-042 ~20 tool call 上限）
- 觀察案例：W1-048.3 sage / W1-081 本 session（PM 三度撞牆 fallback git checkout -b）

### 三方案 WRAP 評估

| 方案 | 描述 | 評估 |
|------|------|------|
| A | 強制 worktree 派發 | 當前 worktree CLI 有 bug（W1-118），不可依賴；即便修復對小型 ticket overhead 仍高 |
| B | 擴大 exempt 加 tests/ | 簡單但風險高：tests 紅燈直接進 main 違反 quality-baseline 規則 1 |
| C | PM prompt 明示 branch | 依賴 PM 自律，但 agent 仍需先在 branch 才能 Edit，問題未解 |
| D | 混合：保護 + PM 派發前自建 feat branch | 本 session 已驗證可行（5 commit fast-forward 整合）|

用戶選擇方案 D（本 session AskUserQuestion 2026-05-29）。

- **Action**：派發前 main 上 `git checkout -b feat/<ticket-id>-<short-desc>`；完成後 `git checkout main && git merge feat/<branch-name> --no-edit`
