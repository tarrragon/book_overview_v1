## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->
### 調查結論：非設計權衡衝突，而是 stale 草案

原疑似「兩 spec 有理由的設計決策衝突」，調查後確認為單純 doc drift：
- 實作 export（src/export/book-data-exporter.js join '; '）與 import（src/overview/import/content-parser.js split '; '）**端到端一致採 `; `**，round-trip 正常。
- csv-export-spec.md 的管道符 `|` 決策**從未進入實作**；它才是 stale/錯誤的一方。
- SSOT 確立為 export-interchange-format-v2.md（status: approved, v2.0.0）§4.2/§5.1 + 實作，皆 `; `。

`;`-locale 顧慮（csv-export-spec 原 `|` 理由）由標準 CSV cell 引號包覆化解：`; ` 在引號內不被視為欄位分隔符。無需改實作。

### 變更摘要（docs/spec/csv-export-spec.md）

1. §陣列序列化策略「決策：管道符 `|` 分隔」→「決策：分號加空格 `; ` 分隔」
2. 加入 SSOT 對齊聲明（引用 export-interchange-format-v2.md，標注 `|` 為已廢止草案、從未實作）
3. 範例值 `作者A|作者B` → `作者A; 作者B`；補 tagNames/tagCategories 衍生欄範例
4. 選擇理由表更新：`; ` 為採用、`|` 標草案已廢止、`;`-locale 風險由引號包覆說明
5. import 對稱性描述對齊實作 content-parser.js split('; ')
6. §Import 對稱性表格「陣列用 `|` 分隔」→「陣列用 `; ` 分隔」

### 自檢結果（Layer 1）
- [x] csv-export-spec.md 分隔符與 export-interchange-format-v2.md(approved) 一致（皆 `; `）
