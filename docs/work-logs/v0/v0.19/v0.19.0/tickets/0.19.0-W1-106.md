## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W1-004 ticket（補 popup logger 5 個缺漏 messageKey）在 W1-105 修復 MessageDictionary constructor bug 之前完成。當時 constructor 簽章為 `constructor(language = 'zh-TW')`，無法接受初始 messages 物件；popup.js 雖然嘗試 `new MessageDictionary({messages...})` 註冊 5 個 popup-specific key（POPUP_INTERFACE_LOADED / POPUP_SCRIPT_LOADED / POPUP_INIT_START / POPUP_INIT_COMPLETE / INITIALIZATION_COMPLETE），但 messages 物件被當作 language 參數，註冊靜默失敗。

W1-004 採表層修復方案：將 5 個 key 直接寫入 `MessageDictionary._loadDefaultMessages()`，讓所有 instance 預設都含這 5 個 key，繞過 constructor bug。W1-105 已修復 constructor 後，popup.js local dict 註冊路徑恢復正常，GlobalMessages 中的 5 個副本變成冗餘且違反模組邊界（popup-specific 不該進跨模組共用 GlobalMessages）。

### 影響範圍

- `src/core/messages/MessageDictionary.js`: `_loadDefaultMessages()` 移除 5 個 key
- `src/popup/popup.js`: 無需變動（local dict 已含這 5 個 key）
- `tests/unit/core/messages/MessageDictionary.test.js`: 無需變動（未斷言這 5 個 key）
- `tests/integration/chrome-extension/popup-interface.test.js`: 無需變動（透過 popup.js local dict 註冊驗證流程）

### 相關 Error Pattern

無新發現的錯誤模式。本 ticket 屬 W1-105 修復後的清理工作，依 quality-baseline 規則 5「所有發現必須追蹤」建立。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

`src/core/messages/MessageDictionary.js` `_loadDefaultMessages()` 中移除以下 5 個 popup-specific key（原 line 87-91 區域）：

| Key | 原中文 | 移除理由 |
|-----|-------|---------|
| POPUP_INTERFACE_LOADED | 'Popup 介面已載入' | popup 模組專屬，由 popup.js local dict 註冊 |
| POPUP_SCRIPT_LOADED | 'Popup 腳本載入完成' | 同上 |
| POPUP_INIT_START | '開始初始化 Popup 介面' | 同上 |
| POPUP_INIT_COMPLETE | 'Popup 介面初始化完成' | 同上 |
| INITIALIZATION_COMPLETE | '初始化流程完成' | 同上（雖名稱通用，但實際僅 popup 用） |

### 保留的跨模組共用 key

- EXTENSION_READY / CONTENT_SCRIPT_LOADED / POPUP_OPENED / BACKGROUND_SCRIPT_ACTIVE（屬 Chrome Extension 跨模組共用語意，保留於 GlobalMessages）

### 設計決策

- W1-105 已修復 MessageDictionary constructor 接受初始 messages 物件的 bug；popup.js local dict 已能正確註冊這 5 個 key，無需在 GlobalMessages 重複定義
- 不修改 popup.js（local dict 維持，避免破壞 W1-004 已驗證的 5 個 messageKey 註冊路徑）
- 不修改其他 caller（本 ticket 範圍僅清理 GlobalMessages 副本）
