## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`tab-state-tracking-service.js` 與 `content-script-coordinator-service.js` 各自實作了近乎相同的 `detectPageType(url)` 方法（W1-029.1 時兩處同步加入 read.readmoo.com hash route 相容性）。共 13 行核心 URL pattern 判斷邏輯重複，未來 URL pattern 變更須兩處同步修改。

兩處差異：tab-state-tracking-service 多了 `isReadmooPage(url)` 前置守門（!url 與 !isReadmooPage 時直接 return null），content-script-coordinator-service 缺此守門但同邏輯。收斂時統一行為（皆走 isReadmooPage 守門）。

### 影響範圍

- `src/background/domains/page/services/tab-state-tracking-service.js` (line 629-657)
- `src/background/domains/page/services/content-script-coordinator-service.js` (line 297-316)
- 既有測試：
  - `tests/unit/background/domains/page/services/tab-state-tracking-service-page-detect.test.js`
  - `tests/unit/background/domains/page/services/content-script-coordinator-service-page-detect.test.js`

範圍邊界：
- **不動**：popup domain 的 `popup-communication-service.isReadmooPage`（不同職責 / 不同呼叫端，且 W1-029 已驗證一致性）
- **不動**：`page-detection-service.detectPageType` 有不同介面（接 url + title + tabId），屬獨立服務不在本 ticket 範圍
- **不動**：content domain 的 `page-detector.js`（content script 內部偵測，與 background service 不同生命週期）

### 相關 Error Pattern

無；本任務為 quality-baseline 規則 5 衍生的純 DRY 收斂，無 bug 觸發。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

新建 `src/background/domains/page/utils/page-type-detector.js` 純函式模組，提供 `isReadmooPage(url)` 與 `detectReadmooPageType(url)` 兩個共用函式；兩個 service 的 `detectPageType` 改為 wrapper 委派此模組。

### 設計決策

| 決策 | 選項 | 採用 | 理由 |
|------|------|------|------|
| 抽出位置 | (a) 新建 `utils/page-type-detector.js` (b) 移到 `page-detection-service.js` (c) 移到 `content/detectors/page-detector.js` | (a) | service 介面零變動需求 + 純函式無需依賴 ctor + 與 background domain 邊界一致；page-detection-service 介面不同（接 url+title+tabId），content/detectors 屬不同生命週期 |
| async 簽名 | (a) 保留 async (b) 改 sync | (a) | Context Bundle 指明「service 介面零變動」；avoid breaking 既有呼叫端 await |
| isReadmooPage 守門統一 | (a) 兩 service 行為對齊（都走守門） (b) 維持差異 | (a) | DRY 收斂的核心動機；非 Readmoo URL 兩處皆應 return null 才是正確語意 |
