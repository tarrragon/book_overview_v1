### 問題根因

`_extractBooksFromData` 行 605-639 走四路徑分流，但實際語意與 SPEC-EXPORT-V2 §2.1 / §3.1 不一致：

| 現況路徑 | 觸發條件 | 對應 spec 語意 |
|---------|---------|--------------|
| 1. v1 JSON converter | `fileFormat='json' && detectFormatVersion==='v1'` | 合法 v1（spec §2.1 Rule 3/4） |
| 2.a DirectArray | Array.isArray(data) | 已被 detectFormatVersion Rule 3 攔截走 v1，**dead branch for JSON** |
| 2.b WrappedBooks | data.books is array | v2 合法形狀 / v1 退化（spec §3.1 / §2.1 Rule 4） |
| 2.c MetadataWrap | data.data is array | **spec 未定義**（歷史包袱，IMP-A~B 容錯加入） |
| 3. 空物件 {} | Object.keys(data).length === 0 | v1 退化合理（合法空匯入） |
| 4. throw | 其他 | 合理錯誤契約 |

實際命中分布：
- JSON 路徑 v1 → 路徑 1（converter）
- JSON 路徑 v2（合法）→ 路徑 2.b WrappedBooks
- JSON 路徑 metadataWrap → 路徑 2.c（**僅此一個情境為 path 2 unique 貢獻**）
- CSV 路徑 → 路徑 2.a DirectArray（fileFormat='csv' bypass detectFormatVersion）
- 空 JSON {} → 路徑 3
- 其他 → 路徑 4

F10「v2 應只有一種合法形狀」實意：v2 path 應由 detectFormatVersion='v2' 顯式分流（如同 v1 path 1），而非依靠形狀辨識 fallthrough。MetadataWrap 屬無 spec 依據的歷史容錯，應決策保留或拒絕。

##### 決策 B：v2 path 顯式化

| 選項 | Why | Consequence |
|------|-----|------------|
| B1 維持 fallthrough（現況） | 改動最小 | v2 與 metadataWrap / DirectArray-for-CSV 共用形狀辨識，語意混雜，F10「v2 應只有一種合法形狀」未解 |
| **B2 顯式 v2 path（採納）** | 對齊 v1 path 1 已 `detectFormatVersion === 'v1'` 顯式分流的設計；使「v2 合法形狀 = WrappedBooks」可在程式碼結構上看見；認知負擔由「形狀辨識→反推版本」改為「版本→形狀」單向 | 新增一個 if-branch，但消除 2.a DirectArray dead branch（detectFormatVersion Rule 3 已將純陣列攔截為 v1）；fallthrough 容錯區塊縮小為「MetadataWrap + 空物件 + throw」三種非 spec 路徑 |

##### 決策 C：CSV path bypass 重構（先 fileFormat 分流 再 detectFormatVersion）

| 選項 | Why | Consequence |
|------|-----|------------|
| C1 維持 `fileFormat === 'json'` 條件單點 gate | 改動最小 | CSV 仍走 v2 path fallthrough（觸發 _isDirectArrayFormat），CSV 語意散落兩處 |
| **C2 頂層先 fileFormat 分流（採納）** | 程式碼結構即文件：讀者第一眼看到 `if (fileFormat === 'csv')` → CSV path 獨立，再讀 JSON 子流程；消除 CSV 走 v2 形狀辨識的隱性耦合；F11 CSV 不走版本偵測在程式碼結構上顯式 | 多一層巢狀但每層責任單一；CSV path 變為「直接取陣列 + 空 tag 區段」單一表達式 |

**採納 C2**：頂層 `if (fileFormat === 'csv')` 直接回傳 `{ books: Array.isArray(data) ? data : [], tagCategories: [], tags: [] }`。JSON 子流程再分 v1 / v2 / 空物件 / MetadataWrap 相容 / throw。

#### 新 path 結構偽碼

```javascript
_extractBooksFromData(data, fileFormat = 'json') {
  // 頂層分流 1：CSV — 不走版本偵測（F11 顯式化）
  if (fileFormat === 'csv') {
    return {
      books: Array.isArray(data) ? data : [],
      tagCategories: [],
      tags: []
    }
  }

  // 頂層分流 2：JSON — 走版本偵測
  const version = detectFormatVersion(data)

  // JSON path A：v1 — converter 完整三區段
  if (version === 'v1') {
    const converted = convertV1ToV2Data(data)
    return {
      books: converted.books,
      tagCategories: this._normalizeToArray(converted.tagCategories),
      tags: this._normalizeToArray(converted.tags)
    }
  }

  // JSON path B：v2 — spec §3.1 唯一合法形狀（data.books）
  if (version === 'v2') {
    return {
      books: data.books,
      tagCategories: this._extractTagSection(data, 'tagCategories'),
      tags: this._extractTagSection(data, 'tags')
    }
  }

  // JSON path C：MetadataWrap 歷史相容（spec 未定義，保留為相容路徑）
  if (this._isMetadataWrapFormat(data)) {
    console.warn('[book-file-importer] 偵測到 metadata-wrap 形狀（非 SPEC-EXPORT-V2 定義），走歷史相容路徑')
    return {
      books: data.data,
      tagCategories: this._extractTagSection(data, 'tagCategories'),
      tags: this._extractTagSection(data, 'tags')
    }
  }

  // JSON path D：空 JSON 物件（v1 退化合理）
  if (data && typeof data === 'object' && Object.keys(data).length === 0) {
    return { books: [], tagCategories: [], tags: [] }
  }

  // JSON path E：throw — 既有錯誤契約（零回歸）
  const error = new Error('JSON 檔案應該包含一個陣列或包含books屬性的物件')
  error.code = ErrorCodes.VALIDATION_ERROR
  error.details = { category: 'validation' }
  throw error
}
```

**helper 處置**：
- `_isDirectArrayFormat`：**移除**（detectFormatVersion Rule 3 已將純陣列判為 v1，dead branch）
- `_isWrappedBooksFormat`：**移除**（v2 path 改為依 `detectFormatVersion === 'v2'`，不再形狀辨識；TC-04 透過 detectFormatVersion Rule 2 命中 v2）
- `_isMetadataWrapFormat`：**保留**（path C 唯一使用者，標註為「歷史相容判定」）
