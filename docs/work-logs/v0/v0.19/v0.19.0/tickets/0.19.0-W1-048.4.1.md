## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`src/overview/book-file-importer.js` `_validateFieldTypes`（L753-756）對 `book.cover` 嚴格要求 `typeof === 'string'`，但 `_validateRequiredFields`（L744-746，W1-047.1 修復）已不要求 cover——形成 selectively-loose validation 反模式。缺 cover 欄位的合法 v2 JSON 書籍會在 `_isValidBook` 串接的 `_validateFieldTypes` 階段被靜默過濾。

### 影響範圍

- `src/overview/book-file-importer.js` `_validateFieldTypes`（單一函式單一行修改）
- `_isValidBook` 過濾鏈下游所有路徑：`parseContent(json)` / `_processBookData` / `_filterValidBooks`
- 缺 cover 的 v2 JSON 書籍從 v1.0 起被靜默丟棄（無錯誤回報、無 warning）

### 相關 Error Pattern

無既有 error pattern；本案屬 W1-047.1 部分修復遺留的局部不一致（_validateRequiredFields 改寬鬆但 _validateFieldTypes 漏改），由 W1-048 Phase 4 評估 F12 識別、W1-048.4 ANA 釐清。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

採 W1-048.4 ANA 結論方案 A（真寬鬆）：`_validateFieldTypes` 容許 `book.cover === undefined`。修改範圍：

| 檔案 | 修改類型 | 行數變動 |
|------|---------|---------|
| `src/overview/book-file-importer.js` | `_validateFieldTypes` 容許 cover undefined + 加註說明 | +9 / -1 |
| `tests/unit/overview/book-file-importer-import-result.test.js` | 新增 TC-12（cover undefined / empty / valid 三情境通過）+ TC-13（cover number / null 防護回歸） | +52 / 0 |

### 設計決策

**為何方案 A（真寬鬆）而非方案 B（真嚴格）或 C（正規化）**：

- 方案 B（_validateRequiredFields 加回 cover required）違反 SPEC-EXPORT-V2 §3.5 明文「cover | string | 否 | "" | 封面圖片 URL」與 data-management.md `required: false`，已被 W1-048.4 ANA 否決
- 方案 C（正規化層補 default）改動範圍大、可能影響其他欄位策略，且最終仍需放寬 `_validateFieldTypes`，方案 A 為其前置條件
- 方案 A 改 1 行核心邏輯、與規格 100% 對齊、不破壞 v1 轉換路徑（v1 轉換已產出 `cover: ''`，string 型別檢查仍通過）

### 防護回歸

TC-13 防止「放寬 undefined 後誤放行其他非法型別」：

- `cover: 123` (number) → 仍被過濾
- `cover: null` (typeof null === 'object') → 仍被過濾

僅 `undefined` 與 `string` 通過，與規格「cover 為 optional string」語意對齊。

### 不在本 ticket 範圍

- 「未來其他選填欄位加入 `_validateFieldTypes`」的同類問題盤點（W1-048.4 ANA Solution P 段建議，但屬另案）
- `_validateRequiredFields` / `_validateFieldTypes` 函式合併或重命名（屬 Phase 4 重構議題，非本 spec 範圍）
