### 修復摘要

依 ticket strategy 完成三事：

#### (a) generateBooksWithProgress 標 @deprecated

在 `tests/helpers/test-data-generator.js:382` `generateBooksWithProgress` 方法上方追加 JSDoc `@deprecated` 標記，明示：

- 廢棄理由：內部 `randomInt` 變動 + 邊界 clamp 在 heavy-reader-device 場景 ~5% 機率產生 avgProgress 浮點偏差觸發 toBeCloseTo flaky（W1-075 根因）
- 替代方法：`buildScenarioBooks`（整數確定性建構，總和精確 = averageProgress × bookCount）
- 保留原因：backward compatibility（IMP-077 子模式 A——死碼但保留以避免破壞潛在外部使用者）
- 違規範例 + 正確範例：JSDoc 內附並列對照

grep 驗證 `generateBooksWithProgress` 全 tests/ 無實際 consumer（僅 .backup 檔案與註解提及），符合 IMP-077 子模式 A 條件。

#### (b) buildScenarioBooks 抽出為 class 方法（已執行）

評估結論：**抽出可行且應執行**。理由：

- TestDataGenerator class 已是測試資料的 single source of truth（含 generateBooks/generateUsers/generateSettings 等同類方法）
- buildScenarioBooks 為純函式（只依賴 scenario 參數，不依賴 random seed），與 class 其他方法風格一致
- 抽出後可供未來其他 scenario 測試複用，避免在多個測試檔重複定義相同 helper

實作：在 `tests/helpers/test-data-generator.js:484` 新增 `buildScenarioBooks(scenario)` 方法，邏輯完全沿襲原 local function（整數下取整商 + 最後一本承接餘數）。

更新點：`tests/integration/workflows/cross-device-sync-workflow.test.js:127-170` 50 行 local function 定義刪除，line 131 改呼叫 `testDataGenerator.buildScenarioBooks(scenario)`。

#### (c) Guard clause 防負 progress 值

在 `buildScenarioBooks` 內部 `inProgressTotalNeeded` 計算後立即追加 guard：

```javascript
if (inProgressTotalNeeded < 0) {
  throw new Error(
    `[buildScenarioBooks] scenario "${name}": ` +
    `inProgressTotalNeeded=${inProgressTotalNeeded} < 0. ` +
    `completedRatio (${completedRatio}) × 100 超過 averageProgress (${averageProgress})，` +
    '無法達成目標平均（會產生負 progress）。' +
    '請調整 completedRatio 或 averageProgress 使 completedRatio × 100 <= averageProgress。'
  )
}
```

防護的失效情境（W1-075 code-explorer 提出的 F-1）：
- completedRatio=0.9 + averageProgress=50 → completedContribution=4500 > totalTarget=2500 → inProgressTotalNeeded=-2000
- 未防護時：inProgress 書籍會被 `Math.max(0, ...)` clamp 為 0，但 reduce 後 avgProgress 變成 (4500+0)/N，遠超 averageProgress=50 → 測試 assertion 失敗 + 訊息模糊
- 加 guard 後：scenario 設計階段即 throw 顯式錯誤，含 scenario 名稱 + 具體數值 + 修正建議，定位時間從 30 分鐘降至秒級

### 設計決策

- **保留 generateBooksWithProgress 不刪除**：採 IMP-077 子模式 A（標 @deprecated 不刪），原因：(1) class 對外暴露於 module.exports，可能有未檢出的外部 consumer (2) @deprecated 標記讓 IDE 自動提示替代方法 (3) 刪除屬 breaking change，需獨立 ticket 評估
- **buildScenarioBooks 不接受 `random` 參數**：保持純整數確定性，不引入隨機性（這正是替代 generateBooksWithProgress 的核心價值——避免 flaky）
- **Guard 訊息含 scenario name**：W1-075 ANA 採行修復方向 C 時已預見「未來新 scenario 設定錯誤難定位」的問題，guard 訊息中包含 scenario name 是核心修復點

### 不在本 ticket 範圍

- `generateBooksWithProgress` 內部 randomInt 變動邏輯重寫（已標 @deprecated，未來若有 consumer 反映 flaky 再開 ticket）
- 其他 test helper 函式的 deprecated/抽出整理（如需擴大範圍應另開 ticket）
