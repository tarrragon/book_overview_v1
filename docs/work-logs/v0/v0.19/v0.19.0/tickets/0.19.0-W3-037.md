### 實作策略（PCB pattern，承接 W3-026.1 commit 7f2ec9e6）

#### 4 個測試路徑（acceptance #1）

| # | 情境 | 輸入 | 預期行為 |
|---|------|------|---------|
| 1 | tty 互動模式 | `sys.stdin.isatty() == True` | `input_data = {}` → fallback 到 started_at 推斷 |
| 2 | JSONDecodeError | stdin 非法 JSON（如 `"abc"` 缺結束） | `input_data = {}` + warning log → fallback |
| 3 | JSON 但非 dict | stdin 為 `[]` / `null` / `"str"` | input_data 傳入 has_background_agents 後被 `isinstance(input_data, dict)` 擋下 → fallback |
| 4 | 整合路徑 | JSON 含 `background_tasks=[{...}]` | generate_hook_output 收到正確 input_data → has_background_agents 回 True |

#### 實作要點（acceptance #2）

- 用 `monkeypatch.setattr("sys.stdin", io.StringIO(...))` 或 mock 物件隔離 stdin
- 用 `monkeypatch.setattr(hook, "generate_hook_output", mock_gen)` mock 下游避開真實 fs 依賴
- 驗證方式：assert `mock_gen` 收到的 input_data 符合預期
- 測試函式命名：`test_main_stdin_tty_fallback` / `test_main_stdin_json_decode_error_fallback` / `test_main_stdin_non_dict_fallback` / `test_main_stdin_with_background_tasks`
