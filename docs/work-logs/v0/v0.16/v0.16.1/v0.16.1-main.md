## 開發日誌

### 2026-03-28：W1 — 核心測試修復（3 個 Ticket）

- W1-001 重新啟用 191 個 skip 測試（分類評估後逐步開啟）
- W1-002 修復 PerformanceAssessment 測試 OOM（performance.now mock 遞增值）
- W1-003 修復 Workflow 測試 Helper 回傳結構不符（8 個測試）

### 2026-03-29：W2 — 大規模測試重啟（16 個 Ticket）

本版本最大的一波，分為三個方向：

**API 測試重寫**：
- W2-001 重寫 API 過時測試套件（58 個測試，3 個檔案）

**E2E 環境測試啟用**（11 個子 Ticket）：
- W2-002 啟用 E2E 環境依賴的 skip 測試（113 個測試，9 個檔案）
- W2-002.1 升級 Puppeteer 至 22.x 並驗證 WebSocket 修復
- W2-002.2 建立 jest.e2e.config.js 獨立 E2E 測試配置
- W2-002.3 ~ W2-002.11 逐項啟用各類 E2E 測試

**品質修復**：
- W2-003 記錄 jsdom 相容性 skip 測試保留決策（29 個測試）
- W2-004 修復 MetricsCollector 與 PerformanceAssessment 記憶體指標結構不匹配
- W2-005 修復 ErrorCodes 項目型別假設錯誤（字串 vs 物件）

**工具修復**：
- W2-006 修復 todolist.yaml 格式與 ticket CLI 解析邏輯不一致
- W2-007 重構 sync-claude-push/pull 腳本改為 merge 機制取代覆蓋
- W2-008 修復 git index.lock 競爭條件自動清理
- W2-009 修正 Hook 中 git 呼叫改用 --no-optional-locks 消除競爭根因

### 2026-03-29：W3 — sync 腳本增強（4 個 Ticket）

- W3-001 新增 sync-push 版本衝突檢測

- W3-004 新增 sync-claude-status 版本+內容 hash 快速比對工具

### 2026-03-29：W4 — sync 腳本收尾（6 個 Ticket）
