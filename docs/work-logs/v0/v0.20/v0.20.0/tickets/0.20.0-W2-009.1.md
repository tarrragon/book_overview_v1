#### TagSchema.js 變更

| 項目 | 內容 |
|------|------|
| 新增常數 | `TAG_TREE_MAX_DEPTH=4`、`ROOT_PARENT_SENTINEL='ROOT'`、`SYSTEM_CATEGORY_ID_PREFIX='sys_cat_'`、`TAG_CATEGORY_ERROR_CODES`（4 碼凍結物件） |
| Schema 欄位 | `TAG_CATEGORY_SCHEMA.fields.parentId = { type:'string', required:false, default:null, nullable:true }`；SchemaValidator 既有邏輯自動處理 parentId 型別檢查（B-validate） |
| `makeCategoryKey(parentId, name)` | 純函式，沿用 makeTagKey 的 `JSON.stringify([normalizedParent, trim+lowercase(name)])` pattern（非 spec 草案 `::` 拼接）；null parentId → ROOT sentinel |
| `depthOfParent(parentId, all)` | 沿 parentId 鏈上溯計算祖先鏈長度（根=0），含環防禦上限 |
| `validateTagCategory(category, siblings, allCategories?)` | 雙模式：2 參數＝欄位+parentId 型別+scoped 兄弟唯一鍵；3 參數追加引用存在性/循環/深度。回傳新增 `codes[]`（錯誤碼），`errors[]` 維持字串（向後相容 book-schema-v2 既有 `.includes` 斷言） |

#### tag-storage-adapter.js 變更

| 項目 | 內容 |
|------|------|
| STORAGE_KEYS | 新增 `CATEGORIES: 'tag_categories'` 別名（樹狀測試以 CATEGORIES 指稱同一 key） |
| 儲存輔助雙模式 | `loadFromStorage`/`saveToStorage`/`getBytesInUse` 改為 callback + Promise 雙相容（既有 mock 為 callback 形式，tag-tree fixture 為 Promise 形式），並以 `readLastError()` 可選鏈防護缺 chrome.runtime 的 mock；getBytesInUse 缺 API 時回 0 bytes |
| `pickCategoryErrorCode(validation)` | 從 codes 依優先序（引用>循環>深度>命名）取單一 error code |
| `createTagCategory` | 先生成 id 再驗證（避免 id 必填誤判）→ scoped siblings 過濾 + validateTagCategory(候選, siblings, all)；寫入 parentId；成功回傳改 `{ success:true, ...category }`（既有測試讀 result.id 向後相容） |
| `updateTagCategory` | merge patch 後 validate（含 parentId 變更引用/循環/深度，linux#2 校正）；fail 不寫入回對應 code |
| `moveTagCategory(id, newParentId)` | 新增，B1 循環防護所需最小移動路徑（循環/深度/引用），fail 不寫入保證樹不變 |
| `computeMergeResult` overload | 首參數含 localCategories/incomingCategories 時走 `computeScopedCategoryMerge`（A3-2 scoped key 去重），與既有三參數形式並存 |

#### 設計決策與範圍說明

1. **系統預裝引用前綴豁免**（A3-1 vs B3-create 調和）：A3-1 在僅含一筆 user category 的 store 下，於未載入的 `sys_cat_1` 下建子並期望成功；B3-create 在 `ghost` 下建子期望 invalid_parent_reference。以 `sys_cat_` 前綴視為合法預裝引用（預裝節點走確定性 ID upsert，可能未 eager 載入）區分二者，符合預裝模型語意。
2. **computeMergeResult overload（A3-2）**：W2-008 標 A3-2/校正點 3 屬 W2-009.3。但 A3-2 案例位於目標檔 tag-tree-unique-key.test.js（禁改 tests/），故以「物件形式 overload + scoped 去重」最小落地使其轉綠，不動既有三參數 merge 管線。完整 merge scoped 重映射（tag/book/tombstone/LWW）仍待 W2-009.3。
3. **moveTagCategory**：屬場景組 F 完整契約，本 ticket 僅實作 B1 循環防護所需最小路徑（深度整體子樹檢查待 F）。

#### ARCH-020 防護
`makeCategoryKey` 為唯一 category 鍵入口；createTagCategory/updateTagCategory/moveTagCategory/computeScopedCategoryMerge 全部 import 同函式，零內聯重寫。
