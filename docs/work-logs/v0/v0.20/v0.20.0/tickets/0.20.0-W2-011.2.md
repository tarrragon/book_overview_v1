## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W2-011 acceptance 3「內嵌 self-test 真陽真陰」未達成：原偵測器只有外部 pytest fixture（test_malformed_tool_call_detector_hook.py），未在 hook 本體內嵌 self-test。缺內嵌 self-test 時，hook 無法獨立自我驗證偵測簽章正確性，CI 須依賴外部測試檔存在。

### 影響範圍

- .claude/hooks/malformed-tool-call-detector-hook.py（加 _self_test() + --self-test 分支 + fixture 常數）
- .claude/hooks/tests/test_malformed_tool_call_detector_hook.py（改引用 hook 常數達 DRY + 新增 self-test 測試）

### 相關 Error Pattern

PC-177（malformed tool-call 被當文字渲染）。本 ticket 為其偵測器的 self-test 內嵌補強，非新增偵測邏輯。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作（thyme F4 設計，非 charset-guard 啟動式 per-event 模式）

1. hook 內新增 module 常數 SELF_TEST_TRUE_NEGATIVES（4 真陰）與 SELF_TEST_TRUE_POSITIVES（3 真陽）：
   - 真陰：four_space_indent（根因 A 縮排）、cross_line_backtick（根因 B 跨行）、fenced_block、inline_backtick
   - 真陽：bare_invoke（sig1）、bare_close_invoke（sig3）、stray_token_invoke（sig4，禁削弱）
2. _self_test() -> list 比對所有 fixture，回傳失敗描述清單（空=全通過）。
3. __main__ 加 --self-test 分支呼叫 _self_test()，有失敗 exit 1 並印出，全過 exit 0 並印通過摘要。非 per-Stop-event。
4. fixture 標記字面以字串拼接組裝（_OPEN = "<in" "voke" 等），避免本 hook 與其他 lint hook 掃描原始碼時連鎖誤判。

### DRY 處理

外部 pytest 改引用 hook 模組常數 SELF_TEST_TRUE_NEGATIVES / SELF_TEST_TRUE_POSITIVES，不再重複定義 fixture。新增 test_self_test_function_passes（呼叫 _self_test()）與 test_self_test_cli_exit_zero（subprocess 跑 --self-test 驗 exit 0）。fixtures 單一事實來源在 hook，消除漂移風險。

### CI 接線

npm run test:hooks → pytest 涵蓋 test_self_test_cli_exit_zero，subprocess 執行 --self-test 路徑。

### Phase 4 評估

重構需要：否。技術債務：無。程式碼品質：A。
結論：fixture 集中於 hook 常數已是 DRY 最簡形式；_self_test 為純比對函式，認知負擔低（1 變數 1 迴圈）。--self-test 分支與 main() 解耦無副作用。無需重構。
