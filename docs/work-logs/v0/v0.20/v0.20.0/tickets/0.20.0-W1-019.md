### 0.19.1 發布實證（2026-06-05）

### 缺陷 A：CHANGELOG 空殼插入
- update_changelog（version_release.py:1313-1358）：偵測既有 "## [version]" 區段則跳過插入（line 1336），但版本區段完全不存在時走 line 1340 插入空殼 placeholder 模板（含 "UC-XX 功能名稱"、"新增功能項目" 等佔位符）。
- 0.19.1 發布時 CHANGELOG 無 0.19.1 區段，若直接 release 會發布空殼。靠發布前手動撰寫 0.19.1 區段（commit 1787743a，彙整 60 ticket）規避。
- 修復方向：版本區段不存在時改 fail-loud（exit non-zero + 提示手動撰寫），不靜默插空殼。

### 缺陷 B：Mark Completed 步驟順序
- release 流程 Step 3 Git Operations（git push origin main）排在 Step "Mark Completed"（todolist active→completed）之前。
- 後果：todolist completed 標記落在已推送 commit 之外，成孤兒未提交變更。0.19.1 發布後遠端 todolist 仍為 active，靠手動補 commit 261c0a56 + push 對齊。
- 修復方向：Mark Completed 移至 push 之前納入同一推送，或標記後自動 commit + push。
