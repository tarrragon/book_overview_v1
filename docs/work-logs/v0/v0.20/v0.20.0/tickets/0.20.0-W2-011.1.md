## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

`strip_code_regions` 剝除不完整，使「被引述的工具標記字面」殘留並命中 signature，造成 main 活躍 false-positive（W2-011 多視角審查 thyme 以重現 script 實證）：

- 根因 A（活躍主路徑）：未剝除 Markdown 4-space／tab 縮排 code block，縮排引述的 `<invoke>` 命中 signature 1（行首裸 invoke）。
- 根因 B（活躍）：inline backtick regex `` `[^`\n]*` `` 排除換行，導致跨行 backtick 引述漏剝，內部 `<invoke>` 命中 signature 1/2。

兩根因經 /tmp 重現 script 確認命中，非推測。

### 影響範圍

- `.claude/hooks/malformed-tool-call-detector-hook.py`（`strip_code_regions`）

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

修正 `strip_code_regions` 剝除順序與範圍（最小範圍，evidence-driven-bugfix，signature patterns 不動）：

1. fenced block（``` ... ```）先剝除，避免後續 backtick 規則誤切三引號內部。
2. backtick 引述改用 `` `[^`]*` `` + `re.DOTALL`，允許跨行（修根因 B）。
3. 新增第三步：`(?m)^(?: {4}|\t).*$` 將 4-space／tab 縮排行整行清空（Markdown 縮排 code block，修根因 A）。

### 設計決策

- 剝除順序固定為 fenced → backtick → indented，避免規則互相吞噬（backtick 不排除換行後，須確保 fenced 已先移除）。
- 不疊加 signature regex 特例（遵守 W2-011 linux taste veto：修「剝除正確性」非加偵測特例）。
- signature 4 完整保留，並由 test_signature_four_preserved 斷言其存在與攔截力。

### 不在本 ticket 範圍

- 內嵌 self-test（--self-test）→ W2-011.2。
- meta-context 豁免（治本 PC-099）→ W2-011.3。

### Phase 4 評估結論

無需重構。本次修改集中於單一函式 `strip_code_regions`，認知負擔指數低（3 條 re.sub 序列，順序註解明示），命名與抽象層級一致。新增測試檔結構化（真陰／真陽 parametrize 分離），無技術債。程式碼品質 A。
