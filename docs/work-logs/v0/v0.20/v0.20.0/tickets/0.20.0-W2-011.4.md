## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W2-011 多視角審查 thyme F5 識別簽章涵蓋缺口：retry-continuation（double-failure 結束狀態仍含 malformed 標記）的偵測簽章未涵蓋「漏 antml 前綴的完整 invoke 字面」形態。

現有 signature 1/3 以**行首**錨點（`(?m)^[ \t]*`）偵測裸 <invoke>/</invoke>。但當完整工具呼叫被當文字渲染、且 <invoke> 前有散文夾住（非行首）時，行首錨點接不到，導致此類 malformed 漏抓——Stop hook 無法觸發 retry-continuation。

### 影響範圍

- `.claude/hooks/malformed-tool-call-detector-hook.py`：SIGNATURE_PATTERNS、self-test fixtures
- `.claude/hooks/tests/test_malformed_tool_call_detector_hook.py`：回歸測試

### 相關 Error Pattern

PC-177（malformed tool-call rendered as text）、PC-099（偵測機制自我誤報，meta-context 豁免對齊）。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 簽章設計（signature 5：成對閉合）

新增 1 條高特異簽章至 SIGNATURE_PATTERNS：

    re.compile(r"<\s*invoke\b[\s\S]*?</\s*invoke\s*>", re.IGNORECASE)

- 同訊息同時出現裸 <invoke …（全域 search 非行首）與裸 </invoke>，中間任意內容（非貪婪）。
- 涵蓋「漏 antml 前綴的完整 invoke 字面」缺口：完整工具呼叫被當文字渲染但 <invoke> 不在行首。
- 特異性靠**成對閉合**：散文單獨引述一個 <invoke> 字面（無對應閉合）不命中；只有開啟＋閉合同時殘留才觸發。
- 比對順序由 detect() 保證：strip_code_regions（剝除 fenced/inline/縮排）+ EXEMPT_MARKER（meta-context 豁免）先行，新 pattern 才比對。故 code 區塊內成對引述、含 exempt marker 的 meta 散文皆不誤觸。

### YAGNI 不修：標記中段斷裂

「標記中段斷裂」（半個 tag、屬性被截斷的部分前綴）記錄為 YAGNI 不實作。理由：形態無窮、特異性低、高誤報；harness in-turn retry 通常不產出半個 tag（要嘛完整渲染 tool-call 結構，要嘛漏前綴的完整標記）。決策同步記於 hook module docstring（retry-continuation 簽章涵蓋章節）。
