## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

依 SPEC-010 場景組 C 將 `tests/unit/storage/chrome-storage-tag-operations.test.js` 中 3 個過時 deleteTagCategory 測試遷移至新 cascade 分層契約。僅改測試斷言，禁改 src。

實作真相來源：`git show f6736c1d` + `src/storage/adapters/tag-storage-adapter.js:573` deleteTagCategory。葉節刪除走 C2 路徑（ensureUncategorizedCategory 惰性建立 Uncategorized 系統 category，tag.categoryId 轉移、tag 不刪、Book.tagIds 不變）；失敗走 catch 回滾，回傳 cascade_partial + rolledBack:true。

### 3 個測試遷移

| 測試 | 舊契約（刪除） | 新契約（C2/C-rollback） |
|------|---------------|------------------------|
| 刪除非系統葉節 category（line 239） | getAllTags 長度 0、category 0、book tag 移除 | tag 保留（長度 2）categoryId 轉 Uncategorized、category 含惰性 Uncategorized、Book.tagIds 不變 |
| 原子性步驟序列（line 854） | 移除書籍引用 -> 刪 tags -> 刪 category | tag 轉 Uncategorized -> 刪 category，書籍引用與 tag 保留 |
| 回滾（line 880） | success:false、category 仍存在 | error=cascade_partial、rolledBack=true、category 與 tag 回滾至刪除前快照 |

### 回滾測試 mock 對齊

新葉刪除寫入序列為先寫 tags 再寫 tag_categories。mock 改為第 2 次寫入（tag_categories）注入 lastError，觸發 saveToStorage reject -> deleteTagCategory catch -> 回滾。`saveToStorage` 透過 readLastError 偵測失敗（adapter line 158-166）。

### 對齊參考

與 `tests/integration/tag-tree-cascade-delete.test.js`（W2-007 新契約測試）C2 / C-rollback 場景一致，無契約矛盾。其餘測試（isSystem 不可刪、不存在回錯誤）保持不動。
