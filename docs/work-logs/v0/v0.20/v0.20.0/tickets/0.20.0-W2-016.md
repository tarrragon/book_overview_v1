### Reality-test 修正（PM reinstall 驗證後，2026-06-06）

原假設「version-release CLI 缺巢狀路徑支援」**經 reality-test 推翻**：

1. **root cause 主因 = stale installed CLI（非 code bug）**：source `scripts/version_release.py` 行 119-123 已有 `worklog_path_pattern` 預設巢狀（`docs/work-logs/v{major}/v{major_minor}/v{version}`），行 371 `resolve_worklog_dir` + 行 404/449 worklog 完成度檢查已正確用 config pattern。但 installed CLI 為 stale（W1-019 改 source 後未 reinstall），用舊扁平邏輯。PM `uv tool install --reinstall` 後，check worklog 完成度從「找不到工作日誌」變「v0.20.0 有 1/26 未完成」（巢狀路徑生效，正確掃描 26 tickets）。

2. **真實殘留 bug（行 495）**：TD 檢查 `tickets_dir = worklog_dir / f"v{version}" / "tickets"`（扁平 `docs/work-logs/v0.20.0/tickets`），未對齊 config pattern。與行 449 worklog 完成度檢查（已用 version_subdir 巢狀）不一致。後果：巢狀票目錄（docs/work-logs/v0/v0.20/v0.20.0/tickets）的 *-TD-*.md 檔被漏檢，TD 檢查永遠 pass（找不到扁平票目錄 = 誤判無 TD）。check 輸出「找不到票目錄: docs/work-logs/v0.20.0/tickets」標 [OK] 即此 false negative。

3. **教訓**：看到 CLI check FAIL 應先驗證 installed vs source（stale）再斷定 code bug（diagnostic rigor before fix）。本次先讀 source 確認已有巢狀支援 + reinstall 驗證，才定位真實殘留在行 495。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 修復摘要

行 495 `tickets_dir = worklog_dir / f"v{version}" / "tickets"`（扁平路徑）對齊行 449 的 config pattern：

1. 載入 config 取 `worklog_path_pattern`（預設巢狀 `docs/work-logs/v{major}/v{major_minor}/v{version}`）。
2. 複用 `resolve_worklog_dir(root, version, pattern)` 解析 `version_subdir`，票目錄取 `version_subdir / "tickets"`。
3. 保留扁平 fallback：若巢狀票目錄不存在，退回 `worklog_dir / f"v{version}" / "tickets"`（相容舊 v0.18.0）。

僅修改 `check_technical_debt_status`（行 492-507），未動其他函式。

### 設計決策

- 複用既有 `resolve_worklog_dir` + `load_version_release_config`（DRY，與行 449 的 worklog 完成度檢查同源），不新增路徑解析邏輯。
- fallback 先試巢狀（current SSOT 結構）再退扁平，與行 423-430 的 worklog 偵測順序一致。

### 不在本 ticket 範圍
