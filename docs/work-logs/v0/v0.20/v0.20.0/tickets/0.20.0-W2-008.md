### 2. 場景組 B：parentId 防護（create + update 雙寫入路徑）

`validateTagCategory(category, siblingCategories, allCategories)` 擴展。新增第三參數 allCategories 供循環/深度計算（spec §2.3 只列 sibling，但循環偵測需全集——校正項）。

虛擬碼:
```
FUNCTION validateTagCategory(category, siblingCategories, allCategories):
    errors = baseFieldValidation(category)
    # parentId 型別
    IF category.parentId != null AND not isString(category.parentId): errors.add('parentId 型別錯誤')
    # 引用存在性（B3/B3-create → invalid_parent_reference）
    IF category.parentId != null AND not exists(allCategories, category.parentId):
        errors.add('invalid_parent_reference')
    # 循環引用（B1/B1-self → circular_reference）：沿 parent 鏈上溯，遇自身即環
    IF category.parentId != null:
        ancestorId = category.parentId
        WHILE ancestorId != null:
            IF ancestorId == category.id: errors.add('circular_reference'); BREAK
            ancestorId = lookup(allCategories, ancestorId).parentId
    # MAX_DEPTH（B2 → max_depth_exceeded）：插入後深度 = 祖先鏈長
    IF depthOf(category.parentId, allCategories) + 1 > TAG_TREE_MAX_DEPTH:
        errors.add('max_depth_exceeded')
    # 兄弟唯一鍵（A2/F1 → duplicate_name）
    key = makeCategoryKey(category.parentId, category.name)
    IF any(siblingCategories where id != category.id, makeCategoryKey(s.parentId,s.name)==key):
        errors.add('duplicate_name')
    RETURN { valid: errors.empty, errors }

FUNCTION depthOf(parentId, all): RETURN length(ancestorChain(parentId, all))   # 根 parentId=null → depth 0
```

**寫入路徑接線**
- `createTagCategory`（B3-create）: 載入全集 → validate → fail 則回對應 error code（無寫入）→ pass 才生成隨機 ID 寫入
- `updateTagCategory`（B3，linux#2 校正）: 移除「直接 Object.assign splice」，改為 merge patch 後 validate（含 parentId 變更）→ fail 不寫入回 `invalid_parent_reference`/`circular_reference`/`max_depth_exceeded`

> error code 對映 §4 表：validate errors 陣列含特定 code 字串 → adapter 轉 `{ success:false, error:<code> }`。建議 validate 回傳結構化 `{ valid, errors:[{code,msg}] }` 讓 adapter 取 code（避免字串解析）。

### 3. 場景組 C：cascade 分層刪除

`deleteTagCategory(id, options={cascadeSubtree:false})` 重寫。虛擬碼:
```
FUNCTION deleteTagCategory(categoryId, options):
    cat = lookup(categoryId); IF not found: RETURN {success:false, error:'not_found'}
    # C4：isSystem 保護（含 Uncategorized C4-uncat）— error code 對齊 spec 'system_protected'
    IF cat.isSystem: RETURN {success:false, error:'system_protected'}
    children = categories where parentId == categoryId
    # C1/C3-noopt：非葉且未 opt-in → 禁止
    IF children.notEmpty AND not options.cascadeSubtree:
        RETURN {success:false, error:'has_children', hint:'請先處理子類'}
    BEGIN atomicRollback(snapshot categories+tags+books):   # 失敗→cascade_partial, rolledBack:true
        IF options.cascadeSubtree:
            subtree = getDescendants(categoryId) + categoryId
            FOR each node in subtree (葉到根序): 刪 node + 其下 tag（轉 Uncategorized 或一併刪由 UI 確認旗標）
        ELSE:  # C2/C2-lazy：刪葉，tag 轉 Uncategorized
            uncatId = ensureUncategorizedCategory()   # 不存在則惰性建立確定性 ID sys_cat_uncategorized
            FOR each tag where categoryId==categoryId: tag.categoryId = uncatId; tag.updatedAt=now()
            刪 cat
        RETURN {success:true}
```
- `ensureUncategorizedCategory`（Q1）: 查 `sys_cat_uncategorized`，無則建 `{id:'sys_cat_uncategorized', name:'Uncategorized', parentId:null, isSystem:true}`
- C-rollback: 沿用既有 `withAtomicRollback`（adapter:352），中途失敗回 `cascade_partial` + `rolledBack:true`
- `error code` 校正：既有 `cannot_delete_system`（346）→ 改 `system_protected`，需同步檢查呼叫端與既有測試是否依賴舊 code（Phase 3b 須 grep `cannot_delete_system`）
