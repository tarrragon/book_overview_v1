## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->

### 問題根因

W2-003 調查結論：v0.20.0 計畫隨 Chrome Extension 預裝散布 110 節點（10 主類 + 100 次類）賴永祥類號-類目對照。類號-類目清單傾向著作權法第 9 條第 3 款（通用名詞/數表/表格）免保護，且 NCL 編目園地免費開放下載；但 NCL 線上頁仍標 all rights reserved、無明示開放授權，形成灰區。內測公開散布前需 NCL 書面或專業法務確認，將「不需授權」從推論轉為定論。

### 決策脈絡（PM × 用戶確認，2026-06-06）

**觸發**：用戶質疑「只是套用圖書館分類依據，為何需授權 / 是否需提供分類表給使用者參考」。

**概念釐清**：
- 「提供分類表」兩義須分離：(a) 附完整分類表 PDF/全文供使用者查閱 = 不需要且最高風險（編輯著作重製）；(b) tag 樹骨架所需分類節點 = 功能資料，使用者透過 UI 點選歸類/篩選，非閱讀一份表。本產品採 (b)，不散布供查閱的分類表文件。
- 無專利問題（圖書分類法非專利標的）；法律維度僅著作權。

**方向決策（用戶拍板）**：採 W2-003 方案 1（預裝 110 節點扁平對照），維持現規劃。否決方案 2（純自訂 tag——規避授權但失標準對齊）與方案 3（僅 10 主類——粒度過粗）。方案 1 維持 → 本 W2-005 函詢保留，trigger 不變（v1.0.0 內測散布前）。本次為決策固化，非執行函詢。
