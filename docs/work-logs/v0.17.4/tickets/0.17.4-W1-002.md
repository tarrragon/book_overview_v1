## Problem Analysis

### Context Bundle

**目標**：新增 4 個 CLI 命令：set-priority、add-acceptance/remove-acceptance、add-spawned、set-decision-tree

**現有模式分析**（已確認）：
- 簡單欄位 set：`fields.py:execute_set_field()` 通用函式，包裝為 `execute_set_who()` 等
- list 欄位 set：`track_relations.py:_execute_set_relation_field()` 支援 --add/--remove
- 命令註冊：`track.py` dispatch dict + subparser

**新命令實作計劃**：

1. **set-priority**（fields.py）
   - 用 `execute_set_field(args, version, 'priority')` 包裝
   - track.py 註冊 subparser + dispatch

2. **add-acceptance / remove-acceptance**（fields.py 新增）
   - acceptance 是 list of strings（如 `['[ ] AC-1', '[ ] AC-2']`）
   - add-acceptance：append 新項到 list
   - remove-acceptance：按 index 移除（從 1 開始編號）
   - 需自訂函式（非通用 execute_set_field）

3. **add-spawned**（track_relations.py 或 fields.py）
   - spawned_tickets 是 list of strings
   - 只需 add（不需 remove）
   - append Ticket ID 到 list

4. **set-decision-tree**（fields.py 新增）
   - decision_tree_path 是 dict with 3 keys
   - CLI: `set-decision-tree <id> --entry <entry> --decision <decision> --rationale <rationale>`
   - 更新 ticket['decision_tree_path'] dict

**測試檔案**：新建 tests/test_fields_extended.py
