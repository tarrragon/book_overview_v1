# TDD Phase 4 重構工作日誌：Logger 架構標準化重構

**版本**: v0.14.2
**TDD 階段**: Phase 4 - 重構階段
**重構日期**: 2025-09-24
**重構類型**: Logger 使用模式標準化與 ErrorCodes 整合

## 🎯 重構動機與目標

### 當前架構的具體問題

1. **Logger 使用模式不一致**：
   - 99% 使用靜態調用 `Logger.info('字串')`（錯誤模式）
   - 1% 使用實例調用 `new Logger().info('messageKey')`（正確模式）
   - PopupErrorHandler 混用兩種模式，導致測試環境中 Logger 無法正常工作

2. **ErrorCodes 系統未整合**：
   - ErrorCodes 系統已建立但與 Logger 完全分離
   - 錯誤記錄缺乏結構化代碼追蹤能力
   - 實體測試需要錯誤代碼比對支援

3. **測試環境支援不足**：
   - 測試環境中 Logger 靜態調用失敗
   - 缺乏自動化錯誤追蹤和比對機制

### 重構後期望達成的狀態

1. **統一 Logger 使用模式**：
   - PopupErrorHandler 完全使用實例模式：`this.logger.info('messageKey')`
   - ErrorCodes 完整整合到所有錯誤記錄
   - 測試環境 100% 支援 Logger 功能

2. **結構化錯誤記錄**：
   - 所有錯誤記錄包含對應的 ErrorCodes
   - LOG 輸出格式支援自動化比對和追蹤
   - 實體測試完整支援錯誤驗證

3. **架構一致性**：
   - Logger 使用模式在整個類別中保持一致
   - 向後相容性保持，不影響現有系統運作

### 這個重構如何解決核心問題

通過建立 Logger 實例模式 + ErrorCodes 整合的標準化架構，解決測試失敗問題並建立完整的錯誤追蹤體系。

## 🚀 重構執行記錄

### 執行的重構動作

#### 1. PopupErrorHandler Logger 使用模式重構

**修改內容**：
```javascript
// 🔥 修改前：混用模式
Logger.warn('[PopupErrorHandler] Unable to load error config, using fallback') // 靜態調用

class PopupErrorHandler {
  // 無 Logger 實例初始化
}

// ✅ 修改後：統一實例模式
class PopupErrorHandler {
  constructor (dependencies = {}) {
    this.logger = new Logger('PopupErrorHandler')  // 建立實例
  }

  handleInitializationError(error) {
    this.logger.error('INITIALIZATION_FAILED', {  // 使用實例
      errorCode: ErrorCodes.INITIALIZATION_ERROR,
      originalError: error.message,
      component: 'popup-error-handler',
      action: 'initialize'
    })
  }
}
```

#### 2. ErrorCodes 整合

**整合模式**：
- 所有錯誤記錄都加入對應的 ErrorCodes
- 結構化錯誤上下文：`component`, `action`, `originalError`
- JSON 格式輸出支援自動化比對

**例子**：
```javascript
// Chrome API 錯誤
this.logger.warn('CHROME_API_RELOAD_FAILED', {
  errorCode: ErrorCodes.CHROME_ERROR,
  originalError: error.message,
  action: 'forceReload'
})

// 診斷模式切換
this.logger.info('DIAGNOSTIC_MODE_TOGGLED', {
  errorCode: ErrorCodes.SYSTEM_ERROR,
  diagnosticMode: this.diagnosticMode,
  action: 'toggleDiagnostic'
})
```

#### 3. 測試環境支援修復

**移除臨時設定**：
```javascript
// 🔥 修改前：測試中的臨時 Logger 設定
beforeEach(() => {
  global.Logger = {  // 臨時全域設定
    info: mockLogFunction,
    warn: mockLogFunction,
    error: mockLogFunction
  }
})

// ✅ 修改後：無需臨時設定，使用標準 Logger 實例
beforeEach(() => {
  errorHandler = new PopupErrorHandler()  // Logger 實例正常工作
})
```
