# 📋 TDD 開發任務清單

## 🎯 開發原則

- 嚴格遵循 **Red-Green-Refactor** 循環
- 每個功能必須先寫測試（紅燈）
- 實現最小可用代碼讓測試通過（綠燈）  
- 重構優化代碼（重構）
- 所有測試必須保持通過狀態
