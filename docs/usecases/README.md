# 用例（Use Cases）

用例文件目錄。定義使用場景、驗收標準和例外處理，供測試設計和驗收參考。

## 使用方式

建立新用例時，複製 `TEMPLATE.md` 並重新命名：

```bash
cp docs/usecases/TEMPLATE.md docs/usecases/UC-{XX}-{簡短描述}.md
```

### 模板欄位說明

| frontmatter 欄位 | 用途 |
|-----------------|------|
| `id` | 用例 ID（UC-XX） |
| `title` | 用例名稱 |
| `status` | draft / review / approved / deprecated |
| `source_proposal` | 來源提案 ID |
| `primary_actor` | 主要行為者 |
| `related_specs` | 對應的規格清單 |
| `ticket_refs` | 實作此用例的 ticket |

### 正文結構

| 章節 | 必填 | 說明 |
|------|------|------|
| 基本資訊 | 是 | 行為者、前置條件、成功保證 |
| 主要成功場景 | 是 | 正常流程的步驟描述 |
| 替代場景 | 否 | 正常但非主要的替代路徑 |
| 例外場景 | 是 | 錯誤情況和處理方式 |
| 驗收條件 | 是 | 功能驗收、邊界條件、效能要求 |
| UI 互動流程 | 否 | 使用者介面流程圖 |
| 變更歷史 | 是 | 用例版本變更記錄 |
