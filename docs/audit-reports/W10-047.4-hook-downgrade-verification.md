## 執行摘要

W10-035.3 ANA 預估 8 hook 降級可達 ~85% 摩擦削減（~5500 觸發/3 天）。本審計以 baseline 前後 hook log 數據實測，結果為 **~57.8% 觸發總量削減**，與 85% 預估差距 27.2 個百分點。差距 > 20%，建議 PM 建立檢討 ticket。

- **Pre-baseline**：2026-05-03 ~ 2026-05-05（3 天），以 log 檔案名稱日期 grep 計數

- **日均化**：pre ÷ 3、post ÷ 1.5，以日均觸發次數比較
- **Action 計數**：grep log 內容中的 DENY / BLOCKED / exit_code 2

### 樣本充足度評估

觀察期 ~36 小時（1.5 天），方法論規定 2 Wave 觀察期。本審計僅有約 0.5-1 Wave 資料，結論標註為**趨勢結論**，非最終定論。

## 實際削減比分析

### 加權觸發削減

- **預估**：~85%（~5500 觸發/3d 削減，來源 W10-035.3）
- **實測**：~57.8%（日均從 2049 降至 865）
- **差距**：27.2 個百分點

### 差距根因分析

| 根因 | 影響程度 | 說明 |
|------|---------|------|
| bash-edit-guard 仍高頻觸發 | 高 | 降級後 bash-edit-guard 仍每次 Bash 呼叫觸發（exit 0 快速放行），佔 pre 總量 26.6%，僅削減 46.2%。原預估可能假設此 hook 完全停用或大幅縮減 matcher 範圍 |
| acceptance-gate 仍高頻觸發 | 高 | 雖加入 fast-path skip（295/453 = 65.1% fast-path），但仍每次 PostToolUse 觸發，僅削減 46.2% |
| 抽樣 hook 削減有限 | 中 | worklog-format / utf8-integrity / comment-qa 各削減 57.2%，但基數較小（各 318/3d），對總量影響有限 |
| parallel-dispatch 完全移除 | 正面 | 唯一完全從 settings.json 移除的 hook，達 83.2% 削減，符合預期 |
| 觀察期樣本不足 | 低 | 1.5 天資料可能存在日間活動量波動偏差，但 3 個高頻 hook 數據量充足（>100 筆） |

### 85% 預估的假設偏差

W10-035.3 的 85% 預估假設「降級後 ~5500 次/3d 觸發被消除」。實際上：
1. **「降級」不等於「移除」**：8 hook 中僅 1 個（parallel-dispatch-verification）從 settings.json 完全移除。其餘 7 個仍註冊、仍觸發，只是內部邏輯改為 fast-path skip 或 sampling。每次觸發仍需 Python 進程啟動 + log 寫入。
2. **Matcher 範圍未縮減**：bash-edit-guard 和 acceptance-gate 的 matcher 觸發範圍未改變，仍匹配全部 Bash / PostToolUse 事件。
3. **預估可能混淆「觸發」與「Action」**：若以 Action（deny/block）計算，pre-baseline 3d = 71 次、post = 15 次/1.5d = 30 次/3d，削減 57.7%——與觸發削減接近。

## 降級機制有效性評估

| 降級機制 | 適用 Hook | 有效性 |
|---------|----------|--------|
| 完全移除（settings.json 取消註冊） | parallel-dispatch-verification | 高（83.2% 削減） |
| Fast-path skip（條件判斷後 exit 0） | acceptance-gate | 中（hook 仍觸發、仍 spawn Python；65% 走 fast-path 但仍產 log） |
| 抽樣 N=10 | worklog-format / utf8-integrity / language-guard | 中（log 顯示仍逐次觸發；sampling 僅控制「是否做完整檢查」，不減觸發） |
| Matcher 限定 | comment-qa | 中（57.2% 削減） |
| 提醒級降級 | file-type-permission | 中（57.2% 削減，但 Action 反增） |
| 邏輯簡化（無 fast-path） | bash-edit-guard | 低（46.2% 削減，每次 Bash 仍觸發） |

### file-type-permission Action 上升觀察

Post-baseline Action 比（5/61 per 1.5d = 5.5%）高於 pre-baseline（2/285 per 3d = 0.7%）。此為 rollback 觸發條件「Action 比異常上升 > baseline x 2」的潛在命中。需 PM 評估是否為偶發波動或系統性變化。

## 決策建議

### 結論

實測削減比 **57.8%** 與預估 **85%** 差距 **27.2 個百分點（> 20%）**。

### 建議 PM 行動

1. **建立檢討 ticket**（acceptance #4 觸發）：分析預估偏差根因，評估是否需要進一步降級（如完全移除 bash-edit-guard / acceptance-gate 的 settings.json 註冊，或縮減 matcher 範圍）
2. **file-type-permission Action 比上升**：納入檢討 ticket 追蹤，評估是否需要 rollback 或調整
3. **延長觀察期**：1.5 天資料為趨勢結論，建議至少再觀察 1 Wave 後更新追蹤表
4. **修正預估方法論**：85% 預估假設「降級 = 消除觸發」，實測顯示「降級 = 減輕處理但仍觸發」。未來預估需區分「觸發消除」vs「處理降級」兩種機制的不同削減效果
