# PROC-001: 錯誤假設 PM 具備人類學習能力

## 錯誤摘要

在設計規則和審查建議時，錯誤假設 PM（主線程）會「用幾次就熟練」「累積經驗」「自行學習」。實際上 PM 是 AI，每個 session 都是全新 context，不具備跨 session 學習能力。

## 症狀

- 審查建議「不需要 references，PM 用幾次就會熟練」
- 規則設計省略具體場景指引，理由是「PM 知道該怎麼做」
- 精簡規範時刪除了 AI PM 需要的具體欄位提示

## 根因分析

linux 審查代理人按人類開發者的心智模型提出建議：人類 PM 會累積經驗、記住上次的做法、根據場景靈活應變。但本專案的 PM 是 AI（rosemary-project-manager），每個 session 從零開始，規則就是它的全部知識。

## 觸發條件

- 審查或設計規則時，出現「PM 會學習」「用過就知道」「不需要額外指引」等表述
- 精簡規範時，刪除了具體場景指引而只保留通用原則

## 正確做法

1. **規範可以精簡**（通用模板），但**必須有對應的 references 補充各場景具體欄位**
2. 設計規則時假設每次都是 PM 第一次使用
3. 寧可 references 有更多細節，也不要有缺失
4. 通用原則放 pm-rules/（自動載入），具體指引放 references/（按需讀取）

## 防護措施

審查代理人（特別是 linux）提出「不需要詳細指引」的建議時，PM 應評估：
- 該規則的使用者是人類還是 AI？
- AI 使用者是否在每個 session 都能自動讀取到此規則？
- 如果不能，是否需要 references 補充？
