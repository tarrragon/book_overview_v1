# 後端類專案測試脈絡

> 本檔承載後端類專案（Go/Python/Node.js 服務等）在各斷言類型上的考量脈絡差異。
> 內容為輕量脈絡提示，不含具體 API 用法或重構步驟。

---

## 適用專案類型

- Go 服務（gRPC、REST API）
- Python 服務（FastAPI、Django、Flask）
- Node.js 服務（Express、Fastify）
- 任何以服務端業務邏輯為主的後端專案

---

## 各斷言類型的後端脈絡

### 類型 8：非同步時序

後端環境中非同步問題最常見於：

- goroutine / async handler 的副作用（例：寫入外部狀態後立即斷言）
- 訊息佇列（message queue）的消費者處理完成前
- 背景工作（background job）的排程執行時序

**脈絡提示**：後端的非同步等待通常需要顯式等待機制（channel 接收、condition variable、polling 或 mock callback）。識別信號：斷言外部狀態（DB、外部服務）時未確認操作已完成。

### 類型 9：亂數輸出

後端環境中隨機性常見於：

- 密碼學函式的 salt / nonce 生成
- 負載均衡的 hash 分配
- 取樣演算法（如日誌取樣率）
- UUID / token 生成

**脈絡提示**：後端的密碼學相關隨機源不應在測試中固定種子（會破壞安全性假設）。正確做法是斷言輸出的格式/長度/分佈特性，而非特定輸出值。

### 類型 10：測試隔離

後端環境中共享狀態最嚴重，常見於：

- 資料庫狀態（測試間的資料殘留）
- 全域 registry / 設定（環境變數、配置物件）
- 進程級 singleton（連線池、快取）

**脈絡提示**：後端測試隔離問題是三類專案中最嚴重的，因資料庫狀態難以自動清理。識別信號：測試順序敏感、在 CI 中隨機失敗。常見防護：每測試用獨立事務並 rollback，或每測試用獨立測試資料庫。

### 類型 11：快照過度覆蓋

後端的快照使用場景：

- API response 的完整 JSON 快照
- 資料庫 schema 的快照
- 序列化輸出的快照

**脈絡提示**：後端 API response 快照較 WEB/APP 少見，但新增欄位時常觸發 false positive。識別信號：API 版本迭代時大量快照需要更新，更新內容主要是新增而非修改。

### 類型 5、6（計時相關）

後端服務的計時問題通常出現在：

- RPC 呼叫的延遲斷言
- 資料庫查詢效能斷言
- 批次處理的吞吐量斷言

**脈絡提示**：後端計時受網路延遲、資料庫連線狀態、CI 機器規格影響尤為明顯。計時斷言在 CI 環境的失敗率遠高於本地環境，是環境依賴 flaky 最典型的場景。
